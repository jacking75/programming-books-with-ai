# 12장. Redis 기초 및 CloudStructures 설치

## 12.1 Redis 소개

### 12.1.1 Redis란?

Redis(Remote Dictionary Server)는 오픈소스 인메모리 데이터 저장소다. 모든 데이터를 메모리에 저장하기 때문에 매우 빠른 읽기/쓰기 성능을 제공한다. 게임 서버에서는 주로 다음과 같은 용도로 사용한다.

- **세션 및 인증 토큰 저장**: 로그인 토큰을 빠르게 검증한다
- **캐시 저장소**: 자주 조회하는 마스터 데이터를 캐싱한다
- **실시간 랭킹**: Sorted Set을 이용한 리더보드 구현에 최적화되어 있다
- **임시 데이터 저장**: 던전 진행 상태 등 휘발성 데이터를 저장한다
- **메시지 큐**: Pub/Sub 기능으로 서버 간 통신을 구현한다

### 12.1.2 Redis의 특징

Redis는 다음과 같은 특징을 가지고 있다.

- **인메모리 저장**: 모든 데이터를 RAM에 저장하여 매우 빠른 속도를 제공한다
- **다양한 데이터 타입**: String, Hash, List, Set, Sorted Set 등 풍부한 자료구조를 지원한다
- **영속성 옵션**: RDB 스냅샷이나 AOF(Append Only File)를 통해 데이터를 디스크에 저장할 수 있다
- **TTL(Time To Live)**: 키마다 만료 시간을 설정할 수 있어 자동으로 데이터가 삭제된다
- **단일 스레드**: 명령어를 순차적으로 처리하여 Race Condition을 방지한다
- **원자성 보장**: 모든 명령어가 원자적으로 실행된다

### 12.1.3 게임 서버에서 Redis를 사용하는 이유

MySQL과 같은 관계형 데이터베이스는 디스크 기반이라 속도가 상대적으로 느리다. 반면 Redis는 메모리 기반이라 다음과 같은 장점이 있다.

- **빠른 응답 속도**: 평균 응답 시간이 1ms 이하다
- **높은 처리량**: 초당 수만~수십만 건의 요청을 처리할 수 있다
- **DB 부하 감소**: 자주 조회하는 데이터를 캐싱하여 MySQL 부하를 줄인다
- **실시간 기능**: 랭킹, 채팅 등 실시간성이 중요한 기능 구현에 적합하다

## 12.2 Redis 설치

### 12.2.1 Windows에 Redis 설치

Redis는 공식적으로 Linux를 지원하지만, Windows에서도 사용할 수 있는 방법이 있다.

#### 방법 1: Memurai 사용 (권장)

Memurai는 Windows용 Redis 호환 서버다. 개발 목적으로 무료로 사용할 수 있다.

1. Memurai 다운로드 페이지에 접속한다: https://www.memurai.com/get-memurai
2. Memurai Developer Edition을 다운로드한다
3. 설치 프로그램을 실행하고 기본 설정으로 설치를 진행한다
4. 설치가 완료되면 자동으로 서비스가 시작된다

설치 확인을 위해 명령 프롬프트를 열고 다음 명령어를 실행한다.

```bash
memurai-cli ping
```

`PONG`이 출력되면 정상적으로 설치된 것이다.

#### 방법 2: WSL2 + Redis 사용

Windows 10/11에서 WSL2(Windows Subsystem for Linux 2)를 사용하여 Redis를 설치할 수 있다.

1. PowerShell을 관리자 권한으로 실행하고 WSL2를 설치한다.

```powershell
wsl --install
```

2. Ubuntu를 설치한다.

```powershell
wsl --install -d Ubuntu
```

3. Ubuntu 터미널을 열고 Redis를 설치한다.

```bash
sudo apt update
sudo apt install redis-server
```

4. Redis 서버를 시작한다.

```bash
sudo service redis-server start
```

5. Redis가 정상적으로 실행되는지 확인한다.

```bash
redis-cli ping
```

### 12.2.2 Docker를 이용한 Redis 설치

Docker를 사용하면 가장 간단하게 Redis를 설치할 수 있다. 이 방법이 가장 권장된다.

#### Docker Desktop 설치

1. Docker Desktop 다운로드 페이지에 접속한다: https://www.docker.com/products/docker-desktop
2. Windows용 설치 프로그램을 다운로드하고 실행한다
3. 설치가 완료되면 Docker Desktop을 실행한다

#### Redis 컨테이너 실행

명령 프롬프트나 PowerShell에서 다음 명령어를 실행한다.

```bash
docker run --name game-redis -p 6379:6379 -d redis:latest
```

각 옵션의 의미는 다음과 같다.

- `--name game-redis`: 컨테이너 이름을 game-redis로 지정한다
- `-p 6379:6379`: 호스트의 6379 포트를 컨테이너의 6379 포트로 연결한다
- `-d`: 백그라운드에서 실행한다
- `redis:latest`: 최신 버전의 Redis 이미지를 사용한다

#### Redis 컨테이너 관리 명령어

컨테이너를 중지할 때는 다음 명령어를 사용한다.

```bash
docker stop game-redis
```

컨테이너를 다시 시작할 때는 다음 명령어를 사용한다.

```bash
docker start game-redis
```

컨테이너를 삭제할 때는 다음 명령어를 사용한다.

```bash
docker rm game-redis
```

#### Redis CLI 접속

Docker로 실행 중인 Redis에 접속하려면 다음 명령어를 사용한다.

```bash
docker exec -it game-redis redis-cli
```

접속 후 `ping` 명령어를 입력하면 `PONG`이 출력된다.

### 12.2.3 Redis 기본 설정

Redis는 기본적으로 별도의 설정 없이 사용할 수 있지만, 게임 서버 용도로 사용할 때는 몇 가지 설정을 변경하는 것이 좋다.

#### 영속성 설정

개발 환경에서는 데이터 영속성이 크게 중요하지 않으므로 기본 설정을 사용한다. 하지만 운영 환경에서는 데이터 손실을 방지하기 위해 RDB 스냅샷이나 AOF를 활성화해야 한다.

Docker로 Redis를 실행할 때 영속성을 활성화하려면 다음과 같이 볼륨을 마운트한다.

```bash
docker run --name game-redis -p 6379:6379 -v redis-data:/data -d redis:latest redis-server --appendonly yes
```

#### 메모리 제한 설정

Redis는 메모리를 무제한으로 사용할 수 있다. 개발 환경에서는 문제가 없지만, 운영 환경에서는 메모리 제한을 설정하는 것이 좋다.

```bash
docker run --name game-redis -p 6379:6379 -d redis:latest redis-server --maxmemory 256mb --maxmemory-policy allkeys-lru
```

- `--maxmemory 256mb`: 최대 메모리를 256MB로 제한한다
- `--maxmemory-policy allkeys-lru`: 메모리가 부족할 때 LRU(Least Recently Used) 알고리즘으로 키를 삭제한다

## 12.3 Redis 기본 명령어

### 12.3.1 Redis CLI 기본 사용법

#### String 타입 기본 명령어

```bash
# 키-값 저장
SET user:1:name "홍길동"

# 값 조회

# 키 존재 여부 확인

# TTL 설정 (초 단위)
SET session:1 "token123" EX 3600

# TTL 확인

# 여러 키-값 동시 저장
MSET user:1:level 10 user:1:gold 5000

# 여러 값 동시 조회
```

## 12.4 CloudStructures 패키지 설치

### 12.4.1 CloudStructures란?

CloudStructures는 .NET용 Redis 클라이언트 라이브러리다. StackExchange.Redis를 기반으로 만들어졌으며, 다음과 같은 장점이 있다.

- **타입 안전성**: 제네릭을 사용하여 타입 안전한 코드를 작성할 수 있다
- **직관적인 API**: Redis의 각 데이터 타입에 대응하는 클래스를 제공한다
- **MessagePack 지원**: 객체를 효율적으로 직렬화할 수 있다
- **async/await 지원**: 비동기 프로그래밍을 완벽하게 지원한다

### 12.4.2 NuGet 패키지 설치

Visual Studio 2022에서 프로젝트를 연 후 다음 방법으로 패키지를 설치한다.

#### 방법 1: NuGet 패키지 관리자 사용

1. 솔루션 탐색기에서 프로젝트를 마우스 오른쪽 버튼으로 클릭한다
2. "NuGet 패키지 관리"를 선택한다
3. "찾아보기" 탭에서 "CloudStructures"를 검색한다
4. CloudStructures를 선택하고 "설치" 버튼을 클릭한다

#### 방법 2: 패키지 관리자 콘솔 사용

Visual Studio 메뉴에서 "도구 > NuGet 패키지 관리자 > 패키지 관리자 콘솔"을 선택한 후 다음 명령어를 입력한다.

```powershell
Install-Package CloudStructures
```

#### 방법 3: .csproj 파일 직접 수정

프로젝트 파일(.csproj)을 열고 다음 내용을 추가한다.

```xml
<ItemGroup>
  <PackageReference Include="CloudStructures" Version="4.2.0" />
</ItemGroup>
```

버전은 최신 버전으로 변경할 수 있다. 파일을 저장하면 자동으로 패키지가 복원된다.

### 12.4.3 설치 확인

패키지가 정상적으로 설치되었는지 확인하기 위해 솔루션 탐색기에서 "종속성 > 패키지"를 확인한다. CloudStructures와 StackExchange.Redis가 표시되어야 한다.

## 12.5 Redis 연결 설정

### 12.5.1 appsettings.json 설정

프로젝트의 `appsettings.json` 파일에 Redis 연결 정보를 추가한다.

```json
{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning"
    }
  },
  "AllowedHosts": "*",
  "ConnectionStrings": {
    "MySQL": "Server=localhost;Database=game_db;Uid=root;Pwd=yourpassword;",
    "Redis": "localhost:6379"
  },
  "Redis": {
    "Host": "localhost",
    "Port": 6379,
    "Database": 0,
    "ConnectTimeout": 5000,
    "SyncTimeout": 5000
  }
}
```

Redis 설정 항목의 의미는 다음과 같다.

- `Host`: Redis 서버 주소다
- `Port`: Redis 서버 포트다 (기본값: 6379)
- `Database`: 사용할 데이터베이스 번호다 (0~15, 기본값: 0)
- `ConnectTimeout`: 연결 타임아웃 시간이다 (밀리초)
- `SyncTimeout`: 동기 작업 타임아웃 시간이다 (밀리초)

### 12.5.2 Redis 설정 모델 클래스 생성

`Models` 폴더에 `RedisConfig.cs` 파일을 생성한다.

```csharp
// Models/RedisConfig.cs
namespace GameApiServer.Models;

public class RedisConfig
{
    public string Host { get; set; } = "localhost";
    public int Port { get; set; } = 6379;
    public int Database { get; set; } = 0;
    public int ConnectTimeout { get; set; } = 5000;
    public int SyncTimeout { get; set; } = 5000;

    public string GetConnectionString()
    {
        return $"{Host}:{Port},connectTimeout={ConnectTimeout},syncTimeout={SyncTimeout}";
    }
}
```

## 12.6 RedisConnection 구성

### 12.6.1 RedisConnection 클래스 생성

CloudStructures에서 Redis에 연결하려면 `RedisConnection` 객체가 필요하다. 이를 생성하고 관리하는 클래스를 만든다.

`Services` 폴더를 생성하고 `RedisService.cs` 파일을 추가한다.

```csharp
// Services/RedisService.cs
using CloudStructures;
using CloudStructures.Structures;
using GameApiServer.Models;
using StackExchange.Redis;

namespace GameApiServer.Services;

public interface IRedisService
{
    RedisConnection GetConnection();
    Task<bool> TestConnectionAsync();
}

public class RedisService : IRedisService
{
    private readonly RedisConnection _connection;
    private readonly ILogger<RedisService> _logger;

    public RedisService(IConfiguration configuration, ILogger<RedisService> logger)
    {
        _logger = logger;

        // appsettings.json에서 Redis 설정 로드
        var redisConfig = configuration.GetSection("Redis").Get<RedisConfig>();
        
        if (redisConfig == null)
        {
            throw new InvalidOperationException("Redis configuration is missing");
        }

        // Redis 연결 구성
        var config = new ConfigurationOptions
        {
            EndPoints = { { redisConfig.Host, redisConfig.Port } },
            ConnectTimeout = redisConfig.ConnectTimeout,
            SyncTimeout = redisConfig.SyncTimeout,
            AbortOnConnectFail = false,
            AllowAdmin = false
        };

        // RedisConnection 생성
        _connection = new RedisConnection(config, redisConfig.Database);

        _logger.LogInformation(
            "Redis connection initialized. Host: {Host}, Port: {Port}, DB: {Database}",
            redisConfig.Host,
            redisConfig.Port,
            redisConfig.Database
        );
    }

    public RedisConnection GetConnection()
    {
        return _connection;
    }

    public async Task<bool> TestConnectionAsync()
    {
        try
        {
            var redis = new RedisString<string>(_connection, "health:check", TimeSpan.FromSeconds(10));
            await redis.SetAsync("ok");
            var result = await redis.GetAsync();
            await redis.DeleteAsync();

            _logger.LogInformation("Redis connection test successful");
            return result.HasValue && result.Value == "ok";
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "Redis connection test failed");
            return false;
        }
    }
}
```

### 12.6.2 Program.cs에 서비스 등록

`Program.cs` 파일을 열고 RedisService를 DI 컨테이너에 등록한다.

```csharp
using GameApiServer.Services;

var builder = WebApplication.CreateBuilder(args);

// 기존 서비스들...
builder.Services.AddControllers();

// Redis 서비스 등록 (싱글톤)
builder.Services.AddSingleton<IRedisService, RedisService>();

var app = builder.Build();

// Redis 연결 테스트
using (var scope = app.Services.CreateScope())
{
    var redisService = scope.ServiceProvider.GetRequiredService<IRedisService>();
    var isConnected = await redisService.TestConnectionAsync();
    
    if (!isConnected)
    {
        var logger = scope.ServiceProvider.GetRequiredService<ILogger<Program>>();
        logger.LogWarning("Redis connection test failed. Some features may not work properly.");
    }
}

app.UseRouting();
app.MapControllers();
app.Run();
```

### 12.6.3 RedisConnection 사용 예제

RedisService를 사용하는 간단한 예제를 만들어본다.

```csharp
// Controllers/TestController.cs
using CloudStructures.Structures;
using GameApiServer.Services;
using Microsoft.AspNetCore.Mvc;

namespace GameApiServer.Controllers;

[ApiController]
[Route("api/[controller]")]
public class TestController : ControllerBase
{
    private readonly IRedisService _redisService;
    private readonly ILogger<TestController> _logger;

    public TestController(IRedisService redisService, ILogger<TestController> logger)
    {
        _redisService = redisService;
        _logger = logger;
    }

    [HttpPost("redis-test")]
    public async Task<RedisTestResponse> RedisTest()
    {
        try
        {
            var connection = _redisService.GetConnection();
            
            // String 타입 테스트
            var stringKey = "test:string";
            var stringRedis = new RedisString<string>(connection, stringKey, TimeSpan.FromMinutes(5));
            
            await stringRedis.SetAsync("Hello Redis!");
            var value = await stringRedis.GetAsync();
            
            if (!value.HasValue)
            {
                return new RedisTestResponse
                {
                    Result = ErrorCode.RedisError,
                    Message = "Failed to get value from Redis"
                };
            }

            await stringRedis.DeleteAsync();

            return new RedisTestResponse
            {
                Result = ErrorCode.None,
                Message = "Redis test successful",
                Value = value.Value
            };
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "Redis test failed");
            return new RedisTestResponse
            {
                Result = ErrorCode.RedisError,
                Message = ex.Message
            };
        }
    }
}

public class RedisTestResponse
{
    public ErrorCode Result { get; set; }
    public string Message { get; set; } = string.Empty;
    public string Value { get; set; } = string.Empty;
}
```

## 12.7 Redis 연결 확인

### 12.7.1 애플리케이션 실행

Visual Studio에서 F5 키를 눌러 애플리케이션을 실행한다. 콘솔 창에 다음과 같은 로그가 출력되어야 한다.

```
info: GameApiServer.Services.RedisService[0]
      Redis connection initialized. Host: localhost, Port: 6379, DB: 0
info: GameApiServer.Services.RedisService[0]
      Redis connection test successful
```

만약 연결에 실패하면 다음 사항을 확인한다.

- Redis 서버가 실행 중인지 확인한다
- `appsettings.json`의 연결 정보가 올바른지 확인한다
- 방화벽에서 6379 포트가 허용되어 있는지 확인한다

### 12.7.2 Postman으로 테스트

Postman에서 다음 요청을 보내본다.


## 12.9 연결 풀 및 성능 최적화

### 12.9.1 연결 풀 설정

StackExchange.Redis는 내부적으로 연결 풀을 관리한다. 기본 설정으로도 충분하지만, 필요에 따라 추가 설정을 할 수 있다.

```csharp
// Services/RedisService.cs
var config = new ConfigurationOptions
{
    EndPoints = { { redisConfig.Host, redisConfig.Port } },
    ConnectTimeout = redisConfig.ConnectTimeout,
    SyncTimeout = redisConfig.SyncTimeout,
    AbortOnConnectFail = false,
    AllowAdmin = false,
    KeepAlive = 60,                    // Keep-alive 간격 (초)
    ConnectRetry = 3,                  // 연결 재시도 횟수
    ReconnectRetryPolicy = new ExponentialRetry(5000), // 재연결 정책
};
```

### 12.9.2 성능 모니터링

Redis의 성능을 모니터링하는 간단한 메서드를 추가한다.

```csharp
// Services/RedisService.cs
public async Task<Dictionary<string, string>> GetServerInfoAsync()
{
    try
    {
        var server = _connection.GetConnection().GetServer(_connection.GetConnection().GetEndPoints()[0]);
        var info = await server.InfoAsync();
        
        var result = new Dictionary<string, string>();
        foreach (var group in info)
        {
            foreach (var item in group)
            {
                result[$"{group.Key}:{item.Key}"] = item.Value;
            }
        }
        
        return result;
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, "Failed to get Redis server info");
        return new Dictionary<string, string>();
    }
}
```

## 12.10 트러블슈팅

### 12.10.1 자주 발생하는 문제

#### 문제 1: Redis 연결 타임아웃

**증상**: `It was not possible to connect to the redis server(s)` 에러가 발생한다.

**해결 방법**:
- Redis 서버가 실행 중인지 확인한다
- 방화벽 설정을 확인한다
- `appsettings.json`의 Host와 Port가 올바른지 확인한다
- ConnectTimeout 값을 늘려본다

#### 문제 2: 직렬화/역직렬화 오류

**증상**: 객체를 Redis에 저장하거나 가져올 때 에러가 발생한다.

**해결 방법**:
- 저장하려는 클래스가 직렬화 가능한지 확인한다
- 클래스의 모든 프로퍼티가 public getter/setter를 가지고 있는지 확인한다
- 순환 참조가 없는지 확인한다

#### 문제 3: 메모리 부족

**증상**: Redis 서버가 메모리 부족으로 데이터를 저장하지 못한다.

**해결 방법**:
- `maxmemory` 설정을 늘린다
- `maxmemory-policy`를 설정하여 오래된 키를 자동 삭제한다
- TTL을 적절히 설정하여 불필요한 데이터가 자동 삭제되도록 한다

### 12.10.2 디버깅 팁

Redis 관련 문제를 디버깅할 때 유용한 팁이다.

#### 로그 레벨 조정

`appsettings.json`에서 StackExchange.Redis의 로그 레벨을 조정한다.

```json
{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning",
      "StackExchange.Redis": "Debug"
    }
  }
}
```

#### Redis 명령어 모니터링

Redis CLI에서 실시간으로 명령어를 모니터링할 수 있다.

```bash
docker exec -it game-redis redis-cli MONITOR
```

이 명령어를 실행하면 Redis 서버에서 실행되는 모든 명령어가 실시간으로 출력된다.

## 12.11 정리

이번 장에서는 Redis의 기본 개념과 설치 방법, CloudStructures 패키지 설정 방법을 학습했다. 주요 내용은 다음과 같다.

- Redis는 인메모리 데이터 저장소로 빠른 응답 속도를 제공한다
- Windows에서는 Docker를 사용하여 Redis를 설치하는 것이 가장 간편하다
- CloudStructures는 .NET에서 Redis를 타입 안전하게 사용할 수 있게 해준다
- RedisService를 싱글톤으로 등록하여 전역적으로 Redis 연결을 관리한다
- appsettings.json에서 Redis 연결 정보를 관리하여 환경별로 다른 설정을 사용할 수 있다

다음 장에서는 Redis의 다양한 데이터 타입을 활용하여 게임 서버의 여러 기능을 구현해본다.
