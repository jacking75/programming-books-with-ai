# 1장. 개발 환경 구축
이 장에서는 ASP.NET Web API 기반 게임 서버 개발을 시작하기 위한 개발 환경을 구축한다. Visual Studio 2022와 .NET 9 SDK를 설치하고, 첫 번째 Web API 프로젝트를 생성한 후 프로젝트 구조를 이해한다.

## 1.1 Visual Studio 2022 설치
Visual Studio 2022는 마이크로소프트에서 제공하는 통합 개발 환경(IDE)이다. ASP.NET Web API 개발에 필요한 모든 도구와 기능을 제공한다.

### 1.1.1 Visual Studio 2022 다운로드
1. 웹 브라우저를 열고 https://visualstudio.microsoft.com/ko/ 에 접속한다.
2. 'Visual Studio 다운로드' 버튼을 클릭한다.
3. Community, Professional, Enterprise 중 하나를 선택한다. 개인 개발자나 학습 목적이라면 무료인 Community 버전을 선택한다.
4. 다운로드된 설치 파일을 실행한다.

### 1.1.2 워크로드 선택
Visual Studio Installer가 실행되면 설치할 워크로드를 선택한다.

1. **ASP.NET 및 웹 개발** 워크로드를 체크한다. 이 워크로드에는 ASP.NET Core, Web API 개발에 필요한 모든 구성 요소가 포함되어 있다.
2. 오른쪽 '설치 세부 정보' 패널에서 다음 항목들이 선택되어 있는지 확인한다:
   - .NET 9.0 Runtime (최신 버전)
   - ASP.NET Core Runtime
   - 개발 시간 IIS 지원
   - HTTP 디버깅
3. '설치' 버튼을 클릭하여 설치를 시작한다.

설치는 인터넷 속도와 시스템 사양에 따라 20분에서 1시간 정도 소요된다.

### 1.1.3 설치 확인
설치가 완료되면 Visual Studio 2022를 실행하여 정상적으로 설치되었는지 확인한다.

1. Visual Studio 2022를 실행한다.
2. 시작 화면에서 '새 프로젝트 만들기'를 클릭한다.
3. 검색창에 'ASP.NET'을 입력하여 'ASP.NET Core Web API' 템플릿이 표시되는지 확인한다.
  

## 1.2 .NET 9 SDK 설치 확인
Visual Studio 2022를 설치하면 .NET 9 SDK가 함께 설치된다. 하지만 별도로 설치가 필요하거나 최신 버전으로 업데이트해야 할 경우가 있다.

### 1.2.1 현재 설치된 .NET SDK 확인
1. Windows 검색에서 'cmd' 또는 '명령 프롬프트'를 입력하여 실행한다.
2. 다음 명령어를 입력한다:

```bash
dotnet --version
```

3. .NET 9.x.x 버전이 표시되면 정상적으로 설치된 것이다.
4. 설치된 모든 SDK 버전을 확인하려면 다음 명령어를 사용한다:

```bash
dotnet --list-sdks
```

### 1.2.2 .NET 9 SDK 별도 설치 (필요한 경우)
만약 .NET 9 SDK가 설치되어 있지 않다면 다음 절차를 따른다.

1. https://dotnet.microsoft.com/download 에 접속한다.
2. '.NET 9.0' 섹션에서 'SDK' 다운로드 링크를 클릭한다.
3. Windows x64 버전을 다운로드한다.
4. 다운로드된 설치 파일을 실행하고 설치를 완료한다.
5. 설치 완료 후 명령 프롬프트를 새로 열어 `dotnet --version` 명령으로 확인한다.
  

## 1.3 ASP.NET Web API 프로젝트 생성
이제 실제로 게임 서버 프로젝트를 생성한다. 우리는 수집형 RPG 게임 서버를 만들 것이므로 프로젝트 이름을 'GameServer'로 지정한다.

### 1.3.1 새 프로젝트 만들기
1. Visual Studio 2022를 실행한다.
2. 시작 화면에서 '새 프로젝트 만들기'를 클릭한다.
3. 프로젝트 템플릿 검색창에 'web api'를 입력한다.
4. 'ASP.NET Core Web API' 템플릿을 선택하고 '다음' 버튼을 클릭한다.

### 1.3.2 프로젝트 구성
프로젝트 구성 화면에서 다음 정보를 입력한다.

- **프로젝트 이름**: GameServer
- **위치**: 원하는 폴더 경로를 선택한다 (예: C:\Projects\)
- **솔루션 이름**: GameServer (프로젝트 이름과 동일하게 자동 입력됨)

'다음' 버튼을 클릭한다.

### 1.3.3 추가 정보 설정
추가 정보 화면에서 다음 옵션을 설정한다.

- **프레임워크**: .NET 9.0 (장기 지원)
- **인증 형식**: 없음 (우리는 커스텀 토큰 기반 인증을 직접 구현할 것이다)
- **HTTPS용 구성**: 체크 해제 (개발 단계에서는 HTTP만 사용)
- **컨트롤러 사용(최소 API 사용 안 함)**: 체크 (컨트롤러 기반 API를 사용)
- **API 컨트롤러용 OpenAPI 지원 사용**: 체크 해제 (이번 교육에서는 Swagger를 사용하지 않음)

'만들기' 버튼을 클릭하면 프로젝트가 생성된다.

### 1.3.4 프로젝트 실행 테스트
프로젝트가 정상적으로 생성되었는지 확인하기 위해 실행해본다.

1. Visual Studio 상단의 '디버그' 메뉴에서 '디버깅 시작'을 선택하거나 F5 키를 누른다.
2. 웹 브라우저가 자동으로 열리며 기본 샘플 API 페이지가 표시된다.
3. 주소창에 표시된 URL을 확인한다 (예: http://localhost:5000).
4. 브라우저를 닫고 Visual Studio에서 'Shift + F5'를 눌러 디버깅을 중지한다.
  

## 1.4 프로젝트 구조 이해
생성된 프로젝트의 구조를 이해하는 것은 매우 중요하다. Visual Studio의 솔루션 탐색기에서 프로젝트 구조를 확인할 수 있다.

### 1.4.1 주요 파일 및 폴더
프로젝트를 생성하면 다음과 같은 구조가 만들어진다.

```
GameServer/
├── Properties/
│   └── launchSettings.json
├── Controllers/
│   └── WeatherForecastController.cs
├── appsettings.json
├── appsettings.Development.json
├── Program.cs
└── WeatherForecast.cs
```

각 파일과 폴더의 역할을 살펴본다.

#### Properties/launchSettings.json
이 파일은 개발 환경에서 애플리케이션을 실행할 때 사용되는 설정을 포함한다.

```json
{
  "$schema": "http://json.schemastore.org/launchsettings.json",
  "profiles": {
    "http": {
      "commandName": "Project",
      "dotnetRunMessages": true,
      "launchBrowser": true,
      "applicationUrl": "http://localhost:5000",
      "environmentVariables": {
        "ASPNETCORE_ENVIRONMENT": "Development"
      }
    }
  }
}
```

주요 설정 항목은 다음과 같다:

- **applicationUrl**: 애플리케이션이 실행될 URL과 포트를 지정한다
- **launchBrowser**: 실행 시 브라우저를 자동으로 열지 여부를 결정한다
- **ASPNETCORE_ENVIRONMENT**: 실행 환경을 지정한다 (Development, Production 등)

#### Controllers 폴더
컨트롤러 클래스들이 위치하는 폴더다. 컨트롤러는 클라이언트의 HTTP 요청을 받아 처리하고 응답을 반환하는 역할을 한다.

기본적으로 생성된 `WeatherForecastController.cs`는 샘플 코드이므로 나중에 삭제할 것이다.

#### appsettings.json
애플리케이션 전체에서 사용되는 설정 정보를 저장하는 파일이다. 데이터베이스 연결 문자열, API 키, 로깅 설정 등을 이곳에 정의한다.

```json
{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning"
    }
  },
  "AllowedHosts": "*"
}
```

이 파일은 나중에 MySQL, Redis 연결 정보를 추가할 것이다.

#### appsettings.Development.json
개발 환경에서만 사용되는 설정 파일이다. `appsettings.json`의 설정을 오버라이드한다.

#### Program.cs
애플리케이션의 진입점이자 구성 파일이다. .NET 6 이후 버전에서는 최소 호스팅 모델을 사용하여 간결한 코드로 애플리케이션을 구성한다.

```csharp
var builder = WebApplication.CreateBuilder(args);

// Add services to the container.
builder.Services.AddControllers();

var app = builder.Build();

// Configure the HTTP request pipeline.
app.UseAuthorization();

app.MapControllers();

app.Run();
```

이 파일에서 다음 작업을 수행한다:

1. **서비스 등록**: 의존성 주입 컨테이너에 서비스를 등록한다
2. **미들웨어 구성**: HTTP 요청 파이프라인을 구성한다
3. **엔드포인트 매핑**: 컨트롤러 라우팅을 설정한다

### 1.4.2 샘플 컨트롤러 분석
생성된 `WeatherForecastController.cs` 파일을 열어보면 다음과 같은 코드를 확인할 수 있다.

```csharp
using Microsoft.AspNetCore.Mvc;

namespace GameServer.Controllers
{
    [ApiController]
    [Route("[controller]")]
    public class WeatherForecastController : ControllerBase
    {
        private static readonly string[] Summaries = new[]
        {
            "Freezing", "Bracing", "Chilly", "Cool", "Mild", "Warm", "Balmy", "Hot", "Sweltering", "Scorching"
        };

        private readonly ILogger<WeatherForecastController> _logger;

        public WeatherForecastController(ILogger<WeatherForecastController> logger)
        {
            _logger = logger;
        }

        [HttpGet(Name = "GetWeatherForecast")]
        public IEnumerable<WeatherForecast> Get()
        {
            return Enumerable.Range(1, 5).Select(index => new WeatherForecast
            {
                Date = DateOnly.FromDateTime(DateTime.Now.AddDays(index)),
                TemperatureC = Random.Shared.Next(-20, 55),
                Summary = Summaries[Random.Shared.Next(Summaries.Length)]
            })
            .ToArray();
        }
    }
}
```

주요 요소를 살펴본다:

#### [ApiController] 특성
이 특성은 컨트롤러가 Web API임을 나타낸다. 자동으로 다음 기능을 활성화한다:

- 모델 유효성 검사 자동 수행
- 바인딩 소스 추론
- 문제 세부 정보 응답 생성

#### [Route] 특성
컨트롤러의 기본 경로를 지정한다. `[Route("[controller]")]`는 컨트롤러 이름에서 'Controller'를 제외한 부분을 경로로 사용한다. 예를 들어 `WeatherForecastController`는 `/weatherforecast` 경로로 매핑된다.

#### ControllerBase 상속
`ControllerBase`는 Web API 컨트롤러의 기본 클래스다. MVC 뷰 렌더링 기능이 없는 경량 버전이다.

#### 의존성 주입
생성자를 통해 `ILogger`를 주입받는다. ASP.NET Core는 자동으로 서비스를 생성하고 주입한다.

#### [HttpGet] 특성
이 메서드가 HTTP GET 요청을 처리함을 나타낸다. 우리는 POST만 사용할 예정이므로 나중에 `[HttpPost]`를 사용할 것이다.

### 1.4.3 프로젝트 구조 정리
이제 우리가 만들 게임 서버에 맞게 프로젝트 구조를 정리한다.

1. 솔루션 탐색기에서 `WeatherForecast.cs` 파일을 삭제한다.
2. `Controllers` 폴더의 `WeatherForecastController.cs` 파일을 삭제한다.
3. 솔루션 탐색기에서 프로젝트 이름을 마우스 오른쪽 버튼으로 클릭하고 '추가' > '새 폴더'를 선택하여 다음 폴더들을 생성한다:

```
GameServer/
├── Controllers/     (이미 존재)
├── Models/         (데이터 모델)
├── Services/       (비즈니스 로직)
├── Repositories/   (데이터 액세스)
├── Middlewares/    (커스텀 미들웨어)
└── Utils/          (유틸리티 클래스)
```

이제 기본적인 개발 환경 구축이 완료되었다. 다음 장에서는 HTTP POST 기반 API 서버의 기초를 학습하고, 첫 번째 API를 만들어본다.

- [ ] .NET 9 SDK가 설치되었다 (`dotnet --version` 명령으로 확인)

- [ ] 프로젝트가 정상적으로 실행된다 (F5 키로 실행 확인)
