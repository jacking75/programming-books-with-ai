# 7장. 데이터 CRUD 작업 (실습)

6장에서 SqlKata의 기본적인 사용법을 배웠다. 이번 장에서는 실제 게임 서버 개발에서 사용하는 패턴인 Repository 패턴을 구현하고, 복잡한 쿼리와 트랜잭션을 처리하는 방법을 배운다. Repository 패턴은 데이터 접근 로직을 비즈니스 로직으로부터 분리하여 코드의 재사용성과 유지보수성을 높여준다.

## 7.1 Repository 패턴 이해

### 7.1.1 Repository 패턴이란?

Repository 패턴은 데이터 소스(데이터베이스, 파일, 외부 API 등)에 대한 접근을 추상화하는 디자인 패턴이다. 컨트롤러나 서비스 클래스는 구체적인 데이터베이스 쿼리를 알 필요 없이 Repository의 메서드를 호출하여 데이터를 다룬다.

**Repository 패턴을 사용하지 않은 경우**

```csharp
// 컨트롤러에서 직접 데이터베이스 접근
[HttpPost("create-user")]
public async Task<CreateUserResponse> CreateUser([FromBody] CreateUserRequest request)
{
    using var db = await _queryFactory.CreateAsync();
    var userId = await db.Query("users").InsertGetIdAsync<long>(new
    {
        email = request.Email,
        password_hash = HashPassword(request.Password),
        nickname = request.Nickname
    });
    return new CreateUserResponse { Result = ErrorCode.None, UserId = userId };
}
```

이 방식은 간단한 경우에는 문제없지만, 다음과 같은 단점이 있다.

- 같은 쿼리를 여러 곳에서 중복 작성하게 된다
- 테스트하기 어렵다 (실제 데이터베이스가 필요함)
- 데이터베이스 구조가 변경되면 모든 컨트롤러를 수정해야 한다
- 비즈니스 로직과 데이터 접근 로직이 섞인다

**Repository 패턴을 사용한 경우**

```csharp
// Repository에서 데이터 접근 로직 처리
public class UserRepository
{
    public async Task<long> CreateUserAsync(string email, string passwordHash, string nickname)
    {
        // 데이터베이스 쿼리 로직
    }
}

// 컨트롤러는 Repository를 통해 데이터 접근
[HttpPost("create-user")]
public async Task<CreateUserResponse> CreateUser([FromBody] CreateUserRequest request)
{
    var passwordHash = HashPassword(request.Password);
    var userId = await _userRepository.CreateUserAsync(
        request.Email, passwordHash, request.Nickname);
    return new CreateUserResponse { Result = ErrorCode.None, UserId = userId };
}
```

이 방식은 다음과 같은 장점이 있다.

- 데이터 접근 로직이 한 곳에 모여 있어 재사용하기 쉽다
- 테스트 시 Repository를 Mock 객체로 대체할 수 있다
- 데이터베이스 구조 변경 시 Repository만 수정하면 된다
- 코드가 더 읽기 쉽고 유지보수하기 좋다

### 7.1.2 Repository 패턴 구조

게임 서버에서 Repository 패턴을 구현할 때 일반적인 구조는 다음과 같다.

```
GameServer/
├── Controllers/          # API 엔드포인트
├── Services/            # 비즈니스 로직
├── Repositories/        # 데이터 접근 로직
│   ├── IUserRepository.cs
│   └── UserRepository.cs
├── Models/              # 데이터 모델
│   └── User.cs
└── Database/            # 데이터베이스 연결 설정
```

각 레이어의 역할은 다음과 같다.

- **Controllers**: HTTP 요청을 받아 Services를 호출하고 응답을 반환한다
- **Services**: 비즈니스 로직을 처리하고 여러 Repository를 조합하여 사용한다
- **Repositories**: 데이터베이스와 직접 통신하여 데이터를 읽고 쓴다
- **Models**: 데이터 구조를 정의한다

## 7.2 사용자 Repository 구현

실제로 사용자 데이터를 관리하는 Repository를 구현해본다.

### 7.2.1 User 모델 정의

먼저 프로젝트에 `Models` 폴더를 만들고 `User.cs` 파일을 생성한다.

```csharp
namespace GameServer.Models
{
    public class User
    {
        public long UserId { get; set; }
        public string Email { get; set; }
        public string PasswordHash { get; set; }
        public string Nickname { get; set; }
        public int Level { get; set; }
        public long Exp { get; set; }
        public long Gold { get; set; }
        public long Gem { get; set; }
        public DateTime CreatedAt { get; set; }
        public DateTime? UpdatedAt { get; set; }
        public DateTime? DeletedAt { get; set; }
    }
}
```

이 모델은 users 테이블의 구조와 일치한다. `DeletedAt`은 소프트 삭제(Soft Delete)를 구현하기 위한 필드다. 실제로 데이터를 삭제하지 않고 삭제 시각을 기록하여 복구 가능하게 한다.

### 7.2.2 IUserRepository 인터페이스 정의

`Repositories` 폴더를 만들고 `IUserRepository.cs` 파일을 생성한다.

```csharp
using GameServer.Models;

namespace GameServer.Repositories
{
    public interface IUserRepository
    {
        Task<long> CreateAsync(User user);
        Task<User?> GetByIdAsync(long userId);
        Task<User?> GetByEmailAsync(string email);
        Task<List<User>> GetAllAsync(int offset = 0, int limit = 100);
        Task<bool> UpdateAsync(User user);
        Task<bool> DeleteAsync(long userId);
        Task<bool> ExistsAsync(string email);
        Task<int> CountAsync();
    }
}
```

인터페이스를 먼저 정의하면 나중에 테스트용 Mock 구현체를 만들거나 다른 데이터 소스로 전환하기 쉽다.

### 7.2.3 UserRepository 구현

`Repositories` 폴더에 `UserRepository.cs` 파일을 생성한다.

```csharp
using GameServer.Database;
using GameServer.Models;
using SqlKata.Execution;

namespace GameServer.Repositories
{
    public class UserRepository : IUserRepository
    {
        private readonly GameQueryFactory _queryFactory;
        private const string TableName = "users";

        public UserRepository(GameQueryFactory queryFactory)
        {
            _queryFactory = queryFactory;
        }

        public async Task<long> CreateAsync(User user)
        {
            using var db = await _queryFactory.CreateAsync();

            var userId = await db.Query(TableName).InsertGetIdAsync<long>(new
            {
                email = user.Email,
                password_hash = user.PasswordHash,
                nickname = user.Nickname,
                level = user.Level,
                exp = user.Exp,
                gold = user.Gold,
                gem = user.Gem,
                created_at = DateTime.UtcNow
            });

            return userId;
        }

        public async Task<User?> GetByIdAsync(long userId)
        {
            using var db = await _queryFactory.CreateAsync();

            var result = await db.Query(TableName)
                .Where("user_id", userId)
                .WhereNull("deleted_at")
                .FirstOrDefaultAsync();

            return MapToUser(result);
        }

        public async Task<User?> GetByEmailAsync(string email)
        {
            using var db = await _queryFactory.CreateAsync();

            var result = await db.Query(TableName)
                .Where("email", email)
                .WhereNull("deleted_at")
                .FirstOrDefaultAsync();

            return MapToUser(result);
        }

        public async Task<List<User>> GetAllAsync(int offset = 0, int limit = 100)
        {
            using var db = await _queryFactory.CreateAsync();

            var results = await db.Query(TableName)
                .WhereNull("deleted_at")
                .OrderByDesc("created_at")
                .Offset(offset)
                .Limit(limit)
                .GetAsync();

            return results.Select(MapToUser).Where(u => u != null).ToList()!;
        }

        public async Task<bool> UpdateAsync(User user)
        {
            using var db = await _queryFactory.CreateAsync();

            var rowsAffected = await db.Query(TableName)
                .Where("user_id", user.UserId)
                .WhereNull("deleted_at")
                .UpdateAsync(new
                {
                    nickname = user.Nickname,
                    level = user.Level,
                    exp = user.Exp,
                    gold = user.Gold,
                    gem = user.Gem,
                    updated_at = DateTime.UtcNow
                });

            return rowsAffected > 0;
        }

        public async Task<bool> DeleteAsync(long userId)
        {
            using var db = await _queryFactory.CreateAsync();

            // 소프트 삭제: deleted_at 필드에 현재 시각 기록
            var rowsAffected = await db.Query(TableName)
                .Where("user_id", userId)
                .WhereNull("deleted_at")
                .UpdateAsync(new
                {
                    deleted_at = DateTime.UtcNow
                });

            return rowsAffected > 0;
        }

        public async Task<bool> ExistsAsync(string email)
        {
            using var db = await _queryFactory.CreateAsync();

            var count = await db.Query(TableName)
                .Where("email", email)
                .WhereNull("deleted_at")
                .CountAsync<int>();

            return count > 0;
        }

        public async Task<int> CountAsync()
        {
            using var db = await _queryFactory.CreateAsync();

            return await db.Query(TableName)
                .WhereNull("deleted_at")
                .CountAsync<int>();
        }

        private User? MapToUser(dynamic? data)
        {
            if (data == null) return null;

            return new User
            {
                UserId = data.user_id,
                Email = data.email,
                PasswordHash = data.password_hash,
                Nickname = data.nickname,
                Level = data.level ?? 1,
                Exp = data.exp ?? 0,
                Gold = data.gold ?? 0,
                Gem = data.gem ?? 0,
                CreatedAt = data.created_at,
                UpdatedAt = data.updated_at,
                DeletedAt = data.deleted_at
            };
        }
    }
}
```

`MapToUser` 메서드는 데이터베이스에서 가져온 동적 객체를 강타입 User 객체로 변환한다. 이 패턴은 코드 전체에서 타입 안정성을 보장한다.

모든 조회 쿼리에 `WhereNull("deleted_at")` 조건을 추가하여 삭제된 데이터는 조회되지 않도록 한다.

### 7.2.4 Repository 서비스 등록

`Program.cs`를 열고 Repository를 의존성 주입 컨테이너에 등록한다.

```csharp
using GameServer.Database;
using GameServer.Repositories;

var builder = WebApplication.CreateBuilder(args);

var connectionString = builder.Configuration.GetConnectionString("MySQL");

builder.Services.AddSingleton(new DbConnectionFactory(connectionString));
builder.Services.AddSingleton<GameQueryFactory>();

// Repository 등록
builder.Services.AddScoped<IUserRepository, UserRepository>();

builder.Services.AddControllers();
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

var app = builder.Build();

if (app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}

app.UseHttpsRedirection();
app.UseAuthorization();
app.MapControllers();

app.Run();
```

`AddScoped`로 등록하면 HTTP 요청당 하나의 인스턴스가 생성되고 요청이 끝나면 해제된다. 이는 웹 애플리케이션에서 일반적으로 사용하는 수명 주기다.

### 7.2.5 Repository를 사용하는 컨트롤러 작성

`Controllers` 폴더에 `UserController.cs` 파일을 생성한다.

```csharp
using GameServer.Models;
using GameServer.Repositories;
using Microsoft.AspNetCore.Mvc;

namespace GameServer.Controllers
{
    [ApiController]
    [Route("[controller]")]
    public class UserController : ControllerBase
    {
        private readonly IUserRepository _userRepository;

        public UserController(IUserRepository userRepository)
        {
            _userRepository = userRepository;
        }

        [HttpPost("create")]
        public async Task<CreateUserResponse> CreateUser([FromBody] CreateUserRequest request)
        {
            try
            {
                // 이메일 중복 확인
                if (await _userRepository.ExistsAsync(request.Email))
                {
                    return new CreateUserResponse
                    {
                        Result = ErrorCode.EmailAlreadyExists,
                        Message = "이미 존재하는 이메일입니다"
                    };
                }

                var user = new User
                {
                    Email = request.Email,
                    PasswordHash = request.Password, // 실제로는 해싱 필요
                    Nickname = request.Nickname,
                    Level = 1,
                    Exp = 0,
                    Gold = 10000,
                    Gem = 100
                };

                var userId = await _userRepository.CreateAsync(user);

                return new CreateUserResponse
                {
                    Result = ErrorCode.None,
                    UserId = userId,
                    Message = "사용자 생성 완료"
                };
            }
            catch (Exception ex)
            {
                return new CreateUserResponse
                {
                    Result = ErrorCode.DatabaseError,
                    Message = "사용자 생성 실패",
                    Error = ex.Message
                };
            }
        }

        [HttpPost("get")]
        public async Task<GetUserResponse> GetUser([FromBody] GetUserRequest request)
        {
            try
            {
                var user = await _userRepository.GetByIdAsync(request.UserId);

                if (user == null)
                {
                    return new GetUserResponse
                    {
                        Result = ErrorCode.UserNotFound,
                        Message = "사용자를 찾을 수 없습니다"
                    };
                }

                return new GetUserResponse
                {
                    Result = ErrorCode.None,
                    User = new UserSummary
                    {
                        UserId = user.UserId,
                        Email = user.Email,
                        Nickname = user.Nickname,
                        Level = user.Level,
                        Exp = user.Exp,
                        Gold = user.Gold,
                        Gem = user.Gem,
                        CreatedAt = user.CreatedAt
                    }
                };
            }
            catch (Exception ex)
            {
                return new GetUserResponse
                {
                    Result = ErrorCode.DatabaseError,
                    Message = "사용자 조회 실패",
                    Error = ex.Message
                };
            }
        }

        [HttpPost("list")]
        public async Task<GetUserListResponse> GetUserList([FromBody] GetUserListRequest request)
        {
            try
            {
                var users = await _userRepository.GetAllAsync(
                    request.Offset, 
                    request.Limit);

                var total = await _userRepository.CountAsync();

                return new GetUserListResponse
                {
                    Result = ErrorCode.None,
                    Total = total,
                    Users = users.Select(u => new UserSummary
                    {
                        UserId = u.UserId,
                        Email = u.Email,
                        Nickname = u.Nickname,
                        Level = u.Level,
                        CreatedAt = u.CreatedAt
                    }).ToList()
                };
            }
            catch (Exception ex)
            {
                return new GetUserListResponse
                {
                    Result = ErrorCode.DatabaseError,
                    Message = "사용자 목록 조회 실패",
                    Error = ex.Message
                };
            }
        }

        [HttpPost("update")]
        public async Task<UpdateUserResponse> UpdateUser([FromBody] UpdateUserRequest request)
        {
            try
            {
                var user = await _userRepository.GetByIdAsync(request.UserId);

                if (user == null)
                {
                    return new UpdateUserResponse
                    {
                        Result = ErrorCode.UserNotFound,
                        Message = "사용자를 찾을 수 없습니다"
                    };
                }

                user.Nickname = request.Nickname;
                user.Level = request.Level;
                user.Exp = request.Exp;
                user.Gold = request.Gold;
                user.Gem = request.Gem;

                var success = await _userRepository.UpdateAsync(user);

                if (!success)
                {
                    return new UpdateUserResponse
                    {
                        Result = ErrorCode.DatabaseError,
                        Message = "사용자 정보 업데이트 실패"
                    };
                }

                return new UpdateUserResponse
                {
                    Result = ErrorCode.None,
                    Message = "사용자 정보 업데이트 완료"
                };
            }
            catch (Exception ex)
            {
                return new UpdateUserResponse
                {
                    Result = ErrorCode.DatabaseError,
                    Message = "사용자 정보 업데이트 실패",
                    Error = ex.Message
                };
            }
        }

        [HttpPost("delete")]
        public async Task<DeleteUserResponse> DeleteUser([FromBody] DeleteUserRequest request)
        {
            try
            {
                var success = await _userRepository.DeleteAsync(request.UserId);

                if (!success)
                {
                    return new DeleteUserResponse
                    {
                        Result = ErrorCode.UserNotFound,
                        Message = "사용자를 찾을 수 없습니다"
                    };
                }

                return new DeleteUserResponse
                {
                    Result = ErrorCode.None,
                    Message = "사용자 삭제 완료"
                };
            }
            catch (Exception ex)
            {
                return new DeleteUserResponse
                {
                    Result = ErrorCode.DatabaseError,
                    Message = "사용자 삭제 실패",
                    Error = ex.Message
                };
            }
        }
    }

    public class CreateUserRequest
    {
        public string Email { get; set; }
        public string Password { get; set; }
        public string Nickname { get; set; }
    }

    public class GetUserRequest
    {
        public long UserId { get; set; }
    }

    public class GetUserListRequest
    {
        public int Offset { get; set; } = 0;
        public int Limit { get; set; } = 20;
    }

    public class UpdateUserRequest
    {
        public long UserId { get; set; }
        public string Nickname { get; set; }
        public int Level { get; set; }
        public long Exp { get; set; }
        public long Gold { get; set; }

    }

    public class DeleteUserRequest
    {
        public long UserId { get; set; }
    }

    public class GetUserListResponse
    {

        public List<UserSummary> Users { get; set; } = new();
        public string Message { get; set; } = string.Empty;

    public class UpdateUserResponse

        public string Error { get; set; } = string.Empty;
    }

    public class UserSummary

        public long UserId { get; set; }
        public string Email { get; set; } = string.Empty;

        public long Exp { get; set; }
        public long Gold { get; set; }
        public long Gem { get; set; }
```

컨트롤러는 Repository를 통해서만 데이터베이스에 접근한다. 비즈니스 로직(이메일 중복 확인, 초기 자원 지급 등)은 컨트롤러나 서비스 레이어에서 처리한다.

## 7.3 복잡한 쿼리 작성

게임 서버에서 자주 사용하는 복잡한 쿼리 패턴을 살펴본다.

### 7.3.1 검색 및 필터링

사용자를 여러 조건으로 검색하는 기능을 추가한다. `IUserRepository` 인터페이스에 메서드를 추가한다.

```csharp
public interface IUserRepository
{
    // 기존 메서드들...
    
    Task<List<User>> SearchAsync(UserSearchOptions options);
}

public class UserSearchOptions
{
    public string? Nickname { get; set; }
    public int? MinLevel { get; set; }
    public int? MaxLevel { get; set; }
    public DateTime? RegisteredAfter { get; set; }
    public DateTime? RegisteredBefore { get; set; }
    public int Offset { get; set; } = 0;
    public int Limit { get; set; } = 20;
}
```

`UserRepository`에 구현을 추가한다.

```csharp
public async Task<List<User>> SearchAsync(UserSearchOptions options)
{
    using var db = await _queryFactory.CreateAsync();

    var query = db.Query(TableName).WhereNull("deleted_at");

    // 닉네임 검색 (부분 일치)
    if (!string.IsNullOrEmpty(options.Nickname))
    {
        query.WhereLike("nickname", $"%{options.Nickname}%");
    }

    // 레벨 범위
    if (options.MinLevel.HasValue)
    {
        query.Where("level", ">=", options.MinLevel.Value);
    }

    if (options.MaxLevel.HasValue)
    {
        query.Where("level", "<=", options.MaxLevel.Value);
    }

    // 가입일 범위
    if (options.RegisteredAfter.HasValue)
    {
        query.Where("created_at", ">=", options.RegisteredAfter.Value);
    }

    if (options.RegisteredBefore.HasValue)
    {
        query.Where("created_at", "<=", options.RegisteredBefore.Value);
    }

    // 정렬 및 페이징
    var results = await query
        .OrderByDesc("level")
        .ThenByDesc("exp")
        .Offset(options.Offset)
        .Limit(options.Limit)
        .GetAsync();

    return results.Select(MapToUser).Where(u => u != null).ToList()!;
}
```

`ThenByDesc`는 첫 번째 정렬 기준이 같을 때 두 번째 기준으로 정렬한다. 여기서는 레벨이 같으면 경험치로 정렬한다.

컨트롤러에 검색 엔드포인트를 추가한다.

## 7.4 트랜잭션 처리

게임 서버에서는 여러 테이블을 동시에 업데이트해야 하는 경우가 많다. 예를 들어 아이템을 구매할 때 사용자의 골드를 차감하고 인벤토리에 아이템을 추가하는 작업은 원자적으로 수행되어야 한다.

### 7.4.1 트랜잭션이란?

트랜잭션은 여러 데이터베이스 작업을 하나의 논리적 단위로 묶는 것이다. 트랜잭션 내의 모든 작업이 성공하면 커밋(Commit)되고, 하나라도 실패하면 모두 롤백(Rollback)된다.

트랜잭션의 ACID 속성은 다음과 같다.

- **Atomicity (원자성)**: 모든 작업이 성공하거나 모두 실패한다
- **Consistency (일관성)**: 트랜잭션 전후로 데이터베이스가 일관된 상태를 유지한다
- **Isolation (격리성)**: 동시에 실행되는 트랜잭션들이 서로 영향을 주지 않는다
- **Durability (지속성)**: 커밋된 트랜잭션은 영구적으로 저장된다

### 7.4.2 SqlKata에서 트랜잭션 사용

SqlKata.Execution은 트랜잭션을 지원한다. 예제로 레벨업 시 보상을 지급하는 기능을 구현한다.

먼저 `IUserRepository`에 메서드를 추가한다.

```csharp
public interface IUserRepository
```

```csharp
public async Task<bool> LevelUpWithRewardAsync(long userId, int newLevel, long goldReward, long gemReward)
{
    using var db = await _queryFactory.CreateAsync();
    var connection = db.Connection;
    
    // 트랜잭션 시작
    using var transaction = connection.BeginTransaction();
    db.Transaction = transaction;

    try
    {
        // 1. 사용자 정보 조회
        var user = await db.Query(TableName)
            .Where("user_id", userId)
            .WhereNull("deleted_at")
            .FirstOrDefaultAsync();

        if (user == null)
        {
            await transaction.RollbackAsync();
            return false;
        }

        // 2. 레벨 및 보상 업데이트
        var rowsAffected = await db.Query(TableName)
            .Where("user_id", userId)
            .UpdateAsync(new
            {
                level = newLevel,
                gold = (long)user.gold + goldReward,
                gem = (long)user.gem + gemReward,
                updated_at = DateTime.UtcNow
            });

        if (rowsAffected == 0)
        {
            await transaction.RollbackAsync();
            return false;
        }

        // 3. 레벨업 이력 기록 (선택적)
        // 별도의 level_up_history 테이블이 있다고 가정
        await db.Query("level_up_history").InsertAsync(new
        {
            user_id = userId,
            old_level = (int)user.level,
            new_level = newLevel,
            gold_reward = goldReward,
            gem_reward = gemReward,
            created_at = DateTime.UtcNow
        });

        // 모든 작업이 성공하면 커밋
        await transaction.CommitAsync();
        return true;
    }
    catch (Exception)
    {
        // 예외 발생 시 롤백
        await transaction.RollbackAsync();
        throw;
    }
}
```

`BeginTransaction()`으로 트랜잭션을 시작하고, `db.Transaction`에 할당하여 이후 모든 쿼리가 해당 트랜잭션 내에서 실행되도록 한다.

모든 작업이 성공하면 `CommitAsync()`를 호출하고, 실패하면 `RollbackAsync()`를 호출하여 변경사항을 취소한다.

### 7.4.3 복잡한 트랜잭션 예제

아이템 거래 시스템을 구현한다. 한 사용자에게서 골드를 차감하고 다른 사용자에게 골드를 지급하는 작업을 트랜잭션으로 처리한다.

```csharp
public async Task<bool> TransferGoldAsync(long fromUserId, long toUserId, long amount)
{
    using var db = await _queryFactory.CreateAsync();
    var connection = db.Connection;
    
    using var transaction = connection.BeginTransaction();
    db.Transaction = transaction;

    try
    {
        // 1. 송신자 조회 및 골드 확인
        var sender = await db.Query(TableName)
            .Where("user_id", fromUserId)
            .WhereNull("deleted_at")
            .FirstOrDefaultAsync();

        if (sender == null || (long)sender.gold < amount)
        {
            await transaction.RollbackAsync();
            return false;
        }

        // 2. 수신자 조회
        var receiver = await db.Query(TableName)
            .Where("user_id", toUserId)
            .WhereNull("deleted_at")
            .FirstOrDefaultAsync();

        if (receiver == null)
        {
            await transaction.RollbackAsync();
            return false;
        }

        // 3. 송신자 골드 차감
        var senderUpdated = await db.Query(TableName)
            .Where("user_id", fromUserId)
            .UpdateAsync(new
            {
                gold = (long)sender.gold - amount,
                updated_at = DateTime.UtcNow
            });

        if (senderUpdated == 0)
        {
            await transaction.RollbackAsync();
            return false;
        }

        // 4. 수신자 골드 추가
        var receiverUpdated = await db.Query(TableName)
            .Where("user_id", toUserId)
            .UpdateAsync(new
            {
                gold = (long)receiver.gold + amount,
                updated_at = DateTime.UtcNow
            });

        if (receiverUpdated == 0)
        {
            await transaction.RollbackAsync();
            return false;
        }

        // 5. 거래 이력 기록
        await db.Query("gold_transfer_history").InsertAsync(new
        {
            from_user_id = fromUserId,
            to_user_id = toUserId,
            amount = amount,
            created_at = DateTime.UtcNow
        });

        await transaction.CommitAsync();
        return true;
    }
    catch (Exception)
    {
        await transaction.RollbackAsync();
        throw;
    }
}
```

이 메서드는 5단계의 작업을 하나의 트랜잭션으로 묶는다. 중간에 하나라도 실패하면 모든 작업이 취소된다.

## 7.5 Repository 패턴의 확장

### 7.5.1 Base Repository 패턴

여러 엔티티에서 공통으로 사용하는 CRUD 메서드를 기본 클래스로 분리할 수 있다.

```csharp
public abstract class BaseRepository<T> where T : class
{
    protected readonly GameQueryFactory _queryFactory;
    protected abstract string TableName { get; }

    protected BaseRepository(GameQueryFactory queryFactory)
    {
        _queryFactory = queryFactory;
    }

    protected abstract T MapToEntity(dynamic data);

    public virtual async Task<T?> GetByIdAsync(long id, string idColumn = "id")
    {
        using var db = await _queryFactory.CreateAsync();

        var result = await db.Query(TableName)
            .Where(idColumn, id)
            .FirstOrDefaultAsync();

        return result != null ? MapToEntity(result) : null;
    }

    public virtual async Task<List<T>> GetAllAsync(int offset = 0, int limit = 100)
    {
        using var db = await _queryFactory.CreateAsync();

        var results = await db.Query(TableName)
            .Offset(offset)
            .Limit(limit)
            .GetAsync();

        return results.Select(r => MapToEntity(r)).ToList();
    }

    public virtual async Task<int> CountAsync()
    {
        using var db = await _queryFactory.CreateAsync();
        return await db.Query(TableName).CountAsync<int>();
    }
}
```

### 7.5.2 Unit of Work 패턴

여러 Repository의 작업을 하나의 트랜잭션으로 묶을 때 Unit of Work 패턴을 사용할 수 있다. 이는 고급 주제이므로 여기서는 개념만 소개한다.

Unit of Work는 여러 Repository를 하나로 묶고, 트랜잭션 관리를 중앙화한다.

## 7.6 정리

이 장에서는 Repository 패턴을 구현하여 데이터 접근 로직을 체계적으로 구조화하는 방법을 배웠다. Repository 패턴은 컨트롤러와 데이터베이스 사이에 추상화 레이어를 제공하여 코드의 재사용성과 테스트 가능성을 높인다.

SqlKata를 사용하여 복잡한 검색 조건, 집계 함수, JOIN 쿼리를 작성하는 방법을 실습했다. 동적 쿼리 구성을 통해 유연한 검색 기능을 구현할 수 있다.

트랜잭션 처리를 통해 여러 데이터베이스 작업을 원자적으로 수행하는 방법을 배웠다. 게임 서버에서 골드 거래, 아이템 구매 등 중요한 작업은 반드시 트랜잭션으로 보호해야 한다.

다음 장에서는 게임 서버의 핵심인 커스텀 토큰 기반 인증 시스템을 설계한다. 사용자를 식별하고 인증하는 메커니즘을 구현하여 보안성 있는 API를 만든다.

지금까지 작성한 코드는 실제 프로젝트의 기반이 된다. 앞으로 캐릭터, 아이템, 던전 등의 게임 시스템을 구현할 때도 동일한 Repository 패턴을 사용한다.
