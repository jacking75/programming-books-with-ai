# 5장. MySQL 연동 기초 (실습)
이 장에서는 ASP.NET Web API 프로젝트에서 MySQL 데이터베이스에 연결하고 기본적인 데이터베이스 작업을 수행하는 방법을 배운다. 4장에서 설계한 데이터베이스 구조를 실제로 코드에서 사용할 수 있도록 연결 설정을 구성한다.

## 5.1 MySqlConnector 패키지 설치
MySQL 데이터베이스와 연동하기 위해서는 ADO.NET 드라이버가 필요하다. .NET에서 MySQL을 사용하는 방법은 여러 가지가 있지만, 여기서는 성능이 우수하고 비동기 작업을 완벽하게 지원하는 `MySqlConnector` 패키지를 사용한다.

### 5.1.1 NuGet 패키지 설치
Visual Studio 2022에서 프로젝트를 열고 다음 방법 중 하나로 패키지를 설치한다.

**방법 1: NuGet 패키지 관리자 UI 사용**

1. 솔루션 탐색기에서 프로젝트를 마우스 오른쪽 버튼으로 클릭한다
2. "NuGet 패키지 관리"를 선택한다
3. "찾아보기" 탭에서 "MySqlConnector"를 검색한다
4. MySqlConnector 패키지를 선택하고 "설치" 버튼을 클릭한다

**방법 2: 패키지 관리자 콘솔 사용**

Visual Studio 상단 메뉴에서 "도구 > NuGet 패키지 관리자 > 패키지 관리자 콘솔"을 선택하고 다음 명령을 입력한다.

```powershell
Install-Package MySqlConnector
```

**방법 3: .NET CLI 사용**

프로젝트 폴더에서 터미널을 열고 다음 명령을 실행한다.

```bash
dotnet add package MySqlConnector
```

설치가 완료되면 프로젝트 파일(.csproj)을 확인하여 패키지가 추가되었는지 확인한다.

```xml
<ItemGroup>
  <PackageReference Include="MySqlConnector" Version="2.3.7" />
</ItemGroup>
```

버전 번호는 설치 시점에 따라 다를 수 있다.
  

## 5.2 Connection String 설정
데이터베이스에 연결하려면 Connection String이 필요하다. Connection String은 데이터베이스 서버의 위치, 데이터베이스 이름, 인증 정보 등을 포함하는 문자열이다.

### 5.2.1 Connection String 구조
MySQL Connection String의 기본 구조는 다음과 같다.

```
Server=서버주소;Port=포트번호;Database=데이터베이스명;Uid=사용자명;Pwd=비밀번호;
```

실제 예시는 다음과 같다.

```
Server=localhost;Port=3306;Database=game_db;Uid=root;Pwd=mypassword;
```

게임 서버에서 자주 사용하는 추가 옵션들이 있다.

```
Server=localhost;Port=3306;Database=game_db;Uid=root;Pwd=mypassword;
Charset=utf8mb4;AllowUserVariables=true;UseCompression=true;
```

각 옵션의 의미는 다음과 같다.

- **Server**: MySQL 서버의 호스트 주소다. localhost는 로컬 개발 환경을 의미한다
- **Port**: MySQL 서버의 포트 번호다. 기본값은 3306이다
- **Database**: 연결할 데이터베이스 이름이다
- **Uid**: 데이터베이스 사용자 이름이다
- **Pwd**: 데이터베이스 비밀번호다
- **Charset**: 문자 인코딩 방식이다. utf8mb4는 이모지를 포함한 모든 UTF-8 문자를 지원한다
- **AllowUserVariables**: 사용자 변수를 사용할 수 있게 한다
- **UseCompression**: 네트워크 트래픽 압축을 사용한다

### 5.2.2 appsettings.json에 Connection String 저장
Connection String은 하드코딩하지 않고 설정 파일에 저장한다. 프로젝트의 `appsettings.json` 파일을 열고 다음과 같이 작성한다.

```json
{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning"
    }
  },
  "AllowedHosts": "*",
  "ConnectionStrings": {
    "MySQL": "Server=localhost;Port=3306;Database=game_db;Uid=root;Pwd=yourpassword;Charset=utf8mb4;"
  }
}
```

`ConnectionStrings` 섹션에 MySQL 연결 정보를 추가한다. 여기서 `Pwd` 값은 실제 MySQL root 계정의 비밀번호로 변경해야 한다.

개발 환경에서만 사용할 별도의 설정을 만들고 싶다면 `appsettings.Development.json` 파일을 생성하여 다음과 같이 작성한다.

```json
{
  "Logging": {
    "LogLevel": {
      "Default": "Debug",
      "Microsoft.AspNetCore": "Information"
    }
  },
  "ConnectionStrings": {
    "MySQL": "Server=localhost;Port=3306;Database=game_db_dev;Uid=root;Pwd=devpassword;Charset=utf8mb4;"
  }
}
```

이렇게 하면 개발 환경에서는 `game_db_dev` 데이터베이스를 사용하고, 운영 환경에서는 `game_db` 데이터베이스를 사용하도록 분리할 수 있다.
  

## 5.3 데이터베이스 연결 관리
데이터베이스 연결을 효율적으로 관리하기 위한 클래스를 작성한다. 프로젝트에 `Database` 폴더를 생성하고 그 안에 `DbConnectionFactory.cs` 파일을 만든다.

### 5.3.1 DbConnectionFactory 클래스 작성

```csharp
using MySqlConnector;
using System.Data;

namespace GameServer.Database
{
    public class DbConnectionFactory
    {
        private readonly string _connectionString;

        public DbConnectionFactory(string connectionString)
        {
            _connectionString = connectionString;
        }

        public async Task<IDbConnection> CreateConnectionAsync()
        {
            var connection = new MySqlConnection(_connectionString);
            await connection.OpenAsync();
            return connection;
        }

        public IDbConnection CreateConnection()
        {
            var connection = new MySqlConnection(_connectionString);
            connection.Open();
            return connection;
        }
    }
}
```

이 클래스는 데이터베이스 연결을 생성하는 팩토리 역할을 한다. 생성자에서 Connection String을 받아 저장하고, `CreateConnectionAsync` 메서드를 통해 비동기적으로 데이터베이스 연결을 생성하고 열어서 반환한다.

비동기 메서드를 사용하는 이유는 네트워크 I/O 작업 중에 스레드를 차단하지 않기 위해서다. 이는 서버의 전체적인 처리 성능을 향상시킨다.

### 5.3.2 Program.cs에 서비스 등록
`DbConnectionFactory`를 의존성 주입 컨테이너에 등록한다. `Program.cs` 파일을 열고 다음과 같이 수정한다.

```csharp
using GameServer.Database;

var builder = WebApplication.CreateBuilder(args);

// Configuration에서 Connection String 가져오기
var connectionString = builder.Configuration.GetConnectionString("MySQL");

// DbConnectionFactory를 싱글톤으로 등록
builder.Services.AddSingleton(new DbConnectionFactory(connectionString));

// 컨트롤러 추가
builder.Services.AddControllers();

// Swagger 설정 (개발 환경)
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

var app = builder.Build();

// 개발 환경에서만 Swagger 사용
if (app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}

app.UseHttpsRedirection();
app.UseAuthorization();
app.MapControllers();

app.Run();
```

`GetConnectionString("MySQL")` 메서드는 `appsettings.json`의 `ConnectionStrings` 섹션에서 "MySQL" 키에 해당하는 값을 가져온다.

`AddSingleton`으로 등록하면 애플리케이션 실행 중에 하나의 인스턴스만 생성되어 모든 요청에서 재사용된다.

### 5.3.3 간단한 연결 테스트 API 작성
데이터베이스 연결이 정상적으로 작동하는지 확인하기 위한 테스트 API를 작성한다. `Controllers` 폴더에 `DatabaseTestController.cs` 파일을 생성한다.

```csharp
using GameServer.Database;
using Microsoft.AspNetCore.Mvc;
using MySqlConnector;

namespace GameServer.Controllers
{
    [ApiController]
    [Route("[controller]")]
    public class DatabaseTestController : ControllerBase
    {
        private readonly DbConnectionFactory _dbConnectionFactory;

        public DatabaseTestController(DbConnectionFactory dbConnectionFactory)
        {
            _dbConnectionFactory = dbConnectionFactory;
        }

        [HttpPost("connection-test")]
        public async Task<TestConnectionResponse> TestConnection()
        {
            try
            {
                using var connection = await _dbConnectionFactory.CreateConnectionAsync();
                
                // 간단한 쿼리 실행으로 연결 확인
                using var command = new MySqlCommand("SELECT VERSION()", (MySqlConnection)connection);
                var version = await command.ExecuteScalarAsync();

                return new TestConnectionResponse
                {
                    Result = ErrorCode.None,
                    Message = "데이터베이스 연결 성공",
                    MysqlVersion = version?.ToString() ?? string.Empty
                };
            }
            catch (Exception ex)
            {
                return new TestConnectionResponse
                {
                    Result = ErrorCode.DatabaseConnectionFailed,
                    Message = "데이터베이스 연결 실패",
                    Error = ex.Message
                };
            }
        }

        [HttpPost("create-tables")]
        public async Task<CreateTablesResponse> CreateTables()
        {
            try
            {
                using var connection = await _dbConnectionFactory.CreateConnectionAsync();

                // users 테이블 생성
                var createUsersTable = @"
                    CREATE TABLE IF NOT EXISTS users (
                        user_id BIGINT AUTO_INCREMENT PRIMARY KEY,
                        email VARCHAR(100) NOT NULL UNIQUE,
                        password_hash VARCHAR(255) NOT NULL,
                        nickname VARCHAR(50) NOT NULL,
                        created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
                        INDEX idx_email (email)
                    ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
                ";

                using var command = new MySqlCommand(createUsersTable, (MySqlConnection)connection);
                await command.ExecuteNonQueryAsync();

                return new CreateTablesResponse
                {
                    Result = ErrorCode.None,
                    Message = "테이블 생성 완료"
                };
            }
            catch (Exception ex)
            {
                return new CreateTablesResponse
                {
                    Result = ErrorCode.DatabaseError,
                    Message = "테이블 생성 실패",
                    Error = ex.Message
                };
            }
        }
    }

    public class TestConnectionResponse
    {
        public ErrorCode Result { get; set; }
        public string Message { get; set; } = string.Empty;
        public string MysqlVersion { get; set; } = string.Empty;
        public string Error { get; set; } = string.Empty;
    }

    public class CreateTablesResponse
    {
        public ErrorCode Result { get; set; }
        public string Message { get; set; } = string.Empty;
        public string Error { get; set; } = string.Empty;
    }
}
```

이 컨트롤러는 두 개의 테스트 엔드포인트를 제공한다.

첫 번째 `/DatabaseTest/connection-test` 엔드포인트는 데이터베이스에 연결하고 MySQL 버전을 조회하여 반환한다. 연결이 성공하면 MySQL 버전 정보를 볼 수 있다.

두 번째 `/DatabaseTest/create-tables` 엔드포인트는 실제로 테이블을 생성하는 예시다. `users` 테이블을 생성하는 DDL 쿼리를 실행한다.

### 5.3.4 API 테스트
Visual Studio에서 F5를 눌러 애플리케이션을 실행한다. 브라우저가 열리면 Swagger UI가 표시된다.

Swagger UI에서 `DatabaseTest` 섹션을 확장하고 `POST /DatabaseTest/connection-test` 엔드포인트를 찾는다. "Try it out" 버튼을 클릭하고 "Execute" 버튼을 누른다.

성공 응답 예시는 다음과 같다.

```json
{
  "success": true,
  "message": "데이터베이스 연결 성공",
  "mysqlVersion": "8.0.35"
}
```

만약 연결이 실패하면 다음과 같은 응답을 받는다.

```json
{
  "success": false,
  "message": "데이터베이스 연결 실패",
  "error": "Unable to connect to any of the specified MySQL hosts."
}
```

연결 실패 시 확인할 사항은 다음과 같다.

1. MySQL 서버가 실행 중인지 확인한다
2. `appsettings.json`의 Connection String이 올바른지 확인한다
3. MySQL 사용자 계정과 비밀번호가 정확한지 확인한다
4. 방화벽이 3306 포트를 차단하고 있지 않은지 확인한다

연결 테스트가 성공했다면 `POST /DatabaseTest/create-tables` 엔드포인트를 실행하여 테이블을 생성할 수 있다.
  

## 5.4 Connection Pool 이해
데이터베이스 연결은 생성하고 해제하는 데 비용이 많이 드는 리소스다. 매 요청마다 새로운 연결을 생성하고 닫는다면 성능이 크게 저하된다.

### 5.4.1 Connection Pool이란
Connection Pool은 미리 생성된 데이터베이스 연결들을 풀(Pool)에 보관하고 재사용하는 메커니즘이다. 애플리케이션이 데이터베이스 연결이 필요할 때 풀에서 유휴 연결을 가져오고, 사용이 끝나면 연결을 닫는 대신 풀에 반환한다.

MySqlConnector는 기본적으로 Connection Pool을 사용한다. 코드에서 `connection.Open()`을 호출하면 실제로는 새 연결을 생성하는 것이 아니라 풀에서 기존 연결을 가져온다. `connection.Close()`나 `using` 블록 종료 시 연결은 풀로 반환된다.

### 5.4.2 Connection Pool 설정
Connection String에 풀 관련 설정을 추가할 수 있다.

```
Server=localhost;Port=3306;Database=game_db;Uid=root;Pwd=mypassword;
Charset=utf8mb4;Pooling=true;MinimumPoolSize=5;MaximumPoolSize=100;
ConnectionIdleTimeout=180;
```

각 옵션의 의미는 다음과 같다.

- **Pooling**: Connection Pool 사용 여부다. 기본값은 true다
- **MinimumPoolSize**: 풀에 유지할 최소 연결 수다. 기본값은 0이다
- **MaximumPoolSize**: 풀에 생성할 최대 연결 수다. 기본값은 100이다
- **ConnectionIdleTimeout**: 유휴 연결이 풀에서 제거되기까지의 시간(초)이다. 기본값은 180초다

게임 서버처럼 동시 접속자가 많은 경우 MinimumPoolSize를 높게 설정하여 초기 연결 비용을 줄일 수 있다. MaximumPoolSize는 데이터베이스 서버가 허용하는 최대 연결 수보다 작게 설정해야 한다.

### 5.4.3 연결 사용 시 주의사항
Connection Pool을 효과적으로 사용하려면 다음 사항을 주의해야 한다.

**반드시 using 블록 사용**

```csharp
// 올바른 사용
using var connection = await _dbConnectionFactory.CreateConnectionAsync();
// 연결 사용
// using 블록 종료 시 자동으로 풀에 반환됨

// 잘못된 사용
var connection = await _dbConnectionFactory.CreateConnectionAsync();
// 연결 사용
// Close()를 명시적으로 호출하지 않으면 풀에 반환되지 않음
```

**장시간 연결 유지 금지**

데이터베이스 연결을 필요한 작업이 끝나면 즉시 반환해야 한다. 연결을 오래 유지하면 풀의 다른 요청이 연결을 얻지 못해 대기하게 된다.

```csharp
// 잘못된 예: 연결을 너무 오래 유지
using var connection = await _dbConnectionFactory.CreateConnectionAsync();
var data = await GetDataFromDatabase(connection);
await ProcessDataForLongTime(data); // 시간이 오래 걸리는 작업
await SaveDataToDatabase(connection, data);

// 올바른 예: 필요한 만큼만 연결 유지
List<Data> data;
using (var connection = await _dbConnectionFactory.CreateConnectionAsync())
{
    data = await GetDataFromDatabase(connection);
}

await ProcessDataForLongTime(data); // 연결 없이 처리

using (var connection = await _dbConnectionFactory.CreateConnectionAsync())
{
    await SaveDataToDatabase(connection, data);
}
```

**비동기 메서드 사용**

데이터베이스 작업은 네트워크 I/O가 발생하므로 항상 비동기 메서드를 사용해야 한다. 동기 메서드는 스레드를 차단하여 서버의 전체 처리량을 감소시킨다.

```csharp
// 올바른 사용
var result = await command.ExecuteScalarAsync();
var reader = await command.ExecuteReaderAsync();
var rowsAffected = await command.ExecuteNonQueryAsync();

// 피해야 할 사용
var result = command.ExecuteScalar(); // 동기 메서드
```

### 5.4.4 Connection Pool 모니터링
개발 중이나 운영 환경에서 Connection Pool의 상태를 모니터링하는 것이 중요하다. 다음 API를 추가하여 풀 상태를 확인할 수 있다.

```csharp
[HttpPost("pool-status")]
public GetPoolStatusResponse GetPoolStatus()
{
    var stats = MySqlConnection.ClearAllPools(); // 풀 초기화 및 통계 반환
    
    return new GetPoolStatusResponse
    {
        Result = ErrorCode.None,
        Message = "Connection Pool 초기화 완료",
        Note = "프로덕션 환경에서는 주의해서 사용해야 함"
    };
}

public class GetPoolStatusResponse
{
    public ErrorCode Result { get; set; }
    public string Message { get; set; } = string.Empty;
    public string Note { get; set; } = string.Empty;
}
```

실제 운영 환경에서는 더 정교한 모니터링 도구를 사용하는 것이 좋다.
  

## 5.5 기본 쿼리 실행 예제
SqlKata를 사용하기 전에 MySqlConnector의 기본 사용법을 이해하는 것이 중요하다. 간단한 CRUD 작업을 직접 구현해본다.

### 5.5.1 데이터 삽입 예제
`DatabaseTestController`에 사용자를 생성하는 메서드를 추가한다.

```csharp
[HttpPost("create-user")]
public async Task<CreateUserResponse> CreateUser([FromBody] CreateUserRequest request)
{
    try
    {
        using var connection = await _dbConnectionFactory.CreateConnectionAsync();

        var query = @"
            INSERT INTO users (email, password_hash, nickname, created_at)
            VALUES (@Email, @PasswordHash, @Nickname, @CreatedAt);
            SELECT LAST_INSERT_ID();
        ";

        using var command = new MySqlCommand(query, (MySqlConnection)connection);
        command.Parameters.AddWithValue("@Email", request.Email);
        command.Parameters.AddWithValue("@PasswordHash", request.Password); // 실제로는 해싱 필요
        command.Parameters.AddWithValue("@Nickname", request.Nickname);
        command.Parameters.AddWithValue("@CreatedAt", DateTime.UtcNow);

        var userId = await command.ExecuteScalarAsync();

        return new CreateUserResponse
        {
            Result = ErrorCode.None,
            UserId = Convert.ToInt64(userId),
            Message = "사용자 생성 완료"
        };
    }
    catch (MySqlException ex) when (ex.Number == 1062) // Duplicate entry
    {
        return new CreateUserResponse
        {
            Result = ErrorCode.EmailAlreadyExists,
            Message = "이미 존재하는 이메일입니다"
        };
    }
    catch (Exception ex)
    {
        return new CreateUserResponse
        {
            Result = ErrorCode.DatabaseError,
            Message = "사용자 생성 실패",
            Error = ex.Message
        };
    }
}

public class CreateUserRequest
{
    public string Email { get; set; }
    public string Password { get; set; }
    public string Nickname { get; set; }
}

public class CreateUserResponse
{
    public ErrorCode Result { get; set; }
    public long UserId { get; set; }
    public string Message { get; set; } = string.Empty;
    public string Error { get; set; } = string.Empty;
}
```

이 코드는 파라미터화된 쿼리를 사용한다. `@Email`, `@PasswordHash` 같은 플레이스홀더를 사용하고 `Parameters.AddWithValue`로 값을 바인딩한다. 이 방식은 SQL Injection 공격을 방지한다.

`ExecuteScalarAsync`는 쿼리의 첫 번째 행의 첫 번째 열 값을 반환한다. 여기서는 `LAST_INSERT_ID()`로 방금 삽입된 행의 자동 증가 ID를 가져온다.

`MySqlException`의 `Number` 속성으로 특정 에러를 처리할 수 있다. 1062는 중복 키 에러 코드다.

### 5.5.2 데이터 조회 예제

```csharp
[HttpPost("get-user")]
public async Task<GetUserResponse> GetUser([FromBody] GetUserRequest request)
{
    try
    {
        using var connection = await _dbConnectionFactory.CreateConnectionAsync();

        var query = "SELECT user_id, email, nickname, created_at FROM users WHERE email = @Email";

        using var command = new MySqlCommand(query, (MySqlConnection)connection);
        command.Parameters.AddWithValue("@Email", request.Email);

        using var reader = await command.ExecuteReaderAsync();

        if (await reader.ReadAsync())
        {
            return new GetUserResponse
            {
                Result = ErrorCode.None,
                User = new UserInfo
                {
                    UserId = reader.GetInt64("user_id"),
                    Email = reader.GetString("email"),
                    Nickname = reader.GetString("nickname"),
                    CreatedAt = reader.GetDateTime("created_at")
                }
            };
        }

        return new GetUserResponse
        {
            Result = ErrorCode.UserNotFound,
            Message = "사용자를 찾을 수 없습니다"
        };
    }
    catch (Exception ex)
    {
        return new GetUserResponse
        {
            Result = ErrorCode.DatabaseError,
            Message = "사용자 조회 실패",
            Error = ex.Message
        };
    }
}

public class GetUserRequest
{
    public string Email { get; set; }
}

public class GetUserResponse
{
    public ErrorCode Result { get; set; }
    public UserInfo? User { get; set; }
    public string Message { get; set; } = string.Empty;
    public string Error { get; set; } = string.Empty;
}

public class UserInfo
{
    public long UserId { get; set; }
    public string Email { get; set; } = string.Empty;
    public string Nickname { get; set; } = string.Empty;
    public DateTime CreatedAt { get; set; }
}
```

`ExecuteReaderAsync`는 `DbDataReader` 객체를 반환한다. `ReadAsync`로 다음 행으로 이동하고, 열 이름이나 인덱스로 값을 읽는다.

여러 행을 조회하려면 `while` 루프를 사용한다.

```csharp
var users = new List<object>();
while (await reader.ReadAsync())
{
    users.Add(new
    {
        userId = reader.GetInt64("user_id"),
        email = reader.GetString("email"),
        nickname = reader.GetString("nickname")
    });
}
```

## 5.6 정리
이 장에서는 MySQL 데이터베이스와 ASP.NET Web API를 연결하는 기초를 배웠다. MySqlConnector 패키지를 설치하고, Connection String을 설정하고, 데이터베이스 연결 팩토리 클래스를 작성했다. Connection Pool의 동작 원리와 올바른 사용법을 이해했고, 기본적인 CRUD 쿼리를 직접 작성하고 실행하는 방법을 배웠다.
