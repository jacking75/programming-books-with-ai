## 11.1 개요

이번 장에서는 기본적인 로그인/로그아웃 외에 실제 게임 서버에서 필요한 인증 관련 부가 기능들을 구현한다. 로그아웃 API를 통한 토큰 무효화, 토큰 갱신, 중복 로그인 방지, 다중 디바이스 관리 등 실무에서 자주 사용되는 기능들을 다룬다.

## 11.2 로그아웃 API 구현

### 11.2.1 로그아웃 요구사항

로그아웃은 단순히 클라이언트에서 토큰을 삭제하는 것이 아니라, 서버에서도 해당 토큰을 무효화해야 한다. 이를 통해 탈취된 토큰이 계속 사용되는 것을 방지할 수 있다.

```csharp
// Repositories/AuthRepository.cs
public async Task<bool> DeleteAuthTokenAsync(long userId)
{
    try
    {
        // MySQL에서 토큰 삭제
        var deletedCount = await _queryFactory.Query("auth_token")
            .Where("user_id", userId)
            .DeleteAsync();

        // Redis에서 토큰 삭제
        var redisKey = $"auth:{userId}";
        var redisDb = new RedisString<string>(_redisConnection, redisKey, null);
        await redisDb.DeleteAsync();

        return deletedCount > 0;
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, "Failed to delete auth token for UserId: {UserId}", userId);
        return false;
    }
}
```

```csharp
using GameApiServer.Models.Auth;
using GameApiServer.Repositories;
using Microsoft.AspNetCore.Mvc;

namespace GameApiServer.Controllers;

[ApiController]
[Route("api/[controller]")]
public class AuthController : ControllerBase
{
    private readonly IAuthRepository _authRepository;
    private readonly ILogger<AuthController> _logger;

    public AuthController(IAuthRepository authRepository, ILogger<AuthController> logger)
    {
        _authRepository = authRepository;
        _logger = logger;
    }

    [HttpPost("logout")]
    public async Task<LogoutResponse> Logout([FromBody] LogoutRequest request)
    {
        // 인증 미들웨어를 거쳤다면 이미 토큰이 검증된 상태

        {

            return new LogoutResponse { Result = ErrorCode.LogoutFail };
        }

        _logger.LogInformation("User logged out successfully. UserId: {UserId}", request.UserId);
        
        return new LogoutResponse { Result = ErrorCode.None };
    }
}
```

## 11.3 토큰 갱신 API 구현

### 11.3.1 토큰 갱신 필요성

게임 서버에서는 보안을 위해 토큰에 만료 시간을 설정한다. 하지만 게임 플레이 중에 토큰이 만료되면 사용자 경험이 나빠진다. 따라서 토큰 갱신 기능이 필요하다.

### 11.3.2 토큰 갱신 요청/응답 모델

### 11.3.3 토큰 갱신 메서드 추가

`AuthRepository`에 토큰 갱신 메서드를 추가한다.

```csharp
// Repositories/AuthRepository.cs
public async Task<(bool Success, string NewToken, DateTime ExpireAt)> RefreshAuthTokenAsync(long userId, string oldToken)
{
    try
    {
        // 기존 토큰 검증
        var isValid = await VerifyAuthTokenAsync(userId, oldToken);
        if (!isValid)
        {
            return (false, string.Empty, DateTime.MinValue);
        }

        // 새 토큰 생성
        var newToken = GenerateToken(userId);
        var expireAt = DateTime.UtcNow.AddHours(24);

        // MySQL에서 토큰 업데이트
        var updated = await _queryFactory.Query("auth_token")
            .Where("user_id", userId)
            .UpdateAsync(new
            {
                token = newToken,
                expire_at = expireAt,
                updated_at = DateTime.UtcNow
            });

        if (updated == 0)
        {
            return (false, string.Empty, DateTime.MinValue);
        }

        // Redis에 새 토큰 저장 (24시간)
        var redisKey = $"auth:{userId}";
        var redisDb = new RedisString<string>(_redisConnection, redisKey, TimeSpan.FromHours(24));
        await redisDb.SetAsync(newToken);

        return (true, newToken, expireAt);
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, "Failed to refresh token for UserId: {UserId}", userId);
        return (false, string.Empty, DateTime.MinValue);
    }
}
```

## 11.4 중복 로그인 방지

### 11.4.1 중복 로그인 방지 전략

게임에서는 보통 동일 계정으로 여러 곳에서 동시에 로그인하는 것을 제한한다. 새로운 로그인이 발생하면 기존 세션을 자동으로 종료시키는 방식을 사용한다.

### 11.4.2 로그인 시 기존 토큰 처리

기존 `LoginAsync` 메서드를 수정하여 중복 로그인을 방지한다.

```csharp
// Repositories/AuthRepository.cs
public async Task<(ErrorCode Result, string Token, DateTime ExpireAt)> LoginAsync(
    string email, 
    string password)
{
    try
    {
        // 사용자 조회
        var user = await _queryFactory.Query("user")
            .Where("email", email)
            .FirstOrDefaultAsync<User>();

        if (user == null)
        {
            return (ErrorCode.LoginFailUserNotExist, string.Empty, DateTime.MinValue);
        }

        // 비밀번호 검증
        var hashedPassword = HashPassword(password, user.Salt);
        if (hashedPassword != user.HashedPassword)
        {
            return (ErrorCode.LoginFailPasswordNotMatch, string.Empty, DateTime.MinValue);
        }

        // 기존 토큰이 있다면 삭제 (중복 로그인 방지)
        await DeleteAuthTokenAsync(user.UserId);

        // 새 토큰 생성
        var token = GenerateToken(user.UserId);
        var expireAt = DateTime.UtcNow.AddHours(24);

        // MySQL에 토큰 저장
        await _queryFactory.Query("auth_token").InsertAsync(new
        {
            user_id = user.UserId,
            token = token,
            expire_at = expireAt,
            created_at = DateTime.UtcNow
        });

        // Redis에 토큰 캐싱
        var redisKey = $"auth:{user.UserId}";
        var redisDb = new RedisString<string>(_redisConnection, redisKey, TimeSpan.FromHours(24));
        await redisDb.SetAsync(token);

        _logger.LogInformation("User logged in successfully. UserId: {UserId}", user.UserId);

        return (ErrorCode.None, token, expireAt);
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, "Login failed for email: {Email}", email);
        return (ErrorCode.LoginFailException, string.Empty, DateTime.MinValue);
    }
}
```

### 11.4.3 중복 로그인 알림

기존 세션이 강제 종료될 때 클라이언트에게 알림을 보내는 것이 좋다. 이를 위해 Redis Pub/Sub이나 SignalR을 사용할 수 있지만, 여기서는 간단하게 Redis에 플래그를 설정하는 방식을 사용한다.

```csharp
// Repositories/AuthRepository.cs
private async Task NotifyDuplicateLoginAsync(long userId)
{
    var redisKey = $"duplicate_login:{userId}";
    var redisDb = new RedisString<bool>(_redisConnection, redisKey, TimeSpan.FromMinutes(5));
    await redisDb.SetAsync(true);
}

public async Task<bool> CheckDuplicateLoginAsync(long userId)
{
    var redisKey = $"duplicate_login:{userId}";
    var redisDb = new RedisString<bool>(_redisConnection, redisKey, null);
    var result = await redisDb.GetAsync();
    
    if (result.HasValue && result.Value)
    {
        // 플래그 삭제
        await redisDb.DeleteAsync();
        return true;
    }
    
    return false;
}
```

수정된 `LoginAsync` 메서드에 알림 추가:

```csharp
// 기존 토큰이 있다면 중복 로그인 알림 설정 후 삭제
var existingToken = await _queryFactory.Query("auth_token")
    .Where("user_id", user.UserId)
    .FirstOrDefaultAsync<AuthToken>();

if (existingToken != null)
{
    await NotifyDuplicateLoginAsync(user.UserId);
    await DeleteAuthTokenAsync(user.UserId);
}
```

### 11.5.6 다중 디바이스 로그인 구현

```csharp
// Repositories/AuthRepository.cs
private const int MaxDevicesPerUser = 3; // 최대 디바이스 수

public async Task<(ErrorCode Result, string Token, DateTime ExpireAt)> LoginWithDeviceAsync(
    string email,
    string password,
    DeviceInfo deviceInfo)
{
    try
    {
        // 사용자 조회 및 비밀번호 검증은 동일
        var user = await _queryFactory.Query("user")
            .Where("email", email)
            .FirstOrDefaultAsync<User>();

        if (user == null)
        {
            return (ErrorCode.LoginFailUserNotExist, string.Empty, DateTime.MinValue);
        }

        var hashedPassword = HashPassword(password, user.Salt);
        if (hashedPassword != user.HashedPassword)
        {
            return (ErrorCode.LoginFailPasswordNotMatch, string.Empty, DateTime.MinValue);
        }

        // 디바이스 정보 저장 또는 업데이트
        var existingDevice = await _queryFactory.Query("device_info")
            .Where("user_id", user.UserId)
            .Where("device_unique_id", deviceInfo.DeviceUniqueId)
            .FirstOrDefaultAsync<DeviceInfo>();

        if (existingDevice == null)
        {
            // 새 디바이스 등록 전 디바이스 수 확인
            var deviceCount = await _queryFactory.Query("device_info")
                .Where("user_id", user.UserId)
                .CountAsync<int>();

            if (deviceCount >= MaxDevicesPerUser)
            {
                // 가장 오래된 디바이스 삭제
                var oldestDevice = await _queryFactory.Query("device_info")
                    .Where("user_id", user.UserId)
                    .OrderBy("last_login_at")
                    .FirstOrDefaultAsync<DeviceInfo>();

                if (oldestDevice != null)
                {
                    // 해당 디바이스의 토큰 삭제
                    await _queryFactory.Query("auth_token")
                        .Where("user_id", user.UserId)
                        .Where("device_unique_id", oldestDevice.DeviceUniqueId)
                        .DeleteAsync();

                    // 디바이스 정보 삭제
                    await _queryFactory.Query("device_info")
                        .Where("device_id", oldestDevice.DeviceId)
                        .DeleteAsync();
                }
            }

            // 새 디바이스 등록
            await _queryFactory.Query("device_info").InsertAsync(new
            {
                user_id = user.UserId,
                device_unique_id = deviceInfo.DeviceUniqueId,
                device_type = deviceInfo.DeviceType,
                device_model = deviceInfo.DeviceModel,
                os_version = deviceInfo.OsVersion,
                app_version = deviceInfo.AppVersion,
                last_login_at = DateTime.UtcNow,
                created_at = DateTime.UtcNow
            });
        }
        else
        {
            // 기존 디바이스 정보 업데이트
            await _queryFactory.Query("device_info")
                .Where("device_id", existingDevice.DeviceId)
                .UpdateAsync(new
                {
                    device_model = deviceInfo.DeviceModel,
                    os_version = deviceInfo.OsVersion,
                    app_version = deviceInfo.AppVersion,
                    last_login_at = DateTime.UtcNow
                });
        }

        // 해당 디바이스의 기존 토큰 삭제
        await _queryFactory.Query("auth_token")
            .Where("user_id", user.UserId)
            .Where("device_unique_id", deviceInfo.DeviceUniqueId)
            .DeleteAsync();

        // 새 토큰 생성
        var token = GenerateToken(user.UserId);
        var expireAt = DateTime.UtcNow.AddHours(24);

        // MySQL에 토큰 저장 (디바이스 정보 포함)
        await _queryFactory.Query("auth_token").InsertAsync(new
        {
            user_id = user.UserId,
            device_unique_id = deviceInfo.DeviceUniqueId,
            token = token,
            expire_at = expireAt,
            created_at = DateTime.UtcNow
        });

        // Redis에 토큰 캐싱 (디바이스별로 구분)
        var redisKey = $"auth:{user.UserId}:{deviceInfo.DeviceUniqueId}";
        var redisDb = new RedisString<string>(_redisConnection, redisKey, TimeSpan.FromHours(24));
        await redisDb.SetAsync(token);

        _logger.LogInformation(
            "User logged in with device. UserId: {UserId}, DeviceId: {DeviceId}",
            user.UserId,
            deviceInfo.DeviceUniqueId
        );

        return (ErrorCode.None, token, expireAt);
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, "Login with device failed for email: {Email}", email);
        return (ErrorCode.LoginFailException, string.Empty, DateTime.MinValue);
    }
}
```

### 11.5.7 디바이스별 토큰 검증 수정

기존 인증 미들웨어도 디바이스 정보를 고려하도록 수정해야 한다.

```csharp
// Repositories/AuthRepository.cs
public async Task<bool> VerifyAuthTokenWithDeviceAsync(
    long userId, 
    string token, 
    string deviceUniqueId)
{
    try
    {
        // Redis에서 먼저 확인 (디바이스별)
        var redisKey = $"auth:{userId}:{deviceUniqueId}";
        var redisDb = new RedisString<string>(_redisConnection, redisKey, null);
        var cachedToken = await redisDb.GetAsync();

        if (cachedToken.HasValue && cachedToken.Value == token)
        {
            return true;
        }

        // Redis 캐시 미스 시 MySQL에서 확인
        var authToken = await _queryFactory.Query("auth_token")
            .Where("user_id", userId)
            .Where("device_unique_id", deviceUniqueId)
            .Where("token", token)
            .Where("expire_at", ">", DateTime.UtcNow)
            .FirstOrDefaultAsync<AuthToken>();

        if (authToken == null)
        {
            return false;
        }

        // Redis에 다시 캐싱
        var ttl = authToken.ExpireAt - DateTime.UtcNow;
        if (ttl.TotalSeconds > 0)
        {
            var redisCacheDb = new RedisString<string>(
                _redisConnection, 
                redisKey, 
                ttl
            );
            await redisCacheDb.SetAsync(token);
        }

        return true;
    }
    catch (Exception ex)
    {
        _logger.LogError(
            ex,
            "Token verification failed. UserId: {UserId}, DeviceId: {DeviceId}",
            userId,
            deviceUniqueId
        );
        return false;
    }
}
```

### 11.6.3 강제 로그아웃 메서드

```csharp
// Repositories/AuthRepository.cs
public async Task<bool> ForceLogoutAsync(long userId, string reason, long adminUserId)
{
    try
    {
        // 모든 디바이스의 토큰 삭제
        var deletedCount = await _queryFactory.Query("auth_token")
            .Where("user_id", userId)
            .DeleteAsync();

        // Redis에서 모든 토큰 삭제
        // 디바이스별 키를 모두 찾아서 삭제
        var devices = await _queryFactory.Query("device_info")
            .Where("user_id", userId)
            .Get<DeviceInfo>();

        foreach (var device in devices)
        {
            var redisKey = $"auth:{userId}:{device.DeviceUniqueId}";
            var redisDb = new RedisString<string>(_redisConnection, redisKey, null);
            await redisDb.DeleteAsync();
        }

        // 강제 로그아웃 기록 저장
        await _queryFactory.Query("force_logout_log").InsertAsync(new
        {
            user_id = userId,
            admin_user_id = adminUserId,
            reason = reason,
            created_at = DateTime.UtcNow
        });

        // 강제 로그아웃 플래그 설정 (5분간 유지)
        var flagKey = $"force_logout:{userId}";
        var flagDb = new RedisString<bool>(_redisConnection, flagKey, TimeSpan.FromMinutes(5));
        await flagDb.SetAsync(true);

        _logger.LogWarning(
            "User force logged out. UserId: {UserId}, AdminId: {AdminId}, Reason: {Reason}",
            userId,
            adminUserId,
            reason
        );

        return deletedCount > 0;
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, "Force logout failed for UserId: {UserId}", userId);
        return false;
    }
}

public async Task<bool> CheckForceLogoutAsync(long userId)
{
    var redisKey = $"force_logout:{userId}";
    var redisDb = new RedisString<bool>(_redisConnection, redisKey, null);
    var result = await redisDb.GetAsync();

    return result.HasValue && result.Value;
}
```

## 11.10 정리

이번 장에서는 인증 시스템의 부가 기능들을 구현했다. 로그아웃 API를 통해 토큰을 안전하게 무효화하고, 토큰 갱신으로 사용자 경험을 개선했다. 중복 로그인 방지와 다중 디바이스 관리를 통해 계정 보안을 강화했으며, 강제 로그아웃 기능으로 운영상 필요한 계정 관리 기능을 추가했다.

주요 구현 내용은 다음과 같다.

- 로그아웃 시 MySQL과 Redis에서 토큰을 모두 삭제하여 완전한 무효화를 보장했다
- 토큰 갱신을 통해 게임 플레이 중 토큰 만료로 인한 불편함을 해소했다
- 중복 로그인 방지로 계정 공유를 차단하고 보안을 강화했다
- 다중 디바이스 관리로 사용자가 여러 기기에서 게임을 할 수 있도록 했다
- 강제 로그아웃으로 운영상 필요한 계정 차단 기능을 구현했다
