# 10장. 인증 미들웨어 구현 (실습)

이번 장에서는 모든 API 요청에 대해 자동으로 인증을 수행하는 미들웨어를 구현한다. 클라이언트가 보낸 UserId와 Token을 검증하고, 인증된 사용자만 API를 사용할 수 있도록 만든다.

## 10.1 미들웨어 개념 이해

### 10.1.1 ASP.NET Core 미들웨어란

미들웨어는 HTTP 요청 파이프라인에서 요청과 응답을 처리하는 컴포넌트다. 각 미들웨어는 요청을 처리하고 다음 미들웨어로 전달하거나, 응답을 즉시 반환할 수 있다.

**미들웨어 실행 순서**

```
HTTP 요청
   ↓
Middleware 1 (예: 로깅)
   ↓
Middleware 2 (예: 인증)
   ↓
Middleware 3 (예: 라우팅)
   ↓
Controller/Action
   ↓
Middleware 3
   ↓
Middleware 2
   ↓
Middleware 1
   ↓
HTTP 응답
```

### 10.1.2 인증 미들웨어의 역할

인증 미들웨어는 다음과 같은 역할을 수행한다.

**요청 검증**
- 요청에서 UserId와 Token을 추출한다.
- 토큰의 유효성을 검증한다.
- 토큰이 만료되었는지 확인한다.

**빠른 검증**
- Redis에서 먼저 토큰을 조회한다.
- Redis에 없으면 MySQL에서 조회한다.
- 검증 결과를 캐싱한다.

**컨텍스트 설정**
- 인증된 사용자 정보를 HttpContext에 저장한다.
- 컨트롤러에서 쉽게 사용자 정보를 가져올 수 있게 한다.

**인증 실패 처리**
- 유효하지 않은 토큰은 401 Unauthorized를 반환한다.
- 적절한 에러 메시지를 제공한다.

### 10.3.2 AuthService 토큰 검증 구현

Redis와 MySQL을 활용한 2단계 토큰 검증을 구현한다.

`Services/AuthService.cs`에 다음 메서드를 추가한다.

```csharp
using CloudStructures;
using CloudStructures.Structures;
using GameServer.Repositories;
using GameServer.Security;

namespace GameServer.Services;

public class AuthService : IAuthService
{
    private readonly IUserRepository _userRepository;
    private readonly IAuthTokenRepository _authTokenRepository;
    private readonly ITokenGenerator _tokenGenerator;
    private readonly RedisConnection _redisConnection;
    private readonly IConfiguration _configuration;

    public AuthService(
        IUserRepository userRepository,
        IAuthTokenRepository authTokenRepository,
        ITokenGenerator tokenGenerator,
        RedisConnection redisConnection,
        IConfiguration configuration)
    {
        _userRepository = userRepository;
        _authTokenRepository = authTokenRepository;
        _tokenGenerator = tokenGenerator;
        _redisConnection = redisConnection;
        _configuration = configuration;
    }

    // 기존 LoginAsync 메서드는 9장과 동일

    public async Task<(bool isValid, string message)> ValidateTokenAsync(long userId, string token)
    {
        try
        {
            // 1. 토큰 자체의 유효성 검증 (구조 및 서명)
            var (isStructureValid, tokenUserId) = _tokenGenerator.ValidateToken(token);
            
            if (!isStructureValid)
            {
                return (false, "Invalid token structure");
            }

            // 2. 토큰의 UserId와 요청의 UserId 일치 확인
            if (tokenUserId != userId)
            {
                return (false, "Token does not match user");
            }

            // 3. Redis에서 토큰 조회 (빠른 검증)
            var redisToken = await GetTokenFromRedisAsync(userId);
            
            if (!string.IsNullOrEmpty(redisToken))
            {
                if (redisToken == token)
                {
                    return (true, "Token validated from Redis");
                }
                else
                {
                    return (false, "Token mismatch in Redis");
                }
            }

            // 4. Redis에 없으면 MySQL에서 조회 (캐시 미스)
            var dbToken = await _authTokenRepository.GetTokenByUserIdAsync(userId);
            
            if (string.IsNullOrEmpty(dbToken))
            {
                return (false, "Token not found");
            }

            if (dbToken != token)
            {
                return (false, "Token mismatch in database");
            }

            // 5. MySQL에서 찾은 토큰을 Redis에 캐싱
            var tokenExpirationHours = _configuration.GetValue<int>("Security:TokenExpirationHours", 24);
            await CacheTokenToRedisAsync(userId, token, tokenExpirationHours);

            return (true, "Token validated from database and cached");
        }
        catch (Exception ex)
        {
            return (false, $"Token validation error: {ex.Message}");
        }
    }

    private async Task<string?> GetTokenFromRedisAsync(long userId)
    {
        try
        {
            var redisKey = $"auth:token:{userId}";
            var redisString = new RedisString<string>(_redisConnection, redisKey, null);
            var result = await redisString.GetAsync();
            return result.HasValue ? result.Value : null;
        }
        catch
        {
            return null;
        }
    }

    private async Task CacheTokenToRedisAsync(long userId, string token, int expirationHours)
    {
        try
        {
            var redisKey = $"auth:token:{userId}";
            var redisString = new RedisString<string>(_redisConnection, redisKey, TimeSpan.FromHours(expirationHours));
            await redisString.SetAsync(token);
        }
        catch
        {
            // Redis 캐싱 실패는 로그만 남기고 계속 진행
        }
    }
}
```

## 10.4 커스텀 미들웨어 작성

### 10.4.1 미들웨어 클래스 생성

`Middlewares` 폴더를 생성하고 `AuthenticationMiddleware.cs` 파일을 만든다.

```csharp
using System.Text;
using System.Text.Json;
using GameServer.Services;

namespace GameServer.Middlewares;

public class AuthenticationMiddleware
{
    private readonly RequestDelegate _next;
    private readonly ILogger<AuthenticationMiddleware> _logger;

    public AuthenticationMiddleware(RequestDelegate next, ILogger<AuthenticationMiddleware> logger)
    {
        _next = next;
        _logger = logger;
    }

    public async Task InvokeAsync(HttpContext context, IAuthService authService)
    {
        // 1. 인증이 필요없는 경로는 건너뛰기
        if (IsPublicEndpoint(context.Request.Path))
        {
            await _next(context);
            return;
        }

        // 2. 요청 바디에서 UserId와 Token 추출
        var (success, userId, token, requestBody) = await ExtractAuthInfoAsync(context.Request);

        if (!success)
        {
            await WriteUnauthorizedResponseAsync(context, "Missing authentication information");
            return;
        }

        // 3. 토큰 검증
        var (isValid, message) = await authService.ValidateTokenAsync(userId, token);

        if (!isValid)
        {
            _logger.LogWarning("Authentication failed for UserId: {UserId}, Message: {Message}", userId, message);
            await WriteUnauthorizedResponseAsync(context, message);
            return;
        }

        // 4. 인증 성공 - HttpContext에 사용자 정보 저장
        context.Items["UserId"] = userId;
        context.Items["Token"] = token;

        // 5. 원본 바디를 복원하여 다음 미들웨어/컨트롤러가 사용할 수 있게 함
        context.Request.Body = new MemoryStream(Encoding.UTF8.GetBytes(requestBody));

        // 6. 다음 미들웨어로 진행
        await _next(context);
    }

    private bool IsPublicEndpoint(PathString path)
    {
        // 인증이 필요없는 엔드포인트 목록
        var publicPaths = new[]
        {
            "/api/auth/login",
            "/api/auth/register",
            "/health",
            "/swagger"
        };

        return publicPaths.Any(p => path.StartsWithSegments(p, StringComparison.OrdinalIgnoreCase));
    }

    private async Task<(bool success, long userId, string token, string requestBody)> ExtractAuthInfoAsync(HttpRequest request)
    {
        try
        {
            // Body를 읽을 수 있도록 설정
            request.EnableBuffering();

            using var reader = new StreamReader(request.Body, Encoding.UTF8, leaveOpen: true);
            var requestBody = await reader.ReadToEndAsync();
            
            // Body 스트림 위치를 처음으로 되돌림
            request.Body.Position = 0;

            if (string.IsNullOrWhiteSpace(requestBody))
            {
                return (false, 0, string.Empty, string.Empty);
            }

            // JSON 파싱하여 UserId와 Token 추출
            using var document = JsonDocument.Parse(requestBody);
            var root = document.RootElement;

            if (!root.TryGetProperty("userId", out var userIdElement) ||
                !root.TryGetProperty("token", out var tokenElement))
            {
                return (false, 0, string.Empty, requestBody);
            }

            var userId = userIdElement.GetInt64();
            var token = tokenElement.GetString() ?? string.Empty;

            return (true, userId, token, requestBody);
        }
        catch
        {
            return (false, 0, string.Empty, string.Empty);
        }
    }

    private async Task WriteUnauthorizedResponseAsync(HttpContext context, string message)
    {
        context.Response.StatusCode = StatusCodes.Status401Unauthorized;
        context.Response.ContentType = "application/json";

        var response = new
        {
            success = false,
            message = message
        };

        await context.Response.WriteAsJsonAsync(response);
    }
}
```

using Microsoft.AspNetCore.Mvc;
using GameServer.Models.Request;
using GameServer.Models.Response;
using GameServer.Repositories;

namespace GameServer.Controllers;

[ApiController]
[Route("api/[controller]")]
public class UserController : ControllerBase
{
    private readonly IUserRepository _userRepository;

    public UserController(IUserRepository userRepository)
    {
        _userRepository = userRepository;
    }

    [HttpPost("profile")]
    public async Task<ActionResult<GetProfileResponse>> GetProfile([FromBody] GetProfileRequest request)
    {

        {

            {
                Success = false,
                Message = "User not found"
            });
        }

        return Ok(new GetProfileResponse
        {
            Success = true,
            UserId = userId,
            Email = "test@example.com",
            Nickname = "TestUser",
            Message = "Profile retrieved successfully"
        });
    }

### 10.6.3 UserRepository 확장

사용자 상세 정보를 조회하는 메서드를 추가한다.

`Repositories/IUserRepository.cs`를 수정한다.

```csharp
namespace GameServer.Repositories;

public interface IUserRepository
{
    Task<(bool exists, long userId, string password)> GetUserByEmailAsync(string email);
    Task<(bool exists, string email, string nickname)> GetUserByIdAsync(long userId);
}
```

`Repositories/UserRepository.cs`에 메서드를 구현한다.

```csharp
using SqlKata.Execution;

namespace GameServer.Repositories;

public class UserRepository : IUserRepository
{
    private readonly QueryFactory _queryFactory;

    public UserRepository(QueryFactory queryFactory)
    {
        _queryFactory = queryFactory;
    }

    public async Task<(bool exists, long userId, string password)> GetUserByEmailAsync(string email)
    {
        var user = await _queryFactory.Query("users")
            .Select("user_id", "password")
            .Where("email", email)
            .FirstOrDefaultAsync();

        if (user == null)
        {
            return (false, 0, string.Empty);
        }

        return (true, (long)user.user_id, (string)user.password);
    }

    public async Task<(bool exists, string email, string nickname)> GetUserByIdAsync(long userId)
    {
        var user = await _queryFactory.Query("users")
            .Select("email", "nickname")
            .Where("user_id", userId)
            .FirstOrDefaultAsync();

        if (user == null)
        {
            return (false, string.Empty, string.Empty);
        }

        return (true, (string)user.email, (string)user.nickname);
    }
}
```

### 10.9.2 캐시 워밍

서버 시작 시 자주 사용되는 토큰을 미리 Redis에 로드할 수 있다.

        _logger = logger;

    {

        {

### 10.9.3 토큰 검증 결과 캐싱

동일한 토큰에 대한 반복적인 검증을 피하기 위해 메모리 캐시를 사용할 수 있다.

```csharp
using Microsoft.Extensions.Caching.Memory;

public class AuthService : IAuthService
{
    private readonly IMemoryCache _memoryCache;

    public async Task<(bool isValid, string message)> ValidateTokenAsync(long userId, string token)
    {
        // 메모리 캐시 확인
        var cacheKey = $"token_validation:{userId}:{token}";
        
        if (_memoryCache.TryGetValue<bool>(cacheKey, out var cachedResult))
        {
            return (cachedResult, "Validated from memory cache");
        }

        // 실제 검증 수행
        var (isValid, message) = await PerformValidationAsync(userId, token);

        // 결과를 메모리 캐시에 저장 (짧은 시간)
        if (isValid)
        {
            _memoryCache.Set(cacheKey, true, TimeSpan.FromMinutes(5));
        }

        return (isValid, message);
    }
}
```

## 10.10 보안 강화

### 10.10.1 요청 빈도 제한

같은 IP에서 너무 많은 인증 실패 요청이 오면 차단한다.

`Middlewares/RateLimitMiddleware.cs` 파일을 생성한다.

```csharp
using Microsoft.Extensions.Caching.Memory;

namespace GameServer.Middlewares;

public class RateLimitMiddleware
{
    private readonly RequestDelegate _next;
    private readonly IMemoryCache _cache;
    private readonly ILogger<RateLimitMiddleware> _logger;
    private const int MaxFailedAttempts = 5;
    private static readonly TimeSpan BlockDuration = TimeSpan.FromMinutes(15);

    public RateLimitMiddleware(
        RequestDelegate next,
        IMemoryCache cache,
        ILogger<RateLimitMiddleware> logger)
    {
        _next = next;
        _cache = cache;
        _logger = logger;
    }

    public async Task InvokeAsync(HttpContext context)
    {
        var ipAddress = context.Connection.RemoteIpAddress?.ToString() ?? "unknown";
        var cacheKey = $"rate_limit:{ipAddress}";

        // 차단 상태 확인
        if (_cache.TryGetValue<int>(cacheKey, out var failedAttempts))
        {
            if (failedAttempts >= MaxFailedAttempts)
            {
                _logger.LogWarning("Rate limit exceeded for IP: {IpAddress}", ipAddress);
                context.Response.StatusCode = StatusCodes.Status429TooManyRequests;
                await context.Response.WriteAsJsonAsync(new
                {
                    success = false,
                    message = "Too many failed attempts. Please try again later."
                });
                return;
            }
        }

        await _next(context);

        // 인증 실패 시 카운트 증가
        if (context.Response.StatusCode == StatusCodes.Status401Unauthorized)
        {
            var currentCount = failedAttempts + 1;
            _cache.Set(cacheKey, currentCount, BlockDuration);
        }
    }
}
```

## 10.11 로깅 추가

### 10.11.1 인증 실패 로깅

인증 미들웨어에 상세한 로깅을 추가한다.

```csharp
public async Task InvokeAsync(HttpContext context, IAuthService authService)
{
    if (IsPublicEndpoint(context.Request.Path))
    {
        await _next(context);
        return;
    }

    var (success, userId, token, requestBody) = await ExtractAuthInfoAsync(context.Request);

    if (!success)
    {
        _logger.LogWarning(
            "Authentication info extraction failed. Path: {Path}, IP: {IP}",
            context.Request.Path,
            context.Connection.RemoteIpAddress);
        
        await WriteUnauthorizedResponseAsync(context, "Missing authentication information");
        return;
    }

    var (isValid, message) = await authService.ValidateTokenAsync(userId, token);

    if (!isValid)
    {
        _logger.LogWarning(
            "Token validation failed. UserId: {UserId}, Message: {Message}, Path: {Path}, IP: {IP}",
            userId,
            message,
            context.Request.Path,
            context.Connection.RemoteIpAddress);
        
        await WriteUnauthorizedResponseAsync(context, message);
        return;
    }

    _logger.LogInformation(
        "Authentication successful. UserId: {UserId}, Path: {Path}",
        userId,
        context.Request.Path);

    context.Items["UserId"] = userId;
    context.Items["Token"] = token;
    context.Request.Body = new MemoryStream(Encoding.UTF8.GetBytes(requestBody));

    await _next(context);
}
```
