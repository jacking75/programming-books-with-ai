# 23장. 로깅 및 모니터링

## 23.1 개요

게임 서버 운영에서 로깅과 모니터링은 필수적인 요소다. 로그는 서버에서 발생하는 모든 이벤트를 기록하여 문제 발생 시 원인을 추적하고, 서비스 품질을 개선하는 데 활용된다. 모니터링은 서버의 현재 상태를 실시간으로 파악하여 장애를 예방하고 신속하게 대응하는 데 도움을 준다.

이번 장에서는 Serilog를 사용하여 체계적인 로깅 시스템을 구축하고, API 요청/응답, 에러, 인증 실패, 데이터베이스 성능 등 다양한 측면을 모니터링하는 방법을 학습한다.

로깅 시스템의 주요 목표는 다음과 같다.

**문제 추적**: 에러 발생 시 원인을 빠르게 파악한다.

**보안 감사**: 인증 실패나 비정상적인 접근 시도를 기록한다.

**성능 분석**: API 응답 시간이나 데이터베이스 쿼리 성능을 측정한다.

**사용자 행동 분석**: 게임 플레이 패턴을 파악하여 서비스를 개선한다.

## 23.2 Serilog 설치 및 설정

Serilog는 구조화된 로깅을 지원하는 .NET 라이브러리로, 다양한 출력 대상(파일, 콘솔, 데이터베이스 등)을 지원한다.

NuGet 패키지를 설치한다.

```bash
dotnet add package Serilog.AspNetCore
dotnet add package Serilog.Sinks.Console
dotnet add package Serilog.Sinks.File
dotnet add package Serilog.Sinks.MySQL
dotnet add package Serilog.Enrichers.Environment
dotnet add package Serilog.Enrichers.Thread
dotnet add package Serilog.Settings.Configuration
```

Program.cs에 Serilog를 설정한다.

```csharp
// Program.cs
using Serilog;
using Serilog.Events;

var builder = WebApplication.CreateBuilder(args);

// Serilog 설정
Log.Logger = new LoggerConfiguration()
    .MinimumLevel.Information()
    .MinimumLevel.Override("Microsoft", LogEventLevel.Warning)
    .MinimumLevel.Override("Microsoft.AspNetCore", LogEventLevel.Warning)
    .Enrich.FromLogContext()
    .Enrich.WithMachineName()
    .Enrich.WithThreadId()
    .Enrich.WithProperty("Application", "GameAPIServer")
    .WriteTo.Console(
        outputTemplate: "[{Timestamp:yyyy-MM-dd HH:mm:ss.fff}] [{Level:u3}] {Message:lj}{NewLine}{Exception}")
    .WriteTo.File(
        path: "logs/log-.txt",
        rollingInterval: RollingInterval.Day,
        outputTemplate: "[{Timestamp:yyyy-MM-dd HH:mm:ss.fff}] [{Level:u3}] [{SourceContext}] {Message:lj}{NewLine}{Exception}",
        retainedFileCountLimit: 30)
    .WriteTo.File(
        path: "logs/error-.txt",
        restrictedToMinimumLevel: LogEventLevel.Error,
        rollingInterval: RollingInterval.Day,
        outputTemplate: "[{Timestamp:yyyy-MM-dd HH:mm:ss.fff}] [{Level:u3}] [{SourceContext}] {Message:lj}{NewLine}{Exception}",
        retainedFileCountLimit: 90)
    .CreateLogger();

builder.Host.UseSerilog();

// 나머지 서비스 등록
builder.Services.AddControllers();

var app = builder.Build();

// Serilog 요청 로깅 미들웨어
app.UseSerilogRequestLogging(options =>
{
    options.MessageTemplate = "HTTP {RequestMethod} {RequestPath} responded {StatusCode} in {Elapsed:0.0000} ms";
    options.EnrichDiagnosticContext = (diagnosticContext, httpContext) =>
    {
        diagnosticContext.Set("RequestHost", httpContext.Request.Host.Value);
        diagnosticContext.Set("RequestScheme", httpContext.Request.Scheme);
        diagnosticContext.Set("RemoteIP", httpContext.Connection.RemoteIpAddress);
        
        // 인증된 사용자 정보 추가
        if (httpContext.Items.TryGetValue("UserId", out var userId))
        {
            diagnosticContext.Set("UserId", userId);
        }
    };
});

app.UseRouting();
app.UseAuthentication();
app.UseAuthorization();
app.MapControllers();

try
{
    Log.Information("게임 API 서버 시작");
    app.Run();
}
catch (Exception ex)
{
    Log.Fatal(ex, "서버 시작 실패");
}
finally
{
    Log.CloseAndFlush();
}
```

설정을 보면 세 가지 출력 대상을 구성했다. 콘솔은 개발 중 실시간으로 로그를 확인하기 위해 사용하고, 일반 로그 파일은 모든 레벨의 로그를 일자별로 저장하며 30일간 보관한다. 에러 로그 파일은 에러 이상의 심각한 로그만 별도로 저장하여 90일간 보관한다.

## 23.3 구조화된 로깅

Serilog의 강력한 기능 중 하나는 구조화된 로깅이다. 단순한 문자열이 아닌 속성과 값을 포함한 로그를 기록하여 나중에 검색과 분석이 용이하다.

```csharp
// Services/LoggingExamples.cs
namespace GameAPIServer.Services;

public class LoggingExamples
{
    private readonly ILogger<LoggingExamples> _logger;
    
    public LoggingExamples(ILogger<LoggingExamples> logger)
    {
        _logger = logger;
    }
    
    // 기본 로깅
    public void BasicLogging()
    {
        _logger.LogInformation("서버 시작됨");
        _logger.LogWarning("메모리 사용량이 높음");
        _logger.LogError("데이터베이스 연결 실패");
    }
    
    // 구조화된 로깅 (권장)
    public void StructuredLogging(long userId, string action)
    {
        _logger.LogInformation(
            "사용자 액션 수행: {UserId}, {Action}, {Timestamp}",
            userId, action, DateTime.UtcNow);
    }
    
    // 예외 로깅
    public void ExceptionLogging(Exception ex, long userId)
    {
        _logger.LogError(ex,
            "사용자 요청 처리 중 오류 발생. UserId: {UserId}",
            userId);
    }
    
    // 성능 측정 로깅
    public void PerformanceLogging(string operation, long userId, TimeSpan elapsed)
    {
        _logger.LogInformation(
            "작업 완료: {Operation}, UserId: {UserId}, Duration: {Duration}ms",
            operation, userId, elapsed.TotalMilliseconds);
        
        if (elapsed.TotalMilliseconds > 1000)
        {
            _logger.LogWarning(
                "느린 작업 감지: {Operation}, UserId: {UserId}, Duration: {Duration}ms",
                operation, userId, elapsed.TotalMilliseconds);
        }
    }
}
```

구조화된 로깅을 사용하면 로그 분석 도구에서 특정 사용자의 모든 활동을 쉽게 추적하거나, 특정 작업의 평균 실행 시간을 계산할 수 있다.

## 23.4 API 요청/응답 로깅 미들웨어

모든 API 요청과 응답을 자동으로 로깅하는 미들웨어를 구현한다.

```csharp
// Middlewares/RequestResponseLoggingMiddleware.cs
using System.Diagnostics;
using System.Text;

namespace GameAPIServer.Middlewares;

public class RequestResponseLoggingMiddleware
{
    private readonly RequestDelegate _next;
    private readonly ILogger<RequestResponseLoggingMiddleware> _logger;
    
    public RequestResponseLoggingMiddleware(
        RequestDelegate next,
        ILogger<RequestResponseLoggingMiddleware> logger)
    {
        _next = next;
        _logger = logger;
    }
    
    public async Task InvokeAsync(HttpContext context)
    {
        var stopwatch = Stopwatch.StartNew();
        
        // 요청 정보 로깅
        await LogRequestAsync(context);
        
        // 원본 응답 스트림 저장
        var originalBodyStream = context.Response.Body;
        
        try
        {
            using var responseBody = new MemoryStream();
            context.Response.Body = responseBody;
            
            // 다음 미들웨어 실행
            await _next(context);
            
            stopwatch.Stop();
            
            // 응답 정보 로깅
            await LogResponseAsync(context, stopwatch.ElapsedMilliseconds);
            
            // 원본 스트림에 응답 복사
            await responseBody.CopyToAsync(originalBodyStream);
        }
        finally
        {
            context.Response.Body = originalBodyStream;
        }
    }
    
    private async Task LogRequestAsync(HttpContext context)
    {
        try
        {
            context.Request.EnableBuffering();
            
            var requestBody = string.Empty;
            if (context.Request.ContentLength > 0)
            {
                using var reader = new StreamReader(
                    context.Request.Body,
                    Encoding.UTF8,
                    detectEncodingFromByteOrderMarks: false,
                    leaveOpen: true);
                
                requestBody = await reader.ReadToEndAsync();
                context.Request.Body.Position = 0;
                
                // 민감한 정보는 마스킹
                requestBody = MaskSensitiveData(requestBody);
            }
            
            var userId = context.Items.TryGetValue("UserId", out var userIdObj)
                ? (long?)userIdObj
                : null;
            
            _logger.LogInformation(
                "API 요청: {Method} {Path}, UserId: {UserId}, IP: {IP}, Body: {Body}",
                context.Request.Method,
                context.Request.Path,
                userId,
                context.Connection.RemoteIpAddress,
                requestBody);
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "요청 로깅 중 오류 발생");
        }
    }
    
    private async Task LogResponseAsync(HttpContext context, long elapsedMs)
    {
        try
        {
            context.Response.Body.Seek(0, SeekOrigin.Begin);
            var responseBody = await new StreamReader(context.Response.Body).ReadToEndAsync();
            context.Response.Body.Seek(0, SeekOrigin.Begin);
            
            var userId = context.Items.TryGetValue("UserId", out var userIdObj)
                ? (long?)userIdObj
                : null;
            
            var logLevel = context.Response.StatusCode >= 400
                ? LogLevel.Warning
                : LogLevel.Information;
            
            _logger.Log(logLevel,
                "API 응답: {Method} {Path}, UserId: {UserId}, StatusCode: {StatusCode}, Duration: {Duration}ms, Body: {Body}",
                context.Request.Method,
                context.Request.Path,
                userId,
                context.Response.StatusCode,
                elapsedMs,
                responseBody.Length > 1000 ? responseBody.Substring(0, 1000) + "..." : responseBody);
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "응답 로깅 중 오류 발생");
        }
    }
    
    private string MaskSensitiveData(string data)
    {
        if (string.IsNullOrEmpty(data))
            return data;
        
        // 비밀번호 마스킹
        data = System.Text.RegularExpressions.Regex.Replace(
            data,
            @"""password""\s*:\s*""[^""]*""",
            @"""password"":""***""",
            System.Text.RegularExpressions.RegexOptions.IgnoreCase);
        
        // 토큰 마스킹
        data = System.Text.RegularExpressions.Regex.Replace(
            data,
            @"""token""\s*:\s*""[^""]*""",
            @"""token"":""***""",
            System.Text.RegularExpressions.RegexOptions.IgnoreCase);
        
        return data;
    }
}

// Program.cs에 미들웨어 등록
app.UseMiddleware<RequestResponseLoggingMiddleware>();
```

이 미들웨어는 모든 API 요청과 응답을 자동으로 기록한다. 요청 시각, 처리 시간, 사용자 ID, 상태 코드 등을 포함하여 API 사용 패턴을 분석하고 성능 문제를 파악하는 데 활용한다. 비밀번호나 토큰 같은 민감한 정보는 자동으로 마스킹하여 보안을 유지한다.

## 23.5 에러 로깅 및 추적

전역 예외 처리 미들웨어를 구현하여 모든 에러를 체계적으로 로깅한다.

```csharp
// Middlewares/GlobalExceptionHandlerMiddleware.cs
using System.Net;
using System.Text.Json;

namespace GameAPIServer.Middlewares;

public class GlobalExceptionHandlerMiddleware
{
    private readonly RequestDelegate _next;
    private readonly ILogger<GlobalExceptionHandlerMiddleware> _logger;
    
    public GlobalExceptionHandlerMiddleware(
        RequestDelegate next,
        ILogger<GlobalExceptionHandlerMiddleware> logger)
    {
        _next = next;
        _logger = logger;
    }
    
    public async Task InvokeAsync(HttpContext context)
    {
        try
        {
            await _next(context);
        }
        catch (Exception ex)
        {
            await HandleExceptionAsync(context, ex);
        }
    }
    
    private async Task HandleExceptionAsync(HttpContext context, Exception exception)
    {
        // 에러 ID 생성 (추적용)
        var errorId = Guid.NewGuid().ToString();
        
        // 사용자 정보 추출
        var userId = context.Items.TryGetValue("UserId", out var userIdObj)
            ? (long?)userIdObj
            : null;
        
        // 에러 로깅
        _logger.LogError(exception,
            "처리되지 않은 예외 발생. ErrorId: {ErrorId}, UserId: {UserId}, Path: {Path}, Method: {Method}",
            errorId,
            userId,
            context.Request.Path,
            context.Request.Method);
        
        // 에러 타입별 상태 코드 결정
        var statusCode = exception switch
        {
            ArgumentException => HttpStatusCode.BadRequest,
            UnauthorizedAccessException => HttpStatusCode.Unauthorized,
            KeyNotFoundException => HttpStatusCode.NotFound,
            InvalidOperationException => HttpStatusCode.Conflict,
            _ => HttpStatusCode.InternalServerError
        };
        
        // 응답 생성
        context.Response.ContentType = "application/json";
        context.Response.StatusCode = (int)statusCode;
        
        var response = new
        {
            errorId = errorId,
            message = "서버 오류가 발생했다",
            detail = context.RequestServices.GetRequiredService<IWebHostEnvironment>().IsDevelopment()
                ? exception.Message
                : "자세한 내용은 관리자에게 문의하라",
            timestamp = DateTime.UtcNow
        };
        
        var jsonResponse = JsonSerializer.Serialize(response);
        await context.Response.WriteAsync(jsonResponse);
    }
}

// Program.cs에 미들웨어 등록
app.UseMiddleware<GlobalExceptionHandlerMiddleware>();
```

전역 예외 처리 미들웨어는 애플리케이션 어디서든 발생하는 예외를 포착하여 일관된 형식으로 로깅하고 응답한다. 에러 ID를 생성하여 클라이언트에 전달함으로써, 사용자가 문의할 때 해당 에러를 쉽게 찾을 수 있다.

## 23.6 인증 실패 로깅

보안 감사를 위해 인증 실패를 별도로 로깅한다.

```csharp
// Middlewares/AuthenticationLoggingMiddleware.cs
namespace GameAPIServer.Middlewares;

public class AuthenticationLoggingMiddleware
{
    private readonly RequestDelegate _next;
    private readonly ILogger<AuthenticationLoggingMiddleware> _logger;
    
    public AuthenticationLoggingMiddleware(
        RequestDelegate next,
        ILogger<AuthenticationLoggingMiddleware> logger)
    {
        _next = next;
        _logger = logger;
    }
    
    public async Task InvokeAsync(HttpContext context)
    {
        await _next(context);
        
        // 인증 실패 로깅
        if (context.Response.StatusCode == 401)
        {
            var token = context.Request.Headers["Authorization"].FirstOrDefault();
            var ip = context.Connection.RemoteIpAddress?.ToString();
            
            _logger.LogWarning(
                "인증 실패: Path: {Path}, IP: {IP}, Token: {Token}",
                context.Request.Path,
                ip,
                string.IsNullOrEmpty(token) ? "없음" : "***");
        }
        
        // 권한 부족 로깅
        if (context.Response.StatusCode == 403)
        {
            var userId = context.Items.TryGetValue("UserId", out var userIdObj)
                ? (long?)userIdObj
                : null;
            
            _logger.LogWarning(
                "권한 부족: Path: {Path}, UserId: {UserId}",
                context.Request.Path,
                userId);
        }
    }
}

// Program.cs에 미들웨어 등록 (인증 미들웨어 이후)
app.UseMiddleware<AuthenticationLoggingMiddleware>();
```

인증 실패 로그를 분석하면 무차별 대입 공격이나 비정상적인 접근 시도를 감지할 수 있다. 특정 IP에서 짧은 시간에 많은 인증 실패가 발생하면 자동으로 차단하는 로직을 추가할 수도 있다.

## 23.7 MySQL 슬로우 쿼리 로깅

데이터베이스 성능을 모니터링하기 위해 느린 쿼리를 로깅하는 기능을 추가한다.

```csharp
// Services/DatabaseLoggingService.cs
using System.Diagnostics;
using SqlKata.Execution;

namespace GameAPIServer.Services;

public class DatabaseLoggingService
{
    private readonly QueryFactory _queryFactory;
    private readonly ILogger<DatabaseLoggingService> _logger;
    private const int SLOW_QUERY_THRESHOLD_MS = 1000;
    
    public DatabaseLoggingService(
        QueryFactory queryFactory,
        ILogger<DatabaseLoggingService> logger)
    {
        _queryFactory = queryFactory;
        _logger = logger;
    }
    
    // 쿼리 실행 및 로깅 래퍼
    public async Task<T> ExecuteWithLoggingAsync<T>(
        Func<Task<T>> queryFunc,
        string queryDescription,
        long? userId = null)
    {
        var stopwatch = Stopwatch.StartNew();
        
        try
        {
            var result = await queryFunc();
            
            stopwatch.Stop();
            
            // 슬로우 쿼리 로깅
            if (stopwatch.ElapsedMilliseconds > SLOW_QUERY_THRESHOLD_MS)
            {
                _logger.LogWarning(
                    "슬로우 쿼리 감지: {Query}, UserId: {UserId}, Duration: {Duration}ms",
                    queryDescription,
                    userId,
                    stopwatch.ElapsedMilliseconds);
            }
            else
            {
                _logger.LogDebug(
                    "쿼리 실행: {Query}, Duration: {Duration}ms",
                    queryDescription,
                    stopwatch.ElapsedMilliseconds);
            }
            
            return result;
        }
        catch (Exception ex)
        {
            stopwatch.Stop();
            
            _logger.LogError(ex,
                "쿼리 실행 실패: {Query}, UserId: {UserId}, Duration: {Duration}ms",
                queryDescription,
                userId,
                stopwatch.ElapsedMilliseconds);
            
            throw;
        }
    }
    
    // 사용 예제
    public async Task<dynamic> GetUserDataExample(long userId)
    {
        return await ExecuteWithLoggingAsync(
            async () =>
            {
                return await _queryFactory.Query("user_account")
                    .Where("user_id", userId)
                    .FirstOrDefaultAsync();
            },
            "사용자 정보 조회",
            userId);
    }
}
```

SqlKata에 로깅을 통합하는 확장 메서드를 만들 수도 있다.

```csharp
// Extensions/QueryFactoryExtensions.cs
using System.Diagnostics;
using SqlKata;
using SqlKata.Execution;

namespace GameAPIServer.Extensions;

public static class QueryFactoryExtensions
{
    public static async Task<IEnumerable<T>> GetWithLoggingAsync<T>(
        this Query query,
        ILogger logger,
        string queryDescription)
    {
        var stopwatch = Stopwatch.StartNew();
        var sql = query.ToSql();
        
        try
        {
            var result = await query.GetAsync<T>();
            stopwatch.Stop();
            
            logger.LogInformation(
                "쿼리 실행: {Description}, SQL: {SQL}, Duration: {Duration}ms, RowCount: {RowCount}",
                queryDescription,
                sql,
                stopwatch.ElapsedMilliseconds,
                result.Count());
            
            return result;
        }
        catch (Exception ex)
        {
            stopwatch.Stop();
            
            logger.LogError(ex,
                "쿼리 실행 실패: {Description}, SQL: {SQL}, Duration: {Duration}ms",
                queryDescription,
                sql,
                stopwatch.ElapsedMilliseconds);
            
            throw;
        }
    }
}
```

이렇게 구현하면 모든 데이터베이스 쿼리의 성능을 추적할 수 있다. 슬로우 쿼리를 발견하면 인덱스를 추가하거나 쿼리를 최적화하여 성능을 개선한다.

## 23.8 Redis 성능 모니터링

Redis 작업의 성능을 모니터링하는 서비스를 구현한다.

```csharp
// Services/RedisLoggingService.cs
using CloudStructures;
using System.Diagnostics;

namespace GameAPIServer.Services;

public class RedisLoggingService
{
    private readonly ILogger<RedisLoggingService> _logger;
    private const int SLOW_REDIS_THRESHOLD_MS = 100;
    
    public RedisLoggingService(ILogger<RedisLoggingService> logger)
    {
        _logger = logger;
    }
    
    // Redis 작업 실행 및 로깅 래퍼
    public async Task<T> ExecuteWithLoggingAsync<T>(
        Func<Task<T>> redisFunc,
        string operation,
        string key,
        long? userId = null)
    {
        var stopwatch = Stopwatch.StartNew();
        
        try
        {
            var result = await redisFunc();
            
            stopwatch.Stop();
            
            // 느린 작업 로깅
            if (stopwatch.ElapsedMilliseconds > SLOW_REDIS_THRESHOLD_MS)
            {
                _logger.LogWarning(
                    "느린 Redis 작업: Operation: {Operation}, Key: {Key}, UserId: {UserId}, Duration: {Duration}ms",
                    operation,
                    key,
                    userId,
                    stopwatch.ElapsedMilliseconds);
            }
            else
            {
                _logger.LogDebug(
                    "Redis 작업: Operation: {Operation}, Key: {Key}, Duration: {Duration}ms",
                    operation,
                    key,
                    stopwatch.ElapsedMilliseconds);
            }
            
            return result;
        }
        catch (Exception ex)
        {
            stopwatch.Stop();
            
            _logger.LogError(ex,
                "Redis 작업 실패: Operation: {Operation}, Key: {Key}, UserId: {UserId}, Duration: {Duration}ms",
                operation,
                key,
                userId,
                stopwatch.ElapsedMilliseconds);
            
            throw;
        }
    }
    
    // 사용 예제
    public async Task<string?> GetStringExample(RedisConnection redis, string key, long userId)
    {
        return await ExecuteWithLoggingAsync(
            async () =>
            {
                var redisString = new CloudStructures.Structures.RedisString<string>(redis, key, null);
                var result = await redisString.GetAsync();
                return result.Value;
            },
            "GET",
            key,
            userId);
    }
}
```

```python
# analyze_logs.py
import re
from collections import defaultdict
from datetime import datetime

def analyze_api_performance(log_file):
    """API 성능 분석"""
    api_stats = defaultdict(lambda: {'count': 0, 'total_time': 0})
    
    with open(log_file, 'r', encoding='utf-8') as f:
        for line in f:
            # API 응답 로그 파싱
            match = re.search(r'API 응답: (\w+) ([^\s,]+).*Duration: ([\d.]+)ms', line)
            if match:
                method, path, duration = match.groups()
                key = f"{method} {path}"
                api_stats[key]['count'] += 1
                api_stats[key]['total_time'] += float(duration)
    
    # 평균 응답 시간 계산
    for key in api_stats:
        avg_time = api_stats[key]['total_time'] / api_stats[key]['count']
        print(f"{key}: {api_stats[key]['count']}회 호출, 평균 {avg_time:.2f}ms")

def analyze_user_activity(log_file):
    """사용자 활동 분석"""
    user_actions = defaultdict(int)
    
    with open(log_file, 'r', encoding='utf-8') as f:
        for line in f:
            match = re.search(r'UserId: (\d+)', line)
            if match:
                user_id = match.group(1)
                user_actions[user_id] += 1
    
    # 활동량 상위 사용자
    sorted_users = sorted(user_actions.items(), key=lambda x: x[1], reverse=True)
    print("\n활동량 상위 10명:")
    for user_id, count in sorted_users[:10]:
        print(f"UserId: {user_id}, 활동: {count}회")
```

## 23.11 로그 보관 및 관리

로그 파일 관리 전략을 수립한다.

**로그 압축**

오래된 로그 파일을 압축하여 디스크 공간을 절약한다.

```csharp
// Services/LogMaintenanceService.cs
using System.IO.Compression;

namespace GameAPIServer.Services;

public class LogMaintenanceService : BackgroundService
{
    private readonly ILogger<LogMaintenanceService> _logger;
    private const string LOG_DIRECTORY = "logs";
    private const int COMPRESS_AFTER_DAYS = 7;
    private const int DELETE_AFTER_DAYS = 90;
    
    public LogMaintenanceService(ILogger<LogMaintenanceService> logger)
    {
        _logger = logger;
    }
    
    protected override async Task ExecuteAsync(CancellationToken stoppingToken)
    {
        while (!stoppingToken.IsCancellationRequested)
        {
            try
            {
                await CompressOldLogsAsync();
                await DeleteOldLogsAsync();
            }
            catch (Exception ex)
            {
                _logger.LogError(ex, "로그 관리 작업 실패");
            }
            
            // 매일 새벽 3시에 실행
            await Task.Delay(TimeSpan.FromHours(24), stoppingToken);
        }
    }
    
    private async Task CompressOldLogsAsync()
    {
        var logDir = new DirectoryInfo(LOG_DIRECTORY);
        if (!logDir.Exists) return;
        
        var filesToCompress = logDir.GetFiles("*.txt")
            .Where(f => f.LastWriteTime < DateTime.Now.AddDays(-COMPRESS_AFTER_DAYS))
            .Where(f => !f.Name.EndsWith(".gz"));
        
        foreach (var file in filesToCompress)
        {
            var compressedPath = file.FullName + ".gz";
            
            await using var input = file.OpenRead();
            await using var output = File.Create(compressedPath);
            await using var gzip = new GZipStream(output, CompressionMode.Compress);
            await input.CopyToAsync(gzip);
            
            file.Delete();
            
            _logger.LogInformation("로그 파일 압축 완료: {FileName}", file.Name);
        }
    }
    
    private Task DeleteOldLogsAsync()
    {
        var logDir = new DirectoryInfo(LOG_DIRECTORY);
        if (!logDir.Exists) return Task.CompletedTask;
        
        var filesToDelete = logDir.GetFiles()
            .Where(f => f.LastWriteTime < DateTime.Now.AddDays(-DELETE_AFTER_DAYS));
        
        foreach (var file in filesToDelete)
        {
            file.Delete();
            _logger.LogInformation("오래된 로그 파일 삭제: {FileName}", file.Name);
        }
        
        return Task.CompletedTask;
    }
}

// Program.cs에 등록
builder.Services.AddHostedService<LogMaintenanceService>();
```

```csharp
                Message = "에러 로그 조회 실패"
            };
        }
    }
```
