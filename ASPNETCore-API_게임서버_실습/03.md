# 3장. 첫 번째 API 만들기 (실습)
이 장에서는 게임 서버의 첫 번째 실제 API인 회원가입 API를 구현한다. 아직 데이터베이스 연동은 하지 않고, 메모리에 데이터를 저장하는 방식으로 구현하여 API 개발의 기본 흐름을 익힌다. 비밀번호 해싱, 요청 데이터 검증, 에러 처리 등 실무에서 반드시 필요한 기술들을 실습한다.

## 3.1 회원가입 API 설계
회원가입 API를 구현하기 전에 먼저 API의 스펙을 명확히 정의한다.

### 3.1.1 API 명세

**엔드포인트**: `POST /api/account/register`

**요청 데이터**:
```json
{
  "email": "player@example.com",
  "password": "SecurePass123!",
  "nickname": "DragonSlayer"
}
```

**성공 응답** (HTTP 200):
```json
{
  "result": 0,
  "message": "Success",
  "data": {
    "userId": "user_123456",
    "email": "player@example.com",
    "nickname": "DragonSlayer",
    "createdAt": "2025-11-22T10:30:00"
  }
}
```

**실패 응답** (HTTP 400):
```json
{
  "result": 4001,
  "message": "Email already exists",
  "data": null
}
```

### 3.1.2 비즈니스 규칙
회원가입 API는 다음 규칙을 따른다.

- 이메일은 유효한 형식이어야 한다
- 이메일은 중복될 수 없다
- 비밀번호는 최소 8자 이상이어야 한다
- 비밀번호는 영문, 숫자, 특수문자를 포함해야 한다
- 닉네임은 2자 이상 20자 이하여야 한다
- 닉네임은 중복될 수 없다
- 비밀번호는 암호화하여 저장한다
   

## 3.2 모델 클래스 작성
요청과 응답에 사용할 모델 클래스를 작성한다.

### 3.2.1 요청 모델 작성
`Models` 폴더에 `AccountModels.cs` 파일을 생성하고 다음 코드를 작성한다.

using System.ComponentModel.DataAnnotations;

namespace GameServer.Models
{
    // 회원가입 요청
    public class RegisterRequest
    {
        [Required(ErrorMessage = "Email is required")]
        [EmailAddress(ErrorMessage = "Invalid email format")]

        [Required(ErrorMessage = "Password is required")]
        [MinLength(8, ErrorMessage = "Password must be at least 8 characters")]
        public string Password { get; set; } = string.Empty;

    {

        public string Nickname { get; set; } = string.Empty;
        public DateTime CreatedAt { get; set; }
    }
}

**Required**: 필수 입력 항목을 지정한다.

**EmailAddress**: 이메일 형식을 검증한다.

**MinLength**: 최소 문자 길이를 검증한다.

## 3.3 비밀번호 해싱
비밀번호는 절대 평문으로 저장해서는 안 된다. 해시 함수를 사용하여 암호화한 후 저장한다.

### 3.3.1 해싱 알고리즘 선택
비밀번호 해싱에는 주로 다음 알고리즘을 사용한다.

**SHA256**: 빠르고 간단하지만, 레인보우 테이블 공격에 취약하다. 솔트(Salt)를 함께 사용해야 한다.

**BCrypt**: 비밀번호 해싱에 특화된 알고리즘이다. 자동으로 솔트를 생성하고, 계산 비용을 조정할 수 있어 브루트포스 공격에 강하다.

**PBKDF2**: .NET에서 기본 제공하는 알고리즘이다. 반복 횟수를 설정하여 보안성을 높일 수 있다.

이 교육에서는 SHA256과 BCrypt를 모두 구현하여 비교한다.

### 3.3.2 SHA256 해싱 구현
SHA256을 사용한 간단한 해싱 유틸리티를 작성한다. `Utils` 폴더를 생성하고 `PasswordHasher.cs` 파일을 추가한다.

```csharp
using System.Security.Cryptography;
using System.Text;

namespace GameServer.Utils
{
    public static class PasswordHasher
    {
        // SHA256을 사용한 해싱
        public static string HashWithSHA256(string password, string salt)
        {
            using var sha256 = SHA256.Create();
            var saltedPassword = password + salt;
            var bytes = Encoding.UTF8.GetBytes(saltedPassword);
            var hash = sha256.ComputeHash(bytes);
            return Convert.ToBase64String(hash);
        }

        // 솔트 생성
        public static string GenerateSalt()
        {
            var bytes = new byte[32];
            using var rng = RandomNumberGenerator.Create();
            rng.GetBytes(bytes);
            return Convert.ToBase64String(bytes);
        }

        // 비밀번호 검증
        public static bool VerifySHA256(string password, string salt, string hashedPassword)
        {
            var hash = HashWithSHA256(password, salt);
            return hash == hashedPassword;
        }
    }
}
```

**코드 설명**:

**HashWithSHA256()**: 비밀번호와 솔트를 결합하여 SHA256 해시를 생성한다.

**GenerateSalt()**: 암호학적으로 안전한 난수 생성기를 사용하여 솔트를 생성한다.

**VerifySHA256()**: 입력된 비밀번호가 저장된 해시와 일치하는지 확인한다.

### 3.3.3 BCrypt 해싱 구현
BCrypt를 사용하려면 NuGet 패키지를 설치해야 한다.

1. Visual Studio의 '도구' 메뉴에서 'NuGet 패키지 관리자' > '솔루션용 NuGet 패키지 관리'를 선택한다.
2. '찾아보기' 탭에서 'BCrypt.Net-Next'를 검색한다.
3. 프로젝트를 선택하고 '설치' 버튼을 클릭한다.

`PasswordHasher.cs`에 BCrypt 관련 메서드를 추가한다.

```csharp
using System.Security.Cryptography;
using System.Text;
using BCrypt.Net;

namespace GameServer.Utils
{
    public static class PasswordHasher
    {
        // SHA256을 사용한 해싱
        public static string HashWithSHA256(string password, string salt)
        {
            using var sha256 = SHA256.Create();
            var saltedPassword = password + salt;
            var bytes = Encoding.UTF8.GetBytes(saltedPassword);
            var hash = sha256.ComputeHash(bytes);
            return Convert.ToBase64String(hash);
        }

        // 솔트 생성
        public static string GenerateSalt()
        {
            var bytes = new byte[32];
            using var rng = RandomNumberGenerator.Create();
            rng.GetBytes(bytes);
            return Convert.ToBase64String(bytes);
        }

        // 비밀번호 검증 (SHA256)
        public static bool VerifySHA256(string password, string salt, string hashedPassword)
        {
            var hash = HashWithSHA256(password, salt);
            return hash == hashedPassword;
        }

        // BCrypt를 사용한 해싱
        public static string HashWithBCrypt(string password)
        {
            return BCrypt.Net.BCrypt.HashPassword(password, workFactor: 12);
        }

        // 비밀번호 검증 (BCrypt)
        public static bool VerifyBCrypt(string password, string hashedPassword)
        {
            return BCrypt.Net.BCrypt.Verify(password, hashedPassword);
        }
    }
}
```

**BCrypt 장점**:

- 솔트가 해시에 자동으로 포함되어 별도 관리가 불필요하다
- workFactor를 조정하여 해싱 시간을 늘려 브루트포스 공격을 어렵게 만든다
- 검증이 간단하다

이 교육에서는 BCrypt를 사용한다.

### 3.3.4 비밀번호 강도 검증
비밀번호가 충분히 강력한지 검증하는 메서드를 추가한다. `Utils` 폴더에 `PasswordValidator.cs` 파일을 생성한다.

```csharp
using System.Text.RegularExpressions;

namespace GameServer.Utils
{
    public static class PasswordValidator
    {
        // 비밀번호 강도 검증
        public static bool IsStrongPassword(string password)
        {
            if (string.IsNullOrEmpty(password) || password.Length < 8)
            {
                return false;
            }

            // 영문 대문자 포함 여부
            bool hasUpperCase = Regex.IsMatch(password, @"[A-Z]");
            
            // 영문 소문자 포함 여부
            bool hasLowerCase = Regex.IsMatch(password, @"[a-z]");
            
            // 숫자 포함 여부
            bool hasDigit = Regex.IsMatch(password, @"[0-9]");
            
            // 특수문자 포함 여부
            bool hasSpecialChar = Regex.IsMatch(password, @"[!@#$%^&*(),.?""':{}|<>]");

            // 최소 3가지 조건 만족
            int conditionsMet = 0;
            if (hasUpperCase) conditionsMet++;
            if (hasLowerCase) conditionsMet++;
            if (hasDigit) conditionsMet++;
            if (hasSpecialChar) conditionsMet++;

            return conditionsMet >= 3;
        }

        // 비밀번호 강도 검증 및 상세 메시지 반환
        public static (bool isValid, string message) ValidatePassword(string password)
        {
            if (string.IsNullOrEmpty(password))
            {
                return (false, "Password is required");
            }

            if (password.Length < 8)
            {
                return (false, "Password must be at least 8 characters");
            }

            if (password.Length > 100)
            {
                return (false, "Password is too long");
            }

            if (!IsStrongPassword(password))
            {
                return (false, "Password must contain at least 3 of: uppercase, lowercase, digit, special character");
            }

            return (true, "Valid password");
        }
    }
}
```
  

## 3.4 메모리 기반 사용자 저장소 구현
데이터베이스 연동 전에 메모리에 사용자 데이터를 저장하는 간단한 저장소를 구현한다.

### 3.4.1 인메모리 저장소 작성
`Services` 폴더를 생성하고 `InMemoryUserService.cs` 파일을 추가한다.

```csharp
using GameServer.Models;
using System.Collections.Concurrent;

namespace GameServer.Services
{
    public class InMemoryUserService
    {
        // 스레드 안전한 컬렉션 사용
        private readonly ConcurrentDictionary<string, UserEntity> _usersByEmail = new();
        private readonly ConcurrentDictionary<string, UserEntity> _usersByNickname = new();
        private readonly ConcurrentDictionary<string, UserEntity> _usersById = new();
        private int _nextUserId = 1;

        // 이메일로 사용자 존재 여부 확인
        public bool ExistsByEmail(string email)
        {
            return _usersByEmail.ContainsKey(email.ToLower());
        }

        // 닉네임으로 사용자 존재 여부 확인
        public bool ExistsByNickname(string nickname)
        {
            return _usersByNickname.ContainsKey(nickname.ToLower());
        }

        // 사용자 생성
        public UserEntity CreateUser(string email, string hashedPassword, string nickname)
        {
            var userId = $"user_{_nextUserId++}";
            var user = new UserEntity
            {
                UserId = userId,
                Email = email.ToLower(),
                HashedPassword = hashedPassword,
                Nickname = nickname,
                CreatedAt = DateTime.Now,
                LastLoginAt = DateTime.Now
            };

            _usersById[userId] = user;
            _usersByEmail[email.ToLower()] = user;
            _usersByNickname[nickname.ToLower()] = user;

            return user;
        }

        // 이메일로 사용자 조회
        public UserEntity? GetUserByEmail(string email)
        {
            _usersByEmail.TryGetValue(email.ToLower(), out var user);
            return user;
        }

        // UserId로 사용자 조회
        public UserEntity? GetUserById(string userId)
        {
            _usersById.TryGetValue(userId, out var user);
            return user;
        }

        // 전체 사용자 수 조회 (테스트용)
        public int GetUserCount()
        {
            return _usersById.Count;
        }
    }
}
```

**코드 설명**:

**ConcurrentDictionary**: 멀티스레드 환경에서 안전한 딕셔너리다. 여러 요청이 동시에 들어와도 데이터 무결성이 보장된다.

**세 개의 인덱스**: 이메일, 닉네임, UserId로 빠르게 검색할 수 있도록 세 개의 딕셔너리를 유지한다.

**ToLower()**: 대소문자 구분 없이 검색하기 위해 소문자로 변환하여 저장한다.

## 3.5 회원가입 API 구현
이제 모든 준비가 완료되었으므로 실제 회원가입 API를 구현한다.

### 3.5.1 AccountController 수정
`Controllers` 폴더의 `AccountController.cs`를 다음과 같이 수정한다.

```csharp
using GameServer.Models;
using GameServer.Services;
using GameServer.Utils;
using Microsoft.AspNetCore.Mvc;

namespace GameServer.Controllers
{
    [Route("api/account")]
    [ApiController]
    public class AccountController : ControllerBase
    {
        private readonly InMemoryUserService _userService;
        private readonly ILogger<AccountController> _logger;

        // 의존성 주입
        public AccountController(InMemoryUserService userService, ILogger<AccountController> logger)
        {
            _userService = userService;
            _logger = logger;
        }

        [HttpPost("register")]
        public RegisterResponse Register([FromBody] RegisterRequest request)
        {
            // 1. 요청 데이터 검증
            if (!ModelState.IsValid)
            {
                var errors = ModelState.Values
                    .SelectMany(v => v.Errors)
                    .Select(e => e.ErrorMessage)
                    .ToList();
                
                _logger.LogWarning("Register validation failed: {Errors}", string.Join(", ", errors));
                
                return new RegisterResponse
                {
                    Result = ErrorCode.InvalidRequest,
                    Message = string.Join(", ", errors)
                };
            }

            // 2. 비밀번호 강도 검증
            var (isValidPassword, passwordMessage) = PasswordValidator.ValidatePassword(request.Password);
            if (!isValidPassword)
            {
                _logger.LogWarning("Weak password attempt for email: {Email}", request.Email);
                
                return new RegisterResponse
                {
                    Result = ErrorCode.InvalidPassword,
                    Message = passwordMessage
                };
            }

            // 3. 이메일 중복 확인
            if (_userService.ExistsByEmail(request.Email))
            {
                _logger.LogWarning("Email already exists: {Email}", request.Email);
                
                return new RegisterResponse
                {
                    Result = ErrorCode.EmailAlreadyExists,
                    Message = "Email already exists"
                };
            }

            // 4. 닉네임 중복 확인
            if (_userService.ExistsByNickname(request.Nickname))
            {
                _logger.LogWarning("Nickname already exists: {Nickname}", request.Nickname);
                
                return new RegisterResponse
                {
                    Result = ErrorCode.NicknameAlreadyExists,
                    Message = "Nickname already exists"
                };
            }

            // 5. 비밀번호 해싱
            var hashedPassword = PasswordHasher.HashWithBCrypt(request.Password);

            // 6. 사용자 생성
            var user = _userService.CreateUser(request.Email, hashedPassword, request.Nickname);

            // 7. 응답 데이터 생성
            var responseData = new RegisterResponseData
            {
                UserId = user.UserId,
                Email = user.Email,
                Nickname = user.Nickname,
                CreatedAt = user.CreatedAt
            };

            _logger.LogInformation("User registered successfully: {UserId}, {Email}", user.UserId, user.Email);

            return new RegisterResponse
            {
                Result = ErrorCode.None,
                Message = "Registration successful",
                Data = responseData
            };
        }
    }

    public class RegisterResponse
    {
        public ErrorCode Result { get; set; }
        public string Message { get; set; } = string.Empty;
        public RegisterResponseData? Data { get; set; }
    }
}
```

**코드 설명**:

**의존성 주입**: 생성자를 통해 `InMemoryUserService`와 `ILogger`를 주입받는다.

**ModelState.IsValid**: Data Annotations로 정의한 유효성 검증을 자동으로 수행한다.

**단계별 검증**: 요청 데이터, 비밀번호 강도, 중복 확인을 순차적으로 검증한다.

**로깅**: 각 단계에서 적절한 로그를 남긴다.

**일관된 응답**: `ApiResponse` 클래스를 사용하여 표준화된 응답을 반환한다.

## 3.7 요청 데이터 검증 고급 기법
더 복잡한 유효성 검증이 필요한 경우 커스텀 검증 로직을 추가할 수 있다.

### 3.7.1 커스텀 Validation Attribute 작성
이메일 도메인을 제한하는 커스텀 검증 특성을 만들어본다. `Utils` 폴더에 `CustomValidations.cs` 파일을 생성한다.

```csharp
using System.ComponentModel.DataAnnotations;

namespace GameServer.Utils
{
    // 허용된 이메일 도메인만 검증
    public class AllowedEmailDomainsAttribute : ValidationAttribute
    {
        private readonly string[] _allowedDomains;

        public AllowedEmailDomainsAttribute(params string[] allowedDomains)
        {
            _allowedDomains = allowedDomains;
        }

        protected override ValidationResult? IsValid(object? value, ValidationContext validationContext)
        {
            if (value is string email)
            {
                var domain = email.Split('@').LastOrDefault();
                if (domain != null && _allowedDomains.Contains(domain.ToLower()))
                {
                    return ValidationResult.Success;
                }
            }

            return new ValidationResult($"Email domain must be one of: {string.Join(", ", _allowedDomains)}");
        }
    }

    // 금지된 단어 검증
    public class ProhibitedWordsAttribute : ValidationAttribute
    {
        private readonly string[] _prohibitedWords;

        public ProhibitedWordsAttribute(params string[] prohibitedWords)
        {
            _prohibitedWords = prohibitedWords.Select(w => w.ToLower()).ToArray();
        }

        protected override ValidationResult? IsValid(object? value, ValidationContext validationContext)
        {
            if (value is string text)
            {
                var lowerText = text.ToLower();
                foreach (var word in _prohibitedWords)
                {
                    if (lowerText.Contains(word))
                    {
                        return new ValidationResult($"Text contains prohibited word: {word}");
                    }
                }
            }

            return ValidationResult.Success;
        }
    }
}
```

## 3.8 응답 코드 및 에러 처리
HTTP 상태 코드를 적절하게 사용하여 클라이언트가 요청 결과를 쉽게 파악할 수 있도록 한다.

### 3.8.1 HTTP 상태 코드 가이드
게임 서버에서 주로 사용하는 상태 코드는 다음과 같다.

- **200 OK**: 요청이 성공적으로 처리됨
- **400 Bad Request**: 잘못된 요청 (유효성 검증 실패, 중복 데이터 등)
- **401 Unauthorized**: 인증되지 않음 (토큰 없음, 만료됨)
- **403 Forbidden**: 권한 없음 (다른 사용자의 리소스 접근 시도)
- **404 Not Found**: 리소스를 찾을 수 없음
- **500 Internal Server Error**: 서버 내부 에러

### 3.8.2 예외 처리 추가
예상치 못한 에러를 처리하기 위해 try-catch 블록을 추가한다.

```csharp
[HttpPost("register")]
public RegisterResponse Register([FromBody] RegisterRequest request)
{
    try
    {
        // 1. 요청 데이터 검증
        if (!ModelState.IsValid)
        {
            var errors = ModelState.Values
                .SelectMany(v => v.Errors)
                .Select(e => e.ErrorMessage)
                .ToList();
            
            _logger.LogWarning("Register validation failed: {Errors}", string.Join(", ", errors));
            
            return new RegisterResponse
            {
                Result = ErrorCode.InvalidRequest,
                Message = string.Join(", ", errors)
            };
        }

        // 2. 비밀번호 강도 검증
        var (isValidPassword, passwordMessage) = PasswordValidator.ValidatePassword(request.Password);
        if (!isValidPassword)
        {
            _logger.LogWarning("Weak password attempt for email: {Email}", request.Email);
            
            return new RegisterResponse
            {
                Result = ErrorCode.InvalidPassword,
                Message = passwordMessage
            };
        }

        // 3. 이메일 중복 확인
        if (_userService.ExistsByEmail(request.Email))
        {
            _logger.LogWarning("Email already exists: {Email}", request.Email);
            
            return new RegisterResponse
            {
                Result = ErrorCode.EmailAlreadyExists,
                Message = "Email already exists"
            };
        }

        // 4. 닉네임 중복 확인
        if (_userService.ExistsByNickname(request.Nickname))
        {
            _logger.LogWarning("Nickname already exists: {Nickname}", request.Nickname);
            
            return new RegisterResponse
            {
                Result = ErrorCode.NicknameAlreadyExists,
                Message = "Nickname already exists"
            };
        }

        // 5. 비밀번호 해싱
        var hashedPassword = PasswordHasher.HashWithBCrypt(request.Password);

        // 6. 사용자 생성
        var user = _userService.CreateUser(request.Email, hashedPassword, request.Nickname);

        // 7. 응답 데이터 생성
        var responseData = new RegisterResponseData
        {
            UserId = user.UserId,
            Email = user.Email,
            Nickname = user.Nickname,
            CreatedAt = user.CreatedAt
        };

        _logger.LogInformation("User registered successfully: {UserId}, {Email}", user.UserId, user.Email);

        return new RegisterResponse
        {
            Result = ErrorCode.None,
            Message = "Registration successful",
            Data = responseData
        };
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, "Unexpected error during registration for email: {Email}", request.Email);
        
        return new RegisterResponse
        {
            Result = ErrorCode.InternalServerError,
            Message = "An unexpected error occurred"
        };
    }
}
```

### 3.8.3 전역 예외 처리 미들웨어
모든 컨트롤러에서 반복적으로 try-catch를 작성하는 대신, 전역 예외 처리 미들웨어를 구현할 수 있다. `Middlewares` 폴더에 `GlobalExceptionMiddleware.cs` 파일을 생성한다.

```csharp
using GameServer.Models;
using System.Net;
using System.Text.Json;

namespace GameServer.Middlewares
{
    public class GlobalExceptionMiddleware
    {
        private readonly RequestDelegate _next;
        private readonly ILogger<GlobalExceptionMiddleware> _logger;

        public GlobalExceptionMiddleware(RequestDelegate next, ILogger<GlobalExceptionMiddleware> logger)
        {
            _next = next;
            _logger = logger;
        }

        public async Task InvokeAsync(HttpContext context)
        {
            try
            {
                await _next(context);
            }
            catch (Exception ex)
            {
                _logger.LogError(ex, "Unhandled exception occurred");
                await HandleExceptionAsync(context, ex);
            }
        }

        private static Task HandleExceptionAsync(HttpContext context, Exception exception)
        {
            context.Response.ContentType = "application/json";
            context.Response.StatusCode = (int)HttpStatusCode.InternalServerError;

            var response = ApiResponse<object>.Error(
                ErrorCode.InternalServerError,
                "An unexpected error occurred"
            );

            var jsonOptions = new JsonSerializerOptions
            {
                PropertyNamingPolicy = JsonNamingPolicy.CamelCase
            };

            return context.Response.WriteAsync(JsonSerializer.Serialize(response, jsonOptions));
        }
    }
}
```

`Program.cs`에서 미들웨어를 등록한다.

```csharp
using GameServer.Middlewares;
using GameServer.Services;
using System.Text.Json;
using System.Text.Json.Serialization;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddControllers()
    .AddJsonOptions(options =>
    {
        options.JsonSerializerOptions.PropertyNamingPolicy = JsonNamingPolicy.CamelCase;
        options.JsonSerializerOptions.DefaultIgnoreCondition = JsonIgnoreCondition.WhenWritingNull;
        options.JsonSerializerOptions.NumberHandling = JsonNumberHandling.AllowReadingFromString;
    });

builder.Services.AddSingleton<InMemoryUserService>();

var app = builder.Build();

// 전역 예외 처리 미들웨어 추가
app.UseMiddleware<GlobalExceptionMiddleware>();

app.UseAuthorization();
app.MapControllers();

app.Run();
```

이제 컨트롤러에서 try-catch를 제거하고 간결한 코드로 작성할 수 있다. 예상치 못한 예외는 자동으로 미들웨어에서 처리된다.

## 3.10 추가 실습 과제
회원가입 API를 더 발전시키기 위해 다음 기능을 추가로 구현해본다.

**이메일 인증**: 실제 서비스에서는 이메일 인증 과정이 필요하다. 인증 코드를 생성하고 검증하는 기능을 추가한다.

**사용자 통계**: 현재 등록된 사용자 수를 조회하는 API를 추가한다.

**닉네임 제안**: 중복된 닉네임 입력 시 사용 가능한 대체 닉네임을 제안하는 기능을 추가한다.

**Rate Limiting**: 같은 IP에서 너무 많은 회원가입 시도를 막는 기능을 추가한다.

이러한 기능들은 나중에 Redis를 학습한 후 구현할 수 있다.
