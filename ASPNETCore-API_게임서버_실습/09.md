이번 장에서는 8장에서 설계한 커스텀 토큰 기반 인증을 실제로 구현한다. 로그인 API를 만들고, 비밀번호를 검증하며, 토큰을 생성하여 MySQL과 Redis에 저장하는 전체 과정을 실습한다.

## 9.1 사전 준비

### 9.1.1 필요한 테이블 생성

로그인 시스템을 구현하기 위해 두 개의 테이블이 필요하다.

**users 테이블**

사용자 계정 정보를 저장하는 테이블이다.

```sql
CREATE TABLE users (
    user_id BIGINT AUTO_INCREMENT PRIMARY KEY,
    email VARCHAR(100) NOT NULL UNIQUE,
    password VARCHAR(100) NOT NULL,
    nickname VARCHAR(50) NOT NULL,
    created_at BIGINT NOT NULL,
    updated_at BIGINT NOT NULL,
    INDEX idx_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**user_auth_token 테이블**

인증 토큰 정보를 저장하는 테이블이다.

```sql
CREATE TABLE user_auth_token (
    user_id BIGINT NOT NULL,
    token VARCHAR(500) NOT NULL,
    created_at BIGINT NOT NULL,
    expired_at BIGINT NOT NULL,
    PRIMARY KEY (user_id),
    INDEX idx_token (token),
    INDEX idx_expired_at (expired_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 9.1.2 테스트 계정 생성

실습을 위해 테스트 계정을 미리 생성한다. 비밀번호는 간단히 평문으로 저장하지만, 실제 서비스에서는 반드시 해시 처리해야 한다.

```sql
INSERT INTO users (email, password, nickname, created_at, updated_at)
VALUES ('test@example.com', 'password123', 'TestUser', UNIX_TIMESTAMP(), UNIX_TIMESTAMP());
```

실제 프로젝트에서는 회원가입 API에서 bcrypt 등을 사용하여 비밀번호를 해시 처리한다. 이 교육에서는 인증 시스템 구현에 집중하기 위해 평문 비밀번호를 사용한다.

### 9.2.2 TokenGenerator 클래스 구현

8장에서 설계한 토큰 생성 알고리즘을 구현한다.

`Security/TokenGenerator.cs` 파일을 생성한다.

```csharp
using System.Security.Cryptography;
using System.Text;

namespace GameServer.Security;

public class TokenGenerator : ITokenGenerator
{
    private readonly string _secretKey;
    private readonly int _tokenExpirationHours;

    public TokenGenerator(IConfiguration configuration)
    {
        _secretKey = configuration["Security:SecretKey"] 
            ?? throw new InvalidOperationException("SecretKey is not configured");
        _tokenExpirationHours = configuration.GetValue<int>("Security:TokenExpirationHours", 24);
    }

    public string GenerateToken(long userId)
    {
        // 1. Salt 생성 (16바이트 랜덤 값)
        var salt = GenerateSalt();
        
        // 2. Timestamp 생성 (Unix timestamp)
        var timestamp = DateTimeOffset.UtcNow.ToUnixTimeSeconds();
        
        // 3. 토큰 데이터 조합
        var tokenData = $"{userId}:{timestamp}:{salt}";
        
        // 4. HMAC-SHA256 해시 생성
        var hash = ComputeHmacSha256(tokenData, _secretKey);
        
        // 5. 최종 토큰 생성 (Base64 인코딩)
        var token = $"{userId}:{timestamp}:{salt}:{hash}";
        return Convert.ToBase64String(Encoding.UTF8.GetBytes(token));
    }

    public (bool isValid, long userId) ValidateToken(string token)
    {
        try
        {
            // 1. Base64 디코딩
            var decodedBytes = Convert.FromBase64String(token);
            var decodedToken = Encoding.UTF8.GetString(decodedBytes);
            
            // 2. 토큰 분리
            var parts = decodedToken.Split(':');
            if (parts.Length != 4)
            {
                return (false, 0);
            }
            
            var userId = long.Parse(parts[0]);
            var timestamp = long.Parse(parts[1]);
            var salt = parts[2];
            var hash = parts[3];
            
            // 3. 만료 시간 확인
            var tokenCreatedAt = DateTimeOffset.FromUnixTimeSeconds(timestamp);
            var expirationTime = tokenCreatedAt.AddHours(_tokenExpirationHours);
            if (DateTimeOffset.UtcNow > expirationTime)
            {
                return (false, 0);
            }
            
            // 4. Hash 재계산
            var tokenData = $"{userId}:{timestamp}:{salt}";
            var calculatedHash = ComputeHmacSha256(tokenData, _secretKey);
            
            // 5. Hash 비교
            if (hash != calculatedHash)
            {
                return (false, 0);
            }
            
            return (true, userId);
        }
        catch
        {
            return (false, 0);
        }
    }

    private string GenerateSalt()
    {
        var saltBytes = new byte[16];
        using (var rng = RandomNumberGenerator.Create())
        {
            rng.GetBytes(saltBytes);
        }
        return Convert.ToBase64String(saltBytes);
    }

    private string ComputeHmacSha256(string data, string key)
    {
        var keyBytes = Encoding.UTF8.GetBytes(key);
        var dataBytes = Encoding.UTF8.GetBytes(data);
        
        using (var hmac = new HMACSHA256(keyBytes))
        {
            var hashBytes = hmac.ComputeHash(dataBytes);
            return Convert.ToBase64String(hashBytes);
        }
    }
}
```

`Repositories/UserRepository.cs` 파일을 생성한다.

```csharp
using SqlKata.Execution;

namespace GameServer.Repositories;

public class UserRepository : IUserRepository
{
    private readonly QueryFactory _queryFactory;

    public UserRepository(QueryFactory queryFactory)
    {
        _queryFactory = queryFactory;
    }

    public async Task<(bool exists, long userId, string password)> GetUserByEmailAsync(string email)
    {
        var user = await _queryFactory.Query("users")
            .Select("user_id", "password")
            .Where("email", email)
            .FirstOrDefaultAsync();

        if (user == null)
        {
            return (false, 0, string.Empty);
        }

        return (true, (long)user.user_id, (string)user.password);
    }
}
```

### 9.3.2 AuthTokenRepository 구현

토큰 정보를 저장하고 조회하는 Repository를 구현한다.

`Repositories/IAuthTokenRepository.cs` 파일을 생성한다.

```csharp
namespace GameServer.Repositories;

public interface IAuthTokenRepository
{
    Task<bool> SaveTokenAsync(long userId, string token, long createdAt, long expiredAt);
    Task<string?> GetTokenByUserIdAsync(long userId);
    Task<bool> DeleteTokenAsync(long userId);
}
```

`Repositories/AuthTokenRepository.cs` 파일을 생성한다.

```csharp
using SqlKata.Execution;

namespace GameServer.Repositories;

public class AuthTokenRepository : IAuthTokenRepository
{
    private readonly QueryFactory _queryFactory;

    public AuthTokenRepository(QueryFactory queryFactory)
    {
        _queryFactory = queryFactory;
    }

    public async Task<bool> SaveTokenAsync(long userId, string token, long createdAt, long expiredAt)
    {
        try
        {
            // UPSERT 수행 (기존 토큰이 있으면 업데이트, 없으면 삽입)
            var exists = await _queryFactory.Query("user_auth_token")
                .Where("user_id", userId)
                .ExistsAsync();

            if (exists)
            {
                await _queryFactory.Query("user_auth_token")
                    .Where("user_id", userId)
                    .UpdateAsync(new
                    {
                        token,
                        created_at = createdAt,
                        expired_at = expiredAt
                    });
            }
            else
            {
                await _queryFactory.Query("user_auth_token")
                    .InsertAsync(new
                    {
                        user_id = userId,
                        token,
                        created_at = createdAt,
                        expired_at = expiredAt
                    });
            }

            return true;
        }
        catch
        {
            return false;
        }
    }

    public async Task<string?> GetTokenByUserIdAsync(long userId)
    {
        var result = await _queryFactory.Query("user_auth_token")
            .Select("token")
            .Where("user_id", userId)
            .FirstOrDefaultAsync();

        return result?.token;
    }

    public async Task<bool> DeleteTokenAsync(long userId)
    {
        try
        {
            await _queryFactory.Query("user_auth_token")
                .Where("user_id", userId)
                .DeleteAsync();
            return true;
        }
        catch
        {
            return false;
        }
    }
}
```

### 9.5.2 AuthService 클래스 구현

로그인 로직을 구현한다.

`Services/AuthService.cs` 파일을 생성한다.

```csharp
using CloudStructures;
using CloudStructures.Structures;
using GameServer.Repositories;
using GameServer.Security;

namespace GameServer.Services;

public class AuthService : IAuthService
{
    private readonly IUserRepository _userRepository;
    private readonly IAuthTokenRepository _authTokenRepository;
    private readonly ITokenGenerator _tokenGenerator;
    private readonly RedisConnection _redisConnection;
    private readonly IConfiguration _configuration;

    public AuthService(
        IUserRepository userRepository,
        IAuthTokenRepository authTokenRepository,
        ITokenGenerator tokenGenerator,
        RedisConnection redisConnection,
        IConfiguration configuration)
    {
        _userRepository = userRepository;
        _authTokenRepository = authTokenRepository;
        _tokenGenerator = tokenGenerator;
        _redisConnection = redisConnection;
        _configuration = configuration;
    }

    public async Task<(bool success, long userId, string token, string message)> LoginAsync(
        string email, string password)
    {
        // 1. 이메일로 사용자 조회
        var (exists, userId, storedPassword) = await _userRepository.GetUserByEmailAsync(email);
        
        if (!exists)
        {
            return (false, 0, string.Empty, "User not found");
        }

        // 2. 비밀번호 검증
        if (storedPassword != password)
        {
            return (false, 0, string.Empty, "Invalid password");
        }

        // 3. 토큰 생성
        var token = _tokenGenerator.GenerateToken(userId);
        var createdAt = DateTimeOffset.UtcNow.ToUnixTimeSeconds();
        var tokenExpirationHours = _configuration.GetValue<int>("Security:TokenExpirationHours", 24);
        var expiredAt = createdAt + (tokenExpirationHours * 3600);

        // 4. MySQL에 토큰 저장
        var savedToDb = await _authTokenRepository.SaveTokenAsync(userId, token, createdAt, expiredAt);
        
        if (!savedToDb)
        {
            return (false, 0, string.Empty, "Failed to save token");
        }

        // 5. Redis에 토큰 캐싱
        await CacheTokenToRedisAsync(userId, token, tokenExpirationHours);

        return (true, userId, token, "Login successful");
    }

    private async Task CacheTokenToRedisAsync(long userId, string token, int expirationHours)
    {
        try
        {
            var redisKey = $"auth:token:{userId}";
            var redisString = new RedisString<string>(_redisConnection, redisKey, TimeSpan.FromHours(expirationHours));
            await redisString.SetAsync(token);
        }
        catch
        {
            // Redis 캐싱 실패는 로그만 남기고 계속 진행
            // 실제 프로젝트에서는 로깅 처리 필요
        }
    }
}
```

## 9.7 로그인 API 컨트롤러 구현

### 9.7.1 AuthController 생성

`Controllers/AuthController.cs` 파일을 생성한다.

```csharp
using Microsoft.AspNetCore.Mvc;
using GameServer.Models.Request;
using GameServer.Models.Response;
using GameServer.Services;

namespace GameServer.Controllers;

[ApiController]
[Route("api/[controller]")]
public class AuthController : ControllerBase
{
    private readonly IAuthService _authService;

    public AuthController(IAuthService authService)
    {
        _authService = authService;
    }

    [HttpPost("login")]
    public async Task<ActionResult<LoginResponse>> Login([FromBody] LoginRequest request)
    {
        if (!ModelState.IsValid)
        {
            return BadRequest(new LoginResponse
            {
                Success = false,
                Message = "Invalid request"
            });
        }

        var (success, userId, token, message) = await _authService.LoginAsync(
            request.Email, 
            request.Password);

        if (!success)
        {
            return Unauthorized(new LoginResponse
            {
                Success = false,
                Message = message
            });
        }

        return Ok(new LoginResponse
        {
            Success = true,
            UserId = userId,
            Token = token,
            Message = message
        });
    }
}
```

## 9.10 토큰 만료 시간 관리

### 9.10.1 만료 시간 설정 전략

토큰의 만료 시간은 게임의 특성에 따라 결정한다.

**짧은 만료 시간 (1-6시간)**
- 보안성이 높다.
- 사용자가 자주 재로그인해야 한다.
- 실시간 대전 게임에 적합하다.

**중간 만료 시간 (12-24시간)**
- 보안과 편의성의 균형이 좋다.
- 대부분의 모바일 게임에 적합하다.
- 이 교육에서 사용하는 기본값이다.

**긴 만료 시간 (7-30일)**
- 사용자 편의성이 높다.
- 보안 위험이 증가한다.
- 캐주얼 게임에 적합하다.

### 9.10.2 appsettings.json에서 설정 변경

만료 시간을 변경하려면 appsettings.json을 수정한다.

```json
{
  "Security": {
    "SecretKey": "YourVerySecureSecretKeyAtLeast32BytesLong!",
    "TokenExpirationHours": 24
  }
}
```

개발/스테이징/운영 환경별로 다른 값을 설정할 수 있다.

### 9.10.3 만료된 토큰 정리

주기적으로 만료된 토큰을 정리하는 배치 작업이 필요하다. 다음 쿼리로 만료된 토큰을 삭제할 수 있다.

```sql
DELETE FROM user_auth_token 
WHERE expired_at < UNIX_TIMESTAMP();
```

실제 프로젝트에서는 백그라운드 서비스나 스케줄러를 사용하여 자동으로 정리한다.

using BCrypt.Net;

public async Task<bool> RegisterAsync(string email, string password, string nickname)
{
    // 비밀번호 해시 생성
    var hashedPassword = BCrypt.Net.BCrypt.HashPassword(password);

## 9.12 에러 처리 및 로깅

### 9.12.1 상세한 에러 처리

AuthService에 에러 처리를 추가한다.

```csharp
public async Task<(bool success, long userId, string token, string message)> LoginAsync(
    string email, string password)
{
    try
    {
        // 1. 입력 검증
        if (string.IsNullOrWhiteSpace(email) || string.IsNullOrWhiteSpace(password))
        {
            return (false, 0, string.Empty, "Email and password are required");
        }

        // 2. 이메일로 사용자 조회
        var (exists, userId, storedPassword) = await _userRepository.GetUserByEmailAsync(email);
        
        if (!exists)
        {
            return (false, 0, string.Empty, "User not found");
        }

        // 3. 비밀번호 검증
        if (storedPassword != password)
        {
            return (false, 0, string.Empty, "Invalid password");
        }

        // 4. 토큰 생성
        var token = _tokenGenerator.GenerateToken(userId);
        var createdAt = DateTimeOffset.UtcNow.ToUnixTimeSeconds();
        var tokenExpirationHours = _configuration.GetValue<int>("Security:TokenExpirationHours", 24);
        var expiredAt = createdAt + (tokenExpirationHours * 3600);

        // 5. MySQL에 토큰 저장
        var savedToDb = await _authTokenRepository.SaveTokenAsync(userId, token, createdAt, expiredAt);
        
        if (!savedToDb)
        {
            return (false, 0, string.Empty, "Failed to save token");
        }

        // 6. Redis에 토큰 캐싱
        await CacheTokenToRedisAsync(userId, token, tokenExpirationHours);

        return (true, userId, token, "Login successful");
    }
    catch (Exception ex)
    {
        // 실제 프로젝트에서는 로깅 처리
        return (false, 0, string.Empty, "Internal server error");
    }
}
```

### 9.12.2 컨트롤러 에러 처리

컨트롤러에서도 적절한 HTTP 상태 코드를 반환한다.

```csharp
[HttpPost("login")]
public async Task<ActionResult<LoginResponse>> Login([FromBody] LoginRequest request)
{
    if (!ModelState.IsValid)
    {
        return BadRequest(new LoginResponse
        {
            Success = false,
            Message = "Invalid request data"
        });
    }

    var (success, userId, token, message) = await _authService.LoginAsync(
        request.Email, 
        request.Password);

    if (!success)
    {
        // 사용자를 찾을 수 없거나 비밀번호가 틀린 경우
        if (message.Contains("not found") || message.Contains("Invalid password"))
        {
            return Unauthorized(new LoginResponse
            {
                Success = false,
                Message = message
            });
        }
        
        // 서버 내부 에러
        return StatusCode(500, new LoginResponse
        {
            Success = false,
            Message = message
        });
    }

    return Ok(new LoginResponse
    {
        Success = true,
        UserId = userId,
        Token = token,
        Message = message
    });
}
```

### 9.14.1 구현한 기능 요약
