# Chapter 8: 연속 집계(Continuous Aggregates) - 실시간 대시보드의 비밀

7장에서 작성했던 분석 쿼리들을 기억하는가? 시간대별 활동량, 던전 인기도, 재화 흐름 같은 통계를 계산하는 복잡한 쿼리들이었다. 이런 쿼리들은 수백만, 수천만 건의 로우를 스캔하고 집계해야 하기 때문에 실행 시간이 길다. 대시보드를 새로고침할 때마다 이런 무거운 쿼리를 실행한다면 데이터베이스에 큰 부하가 걸리고 사용자는 느린 응답 속도를 경험하게 된다.

이 문제를 해결하는 전통적인 방법은 배치 작업을 통해 주기적으로 집계 결과를 별도 테이블에 저장하는 것이다. 하지만 이 방식은 실시간성이 떨어지고, 배치 스케줄 관리가 복잡하며, 원본 데이터와 집계 데이터의 동기화 문제가 발생할 수 있다.

TimescaleDB의 **연속 집계(Continuous Aggregates)**는 이 문제를 우아하게 해결한다. 연속 집계는 복잡한 집계 쿼리를 물리적으로 구체화된 뷰(Materialized View)로 만들되, 원본 데이터가 추가될 때 자동으로 갱신되는 마법 같은 기능이다. 대시보드는 항상 최신 데이터를 빠르게 조회할 수 있고, 개발자는 복잡한 동기화 로직을 작성할 필요가 없다.

이번 챕터에서는 연속 집계의 작동 원리를 이해하고, 게임 서버 대시보드에 필요한 실시간 지표들을 연속 집계로 구현한다. 동시 접속자 통계, 매출 집계, 서버 성능 지표 등을 자동으로 갱신되는 뷰로 만들어서 대시보드의 성능을 극적으로 개선한다.

```
┌─────────────────────────────────────────────────────────────┐
│              연속 집계의 작동 원리                              │
└─────────────────────────────────────────────────────────────┘

원본 데이터 (Hypertable)
┌──────────────────────────────────────┐
│  game_events (수백만 건)              │
│  2025-12-31 14:00:01 | player_login │
│  2025-12-31 14:00:03 | item_acquired│
│  2025-12-31 14:00:05 | dungeon_clear│
│  2025-12-31 14:00:08 | currency_... │
│  ... (계속해서 추가됨)                 │
└──────────┬───────────────────────────┘
           │
           │ 자동 집계 (Background Job)
           ▼
연속 집계 뷰 (Materialized View)
┌──────────────────────────────────────┐
│  hourly_player_stats                 │
│  2025-12-31 14:00 | 2,345 players   │
│  2025-12-31 15:00 | 2,567 players   │
│  ... (자동 갱신됨)                     │
└──────────┬───────────────────────────┘
           │
           │ 빠른 조회 (인덱스된 집계 데이터)
           ▼
대시보드 / API
┌──────────────────────────────────────┐
│  실시간 통계 대시보드                  │
│  - 현재 동접자: 2,567명               │
│  - 시간당 신규 가입: 125명            │
│  - 매출 (이번 시간): $12,345         │
└──────────────────────────────────────┘

성능 비교:
• 원본 쿼리: 3~5초 (수백만 건 스캔)
• 연속 집계: 0.01초 (사전 계산된 결과)
```

## 8.1 연속 집계란 무엇인가?

연속 집계는 PostgreSQL의 물리적 뷰(Materialized View)를 시계열 데이터에 최적화한 TimescaleDB의 특별한 기능이다. 일반적인 Materialized View와의 가장 큰 차이점은 **증분 갱신(Incremental Refresh)**을 지원한다는 것이다.

전통적인 Materialized View는 갱신할 때 전체 데이터를 다시 계산해야 한다. 만약 1억 건의 로그가 있고 새로 1만 건이 추가되었다면, 갱신 시 1억 1만 건 전체를 다시 집계해야 한다. 이는 시간도 오래 걸리고 리소스도 많이 소모한다.

반면 TimescaleDB의 연속 집계는 **마지막 갱신 이후 추가된 데이터만** 처리한다. 위의 예에서는 새로 추가된 1만 건만 집계하여 기존 결과에 병합한다. 이를 통해 갱신 시간을 크게 단축하고, 실시간에 가까운 대시보드를 구현할 수 있다.

연속 집계의 주요 특징은 다음과 같다.

**자동 갱신**: 리프레시 정책을 설정하면 백그라운드에서 자동으로 최신 데이터를 반영한다. 별도의 스케줄러나 크론 작업이 필요 없다.

**증분 계산**: 전체 데이터를 다시 계산하지 않고 변경된 부분만 업데이트한다. 이는 시간과 리소스를 절약한다.

**시간 버킷 기반**: `time_bucket` 함수를 사용하여 시간 단위로 데이터를 그룹화한다. 1분, 5분, 1시간, 1일 등 다양한 간격을 지원한다.

**투명한 쿼리**: 일반 테이블처럼 SELECT 쿼리로 조회할 수 있다. 애플리케이션 코드는 이것이 연속 집계인지 알 필요가 없다.

**데이터 보관 정책과 통합**: 원본 데이터가 압축되거나 삭제되어도 집계 결과는 유지된다. 오래된 상세 로그는 삭제하고 요약 통계만 남길 수 있다.

연속 집계의 제약사항도 이해해야 한다. 첫째, `time_bucket`으로 그룹화하는 쿼리만 지원된다. 시간 기반이 아닌 집계는 사용할 수 없다. 둘째, 집계 함수는 증분 계산이 가능한 것들만 사용할 수 있다. `SUM`, `COUNT`, `AVG`, `MIN`, `MAX`는 가능하지만, `MEDIAN`이나 복잡한 윈도우 함수는 제한적이다. 셋째, JOIN은 제한적으로만 지원된다.

하지만 이런 제약사항은 대부분의 실시간 대시보드 시나리오에서 문제가 되지 않는다. 시간대별 통계, 집계 지표, 추이 분석 등 가장 흔한 사용 사례는 모두 완벽하게 지원된다.

## 8.2 첫 연속 집계 뷰 생성하기

이론은 충분히 배웠으니 실제로 연속 집계를 만들어본다. 가장 기본적인 예제로 시작한다. 플레이어 로그인 이벤트를 시간대별로 집계하는 뷰를 생성한다.

먼저 pgAdmin을 열고 다음 SQL을 실행한다.

```sql
-- 시간별 플레이어 로그인 통계 연속 집계
CREATE MATERIALIZED VIEW hourly_login_stats
WITH (timescaledb.continuous) AS
SELECT 
    time_bucket('1 hour', time) AS bucket,
    COUNT(*) AS login_count,
    COUNT(DISTINCT player_id) AS unique_players
FROM game_events
WHERE event_type = 'player_login'
GROUP BY bucket
WITH NO DATA;

-- 리프레시 정책 설정: 매 5분마다 최근 2시간 데이터 갱신
SELECT add_continuous_aggregate_policy('hourly_login_stats',
    start_offset => INTERVAL '2 hours',
    end_offset => INTERVAL '5 minutes',
    schedule_interval => INTERVAL '5 minutes');
```

이 SQL 문을 하나씩 분석해본다.

`CREATE MATERIALIZED VIEW`는 일반 PostgreSQL 구문이지만, `WITH (timescaledb.continuous)` 옵션이 추가되어 TimescaleDB의 연속 집계로 만들어진다. 뷰의 이름은 `hourly_login_stats`로 지정했다.

`SELECT` 절에서 `time_bucket('1 hour', time)`은 시간을 1시간 단위로 그룹화한다. 예를 들어 14:23, 14:47, 14:55의 이벤트는 모두 14:00 버킷으로 묶인다.

`COUNT(*)`는 해당 시간대의 총 로그인 횟수를, `COUNT(DISTINCT player_id)`는 고유 플레이어 수를 계산한다. 만약 같은 플레이어가 한 시간에 여러 번 로그인했다면 `login_count`는 증가하지만 `unique_players`는 1로만 카운트된다.

`WITH NO DATA`는 생성 시점에 데이터를 채우지 않고 뷰 구조만 만든다는 의미다. 실제 데이터는 첫 리프레시 시에 채워진다.

`add_continuous_aggregate_policy` 함수는 자동 갱신 정책을 설정한다. 파라미터의 의미는 다음과 같다.

`start_offset => INTERVAL '2 hours'`는 "현재 시각부터 2시간 전까지의 데이터를 갱신한다"는 뜻이다. 너무 최신 데이터는 아직 완전히 수집되지 않았을 수 있으므로 약간의 지연을 둔다.

`end_offset => INTERVAL '5 minutes'`는 "현재 시각 5분 전까지"를 의미한다. 즉, 5분 전부터 2시간 전까지의 데이터가 갱신 대상이다.

`schedule_interval => INTERVAL '5 minutes'`는 "5분마다 이 정책을 실행한다"는 뜻이다.

리프레시 정책의 개념을 시각화하면 다음과 같다.

```
현재 시각: 15:00

├─────────┼─────────┼─────────┼─────────┼─────────┤
12:00    13:00    14:00    14:55    15:00
          │←── 갱신 범위 ──→│ 버퍼
          start_offset      end_offset
          (2 hours ago)     (5 min ago)
```

이제 첫 수동 리프레시를 실행하여 데이터를 채운다.

```sql
-- 전체 기간에 대해 수동 리프레시
CALL refresh_continuous_aggregate('hourly_login_stats', NULL, NULL);

-- 결과 확인
SELECT * FROM hourly_login_stats
ORDER BY bucket DESC
LIMIT 10;
```

결과는 다음과 같이 나타난다.

```
         bucket          | login_count | unique_players 
-------------------------+-------------+----------------
 2025-12-31 14:00:00+00 |        4567 |           2345
 2025-12-31 13:00:00+00 |        4321 |           2198
 2025-12-31 12:00:00+00 |        3876 |           1987
 2025-12-31 11:00:00+00 |        3234 |           1654
 ...
```

이제부터 5분마다 자동으로 이 뷰가 갱신된다. 새로운 로그인 이벤트가 `game_events` 테이블에 추가되면, 다음 리프레시 주기에 자동으로 `hourly_login_stats`에 반영된다.

C#에서 이 연속 집계를 사용하는 코드를 작성한다.

```csharp
using System;
using System.Collections.Generic;
using System.Threading.Tasks;
using SqlKata;
using SqlKata.Execution;
using Npgsql;

namespace GameServerDashboard.Services
{
    /// <summary>
    /// 연속 집계를 조회하는 서비스
    /// </summary>
    public class ContinuousAggregateService
    {
        private readonly string _connectionString;
        private readonly QueryFactory _db;

        public ContinuousAggregateService(string connectionString)
        {
            _connectionString = connectionString;
            var connection = new NpgsqlConnection(connectionString);
            var compiler = new SqlKata.Compilers.PostgresCompiler();
            _db = new QueryFactory(connection, compiler);
        }

        /// <summary>
        /// 시간별 로그인 통계를 조회한다
        /// </summary>
        public async Task<List<HourlyLoginStat>> GetHourlyLoginStatsAsync(
            DateTime startTime,
            DateTime endTime)
        {
            var query = new Query("hourly_login_stats")
                .Select("bucket", "login_count", "unique_players")
                .WhereBetween("bucket", startTime, endTime)
                .OrderByDesc("bucket");

            var results = await _db.GetAsync<HourlyLoginStat>(query);
            return results.ToList();
        }

        /// <summary>
        /// 최근 24시간 로그인 추이를 조회한다
        /// </summary>
        public async Task<List<HourlyLoginStat>> GetRecent24HoursLoginAsync()
        {
            var endTime = DateTime.UtcNow;
            var startTime = endTime.AddHours(-24);
            return await GetHourlyLoginStatsAsync(startTime, endTime);
        }

        /// <summary>
        /// 특정 시간의 통계를 조회한다
        /// </summary>
        public async Task<HourlyLoginStat> GetLoginStatsAtTimeAsync(DateTime time)
        {
            // 시간을 1시간 단위로 버킷화
            var bucket = new DateTime(
                time.Year, time.Month, time.Day, time.Hour, 0, 0, DateTimeKind.Utc);

            var query = new Query("hourly_login_stats")
                .Select("bucket", "login_count", "unique_players")
                .Where("bucket", bucket)
                .Limit(1);

            return await _db.FirstOrDefaultAsync<HourlyLoginStat>(query);
        }

        /// <summary>
        /// 피크 타임을 찾는다 (가장 많은 플레이어가 로그인한 시간)
        /// </summary>
        public async Task<List<HourlyLoginStat>> GetPeakHoursAsync(int days = 7, int limit = 10)
        {
            var startTime = DateTime.UtcNow.AddDays(-days);

            var query = new Query("hourly_login_stats")
                .Select("bucket", "login_count", "unique_players")
                .Where("bucket", ">=", startTime)
                .OrderByDesc("unique_players")
                .Limit(limit);

            var results = await _db.GetAsync<HourlyLoginStat>(query);
            return results.ToList();
        }
    }

    /// <summary>
    /// 시간별 로그인 통계 모델
    /// </summary>
    public class HourlyLoginStat
    {
        public DateTime Bucket { get; set; }
        public long LoginCount { get; set; }
        public long UniquePlayers { get; set; }

        /// <summary>
        /// 평균 재접속 횟수 (한 플레이어가 평균 몇 번 로그인했는지)
        /// </summary>
        public double AvgLoginsPerPlayer => 
            UniquePlayers > 0 ? (double)LoginCount / UniquePlayers : 0;
    }
}
```

실제 사용 예제를 작성한다.

```csharp
using System;
using System.Threading.Tasks;

namespace GameServerDashboard.Examples
{
    class ContinuousAggregateExample
    {
        static async Task Main(string[] args)
        {
            var connectionString = "Host=localhost;Port=5432;Database=gamedb;Username=postgres;Password=your_password";
            var service = new ContinuousAggregateService(connectionString);

            Console.WriteLine("=== 연속 집계 조회 예제 ===\n");

            // 1. 최근 24시간 로그인 추이
            Console.WriteLine("📊 최근 24시간 로그인 추이:");
            var recent = await service.GetRecent24HoursLoginAsync();
            foreach (var stat in recent.Take(10))
            {
                Console.WriteLine($"  {stat.Bucket:yyyy-MM-dd HH:00} | " +
                    $"고유 플레이어: {stat.UniquePlayers:N0}, " +
                    $"총 로그인: {stat.LoginCount:N0}, " +
                    $"평균 재접속: {stat.AvgLoginsPerPlayer:F2}회");
            }

            // 2. 현재 시간 통계
            Console.WriteLine("\n📌 현재 시간 통계:");
            var currentStat = await service.GetLoginStatsAtTimeAsync(DateTime.UtcNow);
            if (currentStat != null)
            {
                Console.WriteLine($"  시간: {currentStat.Bucket:yyyy-MM-dd HH:00}");
                Console.WriteLine($"  고유 플레이어: {currentStat.UniquePlayers:N0}");
                Console.WriteLine($"  총 로그인: {currentStat.LoginCount:N0}");
            }
            else
            {
                Console.WriteLine("  아직 데이터가 없습니다.");
            }

            // 3. 최근 7일 피크 타임
            Console.WriteLine("\n🔥 최근 7일 피크 타임 TOP 5:");
            var peakHours = await service.GetPeakHoursAsync(days: 7, limit: 5);
            foreach (var stat in peakHours)
            {
                Console.WriteLine($"  {stat.Bucket:yyyy-MM-dd HH:00} | " +
                    $"{stat.UniquePlayers:N0}명");
            }

            Console.WriteLine("\n✅ 조회 완료!");
        }
    }
}
```

## 8.3 시간대별 동접자 통계 자동화

로그인 통계는 기본적인 예제였다. 이제 더 실용적인 지표인 **동시 접속자 수(CCU, Concurrent Users)**를 추적하는 연속 집계를 만든다.

동접자 수를 계산하는 방법은 여러 가지가 있다. 가장 정확한 방법은 매 시점마다 활성 세션을 카운트하는 것이지만, 이는 복잡하고 비용이 많이 든다. 실용적인 접근은 특정 시간 윈도우 내에 활동한 플레이어 수를 추정하는 것이다.

예를 들어 5분 단위로 "최근 10분 내에 활동한 고유 플레이어 수"를 계산하면 실시간 동접자 수를 근사할 수 있다.

```sql
-- 5분 단위 동시 접속자 추정 연속 집계
CREATE MATERIALIZED VIEW concurrent_users_5min
WITH (timescaledb.continuous) AS
SELECT 
    time_bucket('5 minutes', time) AS bucket,
    COUNT(DISTINCT player_id) AS active_players,
    COUNT(*) AS total_events,
    COUNT(DISTINCT session_id) AS active_sessions
FROM game_events
WHERE event_category IN ('session', 'gameplay', 'item')  -- 의미 있는 활동만
GROUP BY bucket
WITH NO DATA;

-- 리프레시 정책: 매 1분마다 최근 30분 데이터 갱신
SELECT add_continuous_aggregate_policy('concurrent_users_5min',
    start_offset => INTERVAL '30 minutes',
    end_offset => INTERVAL '1 minute',
    schedule_interval => INTERVAL '1 minute');

-- 초기 리프레시
CALL refresh_continuous_aggregate('concurrent_users_5min', NULL, NULL);
```

더 정교한 동접자 계산을 위해 세션 시작/종료를 명시적으로 추적하는 연속 집계를 만든다.

```sql
-- 시간별 세션 시작/종료 추적
CREATE MATERIALIZED VIEW hourly_session_stats
WITH (timescaledb.continuous) AS
SELECT 
    time_bucket('1 hour', time) AS bucket,
    COUNT(*) FILTER (WHERE event_type = 'player_login') AS sessions_started,
    COUNT(*) FILTER (WHERE event_type = 'player_logout') AS sessions_ended,
    COUNT(DISTINCT player_id) FILTER (WHERE event_type = 'player_login') AS unique_logins,
    COUNT(DISTINCT player_id) FILTER (WHERE event_type = 'player_logout') AS unique_logouts
FROM game_events
WHERE event_type IN ('player_login', 'player_logout')
GROUP BY bucket
WITH NO DATA;

-- 리프레시 정책
SELECT add_continuous_aggregate_policy('hourly_session_stats',
    start_offset => INTERVAL '2 hours',
    end_offset => INTERVAL '5 minutes',
    schedule_interval => INTERVAL '5 minutes');

-- 초기 리프레시
CALL refresh_continuous_aggregate('hourly_session_stats', NULL, NULL);
```

## 8.4 매출 집계 실시간 대시보드

게임 운영에서 가장 중요한 지표 중 하나는 매출이다. 실시간 매출 대시보드를 만들기 위해 결제 이벤트를 집계하는 연속 집계를 생성한다.

```sql
-- 시간별 매출 통계 연속 집계
CREATE MATERIALIZED VIEW hourly_revenue_stats
WITH (timescaledb.continuous) AS
SELECT 
    time_bucket('1 hour', time) AS bucket,
    COUNT(*) AS payment_count,
    COUNT(*) FILTER (WHERE (event_data->>'success')::boolean = true) AS successful_payments,
    COUNT(DISTINCT player_id) AS unique_payers,
    SUM((event_data->>'amount')::numeric) FILTER (
        WHERE (event_data->>'success')::boolean = true
    ) AS total_revenue,
    AVG((event_data->>'amount')::numeric) FILTER (
        WHERE (event_data->>'success')::boolean = true
    ) AS avg_transaction_amount,
    -- 결제 수단별 집계 (JSONB 배열로 저장)
    jsonb_agg(DISTINCT event_data->>'payment_method') AS payment_methods_used
FROM game_events
WHERE event_type = 'payment'
GROUP BY bucket
WITH NO DATA;

-- 리프레시 정책
SELECT add_continuous_aggregate_policy('hourly_revenue_stats',
    start_offset => INTERVAL '3 hours',
    end_offset => INTERVAL '10 minutes',
    schedule_interval => INTERVAL '10 minutes');

-- 초기 리프레시
CALL refresh_continuous_aggregate('hourly_revenue_stats', NULL, NULL);
```

상품별 매출 분석을 위한 추가 연속 집계를 생성한다.

```sql
-- 일별 상품별 매출 통계
CREATE MATERIALIZED VIEW daily_product_revenue
WITH (timescaledb.continuous) AS
SELECT 
    time_bucket('1 day', time) AS day,
    event_data->>'product_id' AS product_id,
    COUNT(*) AS purchase_count,
    COUNT(DISTINCT player_id) AS unique_buyers,
    SUM((event_data->>'amount')::numeric) AS product_revenue
FROM game_events
WHERE event_type = 'payment'
  AND (event_data->>'success')::boolean = true
  AND event_data->>'product_id' IS NOT NULL
GROUP BY day, event_data->>'product_id'
WITH NO DATA;

-- 리프레시 정책
SELECT add_continuous_aggregate_policy('daily_product_revenue',
    start_offset => INTERVAL '3 days',
    end_offset => INTERVAL '1 day',
    schedule_interval => INTERVAL '1 hour');

-- 초기 리프레시
CALL refresh_continuous_aggregate('daily_product_revenue', NULL, NULL);
```

C# 매출 분석 서비스를 작성한다.

```csharp
using System;
using System.Collections.Generic;
using System.Linq;
using System.Threading.Tasks;
using SqlKata;
using SqlKata.Execution;

namespace GameServerDashboard.Services
{
    /// <summary>
    /// 매출 분석 서비스
    /// </summary>
    public class RevenueAnalyticsService
    {
        private readonly QueryFactory _db;

        public RevenueAnalyticsService(QueryFactory db)
        {
            _db = db;
        }

        /// <summary>
        /// 시간별 매출 통계를 조회한다
        /// </summary>
        public async Task<List<RevenueStats>> GetHourlyRevenueAsync(
            DateTime startTime,
            DateTime endTime)
        {
            var query = new Query("hourly_revenue_stats")
                .Select("bucket", "payment_count", "successful_payments",
                        "unique_payers", "total_revenue", "avg_transaction_amount")
                .WhereBetween("bucket", startTime, endTime)
                .OrderByDesc("bucket");

            var results = await _db.GetAsync<RevenueStats>(query);
            return results.ToList();
        }

        /// <summary>
        /// 오늘의 매출을 조회한다
        /// </summary>
        public async Task<RevenueSummary> GetTodayRevenueSummaryAsync()
        {
            var today = DateTime.UtcNow.Date;
            var tomorrow = today.AddDays(1);

            var query = new Query("hourly_revenue_stats")
                .SelectRaw("SUM(total_revenue) AS total")
                .SelectRaw("SUM(payment_count) AS transactions")
                .SelectRaw("SUM(unique_payers) AS payers")
                .SelectRaw("AVG(avg_transaction_amount) AS avg_amount")
                .WhereBetween("bucket", today, tomorrow);

            var result = await _db.FirstOrDefaultAsync<dynamic>(query);

            return new RevenueSummary
            {
                Date = today,
                TotalRevenue = result?.total ?? 0m,
                TotalTransactions = result?.transactions ?? 0,
                UniquePayers = result?.payers ?? 0,
                AvgTransactionAmount = result?.avg_amount ?? 0m
            };
        }

        /// <summary>
        /// 최근 7일 매출 추이를 조회한다
        /// </summary>
        public async Task<List<DailyRevenue>> GetLast7DaysRevenueAsync()
        {
            var endDate = DateTime.UtcNow.Date;
            var startDate = endDate.AddDays(-7);

            // 시간별 데이터를 일별로 재집계
            var sql = @"
                SELECT 
                    DATE_TRUNC('day', bucket) AS day,
                    SUM(total_revenue) AS daily_revenue,
                    SUM(payment_count) AS daily_transactions,
                    SUM(unique_payers) AS daily_payers
                FROM hourly_revenue_stats
                WHERE bucket >= @startDate AND bucket < @endDate
                GROUP BY DATE_TRUNC('day', bucket)
                ORDER BY day DESC";

            var results = await _db.SelectAsync<DailyRevenue>(sql, new
            {
                startDate,
                endDate
            });

            return results.ToList();
        }

        /// <summary>
        /// 상품별 매출 순위를 조회한다
        /// </summary>
        public async Task<List<ProductRevenue>> GetTopSellingProductsAsync(
            DateTime startDate,
            DateTime endDate,
            int limit = 10)
        {
            var query = new Query("daily_product_revenue")
                .Select("product_id")
                .SelectRaw("SUM(product_revenue) AS total_revenue")
                .SelectRaw("SUM(purchase_count) AS total_purchases")
                .SelectRaw("SUM(unique_buyers) AS total_buyers")
                .WhereBetween("day", startDate, endDate)
                .GroupBy("product_id")
                .OrderByDesc("total_revenue")
                .Limit(limit);

            var results = await _db.GetAsync<ProductRevenue>(query);
            return results.ToList();
        }

        /// <summary>
        /// 매출 성장률을 계산한다
        /// </summary>
        public async Task<GrowthRate> CalculateRevenueGrowthAsync()
        {
            var today = DateTime.UtcNow.Date;
            var yesterday = today.AddDays(-1);
            var lastWeek = today.AddDays(-7);
            var twoWeeksAgo = today.AddDays(-14);

            // 오늘과 어제 비교
            var todayRevenue = await GetDayRevenueAsync(today, today.AddDays(1));
            var yesterdayRevenue = await GetDayRevenueAsync(yesterday, today);

            // 이번 주와 지난 주 비교
            var thisWeekRevenue = await GetDayRevenueAsync(lastWeek, today);
            var lastWeekRevenue = await GetDayRevenueAsync(twoWeeksAgo, lastWeek);

            return new GrowthRate
            {
                DailyGrowth = CalculatePercentageChange(yesterdayRevenue, todayRevenue),
                WeeklyGrowth = CalculatePercentageChange(lastWeekRevenue, thisWeekRevenue),
                TodayRevenue = todayRevenue,
                YesterdayRevenue = yesterdayRevenue,
                ThisWeekRevenue = thisWeekRevenue,
                LastWeekRevenue = lastWeekRevenue
            };
        }

        private async Task<decimal> GetDayRevenueAsync(DateTime start, DateTime end)
        {
            var query = new Query("hourly_revenue_stats")
                .SelectRaw("COALESCE(SUM(total_revenue), 0) AS total")
                .WhereBetween("bucket", start, end);

            var result = await _db.FirstOrDefaultAsync<dynamic>(query);
            return result?.total ?? 0m;
        }

        private double CalculatePercentageChange(decimal oldValue, decimal newValue)
        {
            if (oldValue == 0) return newValue > 0 ? 100 : 0;
            return (double)((newValue - oldValue) / oldValue * 100);
        }
    }

    // 모델 클래스들
    public class RevenueStats
    {
        public DateTime Bucket { get; set; }
        public long PaymentCount { get; set; }
        public long SuccessfulPayments { get; set; }
        public long UniquePayers { get; set; }
        public decimal TotalRevenue { get; set; }
        public decimal AvgTransactionAmount { get; set; }

        public double SuccessRate =>
            PaymentCount > 0 ? (double)SuccessfulPayments / PaymentCount * 100 : 0;
    }

    public class RevenueSummary
    {
        public DateTime Date { get; set; }
        public decimal TotalRevenue { get; set; }
        public long TotalTransactions { get; set; }
        public long UniquePayers { get; set; }
        public decimal AvgTransactionAmount { get; set; }
    }

    public class DailyRevenue
    {
        public DateTime Day { get; set; }
        public decimal DailyRevenue { get; set; }
        public long DailyTransactions { get; set; }
        public long DailyPayers { get; set; }
    }

    public class ProductRevenue
    {
        public string ProductId { get; set; }
        public decimal TotalRevenue { get; set; }
        public long TotalPurchases { get; set; }
        public long TotalBuyers { get; set; }

        public decimal AvgRevenuePerPurchase =>
            TotalPurchases > 0 ? TotalRevenue / TotalPurchases : 0;
    }

    public class GrowthRate
    {
        public double DailyGrowth { get; set; }
        public double WeeklyGrowth { get; set; }
        public decimal TodayRevenue { get; set; }
        public decimal YesterdayRevenue { get; set; }
        public decimal ThisWeekRevenue { get; set; }
        public decimal LastWeekRevenue { get; set; }
    }
}
```

## 8.5 리프레시 정책 설정

연속 집계의 리프레시 정책은 성능과 실시간성 사이의 균형을 맞추는 중요한 요소다. 정책을 너무 자주 실행하면 데이터베이스에 부하가 걸리고, 너무 드물게 실행하면 대시보드가 오래된 데이터를 보여주게 된다.

리프레시 정책의 파라미터를 이해하고 상황에 맞게 조정하는 방법을 배운다.

```sql
-- 기존 정책 확인
SELECT * FROM timescaledb_information.jobs
WHERE proc_name = 'policy_refresh_continuous_aggregate';

-- 특정 연속 집계의 정책 확인
SELECT * FROM timescaledb_information.continuous_aggregate_stats
WHERE view_name = 'hourly_login_stats';
```

리프레시 정책을 수정하거나 제거하는 방법:

```sql
-- 정책 제거
SELECT remove_continuous_aggregate_policy('hourly_login_stats');

-- 새로운 정책 추가 (더 자주 갱신)
SELECT add_continuous_aggregate_policy('hourly_login_stats',
    start_offset => INTERVAL '1 hour',
    end_offset => INTERVAL '1 minute',
    schedule_interval => INTERVAL '1 minute');
```

상황별 권장 리프레시 전략:

**실시간 대시보드 (CCU, 최근 활동)**
- `schedule_interval`: 1~2분
- `end_offset`: 1~2분 (거의 실시간)
- `start_offset`: 30분~1시간

**매출 통계**
- `schedule_interval`: 5~10분
- `end_offset`: 5~10분 (결제 처리 지연 고려)
- `start_offset`: 2~3시간

**일일 리포트 (DAU, 이탈률 등)**
- `schedule_interval`: 1시간
- `end_offset`: 1일 (완전한 일일 데이터)
- `start_offset`: 7일

**장기 추이 분석**
- `schedule_interval`: 1일
- `end_offset`: 1일
- `start_offset`: 30일

수동 리프레시가 필요한 경우도 있다. 예를 들어 대량의 과거 데이터를 마이그레이션한 후나, 긴급하게 최신 데이터를 반영해야 할 때다.

```sql
-- 전체 기간 리프레시
CALL refresh_continuous_aggregate('hourly_login_stats', NULL, NULL);

-- 특정 기간만 리프레시
CALL refresh_continuous_aggregate('hourly_login_stats', 
    '2025-12-25 00:00:00', 
    '2025-12-31 23:59:59');

-- 최근 24시간만 리프레시
CALL refresh_continuous_aggregate('hourly_login_stats',
    NOW() - INTERVAL '24 hours',
    NOW());
```

C#에서 수동 리프레시를 실행하는 유틸리티를 작성한다.

```csharp
using System;
using System.Threading.Tasks;
using Npgsql;

namespace GameServerDashboard.Utils
{
    /// <summary>
    /// 연속 집계 관리 유틸리티
    /// </summary>
    public class ContinuousAggregateManager
    {
        private readonly string _connectionString;

        public ContinuousAggregateManager(string connectionString)
        {
            _connectionString = connectionString;
        }

        /// <summary>
        /// 연속 집계를 수동으로 리프레시한다
        /// </summary>
        public async Task RefreshAggregateAsync(
            string viewName,
            DateTime? startTime = null,
            DateTime? endTime = null)
        {
            await using var connection = new NpgsqlConnection(_connectionString);
            await connection.OpenAsync();

            var startParam = startTime?.ToString("yyyy-MM-dd HH:mm:ss") ?? "NULL";
            var endParam = endTime?.ToString("yyyy-MM-dd HH:mm:ss") ?? "NULL";

            var sql = $"CALL refresh_continuous_aggregate('{viewName}', '{startParam}', '{endParam}')";

            await using var command = new NpgsqlCommand(sql, connection);
            await command.ExecuteNonQueryAsync();

            Console.WriteLine($"✓ {viewName} 리프레시 완료");
        }

        /// <summary>
        /// 모든 연속 집계를 리프레시한다
        /// </summary>
        public async Task RefreshAllAggregatesAsync()
        {
            var aggregates = new[]
            {
                "hourly_login_stats",
                "concurrent_users_5min",
                "hourly_session_stats",
                "hourly_revenue_stats",
                "daily_product_revenue"
            };

            foreach (var aggregate in aggregates)
            {
                try
                {
                    await RefreshAggregateAsync(aggregate);
                }
                catch (Exception ex)
                {
                    Console.WriteLine($"✗ {aggregate} 리프레시 실패: {ex.Message}");
                }
            }
        }

        /// <summary>
        /// 연속 집계의 상태를 조회한다
        /// </summary>
        public async Task<List<AggregateStatus>> GetAggregateStatusAsync()
        {
            await using var connection = new NpgsqlConnection(_connectionString);
            await connection.OpenAsync();

            var sql = @"
                SELECT 
                    view_name,
                    materialization_hypertable_name,
                    completed_threshold,
                    invalidation_threshold
                FROM timescaledb_information.continuous_aggregates";

            var results = new List<AggregateStatus>();

            await using var command = new NpgsqlCommand(sql, connection);
            await using var reader = await command.ExecuteReaderAsync();

            while (await reader.ReadAsync())
            {
                results.Add(new AggregateStatus
                {
                    ViewName = reader.GetString(0),
                    HypertableName = reader.IsDBNull(1) ? null : reader.GetString(1),
                    CompletedThreshold = reader.IsDBNull(2) ? (DateTime?)null : reader.GetDateTime(2),
                    InvalidationThreshold = reader.IsDBNull(3) ? (DateTime?)null : reader.GetDateTime(3)
                });
            }

            return results;
        }

        /// <summary>
        /// 리프레시 정책을 수정한다
        /// </summary>
        public async Task UpdateRefreshPolicyAsync(
            string viewName,
            TimeSpan startOffset,
            TimeSpan endOffset,
            TimeSpan scheduleInterval)
        {
            await using var connection = new NpgsqlConnection(_connectionString);
            await connection.OpenAsync();

            // 기존 정책 제거
            var removeSql = $"SELECT remove_continuous_aggregate_policy('{viewName}')";
            await using var removeCommand = new NpgsqlCommand(removeSql, connection);
            
            try
            {
                await removeCommand.ExecuteNonQueryAsync();
            }
            catch
            {
                // 정책이 없으면 무시
            }

            // 새 정책 추가
            var addSql = $@"
                SELECT add_continuous_aggregate_policy('{viewName}',
                    start_offset => INTERVAL '{startOffset.TotalHours} hours',
                    end_offset => INTERVAL '{endOffset.TotalMinutes} minutes',
                    schedule_interval => INTERVAL '{scheduleInterval.TotalMinutes} minutes')";

            await using var addCommand = new NpgsqlCommand(addSql, connection);
            await addCommand.ExecuteNonQueryAsync();

            Console.WriteLine($"✓ {viewName} 리프레시 정책 업데이트 완료");
        }
    }

    public class AggregateStatus
    {
        public string ViewName { get; set; }
        public string HypertableName { get; set; }
        public DateTime? CompletedThreshold { get; set; }
        public DateTime? InvalidationThreshold { get; set; }

        /// <summary>
        /// 데이터 지연 시간
        /// </summary>
        public TimeSpan? DataLag =>
            CompletedThreshold.HasValue
                ? DateTime.UtcNow - CompletedThreshold.Value
                : null;
    }
}
```

## 8.6 SQLKata로 연속 집계 조회하기

연속 집계는 일반 테이블처럼 조회할 수 있으므로 SQLKata의 모든 기능을 사용할 수 있다. 복잡한 조회 패턴을 SQLKata로 우아하게 작성하는 방법을 배운다.

```csharp
using System;
using System.Collections.Generic;
using System.Linq;
using System.Threading.Tasks;
using SqlKata;
using SqlKata.Execution;

namespace GameServerDashboard.Services
{
    /// <summary>
    /// SQLKata를 사용한 연속 집계 조회 서비스
    /// </summary>
    public class AggregateQueryService
    {
        private readonly QueryFactory _db;

        public AggregateQueryService(QueryFactory db)
        {
            _db = db;
        }

        /// <summary>
        /// 여러 연속 집계를 조인하여 종합 대시보드 데이터를 생성한다
        /// </summary>
        public async Task<List<ComprehensiveDashboardData>> GetComprehensiveDashboardAsync(
            DateTime startTime,
            DateTime endTime)
        {
            // SQLKata로 복잡한 JOIN 쿼리 작성
            var query = new Query("hourly_login_stats as login")
                .LeftJoin("concurrent_users_5min as ccu", j => j
                    .On("login.bucket", "ccu.bucket"))
                .LeftJoin("hourly_revenue_stats as revenue", j => j
                    .On("login.bucket", "revenue.bucket"))
                .Select(
                    "login.bucket",
                    "login.unique_players",
                    "login.login_count",
                    "ccu.active_players",
                    "revenue.total_revenue",
                    "revenue.successful_payments"
                )
                .WhereBetween("login.bucket", startTime, endTime)
                .OrderByDesc("login.bucket");

            var results = await _db.GetAsync<ComprehensiveDashboardData>(query);
            return results.ToList();
        }

        /// <summary>
        /// 조건부 집계: 특정 조건을 만족하는 시간대만 조회
        /// </summary>
        public async Task<List<HourlyLoginStat>> GetHighActivityPeriodsAsync(
            int days = 7,
            int minPlayers = 1000)
        {
            var startTime = DateTime.UtcNow.AddDays(-days);

            var query = new Query("hourly_login_stats")
                .Select("bucket", "login_count", "unique_players")
                .Where("bucket", ">=", startTime)
                .Where("unique_players", ">=", minPlayers)
                .OrderByDesc("unique_players");

            var results = await _db.GetAsync<HourlyLoginStat>(query);
            return results.ToList();
        }

        /// <summary>
        /// 집계의 집계: 일별 데이터를 주별로 재집계
        /// </summary>
        public async Task<List<WeeklyStats>> GetWeeklyStatsAsync(int weeks = 4)
        {
            var startDate = DateTime.UtcNow.Date.AddDays(-weeks * 7);

            // 시간별 데이터를 주별로 재집계
            var query = new Query("hourly_login_stats")
                .SelectRaw("DATE_TRUNC('week', bucket) AS week")
                .SelectRaw("SUM(login_count) AS total_logins")
                .SelectRaw("AVG(unique_players) AS avg_hourly_players")
                .SelectRaw("MAX(unique_players) AS peak_players")
                .Where("bucket", ">=", startDate)
                .GroupByRaw("DATE_TRUNC('week', bucket)")
                .OrderByRaw("week DESC");

            var results = await _db.GetAsync<WeeklyStats>(query);
            return results.ToList();
        }

        /// <summary>
        /// 비율 계산: 시간대별 전환율 분석
        /// </summary>
        public async Task<List<ConversionStats>> GetConversionStatsAsync(
            DateTime startTime,
            DateTime endTime)
        {
            // 로그인 대비 결제 전환율
            var query = new Query("hourly_login_stats as login")
                .LeftJoin("hourly_revenue_stats as revenue", "login.bucket", "revenue.bucket")
                .SelectRaw(@"
                    login.bucket,
                    login.unique_players,
                    COALESCE(revenue.unique_payers, 0) AS payers,
                    COALESCE(revenue.total_revenue, 0) AS revenue,
                    CASE 
                        WHEN login.unique_players > 0 
                        THEN COALESCE(revenue.unique_payers, 0)::float / login.unique_players * 100
                        ELSE 0
                    END AS conversion_rate,
                    CASE 
                        WHEN COALESCE(revenue.unique_payers, 0) > 0
                        THEN COALESCE(revenue.total_revenue, 0) / revenue.unique_payers
                        ELSE 0
                    END AS arppu
                ")
                .WhereBetween("login.bucket", startTime, endTime)
                .OrderByDesc("login.bucket");

            var results = await _db.GetAsync<ConversionStats>(query);
            return results.ToList();
        }

        /// <summary>
        /// 이동 평균 계산: 시간대별 추이를 부드럽게 표현
        /// </summary>
        public async Task<List<MovingAverageStats>> GetMovingAverageAsync(
            int days = 7,
            int windowSize = 3)
        {
            var startTime = DateTime.UtcNow.AddDays(-days);

            // 윈도우 함수로 이동 평균 계산
            var sql = $@"
                SELECT 
                    bucket,
                    unique_players,
                    AVG(unique_players) OVER (
                        ORDER BY bucket 
                        ROWS BETWEEN {windowSize - 1} PRECEDING AND CURRENT ROW
                    ) AS moving_avg,
                    MAX(unique_players) OVER (
                        ORDER BY bucket 
                        ROWS BETWEEN {windowSize - 1} PRECEDING AND CURRENT ROW
                    ) AS moving_max
                FROM hourly_login_stats
                WHERE bucket >= @startTime
                ORDER BY bucket DESC";

            var results = await _db.SelectAsync<MovingAverageStats>(sql, new { startTime });
            return results.ToList();
        }

        /// <summary>
        /// 상위 N개 선택: 각 날짜별 최고 활동 시간
        /// </summary>
        public async Task<List<DailyPeakHour>> GetDailyPeakHoursAsync(int days = 7)
        {
            var startTime = DateTime.UtcNow.AddDays(-days);

            var sql = @"
                WITH ranked_hours AS (
                    SELECT 
                        DATE_TRUNC('day', bucket) AS day,
                        bucket AS peak_hour,
                        unique_players,
                        ROW_NUMBER() OVER (
                            PARTITION BY DATE_TRUNC('day', bucket)
                            ORDER BY unique_players DESC
                        ) AS rank
                    FROM hourly_login_stats
                    WHERE bucket >= @startTime
                )
                SELECT day, peak_hour, unique_players
                FROM ranked_hours
                WHERE rank = 1
                ORDER BY day DESC";

            var results = await _db.SelectAsync<DailyPeakHour>(sql, new { startTime });
            return results.ToList();
        }
    }

    // 모델 클래스들
    public class ComprehensiveDashboardData
    {
        public DateTime Bucket { get; set; }
        public long UniquePlayers { get; set; }
        public long LoginCount { get; set; }
        public long? ActivePlayers { get; set; }
        public decimal? TotalRevenue { get; set; }
        public long? SuccessfulPayments { get; set; }
    }

    public class WeeklyStats
    {
        public DateTime Week { get; set; }
        public long TotalLogins { get; set; }
        public double AvgHourlyPlayers { get; set; }
        public long PeakPlayers { get; set; }
    }

    public class ConversionStats
    {
        public DateTime Bucket { get; set; }
        public long UniquePlayers { get; set; }
        public long Payers { get; set; }
        public decimal Revenue { get; set; }
        public double ConversionRate { get; set; }
        public decimal Arppu { get; set; } // Average Revenue Per Paying User
    }

    public class MovingAverageStats
    {
        public DateTime Bucket { get; set; }
        public long UniquePlayers { get; set; }
        public double MovingAvg { get; set; }
        public long MovingMax { get; set; }
    }

    public class DailyPeakHour
    {
        public DateTime Day { get; set; }
        public DateTime PeakHour { get; set; }
        public long UniquePlayers { get; set; }
    }
}
```

종합 예제를 실행한다.

```csharp
using System;
using System.Threading.Tasks;

namespace GameServerDashboard.Examples
{
    class AdvancedAggregateExample
    {
        static async Task Main(string[] args)
        {
            var connectionString = "Host=localhost;Port=5432;Database=gamedb;Username=postgres;Password=your_password";
            var connection = new Npgsql.NpgsqlConnection(connectionString);
            var compiler = new SqlKata.Compilers.PostgresCompiler();
            var db = new SqlKata.Execution.QueryFactory(connection, compiler);

            var queryService = new AggregateQueryService(db);

            Console.WriteLine("=== 고급 연속 집계 조회 ===\n");

            // 1. 종합 대시보드 데이터
            Console.WriteLine("📊 최근 6시간 종합 데이터:");
            var comprehensive = await queryService.GetComprehensiveDashboardAsync(
                DateTime.UtcNow.AddHours(-6),
                DateTime.UtcNow
            );
            foreach (var data in comprehensive.Take(3))
            {
                Console.WriteLine($"  {data.Bucket:HH:00} | " +
                    $"플레이어: {data.UniquePlayers:N0}, " +
                    $"활동: {data.ActivePlayers:N0}, " +
                    $"매출: ${data.TotalRevenue:N2}");
            }

            // 2. 주간 통계
            Console.WriteLine("\n📅 주간 통계 (최근 4주):");
            var weekly = await queryService.GetWeeklyStatsAsync(4);
            foreach (var week in weekly)
            {
                Console.WriteLine($"  {week.Week:yyyy-MM-dd} 주 | " +
                    $"총 로그인: {week.TotalLogins:N0}, " +
                    $"평균: {week.AvgHourlyPlayers:N0}, " +
                    $"피크: {week.PeakPlayers:N0}");
            }

            // 3. 전환율 분석
            Console.WriteLine("\n💳 시간대별 결제 전환율:");
            var conversion = await queryService.GetConversionStatsAsync(
                DateTime.UtcNow.AddHours(-12),
                DateTime.UtcNow
            );
            foreach (var stat in conversion.Take(5))
            {
                Console.WriteLine($"  {stat.Bucket:HH:00} | " +
                    $"전환율: {stat.ConversionRate:F2}%, " +
                    $"ARPPU: ${stat.Arppu:F2}");
            }

            // 4. 일별 피크 시간
            Console.WriteLine("\n🔥 일별 피크 타임:");
            var peakHours = await queryService.GetDailyPeakHoursAsync(7);
            foreach (var peak in peakHours.Take(5))
            {
                Console.WriteLine($"  {peak.Day:yyyy-MM-dd} | " +
                    $"피크: {peak.PeakHour:HH:00} " +
                    $"({peak.UniquePlayers:N0}명)");
            }

            Console.WriteLine("\n✅ 조회 완료!");
        }
    }
}
```

이번 챕터에서 배운 연속 집계는 TimescaleDB의 가장 강력한 기능 중 하나다. 복잡한 분석 쿼리를 사전 계산하여 대시보드의 응답 속도를 극적으로 개선하면서도, 실시간성을 유지할 수 있다. 

다음 챕터에서는 데이터 압축과 보관 정책을 배운다. 시간이 지난 오래된 데이터를 압축하여 스토리지 비용을 절감하고, 불필요한 데이터는 자동으로 삭제하는 전략을 다룬다.
