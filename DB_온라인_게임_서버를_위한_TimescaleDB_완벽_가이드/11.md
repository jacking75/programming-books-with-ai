# Chapter 11: 백업과 복구 전략

새벽 3시, 긴급 전화가 울린다. "데이터베이스 서버가 다운됐습니다!" 가슴이 철렁 내려앉는 순간이다. 지난 6개월간 쌓인 플레이어 로그, 결제 기록, 게임 통계... 모든 것이 한순간에 사라질 수 있다. 하지만 제대로 된 백업 전략을 세워두었다면 이런 상황도 침착하게 대응할 수 있다. 30분 안에 서비스를 복구하고, 단 한 건의 데이터도 잃지 않을 수 있다.

백업은 보험과 같다. 평소에는 귀찮고 비용만 들지만, 위기가 닥쳤을 때 회사를 구하는 마지막 보루가 된다. 실제로 많은 게임 서비스가 적절한 백업 없이 운영하다가 하드웨어 장애, 랜섬웨어 공격, 사람의 실수로 인해 치명적인 손실을 입었다. 이번 챕터에서는 라이브 게임 서버에서 사용할 수 있는 완벽한 백업과 복구 전략을 배운다.

## 11.1 백업의 중요성과 전략

백업은 단순히 데이터를 복사하는 것 이상이다. 얼마나 자주 백업할지, 얼마나 오래 보관할지, 어떻게 빠르게 복구할지를 모두 고려한 전략이 필요하다.

**백업이 필요한 재난 시나리오**

게임 서버에서 실제로 발생할 수 있는 재난 상황들을 살펴보자.

```
┌──────────────────────────────────────────────────────────┐
│                  재난 시나리오                            │
├──────────────────────────────────────────────────────────┤
│                                                          │
│ 1. 하드웨어 장애                                         │
│    → 디스크 고장, 서버 다운                              │
│    → 복구 시간: 1~4시간                                  │
│                                                          │
│ 2. 사람의 실수                                           │
│    → DROP TABLE 실수 실행, DELETE WHERE 누락             │
│    → 복구 시간: 10분~2시간                               │
│                                                          │
│ 3. 소프트웨어 버그                                       │
│    → 잘못된 배치 작업, 데이터 손상                       │
│    → 복구 시간: 30분~4시간                               │
│                                                          │
│ 4. 보안 침해                                             │
│    → 랜섬웨어, 해킹, 데이터 삭제                         │
│    → 복구 시간: 2~24시간                                 │
│                                                          │
│ 5. 자연재해                                              │
│    → 화재, 홍수, 지진                                    │
│    → 복구 시간: 클라우드 백업 필수                       │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

각 시나리오마다 필요한 백업 전략이 다르다. 사람의 실수는 몇 시간 전으로 돌아가면 되지만, 하드웨어 장애는 전체 시스템을 새로운 서버로 옮겨야 한다.

**백업의 3가지 핵심 원칙: 3-2-1 규칙**

업계에서 널리 사용되는 백업 원칙이다.

```
3-2-1 백업 규칙:

┌─────────────────────────────────────┐
│ 3개의 복사본                        │
│ → 원본 1개 + 백업 2개               │
└─────────────────────────────────────┘
         ↓
┌─────────────────────────────────────┐
│ 2가지 다른 저장 매체                │
│ → 로컬 디스크 + 클라우드 스토리지   │
└─────────────────────────────────────┘
         ↓
┌─────────────────────────────────────┐
│ 1개는 오프사이트 보관               │
│ → 다른 지역의 데이터센터            │
└─────────────────────────────────────┘
```

예를 들어 게임 서버 데이터베이스는 다음과 같이 구성할 수 있다.

$$\text{원본 데이터} + \text{로컬 백업 (같은 서버의 다른 디스크)} + \text{클라우드 백업 (AWS S3)}$$

**백업 전략의 종류**

```csharp
public enum BackupStrategy
{
    Full,           // 전체 백업: 모든 데이터를 백업
    Incremental,    // 증분 백업: 마지막 백업 이후 변경된 데이터만
    Differential,   // 차등 백업: 마지막 전체 백업 이후 변경된 데이터
    Continuous      // 연속 백업: 모든 변경 사항을 실시간으로 기록
}

public class BackupPlan
{
    public string Name { get; set; }
    public BackupStrategy Strategy { get; set; }
    public TimeSpan Frequency { get; set; }
    public TimeSpan RetentionPeriod { get; set; }
    public string Destination { get; set; }
}
```

게임 서버를 위한 추천 백업 계획은 다음과 같다.

```csharp
public class GameServerBackupPlan
{
    public static List<BackupPlan> GetRecommendedPlans()
    {
        return new List<BackupPlan>
        {
            // 1. 전체 백업: 매일 새벽 3시
            new BackupPlan
            {
                Name = "Daily Full Backup",
                Strategy = BackupStrategy.Full,
                Frequency = TimeSpan.FromDays(1),
                RetentionPeriod = TimeSpan.FromDays(30),
                Destination = "Local Storage + AWS S3"
            },

            // 2. 증분 백업: 매시간
            new BackupPlan
            {
                Name = "Hourly Incremental Backup",
                Strategy = BackupStrategy.Incremental,
                Frequency = TimeSpan.FromHours(1),
                RetentionPeriod = TimeSpan.FromDays(7),
                Destination = "Local Storage"
            },

            // 3. 연속 아카이빙: 실시간
            new BackupPlan
            {
                Name = "Continuous WAL Archiving",
                Strategy = BackupStrategy.Continuous,
                Frequency = TimeSpan.Zero, // 실시간
                RetentionPeriod = TimeSpan.FromDays(7),
                Destination = "Local Storage + AWS S3"
            }
        };
    }
}
```

**복구 시간 목표(RTO)와 복구 시점 목표(RPO)**

백업 전략을 세울 때 가장 중요한 두 가지 지표다.

```
RTO (Recovery Time Objective):
  서비스를 얼마나 빨리 복구할 것인가?
  
  예: "서버 장애 발생 시 30분 안에 서비스 재개"

RPO (Recovery Point Objective):
  얼마나 많은 데이터 손실을 감수할 것인가?
  
  예: "최대 5분치 데이터 손실 허용"

┌───────────────────────────────────────────────────┐
│              비용 vs 목표 관계                    │
├───────────────────────────────────────────────────┤
│                                                   │
│  비용 높음 ▲                                      │
│           │    ╱                                  │
│           │   ╱                                   │
│           │  ╱                                    │
│           │ ╱                                     │
│  비용 낮음 └──────────────────────►               │
│            RTO/RPO 높음      RTO/RPO 낮음         │
│            (느린 복구,       (빠른 복구,          │
│             많은 손실)        적은 손실)          │
│                                                   │
└───────────────────────────────────────────────────┘
```

게임 서비스의 중요도에 따라 목표를 설정한다.

## 11.2 pg_dump로 백업하기

PostgreSQL의 기본 백업 도구인 `pg_dump`는 데이터베이스를 논리적으로 백업한다. SQL 스크립트나 바이너리 형식으로 덤프하여 나중에 복원할 수 있다.

**pg_dump의 장단점**

```
장점:
  ✓ 간단하고 직관적
  ✓ 특정 테이블만 선택적으로 백업 가능
  ✓ 다른 PostgreSQL 버전으로 이식 가능
  ✓ 압축 지원으로 저장 공간 절약

단점:
  ✗ 대용량 데이터베이스는 백업 시간이 오래 걸림
  ✗ 백업 중 데이터베이스 락 발생 가능
  ✗ Point-in-Time Recovery 불가능
```

**기본 pg_dump 사용법**

Windows 11에서 Docker로 실행 중인 TimescaleDB를 백업하는 방법이다.

```bash
# 전체 데이터베이스 백업 (SQL 형식)
docker exec timescaledb pg_dump -U postgres -d gamedb > backup_gamedb.sql

# 압축된 바이너리 형식 (권장)
docker exec timescaledb pg_dump -U postgres -d gamedb -Fc > backup_gamedb.dump

# 특정 테이블만 백업
docker exec timescaledb pg_dump -U postgres -d gamedb -t server_metrics -Fc > backup_server_metrics.dump

# 여러 테이블 백업
docker exec timescaledb pg_dump -U postgres -d gamedb -t server_metrics -t player_login_logs -Fc > backup_logs.dump

# 스키마만 백업 (데이터 제외)
docker exec timescaledb pg_dump -U postgres -d gamedb --schema-only > backup_schema.sql
```

**C#에서 pg_dump 실행하기**

```csharp
using System;
using System.Diagnostics;
using System.IO;
using System.Threading.Tasks;

public class PostgresBackupManager
{
    private readonly string _dockerContainerName;
    private readonly string _databaseName;
    private readonly string _username;
    private readonly string _backupDirectory;

    public PostgresBackupManager(
        string dockerContainerName = "timescaledb",
        string databaseName = "gamedb",
        string username = "postgres",
        string backupDirectory = @"D:\GameBackups")
    {
        _dockerContainerName = dockerContainerName;
        _databaseName = databaseName;
        _username = username;
        _backupDirectory = backupDirectory;

        // 백업 디렉토리 생성
        Directory.CreateDirectory(_backupDirectory);
    }

    // 전체 데이터베이스 백업
    public async Task<BackupResult> CreateFullBackupAsync(string backupName = null)
    {
        backupName ??= $"backup_{_databaseName}_{DateTime.Now:yyyyMMdd_HHmmss}.dump";
        var backupPath = Path.Combine(_backupDirectory, backupName);

        Console.WriteLine($"백업 시작: {backupName}");
        var stopwatch = Stopwatch.StartNew();

        try
        {
            // Docker 컨테이너 내부에서 pg_dump 실행
            var processInfo = new ProcessStartInfo
            {
                FileName = "docker",
                Arguments = $"exec {_dockerContainerName} pg_dump -U {_username} -d {_databaseName} -Fc",
                RedirectStandardOutput = true,
                RedirectStandardError = true,
                UseShellExecute = false,
                CreateNoWindow = true
            };

            using var process = Process.Start(processInfo);
            
            // 백업 데이터를 파일로 저장
            using (var fileStream = File.Create(backupPath))
            {
                await process.StandardOutput.BaseStream.CopyToAsync(fileStream);
            }

            await process.WaitForExitAsync();
            stopwatch.Stop();

            if (process.ExitCode != 0)
            {
                var error = await process.StandardError.ReadToEndAsync();
                throw new Exception($"pg_dump 실패: {error}");
            }

            var fileInfo = new FileInfo(backupPath);

            return new BackupResult
            {
                Success = true,
                BackupPath = backupPath,
                BackupSize = fileInfo.Length,
                Duration = stopwatch.Elapsed,
                Timestamp = DateTime.Now
            };
        }
        catch (Exception ex)
        {
            stopwatch.Stop();
            return new BackupResult
            {
                Success = false,
                ErrorMessage = ex.Message,
                Duration = stopwatch.Elapsed,
                Timestamp = DateTime.Now
            };
        }
    }

    // 특정 테이블만 백업
    public async Task<BackupResult> BackupTableAsync(string tableName)
    {
        var backupName = $"backup_{tableName}_{DateTime.Now:yyyyMMdd_HHmmss}.dump";
        var backupPath = Path.Combine(_backupDirectory, backupName);

        Console.WriteLine($"테이블 백업 시작: {tableName}");
        var stopwatch = Stopwatch.StartNew();

        try
        {
            var processInfo = new ProcessStartInfo
            {
                FileName = "docker",
                Arguments = $"exec {_dockerContainerName} pg_dump -U {_username} -d {_databaseName} -t {tableName} -Fc",
                RedirectStandardOutput = true,
                RedirectStandardError = true,
                UseShellExecute = false,
                CreateNoWindow = true
            };

            using var process = Process.Start(processInfo);
            
            using (var fileStream = File.Create(backupPath))
            {
                await process.StandardOutput.BaseStream.CopyToAsync(fileStream);
            }

            await process.WaitForExitAsync();
            stopwatch.Stop();

            if (process.ExitCode != 0)
            {
                var error = await process.StandardError.ReadToEndAsync();
                throw new Exception($"테이블 백업 실패: {error}");
            }

            var fileInfo = new FileInfo(backupPath);

            return new BackupResult
            {
                Success = true,
                BackupPath = backupPath,
                BackupSize = fileInfo.Length,
                Duration = stopwatch.Elapsed,
                Timestamp = DateTime.Now,
                TableName = tableName
            };
        }
        catch (Exception ex)
        {
            stopwatch.Stop();
            return new BackupResult
            {
                Success = false,
                ErrorMessage = ex.Message,
                Duration = stopwatch.Elapsed,
                Timestamp = DateTime.Now,
                TableName = tableName
            };
        }
    }

    // 백업 파일 복원
    public async Task<RestoreResult> RestoreBackupAsync(string backupFilePath)
    {
        if (!File.Exists(backupFilePath))
        {
            return new RestoreResult
            {
                Success = false,
                ErrorMessage = $"백업 파일을 찾을 수 없음: {backupFilePath}"
            };
        }

        Console.WriteLine($"복원 시작: {Path.GetFileName(backupFilePath)}");
        var stopwatch = Stopwatch.StartNew();

        try
        {
            // 백업 파일을 Docker 컨테이너로 복사
            var tempPath = $"/tmp/{Path.GetFileName(backupFilePath)}";
            var copyProcess = new ProcessStartInfo
            {
                FileName = "docker",
                Arguments = $"cp \"{backupFilePath}\" {_dockerContainerName}:{tempPath}",
                UseShellExecute = false,
                CreateNoWindow = true
            };

            using (var process = Process.Start(copyProcess))
            {
                await process.WaitForExitAsync();
            }

            // pg_restore 실행
            var restoreProcess = new ProcessStartInfo
            {
                FileName = "docker",
                Arguments = $"exec {_dockerContainerName} pg_restore -U {_username} -d {_databaseName} -c {tempPath}",
                RedirectStandardOutput = true,
                RedirectStandardError = true,
                UseShellExecute = false,
                CreateNoWindow = true
            };

            using (var process = Process.Start(restoreProcess))
            {
                await process.WaitForExitAsync();
                stopwatch.Stop();

                var output = await process.StandardOutput.ReadToEndAsync();
                var error = await process.StandardError.ReadToEndAsync();

                // pg_restore는 경고가 있어도 0을 반환할 수 있음
                Console.WriteLine($"복원 출력: {output}");
                if (!string.IsNullOrEmpty(error))
                {
                    Console.WriteLine($"복원 경고/오류: {error}");
                }
            }

            // 임시 파일 삭제
            var cleanupProcess = new ProcessStartInfo
            {
                FileName = "docker",
                Arguments = $"exec {_dockerContainerName} rm {tempPath}",
                UseShellExecute = false,
                CreateNoWindow = true
            };

            using (var process = Process.Start(cleanupProcess))
            {
                await process.WaitForExitAsync();
            }

            return new RestoreResult
            {
                Success = true,
                Duration = stopwatch.Elapsed,
                Timestamp = DateTime.Now
            };
        }
        catch (Exception ex)
        {
            stopwatch.Stop();
            return new RestoreResult
            {
                Success = false,
                ErrorMessage = ex.Message,
                Duration = stopwatch.Elapsed,
                Timestamp = DateTime.Now
            };
        }
    }

    // 백업 파일 목록 조회
    public List<BackupFileInfo> GetBackupFiles()
    {
        var directory = new DirectoryInfo(_backupDirectory);
        if (!directory.Exists)
        {
            return new List<BackupFileInfo>();
        }

        return directory.GetFiles("*.dump")
            .OrderByDescending(f => f.CreationTime)
            .Select(f => new BackupFileInfo
            {
                FileName = f.Name,
                FilePath = f.FullName,
                FileSize = f.Length,
                CreatedDate = f.CreationTime,
                Age = DateTime.Now - f.CreationTime
            })
            .ToList();
    }

    // 오래된 백업 파일 정리
    public int CleanupOldBackups(TimeSpan retentionPeriod)
    {
        var files = GetBackupFiles();
        var deletedCount = 0;

        foreach (var file in files.Where(f => f.Age > retentionPeriod))
        {
            try
            {
                File.Delete(file.FilePath);
                deletedCount++;
                Console.WriteLine($"오래된 백업 삭제: {file.FileName}");
            }
            catch (Exception ex)
            {
                Console.WriteLine($"백업 삭제 실패: {file.FileName} - {ex.Message}");
            }
        }

        return deletedCount;
    }
}

public class BackupResult
{
    public bool Success { get; set; }
    public string BackupPath { get; set; }
    public long BackupSize { get; set; }
    public TimeSpan Duration { get; set; }
    public DateTime Timestamp { get; set; }
    public string TableName { get; set; }
    public string ErrorMessage { get; set; }

    public void Print()
    {
        if (Success)
        {
            Console.WriteLine($"✓ 백업 성공!");
            Console.WriteLine($"  파일: {Path.GetFileName(BackupPath)}");
            Console.WriteLine($"  크기: {BackupSize / 1024.0 / 1024.0:F2} MB");
            Console.WriteLine($"  소요 시간: {Duration.TotalSeconds:F1}초");
            if (!string.IsNullOrEmpty(TableName))
            {
                Console.WriteLine($"  테이블: {TableName}");
            }
        }
        else
        {
            Console.WriteLine($"✗ 백업 실패: {ErrorMessage}");
        }
    }
}

public class RestoreResult
{
    public bool Success { get; set; }
    public TimeSpan Duration { get; set; }
    public DateTime Timestamp { get; set; }
    public string ErrorMessage { get; set; }

    public void Print()
    {
        if (Success)
        {
            Console.WriteLine($"✓ 복원 성공! (소요 시간: {Duration.TotalSeconds:F1}초)");
        }
        else
        {
            Console.WriteLine($"✗ 복원 실패: {ErrorMessage}");
        }
    }
}

public class BackupFileInfo
{
    public string FileName { get; set; }
    public string FilePath { get; set; }
    public long FileSize { get; set; }
    public DateTime CreatedDate { get; set; }
    public TimeSpan Age { get; set; }

    public override string ToString()
    {
        return $"{FileName} ({FileSize / 1024.0 / 1024.0:F2} MB) - {CreatedDate:yyyy-MM-dd HH:mm} ({Age.Days}일 전)";
    }
}
```

## 11.3 자동화된 백업 스크립트 작성

수동으로 백업하는 것은 실수하기 쉽고 귀찮다. 자동화된 백업 스케줄러를 만들어서 정해진 시간에 자동으로 백업이 실행되도록 한다.

```csharp
using System.Threading;
using Microsoft.Extensions.Hosting;
using Microsoft.Extensions.Logging;

public class BackupSchedulerService : BackgroundService
{
    private readonly ILogger<BackupSchedulerService> _logger;
    private readonly PostgresBackupManager _backupManager;
    private readonly BackupConfiguration _config;

    public BackupSchedulerService(
        ILogger<BackupSchedulerService> logger,
        PostgresBackupManager backupManager,
        BackupConfiguration config)
    {
        _logger = logger;
        _backupManager = backupManager;
        _config = config;
    }

    protected override async Task ExecuteAsync(CancellationToken stoppingToken)
    {
        _logger.LogInformation("백업 스케줄러 서비스 시작됨");

        while (!stoppingToken.IsCancellationRequested)
        {
            try
            {
                await PerformScheduledBackupsAsync();
            }
            catch (Exception ex)
            {
                _logger.LogError(ex, "백업 스케줄 실행 중 오류 발생");
            }

            // 다음 실행까지 대기 (기본 1시간)
            await Task.Delay(_config.CheckInterval, stoppingToken);
        }
    }

    private async Task PerformScheduledBackupsAsync()
    {
        var now = DateTime.Now;

        // 1. 매일 새벽 3시 전체 백업
        if (_config.DailyFullBackup && 
            now.Hour == 3 && now.Minute < 10 && 
            !HasBackupTodayAsync("daily"))
        {
            _logger.LogInformation("일일 전체 백업 시작");
            var result = await _backupManager.CreateFullBackupAsync($"daily_backup_{now:yyyyMMdd}.dump");
            
            if (result.Success)
            {
                _logger.LogInformation($"일일 백업 완료: {result.BackupSize / 1024.0 / 1024.0:F2} MB");
                
                // 클라우드 업로드
                if (_config.UploadToCloud)
                {
                    await UploadToCloudAsync(result.BackupPath);
                }
            }
            else
            {
                _logger.LogError($"일일 백업 실패: {result.ErrorMessage}");
                await SendAlertAsync("일일 백업 실패", result.ErrorMessage);
            }
        }

        // 2. 매시간 증분 백업 (WAL 아카이빙)

            _logger.LogInformation("시간별 백업 체크");
            await PerformIncrementalBackupAsync();
        }

        // 3. 오래된 백업 정리
        if (now.Hour == 4 && now.Minute < 10)
        {
            var deleted = _backupManager.CleanupOldBackups(_config.RetentionPeriod);
            _logger.LogInformation($"{deleted}개의 오래된 백업 파일 삭제됨");
        }
    }

    private bool HasBackupTodayAsync(string prefix)
    {
        var today = DateTime.Today;
        var backups = _backupManager.GetBackupFiles();
        return backups.Any(b => 
            b.FileName.StartsWith(prefix) && 
            b.CreatedDate.Date == today);
    }
```

## 11.4 Point-In-Time Recovery

pg_dump는 특정 시점의 스냅샷만 제공한다. 하지만 "오후 2시 23분에 실수로 데이터를 삭제했는데, 2시 20분 상태로 돌아가고 싶다"는 요구사항이 있다면 어떻게 할까? 이때 필요한 것이 PITR(Point-In-Time Recovery)이다.

**WAL (Write-Ahead Log) 아카이빙**

PostgreSQL은 모든 변경 사항을 WAL 파일에 기록한다. 이 WAL 파일을 보관하면 특정 시점으로 정확히 복구할 수 있다.

```
데이터 변경 흐름:

┌──────────────┐
│  변경 요청   │
└──────┬───────┘
       ↓
┌──────────────────────────────┐
│  1. WAL 파일에 먼저 기록     │  ← 여기가 핵심!
└──────┬───────────────────────┘
       ↓
┌──────────────────────────────┐
│  2. 메모리에 변경 적용       │
└──────┬───────────────────────┘
       ↓
┌──────────────────────────────┐
│  3. 나중에 디스크에 씀       │
└──────────────────────────────┘

WAL 아카이빙:
  WAL 파일 → 안전한 저장소에 복사
  
PITR 복구:
  베이스 백업 + WAL 파일 재생 → 특정 시점 복구
```

**WAL 아카이빙 설정**

Docker로 실행 중인 TimescaleDB에서 WAL 아카이빙을 활성화한다.

```bash
# Docker 컨테이너 내부 접속
docker exec -it timescaledb bash

# postgresql.conf 편집
vi /var/lib/postgresql/data/postgresql.conf

# 다음 설정 추가:
wal_level = replica
archive_mode = on
archive_command = 'test ! -f /backup/wal_archive/%f && cp %p /backup/wal_archive/%f'
archive_timeout = 300  # 5분마다 WAL 전환
```

```csharp
public class WALArchivingManager
{
    private readonly string _dockerContainerName;
    private readonly string _walArchiveDirectory;

    public WALArchivingManager(
        string dockerContainerName = "timescaledb",
        string walArchiveDirectory = @"D:\GameBackups\WAL")
    {
        _dockerContainerName = dockerContainerName;
        _walArchiveDirectory = walArchiveDirectory;

        Directory.CreateDirectory(_walArchiveDirectory);
    }

    // WAL 아카이빙 활성화
    public async Task EnableWALArchivingAsync()
    {
        Console.WriteLine("WAL 아카이빙 설정 중...");

        // Docker 볼륨 마운트 확인
        await EnsureWALDirectoryAsync();

        // PostgreSQL 설정 변경
        var commands = new[]
        {
            "ALTER SYSTEM SET wal_level = replica;",
            "ALTER SYSTEM SET archive_mode = on;",
            $"ALTER SYSTEM SET archive_command = 'test ! -f /backup/wal_archive/%f && cp %p /backup/wal_archive/%f';",
            "ALTER SYSTEM SET archive_timeout = 300;",  // 5분
            "SELECT pg_reload_conf();"  // 설정 다시 로드
        };

        foreach (var cmd in commands)
        {
            await ExecutePostgresCommandAsync(cmd);
        }

        Console.WriteLine("✓ WAL 아카이빙 활성화 완료");
        Console.WriteLine("⚠ PostgreSQL 재시작 필요: docker restart timescaledb");
    }

    // 베이스 백업 생성 (PITR의 시작점)
    public async Task<BackupResult> CreateBaseBackupAsync()
    {
        Console.WriteLine("베이스 백업 생성 중...");
        var stopwatch = Stopwatch.StartNew();

        try
        {
            var backupName = $"base_backup_{DateTime.Now:yyyyMMdd_HHmmss}";
            var backupPath = Path.Combine(_walArchiveDirectory, backupName);

            // pg_basebackup 실행
            var processInfo = new ProcessStartInfo
            {
                FileName = "docker",
                Arguments = $"exec {_dockerContainerName} pg_basebackup -U postgres -D /backup/{backupName} -Ft -z -P",
                RedirectStandardOutput = true,
                RedirectStandardError = true,
                UseShellExecute = false,
                CreateNoWindow = true
            };

            using var process = Process.Start(processInfo);
            
            var output = await process.StandardOutput.ReadToEndAsync();
            var error = await process.StandardError.ReadToEndAsync();
            
            await process.WaitForExitAsync();
            stopwatch.Stop();

            if (process.ExitCode != 0)
            {
                throw new Exception($"베이스 백업 실패: {error}");
            }

            Console.WriteLine(output);

            // Docker 컨테이너에서 로컬로 복사
            await CopyFromDockerAsync($"/backup/{backupName}", backupPath);

            var dirInfo = new DirectoryInfo(backupPath);
            var totalSize = dirInfo.GetFiles("*", SearchOption.AllDirectories).Sum(f => f.Length);

            return new BackupResult
            {
                Success = true,
                BackupPath = backupPath,
                BackupSize = totalSize,
                Duration = stopwatch.Elapsed,
                Timestamp = DateTime.Now
            };
        }
        catch (Exception ex)
        {
            stopwatch.Stop();
            return new BackupResult
            {
                Success = false,
                ErrorMessage = ex.Message,
                Duration = stopwatch.Elapsed,
                Timestamp = DateTime.Now
            };
        }
    }

    // 특정 시점으로 복구
    public async Task<RestoreResult> RestoreToPointInTimeAsync(
        string baseBackupPath,
        DateTime targetTime)
    {
        Console.WriteLine($"PITR 복구 시작: {targetTime:yyyy-MM-dd HH:mm:ss}");
        var stopwatch = Stopwatch.StartNew();

        try
        {
            // 1. 현재 데이터베이스 중지
            Console.WriteLine("1. 데이터베이스 중지 중...");
            await StopDatabaseAsync();

            // 2. 기존 데이터 백업 (안전장치)
            Console.WriteLine("2. 현재 데이터 백업 중...");
            await BackupCurrentDataAsync();

            // 3. 베이스 백업 복원
            Console.WriteLine("3. 베이스 백업 복원 중...");
            await RestoreBaseBackupAsync(baseBackupPath);

            // 4. recovery.conf 설정
            Console.WriteLine("4. 복구 설정 중...");
            await CreateRecoveryConfAsync(targetTime);

            // 5. 데이터베이스 시작 (자동으로 WAL 재생)
            Console.WriteLine("5. 데이터베이스 시작 및 복구 중...");
            await StartDatabaseAsync();

            // 6. 복구 완료 대기
            await WaitForRecoveryCompleteAsync();

            stopwatch.Stop();

            Console.WriteLine($"✓ PITR 복구 완료! (소요 시간: {stopwatch.Elapsed.TotalMinutes:F1}분)");

            return new RestoreResult
            {
                Success = true,
                Duration = stopwatch.Elapsed,
                Timestamp = DateTime.Now
            };
        }
        catch (Exception ex)
        {
            stopwatch.Stop();
            Console.WriteLine($"✗ PITR 복구 실패: {ex.Message}");

            return new RestoreResult
            {
                Success = false,
                ErrorMessage = ex.Message,
                Duration = stopwatch.Elapsed,
                Timestamp = DateTime.Now
            };
        }
    }

    // WAL 파일 통계
    public WALStatistics GetWALStatistics()
    {
        var walDir = new DirectoryInfo(_walArchiveDirectory);
        if (!walDir.Exists)
        {
            return new WALStatistics();
        }

        var walFiles = walDir.GetFiles("*", SearchOption.AllDirectories)
            .Where(f => f.Extension != ".backup")
            .OrderBy(f => f.CreationTime)
            .ToList();

        if (!walFiles.Any())
        {
            return new WALStatistics();
        }

        var totalSize = walFiles.Sum(f => f.Length);
        var oldestFile = walFiles.First();
        var newestFile = walFiles.Last();

        return new WALStatistics
        {
            TotalWALFiles = walFiles.Count,
            TotalSize = totalSize,
            OldestWALTime = oldestFile.CreationTime,
            NewestWALTime = newestFile.CreationTime,
            CoverageSpan = newestFile.CreationTime - oldestFile.CreationTime
        };
    }
```

## 11.6 클라우드 백업 연동

로컬 백업만으로는 부족하다. 화재, 홍수 같은 물리적 재난에 대비하려면 클라우드에도 백업을 보관해야 한다.

**AWS S3 백업 연동**

```csharp
using Amazon.S3;
using Amazon.S3.Transfer;
using System.IO;

public class CloudBackupManager
{
    private readonly IAmazonS3 _s3Client;
    private readonly string _bucketName;
    private readonly PostgresBackupManager _backupManager;

    public CloudBackupManager(
        string awsAccessKey,
        string awsSecretKey,
        string bucketName,
        PostgresBackupManager backupManager)
    {
        _s3Client = new AmazonS3Client(
            awsAccessKey,
            awsSecretKey,
            Amazon.RegionEndpoint.APNortheast2 // 서울 리전
        );
        _bucketName = bucketName;
        _backupManager = backupManager;
    }

    // S3에 백업 업로드
    public async Task<bool> UploadBackupToS3Async(string localFilePath)
    {
        try
        {
            var fileInfo = new FileInfo(localFilePath);
            var fileName = fileInfo.Name;
            var s3Key = $"backups/{DateTime.Now:yyyy/MM/dd}/{fileName}";

            Console.WriteLine($"S3 업로드 시작: {fileName}");
            var stopwatch = Stopwatch.StartNew();

            var fileTransferUtility = new TransferUtility(_s3Client);
            
            await fileTransferUtility.UploadAsync(new TransferUtilityUploadRequest
            {
                BucketName = _bucketName,
                FilePath = localFilePath,
                Key = s3Key,
                StorageClass = S3StorageClass.StandardInfrequentAccess, // 비용 절감
                ServerSideEncryptionMethod = ServerSideEncryptionMethod.AES256 // 암호화
            });

            stopwatch.Stop();

            Console.WriteLine($"✓ S3 업로드 완료!");
            Console.WriteLine($"  파일: {fileName}");
            Console.WriteLine($"  크기: {fileInfo.Length / 1024.0 / 1024.0:F2} MB");
            Console.WriteLine($"  S3 Key: {s3Key}");
            Console.WriteLine($"  소요 시간: {stopwatch.Elapsed.TotalSeconds:F1}초");
            Console.WriteLine($"  속도: {fileInfo.Length / 1024.0 / 1024.0 / stopwatch.Elapsed.TotalSeconds:F2} MB/s");

            return true;
        }
        catch (Exception ex)
        {
            Console.WriteLine($"✗ S3 업로드 실패: {ex.Message}");
            return false;
        }
    }

    // S3에서 백업 다운로드
    public async Task<string> DownloadBackupFromS3Async(string s3Key, string localDirectory)
    {
        try
        {
            var fileName = Path.GetFileName(s3Key);
            var localPath = Path.Combine(localDirectory, fileName);

            Console.WriteLine($"S3 다운로드 시작: {s3Key}");
            var stopwatch = Stopwatch.StartNew();

            var fileTransferUtility = new TransferUtility(_s3Client);
            
            await fileTransferUtility.DownloadAsync(localPath, _bucketName, s3Key);

            stopwatch.Stop();

            var fileInfo = new FileInfo(localPath);
            Console.WriteLine($"✓ S3 다운로드 완료!");
            Console.WriteLine($"  파일: {fileName}");
            Console.WriteLine($"  크기: {fileInfo.Length / 1024.0 / 1024.0:F2} MB");
            Console.WriteLine($"  소요 시간: {stopwatch.Elapsed.TotalSeconds:F1}초");

            return localPath;
        }
        catch (Exception ex)
        {
            Console.WriteLine($"✗ S3 다운로드 실패: {ex.Message}");
            return null;
        }
    }

    // S3 백업 목록 조회
    public async Task<List<S3BackupInfo>> ListS3BackupsAsync(string prefix = "backups/")
    {
        var backups = new List<S3BackupInfo>();

        try
        {
            var request = new Amazon.S3.Model.ListObjectsV2Request
            {
                BucketName = _bucketName,
                Prefix = prefix
            };

            var response = await _s3Client.ListObjectsV2Async(request);

            foreach (var obj in response.S3Objects.OrderByDescending(o => o.LastModified))
            {
                backups.Add(new S3BackupInfo
                {
                    Key = obj.Key,
                    Size = obj.Size,
                    LastModified = obj.LastModified,
                    StorageClass = obj.StorageClass
                });
            }
        }
        catch (Exception ex)
        {
            Console.WriteLine($"S3 목록 조회 실패: {ex.Message}");
        }

        return backups;
    }

    // 자동 클라우드 백업 (백업 생성 후 자동 업로드)
    public async Task<bool> CreateAndUploadBackupAsync()
    {
        Console.WriteLine("=== 자동 클라우드 백업 시작 ===\n");

        // 1. 로컬 백업 생성
        var backupResult = await _backupManager.CreateFullBackupAsync();
        backupResult.Print();

        if (!backupResult.Success)
        {
            return false;
        }

        Console.WriteLine();

        // 2. S3 업로드
        var uploadSuccess = await UploadBackupToS3Async(backupResult.BackupPath);

        if (uploadSuccess)
        {
            Console.WriteLine("\n✓ 클라우드 백업 완료!");
            
            // 3. 로컬 파일 삭제 (선택사항)
            // File.Delete(backupResult.BackupPath);
            // Console.WriteLine("로컬 백업 파일 삭제됨 (공간 절약)");
        }

        return uploadSuccess;
    }

    // S3 백업을 로컬로 복원
    public async Task<RestoreResult> RestoreFromS3BackupAsync(string s3Key)
    {
        Console.WriteLine($"S3 백업에서 복원 시작: {s3Key}");

        // 1. S3에서 다운로드
        var localPath = await DownloadBackupFromS3Async(
            s3Key,
            _backupManager.GetBackupDirectory()
        );

        if (localPath == null)
        {
            return new RestoreResult
            {
                Success = false,
                ErrorMessage = "S3 다운로드 실패"
            };
        }

        // 2. 로컬 백업 복원
        return await _backupManager.RestoreBackupAsync(localPath);
    }
}

public class S3BackupInfo
{
    public string Key { get; set; }
    public long Size { get; set; }
    public DateTime LastModified { get; set; }
    public string StorageClass { get; set; }

    public override string ToString()
    {
        return $"{Path.GetFileName(Key)} ({Size / 1024.0 / 1024.0:F2} MB) - {LastModified:yyyy-MM-dd HH:mm}";
    }
}
```

```csharp
public class ComprehensiveBackupSolution

        Console.WriteLine($"   클라우드 백업: {s3Backups.Count}개");
        
        if (localBackups.Any())
        {
            var newestLocal = localBackups.OrderByDescending(b => b.CreatedDate).First();

        }

        if (s3Backups.Any())
        {
            var newestS3 = s3Backups.OrderByDescending(b => b.LastModified).First();
            Console.WriteLine($"   최신 클라우드 백업: {newestS3.LastModified:yyyy-MM-dd HH:mm}");
        }
    }
```

**핵심 요약**

이번 챕터에서는 라이브 게임 서버를 위한 완벽한 백업과 복구 전략을 배웠다. 백업의 3-2-1 규칙을 따라 데이터를 안전하게 보호하고, pg_dump로 정기적인 전체 백업을 생성하며, WAL 아카이빙으로 Point-In-Time Recovery를 구현하는 방법을 다뤘다. C#으로 자동화된 백업 스케줄러를 만들어 매일 새벽 자동으로 백업이 실행되도록 설정했다. AWS S3 같은 클라우드 스토리지에 백업을 저장하여 물리적 재난에도 대비할 수 있다. 다양한 재난 시나리오를 시뮬레이션하고 복구하는 훈련을 통해 실제 장애 상황에서도 침착하게 대응할 수 있다. 제대로 된 백업 전략은 게임 서비스의 마지막 보루이며, 언젠가 반드시 그 가치를 증명하는 날이 온다.

다음 챕터에서는 실시간으로 게임 서버의 이상 징후를 감지하고 자동으로 알림을 보내는 시스템을 구축한다.
