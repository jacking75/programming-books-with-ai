## Day 1-2: 소유권(Ownership) 개념

### 🎯 **C++ 개발자가 알아야 할 핵심**
C++에서 10년간 개발하면서 다음과 같은 메모리 관련 버그들을 경험해보았을 것이다:

```cpp
// C++에서 흔히 발생하는 메모리 문제들
void dangerous_cpp_code() {
    std::string* ptr = new std::string("Hello");
    std::string* ptr2 = ptr;  // 얕은 복사
    
    delete ptr;               // ptr 해제
    std::cout << *ptr2;       // 💥 Use after free!
}

void dangling_reference() {
    std::string& get_reference() {
        std::string local = "temporary";
        return local;         // 💥 Dangling reference!
    }
}

void double_free_problem() {
    std::string* ptr = new std::string("Hello");
    delete ptr;
    delete ptr;               // 💥 Double free!
}
```

Rust의 소유권 시스템은 이러한 모든 문제를 **컴파일 타임**에 방지한다.

  
### 🔍 **소유권의 3가지 규칙**
Rust의 소유권 시스템은 다음 3가지 규칙을 기반으로 한다:

1. **각 값은 하나의 소유자(owner)를 가진다**
2. **한 번에 하나의 소유자만 존재할 수 있다**
3. **소유자가 스코프를 벗어나면 값이 자동으로 해제된다**

### 💡 **C++과 Rust 비교: 기본 소유권**

```rust
// Rust: 소유권 이전 (Move)
fn rust_ownership_example() {
    let s1 = String::from("hello");    // s1이 "hello" 소유
    let s2 = s1;                       // 소유권이 s1에서 s2로 이동
    
    // println!("{}", s1);             // 💥 컴파일 에러! s1은 더 이상 유효하지 않음
    println!("{}", s2);                // ✅ OK: s2가 소유권을 가짐
}
```

```cpp
// C++: 비교 예제
void cpp_comparison() {
    // Case 1: 기본 포인터 (위험)
    std::string* s1 = new std::string("hello");
    std::string* s2 = s1;              // 두 포인터가 같은 객체를 가리킴
    delete s1;                         // s1 해제
    // std::cout << *s2;               // 💥 런타임 에러!
    
    // Case 2: unique_ptr (안전하지만 명시적)
    std::unique_ptr<std::string> u1 = std::make_unique<std::string>("hello");
    std::unique_ptr<std::string> u2 = std::move(u1);  // 명시적 이동
    
    // std::cout << *u1;               // 💥 런타임 에러 (u1은 nullptr)
    std::cout << *u2;                  // ✅ OK
}
```

```rust
}
```

## Day 3-4: 빌림(Borrowing) & 참조

### 🎯 **C++ 참조/포인터 vs Rust 참조**
C++에서는 참조와 포인터를 다음과 같이 사용한다:

```cpp
// C++ 참조와 포인터
void cpp_references() {
    std::string data = "Hello World";
    
    // 참조 (Reference)
    std::string& ref = data;            // 별명(alias)
    ref += " C++";                      // 원본 수정 가능
    
    // 포인터 (Pointer)
    std::string* ptr = &data;           // 주소 저장
    *ptr += " Pointer";                 // 역참조 후 수정
    
    // const 참조
    const std::string& const_ref = data;
    // const_ref += "x";                // 컴파일 에러
    
    std::cout << data;                  // "Hello World C++ Pointer"
}
```

Rust는 이를 더 안전하게 만듭니다:

```rust
fn rust_borrowing() {
    let mut data = String::from("Hello World");
    
    // 불변 참조 (Immutable Reference)
    let ref1 = &data;                   // 읽기만 가능
    let ref2 = &data;                   // 여러 개의 불변 참조 가능
    
    println!("{} and {}", ref1, ref2);  // ✅ OK
    
    // 가변 참조 (Mutable Reference)
    let ref_mut = &mut data;            // 하나만 허용
    ref_mut.push_str(" Rust");
    
    // println!("{}", ref1);             // 💥 컴파일 에러!
    println!("{}", ref_mut);            // ✅ OK: "Hello World Rust"
}
```

### 🔒 **빌림 규칙 (Borrowing Rules)**
Rust의 빌림 시스템은 다음 규칙을 따른다:

1. **동시에 여러 개의 불변 참조 가능**
2. **가변 참조는 단 하나만 허용**
3. **불변 참조와 가변 참조는 동시에 존재할 수 없음**
4. **참조의 생명주기는 참조 대상보다 짧아야 함**

### 💡 **ChatServer에서의 참조 활용**
ChatServer의 UserManager에서 참조가 어떻게 사용되는지 살펴보겠다:

```rust
// ChatServer UserManager의 참조 활용 예제
use std::collections::HashMap;

pub struct UserManager {
    users: HashMap<String, User>,
}

impl UserManager {
    // 불변 참조 반환 - 읽기 전용 접근
    pub fn get_user(&self, user_id: &str) -> Option<&User> {
        self.users.get(user_id)
    }
    
    // 가변 참조 반환 - 수정 가능
    pub fn get_user_mut(&mut self, user_id: &str) -> Option<&mut User> {
        self.users.get_mut(user_id)
    }
    
    // 소유권 이전 없이 함수에 전달
    pub fn validate_user(&self, user: &User) -> bool {
        user.is_confirmed && !user.id.is_empty()
    }
    
    // 가변 참조로 사용자 수정
    pub fn confirm_user(&mut self, user_id: &str) -> Result<(), String> {
        match self.get_user_mut(user_id) {
            Some(user) => {
                user.is_confirmed = true;   // 원본 수정
                Ok(())
            }
            None => Err("User not found".to_string())
        }
    }
}
```

### 🎪 **실습: C++ vs Rust 참조 비교**
동일한 기능을 C++과 Rust로 구현해보겠다:

```cpp
// C++ 버전: 위험성 내포
class UserManagerCpp {
private:
    std::unordered_map<std::string, User> users;
    
public:
    // 포인터 반환 - 위험할 수 있음
    User* getUser(const std::string& userId) {
        auto it = users.find(userId);
        return it != users.end() ? &it->second : nullptr;
    }
    
    // const 참조 반환
    const User* getUserConst(const std::string& userId) const {
        auto it = users.find(userId);
        return it != users.end() ? &it->second : nullptr;
    }
    
    void dangerousExample() {
        User* user = getUser("player1");
        users.clear();                  // 💥 user 포인터가 무효화됨!
        
        if (user) {
            std::cout << user->id;      // 💥 Undefined Behavior!
        }
    }
};
```

```rust
// Rust 버전: 컴파일 타임 안전성 보장
```

### 📝 **생명주기(Lifetimes) 기초**

### 🔥 **열거형의 강력함: C++ vs Rust**
C++의 enum과 Rust의 enum은 완전히 다른 수준이다:

```cpp
// C++ enum class (제한적)
enum class GameState {
    None,
    Login,
    Lobby,
    Room
};

class User {
    GameState state;
    std::string lobbyId;    // 별도 필드 필요
    std::string roomId;     // 별도 필드 필요
    
public:
    bool isInLobby() const {
        return state == GameState::Lobby && !lobbyId.empty();
    }
};
```

```rust
// Rust enum (데이터를 포함할 수 있음!)
#[derive(Debug, Clone, PartialEq)]
pub enum UserState {
    None,
    Login,
    Lobby(String),              // 로비 ID를 직접 포함
    Room { lobby: String, room: String },  // 여러 데이터 포함 가능
}

impl User {
    pub fn is_in_lobby(&self) -> bool {
        matches!(self.state, UserState::Lobby(_))
    }
    
    pub fn get_lobby_id(&self) -> Option<&String> {
        match &self.state {
            UserState::Lobby(id) => Some(id),
            UserState::Room { lobby, .. } => Some(lobby),
            _ => None,
        }
    }
}
```

### 🎪 **ChatServer의 패킷 시스템 분석**
ChatServer에서 enum과 패턴 매칭이 어떻게 활용되는지 살펴보겠다:

```rust
// ChatServer의 패킷 시스템
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum PacketId {
    // 시스템 패킷
    NtfSysConnectSession = 2,
    NtfSysCloseSession = 3,
    
    // 로그인 패킷
    LoginInReq = 21,
    LoginInRes = 22,
    
    // 로비 패킷
    LobbyListReq = 26,
    LobbyListRes = 27,
    LobbyEnterReq = 31,
    LobbyEnterRes = 32,
    
    // 방 패킷
    RoomEnterReq = 61,
    RoomEnterRes = 62,
}

// 패킷 처리 - 패턴 매칭의 위력
impl ChatServer {
    async fn process_packet(&self, session_id: Uuid, packet: Packet) -> Result<(), ServerError> {
        match packet.header.id {
            PacketId::NtfSysConnectSession => {
                info!("New session connected: {}", session_id);
                Ok(())
            }
            
            PacketId::LoginInReq => {
                self.handle_login(session_id, packet.body).await
            }
            
            PacketId::LobbyEnterReq => {
                // 사용자 상태 확인
                let user = self.user_manager.get_user(session_id).await?;
                let user_guard = user.read().await;
                
                match &user_guard.as_ref().unwrap().state {
                    UserState::Login => {
                        // 로비 입장 처리
                        self.handle_lobby_enter(session_id, packet.body).await
                    }
                    UserState::Lobby(_) => {
                        Err(ServerError::LobbyEnterInvalidDomain)
                    }
                    _ => Err(ServerError::LobbyEnterInvalidDomain)
                }
            }
            
            // 컴파일러가 모든 경우를 확인함!
            _ => {
                warn!("Unhandled packet: {:?}", packet.header.id);
                Ok(())
            }
        }
    }
}
```

```rust
#[derive(Debug, PartialEq)]
pub enum PacketType {
    Login { username: String, password: String },
    Logout,
    ChatMessage { message: String },
    JoinRoom { room_id: u32 },
    LeaveRoom,
}

#[derive(Debug)]
pub struct Packet {
    pub id: u32,
    pub packet_type: PacketType,
}

#[derive(Debug, PartialEq)]
pub enum ProcessError {
    InvalidCredentials,
    NotLoggedIn,
    RoomNotFound,
    EmptyMessage,
}

pub struct PacketProcessor {
    logged_in: bool,
    current_room: Option<u32>,
}

impl PacketProcessor {
    pub fn new() -> Self {
        Self {
            logged_in: false,
            current_room: None,
        }
    }
    
    pub fn process_packet(&mut self, packet: Packet) -> Result<String, ProcessError> {
        match packet.packet_type {
            PacketType::Login { username, password } => {
                if username == "admin" && password == "password" {
                    self.logged_in = true;
                    Ok(format!("Welcome, {}!", username))
                } else {
                    Err(ProcessError::InvalidCredentials)
                }
            }
            
            PacketType::Logout => {
                if !self.logged_in {
                    return Err(ProcessError::NotLoggedIn);
                }
                self.logged_in = false;
                self.current_room = None;
                Ok("Logged out successfully".to_string())
            }
            
            PacketType::ChatMessage { message } => {
                if !self.logged_in {
                    return Err(ProcessError::NotLoggedIn);
                }
                if message.trim().is_empty() {
                    return Err(ProcessError::EmptyMessage);
                }
                Ok(format!("Message sent: {}", message))
            }
            
            PacketType::JoinRoom { room_id } => {
                if !self.logged_in {
                    return Err(ProcessError::NotLoggedIn);
                }
                if room_id > 100 {
                    return Err(ProcessError::RoomNotFound);
                }
                self.current_room = Some(room_id);
                Ok(format!("Joined room {}", room_id))
            }
            
            PacketType::LeaveRoom => {
                match self.current_room {
                    Some(room_id) => {
                        self.current_room = None;
                        Ok(format!("Left room {}", room_id))
                    }
                    None => Ok("Not in any room".to_string())
                }
            }
        }
    }
}
```
