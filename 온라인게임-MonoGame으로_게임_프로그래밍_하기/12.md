# 12장. 벽돌깨기 게임 만들기 (1/3)

## 12.1 게임 기획
벽돌깨기는 클래식 아케이드 게임으로, 게임 프로그래밍의 핵심 개념들을 모두 포함하고 있다. 이 장에서는 플레이어가 패들을 조작하여 공을 튕겨내고, 공이 벽돌에 부딪혀 벽돌을 깨뜨리는 게임을 만들 것이다. 먼저 게임의 요구사항과 설계를 정리해보자.

### 게임의 핵심 요소
벽돌깨기 게임은 다음과 같은 요소들로 구성된다.

**게임 오브젝트:** 플레이어가 조작하는 패들(paddle), 화면을 움직이는 공(ball), 그리고 파괴 대상인 벽돌(brick)이 필요하다. 각 벽돌은 충돌 시 제거되며, 공은 게임 공간의 벽과 바닥에 부딪혀 반사된다.

**게임 로직:** 공이 화면 하단(바닥)에 도달하면 플레이어가 생명을 잃게 된다. 패들로 공을 받지 못하면 게임 오버가 되고, 모든 벽돌을 깨뜨리면 게임을 클리어한다.

**제어:** 플레이어는 키보드의 좌우 화살표 키나 마우스 움직임으로 패들을 조작한다.

### 게임 디자인 문서

다음은 이 게임의 기본 설계를 다이어그램으로 나타낸 것이다.

```
┌─────────────────────────────────────┐
│      게임 윈도우 (1280x720)           │
├─────────────────────────────────────┤
│                                     │
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ...        │
│  │ B │ │ B │ │ B │ │ B │ (벽돌들 )   │
│  └───┘ └───┘ └───┘ └───┘            │
│                                     │
│                                     │
│                ◯                   │  (공)
│                                     │
│  ═══════════════════════════════    │  (패들)
│                                     │
└─────────────────────────────────────┘
```

게임의 해상도는 1280x720 픽셀로 설정하고, 벽돌은 여러 줄로 배치한다. 공은 초기에 패들 위에 위치하며, 게임 시작 시 패들에서 떨어져 움직이기 시작한다.

## 12.2 플레이어 패들 구현
패들은 플레이어가 직접 조작하는 게임 오브젝트로, 공을 반사시키는 핵심 요소다. 이전 장에서 배운 GameObject 클래스를 상속하여 패들을 구현해보자.

### 패들 클래스 설계
패들은 수평 이동만 하며, 게임 화면 하단에 고정된다. 패들의 위치는 중심 좌표로 관리하고, 너비와 높이를 가진 직사각형으로 표현한다.

```csharp
public class Paddle : GameObject
{
    public float Width { get; set; }
    public float Height { get; set; }
    private float _maxSpeed;
    private float _acceleration;

    public Paddle(Vector2 position, float width, float height) 
        : base(position)
    {
        Width = width;
        Height = height;
        _maxSpeed = 500f;
        _acceleration = 2000f;
        Velocity = Vector2.Zero;
    }

    public override void Update(GameTime gameTime)
    {
        float deltaTime = (float)gameTime.ElapsedGameTime.TotalSeconds;
        KeyboardState keyboardState = Keyboard.GetState();

        // 입력 처리
        if (keyboardState.IsKeyDown(Keys.Left))
        {
            Velocity.X = -_maxSpeed;
        }
        else if (keyboardState.IsKeyDown(Keys.Right))
        {
            Velocity.X = _maxSpeed;
        }
        else
        {
            Velocity.X = 0;
        }

        // 위치 업데이트
        Position += Velocity * deltaTime;

        // 경계 제한
        Position.X = MathHelper.Clamp(
            Position.X, 
            Width / 2, 
            1280 - Width / 2
        );
    }

    public override void Draw(SpriteBatch spriteBatch)
    {
        // 패들을 흰색 직사각형으로 그린다
        Rectangle paddleRect = new Rectangle(
            (int)(Position.X - Width / 2),
            (int)(Position.Y - Height / 2),
            (int)Width,
            (int)Height
        );
        spriteBatch.Draw(
            texture: Texture2D.FromColor(GraphicsDevice, 
                1, 1, Color.White),
            destinationRectangle: paddleRect,
            color: Color.White
        );
    }

    public Rectangle GetBounds()
    {
        return new Rectangle(
            (int)(Position.X - Width / 2),
            (int)(Position.Y - Height / 2),
            (int)Width,
            (int)Height
        );
    }
}
```

**코드 설명:**

위 코드는 패들의 기본 구조를 정의한다. `GameObject` 클래스를 상속받았으므로 위치(Position)와 속도(Velocity) 프로퍼티를 이용한다.

`Update` 메서드에서는 키보드 입력을 확인한다. 좌우 화살표 키가 눌렸을 때 속도를 설정하고, 입력이 없으면 속도를 0으로 만든다. 패들이 화면을 벗어나지 않도록 `MathHelper.Clamp`를 사용하여 X 위치를 제한한다.

`Draw` 메서드는 패들을 흰색 직사각형으로 렌더링한다. 실제 게임 개발에서는 텍스처를 사용하겠지만, 여기서는 간단하게 색깔로 표현한다.

`GetBounds` 메서드는 패들의 현재 충돌 범위를 반환한다. 이 메서드는 나중에 충돌 감지 시스템에서 사용될 것이다.

### 패들 초기화 및 사용
게임의 `LoadContent` 메서드에서 패들을 생성한다.

```csharp
private Paddle paddle;

protected override void LoadContent()
{
    _spriteBatch = new SpriteBatch(GraphicsDevice);
    
    // 패들을 화면 하단 중앙에 생성
    paddle = new Paddle(
        position: new Vector2(640, 650),
        width: 120,
        height: 20
    );
}
```

게임의 `Update` 메서드에서는 패들의 `Update` 메서드를 호출한다.

```csharp
protected override void Update(GameTime gameTime)
{
    if (GamePad.GetState(PlayerIndex.One).Buttons.Back == ButtonState.Pressed 
        || Keyboard.GetState().IsKeyDown(Keys.Escape))
        Exit();

    paddle.Update(gameTime);

    base.Update(gameTime);
}
```

`Draw` 메서드에서는 패들을 렌더링한다.

```csharp
protected override void Draw(GameTime gameTime)
{
    GraphicsDevice.Clear(Color.Black);

    _spriteBatch.Begin();
    paddle.Draw(_spriteBatch);
    _spriteBatch.End();

    base.Draw(gameTime);
}
```

## 12.3 공 물리 구현
공은 게임의 중심 요소로, 선형 운동과 벽면 반사를 구현해야 한다. 공의 물리 시뮬레이션은 게임의 재미에 큰 영향을 미친다.

### 공 클래스 설계
공은 원형 오브젝트로 표현되며, 속도 벡터에 따라 움직인다. 벽이나 패들에 부딪혔을 때 반사 각도를 계산해야 한다.

```csharp
public class Ball : GameObject
{
    public float Radius { get; set; }
    private float _speed;
    private bool _isLaunched;
    private Paddle _paddle;

    public Ball(Vector2 position, float radius, Paddle paddle) 
        : base(position)
    {
        Radius = radius;
        _speed = 350f;
        _isLaunched = false;
        _paddle = paddle;
        Velocity = Vector2.Zero;
    }

    public override void Update(GameTime gameTime)
    {
        float deltaTime = (float)gameTime.ElapsedGameTime.TotalSeconds;

        if (!_isLaunched)
        {
            // 공이 패들 위에 있으면 패들을 따라 이동
            Position = new Vector2(_paddle.Position.X, _paddle.Position.Y - 30);

            // 스페이스바로 공을 발사
            if (Keyboard.GetState().IsKeyDown(Keys.Space))
            {
                LaunchBall();
            }
        }
        else
        {
            // 공의 위치 업데이트
            Position += Velocity * deltaTime;

            // 좌우 벽 충돌 처리
            if (Position.X - Radius < 0 || Position.X + Radius > 1280)
            {
                Velocity.X *= -1;
                Position.X = MathHelper.Clamp(
                    Position.X, 
                    Radius, 
                    1280 - Radius
                );
            }

            // 상단 벽 충돌 처리
            if (Position.Y - Radius < 0)
            {
                Velocity.Y *= -1;
                Position.Y = Radius;
            }

            // 하단 벗어남 처리 (게임 오버)
            if (Position.Y - Radius > 720)
            {
                ResetBall();
            }
        }
    }

    public void LaunchBall()
    {
        _isLaunched = true;
        
        // 45도 각도로 발사
        float angle = MathHelper.PiOver4;
        Velocity = new Vector2(
            (float)Math.Cos(angle) * _speed,
            -(float)Math.Sin(angle) * _speed
        );
    }

    public void ResetBall()
    {
        _isLaunched = false;
        Velocity = Vector2.Zero;
    }

    public void ReflectOffPaddle(Paddle paddle)
    {
        // 패들과의 충돌 위치에 따라 반사각 결정
        float paddleCenter = paddle.Position.X;
        float hitOffset = Position.X - paddleCenter;
        float normalizedOffset = hitOffset / (paddle.Width / 2);
        normalizedOffset = MathHelper.Clamp(normalizedOffset, -1, 1);

        // 반사각 계산 (최대 60도까지)
        float reflectAngle = normalizedOffset * MathHelper.PiOver3;
        
        Velocity = new Vector2(
            (float)Math.Sin(reflectAngle) * _speed,
            -(float)Math.Abs(Math.Cos(reflectAngle)) * _speed
        );

        // 공이 패들 내부에 갇히지 않도록 위치 조정
        Position.Y = paddle.Position.Y - paddle.Height / 2 - Radius - 1;
    }

    public override void Draw(SpriteBatch spriteBatch)
    {
        // 공을 노란색 원으로 그린다
        DrawCircle(spriteBatch, Position, Radius, Color.Yellow);
    }

    private void DrawCircle(SpriteBatch spriteBatch, Vector2 center, 
        float radius, Color color)
    {
        // 간단한 원 그리기 (완벽한 원은 아니지만 시각적으로 충분함)
        int segments = (int)(radius * 2);
        for (int i = 0; i < segments; i++)
        {
            float angle1 = (float)i / segments * MathHelper.TwoPi;
            float angle2 = (float)(i + 1) / segments * MathHelper.TwoPi;

            Vector2 p1 = center + new Vector2(
                (float)Math.Cos(angle1) * radius,
                (float)Math.Sin(angle1) * radius
            );
            Vector2 p2 = center + new Vector2(
                (float)Math.Cos(angle2) * radius,
                (float)Math.Sin(angle2) * radius
            );

            spriteBatch.DrawLine(p1, p2, color, 2);
        }
    }

    public Rectangle GetBounds()
    {
        return new Rectangle(
            (int)(Position.X - Radius),
            (int)(Position.Y - Radius),
            (int)(Radius * 2),
            (int)(Radius * 2)
        );
    }

    public bool IsLaunched => _isLaunched;
}
```

**코드 설명:**

공 클래스는 여러 상태를 관리한다. `_isLaunched` 플래그는 공이 발사되었는지 여부를 나타낸다. 발사 전에는 공이 패들 위에서 패들을 따라 움직이고, 스페이스바로 발사한다.

`Update` 메서드는 발사 전후로 다른 로직을 실행한다. 발사 후에는 속도에 따라 위치를 업데이트하고, 벽과의 충돌을 처리한다. 좌우 벽과 상단 벽에 부딪히면 해당 방향의 속도를 반전시킨다. 하단으로 벗어나면 공을 리셋한다.

`LaunchBall` 메서드는 공을 45도 각도로 발사한다. 각도를 라디안 단위로 변환하여 속도 벡터를 계산한다.

`ReflectOffPaddle` 메서드는 공이 패들과 충돌했을 때 반사각을 계산한다. 공이 패들의 중심에서 얼마나 떨어져 있는지에 따라 반사각이 달라진다. 패들의 끝에 가까울수록 더 큰 각도로 반사되어, 플레이어가 공의 궤적을 조절할 수 있게 한다. 반사각은 최대 60도로 제한하여 게임 난이도의 균형을 맞춘다.

`DrawCircle` 메서드는 삼각함수를 이용하여 원을 그린다. 원 주위에 작은 선분들을 그려서 원형 모양을 표현한다.

### 공의 초기화 및 사용
게임의 `LoadContent` 메서드에 공 생성 코드를 추가한다.

```csharp
private Ball ball;

protected override void LoadContent()
{
    _spriteBatch = new SpriteBatch(GraphicsDevice);
    
    paddle = new Paddle(
        position: new Vector2(640, 650),
        width: 120,
        height: 20
    );
    
    // 공을 패들 위에 생성
    ball = new Ball(
        position: new Vector2(640, 620),
        radius: 8,
        paddle: paddle
    );
}
```

게임의 `Update` 메서드에 공 업데이트와 충돌 처리를 추가한다.

```csharp
protected override void Update(GameTime gameTime)
{
    if (GamePad.GetState(PlayerIndex.One).Buttons.Back == ButtonState.Pressed 
        || Keyboard.GetState().IsKeyDown(Keys.Escape))
        Exit();

    paddle.Update(gameTime);
    ball.Update(gameTime);

    // 공과 패들의 충돌 감지 (AABB 방식)
    if (ball.IsLaunched && 
        ball.GetBounds().Intersects(paddle.GetBounds()))
    {
        ball.ReflectOffPaddle(paddle);
    }

    base.Update(gameTime);
}
```

게임의 `Draw` 메서드를 수정하여 공도 렌더링한다.

```csharp
protected override void Draw(GameTime gameTime)
{
    GraphicsDevice.Clear(Color.Black);

    _spriteBatch.Begin();
    paddle.Draw(_spriteBatch);
    ball.Draw(_spriteBatch);
    _spriteBatch.End();

    base.Draw(gameTime);
}
```

### 물리 개념 설명
공의 움직임은 기본적인 선형 물리학을 따른다. 속도 벡터는 시간에 따른 위치 변화를 나타내며, 다음과 같은 공식으로 계산된다.

```
새로운 위치 = 현재 위치 + 속도 벡터 × 경과 시간
```

벽면 반사는 법선 벡터(normal vector)를 이용하여 계산한다. 수평 벽(좌우)에 충돌했을 때는 X 속도의 부호를 반전시키고, 수직 벽(상단)에 충돌했을 때는 Y 속도의 부호를 반전시킨다.

패들과의 충돌에서 중요한 것은 충돌 위치에 따른 반사각 조절이다. 이를 통해 플레이어가 공의 방향을 제어할 수 있게 된다. 패들의 중심에 맞으면 수직으로 올라가고, 끝에 맞으면 사각으로 튕겨나간다.

## 과제
이 장에서는 패들과 공의 기본 구현을 학습했다. 다음 과제들을 통해 학습 내용을 확인하고 게임을 확장해보자.

**과제 1: 공의 속도 증가 기능**

현재 공은 항상 같은 속도로 움직인다. 시간이 지날수록 공의 속도가 조금씩 증가하는 기능을 구현하자. 단, 속도가 너무 빨라지지 않도록 상한선을 설정해야 한다. `Ball` 클래스에 `_speed` 변수를 활용하여 구현하되, 초당 50픽셀씩 증가하고 최대 600픽셀/초로 제한하도록 하자.

**과제 2: 패들 크기 조절**

패들의 너비와 높이를 게임 상황에 따라 변경할 수 있는 메서드를 구현하자. 예를 들어, 특정 조건에서 패들이 작아지거나 커지는 파워업/파워다운 시스템을 생각하며 구현하면 된다. `Resize(float newWidth, float newHeight)` 메서드를 `Paddle` 클래스에 추가하자.

**과제 3: 공의 회전**

공을 렌더링할 때 속도 벡터에 따라 회전하는 효과를 추가해보자. 공이 빠르게 움직일수록 더 빠르게 회전하는 모습을 시각화하면 게임의 느낌이 더 살아난다. `Ball` 클래스에 회전 각도를 나타내는 프로퍼티를 추가하고, `Update` 메서드에서 속도에 비례하게 증가시키자. 원의 표현 방식을 변경하여 회전을 시각화할 수 있다.

**과제 4: 생명 시스템 구현**

게임에 생명 시스템을 추가하자. 공이 화면 하단으로 떨어지면 생명이 1개 감소하고, 생명이 0이 되면 게임 오버가 되도록 구현하자. 게임의 `Game` 클래스에 생명을 관리하는 변수를 추가하고, 화면에 남은 생명을 표시하자. (이전 장에서 배운 텍스트 렌더링 기능을 활용할 수 있다.)

**과제 5: 공의 회전 방향 계산**

앞서 공이 패들의 어느 위치에 맞혔는지에 따라 반사각이 결정된다고 배웠다. 현재 구현에서는 반사각만 계산하지만, 패들의 움직임을 고려한 공의 반사를 구현해보자. 만약 패들이 오른쪽으로 움직이고 있을 때 공이 충돌했다면, 공이 더 오른쪽으로 튕겨나가도록 반사각을 미세하게 조정하자. (힌트: `Paddle`의 속도 정보를 활용하자.)
