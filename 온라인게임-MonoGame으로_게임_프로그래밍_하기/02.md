# 2장. Game 클래스와 게임 루프
이전 장에서 우리는 MonoGame 개발 환경을 구축하고 첫 번째 프로젝트를 성공적으로 실행했다. 이번 장에서는 MonoGame 게임의 심장이라 할 수 있는 Game 클래스와 게임 루프(Game Loop)에 대해 깊이 있게 배워볼 것이다. 게임 루프를 이해하면, 모든 게임이 어떻게 작동하는지 명확하게 알 수 있을 것이다.


## 2.1 Game 클래스의 구조
MonoGame의 모든 게임은 Game 클래스를 상속받는 클래스에서 시작된다. 1장에서 생성한 프로젝트의 Game1 클래스도 Game을 상속받고 있다. Game 클래스가 제공하는 것들을 이해해보자.

```csharp
public class Game1 : Game
{
    // 게임 클래스를 상속받는다
}
```

Game 클래스는 다음과 같은 핵심 기능들을 제공한다.

**GraphicsDevice** - GPU와 통신하여 화면에 그리는 작업을 담당한다. 모든 렌더링 작업은 GraphicsDevice를 통해 이루어진다.

**GameTime** - 경과 시간 정보를 제공한다. 프레임 간 시간 차이(DeltaTime)를 얻을 수 있으며, 이를 통해 프레임에 관계없이 일정한 속도로 객체를 움직일 수 있다.

**Content** - 이미지, 사운드, 폰트 등 게임 에셋을 로드한다.

**IsMouseVisible** - 마우스 커서의 표시 여부를 제어한다.

**Window** - 게임 윈도우의 속성(제목, 크기, 위치 등)을 제어한다.

**TargetElapsedTime** - 프레임 시간을 설정한다. 기본값은 1/60초(60 FPS)다.

Game 클래스는 이 외에도 많은 메서드와 속성을 가지고 있지만, 게임 개발 초기에는 위의 것들이 가장 중요하다.

</br>    

## 2.2 게임 루프의 이해
게임의 가장 중요한 개념은 **게임 루프(Game Loop)**다. 게임 루프는 게임이 시작된 후 종료될 때까지 반복적으로 실행되는 프로세스다. 초당 60번 이상 반복되기도 한다.

게임 루프의 흐름을 다이어그램으로 나타내면 다음과 같다.

```
┌─────────────────────────────────────┐
│   게임 시작 (Program.cs)             │
└─────────────┬───────────────────────┘
              │
              ▼
┌─────────────────────────────────────┐
│   Game1 생성자 (Constructor)         │
│   - 게임 기본 설정                    │
└─────────────┬───────────────────────┘
              │
              ▼
┌─────────────────────────────────────┐
│   Initialize() 호출 (한 번만)         │
│   - 게임 변수 초기화                  │  
└─────────────┬───────────────────────┘
              │
              ▼
┌─────────────────────────────────────┐
│   LoadContent() 호출 (한 번만)        │
│   - 에셋 로드 (이미지, 소리 등)        │
└─────────────┬───────────────────────┘
              │
              ▼
        ┌─────────────────┐
        │  게임 루프 시작   │
        └─────────┬───────┘
                  │
        ┌─────────▼──────────┐
        │ Update() 호출       │ (매 프레임)
        │ - 게임 로직 실행     │
        │ - 입력 처리         │
        │ - 충돌 감지 등       │
        └─────────┬──────────┘
                  │
        ┌─────────▼──────────┐
        │ Draw() 호출         │ (매 프레임)
        │ - 화면 렌더링        │
        │ - 오브젝트 그리기     │
        └─────────┬──────────┘
                  │
        ┌─────────▼──────────────────┐
        │ 게임 종료 조건 확인?          │
        └─────────┬──────────────────┘
                  │
        ┌─────────▼──────────────────┐
        │ 아니오 → 루프 반복           │ (계속)
        └─────────┬──────────────────┘
                  │
        ┌─────────▼──────────────────┐
        │ 예 → UnloadContent()       │
        │      게임 루프 종료          │
        └─────────┬──────────────────┘
                  │
                  ▼
       ┌──────────────────────┐
       │   게임 종료           │
       └──────────────────────┘
```

이 흐름을 보면 게임 루프가 몇 가지 단계로 나뉨을 알 수 있다. 각 단계를 자세히 알아보자.

### Initialize()
Initialize() 메서드는 게임이 시작된 직후, **단 한 번만** 호출된다. 이 메서드에서는 게임에 필요한 모든 변수를 초기화한다.

```csharp
protected override void Initialize()
{
    // 게임에 사용할 변수들을 초기화한다
    playerX = 100;
    playerY = 100;
    playerSpeed = 5;
    
    base.Initialize();
}
```

Initialize()에서는 게임의 논리적 초기화를 담당하지만, 이미지나 사운드 같은 파일 기반의 리소스는 로드하지 않는다. 파일 로드는 다음 단계인 LoadContent()에서 수행된다.

### LoadContent()
LoadContent() 메서드는 Initialize() 이후에 **한 번만** 호출된다. 이 메서드에서는 게임에 사용될 모든 에셋(이미지, 사운드, 폰트 등)을 메모리에 로드한다.

```csharp
protected override void LoadContent()
{
    _spriteBatch = new SpriteBatch(GraphicsDevice);
    
    // 이미지 로드
    playerTexture = Content.Load<Texture2D>("player");
    
    // 사운드 로드
    jumpSound = Content.Load<SoundEffect>("sounds/jump");
    
    // 폰트 로드
    gameFont = Content.Load<SpriteFont>("fonts/arial");
}
```

에셋 로드는 시간이 걸리는 작업이므로, 게임이 시작되기 전에 LoadContent()에서 한 번에 처리하는 것이 효율적이다. 만약 게임 중간에 에셋을 로드하면 프레임이 끊길 수 있다.

### Update()
Update() 메서드는 게임이 실행되는 동안 매 프레임마다 호출된다. 기본적으로 초당 60번(60 FPS) 호출된다. 이 메서드에서는 게임의 모든 로직이 처리된다.

```csharp
protected override void Update(GameTime gameTime)
{
    // 게임 로직
    if (Keyboard.GetState().IsKeyDown(Keys.Left))
    {
        playerX -= playerSpeed;
    }
    
    // 충돌 감지
    if (playerBounds.Intersects(enemyBounds))
    {
        health--;
    }
    
    // 점수 증가
    score += deltaTime;
    
    base.Update(gameTime);
}
```

Update()에서 주의할 점은 화면에 아무것도 그리면 안 된다는 것이다. Update()는 순수하게 게임의 상태를 업데이트하는 역할만 한다. 화면 렌더링은 Draw() 메서드에서 담당한다.

Update() 메서드의 매개변수로 전달되는 GameTime 객체는 매우 중요하다. 이 객체에는 경과 시간 정보가 담겨 있으며, 다음과 같이 활용할 수 있다.

```csharp
// 경과한 총 시간
float totalSeconds = (float)gameTime.TotalGameTime.TotalSeconds;

// 이전 프레임과 현재 프레임 사이의 시간 (델타 타임)
float deltaTime = (float)gameTime.ElapsedGameTime.TotalSeconds;

// 이를 이용해 프레임에 관계없이 일정한 속도를 유지할 수 있다
playerX += playerSpeed * deltaTime;  // deltaTime를 곱하면 초 단위로 움직임
```

### Draw()
Draw() 메서드도 매 프레임마다 호출되며, 이 메서드에서 게임화면에 모든 것을 그린다. 게임의 시각적 표현은 모두 이 메서드에서 이루어진다.

```csharp
protected override void Draw(GameTime gameTime)
{
    // 화면을 배경색으로 지운다
    GraphicsDevice.Clear(Color.Black);
    
    // SpriteBatch 시작
    _spriteBatch.Begin();
    
    // 플레이어 그리기
    _spriteBatch.Draw(playerTexture, new Vector2(playerX, playerY), Color.White);
    
    // 텍스트 그리기
    _spriteBatch.DrawString(gameFont, "Score: " + score, 
        new Vector2(10, 10), Color.White);
    
    // SpriteBatch 종료
    _spriteBatch.End();
    
    base.Draw(gameTime);
}
```

Draw() 메서드는 다음과 같은 순서로 진행된다.

1. **Clear()** - 이전 프레임의 내용을 지운다. 이 작업을 하지 않으면 화면에 이전 프레임의 이미지가 겹쳐서 나타난다.

2. **Begin()** - SpriteBatch를 시작한다. 이 호출 이후부터 그리기 작업을 할 수 있다.

3. **Draw() / DrawString()** - 실제로 이미지나 텍스트를 화면에 그린다.

4. **End()** - SpriteBatch를 종료한다. 이 호출 이전까지의 모든 그리기 명령이 실제로 화면에 렌더링된다.

이 순서가 중요한 이유는 그리기 성능 때문이다. 모든 그리기를 Begin()과 End() 사이에 모아 한 번에 처리하면, 게임의 성능이 크게 향상된다.

## 2.3 간단한 창 띄우기
이제 우리가 학습한 Game 클래스의 메서드들을 직접 사용해보자. 먼저 게임 윈도우의 제목과 크기를 설정하는 간단한 예제를 만들어보겠다.

1장에서 생성한 MyFirstGame 프로젝트를 다시 열어서, Game1.cs 파일을 다음과 같이 수정해보자.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;

namespace MyFirstGame
{
    public class Game1 : Game
    {
        private GraphicsDeviceManager _graphics;
        private SpriteBatch _spriteBatch;
        
        // 게임 변수들
        private int windowWidth = 800;
        private int windowHeight = 600;
        private string gameTitle = "My First MonoGame!";

        public Game1()
        {
            _graphics = new GraphicsDeviceManager(this);
            Content.RootDirectory = "Content";
            IsMouseVisible = true;
            
            // 윈도우 크기 설정
            _graphics.PreferredBackBufferWidth = windowWidth;
            _graphics.PreferredBackBufferHeight = windowHeight;
        }

        protected override void Initialize()
        {
            // 윈도우 제목 설정
            Window.Title = gameTitle;
            
            base.Initialize();
        }

        protected override void LoadContent()
        {
            _spriteBatch = new SpriteBatch(GraphicsDevice);
        }

        protected override void Update(GameTime gameTime)
        {
            // ESC 키로 게임 종료
            if (Keyboard.GetState().IsKeyDown(Keys.Escape))
                Exit();

            base.Update(gameTime);
        }

        protected override void Draw(GameTime gameTime)
        {
            // 화면을 흰색으로 채운다
            GraphicsDevice.Clear(Color.White);

            base.Draw(gameTime);
        }
    }
}
```

**코드 설명:**

이 코드의 핵심은 GraphicsDeviceManager를 사용하여 게임 윈도우의 크기를 설정하는 것이다. PreferredBackBufferWidth와 PreferredBackBufferHeight는 게임 화면의 해상도를 의미한다. 이 값들은 생성자에서 설정되어야 하며, Initialize() 호출 이전에 적용되어야 한다.

Window.Title은 Initialize() 메서드에서 설정한다. 이 속성은 게임 윈도우의 제목 표시줄에 표시되는 텍스트를 설정한다.

이제 이 코드를 실행해보면, 다음을 확인할 수 있을 것이다.

1. 게임 윈도우의 크기가 800x600 픽셀이다.
2. 제목 표시줄에 "My First MonoGame!"이라는 텍스트가 표시된다.
3. 배경색이 흰색이다.
4. ESC 키를 누르면 게임이 종료된다.

**프로젝트 실행:**

Visual Studio의 상단에 있는 "시작" 버튼(재생 버튼)을 클릭하거나, F5 키를 눌러서 프로젝트를 실행한다. 또는 "디버그" 메뉴에서 "디버깅 시작"을 선택한다.

## 2.4 배경색 변경하기
MonoGame 게임의 배경색을 동적으로 변경하는 것은 매우 간단하다. 이 예제를 통해 Update()와 Draw() 메서드 간의 관계를 더 잘 이해할 수 있을 것이다.

다음 코드는 시간에 따라 배경색이 변하는 게임을 구현한다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;

namespace MyFirstGame
{
    public class Game1 : Game
    {
        private GraphicsDeviceManager _graphics;
        private SpriteBatch _spriteBatch;
        
        private float elapsedTime = 0;
        private Color currentColor = Color.White;
        private float colorChangeInterval = 2.0f;  // 2초마다 색상 변경

        public Game1()
        {
            _graphics = new GraphicsDeviceManager(this);
            Content.RootDirectory = "Content";
            IsMouseVisible = true;
            
            _graphics.PreferredBackBufferWidth = 800;
            _graphics.PreferredBackBufferHeight = 600;
        }

        protected override void Initialize()
        {
            Window.Title = "Color Changing Game";
            base.Initialize();
        }

        protected override void LoadContent()
        {
            _spriteBatch = new SpriteBatch(GraphicsDevice);
        }

        protected override void Update(GameTime gameTime)
        {
            // 경과 시간을 누적한다
            elapsedTime += (float)gameTime.ElapsedGameTime.TotalSeconds;
            
            // 색상 변경 시간이 지났는가?
            if (elapsedTime >= colorChangeInterval)
            {
                // 색상을 무작위로 변경한다
                var random = new System.Random();
                currentColor = new Color(
                    random.Next(256),
                    random.Next(256),
                    random.Next(256)
                );
                
                // 경과 시간을 초기화한다
                elapsedTime = 0;
            }
            
            // ESC 키로 게임 종료
            if (Keyboard.GetState().IsKeyDown(Keys.Escape))
                Exit();

            base.Update(gameTime);
        }

        protected override void Draw(GameTime gameTime)
        {
            // 현재 색상으로 화면을 채운다
            GraphicsDevice.Clear(currentColor);

            base.Draw(gameTime);
        }
    }
}
```

**코드 설명 및 동작 원리:**

이 예제는 게임 루프의 핵심을 보여준다. Update() 메서드에서는 게임의 상태(이 경우, currentColor 변수)를 변경하고, Draw() 메서드에서는 그 상태를 화면에 표현한다.

**elapsedTime 변수** - 경과 시간을 누적한다. Update()가 호출될 때마다 gameTime.ElapsedGameTime.TotalSeconds를 더한다. 이 값은 이전 프레임과 현재 프레임 사이의 시간 차이다.

**색상 변경 로직** - elapsedTime이 colorChangeInterval(2초)에 도달하면, Random을 사용하여 무작위 색상을 생성한다. Color 클래스의 생성자는 RGB 값(0~255)을 받으므로, 각각을 0~255 사이의 무작위 숫자로 설정한다.

**경과 시간 초기화** - 색상을 변경한 후 elapsedTime을 0으로 초기화한다. 그래야 다음 2초 후에 또 다시 색상이 변경된다.

**GameTime의 중요성** - gameTime.ElapsedGameTime을 사용함으로써, 게임이 60 FPS에서 실행되든 120 FPS에서 실행되든 관계없이 항상 2초마다 색상이 변하도록 보장된다. 만약 단순히 frame counter를 사용했다면, 다른 프레임레이트에서는 색상 변경 시간이 달라질 것이다.

이 예제를 실행하면, 게임 윈도우의 배경색이 2초마다 무작위의 색상으로 변할 것이다.

**더 나은 구현:**

위 코드에는 매 프레임마다 Random 객체를 생성하는 비효율이 있다. 프로덕션 코드라면 다음과 같이 개선하는 것이 좋다.

```csharp
public class Game1 : Game
{
    private System.Random random = new System.Random();  // 클래스 변수로 선언
    
    protected override void Update(GameTime gameTime)
    {
        elapsedTime += (float)gameTime.ElapsedGameTime.TotalSeconds;
        
        if (elapsedTime >= colorChangeInterval)
        {
            // 이제 생성된 Random 객체를 재사용한다
            currentColor = new Color(
                random.Next(256),
                random.Next(256),
                random.Next(256)
            );
            
            elapsedTime = 0;
        }
        
        if (Keyboard.GetState().IsKeyDown(Keys.Escape))
            Exit();

        base.Update(gameTime);
    }
}
```

Random을 한 번만 생성하고 재사용하면 매 프레임마다 객체를 생성하는 오버헤드가 사라진다.

**과제 1: 게임 루프의 호출 순서 확인**

다음 코드를 Game1.cs에 추가하여 각 메서드가 얼마나 자주 호출되는지 확인해보자.

```csharp
private int updateCount = 0;
private int drawCount = 0;

protected override void Initialize()
{
    System.Diagnostics.Debug.WriteLine("Initialize() called");
    base.Initialize();
}

protected override void LoadContent()
{
    System.Diagnostics.Debug.WriteLine("LoadContent() called");
    _spriteBatch = new SpriteBatch(GraphicsDevice);
}

protected override void Update(GameTime gameTime)
{
    updateCount++;
    System.Diagnostics.Debug.WriteLine($"Update() called - Count: {updateCount}");
    base.Update(gameTime);
}

protected override void Draw(GameTime gameTime)
{
    drawCount++;
    GraphicsDevice.Clear(Color.CornflowerBlue);
    base.Draw(gameTime);
}
```

게임을 실행한 후 2~3초 정도 지난 후 ESC 키를 눌러 게임을 종료한다. Visual Studio의 출력 창(Output Window)을 보고 다음을 확인해보자.

**과제 5: 문제 해결**

다음 코드에는 문제가 있다. 문제를 찾아내고 수정해보자.

```csharp
private Color[] colors = { Color.Red, Color.Green, Color.Blue };
private int colorIndex = 0;
private float elapsedTime = 0;

protected override void Update(GameTime gameTime)
{
    elapsedTime += (float)gameTime.ElapsedGameTime.TotalSeconds;
    
    // 이 코드에는 문제가 있다
    if (elapsedTime >= 2.0f)
    {
        colorIndex++;
        // elapsedTime을 리셋하지 않음!
    }
    
    if (colorIndex >= colors.Length)
    {
        colorIndex = 0;
    }
    
    if (Keyboard.GetState().IsKeyDown(Keys.Escape))
        Exit();

    base.Update(gameTime);
}

protected override void Draw(GameTime gameTime)
{
    GraphicsDevice.Clear(colors[colorIndex]);
    base.Draw(gameTime);
}
```

**문제점:** elapsedTime을 리셋하지 않았으므로, 2.0f를 한 번 넘으면 계속 2.0f보다 크기 때문에 매 프레임마다 colorIndex가 증가한다. 색상이 정상적으로 변경되지 않을 것이다.

이 코드를 수정하고, 수정된 코드가 정상적으로 작동하는지 확인해보자.
