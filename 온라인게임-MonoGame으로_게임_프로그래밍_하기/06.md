# 6장. 키보드와 마우스 입력
이번 장에서는 게임에서 가장 기본이 되는 입력 처리를 배운다. 게임이 플레이어의 의도에 반응하지 않으면 그것은 게임이 아니다. MonoGame에서는 키보드와 마우스 입력을 간단하게 처리할 수 있는 API를 제공한다. 이 장을 통해 플레이어가 게임과 상호작용하는 기초를 다진다.

## 6.1 키보드 상태 읽기

### KeyboardState 이해하기
MonoGame에서 키보드 입력을 처리하려면 Keyboard.GetState() 메서드를 사용하여 현재 키보드의 상태를 얻는다. 이것이 KeyboardState 객체인데, 이것은 모든 키의 현재 상태(눌려있음 또는 안 눌려있음)를 포함한다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Input;

public class KeyboardInputExample
{
    public void Update()
    {
        // 현재 키보드 상태 가져오기
        KeyboardState keyboardState = Keyboard.GetState();

        // 스페이스 키가 눌려있는지 확인
        if (keyboardState.IsKeyDown(Keys.Space))
        {
            // 스페이스 키가 눌려있을 때 실행
            System.Console.WriteLine("스페이스 키가 눌려있습니다!");
        }
    }
}
```

IsKeyDown() 메서드는 특정 키가 현재 눌려있으면 true를 반환한다. 이것은 매 프레임 호출되므로, 키가 눌려있는 동안 계속 true를 반환한다.

### 키의 종류
MonoGame의 Keys 열거형은 키보드의 모든 키를 정의한다. 자주 사용되는 키들은 다음과 같다.


### 캐릭터 이동 구현
가장 흔한 입력 처리인 캐릭터 이동을 구현해보자. 방향키나 WASD 키로 캐릭터를 이동시키는 방식이다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Content;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;

public class Player
{
    private Texture2D texture;
    public Vector2 Position { get; set; }
    private float moveSpeed = 200f;  // 초당 픽셀

    public Player(Vector2 initialPosition)
    {
        Position = initialPosition;
    }

    public void LoadContent(ContentManager content)
    {
        texture = content.Load<Texture2D>("Player");
    }

    public void Update(GameTime gameTime)
    {
        KeyboardState keyboardState = Keyboard.GetState();
        
        // 이동 벡터 (기본값은 0)
        Vector2 movement = Vector2.Zero;

        // 위쪽 이동 (W 또는 위쪽 방향키)
        if (keyboardState.IsKeyDown(Keys.W) || keyboardState.IsKeyDown(Keys.Up))
        {
            movement.Y -= 1;
        }

        // 아래쪽 이동 (S 또는 아래쪽 방향키)
        if (keyboardState.IsKeyDown(Keys.S) || keyboardState.IsKeyDown(Keys.Down))
        {
            movement.Y += 1;
        }

        // 왼쪽 이동 (A 또는 왼쪽 방향키)
        if (keyboardState.IsKeyDown(Keys.A) || keyboardState.IsKeyDown(Keys.Left))
        {
            movement.X -= 1;
        }

        // 오른쪽 이동 (D 또는 오른쪽 방향키)
        if (keyboardState.IsKeyDown(Keys.D) || keyboardState.IsKeyDown(Keys.Right))
        {
            movement.X += 1;
        }

        // 대각선 이동 시 정규화 (모든 방향에서 같은 속도 유지)
        if (movement != Vector2.Zero)
        {
            movement.Normalize();
        }

        // 위치 업데이트
        float deltaTime = (float)gameTime.ElapsedGameTime.TotalSeconds;
        Position += movement * moveSpeed * deltaTime;
    }

    public void Draw(SpriteBatch spriteBatch)
    {
        spriteBatch.Draw(texture, Position, Color.White);
    }
}
```

**코드 설명:**

이 코드의 핵심은 movement 벡터다. 각 키의 입력에 따라 movement의 X 또는 Y 값을 조정한다. 대각선으로 이동할 때(예: W와 D를 동시에 누름) movement 벡터는 (1, -1)이 되는데, 이를 그대로 사용하면 대각선 이동이 가로나 세로 이동보다 약 1.414배 빠르다. 따라서 Normalize() 메서드로 벡터의 크기를 1로 정규화하여 모든 방향에서 같은 속도를 유지한다. 마지막으로 ElapsedGameTime을 곱하여 프레임 독립적인 이동을 구현한다.

### IsKeyDown과 IsKeyUp의 차이
IsKeyDown()은 키가 현재 눌려있는지를 나타내고, IsKeyUp()은 키가 눌려있지 않은지를 나타낸다. 단순 상태 확인에는 IsKeyDown()을 사용하고, 특정 키가 눌려있지 않은 상태를 확인할 때는 IsKeyUp()을 사용한다.

```csharp
KeyboardState keyboardState = Keyboard.GetState();

// 스페이스 키가 눌려있지 않으면
if (keyboardState.IsKeyUp(Keys.Space))
{
    // 점프 준비 상태
}

// 더 간결한 표현 (같은 의미)
if (!keyboardState.IsKeyDown(Keys.Space))
{
    // 점프 준비 상태
}
```

## 6.2 마우스 입력 처리

### MouseState 이해하기
마우스 입력도 키보드와 유사하게 MouseState 객체로 처리한다. 마우스는 위치, 버튼 상태, 스크롤 휠 상태 등을 제공한다.

```csharp
using Microsoft.Xna.Framework.Input;

public class MouseInputExample
{
    public void Update()
    {
        // 현재 마우스 상태 가져오기
        MouseState mouseState = Mouse.GetState();

        // 마우스 위치 (화면 좌표)
        int mouseX = mouseState.X;
        int mouseY = mouseState.Y;
        Vector2 mousePosition = new Vector2(mouseX, mouseY);

        // 왼쪽 버튼이 눌려있는지 확인
        if (mouseState.LeftButton == ButtonState.Pressed)
        {
            System.Console.WriteLine($"왼쪽 버튼 눌림: ({mouseX}, {mouseY})");
        }

        // 오른쪽 버튼이 눌려있는지 확인
        if (mouseState.RightButton == ButtonState.Pressed)
        {
            System.Console.WriteLine("오른쪽 버튼 눌림");
        }

        // 중간 버튼(휠)이 눌려있는지 확인
        if (mouseState.MiddleButton == ButtonState.Pressed)
        {
            System.Console.WriteLine("중간 버튼 눌림");
        }

        // 스크롤 휠 위치 (양수면 위로, 음수면 아래로)
        int scrollDelta = mouseState.ScrollWheelValue;
    }
}
```

**마우스 상태 속성:**

X와 Y는 마우스 커서의 화면 좌표다. 화면의 왼쪽 위가 (0, 0)이고 오른쪽 아래로 갈수록 값이 증가한다. LeftButton, RightButton, MiddleButton은 ButtonState 열거형을 반환하는데, Pressed 또는 Released 값을 가진다. ScrollWheelValue는 누적된 스크롤 휠 값이므로, 이전 프레임과의 차이를 계산하여 스크롤 량을 파악한다.

### 마우스로 객체 드래그하기
게임에서 흔한 상호작용은 마우스로 객체를 드래그하는 것이다. 다음은 그러한 예제다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Content;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;

public class DraggableObject
{
    private Texture2D texture;
    public Vector2 Position { get; set; }
    private float size = 50f;
    private bool isDragging = false;
    private Vector2 dragOffset = Vector2.Zero;

    public DraggableObject(Vector2 initialPosition)
    {
        Position = initialPosition;
    }

    public void LoadContent(ContentManager content)
    {
        texture = content.Load<Texture2D>("Circle");
    }

    public void Update(GameTime gameTime)
    {
        MouseState mouseState = Mouse.GetState();
        Vector2 mousePosition = new Vector2(mouseState.X, mouseState.Y);

        // 객체의 바운딩 박스 (중앙을 기준으로)
        Rectangle boundingBox = new Rectangle(
            (int)(Position.X - size / 2),
            (int)(Position.Y - size / 2),
            (int)size,
            (int)size
        );

        // 마우스 위치가 객체 범위 내에 있는지 확인
        bool isMouseOver = boundingBox.Contains(mousePosition.ToPoint());

        // 왼쪽 버튼을 눌렀을 때
        if (mouseState.LeftButton == ButtonState.Pressed)
        {
            if (isMouseOver && !isDragging)
            {
                // 드래그 시작
                isDragging = true;
                dragOffset = Position - mousePosition;
            }
        }
        else
        {
            // 버튼을 놓았을 때
            isDragging = false;
        }

        // 드래그 중일 때 위치 업데이트
        if (isDragging)
        {
            Position = mousePosition + dragOffset;
        }
    }

    public void Draw(SpriteBatch spriteBatch)
    {
        Color drawColor = isDragging ? Color.Yellow : Color.White;
        spriteBatch.Draw(
            texture,
            Position,
            null,
            drawColor,
            0f,
            new Vector2(size / 2, size / 2),
            1f,
            SpriteEffects.None,
            0f
        );
    }
}
```

**코드 설명:**

이 클래스는 마우스로 드래그 가능한 객체를 구현한다. isDragging 플래그로 드래그 중인지 추적한다. 드래그 시작 시 dragOffset에 객체 위치와 마우스 위치의 차이를 저장한다. 이렇게 하면 마우스의 정확한 누른 지점에서 드래그가 시작되어 자연스러운 상호작용이 된다. 드래그 중일 때는 마우스 위치에 저장된 오프셋을 더하여 객체의 위치를 업데이트한다. 드래그 중이면 색상을 노란색으로 변경하여 시각적 피드백을 준다.

### 마우스로 총 쏘기
슈팅 게임에서 마우스 클릭으로 총을 쏘는 것도 흔한 입력이다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Content;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;
using System;

public class GunController
{
    private Vector2 gunPosition;
    private Texture2D bulletTexture;

    public GunController(Vector2 position)
    {
        gunPosition = position;
    }

    public void LoadContent(ContentManager content)
    {
        bulletTexture = content.Load<Texture2D>("Bullet");
    }

    public Bullet TryFire(GameTime gameTime)
    {
        MouseState mouseState = Mouse.GetState();
        Vector2 mousePosition = new Vector2(mouseState.X, mouseState.Y);

        // 왼쪽 버튼이 클릭되었을 때만 총 발사
        if (mouseState.LeftButton == ButtonState.Pressed)
        {
            // 발사 방향 계산 (총 위치에서 마우스 위치로)
            Vector2 direction = mousePosition - gunPosition;
            
            if (direction.Length() > 0)
            {
                direction.Normalize();
            }

            // 총알 생성 및 반환
            return new Bullet(
                gunPosition,
                direction,
                bulletTexture,
                speed: 500f
            );
        }

        return null;
    }

    public void Draw(SpriteBatch spriteBatch)
    {
        // 총의 위치에 십자 표시
        spriteBatch.DrawString(
            null,  // 폰트 (여기서는 생략)
            "+",
            gunPosition,
            Color.Red
        );
    }
}

public class Bullet
{
    public Vector2 Position { get; set; }
    public Vector2 Direction { get; set; }
    private Texture2D texture;
    private float speed;
    private float lifetime = 5f;  // 5초
    private float elapsedTime = 0;

    public Bullet(Vector2 position, Vector2 direction, Texture2D texture, float speed)
    {
        Position = position;
        Direction = direction;
        this.texture = texture;
        this.speed = speed;
    }

    public void Update(GameTime gameTime)
    {
        float deltaTime = (float)gameTime.ElapsedGameTime.TotalSeconds;
        Position += Direction * speed * deltaTime;
        elapsedTime += deltaTime;
    }

    public bool IsAlive => elapsedTime < lifetime;

    public void Draw(SpriteBatch spriteBatch)
    {
        spriteBatch.Draw(texture, Position, Color.White);
    }
}
```

## 6.3 입력 상태 관리

### 입력 상태 추적의 문제점
지금까지의 예제에서는 단순히 IsKeyDown()으로 키가 눌려있는지만 확인했다. 하지만 게임 개발에서는 종종 "키가 방금 눌렸는가?"를 구분해야 한다. 예를 들어, 점프는 키가 눌린 순간에만 발생해야 하고, 계속 누르고 있다고 해서 반복해서 점프하면 안 된다.

```csharp
// 문제 있는 코드
KeyboardState keyboardState = Keyboard.GetState();

if (keyboardState.IsKeyDown(Keys.Space))
{
    Jump();  // 프레임마다 호출됨 (잘못된 동작)
}
```

이런 문제를 해결하려면 현재 프레임과 이전 프레임의 입력 상태를 비교해야 한다.

### InputManager 클래스 구현
프레임마다 입력을 추적하는 InputManager 클래스를 만들자.

```csharp
using Microsoft.Xna.Framework.Input;

public class InputManager
{
    private KeyboardState previousKeyboardState;
    private KeyboardState currentKeyboardState;
    private MouseState previousMouseState;
    private MouseState currentMouseState;

    public void Update()
    {
        // 이전 상태를 현재 상태로 저장
        previousKeyboardState = currentKeyboardState;
        previousMouseState = currentMouseState;

        // 현재 상태 갱신
        currentKeyboardState = Keyboard.GetState();
        currentMouseState = Mouse.GetState();
    }

    // 키가 현재 눌려있는지
    public bool IsKeyDown(Keys key)
    {
        return currentKeyboardState.IsKeyDown(key);
    }

    // 키가 현재 눌려있지 않은지
    public bool IsKeyUp(Keys key)
    {
        return currentKeyboardState.IsKeyUp(key);
    }

    // 키가 방금 눌렸는지 (이전에는 놓여있었고, 현재는 눌려있음)
    public bool IsKeyPressed(Keys key)
    {
        return currentKeyboardState.IsKeyDown(key) && 
               previousKeyboardState.IsKeyUp(key);
    }

    // 키가 방금 놓였는지 (이전에는 눌려있었고, 현재는 놓여있음)
    public bool IsKeyReleased(Keys key)
    {
        return currentKeyboardState.IsKeyUp(key) && 
               previousKeyboardState.IsKeyDown(key);
    }

    // 마우스 위치
    public int MouseX => currentMouseState.X;
    public int MouseY => currentMouseState.Y;

    // 왼쪽 버튼이 눌려있는지
    public bool IsLeftButtonDown => currentMouseState.LeftButton == ButtonState.Pressed;

    // 왼쪽 버튼이 방금 눌렸는지
    public bool IsLeftButtonPressed => 
        currentMouseState.LeftButton == ButtonState.Pressed && 
        previousMouseState.LeftButton == ButtonState.Released;

    // 오른쪽 버튼이 눌려있는지
    public bool IsRightButtonDown => currentMouseState.RightButton == ButtonState.Pressed;

    // 오른쪽 버튼이 방금 눌렸는지
    public bool IsRightButtonPressed => 
        currentMouseState.RightButton == ButtonState.Pressed && 
        previousMouseState.RightButton == ButtonState.Released;
}
```

**코드 설명:**

이 InputManager 클래스는 이전과 현재의 입력 상태를 저장한다. Update() 메서드는 매 프레임 호출되어야 하며, 현재 상태를 이전 상태로 업데이트한 후 새로운 현재 상태를 읽는다. IsKeyPressed() 메서드는 현재 누르고 있으면서 이전 프레임에는 누르지 않은 경우에만 true를 반환한다. 이를 통해 "방금 눌렸는가?"를 정확히 판단할 수 있다.

### InputManager를 사용한 점프 구현
이제 InputManager를 사용하여 올바른 점프 기능을 구현해보자.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Content;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;

public class Player
{
    private Texture2D texture;
    public Vector2 Position { get; set; }
    public Vector2 Velocity { get; set; }
    private float gravity = 800f;
    private float jumpPower = 400f;
    private bool isJumping = false;
    private InputManager inputManager;

    public Player(Vector2 initialPosition, InputManager inputManager)
    {
        Position = initialPosition;
        this.inputManager = inputManager;
        Velocity = Vector2.Zero;
    }

    public void LoadContent(ContentManager content)
    {
        texture = content.Load<Texture2D>("Player");
    }

    public void Update(GameTime gameTime)
    {
        float deltaTime = (float)gameTime.ElapsedGameTime.TotalSeconds;

        // 점프 입력 처리 (방향키나 스페이스 키)
        if ((inputManager.IsKeyPressed(Keys.Up) || 
             inputManager.IsKeyPressed(Keys.Space)) && !isJumping)
        {
            // 점프 시작 (IsKeyPressed이므로 한 번만 실행)
            Velocity.Y = -jumpPower;
            isJumping = true;
        }

        // 중력 적용
        Velocity.Y += gravity * deltaTime;

        // 위치 업데이트
        Position += Velocity * deltaTime;

        // 바닥 충돌 (y가 400 이상이면 바닥)
        if (Position.Y >= 400)
        {
            Position.Y = 400;
            Velocity.Y = 0;
            isJumping = false;
        }
    }

    public void Draw(SpriteBatch spriteBatch)
    {
        spriteBatch.Draw(texture, Position, Color.White);
    }
}
```

**코드 설명:**

IsKeyPressed()를 사용하면 스페이스 키가 눌린 정확한 순간에만 점프가 발생한다. 계속 누르고 있어도 한 번만 점프한다. isJumping 플래그로 공중에 있을 때 다시 점프하는 것을 방지한다. 위치가 바닥에 닿으면 isJumping을 false로 설정하여 다시 점프할 수 있게 한다.

### Game 클래스에 InputManager 통합
이제 게임의 메인 클래스에서 InputManager를 사용하는 방법을 보자.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;

public class MyGame : Game
{
    private GraphicsDeviceManager graphics;
    private SpriteBatch spriteBatch;
    private InputManager inputManager;
    private Player player;

    public MyGame()
    {
        graphics = new GraphicsDeviceManager(this);
        Content.RootDirectory = "Content";
    }

    protected override void Initialize()
    {
        inputManager = new InputManager();
        player = new Player(new Vector2(400, 300), inputManager);

        base.Initialize();
    }

    protected override void LoadContent()
    {
        spriteBatch = new SpriteBatch(GraphicsDevice);
        player.LoadContent(Content);
    }

    protected override void Update(GameTime gameTime)
    {
        // InputManager 업데이트 (가장 먼저 호출)
        inputManager.Update();

        // ESC 키로 게임 종료
        if (inputManager.IsKeyPressed(Microsoft.Xna.Framework.Input.Keys.Escape))
        {
            Exit();
        }

        player.Update(gameTime);

        base.Update(gameTime);
    }

    protected override void Draw(GameTime gameTime)
    {
        GraphicsDevice.Clear(Color.CornflowerBlue);

        spriteBatch.Begin();
        player.Draw(spriteBatch);
        spriteBatch.End();

        base.Draw(gameTime);
    }
}
```

**중요한 포인트:**

Update() 메서드의 가장 첫 번째에 inputManager.Update()를 호출해야 한다. 그래야 최신의 입력 상태를 읽을 수 있다.

### 마우스 스크롤 처리
마지막으로 마우스 스크롤 휠을 처리하는 방법을 보자.

```csharp
public class InputManager
{
    private MouseState previousMouseState;
    private MouseState currentMouseState;
    private int previousScrollValue = 0;

    public void Update()
    {
        previousMouseState = currentMouseState;
        currentMouseState = Mouse.GetState();
    }

    // 스크롤 휠의 변화량 (양수면 위로, 음수면 아래로)
    public int ScrollDelta
    {
        get
        {
            return currentMouseState.ScrollWheelValue - previousScrollValue;
        }
    }

    public void UpdateScrollValue()
    {
        previousScrollValue = currentMouseState.ScrollWheelValue;
    }
}
```

게임 루프에서 마우스 처리 후 UpdateScrollValue()를 호출하여 스크롤 값을 업데이트한다.

## 과제
이번 장에서 배운 내용을 확인하기 위한 과제다.

### 과제: 마우스와 키보드로 조종하는 인벤토리 시스템

**요구사항:**

1. 게임 화면에 3x3 격자 인벤토리를 그린다(총 9개 슬롯).
2. 각 슬롯은 50x50 픽셀 크기, 10 픽셀 간격으로 배치한다.
3. 마우스로 아이템을 드래그하여 슬롯 간에 이동할 수 있다.
4. 아이템이 슬롯 안에 있을 때만 인식된다.

**키보드 입력:**
- A 키: 현재 마우스 위치의 슬롯에 새 아이템 추가(있으면 무시)
- D 키: 현재 마우스 위치의 슬롯에서 아이템 제거
- C 키: 인벤토리 비우기
- 오른쪽 화살표 키: 선택된 슬롯을 오른쪽으로 이동
- 왼쪽 화살표 키: 선택된 슬롯을 왼쪽으로 이동

**마우스 입력:**
- 왼쪽 클릭 드래그: 아이템을 다른 슬롯으로 이동
- 오른쪽 클릭: 아이템 회전(시각적 표시만)

**표시 사항:**
1. 현재 마우스가 있는 슬롯은 밝은 회색으로 하이라이트한다.
2. 선택된(키보드로 선택된) 슬롯은 초록색 테두리로 표시한다.
3. 아이템이 있는 슬롯은 파란색으로 채워진다.
4. 화면 상단에 "인벤토리: X/9" 형식으로 아이템 개수를 표시한다.
5. 현재 마우스 좌표를 화면 우상단에 표시한다.

**구현해야 할 클래스:**
- InventorySlot: 슬롯 정보 관리
- Inventory: 인벤토리 전체 관리
- InputManager: 입력 상태 관리
- MyGame: 통합 관리

**테스트 항목:**
- A 키를 누르면 마우스 위치의 슬롯에 아이템이 추가되는가?
- D 키를 누르면 마우스 위치의 슬롯에서 아이템이 제거되는가?
- C 키를 누르면 모든 아이템이 제거되는가?
- 마우스로 아이템을 드래그할 수 있는가?
- 오른쪽 화살표 키로 선택 슬롯을 이동할 수 있는가?
- 마우스가 슬롯 위에 있을 때 하이라이트되는가?
- 아이템 개수가 올바르게 표시되는가?

**팁:**
- 슬롯의 격자 좌표(행, 열)를 계산하면 쉽다: 행 = Y / 60, 열 = X / 60
- 아이템 상태는 bool 배열로 관리할 수 있다.
- 드래그 중인 아이템의 슬롯도 추적해야 한다.
- 마우스 좌표가 슬롯 범위 내인지 확인하려면 Rectangle.Contains()를 사용한다.

이 과제를 완성하면 키보드와 마우스 입력 처리, 입력 상태 추적, 객체 상호작용을 모두 구현한 것이 된다. 또한 인벤토리 시스템은 실제 게임에서 자주 사용되는 UI 패턴이므로 유용한 경험을 얻을 수 있다.
