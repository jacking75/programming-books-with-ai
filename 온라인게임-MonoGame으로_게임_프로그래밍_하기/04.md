# 4장. 스프라이트와 애니메이션
이번 장에서는 게임에서 가장 중요한 시각 요소인 스프라이트와 애니메이션을 다룬다. 이전 장에서는 개별 이미지를 그렸지만, 이번 장에서는 여러 이미지를 조합하여 움직이는 캐릭터를 만드는 방법을 배운다. 게임에서 애니메이션은 캐릭터가 살아있다고 느끼게 해주는 핵심 요소다.
  
## 4.1 스프라이트 시트 활용

### 스프라이트 시트란?
스프라이트 시트는 여러 개의 이미지를 하나의 큰 이미지 파일에 격자 형태로 배열한 것이다. 예를 들어, 캐릭터가 4방향으로 움직일 때 각 방향마다 8개의 프레임이 필요하다면, 총 32개의 이미지를 개별 파일로 관리하기보다는 하나의 스프라이트 시트에 모두 담아 관리한다.

```
스프라이트 시트 구조 예시 (4x8 그리드):

┌─────────────────────────────────────────────┐
│ 위쪽 방향 (8 프레임)                          │
├─────┬─────┬─────┬─────┬─────┬─────┬─────┬───┤
│ [0] │ [1] │ [2] │ [3] │ [4] │ [5] │ [6] │[7]
├─────────────────────────────────────────────┤
│ 아래쪽 방향 (8 프레임)                         │
├─────┬─────┬─────┬─────┬─────┬─────┬─────┬───┤
│ [0] │ [1] │ [2] │ [3] │ [4] │ [5] │ [6] │[7]
├─────────────────────────────────────────────┤
│ 왼쪽 방향 (8 프레임)                          │
├─────┬─────┬─────┬─────┬─────┬─────┬─────┬───┤
│ [0] │ [1] │ [2] │ [3] │ [4] │ [5] │ [6] │[7]
├─────────────────────────────────────────────┤
│ 오른쪽 방향 (8 프레임)                         │
├─────┬─────┬─────┬─────┬─────┬─────┬─────┬───┤
│ [0] │ [1] │ [2] │ [3] │ [4] │ [5] │ [6] │[7]
└─────────────────────────────────────────────┘

각 셀의 크기: 64x64 픽셀
전체 크기: 512x256 픽셀
```

스프라이트 시트를 사용하면 여러 이미지를 한 번에 로드하므로 성능이 더 좋다. 또한 관리가 편하고, 특히 모바일 게임에서 메모리 사용을 줄일 수 있다.

### 스프라이트 시트에서 특정 부분 그리기
스프라이트 시트에서 원하는 부분만 그려내는 방법을 배운다. MonoGame의 SpriteBatch.Draw() 메서드는 원본 이미지의 일부만 그리도록 하는 sourceRectangle 매개변수를 제공한다.

다음 예제를 보자. 이 코드는 스프라이트 시트에서 특정 셀을 추출하여 화면에 그리는 기본 방법을 보여준다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Content;
using Microsoft.Xna.Framework.Graphics;

public class SpriteSheetExample
{
    private Texture2D spriteSheet;
    private int frameWidth = 64;      // 각 프레임의 너비
    private int frameHeight = 64;     // 각 프레임의 높이
    private int currentFrame = 0;     // 현재 표시할 프레임 인덱스
    private Vector2 position = new Vector2(100, 100);

    public void LoadContent(ContentManager content)
    {
        spriteSheet = content.Load<Texture2D>("Characters/Player");
    }

    public void Draw(SpriteBatch spriteBatch)
    {
        // 현재 프레임이 어느 행에 있는지 계산
        int row = currentFrame / 8;  // 한 행에 8개의 프레임
        int column = currentFrame % 8;

        // 스프라이트 시트에서 그릴 영역 정의
        Rectangle sourceRectangle = new Rectangle(
            column * frameWidth,
            row * frameHeight,
            frameWidth,
            frameHeight
        );

        // 화면에 그리기
        spriteBatch.Draw(
            spriteSheet,
            position,
            sourceRectangle,
            Color.White,
            0f,
            Vector2.Zero,
            1f,
            SpriteEffects.None,
            0f
        );
    }

    public void NextFrame()
    {
        currentFrame = (currentFrame + 1) % 32;  // 총 32개 프레임
    }
}
```

**코드 설명:**

이 코드의 핵심은 Rectangle 타입의 sourceRectangle를 만드는 부분이다. sourceRectangle는 스프라이트 시트에서 그릴 영역을 정의한다. 현재 프레임 인덱스를 바탕으로 행(row)과 열(column)을 계산한다. 예를 들어 currentFrame이 10이라면, 10 / 8 = 1행, 10 % 8 = 2열이 된다. 그리고 이 위치에 frameWidth와 frameHeight를 곱하여 실제 픽셀 좌표를 계산한다. SpriteBatch.Draw() 메서드에 sourceRectangle을 전달하면 스프라이트 시트의 그 부분만 화면에 그려진다.

## 4.2 프레임 기반 애니메이션

### 애니메이션의 개념
프레임 기반 애니메이션은 정적인 여러 이미지를 빠르게 연속으로 표시하여 움직임을 표현하는 기법이다. 영화가 초당 24프레임으로 동작하는 것처럼, 게임에서도 초당 60프레임 중에서 일부만 사용하여 애니메이션을 만든다.

예를 들어, 캐릭터의 걷기 애니메이션이 8개의 프레임으로 이루어져 있다면, 4프레임마다 다음 프레임을 표시하면 게임은 60fps로 실행되면서도 애니메이션은 15fps로 표시된다(60 / 4 = 15).

### 애니메이션 클래스 설계
이제 재사용 가능한 애니메이션 클래스를 만들어보자. 이 클래스는 스프라이트 시트와 애니메이션 속도를 관리한다.

```csharp
using Microsoft.Xna.Framework;

public class Animation
{
    private int frameWidth;
    private int frameHeight;
    private int totalFrames;
    private int framesPerRow;
    private int currentFrame;
    private double elapsedTime;
    private double frameTime;  // 각 프레임이 표시되는 시간 (초 단위)
    private bool isPlaying;
    private bool isLooping;

    public Animation(
        int frameWidth,
        int frameHeight,
        int totalFrames,
        int framesPerRow,
        double framesPerSecond,
        bool isLooping = true)
    {
        this.frameWidth = frameWidth;
        this.frameHeight = frameHeight;
        this.totalFrames = totalFrames;
        this.framesPerRow = framesPerRow;
        this.isLooping = isLooping;
        
        // framesPerSecond(애니메이션 속도)로부터 각 프레임 시간 계산
        this.frameTime = 1.0 / framesPerSecond;
        
        this.currentFrame = 0;
        this.elapsedTime = 0;
        this.isPlaying = true;
    }

    public void Update(GameTime gameTime)
    {
        if (!isPlaying)
            return;

        elapsedTime += gameTime.ElapsedGameTime.TotalSeconds;

        // 충분한 시간이 경과했으면 다음 프레임으로 이동
        if (elapsedTime >= frameTime)
        {
            elapsedTime -= frameTime;
            currentFrame++;

            // 애니메이션 종료 처리
            if (currentFrame >= totalFrames)
            {
                if (isLooping)
                {
                    currentFrame = 0;
                }
                else
                {
                    currentFrame = totalFrames - 1;
                    isPlaying = false;
                }
            }
        }
    }

    public Rectangle GetSourceRectangle()
    {
        int row = currentFrame / framesPerRow;
        int column = currentFrame % framesPerRow;

        return new Rectangle(
            column * frameWidth,
            row * frameHeight,
            frameWidth,
            frameHeight
        );
    }

    public void Play()
    {
        isPlaying = true;
    }

    public void Stop()
    {
        isPlaying = false;
    }

    public void Reset()
    {
        currentFrame = 0;
        elapsedTime = 0;
        isPlaying = true;
    }

    public bool IsFinished => !isPlaying && currentFrame == totalFrames - 1;
}
```

**코드 설명:**

이 Animation 클래스는 애니메이션 재생을 관리하는 핵심 로직을 담고 있다. 먼저 frameTime을 계산할 때 framesPerSecond의 역수를 취하여 각 프레임이 얼마나 오래 표시되어야 하는지를 구한다. Update() 메서드에서는 경과 시간을 누적하여 frameTime만큼 시간이 경과했으면 다음 프레임으로 넘어간다. 이때 isLooping 플래그로 애니메이션이 반복되는지 한 번만 재생되는지를 제어한다. GetSourceRectangle() 메서드는 현재 프레임의 위치를 Rectangle로 반환하므로 SpriteBatch.Draw()에 바로 사용할 수 있다.

### 애니메이션을 사용하는 게임 오브젝트
이제 Animation 클래스를 실제로 사용하는 캐릭터 클래스를 만든다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Content;
using Microsoft.Xna.Framework.Graphics;

public class AnimatedCharacter
{
    private Texture2D spriteSheet;
    private Animation[] animations;  // 여러 애니메이션 저장 (걷기, 점프 등)
    private int currentAnimationIndex;
    public Vector2 Position { get; set; }

    public AnimatedCharacter(Vector2 initialPosition)
    {
        Position = initialPosition;
        animations = new Animation[4];
        currentAnimationIndex = 0;
    }

    public void LoadContent(ContentManager content)
    {
        spriteSheet = content.Load<Texture2D>("Characters/Player");

        // 각 방향별로 애니메이션 생성
        // 매개변수: 너비, 높이, 전체 프레임, 한 행의 프레임, 초당 프레임 수
        animations[0] = new Animation(64, 64, 8, 8, 10.0);  // 위쪽 방향
        animations[1] = new Animation(64, 64, 8, 8, 10.0);  // 아래쪽 방향
        animations[2] = new Animation(64, 64, 8, 8, 10.0);  // 왼쪽 방향
        animations[3] = new Animation(64, 64, 8, 8, 10.0);  // 오른쪽 방향
    }

    public void Update(GameTime gameTime)
    {
        animations[currentAnimationIndex].Update(gameTime);
    }

    public void Draw(SpriteBatch spriteBatch)
    {
        Animation currentAnimation = animations[currentAnimationIndex];
        spriteBatch.Draw(
            spriteSheet,
            Position,
            currentAnimation.GetSourceRectangle(),
            Color.White
        );
    }

    public void SetAnimation(int index)
    {
        if (index != currentAnimationIndex)
        {
            currentAnimationIndex = index;
            animations[currentAnimationIndex].Reset();
        }
    }
}
```

**코드 설명:**

AnimatedCharacter 클래스는 여러 개의 Animation 객체를 배열로 관리한다. 각 배열 요소는 다른 방향이나 다른 동작(걷기, 점프 등)에 대한 애니메이션을 나타낸다. SetAnimation() 메서드로 현재 재생할 애니메이션을 전환할 수 있으며, 애니메이션이 바뀔 때 Reset()을 호출하여 새 애니메이션을 처음부터 시작하도록 한다.

## 4.3 회전과 크기 변환

### 회전 변환
게임에서 스프라이트를 회전시키는 것은 매우 흔하다. 미사일이 이동 방향으로 회전한다거나, 캐릭터가 마우스를 향해 몸을 돌린다거나, 회전하는 별 모양 장애물 등이 그 예다.

SpriteBatch.Draw() 메서드의 rotation 매개변수를 사용하여 회전을 구현한다. 각도는 라디안(radian) 단위로 지정하며, 시계 반대 방향이 양수다.

다음은 마우스 위치를 향해 회전하는 화살표를 그리는 예제다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;

public class RotatingArrow
{
    private Texture2D arrowTexture;
    private Vector2 position;
    private float rotation;

    public void LoadContent(Microsoft.Xna.Framework.Content.ContentManager content)
    {
        arrowTexture = content.Load<Texture2D>("Arrow");
    }

    public void Update(GameTime gameTime)
    {
        // 마우스 위치 가져오기
        MouseState mouseState = Mouse.GetState();
        Vector2 mousePosition = new Vector2(mouseState.X, mouseState.Y);

        // 화살표에서 마우스로의 방향 벡터 계산
        Vector2 direction = mousePosition - position;

        // 방향 벡터로부터 각도 계산 (라디안)
        // Atan2는 y, x 순서로 매개변수를 받음
        rotation = (float)System.Math.Atan2(direction.Y, direction.X);
    }

    public void Draw(SpriteBatch spriteBatch)
    {
        // 회전의 중심점: 이미지의 중앙
        Vector2 origin = new Vector2(arrowTexture.Width / 2f, arrowTexture.Height / 2f);

        spriteBatch.Draw(
            arrowTexture,
            position,
            null,
            Color.White,
            rotation,      // 라디안 단위의 회전 각도
            origin,        // 회전의 중심점
            1f,
            SpriteEffects.None,
            0f
        );
    }
}
```

**회전 매개변수 설명:**

SpriteBatch.Draw()에서 rotation 매개변수는 라디안 단위로 지정한다. 도(degree)를 라디안으로 변환하려면 (도 * π / 180) 공식을 사용하거나, MonoGame의 MathHelper 클래스를 사용할 수 있다. 예를 들어 90도 회전은 MathHelper.PiOver2와 같다.

또한 origin 매개변수는 매우 중요하다. 이것은 회전의 중심점을 지정하는데, (0, 0)이면 이미지의 왼쪽 위에서 회전하고, (width/2, height/2)이면 이미지의 중앙에서 회전한다. 대부분의 경우 이미지의 중앙을 중심으로 회전하는 것이 자연스럽다.

### 크기 변환 (스케일)
스프라이트의 크기를 확대하거나 축소하는 것도 자주 필요하다. SpriteBatch.Draw()의 scale 매개변수로 이를 구현한다. scale이 1f면 원래 크기, 2f면 2배, 0.5f면 절반 크기다.

다음 예제는 시간에 따라 커졌다가 작아지는 펄싱(pulsing) 효과를 보여준다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;

public class PulsingSprite
{
    private Texture2D texture;
    private Vector2 position;
    private double elapsedTime;
    private double pulseCycle = 1.0;  // 1초마다 반복

    public void LoadContent(Microsoft.Xna.Framework.Content.ContentManager content)
    {
        texture = content.Load<Texture2D>("Particle");
    }

    public void Update(GameTime gameTime)
    {
        elapsedTime += gameTime.ElapsedGameTime.TotalSeconds;

        // 사이클이 1초이므로, elapsedTime % pulseCycle은 0 ~ 1 사이의 값
        double cycleProgress = elapsedTime % pulseCycle;

        // 0부터 1까지 갔다가 다시 0으로 돌아오는 값 (사인파 사용)
        // Sin() 함수는 라디안을 받으므로, cycleProgress에 π를 곱함
        float pulseFactor = (float)System.Math.Sin(cycleProgress * System.Math.PI);

        // pulseFactor는 0 ~ 1 ~ 0이므로, 0.8 ~ 1.8 범위로 스케일 조정
        float scale = 0.8f + pulseFactor;

        // 위 코드에서 scale을 사용하여 Draw()를 호출
    }

    public void Draw(SpriteBatch spriteBatch)
    {
        double cycleProgress = elapsedTime % pulseCycle;
        float pulseFactor = (float)System.Math.Sin(cycleProgress * System.Math.PI);
        float scale = 0.8f + pulseFactor;

        Vector2 origin = new Vector2(texture.Width / 2f, texture.Height / 2f);

        spriteBatch.Draw(
            texture,
            position,
            null,
            Color.White,
            0f,
            origin,
            scale,         // 크기 변환
            SpriteEffects.None,
            0f
        );
    }
}
```

**코드 설명:**

이 코드의 핵심은 사인파를 이용한 부드러운 펄싱 효과다. elapsedTime을 pulseCycle로 나눈 나머지를 구하면 0부터 1까지의 주기적인 값을 얻는다. 이 값에 π를 곱하여 라디안으로 변환한 후 Sin() 함수에 전달하면, 0부터 1까지 올라갔다가 다시 0으로 내려오는 파형을 얻는다. 이 값을 크기 변환에 사용하면 자연스러운 펄싱 효과가 나타난다.

### 회전과 크기를 함께 사용
게임에서는 회전과 크기를 동시에 사용하는 경우가 많다. 다음은 회전하면서 동시에 커졌다가 작아지는 효과를 보여준다.

```csharp
public void Draw(SpriteBatch spriteBatch)
{
    // 회전 계산
    float rotationSpeed = 2f;  // 초당 라디안
    float rotation = (float)(elapsedTime * rotationSpeed);

    // 크기 계산
    double cycleProgress = elapsedTime % 1.0;
    float pulseFactor = (float)System.Math.Sin(cycleProgress * System.Math.PI);
    float scale = 0.8f + pulseFactor;

    Vector2 origin = new Vector2(texture.Width / 2f, texture.Height / 2f);

    spriteBatch.Draw(
        texture,
        position,
        null,
        Color.White,
        rotation,
        origin,
        scale,
        SpriteEffects.None,
        0f
    );
}
```

## 4.4 레이어링과 깊이

### 드로우 오더 문제
여러 스프라이트를 그릴 때, 어떤 것을 먼저 그려야 할까? 2D 게임에서도 깊이(depth)가 있어서, 뒤에 있는 것은 먼저 그려야 하고, 앞에 있는 것은 나중에 그려야 한다. 예를 들어, 배경, 땅 위의 캐릭터, 캐릭터 위의 던전 천장 순서로 그려야 자연스럽다.

```
그릴 순서 (Z 깊이 기준):

    ↓ 그리는 순서
    
[깊이 0.0] ████████ 배경
[깊이 0.2] ████████ 앞 나무들
[깊이 0.5] ████████ 캐릭터
[깊이 0.8] ████████ 캐릭터 위 천장
[깊이 1.0] ████████ UI (화면 최상단)
```

### SpriteBatch의 layerDepth 매개변수
SpriteBatch.Draw() 메서드의 마지막 매개변수인 layerDepth를 사용하면 깊이를 지정할 수 있다. 이 값은 0.0f부터 1.0f 사이로, 값이 작을수록 뒤에 있고 값이 클수록 앞에 있다. 

SpriteBatch를 사용할 때 layerDepth를 활용하려면 SpriteBatchOptions.None이나 SpriteSortMode를 지정해야 한다.

다음 예제는 여러 스프라이트를 깊이 순서대로 그리는 방법을 보여준다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using System.Collections.Generic;

public class LayeredScene
{
    public class DrawableObject
    {
        public Texture2D Texture { get; set; }
        public Vector2 Position { get; set; }
        public float LayerDepth { get; set; }  // 깊이 (0.0 ~ 1.0)
        public string Name { get; set; }
    }

    private List<DrawableObject> objects;

    public LayeredScene()
    {
        objects = new List<DrawableObject>();
    }

    public void AddObject(Texture2D texture, Vector2 position, 
                         float layerDepth, string name)
    {
        objects.Add(new DrawableObject
        {
            Texture = texture,
            Position = position,
            LayerDepth = layerDepth,
            Name = name
        });
    }

    public void Draw(SpriteBatch spriteBatch)
    {
        // layerDepth로 정렬하여 깊이 순서대로 그리기
        // 작은 값부터 큰 값 순서로 정렬 (뒤에서 앞으로)
        var sortedObjects = new List<DrawableObject>(objects);
        sortedObjects.Sort((a, b) => a.LayerDepth.CompareTo(b.LayerDepth));

        foreach (var obj in sortedObjects)
        {
            spriteBatch.Draw(
                obj.Texture,
                obj.Position,
                null,
                Color.White,
                0f,
                Vector2.Zero,
                1f,
                SpriteEffects.None,
                obj.LayerDepth  // layerDepth 지정
            );
        }
    }
}
```

**코드 설명:**

이 코드는 여러 DrawableObject를 리스트에 저장하고, layerDepth로 정렬한 후 그 순서대로 그린다. Sort() 메서드의 람다식에서 a.LayerDepth를 b.LayerDepth와 비교하므로, 작은 값이 먼저 그려진다. Draw() 메서드에서는 정렬된 순서대로 SpriteBatch.Draw()를 호출한다.

### 실전 예제: 타일 기반 게임의 레이어링
많은 2D 게임에서는 타일 기반 맵을 사용한다. 이 경우 캐릭터의 Y 좌표를 기반으로 깊이를 결정하는 것이 효과적이다. Y 값이 클수록 화면 아래에 있으므로, 뒤에 있는 것처럼 보이게 하려면 Y 값을 깊이로 정규화하여 사용한다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;

public class TileBasedScene
{
    private const int MapHeight = 20;  // 맵의 높이 (픽셀)

    public float CalculateDepth(Vector2 position)
    {
        // Y 좌표를 0 ~ 1 범위로 정규화
        // Y가 클수록 깊이도 커짐 (화면 아래가 앞)
        return position.Y / (MapHeight * 32);  // 32는 타일 크기
    }

    public void DrawCharacter(SpriteBatch spriteBatch, 
                             Texture2D texture, Vector2 position)
    {
        float depth = CalculateDepth(position);

        // 깊이가 1.0을 초과하지 않도록 제한
        depth = MathHelper.Clamp(depth, 0f, 0.99f);

        spriteBatch.Draw(
            texture,
            position,
            null,
            Color.White,
            0f,
            Vector2.Zero,
            1f,
            SpriteEffects.None,
            depth
        );
    }
}
```

**코드 설명:**

이 방식은 일반적인 2D 게임에서 매우 유용하다. Y 좌표가 클수록(화면 아래) 깊이 값도 커지므로, 같은 Y 좌표에 있는 객체들은 자동으로 같은 깊이를 갖는다. MathHelper.Clamp()를 사용하여 깊이를 0.0f부터 0.99f 범위로 제한하는데, 1.0f는 보통 UI 같은 최상단 요소용으로 예약해둔다.

### 과제: 애니메이션된 캐릭터 제작

**요구사항:**

3. 이동 방향에 따라 적절한 애니메이션을 재생한다.
4. 캐릭터가 정지하면 애니메이션도 정지한다.
5. 캐릭터 주변에 작은 파티클들이 회전하면서 펄싱 효과를 낸다.
6. 파티클들은 캐릭터보다 뒤에 표시되어야 한다.

**해야 할 작업:**

- Animation 클래스를 작성한다.
- 애니메이션된 캐릭터를 그리는 클래스를 작성한다.
- Game 클래스의 Update()와 Draw()에서 키 입력을 처리하고 화면에 그린다.
- 배경색은 밝은 회색(LightGray)으로 설정한다.

**테스트 항목:**

- 화살표 키로 캐릭터가 4방향으로 이동하는가?
- 각 방향 이동 시 맞는 애니메이션이 재생되는가?
- 이동을 멈추면 애니메이션도 멈추는가?
- 파티클이 캐릭터 주변에서 회전하고 펄싱하는가?
- 렌더링 순서(깊이)가 올바른가?
