## 13.1 벽돌 배치
이전 장에서는 패들과 공의 기본 물리를 구현했다. 이제 게임의 주요 요소인 벽돌을 만들고 화면에 배치해야 한다. 벽돌은 게임의 목표물이며, 공과의 충돌을 통해 파괴된다. 효율적인 벽돌 관리를 위해 리스트를 사용하여 여러 벽돌을 동시에 처리할 수 있도록 설계하겠다.

### 벽돌 클래스 설계
벽돌은 고정된 위치에 있는 직사각형 오브젝트로, 공과의 충돌 시 제거된다. 각 벽돌은 색깔과 점수 정보를 가질 수 있다.

```csharp
public class Brick : GameObject
{
    public float Width { get; set; }
    public float Height { get; set; }
    public int Points { get; set; }
    public Color BrickColor { get; set; }
    public bool IsDestroyed { get; set; }

    public Brick(Vector2 position, float width, float height, 
        int points, Color color) 
        : base(position)
    {
        Width = width;
        Height = height;
        Points = points;
        BrickColor = color;
        IsDestroyed = false;
    }

    public override void Update(GameTime gameTime)
    {
        // 벽돌은 움직이지 않으므로 특별한 업데이트가 필요 없다
    }

    public override void Draw(SpriteBatch spriteBatch)
    {
        if (!IsDestroyed)
        {
            Rectangle brickRect = new Rectangle(
                (int)(Position.X - Width / 2),
                (int)(Position.Y - Height / 2),
                (int)Width,
                (int)Height
            );
            
            // 벽돌을 색칠된 직사각형으로 그린다
            spriteBatch.Draw(
                texture: CreateColorTexture(1, 1, BrickColor),
                destinationRectangle: brickRect,
                color: BrickColor
            );

            // 벽돌의 경계선을 검은색으로 그린다
            spriteBatch.DrawRectangle(brickRect, Color.Black, 2);
        }
    }

    public Rectangle GetBounds()
    {
        return new Rectangle(
            (int)(Position.X - Width / 2),
            (int)(Position.Y - Height / 2),
            (int)Width,
            (int)Height
        );
    }

    public void Destroy()
    {
        IsDestroyed = true;
    }

    private Texture2D CreateColorTexture(int width, int height, Color color)
    {
        Texture2D texture = new Texture2D(GraphicsDevice, width, height);
        Color[] data = new Color[width * height];
        for (int i = 0; i < data.Length; i++)
        {
            data[i] = color;
        }
        texture.SetData(data);
        return texture;
    }
}
```

**코드 설명:**

`Brick` 클래스는 벽돌의 속성과 동작을 정의한다. `IsDestroyed` 플래그는 벽돌이 파괴되었는지 여부를 나타낸다. 처음에는 false로 시작하며, 공과의 충돌 시 true로 변경되어 화면에서 사라진다.

`Draw` 메서드는 파괴되지 않은 벽돌만 렌더링한다. `IsDestroyed`가 false일 때만 벽돌 내부를 색칠하고 경계선을 그린다. 이를 통해 파괴된 벽돌은 자동으로 화면에서 사라진다.

`Destroy` 메서드는 벽돌을 파괴 상태로 변경한다. 이 메서드는 나중에 충돌 처리 로직에서 호출될 것이다.

`CreateColorTexture` 메서드는 단색 텍스처를 만드는 헬퍼 메서드다. MonoGame에서는 텍스처 없이 직접 색깔을 그릴 수 없으므로, 1x1 픽셀의 흰 텍스처를 만든 후 색깔을 지정하여 직사각형을 그린다.

### 벽돌 배치 로직
게임 화면에 여러 벽돌을 격자 형태로 배치해야 한다. 게임의 `Game` 클래스에 벽돌 관리 코드를 추가하자.

```csharp
private List<Brick> bricks;
private const int BRICK_ROWS = 5;
private const int BRICK_COLS = 8;
private const float BRICK_WIDTH = 140;
private const float BRICK_HEIGHT = 25;
private const float BRICK_PADDING = 10;
private const float BRICK_START_X = 50;
private const float BRICK_START_Y = 50;

protected override void LoadContent()
{
    _spriteBatch = new SpriteBatch(GraphicsDevice);
    
    paddle = new Paddle(
        position: new Vector2(640, 650),
        width: 120,
        height: 20
    );
    
    ball = new Ball(
        position: new Vector2(640, 620),
        radius: 8,
        paddle: paddle
    );

    // 벽돌 배치
    InitializeBricks();
}

private void InitializeBricks()
{
    bricks = new List<Brick>();
    
    // 색깔 배열 (행마다 다른 색깔 사용)
    Color[] rowColors = new Color[]
    {
        Color.Red,
        Color.Orange,
        Color.Yellow,
        Color.Green,
        Color.Blue
    };

    // 격자 형태로 벽돌 생성
    for (int row = 0; row < BRICK_ROWS; row++)
    {
        for (int col = 0; col < BRICK_COLS; col++)
        {
            float x = BRICK_START_X + col * (BRICK_WIDTH + BRICK_PADDING) 
                + BRICK_WIDTH / 2;
            float y = BRICK_START_Y + row * (BRICK_HEIGHT + BRICK_PADDING) 
                + BRICK_HEIGHT / 2;
            
            // 행에 따라 점수를 다르게 설정 (아래쪽이 높은 점수)
            int points = (BRICK_ROWS - row) * 10;
            
            Brick brick = new Brick(
                position: new Vector2(x, y),
                width: BRICK_WIDTH,
                height: BRICK_HEIGHT,
                points: points,
                color: rowColors[row]
            );
            
            bricks.Add(brick);
        }
    }
}
```

**코드 설명:**

`InitializeBricks` 메서드는 게임 시작 시 모든 벽돌을 생성한다. 상수 값들을 이용하여 벽돌의 개수, 크기, 간격을 쉽게 조정할 수 있도록 설계했다.

벽돌을 격자 형태로 배치하기 위해 중첩 반복문을 사용한다. 바깥쪽 반복문은 행(row)을, 안쪽 반복문은 열(col)을 처리한다. 각 벽돌의 위치는 다음과 같이 계산된다.

```
X 좌표 = 시작 X + 열 × (벽돌 너비 + 간격) + 벽돌 너비 / 2
Y 좌표 = 시작 Y + 행 × (벽돌 높이 + 간격) + 벽돌 높이 / 2
```

행마다 다른 색깔을 지정하여 게임의 시각적 매력을 높인다. 또한 행 번호에 따라 점수를 다르게 설정하여, 위쪽의 어려운 벽돌일수록 높은 점수를 얻을 수 있게 한다.

## 13.2 충돌 처리
벽돌이 화면에 배치되었으니, 이제 공과 벽돌 사이의 충돌을 감지하고 처리해야 한다. 충돌 감지는 게임의 핵심 로직이며, 정확한 충돌 처리가 게임의 재미를 결정한다.

### 충돌 감지 알고리즘
앞서 9장에서 배운 AABB(Axis-Aligned Bounding Box) 충돌 감지를 사용한다. 원과 직사각형의 충돌을 정확하게 감지하기 위해, 원의 중심에서 직사각형까지의 최단 거리를 계산한다.

```csharp
public class CollisionDetection
{
    // 원과 직사각형의 충돌 감지
    public static bool CircleRectangleCollision(Vector2 circleCenter, 
        float circleRadius, Rectangle rect)
    {
        // 원의 중심에서 직사각형까지의 최단 거리 계산
        float closestX = MathHelper.Clamp(circleCenter.X, rect.Left, rect.Right);
        float closestY = MathHelper.Clamp(circleCenter.Y, rect.Top, rect.Bottom);

        // 최단 거리 계산
        float distanceX = circleCenter.X - closestX;
        float distanceY = circleCenter.Y - closestY;
        float distanceSquared = distanceX * distanceX + distanceY * distanceY;

        // 거리가 반지름보다 작으면 충돌
        return distanceSquared < circleRadius * circleRadius;
    }

    // 충돌 방향 판단
    public enum CollisionDirection
    {
        Top,
        Bottom,
        Left,
        Right,
        None
    }

    public static CollisionDirection GetCollisionDirection(
        Vector2 circleCenter, float circleRadius, Rectangle rect)
    {
        // 원의 중심을 직사각형 중심과의 관계로 판단
        float rectCenterX = rect.X + rect.Width / 2f;
        float rectCenterY = rect.Y + rect.Height / 2f;

        float dx = circleCenter.X - rectCenterX;
        float dy = circleCenter.Y - rectCenterY;

        float width = rect.Width / 2f;
        float height = rect.Height / 2f;

        // 원이 직사각형과 충돌했을 때 충돌한 변 판단
        float crossWidth = width * dy;
        float crossHeight = height * dx;

        if (crossWidth > crossHeight)
        {
            return crossWidth > -crossHeight ? CollisionDirection.Right : CollisionDirection.Top;
        }
        else
        {
            return crossWidth > -crossHeight ? CollisionDirection.Bottom : CollisionDirection.Left;
        }
    }
}
```

**코드 설명:**

`CircleRectangleCollision` 메서드는 원과 직사각형의 충돌을 정확하게 감지한다. 알고리즘의 핵심은 원의 중심에서 직사각형까지의 최단점을 찾는 것이다. `MathHelper.Clamp`를 사용하여 원의 중심 좌표를 직사각형의 범위 내로 제한하면, 그 결과가 직사각형과 원 사이의 최단점이 된다.

`GetCollisionDirection` 메서드는 충돌이 발생한 변을 판단한다. 원의 중심과 직사각형의 중심 사이의 상대적 위치를 계산하여, 어느 변에 충돌했는지 결정한다. 이 정보는 공의 반사 방향을 결정할 때 사용된다.

### 벽돌과의 충돌 처리
게임의 `Update` 메서드에 벽돌과의 충돌 처리 로직을 추가한다.

```csharp
protected override void Update(GameTime gameTime)
{
    if (GamePad.GetState(PlayerIndex.One).Buttons.Back == ButtonState.Pressed 
        || Keyboard.GetState().IsKeyDown(Keys.Escape))
        Exit();

    paddle.Update(gameTime);
    ball.Update(gameTime);

    // 공과 패들의 충돌 감지
    if (ball.IsLaunched && 
        ball.GetBounds().Intersects(paddle.GetBounds()))
    {
        ball.ReflectOffPaddle(paddle);
    }

    // 공과 벽돌의 충돌 감지
    foreach (Brick brick in bricks)
    {
        if (!brick.IsDestroyed && 
            CollisionDetection.CircleRectangleCollision(
                ball.Position, ball.Radius, brick.GetBounds()))
        {
            brick.Destroy();
            
            // 충돌 방향에 따라 공의 반사 방향 결정
            CollisionDetection.CollisionDirection direction = 
                CollisionDetection.GetCollisionDirection(
                    ball.Position, ball.Radius, brick.GetBounds());
            
            ReflectBallFromBrick(direction);
        }
    }

    base.Update(gameTime);
}

private void ReflectBallFromBrick(
    CollisionDetection.CollisionDirection direction)
{
    switch (direction)
    {
        case CollisionDetection.CollisionDirection.Top:
        case CollisionDetection.CollisionDirection.Bottom:
            ball.Velocity.Y *= -1;
            break;
        case CollisionDetection.CollisionDirection.Left:
        case CollisionDetection.CollisionDirection.Right:
            ball.Velocity.X *= -1;
            break;
    }
}
```

**코드 설명:**

벽돌과의 충돌 처리는 패들과의 충돌과 유사하지만, 더 간단하다. 먼저 `CollisionDetection.CircleRectangleCollision`으로 충돌 여부를 확인하고, 충돌했다면 해당 벽돌을 파괴한다.

충돌한 변의 방향을 `GetCollisionDirection`으로 판단하여, 공의 속도 벡터를 적절히 반사시킨다. 위아래 변에 충돌했으면 Y 속도를 반전시키고, 좌우 변에 충돌했으면 X 속도를 반전시킨다.

### 충돌 방향 판단 로직의 이해
다음 다이어그램은 원과 직사각형의 충돌 방향을 판단하는 방식을 보여준다.

```
┌─────────────────┐
│   Top           │
│                 │
│ L     Center    R│
│                 │
│   Bottom        │
└─────────────────┘

원의 중심이 직사각형의:
- 위쪽 절반에 있으면: Top 또는 Left/Right
- 아래쪽 절반에 있으면: Bottom 또는 Left/Right
- 왼쪽 절반에 있으면: Left 또는 Top/Bottom
- 오른쪽 절반에 있으면: Right 또는 Top/Bottom

정확한 판단은 교차곱(cross product) 계산으로 결정된다.
```

## 13.3 점수 시스템
게임의 재미를 높이기 위해 점수 시스템을 구현해야 한다. 플레이어가 벽돌을 깨뜨릴 때마다 점수를 얻고, 현재 점수를 화면에 표시할 것이다.

### 점수 관리 클래스
게임 상태를 관리하는 클래스를 만들어 점수, 생명, 레벨 등을 관리하도록 하자.

```csharp
public class GameState
{
    public int Score { get; set; }
    public int Lives { get; set; }
    public int Level { get; set; }
    public int BricksDestroyed { get; set; }
    public int TotalBricks { get; set; }

    public GameState()
    {
        Score = 0;
        Lives = 3;
        Level = 1;
        BricksDestroyed = 0;
        TotalBricks = 0;
    }

    public void AddScore(int points)
    {
        Score += points;
    }

    public void DecreaseLives()
    {
        if (Lives > 0)
        {
            Lives--;
        }
    }

    public bool IsGameOver()
    {
        return Lives <= 0;
    }

    public bool IsClear()
    {
        return BricksDestroyed == TotalBricks && TotalBricks > 0;
    }

    public void Reset()
    {
        Score = 0;
        Lives = 3;
        Level = 1;
        BricksDestroyed = 0;
    }
}
```

**코드 설명:**

`GameState` 클래스는 게임의 현재 상태를 관리한다. 점수, 생명, 레벨, 파괴된 벽돌의 개수 등을 추적한다.

`AddScore` 메서드는 벽돌 파괴 시 점수를 추가한다. `DecreaseLives` 메서드는 생명을 감소시키되, 생명이 이미 0이면 음수가 되지 않도록 확인한다.

`IsGameOver`와 `IsClear` 메서드는 각각 게임 오버와 게임 클리어 조건을 확인한다. 이 메서드들은 나중에 게임 상태 관리 로직에서 사용될 것이다.

### 점수 시스템 통합
게임의 주 클래스에 게임 상태를 통합한다.

```csharp
private GameState gameState;
```

벽돌 파괴 시 점수를 추가하도록 `Update` 메서드를 수정한다.

```csharp
protected override void Update(GameTime gameTime)
{
    if (GamePad.GetState(PlayerIndex.One).Buttons.Back == ButtonState.Pressed 
        || Keyboard.GetState().IsKeyDown(Keys.Escape))
        Exit();

    paddle.Update(gameTime);
    ball.Update(gameTime);

    // 공과 패들의 충돌 감지
    if (ball.IsLaunched && 
        ball.GetBounds().Intersects(paddle.GetBounds()))
    {
        ball.ReflectOffPaddle(paddle);
    }

    // 공과 벽돌의 충돌 감지
    foreach (Brick brick in bricks)
    {
        if (!brick.IsDestroyed && 
            CollisionDetection.CircleRectangleCollision(
                ball.Position, ball.Radius, brick.GetBounds()))
        {
            brick.Destroy();
            gameState.AddScore(brick.Points);
            gameState.BricksDestroyed++;
            
            CollisionDetection.CollisionDirection direction = 
                CollisionDetection.GetCollisionDirection(
                    ball.Position, ball.Radius, brick.GetBounds());
            
            ReflectBallFromBrick(direction);
        }
    }

    // 공이 화면 하단으로 떨어졌을 때
    if (ball.Position.Y > 720)
    {
        gameState.DecreaseLives();
        ball.ResetBall();
        
        if (gameState.IsGameOver())
        {
            // 게임 오버 처리 (14장에서 구현)
        }
    }

    base.Update(gameTime);
}
```

### 점수 화면 표시
점수를 화면에 표시하려면 텍스트 렌더링이 필요하다. 이전 장에서 배운 폰트를 사용하여 점수를 표시하자.

```csharp
private SpriteFont gameFont;

protected override void LoadContent()
{
    _spriteBatch = new SpriteBatch(GraphicsDevice);

    _spriteBatch.DrawString(gameFont, livesText,
```

### 점수 시스템의 게임 디자인
벽돌의 점수를 설계할 때는 게임의 난이도와 균형을 고려해야 한다. 일반적으로 다음과 같은 원칙을 따른다.

```
점수 배분 규칙:
- 접근하기 쉬운 아래쪽 벽돌: 낮은 점수 (10점)
- 중간 난이도 벽돌: 중간 점수 (20~30점)
- 도달하기 어려운 위쪽 벽돌: 높은 점수 (40~50점)

총 벽돌 40개 (5행 × 8열) 기준:
- 행 1 (맨 아래): 8개 × 10점 = 80점
- 행 2: 8개 × 20점 = 160점
- 행 3: 8개 × 30점 = 240점
- 행 4: 8개 × 40점 = 320점
- 행 5 (맨 위): 8개 × 50점 = 400점

게임 총 최대 점수: 1,200점
```

이런 점수 체계는 플레이어에게 모든 벽돌을 파괴하려는 동기를 부여하고, 어려운 벽돌에 도달하기 위한 기술 습득을 장려한다.

## 과제
이 장에서는 벽돌 배치, 충돌 처리, 점수 시스템을 구현했다. 다음 과제들을 통해 학습한 내용을 심화시켜보자.

**과제 1: 특수 벽돌 구현**

현재는 모든 벽돌이 동일하지만, 특수한 속성을 가진 벽돌을 추가해보자. 다음과 같은 특수 벽돌들을 구현하자.

- 강화 벽돌(Strong Brick): 한 번의 충돌로 파괴되지 않고, 두 번 맞아야 파괴되는 벽돌. 기본 벽돌과는 다른 색으로 표시하자. (예: 회색)
- 생명 회복 벽돌(Life Brick): 파괴 시 플레이어의 생명을 1개 증가시키는 벽돌. (예: 녹색)
- 보너스 점수 벽돌(Bonus Brick): 일반 벽돌보다 높은 점수를 주는 벽돌. (예: 금색)

`Brick` 클래스를 상속하여 `StrongBrick`, `LifeBrick`, `BonusBrick` 클래스를 만들고, 각 벽돌의 고유한 동작을 구현하자.

**과제 2: 동적 벽돌 배치**

현재는 게임 시작 시 항상 같은 패턴으로 벽돌이 배치된다. 난이도 단계에 따라 다른 벽돌 배치를 생성하는 메서드를 구현해보자. 예를 들어, 레벨 1에서는 일반 벽돌만 배치하고, 레벨 2에서는 강화 벽돌을 추가하는 식으로 구현할 수 있다.

`InitializeBricks()` 메서드를 확장하여 `InitializeBricks(int level)` 형태로 변경하고, 레벨에 따라 다른 벽돌을 배치하도록 하자.

**과제 3: 충돌 음향 효과**

현재는 시각적 충돌 처리만 구현했다. 벽돌과의 충돌, 패들과의 충돌, 벽면과의 충돌 시 각각 다른 음향 효과를 재생하도록 코드를 수정해보자. (이를 위해 11장의 사운드 이펙트 로드와 재생 방법을 참고하자.)
