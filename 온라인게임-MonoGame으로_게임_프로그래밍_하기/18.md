# 18장. 성능 최적화
게임 개발에서 성능 최적화는 매우 중요한 주제다. 아무리 멋진 기능을 구현했더라도 게임이 버벅거리거나 느리면 플레이어의 경험이 급격히 떨어진다. 특히 모바일 플랫폼이나 저사양 PC에서 게임을 배포할 계획이라면 성능 관리는 선택이 아닌 필수다.

이 장에서는 MonoGame에서 게임의 성능을 효과적으로 관리하고 개선하는 방법을 배운다. 프레임레이트 관리부터 시작해서 메모리 낭비를 줄이는 오브젝트 풀링, 그리고 게임의 병목 지점을 찾아내는 프로파일링 기법까지 다룬다.
  

## 18.1 프레임레이트 관리

### 프레임레이트란 무엇인가
프레임레이트(Frame Rate)는 게임이 1초에 몇 번 화면을 그리는지를 나타내는 지표다. 일반적으로 FPS(Frames Per Second)라고 표현한다. 예를 들어 60 FPS는 1초에 60번 화면을 그린다는 의미다.

사람의 눈은 보통 60 FPS 이상의 화면을 봤을 때 부드럽다고 느낀다. 반면 30 FPS 이하면 끊기는 느낌을 받는다. 따라서 대부분의 게임은 60 FPS를 목표로 한다. 다만 하드웨어 제약이 있다면 30 FPS로 설정하기도 한다.

### TargetElapsedTime으로 프레임레이트 고정하기
MonoGame에서는 `TargetElapsedTime` 속성을 사용해서 목표 프레임레이트를 설정한다. 이 값을 작게 할수록 더 빈번하게 업데이트가 호출된다.

```csharp
protected override void Initialize()
{
    base.Initialize();
    
    // 목표 프레임레이트를 60 FPS로 설정
    // TimeSpan.FromSeconds(1.0 / 60.0)은 약 0.0167초
    TargetElapsedTime = TimeSpan.FromSeconds(1.0 / 60.0);
    
    // IsFixedTimeStep을 true로 유지하면 위 설정이 적용된다
    IsFixedTimeStep = true;
}
```

`IsFixedTimeStep` 속성은 게임이 고정된 시간 간격으로 업데이트되는지를 제어한다. `true`로 설정하면 `TargetElapsedTime`에 설정된 시간 간격으로 정확하게 `Update` 메서드가 호출된다. `false`로 설정하면 프로세서가 가능한 한 빠르게 업데이트를 실행한다.

고정된 업데이트는 게임 로직의 일관성을 보장하기 때문에 대부분의 게임에서 권장된다. 프레임이 떨어지는 상황에서도 게임 로직이 예측 가능하게 동작하기 때문이다.

### 실제 FPS 측정하기
게임이 실제로 몇 FPS로 동작하는지 측정하려면 경과 시간을 추적해야 한다. 다음은 간단한 FPS 카운터 구현 예제다.

```csharp
public class Game1 : Game
{
    private GraphicsDeviceManager _graphics;
    private SpriteBatch _spriteBatch;
    private SpriteFont _font;
    
    private double _elapsedTime = 0;
    private int _frameCount = 0;
    private int _currentFPS = 0;

    public Game1()
    {
        _graphics = new GraphicsDeviceManager(this);
        Content.RootDirectory = "Content";
    }

    protected override void Initialize()
    {
        base.Initialize();
        TargetElapsedTime = TimeSpan.FromSeconds(1.0 / 60.0);
        IsFixedTimeStep = true;
    }

    protected override void LoadContent()
    {
        _spriteBatch = new SpriteBatch(GraphicsDevice);
        _font = Content.Load<SpriteFont>("Arial");
    }

    protected override void Update(GameTime gameTime)
    {
        if (GamePad.GetState(PlayerIndex.One).Buttons.Back == ButtonState.Pressed)
            Exit();

        base.Update(gameTime);
    }

    protected override void Draw(GameTime gameTime)
    {
        GraphicsDevice.Clear(Color.CornflowerBlue);

        // FPS 계산
        _elapsedTime += gameTime.ElapsedGameTime.TotalSeconds;
        _frameCount++;

        if (_elapsedTime >= 1.0)
        {
            _currentFPS = _frameCount;
            _frameCount = 0;
            _elapsedTime -= 1.0;
        }

        _spriteBatch.Begin();
        _spriteBatch.DrawString(_font, $"FPS: {_currentFPS}", 
            new Vector2(10, 10), Color.White);
        _spriteBatch.End();

        base.Draw(gameTime);
    }
}
```

이 코드는 1초 동안 몇 번 `Draw` 메서드가 호출되었는지 세어서 실제 FPS를 계산한다. 게임의 우측 상단에 현재 FPS를 표시한다. 하드웨어 성능이나 다른 요인에 따라 목표 프레임레이트와 실제 FPS가 다를 수 있으므로, 이런 식으로 모니터링하면 최적화의 효과를 측정할 수 있다.

### VSync와 배터리 절약
MonoGame은 기본적으로 VSync(수직 동기화)를 활성화하고 있다. VSync는 모니터의 재생 빈도에 맞춰서 화면을 그린다. 예를 들어 60Hz 모니터라면 최대 60 FPS로 제한된다. 이는 화면 테어링(Tearing) 현상을 방지한다.

배터리 절약이 중요한 모바일 게임에서는 `IsMouseVisible`과 함께 `InactiveSleepTime`을 설정해서 게임이 포커스를 잃었을 때 업데이트 빈도를 낮출 수 있다.

```csharp
protected override void Initialize()
{
    base.Initialize();
    
    // 게임이 비활성화되었을 때 대기 시간 설정
    // 배터리를 절약하기 위해 업데이트 빈도를 낮춤
    InactiveSleepTime = TimeSpan.FromSeconds(1.0 / 30.0);
}
```

## 18.2 오브젝트 풀링

### 오브젝트 풀링이 필요한 이유
게임에서 총알, 폭발, 파티클 같은 오브젝트를 자주 생성하고 삭제한다면 메모리 할당과 해제가 반복된다. C#의 가비지 컬렉션(Garbage Collection) 때문에 이런 작업이 발생할 때마다 게임의 성능이 저하된다.

오브젝트 풀링은 미리 일정한 수의 오브젝트를 생성해 두고, 필요할 때마다 그 오브젝트를 재사용하는 기법이다. 이렇게 하면 런타임에 메모리 할당과 해제가 줄어들어서 성능이 개선된다.

### 간단한 오브젝트 풀 구현
먼저 풀로 관리할 기본 오브젝트를 정의한다.

```csharp
public class Bullet
{
    public Vector2 Position { get; set; }
    public Vector2 Velocity { get; set; }
    public bool IsActive { get; set; }
    public float Speed { get; set; } = 300f;

    public Bullet()
    {
        IsActive = false;
    }

    public void Update(GameTime gameTime)
    {
        if (!IsActive) return;

        Position += Velocity * (float)gameTime.ElapsedGameTime.TotalSeconds;
    }

    public void Draw(SpriteBatch spriteBatch, Texture2D texture)
    {
        if (!IsActive) return;
        spriteBatch.Draw(texture, Position, Color.White);
    }
}
```

이제 이 총알을 관리하는 오브젝트 풀을 만든다. 풀은 일정한 크기를 유지하면서 비활성 오브젝트를 재사용한다.

```csharp
public class BulletPool
{
    private List<Bullet> _bullets;
    private readonly int _poolSize;

    public BulletPool(int poolSize = 100)
    {
        _poolSize = poolSize;
        _bullets = new List<Bullet>(poolSize);

        // 미리 모든 총알 객체를 생성
        for (int i = 0; i < poolSize; i++)
        {
            _bullets.Add(new Bullet());
        }
    }

    /// <summary>
    /// 비활성 상태의 총알을 풀에서 가져온다.
    /// 비활성 총알이 없으면 새로운 총알을 생성한다.
    /// </summary>
    public Bullet GetBullet()
    {
        // 이미 생성된 비활성 총알을 찾는다
        foreach (var bullet in _bullets)
        {
            if (!bullet.IsActive)
            {
                bullet.IsActive = true;
                return bullet;
            }
        }

        // 풀에 여유가 있으면 새 총알을 생성해서 반환
        if (_bullets.Count < _poolSize)
        {
            var newBullet = new Bullet { IsActive = true };
            _bullets.Add(newBullet);
            return newBullet;
        }

        // 풀이 가득 찬 경우 경고 후 null 반환
        System.Diagnostics.Debug.WriteLine("Bullet pool is full!");
        return null;
    }

    /// <summary>
    /// 주어진 위치와 속도로 새 총알을 발사한다.
    /// </summary>
    public void Fire(Vector2 position, Vector2 direction)
    {
        var bullet = GetBullet();
        if (bullet != null)
        {
            bullet.Position = position;
            bullet.Velocity = direction * bullet.Speed;
        }
    }

    /// <summary>
    /// 모든 활성 총알을 업데이트하고,
    /// 화면을 벗어난 총알을 비활성화한다.
    /// </summary>
    public void Update(GameTime gameTime, int screenWidth, int screenHeight)
    {
        foreach (var bullet in _bullets)
        {
            if (!bullet.IsActive) continue;

            bullet.Update(gameTime);

            // 화면을 벗어난 총알 비활성화
            if (bullet.Position.X < -10 || bullet.Position.X > screenWidth + 10 ||
                bullet.Position.Y < -10 || bullet.Position.Y > screenHeight + 10)
            {
                bullet.IsActive = false;
            }
        }
    }

    /// <summary>
    /// 모든 활성 총알을 그린다.
    /// </summary>
    public void Draw(SpriteBatch spriteBatch, Texture2D bulletTexture)
    {
        foreach (var bullet in _bullets)
        {
            bullet.Draw(spriteBatch, bulletTexture);
        }
    }

    /// <summary>
    /// 모든 총알을 비활성화한다.
    /// </summary>
    public void Clear()
    {
        foreach (var bullet in _bullets)
        {
            bullet.IsActive = false;
        }
    }

    /// <summary>
    /// 현재 활성 상태인 총알의 개수를 반환한다.
    /// </summary>
    public int GetActiveBulletCount()
    {
        int count = 0;
        foreach (var bullet in _bullets)
        {
            if (bullet.IsActive) count++;
        }
        return count;
    }
}
```

이 코드는 총알을 효율적으로 관리한다. 생성자에서 지정된 크기만큼 총알 객체를 미리 만들어 두고, `GetBullet` 메서드로 비활성 총알을 재사용한다. 만약 모든 총알이 활성 상태라면 새로운 총알을 생성한다. 화면을 벗어난 총알은 자동으로 비활성화되어서 재사용을 기다린다.

이제 게임 클래스에서 이 풀을 사용하는 방법을 보자.

```csharp
public class Game1 : Game
{
    private GraphicsDeviceManager _graphics;
    private SpriteBatch _spriteBatch;
    private Texture2D _bulletTexture;
    private BulletPool _bulletPool;
    private Vector2 _playerPosition;

    public Game1()
    {
        _graphics = new GraphicsDeviceManager(this);
        Content.RootDirectory = "Content";
    }

    protected override void Initialize()
    {
        base.Initialize();
        // 최대 500개의 총알을 풀에서 관리
        _bulletPool = new BulletPool(500);
        _playerPosition = new Vector2(400, 350);
    }

    protected override void LoadContent()
    {
        _spriteBatch = new SpriteBatch(GraphicsDevice);
        _bulletTexture = Content.Load<Texture2D>("bullet");
    }

    protected override void Update(GameTime gameTime)
    {
        var keyState = Keyboard.GetState();
        if (keyState.IsKeyDown(Keys.Space))
        {
            // 위쪽 방향으로 총알 발사
            _bulletPool.Fire(_playerPosition, Vector2.UnitY * -1);
        }

        _bulletPool.Update(gameTime, 800, 700);

        base.Update(gameTime);
    }

    protected override void Draw(GameTime gameTime)
    {
        GraphicsDevice.Clear(Color.Black);

        _spriteBatch.Begin();
        _bulletPool.Draw(_spriteBatch, _bulletTexture);
        _spriteBatch.End();

        base.Draw(gameTime);
    }
}
```

오브젝트 풀을 사용하면 메모리 할당과 해제가 크게 줄어든다. 특히 초당 수십 개의 오브젝트가 생성되고 삭제되는 상황에서는 성능 차이가 눈에 띄게 난다.
    
### 다양한 오브젝트 타입 풀링
게임에서는 여러 종류의 오브젝트를 동시에 풀링해야 할 수도 있다. 예를 들어 총알, 폭발, 아이템 등을 각각 관리해야 한다면 제네릭을 사용한 범용 풀을 만드는 것이 효율적이다.

```csharp
/// <summary>
/// 모든 오브젝트 타입을 관리할 수 있는 범용 오브젝트 풀이다.
/// T는 풀에서 관리할 오브젝트의 타입이다.
/// </summary>
public class ObjectPool<T> where T : class, new()
{
    private Stack<T> _availableObjects;
    private HashSet<T> _activeObjects;
    private readonly int _initialSize;

    public ObjectPool(int initialSize = 50)
    {
        _initialSize = initialSize;
        _availableObjects = new Stack<T>(initialSize);
        _activeObjects = new HashSet<T>();

        // 초기 객체들을 미리 생성
        for (int i = 0; i < initialSize; i++)
        {
            _availableObjects.Push(new T());
        }
    }

    /// <summary>
    /// 풀에서 객체를 가져온다.
    /// 사용 가능한 객체가 없으면 새로운 객체를 생성한다.
    /// </summary>
    public T Get()
    {
        T obj;

        if (_availableObjects.Count > 0)
        {
            obj = _availableObjects.Pop();
        }
        else
        {
            obj = new T();
        }

        _activeObjects.Add(obj);
        return obj;
    }

    /// <summary>
    /// 사용을 마친 객체를 풀에 반환한다.
    /// </summary>
    public void Return(T obj)
    {
        if (_activeObjects.Remove(obj))
        {
            _availableObjects.Push(obj);
        }
    }

    /// <summary>
    /// 현재 활성 상태인 객체의 개수를 반환한다.
    /// </summary>
    public int GetActiveCount()
    {
        return _activeObjects.Count;
    }

    /// <summary>
    /// 현재 사용 가능한 객체의 개수를 반환한다.
    /// </summary>
    public int GetAvailableCount()
    {
        return _availableObjects.Count;
    }
}
```

`Stack` 자료구조를 사용하면 객체 추가와 제거가 O(1) 시간에 이루어진다. `HashSet`으로 활성 객체를 추적하면 어떤 객체가 사용 중인지 빠르게 파악할 수 있다.

## 18.3 프로파일링 기법

### 프로파일링이란
프로파일링(Profiling)은 게임의 어느 부분에서 가장 많은 시간과 메모리를 사용하는지 측정하는 과정이다. 성능 최적화를 시작하기 전에 반드시 프로파일링을 해야 한다. 왜냐하면 병목 지점을 정확히 파악하지 못한 채 최적화하면 오히려 코드가 복잡해질 수 있기 때문이다.

### 간단한 성능 측정 도구 만들기
게임에서 특정 작업의 실행 시간을 측정하는 간단한 도구를 만들어 보자.

```csharp
/// <summary>
/// 게임의 특정 구간의 실행 시간을 측정하고 기록하는 클래스다.
/// 여러 구간을 동시에 측정할 수 있다.
/// </summary>
public class PerformanceMonitor
{
    private Dictionary<string, Stopwatch> _stopwatches;
    private Dictionary<string, long> _totalTicks;
    private Dictionary<string, int> _callCounts;

    public PerformanceMonitor()
    {
        _stopwatches = new Dictionary<string, Stopwatch>();
        _totalTicks = new Dictionary<string, long>();
        _callCounts = new Dictionary<string, int>();
    }

    /// <summary>
    /// 주어진 이름으로 새로운 측정 구간을 시작한다.
    /// </summary>
    public void Begin(string label)
    {
        if (!_stopwatches.ContainsKey(label))
        {
            _stopwatches[label] = new Stopwatch();
            _totalTicks[label] = 0;
            _callCounts[label] = 0;
        }

        _stopwatches[label].Restart();
    }

    /// <summary>
    /// 주어진 이름의 측정 구간을 종료하고 결과를 기록한다.
    /// </summary>
    public void End(string label)
    {
        if (_stopwatches.ContainsKey(label))
        {
            _stopwatches[label].Stop();
            _totalTicks[label] += _stopwatches[label].ElapsedTicks;
            _callCounts[label]++;
        }
    }

    /// <summary>
    /// 주어진 이름의 측정 결과를 밀리초 단위로 반환한다.
    /// </summary>
    public double GetMilliseconds(string label)
    {
        if (_totalTicks.ContainsKey(label) && _callCounts[label] > 0)
        {
            return (_totalTicks[label] / (double)Stopwatch.Frequency) * 1000.0;
        }
        return 0;
    }

    /// <summary>
    /// 주어진 이름의 호출 횟수를 반환한다.
    /// </summary>
    public int GetCallCount(string label)
    {
        return _callCounts.ContainsKey(label) ? _callCounts[label] : 0;
    }

    /// <summary>
    /// 모든 측정 통계를 문자열로 반환한다.
    /// </summary>
    public string GetReport()
    {
        var sb = new StringBuilder();
        sb.AppendLine("=== Performance Report ===");

        foreach (var label in _totalTicks.Keys)
        {
            double totalMs = (_totalTicks[label] / (double)Stopwatch.Frequency) * 1000.0;
            double avgMs = totalMs / _callCounts[label];

            sb.AppendLine($"{label}:");
            sb.AppendLine($"  Total: {totalMs:F2}ms");
            sb.AppendLine($"  Average: {avgMs:F4}ms");
            sb.AppendLine($"  Calls: {_callCounts[label]}");
        }

        return sb.ToString();
    }

    /// <summary>
    /// 모든 측정 통계를 초기화한다.
    /// </summary>
    public void Reset()
    {
        _stopwatches.Clear();
        _totalTicks.Clear();
        _callCounts.Clear();
    }
}
```

이 클래스는 `System.Diagnostics.Stopwatch`를 사용해서 정확한 시간 측정을 한다. 여러 구간을 동시에 측정하고, 누적 시간과 평균 시간을 계산할 수 있다.

### 게임에서 프로파일링 사용하기
이제 게임 클래스에서 이 모니터를 사용해 보자.

```csharp
using System;
using System.Diagnostics;
using System.Text;
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;

public class Game1 : Game
{
    private GraphicsDeviceManager _graphics;
    private SpriteBatch _spriteBatch;
    private SpriteFont _font;
    private PerformanceMonitor _monitor;
    
    private BulletPool _bulletPool;
    private List<GameObject> _gameObjects;
    private Vector2 _playerPosition;

    public Game1()
    {
        _graphics = new GraphicsDeviceManager(this);
        Content.RootDirectory = "Content";
    }

    protected override void Initialize()
    {
        base.Initialize();
        
        _monitor = new PerformanceMonitor();
        _bulletPool = new BulletPool(1000);
        _gameObjects = new List<GameObject>(500);
        _playerPosition = new Vector2(400, 350);

        // 테스트용 게임 오브젝트 100개 생성
        for (int i = 0; i < 100; i++)
        {
            _gameObjects.Add(new GameObject 
            { 
                Position = new Vector2(i % 10 * 80, i / 10 * 80) 
            });
        }
    }

    protected override void LoadContent()
    {
        _spriteBatch = new SpriteBatch(GraphicsDevice);
        _font = Content.Load<SpriteFont>("Arial");
    }

    protected override void Update(GameTime gameTime)
    {
        // 전체 업데이트 시간 측정
        _monitor.Begin("Total Update");

        var keyState = Keyboard.GetState();
        if (keyState.IsKeyDown(Keys.Space))
        {
            _monitor.Begin("Fire Bullet");
            _bulletPool.Fire(_playerPosition, Vector2.UnitY * -1);
            _monitor.End("Fire Bullet");
        }

        // 총알 업데이트 시간 측정
        _monitor.Begin("Bullet Update");
        _bulletPool.Update(gameTime, 800, 600);
        _monitor.End("Bullet Update");

        // 게임 오브젝트 업데이트 시간 측정
        _monitor.Begin("GameObject Update");
        foreach (var obj in _gameObjects)
        {
            obj.Update(gameTime);
        }
        _monitor.End("GameObject Update");

        _monitor.End("Total Update");

        base.Update(gameTime);
    }

    protected override void Draw(GameTime gameTime)
    {
        GraphicsDevice.Clear(Color.Black);

        // 전체 드로우 시간 측정
        _monitor.Begin("Total Draw");

        _spriteBatch.Begin();

        // 게임 오브젝트 렌더링
        _monitor.Begin("GameObject Draw");
        foreach (var obj in _gameObjects)
        {
            obj.Draw(_spriteBatch);
        }
        _monitor.End("GameObject Draw");

        // 성능 정보 표시
        _spriteBatch.DrawString(_font, _monitor.GetReport(), 
            new Vector2(10, 10), Color.Lime);

        _spriteBatch.End();

        _monitor.End("Total Draw");

        base.Draw(gameTime);
    }
}
```

이 코드에서 각 주요 작업 구간을 측정하고 있다. `Begin`으로 시작하고 `End`로 종료하면 자동으로 시간이 기록된다. 게임 화면에 실시간 성능 리포트가 표시되므로 최적화 전후의 성능 변화를 명확하게 볼 수 있다.

### 메모리 사용량 추적

프로파일링은 시간뿐만 아니라 메모리도 측정해야 한다. 다음은 메모리 사용량을 간단히 추적하는 방법이다.

```csharp
/// <summary>
/// 게임의 메모리 사용량을 추적하는 클래스다.
/// </summary>
public class MemoryMonitor
{
    private long _initialMemory;

    public MemoryMonitor()
    {
        _initialMemory = GC.GetTotalMemory(false);
    }

    /// <summary>
    /// 현재 메모리 사용량을 바이트 단위로 반환한다.
    /// </summary>
    public long GetCurrentMemory()
    {
        return GC.GetTotalMemory(false);
    }

    /// <summary>
    /// 시작 이후 증가한 메모리 양을 바이트 단위로 반환한다.
    /// </summary>
    public long GetMemoryDelta()
    {
        return GetCurrentMemory() - _initialMemory;
    }

    /// <summary>
    /// 메모리 사용량을 메가바이트 단위로 포맷된 문자열로 반환한다.
    /// </summary>
    public string GetMemoryReport()
    {
        long currentMb = GetCurrentMemory() / (1024 * 1024);
        long deltaMb = GetMemoryDelta() / (1024 * 1024);
        return $"Memory: {currentMb}MB (Delta: {deltaMb:+#;-#;0}MB)";
    }
}
```

`GC.GetTotalMemory`는 현재 관리되는 메모리의 총량을 반환한다. 게임 시작 시점과 현재를 비교하면 메모리가 얼마나 증가했는지 알 수 있다. 메모리가 계속 증가한다면 메모리 누수가 있을 수 있으므로 조사해야 한다.

### 프로파일링 결과 해석
프로파일링 결과를 해석할 때는 다음 사항을 고려해야 한다.

첫째, 가장 시간을 많이 사용하는 부분부터 최적화한다. 전체 실행 시간의 80%를 차지하는 20%의 코드에 집중하는 것이 효율적이다.

둘째, 호출 횟수가 많은 코드를 주의해야 한다. 함수가 초당 수천 번 호출된다면 아주 작은 성능 개선도 전체 게임 성능에 큰 영향을 미친다.

셋째, 메모리 누수를 찾는다. 메모리가 계속 증가하면 결국 게임이 종료될 때까지 성능이 계속 저하된다. 특히 `Update` 메서드 안에서 반복적으로 새 객체를 생성하는 코드가 있는지 확인한다.

```csharp
// 나쁜 예: Update에서 매번 새 리스트를 생성한다
protected override void Update(GameTime gameTime)
{
    var nearbyObjects = new List<GameObject>(); // 메모리 낭비!
    // ...
}

// 좋은 예: 미리 생성한 리스트를 재사용한다
private List<GameObject> _nearbyObjects = new List<GameObject>();

protected override void Update(GameTime gameTime)
{
    _nearbyObjects.Clear(); // 내용만 비우고 객체는 재사용
    // ...
}
```

## 과제
이제 배운 성능 최적화 기법을 실제로 적용해 보자.

**과제 1: FPS 모니터 구현**

앞서 배운 FPS 카운터 구현을 기반으로, 다음 정보를 화면에 표시하는 프로그램을 만들어라:
- 현재 FPS
- 최대 FPS (지금까지 측정한 최고값)
- 최소 FPS (지금까지 측정한 최저값)
- 평균 FPS

마우스로 원을 드래그하면서 성능이 어떻게 변하는지 관찰하면 좋다.

**과제 2: 오브젝트 풀 활용**

벽돌깨기 게임에서 사용한 `Bullet` 클래스를 `ObjectPool<T>` 제네릭 풀로 관리하도록 리팩토링하라. 다음을 확인해야 한다:
- 총알을 발사해도 메모리가 계속 증가하지 않아야 한다
- 풀에서 비활성 총알의 개수를 모니터링하라
- 풀 크기를 변경해서 성능 변화를 관찰하라

**과제 3: 프로파일링 분석**

앞서 만든 게임에서 다음을 프로파일링하라:
- 오브젝트 업데이트 시간
- 충돌 감지 시간
- 렌더링 시간

어느 부분이 가장 많은 시간을 소비하는지 찾아내고, 그 결과를 정리해서 제출하라. 최적화 가능한 부분이 있다면 제안하라.

**과제 4: 메모리 누수 찾기**

다음 코드에서 메모리 누수를 찾아내고 수정하라:

```csharp
public class GameManager : Game
{
    private List<Particle> _particles;

    protected override void Update(GameTime gameTime)
    {
        // 매 프레임 새 리스트를 생성한다
        var activeParticles = new List<Particle>();
        
        foreach (var p in _particles)
        {
            p.Update(gameTime);
            if (p.IsAlive)
            {
                activeParticles.Add(p);
            }
        }

        _particles = activeParticles; // 기존 리스트는 가비지가 된다
    }
}
```

이 코드의 문제점을 분석하고, 메모리 효율적으로 수정한 버전을 만들어라.
