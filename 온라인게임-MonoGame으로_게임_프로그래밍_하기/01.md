# 1장. MonoGame 개발 환경 구축
이제 우리는 MonoGame의 세계로 들어갈 준비를 할 것이다. 이 장에서는 게임 개발을 위한 환경을 완벽하게 설정하고, 첫 번째 MonoGame 프로젝트를 만들어 볼 것이다. 함께 시작해보자.


## 1.1 MonoGame이란?
MonoGame은 C#을 사용한 게임 개발 프레임워크다. 원래 XNA(Microsoft의 게임 개발 프레임워크)의 후속 프로젝트로 시작되었으며, XNA의 API를 기반으로 하면서도 더욱 강력한 기능들을 제공한다.

MonoGame의 가장 큰 특징은 **크로스 플랫폼 지원**이다. 한 번의 코딩으로 Windows, macOS, Linux, iOS, Android 등 다양한 플랫폼에서 게임을 실행할 수 있다. 우리가 배우는 C# 언어와도 완벽하게 호환되므로, 이미 알고 있는 C# 문법을 그대로 활용하면 된다.

```
┌─────────────────────────────────┐
│      MonoGame Framework         │
│                                 │
│  ┌──────────────────────────┐  │
│  │  Graphics (2D/3D)        │  │
│  └──────────────────────────┘  │
│  ┌──────────────────────────┐  │
│  │  Audio                   │  │
│  └──────────────────────────┘  │
│  ┌──────────────────────────┐  │
│  │  Input (Keyboard/Mouse)  │  │
│  └──────────────────────────┘  │
│  ┌──────────────────────────┐  │
│  │  Content Management      │  │
│  └──────────────────────────┘  │
│                                 │
│  ↓                              │
│  Windows / macOS / Linux / etc  │
└─────────────────────────────────┘
```

MonoGame은 게임에 필요한 그래픽 렌더링, 사운드 처리, 입력 관리, 게임 루프 등 핵심 기능을 모두 제공한다. 개발자는 MonoGame의 기능들을 활용하여 로직에만 집중할 수 있다.

## 1.2 개발 환경 준비
게임을 만들기 위해서는 먼저 필요한 도구들을 설치해야 한다. 우리는 .NET 9.0, Visual Studio 2022, MonoGame 3.8을 사용할 것이다.

### .NET 9.0 SDK 설치
.NET은 Microsoft에서 만든 크로스 플랫폼 개발 플랫폼이다. MonoGame 3.8은 .NET 9.0 이상에서 최적으로 동작한다.

**설치 단계:**

1. 웹 브라우저를 열고 [https://dotnet.microsoft.com/download](https://dotnet.microsoft.com/download)에 접속한다.
2. .NET 9.0 SDK를 다운로드한다. (운영 체제에 맞는 버전을 선택하자)
3. 다운로드한 설치 파일을 실행한다.
4. 설치 마법사를 따라 기본 설정으로 설치를 진행한다.

설치가 완료되었는지 확인하려면 명령 프롬프트나 터미널에서 다음 명령어를 입력한다.

```bash
dotnet --version
```

출력되는 버전이 9.0 이상이면 정상적으로 설치된 것이다.

### Visual Studio 2022 설치
Visual Studio는 C# 코드를 작성하고 디버깅하기 위한 통합 개발 환경(IDE)이다. MonoGame 게임 개발에 최적화되어 있다.

**설치 단계:**

1. [https://visualstudio.microsoft.com/](https://visualstudio.microsoft.com/)에 접속한다.
2. Visual Studio Community 2022를 다운로드한다. (학생은 무료로 사용 가능하다)
3. 설치 파일을 실행하면 Visual Studio Installer가 열린다.
4. "게임 개발(C#과 C++)" 워크로드를 선택한다.
5. 오른쪽 하단의 "설치" 버튼을 클릭하고 설치를 기다린다.

설치에는 시간이 걸릴 수 있으니 차를 한 잔 마시면서 기다리자. 설치가 완료되면 Visual Studio를 처음으로 실행할 때 추가 설정을 진행하게 된다.

### MonoGame 3.8 설치
MonoGame은 Visual Studio의 확장 프로그램(Extension) 형태로 설치된다.

**설치 단계:**

1. Visual Studio 2022를 실행한다.
2. 메뉴에서 "확장" → "확장 관리"를 선택한다.
3. 오른쪽 위의 검색 창에 "MonoGame"을 입력한다.
4. "MonoGame 3.8" (또는 최신 버전)을 찾아서 "다운로드" 버튼을 클릭한다.
5. Visual Studio를 다시 시작하면 설치가 완료된다.

설치 확인은 나중에 프로젝트를 생성할 때 하게 되니 지금은 걱정하지 않아도 된다.

**또는 명령 프롬프트를 통한 설치:**

고급 사용자라면 명령줄을 통해서도 MonoGame 템플릿을 설치할 수 있다.

```bash
dotnet new install MonoGame.Templates.CSharp
```

이 명령어를 실행하면 .NET CLI에서 MonoGame 프로젝트 템플릿을 사용할 수 있게 된다.

## 1.3 첫 번째 MonoGame 프로젝트 생성
이제 우리의 첫 번째 MonoGame 게임 프로젝트를 만들어보자.

**Visual Studio를 통한 프로젝트 생성:**

1. Visual Studio 2022를 열고 "새 프로젝트 만들기"를 클릭한다.
2. 프로젝트 템플릿 검색 창에 "MonoGame"을 입력한다.
3. "MonoGame Windows Desktop Application (MGDESKTOP)"을 선택한다.
4. "다음"을 클릭한다.
5. 프로젝트 이름을 "MyFirstGame"으로 설정한다. (또는 원하는 이름으로)
6. 프로젝트 위치를 설정한 후 "만들기"를 클릭한다.
7. .NET 버전 선택에서 ".NET 9.0"을 선택한다.
8. "만들기" 버튼을 클릭하면 프로젝트가 생성된다.

**명령 프롬프트를 통한 프로젝트 생성:**

터미널이나 명령 프롬프트에서도 프로젝트를 만들 수 있다.

```bash
dotnet new mgdesktop -n MyFirstGame
cd MyFirstGame
```

이 명령어들을 실행하면 "MyFirstGame"이라는 폴더가 생성되고, 그 안에 기본 MonoGame 프로젝트 구조가 만들어진다.

**프로젝트 실행:**

프로젝트가 생성된 후, 다음과 같이 실행할 수 있다.

1. Visual Studio에서 프로젝트를 열었다면, 상단의 "시작" 버튼(재생 버튼)을 클릭한다.
2. 또는 "디버그" 메뉴에서 "디버깅 시작"을 선택한다.
3. 명령 프롬프트에서는 프로젝트 폴더에서 `dotnet run` 명령어를 입력한다.

잠시 후 게임 윈도우가 나타날 것이다. 창의 배경색이 어떤 색인가? 이것은 게임 루프가 정상적으로 작동하고 있다는 의미다.

## 1.4 프로젝트 구조 이해하기
MonoGame 프로젝트가 생성되면 여러 파일과 폴더가 자동으로 만들어진다. 각각이 어떤 역할을 하는지 알아보자.

```
MyFirstGame/
├── Content/
│   └── Content.mgcb
├── bin/
├── obj/
├── Program.cs
├── Game1.cs
├── MyFirstGame.csproj
└── MyFirstGame.sln
```

**주요 파일 설명:**

**Game1.cs** - 가장 중요한 파일이다. 이 파일에는 게임의 핵심 로직이 들어간다. MonoGame의 Game 클래스를 상속받으며, Initialize(), LoadContent(), Update(), Draw() 같은 메서드를 포함한다. 우리는 대부분의 시간을 이 파일에서 코드를 작성할 것이다.

**Program.cs** - 게임 프로그램의 진입점(Entry Point)이다. 게임이 시작될 때 이 파일의 코드가 먼저 실행되며, Game1 인스턴스를 생성하고 게임 루프를 시작한다. 보통은 이 파일을 수정할 필요가 없다.

**Content/Content.mgcb** - 게임에 사용될 이미지, 음성, 폰트 등의 에셋을 관리하는 파일이다. 확장자가 .mgcb인데, 이것은 "MonoGame Content Builder"의 약자다. 게임에 그래픽이나 사운드를 추가할 때 이 파일을 수정하게 된다.

**MyFirstGame.csproj** - C# 프로젝트 설정 파일이다. 프로젝트의 메타데이터, 참조 라이브러리, 빌드 설정 등이 포함되어 있다. 일반적으로는 Visual Studio가 자동으로 관리한다.

**MyFirstGame.sln** - 솔루션 파일다. 프로젝트를 Visual Studio에서 열 때 이 파일을 열게 된다.

**bin/ 과 obj/** - 컴파일 결과물과 임시 파일이 저장되는 폴더다. 게임을 실행할 때 필요한 .dll 파일과 실행 파일이 bin 폴더에 생성된다.

이제 기본 프로젝트 구조를 이해했으므로, Game1.cs 파일을 살펴보자. 자동 생성된 기본 코드는 다음과 같은 구조를 가지고 있다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;

namespace MyFirstGame
{
    public class Game1 : Game
    {
        private GraphicsDeviceManager _graphics;
        private SpriteBatch _spriteBatch;

        public Game1()
        {
            _graphics = new GraphicsDeviceManager(this);
            Content.RootDirectory = "Content";
            IsMouseVisible = true;
        }

        protected override void Initialize()
        {
            base.Initialize();
        }

        protected override void LoadContent()
        {
            _spriteBatch = new SpriteBatch(GraphicsDevice);
        }

        protected override void Update(GameTime gameTime)
        {
            if (GamePad.GetState(PlayerIndex.One).Buttons.Back == ButtonState.Pressed ||
                Keyboard.GetState().IsKeyDown(Keys.Escape))
                Exit();

            base.Update(gameTime);
        }

        protected override void Draw(GameTime gameTime)
        {
            GraphicsDevice.Clear(Color.CornflowerBlue);
            base.Draw(gameTime);
        }
    }
}
```

**코드의 역할 설명:**

각 메서드가 언제 호출되는지 이해하는 것이 중요하다. Initialize()는 게임 시작 시 한 번만 호출되어 초기 설정을 한다. LoadContent()는 Initialize() 이후에 호출되며, 이미지나 사운드 같은 게임 에셋을 로드한다. Update()는 매 프레임마다 호출되어 게임 로직을 처리한다. Draw()도 매 프레임마다 호출되며, 화면에 그릴 것들을 렌더링한다.

**using 문장들:**

코드 맨 위의 using 문장들은 필요한 라이브러리를 import하는 것이다. Microsoft.Xna.Framework는 MonoGame이 XNA를 기반으로 하기 때문에 포함되어 있다. Graphics는 화면 렌더링, Input은 키보드/마우스 입력을 처리하기 위한 네임스페이스다.

**GraphicsDeviceManager와 SpriteBatch:**

GraphicsDeviceManager는 화면 해상도, 전체 화면 모드 등을 관리한다. SpriteBatch는 2D 그래픽을 그리기 위한 도구로, 이미지를 화면에 출력할 때 반드시 필요하다.

3. MonoGame 3.8이 설치되었는지 확인한다. Visual Studio의 새 프로젝트 만들기 화면에서 "MonoGame"이라고 검색했을 때 템플릿이 나타나는가?

**과제 2: 첫 번째 프로젝트 실행**

1. 윈도우의 제목이 무엇인가?
2. 배경색이 무엇인가? (CornflowerBlue라는 색상이다)

**과제 3: 프로젝트 구조 탐색**

생성된 MyFirstGame 프로젝트의 폴더 구조를 살펴보고, 다음을 답해라.

3. MyFirstGame.csproj 파일을 메모장으로 열어보면, 어떤 정보가 포함되어 있는가?

**과제 4: Game1.cs 분석**

자동 생성된 Game1.cs 파일의 코드를 읽고, 다음 질문에 답해라.

1. Game1 클래스는 어느 클래스를 상속받고 있는가?
2. GraphicsDeviceManager와 SpriteBatch는 각각 어떤 역할을 하는가?
3. IsMouseVisible = true가 의미하는 바는 무엇인가?
4. ESC 키를 누르면 게임이 종료되는 코드는 어디에 있는가?

**과제 5: 코드 수정 및 테스트**

Game1.cs 파일을 수정하여, 다음을 실행해보자.

1. Draw() 메서드의 GraphicsDevice.Clear(Color.CornflowerBlue)를 Color.Black으로 변경하고, 게임을 실행했을 때 배경색이 검은색으로 변하는지 확인한다.
2. 게임 윈도우의 제목을 변경해본다. (힌트: 게임 윈도우의 제목은 Game1의 생성자 또는 Initialize 메서드에서 설정할 수 있다. 하지만 자동 생성 코드에는 없으므로, 직접 추가해야 한다. Window 속성을 살펴봐라.)
