# 5장. 텍스트와 폰트
이번 장에서는 게임 화면에 텍스트를 표시하는 방법을 배운다. 점수, 체력, 게임 오버 메시지 등 대부분의 게임에서 텍스트는 필수 요소다. MonoGame에서 텍스트를 렌더링하는 것은 이미지를 그리는 것만큼 간단하지만, 폰트 파일을 준비하고 관리해야 한다는 점에서 조금 다르다.

## 5.1 SpriteFont 생성하기

### SpriteFont란?
SpriteFont는 MonoGame에서 사용하는 폰트 파일 형식이다. 게임에서 텍스트를 빠르게 렌더링하기 위해 미리 폰트를 이미지로 변환하여 저장하는 방식이다. 이 방식은 TrueType 폰트를 직접 렌더링하는 것보다 훨씬 빠르고, 게임 내에서 일관성 있는 글꼴을 사용할 수 있다.

### Content Pipeline을 통한 SpriteFont 생성
MonoGame 프로젝트에서 텍스트를 사용하려면 먼저 Content 폴더에 SpriteFont 파일(.spritefont)을 생성해야 한다. 다음은 그 과정이다.

1. Visual Studio에서 Content 폴더를 우클릭한다.
2. "추가" → "새 항목"을 선택한다.
3. "SpriteFont"를 선택하고 파일명을 지정한다(예: "Arial.spritefont").

또는 명령어로도 생성할 수 있다.

```bash
dotnet new sdf -n Arial
```

생성된 SpriteFont 파일은 XML 형식이다. 다음은 기본 구조를 보여준다.

```xml
<?xml version="1.0" encoding="utf-8"?>
<XmlRoot xmlns:xsd="http://www.w3.org/2001/XMLSchema" 
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <Asset Type="Microsoft.Xna.Framework.Content.Pipeline.Graphics.FontDescription">
    <FontName>Arial</FontName>
    <Size>24</Size>
    <Spacing>0</Spacing>
    <UseKerning>true</UseKerning>
    <DefaultCharacter>32</DefaultCharacter>
    <CharacterRegions>
      <CharacterRegion>
        <Start>32</Start>
        <End>126</End>
      </CharacterRegion>
    </CharacterRegions>
  </Asset>
</XmlRoot>
```

**주요 속성 설명:**

FontName은 시스템에 설치된 폰트의 이름이다. Windows에서는 Arial, Times New Roman, Courier New 등 기본 폰트를 사용할 수 있다. macOS에서는 Helvetica, Monaco 등을 사용한다. Size는 픽셀 단위의 폰트 크기다. Spacing은 문자 간 간격을 조정하는 옵션이다. CharacterRegion은 포함할 문자의 범위를 지정하는데, 위의 예에서는 ASCII 32(공백)부터 126(물결표)까지 총 95개 문자를 포함한다.

### 한글 텍스트 지원
한글을 포함한 폰트를 만들려면 CharacterRegion을 확장해야 한다. 한글 코드 범위는 44032부터 55203까지다. 다음은 한글을 지원하는 SpriteFont 파일의 예다.

```xml
<?xml version="1.0" encoding="utf-8"?>
<XmlRoot xmlns:xsd="http://www.w3.org/2001/XMLSchema" 
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <Asset Type="Microsoft.Xna.Framework.Content.Pipeline.Graphics.FontDescription">
    <FontName>나눔고딕</FontName>
    <Size>28</Size>
    <Spacing>0</Spacing>
    <UseKerning>true</UseKerning>
    <DefaultCharacter>32</DefaultCharacter>
    <CharacterRegions>
      <CharacterRegion>
        <Start>32</Start>
        <End>126</End>
      </CharacterRegion>
      <CharacterRegion>
        <Start>44032</Start>
        <End>55203</End>
      </CharacterRegion>
    </CharacterRegions>
  </Asset>
</XmlRoot>
```

Windows 시스템에는 "나눔고딕", "맑은 고딕" 등 한글 폰트가 기본 설치되어 있다. FontName 속성에 이 이름들을 명시하면 된다.

### 여러 크기의 폰트 생성
게임에서는 제목, 일반 텍스트, 작은 텍스트 등 여러 크기의 폰트가 필요하다. 따라서 같은 폰트 이름으로 다양한 크기의 SpriteFont 파일을 생성하는 것이 좋다.

```
Content/
├── Fonts/
│   ├── Arial_Large.spritefont    (크기 48)
│   ├── Arial_Normal.spritefont   (크기 24)
│   └── Arial_Small.spritefont    (크기 14)
```

## 5.2 텍스트 렌더링

### 기본 텍스트 렌더링
SpriteFont를 로드했다면, 이제 SpriteBatch.DrawString() 메서드로 텍스트를 화면에 그릴 수 있다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Content;
using Microsoft.Xna.Framework.Graphics;

public class TextRenderer
{
    private SpriteFont spriteFont;
    private string textToDraw = "안녕하세요!";
    private Vector2 textPosition = new Vector2(100, 100);

    public void LoadContent(ContentManager content)
    {
        spriteFont = content.Load<SpriteFont>("Fonts/Arial_Normal");
    }

    public void Draw(SpriteBatch spriteBatch)
    {
        // 기본 텍스트 렌더링
        spriteBatch.DrawString(
            spriteFont,
            textToDraw,
            textPosition,
            Color.White
        );
    }
}
```

**DrawString() 메서드 매개변수:**

첫 번째 매개변수는 SpriteFont 객체다. 두 번째는 그릴 문자열이다. 세 번째는 화면상의 위치(Vector2)다. 네 번째는 텍스트 색상이다.

### 고급 텍스트 렌더링 옵션
DrawString() 메서드는 더 많은 매개변수를 받을 수 있다. 이를 이용하면 텍스트를 회전시키거나 크기를 조정하거나 정렬 기준점을 변경할 수 있다.

```csharp
public void Draw(SpriteBatch spriteBatch)
{
    // 고급 옵션을 포함한 텍스트 렌더링
    Vector2 origin = spriteFont.MeasureString(textToDraw) / 2f;  // 텍스트 중앙
    
    spriteBatch.DrawString(
        spriteFont,
        textToDraw,
        textPosition,
        Color.White,
        rotation: 0f,                      // 회전 각도 (라디안)
        origin: origin,                    // 회전/스케일의 중심점
        scale: 1f,                         // 크기 배수
        effects: SpriteEffects.None,       // 플립 효과
        layerDepth: 0.5f                   // 렌더링 순서
    );
}
```

**중요한 매개변수들:**

origin 매개변수는 텍스트의 기준점을 지정한다. (0, 0)이면 왼쪽 위, MeasureString() / 2f면 텍스트의 중앙이다. MeasureString() 메서드는 문자열의 너비와 높이를 픽셀 단위로 반환한다. scale은 텍스트의 크기를 배수로 지정한다. layerDepth는 4장에서 배운 렌더링 순서를 지정한다.

### 텍스트 색상 효과
게임에서는 텍스트에 다양한 색상 효과를 주는 것이 흔하다. 다음은 시간에 따라 색상이 변하는 텍스트를 보여준다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;

public class ColorfulText
{
    private SpriteFont spriteFont;
    private string text = "게임 타이틀";
    private Vector2 position = new Vector2(400, 200);
    private double elapsedTime = 0;

    public void Update(GameTime gameTime)
    {
        elapsedTime += gameTime.ElapsedGameTime.TotalSeconds;
    }

    public void Draw(SpriteBatch spriteBatch)
    {
        // 시간에 따라 색상 계산
        // 빨강, 초록, 파랑 채널을 각각 계산
        float red = (float)(System.Math.Sin(elapsedTime * 2) * 0.5f + 0.5f);
        float green = (float)(System.Math.Sin(elapsedTime * 2 + 2.094) * 0.5f + 0.5f);
        float blue = (float)(System.Math.Sin(elapsedTime * 2 + 4.188) * 0.5f + 0.5f);

        Color dynamicColor = new Color(red, green, blue);

        spriteBatch.DrawString(
            spriteFont,
            text,
            position,
            dynamicColor
        );
    }
}
```

**코드 설명:**

이 코드는 사인파를 이용하여 각 색상 채널을 주기적으로 변화시킨다. 사인 함수는 -1부터 1까지의 값을 반환하는데, 여기서는 0부터 1 사이의 값으로 정규화했다. elapsedTime * 2 + (값)에서 덧셈 항을 사용하여 각 채널의 위상을 다르게 하므로, 색상이 부드럽게 변화한다.

### 텍스트 그림자 효과
텍스트를 더 가독성 좋게 만들기 위해 그림자 효과를 주는 것도 일반적이다.

```csharp
public void DrawTextWithShadow(
    SpriteBatch spriteBatch,
    SpriteFont font,
    string text,
    Vector2 position,
    Color textColor,
    Color shadowColor,
    int shadowOffset = 2)
{
    // 그림자를 먼저 그리기
    Vector2 shadowPosition = position + new Vector2(shadowOffset, shadowOffset);
    spriteBatch.DrawString(font, text, shadowPosition, shadowColor);

    // 그 위에 실제 텍스트 그리기
    spriteBatch.DrawString(font, text, position, textColor);
}
```

이 메서드를 사용하면 다음과 같이 호출할 수 있다.

```csharp
DrawTextWithShadow(
    spriteBatch,
    spriteFont,
    "점수: 1000",
    new Vector2(50, 50),
    Color.Yellow,
    Color.Black,
    shadowOffset: 3
);
```

## 5.3 동적 텍스트 표시

### 게임 상태에 따라 변하는 텍스트
게임에서 가장 많이 사용되는 텍스트는 점수, 체력, 아이템 개수 등 게임 상태를 반영하는 것들이다. 이러한 텍스트들은 Update() 메서드에서 계산된 값을 기반으로 매 프레임 업데이트된다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Content;
using Microsoft.Xna.Framework.Graphics;

public class GameUI
{
    private SpriteFont font;
    private int score = 0;
    private int health = 100;
    private int ammo = 30;
    private Vector2 uiPosition = new Vector2(10, 10);
    private float lineHeight = 30f;

    public void LoadContent(ContentManager content)
    {
        font = content.Load<SpriteFont>("Fonts/Arial_Normal");
    }

    public void Update(GameTime gameTime)
    {
        // 게임 로직에서 값 변경
        // (예: 적 처치 시 점수 증가, 피해 입을 시 체력 감소 등)
    }

    public void Draw(SpriteBatch spriteBatch)
    {
        // 점수 표시
        spriteBatch.DrawString(
            font,
            $"점수: {score}",
            uiPosition,
            Color.Yellow
        );

        // 체력 표시
        spriteBatch.DrawString(
            font,
            $"체력: {health}",
            uiPosition + new Vector2(0, lineHeight),
            health > 50 ? Color.Green : Color.Red
        );

        // 탄약 표시
        spriteBatch.DrawString(
            font,
            $"탄약: {ammo}",
            uiPosition + new Vector2(0, lineHeight * 2),
            ammo > 10 ? Color.White : Color.Orange
        );
    }

    public void IncreaseScore(int points) => score += points;
    public void TakeDamage(int damage) => health -= damage;
    public void UseAmmo(int amount) => ammo -= amount;
}
```

**코드 설명:**

이 클래스는 게임의 UI 정보를 관리한다. Update() 메서드는 예시를 위해 비어있지만, 실제로는 게임 로직에서 호출하여 값을 변경한다. Draw() 메서드에서는 문자열 보간(string interpolation)을 사용하여 동적으로 텍스트를 생성한다. 체력이 50 이하면 빨강색, 탄약이 10 이하면 주황색으로 표시하여 플레이어에게 시각적 피드백을 준다.

### 카운트다운 타이머 텍스트
많은 게임에서 남은 시간을 표시하는 카운트다운이 필요하다. 다음은 분과 초 형식으로 표시하는 예제다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Content;
using Microsoft.Xna.Framework.Graphics;

public class TimerDisplay
{
    private SpriteFont font;
    private double remainingTime = 120.0;  // 2분
    private Vector2 position = new Vector2(400, 50);

    public void LoadContent(ContentManager content)
    {
        font = content.Load<SpriteFont>("Fonts/Arial_Large");
    }

    public void Update(GameTime gameTime)
    {
        // 시간 감소
        remainingTime -= gameTime.ElapsedGameTime.TotalSeconds;

        // 시간이 0 이하가 되지 않도록 제한
        if (remainingTime < 0)
            remainingTime = 0;
    }

    public void Draw(SpriteBatch spriteBatch)
    {
        // 분과 초로 변환
        int minutes = (int)(remainingTime / 60);
        int seconds = (int)(remainingTime % 60);

        // mm:ss 형식으로 표시
        string timeText = $"{minutes:00}:{seconds:00}";

        // 시간이 10초 이하면 빨강색으로 표시
        Color timeColor = remainingTime <= 10 ? Color.Red : Color.White;

        // 텍스트의 중앙을 중심으로 렌더링
        Vector2 textSize = font.MeasureString(timeText);
        Vector2 origin = textSize / 2f;

        spriteBatch.DrawString(
            font,
            timeText,
            position,
            timeColor,
            0f,
            origin,
            1f,
            SpriteEffects.None,
            0f
        );
    }

    public bool IsTimeUp => remainingTime <= 0;
}
```

**코드 설명:**

remainingTime은 초 단위로 남은 시간을 저장한다. Update() 메서드에서 GameTime.ElapsedGameTime을 빼서 프레임마다 시간을 감소시킨다. Draw() 메서드에서는 remainingTime을 분과 초로 변환하는데, {minutes:00}은 정수를 00 형식(항상 2자리)으로 포맷팅하는 것이다. 예를 들어 minutes가 5면 "05"로 표시된다.

### 플로팅 텍스트 (Floating Text)
게임에서 데미지나 회복량 같은 정보를 일시적으로 표시하는 것이 흔하다. 이런 텍스트는 위로 떠오르면서 서서히 사라진다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using System.Collections.Generic;

public class FloatingText
{
    private class FloatingTextItem
    {
        public string Text { get; set; }
        public Vector2 Position { get; set; }
        public double RemainingLifetime { get; set; }
        public double TotalLifetime { get; set; }
        public Color Color { get; set; }
    }

    private SpriteFont font;
    private List<FloatingTextItem> floatingTexts = new List<FloatingTextItem>();
    private float floatSpeed = 50f;  // 초당 픽셀

    public void LoadContent(Microsoft.Xna.Framework.Content.ContentManager content)
    {
        font = content.Load<SpriteFont>("Fonts/Arial_Normal");
    }

    public void AddFloatingText(
        string text,
        Vector2 position,
        Color color,
        double lifetime = 2.0)
    {
        floatingTexts.Add(new FloatingTextItem
        {
            Text = text,
            Position = position,
            RemainingLifetime = lifetime,
            TotalLifetime = lifetime,
            Color = color
        });
    }

    public void Update(GameTime gameTime)
    {
        for (int i = floatingTexts.Count - 1; i >= 0; i--)
        {
            var item = floatingTexts[i];
            item.RemainingLifetime -= gameTime.ElapsedGameTime.TotalSeconds;

            // 위로 올라가기
            float moveDistance = floatSpeed * (float)gameTime.ElapsedGameTime.TotalSeconds;
            item.Position.Y -= moveDistance;

            // 생명 시간이 다하면 제거
            if (item.RemainingLifetime <= 0)
            {
                floatingTexts.RemoveAt(i);
            }
        }
    }

    public void Draw(SpriteBatch spriteBatch)
    {
        foreach (var item in floatingTexts)
        {
            // 남은 시간 비율에 따라 투명도 계산
            float alphaFactor = (float)(item.RemainingLifetime / item.TotalLifetime);
            
            // 생명 시간이 다할수록 투명해짐
            Color fadeColor = new Color(
                item.Color.R,
                item.Color.G,
                item.Color.B,
                (byte)(255 * alphaFactor)
            );

            spriteBatch.DrawString(
                font,
                item.Text,
                item.Position,
                fadeColor
            );
        }
    }
}
```

**코드 설명:**

이 클래스는 여러 개의 플로팅 텍스트를 동시에 관리한다. FloatingTextItem 내부 클래스는 각 텍스트의 생명 시간, 위치, 색상을 추적한다. Update() 메서드에서는 루프를 역순으로 순회하면서 항목을 제거하는데, 이는 컬렉션의 크기가 변하는 동안 정방향 순회를 할 때의 인덱스 오류를 방지하기 위함이다. 투명도는 남은 생명 시간의 비율에 따라 계산되므로, 텍스트가 서서히 페이드아웃된다.

### 게임 클래스에서 통합하기
위의 모든 텍스트 렌더링 클래스들을 실제 게임에 통합하는 방법을 보자.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;

public class MyGame : Game
{
    private GraphicsDeviceManager graphics;
    private SpriteBatch spriteBatch;
    private GameUI gameUI;
    private TimerDisplay timerDisplay;
    private FloatingText floatingText;

    public MyGame()
    {
        graphics = new GraphicsDeviceManager(this);
        Content.RootDirectory = "Content";
    }

    protected override void Initialize()
    {
        gameUI = new GameUI();
        timerDisplay = new TimerDisplay();
        floatingText = new FloatingText();

        base.Initialize();
    }

    protected override void LoadContent()
    {
        spriteBatch = new SpriteBatch(GraphicsDevice);
        gameUI.LoadContent(Content);
        timerDisplay.LoadContent(Content);
        floatingText.LoadContent(Content);
    }

    protected override void Update(GameTime gameTime)
    {
        if (GamePad.GetState(PlayerIndex.One).Buttons.Back == ButtonState.Pressed)
            Exit();

        // 스페이스 키 누르면 점수 증가
        if (Keyboard.GetState().IsKeyDown(Keys.Space))
        {
            gameUI.IncreaseScore(10);
            // 플로팅 텍스트 생성
            floatingText.AddFloatingText(
                "+10",
                new Vector2(640, 200),
                Color.Yellow,
                lifetime: 1.5
            );
        }

        gameUI.Update(gameTime);
        timerDisplay.Update(gameTime);
        floatingText.Update(gameTime);

        base.Update(gameTime);
    }

    protected override void Draw(GameTime gameTime)
    {
        GraphicsDevice.Clear(Color.Black);

        spriteBatch.Begin();

        gameUI.Draw(spriteBatch);
        timerDisplay.Draw(spriteBatch);
        floatingText.Draw(spriteBatch);

        spriteBatch.End();

        base.Draw(gameTime);
    }
}
```

**코드 설명:**

게임의 Main Game 클래스에서 세 개의 텍스트 관련 클래스를 모두 생성하고 관리한다. LoadContent()에서 각 클래스의 LoadContent() 메서드를 호출하여 폰트를 로드한다. Update()에서는 키 입력에 따라 게임 상태를 변경하고, 점수가 증가할 때마다 플로팅 텍스트를 생성한다. Draw()에서는 spriteBatch.Begin()과 End() 사이에 모든 텍스트를 그린다.

## 과제
이번 장에서 배운 내용을 확인하기 위한 과제다.

### 과제: 게임 점수 시스템 UI 만들기

**요구사항:**

1. 화면 좌상단에 다음 정보를 표시한다.
   - 점수 (Score): 0부터 시작
   - 체력 (Health): 100부터 시작
   - 남은 시간 (Time): 60초부터 시작

2. 키보드 입력 처리:
   - A 키: 점수 100 증가, 노란색 "+100" 플로팅 텍스트 생성
   - S 키: 체력 10 감소, 빨간색 "-10" 플로팅 텍스트 생성
   - D 키: 아이템 습득, 초록색 "+1 ITEM" 플로팅 텍스트 생성

3. 텍스트 표시:
   - 일반 정보(점수, 체력, 시간)는 하얀색 기본 크기로 표시
   - 체력이 30 이하면 빨강색으로 표시
   - 남은 시간이 10초 이하면 빨강색으로 표시하고 깜빡인다 (명멸 효과)

4. 플로팅 텍스트:
   - 키를 누르면 마우스 위치에서 위로 떠오르는 플로팅 텍스트 생성
   - 1.5초 동안 위로 떠오르면서 투명도 감소
   - 최대 20개까지만 동시 표시 (초과하면 가장 오래된 것부터 제거)

5. 게임 종료 조건:
   - 시간이 0이 되면 "GAME OVER" 메시지를 화면 중앙에 표시
   - 체력이 0이 되면 "YOU DIED" 메시지를 화면 중앙에 표시
   - R 키로 게임을 초기화하여 다시 시작할 수 있도록 한다

6. 배경색: 어두운 파란색 (DarkBlue)

**구현해야 할 클래스:**

- GameUI: 점수, 체력, 아이템 관리
- TimerDisplay: 시간 표시 및 남은 시간 관리
- FloatingText: 플로팅 텍스트 관리
- MyGame: 통합 관리

**테스트 항목:**

- A 키를 누르면 점수가 100씩 증가하는가?
- S 키를 누르면 체력이 10씩 감소하는가?
- D 키를 누르면 아이템이 증가하는가?
- 체력이 30 이하일 때 빨강색으로 표시되는가?
- 남은 시간이 10초 이하일 때 깜빡이는 효과가 나타나는가?
- 시간이 0이 되면 "GAME OVER"이 표시되는가?
- 체력이 0이 되면 "YOU DIED"가 표시되는가?
- 플로팅 텍스트가 위로 떠오르면서 투명해지는가?

**팁:**

- 깜빡임 효과는 Sin 함수를 사용하여 alpha 값을 0과 1 사이에서 변하게 하면 된다.
- FloatingText 클래스의 최대 개수 제한은 Update() 메서드에서 RemoveAt(0)으로 첫 번째 항목부터 제거하면 된다.
- 마우스 위치는 Mouse.GetState().Position으로 얻을 수 있다.
- 게임 상태(Playing, GameOver, YouDied)를 enum으로 정의하여 관리하면 더 깔끔하다.

이 과제를 완성하면 게임에서 필수적인 UI 표시 기능을 모두 구현한 것이 된다. 텍스트 렌더링, 동적 색상 변경, 플로팅 텍스트, 게임 상태 관리 등 이후 장들에서 자주 사용될 개념들을 모두 다룬다.
