## 3.1 SpriteBatch 이해하기
SpriteBatch는 MonoGame에서 2D 그래픽을 렌더링하는 가장 기본적인 도구다. SpriteBatch의 개념을 정확하게 이해하면, 모든 2D 게임 그래픽 작업이 훨씬 수월해진다.

### SpriteBatch란 무엇인가?
SpriteBatch는 그림을 그리기 위한 "배치"(일괄 처리)이다. 배치라는 말이 생소할 수 있으니 현실의 예로 설명하겠다. 만약 100개의 그림을 종이에 그려야 한다고 생각해보자. 그림을 하나 그릴 때마다 모든 도구를 꺼냈다가 다시 정리한다면 매우 비효율적일 것이다. 하지만 모든 도구를 먼저 준비한 후 100개를 연속으로 그리고 나중에 정리한다면 훨씬 효율적이다. SpriteBatch도 같은 원리다.

SpriteBatch는 다음과 같이 작동한다.

```
Begin() 호출
   ↓
Draw() / DrawString() 여러 번 호출 (메모리에만 저장됨)
   ↓
End() 호출
   ↓
모든 그리기가 한 번에 GPU로 전송되고 화면에 렌더링됨
```

이렇게 배치 처리하면 GPU 통신 횟수가 줄어들어 성능이 크게 향상된다.

### SpriteBatch의 기본 사용 방법
2장에서 자동 생성된 코드에서 이미 SpriteBatch를 봤을 것이다. 기본 구조는 다음과 같다.

```csharp
protected override void Draw(GameTime gameTime)
{
    GraphicsDevice.Clear(Color.CornflowerBlue);
    
    // SpriteBatch 시작
    _spriteBatch.Begin();
    
    // 여기에서 Draw() 메서드들을 호출한다
    // _spriteBatch.Draw(...);
    
    // SpriteBatch 종료
    _spriteBatch.End();
    
    base.Draw(gameTime);
}
```

SpriteBatch는 항상 Begin()과 End() 사이에서만 사용할 수 있다. Begin()을 호출하지 않은 상태에서 Draw()를 호출하면 예외가 발생한다.

### SpriteBatch.Begin()의 옵션

SpriteBatch.Begin()은 여러 옵션을 받을 수 있다. 기본적인 옵션들을 살펴보자.

```csharp
_spriteBatch.Begin(
    sortMode: SpriteSortMode.Deferred,           // 그리기 순서
    blendState: BlendState.AlphaBlend,           // 색상 합성 방식
    samplerState: SamplerState.LinearClamp,      // 텍스처 샘플링 방식
    depthStencilState: DepthStencilState.None,   // 깊이 테스트
    rasterizerState: RasterizerState.CullCounterClockwise  // 래스터화 설정
);
```

이 옵션들은 대부분의 경우 기본값(Begin())으로 충분하다. 나중에 고급 그래픽 효과가 필요할 때 이러한 옵션들을 사용한다. 지금은 기본값으로 사용해도 문제없다.

### SpriteSortMode의 종류
SpriteSortMode는 여러 개의 그리기 명령이 있을 때 그 순서를 결정한다.

**SpriteSortMode.Deferred** - 기본값이다. 모든 그리기 명령을 메모리에 저장했다가 End()에서 한 번에 처리한다. 가장 빠르고 효율적이다.

**SpriteSortMode.Immediate** - 각 Draw() 호출마다 즉시 GPU로 전송된다. 특수한 상황에서만 사용한다.

**SpriteSortMode.BackToFront** - 앞에서 그린 것부터 뒤에 그린 것 순서로 렌더링한다. 투명도가 있는 이미지를 여러 겹 그릴 때 사용한다.

**SpriteSortMode.FrontToBack** - 뒤에서 그린 것부터 앞에 그린 것 순서로 렌더링한다.

## 3.2 Texture2D로 이미지 그리기
Texture2D는 메모리에 로드된 이미지를 나타낸다. 게임에서 보는 모든 스프라이트(이미지)는 Texture2D 형태로 존재한다.

### 이미지 에셋 준비
게임에 이미지를 사용하려면 먼저 이미지 파일을 프로젝트에 추가해야 한다. 우리는 간단한 테스트용 이미지를 프로그래밍으로 직접 생성할 것이다.

다음은 빨간색 사각형 이미지를 런타임에 생성하는 방법이다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;

namespace MyGraphicsGame
{
    public class Game1 : Game
    {
        private GraphicsDeviceManager _graphics;
        private SpriteBatch _spriteBatch;
        
        private Texture2D redSquareTexture;
        private Vector2 squarePosition = new Vector2(100, 100);

        public Game1()
        {
            _graphics = new GraphicsDeviceManager(this);
            Content.RootDirectory = "Content";
            IsMouseVisible = true;
            
            _graphics.PreferredBackBufferWidth = 800;
            _graphics.PreferredBackBufferHeight = 600;
        }

        protected override void Initialize()
        {
            Window.Title = "2D Graphics Game";
            base.Initialize();
        }

        protected override void LoadContent()
        {
            _spriteBatch = new SpriteBatch(GraphicsDevice);
            
            // 빨간색 사각형 텍스처 생성
            redSquareTexture = new Texture2D(GraphicsDevice, 100, 100);
            
            // 텍스처 데이터 배열 생성
            Color[] data = new Color[100 * 100];
            
            // 모든 픽셀을 빨간색으로 설정
            for (int i = 0; i < data.Length; i++)
            {
                data[i] = Color.Red;
            }
            
            // 텍스처에 데이터 적용
            redSquareTexture.SetData(data);
        }

        protected override void Update(GameTime gameTime)
        {
            // 키보드로 사각형 이동
            KeyboardState keyboardState = Keyboard.GetState();
            
            if (keyboardState.IsKeyDown(Keys.Left))
                squarePosition.X -= 5;
            if (keyboardState.IsKeyDown(Keys.Right))
                squarePosition.X += 5;
            if (keyboardState.IsKeyDown(Keys.Up))
                squarePosition.Y -= 5;
            if (keyboardState.IsKeyDown(Keys.Down))
                squarePosition.Y += 5;
            
            if (keyboardState.IsKeyDown(Keys.Escape))
                Exit();

            base.Update(gameTime);
        }

        protected override void Draw(GameTime gameTime)
        {
            GraphicsDevice.Clear(Color.CornflowerBlue);
            
            _spriteBatch.Begin();
            
            // 텍스처를 지정된 위치에 그린다
            _spriteBatch.Draw(redSquareTexture, squarePosition, Color.White);
            
            _spriteBatch.End();
            
            base.Draw(gameTime);
        }
    }
}
```

**코드 설명:**

이 코드의 핵심은 Texture2D를 생성하고 그리는 것이다. LoadContent()에서 텍스처를 만든다.

**new Texture2D(GraphicsDevice, 100, 100)** - 100x100 픽셀 크기의 빈 텍스처를 생성한다. GraphicsDevice는 GPU와의 통신을 담당한다.

**Color[] data** - 텍스처의 모든 픽셀 정보를 저장하는 배열이다. 100x100이므로 10,000개의 픽셀이 있고, 배열의 길이도 10,000이다.

**SetData(data)** - 배열의 색상 정보를 텍스처에 적용한다. 이 메서드 호출 후에야 텍스처가 실제 이미지로 된다.

**_spriteBatch.Draw()** - 텍스처를 화면에 그린다. 첫 번째 매개변수는 그릴 텍스처, 두 번째는 위치, 세 번째는 색상 필터다.

이 코드를 실행하면, 빨간색 정사각형이 화면에 나타나고, 화살표 키로 움직일 수 있을 것이다.

### 파일에서 이미지 로드
실제 게임에서는 보통 PNG나 JPG 파일로 된 이미지를 사용한다. 이를 위해서는 Content Pipeline을 사용해야 한다.

Content Pipeline은 이미지 파일을 게임에 최적화된 형식으로 변환하는 도구다. 다음은 파일 기반 이미지를 로드하는 방법이다.

```csharp
protected override void LoadContent()
{
    _spriteBatch = new SpriteBatch(GraphicsDevice);
    
    // Content 폴더에 있는 "player.png" 파일을 로드한다
    playerTexture = Content.Load<Texture2D>("player");
    
    // Content 폴더의 "Images" 서브폴더에 있는 파일
    backgroundTexture = Content.Load<Texture2D>("Images/background");
}
```

파일을 로드하려면:

1. 이미지 파일을 프로젝트의 Content 폴더에 복사한다.
2. Visual Studio의 솔루션 탐색기에서 Content.mgcb 파일을 더블클릭한다.
3. MonoGame Content Editor가 열리면, "Add" → "Existing Item"을 선택하여 이미지를 추가한다.
4. 저장하고 닫는다.

이 과정을 거치면 게임을 빌드할 때 이미지가 자동으로 최적화되고, Content.Load()로 로드할 수 있게 된다.

## 3.3 좌표계 이해하기
2D 게임에서 좌표계를 정확하게 이해하는 것은 매우 중요하다. 잘못된 좌표 이해는 게임의 기본이 흔들린다.

### MonoGame의 좌표계
MonoGame의 좌표계는 다음과 같다.

```
(0, 0) ─────── X 증가 ─────→

   │
   │
   Y 증가
   │
   ▼

화면 좌측 상단이 원점(0, 0)이다.
```

구체적으로 800x600 해상도라면:

```
(0, 0) ────────────────── (800, 0)
  ┌─────────────────────────┐
  │                         │
  │                         │
  │     게임 화면           │
  │                         │
  │                         │
(0, 600) ──────────────── (800, 600)
```

이는 수학에서 배운 좌표계와 다르다. 수학의 좌표계는 왼쪽 아래가 원점인데, 게임의 좌표계는 왼쪽 위가 원점이다. 이 차이를 꼭 기억해야 한다.

### Vector2를 사용한 위치 표현
Vector2는 X와 Y 좌표를 나타내는 구조체다.

```csharp
// Vector2 생성하기
Vector2 position = new Vector2(100, 50);  // X=100, Y=50

// Vector2 속성 접근
float x = position.X;
float y = position.Y;

// Vector2 수정
position.X = 200;
position.Y = 150;

// 또는 다시 생성
position = new Vector2(200, 150);

// Vector2 연산
Vector2 velocity = new Vector2(5, 0);  // 초당 5픽셀 오른쪽으로 이동
position = position + velocity;        // 위치 업데이트

// 또는
position += velocity;  // 더 간단한 방식
```

### 오브젝트의 위치와 크기
SpriteBatch.Draw() 메서드에는 여러 오버로드(버전)가 있다. 가장 기본적인 것들을 살펴보자.

```csharp
// 버전 1: 기본 그리기
_spriteBatch.Draw(texture, position, color);
// texture: 그릴 텍스처
// position: 그릴 위치 (Vector2)
// color: 색상 필터

// 버전 2: 크기 조절
_spriteBatch.Draw(texture, position, sourceRectangle, color);
// sourceRectangle: 텍스처의 일부만 그릴 때 사용 (Rectangle)

// 버전 3: 완전한 제어
_spriteBatch.Draw(
    texture,                    // 그릴 텍스처
    position,                   // 그릴 위치
    sourceRectangle,            // 텍스처의 일부 (null이면 전체)
    color,                      // 색상 필터
    rotation,                   // 회전 각도 (라디안)
    origin,                     // 회전/확대축 (Vector2)
    scale,                      // 크기 배율 (1.0 = 원래 크기)
    effects,                    // 좌우/상하 반전
    layerDepth                  // 레이어 깊이 (0.0~1.0)
);
```

다음은 다양한 방식으로 이미지를 그리는 예제다.

```csharp
protected override void Draw(GameTime gameTime)
{
    GraphicsDevice.Clear(Color.CornflowerBlue);
    
    _spriteBatch.Begin();
    
    // 1. 기본 그리기 - 좌측 상단 정렬
    _spriteBatch.Draw(texture, new Vector2(50, 50), Color.White);
    
    // 2. 2배 크기로 그리기
    _spriteBatch.Draw(
        texture,
        new Vector2(200, 50),
        null,
        Color.White,
        0,                              // 회전 없음
        Vector2.Zero,                   // 좌측 상단 기준
        2.0f,                           // 2배 크기
        SpriteEffects.None,             // 반전 없음
        0                               // 레이어 깊이
    );
    
    // 3. 중심 기준으로 그리기
    Vector2 centerPosition = new Vector2(400, 300);
    Vector2 origin = new Vector2(texture.Width / 2, texture.Height / 2);  // 텍스처 중심
    
    _spriteBatch.Draw(
        texture,
        centerPosition,
        null,
        Color.White,
        0,
        origin,                         // 중심 기준으로 회전/확대
        1.0f,
        SpriteEffects.None,
        0
    );
    
    _spriteBatch.End();
    
    base.Draw(gameTime);
}
```

**좌표계 팁:**

위 코드에서 중심 기준으로 그리려면 origin을 텍스처의 중심으로 설정해야 한다. 만약 origin이 (0, 0)이면 텍스처의 좌측 상단이 position에 배치된다. 하지만 origin이 (width/2, height/2)이면 텍스처의 중심이 position에 배치된다.

## 3.4 Color와 투명도
MonoGame의 Color 구조체는 RGB(Red, Green, Blue) 값과 Alpha(투명도) 값으로 이루어져 있다. 색상을 이해하는 것은 게임 그래픽의 기본이다.

### Color 구조체
Color는 다음과 같이 표현된다.

```csharp
// 기본 생성자
Color color = new Color(red, green, blue, alpha);
// red: 0~255 (또는 0~1 범위)
// green: 0~255 (또는 0~1 범위)
// blue: 0~255 (또는 0~1 범위)
// alpha: 0~255 (또는 0~1 범위, 생략 가능)

// 예시
Color red = new Color(255, 0, 0);           // 순수 빨강
Color green = new Color(0, 255, 0);         // 순수 초록
Color blue = new Color(0, 0, 255);          // 순수 파랑
Color white = new Color(255, 255, 255);     // 흰색
Color black = new Color(0, 0, 0);           // 검은색
Color gray = new Color(128, 128, 128);      // 회색
```

### 미리 정의된 색상
MonoGame은 자주 사용하는 색상을 미리 정의해두었다.

```csharp
Color.White       // 흰색
Color.Black       // 검은색
Color.Red         // 빨강
Color.Green       // 초록
Color.Blue        // 파랑
Color.Yellow      // 노랑
Color.Cyan        // 시안 (밝은 파랑)
Color.Magenta     // 마젠타 (자주색)
Color.Gray        // 회색
Color.Transparent // 완전 투명
Color.CornflowerBlue  // 옥수수 색 파랑 (MonoGame 기본 배경색)
```

### Alpha와 투명도
Alpha 값은 색상의 투명도를 나타낸다.

```
Alpha = 255 (또는 1.0) → 완전 불투명
Alpha = 128 (또는 0.5) → 50% 투명도
Alpha = 0 (또는 0.0)   → 완전 투명
```

```csharp
// 반투명한 빨간색
Color semiTransparentRed = new Color(255, 0, 0, 128);  // 50% 투명

// 완전 투명한 초록
Color fullyTransparent = new Color(0, 255, 0, 0);

// 기존 색상의 알파값을 변경
Color originalColor = Color.Red;
Color transparentVersion = originalColor * 0.5f;  // 50% 투명도로 만들기
```

### Draw() 메서드의 색상 필터
Draw() 메서드에 전달하는 색상은 텍스처의 각 픽셀과 곱해진다(Multiply).

```csharp
_spriteBatch.Begin();

// 흰색(255, 255, 255)으로 그리면 원본 색상 유지
_spriteBatch.Draw(texture, new Vector2(50, 50), Color.White);

// 빨간색(255, 0, 0)으로 그리면 파랑 채널만 0이 되어 빨강/초록이 제거됨
_spriteBatch.Draw(texture, new Vector2(200, 50), Color.Red);

// 반투명(Alpha = 128)하게 그림
Color semiTransparent = Color.White * 0.5f;
_spriteBatch.Draw(texture, new Vector2(350, 50), semiTransparent);

_spriteBatch.End();
```

### 종합 예제: 색상 필터를 이용한 이미지 렌더링
다음은 다양한 색상과 투명도를 사용하여 같은 이미지를 여러 방식으로 그리는 예제다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;

namespace ColorAndTransparencyGame
{
    public class Game1 : Game
    {
        private GraphicsDeviceManager _graphics;
        private SpriteBatch _spriteBatch;
        private Texture2D circleTexture;

        public Game1()
        {
            _graphics = new GraphicsDeviceManager(this);
            Content.RootDirectory = "Content";
            IsMouseVisible = true;
            
            _graphics.PreferredBackBufferWidth = 800;
            _graphics.PreferredBackBufferHeight = 600;
        }

        protected override void Initialize()
        {
            Window.Title = "Color and Transparency";
            base.Initialize();
        }

        protected override void LoadContent()
        {
            _spriteBatch = new SpriteBatch(GraphicsDevice);
            
            // 흰색 원 텍스처 생성
            circleTexture = new Texture2D(GraphicsDevice, 50, 50);
            Color[] data = new Color[50 * 50];
            
            Vector2 center = new Vector2(25, 25);
            float radius = 25;
            
            for (int y = 0; y < 50; y++)
            {
                for (int x = 0; x < 50; x++)
                {
                    float distance = Vector2.Distance(
                        new Vector2(x, y), 
                        center
                    );
                    
                    if (distance <= radius)
                    {
                        data[y * 50 + x] = Color.White;
                    }
                    else
                    {
                        data[y * 50 + x] = Color.Transparent;
                    }
                }
            }
            
            circleTexture.SetData(data);
        }

        protected override void Update(GameTime gameTime)
        {
            if (Keyboard.GetState().IsKeyDown(Keys.Escape))
                Exit();

            base.Update(gameTime);
        }

        protected override void Draw(GameTime gameTime)
        {
            GraphicsDevice.Clear(Color.Black);
            
            _spriteBatch.Begin();
            
            // 1. 순수 빨강
            _spriteBatch.Draw(
                circleTexture, 
                new Vector2(50, 50), 
                Color.Red
            );
            
            // 2. 순수 초록
            _spriteBatch.Draw(
                circleTexture, 
                new Vector2(150, 50), 
                Color.Green
            );
            
            // 3. 순수 파랑
            _spriteBatch.Draw(
                circleTexture, 
                new Vector2(250, 50), 
                Color.Blue
            );
            
            // 4. 노랑 (빨강 + 초록)
            _spriteBatch.Draw(
                circleTexture, 
                new Vector2(350, 50), 
                Color.Yellow
            );
            
            // 5. 50% 투명도의 하얀색
            _spriteBatch.Draw(
                circleTexture, 
                new Vector2(450, 50), 
                Color.White * 0.5f
            );
            
            // 6. 75% 투명도의 하얀색
            _spriteBatch.Draw(
                circleTexture, 
                new Vector2(550, 50), 
                Color.White * 0.25f
            );
            
            // 7. 색상과 투명도 조합
            Color customColor = new Color(255, 200, 0, 128);  // 주황색, 50% 투명
            _spriteBatch.Draw(
                circleTexture, 
                new Vector2(650, 50), 
                customColor
            );
            
            // 8. 여러 원을 겹쳐서 그리기 (투명도로 인한 혼합)
            for (int i = 0; i < 5; i++)
            {
                Color blendColor = Color.Red * 0.4f;  // 빨강, 40% 투명
                _spriteBatch.Draw(
                    circleTexture, 
                    new Vector2(100 + i * 20, 200 + i * 20), 
                    blendColor
                );
            }
            
            _spriteBatch.End();
            
            base.Draw(gameTime);
        }
    }
}
```

**코드 설명:**

이 코드는 여러 개의 원형 이미지를 다양한 색상과 투명도로 렌더링한다.

**원 텍스처 생성** - LoadContent()에서 거리를 계산하여 원형 텍스처를 생성한다. 중심에서 거리가 반지름보다 작으면 흰색, 아니면 투명하게 설정한다.

**색상 필터** - Draw()의 color 매개변수를 다양하게 설정하여 같은 원 텍스처를 여러 색상으로 렌더링한다.

**투명도 조절** - Color.White * 0.5f는 하얀색을 50% 투명하게 만든다. 0.5 대신 0.25를 사용하면 75% 투명해진다.

**겹침 효과** - 투명도가 있는 이미지를 여러 번 그리면, 겹친 부분이 더 진해진다. 이는 BlendState.AlphaBlend에 의해 색상이 누적되기 때문이다.
