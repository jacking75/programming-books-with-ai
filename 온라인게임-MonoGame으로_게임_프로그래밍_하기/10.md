# 10장. 타이머와 게임 상태
게임을 만들면서 가장 중요한 개념 중 하나가 바로 시간 관리이다. 플레이어가 얼마나 오래 게임을 했는지, 특정 이벤트가 언제 발생해야 하는지, 그리고 게임이 어느 단계에 있는지를 추적해야 한다. 이 장에서는 MonoGame의 GameTime을 활용하여 시간을 관리하고, 게임의 여러 상태(메뉴, 플레이, 종료)를 효과적으로 관리하는 방법을 배운다.

## 10.1 GameTime 활용
MonoGame의 게임 루프는 매 프레임마다 Update와 Draw 메서드를 호출한다. 이때 GameTime 객체가 전달되는데, 이 객체에는 현재 프레임의 시간 정보가 담겨있다.

### GameTime의 주요 속성
GameTime 객체에는 다음과 같은 중요한 속성들이 있다.

**ElapsedGameTime**은 이전 프레임부터 현재 프레임까지 경과한 시간을 나타낸다. 이 값은 TimeSpan 타입이며, 플레이어의 동작을 시간에 독립적으로 움직일 수 있게 해준다.

**TotalGameTime**은 게임이 시작된 후부터 현재까지의 총 경과 시간이다. 게임 전체의 진행 상황을 추적할 때 사용한다.

**IsRunningSlowly**는 프레임 레이트가 충분하지 않을 때 true가 된다. 게임이 버벅거리고 있음을 감지할 수 있다.

다음 코드는 GameTime을 활용하여 플레이어를 일정한 속도로 움직이는 예제이다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Input;

public class Player
{
    public Vector2 Position { get; set; }
    private float Speed = 200f; // 초당 200 픽셀로 이동
    
    public void Update(GameTime gameTime)
    {
        float deltaTime = (float)gameTime.ElapsedGameTime.TotalSeconds;
        KeyboardState keyboardState = Keyboard.GetState();
        
        Vector2 movement = Vector2.Zero;
        
        if (keyboardState.IsKeyDown(Keys.Left))
            movement.X -= 1;
        if (keyboardState.IsKeyDown(Keys.Right))
            movement.X += 1;
        if (keyboardState.IsKeyDown(Keys.Up))
            movement.Y -= 1;
        if (keyboardState.IsKeyDown(Keys.Down))
            movement.Y += 1;
        
        // 벡터를 정규화하여 대각선 이동 시 속도가 같도록 함
        if (movement != Vector2.Zero)
            movement.Normalize();
        
        Position += movement * Speed * deltaTime;
    }
}
```

이 코드의 핵심은 `deltaTime`을 구하는 부분이다. `gameTime.ElapsedGameTime.TotalSeconds`는 이전 프레임부터 현재 프레임까지의 경과 시간을 초 단위로 나타낸다. 이를 사용하면 프레임 레이트가 60fps이든 120fps이든 플레이어의 이동 속도는 항상 초당 200 픽셀로 일정하게 유지된다.

예를 들어 프레임 레이트가 60fps라면 각 프레임의 deltaTime은 약 0.0167초이고, 120fps라면 약 0.0083초이다. 이 값을 속도(200)와 곱하면 프레임 레이트에 관계없이 일관된 이동이 보장된다.

### 누적 시간 활용
TotalGameTime을 활용하면 게임 시작 후 경과한 총 시간을 추적할 수 있다. 다음은 일정 시간마다 특정 동작을 수행하는 예제이다.

```csharp
public class ScoreDisplay
{
    private double timeSinceLastUpdate = 0;
    private int displayValue = 0;
    
    public void Update(GameTime gameTime)
    {
        timeSinceLastUpdate += gameTime.ElapsedGameTime.TotalSeconds;
        
        // 0.5초마다 점수 표시를 업데이트
        if (timeSinceLastUpdate >= 0.5)
        {
            displayValue++;
            timeSinceLastUpdate = 0;
        }
    }
}
```

## 10.2 타이머 구현
게임에서는 특정 기능을 위한 타이머가 자주 필요하다. 예를 들어 게임이 시작되기 전에 3초 카운트다운을 표시하거나, 플레이어가 특정 상태에 얼마나 오래 있었는지 추적해야 한다. 재사용 가능한 타이머 클래스를 만들어보자.

```csharp
using System;

/// <summary>
/// 게임에서 사용할 수 있는 재사용 가능한 타이머 클래스이다.
/// </summary>
public class Timer
{
    /// <summary>
    /// 타이머가 설정된 시간(초 단위)
    /// </summary>
    private float totalDuration;
    
    /// <summary>
    /// 현재 경과한 시간(초 단위)
    /// </summary>
    private float elapsedTime;
    
    /// <summary>
    /// 타이머가 현재 실행 중인지 여부
    /// </summary>
    private bool isActive;
    
    /// <summary>
    /// 타이머의 현재 진행률(0.0 ~ 1.0)
    /// </summary>
    public float Progress => totalDuration > 0 ? elapsedTime / totalDuration : 0;
    
    /// <summary>
    /// 타이머가 완료되었는지 여부
    /// </summary>
    public bool IsFinished => elapsedTime >= totalDuration;
    
    /// <summary>
    /// 남은 시간(초 단위)
    /// </summary>
    public float RemainingTime => Math.Max(0, totalDuration - elapsedTime);
    
    public Timer(float duration)
    {
        totalDuration = duration;
        elapsedTime = 0;
        isActive = false;
    }
    
    /// <summary>
    /// 타이머를 시작한다.
    /// </summary>
    public void Start()
    {
        isActive = true;
    }
    
    /// <summary>
    /// 타이머를 일시 중지한다.
    /// </summary>
    public void Pause()
    {
        isActive = false;
    }
    
    /// <summary>
    /// 타이머를 초기화하고 0부터 시작한다.
    /// </summary>
    public void Reset()
    {
        elapsedTime = 0;
        isActive = false;
    }
    
    /// <summary>
    /// 타이머를 초기화하고 바로 시작한다.
    /// </summary>
    public void Restart()
    {
        elapsedTime = 0;
        isActive = true;
    }
    
    /// <summary>
    /// 매 프레임마다 호출하여 타이머를 업데이트한다.
    /// </summary>
    public void Update(Microsoft.Xna.Framework.GameTime gameTime)
    {
        if (!isActive)
            return;
        
        elapsedTime += (float)gameTime.ElapsedGameTime.TotalSeconds;
        
        // 시간이 초과되면 totalDuration에서 멈춘다
        if (elapsedTime >= totalDuration)
        {
            elapsedTime = totalDuration;
            isActive = false;
        }
    }
}
```

이 Timer 클래스의 설계 목적은 게임에서 자주 사용되는 타이밍 로직을 일반화하는 것이다. Start로 시작하고, Update로 시간을 누적하고, IsFinished로 완료 여부를 확인하는 흐름을 표준화했다. Progress 속성은 로딩 바를 그릴 때 유용하고, RemainingTime은 카운트다운 디스플레이에 사용할 수 있다.

다음은 타이머를 실제로 사용하는 예제이다.

```csharp
public class GameLogic
{
    private Timer countdownTimer;
    private Timer invincibilityTimer;
    
    public GameLogic()
    {
        countdownTimer = new Timer(3.0f); // 3초 카운트다운
        invincibilityTimer = new Timer(2.0f); // 2초 무적 상태
    }
    
    public void StartRound()
    {
        countdownTimer.Restart();
    }
    
    public void MakePlayerInvincible()
    {
        invincibilityTimer.Restart();
    }
    
    public void Update(GameTime gameTime)
    {
        countdownTimer.Update(gameTime);
        invincibilityTimer.Update(gameTime);
        
        // 카운트다운이 끝나면 게임 시작
        if (countdownTimer.IsFinished)
        {
            // 게임 시작 로직
        }
        
        // 무적 상태 여부 확인
        bool isInvincible = !invincibilityTimer.IsFinished;
    }
}
```

## 10.3 게임 상태 관리 (메뉴, 플레이, 종료)
게임은 여러 상태를 가진다. 메뉴 화면에서는 플레이 화면과 다른 입력을 처리해야 하고, 게임 오버 화면에서도 다르게 동작한다. 이런 상태들을 효과적으로 관리하려면 상태 패턴(State Pattern)을 사용하는 것이 좋다.

먼저 게임 상태의 종류를 열거형으로 정의하자.

```csharp
public enum GameState
{
    Menu,      // 메뉴 화면
    Playing,   // 게임 진행 중
    Paused,    // 일시 중지
    GameOver   // 게임 오버
}
```

게임 상태를 관리하는 GameStateManager 클래스를 만들어보자.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;

/// <summary>
/// 게임의 여러 상태를 관리하는 클래스이다.
/// 각 상태에 따라 다른 업데이트와 렌더링 로직을 수행한다.
/// </summary>
public class GameStateManager
{
    private GameState currentState;
    private GameState nextState;
    
    // 각 상태에서 필요한 데이터
    private int playerScore;
    private string gameOverReason;
    private Timer stateTransitionTimer;
    
    public GameState CurrentState => currentState;
    public int PlayerScore => playerScore;
    
    public GameStateManager()
    {
        currentState = GameState.Menu;
        nextState = GameState.Menu;
        playerScore = 0;
        gameOverReason = "";
        stateTransitionTimer = new Timer(0.5f);
    }
    
    /// <summary>
    /// 게임 상태를 변경한다.
    /// 상태 전환 시 초기화 로직이 필요하면 이 메서드에서 처리한다.
    /// </summary>
    public void ChangeState(GameState newState)
    {
        if (currentState == newState)
            return;
        
        // 이전 상태 정리
        OnStateExit();
        
        currentState = newState;
        
        // 새 상태 초기화
        OnStateEnter();
    }
    
    /// <summary>
    /// 현재 상태에서 벗어날 때 호출된다.
    /// </summary>
    private void OnStateExit()
    {
        switch (currentState)
        {
            case GameState.Playing:
                // 게임 재시작 전 필요한 정리
                break;
            case GameState.Menu:
                // 메뉴에서 벗어날 때 처리
                break;
        }
    }
    
    /// <summary>
    /// 새로운 상태로 진입할 때 호출된다.
    /// </summary>
    private void OnStateEnter()
    {
        switch (currentState)
        {
            case GameState.Menu:
                playerScore = 0;
                gameOverReason = "";
                break;
            case GameState.Playing:
                playerScore = 0;
                break;
            case GameState.Paused:
                // 일시 중지 상태 초기화
                break;
            case GameState.GameOver:
                // 게임 오버 상태 초기화
                break;
        }
    }
    
    /// <summary>
    /// 현재 상태에 따라 업데이트 로직을 수행한다.
    /// </summary>
    public void Update(GameTime gameTime)
    {
        switch (currentState)
        {
            case GameState.Menu:
                UpdateMenu(gameTime);
                break;
            case GameState.Playing:
                UpdatePlaying(gameTime);
                break;
            case GameState.Paused:
                UpdatePaused(gameTime);
                break;
            case GameState.GameOver:
                UpdateGameOver(gameTime);
                break;
        }
    }
    
    /// <summary>
    /// 메뉴 상태에서의 업데이트 로직이다.
    /// </summary>
    private void UpdateMenu(GameTime gameTime)
    {
        KeyboardState keyboardState = Keyboard.GetState();
        
        if (keyboardState.IsKeyDown(Keys.Enter))
        {
            ChangeState(GameState.Playing);
        }
    }
    
    /// <summary>
    /// 게임 진행 중의 업데이트 로직이다.
    /// </summary>
    private void UpdatePlaying(GameTime gameTime)
    {
        KeyboardState keyboardState = Keyboard.GetState();
        
        // 일시 중지 처리
        if (keyboardState.IsKeyDown(Keys.P))
        {
            ChangeState(GameState.Paused);
        }
        
        // 게임 로직: 점수 증가 등
        playerScore += 10;
        
        // 특정 조건에서 게임 오버
        if (playerScore >= 1000)
        {
            gameOverReason = "승리! 1000점을 달성했습니다!";
            ChangeState(GameState.GameOver);
        }
    }
    
    /// <summary>
    /// 일시 중지 상태에서의 업데이트 로직이다.
    /// </summary>
    private void UpdatePaused(GameTime gameTime)
    {
        KeyboardState keyboardState = Keyboard.GetState();
        
        if (keyboardState.IsKeyDown(Keys.P))
        {
            ChangeState(GameState.Playing);
        }
        
        if (keyboardState.IsKeyDown(Keys.M))
        {
            ChangeState(GameState.Menu);
        }
    }
    
    /// <summary>
    /// 게임 오버 상태에서의 업데이트 로직이다.
    /// </summary>
    private void UpdateGameOver(GameTime gameTime)
    {
        KeyboardState keyboardState = Keyboard.GetState();
        
        if (keyboardState.IsKeyDown(Keys.R))
        {
            ChangeState(GameState.Menu);
        }
    }
    
    /// <summary>
    /// 현재 상태에 따라 화면을 렌더링한다.
    /// </summary>
    public void Draw(SpriteBatch spriteBatch, SpriteFont font)
    {
        switch (currentState)
        {
            case GameState.Menu:
                DrawMenu(spriteBatch, font);
                break;
            case GameState.Playing:
                DrawPlaying(spriteBatch, font);
                break;
            case GameState.Paused:
                DrawPaused(spriteBatch, font);
                break;
            case GameState.GameOver:
                DrawGameOver(spriteBatch, font);
                break;
        }
    }
    
    private void DrawMenu(SpriteBatch spriteBatch, SpriteFont font)
    {
        spriteBatch.DrawString(font, "메뉴", new Vector2(100, 100), Color.White);
        spriteBatch.DrawString(font, "Enter를 눌러 시작", new Vector2(100, 150), Color.White);
    }
    
    private void DrawPlaying(SpriteBatch spriteBatch, SpriteFont font)
    {
        spriteBatch.DrawString(font, $"점수: {playerScore}", new Vector2(10, 10), Color.White);
        spriteBatch.DrawString(font, "P: 일시중지", new Vector2(10, 40), Color.White);
    }
    
    private void DrawPaused(SpriteBatch spriteBatch, SpriteFont font)
    {
        spriteBatch.DrawString(font, "일시 중지", new Vector2(100, 100), Color.White);
        spriteBatch.DrawString(font, "P: 재개", new Vector2(100, 150), Color.White);
        spriteBatch.DrawString(font, "M: 메뉴로", new Vector2(100, 200), Color.White);
    }
    
    private void DrawGameOver(SpriteBatch spriteBatch, SpriteFont font)
    {
        spriteBatch.DrawString(font, "게임 오버", new Vector2(100, 100), Color.White);
        spriteBatch.DrawString(font, gameOverReason, new Vector2(100, 150), Color.White);
        spriteBatch.DrawString(font, $"최종 점수: {playerScore}", new Vector2(100, 200), Color.White);
        spriteBatch.DrawString(font, "R: 다시 시작", new Vector2(100, 250), Color.White);
    }
}
```

이 GameStateManager 클래스의 설계는 상태 패턴에 기반하고 있다. 각 상태마다 별도의 Update와 Draw 메서드를 가지고 있어서, 상태별로 다른 로직을 명확하게 분리할 수 있다. ChangeState 메서드는 상태 전환 시 정리와 초기화 작업을 자동으로 처리한다.

다음은 MonoGame의 Game 클래스에서 GameStateManager를 사용하는 방법이다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;

public class MyGame : Game
{
    private GraphicsDeviceManager graphics;
    private SpriteBatch spriteBatch;
    private GameStateManager stateManager;
    private SpriteFont font;
    
    public MyGame()
    {
        graphics = new GraphicsDeviceManager(this);
        Content.RootDirectory = "Content";
    }
    
    protected override void Initialize()
    {
        stateManager = new GameStateManager();
        base.Initialize();
    }
    
    protected override void LoadContent()
    {
        spriteBatch = new SpriteBatch(GraphicsDevice);
        font = Content.Load<SpriteFont>("Arial");
    }
    
    protected override void Update(GameTime gameTime)
    {
        stateManager.Update(gameTime);
        base.Update(gameTime);
    }
    
    protected override void Draw(GameTime gameTime)
    {
        GraphicsDevice.Clear(Color.Black);
        
        spriteBatch.Begin();
        stateManager.Draw(spriteBatch, font);
        spriteBatch.End();
        
        base.Draw(gameTime);
    }
}
```

## 과제

### 과제 1: 타이머를 활용한 카운트다운 표시

**목표:** Timer 클래스를 사용하여 게임 시작 전 3초 카운트다운을 구현하고, 화면에 남은 시간을 초 단위로 표시하는 프로그램을 작성하자.

**요구사항:**
- Timer 클래스를 사용하여 3초 타이머를 만든다.
- GameTime을 활용하여 매 프레임마다 타이머를 업데이트한다.
- 화면에 남은 시간을 정수로 표시한다. (예: "3", "2", "1", "0")
- 타이머가 완료되면 "게임 시작!" 메시지를 표시한다.
- 스페이스 바를 누르면 카운트다운을 다시 시작한다.

### 과제 2: 게임 상태 관리 시스템 확장
**목표:** GameStateManager를 확장하여 하이스코어를 저장하고, 메뉴에서 하이스코어를 표시하는 기능을 추가하자.

**요구사항:**
- GameStateManager에 highScore 필드를 추가한다.
- 게임 오버 시 현재 점수가 하이스코어보다 높으면 하이스코어를 업데이트한다.
- 메뉴 화면에 "하이스코어: XXXX" 형태로 표시한다.
- 게임을 다시 시작해도 하이스코어는 유지된다. (메뉴 상태에서만 초기화되지 않음)
- 게임 오버 화면에 "개인 최고 기록: XXXX"를 표시한다.

### 과제 3: 복합 타이머 시스템

**목표:** 여러 개의 타이머를 동시에 관리하고, 특정 이벤트를 시간에 따라 순차적으로 발생시키는 시스템을 만들자.

**요구사항:**
- 적어도 3개 이상의 Timer를 생성한다. (예: powerUpTimer, invincibilityTimer, countdownTimer)
- 각 타이머의 남은 시간을 화면에 표시한다.
- 각 타이머가 완료될 때마다 다른 텍스트를 표시한다. (예: "파워업!", "무적 해제!", "라운드 시작!")
- Spacebar를 누르면 모든 타이머를 일시 중지하고, 다시 누르면 재개한다.
- 각 타이머의 진행 상태를 시각적인 진행 막대로 표시한다. (너비가 변하는 사각형 등)

**과제 제출 시 다음 사항을 확인하자.**
- 코드가 컴파일되고 에러 없이 실행되는가?
- 모든 요구사항을 충족하는가?
- 변수 이름과 메서드 이름이 명확하고 이해하기 쉬운가?
- 중복된 코드가 없는가?
- 주석이나 설명이 필요한 부분에 적절한 주석을 달았는가?

이 장에서 배운 GameTime과 Timer, 그리고 상태 관리 개념은 게임 개발의 기초가 되는 매우 중요한 부분이다. 특히 상태 관리는 게임의 복잡도가 증가할수록 더욱 중요해지므로, 지금부터 좋은 습관을 들이는 것이 좋다.
