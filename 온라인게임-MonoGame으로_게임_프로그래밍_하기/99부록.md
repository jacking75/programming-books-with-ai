# 부록

## 부록 A. Content Pipeline 도구
MonoGame에서 게임에 사용할 이미지, 사운드, 폰트 같은 에셋(Asset)을 관리하는 방법은 매우 중요하다. 이러한 에셋들을 효율적으로 처리하고 최적화하는 과정을 Content Pipeline이라고 부른다. 이 부록에서는 MonoGame의 Content Pipeline 도구를 깊이 있게 살펴본다.

### A.1 Content.mgcb 편집기

#### Content Pipeline이란
게임 개발에서 에셋은 다양한 형식으로 제공된다. 예를 들어 이미지는 PNG, JPG, BMP 등 여러 형식이 있고, 사운드는 WAV, MP3, OGG 등이 있다. 이런 다양한 형식의 에셋을 게임이 직접 읽기는 어렵다. 따라서 MonoGame은 이들을 통일된 형식으로 변환하는 파이프라인을 제공한다.

Content Pipeline의 역할을 다이어그램으로 나타내면 다음과 같다.

```
원본 에셋 (PNG, WAV, OGG)
        ↓
   Content Pipeline
        ↓
  MonoGame 바이너리 형식
        ↓
   게임에서 로드 및 사용
```

Content.mgcb 파일은 이 파이프라인의 설정을 저장하는 파일이다. 프로젝트를 생성하면 자동으로 이 파일이 생성되는데, 이 파일을 편집해서 에셋을 추가하고 변환 설정을 할 수 있다.

#### Content.mgcb 파일 구조
Content.mgcb 파일은 텍스트 기반의 설정 파일이다. 기본적인 구조는 다음과 같다.

```
#----------------------------- Global Properties ----------------------------#

/outputDir:bin/$(Platform)
/intermediateDir:obj
/platform:DesktopGL
/config:
/profile:Reach
/compress:False

#-------------------------------- References --------------------------------#

#---------------------------------- Content ---------------------------------#

#begin Assets/Images/player.png
/importer:TextureImporter
/processor:TextureProcessor
/build:player

#begin Assets/Sounds/jump.wav
/importer:WavImporter
/processor:SoundEffectProcessor
/build:jump
```

파일의 구조를 설명하면 다음과 같다.

**Global Properties 섹션**에서는 전체 파이프라인의 기본 설정을 정의한다. `outputDir`은 변환된 에셋이 저장될 위치를 지정하고, `platform`은 빌드 대상 플랫폼을 지정한다. MonoGame은 DesktopGL, DirectX, Android, iOS 등 다양한 플랫폼을 지원한다.

**Content 섹션**에서는 각 에셋의 변환 설정을 정의한다. `#begin`으로 시작해서 에셋의 경로를 지정하고, `importer`로 파일을 읽는 방식을 정의하며, `processor`로 변환하는 방식을 정의한다. `build` 옵션은 게임에서 사용할 이름을 지정한다.

#### MonoGame Content Builder 사용하기
.NET 환경에서는 명령줄을 통해 Content Pipeline을 실행한다. 프로젝트 폴더에서 다음 명령을 실행하면 모든 에셋이 변환된다.

```bash
dotnet build-content
```

이 명령은 Content 폴더의 Content.mgcb 파일을 읽고, 정의된 모든 에셋을 변환한다. 변환된 에셋은 `bin` 폴더에 저장된다.

Visual Studio에서 개발 중이라면 프로젝트 파일에 다음 설정을 추가해야 한다.

```xml
<ItemGroup>
  <MonoGameContentReference Include="Content\Content.mgcb" />
</ItemGroup>

<Target Name="RestoreToolVersion" BeforeTargets="Build">
  <Exec Command="dotnet tool restore" />
</Target>
```

이 설정을 추가하면 프로젝트를 빌드할 때 자동으로 Content Pipeline이 실행된다.

#### 에셋 추가하기
새로운 에셋을 프로젝트에 추가하는 절차는 다음과 같다.

먼저 에셋 파일을 `Content` 폴더의 적절한 하위 폴더에 배치한다. 예를 들어 이미지는 `Content/Images`, 사운드는 `Content/Sounds`, 폰트는 `Content/Fonts` 같은 식으로 정리한다.

그 다음 Content.mgcb 파일을 텍스트 에디터로 열고, Content 섹션에 다음과 같은 항목을 추가한다.

```
#begin Images/enemy.png
/importer:TextureImporter
/processor:TextureProcessor
/build:enemy
```

여기서 중요한 것은 `/build:` 옵션이다. 이 값이 게임 코드에서 에셋을 로드할 때 사용하는 이름이다. 따라서 파일 이름과 상관없이 원하는 이름을 지정할 수 있다.

```csharp
// Content.mgcb에서 /build:enemy로 지정했다면
Texture2D enemyTexture = Content.Load<Texture2D>("enemy");
```

### A.2 에셋 임포트 설정

#### 이미지 임포트 설정
이미지 에셋은 `TextureImporter`로 읽고 `TextureProcessor`로 변환한다. 이 과정에서 다양한 옵션을 설정할 수 있다.

```
#begin Images/background.png
/importer:TextureImporter
/processor:TextureProcessor
/processorParam:ColorKeyEnabled=True
/processorParam:ColorKey=255,0,255
/processorParam:TextureFormat=Compressed
/build:background
```

각 파라미터를 설명하면 다음과 같다.

`ColorKeyEnabled`를 True로 설정하면 특정 색상을 투명 처리할 수 있다. `ColorKey` 파라미터로 지정한 RGB 색상이 투명하게 된다. 일반적으로 마젠타색(255,0,255)을 배경색으로 사용하고 투명 처리한다.

`TextureFormat`은 메모리에 어떤 형식으로 저장할지를 결정한다. `Compressed`를 지정하면 DXT 압축을 사용해서 메모리 사용량을 줄인다. 고품질이 필요하면 `Color` 형식을 사용한다.

```
/processorParam:TextureFormat=Color
```

또한 텍스처를 미리 곱하기(Premultiply) 처리할 수 있다. 이는 반투명 처리가 있는 이미지에서 렌더링 성능을 개선한다.

```
/processorParam:PremultiplyAlpha=True
```

#### 사운드 임포트 설정

사운드 파일도 최적화 옵션을 제공한다.

```
#begin Sounds/music.mp3
/importer:Mp3Importer
/processor:SoundEffectProcessor
/processorParam:Quality=Best
/build:music

#begin Sounds/jump.wav
/importer:WavImporter
/processor:SoundEffectProcessor
/processorParam:Quality=Medium
/build:jump
```

`Quality` 파라미터는 음질을 조정한다. `Best`, `Medium`, `Low` 세 가지 옵션이 있다. 음악 파일은 `Best`를 사용하고, 효과음은 `Medium` 정도면 충분하다.

배경 음악은 `SongProcessor`를 사용해서 따로 처리할 수 있다. 이렇게 하면 게임 중간에 음악을 일시정지하고 재개할 수 있다.

```
#begin Sounds/background_music.ogg
/importer:OggImporter
/processor:SongProcessor
/build:bgm
```

#### 폰트 임포트 설정
게임에서 사용할 폰트도 Content Pipeline을 통해 변환된다. MonoGame은 XNA Font Format(.xnb)을 지원한다.

```
#begin Fonts/Arial.spritefont
/importer:FontDescriptionImporter
/processor:FontDescriptionProcessor
/processorParam:TextureFormat=Compressed
/build:Arial
```

폰트 설정 파일은 XML 형식의 `.spritefont` 파일이다. 기본 구조는 다음과 같다.

```xml
<?xml version="1.0" encoding="utf-8"?>
<XnaContent xmlns:n="Microsoft.Xna.Framework.Content.Pipeline.Graphics">
  <Asset Type="FontDescriptionProcessor">
    <FontName>Arial</FontName>
    <Size>24</Size>
    <Spacing>0</Spacing>
    <UseKerning>true</UseKerning>
    <CharacterRegions>
      <CharacterRegion>
        <Start>32</Start>
        <End>126</End>
      </CharacterRegion>
      <CharacterRegion>
        <Start>0xAC00</Start>
        <End>0xD7A3</End>
      </CharacterRegion>
    </CharacterRegions>
  </Asset>
</XnaContent>
```

`FontName`은 시스템에 설치된 폰트명을 지정한다. `Size`는 픽셀 단위 크기다. `CharacterRegion`은 포함할 문자의 범위를 정의한다. 한글을 포함하려면 0xAC00부터 0xD7A3까지의 범위를 추가해야 한다.

### A.3 커스텀 프로세서

#### 커스텀 프로세서 만들기
때로는 기본 제공되는 프로세서로는 부족할 수 있다. 예를 들어 특수한 이미지 형식이나 맞춤형 데이터 형식을 처리해야 한다면 커스텀 프로세서를 만들어야 한다.

커스텀 프로세서를 만들기 위해 먼저 `ContentProcessor` 기본 클래스를 상속받는다.

```csharp
using Microsoft.Xna.Framework.Content.Pipeline;
using Microsoft.Xna.Framework.Content.Pipeline.Graphics;
using Microsoft.Xna.Framework.Graphics;

namespace ContentProcessors
{
    /// <summary>
    /// 텍스처에 추가 정보를 저장하는 커스텀 프로세서다.
    /// 기본 텍스처 프로세서에 메타데이터를 추가한다.
    /// </summary>
    [ContentProcessor(DisplayName = "Custom Texture Processor")]
    public class CustomTextureProcessor : TextureProcessor
    {
        /// <summary>
        /// 프로세서의 입력 형식을 지정한다.
        /// 이 경우 TextureContent를 입력받는다.
        /// </summary>
        public override string InputType
        {
            get { return "TextureContent"; }
        }

        /// <summary>
        /// 프로세서의 출력 형식을 지정한다.
        /// Texture2D로 변환하여 출력한다.
        /// </summary>
        public override string OutputType
        {
            get { return typeof(Texture2D).FullName; }
        }

        /// <summary>
        /// 텍스처의 가로 크기를 읽어 저장한다.
        /// </summary>
        [ContentProcessorParameter]
        public int TextureWidth { get; set; }

        /// <summary>
        /// 텍스처의 세로 크기를 읽어 저장한다.
        /// </summary>
        [ContentProcessorParameter]
        public int TextureHeight { get; set; }

        /// <summary>
        /// 실제 처리 로직을 구현한다.
        /// </summary>
        public override TextureContent Process(TextureContent input, 
            ContentProcessorContext context)
        {
            // 기본 텍스처 프로세싱 수행
            var processedTexture = base.Process(input, context);

            // 텍스처 크기 검증
            if (input.Faces[0][0].Width % 2 != 0 || 
                input.Faces[0][0].Height % 2 != 0)
            {
                context.Logger.LogWarning(null, null,
                    "Texture dimensions should be power of 2!");
            }

            return processedTexture;
        }
    }
}
```

이 커스텀 프로세서는 기본 `TextureProcessor`를 상속받고, 텍스처 크기가 2의 거듭제곱인지 검증한다. 2의 거듭제곱 크기는 그래픽스 카드에서 더 효율적으로 처리되기 때문에 게임 성능에 중요하다.

커스텀 프로세서를 사용하려면 Content.mgcb 파일에서 `/processor` 옵션을 지정해야 한다.

```
#begin Images/special_texture.png
/importer:TextureImporter
/processor:CustomTextureProcessor
/build:special_texture
```

#### 데이터 구조 직렬화
게임에서 복잡한 데이터 구조를 Content Pipeline으로 처리하려면 커스텀 프로세서를 통해 직렬화해야 한다. 예를 들어 맵 데이터를 JSON으로 작성하고, 이를 게임이 읽을 수 있는 형식으로 변환하는 경우를 생각해 보자.

먼저 게임에서 사용할 데이터 클래스를 정의한다.

```csharp
namespace GameContent
{
    /// <summary>
    /// 타일맵 데이터를 저장하는 클래스다.
    /// 각 타일의 정보와 맵의 크기를 포함한다.
    /// </summary>
    public class TileMapData
    {
        public int Width { get; set; }
        public int Height { get; set; }
        public int[] Tiles { get; set; }

        public TileMapData()
        {
        }

        public TileMapData(int width, int height)
        {
            Width = width;
            Height = height;
            Tiles = new int[width * height];
        }

        /// <summary>
        /// 주어진 좌표의 타일 ID를 반환한다.
        /// </summary>
        public int GetTile(int x, int y)
        {
            if (x < 0 || x >= Width || y < 0 || y >= Height)
                return 0;
            return Tiles[y * Width + x];
        }

        /// <summary>
        /// 주어진 좌표의 타일 ID를 설정한다.
        /// </summary>
        public void SetTile(int x, int y, int tileId)
        {
            if (x >= 0 && x < Width && y >= 0 && y < Height)
                Tiles[y * Width + x] = tileId;
        }
    }
}
```

이제 JSON을 읽어서 이 데이터 구조로 변환하는 커스텀 프로세서를 만든다.

```csharp
using Microsoft.Xna.Framework.Content.Pipeline;
using System.IO;
using System.Text.Json;
using GameContent;

namespace ContentProcessors
{
    /// <summary>
    /// JSON 형식의 타일맵 파일을 읽어 TileMapData로 변환하는 프로세서다.
    /// JSON 파일은 다음과 같은 구조여야 한다:
    /// {
    ///   "width": 10,
    ///   "height": 10,
    ///   "tiles": [0, 1, 2, ...]
    /// }
    /// </summary>
    [ContentImporter(".json", DisplayName = "JSON Importer")]
    public class JsonImporter : ContentImporter<string>
    {
        public override string Import(string filename, 
            ContentImporterContext context)
        {
            return File.ReadAllText(filename);
        }
    }

    /// <summary>
    /// JSON 형식의 타일맵 데이터를 처리하는 프로세서다.
    /// </summary>
    [ContentProcessor(DisplayName = "TileMap Processor")]
    public class TileMapProcessor : ContentProcessor<string, TileMapData>
    {
        public override TileMapData Process(string input, 
            ContentProcessorContext context)
        {
            var options = new JsonSerializerOptions { PropertyNameCaseInsensitive = true };
            var jsonData = JsonSerializer.Deserialize<TileMapData>(input, options);
            return jsonData;
        }
    }
}
```

이 프로세서를 사용하려면 Content.mgcb 파일에 다음과 같이 추가한다.

```
#begin Maps/level1.json
/importer:JsonImporter
/processor:TileMapProcessor
/build:level1
```

게임 코드에서는 다음과 같이 로드한다.

```csharp
TileMapData mapData = Content.Load<TileMapData>("level1");
```

## 부록 B. 크로스 플랫폼 빌드

MonoGame의 가장 큰 장점 중 하나는 다양한 플랫폼에서 게임을 실행할 수 있다는 것이다. 같은 코드로 Windows, macOS, Linux는 물론 모바일 플랫폼까지 지원한다.

### B.1 Windows 빌드

#### .NET 9.0 SDK 설치

Windows에서 MonoGame 게임을 빌드하려면 먼저 .NET 9.0 SDK를 설치해야 한다. 공식 웹사이트(https://dotnet.microsoft.com/download)에서 최신 버전을 다운로드할 수 있다.

설치 후 명령 프롬프트에서 다음을 실행해 설치 확인한다.

```bash
dotnet --version
```

#### 프로젝트 파일 설정

.NET 환경에서 Windows 빌드를 위해 프로젝트 파일(.csproj)을 다음과 같이 설정한다.

```xml
<Project Sdk="Microsoft.NET.Sdk.WindowsDesktop">
  <PropertyGroup>
    <OutputType>WinExe</OutputType>
    <TargetFramework>net9.0-windows</TargetFramework>
    <UseDesktopSWF>true</UseDesktopSWF>
    <PublishReadyToRun>true</PublishReadyToRun>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="MonoGame.Framework.DesktopGL" Version="3.8.1.303" />
    <PackageReference Include="MonoGame.Content.Builder.Task" Version="3.8.1.303" />
  </ItemGroup>

  <ItemGroup>
    <MonoGameContentReference Include="Content\Content.mgcb" />
  </ItemGroup>
</Project>
```

`TargetFramework`을 `net9.0-windows`로 지정하면 Windows 전용 기능을 사용할 수 있다. `PublishReadyToRun`을 true로 설정하면 JIT 컴파일 없이 실행되어 시작 속도가 빨라진다.

#### 빌드 및 실행

개발 중에는 다음 명령으로 게임을 실행한다.

```bash
dotnet run
```

릴리스 모드로 빌드하려면 다음을 사용한다.

```bash
dotnet publish -c Release
```

이 명령은 `bin/Release/net9.0-windows/publish` 폴더에 배포 가능한 파일을 생성한다. 이 폴더의 모든 파일을 압축하면 배포 패키지가 된다.

#### 독립 실행형 빌드

사용자가 .NET 런타임을 따로 설치하지 않아도 되도록 독립 실행형으로 빌드할 수 있다.

```xml
<PropertyGroup>
  <PublishSelfContained>true</PublishSelfContained>
  <RuntimeIdentifier>win-x64</RuntimeIdentifier>
</PropertyGroup>
```

이렇게 설정하고 다음 명령을 실행한다.

```bash
dotnet publish -c Release -r win-x64
```

생성된 exe 파일은 .NET 없이도 독립적으로 실행된다.

### B.2 macOS 빌드

#### macOS 환경 준비

macOS에서도 .NET SDK를 설치해야 한다. Homebrew를 사용하면 쉽게 설치할 수 있다.

```bash
brew install dotnet
```

M1/M2 칩과 Intel 칩 모두에서 실행 가능한 범용 바이너리를 만들 수 있다.

#### 프로젝트 파일 설정

macOS용 프로젝트 파일 설정은 다음과 같다.

```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <OutputType>WinExe</OutputType>
    <TargetFramework>net9.0</TargetFramework>
    <RuntimeIdentifier>osx-arm64</RuntimeIdentifier>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="MonoGame.Framework.DesktopGL" Version="3.8.1.303" />
    <PackageReference Include="MonoGame.Content.Builder.Task" Version="3.8.1.303" />
  </ItemGroup>
</Project>
```

`RuntimeIdentifier`를 `osx-arm64`(Apple Silicon)로 설정하거나 `osx-x64`(Intel)로 설정한다.

#### 앱 번들 생성
macOS에서 배포하려면 앱 번들 형식으로 만들어야 한다. 다음 스크립트를 사용한다.

```bash
#!/bin/bash

APP_NAME="MyGame"
BUNDLE_NAME="${APP_NAME}.app"

# 앱 번들 디렉토리 구조 생성
mkdir -p "${BUNDLE_NAME}/Contents/MacOS"
mkdir -p "${BUNDLE_NAME}/Contents/Resources"

# 실행 파일 복사
cp "bin/Release/net9.0/MyGame" "${BUNDLE_NAME}/Contents/MacOS/"

# Info.plist 생성
cat > "${BUNDLE_NAME}/Contents/Info.plist" << EOF
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" 
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>CFBundleName</key>
  <string>${APP_NAME}</string>
  <key>CFBundleExecutable</key>
  <string>${APP_NAME}</string>
  <key>CFBundleVersion</key>
  <string>1.0</string>
</dict>
</plist>
EOF

echo "앱 번들 생성 완료: ${BUNDLE_NAME}"
```

이 스크립트를 실행하면 macOS 앱 번들이 생성되고, Finder에서 실행할 수 있다.

#### 범용 바이너리 생성

Intel과 Apple Silicon 모두에서 실행되는 범용 앱을 만들려면 두 플랫폼 모두에서 빌드한 후 합쳐야 한다.

```bash
# Intel 빌드
dotnet publish -c Release -r osx-x64

# Apple Silicon 빌드
dotnet publish -c Release -r osx-arm64

# 바이너리 합치기
lipo -create \
  bin/Release/net9.0/osx-x64/publish/MyGame \
  bin/Release/net9.0/osx-arm64/publish/MyGame \
  -output universal_MyGame
```

### B.3 Linux 빌드

#### Linux 환경 준비

Linux에서 MonoGame을 사용하려면 추가 라이브러리를 설치해야 한다. Ubuntu/Debian의 경우:

```bash
sudo apt-get install libsdl2-2.0-0 libsdl2-dev openal-soft libopenal-dev
```

Red Hat/Fedora의 경우:

```bash
sudo dnf install SDL2 SDL2-devel OpenAL-soft openal-soft-devel
```

#### 프로젝트 파일 설정

Linux 빌드를 위한 프로젝트 파일 설정은 다음과 같다.

```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>net9.0</TargetFramework>
    <RuntimeIdentifier>linux-x64</RuntimeIdentifier>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="MonoGame.Framework.DesktopGL" Version="3.8.1.303" />
    <PackageReference Include="MonoGame.Content.Builder.Task" Version="3.8.1.303" />
  </ItemGroup>
</Project>
```

#### 빌드 및 실행
Linux에서도 다른 플랫폼과 마찬가지로 빌드한다.

```bash
dotnet publish -c Release -r linux-x64
```

생성된 실행 파일에 실행 권한을 부여해야 한다.

```bash
chmod +x bin/Release/net9.0/linux-x64/publish/MyGame
./bin/Release/net9.0/linux-x64/publish/MyGame
```

#### Snap 패키지로 배포
Linux에서 널리 사용되는 Snap 형식으로 배포할 수 있다. 프로젝트 루트에 `snapcraft.yaml` 파일을 생성한다.

```yaml
name: mygame
version: '1.0'
summary: MonoGame으로 만든 게임
description: |
  MonoGame 엔진으로 만든 재미있는 게임입니다.

grade: stable
confinement: strict

apps:
  mygame:
    command: bin/MyGame
    plugs: [opengl, audio-playback]

parts:
  mygame:
    plugin: dotnet
    source: .
    dotnet-version: net9.0
    dotnet-build-configuration: Release
```

Snap을 빌드하려면 `snapcraft` 도구를 설치하고 다음을 실행한다.

```bash
snapcraft
```

### C.3 무료 에셋 사이트

게임 개발에는 다양한 에셋이 필요하다. 다음 사이트에서 무료로 에셋을 다운로드할 수 있다.

#### 이미지 에셋

**OpenGameArt.org**: https://opengameart.org/

고품질의 무료 게임 그래픽을 제공한다.

- Sprites: 캐릭터, 적, 환경 그래픽
- Textures: 배경 텍스처
- Tile Sets: 타일맵용 타일 세트

**Itch.io Game Assets**: https://itch.io/game-assets/free

아마추어부터 전문가까지 다양한 디자이너가 올린 에셋이 있다.

- 검색 기능으로 원하는 에셋을 쉽게 찾을 수 있다
- 일부는 상업용으로도 사용 가능하다

**Freepik**: https://www.freepik.com/

벡터 그래픽과 일러스트레이션이 풍부하다.

- AI 생성 이미지도 포함되어 있다
- 무료 버전과 프리미엄 버전이 있다

#### 사운드 에셋

**Freesound.org**: https://freesound.org/

다양한 효과음과 음악을 제공한다.

- 효과음: 발소리, 충돌음, 파괴음 등
- 음악: 다양한 장르의 배경음악
- 모두 크리에이티브 커먼즈 라이선스를 사용한다

**Zapsplat**: https://www.zapsplat.com/

무료로 이용할 수 있는 사운드 라이브러리다.

- 효과음: 1,500개 이상
- 음악: 5,000개 이상
- 가입하면 모두 무료로 사용 가능하다

**YouTube Audio Library**: https://www.youtube.com/audiolibrary

YouTube 채널에서 제공하는 무료 음악과 효과음이다.

- YouTube 비디오에 삽입할 수 있다
- 일부는 게임에서도 사용 가능하다

#### 폰트

**Google Fonts**: https://fonts.google.com/

무료로 사용할 수 있는 오픈 소스 폰트 1,000개 이상을 제공한다.

- 한글 폰트도 많다
- 상업용으로도 자유롭게 사용 가능하다
- CSS를 통해 웹에서도 사용할 수 있다

**DaFont**: https://www.dafont.com/

더 창의적인 폰트를 찾을 수 있다.

- 다양한 스타일의 폰트가 있다
- 라이선스 확인 후 사용해야 한다
- 일부는 상업용 사용이 제한될 수 있다

#### 배경 음악

**Incompetech**: https://incompetech.com/

Kevin MacLeod가 만든 로열티 프리 음악을 제공한다.

- 게임, 동영상, 팟캐스트 등에 자유롭게 사용 가능하다
- 크레딧 표시 필요 (선택 사항)
- 여러 장르와 분위기의 음악이 있다

**Bensound**: https://www.bensound.com/

고품질의 로열티 프리 배경음악을 제공한다.

- 분위기별로 분류되어 있어 찾기 쉽다
- 크레딧 표시 권장 (필수 아님)
- 프리미엄 라이선스도 제공한다

### 학습 방향 제안

MonoGame을 배우고 나서 다음과 같은 방향으로 학습을 진행하길 권한다.

**초급 단계 완료 후**: 간단한 게임 여러 개를 만들어 보자. 예를 들어 Space Shooter, 2048, Snake 같은 클래식 게임들이 좋다.

**중급 단계**: 게임 엔진의 내부 동작을 이해해 보자. 카메라 시스템, 파티클 시스템, 음향 시스템 등을 깊이 있게 학습한다.

**고급 단계**: 자신만의 게임 프레임워크를 만들어 보자. 재사용 가능한 컴포넌트와 시스템을 설계하는 경험이 도움이 된다.

**최종 목표**: 완성도 있는 게임을 만들어 배포하자. itch.io 같은 플랫폼에 올려서 피드백을 받으면 실력 향상에 큰 도움이 된다.

게임 개발은 즐거운 여정이다. 실패를 두려워하지 말고 계속 도전하기 바란다.
