# Chapter 3. 플레이어 세션과 인증
온라인 게임에서 플레이어 세션 관리는 가장 기본적이면서도 중요한 기능이다. 플레이어가 로그인하면 세션을 생성하고, 게임을 플레이하는 동안 세션을 유지하며, 로그아웃하거나 일정 시간 비활성 상태가 되면 세션을 종료한다. Redis는 빠른 속도와 TTL(Time To Live) 기능으로 세션 관리에 최적화되어 있다.

## 3.1 로그인 시스템 구현

### 세션 토큰 생성 및 저장
플레이어가 로그인에 성공하면 고유한 세션 토큰을 생성하여 Redis에 저장한다. 이 토큰은 이후 모든 API 요청에서 플레이어를 식별하는 데 사용된다.

**세션 토큰 생성 전략**

세션 토큰은 예측 불가능하고 충돌 가능성이 낮아야 한다. GUID나 암호학적으로 안전한 난수를 사용한다.

```csharp
public class SessionService
{
    private readonly RedisConnection _connection;
    
    public SessionService(RedisConnection connection)
    {
        _connection = connection;
    }
    
    // 세션 토큰 생성
    public string GenerateSessionToken()
    {
        // GUID 기반 토큰 생성
        return Guid.NewGuid().ToString("N"); // 32자 16진수 문자열
        
        // 또는 더 긴 보안 토큰
        // var bytes = new byte[32];
        // using var rng = RandomNumberGenerator.Create();
        // rng.GetBytes(bytes);
        // return Convert.ToBase64String(bytes);
    }
}
```

**세션 데이터 모델**

세션에는 플레이어 ID, 로그인 시간, 디바이스 정보 등을 저장한다.

```csharp
public class SessionData
{
    public string PlayerId { get; set; }
    public DateTime LoginTime { get; set; }
    public string DeviceId { get; set; }
    public string IpAddress { get; set; }
```

**로그인 프로세스**

```
┌─────────────┐
│   Client    │
└──────┬──────┘
       │ 1. Login Request
       │    (ID, Password)
       ↓
┌─────────────────────┐
│   Game Server       │

```

**로그인 구현**

```csharp
public async Task<string> LoginAsync(string playerId, string password, 
    string deviceId, string ipAddress, string platform)
{
    // 1. 패스워드 검증 (실제로는 DB에서 해시 비교)
    // var isValid = await VerifyPasswordAsync(playerId, password);
    // if (!isValid) throw new UnauthorizedException();
    
    // 2. 세션 토큰 생성
    var sessionToken = GenerateSessionToken();
    
    // 3. 세션 데이터 생성
    var sessionData = new SessionData
    {
        PlayerId = playerId,
        LoginTime = DateTime.UtcNow,
        DeviceId = deviceId,
        IpAddress = ipAddress,
        Platform = platform
    };
    
    // 4. Redis에 세션 저장 (1시간 TTL)
    var sessionKey = $"session:{sessionToken}";
    var redis = new RedisString<SessionData>(
        _connection, 
        sessionKey, 
        TimeSpan.FromHours(1)
    );
    await redis.SetAsync(sessionData);
    
    // 5. 플레이어 ID로 세션 토큰 역참조 저장
    var playerSessionKey = $"player:{playerId}:session";
    var playerSession = new RedisString<string>(
        _connection, 
        playerSessionKey, 
        TimeSpan.FromHours(1)
    );
    await playerSession.SetAsync(sessionToken);
    
    return sessionToken;
}
```

### String과 Hash를 활용한 세션 관리
세션 데이터를 저장하는 방법은 두 가지가 있다. String에 직렬화된 객체를 저장하거나, Hash로 필드별로 저장할 수 있다.

**String 방식의 장단점**

장점:
- 구현이 간단하다
- 전체 데이터를 한 번에 읽고 쓴다
- CloudStructures의 자동 직렬화를 활용한다

단점:
- 일부 필드만 수정하려면 전체 데이터를 읽어서 다시 써야 한다
- 네트워크 트래픽이 약간 더 크다

**Hash 방식의 장단점**

장점:
- 필드별로 독립적으로 읽고 쓸 수 있다
- 메모리 효율이 좋다 (작은 Hash는 ziplist로 압축)
- 특정 필드만 증감할 수 있다 (HINCRBY)

단점:
- 코드가 약간 복잡하다
- 전체 데이터를 조회할 때 String보다 느릴 수 있다

**Hash로 세션 관리하기**

```csharp
public async Task<string> LoginWithHashAsync(string playerId, string password, 
    string deviceId, string ipAddress, string platform)
{
    var sessionToken = GenerateSessionToken();
    var sessionKey = $"session:{sessionToken}";
    
    // Hash에 필드별로 저장
    var db = _connection.GetConnection().GetDatabase();
    
    var sessionFields = new HashEntry[]
    {
        new HashEntry("player_id", playerId),
        new HashEntry("login_time", DateTime.UtcNow.ToString("o")),
        new HashEntry("device_id", deviceId),
        new HashEntry("ip_address", ipAddress),
        new HashEntry("platform", platform),
        new HashEntry("last_activity", DateTime.UtcNow.ToString("o"))
    };
    
    await db.HashSetAsync(sessionKey, sessionFields);
    await db.KeyExpireAsync(sessionKey, TimeSpan.FromHours(1));
    
    return sessionToken;
}
```

**Hash 세션 조회**

```csharp
public async Task<SessionData> GetSessionFromHashAsync(string sessionToken)
{
    var sessionKey = $"session:{sessionToken}";
    var db = _connection.GetConnection().GetDatabase();
    
    var sessionHash = await db.HashGetAllAsync(sessionKey);
    
    if (sessionHash.Length == 0)
    {
        throw new UnauthorizedException("Invalid or expired session");
    }
    
    // Hash를 객체로 변환
    var sessionData = new SessionData
    {
        PlayerId = sessionHash.First(x => x.Name == "player_id").Value,
        LoginTime = DateTime.Parse(sessionHash.First(x => x.Name == "login_time").Value),
        DeviceId = sessionHash.First(x => x.Name == "device_id").Value,
        IpAddress = sessionHash.First(x => x.Name == "ip_address").Value,
        Platform = sessionHash.First(x => x.Name == "platform").Value
    };
    
    // 마지막 활동 시간 업데이트
    await db.HashSetAsync(sessionKey, "last_activity", DateTime.UtcNow.ToString("o"));
    await db.KeyExpireAsync(sessionKey, TimeSpan.FromHours(1));
    
    return sessionData;
}
```

### 세션 만료 처리 (TTL)
Redis의 TTL 기능을 활용하면 일정 시간 후 자동으로 세션을 삭제할 수 있다. 수동으로 정리 작업을 할 필요가 없어 메모리 관리가 쉽다.

**TTL 전략**

게임 특성에 따라 TTL을 다르게 설정한다:

- **모바일 게임**: 1~4시간 (배터리 소모 고려)
- **PC MMORPG**: 8~24시간 (장시간 플레이)
- **실시간 대전 게임**: 30분~1시간 (빠른 세션 회전)

**활동 기반 TTL 연장**

플레이어가 활동할 때마다 TTL을 연장하여 세션을 유지한다.

```csharp
public async Task<SessionData> ValidateAndRefreshSessionAsync(string sessionToken)
{
    var sessionKey = $"session:{sessionToken}";
    var redis = new RedisString<SessionData>(_connection, sessionKey, null);
    
    var session = await redis.GetAsync();
    
    if (!session.HasValue)
    {
        throw new UnauthorizedException("Session expired");
    }
    
    // 마지막 활동으로부터 1시간 연장
    await redis.ExpireAsync(TimeSpan.FromHours(1));
    
    return session.Value;
}
```

**슬라이딩 만료 vs 절대 만료**

슬라이딩 만료(Sliding Expiration)는 활동할 때마다 연장되지만, 절대 만료(Absolute Expiration)는 최초 로그인부터 정해진 시간 후 무조건 만료된다.

```csharp
public class SessionWithAbsoluteExpiration
{
    public string PlayerId { get; set; }
    public DateTime LoginTime { get; set; }
    public DateTime AbsoluteExpireTime { get; set; }
}

public async Task ValidateWithAbsoluteExpirationAsync(string sessionToken)
{
    var session = await GetSessionAsync(sessionToken);
    
    // 절대 만료 시간 확인
    if (DateTime.UtcNow > session.AbsoluteExpireTime)
    {
        await LogoutAsync(sessionToken);
        throw new UnauthorizedException("Session absolutely expired");
    }
    
    // 슬라이딩 TTL은 연장하지만 절대 시간은 변경하지 않음
    await RefreshSessionAsync(sessionToken);
}
```

**TTL 모니터링**

남은 TTL을 확인하여 클라이언트에 알림을 줄 수 있다.

```csharp
public async Task<TimeSpan?> GetRemainingSessionTimeAsync(string sessionToken)
```

**로그아웃 처리**

명시적 로그아웃 시 세션을 즉시 삭제한다.

## 3.2 중복 로그인 방지
온라인 게임에서는 보안과 공정성을 위해 중복 로그인을 제한해야 한다. 한 계정으로 여러 디바이스에서 동시 접속하는 것을 막거나, 제한된 수의 디바이스만 허용한다.

### Set을 활용한 동시 접속 관리

**중복 로그인 정책**

게임마다 정책이 다르다:

1. **완전 차단**: 새 로그인 시 기존 세션 강제 종료
2. **디바이스 제한**: 최대 N개 디바이스까지 허용
3. **플랫폼별 허용**: PC 1개 + 모바일 1개

**완전 차단 방식**

가장 간단한 방법은 플레이어당 하나의 세션만 허용하는 것이다.

```csharp
public async Task<string> LoginWithSingleSessionAsync(
    string playerId, 
    string deviceId, 
    string platform)
{
    var db = _connection.GetConnection().GetDatabase();
    
    // 1. 기존 세션 확인
    var playerSessionKey = $"player:{playerId}:session";
    var existingToken = await db.StringGetAsync(playerSessionKey);
    
    if (existingToken.HasValue)
    {
        // 2. 기존 세션 강제 종료
        var oldSessionKey = $"session:{existingToken}";
        await db.KeyDeleteAsync(oldSessionKey);
        
        Console.WriteLine($"Kicked existing session for player {playerId}");
    }
    
    // 3. 새 세션 생성
    var newToken = GenerateSessionToken();
    var sessionData = new SessionData
    {
        PlayerId = playerId,
        LoginTime = DateTime.UtcNow,
        DeviceId = deviceId,
        Platform = platform
    };
    
    var sessionKey = $"session:{newToken}";
    var redis = new RedisString<SessionData>(_connection, sessionKey, TimeSpan.FromHours(1));
    await redis.SetAsync(sessionData);
    
    // 4. 플레이어 역참조 업데이트
    await db.StringSetAsync(playerSessionKey, newToken, TimeSpan.FromHours(1));
    
    return newToken;
}
```

**중복 로그인 감지 흐름**

```
[플레이어 A가 PC에서 로그인]
  ↓
player:A:session → "token-pc-123"
session:token-pc-123 → {PlayerId: "A", Platform: "PC"}
  
[플레이어 A가 모바일에서 로그인 시도]
  ↓
player:A:session 조회 → "token-pc-123" 발견
  ↓
session:token-pc-123 삭제 (PC 세션 강제 종료)
  ↓
새 세션 생성
player:A:session → "token-mobile-456"
session:token-mobile-456 → {PlayerId: "A", Platform: "Mobile"}
```

### 디바이스별 로그인 제한
여러 디바이스를 허용하되 개수를 제한한다. Set을 사용하여 활성 세션 목록을 관리한다.

**다중 세션 데이터 구조**

```
player:1000:sessions (Set)
  ├─ session:token-1
  ├─ session:token-2
  └─ session:token-3

session:token-1 (Hash)
  ├─ player_id: "1000"
  ├─ device_id: "device-abc"
  └─ platform: "iOS"

session:token-2 (Hash)
  ├─ player_id: "1000"
  ├─ device_id: "device-xyz"
  └─ platform: "Android"
```

**최대 3개 디바이스 허용**

```csharp
public async Task<string> LoginWithMultipleDevicesAsync(
    string playerId, 
    string deviceId, 
    string platform,
    int maxDevices = 3)
{
    var db = _connection.GetConnection().GetDatabase();
    
    // 1. 플레이어의 활성 세션 목록
    var playerSessionsKey = $"player:{playerId}:sessions";
    var activeSessions = new RedisSet<string>(_connection, playerSessionsKey, null);
    
    // 2. 현재 세션 수 확인
    var sessionCount = await activeSessions.LengthAsync();
    
    if (sessionCount >= maxDevices)
    {
        // 3. 가장 오래된 세션 제거
        var oldestSession = await GetOldestSessionAsync(playerId);
        
        if (oldestSession != null)
        {
            await RemoveSessionAsync(oldestSession, playerId);
            Console.WriteLine($"Removed oldest session: {oldestSession}");
        }
    }
    
    // 4. 새 세션 생성
    var newToken = GenerateSessionToken();
    var sessionKey = $"session:{newToken}";
    
    var sessionData = new SessionData
    {
        PlayerId = playerId,
        LoginTime = DateTime.UtcNow,
        DeviceId = deviceId,
        Platform = platform
    };
    
    var redis = new RedisString<SessionData>(_connection, sessionKey, TimeSpan.FromHours(1));
    await redis.SetAsync(sessionData);
    
    // 5. 활성 세션 목록에 추가
    await activeSessions.AddAsync(newToken);
    
    // 6. Set에도 TTL 설정 (가장 긴 세션 시간으로)
    var setTtl = TimeSpan.FromHours(1);
    await db.KeyExpireAsync(playerSessionsKey, setTtl);
    
    return newToken;
}
```

**가장 오래된 세션 찾기**

Sorted Set으로 로그인 시간을 추적하면 가장 오래된 세션을 쉽게 찾을 수 있다.

```csharp
public async Task<string> GetOldestSessionAsync(string playerId)
{
    var playerSessionsKey = $"player:{playerId}:sessions";
    var sessions = new RedisSet<string>(_connection, playerSessionsKey, null);
    
    var allSessions = await sessions.MembersAsync();
    
    if (allSessions.Length == 0)
    {
        return null;
    }
    
    // 각 세션의 로그인 시간 조회
    string oldestToken = null;
    DateTime oldestTime = DateTime.MaxValue;
    
    foreach (var sessionToken in allSessions)
    {
        var sessionKey = $"session:{sessionToken}";
        var redis = new RedisString<SessionData>(_connection, sessionKey, null);
        var session = await redis.GetAsync();
        
        if (session.HasValue && session.Value.LoginTime < oldestTime)
        {
            oldestTime = session.Value.LoginTime;
            oldestToken = sessionToken;
        }
    }
    
    return oldestToken;
}
```

**더 효율적인 방법: Sorted Set 활용**

로그인 시간을 점수로 하는 Sorted Set을 사용하면 가장 오래된 세션을 O(1)로 찾을 수 있다.

```csharp
public async Task<string> LoginWithSortedSessionsAsync(
    string playerId, 
    string deviceId, 
    string platform,
    int maxDevices = 3)
{
    var db = _connection.GetConnection().GetDatabase();
    
    // Sorted Set으로 세션 관리 (점수 = 로그인 시간)
    var playerSessionsKey = $"player:{playerId}:sessions_sorted";
    var sessionsSorted = new RedisSortedSet<string>(_connection, playerSessionsKey, null);
    
    // 현재 세션 수
    var sessionCount = await sessionsSorted.LengthAsync();
    
    if (sessionCount >= maxDevices)
    {
        // 가장 오래된 세션 (점수가 가장 낮은)
        var oldestSessions = await sessionsSorted.RangeByRankAsync(0, 0);
        
        if (oldestSessions.Length > 0)
        {
            var oldestToken = oldestSessions[0];
            await RemoveSessionAsync(oldestToken, playerId);
            await sessionsSorted.RemoveAsync(oldestToken);
        }
    }
    
    // 새 세션 생성
    var newToken = GenerateSessionToken();
    var sessionKey = $"session:{newToken}";
    
    var sessionData = new SessionData
    {
        PlayerId = playerId,
        LoginTime = DateTime.UtcNow,
        DeviceId = deviceId,
        Platform = platform
    };
    
    var redis = new RedisString<SessionData>(_connection, sessionKey, TimeSpan.FromHours(1));
    await redis.SetAsync(sessionData);
    
    // Sorted Set에 추가 (점수 = Unix timestamp)
    var timestamp = DateTimeOffset.UtcNow.ToUnixTimeSeconds();
    await sessionsSorted.AddAsync(newToken, timestamp);
    
    // TTL 설정
    await db.KeyExpireAsync(playerSessionsKey, TimeSpan.FromHours(1));
    
    return newToken;
}
```

**플랫폼별 제한**

PC는 1개, 모바일은 2개까지 허용하는 정책을 구현할 수 있다.

```csharp
public async Task<string> LoginWithPlatformLimitAsync(
    string playerId, 
    string deviceId, 
    string platform)
{
    var db = _connection.GetConnection().GetDatabase();
    
    // 플랫폼별 세션 목록
    var platformSessionKey = $"player:{playerId}:sessions:{platform.ToLower()}";
    var platformSessions = new RedisSet<string>(_connection, platformSessionKey, null);
    
    // 플랫폼별 제한
    int maxSessions = platform.ToLower() == "pc" ? 1 : 2;
    
    var sessionCount = await platformSessions.LengthAsync();
    
    if (sessionCount >= maxSessions)
    {
        // 가장 오래된 세션 제거
        var sessions = await platformSessions.MembersAsync();
        
        if (sessions.Length > 0)
        {
            var oldestToken = sessions[0]; // 실제로는 Sorted Set 사용 권장
            await RemoveSessionAsync(oldestToken, playerId);
            await platformSessions.RemoveAsync(oldestToken);
        }
    }
    
    // 새 세션 생성
    var newToken = GenerateSessionToken();
    // ... (세션 생성 로직)
    
    await platformSessions.AddAsync(newToken);
    
    return newToken;
}
```

## 3.3 온라인 유저 목록 관리
실시간으로 누가 접속해 있는지 추적하는 것은 게임 운영과 소셜 기능에 중요하다. 친구가 온라인인지, 현재 접속자가 몇 명인지 빠르게 알아야 한다.

### Sorted Set으로 최근 접속 시간 추적
Sorted Set을 사용하면 플레이어의 마지막 활동 시간을 점수로 저장하여 최근 접속자를 효율적으로 조회할 수 있다.

**온라인 사용자 추적 구조**

```
online_users (Sorted Set)
  ├─ player:1000 → 1703584800 (timestamp)
  ├─ player:1001 → 1703584850
  └─ player:1005 → 1703584900

점수 = Unix timestamp (마지막 활동 시간)
```

```csharp
public async Task<bool> IsPlayerOnlineAsync(string playerId, TimeSpan onlineThreshold)
{
    var onlineUsers = new RedisSortedSet<string>(
        _connection, 
        "online_users", 
        null
    );
    
    var lastActivity = await onlineUsers.ScoreAsync(playerId);
    
    if (!lastActivity.HasValue)
    {
        return false; // 목록에 없음
    }
    
    // 마지막 활동이 임계값 이내인지 확인
    var cutoffTime = DateTimeOffset.UtcNow.Subtract(onlineThreshold).ToUnixTimeSeconds();
    return lastActivity.Value >= cutoffTime;
}
```

## 정리
이번 장에서는 Redis를 활용한 플레이어 세션과 인증 시스템을 구현했다.

**핵심 내용**:

1. **세션 관리**
   - String 또는 Hash로 세션 데이터 저장
   - TTL을 활용한 자동 만료
   - 슬라이딩/절대 만료 전략

2. **중복 로그인 방지**
   - Set으로 활성 세션 목록 관리
   - Sorted Set으로 효율적인 세션 정리
   - 플랫폼별, 디바이스별 제한 정책

3. **온라인 유저 관리**
   - Sorted Set으로 최근 활동 시간 추적
   - 비활성 사용자 자동 정리
   - 실시간 접속자 수 집계
   - HyperLogLog로 대규모 집계

**실전 팁**:

- 세션 토큰은 예측 불가능하게 생성한다
- TTL을 적절히 설정하여 메모리를 효율적으로 관리한다
- 비활성 사용자를 정기적으로 정리한다
- 중요한 작업 전에는 항상 세션을 검증한다
- 플랫폼과 지역별로 세션을 분리하여 관리한다

다음 장에서는 플레이어 데이터와 인벤토리 시스템을 다룬다. Cache-Aside 패턴으로 DB와 Redis를 연동하고, Hash를 활용하여 효율적인 인벤토리 관리를 구현한다.
