# 10장. 모니터링과 로깅

게임 서버를 운영할 때 가장 중요한 것 중 하나가 바로 서버의 상태를 실시간으로 파악하는 것이다. 플레이어가 갑자기 접속이 안 된다거나, 서버가 느려진다는 신고가 들어왔을 때 빠르게 원인을 찾아 대응해야 한다. 이 장에서는 Docker 컨테이너로 실행 중인 게임 서버를 모니터링하고 로그를 효과적으로 관리하는 방법을 실습한다.

## 10.1 컨테이너 리소스 모니터링

### 컨테이너 상태 확인하기

먼저 실행 중인 컨테이너의 리소스 사용량을 확인하는 기본 명령어를 알아본다.

```bash
# 실행 중인 컨테이너의 리소스 사용량 실시간 확인
docker stats

# 특정 컨테이너만 확인
docker stats game-server-1

# 한 번만 출력 (실시간 모드 종료)
docker stats --no-stream
```

`docker stats` 명령을 실행하면 다음과 같은 정보를 볼 수 있다.

```
CONTAINER ID   NAME            CPU %   MEM USAGE / LIMIT   MEM %   NET I/O     BLOCK I/O
a1b2c3d4e5f6   game-server-1   25.5%   512MiB / 2GiB      25.6%   1.2MB/800KB 0B/0B
```

여기서 확인할 수 있는 주요 지표는 다음과 같다.

- **CPU %**: CPU 사용률
- **MEM USAGE / LIMIT**: 현재 메모리 사용량과 제한값
- **MEM %**: 메모리 사용 비율
- **NET I/O**: 네트워크 입출력량
- **BLOCK I/O**: 디스크 입출력량

### 리소스 제한 설정하기

게임 서버가 무제한으로 리소스를 사용하면 다른 서비스에 영향을 줄 수 있다. 컨테이너 실행 시 리소스 제한을 설정한다.

```bash
# 메모리와 CPU 제한을 두고 실행
docker run -d \
  --name game-server-limited \
  --memory="1g" \
  --cpus="1.5" \
  -p 5000:5000 \
  game-server:latest
```

Docker Compose에서는 다음과 같이 설정한다.

```yaml
services:
  game-server:
    image: game-server:latest
    ports:
      - "5000:5000"
    deploy:
      resources:
        limits:
          cpus: '1.5'
          memory: 1G
        reservations:
          cpus: '0.5'
          memory: 512M
```

### 리소스 사용량 로그 저장하기

모니터링 데이터를 파일로 저장하여 나중에 분석할 수도 있다.

```bash

# 또는 실시간으로 계속 저장
docker stats --format "table {{.Container}}\t{{.CPUPerc}}\t{{.MemUsage}}\t{{.NetIO}}" >> monitor.log
```

## 10.2 로그 수집 전략

### 기본 로그 확인

Docker는 컨테이너의 표준 출력(stdout)과 표준 에러(stderr)를 자동으로 수집한다.

```bash
# 전체 로그 확인
docker logs game-server-1

# 실시간 로그 확인 (tail -f와 유사)
docker logs -f game-server-1

# 최근 100줄만 보기
docker logs --tail 100 game-server-1

# 타임스탬프 포함
docker logs -t game-server-1

# 특정 시간 이후 로그
docker logs --since 2024-11-22T10:00:00 game-server-1
```

### ASP.NET 게임 서버의 로깅 설정

게임 서버 코드에서 표준 출력으로 로그를 출력하도록 구성한다.

```csharp
// Program.cs
var builder = WebApplication.CreateBuilder(args);

// 콘솔 로깅 설정
builder.Logging.ClearProviders();
builder.Logging.AddConsole();
builder.Logging.SetMinimumLevel(LogLevel.Information);

var app = builder.Build();

app.MapGet("/api/player/login", (ILogger<Program> logger) =>
{
    var playerId = Guid.NewGuid();
    logger.LogInformation("Player login: {PlayerId} at {Time}", 
        playerId, DateTime.UtcNow);
    
    return new { PlayerId = playerId, Success = true };
});

app.Run();
```

Dockerfile에서도 로깅이 잘 되도록 설정한다.

```dockerfile
FROM mcr.microsoft.com/dotnet/aspnet:8.0 AS base
WORKDIR /app
EXPOSE 5000

# 환경 변수로 로그 레벨 설정 가능하게

FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src
COPY ["GameServer.csproj", "./"]

COPY . .

FROM build AS publish

FROM base AS final
WORKDIR /app

```

### 소켓 게임 서버 로깅

소켓 서버도 표준 출력으로 로그를 출력한다.

```csharp
// SocketGameServer.cs
public class SocketGameServer
{
    public async Task StartAsync(int port)
    {
        var listener = new TcpListener(IPAddress.Any, port);
        listener.Start();
        
        Console.WriteLine($"[{DateTime.UtcNow:yyyy-MM-dd HH:mm:ss}] Socket server started on port {port}");
        
        while (true)
        {
            var client = await listener.AcceptTcpClientAsync();
            var endpoint = client.Client.RemoteEndPoint;
            
            Console.WriteLine($"[{DateTime.UtcNow:yyyy-MM-dd HH:mm:ss}] Client connected: {endpoint}");
            
            _ = HandleClientAsync(client);
        }
    }
    
    private async Task HandleClientAsync(TcpClient client)
    {
        try
        {
            // 클라이언트 처리 로직
            Console.WriteLine($"[{DateTime.UtcNow:yyyy-MM-dd HH:mm:ss}] Handling client...");
        }
        catch (Exception ex)
        {
            Console.Error.WriteLine($"[{DateTime.UtcNow:yyyy-MM-dd HH:mm:ss}] Error: {ex.Message}");
        }
        finally
        {
            client.Close();
            Console.WriteLine($"[{DateTime.UtcNow:yyyy-MM-dd HH:mm:ss}] Client disconnected");
        }
    }
}
```

### 로그 드라이버 설정

Docker는 다양한 로그 드라이버를 지원한다. 기본은 `json-file`이지만, 필요에 따라 변경할 수 있다.

```bash
# 로컬 파일로 저장 (최대 크기와 파일 수 제한)
docker run -d \
  --name game-server-log \
  --log-driver json-file \
  --log-opt max-size=10m \
  --log-opt max-file=3 \
  -p 5000:5000 \
  game-server:latest
```

Docker Compose에서는 다음과 같이 설정한다.

```yaml
services:
  game-server:
    image: game-server:latest
    ports:
      - "5000:5000"
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"
        labels: "game-server"
        
  socket-server:
    image: socket-server:latest
    ports:
      - "9000:9000"
    logging:
      driver: "json-file"
      options:
        max-size: "5m"
        max-file: "5"
```

### 소켓 서버 헬스체크

소켓 서버는 HTTP 엔드포인트가 없으므로, 간단한 TCP 연결 테스트를 한다.

```dockerfile
FROM mcr.microsoft.com/dotnet/runtime:8.0
WORKDIR /app
COPY --from=build /app/publish .

# nc(netcat) 설치
RUN apt-get update && apt-get install -y netcat-openbsd && rm -rf /var/lib/apt/lists/*

# TCP 포트로 헬스체크
HEALTHCHECK --interval=30s --timeout=3s --start-period=10s --retries=3 \
  CMD nc -z localhost 9000 || exit 1

ENTRYPOINT ["dotnet", "SocketServer.dll"]
```

또는 간단한 HTTP 상태 엔드포인트를 별도로 만들 수도 있다.

```csharp
// SocketGameServer.cs
public class SocketGameServer
{
    private bool _isHealthy = true;
    
    public async Task StartAsync(int gamePort, int healthPort)
    {
        // 게임 소켓 서버
        _ = Task.Run(() => StartGameServerAsync(gamePort));
        
        // 헬스체크용 HTTP 서버
        _ = Task.Run(() => StartHealthServerAsync(healthPort));
        
        await Task.Delay(-1);
    }
    
    private async Task StartGameServerAsync(int port)
    {
        var listener = new TcpListener(IPAddress.Any, port);
        listener.Start();
        _isHealthy = true;
        
        Console.WriteLine($"Game server started on port {port}");
        
        while (true)
        {
            var client = await listener.AcceptTcpClientAsync();
            _ = HandleClientAsync(client);
        }
    }
    
    private async Task StartHealthServerAsync(int port)
    {
        var listener = new HttpListener();
        listener.Prefixes.Add($"http://+:{port}/health/");
        listener.Start();
        
        Console.WriteLine($"Health server started on port {port}");
        
        while (true)
        {
            var context = await listener.GetContextAsync();
            var response = context.Response;
            
            var status = _isHealthy ? "Healthy" : "Unhealthy";
            var json = $"{{\"status\":\"{status}\",\"timestamp\":\"{DateTime.UtcNow:O}\"}}";
            
            var buffer = System.Text.Encoding.UTF8.GetBytes(json);
            response.ContentLength64 = buffer.Length;
            response.StatusCode = _isHealthy ? 200 : 503;
            await response.OutputStream.WriteAsync(buffer, 0, buffer.Length);
            response.Close();
        }
    }
}
```

## 10.4 간단한 모니터링 스택 구성

### Prometheus와 Grafana 구성

게임 서버 모니터링을 위한 기본적인 스택을 구성해본다. Prometheus로 메트릭을 수집하고 Grafana로 시각화한다.

프로젝트 구조:

```
monitoring-stack/
├── docker-compose.yml
├── prometheus/
│   └── prometheus.yml
├── grafana/
│   └── dashboards/
└── game-server/
    └── Dockerfile
```

먼저 게임 서버에 메트릭 엔드포인트를 추가한다.

```bash
# ASP.NET 프로젝트에 Prometheus 패키지 추가
dotnet add package prometheus-net.AspNetCore
```

```csharp
// Program.cs
using Prometheus;

var builder = WebApplication.CreateBuilder(args);
builder.Logging.AddConsole();

var app = builder.Build();

// Prometheus 메트릭 엔드포인트
app.UseMetricServer();
app.UseHttpMetrics();

// 커스텀 메트릭
var loginCounter = Metrics.CreateCounter("game_player_logins_total", 
    "Total number of player logins");
var activePlayersGauge = Metrics.CreateGauge("game_active_players", 
    "Number of active players");

app.MapGet("/health", () => Results.Ok(new 
{ 
    Status = "Healthy", 
    Timestamp = DateTime.UtcNow 
}));

app.MapGet("/api/player/login", (ILogger<Program> logger) =>
{
    var playerId = Guid.NewGuid();
    
    loginCounter.Inc();
    activePlayersGauge.Inc();
    
    logger.LogInformation("Player login: {PlayerId}", playerId);
    
    return new { PlayerId = playerId, Success = true };
});

app.MapGet("/api/player/logout", (ILogger<Program> logger) =>
{
    activePlayersGauge.Dec();
    logger.LogInformation("Player logout");
    
    return new { Success = true };
});

app.Run();
```

Prometheus 설정 파일을 작성한다.

```yaml

      - monitoring

  monitoring:
    driver: bridge

  prometheus-data:
  grafana-data:
```


실행 후 접속할 수 있는 주소는 다음과 같다.

- 게임 API: http://localhost:5000
- Prometheus: http://localhost:9090
- Grafana: http://localhost:3000 (admin/admin)

### Grafana 대시보드 설정

Grafana에 접속하여 데이터 소스를 추가한다.

1. Grafana (http://localhost:3000) 접속
2. Configuration → Data Sources → Add data source
3. Prometheus 선택
4. URL에 `http://prometheus:9090` 입력
5. Save & Test

간단한 대시보드를 만든다.

- 새 대시보드 생성
- Add Panel 클릭
- 메트릭 쿼리 입력:
  - `game_player_logins_total` (누적 로그인 수)
  - `game_active_players` (현재 접속자 수)
  - `rate(game_player_logins_total[5m])` (5분간 로그인 비율)

### 로그 수집 스택 (선택사항)

### 간단한 알림 설정

Prometheus에서 간단한 알림 규칙을 만들 수 있다.

```yaml
# prometheus/alert-rules.yml
groups:
  - name: game_server_alerts
    interval: 30s
    rules:
      - alert: HighMemoryUsage
        expr: container_memory_usage_bytes{name="game-api"} > 500000000
        for: 2m
        labels:
          severity: warning
        annotations:
          summary: "High memory usage on game server"
          description: "Memory usage is above 500MB for 2 minutes"
          
      - alert: NoPlayerLogins
        expr: rate(game_player_logins_total[5m]) == 0
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "No player logins detected"
          description: "No players have logged in for 5 minutes"
```

Prometheus 설정에 추가한다.

```yaml
# prometheus/prometheus.yml
global:
  scrape_interval: 15s

rule_files:
  - /etc/prometheus/alert-rules.yml

scrape_configs:
  - job_name: 'game-api'
    static_configs:
      - targets: ['game-api:5000']
```

### 정리

이번 장에서는 Docker 컨테이너로 실행되는 게임 서버를 모니터링하고 로그를 관리하는 방법을 배웠다.

주요 내용을 정리하면 다음과 같다.

**리소스 모니터링**
- `docker stats`로 CPU, 메모리, 네트워크 사용량을 실시간으로 확인한다
- `--memory`, `--cpus` 옵션으로 리소스 제한을 설정한다
- Docker Compose의 `deploy.resources`로 리소스를 관리한다

**로그 관리**
- `docker logs`로 컨테이너의 표준 출력/에러를 확인한다
- 로그 드라이버와 옵션으로 로그 크기와 보관 정책을 설정한다
- 게임 서버는 표준 출력으로 로그를 작성하도록 구현한다

**헬스체크**
- Dockerfile의 `HEALTHCHECK` 지시어로 컨테이너 상태를 자동 확인한다
- 게임 서버에 `/health` 엔드포인트를 구현한다
- Docker Compose에서 `depends_on`과 함께 사용하여 의존성을 관리한다

**모니터링 스택**
- Prometheus로 메트릭을 수집하고 Grafana로 시각화한다
- 게임 서버에 커스텀 메트릭(로그인 수, 동접자 등)을 추가한다
- 알림 규칙으로 문제 상황을 미리 감지한다
