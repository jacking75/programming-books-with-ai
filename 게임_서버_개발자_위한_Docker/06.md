# 6장. 데이터 영속성과 볼륨

게임 서버에서 플레이어 데이터, 게임 로그, 설정 파일 등은 컨테이너가 재시작되어도 유지되어야 한다. 이 장에서는 Docker 볼륨을 사용하여 데이터를 영구적으로 저장하는 방법을 실습한다.

## 6.1 볼륨의 필요성

Docker 컨테이너는 기본적으로 상태를 저장하지 않는 ephemeral(일시적) 특성을 가진다. 컨테이너가 삭제되면 내부 데이터도 함께 사라진다.

### 컨테이너 데이터의 문제점

컨테이너 파일시스템의 동작 방식을 이해해본다.

```
┌─────────────────────────────────────────────┐
│         Container Lifecycle                 │
├─────────────────────────────────────────────┤
│                                             │
│  ┌────────────┐      ┌────────────┐         │
│  │ Container  │      │ Container  │         │
│  │  Running   │ ───> │  Stopped   │         │
│  │            │      │            │         │
│  │ Data: ✓    │      │ Data: ✓    │         │
│  └────────────┘      └────────────┘         │
│                             │               │
│                             │ docker rm     │
│                             ▼               │
│                      ┌────────────┐         │
│                      │  Deleted   │         │
│                      │            │         │
│                      │ Data: ✗    │         │
│                      └────────────┘         │
│                                             │
└─────────────────────────────────────────────┘
```

### 데이터 손실 시연

실제로 데이터가 손실되는 상황을 확인한다.

```bash
# 컨테이너 실행 및 파일 생성
docker run -it --name test-container ubuntu bash
```

컨테이너 내부에서 파일을 생성한다.

```bash
echo "Important game data" > /data.txt
cat /data.txt
exit
```

컨테이너를 삭제하고 다시 생성한다.

```bash
# 컨테이너 삭제
docker rm test-container

# 동일한 이미지로 새 컨테이너 생성
docker run -it --name test-container ubuntu bash

# 이전 데이터 확인 시도
cat /data.txt
# cat: /data.txt: No such file or directory
```

파일이 사라진 것을 확인할 수 있다.

### 게임 서버에서 보존해야 할 데이터

게임 서버에서 영구적으로 저장해야 하는 데이터 유형은 다음과 같다.

**플레이어 데이터**: 프로필, 인벤토리, 진행 상황, 업적 등의 정보다.

**게임 로그**: 플레이어 행동, 트랜잭션, 오류 로그 등 분석과 디버깅에 필요하다.

**설정 파일**: 서버 설정, 게임 밸런스 데이터, 이벤트 정보 등이다.

**세션 정보**: 활성 세션, 임시 상태 등을 저장한다.

**백업 데이터**: 정기적인 백업과 복구를 위한 스냅샷이다.

### Docker 볼륨 vs 바인드 마운트

데이터를 보존하는 두 가지 주요 방법을 비교한다.

```
┌──────────────────────────────────────────────────┐
│                                                  │
│  Docker Volume              Bind Mount           │
│  ┌─────────────┐           ┌──────────────┐     │
│  │             │           │              │     │
│  │  Docker가   │           │  호스트 경로 │     │
│  │  관리하는   │           │  직접 지정   │     │
│  │  저장소     │           │              │     │
│  │             │           │              │     │
│  └─────────────┘           └──────────────┘     │
│       │                          │              │
│       │                          │              │
│       ▼                          ▼              │
│  /var/lib/docker/            /home/user/        │
│  volumes/                    project/           │
│                                                  │
│  장점:                      장점:               │
│  - Docker가 관리           - 개발 중 편리       │
│  - 백업/복원 쉬움          - 파일 직접 수정     │
│  - 플랫폼 독립적           - 실시간 반영        │
│                                                  │
│  단점:                      단점:               │
│  - 직접 접근 어려움        - 경로 의존성        │
│  - 수동 수정 불편          - 권한 문제          │
│                                                  │
└──────────────────────────────────────────────────┘
```

## 6.2 볼륨 생성과 마운트 실습

Docker 볼륨을 생성하고 컨테이너에 마운트하는 방법을 실습한다.

### 볼륨 생성하기

명령어로 볼륨을 생성한다.

```bash
# 게임 데이터용 볼륨 생성
docker volume create game-data

# 로그용 볼륨 생성
docker volume create game-logs

# 설정 파일용 볼륨 생성
docker volume create game-config
```

### 볼륨 목록 확인

생성된 볼륨을 확인한다.

```bash
docker volume ls
```

출력 예시는 다음과 같다.

```
DRIVER    VOLUME NAME
local     game-config
local     game-data
local     game-logs
```

### 볼륨 상세 정보 확인

볼륨의 세부 정보를 조회한다.

```bash
docker volume inspect game-data
```

출력 결과는 다음과 유사하다.

```json
[
    {
        "CreatedAt": "2025-11-22T10:00:00Z",
        "Driver": "local",
        "Labels": {},
        "Mountpoint": "/var/lib/docker/volumes/game-data/_data",
        "Name": "game-data",
        "Options": {},
        "Scope": "local"
    }
]
```

### 볼륨을 컨테이너에 마운트

볼륨을 사용하여 컨테이너를 실행한다.

```bash
docker run -d \
  -v game-data:/app/data \
  -v game-logs:/app/logs \
  --name test-volume-container \
  ubuntu sleep 3600
```

옵션 설명은 다음과 같다.

- `-v game-data:/app/data`: game-data 볼륨을 컨테이너의 /app/data 경로에 마운트한다.
- `-v game-logs:/app/logs`: game-logs 볼륨을 컨테이너의 /app/logs 경로에 마운트한다.

### 볼륨 마운트 구조

볼륨 마운트의 동작 방식을 다이어그램으로 표현하면 다음과 같다.

```
┌────────────────────────────────────────────┐
│           Docker Host                      │
│                                            │
│  ┌──────────────────────────────────┐      │
│  │  Docker Volume Storage           │      │
│  │  /var/lib/docker/volumes/        │      │
│  │                                  │      │
│  │  ┌─────────────────────────┐     │      │
│  │  │  game-data/_data        │     │      │
│  │  │  - player1.json         │◄────┼──┐   │
│  │  │  - player2.json         │     │  │   │
│  │  └─────────────────────────┘     │  │   │
│  │                                  │  │   │
│  │  ┌─────────────────────────┐     │  │   │
│  │  │  game-logs/_data        │     │  │   │
│  │  │  - server.log           │◄────┼──┼─┐ │
│  │  │  - error.log            │     │  │ │ │
│  │  └─────────────────────────┘     │  │ │ │
│  └──────────────────────────────────┘  │ │ │
│                                        │ │ │
│  ┌──────────────────────────────────┐  │ │ │
│  │  Container                       │  │ │ │
│  │  test-volume-container           │  │ │ │
│  │                                  │  │ │ │
│  │  /app/data ─────────────────────────┘ │ │
│  │  /app/logs ───────────────────────────┘ │
│  └──────────────────────────────────┘      │
└────────────────────────────────────────────┘
```

### 볼륨에 데이터 저장 테스트

컨테이너 내부에서 데이터를 생성한다.

```bash
# 컨테이너 접속
docker exec -it test-volume-container bash

# 데이터 파일 생성
echo "Player 1 Score: 1000" > /app/data/player1.txt
echo "Player 2 Score: 2000" > /app/data/player2.txt

# 로그 파일 생성
echo "Server started at $(date)" > /app/logs/server.log

# 파일 확인
ls -la /app/data/
ls -la /app/logs/

exit
```

### 볼륨 데이터 영속성 확인

컨테이너를 삭제하고 새로 생성하여 데이터가 유지되는지 확인한다.

```bash
# 컨테이너 삭제
docker rm -f test-volume-container

# 동일한 볼륨으로 새 컨테이너 생성
docker run -d \
  -v game-data:/app/data \
  -v game-logs:/app/logs \
  --name new-volume-container \
  ubuntu sleep 3600

# 데이터 확인
docker exec new-volume-container cat /app/data/player1.txt
docker exec new-volume-container cat /app/logs/server.log
```

이전에 저장한 데이터가 그대로 유지된 것을 확인할 수 있다.

### 읽기 전용 볼륨 마운트

설정 파일처럼 수정하지 않아야 할 데이터는 읽기 전용으로 마운트한다.

```bash
docker run -d \
  -v game-config:/app/config:ro \
  --name readonly-test \
  ubuntu sleep 3600
```

`:ro` 옵션은 read-only를 의미한다.

읽기 전용 확인을 테스트한다.

```bash
docker exec readonly-test sh -c "echo 'test' > /app/config/test.txt"
# 오류 발생: Read-only file system
```

## 6.3 게임 데이터 저장하기

실제 게임 서버에서 플레이어 데이터를 볼륨에 저장하는 예제를 만든다.

### 데이터 저장 기능이 있는 게임 서버 만들기

4장에서 만든 REST API 게임 서버를 수정하여 데이터를 파일로 저장하는 기능을 추가한다.

**Services/PlayerDataService.cs** 파일을 생성한다.

```csharp
using System.Text.Json;
using GameServer.Models;

namespace GameServer.Services
{
    public class PlayerDataService
    {
        private readonly string _dataPath;
        private readonly Dictionary<int, Player> _players = new();
        private int _nextId = 1;

        public PlayerDataService(IConfiguration configuration)
        {
            _dataPath = configuration["DataPath"] ?? "/app/data";
            Directory.CreateDirectory(_dataPath);
            LoadPlayers();
        }

        public Player CreatePlayer(string name)
        {
            var player = new Player
            {
                Id = _nextId++,
                Name = name,
                Score = 0,
                CreatedAt = DateTime.UtcNow
            };
            
            _players[player.Id] = player;
            SavePlayer(player);
            
            return player;
        }

        public Player? GetPlayer(int id)
        {
            _players.TryGetValue(id, out var player);
            return player;
        }

        public List<Player> GetAllPlayers()
        {
            return _players.Values.ToList();
        }

        public bool UpdateScore(int id, int score)
        {
            if (!_players.TryGetValue(id, out var player))
                return false;
            
            player.Score = score;
            SavePlayer(player);
            
            return true;
        }

        private void SavePlayer(Player player)
        {
            var filePath = Path.Combine(_dataPath, $"player_{player.Id}.json");
            var json = JsonSerializer.Serialize(player, new JsonSerializerOptions 
            { 
                WriteIndented = true 
            });
            
            File.WriteAllText(filePath, json);
            Console.WriteLine($"[DataService] Saved player {player.Id} to {filePath}");
        }

        private void LoadPlayers()
        {
            var files = Directory.GetFiles(_dataPath, "player_*.json");
            
            foreach (var file in files)
            {
                try
                {
                    var json = File.ReadAllText(file);
                    var player = JsonSerializer.Deserialize<Player>(json);
                    
                    if (player != null)
                    {
                        _players[player.Id] = player;
                        if (player.Id >= _nextId)
                            _nextId = player.Id + 1;
                    }
                }
                catch (Exception ex)
                {
                    Console.WriteLine($"[DataService] Error loading {file}: {ex.Message}");
                }
            }
            
            Console.WriteLine($"[DataService] Loaded {_players.Count} players from {_dataPath}");
        }

        public void SaveAllPlayers()
        {
            foreach (var player in _players.Values)
            {
                SavePlayer(player);
            }
        }
    }
}
```

### Program.cs 수정

PlayerDataService를 사용하도록 Program.cs를 수정한다.

```csharp
using GameServer.Services;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddControllers();
builder.Services.AddSingleton<PlayerDataService>();
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

var app = builder.Build();

// 종료 시 모든 데이터 저장
var lifetime = app.Services.GetRequiredService<IHostApplicationLifetime>();
lifetime.ApplicationStopping.Register(() =>
{
    var dataService = app.Services.GetRequiredService<PlayerDataService>();
    dataService.SaveAllPlayers();
    Console.WriteLine("[Server] All player data saved");
});

if (app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}

app.MapControllers();

app.Run();
```

### Dockerfile에 볼륨 지정 추가

Dockerfile을 수정하여 데이터 디렉토리를 볼륨으로 지정한다.

```dockerfile
FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build

WORKDIR /app

COPY *.csproj ./
RUN dotnet restore

COPY . ./
RUN dotnet publish -c Release -o out

FROM mcr.microsoft.com/dotnet/aspnet:8.0

WORKDIR /app

COPY --from=build /app/out .

# 데이터 디렉토리 생성
RUN mkdir -p /app/data /app/logs

# 볼륨 포인트 선언
VOLUME ["/app/data", "/app/logs"]

EXPOSE 8080

ENV ASPNETCORE_URLS=http://+:8080

ENTRYPOINT ["dotnet", "GameServer.dll"]
```

## 6.4 바인드 마운트 활용

개발 환경에서는 호스트의 디렉토리를 직접 마운트하여 실시간으로 파일을 수정할 수 있다.

### 바인드 마운트 기본 사용법

호스트의 특정 경로를 컨테이너에 마운트한다.

```bash
# 호스트에 데이터 디렉토리 생성
mkdir -p ~/game-server-data
mkdir -p ~/game-server-config

# 바인드 마운트로 컨테이너 실행
docker run -d \
  -p 8081:8080 \
  -v ~/game-server-data:/app/data \
  -v ~/game-server-config:/app/config \
  --name game-server-bind \
  game-server:v2-persistent
```

### 바인드 마운트 vs 볼륨 비교

두 방식의 차이를 정리하면 다음과 같다.

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  Named Volume                                       │
│  ┌───────────────────────────────────────────────┐  │
│  │  docker run -v game-data:/app/data ...        │  │
│  └───────────────────────────────────────────────┘  │
│                                                     │
│  - Docker가 저장 위치 관리                          │
│  - /var/lib/docker/volumes/game-data/_data         │
│  - 운영 환경에 적합                                 │
│  - 플랫폼 독립적                                    │
│                                                     │
├─────────────────────────────────────────────────────┤
│                                                     │
│  Bind Mount                                         │
│  ┌───────────────────────────────────────────────┐  │
│  │  docker run -v ~/data:/app/data ...           │  │
│  └───────────────────────────────────────────────┘  │
│                                                     │
│  - 호스트 경로 직접 지정                            │
│  - /home/user/data                                 │
│  - 개발 환경에 적합                                 │
│  - 파일 직접 수정 가능                              │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 설정 파일 바인드 마운트 실습

게임 서버 설정 파일을 바인드 마운트로 관리한다.

호스트에 설정 파일을 생성한다.

```bash
# 설정 디렉토리 생성
```

설정 파일을 읽는 코드를 추가한다.

**Services/GameConfigService.cs** 파일을 생성한다.

```csharp
using System.Text.Json;

namespace GameServer.Services
{
    public class GameConfig
    {
        public int MaxPlayers { get; set; }
        public string ServerName { get; set; } = string.Empty;
        public int TickRate { get; set; }
        public Dictionary<string, bool> Features { get; set; } = new();
    }

    public class GameConfigService
    {
        private readonly string _configPath;
        private GameConfig? _config;

        public GameConfigService(IConfiguration configuration)
        {
            _configPath = configuration["ConfigPath"] ?? "/app/config";
            LoadConfig();
        }

        public GameConfig GetConfig()
        {
            return _config ?? new GameConfig();
        }

        private void LoadConfig()
        {
            var filePath = Path.Combine(_configPath, "game-settings.json");
            
            if (File.Exists(filePath))
            {
                try
                {
                    var json = File.ReadAllText(filePath);
                    _config = JsonSerializer.Deserialize<GameConfig>(json);
                    Console.WriteLine($"[ConfigService] Loaded config from {filePath}");
                    Console.WriteLine($"[ConfigService] Server: {_config?.ServerName}, Max Players: {_config?.MaxPlayers}");
                }
                catch (Exception ex)
                {
                    Console.WriteLine($"[ConfigService] Error loading config: {ex.Message}");
                    _config = new GameConfig();
                }
            }
            else
            {
                Console.WriteLine($"[ConfigService] Config file not found: {filePath}");
                _config = new GameConfig();
            }
        }

        public void ReloadConfig()
        {
            LoadConfig();
        }
    }
}
```

```csharp
using GameServer.Services;

namespace GameServer.Controllers

    [ApiController]
```

### 볼륨 백업과 복원

게임 데이터를 백업하고 복원하는 방법을 실습한다.

**볼륨 백업**을 수행한다.

```bash
# 임시 컨테이너로 볼륨 데이터를 tar로 압축
docker run --rm \
  -v game-player-data:/data \
  -v $(pwd):/backup \
  ubuntu \
  tar czf /backup/game-data-backup-$(date +%Y%m%d).tar.gz -C /data .

# 백업 파일 확인
ls -lh game-data-backup-*.tar.gz
```

**볼륨 복원**을 수행한다.

```bash
# 새 볼륨 생성
docker volume create game-player-data-restored

# 백업 파일에서 데이터 복원
docker run --rm \
  -v game-player-data-restored:/data \
  -v $(pwd):/backup \
  ubuntu \
  tar xzf /backup/game-data-backup-20251122.tar.gz -C /data

# 복원된 데이터로 서버 실행
docker run -d \
  -p 8085:8080 \
  -v game-player-data-restored:/app/data \
  --name game-server-restored \
  game-server:v2-persistent
```

### 볼륨 간 데이터 복사

한 볼륨에서 다른 볼륨으로 데이터를 복사한다.

```bash
# 소스 볼륨에서 대상 볼륨으로 복사
docker run --rm \
  -v game-player-data:/source:ro \
  -v game-player-data-copy:/destination \
  ubuntu \
  cp -av /source/. /destination/
```

### 볼륨 정리

사용하지 않는 볼륨을 정리한다.

```bash
# 특정 볼륨 삭제 (컨테이너가 사용 중이면 삭제 불가)
docker volume rm game-player-data-copy

# 사용하지 않는 모든 볼륨 삭제
docker volume prune

# 강제 삭제 (주의: 데이터 손실)
docker volume rm -f game-player-data
```
