# Chapter 1: Singleton Pattern (싱글톤 패턴)

## 게임 개발 현장에서...
당신은 2D 슈팅 게임을 만들고 있다. 게임에는 배경음악과 효과음을 관리하는 SoundManager가 필요하다. 

처음에는 간단했다. Player 클래스에서 총을 쏠 때 효과음을 재생하면 됐다.

```csharp
public class Player : MonoBehaviour 
{
    private SoundManager soundManager;
    
    void Start() 
    {
        soundManager = new SoundManager();
    }
    
    void Shoot() 
    {
        soundManager.PlaySound("shoot");
    }
}
```

그런데 문제가 생겼다. Enemy 클래스에서도 SoundManager가 필요했다. UI 버튼 클릭 시에도 소리를 재생해야 했다. GameOver 화면에서도 효과음이 필요했다.

각 클래스마다 SoundManager를 새로 만들었더니 이상한 일이 벌어졌다. 배경음악이 3개씩 동시에 재생됐다. 볼륨 설정을 바꿔도 일부 소리만 바뀌었다. 게임이 느려졌다.

"왜 이런 일이 생기는 걸까? SoundManager는 게임 전체에 하나만 있으면 되는데..."

바로 이 순간, Singleton Pattern이 필요하다.

## 문제점 분석

### 문제 1: 여러 개의 인스턴스 생성

```
메모리 상태:

[SoundManager #1] ← Player가 참조
  └─ volume: 1.0
  └─ audioSources: [...대용량 데이터...]

[SoundManager #2] ← Enemy가 참조
  └─ volume: 1.0
  └─ audioSources: [...대용량 데이터...]

[SoundManager #3] ← UIManager가 참조
  └─ volume: 1.0
  └─ audioSources: [...대용량 데이터...]

[SoundManager #4] ← SettingsMenu가 참조
  └─ volume: 0.5  ← 이것만 변경됨!
  └─ audioSources: [...대용량 데이터...]
```

게임 전체에서 하나만 있으면 되는 SoundManager가 4개나 생성되었다. 각각 독립적인 객체이므로 메모리를 4배로 낭비한다.

### 문제 2: 설정 공유 불가

```csharp
settings.ChangeVolume(0.5f);  // SettingsMenu의 SoundManager 볼륨만 변경

player.Shoot();   // Player의 SoundManager는 여전히 볼륨 1.0
enemy.Attack();   // Enemy의 SoundManager도 여전히 볼륨 1.0
```

각 객체가 서로 다른 SoundManager를 참조하므로 설정이 공유되지 않는다. 플레이어가 설정 메뉴에서 볼륨을 조절해도 실제 게임에서는 반영되지 않는다.

### 문제 3: 초기화 비용 증가

```
시간    0ms   100ms  200ms  300ms  400ms
        |      |      |      |      |
Player  [생성---------로딩--------완료]
Enemy         [생성---------로딩--------완료]
UI                   [생성---------로딩--------완료]
Settings                    [생성---------로딩--------완료]

총 소요 시간: 400ms
실제 필요 시간: 100ms (한 번만 생성했다면)
```

사운드 파일을 로드하는 무거운 작업이 4번 반복된다. 게임 시작이 느려지고, 메모리도 낭비된다.

### 문제 4: 배경음악 중복 재생

```csharp
public void PlayBackgroundMusic(string musicName) 
{
    // 배경음악 재생
}
```

만약 SoundManager가 배경음악도 관리한다면, 4개의 인스턴스가 각각 배경음악을 재생한다. 같은 음악이 4번 동시에 재생되어 귀가 아프다.

### 문제 5: 코드 관리의 어려움

```csharp
// 다른 개발자가 작성한 코드
public class Boss 
{
    private SoundManager soundManager;
    
    public Boss() 
    {
        // 또 새로운 인스턴스를 만들까? 
        // 아니면 다른 곳에서 받아올까?
        // 어떻게 해야 하지?
        soundManager = new SoundManager();  // 결국 또 만듦
    }
}
```

개발자마다 SoundManager를 어떻게 얻어야 할지 혼란스럽다. 결국 계속 새로운 인스턴스를 만들게 된다.
  

## 패턴 소개
Singleton Pattern은 클래스의 인스턴스가 프로그램 전체에서 단 하나만 존재하도록 보장하는 패턴이다.

### 핵심 개념

```
일반 클래스:
  new MyClass() → 인스턴스 1
  new MyClass() → 인스턴스 2
  new MyClass() → 인스턴스 3
  ...계속 생성 가능

Singleton:
  MyClass.Instance → 인스턴스 1
  MyClass.Instance → 인스턴스 1 (같은 것!)
  MyClass.Instance → 인스턴스 1 (계속 같은 것!)
  ...항상 같은 인스턴스 반환
```

### 구조 다이어그램

```
┌─────────────────────────────────┐
│      <<Singleton>>              │
│      SoundManager               │
├─────────────────────────────────┤
│ - instance : SoundManager       │  ← private static (자기 자신 저장)
│ - volume : float                │
├─────────────────────────────────┤
│ - SoundManager()                │  ← private 생성자 (외부 생성 차단)
│ + Instance : SoundManager       │  ← public static property (접근점)
│ + PlaySound(name)               │
│ + SetVolume(vol)                │
└─────────────────────────────────┘
         ▲
         │ 참조
         │
┌────────┴────────┬───────────┬──────────┐
│                 │           │          │
Player          Enemy      UIManager  SettingsMenu
│                 │           │          │
└─────────────────┴───────────┴──────────┘
        모두 같은 인스턴스를 참조
```

### Singleton 패턴의 3가지 핵심 요소

**1. private static 인스턴스 변수**
```csharp
private static SoundManager instance;
```
클래스 자신의 유일한 인스턴스를 저장한다. static이므로 클래스 레벨에서 단 하나만 존재한다.

**2. private 생성자**
```csharp
private SoundManager() 
{
    // 외부에서 new SoundManager() 호출 불가!
}
```
외부에서 인스턴스를 생성하지 못하도록 막는다.

**3. public static 접근 메서드/프로퍼티**
```csharp
public static SoundManager Instance 
{
    get 
    {
        if (instance == null) 
        {
            instance = new SoundManager();
        }
        return instance;
    }
}
```
유일한 인스턴스에 접근할 수 있는 전역 접근점을 제공한다.

### 동작 원리

```
첫 번째 호출: SoundManager.Instance
  ┌─────────────────────────────┐
  │ instance가 null인가?        │
  │ YES → 새로 생성             │
  └─────────────────────────────┘
           ↓
  [SoundManager 인스턴스 생성]
           ↓
  instance에 저장
           ↓
  반환

두 번째 호출: SoundManager.Instance
  ┌─────────────────────────────┐
  │ instance가 null인가?        │
  │ NO → 기존 인스턴스 사용     │
  └─────────────────────────────┘
           ↓
  기존 instance 반환

세 번째 호출: SoundManager.Instance
  ┌─────────────────────────────┐
  │ instance가 null인가?        │
  │ NO → 기존 인스턴스 사용     │
  └─────────────────────────────┘
           ↓
  기존 instance 반환
  
...계속 같은 인스턴스 반환
```
  

## 패턴 적용하기

### 기본 Singleton 구현

```csharp
// SoundManager.cs - Singleton 패턴 적용
public class SoundManager 
{
    // 1. private static 인스턴스 변수
    private static SoundManager instance;
    
    // 2. private 생성자
    private SoundManager() 
    {
        Console.WriteLine("SoundManager 생성됨! (단 한 번만)");
        LoadSounds();
    }
    
    // 3. public static 접근 프로퍼티
    public static SoundManager Instance 
    {
        get 
        {
            if (instance == null) 
            {
                instance = new SoundManager();
            }
            return instance;
        }
    }
    
    // 실제 기능들
    private float volume = 1.0f;
    private List<string> loadedSounds = new List<string>();
    
    private void LoadSounds() 
    {
        Console.WriteLine("사운드 파일 로딩 중...");
        loadedSounds.Add("shoot");
        loadedSounds.Add("attack");
        loadedSounds.Add("button_click");
    }
    
    public void PlaySound(string soundName) 
    {
        Console.WriteLine($"[볼륨 {volume}] {soundName} 재생");
    }
    
    public void SetVolume(float newVolume) 
    {
        volume = newVolume;
        Console.WriteLine($"전역 볼륨 변경: {volume}");
    }
    
    public void PlayBackgroundMusic(string musicName) 
    {
        Console.WriteLine($"배경음악 재생: {musicName}");
    }
}
```

### 게임 설정 관리 시스템 예제
실전에서 자주 사용하는 GameSettings 클래스를 Singleton으로 구현해보자.

```csharp
// GameSettings.cs
public class GameSettings 
{
    private static GameSettings instance;
    
    private GameSettings() 
    {
        LoadSettings();
    }
    
    public static GameSettings Instance 
    {
        get 
        {
            if (instance == null) 
            {
                instance = new GameSettings();
            }
            return instance;
        }
    }
    
    // 설정 값들
    public float MasterVolume { get; private set; } = 1.0f;
    public float MusicVolume { get; private set; } = 1.0f;
    public float SFXVolume { get; private set; } = 1.0f;
    public int GraphicsQuality { get; private set; } = 2; // 0:Low, 1:Medium, 2:High
    public bool IsFullscreen { get; private set; } = true;
    public int TargetFrameRate { get; private set; } = 60;
    public string Language { get; private set; } = "Korean";
    
    private void LoadSettings() 
    {
        // 실제로는 파일이나 PlayerPrefs에서 읽어옴
        Console.WriteLine("설정 로드 완료");
    }
    
    public void SetMasterVolume(float volume) 
    {
        MasterVolume = Mathf.Clamp01(volume);
        SaveSettings();
    }
    
    public void SetMusicVolume(float volume) 
    {
        MusicVolume = Mathf.Clamp01(volume);
        SaveSettings();
    }
    
    public void SetSFXVolume(float volume) 
    {
        SFXVolume = Mathf.Clamp01(volume);
        SaveSettings();
    }
    
    public void SetGraphicsQuality(int quality) 
    {
        GraphicsQuality = Mathf.Clamp(quality, 0, 2);
        ApplyGraphicsSettings();
        SaveSettings();
    }
    
    public void SetFullscreen(bool fullscreen) 
    {
        IsFullscreen = fullscreen;
        ApplyDisplaySettings();
        SaveSettings();
    }
    
    public void SetTargetFrameRate(int fps) 
    {
        TargetFrameRate = fps;
        Application.targetFrameRate = fps;
        SaveSettings();
    }
    
    public void SetLanguage(string language) 
    {
        Language = language;
        SaveSettings();
    }
    
    private void ApplyGraphicsSettings() 
    {
        Console.WriteLine($"그래픽 품질 적용: {GraphicsQuality}");
        // QualitySettings.SetQualityLevel(GraphicsQuality);
    }
    
    private void ApplyDisplaySettings() 
    {
        Console.WriteLine($"전체화면 모드: {IsFullscreen}");
        // Screen.fullScreen = IsFullscreen;
    }
    
    private void SaveSettings() 
    {
        // 실제로는 파일이나 PlayerPrefs에 저장
        Console.WriteLine("설정 저장됨");
    }
    
    public void ResetToDefault() 
    {
        MasterVolume = 1.0f;
        MusicVolume = 1.0f;
        SFXVolume = 1.0f;
        GraphicsQuality = 2;
        IsFullscreen = true;
        TargetFrameRate = 60;
        Language = "Korean";
        SaveSettings();
        Console.WriteLine("설정 초기화됨");
    }
}

// Mathf 유틸리티 (Unity의 Mathf.Clamp01과 유사)
public static class Mathf 
{
    public static float Clamp01(float value) 
    {
        if (value < 0f) return 0f;
        if (value > 1f) return 1f;
        return value;
    }
    
    public static int Clamp(int value, int min, int max) 
    {
        if (value < min) return min;
        if (value > max) return max;
        return value;
    }
}

// Application 유틸리티 (Unity의 Application과 유사)
public static class Application 
{
    public static int targetFrameRate;
}
```

## 실전 팁

### 1. Thread-Safe 구현
멀티스레드 환경에서는 기본 Singleton이 문제가 될 수 있다.

**문제 상황:**
```csharp
// 스레드 1과 스레드 2가 동시에 접근
Thread 1: if (instance == null)  // null이라고 판단
Thread 2: if (instance == null)  // null이라고 판단
Thread 1: instance = new SoundManager();  // 생성
Thread 2: instance = new SoundManager();  // 또 생성!

결과: 2개의 인스턴스가 만들어짐!
```

**해결책 1: Lock 사용**
```csharp
public class SoundManager 
{
    private static SoundManager instance;
    private static readonly object lockObject = new object();
    
    private SoundManager() { }
    
    public static SoundManager Instance 
    {
        get 
        {
            lock (lockObject)  // 한 번에 하나의 스레드만 접근
            {
                if (instance == null) 
                {
                    instance = new SoundManager();
                }
                return instance;
            }
        }
    }
}
```

**해결책 2: Double-Check Locking (더 빠름)**
```csharp
public class SoundManager 
{
    private static SoundManager instance;
    private static readonly object lockObject = new object();
    
    private SoundManager() { }
    
    public static SoundManager Instance 
    {
        get 
        {
            if (instance == null)  // 첫 번째 체크 (빠름)
            {
                lock (lockObject) 
                {
                    if (instance == null)  // 두 번째 체크 (안전함)
                    {
                        instance = new SoundManager();
                    }
                }
            }
            return instance;
        }
    }
}
```

**해결책 3: Lazy<T> 사용 (가장 간단함)**
```csharp
public class SoundManager 
{
    private static readonly Lazy<SoundManager> lazy = 
        new Lazy<SoundManager>(() => new SoundManager());
    
    private SoundManager() { }
    
    public static SoundManager Instance => lazy.Value;
}
```

### 2. Unity에서의 주의점
Unity에서 Singleton을 사용할 때는 MonoBehaviour와 결합해야 한다.

**Unity Singleton 패턴:**
```csharp
// Unity용 Singleton 베이스 클래스
public class SingletonMonoBehaviour<T> : MonoBehaviour where T : MonoBehaviour 
{
    private static T instance;
    private static readonly object lockObject = new object();
    
    public static T Instance 
    {
        get 
        {
            if (instance == null) 
            {
                lock (lockObject) 
                {
                    if (instance == null) 
                    {
                        // 씬에서 찾기
                        instance = FindObjectOfType<T>();
                        
                        if (instance == null) 
                        {
                            // 없으면 새로 만들기
                            GameObject singleton = new GameObject(typeof(T).Name);
                            instance = singleton.AddComponent<T>();
                            DontDestroyOnLoad(singleton);
                        }
                    }
                }
            }
            return instance;
        }
    }
    
    protected virtual void Awake() 
    {
        if (instance == null) 
        {
            instance = this as T;
            DontDestroyOnLoad(gameObject);
        }
        else if (instance != this) 
        {
            Destroy(gameObject);  // 중복 제거
        }
    }
}
```

**사용 예시:**

### 7. Singleton의 대안
때로는 Singleton 대신 다른 패턴이 더 나을 수 있다.

**Dependency Injection (의존성 주입):**
```csharp
public class Player 
{
    private readonly ISoundManager soundManager;
    
    // 생성자를 통해 주입받음
    public Player(ISoundManager soundManager) 
    {
        this.soundManager = soundManager;
    }
    
    public void Shoot() 
    {
        soundManager.PlaySound("shoot");
    }
}

// 사용
var soundManager = new SoundManager();
var player = new Player(soundManager);  // 명시적으로 전달
```

장점: 테스트하기 쉽고, 의존성이 명확함
단점: 코드가 조금 더 복잡해짐

**Service Locator (서비스 로케이터):**
```csharp
public class ServiceLocator 
{
    private static Dictionary<Type, object> services = new Dictionary<Type, object>();
    
    public static void Register<T>(T service) 
    {
        services[typeof(T)] = service;
    }
    
    public static T Get<T>() 
    {
        return (T)services[typeof(T)];
    }
}

// 사용
ServiceLocator.Register<ISoundManager>(new SoundManager());
var soundManager = ServiceLocator.Get<ISoundManager>();
```

public class InputManager 
{
    private static InputManager instance;
    private static readonly object lockObject = new object();
    
    private bool inputEnabled = true;
    private HashSet<string> pressedKeys = new HashSet<string>();
    private bool mouseClicked = false;
    
    private InputManager() { }
    
    public static InputManager Instance 
    {
        get 
        {
            if (instance == null) 
            {
                lock (lockObject) 
                {
                    if (instance == null) 
                    {
                        instance = new InputManager();
                    }
                }
            }
            return instance;
        }
    }
    
    public void Update() 
    {
        if (!inputEnabled) return;
        
        // 실제 구현에서는 Unity의 Input.GetKey 등을 사용
        // 여기서는 시뮬레이션
    }
    
    public bool IsKeyPressed(string key) 
    {
        if (!inputEnabled) return false;
        return pressedKeys.Contains(key);
    }
    
    public bool IsMouseClicked() 
    {
        if (!inputEnabled) return false;
        return mouseClicked;
    }
    
    public void EnableInput() 
    {
        inputEnabled = true;
        Console.WriteLine("입력 활성화");
    }
    
    public void DisableInput() 
    {
        inputEnabled = false;
        pressedKeys.Clear();
        mouseClicked = false;
        Console.WriteLine("입력 비활성화");
    }
    
    // 테스트용 메서드들
    public void SimulateKeyPress(string key) 
    {
        pressedKeys.Add(key);
    }
    
    public void SimulateKeyRelease(string key) 
    {
        pressedKeys.Remove(key);
    }
    
    public void SimulateMouseClick() 
    {
        mouseClicked = true;
    }
}

// 사용 예시

        // 키 입력 시뮬레이션
        input.SimulateKeyPress("W");
        Console.WriteLine($"W 키 눌림: {input.IsKeyPressed("W")}");

        input.DisableInput();
        Console.WriteLine($"비활성화 후 W 키: {input.IsKeyPressed("W")}");
