# 10장. 성능 분석과 디버깅

## 10.1 Windows Performance Toolkit 활용
Windows Performance Toolkit(WPT)은 Microsoft에서 제공하는 강력한 성능 분석 도구로, Windows 시스템의 성능 문제를 진단하고 최적화하기 위해 설계되었다. 이 도구는 CPU 사용률, 메모리 할당, 디스크 I/O, 네트워크 활동 등 시스템의 다양한 측면을 상세하게 기록하고 분석할 수 있다.

WPT는 두 개의 주요 구성 요소로 이루어져 있다. Windows Performance Recorder(WPR)는 시스템 성능 데이터를 수집하는 도구이며, Windows Performance Analyzer(WPA)는 수집된 데이터를 시각화하고 분석하는 도구다. 이 두 도구의 조합을 통해 사용자는 시스템의 병목 지점을 파악하고 성능 개선 방안을 도출할 수 있다.

WPT는 커널 레벨에서 데이터를 수집하기 때문에 매우 낮은 오버헤드로 정확한 성능 정보를 제공한다. 개발자, 시스템 관리자, 성능 엔지니어 등 다양한 전문가들이 프로세스 최적화, 시스템 튜닝, 문제 해결 등의 작업에 활용할 수 있다.

### 설치
Windows Performance Toolkit은 Windows Assessment and Deployment Kit(ADK) 또는 Windows Assessment and Deployment Kit for Windows Insider Preview(ADK for Windows Insider Preview)의 일부로 배포된다. 설치 방법은 다음과 같다.

먼저 Microsoft의 공식 웹사이트에서 해당 Windows 버전에 맞는 ADK 설치 프로그램을 다운로드해야 한다. 다운로드 완료 후 설치 프로그램을 실행하면 설치 마법사가 시작된다. 마법사에서 설치 위치를 선택한 후 "기능 선택" 단계에서 "Windows Performance Toolkit" 옵션을 체크해야 한다. 반드시 이 옵션을 선택해야만 WPR과 WPA가 설치된다.

선택 완료 후 "설치" 버튼을 클릭하면 필요한 파일들이 컴퓨터에 다운로드되고 설치된다. 설치 과정에서 관리자 권한이 필요하므로 미리 확인해야 한다. 설치가 완료되면 시작 메뉴에서 "Windows Performance Recorder"와 "Windows Performance Analyzer"를 찾을 수 있다.

일부 기능의 경우 추가 심볼 파일이 필요할 수 있다. 분석의 정확도를 높이기 위해 Microsoft 심볼 서버로부터 디버깅 심볼을 다운로드하는 것이 권장된다. WPA의 설정 메뉴에서 심볼 경로를 구성하여 자동으로 심볼을 다운로드하도록 설정할 수 있다.

### 프로파일링 하는 방법
Windows Performance Recorder를 사용하여 시스템 성능 데이터를 수집하는 과정을 프로파일링이라고 한다. 프로파일링을 시작하기 위해서는 먼저 Windows Performance Recorder를 관리자 권한으로 실행해야 한다.

WPR을 실행하면 여러 프로파일링 옵션을 선택할 수 있는 창이 나타난다. 상단의 드롭다운 메뉴에서 성능 분석의 목적에 맞는 프로필을 선택해야 한다. "CPU usage" 프로필은 CPU 사용률 분석을 위해 사용되며, "Disk I/O" 프로필은 디스크 성능 분석에 특화되어 있다. "Memory usage" 프로필은 메모리 할당 및 사용 패턴을 분석하는 데 도움이 된다.

프로필 선택 후 "시작" 버튼을 클릭하면 백그라운드에서 성능 데이터 수집이 시작된다. 이 시점부터 분석하고자 하는 애플리케이션 또는 작업을 실행해야 한다. 측정하려는 작업이 충분히 진행되었다고 판단되면 WPR 창에서 "중지" 버튼을 클릭한다.

중지 버튼을 클릭하면 수집된 데이터가 파일로 저장되는 과정이 시작된다. 파일명과 저장 위치를 지정하는 대화상자가 나타나며, 일반적으로 .etl(Event Trace Log) 형식으로 저장된다. 파일 저장이 완료될 때까지 기다렸다가 "저장" 버튼을 누르면 프로파일링 작업이 완료된다.

더 세밀한 제어가 필요한 경우 "옵션" 탭에서 추가 설정을 할 수 있다. 버퍼 크기, 수집 범위, 특정 커널 이벤트의 추적 여부 등을 상세하게 조정할 수 있다. 고급 사용자는 자신의 분석 목적에 맞게 커스텀 프로필을 만들어 사용할 수도 있다.

### 성능분석 방법
Windows Performance Analyzer는 WPR에서 수집한 .etl 파일을 열어 상세한 성능 분석을 수행한다. WPA를 실행한 후 "파일" 메뉴에서 "열기"를 선택하여 저장된 .etl 파일을 로드해야 한다.

파일이 로드되면 WPA의 메인 창에 다양한 분석 그래프와 표가 표시된다. 좌측 패널의 "그래프" 섹션에는 CPU, 메모리, 디스크 I/O 등 여러 가지 성능 지표가 나열되어 있다. 분석하고자 하는 항목을 클릭하면 해당 데이터가 시간대별로 표시된다.

CPU 분석의 경우 CPU 사용률 그래프에서 피크 지점을 확인할 수 있다. 그래프의 특정 구간을 선택하여 확대 보기를 할 수 있으며, 어떤 프로세스가 CPU를 가장 많이 사용했는지 상세하게 파악할 수 있다. 좌측 패널에서 "프로세스별 CPU 사용률" 옵션을 선택하면 각 프로세스가 사용한 CPU 시간을 정렬된 목록 형태로 볼 수 있다.

메모리 분석은 메모리 사용 패턴을 시간에 따라 추적한다. 메모리가 지속적으로 증가하는 경향이 보인다면 메모리 누수의 가능성을 의심할 수 있다. "메모리 할당" 그래프를 통해 어느 시점에서 메모리 할당이 일어났는지, 어떤 프로세스가 메모리를 할당했는지 확인할 수 있다.

디스크 I/O 분석은 디스크 읽기 및 쓰기 작업의 빈도와 규모를 보여준다. 디스크 I/O가 높은 구간을 선택하여 해당 시간에 어떤 프로세스가 디스크에 접근했는지 파악할 수 있다. 이를 통해 불필요한 디스크 작업을 최적화할 수 있다.

WPA의 강력한 기능 중 하나는 사용자가 분석 뷰를 커스터마이징할 수 있다는 점이다. 여러 그래프를 동시에 비교하거나, 특정 프로세스만 필터링하여 분석할 수 있다. 분석 결과를 더욱 깊이 있게 검토하기 위해 스택 트레이스 분석 기능도 활용할 수 있다.

성능 분석 완료 후 발견 사항을 정리하고 성능 개선 방안을 도출해야 한다. WPA에서 생성한 그래프와 데이터를 스크린샷으로 캡처하거나 보고서 형식으로 내보낼 수 있으며, 이를 통해 팀 내에서 성능 개선 방향을 논의할 수 있다.


### 실습: 테스트용 샘플 애플리케이션 작성
C++를 사용하여 성능 분석을 위한 테스트 프로그램을 작성한다. 이 프로그램은 CPU 부하, 메모리 할당, 디스크 I/O를 발생시킨다.

```cpp
#include <iostream>
#include <vector>
#include <cmath>
#include <chrono>
#include <fstream>
#include <windows.h>
#include <cstdint>

using namespace std;

// CPU 부하를 발생시키는 함수
void CPULoadTest()
{
    cout << "[1단계] CPU 부하 테스트 시작..." << endl;
    
    auto startTime = chrono::high_resolution_clock::now();
    auto endTime = startTime + chrono::seconds(5);
    
    while (chrono::high_resolution_clock::now() < endTime)
    {
        // 계산 집약적 작업
        double result = 0.0;
        for (int i = 0; i < 1000000; i++)
        {
            result += sqrt(static_cast<double>(i)) * sin(static_cast<double>(i));
        }
    }
    
    cout << "CPU 부하 테스트 완료 (5초 동안 실행)" << endl;
}

// 메모리 할당을 수행하는 함수
void MemoryAllocationTest()
{
    cout << "\n[2단계] 메모리 할당 테스트 시작..." << endl;
    
    vector<uint8_t*> memoryBlocks;
    const int blockSize = 10 * 1024 * 1024;  // 10MB
    const int blockCount = 50;
    
    for (int i = 0; i < blockCount; i++)
    {
        uint8_t* block = new uint8_t[blockSize];
        
        // 메모리를 실제로 사용하여 할당을 확실히 함
        for (int j = 0; j < blockSize; j += 4096)
        {
            block[j] = static_cast<uint8_t>(i);
        }
        
        memoryBlocks.push_back(block);
        cout << "메모리 할당: " << (i + 1) << " / " << blockCount 
             << " (" << (i + 1) * 10 << "MB)" << endl;
        
        Sleep(100);  // 100ms 대기
    }
    
    cout << "메모리 할당 테스트 완료" << endl;
    Sleep(2000);
    
    // 메모리 해제
    for (auto block : memoryBlocks)
    {
        delete[] block;
    }
    memoryBlocks.clear();
}

// 디스크 I/O를 수행하는 함수
void DiskIOTest()
{
    cout << "\n[3단계] 디스크 I/O 테스트 시작..." << endl;
    
    const char* testFilePath = "C:\\Temp\\performance_test.txt";
    
    // 파일에 데이터 쓰기
    cout << "디스크 쓰기 작업 시작..." << endl;
    ofstream outFile(testFilePath, ios::app);
    
    for (int i = 0; i < 100; i++)
    {
        string timestamp = to_string(chrono::system_clock::now().time_since_epoch().count());
        string data = "테스트 데이터 " + to_string(i) + ": " + timestamp + "\n";
        outFile << data;
        outFile.flush();
        
        cout << "쓰기 작업: " << (i + 1) << " / 100" << endl;
        Sleep(50);
    }
    outFile.close();
    
    // 파일에서 데이터 읽기
    cout << "디스크 읽기 작업 시작..." << endl;
    for (int i = 0; i < 50; i++)
    {
        ifstream inFile(testFilePath);
        string line;
        while (getline(inFile, line))
        {
            // 데이터 읽기
        }
        inFile.close();
        
        cout << "읽기 작업: " << (i + 1) << " / 50" << endl;
        Sleep(50);
    }
    
    // 테스트 파일 삭제
    remove(testFilePath);
    
    cout << "디스크 I/O 테스트 완료" << endl;
}

// 메인 함수
int main()
{
    cout << "성능 테스트 애플리케이션 시작" << endl;
    cout << "WPR에서 프로파일링을 시작한 후 아무 키나 누르세요..." << endl;
    cin.get();
    
    CPULoadTest();
    MemoryAllocationTest();
    DiskIOTest();
    
    cout << "\n모든 테스트 완료" << endl;
    cout << "종료하려면 아무 키나 누르세요..." << endl;
    cin.get();
    
    return 0;
}
```

### Windows API를 활용한 고급 성능 테스트
더 세밀한 성능 분석을 위해 Windows API를 직접 사용하는 고급 버전을 작성한다.

```cpp
#include <iostream>
#include <vector>
#include <windows.h>
#include <psapi.h>
#include <cmath>
#include <fstream>
#include <chrono>

#pragma comment(lib, "psapi.lib")

using namespace std;

// 현재 프로세스의 메모리 정보를 출력하는 함수
void PrintMemoryInfo()
{
    PROCESS_MEMORY_COUNTERS pmc;
    HANDLE hProcess = GetCurrentProcess();
    
    if (GetProcessMemoryInfo(hProcess, &pmc, sizeof(pmc)))
    {
        cout << "메모리 사용량: " << (pmc.WorkingSetSize / 1024 / 1024) << " MB" << endl;
        cout << "페이지 파일 사용량: " << (pmc.PagefileUsage / 1024 / 1024) << " MB" << endl;
    }
}

// CPU 사이클 측정을 포함한 CPU 부하 테스트
void AdvancedCPULoadTest()
{
    cout << "[CPU 부하 테스트]" << endl;
    
    LARGE_INTEGER frequency;
    LARGE_INTEGER startCounter, endCounter;
    
    QueryPerformanceFrequency(&frequency);
    QueryPerformanceCounter(&startCounter);
    
    auto startTime = chrono::high_resolution_clock::now();
    auto endTime = startTime + chrono::seconds(10);
    
    uint64_t operationCount = 0;
    
    while (chrono::high_resolution_clock::now() < endTime)
    {
        for (int i = 0; i < 100000; i++)
        {
            double result = sqrt(static_cast<double>(i)) * cos(static_cast<double>(i));
            result = result * sin(static_cast<double>(i));
            operationCount++;
        }
    }
    
    QueryPerformanceCounter(&endCounter);
    
    double elapsedTime = static_cast<double>(endCounter.QuadPart - startCounter.QuadPart) 
                         / static_cast<double>(frequency.QuadPart);
    double operationsPerSecond = operationCount / elapsedTime;
    
    cout << "실행 시간: " << elapsedTime << "초" << endl;
    cout << "총 연산 수: " << operationCount << endl;
    cout << "초당 연산 수: " << operationsPerSecond << endl;
    
    PrintMemoryInfo();
}

// 다양한 크기의 메모리 할당 테스트
void AdvancedMemoryAllocationTest()
{
    cout << "\n[메모리 할당 테스트]" << endl;
    
    vector<void*> allocations;
    
    // 1단계: 순차 할당
    cout << "순차 메모리 할당..." << endl;
    for (int i = 0; i < 30; i++)
    {
        size_t size = (i + 1) * 1024 * 1024;  // 1MB부터 30MB까지 점진적 증가
        void* ptr = malloc(size);
        
        if (ptr != nullptr)
        {
            // 메모리 초기화
            memset(ptr, 0xAA, size);
            allocations.push_back(ptr);
            cout << "할당됨: " << size / 1024 / 1024 << " MB" << endl;
            Sleep(200);
        }
    }
    
    PrintMemoryInfo();
    
    // 2단계: 일부 해제
    cout << "\n부분 메모리 해제..." << endl;
    for (int i = 0; i < allocations.size() / 2; i++)
    {
        free(allocations[i]);
        allocations[i] = nullptr;
        cout << "해제됨: " << (i + 1) << " / " << (allocations.size() / 2) << endl;
        Sleep(100);
    }
    
    PrintMemoryInfo();
    
    // 3단계: 나머지 해제
    cout << "\n남은 메모리 해제..." << endl;
    for (size_t i = allocations.size() / 2; i < allocations.size(); i++)
    {
        if (allocations[i] != nullptr)
        {
            free(allocations[i]);
            allocations[i] = nullptr;
        }
    }
    allocations.clear();
    
    PrintMemoryInfo();
}

// 순차 디스크 I/O 테스트
void SequentialDiskIOTest()
{
    cout << "\n[순차 디스크 I/O 테스트]" << endl;
    
    const char* filename = "C:\\Temp\\sequential_io_test.dat";
    const int fileSize = 100 * 1024 * 1024;  // 100MB
    const int chunkSize = 1024 * 1024;  // 1MB
    
    cout << "순차 쓰기 시작..." << endl;
    
    LARGE_INTEGER frequency, startCounter, endCounter;
    QueryPerformanceFrequency(&frequency);
    QueryPerformanceCounter(&startCounter);
    
    ofstream outFile(filename, ios::binary);
    char* buffer = new char[chunkSize];
    memset(buffer, 0xAA, chunkSize);
    
    for (int i = 0; i < (fileSize / chunkSize); i++)
    {
        outFile.write(buffer, chunkSize);
        cout << "쓰기 진행: " << ((i + 1) * 100 / (fileSize / chunkSize)) << "%" << endl;
        Sleep(10);
    }
    
    outFile.close();
    
    QueryPerformanceCounter(&endCounter);
    double writeTime = static_cast<double>(endCounter.QuadPart - startCounter.QuadPart) 
                       / static_cast<double>(frequency.QuadPart);
    double writeMBps = (fileSize / (1024 * 1024)) / writeTime;
    
    cout << "쓰기 완료 - 시간: " << writeTime << "초, 속도: " << writeMBps << " MB/s" << endl;
    
    // 읽기 테스트
    cout << "\n순차 읽기 시작..." << endl;
    QueryPerformanceCounter(&startCounter);
    
    ifstream inFile(filename, ios::binary);
    for (int i = 0; i < (fileSize / chunkSize); i++)
    {
        inFile.read(buffer, chunkSize);
        cout << "읽기 진행: " << ((i + 1) * 100 / (fileSize / chunkSize)) << "%" << endl;
        Sleep(10);
    }
    
    inFile.close();
    
    QueryPerformanceCounter(&endCounter);
    double readTime = static_cast<double>(endCounter.QuadPart - startCounter.QuadPart) 
                      / static_cast<double>(frequency.QuadPart);
    double readMBps = (fileSize / (1024 * 1024)) / readTime;
    
    cout << "읽기 완료 - 시간: " << readTime << "초, 속도: " << readMBps << " MB/s" << endl;
    
    delete[] buffer;
    remove(filename);
}

// 무작위 디스크 I/O 테스트
void RandomDiskIOTest()
{
    cout << "\n[무작위 디스크 I/O 테스트]" << endl;
    
    const char* filename = "C:\\Temp\\random_io_test.dat";
    const int fileSize = 50 * 1024 * 1024;  // 50MB
    const int requestSize = 4096;  // 4KB
    const int requestCount = 1000;
    
    // 파일 생성
    ofstream createFile(filename, ios::binary);
    char* initBuffer = new char[fileSize];
    memset(initBuffer, 0xBB, fileSize);
    createFile.write(initBuffer, fileSize);
    createFile.close();
    delete[] initBuffer;
    
    cout << "무작위 읽기 시작..." << endl;
    
    LARGE_INTEGER frequency, startCounter, endCounter;
    QueryPerformanceFrequency(&frequency);
    QueryPerformanceCounter(&startCounter);
    
    HANDLE hFile = CreateFileA(filename, GENERIC_READ, FILE_SHARE_READ,
                               nullptr, OPEN_EXISTING, FILE_FLAG_RANDOM_ACCESS, nullptr);
    
    char* buffer = new char[requestSize];
    DWORD bytesRead;
    
    for (int i = 0; i < requestCount; i++)
    {
        DWORD offset = (rand() % (fileSize - requestSize)) / requestSize * requestSize;
        LARGE_INTEGER fileOffset;
        fileOffset.QuadPart = offset;
        
        SetFilePointerEx(hFile, fileOffset, nullptr, FILE_BEGIN);
        ReadFile(hFile, buffer, requestSize, &bytesRead, nullptr);
        
        if ((i + 1) % 100 == 0)
        {
            cout << "요청: " << (i + 1) << " / " << requestCount << endl;
        }
        Sleep(5);
    }
    
    CloseHandle(hFile);
    
    QueryPerformanceCounter(&endCounter);
    double ioTime = static_cast<double>(endCounter.QuadPart - startCounter.QuadPart) 
                    / static_cast<double>(frequency.QuadPart);
    double iopsPerSecond = requestCount / ioTime;
    
    cout << "무작위 읽기 완료 - 시간: " << ioTime << "초, IOPS: " << iopsPerSecond << endl;
    
    delete[] buffer;
    remove(filename);
}
```

### Windows Performance Recorder를 사용한 프로파일링 실습
C++ 샘플 애플리케이션을 프로파일링하는 단계별 방법을 설명한다.

**Step 1: 프로젝트 설정 및 컴파일**

Visual Studio에서 새로운 콘솔 애플리케이션 프로젝트를 생성한다. 위의 C++ 코드를 프로젝트에 추가하고 컴파일한다. 릴리스 모드로 빌드하는 것이 성능 측정 정확도를 높인다. 컴파일 완료 후 생성된 실행 파일의 경로를 기억해두어야 한다.

**Step 2: 관리자 권한으로 Windows Performance Recorder 실행**

시작 메뉴에서 "Windows Performance Recorder"를 찾아서 마우스 오른쪽 버튼으로 클릭한 후 "관리자 권한으로 실행"을 선택한다. WPR 메인 윈도우가 나타나면 다양한 프로필 옵션이 보인다.

**Step 3: 프로필 선택 및 옵션 구성**

WPR 윈도우 상단의 드롭다운 메뉴에서 "CPU usage" 프로필을 선택한다. 좌측 패널에서 다음 항목들의 체크박스를 활성화한다: Memory (Commit), Disk I/O, File I/O, Registry I/O. 이렇게 하면 CPU, 메모리, 디스크 활동을 모두 수집하게 된다.

**Step 4: 버퍼 크기 설정**

WPR 윈도우의 "옵션" 탭을 클릭한다. "버퍼 크기" 항목에서 "중간" 이상으로 설정하여 많은 양의 데이터를 수집할 수 있도록 한다. "수집 모드"는 기본값인 "실시간"으로 둔다.

**Step 5: 프로파일링 시작**

WPR 윈도우에서 "시작" 버튼을 클릭한다. 상태 표시줄에 "프로파일링 중" 메시지가 표시되고, 프로파일링이 백그라운드에서 진행된다.

**Step 6: 테스트 애플리케이션 실행**

명령 프롬프트 또는 PowerShell에서 컴파일된 C++ 실행 파일을 실행한다. 프로그램이 시작되어 "아무 키나 누르세요" 메시지를 표시하면 Enter 키를 누르거나 아무 키나 누른다. 프로그램은 CPU, 메모리, 디스크 테스트를 순차적으로 진행한다. 모든 테스트가 완료될 때까지 기다린다.

**Step 7: 프로파일링 중지**

테스트 애플리케이션이 완료되면 WPR 윈도우로 돌아가서 "중지" 버튼을 클릭한다. 수집된 데이터가 처리되는 동안 잠시 기다린다. 파일 저장 대화상자가 나타나면 파일 이름을 "cpp_performance_test.etl"로 지정하고 C:\Temp 폴더에 저장한다.
  

### Windows Performance Analyzer를 사용한 분석 실습
수집한 C++ 프로파일링 데이터를 분석하는 방법이다.

**Step 1: 데이터 파일 열기**

Windows Performance Analyzer를 실행한다. "파일" 메뉴에서 "열기"를 선택하고 앞서 저장한 "cpp_performance_test.etl" 파일을 선택한다. 파일 크기에 따라 로딩에 수십 초가 소요될 수 있다. 로딩 완료 후 WPA의 메인 창에 여러 그래프와 데이터가 표시된다.

**Step 2: CPU 분석**

좌측 패널의 "그래프" 섹션에서 "CPU (Sampled)" 옵션을 더블클릭한다. 메인 영역에 CPU 사용률이 시간에 따라 그래프로 표시된다. 첫 번째 10초 구간에서 높은 CPU 사용률을 보여야 한다. CPU 그래프의 피크 부분을 마우스로 드래그하여 선택하면 자세한 정보가 표시된다.

CPU 그래프 아래의 "프로세스별" 탭을 클릭하여 어떤 프로세스가 CPU를 가장 많이 사용했는지 확인한다. C++ 테스트 애플리케이션의 이름이 가장 높은 CPU 사용률을 보여야 한다.

**Step 3: 함수별 CPU 사용률 분석**

"스택" 탭을 클릭하면 함수 호출 스택이 표시된다. sqrt, sin, cos 등의 수학 함수들이 높은 CPU 사용률을 보여야 한다. 이를 통해 어느 함수가 가장 많은 CPU 시간을 소비했는지 파악할 수 있다.

**Step 4: 메모리 분석**

좌측 패널에서 "VirtualAlloc" 또는 "Heap Allocations" 그래프를 찾아 선택한다. 메모리 할당 구간(프로그램 실행 후 약 5~15초)에서 메모리 사용량이 급증하는 패턴이 보인다. 그래프의 상승 구간에서 마우스를 올리면 할당 크기 정보가 표시된다.

"프로세스별" 탭에서는 어떤 프로세스가 메모리를 할당했는지 확인할 수 있다. "함수별" 탭에서는 new 또는 malloc 함수가 높은 할당률을 보인다.

**Step 5: 디스크 I/O 분석**

좌속 패널에서 "Disk I/O" 또는 "File I/O" 그래프를 선택한다. 프로그램 실행 후 약 15초 이후 부분에서 디스크 I/O 활동이 증가해야 한다. 피크 부분을 선택하여 확대하면 성능_test.dat 파일에 대한 접근이 표시된다.

**Step 6: I/O 성능 메트릭 확인**

"I/O File" 탭의 테이블에서 다음 정보를 확인할 수 있다: 파일 이름, 읽기 횟수, 쓰기 횟수, 평균 I/O 크기, 총 I/O 시간. 이 정보를 통해 순차 I/O와 무작위 I/O의 성능 차이를 비교할 수 있다.

**Step 7: 멀티스레드 분석**

"스레드별" 탭을 클릭하면 각 스레드의 활동을 확인할 수 있다. C++ 애플리케이션이 단일 스레드로 실행되었으므로 하나의 스레드 항목만 표시되어야 한다.

**Step 8: 이벤트 추적**

특정 시간 구간의 상세한 이벤트를 보려면 "이벤트" 뷰를 사용한다. "View" 메뉴에서 "Event List"를 선택하면 모든 발생한 이벤트가 시간 순서대로 나열된다. 특정 이벤트를 클릭하면 해당 이벤트의 상세 정보가 표시된다.

```cpp
#include <windows.h> // Windows API 사용 (SetThreadDescription, DebugBreak 등)
#include <map>         // 맵 (스레드 ID -> 스레드 정보, 락 주소 -> 락 정보)
#include <set>         // 셋 (데드락 검사 시 방문한 스레드 추적)

// 데드락 감지기 클래스
class DeadlockDetector {
private:
    // 락(뮤텍스)에 대한 정보를 저장하는 구조체
    struct LockInfo {
        std::thread::id threadId; // 락을 소유한 스레드 ID
        std::string lockName;       // 락의 이름 (디버깅용)
        std::chrono::steady_clock::time_point acquireTime; // 락을 획득한 시간
        void* lockAddress;      // 락 객체의 메모리 주소 (고유 식별자)
        std::string sourceLocation; // 락을 획득/대기하는 코드 위치 (파일명:줄번호)
    };
    
    // 스레드의 상태 정보를 저장하는 구조체
    struct ThreadInfo {
        std::thread::id threadId;            // 이 정보를 소유한 스레드 ID
        std::vector<LockInfo> heldLocks;   // 현재 이 스레드가 보유한 락 목록
        LockInfo waitingFor;               // 현재 이 스레드가 대기 중인 락 정보
        bool isWaiting = false;            // 락을 대기 중인지 여부
    };
    
    // DeadlockDetector의 내부 데이터를 보호하기 위한 뮤텍스
    std::mutex detectorMutex_; 
    // 모든 스레드의 상태 정보를 관리 (스레드 ID -> ThreadInfo)
    std::map<std::thread::id, ThreadInfo> threadStates_;
    // 현재 락이 어떤 스레드에 의해 소유되었는지 관리 (락 주소 -> LockInfo)
    std::map<void*, LockInfo> lockOwnership_;
    // 데드락 감지 기능 활성화 여부
    std::atomic<bool> detectionEnabled_{true};
    
    // 경고 임계값
    // 락 대기 시간이 이 값을 초과하면 'Long Wait' 경고 출력
    static constexpr auto LONG_WAIT_THRESHOLD = std::chrono::milliseconds(1000);
    // 백그라운드 스레드가 데드락 검사를 수행하는 주기
    static constexpr auto DEADLOCK_CHECK_INTERVAL = std::chrono::milliseconds(100);
    
public:
    // 싱글톤(Singleton) 패턴: DeadlockDetector의 유일한 인스턴스를 반환
    static DeadlockDetector& GetInstance() {
        static DeadlockDetector instance; // 프로그램 전체에서 유일한 인스턴스
        return instance;
    }
    
    // 생성자
    DeadlockDetector() {
        // 백그라운드에서 주기적으로 데드락을 감지하는 스레드 시작
        std::thread detectionThread([this]() {
            DeadlockDetectionLoop();
        });
        detectionThread.detach(); // 메인 스레드와 분리 (독립적으로 실행)
    }
    
    // (실제 락 획득 시도 *전*에 호출)
    void BeforeAcquire(void* lockPtr, const std::string& lockName, 
                      const std::string& sourceLocation = "") {
        if (!detectionEnabled_.load()) return; // 기능 비활성화 시 반환
        
        std::lock_guard<std::mutex> lock(detectorMutex_); // 내부 데이터 보호
        auto threadId = std::this_thread::get_id(); // 현재 스레드 ID
        
        // 현재 스레드의 정보 가져오기 (없으면 새로 생성)
        ThreadInfo& threadInfo = threadStates_[threadId];
        threadInfo.threadId = threadId;
        // 어떤 락을 기다리는지 상태 업데이트
        threadInfo.waitingFor = {threadId, lockName, std::chrono::steady_clock::now(), 
                                lockPtr, sourceLocation};
        threadInfo.isWaiting = true;
        
        // 락 대기 시작 즉시 데드락 검사
        if (DetectDeadlockForThread(threadId)) {
            ReportDeadlock(threadId); // 데드락 발견 시 보고
        }
    }
    
    // (실제 락 획득 *후*에 호출)
    void AfterAcquire(void* lockPtr, const std::string& lockName,
                     const std::string& sourceLocation = "") {
        if (!detectionEnabled_.load()) return;
        
        std::lock_guard<std::mutex> lock(detectorMutex_);
        auto threadId = std::this_thread::get_id();
        
        ThreadInfo& threadInfo = threadStates_[threadId];
        threadInfo.isWaiting = false; // 대기 상태 해제
        
        // 획득한 락 정보 생성
        LockInfo lockInfo = {threadId, lockName, std::chrono::steady_clock::now(),
                            lockPtr, sourceLocation};
        
        // 스레드가 보유한 락 목록에 추가
        threadInfo.heldLocks.push_back(lockInfo);
        // 락 소유권 맵에 등록
        lockOwnership_[lockPtr] = lockInfo;
    }
    
    // (실제 락 해제 *후*에 호출)
    void AfterRelease(void* lockPtr) {
        if (!detectionEnabled_.load()) return;
        
        std::lock_guard<std::mutex> lock(detectorMutex_);
        auto threadId = std::this_thread::get_id();
        
        ThreadInfo& threadInfo = threadStates_[threadId];
        
        // 스레드가 보유한 락 목록에서 해당 락 제거
        threadInfo.heldLocks.erase(
            std::remove_if(threadInfo.heldLocks.begin(), threadInfo.heldLocks.end(),
                [lockPtr](const LockInfo& info) { return info.lockAddress == lockPtr; }),
            threadInfo.heldLocks.end()
        );
        
        // 락 소유권 맵에서 제거
        lockOwnership_.erase(lockPtr);
    }
    
private:
    // 백그라운드 스레드에서 실행되는 메인 루프
    void DeadlockDetectionLoop() {
        // 스레드 이름을 "DeadlockDetector"로 설정 (디버깅 시 유용)
        SetThreadDescription(GetCurrentThread(), L"DeadlockDetector");
        
        // 감지 기능이 활성화되어 있는 동안 무한 반복
        while (detectionEnabled_.load()) {
            // 설정된 주기(INTERVAL)만큼 대기
            std::this_thread::sleep_for(DEADLOCK_CHECK_INTERVAL);
            
            std::lock_guard<std::mutex> lock(detectorMutex_);
            
            // 모든 스레드 상태를 순회
            for (const auto& [threadId, threadInfo] : threadStates_) {
                // 현재 락을 대기 중인 스레드만 검사
                if (threadInfo.isWaiting) {
                    // 대기 시간 계산
                    auto waitTime = std::chrono::steady_clock::now() - threadInfo.waitingFor.acquireTime;
                    
                    // 대기 시간이 임계값을 넘으면 경고 리포트
                    if (waitTime > LONG_WAIT_THRESHOLD) {
                        ReportLongWait(threadId, waitTime);
                    }
                    
                    // 이 스레드로부터 시작하는 데드락(순환)이 있는지 검사
                    if (DetectDeadlockForThread(threadId)) {
                        ReportDeadlock(threadId); // 데드락 리포트
                    }
                }
            }
        }
    }
    
    // 특정 스레드에서 시작하는 데드락(순환)이 있는지 검사하는 진입점
    bool DetectDeadlockForThread(std::thread::id startThreadId) {
        // DFS(깊이 우선 탐색)를 위한 자료구조
        std::set<std::thread::id> visited;        // 이미 방문(검사)한 노드
        std::set<std::thread::id> recursionStack; // 현재 탐색 경로에 포함된 노드
        
        // 재귀적으로 순환(Cycle)이 있는지 검사
        return HasCycle(startThreadId, visited, recursionStack);
    }
    
    // 데드락(순환)을 찾는 핵심 로직 (DFS 알고리즘)
    bool HasCycle(std::thread::id currentThreadId, 
                  std::set<std::thread::id>& visited,
                  std::set<std::thread::id>& recursionStack) {
        
        // 1. 현재 탐색 경로(recursionStack)에 이미 이 스레드가 있다면,
        //    이는 순환(A -> B -> ... -> A)을 의미. 데드락 발견!
        if (recursionStack.find(currentThreadId) != recursionStack.end()) {
            return true; // 사이클 발견
        }
        
        // 2. 이미 검사가 끝난(visited) 스레드라면,
        //    이 경로는 이전에 검사했으므로 더 이상 탐색 불필요.
        if (visited.find(currentThreadId) != visited.end()) {
            return false; // 이미 방문한 노드
        }
        
        // 3. 현재 스레드를 '방문' 및 '현재 경로'에 추가
        visited.insert(currentThreadId);
        recursionStack.insert(currentThreadId);
        
        // 4. 현재 스레드가 대기 중인 락을 찾는다
        auto it = threadStates_.find(currentThreadId);
        if (it != threadStates_.end() && it->second.isWaiting) {
            void* waitingForLock = it->second.waitingFor.lockAddress;
            
            // 5. 이 락을 소유한 '다음 스레드'를 찾는다
            auto lockIt = lockOwnership_.find(waitingForLock);
            if (lockIt != lockOwnership_.end()) {
                std::thread::id ownerThreadId = lockIt->second.threadId;
                
                // 6. '다음 스레드'에 대해 재귀적으로 순환 검사
                if (HasCycle(ownerThreadId, visited, recursionStack)) {
                    return true; // 순환 발견 시 true 반환
                }
            }
        }
        
        // 7. 이 스레드에서 시작하는 모든 경로 탐색이 끝났으므로, '현재 경로'에서 제거
        recursionStack.erase(currentThreadId);
        return false; // 순환(데드락) 없음
    }
    
    // 데드락 발견 시 호출되는 리포트 함수
    void ReportDeadlock(std::thread::id threadId) {
        std::ostringstream oss;
        oss << "\n🚨 DEADLOCK DETECTED! 🚨\n";
        oss << "Thread ID: " << threadId << "\n";
        oss << "Deadlock chain:\n";
        
        // 데드락 순환 고리를 출력
        PrintDeadlockChain(threadId, oss);
        
        std::cout << oss.str() << std::endl;
        
        // (실제로는 MiniDumpWriteDump 같은 함수로 크래시 덤프 생성)
        GenerateCrashDump("deadlock");
    }
    
    // 장기 대기(Long Wait) 발견 시 호출되는 리포트 함수
    void ReportLongWait(std::thread::id threadId, std::chrono::milliseconds waitTime) {
        auto it = threadStates_.find(threadId);
        if (it != threadStates_.end()) {
            // 어떤 스레드가 어떤 락을 얼마나 오래 기다리는지 출력
            std::cout << "⚠️  Long wait detected: Thread " << threadId 
                      << " waiting for '" << it->second.waitingFor.lockName 
                      << "' for " << waitTime.count() << "ms\n";
        }
    }
    
    // 데드락에 연루된 스레드와 락의 관계를 추적하여 출력
    void PrintDeadlockChain(std::thread::id startThreadId, std::ostringstream& oss) {
        std::set<std::thread::id> visited;
        std::thread::id currentThreadId = startThreadId;
        
        // 순환 고리를 따라가며, 이미 방문한 스레드를 다시 만날 때까지 반복
        while (visited.find(currentThreadId) == visited.end()) {
            visited.insert(currentThreadId);
            
            auto it = threadStates_.find(currentThreadId);
            if (it == threadStates_.end() || !it->second.isWaiting) break;
            
            const auto& threadInfo = it->second;
            // "스레드 A가 'Lock B'를 기다림"
            oss << "  Thread " << currentThreadId 
                << " waiting for '" << threadInfo.waitingFor.lockName << "'\n";
            
            // "    (스레드 A는 'Lock A'를 보유 중)"
            oss << "    Holds: ";
            for (const auto& lock : threadInfo.heldLocks) {
                oss << "'" << lock.lockName << "' ";
            }
            oss << "\n";
            
            // 다음 스레드(이 스레드가 기다리는 락을 소유한 스레드) 찾기
            void* waitingForLock = threadInfo.waitingFor.lockAddress;
            auto lockIt = lockOwnership_.find(waitingForLock);
            if (lockIt != lockOwnership_.end()) {
                currentThreadId = lockIt->second.threadId; // 다음 스레드로 이동
            } else {
                break; // 소유자가 없으면 종료 (이론상 데드락이면 발생 안 함)
            }
        }
    }
    
    // 크래시 덤프 생성 (시뮬레이션)
    void GenerateCrashDump(const std::string& reason) {
        // 실제 구현에서는 <dbghelp.h>의 MiniDumpWriteDump 함수 사용
        std::string dumpFileName = "gameserver_" + reason + "_" + 
                                  std::to_string(GetCurrentProcessId()) + ".dmp";
        
        std::cout << "Generating crash dump: " << dumpFileName << "\n";
        
        // Windows 디버거가 연결되어 있으면 중단점(Breakpoint)을 발생시켜 즉시 디버깅
        if (IsDebuggerPresent()) {
            DebugBreak();
        }
    }
};

// 매크로를 통한 편리한 사용
// __FILE__ (파일명)과 __LINE__ (줄번호)을 자동으로 넘겨줌
#define DEADLOCK_DETECTOR_BEFORE_ACQUIRE(lock, name) \
    DeadlockDetector::GetInstance().BeforeAcquire(lock, name, __FILE__ ":" + std::to_string(__LINE__))

#define DEADLOCK_DETECTOR_AFTER_ACQUIRE(lock, name) \
    DeadlockDetector::GetInstance().AfterAcquire(lock, name, __FILE__ ":" + std::to_string(__LINE__))

#define DEADLOCK_DETECTOR_AFTER_RELEASE(lock) \
    DeadlockDetector::GetInstance().AfterRelease(lock)
```

이 코드는 C++로 작성된 **실시간 데드락(Deadlock) 감지기**이다. 멀티스레드 환경에서 발생할 수 있는 교착 상태(데드락)를 찾아내어 리포트하고, 필요시 디버깅을 위한 크래시 덤프를 생성하는 디버깅 도구이다.

#### 핵심 목적
데드락은 둘 이상의 스레드가 서로가 점유하고 있는 자원(락)을 기다리며 무한 대기 상태에 빠지는 현상이다. 이 코드는 이러한 상황을 런타임 중에 감지하여 개발자에게 알리는 것이 목적이다.

#### 작동 원리
이 감지기는 **자원 할당 그래프(Resource-Allocation Graph)** 또는 **대기 그래프(Wait-for Graph)**의 원리를 이용한다.  

1.  **상태 추적 (State Tracking)**

      * `threadStates_` (맵): 어떤 스레드(`thread::id`)가 현재 어떤 락(`LockInfo`)들을 보유하고 있으며, 어떤 락을 기다리고 있는지(`waitingFor`) 관리한다.
      * `lockOwnership_` (맵): 어떤 락(`void* lockAddress`)이 현재 어떤 스레드에 의해 소유되었는지 관리합한니다.

2.  **락 연산 감시 (Hooking)**

      * 개발자는 실제 락(예: `std::mutex`)을 `lock()` 하거나 `unlock()` 하는 코드 **주변**에 이 감지기 함수들을 수동으로 호출해야 한다.
      * `BeforeAcquire()`: 스레드가 락 획득을 **시도하기 직전**에 호출. "이제부터 이 락을 기다립니다."라고 감지기에 등록한다.
      * `AfterAcquire()`: 스레드가 락 획득에 **성공한 직후**에 호출. "대기 상태가 끝났고, 이제 이 락은 내 소유입니다."라고 등록한다.
      * `AfterRelease()`: 스레드가 락을 **해제한 직후**에 호출. "이 락을 더 이상 소유하지 않습니다."라고 등록한다.

3.  **데드락 감지 (Cycle Detection)**

      * 데드락은 "대기 그래프"에서 **순환(Cycle)**이 발생한 것과 같다.
          * 예: 스레드 A가 락 B를 기다림.
          * 락 B는 스레드 B가 소유함.
          * 스레드 B는 락 A를 기다림.
          * 락 A는 스레드 A가 소유함.
          * 이는 `스레드 A -> (락 B) -> 스레드 B -> (락 A) -> 스레드 A` 형태의 순환 고리를 만든다.
      * `HasCycle()` 함수가 이 순환을 찾는 핵심 로직이다. **DFS(깊이 우선 탐색)** 알고리즘을 사용하여, 특정 스레드에서 시작하여 자신에게 다시 돌아오는 대기 경로가 있는지 탐색한다.

#### 주요 구성 요소

  * **`DeadlockDetector` (싱글톤 클래스)**
      * `GetInstance()`: 프로그램 전역에서 단 하나의 감지기 인스턴스만 존재하도록 보장합니다 (싱글톤 패턴).
  * **`DeadlockDetectionLoop` (백그라운드 스레드)**
      * 생성자에서 별도의 스레드로 실행되어, `DEADLOCK_CHECK_INTERVAL` (100ms)마다 모든 스레드 상태를 검사한다.
      * **주기적 검사:** `BeforeAcquire` 시점의 즉각적인 검사에서 놓칠 수 있는 복잡한 데드락을 주기적으로 검사한다.
      * **장기 대기 검사:** 특정 스레드가 `LONG_WAIT_THRESHOLD` (1초) 이상 락을 기다리면, 데드락은 아니더라도 성능 문제를 유발할 수 있으므로 `ReportLongWait`를 호출해 경고한다.
  * **`DetectDeadlockForThread` / `HasCycle`**
      * "스레드 A가 락 B를 기다리고, 락 B는 스레드 C가 가졌고, 스레드 C는 락 D를 기다리고..." 이 관계를 재귀적으로 따라가며 순환(Cycle)을 찾는 그래프 탐색 알고리즘이다.
  * **`ReportDeadlock` / `PrintDeadlockChain`**
      * 데드락이 감지되면, 어떤 스레드가 어떤 락을 기다리고, 그 락은 누가 가졌는지를 추적하여 순환 고리 전체를 콘솔에 출력한다.
  * **매크로 (Macros)**
      * `DEADLOCK_DETECTOR_BEFORE_ACQUIRE` 등: 사용자가 더 편리하게 감지기 함수를 호출하도록 돕는다. 자동으로 `__FILE__` (파일명)과 `__LINE__` (줄 번호) 정보를 넘겨주어, 데드락 발생 시 정확한 코드 위치를 리포트할 수 있게 한다.

#### 사용 방법 (예시)
이 감지기를 사용하려면, 기존의 락 코드를 다음과 같이 수정해야 한다.  

**[수정 전]**

```cpp
std::mutex myMutex;
// ...
myMutex.lock();
// 임계 영역 (Critical Section)
myMutex.unlock();
```

**[수정 후]**

```cpp
std::mutex myMutex;
// ...
DEADLOCK_DETECTOR_BEFORE_ACQUIRE(&myMutex, "MyMutex");
myMutex.lock();
DEADLOCK_DETECTOR_AFTER_ACQUIRE(&myMutex, "MyMutex");

// 임계 영역 (Critical Section)

myMutex.unlock();
DEADLOCK_DETECTOR_AFTER_RELEASE(&myMutex);
```  

  
### 스마트 락 래퍼
데드락 감지 기능이 내장된 스마트 락 래퍼를 구현한다:

```cpp
/*
 * 템플릿(Template)을 사용하여 다양한 종류의 뮤텍스(예: std::mutex,
 * std::recursive_mutex)를 래핑(wrapping)할 수 있는 디버깅용 뮤텍스 클래스.
 */
template<typename MutexType>
class DebugMutex {
private:
    MutexType mutex_; // 실제 내부에서 사용할 뮤텍스 객체
    std::string name_; // 디버깅 시 식별을 위한 뮤텍스 이름
    
    // 현재 이 뮤텍스를 몇 번 잠갔는지 카운트 (재귀 뮤텍스인 경우 1보다 클 수 있음)
    // mutable: const 멤버 함수에서도 값을 변경할 수 있게 하지만, 여기서는 atomic을 사용함.
    mutable std::atomic<int> lockCount_{0}; 
    
    // 현재 이 뮤텍스를 소유한 스레드의 ID
    mutable std::thread::id ownerThread_;
    
public:
    // 생성자: 디버깅용 이름을 받음 (기본값 "UnnamedMutex")
    // explicit: DebugMutex m = "MyMutex"; 와 같은 암시적 형변환을 방지.
    explicit DebugMutex(const std::string& name = "UnnamedMutex") 
        : name_(name) {}
    
    // 락(lock) 획득
    void lock() {
        // 1. 데드락 감지기: 락 획득 시도 *전*에 알림
        // 락의 주소(&mutex_)와 이름(name_)을 전달
        DEADLOCK_DETECTOR_BEFORE_ACQUIRE(&mutex_, name_);
        
        // 2. 락 대기 시간 측정을 위해 시작 시간 기록
        auto start = std::chrono::high_resolution_clock::now();
        
        // 3. 실제 뮤텍스 락 획득 (이 부분에서 스레드가 대기(block)될 수 있음)
        mutex_.lock();
        
        // 4. 락 획득 성공 후 시간 기록
        auto end = std::chrono::high_resolution_clock::now();
        
        // 5. 디버깅 정보 업데이트 (락 카운트, 소유 스레드 ID)
        lockCount_.fetch_add(1); // 아토믹 연산으로 1 증가
        ownerThread_ = std::this_thread::get_id(); // 현재 스레드 ID 저장
        
        // 6. 락 대기 시간 계산
        auto duration = std::chrono::duration_cast<std::chrono::milliseconds>(end - start);
        if (duration.count() > 100) { // 대기 시간이 100ms를 초과하면
            // 경고 메시지 출력 (성능 문제 가능성)
            std::cout << "⚠️  Long mutex wait: " << name_ << " took " 
                      << duration.count() << "ms\n";
        }
        
        // 7. 데드락 감지기: 락 획득 *후*에 알림
        DEADLOCK_DETECTOR_AFTER_ACQUIRE(&mutex_, name_);
    }
    
    // 락 획득 시도 (non-blocking)
    bool try_lock() {
        // 1. 데드락 감지기: 락 획득 시도 *전*에 알림
        DEADLOCK_DETECTOR_BEFORE_ACQUIRE(&mutex_, name_);
        
        // 2. 실제 뮤텍스 try_lock() 호출 (대기하지 않음)
        if (mutex_.try_lock()) {
            // 3. 락 획득 성공 시 디버깅 정보 업데이트
            lockCount_.fetch_add(1);
            ownerThread_ = std::this_thread::get_id();
            
            // 4. 데드락 감지기: 락 획득 *후*에 알림
            DEADLOCK_DETECTOR_AFTER_ACQUIRE(&mutex_, name_);
            return true; // 락 획득 성공
        }
        
        return false; // 락 획득 실패
    }
    
    // 락(lock) 해제
    void unlock() {
        // 1. 디버깅 정보 업데이트
        lockCount_.fetch_sub(1); // 락 카운트 1 감소
        ownerThread_ = std::thread::id{}; // 소유 스레드 ID 초기화
        
        // 2. 데드락 감지기: 락 해제 *후*에 알림
        // (실제 락 해제 *전*에 호출되어야 함)
        DEADLOCK_DETECTOR_AFTER_RELEASE(&mutex_);
        
        // 3. 실제 뮤텍스 락 해제
        mutex_.unlock();
    }
    
    // --- 통계 정보 제공 함수들 ---
    int GetLockCount() const { return lockCount_.load(); }
    std::thread::id GetOwner() const { return ownerThread_; }
    const std::string& GetName() const { return name_; }
};

// ==========================================================
// 사용 예제: 데드락 감지 기능이 포함된 게임 서버
// ==========================================================
class GameServerWithDeadlockDetection {
private:
    // std::mutex 대신 DebugMutex<std::mutex>를 사용
    DebugMutex<std::mutex> playerMutex_{"PlayerMutex"};
    DebugMutex<std::mutex> worldMutex_{"WorldMutex"};
    DebugMutex<std::mutex> networkMutex_{"NetworkMutex"};
    
    // 서버가 관리하는 데이터 (예시)
    std::map<int, int> playerData_;
    std::vector<int> worldState_;
    std::queue<int> networkQueue_;
    
public:
    // [데드락 유발 함수 1]
    // 락 획득 순서: playerMutex_ -> worldMutex_
    void ProcessPlayerAction(int playerId, int action) {
        // std::lock_guard를 사용하면 생성자에서 lock(), 소멸자에서 unlock()이 자동 호출됨
        std::lock_guard<DebugMutex<std::mutex>> playerLock(playerMutex_);
        std::this_thread::sleep_for(std::chrono::milliseconds(10)); // 데드락 발생 타이밍 조절
        
        {
            std::lock_guard<DebugMutex<std::mutex>> worldLock(worldMutex_);
            // 플레이어 액션 처리
            playerData_[playerId] = action;
            worldState_.push_back(action);
        }
    }
    
    // [데드락 유발 함수 2]
    // 락 획득 순서: worldMutex_ -> playerMutex_ (위 함수와 순서가 반대)
    void UpdateWorld() {
        std::lock_guard<DebugMutex<std::mutex>> worldLock(worldMutex_);
        std::this_thread::sleep_for(std::chrono::milliseconds(15)); // 데드락 발생 타이밍 조절
        
        {
            std::lock_guard<DebugMutex<std::mutex>> playerLock(playerMutex_);
            // 월드 업데이트에 따른 플레이어 상태 변경
            for (auto& [playerId, data] : playerData_) {
                data += 1;
            }
        }
    }
    
    // [안전한 버전 1] - 락 순서 고정
    void ProcessPlayerActionSafe(int playerId, int action) {
        // 항상 playerMutex_ -> worldMutex_ 순서로 획득
        std::lock_guard<DebugMutex<std::mutex>> playerLock(playerMutex_);
        std::lock_guard<DebugMutex<std::mutex>> worldLock(worldMutex_);
        
        playerData_[playerId] = action;
        worldState_.push_back(action);
    }
    
    // [안전한 버전 2] - 락 순서 고정
    void UpdateWorldSafe() {
        // 동일한 락 순서 유지: playerMutex_ -> worldMutex_
        std::lock_guard<DebugMutex<std::mutex>> playerLock(playerMutex_);
        std::lock_guard<DebugMutex<std::mutex>> worldLock(worldMutex_);
        
        for (auto& [playerId, data] : playerData_) {
            data += 1;
        }
    }
    
    // 데드락 시뮬레이션
    void SimulateDeadlock() {
        std::cout << "\n=== Deadlock Simulation ===\n";
        
        // 스레드 1: ProcessPlayerAction (P -> W 락 순서) 실행
        std::thread t1([this]() {
            for (int i = 0; i < 100; ++i) {
                ProcessPlayerAction(i, i * 2);
                std::this_thread::sleep_for(std::chrono::milliseconds(1));
            }
        });
        
        // 스레드 2: UpdateWorld (W -> P 락 순서) 실행
        std::thread t2([this]() {
            for (int i = 0; i < 100; ++i) {
                UpdateWorld();
                std::this_thread::sleep_for(std::chrono::milliseconds(1));
            }
        });
        
        // 두 스레드가 종료될 때까지 대기
        // (데드락 발생 시, 두 스레드 모두 여기서 영원히 종료되지 않음)
        t1.join();
        t2.join();
        
        std::cout << "Deadlock simulation completed\n";
    }
    
    // 락 통계 출력
    void PrintLockStatistics() {
        std::cout << "\n=== Lock Statistics ===\n";
        std::cout << "Player Mutex: " << playerMutex_.GetLockCount() << " locks\n";
        std::cout << "World Mutex: " << worldMutex_.GetLockCount() << " locks\n";
        std::cout << "Network Mutex: " << networkMutex_.GetLockCount() << " locks\n";
    }
};
```
  
이 코드는 C++ 표준 라이브러리의 뮤텍스(예: `std::mutex`)를 감싸는(wrap) `DebugMutex` 템플릿 클래스를 정의하고, 이를 활용하여 실제 데드락(Deadlock) 상황을 시뮬레이션하는 예제를 보여준다.

#### 1. `DebugMutex` 클래스
이 클래스는 일반 뮤텍스에 여러 디버깅 기능을 추가한 래퍼(Wrapper) 클래스다.

  * **템플릿 사용**: `template<typename MutexType>`을 사용하여 `std::mutex`뿐만 아니라 `std::recursive_mutex` 등 다양한 뮤텍스 타입을 지원할 수 있다.
  * **주요 기능**:
    1.  **데드락 감지 연동**: `lock()`, `try_lock()`, `unlock()` 함수 내부에서 이전에 정의한 `DEADLOCK_DETECTOR_...` 매크로를 호출한다. 이를 통해 `DeadlockDetector`가 모든 락의 획득/대기/해제 상태를 실시간으로 추적할 수 있게 한다.
    2.  **긴 대기 시간 경고**: `lock()` 함수가 락을 획득하기까지 100ms 이상 대기한 경우, 콘솔에 경고 메시지를 출력한다. 이는 데드락은 아니지만 심각한 성능 저하(Lock Contention)를 유발하는 지점을 찾는 데 도움을 준다.
    3.  **통계 추적**: `lockCount_` (현재 락 카운트)와 `ownerThread_` (현재 락 소유 스레드) 같은 간단한 통계 정보를 `std::atomic` 변수로 관리한다.

#### 2. `GameServerWithDeadlockDetection` 클래스
이 클래스는 `DebugMutex`를 실제로 사용하는 예제이며, 의도적으로 데드락을 유발하는 시나리오를 포함한다.

  * **뮤텍스 정의**: `playerData_`를 보호하는 `playerMutex_`와 `worldState_`를 보호하는 `worldMutex_`를 `DebugMutex` 타입으로 정의한다.
  * **데드락 유발 시나리오**:
      * `ProcessPlayerAction` 함수: 락을 `playerMutex_` -> `worldMutex_` 순서로 획득한다. (P -> W)
      * `UpdateWorld` 함수: 락을 `worldMutex_` -\> `playerMutex_` 순서로 획득한다. (W -> P)
  * **데드락 발생 과정**:
    1.  스레드 1이 `ProcessPlayerAction`을 실행하여 `playerMutex_`를 획득한다.
    2.  동시에 스레드 2가 `UpdateWorld`를 실행하여 `worldMutex_`를 획득한다.
    3.  스레드 1은 `worldMutex_`를 획득하려 하지만, 스레드 2가 소유 중이므로 대기한다.
    4.  스레드 2는 `playerMutex_`를 획득하려 하지만, 스레드 1이 소유 중이므로 대기한다.
    5.  두 스레드(1과 2)는 서로가 가진 락을 무한히 기다리는 **데드락** 상태에 빠진다.

#### 3\. `SimulateDeadlock` 함수

이 함수는 위에서 설명한 데드락 시나리오를 실제로 실행한다.

  * `t1` 스레드는 `ProcessPlayerAction`을 반복 실행한다.
  * `t2` 스레드는 `UpdateWorld`를 반복 실행한다.
  * 두 스레드가 실행되다가 특정 타이밍(코드 내 `sleep_for`로 조절)에 데드락에 빠지게 된다.
  * `DebugMutex` 내부에서 호출하는 `DeadlockDetector`가 이 순환 대기 상태를 감지하고 콘솔에 데드락 리포트를 출력하게 된다.

#### 4\. 데드락 해결 (Safe 함수)

  * `ProcessPlayerActionSafe`와 `UpdateWorldSafe` 함수는 데드락을 해결한 예시를 보여준다.
  * 해결책은 **락 획득 순서를 항상 동일하게** 맞추는 것이다. 두 함수 모두 `playerMutex_`를 먼저 획득하고 `worldMutex_`를 나중에 획득하도록 (P -\> W) 순서를 통일했다.
  * 이렇게 하면 한 스레드가 두 락을 모두 획득하기 전까지 다른 스레드는 첫 번째 락(`playerMutex_`)에서 대기하게 되므로 순환 대기가 발생하지 않는다.

#### 결론
이 코드는 `DeadlockDetector`를 실제 뮤텍스 래퍼 클래스(`DebugMutex`)에 통합하여, 사용자가 `std::mutex` 대신 `DebugMutex`를 사용하기만 하면 자동으로 데드락 감지 및 성능 모니터링이 가능하도록 만든 실용적인 디버깅 도구의 예시다.

## 10.3 ETW를 이용한 커스텀 프로파일링

### ETW 프로바이더 구현
Event Tracing for Windows(ETW)를 활용하여 게임 서버 특화 이벤트를 추적하는 시스템을 구현한다.

이 코드는 Windows의 고성능 로깅 시스템인 ETW(Event Tracing for Windows)를 활용하여 게임 서버의 주요 이벤트를 추적하는 '프로바이더(Provider)'를 구현한 예제다.

코드는 크게 두 부분으로 나뉜다.

1.  **`GameServerETWProvider`**: ETW와 직접 통신하며 이벤트를 등록하고 전송하는 핵심 로깅 클래스다.
2.  **`GameServerWithETW`**: 실제 게임 서버의 로직을 시뮬레이션하며, `GameServerETWProvider`를 사용하여 다양한 이벤트를 기록하는 예제 클래스다.

  
```cpp
#include <windows.h>      // Windows API 기본 헤더
#include <evntprov.h>     // ETW 프로바이더 API (EventRegister, EventWrite 등)
#include <evntcons.h>     // ETW 컨슈머 API (이 예제에서는 직접 사용 안 함)
#include <iostream>       // 콘솔 출력을 위한 C++ 표준 입출력
#include <string>         // C++ 문자열 클래스
#include <atomic>         // 스레드 안전한 변수(enabled_)를 위한 C++ atomic

// ETW 프로바이더 GUID (실제 사용 시에는 guidgen.exe로 고유 ID 생성 필요)
// 이 GUID는 이 프로바이더를 식별하는 고유한 ID다.
// {12345678-1234-5678-9012-123456789012}
DEFINE_GUID(GameServerProviderGuid,
    0x12345678, 0x1234, 0x5678, 0x90, 0x12, 0x12, 0x34, 0x56, 0x78, 0x90, 0x12);

// 게임 서버 이벤트를 ETW에 전송하는 프로바이더 클래스
class GameServerETWProvider {
private:
    // ETW에 프로바이더를 등록할 때 받는 핸들. 이벤트 쓰기/등록 해제 시 사용된다.
    REGHANDLE registrationHandle_;
    // ETW 세션이 이 프로바이더를 활성화했는지 여부.
    // atomic을 사용하여 여러 스레드에서 안전하게 접근할 수 있다.
    std::atomic<bool> enabled_{false};
    
    // 발생시킬 이벤트의 종류를 식별하는 ID. 
    // 매니페스트(XML) 파일로 정의하는 것이 일반적이나, 여기서는 간단히 enum으로 정의.
    enum class EventId : UCHAR {
        PLAYER_CONNECT = 1,
        PLAYER_DISCONNECT = 2,
        PACKET_RECEIVED = 3,
        PACKET_SENT = 4,
        GAME_LOGIC_START = 5,
        GAME_LOGIC_END = 6,
        DATABASE_QUERY_START = 7,
        DATABASE_QUERY_END = 8,
        MEMORY_ALLOCATION = 9,
        MEMORY_DEALLOCATION = 10,
        PERFORMANCE_COUNTER = 11
    };
    
public:
    // 생성자: 핸들을 0으로 초기화하고 Initialize()를 호출한다.
    GameServerETWProvider() : registrationHandle_(0) {
        Initialize();
    }
    
    // 소멸자: Cleanup()을 호출하여 리소스를 정리한다. (RAII 패턴)
    ~GameServerETWProvider() {
        Cleanup();
    }
    
    // ETW 프로바이더를 Windows 시스템에 등록한다.
    bool Initialize() {
        // EventRegister API를 호출하여 이 프로바이더(GUID)를 ETW에 등록한다.
        ULONG status = EventRegister(
            &GameServerProviderGuid, // 등록할 프로바이더의 GUID
            nullptr,                 // 활성화/비활성화 콜백 (사용 안 함)
            nullptr,                 // 콜백에 전달될 컨텍스트 (사용 안 함)
            &registrationHandle_     // 등록 성공 시 핸들을 받을 변수
        );
        
        if (status == ERROR_SUCCESS) {
            enabled_.store(true); // 등록 성공 시 활성화 상태로 설정
            std::cout << "✅ ETW Provider registered successfully\n";
            return true;
        } else {
            std::cout << "❌ Failed to register ETW Provider. Error: " << status << "\n";
            return false;
        }
    }
    
    // ETW 프로바이더 등록을 해제한다.
    void Cleanup() {
        if (registrationHandle_ != 0) {
            EventUnregister(registrationHandle_); // 핸들을 사용하여 등록 해제
            registrationHandle_ = 0;
            enabled_.store(false);
        }
    }
    
    // --- 이벤트 로깅 함수들 ---

    // 플레이어 연결 이벤트를 로깅한다.
    void LogPlayerConnect(int playerId, const std::string& playerName, 
                          const std::string& ipAddress) {
        // 프로바이더가 활성화되지 않았으면 (즉, 아무도 이벤트를 수집하고 있지 않으면) 아무것도 안 함.
        // 이것이 ETW의 큰 장점 (성능 저하 없음).
        if (!enabled_.load()) return;
        
        // ETW에 보낼 데이터 조각들을 설명하는 구조체 배열 (데이터 3개)
        EVENT_DATA_DESCRIPTOR eventData[3];
        
        // 1. 플레이어 ID (int)
        EventDataDescCreate(&eventData[0], &playerId, sizeof(playerId));
        
        // 2. 플레이어 이름 (string -> wstring)
        // ETW는 기본적으로 유니코드(wchar_t*) 문자열을 사용한다.
        std::wstring wPlayerName(playerName.begin(), playerName.end());
        EventDataDescCreate(&eventData[1], wPlayerName.c_str(), 
                            static_cast<ULONG>((wPlayerName.length() + 1) * sizeof(wchar_t))); // 널 종료 문자 포함
        
        // 3. IP 주소 (string -> wstring)
        std::wstring wIpAddress(ipAddress.begin(), ipAddress.end());
        EventDataDescCreate(&eventData[2], wIpAddress.c_str(),
                            static_cast<ULONG>((wIpAddress.length() + 1) * sizeof(wchar_t)));
        
        // EventWrite를 호출하여 실제 이벤트를 전송한다.
        EventWrite(registrationHandle_, // 등록 핸들
                   CreateEventDescriptor(EventId::PLAYER_CONNECT, TRACE_LEVEL_INFORMATION), // 이벤트 설명자
                   3, eventData); // 데이터 개수 및 데이터 디스크립터 배열
    }
    
    // 패킷 수신 이벤트를 로깅한다.
    void LogPacketReceived(int playerId, int packetType, int packetSize, 
                           ULONGLONG processingTimeUs) {
        if (!enabled_.load()) return;
        
        EVENT_DATA_DESCRIPTOR eventData[4]; // 데이터 4개
        
        // int, int, int, ULONGLONG 타입은 변환 없이 바로 전달 가능
        EventDataDescCreate(&eventData[0], &playerId, sizeof(playerId));
        EventDataDescCreate(&eventData[1], &packetType, sizeof(packetType));
        EventDataDescCreate(&eventData[2], &packetSize, sizeof(packetSize));
        EventDataDescCreate(&eventData[3], &processingTimeUs, sizeof(processingTimeUs));
        
        EventWrite(registrationHandle_,
                   CreateEventDescriptor(EventId::PACKET_RECEIVED, TRACE_LEVEL_VERBOSE), // 레벨을 Verbose로 설정
                   4, eventData);
    }
    
    // 게임 로직 성능 측정 이벤트를 로깅한다.
    void LogGameLogicPerformance(const std::string& functionName, 
                                 ULONGLONG durationUs, int objectCount) {
        if (!enabled_.load()) return;
        
        EVENT_DATA_DESCRIPTOR eventData[3];
        
        // 1. 함수 이름 (wstring)
        std::wstring wFunctionName(functionName.begin(), functionName.end());
        EventDataDescCreate(&eventData[0], wFunctionName.c_str(),
                            static_cast<ULONG>((wFunctionName.length() + 1) * sizeof(wchar_t)));
        // 2. 소요 시간 (마이크로초)
        EventDataDescCreate(&eventData[1], &durationUs, sizeof(durationUs));
        // 3. 처리된 객체 수
        EventDataDescCreate(&eventData[2], &objectCount, sizeof(objectCount));
        
        EventWrite(registrationHandle_,
                   CreateEventDescriptor(EventId::GAME_LOGIC_END, TRACE_LEVEL_INFORMATION),
                   3, eventData);
    }
    
    // 메모리 할당 추적 이벤트를 로깅한다.
    void LogMemoryAllocation(void* address, size_t size, const std::string& category) {
        if (!enabled_.load()) return;
        
        EVENT_DATA_DESCRIPTOR eventData[3];
        
        // 1. 할당된 주소 (포인터를 64비트 정수로 변환)
        ULONGLONG addr = reinterpret_cast<ULONGLONG>(address);
        EventDataDescCreate(&eventData[0], &addr, sizeof(addr));
        // 2. 할당 크기
        EventDataDescCreate(&eventData[1], &size, sizeof(size));
        // 3. 할당 카테고리 (wstring)
        std::wstring wCategory(category.begin(), category.end());
        EventDataDescCreate(&eventData[2], wCategory.c_str(),
                            static_cast<ULONG>((wCategory.length() + 1) * sizeof(wchar_t)));
        
        EventWrite(registrationHandle_,
                   CreateEventDescriptor(EventId::MEMORY_ALLOCATION, TRACE_LEVEL_VERBOSE),
                   3, eventData);
    }
    
    // 주기적인 성능 카운터 값을 로깅한다.
    void LogPerformanceCounter(const std::string& counterName, double value) {
        if (!enabled_.load()) return;
        
        EVENT_DATA_DESCRIPTOR eventData[2];
        
        // 1. 카운터 이름 (wstring)
        std::wstring wCounterName(counterName.begin(), counterName.end());
        EventDataDescCreate(&eventData[0], wCounterName.c_str(),
                            static_cast<ULONG>((wCounterName.length() + 1) * sizeof(wchar_t)));
        // 2. 카운터 값 (double)
        EventDataDescCreate(&eventData[1], &value, sizeof(value));
        
        EventWrite(registrationHandle_,
                   CreateEventDescriptor(EventId::PERFORMANCE_COUNTER, TRACE_LEVEL_INFORMATION),
                   2, eventData);
    }
    
private:
    // EventWrite에 필요한 EVENT_DESCRIPTOR 구조체를 생성하는 헬퍼 함수
    EVENT_DESCRIPTOR CreateEventDescriptor(EventId id, UCHAR level) {
        EVENT_DESCRIPTOR desc = {0};
        desc.Id = static_cast<USHORT>(id); // 이벤트 ID (Enum 값)
        desc.Version = 1;                  // 이벤트 스키마 버전
        desc.Channel = 16;                 // 채널 (16은 Application 채널을 의미할 수 있음)
        desc.Level = level;                // 이벤트 심각도 (Information, Verbose, Warning, Error 등)
        desc.Opcode = 0;                   // 작업 코드 (사용 안 함)
        desc.Task = 0;                     // 작업 ID (사용 안 함)
        desc.Keyword = 0;                  // 키워드 (필터링용, 사용 안 함)
        return desc;
    }
};

// ETW 로깅이 통합된 가상의 게임 서버 클래스
class GameServerWithETW {
private:
    GameServerETWProvider etwProvider_; // 위에서 정의한 ETW 프로바이더 인스턴스
    std::atomic<int> connectedPlayers_{0};
    std::atomic<uint64_t> totalPacketsReceived_{0};
    std::atomic<uint64_t> totalPacketsSent_{0};
    
public:
    // 특정 코드 블록의 실행 시간을 자동으로 측정하고 ETW로 로깅하는 RAII 타이머 클래스
    class PerformanceTimer {
    private:
        std::chrono::high_resolution_clock::time_point startTime_; // 시작 시간
        std::string name_; // 측정할 대상(함수) 이름
        GameServerETWProvider* provider_; // 로깅을 수행할 프로바이더 포인터
        
    public:
        // 생성자: 이름과 프로바이더를 받고 시작 시간을 기록
        PerformanceTimer(const std::string& name, GameServerETWProvider* provider)
            : name_(name), provider_(provider) {
            startTime_ = std::chrono::high_resolution_clock::now();
        }
        
        // 소멸자: 객체가 파괴될 때 (즉, 범위를 벗어날 때) 자동으로 호출됨
        ~PerformanceTimer() {
            auto endTime = std::chrono::high_resolution_clock::now();
            // 경과 시간을 마이크로초(us) 단위로 계산
            auto duration = std::chrono::duration_cast<std::chrono::microseconds>(
                endTime - startTime_);
            
            // 프로바이더가 유효하면 성능 로그를 전송
            if (provider_) {
                provider_->LogGameLogicPerformance(name_, duration.count(), 1); // objectCount는 1로 고정
            }
        }
    };
    
    // 플레이어 연결 처리 로직
    void ProcessPlayerConnection(int playerId, const std::string& playerName,
                                 const std::string& ipAddress) {
        // 이 함수가 실행되는 동안의 시간을 자동으로 측정하고 로깅
        PerformanceTimer timer("ProcessPlayerConnection", &etwProvider_);
        
        // 실제 연결 처리 로직 시뮬레이션 (10ms 대기)
        std::this_thread::sleep_for(std::chrono::milliseconds(10));
        
        connectedPlayers_.fetch_add(1);
        // ETW 이벤트 발생
        etwProvider_.LogPlayerConnect(playerId, playerName, ipAddress);
        
        std::cout << "Player " << playerName << " connected from " << ipAddress << "\n";
    }
    
    // 수신 패킷 처리 로직
    void ProcessIncomingPacket(int playerId, int packetType, const std::vector<char>& data) {
        // 이 함수는 수동으로 시간 측정 (시작)
        auto startTime = std::chrono::high_resolution_clock::now();
        
        // 패킷 처리 시뮬레이션
        switch (packetType) {
            case 1: ProcessMovePacket(playerId, data); break;
            case 2: ProcessChatPacket(playerId, data); break;
            case 3: ProcessActionPacket(playerId, data); break;
            default: std::this_thread::sleep_for(std::chrono::microseconds(100)); break;
        }
        
        // 시간 측정 (종료)
        auto endTime = std::chrono::high_resolution_clock::now();
        auto duration = std::chrono::duration_cast<std::chrono::microseconds>(endTime - startTime);
        
        totalPacketsReceived_.fetch_add(1);
        // ETW 이벤트 발생 (처리 시간 포함)
        etwProvider_.LogPacketReceived(playerId, packetType, static_cast<int>(data.size()), 
                                       duration.count());
    }
    
    // 주기적인 게임 로직 업데이트
    void UpdateGameLogic() {
        // RAII 타이머로 함수 실행 시간 측정
        PerformanceTimer timer("UpdateGameLogic", &etwProvider_);
        
        // 게임 로직 업데이트 시뮬레이션 (5ms 대기)
        std::this_thread::sleep_for(std::chrono::milliseconds(5));
        
        // 현재 서버 상태를 성능 카운터로 ETW에 로깅
        etwProvider_.LogPerformanceCounter("ConnectedPlayers", connectedPlayers_.load());
        etwProvider_.LogPerformanceCounter("PacketsPerSecond", 
                                           static_cast<double>(totalPacketsReceived_.load()));
        // (참고: PacketsPerSecond는 실제로는 누적 값이므로, 분석 시 delta 계산 필요)
    }
    
    // ETW 로깅이 포함된 메모리 할당 래퍼(Wrapper)
    void* AllocateGameMemory(size_t size, const std::string& category) {
        void* ptr = malloc(size); // 실제 메모리 할당
        
        if (ptr) {
            // ETW에 메모리 할당 이벤트 로깅
            etwProvider_.LogMemoryAllocation(ptr, size, category);
        }
        
        return ptr;
    }
    
    // ETW 로깅이 포함된 메모리 해제 래퍼
    void FreeGameMemory(void* ptr) {
        if (ptr) {
            // ETW에 메모리 해제 이벤트 로깅 (size=0, category="FREE")
            etwProvider_.LogMemoryAllocation(ptr, 0, "FREE");
            free(ptr); // 실제 메모리 해제
        }
    }
    
private:
    // 패킷 처리 함수 시뮬레이션 (단순히 대기)
    void ProcessMovePacket(int playerId, const std::vector<char>& data) {
        std::this_thread::sleep_for(std::chrono::microseconds(200));
    }
    
    void ProcessChatPacket(int playerId, const std::vector<char>& data) {
        std::this_thread::sleep_for(std::chrono::microseconds(150));
    }
    
    void ProcessActionPacket(int playerId, const std::vector<char>& data) {
        std::this_thread::sleep_for(std::chrono::microseconds(500));
    }
    
public:
    // ETW 로깅 데모 실행 함수
    void RunETWDemo() {
        std::cout << "\n=== ETW Logging Demo ===\n";
        // ETW 추적을 시작하는 방법을 사용자에게 안내
        std::cout << "ETW 세션을 시작하려면 관리자 권한으로 다음 명령어를 실행하세요:\n";
        std::cout << "wpr.exe -start custom.wprp -filemode\n";
        std::cout << "또는\n";
        std::cout << "xperf.exe -start GameServerTrace -on {12345678-1234-5678-9012-123456789012}\n\n";
        
        // 플레이어 연결 시뮬레이션
        for (int i = 1; i <= 5; ++i) {
            ProcessPlayerConnection(i, "Player" + std::to_string(i), 
                                    "192.168.1." + std::to_string(100 + i));
        }
        
        // 패킷 처리 시뮬레이션
        for (int i = 0; i < 100; ++i) {
            int playerId = (i % 5) + 1;
            int packetType = (i % 3) + 1;
            std::vector<char> data(64 + (i % 128), 'A' + (i % 26));
            
            ProcessIncomingPacket(playerId, packetType, data);
            
            // 20 패킷마다 게임 로직 업데이트 및 성능 카운터 로깅
            if (i % 20 == 0) {
                UpdateGameLogic();
            }
        }
        
        // 메모리 할당/해제 시뮬레이션
        std::vector<void*> allocations;
        for (int i = 0; i < 10; ++i) {
            void* ptr = AllocateGameMemory(1024 * (i + 1), "GameObjects");
            allocations.push_back(ptr);
        }
        
        for (void* ptr : allocations) {
            FreeGameMemory(ptr);
        }
        
        // ETW 추적을 중지하고 파일로 저장하는 방법 안내
        std::cout << "\nETW 세션을 중지하려면:\n";
        std::cout << "wpr.exe -stop gameserver_trace.etl\n";
        std::cout << "또는\n";
        std::cout << "xperf.exe -stop GameServerTrace -d gameserver_trace.etl\n\n";
        
        // 저장된 .etl 파일을 분석하는 방법 안내 (Windows Performance Analyzer)
        std::cout << "분석을 위해:\n";
        std::cout << "wpa.exe gameserver_trace.etl\n";
    }
};
```

#### `GameServerETWProvider` 클래스
이 클래스는 ETW 프로바이더의 핵심 기능을 캡슐화한다.

1.  **GUID 정의**: `DEFINE_GUID` 매크로를 사용하여 이 프로바이더를 시스템 전체에서 식별할 수 있는 고유 ID(GUID)를 정의한다. ETW 컨트롤러(xperf, wpr 등)는 이 GUID를 사용하여 특정 프로바이더의 이벤트를 수집한다.
2.  **초기화 및 정리 (Initialize, Cleanup)**:
      * `Initialize`는 `EventRegister` API를 호출하여 프로바이더를 ETW 시스템에 등록한다. 성공 시 `registrationHandle_`에 유효한 핸들을 받는다.
      * `Cleanup`은 프로그램 종료 시 `EventUnregister`를 호출하여 프로바이더 등록을 해제하고 리소스를 정리한다.
      * 생성자와 소멸자에서 이 함수들을 호출하여 RAII(Resource Acquisition Is Initialization) 패턴을 따른다.
3.  **이벤트 ID 정의**: `EventId` 열거형(enum)은 발생시킬 이벤트의 종류를 숫자로 정의한다. (예: `PLAYER_CONNECT`는 1번)
4.  **이벤트 로깅 함수 (Log\*)**:
      * `LogPlayerConnect`, `LogPacketReceived` 등 서버에서 발생할 수 있는 구체적인 이벤트들을 로깅하는 함수들이다.
      * 가장 먼저 `enabled_.load()`를 확인한다. 만약 ETW 세션이 이 프로바이더의 이벤트를 수집하고 있지 않다면, 이 값은 `false`가 되며 함수는 즉시 반환된다. 이 '조기 반환' 덕분에 이벤트 수집을 안 할 때의 성능 부하(오버헤드)가 거의 0에 가깝다.
      * 이벤트에 포함될 데이터(예: 플레이어 ID, IP 주소)를 `EVENT_DATA_DESCRIPTOR` 배열로 구성한다. 이 구조체는 ETW에게 데이터의 메모리 주소와 크기를 알려준다.
      * `std::string`은 `std::wstring`(유니코드)으로 변환해야 한다.
      * `EventWrite` API를 호출하여 등록 핸들, 이벤트 설명자(ID, 레벨 등), 그리고 데이터 디스크립터 배열을 전달함으로써 이벤트를 ETW 시스템에 전송한다.
5.  **`CreateEventDescriptor` 헬퍼 함수**:
      * `EventWrite`에 필요한 `EVENT_DESCRIPTOR` 구조체를 쉽게 만들도록 도와준다.
      * 이벤트 ID, 버전, 채널(Application), 레벨(Information, Verbose 등)을 설정한다.

#### `GameServerWithETW` 클래스
이 클래스는 `GameServerETWProvider`를 실제로 사용하는 가상의 게임 서버다.

1.  **`PerformanceTimer` 중첩 클래스**:
      * RAII 패턴을 활용한 매우 유용한 타이머 클래스다.
      * 객체가 생성될 때(함수나 범위의 시작) 현재 시간을 기록한다.
      * 객체가 소멸될 때(함수나 범위의 끝) 자동으로 경과 시간을 계산하여 `LogGameLogicPerformance` 이벤트를 발생시킨다.
      * `ProcessPlayerConnection`이나 `UpdateGameLogic` 함수 맨 위에 `PerformanceTimer timer(...)` 한 줄만 추가하면, 해당 함수의 실행 시간을 별도 코드 없이 자동으로 로깅할 수 있다.
2.  **게임 로직 함수 (Process\*, Update\*)**:
      * `ProcessPlayerConnection`에서는 플레이어 연결 처리를 시뮬레이션하고, `etwProvider_.LogPlayerConnect`를 호출하여 관련 정보를 ETW에 기록한다.
      * `ProcessIncomingPacket`에서는 패킷 처리 시간을 수동으로 측정한 뒤 `etwProvider_.LogPacketReceived`를 호출하여 처리 시간(duration)을 ETW에 기록한다.
      * `UpdateGameLogic`에서는 주기적으로 현재 서버 상태(연결된 플레이어 수 등)를 `etwProvider_.LogPerformanceCounter`로 기록한다.
3.  **메모리 래퍼 (Allocate\*, Free\*)**:
      * `malloc`과 `free`를 감싸는 래퍼 함수를 제공한다.
      * 메모리를 할당하거나 해제할 때 `etwProvider_.LogMemoryAllocation`을 호출하여 할당 주소, 크기, 카테고리를 ETW에 기록한다. 이는 메모리 누수나 단편화를 분석하는 데 매우 유용하다.
4.  **`RunETWDemo` 함수**:
      * 이 모든 기능을 시뮬레이션하는 메인 함수다.
      * ETW 추적을 시작하고 중지하는 명령어(xperf.exe, wpr.exe)와 분석 도구(wpa.exe) 사용법을 콘솔에 출력하여 사용자가 실제로 이벤트를 캡처하고 분석할 수 있도록 안내한다.
      * 플레이어 연결, 패킷 처리, 게임 로직 업데이트, 메모리 할당/해제를 순차적으로 시뮬레이션하며 정의된 ETW 이벤트들을 발생시킨다.
  


### 실시간 성능 모니터링
아래 코드는 게임 서버의 성능을 실시간으로 모니터링하고, 설정된 임계값을 넘으면 알람을 발생시키는 `RealTimePerformanceMonitor` 클래스를 구현한 것이다.
  
```cpp
#include <windows.h>      // GlobalMemoryStatusEx, SetThreadDescription 등 Windows API
#include <evntprov.h>     // ETW 프로바이더 (이전 코드에서 정의됨)
#include <iostream>       // 콘솔 출력
#include <string>         // C++ 문자열
#include <atomic>         // 스레드 안전한 원자적 연산
#include <thread>         // 스레드 생성 및 관리
#include <chrono>         // 시간 관리 (sleep_for)
#include <cmath>          // sin 함수 (CPU 시뮬레이션용)
#include <iomanip>        // 출력 포맷팅 (setprecision)

// (GameServerETWProvider 클래스는 이전 코드에 정의되어 있다고 가정)

// 실시간 성능 모니터링 클래스
class RealTimePerformanceMonitor {
private:
    // 서버 성능 지표를 저장하는 구조체
    struct PerformanceMetrics {
        // 모든 멤버는 std::atomic을 사용하여 여러 스레드에서 
        // 잠금(lock) 없이 안전하게 읽고 쓸 수 있다.
        std::atomic<double> cpuUsage{0.0};              // CPU 사용률 (%)
        std::atomic<uint64_t> memoryUsage{0};           // 메모리 사용량 (Bytes)
        std::atomic<uint64_t> networkBytesSent{0};      // 누적 네트워크 전송량
        std::atomic<uint64_t> networkBytesReceived{0};  // 누적 네트워크 수신량
        std::atomic<int> activeConnections{0};          // 현재 활성 연결 수
        std::atomic<double> averageLatency{0.0};        // 평균 지연 시간 (ms)
        std::atomic<uint64_t> packetsPerSecond{0};      // 초당 패킷 수 (이 예제에서는 직접 업데이트 안 됨)
    };
    
    PerformanceMetrics metrics_; // 성능 지표 인스턴스
    GameServerETWProvider* etwProvider_; // ETW 로깅을 위한 프로바이더 포인터
    std::atomic<bool> monitoring_{true}; // 모니터링 루프 제어 플래그
    
public:
    // 생성자: ETW 프로바이더를 받고 모니터링 스레드를 시작한다.
    RealTimePerformanceMonitor(GameServerETWProvider* provider) 
        : etwProvider_(provider) {
        
        // 별도의 스레드에서 모니터링 루프를 실행한다.
        std::thread monitoringThread([this]() {
            MonitoringLoop();
        });
        // 스레드를 분리(detach)한다. 
        // 이렇게 하면 메인 스레드가 이 스레드의 종료를 기다릴(join) 필요가 없다.
        // 스레드는 monitoring_ 플래그가 false가 되거나 프로그램이 종료될 때까지 실행된다.
        monitoringThread.detach();
    }
    
    // 외부에서 메트릭을 업데이트하기 위한 함수 (Push 방식)
    void UpdateMetrics(const std::string& metricName, double value) {
        // 메트릭 이름에 따라 적절한 atomic 변수를 업데이트한다.
        if (metricName == "CPU") {
            metrics_.cpuUsage.store(value);
        } else if (metricName == "Memory") {
            metrics_.memoryUsage.store(static_cast<uint64_t>(value));
        } else if (metricName == "Latency") {
            metrics_.averageLatency.store(value);
        } else if (metricName == "Connections") {
            metrics_.activeConnections.store(static_cast<int>(value));
        }
        
        // ETW로도 메트릭을 전송한다.
        if (etwProvider_) {
            etwProvider_->LogPerformanceCounter(metricName, value);
        }
    }
    
private:
    // 모니터링 스레드에서 실행되는 메인 루프
    void MonitoringLoop() {
        // 디버거(Visual Studio, xperf 등)에서 이 스레드를 쉽게 식별할 수 있도록 이름을 설정한다.
        SetThreadDescription(GetCurrentThread(), L"PerformanceMonitor");
        
        // monitoring_ 플래그가 true인 동안 계속 반복한다.
        while (monitoring_.load()) {
            // 시스템 메트릭 수집 (Pull 방식)
            CollectSystemMetrics();
            
            // 수집된 메트릭을 기준으로 알람 조건 확인
            CheckAlarms();
            
            // 1초마다 반복
            std::this_thread::sleep_for(std::chrono::seconds(1));
        }
    }
    
    // 시스템 레벨의 메트릭을 수집하는 함수
    void CollectSystemMetrics() {
        // [시뮬레이션] 실제 CPU 사용률 측정
        // (실제 환경에서는 PDH(Performance Data Helper) API나 WMI를 사용해야 한다)
        static int cpuCounter = 0;
        double cpuUsage = 20.0 + sin(cpuCounter * 0.1) * 30.0; // 20-50% 사이에서 변동
        metrics_.cpuUsage.store(cpuUsage);
        
        // [실제] 시스템 메모리 사용률 측정 (Windows API 사용)
        MEMORYSTATUSEX memStatus = {0};
        memStatus.dwLength = sizeof(memStatus);
        if (GlobalMemoryStatusEx(&memStatus)) {
            // 전체 물리 메모리 - 사용 가능한 물리 메모리 = 사용 중인 메모리
            uint64_t usedMemory = memStatus.ullTotalPhys - memStatus.ullAvailPhys;
            metrics_.memoryUsage.store(usedMemory);
        }
        
        // [시뮬레이션] 네트워크 메트릭 (단순히 값 증가)
        metrics_.networkBytesSent.fetch_add(1024 * 100); // 100KB/s
        metrics_.networkBytesReceived.fetch_add(1024 * 80); // 80KB/s
        
        cpuCounter++;
    }
    
    // 설정된 임계값을 기준으로 알람을 확인하는 함수
    void CheckAlarms() {
        // CPU 사용률 알람 (80% 초과 시)
        if (metrics_.cpuUsage.load() > 80.0) {
            TriggerAlarm("HIGH_CPU", metrics_.cpuUsage.load());
        }
        
        // 메모리 사용률 알람 (2GB 초과 시)
        if (metrics_.memoryUsage.load() > 1024LL * 1024 * 1024 * 2) { 
            TriggerAlarm("HIGH_MEMORY", static_cast<double>(metrics_.memoryUsage.load()));
        }
        
        // 레이턴시 알람 (100ms 초과 시)
        if (metrics_.averageLatency.load() > 100.0) { 
            TriggerAlarm("HIGH_LATENCY", metrics_.averageLatency.load());
        }
    }
    
    // 알람을 발생시키는 함수
    void TriggerAlarm(const std::string& alarmType, double value) {
        // 콘솔에 알람 출력
        std::cout << "🚨 ALARM: " << alarmType << " = " << value << "\n";
        
        // ETW에도 알람 이벤트를 로깅한다. (접두사 "ALARM_" 사용)
        if (etwProvider_) {
            etwProvider_->LogPerformanceCounter("ALARM_" + alarmType, value);
        }
    }
    
public:
    // 현재 메트릭 상태를 콘솔에 대시보드 형태로 출력한다.
    void PrintDashboard() {
        std::cout << "\n=== Real-Time Performance Dashboard ===\n";
        std::cout << "CPU Usage:        " << std::fixed << std::setprecision(1) 
                  << metrics_.cpuUsage.load() << "%\n";
        std::cout << "Memory Usage:     " << (metrics_.memoryUsage.load() / 1024 / 1024) << " MB\n";
        std::cout << "Active Connections: " << metrics_.activeConnections.load() << "\n";
        std::cout << "Average Latency:  " << std::fixed << std::setprecision(2) 
                  << metrics_.averageLatency.load() << " ms\n";
        std::cout << "Network TX:       " << (metrics_.networkBytesSent.load() / 1024) << " KB\n";
        std::cout << "Network RX:       " << (metrics_.networkBytesReceived.load() / 1024) << " KB\n";
        
        // CPU 사용률을 ASCII 그래프로 표시
        PrintCPUGraph();
    }
    
private:
    // CPU 사용률을 시각적인 ASCII 그래프로 출력하는 헬퍼 함수
    void PrintCPUGraph() {
        double cpu = metrics_.cpuUsage.load();
        int bars = static_cast<int>(cpu / 5); // 5%당 1개의 바로 표시 (총 20칸)
        
        std::cout << "CPU Graph: [";
        for (int i = 0; i < 20; ++i) {
            if (i < bars) {
                if (i < 10) std::cout << "█";      // 0-50%: 정상 (█)
                else if (i < 16) std::cout << "▓"; // 50-80%: 주의 (▓)
                else std::cout << "▒";             // 80-100%: 경고 (▒)
            } else {
                std::cout << "░"; // 빈 칸
            }
        }
        std::cout << "] " << std::fixed << std::setprecision(1) << cpu << "%\n";
    }
};
```

`RealTimePerformanceMonitor` 클래스는 서버의 주요 성능 지표를 실시간으로 수집하고 모니터링하는 역할을 한다.

#### 1\. 주요 구성 요소

  * **`PerformanceMetrics` 구조체**:

      * 서버의 핵심 성능 지표(CPU, 메모리, 네트워크, 연결 수, 지연 시간)를 저장한다.
      * 모든 멤버 변수가 `std::atomic`으로 선언되어 있다. 이는 여러 스레드(모니터링 스레드, 메인 게임 루프 스레드 등)가 동시에 이 값들에 접근(읽기/쓰기)할 때, 데이터가 깨지지 않도록(Race Condition 방지) 보장한다.

  * **백그라운드 모니터링 스레드**:

      * `RealTimePerformanceMonitor` 객체가 생성될 때, 생성자에서 `std::thread`를 생성하여 `MonitoringLoop` 함수를 즉시 실행시킨다.
      * 스레드는 `detach()`를 통해 분리되어, 메인 스레드와 독립적으로 백그라운드에서 계속 실행된다.
      * `SetThreadDescription` API를 호출하여 디버깅 시 이 스레드의 용도를 쉽게 파악할 수 있도록 "PerformanceMonitor"라는 이름을 부여한다.

#### 2\. 핵심 동작

  * **`MonitoringLoop` (Pull 방식)**:

      * `monitoring_` 플래그가 `true`인 동안 1초마다 `CollectSystemMetrics`와 `CheckAlarms`를 반복 호출한다.
      * `CollectSystemMetrics`:
          * `GlobalMemoryStatusEx` 같은 Windows API를 호출하여 시스템의 *실제* 메모리 사용량을 가져온다 (Pull).
          * (참고) 이 코드에서 CPU와 네트워크 사용량은 `sin` 함수와 `fetch_add`를 사용한 *시뮬레이션*이다. 실제 환경에서는 CPU 사용률을 얻기 위해 PDH(Performance Data Helper) API 등을 사용해야 한다.
      * `CheckAlarms`:
          * 수집된 메트릭 값이 미리 정해진 임계값(예: CPU 80% 초과)을 넘었는지 확인한다.
          * 임계값을 넘으면 `TriggerAlarm` 함수를 호출한다.
      * `TriggerAlarm`:
          * 콘솔에 "🚨 ALARM" 메시지를 출력한다.
          * `etwProvider_`를 통해 "ALARM\_HIGH\_CPU"와 같은 특별한 이름으로 ETW 이벤트를 로깅하여, 나중에 성능 분석(WPA) 시 알람이 발생한 시점을 쉽게 인지할 수 있게 한다.

  * **`UpdateMetrics` (Push 방식)**:

      * `MonitoringLoop`가 주기적으로 수집하는 데이터(CPU, 메모리)와 달리, "평균 지연 시간(Latency)"이나 "활성 연결 수(Connections)"처럼 다른 모듈에서 비동기적으로 계산되는 값들이 있다.
      * 이런 값들은 해당 모듈이 `UpdateMetrics` 함수를 직접 호출하여 모니터에 값을 *밀어넣는(Push)* 방식으로 업데이트한다.
      * 이 함수는 값을 업데이트함과 동시에 `etwProvider_`를 호출하여 ETW에도 실시간으로 로깅한다.

  * **`PrintDashboard`**:

      * 현재 `metrics_`에 저장된 최신 값들을 읽어와 콘솔에 보기 좋은 대시보드 형태로 출력한다.
      * `PrintCPUGraph` 헬퍼 함수는 CPU 사용률을 ASCII 문자를 이용한 막대그래프로 시각화하여 보여준다.

```cpp
class MemoryLeakDetector {
private:
    struct AllocationInfo {
        size_t size;
        std::string file;
        int line;
        std::chrono::steady_clock::time_point timestamp;
        std::thread::id threadId;
    };
    
    std::mutex allocationMutex_;
    std::unordered_map<void*, AllocationInfo> allocations_;
    std::atomic<size_t> totalAllocated_{0};
    std::atomic<size_t> totalFreed_{0};
    std::atomic<int> allocationCount_{0};
    
public:
    static MemoryLeakDetector& GetInstance() {
        static MemoryLeakDetector instance;
        return instance;
    }
    
    void RecordAllocation(void* ptr, size_t size, const char* file, int line) {
        std::lock_guard<std::mutex> lock(allocationMutex_);
        
        allocations_[ptr] = {
            size, 
            file ? file : "unknown", 
            line,
            std::chrono::steady_clock::now(),
            std::this_thread::get_id()
        };
        
        totalAllocated_.fetch_add(size);
        allocationCount_.fetch_add(1);
    }
    
    void RecordDeallocation(void* ptr) {
        std::lock_guard<std::mutex> lock(allocationMutex_);
        
        auto it = allocations_.find(ptr);
        if (it != allocations_.end()) {
            totalFreed_.fetch_add(it->second.size);
            allocations_.erase(it);
            allocationCount_.fetch_sub(1);
        }
    }
    
    void PrintLeakReport() {
        std::lock_guard<std::mutex> lock(allocationMutex_);
        
        std::cout << "\n=== Memory Leak Report ===\n";
        std::cout << "Total allocated: " << totalAllocated_.load() << " bytes\n";
        std::cout << "Total freed: " << totalFreed_.load() << " bytes\n";
        std::cout << "Current leaks: " << (totalAllocated_.load() - totalFreed_.load()) << " bytes\n";
        std::cout << "Outstanding allocations: " << allocations_.size() << "\n\n";
        
        if (!allocations_.empty()) {
            std::cout << "🔍 Leak Details:\n";
            
            // 크기별로 정렬
            std::vector<std::pair<void*, AllocationInfo>> sortedLeaks(
                allocations_.begin(), allocations_.end());
            
            std::sort(sortedLeaks.begin(), sortedLeaks.end(),
                [](const auto& a, const auto& b) {
                    return a.second.size > b.second.size;
                });
            
            int count = 0;
            for (const auto& [ptr, info] : sortedLeaks) {
                if (count++ >= 10) break; // 상위 10개만 표시
                
                auto duration = std::chrono::steady_clock::now() - info.timestamp;
                auto seconds = std::chrono::duration_cast<std::chrono::seconds>(duration).count();
                
                std::cout << "  " << ptr << ": " << info.size << " bytes"
                          << " (" << info.file << ":" << info.line << ")"
                          << " [" << seconds << "s ago]\n";
            }
        }
    }
    
    // 주기적으로 메모리 상태 확인
    void StartPeriodicCheck() {
        std::thread checkThread([this]() {
            while (true) {
                std::this_thread::sleep_for(std::chrono::seconds(30));
                
                size_t currentUsage = totalAllocated_.load() - totalFreed_.load();
                if (currentUsage > 100 * 1024 * 1024) { // 100MB 이상
                    std::cout << "⚠️  High memory usage detected: " 
                              << (currentUsage / 1024 / 1024) << " MB\n";
                    PrintLeakReport();
                }
            }
        });
        checkThread.detach();
    }
};
```

```cpp
class PerformanceBottleneckAnalyzer {
private:
    struct FunctionProfile {
        std::string name;
        std::atomic<uint64_t> callCount{0};
        std::atomic<uint64_t> totalTime{0}; // 마이크로초
        std::atomic<uint64_t> maxTime{0};
        std::atomic<uint64_t> minTime{UINT64_MAX};
    };
    
    std::mutex profileMutex_;
    std::unordered_map<std::string, std::unique_ptr<FunctionProfile>> profiles_;
    
public:
    class ScopedProfiler {
    private:
        std::chrono::high_resolution_clock::time_point startTime_;
        std::string functionName_;
        PerformanceBottleneckAnalyzer* analyzer_;
        
    public:
        ScopedProfiler(const std::string& name, PerformanceBottleneckAnalyzer* analyzer)
            : functionName_(name), analyzer_(analyzer) {
            startTime_ = std::chrono::high_resolution_clock::now();
        }
        
        ~ScopedProfiler() {
            auto endTime = std::chrono::high_resolution_clock::now();
            auto duration = std::chrono::duration_cast<std::chrono::microseconds>(
                endTime - startTime_).count();
            
            analyzer_->RecordExecution(functionName_, duration);
        }
    };
    
    static PerformanceBottleneckAnalyzer& GetInstance() {
        static PerformanceBottleneckAnalyzer instance;
        return instance;
    }
    
    void RecordExecution(const std::string& functionName, uint64_t durationUs) {
        std::lock_guard<std::mutex> lock(profileMutex_);
        
        auto& profile = profiles_[functionName];
        if (!profile) {
            profile = std::make_unique<FunctionProfile>();
            profile->name = functionName;
        }
        
        profile->callCount.fetch_add(1);
        profile->totalTime.fetch_add(durationUs);
        
        // 최대/최소 시간 업데이트
        uint64_t currentMax = profile->maxTime.load();
        while (durationUs > currentMax && 
               !profile->maxTime.compare_exchange_weak(currentMax, durationUs));
        
        uint64_t currentMin = profile->minTime.load();
        while (durationUs < currentMin && 
               !profile->minTime.compare_exchange_weak(currentMin, durationUs));
    }
    
    void PrintBottleneckReport() {
        std::lock_guard<std::mutex> lock(profileMutex_);
        
        std::cout << "\n=== Performance Bottleneck Analysis ===\n";
        
        // 총 실행 시간별로 정렬
        std::vector<FunctionProfile*> sortedProfiles;
        for (const auto& [name, profile] : profiles_) {
            sortedProfiles.push_back(profile.get());
        }
        
        std::sort(sortedProfiles.begin(), sortedProfiles.end(),
            [](const FunctionProfile* a, const FunctionProfile* b) {
                return a->totalTime.load() > b->totalTime.load();
            });
        
        std::cout << "┌────────────────────────────┬─────────┬─────────────┬─────────────┬─────────────┐\n";
        std::cout << "│ Function Name              │ Calls   │ Total (ms)  │ Avg (μs)    │ Max (μs)    │\n";
        std::cout << "├────────────────────────────┼─────────┼─────────────┼─────────────┼─────────────┤\n";
        
        for (size_t i = 0; i < std::min<size_t>(10, sortedProfiles.size()); ++i) {
            const auto* profile = sortedProfiles[i];
            
            uint64_t calls = profile->callCount.load();
            uint64_t totalMs = profile->totalTime.load() / 1000;
            uint64_t avgUs = calls > 0 ? profile->totalTime.load() / calls : 0;
            uint64_t maxUs = profile->maxTime.load();
            
            std::cout << "│ " << std::setw(26) << std::left << profile->name.substr(0, 26)
                      << " │ " << std::setw(7) << std::right << calls
                      << " │ " << std::setw(11) << totalMs
                      << " │ " << std::setw(11) << avgUs
                      << " │ " << std::setw(11) << maxUs << " │\n";
        }
        
        std::cout << "└────────────────────────────┴─────────┴─────────────┴─────────────┴─────────────┘\n";
        
        // 병목 지점 식별
        IdentifyBottlenecks(sortedProfiles);
    }
    
private:
    void IdentifyBottlenecks(const std::vector<FunctionProfile*>& profiles) {
        std::cout << "\n🔍 Bottleneck Analysis:\n";
        
        if (profiles.empty()) return;
        
        uint64_t totalExecutionTime = 0;
        for (const auto* profile : profiles) {
            totalExecutionTime += profile->totalTime.load();
        }
        
        for (size_t i = 0; i < std::min<size_t>(5, profiles.size()); ++i) {
            const auto* profile = profiles[i];
            double percentage = static_cast<double>(profile->totalTime.load()) / 
                              totalExecutionTime * 100.0;
            
            if (percentage > 20.0) {
                std::cout << "🔴 CRITICAL: " << profile->name 
                          << " consumes " << std::fixed << std::setprecision(1) 
                          << percentage << "% of total execution time\n";
            } else if (percentage > 10.0) {
                std::cout << "🟡 WARNING: " << profile->name 
                          << " consumes " << std::fixed << std::setprecision(1) 
                          << percentage << "% of total execution time\n";
            }
        }
    }
};
```

### 시나리오 3: 종합 디버깅 시스템

```cpp
class GameServerDebugSystem {
private:
    GameServerETWProvider etwProvider_;
    RealTimePerformanceMonitor perfMonitor_;
    std::atomic<bool> debugMode_{true};
    
public:
    GameServerDebugSystem() : perfMonitor_(&etwProvider_) {
        // 메모리 누수 감지 시작
        MemoryLeakDetector::GetInstance().StartPeriodicCheck();
        
        std::cout << "🔧 Game Server Debug System initialized\n";
    }
    
    // 문제가 있는 게임 서버 함수들 (디버깅 대상)
    void ProblematicFunction1() {
        PROFILE_FUNCTION();
        
        // 메모리 누수 시뮬레이션
        void* leak = DEBUG_NEW(1024 * 1024); // 1MB 누수
        (void)leak; // 의도적으로 해제하지 않음

        volatile long long sum = 0;
        for (int i = 0; i < 1000000; ++i) {
            sum += i * i;
        }
        
        std::this_thread::sleep_for(std::chrono::milliseconds(50));
    }

        PROFILE_FUNCTION();
        
        // 데드락 위험이 있는 코드
        static std::mutex mutex1, mutex2;
        
        std::lock_guard<std::mutex> lock1(mutex1);
        std::this_thread::sleep_for(std::chrono::milliseconds(10));
        
        // 다른 스레드가 mutex2 -> mutex1 순서로 잠그면 데드락
        std::lock_guard<std::mutex> lock2(mutex2);

        PROFILE_FUNCTION();
        
        // I/O 병목 시뮬레이션

        for (size_t i = 0; i < allocations.size(); i += 2) {
            DEBUG_DELETE(allocations[i]);

    void RunDiagnosticWorkload() {
        std::cout << "\n🚀 Running diagnostic workload...\n";
        
        std::vector<std::thread> workers;
        
        // 문제가 있는 함수들을 여러 스레드에서 실행
        for (int i = 0; i < 4; ++i) {
            workers.emplace_back([this, i]() {
                SetThreadDescription(GetCurrentThread(), 
                    (L"DiagnosticWorker" + std::to_wstring(i)).c_str());
                
                for (int j = 0; j < 20; ++j) {
                    switch (j % 3) {
                        case 0: ProblematicFunction1(); break;
                        case 1: ProblematicFunction2(); break;
                        case 2: ProblematicFunction3(); break;
                    }
                    
                    // 성능 메트릭 업데이트
                    perfMonitor_.UpdateMetrics("CPU", 50.0 + (j % 40));
                    perfMonitor_.UpdateMetrics("Latency", 20.0 + (j % 80));
                    perfMonitor_.UpdateMetrics("Connections", 100 + (j % 50));
                    
                    std::this_thread::sleep_for(std::chrono::milliseconds(100));
                }
            });
        }
        
        // 모니터링 스레드
        std::thread monitorThread([this]() {
            for (int i = 0; i < 30; ++i) { // 30초간 모니터링
                std::this_thread::sleep_for(std::chrono::seconds(1));
                
                if (i % 5 == 0) { // 5초마다 대시보드 출력
                    perfMonitor_.PrintDashboard();
                }
            }
        });
        
        // 모든 스레드 완료 대기
        for (auto& worker : workers) {
            worker.join();
        }
        monitorThread.join();
        
        // 최종 보고서 생성
        GenerateFinalReport();
    }
```
