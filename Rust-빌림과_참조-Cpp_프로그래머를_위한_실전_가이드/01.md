# 1장. 기초 개념 훑어보기
이 장에서는 Rust의 소유권과 빌림 시스템의 기본을 빠르게 훑어본다. C++ 프로그래머로서 이미 메모리 관리와 포인터에 익숙하므로, 핵심 개념만 간결하게 다루고 곧바로 실전 문제로 넘어간다.

## 1.1 소유권의 기본 원칙
Rust는 가비지 컬렉터 없이 메모리 안전성을 보장하기 위해 소유권 시스템을 사용한다. C++에서 `std::unique_ptr`을 사용해봤다면 개념적으로 유사하다고 느낄 것이다.

**세 가지 핵심 규칙:**
1. 각 값은 정확히 하나의 소유자를 가진다
2. 소유자가 스코프를 벗어나면 값은 자동으로 해제된다
3. 값은 이동(move)되거나 빌려질(borrow) 수 있다

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;  // s1의 소유권이 s2로 이동
    
    // println!("{}", s1);  // 컴파일 에러! s1은 더 이상 유효하지 않다
    println!("{}", s2);  // OK
}
```

C++와의 비교:

```cpp
// C++에서는 복사가 일어난다
std::string s1 = "hello";
std::string s2 = s1;  // 깊은 복사
std::cout << s1;  // OK, s1은 여전히 유효하다

// Rust와 유사한 동작을 원한다면 move를 사용한다
std::string s1 = "hello";
std::string s2 = std::move(s1);  // 이동
// std::cout << s1;  // 정의되지 않은 동작
```

Rust에서는 기본적으로 모든 대입이 이동이다. 복사를 원한다면 명시적으로 `.clone()`을 호출해야 한다:

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1.clone();  // 명시적 복사
    
    println!("{}", s1);  // OK
    println!("{}", s2);  // OK
}
```

**정수형 같은 단순 타입은 예외다:**

```rust
fn main() {
    let x = 5;
    let y = x;  // 복사가 일어난다 (Copy 트레이트 구현)
    
    println!("{}", x);  // OK
    println!("{}", y);  // OK
}
```

이는 스택에 저장되는 고정 크기 타입들이 `Copy` 트레이트를 구현하기 때문이다. C++의 POD(Plain Old Data) 타입과 유사하다.
  

## 1.2 빌림과 참조의 핵심 규칙
소유권을 넘기지 않고 값에 접근하려면 참조를 사용한다. Rust의 빌림 검사기는 컴파일 타임에 다음 규칙을 강제한다:

**빌림의 두 가지 규칙:**
1. 어느 시점에든 **하나의 가변 참조** 또는 **여러 개의 불변 참조**만 가질 수 있다
2. 참조는 항상 유효해야 한다 (댕글링 참조 불가)

```rust
fn main() {
    let mut s = String::from("hello");
    
    // 불변 참조는 여러 개 가능하다
    let r1 = &s;
    let r2 = &s;
    println!("{} and {}", r1, r2);  // OK
    
    // 가변 참조는 하나만 가능하다
    let r3 = &mut s;
    r3.push_str(", world");
    println!("{}", r3);  // OK
}
```

**규칙 위반 예시:**

```rust
fn main() {
    let mut s = String::from("hello");
    
    let r1 = &s;      // 불변 참조
    let r2 = &mut s;  // 컴파일 에러! 불변 참조와 가변 참조 동시 존재
    
    println!("{}, {}", r1, r2);
}
```

에러 메시지:
```
error[E0502]: cannot borrow `s` as mutable because it is also borrowed as immutable
```

**중요한 점: 참조의 스코프**

Rust의 빌림 검사기는 참조가 마지막으로 사용된 지점까지만 스코프를 계산한다 (Non-Lexical Lifetimes, NLL):

```rust
fn main() {
    let mut s = String::from("hello");
    
    let r1 = &s;
    let r2 = &s;
    println!("{} and {}", r1, r2);
    // r1과 r2는 여기서 마지막으로 사용된다
    
    let r3 = &mut s;  // OK! r1, r2는 더 이상 사용되지 않는다
    r3.push_str(", world");
    println!("{}", r3);
}
```

이는 C++에서 스코프가 중괄호로만 결정되는 것과 다르다.
  

## 1.3 C++의 참조/포인터와 Rust의 차이점
C++ 프로그래머로서 혼란스러울 수 있는 주요 차이점들을 정리한다.

**1. 기본 동작의 차이**

| 상황 | C++ | Rust |
|------|-----|------|
| 변수 대입 | 복사 (또는 복사 생성자) | 이동 (Copy 타입 제외) |
| 함수 인자 | 값 복사 | 소유권 이동 |
| 참조 생성 | `&` 연산자 | `&` 연산자 (유사) |

**2. 가변성의 차이**

C++:
```cpp
int x = 5;
int& r = x;      // 참조 자체는 재할당 불가하지만
r = 10;          // 참조된 값은 변경 가능

const int y = 5;
const int& cr = y;  // 상수 참조
```

Rust:
```rust
let mut x = 5;
let r = &mut x;  // 가변 참조
*r = 10;         // OK

let y = 5;
let r = &y;      // 불변 참조
// *r = 10;      // 컴파일 에러!
```

Rust에서는 참조를 역참조(`*`)해서 값을 변경하려면 `&mut`가 필요하다. 단, 많은 경우 자동 역참조가 일어나 `*`를 생략할 수 있다.

**3. 댕글링 참조 방지**

C++에서 흔한 버그:
```cpp
int* danglingPointer() {
    int x = 5;
    return &x;  // 위험! 스택 변수에 대한 포인터 반환
}

std::string& danglingReference() {
    std::string s = "hello";
    return s;  // 위험! 로컬 변수에 대한 참조 반환
}
```

Rust에서는 컴파일 타임에 방지된다:
```rust
fn dangle() -> &String {  // 컴파일 에러!
    let s = String::from("hello");
    &s
}  // s가 여기서 해제되므로 참조는 유효하지 않다
```

올바른 방법:
```rust
fn no_dangle() -> String {
    let s = String::from("hello");
    s  // 소유권을 이동시킨다
}
```

**4. null 포인터의 부재**

C++:
```cpp
int* ptr = nullptr;  // null 포인터 가능
if (ptr != nullptr) {
    // 사용
}
```

Rust:
```rust
// 참조는 절대 null이 될 수 없다
let r: &i32;  // 초기화되지 않은 참조는 사용 불가

// null 가능성을 표현하려면 Option을 사용한다
let opt: Option<&i32> = None;
match opt {
    Some(r) => println!("{}", r),
    None => println!("No value"),
}
```
  

## 1.4 간단한 예제로 보는 빌림 검사기
실전에서 자주 마주치는 간단한 패턴들을 살펴본다.

**예제 1: 함수에 참조 전달하기**

```rust
fn calculate_length(s: &String) -> usize {
    s.len()
}  // s는 빌린 것이므로 여기서 해제되지 않는다

fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1);
    println!("Length of '{}' is {}", s1, len);  // s1은 여전히 유효하다
}
```

C++와 유사하지만, Rust는 함수 시그니처만 보고도 소유권이 이동하는지 빌려지는지 명확히 알 수 있다.

**예제 2: 가변 참조로 값 수정하기**

```rust
fn append_world(s: &mut String) {
    s.push_str(", world");
}

fn main() {
    let mut s = String::from("hello");
    append_world(&mut s);
    println!("{}", s);  // "hello, world"
}
```

**예제 3: 흔한 실수 - 가변 참조 중복**

```rust
fn main() {
    let mut s = String::from("hello");
    
    let r1 = &mut s;
    let r2 = &mut s;  // 컴파일 에러!
    
    println!("{}, {}", r1, r2);
}
```

에러:
```
error[E0499]: cannot borrow `s` as mutable more than once at a time
```

해결 방법 - 스코프를 분리한다:
```rust
fn main() {
    let mut s = String::from("hello");
    
    {
        let r1 = &mut s;
        r1.push_str(", world");
    }  // r1이 여기서 스코프를 벗어난다
    
    let r2 = &mut s;
    r2.push_str("!");
    
    println!("{}", s);  // "hello, world!"
}
```

**예제 4: 불변 참조와 가변 참조의 충돌**

```rust
fn main() {
    let mut v = vec![1, 2, 3, 4, 5];
    
    let first = &v[0];  // 불변 참조
    v.push(6);          // 컴파일 에러! 가변 참조 시도
    
    println!("The first element is: {}", first);
}
```

에러:
```
error[E0502]: cannot borrow `v` as mutable because it is also borrowed as immutable
```

왜 이런 제약이 있을까? `v.push(6)`는 벡터를 재할당할 수 있고, 그러면 `first`가 가리키는 메모리가 무효화될 수 있기 때문이다.

해결 방법 - 불변 참조 사용을 앞당긴다:
```rust
fn main() {
    let mut v = vec![1, 2, 3, 4, 5];
    
    let first = v[0];  // 값을 복사한다 (i32는 Copy)
    v.push(6);
    
    println!("The first element is: {}", first);
}
```

또는:
```rust
fn main() {
    let mut v = vec![1, 2, 3, 4, 5];
    
    {
        let first = &v[0];
        println!("The first element is: {}", first);
    }  // first는 여기서 끝난다
    
    v.push(6);  // OK
}
```

**예제 5: 메서드 호출에서의 자동 빌림**

Rust는 메서드 호출 시 자동으로 빌림/역참조를 수행한다:

```rust
fn main() {
    let mut s = String::from("hello");
    
    // 다음 세 가지는 모두 동일하다
    s.push_str(", world");
    (&mut s).push_str(", world");
    String::push_str(&mut s, ", world");
}
```

이는 C++의 `.`와 `->`의 차이를 자동으로 처리하는 것과 유사하지만, Rust는 더 나아가 `&`와 `&mut`도 자동으로 추론한다.

**1장 요약**

이 장에서는 Rust의 소유권과 빌림 시스템의 기초를 다뤘다:

- 소유권: 각 값은 하나의 소유자를 가지며, 소유자가 스코프를 벗어나면 자동으로 해제된다
- 빌림 규칙: 하나의 가변 참조 또는 여러 불변 참조만 허용된다
- C++와의 차이: 기본 이동 의미론, null 불가능한 참조, 컴파일 타임 댕글링 방지
- 빌림 검사기: 컴파일 타임에 메모리 안전성을 보장하며, 참조의 실제 사용 범위를 추적한다
