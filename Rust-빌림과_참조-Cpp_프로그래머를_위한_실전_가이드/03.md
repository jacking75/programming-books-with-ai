# 3장. 멤버 변수 접근 패턴
이 장에서는 구조체의 필드에 접근할 때 발생하는 빌림 문제들을 심층적으로 다룬다. C++에서는 자유롭게 멤버 변수를 읽고 쓸 수 있지만, Rust에서는 빌림 규칙 때문에 예상치 못한 제약을 만나게 된다. 특히 큰 규모의 프로그램에서는 이런 문제가 복잡하게 얽히므로, 실전적인 해결 패턴을 익히는 것이 중요하다.

## 3.1 필드 빌림과 구조체 전체 빌림의 충돌
Rust의 빌림 검사기는 기본적으로 구조체 단위로 빌림을 추적한다. 하지만 실제로는 필드 단위로도 빌림을 분석할 수 있다. 이 차이를 이해하는 것이 중요하다.

**기본 상황: 필드별 빌림은 가능하다**

```rust
struct Player {
    name: String,
    health: i32,
    mana: i32,
    position: (f32, f32),
}

fn main() {
    let mut player = Player {
        name: String::from("Hero"),
        health: 100,
        mana: 50,
        position: (0.0, 0.0),
    };
    
    // 서로 다른 필드에 동시에 접근 가능
    let health_ref = &player.health;
    let mana_ref = &player.mana;
    println!("Health: {}, Mana: {}", health_ref, mana_ref);
    
    // 가변 참조도 서로 다른 필드라면 가능
    let health_mut = &mut player.health;
    let mana_mut = &mut player.mana;
    *health_mut -= 10;
    *mana_mut -= 5;
    println!("Health: {}, Mana: {}", player.health, player.mana);
}
```

이는 C++에서 여러 멤버 변수를 동시에 조작하는 것과 유사하다.

**문제 상황 1: 메서드가 전체 구조체를 빌린다**

```rust
impl Player {
    fn get_health(&self) -> i32 {
        self.health
    }
    
    fn get_mana(&self) -> i32 {
        self.mana
    }
    
    // 문제가 되는 메서드
    fn heal_with_mana_wrong(&mut self) {
        let current_health = self.get_health();  // &self로 전체 구조체를 빌린다
        
        if current_health < 100 {
            self.mana -= 20;  // 컴파일 에러!
            self.health += 10;
            // cannot borrow `*self` as mutable because it is also borrowed as immutable
        }
    }
}
```

이 에러는 `get_health()`가 `&self`를 받기 때문에 전체 구조체가 불변으로 빌려진다. 그 상태에서 `self.mana`나 `self.health`를 변경하려 하면 가변 빌림이 발생하여 충돌한다.

**해결책 1: 필드에 직접 접근한다**

```rust
impl Player {
    fn heal_with_mana(&mut self) {
        // 메서드를 호출하지 않고 직접 접근
        let current_health = self.health;
        
        if current_health < 100 {
            self.mana -= 20;
            self.health += 10;
        }
    }
}

fn main() {
    let mut player = Player {
        name: String::from("Hero"),
        health: 80,
        mana: 50,
        position: (0.0, 0.0),
    };
    
    player.heal_with_mana();
    println!("Health: {}, Mana: {}", player.health, player.mana);
}
```

**해결책 2: 값을 복사하여 빌림을 종료한다**

```rust
impl Player {
    fn heal_with_mana_v2(&mut self) {
        let current_health = self.get_health();  // 값을 복사
        // get_health() 호출이 끝나면서 &self 빌림이 종료됨
        
        if current_health < 100 {
            self.mana -= 20;  // OK
            self.health += 10;
        }
    }
}
```

`i32`는 `Copy` 트레이트를 구현하므로, `get_health()`의 반환값을 받는 순간 빌림이 종료된다.

**문제 상황 2: 참조를 반환하는 게터**

```rust
impl Player {
    fn get_name(&self) -> &String {
        &self.name
    }
    
    fn get_position(&self) -> &(f32, f32) {
        &self.position

        self.position.0 += 1.0;  // 컴파일 에러!
```

**해결책 1: 값을 복사한다**

```rust
impl Player {

        // get_position() 빌림 종료
        
        self.position.0 += 1.0;
        self.position.1 += 1.0;
        
        println!("Moved from {:?} to {:?}", old_pos, self.position);
    }
}
```

**해결책 2: 빌림 스코프를 명시적으로 제한한다**

```rust
impl Player {

        let old_pos = {
            let pos = self.get_position();
            *pos  // 복사
        };  // pos 빌림이 여기서 종료
        
        self.position.0 += 1.0;
        self.position.1 += 1.0;

    }
}
```

## 3.2 여러 필드를 동시에 접근하기
큰 규모의 프로그램에서는 여러 필드를 동시에 읽고 쓰는 상황이 자주 발생한다. Rust의 빌림 규칙은 이를 허용하지만, 메서드를 통한 간접 접근에서는 제약이 있다.

**문제 상황: 여러 필드 간의 복잡한 상호작용**

```rust
struct GameState {
    player_gold: i32,
    shop_inventory: Vec<String>,
    player_inventory: Vec<String>,
    transaction_log: Vec<String>,
}

impl GameState {
    // 문제가 되는 메서드
    fn buy_item_wrong(&mut self, item: &str, price: i32) {
        if self.can_afford(price) {  // &self로 빌림
            self.player_gold -= price;  // 컴파일 에러!
            // cannot borrow `*self` as mutable because it is also borrowed as immutable
        }
    }
    
    fn can_afford(&self, price: i32) -> bool {
        self.player_gold >= price
    }
}
```

**해결책 1: 조건문을 단순화한다**

```rust
impl GameState {
    fn buy_item(&mut self, item: &str, price: i32) {
        if self.player_gold >= price {  // 직접 접근
            self.player_gold -= price;
            
            // 아이템 이동
            if let Some(pos) = self.shop_inventory.iter().position(|i| i == item) {
                let bought_item = self.shop_inventory.remove(pos);
                self.player_inventory.push(bought_item);
                
                // 로그 기록
                self.transaction_log.push(format!("Bought {} for {}", item, price));
            }
        }
    }
}

fn main() {
    let mut game = GameState {
        player_gold: 100,
        shop_inventory: vec![
            String::from("Sword"),
            String::from("Shield"),
            String::from("Potion"),
        ],
        player_inventory: vec![],
        transaction_log: vec![],
    };
    
    game.buy_item("Sword", 50);
    println!("Gold remaining: {}", game.player_gold);
    println!("Player inventory: {:?}", game.player_inventory);
}
```

**문제 상황 2: 필드 간 데이터 이동**

```rust
struct DataProcessor {
    input_queue: Vec<String>,
    output_queue: Vec<String>,
    error_queue: Vec<String>,
    statistics: ProcessStats,
}

struct ProcessStats {
    processed: usize,
    errors: usize,
}

impl DataProcessor {
    // 복잡한 처리 로직
    fn process_next_wrong(&mut self) {
        if let Some(item) = self.input_queue.pop() {
            // 처리 시뮬레이션
            if item.len() > 10 {
                self.output_queue.push(item.clone());
                self.increment_processed();  // &mut self를 다시 빌림
            } else {
                self.error_queue.push(item.clone());
                self.increment_errors();
            }
        }
    }
    
    fn increment_processed(&mut self) {
        self.statistics.processed += 1;
    }
    
    fn increment_errors(&mut self) {
        self.statistics.errors += 1;
    }
}
```

위 코드는 실제로는 컴파일된다. 왜냐하면 `increment_processed()`와 `increment_errors()`를 호출할 때 이미 `input_queue.pop()`의 빌림이 끝났기 때문이다.

**하지만 이런 경우는 문제가 된다:**

```rust
impl DataProcessor {
    fn process_batch_wrong(&mut self) {
        let input_ref = &self.input_queue;  // 불변 빌림
        
        for item in input_ref {
            if item.len() > 10 {
                self.output_queue.push(item.clone());  // 컴파일 에러!
                // cannot borrow `self.output_queue` as mutable because `*self` is also borrowed as immutable
            }
        }
    }
}
```

**해결책: 필드만 따로 빌린다**

```rust
impl DataProcessor {
    fn process_batch(&mut self) {
        // 두 필드를 동시에 가변으로 빌린다
        let input = &mut self.input_queue;
        let output = &mut self.output_queue;
        let errors = &mut self.error_queue;
        let stats = &mut self.statistics;
        
        // drain을 사용하여 소유권을 가져온다
        let items: Vec<_> = input.drain(..).collect();
        
        for item in items {
            if item.len() > 10 {
                output.push(item);
                stats.processed += 1;
            } else {
                errors.push(item);
                stats.errors += 1;
            }
        }
    }
}

fn main() {
    let mut processor = DataProcessor {
        input_queue: vec![
            String::from("short"),
            String::from("this is a long string"),
            String::from("test"),
        ],
        output_queue: vec![],
        error_queue: vec![],
        statistics: ProcessStats {
            processed: 0,
            errors: 0,
        },
    };
    
    processor.process_batch();
    println!("Processed: {}, Errors: {}", 
             processor.statistics.processed, 
             processor.statistics.errors);
}
```

**실전 예제: 복잡한 상태 전환**

```rust
struct NetworkConnection {
    send_buffer: Vec<u8>,
    receive_buffer: Vec<u8>,
    pending_requests: Vec<Request>,
    completed_responses: Vec<Response>,
    state: ConnectionState,
}

#[derive(Debug)]
enum ConnectionState {
    Disconnected,
    Connecting,
    Connected,
    Error(String),
}

struct Request {
    id: u32,
    data: Vec<u8>,
}

struct Response {
    id: u32,
    data: Vec<u8>,
}

impl NetworkConnection {
    fn new() -> Self {
        NetworkConnection {
            send_buffer: Vec::new(),
            receive_buffer: Vec::new(),
            pending_requests: Vec::new(),
            completed_responses: Vec::new(),
            state: ConnectionState::Disconnected,
        }
    }
    
    // 여러 필드를 동시에 조작하는 복잡한 메서드
    fn process_incoming_data(&mut self, data: &[u8]) {
        // 버퍼에 데이터 추가
        self.receive_buffer.extend_from_slice(data);
        
        // 완전한 응답이 있는지 확인
        while self.receive_buffer.len() >= 8 {
            // 간단한 프로토콜: [id: 4바이트][length: 4바이트][data]
            let id = u32::from_be_bytes([
                self.receive_buffer[0],
                self.receive_buffer[1],
                self.receive_buffer[2],
                self.receive_buffer[3],
            ]);
            
            let length = u32::from_be_bytes([
                self.receive_buffer[4],
                self.receive_buffer[5],
                self.receive_buffer[6],
                self.receive_buffer[7],
            ]) as usize;
            
            if self.receive_buffer.len() < 8 + length {
                break;  // 아직 데이터가 충분하지 않음
            }
            
            // 응답 추출
            let response_data = self.receive_buffer[8..8+length].to_vec();
            self.receive_buffer.drain(..8+length);
            
            // 대기 중인 요청 제거
            if let Some(pos) = self.pending_requests.iter().position(|r| r.id == id) {
                self.pending_requests.remove(pos);
            }
            
            // 완료된 응답 추가
            self.completed_responses.push(Response {
                id,
                data: response_data,
            });
            
            // 상태 업데이트
            self.state = ConnectionState::Connected;
        }
    }
    
    fn send_request(&mut self, request: Request) {
        // 요청을 버퍼에 추가
        let id_bytes = request.id.to_be_bytes();
        let length_bytes = (request.data.len() as u32).to_be_bytes();
        
        self.send_buffer.extend_from_slice(&id_bytes);
        self.send_buffer.extend_from_slice(&length_bytes);
        self.send_buffer.extend_from_slice(&request.data);
        
        // 대기 목록에 추가
        self.pending_requests.push(request);
    }
}

fn main() {
    let mut conn = NetworkConnection::new();
    
    conn.send_request(Request {
        id: 1,
        data: vec![1, 2, 3, 4],
    });
    
    // 응답 시뮬레이션
    let response_data = [
        0, 0, 0, 1,  // id: 1
        0, 0, 0, 4,  // length: 4
        5, 6, 7, 8,  // data
    ];
    
    conn.process_incoming_data(&response_data);
    
    println!("Pending: {}, Completed: {}", 
             conn.pending_requests.len(),
             conn.completed_responses.len());
}
```
  

## 3.3 메서드 내에서 다른 메서드 호출 시 발생하는 문제
객체지향 프로그래밍에서는 메서드가 다른 메서드를 호출하는 것이 일반적이다. 하지만 Rust에서는 이로 인해 빌림 충돌이 자주 발생한다.

**문제 상황 1: Helper 메서드 호출**

```rust
struct TextEditor {
    content: String,
    cursor_position: usize,
    is_modified: bool,
}

impl TextEditor {
    fn get_current_line(&self) -> &str {
        // 간단화를 위해 첫 줄만 반환
        self.content.lines().next().unwrap_or("")
    }
    
    fn get_cursor_position(&self) -> usize {
        self.cursor_position
    }
    
    // 문제가 되는 메서드
    fn insert_at_cursor_wrong(&mut self, text: &str) {
        let current_line = self.get_current_line();  // &self로 빌림
        let cursor = self.get_cursor_position();
        
        // 현재 라인 정보를 로그로 출력
        println!("Inserting into line: {}", current_line);
        
        self.content.insert_str(cursor, text);  // 컴파일 에러!
        self.is_modified = true;
        // cannot borrow `self.content` as mutable because it is also borrowed as immutable
    }
}
```

`get_current_line()`이 `&self.content`를 참조로 반환하고, 그 참조(`current_line`)를 나중에 사용하기 때문에 빌림이 지속된다.

**해결책 1: 참조 대신 값을 반환한다**

```rust
impl TextEditor {
    fn get_current_line_owned(&self) -> String {
        self.content.lines().next().unwrap_or("").to_string()
    }
    
    fn insert_at_cursor(&mut self, text: &str) {
        let current_line = self.get_current_line_owned();  // String 복사
        let cursor = self.get_cursor_position();
        
        println!("Inserting into line: {}", current_line);
        
        self.content.insert_str(cursor, text);  // OK
        self.is_modified = true;
    }
}

fn main() {
    let mut editor = TextEditor {
        content: String::from("Hello, world!\nSecond line"),
        cursor_position: 5,
        is_modified: false,
    };
    
    editor.insert_at_cursor(" Rust");
    println!("Content: {}", editor.content);
}
```

**해결책 2: 빌림 스코프를 분리한다**

```rust
impl TextEditor {
    fn insert_at_cursor_v2(&mut self, text: &str) {
        // 빌림을 먼저 끝낸다
        {
            let current_line = self.get_current_line();
            println!("Inserting into line: {}", current_line);
        }  // current_line 빌림이 여기서 종료
        
        let cursor = self.get_cursor_position();
        self.content.insert_str(cursor, text);  // OK
        self.is_modified = true;
    }
}
```

**문제 상황 2: 유효성 검사 메서드**

```rust
struct FormData {
    username: String,
    email: String,
    age: u32,
    validation_errors: Vec<String>,
}

impl FormData {
    fn is_username_valid(&self) -> bool {
        self.username.len() >= 3 && self.username.len() <= 20
    }
    
    fn is_email_valid(&self) -> bool {
        self.email.contains('@')
    }
    
    fn is_age_valid(&self) -> bool {
        self.age >= 18 && self.age <= 100
    }
    
    // 문제가 되는 메서드
    fn validate_wrong(&mut self) {
        self.validation_errors.clear();
        
        if !self.is_username_valid() {  // &self로 빌림
            self.validation_errors.push(  // 컴파일 에러!
                String::from("Username must be 3-20 characters")
            );
        }
    }
}
```

**해결책: 검증 결과를 먼저 수집한다**

```rust
impl FormData {
    fn validate(&mut self) {
        self.validation_errors.clear();
        
        // 모든 검증을 먼저 수행
        let username_valid = self.is_username_valid();
        let email_valid = self.is_email_valid();
        let age_valid = self.is_age_valid();
        
        // 그 후 에러 메시지 추가
        if !username_valid {
            self.validation_errors.push(
                String::from("Username must be 3-20 characters")
            );
        }
        
        if !email_valid {
            self.validation_errors.push(
                String::from("Email must contain @")
            );
        }
        
        if !age_valid {
            self.validation_errors.push(
                String::from("Age must be between 18 and 100")
            );
        }
    }
    
    fn is_valid(&self) -> bool {
        self.is_username_valid() && self.is_email_valid() && self.is_age_valid()
    }
}

fn main() {
    let mut form = FormData {
        username: String::from("ab"),  // 너무 짧음
        email: String::from("invalid-email"),  // @ 없음
        age: 25,
        validation_errors: vec![],
    };
    
    form.validate();
    
    if form.validation_errors.is_empty() {
        println!("Form is valid!");
    } else {
        println!("Validation errors:");
        for error in &form.validation_errors {
            println!("  - {}", error);
        }
    }
}
```

**실전 예제: 복잡한 비즈니스 로직**

```rust
struct ShoppingCart {
    items: Vec<CartItem>,
    discount_code: Option<String>,
    customer_tier: CustomerTier,
    total_cache: Option<f64>,
}

struct CartItem {
    name: String,
    price: f64,
    quantity: u32,
}

#[derive(Clone, Copy)]
enum CustomerTier {
    Regular,
    Silver,
    Gold,
}

impl ShoppingCart {
    fn calculate_subtotal(&self) -> f64 {
        self.items.iter()
            .map(|item| item.price * item.quantity as f64)
            .sum()
    }
    
    fn get_discount_percentage(&self) -> f64 {
        match self.customer_tier {
            CustomerTier::Regular => 0.0,
            CustomerTier::Silver => 5.0,
            CustomerTier::Gold => 10.0,
        }
    }
    
    fn has_discount_code(&self) -> bool {
        self.discount_code.is_some()
    }
    
    // 복잡한 총액 계산
    fn calculate_total(&mut self) -> f64 {
        // 캐시가 있으면 반환
        if let Some(cached) = self.total_cache {
            return cached;
        }
        
        // 필요한 값들을 먼저 계산
        let subtotal = self.calculate_subtotal();
        let discount_pct = self.get_discount_percentage();
        let has_code = self.has_discount_code();
        
        // 할인 적용
        let mut total = subtotal;
        total *= 1.0 - (discount_pct / 100.0);
        
        // 할인 코드가 있으면 추가 10% 할인
        if has_code {
            total *= 0.9;
        }
        
        // 캐시에 저장
        self.total_cache = Some(total);
        
        total
    }
    
    fn add_item(&mut self, item: CartItem) {
        self.items.push(item);
        self.total_cache = None;  // 캐시 무효화
    }
    
    fn apply_discount_code(&mut self, code: String) {
        self.discount_code = Some(code);
        self.total_cache = None;  // 캐시 무효화
    }
}

fn main() {
    let mut cart = ShoppingCart {
        items: vec![],
        discount_code: None,
        customer_tier: CustomerTier::Gold,
        total_cache: None,
    };
    
    cart.add_item(CartItem {
        name: String::from("Laptop"),
        price: 1000.0,
        quantity: 1,
    });
    
    cart.add_item(CartItem {
        name: String::from("Mouse"),
        price: 50.0,
        quantity: 2,
    });
    
    cart.apply_discount_code(String::from("SAVE10"));
    
    let total = cart.calculate_total();
    println!("Total: ${:.2}", total);
    
    // 캐시된 값 사용
    let total2 = cart.calculate_total();
    println!("Total (cached): ${:.2}", total2);
}
```
  

## 3.4 분할 빌림(Split Borrowing) 패턴
분할 빌림은 Rust의 강력한 기능으로, 구조체의 서로 다른 필드를 동시에 가변으로 빌릴 수 있게 한다. 이는 C++에서 여러 멤버 변수를 동시에 조작하는 것과 유사하지만, 더 안전하다.

**기본 개념**

```rust
struct Rectangle {
    width: f64,
    height: f64,
    color: String,
    position: (f64, f64),
}

fn main() {
    let mut rect = Rectangle {
        width: 100.0,
        height: 50.0,
        color: String::from("red"),
        position: (0.0, 0.0),
    };
    
    // 분할 빌림: 서로 다른 필드를 동시에 가변으로 빌린다
    let width = &mut rect.width;
    let height = &mut rect.height;
    let color = &mut rect.color;
    
    *width *= 2.0;
    *height *= 2.0;
    *color = String::from("blue");
    
    println!("Rectangle: {}x{}, color: {}", rect.width, rect.height, rect.color);
}
```

**문제: 메서드에서는 자동으로 분할 빌림이 안 된다**

```rust
impl Rectangle {
    // 이 메서드는 전체 구조체를 빌린다
    fn resize_wrong(&mut self, factor: f64) {
        let w = &mut self.width;  // width만 빌리려 했지만
        *w *= factor;
        
        self.height *= factor;  // 여전히 작동한다! (컴파일러가 분석 가능)
    }
    
    // 하지만 이런 경우는 문제가 된다
    fn process_wrong(&mut self) {
        self.update_dimensions(&mut self.width);  // 컴파일 에러!
        // cannot borrow `*self` as immutable because it is also borrowed as mutable
    }
    
    fn update_dimensions(&self, width: &mut f64) {
        *width = self.height * 2.0;
    }
}
```

**해결책 1: 함수로 추출하여 분할 빌림을 명시한다**

```rust
impl Rectangle {
    fn process(&mut self) {
        // 함수를 통해 분할 빌림을 명시
        update_dimensions_helper(&mut self.width, &self.height);
    }
}

fn update_dimensions_helper(width: &mut f64, height: &f64) {
    *width = *height * 2.0;
}

fn main() {
    let mut rect = Rectangle {
        width: 100.0,
        height: 50.0,
        color: String::from("red"),
        position: (0.0, 0.0),
    };
    
    rect.process();
    println!("Width: {}, Height: {}", rect.width, rect.height);
}
```

**해결책 2: 튜플이나 구조체로 필드를 그룹화한다**

```rust

    color: String,

        self.dimensions.width *= factor;
        self.dimensions.height *= factor;
    }
    
    fn change_color(&mut self, color: String) {

        self.color = color;
    }

        let col = &mut self.color;
        
        dims.width *= factor;
        dims.height *= factor;
        *col = color;
    }
}
```

**실전 예제 1: 게임 엔티티 시스템**

```rust
struct Entity {
    transform: Transform,
    physics: Physics,
    render: RenderData,
    ai: Option<AIComponent>,
}

struct Transform {
    position: (f32, f32, f32),
    rotation: (f32, f32, f32),
    scale: (f32, f32, f32),
}

struct Physics {
    velocity: (f32, f32, f32),
    acceleration: (f32, f32, f32),
    mass: f32,
}

struct RenderData {
    mesh_id: u32,
    material_id: u32,
    visible: bool,
}

struct AIComponent {
    state: AIState,
    target: Option<(f32, f32, f32)>,
}

#[derive(Debug)]
enum AIState {
    Idle,
    Moving,
    Attacking,
}

impl Entity {
    // 물리 업데이트: transform과 physics를 동시에 수정
    fn update_physics(&mut self, delta_time: f32) {
        // 분할 빌림으로 두 필드를 동시에 접근
        let transform = &mut self.transform;
        let physics = &mut self.physics;
        
        // 가속도로 속도 업데이트
        physics.velocity.0 += physics.acceleration.0 * delta_time;
        physics.velocity.1 += physics.acceleration.1 * delta_time;
        physics.velocity.2 += physics.acceleration.2 * delta_time;
        
        // 속도로 위치 업데이트
        transform.position.0 += physics.velocity.0 * delta_time;
        transform.position.1 += physics.velocity.1 * delta_time;
        transform.position.2 += physics.velocity.2 * delta_time;
    }
    
    // AI 업데이트: ai와 transform, physics를 동시에 수정
    fn update_ai(&mut self, delta_time: f32) {
        if let Some(ai) = &mut self.ai {
            match ai.state {
                AIState::Moving => {
                    if let Some(target) = ai.target {
                        // 목표를 향해 이동
                        let dx = target.0 - self.transform.position.0;
                        let dy = target.1 - self.transform.position.1;
                        let dz = target.2 - self.transform.position.2;
                        
                        let distance = (dx*dx + dy*dy + dz*dz).sqrt();
                        
                        if distance < 0.1 {
                            ai.state = AIState::Idle;
                            ai.target = None;
                        } else {
                            // 정규화하여 방향 계산
                            self.physics.acceleration.0 = dx / distance;
                            self.physics.acceleration.1 = dy / distance;
                            self.physics.acceleration.2 = dz / distance;
                        }
                    }
                }
                _ => {}
            }
        }
    }
}

fn main() {
    let mut entity = Entity {
        transform: Transform {
            position: (0.0, 0.0, 0.0),
            rotation: (0.0, 0.0, 0.0),
            scale: (1.0, 1.0, 1.0),
        },
        physics: Physics {
            velocity: (0.0, 0.0, 0.0),
            acceleration: (0.0, 0.0, 0.0),
            mass: 1.0,
        },
        render: RenderData {
            mesh_id: 0,
            material_id: 0,
            visible: true,
        },
        ai: Some(AIComponent {
            state: AIState::Moving,
            target: Some((10.0, 0.0, 0.0)),
        }),
    };
    
    // 게임 루프 시뮬레이션
    for _ in 0..10 {
        entity.update_ai(0.1);
        entity.update_physics(0.1);
        
        println!("Position: {:?}", entity.transform.position);
    }
}
```

**실전 예제 2: 데이터베이스 연결 풀**

```rust
struct ConnectionPool {
    available: Vec<Connection>,
    in_use: Vec<Connection>,
    config: PoolConfig,
    statistics: PoolStats,
}

struct Connection {
    id: u32,
    connected: bool,
}

struct PoolConfig {
    max_connections: usize,
    min_connections: usize,
}

struct PoolStats {
    total_requests: u64,
    successful_gets: u64,
    failed_gets: u64,
}

impl ConnectionPool {
    fn new(config: PoolConfig) -> Self {
        let mut available = Vec::new();
        for i in 0..config.min_connections {
            available.push(Connection {
                id: i as u32,
                connected: true,
            });
        }
        
        ConnectionPool {
            available,
            in_use: Vec::new(),
            config,
            statistics: PoolStats {
                total_requests: 0,
                successful_gets: 0,
                failed_gets: 0,
            },
        }
    }
    
    fn get_connection(&mut self) -> Option<Connection> {
        // 분할 빌림: 여러 필드를 동시에 수정
        let available = &mut self.available;
        let in_use = &mut self.in_use;
        let stats = &mut self.statistics;
        let config = &self.config;
        
        stats.total_requests += 1;
        
        if let Some(conn) = available.pop() {
            in_use.push(Connection {
                id: conn.id,
                connected: conn.connected,
            });
            stats.successful_gets += 1;
            Some(conn)
        } else if in_use.len() + available.len() < config.max_connections {
            // 새 연결 생성
            let new_id = (in_use.len() + available.len()) as u32;
            let conn = Connection {
                id: new_id,
                connected: true,
            };
            in_use.push(Connection {
                id: conn.id,
                connected: conn.connected,
            });
            stats.successful_gets += 1;
            Some(conn)
        } else {
            stats.failed_gets += 1;
            None
        }
    }
    
    fn return_connection(&mut self, conn: Connection) {
        // in_use에서 제거하고 available로 이동
        if let Some(pos) = self.in_use.iter().position(|c| c.id == conn.id) {
            self.in_use.remove(pos);
            self.available.push(conn);
        }
    }
    
    fn print_stats(&self) {
        println!("Pool Statistics:");
        println!("  Total requests: {}", self.statistics.total_requests);
        println!("  Successful: {}", self.statistics.successful_gets);
        println!("  Failed: {}", self.statistics.failed_gets);
        println!("  Available: {}", self.available.len());
        println!("  In use: {}", self.in_use.len());
    }
}

fn main() {
    let mut pool = ConnectionPool::new(PoolConfig {
        max_connections: 5,
        min_connections: 2,
    });
    
    let conn1 = pool.get_connection();
    let conn2 = pool.get_connection();
    let conn3 = pool.get_connection();
    
    pool.print_stats();
    
    if let Some(c) = conn1 {
        pool.return_connection(c);
    }
    
    pool.print_stats();
}
```

**패턴: 헬퍼 함수로 분할 빌림 명시화**

큰 규모의 프로그램에서는 분할 빌림을 명시적으로 만드는 것이 좋다:

```rust
struct ComplexState {
    cache: Cache,
    database: Database,
    network: Network,
    logger: Logger,
}

struct Cache { data: Vec<String> }
struct Database { connection: String }
struct Network { socket: u32 }
struct Logger { log_file: String }

impl ComplexState {
    fn process_request(&mut self, request: &str) {
        // 헬퍼 함수로 필요한 부분만 전달
        process_with_cache_and_db(
            &mut self.cache,
            &mut self.database,
            &self.logger,
            request
        );
        
        process_with_network_and_logger(
            &mut self.network,
            &mut self.logger,
            request
        );
    }
}

fn process_with_cache_and_db(
    cache: &mut Cache,
    db: &mut Database,
    logger: &Logger,
    request: &str
) {
    // cache와 db를 동시에 수정 가능
    cache.data.push(request.to_string());
    db.connection = format!("Processing: {}", request);
    
    // logger는 읽기만
    println!("Log file: {}", logger.log_file);
}

fn process_with_network_and_logger(
    network: &mut Network,
    logger: &mut Logger,
    request: &str
) {
    network.socket += 1;
    logger.log_file.push_str(&format!("\n{}", request));
}
```

**3장 요약**

이 장에서는 멤버 변수 접근 시 발생하는 복잡한 빌림 문제들을 다뤘다:

- **필드 vs 구조체 빌림**: 메서드는 기본적으로 전체 구조체를 빌리지만, 직접 필드에 접근하면 필드 단위 빌림이 가능하다

- **여러 필드 동시 접근**: 서로 다른 필드는 동시에 가변으로 빌릴 수 있지만, 메서드를 통한 간접 접근에서는 제약이 있다

- **메서드 간 호출 문제**: 헬퍼 메서드를 호출할 때 빌림 충돌을 피하려면 값을 복사하거나, 스코프를 분리하거나, 필요한 필드만 전달해야 한다

- **분할 빌림 패턴**: 헬퍼 함수나 구조체 그룹화를 통해 명시적으로 분할 빌림을 활용하면, 복잡한 상태를 안전하게 관리할 수 있다

다음 장에서는 컬렉션과 반복자에서 발생하는 빌림 문제와 그 해결 방법을 살펴본다.
