# 5장. 객체 간 참조 관계 설계
C++에서는 객체들이 서로를 포인터나 참조로 가리키는 것이 자연스럽다. 하지만 Rust의 소유권 시스템에서는 이런 참조 관계를 만드는 것이 가장 어려운 도전 중 하나다. 이 장에서는 객체 간 복잡한 참조 관계를 Rust에서 어떻게 설계하고 구현하는지 실전 예제를 통해 살펴본다.

## 5.1 상호 참조하는 객체들 (양방향 참조)

### 문제 상황: 양방향 연결 리스트
C++에서는 노드가 이전 노드와 다음 노드를 모두 가리키는 양방향 연결 리스트를 쉽게 만들 수 있다. Rust에서 같은 방식을 시도하면 즉시 막힌다.

```rust
// 컴파일 실패!
struct Node {
    value: i32,
    next: Option<Box<Node>>,
    prev: Option<&Node>,  // 에러: 라이프타임 명시 필요
}
```

라이프타임을 추가해도 문제는 해결되지 않는다.

```rust
// 여전히 문제가 많음
struct Node<'a> {
    value: i32,
    next: Option<Box<Node<'a>>>,
    prev: Option<&'a Node<'a>>,
}

// 실제로 사용하려고 하면...
fn create_linked_list() {
    let mut first = Node {
        value: 1,
        next: None,
        prev: None,
    };
    
    let mut second = Node {
        value: 2,
        next: None,
        prev: Some(&first),  // first를 빌림
    };
    
    first.next = Some(Box::new(second));  // 에러: second가 이미 빌려졌음
}
```

`second`를 만들 때 `first`를 빌리고, 동시에 `second`를 `first.next`로 이동시키려고 하므로 소유권 규칙을 위반한다.

### 해결책 1: 참조 대신 인덱스 사용
가장 Rust다운 해결책은 참조 대신 인덱스를 사용하는 것이다.

```rust
struct Node {
    value: i32,
    next: Option<usize>,
    prev: Option<usize>,
}

struct LinkedList {
    nodes: Vec<Node>,
}

impl LinkedList {
    fn new() -> Self {
        LinkedList { nodes: Vec::new() }
    }
    
    fn add_node(&mut self, value: i32) -> usize {
        let index = self.nodes.len();
        self.nodes.push(Node {
            value,
            next: None,
            prev: None,
        });
        index
    }
    
    fn link_nodes(&mut self, first: usize, second: usize) {
        if first < self.nodes.len() && second < self.nodes.len() {
            self.nodes[first].next = Some(second);
            self.nodes[second].prev = Some(first);
        }
    }
    
    fn get_value(&self, index: usize) -> Option<i32> {
        self.nodes.get(index).map(|node| node.value)
    }
    
    fn traverse_forward(&self, start: usize) -> Vec<i32> {
        let mut result = Vec::new();
        let mut current = Some(start);
        
        while let Some(idx) = current {
            if let Some(node) = self.nodes.get(idx) {
                result.push(node.value);
                current = node.next;
            } else {
                break;
            }
        }
        result
    }
    
    fn traverse_backward(&self, start: usize) -> Vec<i32> {
        let mut result = Vec::new();
        let mut current = Some(start);
        
        while let Some(idx) = current {
            if let Some(node) = self.nodes.get(idx) {
                result.push(node.value);
                current = node.prev;
            } else {
                break;
            }
        }
        result
    }
}

fn main() {
    let mut list = LinkedList::new();
    
    let first = list.add_node(10);
    let second = list.add_node(20);
    let third = list.add_node(30);
    
    list.link_nodes(first, second);
    list.link_nodes(second, third);
    
    println!("Forward: {:?}", list.traverse_forward(first));
    println!("Backward: {:?}", list.traverse_backward(third));
}
```

이 방식은 메모리 안전성을 유지하면서도 양방향 순회가 가능하다. 인덱스는 빌림 검사기의 제약을 받지 않으므로 자유롭게 저장하고 사용할 수 있다.

### 해결책 2: Rc와 RefCell 조합
참조 카운팅과 내부 가변성을 조합하면 C++과 비슷한 방식으로 구현할 수 있다.

```rust
use std::rc::Rc;
use std::cell::RefCell;

struct Node {
    value: i32,
    next: Option<Rc<RefCell<Node>>>,
    prev: Option<Rc<RefCell<Node>>>,
}

impl Node {
    fn new(value: i32) -> Rc<RefCell<Node>> {
        Rc::new(RefCell::new(Node {
            value,
            next: None,
            prev: None,
        }))
    }
}

fn create_bidirectional_link() {
    let first = Node::new(10);
    let second = Node::new(20);
    
    // first -> second
    first.borrow_mut().next = Some(Rc::clone(&second));
    // second -> first
    second.borrow_mut().prev = Some(Rc::clone(&first));
    
    // 순회
    let mut current = Some(Rc::clone(&first));
    while let Some(node) = current {
        println!("Value: {}", node.borrow().value);
        current = node.borrow().next.as_ref().map(|n| Rc::clone(n));
    }
}
```

이 방식은 작동하지만 순환 참조를 만든다. `first`와 `second`가 서로를 참조하므로 참조 카운트가 절대 0이 되지 않아 메모리 누수가 발생한다. 이 문제는 5.5절에서 `Weak` 참조로 해결한다.

### 실전 예제: 편집기의 문서-뷰 구조
텍스트 편집기에서 문서(Document)와 뷰(View)가 서로를 참조해야 하는 경우를 보자.

```rust
// 잘못된 시도
struct Document {
    content: String,
    views: Vec<&View>,  // 에러: 라이프타임 필요
}

struct View {
    document: &Document,  // 에러: 라이프타임 필요
    cursor_pos: usize,
}
```

인덱스 기반 접근으로 재설계한다.

```rust
type DocumentId = usize;
type ViewId = usize;

struct Document {
    content: String,
    views: Vec<ViewId>,
}

struct View {
    document_id: DocumentId,
    cursor_pos: usize,
}

struct Editor {
    documents: Vec<Document>,
    views: Vec<View>,
}

impl Editor {
    fn new() -> Self {
        Editor {
            documents: Vec::new(),
            views: Vec::new(),
        }
    }
    
    fn create_document(&mut self, content: String) -> DocumentId {
        let id = self.documents.len();
        self.documents.push(Document {
            content,
            views: Vec::new(),
        });
        id
    }
    
    fn create_view(&mut self, document_id: DocumentId) -> Option<ViewId> {
        if document_id >= self.documents.len() {
            return None;
        }
        
        let view_id = self.views.len();
        self.views.push(View {
            document_id,
            cursor_pos: 0,
        });
        
        self.documents[document_id].views.push(view_id);
        Some(view_id)
    }
    
    fn update_view_cursor(&mut self, view_id: ViewId, new_pos: usize) -> bool {
        if let Some(view) = self.views.get_mut(view_id) {
            if let Some(doc) = self.documents.get(view.document_id) {
                if new_pos <= doc.content.len() {
                    view.cursor_pos = new_pos;
                    return true;
                }
            }
        }
        false
    }
    
    fn get_view_content(&self, view_id: ViewId) -> Option<&str> {
        let view = self.views.get(view_id)?;
        let doc = self.documents.get(view.document_id)?;
        Some(&doc.content)
    }
}

fn main() {
    let mut editor = Editor::new();
    
    let doc = editor.create_document("Hello, Rust!".to_string());
    let view1 = editor.create_view(doc).unwrap();
    let view2 = editor.create_view(doc).unwrap();
    
    editor.update_view_cursor(view1, 5);
    editor.update_view_cursor(view2, 7);
    
    println!("View 1 sees: {}", editor.get_view_content(view1).unwrap());
    println!("View 2 sees: {}", editor.get_view_content(view2).unwrap());
}
```

이 패턴은 중앙 컨테이너(`Editor`)가 모든 객체를 소유하고, 객체들은 ID를 통해 서로를 참조한다. 빌림 검사기의 제약을 우아하게 우회하면서도 타입 안전성을 유지한다.
  

## 5.2 부모-자식 관계의 구현

### 문제 상황: 트리 구조
부모 노드가 자식 노드들을 소유하고, 자식이 부모를 참조하는 트리 구조는 매우 흔하다.

```rust
// 컴파일 실패!
struct TreeNode {
    value: i32,
    parent: Option<&TreeNode>,  // 에러
    children: Vec<TreeNode>,
}
```

부모와 자식이 서로를 참조하는 순환 구조라 단순 참조로는 불가능하다.

### 해결책 1: 부모 참조 제거
가장 간단한 해결책은 부모 참조를 아예 제거하는 것이다.

```rust
#[derive(Debug)]
struct TreeNode {
    value: i32,
    children: Vec<TreeNode>,
}

impl TreeNode {
    fn new(value: i32) -> Self {
        TreeNode {
            value,
            children: Vec::new(),
        }
    }
    
    fn add_child(&mut self, child: TreeNode) {
        self.children.push(child);
    }
    
    fn find_path_to_value(&self, target: i32, path: &mut Vec<i32>) -> bool {
        path.push(self.value);
        
        if self.value == target {
            return true;
        }
        
        for child in &self.children {
            if child.find_path_to_value(target, path) {
                return true;
            }
        }
        
        path.pop();
        false
    }
}

fn main() {
    let mut root = TreeNode::new(1);
    
    let mut child1 = TreeNode::new(2);
    child1.add_child(TreeNode::new(4));
    child1.add_child(TreeNode::new(5));
    
    let mut child2 = TreeNode::new(3);
    child2.add_child(TreeNode::new(6));
    
    root.add_child(child1);
    root.add_child(child2);
    
    let mut path = Vec::new();
    if root.find_path_to_value(5, &mut path) {
        println!("Path to 5: {:?}", path);
    }
}
```

부모로 올라가는 탐색이 필요 없다면 이 방식으로 충분하다. 경로는 재귀 호출 스택이나 별도 벡터로 추적할 수 있다.

### 해결책 2: 인덱스 기반 아레나
부모 참조가 꼭 필요하다면 인덱스 기반 아레나를 사용한다.

```rust
type NodeId = usize;

#[derive(Debug)]
struct TreeNode {
    value: i32,
    parent: Option<NodeId>,
    children: Vec<NodeId>,
}

struct Tree {
    nodes: Vec<TreeNode>,
}

impl Tree {
    fn new() -> Self {
        Tree { nodes: Vec::new() }
    }
    
    fn create_node(&mut self, value: i32, parent: Option<NodeId>) -> NodeId {
        let id = self.nodes.len();
        self.nodes.push(TreeNode {
            value,
            parent,
            children: Vec::new(),
        });
        
        if let Some(parent_id) = parent {
            if parent_id < self.nodes.len() {
                self.nodes[parent_id].children.push(id);
            }
        }
        
        id
    }
    
    fn get_ancestors(&self, node_id: NodeId) -> Vec<i32> {
        let mut ancestors = Vec::new();
        let mut current = node_id;
        
        while let Some(node) = self.nodes.get(current) {
            if let Some(parent_id) = node.parent {
                if let Some(parent) = self.nodes.get(parent_id) {
                    ancestors.push(parent.value);
                    current = parent_id;
                } else {
                    break;
                }
            } else {
                break;
            }
        }
        
        ancestors
    }
    
    fn get_descendants(&self, node_id: NodeId) -> Vec<i32> {
        let mut descendants = Vec::new();
        self.collect_descendants(node_id, &mut descendants);
        descendants
    }
    
    fn collect_descendants(&self, node_id: NodeId, result: &mut Vec<i32>) {
        if let Some(node) = self.nodes.get(node_id) {
            for &child_id in &node.children {
                if let Some(child) = self.nodes.get(child_id) {
                    result.push(child.value);
                    self.collect_descendants(child_id, result);
                }
            }
        }
    }
    
    fn print_tree(&self, node_id: NodeId, indent: usize) {
        if let Some(node) = self.nodes.get(node_id) {
            println!("{}{}", "  ".repeat(indent), node.value);
            for &child_id in &node.children {
                self.print_tree(child_id, indent + 1);
            }
        }
    }
}

fn main() {
    let mut tree = Tree::new();
    
    let root = tree.create_node(1, None);
    let child1 = tree.create_node(2, Some(root));
    let child2 = tree.create_node(3, Some(root));
    let grandchild1 = tree.create_node(4, Some(child1));
    let grandchild2 = tree.create_node(5, Some(child1));
    let grandchild3 = tree.create_node(6, Some(child2));
    
    println!("Tree structure:");
    tree.print_tree(root, 0);
    
    println!("\nAncestors of node 4:");
    println!("{:?}", tree.get_ancestors(grandchild1));
    
    println!("\nDescendants of node 2:");
    println!("{:?}", tree.get_descendants(child1));
}
```

이 패턴은 매우 강력하다. 부모와 자식 양방향 순회가 모두 가능하고, 빌림 충돌도 없으며, 메모리 안전성도 보장된다.

### 해결책 3: Rc와 Weak 조합 (미리보기)
나중에 자세히 다루겠지만, `Rc`와 `Weak`를 조합하면 참조 카운팅 기반 트리를 만들 수 있다.

```rust
use std::rc::{Rc, Weak};
use std::cell::RefCell;

struct TreeNode {
    value: i32,
    parent: RefCell<Weak<TreeNode>>,
    children: RefCell<Vec<Rc<TreeNode>>>,
}

impl TreeNode {
    fn new(value: i32) -> Rc<Self> {
        Rc::new(TreeNode {
            value,
            parent: RefCell::new(Weak::new()),
            children: RefCell::new(Vec::new()),
        })
    }
    
    fn add_child(parent: &Rc<TreeNode>, value: i32) -> Rc<TreeNode> {
        let child = TreeNode::new(value);
        *child.parent.borrow_mut() = Rc::downgrade(parent);
        parent.children.borrow_mut().push(Rc::clone(&child));
        child
    }
}
```

이 방식은 복잡하지만 동적 트리 구조가 필요할 때 유용하다.
  

## 5.3 그래프 구조 표현하기

### 문제 상황: 일반 그래프
트리보다 더 복잡한 그래프 구조는 노드 간 임의의 연결을 허용한다.

```rust
// 작동하지 않음
struct GraphNode {
    value: i32,
    neighbors: Vec<&GraphNode>,  // 에러
}
```

### 해결책: 인접 리스트와 인덱스

그래프는 인접 리스트 방식으로 표현하는 것이 가장 자연스럽다.

```rust
type NodeId = usize;

#[derive(Debug, Clone)]
struct Node {
    value: i32,
}

struct Graph {
    nodes: Vec<Node>,
    edges: Vec<Vec<NodeId>>,  // edges[i]는 노드 i의 이웃들
}

impl Graph {
    fn new() -> Self {
        Graph {
            nodes: Vec::new(),
            edges: Vec::new(),
        }
    }
    
    fn add_node(&mut self, value: i32) -> NodeId {
        let id = self.nodes.len();
        self.nodes.push(Node { value });
        self.edges.push(Vec::new());
        id
    }
    
    fn add_edge(&mut self, from: NodeId, to: NodeId) {
        if from < self.nodes.len() && to < self.nodes.len() {
            self.edges[from].push(to);
        }
    }
    
    fn add_bidirectional_edge(&mut self, a: NodeId, b: NodeId) {
        self.add_edge(a, b);
        self.add_edge(b, a);
    }
    
    fn neighbors(&self, node_id: NodeId) -> &[NodeId] {
        if node_id < self.edges.len() {
            &self.edges[node_id]
        } else {
            &[]
        }
    }
    
    fn bfs(&self, start: NodeId) -> Vec<i32> {
        use std::collections::VecDeque;
        
        let mut visited = vec![false; self.nodes.len()];
        let mut queue = VecDeque::new();
        let mut result = Vec::new();
        
        queue.push_back(start);
        visited[start] = true;
        
        while let Some(current) = queue.pop_front() {
            if let Some(node) = self.nodes.get(current) {
                result.push(node.value);
                
                for &neighbor in self.neighbors(current) {
                    if !visited[neighbor] {
                        visited[neighbor] = true;
                        queue.push_back(neighbor);
                    }
                }
            }
        }
        
        result
    }
    
    fn dfs(&self, start: NodeId) -> Vec<i32> {
        let mut visited = vec![false; self.nodes.len()];
        let mut result = Vec::new();
        self.dfs_recursive(start, &mut visited, &mut result);
        result
    }
    
    fn dfs_recursive(&self, current: NodeId, visited: &mut Vec<bool>, result: &mut Vec<i32>) {
        if visited[current] {
            return;
        }
        
        visited[current] = true;
        
        if let Some(node) = self.nodes.get(current) {
            result.push(node.value);
        }
        
        for &neighbor in self.neighbors(current) {
            self.dfs_recursive(neighbor, visited, result);
        }
    }
    
    fn find_path(&self, start: NodeId, end: NodeId) -> Option<Vec<NodeId>> {
        use std::collections::{VecDeque, HashMap};
        
        let mut visited = vec![false; self.nodes.len()];
        let mut queue = VecDeque::new();
        let mut parent: HashMap<NodeId, NodeId> = HashMap::new();
        
        queue.push_back(start);
        visited[start] = true;
        
        while let Some(current) = queue.pop_front() {
            if current == end {
                // 경로 재구성
                let mut path = Vec::new();
                let mut node = end;
                path.push(node);
                
                while node != start {
                    if let Some(&p) = parent.get(&node) {
                        path.push(p);
                        node = p;
                    } else {
                        break;
                    }
                }
                
                path.reverse();
                return Some(path);
            }
            
            for &neighbor in self.neighbors(current) {
                if !visited[neighbor] {
                    visited[neighbor] = true;
                    parent.insert(neighbor, current);
                    queue.push_back(neighbor);
                }
            }
        }
        
        None
    }
}

fn main() {
    let mut graph = Graph::new();
    
    // 노드 생성
    let n0 = graph.add_node(0);
    let n1 = graph.add_node(1);
    let n2 = graph.add_node(2);
    let n3 = graph.add_node(3);
    let n4 = graph.add_node(4);
    
    // 간선 추가
    graph.add_bidirectional_edge(n0, n1);
    graph.add_bidirectional_edge(n0, n2);
    graph.add_bidirectional_edge(n1, n3);
    graph.add_bidirectional_edge(n2, n3);
    graph.add_bidirectional_edge(n3, n4);
    
    println!("BFS from 0: {:?}", graph.bfs(n0));
    println!("DFS from 0: {:?}", graph.dfs(n0));
    
    if let Some(path) = graph.find_path(n0, n4) {
        println!("Path from 0 to 4: {:?}", path);
    }
}
```

이 구현은 효율적이고 안전하며, 대부분의 그래프 알고리즘을 구현할 수 있다.

### 실전 예제: 소셜 네트워크 그래프

```rust
#[derive(Debug, Clone)]
struct User {
    name: String,
    email: String,
}

struct SocialNetwork {
    users: Vec<User>,
    friendships: Vec<Vec<NodeId>>,
}

impl SocialNetwork {
    fn new() -> Self {
        SocialNetwork {
            users: Vec::new(),
            friendships: Vec::new(),
        }
    }
    
    fn add_user(&mut self, name: String, email: String) -> NodeId {
        let id = self.users.len();
        self.users.push(User { name, email });
        self.friendships.push(Vec::new());
        id
    }
    
    fn add_friendship(&mut self, user1: NodeId, user2: NodeId) {
        if user1 < self.users.len() && user2 < self.users.len() && user1 != user2 {
            self.friendships[user1].push(user2);
            self.friendships[user2].push(user1);
        }
    }
    
    fn get_friends(&self, user_id: NodeId) -> Vec<&User> {
        let mut friends = Vec::new();
        if let Some(friend_ids) = self.friendships.get(user_id) {
            for &friend_id in friend_ids {
                if let Some(user) = self.users.get(friend_id) {
                    friends.push(user);
                }
            }
        }
        friends
    }
    
    fn find_mutual_friends(&self, user1: NodeId, user2: NodeId) -> Vec<&User> {
        use std::collections::HashSet;
        
        let friends1: HashSet<NodeId> = self.friendships
            .get(user1)
            .map(|v| v.iter().copied().collect())
            .unwrap_or_default();
        
        let friends2: HashSet<NodeId> = self.friendships
            .get(user2)
            .map(|v| v.iter().copied().collect())
            .unwrap_or_default();
        
        friends1.intersection(&friends2)
            .filter_map(|&id| self.users.get(id))
            .collect()
    }
    
    fn suggest_friends(&self, user_id: NodeId) -> Vec<(NodeId, &User, usize)> {
        use std::collections::HashMap;
        
        let friends: HashSet<NodeId> = self.friendships
            .get(user_id)
            .map(|v| v.iter().copied().collect())
            .unwrap_or_default();
        
        let mut mutual_count: HashMap<NodeId, usize> = HashMap::new();
        
        // 친구의 친구들을 세기
        for &friend_id in &friends {
            if let Some(friends_of_friend) = self.friendships.get(friend_id) {
                for &fof_id in friends_of_friend {
                    if fof_id != user_id && !friends.contains(&fof_id) {
                        *mutual_count.entry(fof_id).or_insert(0) += 1;
                    }
                }
            }
        }
        
        let mut suggestions: Vec<_> = mutual_count
            .into_iter()
            .filter_map(|(id, count)| {
                self.users.get(id).map(|user| (id, user, count))
            })
            .collect();
        
        suggestions.sort_by(|a, b| b.2.cmp(&a.2));
        suggestions
    }
}

fn main() {
    let mut network = SocialNetwork::new();
    
    let alice = network.add_user("Alice".to_string(), "alice@example.com".to_string());
    let bob = network.add_user("Bob".to_string(), "bob@example.com".to_string());
    let carol = network.add_user("Carol".to_string(), "carol@example.com".to_string());
    let dave = network.add_user("Dave".to_string(), "dave@example.com".to_string());
    
    network.add_friendship(alice, bob);
    network.add_friendship(alice, carol);
    network.add_friendship(bob, carol);
    network.add_friendship(bob, dave);
    
    println!("Alice's friends:");
    for friend in network.get_friends(alice) {
        println!("  - {}", friend.name);
    }
    
    println!("\nMutual friends of Alice and Bob:");
    for friend in network.find_mutual_friends(alice, bob) {
        println!("  - {}", friend.name);
    }
    
    println!("\nFriend suggestions for Alice:");
    for (_, user, count) in network.suggest_friends(alice) {
        println!("  - {} ({} mutual friends)", user.name, count);
    }
}
```

이 패턴은 실전에서 매우 유용하다. 참조 대신 인덱스를 사용하므로 빌림 문제가 전혀 없고, 알고리즘 구현도 명확하다.
  

## 5.4 Rc와 RefCell을 사용한 해결책

### Rc의 개념
`Rc`(Reference Counted)는 참조 카운팅을 통해 여러 소유자가 같은 데이터를 공유할 수 있게 한다. C++의 `shared_ptr`과 비슷하다.

```rust
use std::rc::Rc;

fn basic_rc_example() {
    let data = Rc::new(vec![1, 2, 3, 4, 5]);
    
    let ref1 = Rc::clone(&data);
    let ref2 = Rc::clone(&data);
    
    println!("Reference count: {}", Rc::strong_count(&data));  // 3
    println!("Data: {:?}", data);
    
    drop(ref1);
    println!("After dropping ref1: {}", Rc::strong_count(&data));  // 2
}
```

`Rc::clone()`은 포인터만 복사하고 참조 카운트를 증가시킨다. 실제 데이터 복사는 일어나지 않는다.

### RefCell의 개념
`RefCell`은 내부 가변성을 제공한다. 컴파일 타임이 아닌 런타임에 빌림 규칙을 검사한다.

```rust
use std::cell::RefCell;

fn basic_refcell_example() {
    let data = RefCell::new(vec![1, 2, 3]);
    
    // 불변 빌림
    {
        let borrowed = data.borrow();
        println!("Data: {:?}", *borrowed);
    }  // borrowed 스코프 종료
    
    // 가변 빌림
    {
        let mut borrowed = data.borrow_mut();
        borrowed.push(4);
    }
    
    println!("Modified: {:?}", data.borrow());
}
```

`borrow()`는 `Ref<T>`를, `borrow_mut()`는 `RefMut<T>`를 반환한다. 빌림 규칙 위반 시 런타임 패닉이 발생한다.

### Rc와 RefCell 조합
둘을 조합하면 공유 가능하고 수정도 가능한 데이터를 만들 수 있다.

```rust
use std::rc::Rc;
use std::cell::RefCell;

#[derive(Debug)]
struct SharedCounter {
    value: RefCell<i32>,
}

impl SharedCounter {
    fn new(initial: i32) -> Rc<Self> {
        Rc::new(SharedCounter {
            value: RefCell::new(initial),
        })
    }
    
    fn increment(&self) {
        *self.value.borrow_mut() += 1;
    }
    
    fn get(&self) -> i32 {
        *self.value.borrow()
    }
}

fn main() {
    let counter = SharedCounter::new(0);
    let counter_ref1 = Rc::clone(&counter);
    let counter_ref2 = Rc::clone(&counter);
    
    counter.increment();
    counter_ref1.increment();
    counter_ref2.increment();
    
    println!("Final value: {}", counter.get());  // 3
}
```

### 실전 예제: 옵저버 패턴
옵저버 패턴은 객체 간 참조가 복잡하게 얽힌 대표적인 사례다.

```rust
use std::rc::Rc;
use std::cell::RefCell;

trait Observer {
    fn update(&self, value: i32);
}

struct Subject {
    value: RefCell<i32>,
    observers: RefCell<Vec<Rc<dyn Observer>>>,
}

impl Subject {
    fn new(initial: i32) -> Rc<Self> {
        Rc::new(Subject {
            value: RefCell::new(initial),
            observers: RefCell::new(Vec::new()),
        })
    }
    
    fn attach(&self, observer: Rc<dyn Observer>) {
        self.observers.borrow_mut().push(observer);
    }
    
    fn set_value(&self, new_value: i32) {
        *self.value.borrow_mut() = new_value;
        self.notify();
    }
    
    fn notify(&self) {
        let value = *self.value.borrow();
        for observer in self.observers.borrow().iter() {
            observer.update(value);
        }
    }
}

struct ConcreteObserver {
    name: String,
}

impl ConcreteObserver {
    fn new(name: &str) -> Rc<Self> {
        Rc::new(ConcreteObserver {
            name: name.to_string(),
        })
    }
}

impl Observer for ConcreteObserver {
    fn update(&self, value: i32) {
        println!("{} received update: {}", self.name, value);
    }
}

fn main() {
    let subject = Subject::new(0);
    
    let observer1 = ConcreteObserver::new("Observer1");
    let observer2 = ConcreteObserver::new("Observer2");
    
    subject.attach(observer1);
    subject.attach(observer2);
    
    subject.set_value(10);
    subject.set_value(20);
}
```

이 패턴은 GUI 프레임워크나 이벤트 시스템에서 자주 사용된다.

### 주의사항: 런타임 패닉
`RefCell`은 런타임에 검사하므로 잘못 사용하면 패닉이 발생한다.

```rust
use std::cell::RefCell;

fn panic_example() {
    let data = RefCell::new(vec![1, 2, 3]);
    
    let _borrow1 = data.borrow();
    let _borrow2 = data.borrow();  // OK: 여러 불변 빌림
    
    // let _borrow_mut = data.borrow_mut();  // 패닉! 이미 불변 빌림 중
}

fn another_panic() {
    let data = RefCell::new(vec![1, 2, 3]);
    
    let _borrow_mut1 = data.borrow_mut();
    // let _borrow_mut2 = data.borrow_mut();  // 패닉! 이미 가변 빌림 중
}
```

이런 패닉을 피하려면 빌림 스코프를 신중하게 관리해야 한다.
  

## 5.5 Weak 참조로 순환 참조 방지하기

### 문제 상황: 순환 참조로 인한 메모리 누수
`Rc`만 사용하면 순환 참조가 생겨 메모리가 누수될 수 있다.

```rust
use std::rc::Rc;
use std::cell::RefCell;

struct Node {
    value: i32,
    next: RefCell<Option<Rc<Node>>>,
}

fn create_cycle() {
    let first = Rc::new(Node {
        value: 1,
        next: RefCell::new(None),
    });
    
    let second = Rc::new(Node {
        value: 2,
        next: RefCell::new(Some(Rc::clone(&first))),
    });
    
    *first.next.borrow_mut() = Some(Rc::clone(&second));
    
    println!("first rc count: {}", Rc::strong_count(&first));  // 2
    println!("second rc count: {}", Rc::strong_count(&second));  // 2
    
    // first와 second가 스코프를 벗어나도 참조 카운트가 0이 되지 않음
    // 메모리 누수 발생!
}
```

### Weak 참조의 개념
`Weak`는 참조 카운트를 증가시키지 않는 약한 참조다. 순환 참조의 일부를 `Weak`로 바꾸면 메모리 누수를 방지할 수 있다.

```rust
use std::rc::{Rc, Weak};
use std::cell::RefCell;

struct Node {
    value: i32,
    next: RefCell<Option<Rc<Node>>>,
    prev: RefCell<Weak<Node>>,  // Weak 사용
}

fn no_cycle() {
    let first = Rc::new(Node {
        value: 1,
        next: RefCell::new(None),
        prev: RefCell::new(Weak::new()),
    });
    
    let second = Rc::new(Node {
        value: 2,
        next: RefCell::new(None),
        prev: RefCell::new(Rc::downgrade(&first)),  // Weak 참조 생성
    });
    
    *first.next.borrow_mut() = Some(Rc::clone(&second));
    
    println!("first strong count: {}", Rc::strong_count(&first));  // 1
    println!("first weak count: {}", Rc::weak_count(&first));  // 1
    println!("second strong count: {}", Rc::strong_count(&second));  // 2
    
    // 스코프를 벗어나면 정상적으로 해제됨
}
```

`Weak`에서 실제 데이터를 접근하려면 `upgrade()`로 `Option<Rc<T>>`를 얻어야 한다.

```rust
fn access_weak() {
    let strong = Rc::new(42);
    let weak = Rc::downgrade(&strong);
    
    // Weak에서 Strong으로 업그레이드
    if let Some(value) = weak.upgrade() {
        println!("Value is still alive: {}", value);
    } else {
        println!("Value has been dropped");
    }
    
    drop(strong);
    
    // Strong 참조가 사라진 후
    if weak.upgrade().is_none() {
        println!("Now the value is gone");
    }
}
```

### 실전 예제: 부모-자식 트리 (개선된 버전)

```rust
use std::rc::{Rc, Weak};
use std::cell::RefCell;

struct TreeNode {
    value: i32,
    parent: RefCell<Weak<TreeNode>>,
    children: RefCell<Vec<Rc<TreeNode>>>,
}

impl TreeNode {
    fn new(value: i32) -> Rc<Self> {
        Rc::new(TreeNode {
            value,
            parent: RefCell::new(Weak::new()),
            children: RefCell::new(Vec::new()),
        })
    }
    
    fn add_child(parent: &Rc<TreeNode>, value: i32) -> Rc<TreeNode> {
        let child = TreeNode::new(value);
        *child.parent.borrow_mut() = Rc::downgrade(parent);
        parent.children.borrow_mut().push(Rc::clone(&child));
        child
    }
    
    fn get_path_to_root(&self) -> Vec<i32> {
        let mut path = vec![self.value];
        let mut current = self.parent.borrow().upgrade();
        
        while let Some(parent) = current {
            path.push(parent.value);
            current = parent.parent.borrow().upgrade();
        }
        
        path.reverse();
        path
    }
    
    fn print_tree(&self, indent: usize) {
        println!("{}{}", "  ".repeat(indent), self.value);
        for child in self.children.borrow().iter() {
            child.print_tree(indent + 1);
        }
    }
}

fn main() {
    let root = TreeNode::new(1);
    let child1 = TreeNode::add_child(&root, 2);
    let child2 = TreeNode::add_child(&root, 3);
    let grandchild = TreeNode::add_child(&child1, 4);
    
    println!("Tree structure:");
    root.print_tree(0);
    
    println!("\nPath from grandchild to root:");
    println!("{:?}", grandchild.get_path_to_root());
    
    println!("\nReference counts:");
    println!("root strong: {}", Rc::strong_count(&root));
    println!("child1 strong: {}", Rc::strong_count(&child1));
    println!("child1 weak: {}", Rc::weak_count(&child1));
}
```

자식은 부모를 `Weak`로 참조하므로 순환 참조가 생기지 않는다. 부모가 사라지면 자식들도 함께 정리된다.

### 복잡한 예제: 양방향 연결 리스트

```rust
use std::rc::{Rc, Weak};
use std::cell::RefCell;

struct ListNode {
    value: i32,
    next: RefCell<Option<Rc<ListNode>>>,
    prev: RefCell<Weak<ListNode>>,
}

struct LinkedList {
    head: Option<Rc<ListNode>>,
    tail: Weak<ListNode>,
}

impl LinkedList {
    fn new() -> Self {
        LinkedList {
            head: None,
            tail: Weak::new(),
        }
    }
    
    fn append(&mut self, value: i32) {
        let new_node = Rc::new(ListNode {
            value,
            next: RefCell::new(None),
            prev: RefCell::new(Weak::new()),
        });
        
        if let Some(old_tail) = self.tail.upgrade() {
            *old_tail.next.borrow_mut() = Some(Rc::clone(&new_node));
            *new_node.prev.borrow_mut() = Rc::downgrade(&old_tail);
        } else {
            self.head = Some(Rc::clone(&new_node));
        }
        
        self.tail = Rc::downgrade(&new_node);
    }
    
    fn traverse_forward(&self) -> Vec<i32> {
        let mut result = Vec::new();
        let mut current = self.head.as_ref().map(|n| Rc::clone(n));
        
        while let Some(node) = current {
            result.push(node.value);
            current = node.next.borrow().as_ref().map(|n| Rc::clone(n));
        }
        
        result
    }
    
    fn traverse_backward(&self) -> Vec<i32> {
        let mut result = Vec::new();
        let mut current = self.tail.upgrade();
        
        while let Some(node) = current {
            result.push(node.value);
            current = node.prev.borrow().upgrade();
        }
        
        result
    }
}

fn main() {
    let mut list = LinkedList::new();
    
    list.append(10);
    list.append(20);
    list.append(30);
    list.append(40);
    
    println!("Forward: {:?}", list.traverse_forward());
    println!("Backward: {:?}", list.traverse_backward());
}
```

### 언제 Rc/Weak를 사용할까
`Rc`와 `Weak`는 강력하지만 비용이 있다. 다음 경우에 고려한다:

1. 소유권이 명확하지 않은 공유 데이터
2. 순환 참조가 필요한 복잡한 그래프 구조
3. 동적으로 변하는 참조 관계

하지만 가능하면 인덱스 기반 접근을 먼저 고려하는 것이 좋다. 인덱스 방식이 더 간단하고 효율적인 경우가 많다.

객체 간 참조 관계를 설계할 때는 먼저 소유권 구조를 명확히 하고, 정말 양방향 참조가 필요한지 검토하며, 필요하다면 인덱스나 `Rc`/`Weak` 중 상황에 맞는 방법을 선택해야 한다. Rust의 제약은 처음엔 불편하지만, 설계를 더 명확하고 안전하게 만들어준다.
