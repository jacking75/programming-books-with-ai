## 이 책에 대하여

이 책은 **23년 경력의 게임 서버 개발자**가 Rust를 빠르게 습득하고 AI(바이브 코딩)와 함께 효과적으로 사용하는 방법을 다룬다.

### 대상 독자
- C++ 또는 C# 경력자로 Rust를 처음 배우는 개발자
- 게임 서버, 백엔드 서버를 Rust로 구현하려는 개발자
- AI를 활용한 Rust 개발(바이브 코딩)에 관심 있는 개발자

### 전제 지식
- C++ 또는 C# 중급 이상
- 멀티스레드·네트워크 프로그래밍 기초
- Git, 빌드 시스템 사용 경험

### 학습 경로

```
사고방식 이해 (Ch.01)
    ↓
핵심 개념 습득 (Ch.02~04)
    ↓
실전 패턴 학습 (Ch.05~08)
    ↓
생태계 파악 (Ch.09~11)
    ↓
AI 협업 마스터 (Ch.12~14)
```

### 각 챕터 구성

모든 챕터는 다음 구조를 따른다:

1. **개념 설명** — Rust의 철학과 설계 의도
2. **코드 예제** — C++/C# 방식 → Rust 올바른 방식 대조
3. **게임 서버 적용** — 실제 서버 코드에서의 활용
4. **AI 지시 방법** — 바이브 코딩에서 이 개념을 다루는 법
5. **핵심 정리** — 챕터 요약과 체크리스트

### 동시성 타입 선택 가이드

| 상황 | 타입 |
|------|------|
| 단일 스레드, 단순 소유권 | `Box<T>` |
| 단일 스레드, 공유 참조 | `Rc<T>` |
| 멀티 스레드, 공유 소유권 | `Arc<T>` |
| 멀티 스레드, 독점 쓰기 | `Arc<Mutex<T>>` |
| 멀티 스레드, 읽기 중심 | `Arc<RwLock<T>>` |
| 고성능 동시성 맵 | `DashMap<K,V>` |

### 채널 선택 가이드

| 패턴 | 채널 |
|------|------|
| 태스크 큐, 이벤트 디스패치 | `mpsc` |
| 요청-응답, RPC 결과 반환 | `oneshot` |
| 게임 이벤트 브로드캐스트 | `broadcast` |
| 설정값 공유, 상태 모니터링 | `watch` |

### 에러 처리 계층

```
라이브러리 크레이트: thiserror (타입 안전 에러)
    ↓
애플리케이션 크레이트: anyhow (컨텍스트 추가)
    ↓
최상위 핸들러: 로깅 후 graceful shutdown
```
