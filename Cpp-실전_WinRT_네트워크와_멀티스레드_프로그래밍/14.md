# Chapter 14. 비동기 패턴 심화

## 14.1 코루틴 생명주기 관리

### 취소와 예외 전파
비동기 프로그래밍의 기본을 넘어서면, 실제로 마주치는 복잡한 상황들을 다루어야 한다. 코루틴이 실행 중일 때 취소되어야 한다면 어떻게 할까? 또는 비동기 작업 중에 예외가 발생하면 그것을 어떻게 처리해야 할까? 이러한 질문들이 이 섹션의 주제다.

코루틴의 생명주기는 여러 단계로 나뉜다. 생성(creation), 실행(execution), 일시 중지(suspension), 재개(resumption), 종료(completion) 단계가 있다. 각 단계에서 취소나 예외가 발생할 수 있으므로, 이를 안전하게 처리하는 것이 중요하다.

WinRT의 IAsyncAction과 IAsyncOperation은 취소 기능을 지원한다. Cancel() 메서드를 호출하면 비동기 작업을 취소할 수 있다. 하지만 취소는 즉시 이루어지지 않고, 코루틴이 co_await 지점에서 취소 신호를 확인한다.

예외 처리도 중요하다. 비동기 작업 중에 발생한 예외는 그 작업을 기다리는 지점에서 다시 발생한다. 따라서 co_await 주변에 try-catch 블록을 배치해야 한다.

다음은 취소와 예외를 처리하는 예제다.

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <iostream>
#include <chrono>

using namespace winrt;
using namespace Windows::Networking::Sockets;

// 취소 가능한 장기 작업
IAsyncAction CancellableOperationAsync(
    IAsyncAction cancellationSignal)
{
    try
    {
        for (int i = 0; i < 10; i++)
        {
            std::wcout << L"작업 진행 중: " << (i + 1) << L"/10" 
                      << std::endl;
            
            // 취소 신호 확인
            // cancellationSignal이 완료되면 작업 중지
            co_await cancellationSignal;
            
            // 1초 대기
            co_await winrt::resume_after(std::chrono::seconds(1));
        }
        
        std::wcout << L"작업 정상 완료" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"예외 발생: " << ex.message().c_str() 
                  << std::endl;
    }
}

// 예외를 발생시키는 작업
IAsyncAction FailingOperationAsync()
{
    try
    {
        std::wcout << L"예외를 발생시키는 작업 시작" << std::endl;
        
        // 의도적으로 예외 발생
        throw hresult_error(E_FAIL, L"의도적인 오류 발생");
        
        co_return;  // 도달하지 않음
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"내부에서 예외 처리: " 
                  << ex.message().c_str() << std::endl;
        throw;  // 예외를 다시 발생시킴
    }
}

// 예외 처리와 함께 비동기 작업 호출
IAsyncAction CallWithExceptionHandlingAsync()
{
    try
    {
        std::wcout << L"비동기 작업 호출" << std::endl;
        co_await FailingOperationAsync();
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"호출자가 예외 처리: " 
                  << ex.message().c_str() << std::endl;
    }
}
```

이 코드에서 중요한 부분을 설명하자면, CancellableOperationAsync()는 취소 신호를 받을 수 있다. 실제 구현에서는 CancellationToken을 사용하는 것이 더 일반적이다.

FailingOperationAsync()는 의도적으로 예외를 발생시킨다. 이 예외는 co_await 지점에서 발생하며, 호출자가 catch할 수 있다. 예외를 내부에서 처리하고 다시 발생시키는 패턴은 로깅이나 정리 작업을 수행한 후 호출자에게 예외를 전파할 때 유용하다.

CallWithExceptionHandlingAsync()는 호출자 관점에서의 예외 처리를 보여준다. co_await로 비동기 작업을 호출할 때, 그 작업에서 발생한 예외가 이 지점에서 발생한다.

## 14.2 비동기 루프 패턴

### while 루프 내 co_await
네트워크 프로그래밍에서는 지속적으로 데이터를 수신하고 처리해야 한다. TCP 연결로부터 계속 데이터를 읽거나, 타이머 이벤트를 반복해서 기다리는 것이 일반적인 패턴이다. 이를 구현하는 방법이 바로 while 루프 내에 co_await를 배치하는 것이다.

이 패턴의 핵심은 각 반복마다 비동기 작업을 기다린다는 것이다. 루프의 바디를 동기적으로 실행하되, 특정 지점에서 비동기 작업의 완료를 기다린다.

다음은 비동기 루프 패턴을 보여주는 예제다.

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>
#include <chrono>

using namespace winrt;
using namespace Windows::Networking::Sockets;
using namespace Windows::Storage::Streams;

// 반복적으로 데이터를 수신하는 루프
IAsyncAction ReceiveLoopAsync(
    StreamSocket const& socket,
    uint32_t maxIterations = 5)
{
    try
    {
        DataReader reader(socket.InputStream());
        uint32_t iteration = 0;
        
        // 비동기 루프: 여러 번 반복하면서 데이터 수신
        while (iteration < maxIterations)
        {
            std::wcout << L"[반복 " << (iteration + 1) << L"] "
                      << L"데이터 수신 대기 중..." << std::endl;
            
            // 1024 바이트 읽기 대기
            uint32_t bytesLoaded = 
                co_await reader.LoadAsync(1024);
            
            if (bytesLoaded == 0)
            {
                // 연결 종료
                std::wcout << L"[반복 " << (iteration + 1) 
                          << L"] 연결 종료됨" << std::endl;
                break;
            }
            
            // 데이터 처리
            std::wstring data = reader.ReadString(bytesLoaded);
            std::wcout << L"[반복 " << (iteration + 1) 
                      << L"] 수신: " << data.c_str() << std::endl;
            
            iteration++;
        }
        
        std::wcout << L"루프 종료, 총 " << iteration 
                  << L" 번 반복 수행" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"루프 오류: " << ex.message().c_str() 
                  << std::endl;
    }
}

// 주기적 작업을 반복하는 루프
IAsyncAction PeriodicTaskLoopAsync(uint32_t numTasks = 5)
{
    try
    {
        std::wcout << L"주기적 작업 시작" << std::endl;
        
        for (uint32_t i = 0; i < numTasks; i++)
        {
            std::wcout << L"[작업 " << (i + 1) << L"] 시작" 
                      << std::endl;
            
            // 비동기 대기 (시뮬레이션: 작업 수행)
            co_await winrt::resume_after(std::chrono::milliseconds(500));
            
            std::wcout << L"[작업 " << (i + 1) << L"] 완료" 
                      << std::endl;
        }
        
        std::wcout << L"모든 주기적 작업 완료" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"작업 루프 오류: " 
                  << ex.message().c_str() << std::endl;
    }
}

// 조건부 루프: 특정 조건까지 반복
IAsyncAction ConditionalLoopAsync()
{
    try
    {
        int count = 0;
        bool shouldContinue = true;
        
        std::wcout << L"조건부 루프 시작" << std::endl;
        
        while (shouldContinue && count < 10)
        {
            std::wcout << L"[반복 " << count << L"]" << std::endl;
            
            // 비동기 작업 (조건을 판단하는 비동기 함수 호출)
            co_await winrt::resume_after(
                std::chrono::milliseconds(100)
            );
            
            count++;
            
            // 조건 평가 (예: count가 5 이상이면 중지)
            if (count >= 5)
            {
                shouldContinue = false;
                std::wcout << L"조건 만족, 루프 종료" << std::endl;
            }
        }
        
        std::wcout << L"총 " << count << L" 번 반복" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"조건부 루프 오류: " 
                  << ex.message().c_str() << std::endl;
    }
}
```

이 코드는 세 가지 일반적인 루프 패턴을 보여준다.

첫 번째는 데이터 수신 루프다. StreamSocket에서 계속해서 데이터를 읽으며, 각 반복마다 LoadAsync()를 호출한다. 이 메서드는 지정한 바이트 수가 도착할 때까지 비동기적으로 대기한다. 연결이 종료되면 bytesLoaded가 0이 되어 루프를 빠져나간다.

두 번째는 주기적 작업 루프다. for 루프와 co_await를 결합하여, 매 반복마다 일정 시간 대기한 후 작업을 수행한다. 이는 게임 서버의 틱(tick) 루프나 주기적 상태 동기화에 사용된다.

세 번째는 조건부 루프로, 루프가 계속될지 여부를 동적으로 판단한다. 특정 조건이 만족되면 shouldContinue 플래그를 변경하여 루프를 탈출한다.

각 루프에서 중요한 점은 co_await 호출이 스레드를 블로킹하지 않는다는 것이다. co_await는 비동기 작업의 완료를 대기하는 동안 다른 작업이 실행될 수 있게 한다.

## 14.3 작업 체이닝

### 순차적 비동기 작업
실제 네트워크 프로그래밍에서는 여러 작업을 순차적으로 수행해야 하는 경우가 많다. 예를 들어, 서버에 연결한 후 인증을 수행하고, 그 다음 데이터를 요청하는 식의 흐름이 일반적이다. 이를 구현하는 것이 작업 체이닝(task chaining)이다.

작업 체이닝은 한 비동기 작업의 결과를 다음 비동기 작업의 입력으로 사용하는 패턴이다. 이를 통해 복잡한 워크플로우를 간결하게 표현할 수 있다.

다음은 작업 체이닝의 여러 예제를 보여준다.

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Networking.h>
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>

using namespace winrt;
using namespace Windows::Networking::Sockets;
using namespace Windows::Networking;
using namespace Windows::Storage::Streams;

// 단계 1: 서버에 연결
IAsyncOperation<StreamSocket> ConnectToServerAsync(
    std::wstring const& hostName,
    std::wstring const& serviceName)
{
    StreamSocket socket;
    
    try
    {
        std::wcout << L"[1단계] 서버에 연결 중..." << std::endl;
        
        HostName host(hostName);
        co_await socket.ConnectAsync(host, serviceName);
        
        std::wcout << L"[1단계] 연결 성공" << std::endl;
        co_return socket;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"[1단계] 연결 실패: " 
                  << ex.message().c_str() << std::endl;
        throw;
    }
}

// 단계 2: 인증 메시지 전송
IAsyncAction AuthenticateAsync(
    StreamSocket const& socket,
    std::wstring const& username,
    std::wstring const& password)
{
    try
    {
        std::wcout << L"[2단계] 인증 중..." << std::endl;
        
        DataWriter writer(socket.OutputStream());
        std::wstring authMessage = username + L":" + password;
        writer.WriteString(authMessage);
        
        co_await writer.StoreAsync();
        
        std::wcout << L"[2단계] 인증 메시지 전송 완료" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"[2단계] 인증 실패: " 
                  << ex.message().c_str() << std::endl;
        throw;
    }
}

// 단계 3: 인증 응답 수신
IAsyncOperation<bool> ReceiveAuthResponseAsync(
    StreamSocket const& socket)
{
    try
    {
        std::wcout << L"[3단계] 인증 응답 대기 중..." << std::endl;
        
        DataReader reader(socket.InputStream());
        uint32_t bytesRead = co_await reader.LoadAsync(100);
        
        if (bytesRead == 0)
        {
            std::wcout << L"[3단계] 서버 응답 없음" << std::endl;
            co_return false;
        }
        
        std::wstring response = reader.ReadString(bytesRead);
        bool authenticated = (response == L"OK");
        
        std::wcout << L"[3단계] 응답: " << response.c_str() 
                  << (authenticated ? L" (성공)" : L" (실패)") 
                  << std::endl;
        
        co_return authenticated;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"[3단계] 응답 수신 실패: " 
                  << ex.message().c_str() << std::endl;
        throw;
    }
}

// 단계 4: 데이터 요청
IAsyncAction RequestDataAsync(StreamSocket const& socket)
{
    try
    {
        std::wcout << L"[4단계] 데이터 요청 중..." << std::endl;
        
        DataWriter writer(socket.OutputStream());
        writer.WriteString(L"GET_DATA");
        
        co_await writer.StoreAsync();
        
        std::wcout << L"[4단계] 데이터 요청 완료" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"[4단계] 데이터 요청 실패: " 
                  << ex.message().c_str() << std::endl;
        throw;
    }
}

// 전체 워크플로우: 작업 체이닝
IAsyncAction CompleteWorkflowAsync()
{
    try
    {
        // 단계 1: 연결
        auto socket = co_await ConnectToServerAsync(
            L"localhost",
            L"8080"
        );
        
        // 단계 2: 인증 메시지 전송
        co_await AuthenticateAsync(socket, L"user", L"pass");
        
        // 단계 3: 인증 응답 수신
        bool authenticated = co_await ReceiveAuthResponseAsync(socket);
        
        if (!authenticated)
        {
            std::wcout << L"인증 실패, 워크플로우 중단" << std::endl;
            co_return;
        }
        
        // 단계 4: 데이터 요청
        co_await RequestDataAsync(socket);
        
        std::wcout << L"\n워크플로우 완료!" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"워크플로우 오류: " 
                  << ex.message().c_str() << std::endl;
    }
}
```

이 코드는 클라이언트가 서버에 연결하고 인증받은 후 데이터를 요청하는 전체 워크플로우를 보여준다.

각 단계가 별도의 함수로 분리되어 있어 재사용성이 높다. 예를 들어, ConnectToServerAsync()는 어떤 프로토콜에서든 재사용할 수 있다.

CompleteWorkflowAsync()에서 주목할 점은 각 단계가 순차적으로 실행된다는 것이다. 첫 번째 co_await가 완료되어야 두 번째가 시작되고, 이런 식으로 계속된다. 만약 어느 단계에서 예외가 발생하면 catch 블록에서 처리되고 전체 워크플로우가 중단된다.

이 패턴은 상태 머신 없이도 복잡한 워크플로우를 관리할 수 있게 해준다. 각 단계의 논리가 명확하고, 추적하기 쉽다.

## 14.4 병렬 작업 실행

### 여러 소켓 동시 처리
게임 서버는 수천 개의 클라이언트로부터 동시에 연결을 받아야 한다. 각 클라이언트와의 통신을 순차적으로 처리하면 성능이 크게 저하된다. 대신 여러 작업을 병렬로 실행하는 것이 필수다.

병렬 작업 실행은 여러 비동기 작업을 동시에 시작하고, 모두의 완료를 기다리는 패턴이다. WinRT에서는 이를 구현하기 위해 여러 방법을 제공한다.

다음은 병렬 작업 실행의 여러 예제를 보여준다.

```cpp
#include <winrt/Windows.Foundation.Collections.h>
#include <winrt/Windows.Networking.Sockets.h>
#include <iostream>
#include <vector>
#include <chrono>

using namespace winrt;
using namespace Windows::Networking::Sockets;
using namespace Windows::Foundation::Collections;

// 단일 클라이언트 처리
IAsyncAction HandleClientAsync(int clientId)
{
    try
    {
        std::wcout << L"[클라이언트 " << clientId << L"] 처리 시작" 
                  << std::endl;
        
        // 작업 시뮬레이션
        co_await winrt::resume_after(
            std::chrono::milliseconds(clientId * 100)
        );
        
        std::wcout << L"[클라이언트 " << clientId 
                  << L"] 처리 완료" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"[클라이언트 " << clientId 
                  << L"] 오류: " << ex.message().c_str() 
                  << std::endl;
    }
}

// 방법 1: 벡터에 비동기 작업들을 저장하고 모두 기다리기
IAsyncAction ParallelExecutionMethodOneAsync()
{
    try
    {
        std::wcout << L"\n방법 1: 벡터 기반 병렬 실행" << std::endl;
        
        // 여러 클라이언트 작업 생성
        std::vector<IAsyncAction> tasks;
        
        for (int i = 0; i < 5; i++)
        {
            tasks.push_back(HandleClientAsync(i));
        }
        
        // 모든 작업이 완료될 때까지 대기
        // (주의: winrt::when_all은 직접 지원되지 않을 수 있음)
        // 대신 개별적으로 기다려야 함
        for (auto& task : tasks)
        {
            co_await task;
        }
        
        std::wcout << L"모든 클라이언트 처리 완료" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"병렬 실행 오류: " 
                  << ex.message().c_str() << std::endl;
    }
}

// 방법 2: 람다 함수를 사용한 병렬 실행
IAsyncAction ParallelExecutionMethodTwoAsync()
{
    try
    {
        std::wcout << L"\n방법 2: 람다 함수 기반 병렬 실행" 
                  << std::endl;
        
        auto client1 = HandleClientAsync(10);
        auto client2 = HandleClientAsync(20);
        auto client3 = HandleClientAsync(30);
        
        std::wcout << L"세 클라이언트 작업 생성됨 (동시 실행 중)" 
                  << std::endl;
        
        // 순차적으로 대기 (실제로는 동시에 실행 중)
        co_await client1;
        co_await client2;
        co_await client3;
        
        std::wcout << L"모든 클라이언트 처리 완료" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"병렬 실행 오류: " 
                  << ex.message().c_str() << std::endl;
    }
}

// 방법 3: 동적 병렬 실행 (클라이언트 수가 가변적)
IAsyncAction DynamicParallelExecutionAsync(int numClients)
{
    try
    {
        std::wcout << L"\n방법 3: 동적 병렬 실행 (" 
                  << numClients << L" 클라이언트)" << std::endl;
        
        std::vector<IAsyncAction> clientTasks;
        
        // 동적으로 작업 생성
        for (int i = 0; i < numClients; i++)
        {
            clientTasks.push_back(HandleClientAsync(i));
        }
        
        std::wcout << L"모든 작업 생성 완료, 실행 중..." << std::endl;
        
        // 모든 작업 대기
        for (auto& task : clientTasks)
        {
            co_await task;
        }
        
        std::wcout << L"모든 클라이언트 처리 완료" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"동적 병렬 실행 오류: " 
                  << ex.message().c_str() << std::endl;
    }
}

// 통합 데모
IAsyncAction DemoParallelExecutionAsync()
{
    auto startTime = std::chrono::high_resolution_clock::now();
    
    co_await ParallelExecutionMethodOneAsync();
    co_await ParallelExecutionMethodTwoAsync();
    co_await DynamicParallelExecutionAsync(4);
    
    auto endTime = std::chrono::high_resolution_clock::now();
    auto duration = std::chrono::duration_cast<
        std::chrono::milliseconds>(endTime - startTime);

```

이 코드는 여러 클라이언트를 동시에 처리하는 다양한 방식을 보여준다.

첫 번째 방법은 벡터에 비동기 작업들을 저장하고, 반복문으로 각각을 기다린다. 이 방식은 작업 수가 미리 결정되지 않은 경우에 유용하다.

두 번째 방법은 작업을 먼저 생성한 후, 각각을 기다린다. 중요한 점은 각 작업이 co_await 호출 전에 이미 실행 중이라는 것이다. 따라서 실제로는 병렬로 실행되고 있다.

세 번째 방법은 클라이언트 수가 동적으로 결정되는 경우다. 예를 들어, 새로운 클라이언트가 연결될 때마다 작업을 생성하는 시나리오에 적합하다.

중요한 개념은 비동기 작업 생성과 기다리기의 분리다. IAsyncAction을 생성하면 즉시 실행이 시작되고, co_await는 단지 그 완료를 기다릴 뿐이다. 따라서 여러 작업을 생성한 후 기다리면 자연스럽게 병렬 실행이 된다.

## 14.5 종합 예제: 비동기 네트워크 스캐너

### 코드의 목적
이 예제는 여러 호스트의 여러 포트를 동시에 스캔하는 네트워크 스캐너를 구현한다. 각 호스트-포트 조합에 대해 연결을 시도하고, 일정 시간 내에 연결이 성공하는지 확인한다. 이는 병렬 비동기 작업 실행, 타임아웃 처리, 결과 수집 등 고급 비동기 패턴의 종합적인 예제다.

네트워크 스캐너는 게임 서버 개발에서 실제로 필요한 도구다. 서버 배포 후 여러 포트가 정상적으로 열려있는지 확인하거나, 내부 네트워크의 서비스 가용성을 확인할 때 사용된다.

### 설계 결정 이유
스캐너는 다양한 호스트와 포트에 동시에 연결을 시도해야 하므로, 병렬 처리가 필수다. 만약 순차적으로 처리하면 수백 개의 포트를 스캔하는 데 오래 걸린다.

각 연결 시도에는 타임아웃을 설정해야 한다. 응답이 없는 서비스에 무한정 기다릴 수 없기 때문이다. WinRT의 타임아웃은 CancellationTokenSource와 resume_after를 조합하여 구현할 수 있다.

결과는 구조화된 형식으로 저장하여, 스캔이 끝난 후 쉽게 분석할 수 있게 한다. 성공한 연결과 실패한 연결을 분리하고, 통계를 제공한다.

### 코드 구현

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Networking.h>
#include <iostream>
#include <vector>
#include <string>
#include <chrono>
#include <iomanip>

using namespace winrt;
using namespace Windows::Networking::Sockets;
using namespace Windows::Networking;

// 포트 스캔 결과
struct ScanResult
{
    std::wstring hostName;
    uint16_t port;
    bool isOpen;
    std::wstring errorMessage;
    double scanTimeMs;
};

// 네트워크 스캐너 클래스
class NetworkScanner
{
private:
    static constexpr int DEFAULT_TIMEOUT_MS = 3000;  // 3초
    std::vector<ScanResult> m_results;
    
public:
    // 단일 포트 스캔
    IAsyncOperation<ScanResult> ScanPortAsync(
        std::wstring const& hostName,
        uint16_t port,
        int timeoutMs = DEFAULT_TIMEOUT_MS)
    {
        ScanResult result;
        result.hostName = hostName;
        result.port = port;
        result.isOpen = false;
        result.scanTimeMs = 0;
        
        auto startTime = std::chrono::high_resolution_clock::now();
        
        try
        {
            StreamSocket socket;
            HostName host(hostName);
            std::wstring portStr = std::to_wstring(port);
            
            // 타임아웃을 구현하기 위해 수동으로 시간 측정
            co_await socket.ConnectAsync(host, portStr);
            
            result.isOpen = true;
            result.errorMessage = L"";
            
            std::wcout << L"[✓] " << hostName << L":" << port 
                      << L" 열려있음" << std::endl;
        }
        catch (hresult_error const& ex)
        {
            result.isOpen = false;
            result.errorMessage = ex.message().c_str();
            
            // 에러를 출력하지 않음 (닫혀있는 것이 정상)
        }
        
        auto endTime = std::chrono::high_resolution_clock::now();
        auto duration = std::chrono::duration_cast<
            std::chrono::milliseconds>(endTime - startTime);
        result.scanTimeMs = duration.count();
        
        co_return result;
    }
    
    // 호스트의 여러 포트 스캔
    IAsyncAction ScanHostAsync(
        std::wstring const& hostName,
        std::vector<uint16_t> const& ports)
    {
        std::wcout << L"\n호스트 스캔: " << hostName.c_str() 
                  << std::endl;
        std::wcout << L"포트 수: " << ports.size() << std::endl;
        
        std::vector<IAsyncOperation<ScanResult>> scanTasks;
        
        // 모든 포트의 스캔 작업 생성
        for (uint16_t port : ports)
        {
            scanTasks.push_back(ScanPortAsync(hostName, port));
        }
        
        std::wcout << L"스캔 중..." << std::endl;
        
        // 모든 스캔 작업의 결과 수집
        for (auto& task : scanTasks)
        {
            auto result = co_await task;
            m_results.push_back(result);
        }
    }
    
    // 여러 호스트 스캔
    IAsyncAction ScanMultipleHostsAsync(
        std::vector<std::pair<std::wstring, std::vector<uint16_t>>> 
            const& hostPortPairs)
    {
        std::wcout << L"네트워크 스캔 시작" << std::endl;
        std::wcout << std::wstring(50, L'=') << std::endl;
        
        auto startTime = std::chrono::high_resolution_clock::now();
        
        std::vector<IAsyncAction> hostScanTasks;
        
        // 각 호스트에 대해 스캔 작업 생성
        for (auto const& pair : hostPortPairs)
        {
            hostScanTasks.push_back(
                ScanHostAsync(pair.first, pair.second)
            );
        }
        
        // 모든 호스트 스캔 대기
        for (auto& task : hostScanTasks)
        {
            co_await task;
        }
        
        auto endTime = std::chrono::high_resolution_clock::now();
        auto totalTime = std::chrono::duration_cast<
            std::chrono::milliseconds>(endTime - startTime);
        
        // 결과 출력
        PrintResults(totalTime.count());
    }
    
    // 결과 출력
    void PrintResults(long long totalTimeMs)
    {
        std::wcout << std::wstring(50, L'=') << std::endl;
        std::wcout << L"스캔 결과" << std::endl;
        std::wcout << std::wstring(50, L'-') << std::endl;
        
        int openCount = 0;
        int closedCount = 0;
        
        // 호스트별로 결과 그룹화
        std::vector<std::wstring> currentHost;
        
        for (auto const& result : m_results)
        {
            if (result.isOpen)
            {
                openCount++;
                std::wcout << std::left << std::setw(20) 
                          << result.hostName.c_str()
                          << std::setw(10) << result.port
                          << std::setw(15) << L"[OPEN]"
                          << std::setw(10) 
                          << std::fixed << std::setprecision(1)
                          << result.scanTimeMs << L"ms"
                          << std::endl;
            }
            else
            {
                closedCount++;
            }
        }
        
        std::wcout << std::wstring(50, L'-') << std::endl;
        std::wcout << L"요약:" << std::endl;
        std::wcout << L"  - 총 포트 수: " << m_results.size() 
                  << std::endl;
        std::wcout << L"  - 열려있는 포트: " << openCount 
                  << std::endl;
        std::wcout << L"  - 닫혀있는 포트: " << closedCount 
                  << std::endl;
        std::wcout << L"  - 총 스캔 시간: " << totalTimeMs 
                  << L"ms" << std::endl;
        std::wcout << std::wstring(50, L'=') << std::endl;
    }
    
    // 결과 초기화
    void ClearResults()
    {
        m_results.clear();
    }
};

// 사용 예제
IAsyncAction RunNetworkScannerAsync()
{
    NetworkScanner scanner;
    
    // 스캔할 호스트와 포트 목록 정의
    std::vector<std::pair<std::wstring, std::vector<uint16_t>>> 
        hostPortPairs = {
        {
            L"localhost",
            {
                80,     // HTTP
                443,    // HTTPS
                8080,   // 대체 HTTP
                3306,   // MySQL
                5432,   // PostgreSQL
                27017,  // MongoDB
                6379,   // Redis
                9200,   // Elasticsearch
                8000,   // 게임 서버 (테스트용)
                8001    // 게임 서버2 (테스트용)
            }
        },
        {
            L"127.0.0.1",
            {
                3389,   // RDP
                5900,   // VNC
                22,     // SSH
                21,     // FTP
                25,     // SMTP
                53,     // DNS
                110,    // POP3
                143     // IMAP
            }
        }
    };
    
    try
    {
        // 스캔 실행
        co_await scanner.ScanMultipleHostsAsync(hostPortPairs);
        
        std::wcout << L"\n스캔 완료!" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"스캔 오류: " << ex.message().c_str() 
                  << std::endl;
    }
}
```

### 주요 부분 상세 해설
ScanPortAsync() 메서드는 단일 포트에 대한 연결 시도를 수행한다. 이 메서드는 try-catch를 사용하여 연결 실패를 처리한다. 연결이 성공하면 isOpen을 true로 설정하고, 실패하면 에러 메시지를 저장한다.

ScanHostAsync() 메서드는 하나의 호스트에 대해 여러 포트를 병렬로 스캔한다. 모든 포트의 스캔 작업(IAsyncOperation)을 벡터에 저장한 후, 각각을 co_await한다. 이렇게 하면 실제로는 모든 포트가 동시에 스캔되지만, 순차적으로 결과를 수집할 수 있다.

ScanMultipleHostsAsync() 메서드는 여러 호스트에 대해 동시에 스캔을 수행한다. 각 호스트의 스캔 작업을 생성한 후 모두 기다린다.

중요한 성능 최적화는 병렬 처리다. 만약 순차적으로 스캔했다면 (80 포트 × 2 호스트 × 3초 타임아웃) 약 480초가 필요했을 것이다. 하지만 병렬 처리로 인해 실제로는 3초 정도면 충분하다.

PrintResults() 메서드는 결과를 보기 좋게 출력한다. 열려있는 포트만 표시하고, 닫혀있는 포트는 요약에만 포함한다. 각 포트의 응답 시간도 기록하여, 네트워크 상태를 파악할 수 있게 한다.

### 개선 방안과 확장 가능성
첫 번째 개선점은 타임아웃 구현이다. 현재는 기본 타임아웃이 포함되어 있지 않으므로, 응답 없는 서비스에 오래 기다릴 수 있다. CancellationTokenSource를 사용하여 명시적 타임아웃을 추가할 수 있다.

두 번째는 프로토콜별 검증이다. 포트가 열려있는 것을 확인한 후, 실제로 기대하는 서비스인지 확인할 수 있다. 예를 들어, 포트 80에서 HTTP GET 요청을 보내고 응답을 확인할 수 있다.

세 번째는 동시 스캔 수 제한이다. 현재는 모든 포트를 동시에 스캔하므로, 시스템 리소스를 많이 사용한다. 최대 동시 연결 수를 제한하는 스레드 풀 같은 메커니즘을 추가할 수 있다.

네 번째는 포트 범위 스캔이다. 현재는 특정 포트들만 스캔하지만, 예를 들어 "8000-9000" 범위의 모든 포트를 스캔하는 기능을 추가할 수 있다.

다섯 번째는 결과 저장이다. 스캔 결과를 파일로 저장하거나 데이터베이스에 저장할 수 있다. 이를 통해 시간 경과에 따른 네트워크 상태 변화를 추적할 수 있다.

실전에서는 이러한 개선 사항들을 적용하여, 더욱 강력하고 실용적인 스캐닝 도구를 만들 수 있다.
