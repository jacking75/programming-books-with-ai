# Chapter 21. 채팅 서버 구축
채팅 서버는 온라인 게임의 필수 구성 요소다. 게임 내 채팅을 통해 플레이어들이 소통하고, 파티원들과 협력하며, 길드 공지를 받는다. 채팅 서버는 메시지 프레이밍, 세션 관리, 메시지 브로드캐스트 등 이전 장에서 배운 개념들을 통합하는 실전 프로젝트다.

단순해 보이지만, 실제로는 매우 복잡한 요구사항을 가지고 있다. 수천 명의 플레이어가 동시에 메시지를 보낼 수 있고, 각 메시지는 특정 채팅방의 모든 사용자에게 전달되어야 하며, 네트워크 지연으로 인한 메시지 순서 문제도 처리해야 한다. 또한 스팸 필터링, 금지 단어 검사, 채팅 기록 저장 등 부가 기능도 있다.

이 장에서는 기본적인 채팅 기능부터 실전 레벨까지 단계적으로 구현하며, 프로덕션 환경에서 안정적으로 작동하는 채팅 서버를 만들어본다.

---

## 21.1 채팅 서버 요구사항 분석
채팅 서버를 설계하기 전에 요구사항을 명확하게 정의해야 한다. 불필요한 기능을 제외하고 핵심 기능에 집중하는 것이 중요하다.

**필수 기능 (Priority 1)**

1. **사용자 인증**: 플레이어가 계정으로 로그인/로그아웃할 수 있어야 한다. 인증되지 않은 사용자는 채팅에 접근할 수 없다.

2. **채팅방 관리**: 플레이어가 채팅방을 생성하고 입장/퇴장할 수 있어야 한다. 각 채팅방은 독립적인 메시지 스트림을 유지한다.

3. **메시지 브로드캐스트**: 한 플레이어가 보낸 메시지를 같은 채팅방의 모든 플레이어에게 전달해야 한다.

4. **메시지 순서 보장**: 같은 채팅방에서 메시지는 서버에 도착한 순서대로 처리되어야 한다.

**중요 기능 (Priority 2)**

5. **타임아웃 처리**: 응답이 없는 플레이어를 자동으로 채팅방에서 제거한다.

6. **오류 처리 및 복구**: 네트워크 오류나 서버 오류가 발생해도 서버가 정상 작동해야 한다.

7. **채팅방 정보 조회**: 플레이어가 현재 채팅방의 참가자 목록, 방 정보 등을 조회할 수 있어야 한다.

**선택 기능 (Priority 3)**

8. **스팸 필터링**: 같은 메시지를 반복으로 보내는 것을 방지한다.

9. **금지 단어 필터**: 부적절한 단어를 자동으로 검열한다.

10. **채팅 기록**: 메시지를 로그 파일이나 데이터베이스에 저장한다.

11. **귓속말(Private Message)**: 특정 플레이어에게만 메시지를 보낼 수 있다.

이 장에서는 필수 기능 1-4와 중요 기능 5-7을 구현하는 데 집중한다. 프로덕션급 채팅 서버의 핵심을 다루기 때문이다.

## 21.2 아키텍처 설계
채팅 서버의 전체 구조를 이해하기 위해 컴포넌트 다이어그램을 살펴보자.

```
┌─────────────────────────────────────────────────────────────┐
│                     Chat Server                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────────────┐        ┌──────────────────┐          │
│  │  Socket Layer    │        │ Message Framing  │          │
│  │  (StreamSocket)  │◄──────►│  (Length Prefix) │          │
│  └────────┬─────────┘        └────────┬─────────┘          │
│           │                           │                    │
│           ▼                           ▼                    │
│  ┌────────────────────────────────────────────────┐       │
│  │    Session Manager                            │       │
│  │  - User authentication                        │       │
│  │  - Session lifecycle                          │       │
│  │  - Timeout handling                           │       │
│  └────────────┬─────────────────────────────────┘       │
│               │                                           │
│               ▼                                           │
│  ┌────────────────────────────────────────────────┐       │
│  │    Command Processor                          │       │
│  │  - LOGIN, LOGOUT                              │       │
│  │  - CREATE_ROOM, JOIN_ROOM, LEAVE_ROOM        │       │
│  │  - SEND_MESSAGE, GET_ROOM_LIST               │       │
│  └────────────┬─────────────────────────────────┘       │
│               │                                           │
│               ▼                                           │
│  ┌────────────────────────────────────────────────┐       │
│  │    Room Manager                               │       │
│  │  - Room creation/deletion                     │       │
│  │  - Member management                         │       │
│  │  - Message broadcasting                      │       │
│  └────────────┬─────────────────────────────────┘       │
│               │                                           │
│               ▼                                           │
│  ┌────────────────────────────────────────────────┐       │
│  │    Persistence Layer                          │       │
│  │  - Log chat messages                          │       │
│  │  - Store room information                     │       │
│  └────────────────────────────────────────────────┘       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**계층별 설명**

**1. Socket Layer**: WinRT의 StreamSocket을 사용하여 네트워크 연결을 관리한다. 비동기 I/O로 여러 클라이언트를 동시에 처리한다.

**2. Message Framing**: Chapter 19에서 배운 길이 접두사 방식을 사용하여 메시지의 경계를 명확하게 한다.

**3. Session Manager**: Chapter 20에서 구현한 세션 관리 시스템을 사용하여 사용자 인증과 세션 라이프사이클을 관리한다.

**4. Command Processor**: 클라이언트에서 받은 명령어를 파싱하고 처리한다. 각 명령어는 특정 비즈니스 로직을 트리거한다.

**5. Room Manager**: 채팅방을 생성하고 관리하며, 각 방의 멤버에게 메시지를 브로드캐스트한다.

**6. Persistence Layer**: 채팅 메시지를 영구 저장소에 기록한다. (이 장에서는 간단한 파일 로깅만 구현)

이러한 계층 구조는 각 컴포넌트가 독립적으로 테스트되고, 변경될 수 있도록 한다. 예를 들어, 데이터베이스를 추가하려면 Persistence Layer만 수정하면 된다.

## 21.3 사용자 인증 구현
채팅 서버의 첫 번째 단계는 사용자 인증이다. 클라이언트가 접속하면 반드시 로그인 명령어를 전송해야 하고, 서버는 이를 검증해야 한다.

```cpp
#include <string>
#include <map>
#include <memory>
#include <mutex>
#include <vector>

enum class ChatCommand
{
    LOGIN = 1,
    LOGOUT = 2,
    CREATE_ROOM = 3,
    JOIN_ROOM = 4,
    LEAVE_ROOM = 5,
    SEND_MESSAGE = 6,
    GET_ROOM_LIST = 7,
    ERROR_RESPONSE = 100
};

struct LoginRequest
{
    std::string username;
    std::string password;  // 실제로는 암호화되어야 함
};

struct LoginResponse
{
    bool success;
    std::string sessionId;
    std::string errorMessage;
};

class AuthenticationManager
{
private:
    // 실제 서버에서는 데이터베이스 사용
    std::map<std::string, std::string> userDatabase;
    std::mutex dbMutex;

public:
    AuthenticationManager()
    {
        // 테스트용 계정
        userDatabase["player1"] = "password1";
        userDatabase["player2"] = "password2";
        userDatabase["admin"] = "adminpass";
    }

    // 사용자 자격증명 검증
    bool ValidateCredentials(const std::string& username, 
                            const std::string& password)
    {
        std::lock_guard<std::mutex> lock(dbMutex);
        
        auto it = userDatabase.find(username);
        if (it == userDatabase.end())
            return false;

        // 실제로는 해시 비교를 해야 함
        return it->second == password;
    }

    // 새 사용자 등록 (선택 사항)
    bool RegisterUser(const std::string& username, 
                     const std::string& password)
    {
        std::lock_guard<std::mutex> lock(dbMutex);
        
        if (userDatabase.find(username) != userDatabase.end())
            return false;  // 이미 존재하는 계정

        userDatabase[username] = password;
        return true;
    }

    // 사용자 존재 확인
    bool UserExists(const std::string& username)
    {
        std::lock_guard<std::mutex> lock(dbMutex);
        return userDatabase.find(username) != userDatabase.end();
    }
};

// 로그인 요청/응답 처리 클래스
class LoginHandler
{
private:
    AuthenticationManager& authManager;

public:
    explicit LoginHandler(AuthenticationManager& auth)
        : authManager(auth)
    {
    }

    // 로그인 요청 처리
    LoginResponse HandleLogin(const LoginRequest& request)
    {
        LoginResponse response;

        // 사용자명 검증
        if (request.username.empty())
        {
            response.success = false;
            response.errorMessage = "Username cannot be empty";
            return response;
        }

        // 비밀번호 검증
        if (request.password.length() < 4)
        {
            response.success = false;
            response.errorMessage = "Invalid password";
            return response;
        }

        // 자격증명 검증
        if (!authManager.ValidateCredentials(request.username, 
                                            request.password))
        {
            response.success = false;
            response.errorMessage = "Authentication failed";
            return response;
        }

        // 로그인 성공
        response.success = true;
        response.sessionId = GenerateSessionId(request.username);
        return response;
    }

private:
    std::string GenerateSessionId(const std::string& username)
    {
        // 간단한 세션 ID 생성 (실제로는 더 복잡해야 함)
        auto timestamp = std::chrono::system_clock::now()
            .time_since_epoch().count();
        return username + "_" + std::to_string(timestamp);
    }
};
```

이 인증 시스템의 핵심은 다음과 같다.

**Separation of Concerns**: `AuthenticationManager`는 데이터베이스 접근만 담당하고, `LoginHandler`는 비즈니스 로직만 담당한다. 이렇게 분리하면 각 부분을 독립적으로 테스트할 수 있다.

**Thread Safety**: `std::mutex`를 사용하여 여러 스레드가 동시에 사용자 데이터베이스에 접근하는 것을 보호한다.

**명확한 에러 메시지**: 클라이언트가 왜 로그인에 실패했는지 알 수 있도록 상세한 에러 메시지를 반환한다.

**실제 구현 시 주의사항**: 예제에서는 평문 비밀번호를 저장했지만, 실제로는 bcrypt, Argon2 등의 해시 알고리즘을 사용해야 한다.

## 21.4 채팅방 관리
채팅방은 메시지를 공유하는 공간이다. 플레이어는 채팅방을 생성하고, 다른 플레이어는 그 방에 입장하여 메시지를 주고받을 수 있다.

```cpp
#include <string>
#include <map>
#include <set>
#include <memory>
#include <mutex>
#include <vector>

class ChatRoom
{
private:
    std::string roomId;
    std::string roomName;
    std::string ownerId;
    std::set<std::string> members;  // 방의 멤버 목록
    std::mutex memberMutex;
    
    static const size_t MAX_MEMBERS = 100;

public:
    ChatRoom(const std::string& id, const std::string& name, 
             const std::string& owner)
        : roomId(id), roomName(name), ownerId(owner)
    {
        // 방 소유자를 자동으로 추가
        members.insert(owner);
    }

    // Getter 메서드들
    const std::string& GetRoomId() const { return roomId; }
    const std::string& GetRoomName() const { return roomName; }
    const std::string& GetOwnerId() const { return ownerId; }

    // 멤버 추가
    bool AddMember(const std::string& userId)
    {
        std::lock_guard<std::mutex> lock(memberMutex);
        
        // 이미 방에 있는 사용자
        if (members.find(userId) != members.end())
            return false;

        // 방이 가득 찼는지 확인
        if (members.size() >= MAX_MEMBERS)
            return false;

        members.insert(userId);
        return true;
    }

    // 멤버 제거
    bool RemoveMember(const std::string& userId)
    {
        std::lock_guard<std::mutex> lock(memberMutex);
        return members.erase(userId) > 0;
    }

    // 멤버 확인
    bool HasMember(const std::string& userId) const
    {
        std::lock_guard<std::mutex> lock(memberMutex);
        return members.find(userId) != members.end();
    }

    // 멤버 목록 반환
    std::vector<std::string> GetMembers() const
    {
        std::lock_guard<std::mutex> lock(memberMutex);
        return std::vector<std::string>(members.begin(), members.end());
    }

    // 현재 멤버 수
    size_t GetMemberCount() const
    {
        std::lock_guard<std::mutex> lock(memberMutex);
        return members.size();
    }

    // 방이 비었는지 확인 (방 소유자 제외)
    bool IsEmpty() const
    {
        std::lock_guard<std::mutex> lock(memberMutex);
        return members.size() <= 1;  // 소유자만 남음
    }
};

class RoomManager
{
private:
    std::map<std::string, std::shared_ptr<ChatRoom>> rooms;
    std::map<std::string, std::string> userToRoom;  // userId -> roomId
    std::mutex roomMutex;

public:
    // 채팅방 생성
    std::shared_ptr<ChatRoom> CreateRoom(const std::string& roomName, 
                                         const std::string& ownerId)
    {
        std::lock_guard<std::mutex> lock(roomMutex);
        
        // 사용자가 이미 다른 방에 있는지 확인
        if (userToRoom.find(ownerId) != userToRoom.end())
            return nullptr;

        std::string roomId = GenerateRoomId();
        auto room = std::make_shared<ChatRoom>(roomId, roomName, ownerId);
        rooms[roomId] = room;
        userToRoom[ownerId] = roomId;
        return room;
    }

    // 방에 입장
    std::shared_ptr<ChatRoom> JoinRoom(const std::string& roomId, 
                                       const std::string& userId)
    {
        std::lock_guard<std::mutex> lock(roomMutex);
        
        auto it = rooms.find(roomId);
        if (it == rooms.end())
            return nullptr;

        auto room = it->second;

        // 이미 다른 방에 있는지 확인
        if (userToRoom.find(userId) != userToRoom.end())
            return nullptr;

        // 멤버 추가
        if (!room->AddMember(userId))
            return nullptr;

        userToRoom[userId] = roomId;
        return room;
    }

    // 방에서 퇴장
    bool LeaveRoom(const std::string& userId)
    {
        std::lock_guard<std::mutex> lock(roomMutex);
        
        auto userIt = userToRoom.find(userId);
        if (userIt == userToRoom.end())
            return false;

        std::string roomId = userIt->second;
        auto roomIt = rooms.find(roomId);
        if (roomIt == rooms.end())
            return false;

        auto room = roomIt->second;
        room->RemoveMember(userId);
        userToRoom.erase(userIt);

        // 방이 비었으면 삭제
        if (room->IsEmpty())
            rooms.erase(roomIt);

        return true;
    }

    // 사용자가 속한 방 조회
    std::shared_ptr<ChatRoom> GetUserRoom(const std::string& userId)
    {
        std::lock_guard<std::mutex> lock(roomMutex);
        
        auto it = userToRoom.find(userId);
        if (it == userToRoom.end())
            return nullptr;

        auto roomIt = rooms.find(it->second);
        if (roomIt == rooms.end())
            return nullptr;

        return roomIt->second;
    }

    // 방 ID로 직접 조회
    std::shared_ptr<ChatRoom> GetRoom(const std::string& roomId)
    {
        std::lock_guard<std::mutex> lock(roomMutex);
        
        auto it = rooms.find(roomId);
        if (it == rooms.end())
            return nullptr;

        return it->second;
    }

    // 모든 방 목록 (로비 표시용)
    std::vector<std::pair<std::string, size_t>> GetRoomList() const
    {
        std::lock_guard<std::mutex> lock(roomMutex);
        
        std::vector<std::pair<std::string, size_t>> roomList;
        for (const auto& entry : rooms)
        {
            roomList.emplace_back(
                entry.second->GetRoomName(),
                entry.second->GetMemberCount()
            );
        }
        return roomList;
    }

    // 현재 방의 개수
    size_t GetRoomCount() const
    {
        std::lock_guard<std::mutex> lock(roomMutex);
        return rooms.size();
    }

private:
    std::string GenerateRoomId()
    {
        static int roomCounter = 0;
        return "room_" + std::to_string(roomCounter++);
    }
};
```

채팅방 관리의 핵심 설계 원칙은 다음과 같다.

**멀티플 인덱싱**: 방을 방 ID로도, 사용자를 통해서도 빠르게 찾을 수 있도록 두 가지 맵을 유지한다.

**자동 정리**: 방이 비어지면 자동으로 삭제하여 메모리 누수를 방지한다.

**사용자당 한 방**: 한 사용자는 한 번에 한 개의 방에만 속할 수 있다. 이는 상태 관리를 단순하게 한다.

**스레드 안전**: 모든 맵 접근을 뮤텍스로 보호한다.

## 21.5 메시지 브로드캐스트
채팅 서버의 핵심은 메시지를 같은 방의 모든 사용자에게 전달하는 것이다. 이를 효율적으로 하려면 비동기 네트워크 I/O가 필수다.

```cpp
#include <string>
#include <vector>
#include <memory>
#include <mutex>

struct ChatMessage
{
    std::string senderId;
    std::string senderName;
    std::string content;
    int64_t timestamp;
    int32_t messageId;
};

class MessageBroadcaster
{
private:
    struct ClientConnection
    {
        std::string userId;
        std::string sessionId;
        // WinRT StreamSocket이 여기에 들어감
        // StreamSocket socket;
    };

    std::vector<ClientConnection> connections;
    std::mutex connectionMutex;

public:
    // 클라이언트 연결 등록
    void RegisterClient(const std::string& userId, 
                       const std::string& sessionId)
    {
        std::lock_guard<std::mutex> lock(connectionMutex);
        connections.push_back({userId, sessionId});
    }

    // 클라이언트 연결 해제
    void UnregisterClient(const std::string& userId)
    {
        std::lock_guard<std::mutex> lock(connectionMutex);
        auto it = std::remove_if(connections.begin(), connections.end(),
            [&userId](const ClientConnection& conn) { 
                return conn.userId == userId; 
            });
        connections.erase(it, connections.end());
    }

    // 방의 모든 멤버에게 메시지 브로드캐스트
    void BroadcastToRoom(const std::shared_ptr<ChatRoom>& room, 
                         const ChatMessage& message)
    {
        if (!room)
            return;

        std::vector<std::string> members = room->GetMembers();
        
        for (const auto& userId : members)
        {
            // 발신자에게는 ACK 전송
            if (userId == message.senderId)
            {
                SendMessageAck(userId, message.messageId);
            }
            else
            {
                // 다른 사용자에게는 메시지 전송
                SendMessageToUser(userId, message);
            }
        }
    }

    // 특정 사용자에게 메시지 전송
    void SendMessageToUser(const std::string& userId, 
                          const ChatMessage& message)
    {
        std::lock_guard<std::mutex> lock(connectionMutex);
        
        for (auto& conn : connections)
        {
            if (conn.userId == userId)
            {
                // 실제 전송: socket.SendAsync(serialized_message)
                // 여기서는 의사 코드
                std::string serialized = SerializeMessage(message);
                // SendAsync(conn.socket, serialized);
                break;
            }
        }
    }

    // 메시지 수신 확인 (ACK)
    void SendMessageAck(const std::string& userId, 
                       int32_t messageId)
    {
        // 클라이언트에게 메시지가 성공적으로 서버에 도착했음을 알림
        // 이는 클라이언트의 메시지 전송 UI를 업데이트할 때 사용됨
    }

    // 현재 연결된 사용자 수
    size_t GetConnectedUserCount() const
    {
        std::lock_guard<std::mutex> lock(connectionMutex);
        return connections.size();
    }

    // 사용자가 연결되어 있는지 확인
    bool IsUserConnected(const std::string& userId) const
    {
        std::lock_guard<std::mutex> lock(connectionMutex);
        for (const auto& conn : connections)
        {
            if (conn.userId == userId)
                return true;
        }
        return false;
    }

private:
    std::string SerializeMessage(const ChatMessage& message)
    {
        // 메시지를 네트워크 형식으로 직렬화
        // Chapter 18의 직렬화 기법 사용
        std::string result;
        result += message.senderId + "|";
        result += message.senderName + "|";
        result += message.content + "|";
        result += std::to_string(message.timestamp);
        return result;
    }
};

// 메시지 처리 클래스
class MessageProcessor
{
private:
    RoomManager& roomManager;
    MessageBroadcaster& broadcaster;
    static int32_t messageIdCounter;

public:
    MessageProcessor(RoomManager& rm, MessageBroadcaster& mb)
        : roomManager(rm), broadcaster(mb)
    {
    }

    // 메시지 처리 (메인 로직)
    bool ProcessMessage(const std::string& userId, 
                       const std::string& senderName,
                       const std::string& content)
    {
        // 사용자가 방에 있는지 확인
        auto room = roomManager.GetUserRoom(userId);
        if (!room)
            return false;

        // 메시지 유효성 검사
        if (content.empty() || content.length() > 500)
            return false;

        // 채팅 메시지 생성
        ChatMessage message;
        message.senderId = userId;
        message.senderName = senderName;
        message.content = content;
        message.timestamp = std::chrono::system_clock::now()
            .time_since_epoch().count();
        message.messageId = messageIdCounter++;

        // 같은 방의 모든 사용자에게 브로드캐스트
        broadcaster.BroadcastToRoom(room, message);

        return true;
    }
};

int32_t MessageProcessor::messageIdCounter = 0;
```

메시지 브로드캐스트의 핵심 설계는 다음과 같다.

**비동기 전송**: 한 사용자에게 메시지 전송이 느리더라도 다른 사용자의 전송을 차단하지 않는다. 이는 `SendAsync()`를 사용하여 구현된다.

**연결 추상화**: 실제 소켓 연결 세부사항을 `ClientConnection` 구조체로 숨겨서, 나머지 코드는 구체적인 네트워크 구현에 의존하지 않는다.

**메시지 ACK**: 발신자에게는 메시지 수신 확인을 보내서, 클라이언트가 메시지가 제대로 전송됐는지 알 수 있게 한다.

**멀티스레드 안전**: 연결 목록을 뮤텍스로 보호하여 여러 스레드에서 동시에 접근해도 안전하다.

## 21.6 종합 예제: 멀티룸 채팅 서버
이제 모든 구성 요소를 통합하여 실제로 작동하는 멀티룸 채팅 서버를 만들어보자.

**코드의 목적**: 로그인부터 채팅까지의 전체 흐름을 처리하는 완전한 채팅 서버를 구현한다. 다양한 명령어(LOGIN, CREATE_ROOM, SEND_MESSAGE 등)를 처리하고, 각 명령어에 대해 적절한 응답을 클라이언트에게 보낸다. 또한 멀티스레드 환경에서 안전하게 작동해야 한다.

**설계 결정 이유**:
1. **명령어 기반 프로토콜**: 각 메시지는 명령어 타입으로 시작되고, 뒤에 인자가 따른다. 이는 확장성이 좋고 클라이언트 구현도 간단하다.
2. **이벤트 기반 알림**: 채팅방에 사용자가 입장하면 다른 사용자들에게 알린다. 이는 UI 갱신을 자동으로 할 수 있게 해준다.
3. **에러 응답**: 모든 실패는 명확한 에러 코드와 메시지로 클라이언트에 알려진다.
4. **세션 검증**: 모든 요청에서 세션 ID를 확인하여 인증되지 않은 접근을 방지한다.

```cpp
#include <string>
#include <memory>
#include <map>
#include <mutex>
#include <vector>
#include <sstream>
#include <chrono>
#include <algorithm>

// ============ 프로토콜 정의 ============

enum class CommandType
{
    LOGIN = 1,
    LOGOUT = 2,
    CREATE_ROOM = 3,
    JOIN_ROOM = 4,
    LEAVE_ROOM = 5,
    SEND_MESSAGE = 6,
    GET_ROOM_LIST = 7,
    NOTIFY_USER_JOIN = 20,
    NOTIFY_USER_LEAVE = 21,
    NOTIFY_MESSAGE = 22,
    RESPONSE_SUCCESS = 100,
    RESPONSE_ERROR = 101
};

struct Request
{
    CommandType command;
    std::string sessionId;
    std::vector<std::string> args;
};

struct Response
{
    bool success;
    CommandType command;
    std::string message;
    std::vector<std::string> data;
};

// ============ 명령어 파서 ============

class CommandParser
{
public:
    static Request Parse(const std::string& rawData)
    {
        Request req;
        std::stringstream ss(rawData);
        std::string token;

        // 첫 번째 토큰은 명령어 타입
        if (std::getline(ss, token, '|'))
            req.command = static_cast<CommandType>(std::stoi(token));

        // 두 번째 토큰은 세션 ID
        if (std::getline(ss, token, '|'))
            req.sessionId = token;

        // 나머지는 인자
        while (std::getline(ss, token, '|'))
            req.args.push_back(token);

        return req;
    }

    static std::string Serialize(const Response& resp)
    {
        std::stringstream ss;
        ss << static_cast<int>(resp.command) << "|"
           << (resp.success ? "1" : "0") << "|"
           << resp.message;

        for (const auto& data : resp.data)
            ss << "|" << data;

        return ss.str();
    }
};

// ============ 세션 저장소 ============

class SessionStore
{
private:
    std::map<std::string, std::string> sessionToUser;  // sessionId -> userId
    std::mutex mutex;

public:
    void AddSession(const std::string& sessionId, 
                   const std::string& userId)
    {
        std::lock_guard<std::mutex> lock(mutex);
        sessionToUser[sessionId] = userId;
    }

    std::string GetUserId(const std::string& sessionId)
    {
        std::lock_guard<std::mutex> lock(mutex);
        auto it = sessionToUser.find(sessionId);
        if (it != sessionToUser.end())
            return it->second;
        return "";
    }

    void RemoveSession(const std::string& sessionId)
    {
        std::lock_guard<std::mutex> lock(mutex);
        sessionToUser.erase(sessionId);
    }

    bool IsValidSession(const std::string& sessionId)
    {
        std::lock_guard<std::mutex> lock(mutex);
        return sessionToUser.find(sessionId) != sessionToUser.end();
    }
};

// ============ 메인 채팅 서버 ============

class ChatServer
{
private:
    AuthenticationManager authManager;
    SessionStore sessionStore;
    RoomManager roomManager;
    MessageBroadcaster broadcaster;
    MessageProcessor messageProcessor;

    std::mutex userNameMutex;
    std::map<std::string, std::string> userIdToName;  // userId -> username

public:
    ChatServer()
        : messageProcessor(roomManager, broadcaster)
    {
    }

    // ===== 메인 요청 처리 =====

    Response ProcessRequest(const std::string& rawData)
    {
        Request req;
        try
        {
            req = CommandParser::Parse(rawData);
        }
        catch (...)
        {
            return CreateErrorResponse(
                CommandType::RESPONSE_ERROR, 
                "Invalid request format"
            );
        }

        // 세션 검증 (로그인 제외)
        if (req.command != CommandType::LOGIN)
        {
            if (!sessionStore.IsValidSession(req.sessionId))
            {
                return CreateErrorResponse(
                    CommandType::RESPONSE_ERROR,
                    "Invalid or expired session"
                );
            }
        }

        // 명령어별 처리
        switch (req.command)
        {
        case CommandType::LOGIN:
            return HandleLogin(req);

        case CommandType::LOGOUT:
            return HandleLogout(req);

        case CommandType::CREATE_ROOM:
            return HandleCreateRoom(req);

        case CommandType::JOIN_ROOM:
            return HandleJoinRoom(req);

        case CommandType::LEAVE_ROOM:
            return HandleLeaveRoom(req);

        case CommandType::SEND_MESSAGE:
            return HandleSendMessage(req);

        case CommandType::GET_ROOM_LIST:
            return HandleGetRoomList(req);

        default:
            return CreateErrorResponse(
                CommandType::RESPONSE_ERROR,
                "Unknown command"
            );
        }
    }

    // ===== 명령어 핸들러 =====

private:
    Response HandleLogin(const Request& req)
    {
        if (req.args.size() < 2)
        {
            return CreateErrorResponse(
                CommandType::LOGIN,
                "Username and password required"
            );
        }

        const std::string& username = req.args[0];
        const std::string& password = req.args[1];

        // 자격증명 검증
        if (!authManager.ValidateCredentials(username, password))
        {
            return CreateErrorResponse(
                CommandType::LOGIN,
                "Authentication failed"
            );
        }

        // 세션 생성
        std::string sessionId = GenerateSessionId(username);
        sessionStore.AddSession(sessionId, username);

        // 사용자명 저장
        {
            std::lock_guard<std::mutex> lock(userNameMutex);
            userIdToName[username] = username;
        }

        // 클라이언트에 연결 등록
        broadcaster.RegisterClient(username, sessionId);

        Response resp;
        resp.success = true;
        resp.command = CommandType::LOGIN;
        resp.message = "Login successful";
        resp.data.push_back(sessionId);

        return resp;
    }

    Response HandleLogout(const Request& req)
    {
        std::string userId = sessionStore.GetUserId(req.sessionId);

        // 사용자가 속한 방에서 퇴장
        roomManager.LeaveRoom(userId);

        // 세션 제거
        sessionStore.RemoveSession(req.sessionId);

        // 연결 해제
        broadcaster.UnregisterClient(userId);

        Response resp;
        resp.success = true;
        resp.command = CommandType::LOGOUT;
        resp.message = "Logout successful";

        return resp;
    }

    Response HandleCreateRoom(const Request& req)
    {
        if (req.args.empty())
        {
            return CreateErrorResponse(
                CommandType::CREATE_ROOM,
                "Room name required"
            );
        }

        std::string userId = sessionStore.GetUserId(req.sessionId);
        std::string roomName = req.args[0];

        // 사용자가 이미 다른 방에 있으면 거기서 퇴장
        roomManager.LeaveRoom(userId);

        // 새 방 생성
        auto room = roomManager.CreateRoom(roomName, userId);
        if (!room)
        {
            return CreateErrorResponse(
                CommandType::CREATE_ROOM,
                "Failed to create room"
            );
        }

        // 다른 사용자들에게 로비 업데이트 알림
        NotifyRoomListChanged();

        Response resp;
        resp.success = true;
        resp.command = CommandType::CREATE_ROOM;
        resp.message = "Room created";
        resp.data.push_back(room->GetRoomId());

        return resp;
    }

    Response HandleJoinRoom(const Request& req)
    {
        if (req.args.empty())
        {
            return CreateErrorResponse(
                CommandType::JOIN_ROOM,
                "Room ID required"
            );
        }

        std::string userId = sessionStore.GetUserId(req.sessionId);
        std::string roomId = req.args[0];

        // 사용자가 이미 다른 방에 있으면 거기서 퇴장
        auto currentRoom = roomManager.GetUserRoom(userId);
        if (currentRoom)
            roomManager.LeaveRoom(userId);

        // 새 방에 입장
        auto room = roomManager.JoinRoom(roomId, userId);
        if (!room)
        {
            return CreateErrorResponse(
                CommandType::JOIN_ROOM,
                "Failed to join room"
            );
        }

        // 방의 다른 사용자들에게 입장 알림
        ChatMessage notification;
        notification.senderId = "SYSTEM";
        notification.senderName = "SYSTEM";
        notification.content = userId + " has joined the room";
        notification.timestamp = std::chrono::system_clock::now()
            .time_since_epoch().count();

        broadcaster.BroadcastToRoom(room, notification);

        Response resp;
        resp.success = true;
        resp.command = CommandType::JOIN_ROOM;
        resp.message = "Joined room";
        resp.data.push_back(roomId);

        // 방의 멤버 목록도 전송
        for (const auto& member : room->GetMembers())
            resp.data.push_back(member);

        return resp;
    }

    Response HandleLeaveRoom(const Request& req)
    {
        std::string userId = sessionStore.GetUserId(req.sessionId);

        auto room = roomManager.GetUserRoom(userId);
        if (!room)
        {
            return CreateErrorResponse(
                CommandType::LEAVE_ROOM,
                "Not in any room"
            );
        }

        // 퇴장 알림
        ChatMessage notification;
        notification.senderId = "SYSTEM";
        notification.senderName = "SYSTEM";
        notification.content = userId + " has left the room";
        notification.timestamp = std::chrono::system_clock::now()
            .time_since_epoch().count();

        broadcaster.BroadcastToRoom(room, notification);

        // 방에서 퇴장
        roomManager.LeaveRoom(userId);

        Response resp;
        resp.success = true;
        resp.command = CommandType::LEAVE_ROOM;
        resp.message = "Left room";

        return resp;
    }

    Response HandleSendMessage(const Request& req)
    {
        if (req.args.empty())
        {
            return CreateErrorResponse(
                CommandType::SEND_MESSAGE,
                "Message content required"
            );
        }

        std::string userId = sessionStore.GetUserId(req.sessionId);
        std::string content = req.args[0];

        // 메시지 처리 (브로드캐스트 포함)
        if (!messageProcessor.ProcessMessage(
            userId,
            userId,  // 사용자명 (실제로는 별도 저장소에서 조회)
            content))
        {
            return CreateErrorResponse(
                CommandType::SEND_MESSAGE,
                "Failed to send message"
            );
        }

        Response resp;
        resp.success = true;
        resp.command = CommandType::SEND_MESSAGE;
        resp.message = "Message sent";

        return resp;
    }

    Response HandleGetRoomList(const Request& req)
    {
        auto roomList = roomManager.GetRoomList();

        Response resp;
        resp.success = true;
        resp.command = CommandType::GET_ROOM_LIST;
        resp.message = "Room list";

        for (const auto& room : roomList)
        {
            resp.data.push_back(room.first + " (" + 
                               std::to_string(room.second) + ")");
        }

        return resp;
    }

    // ===== 헬퍼 메서드 =====

    Response CreateErrorResponse(CommandType cmdType, 
                                const std::string& message)
    {
        Response resp;
        resp.success = false;
        resp.command = CommandType::RESPONSE_ERROR;
        resp.message = message;
        return resp;
    }

    std::string GenerateSessionId(const std::string& username)
    {
        auto timestamp = std::chrono::system_clock::now()
            .time_since_epoch().count();
        return username + "_" + std::to_string(timestamp);
    }

    void NotifyRoomListChanged()
    {
        // 모든 클라이언트에게 로비가 변경됐음을 알림
        // (선택 사항: 성능상 이유로 생략할 수 있음)
    }
};
```

**주요 부분 상세 해설**:

1. **CommandParser**: 클라이언트에서 받은 문자열을 파싱하여 Request 구조체로 변환한다. 프로토콜은 파이프(|)로 필드를 분리한다. 예: `1|sessionId123|room1|Hello`

2. **SessionStore**: 세션 ID와 사용자 ID의 매핑을 관리한다. 모든 요청(로그인 제외)은 세션 검증을 거쳐야 한다.

3. **ProcessRequest**: 메인 진입점으로, 요청을 파싱하고 세션을 검증한 후 적절한 핸들러로 라우팅한다.

4. **HandleCreateRoom**: 사용자가 이미 다른 방에 있으면 먼저 그곳에서 퇴장한다. 이는 "한 사용자, 한 방" 규칙을 유지한다.

5. **HandleJoinRoom**: 입장 후 방의 모든 멤버에게 시스템 메시지를 브로드캐스트한다. 이는 다른 사용자들의 UI를 즉시 갱신할 수 있게 한다.

6. **HandleSendMessage**: MessageProcessor를 사용하여 메시지를 처리하고 같은 방의 모든 사용자에게 브로드캐스트한다.

**개선 방안과 확장 가능성**:

첫째, **채팅 기록 저장**: 메시지를 처리할 때마다 파일이나 데이터베이스에 저장하여, 사용자가 이전 메시지를 조회할 수 있도록 한다. SQLite나 MySQL을 사용할 수 있다.

둘째, **귓속말(Whisper) 기능**: `SEND_PRIVATE_MESSAGE` 명령어를 추가하여 특정 사용자에게만 메시지를 보낼 수 있게 한다.

셋째, **스팸 필터링**: 같은 메시지를 반복으로 보내거나 너무 많은 메시지를 짧은 시간에 보내는 것을 감지하고 제한한다. 사용자당 메시지 전송률 제한(Rate Limiting)을 구현한다.

넷째, **금지 단어 검사**: 메시지에 부적절한 단어가 포함되면 "*" 등으로 치환하거나 메시지 전송을 차단한다.

다섯째, **관리자 명령어**: `/ban`, `/kick` 같은 관리자용 명령어를 추가하여, 서버 운영자가 문제 사용자를 처리할 수 있도록 한다.

여섯째, **읽음 확인(Read Receipt)**: 메시지 옆에 "읽음" 표시를 하거나, 특정 사용자가 읽었는지 확인할 수 있게 한다.

일곱째, **이모지와 서식**: 메시지에 색상, 굵기, 이모지 등을 포함할 수 있도록 확장한다.

여덟째, **모듈화된 구조**: 인증, 방 관리, 메시지 처리를 플러그인처럼 구현하여, 기능 추가/제거를 쉽게 할 수 있도록 한다.
