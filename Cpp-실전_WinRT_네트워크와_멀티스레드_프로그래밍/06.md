# Chapter 6. 타이머 프로그래밍
타이머는 게임 서버의 심장이다. 게임 루프는 일정한 간격으로 반복되어야 하고, 정기적인 작업(플레이어 상태 업데이트, 저장, 통계 수집)은 정확한 시간에 실행되어야 한다. WinRT의 `ThreadPoolTimer`는 이러한 시간 기반 작업을 효율적으로 관리할 수 있게 해준다. 이 장에서는 타이머의 기초부터 고급 사용법까지 학습한다.
  

## 6.1 ThreadPoolTimer API 설명

### API 상세 설명
`ThreadPoolTimer`는 WinRT의 타이머 API다. ThreadPool 위에서 작동하므로, 별도의 타이머 스레드를 생성하지 않으면서도 정확한 타이밍을 제공한다. 타이머가 실행되면 ThreadPool의 작업으로 등록되어 스레드 풀의 스레드에서 실행된다.

`ThreadPoolTimer`의 생성은 정적 메서드 `CreateTimer()` 또는 `CreatePeriodicTimer()`를 사용한다. 두 메서드 모두 `TimerElapsedHandler`(실행할 콜백)와 `TimeSpan`(대기 시간)을 매개변수로 받는다.

생성된 타이머는 즉시 시작되며, 지정된 시간이 경과하면 콜백 함수가 호출된다. `Period` 속성으로 타이머의 주기를 수정할 수 있으며, `Cancel()` 메서드로 타이머를 중단할 수 있다.

타이머의 콜백 함수는 `void(IThreadPoolTimer)`의 시그니처를 가진다. 콜백이 실행되는 동안 타이머는 일시 중단되지 않는다는 점을 기억하자. 콜백이 100ms 걸리고 주기가 50ms라면, 콜백이 끝난 후에 다음 주기가 시작된다.

### 기본 타이머 생성 예제 

```cpp
#include <winrt/Windows.System.Threading.h>
#include <iostream>
#include <chrono>

using namespace winrt;
using namespace Windows::System::Threading;

int main()
{
    init_apartment();
    
    // 1초(1000밀리초) 후에 한 번 실행되는 타이머
    auto timer = ThreadPoolTimer::CreateTimer(
        [](IThreadPoolTimer const& timer)
        {
            std::cout << "타이머 콜백 실행!" << std::endl;
        },
        std::chrono::milliseconds(1000));
    
    // 타이머 완료 대기
    Sleep(2000);
    
    return 0;
}
```

이 예제는 1초 후에 콜백을 한 번 실행하는 일회성 타이머다. `Sleep(2000)`으로 2초를 기다리는 것은 타이머가 실행될 시간을 충분히 주기 위함이다.
  


## 6.2 CreateTimer vs CreatePeriodicTimer

### API 설명
`CreateTimer()`와 `CreatePeriodicTimer()`는 타이머의 동작 방식을 결정한다.

**CreateTimer()**는 지정된 시간 후 한 번만 실행되고 자동으로 종료되는 일회성 타이머다. 초기 지연 후 콜백이 정확히 한 번 실행된다.

**CreatePeriodicTimer()**는 지정된 간격으로 반복 실행되는 주기적 타이머다. `Period` 속성으로 반복 주기를 설정한다. 타이머가 명시적으로 `Cancel()`로 취소될 때까지 계속 실행된다.

주의할 점은 주기적 타이머의 주기는 다음과 같이 계산된다는 것이다: 이전 콜백의 시작 시간 + Period = 다음 콜백의 시작 시간. 만약 콜백이 Period보다 오래 걸리면, 다음 콜백은 즉시 실행된다(어떤 대기도 없음).

### 일회성/주기적 타이머 비교 예제 

```cpp
#include <winrt/Windows.System.Threading.h>
#include <iostream>
#include <chrono>
#include <thread>

using namespace winrt;
using namespace Windows::System::Threading;

IAsyncAction DemonstrateTimers()
{
    std::cout << "=== 타이머 비교 데모 ===" << std::endl;
    
    int oneTimeCallCount = 0;
    
    // 1. 일회성 타이머
    std::cout << "\n[일회성 타이머] 500ms 후 한 번 실행" << std::endl;
    auto oneTimeTimer = ThreadPoolTimer::CreateTimer(
        [&oneTimeCallCount](IThreadPoolTimer const&)
        {
            oneTimeCallCount++;
            std::cout << "일회성 타이머 콜백 (호출 횟수: " << oneTimeCallCount << ")" << std::endl;
        },
        std::chrono::milliseconds(500));
    
    // 일회성 타이머가 실행될 때까지 대기
    co_await resume_after(std::chrono::milliseconds(1000));
    
    std::cout << "일회성 타이머 최종 호출 횟수: " << oneTimeCallCount << std::endl;
    
    // 2. 주기적 타이머
    std::cout << "\n[주기적 타이머] 300ms 주기로 반복 실행" << std::endl;
    
    int periodicCallCount = 0;
    auto periodicTimer = ThreadPoolTimer::CreatePeriodicTimer(
        [&periodicCallCount](IThreadPoolTimer const&)
        {
            periodicCallCount++;
            std::cout << "주기적 타이머 콜백 (호출 횟수: " << periodicCallCount << ")" << std::endl;
        },
        std::chrono::milliseconds(300));
    
    // 1초 동안 주기적 타이머 실행
    co_await resume_after(std::chrono::milliseconds(1000));
    
    // 주기적 타이머 취소
    periodicTimer.Cancel();
    std::cout << "주기적 타이머 취소" << std::endl;
    
    std::cout << "주기적 타이머 최종 호출 횟수: " << periodicCallCount << std::endl;
}

int main()
{
    init_apartment();
    
    auto demo = DemonstrateTimers();
    demo.get();
    
    return 0;
}
```

이 예제를 실행하면 일회성 타이머는 한 번만 호출되지만, 주기적 타이머는 여러 번 호출되는 것을 확인할 수 있다. 주기적 타이머는 명시적으로 `Cancel()`을 호출해야 종료된다. 만약 취소하지 않으면 프로그램이 종료될 때까지 계속 실행된다.
  


## 6.3 타이머 정확도와 해상도

### API 설명
타이머의 정확도는 절대적이지 않다. 운영체제는 모든 타이머 요청을 정확하게 처리하려고 노력하지만, 시스템 부하, CPU 사용률, 타이머 콜백의 실행 시간 등 여러 요인에 의해 영향을 받는다.

타이머의 정확도는 두 가지 개념으로 나뉜다:
- **지연(Latency)**: 타이머가 실행되어야 할 시간과 실제로 실행된 시간의 차이
- **지터(Jitter)**: 여러 타이머 호출 사이의 시간 간격 변동

게임 서버에서는 보통 지터를 최소화하는 것이 중요하다. 게임 틱의 시간 간격이 일정해야 게임 로직이 예측 가능하기 때문이다.

WinRT의 타이머는 일반적으로 밀리초 수준의 정확도를 제공한다. 마이크로초 수준의 정확도가 필요하면 저수준 Windows API(예: `QueryPerformanceCounter`)를 사용해야 한다.

### 타이머 정확도 측정 예제 

```cpp
#include <winrt/Windows.System.Threading.h>
#include <iostream>
#include <chrono>
#include <vector>

using namespace winrt;
using namespace Windows::System::Threading;

IAsyncAction MeasureTimerAccuracy()
{
    std::cout << "=== 타이머 정확도 측정 ===" << std::endl;
    
    const int NUM_SAMPLES = 10;
    std::vector<long long> intervals;
    auto lastTime = std::chrono::high_resolution_clock::now();
    
    std::cout << "300ms 주기로 타이머 정확도 측정 중..." << std::endl;
    
    auto timer = ThreadPoolTimer::CreatePeriodicTimer(
        [&intervals, &lastTime](IThreadPoolTimer const&)
        {
            auto currentTime = std::chrono::high_resolution_clock::now();
            auto interval = std::chrono::duration_cast<std::chrono::milliseconds>(
                currentTime - lastTime).count();
            intervals.push_back(interval);
            lastTime = currentTime;
            
            std::cout << "  호출 " << intervals.size() << ": " 
                     << interval << "ms" << std::endl;
        },
        std::chrono::milliseconds(300));
    
    // 샘플 수집
    co_await resume_after(std::chrono::milliseconds(3000));
    
    timer.Cancel();
    
    // 통계 계산
    if (intervals.size() > 1)
    {
        long long minInterval = intervals[0];
        long long maxInterval = intervals[0];
        long long sumInterval = 0;
        
        for (auto interval : intervals)
        {
            minInterval = std::min(minInterval, interval);
            maxInterval = std::max(maxInterval, interval);
            sumInterval += interval;
        }
        
        long long avgInterval = sumInterval / intervals.size();
        
        std::cout << "\n=== 측정 결과 ===" << std::endl;
        std::cout << "최소 간격: " << minInterval << "ms" << std::endl;
        std::cout << "최대 간격: " << maxInterval << "ms" << std::endl;
        std::cout << "평균 간격: " << avgInterval << "ms" << std::endl;
        std::cout << "지터(최대-최소): " << (maxInterval - minInterval) << "ms" << std::endl;
    }
}

int main()
{
    init_apartment();
    
    auto measure = MeasureTimerAccuracy();
    measure.get();
    
    return 0;
}
```

이 예제는 타이머의 각 호출 시간을 측정하고, 간격의 통계를 계산한다. 측정 결과를 보면 타이머의 지터가 얼마나 되는지 알 수 있다. 일반적으로 몇 밀리초의 지터가 있을 것으로 예상된다.
  


## 6.4 타이머 취소와 정리

### API 설명
타이머를 더 이상 필요 없을 때는 `Cancel()` 메서드로 취소해야 한다. 특히 주기적 타이머는 명시적으로 취소하지 않으면 프로그램 종료 시까지 계속 실행된다.

타이머를 취소할 때 주의할 점이 있다:
- 이미 실행 중인 콜백은 완료될 때까지 계속 실행된다. `Cancel()`을 호출해도 콜백이 즉시 중단되지 않는다.
- `Cancel()` 호출 후에 새로운 콜백은 실행되지 않는다.
- 주의하지 않으면 데드락이 발생할 수 있다. 콜백 내에서 타이머를 취소하면서 동시에 타이머가 완료될 때까지 기다리는 경우 등이다.

안전한 타이머 정리는 다음 원칙을 따른다:
1. 타이머를 멤버 변수로 보유하고, 객체가 소멸할 때 취소한다.
2. 멀티스레딩 환경에서는 타이머 취소를 스레드 안전하게 구현한다.
3. 취소 후에는 타이머 변수를 안전하게 처리한다(예: nullptr 할당).

### 안전한 리소스 해제 예제 

```cpp
#include <winrt/Windows.System.Threading.h>
#include <iostream>
#include <memory>

using namespace winrt;
using namespace Windows::System::Threading;

class ManagedTimer
{
private:
    IThreadPoolTimer m_timer = nullptr;
    
public:
    void Start(int periodMs)
    {
        std::cout << "타이머 시작" << std::endl;
        m_timer = ThreadPoolTimer::CreatePeriodicTimer(
            [](IThreadPoolTimer const&)
            {
                std::cout << "  타이머 콜백 실행" << std::endl;
            },
            std::chrono::milliseconds(periodMs));
    }
    
    void Stop()
    {
        if (m_timer != nullptr)
        {
            std::cout << "타이머 취소" << std::endl;
            m_timer.Cancel();
            m_timer = nullptr;  // 안전하게 nullptr로 설정
        }
    }
    
    ~ManagedTimer()
    {
        Stop();  // 소멸자에서 자동으로 정리
    }
};

IAsyncAction SafeTimerCleanupDemo()
{
    {
        ManagedTimer timer;
        timer.Start(500);
        
        // 2초 동안 실행
        co_await resume_after(std::chrono::seconds(2));
        
        // 블록 벗어나면서 소멸자가 호출되어 자동 정리
    }
    
    std::cout << "타이머가 정리됨" << std::endl;
}

int main()
{
    init_apartment();
    
    auto demo = SafeTimerCleanupDemo();
    demo.get();
    
    return 0;
}
```

`ManagedTimer` 클래스는 RAII(Resource Acquisition Is Initialization) 패턴을 구현한다. 타이머가 생성될 때 시작되고, 객체가 소멸될 때 자동으로 정리된다. 이 패턴을 사용하면 타이머 누수를 방지할 수 있다.
  


## 6.5 종합 예제: 게임 틱 시뮬레이터

### 코드의 목적
이 예제는 온라인 게임 서버의 게임 루프를 시뮬레이션한다. 게임 서버는 고정된 프레임레이트(예: 30 FPS, 60 FPS)로 동작해야 한다. 각 틱마다 플레이어 입력을 처리하고, 게임 상태를 업데이트하고, 변경사항을 클라이언트에 전송해야 한다.

이 예제에서는 다음을 구현한다:
1. 고정 프레임레이트 유지(목표: 60 FPS = 16.67ms 주기)
2. 델타 타임(Delta Time) 계산: 이전 틱과의 시간 차이
3. 타이머 드리프트 보정: 누적된 오차를 보정해서 평균 프레임레이트 유지

### 설계 결정 이유
첫 번째, 목표 프레임레이트를 설정한다. 60 FPS는 많은 게임 서버의 표준이며, 16.67ms마다 한 틱을 실행한다는 의미다.

두 번째, 델타 타임을 추적한다. 이전 틱 이후 경과한 실제 시간을 계산해서 게임 로직에 전달한다. 이렇게 하면 프레임레이트 변동에 관계없이 게임 로직이 일관되게 동작한다.

세 번째, 타이머 드리프트를 보정한다. 타이머는 완벽하지 않으므로, 오차가 누적될 수 있다. 정기적으로 오차를 조정해서 평균 프레임레이트를 유지한다.

네 번째, 게임 상태와 통계를 추적한다. 틱 수, 평균 델타 타임, 프레임 드롭(목표보다 느린 틱) 등을 기록한다.

### 주요 부분 상세 해설
게임 루프는 주기적 타이머로 구현된다. 매 틱마다 다음이 수행된다:
1. 현재 시간 기록
2. 델타 타임 계산 (현재 시간 - 이전 시간)
3. 게임 로직 업데이트 (플레이어 입력 처리, 상태 계산)
4. 클라이언트에 상태 전송 (시뮬레이션)

드리프트 보정은 다음과 같이 작동한다:
- 누적 오차(accumulated error)를 추적한다.
- 매 틱마다 오차를 누적한다: 오류 = (목표 시간 - 실제 시간)
- 정기적으로(예: 매 1초) 오차가 커지면, 목표 시간을 일시적으로 조정한다.

### 전체 코드 

```cpp
#include <winrt/Windows.System.Threading.h>
#include <iostream>
#include <chrono>
#include <iomanip>

using namespace winrt;
using namespace Windows::System::Threading;

class GameTickSimulator
{
private:
    static const int TARGET_FPS = 60;
    static const int TARGET_FRAME_TIME_MS = 1000 / TARGET_FPS;  // ~16.67ms
    
    struct GameState
    {
        uint64_t tickCount = 0;
        double deltaTime = 0.0;
        double totalTime = 0.0;
    };
    
    struct Statistics
    {
        uint64_t totalTicks = 0;
        double totalDeltaTime = 0.0;
        double averageDeltaTime = 0.0;
        int frameDropCount = 0;  // 목표 시간보다 느린 틱
        double maxDeltaTime = 0.0;
        double minDeltaTime = 999999.0;
    };
    
    GameState m_gameState;
    Statistics m_stats;
    IThreadPoolTimer m_gameTimer = nullptr;
    std::chrono::high_resolution_clock::time_point m_lastTickTime;
    std::chrono::high_resolution_clock::time_point m_startTime;
    double m_accumulatedError = 0.0;
    
public:
    void Start()
    {
        std::cout << "=== 게임 틱 시뮬레이터 ===" << std::endl;
        std::cout << "목표 FPS: " << TARGET_FPS << " (프레임당 " 
                 << TARGET_FRAME_TIME_MS << "ms)" << std::endl << std::endl;
        
        m_startTime = std::chrono::high_resolution_clock::now();
        m_lastTickTime = m_startTime;
        
        m_gameTimer = ThreadPoolTimer::CreatePeriodicTimer(
            [this](IThreadPoolTimer const&)
            {
                OnGameTick();
            },
            std::chrono::milliseconds(TARGET_FRAME_TIME_MS));
    }
    
    void Stop()
    {
        if (m_gameTimer != nullptr)
        {
            m_gameTimer.Cancel();
            m_gameTimer = nullptr;
        }
    }
    
    void PrintStatistics() const
    {
        std::cout << "\n=== 게임 통계 ===" << std::endl;
        std::cout << "총 틱 수: " << m_stats.totalTicks << std::endl;
        std::cout << "실행 시간: " << m_stats.totalDeltaTime << "초" << std::endl;
        
        if (m_stats.totalTicks > 0)
        {
            std::cout << std::fixed << std::setprecision(3);
            std::cout << "평균 델타 타임: " << m_stats.averageDeltaTime << "ms" << std::endl;
            std::cout << "최소 델타 타임: " << m_stats.minDeltaTime << "ms" << std::endl;
            std::cout << "최대 델타 타임: " << m_stats.maxDeltaTime << "ms" << std::endl;
            std::cout << "프레임 드롭 수: " << m_stats.frameDropCount << std::endl;
            
            double actualFPS = m_stats.totalTicks / m_stats.totalDeltaTime;
            std::cout << "실제 평균 FPS: " << actualFPS << std::endl;
        }
    }
    
private:
    void OnGameTick()
    {
        auto currentTime = std::chrono::high_resolution_clock::now();
        
        // 델타 타임 계산 (ms)
        double deltaTimeMs = std::chrono::duration_cast<
            std::chrono::duration<double, std::milli>>(
            currentTime - m_lastTickTime).count();
        
        m_lastTickTime = currentTime;
        
        // 오차 누적
        m_accumulatedError += (TARGET_FRAME_TIME_MS - deltaTimeMs);
        
        // 통계 업데이트
        m_stats.totalTicks++;
        m_stats.totalDeltaTime += deltaTimeMs / 1000.0;
        m_stats.maxDeltaTime = std::max(m_stats.maxDeltaTime, deltaTimeMs);
        m_stats.minDeltaTime = std::min(m_stats.minDeltaTime, deltaTimeMs);
        
        if (deltaTimeMs > TARGET_FRAME_TIME_MS * 1.1)  // 10% 이상 초과
        {
            m_stats.frameDropCount++;
        }
        
        // 게임 상태 업데이트
        m_gameState.tickCount++;
        m_gameState.deltaTime = deltaTimeMs / 1000.0;  // 초 단위로 변환
        m_gameState.totalTime += m_gameState.deltaTime;
        
        // 드리프트 보정: 1초마다 오차 확인
        if (m_gameState.tickCount % TARGET_FPS == 0)
        {
            PrintTickReport();
            
            // 누적 오차가 크면 조정
            if (std::abs(m_accumulatedError) > TARGET_FRAME_TIME_MS * 0.5)
            {
                std::cout << "    [드리프트 보정] 누적 오차: " 
                         << std::fixed << std::setprecision(1)
                         << m_accumulatedError << "ms" << std::endl;
                m_accumulatedError = 0.0;  // 오차 리셋
            }
        }
        
        // 게임 로직 시뮬레이션
        SimulateGameLogic();
    }
    
    void PrintTickReport()
    {
        double fps = TARGET_FPS;  // 이상적인 경우
        if (m_stats.totalTicks > 0)
        {
            fps = 1000.0 / m_stats.averageDeltaTime;
        }
        
        m_stats.averageDeltaTime = m_stats.totalDeltaTime * 1000.0 / m_stats.totalTicks;
        
        std::cout << "[틱 " << m_stats.totalTicks << "] "
                 << "델타: " << std::fixed << std::setprecision(2)
                 << m_stats.averageDeltaTime << "ms, "
                 << "FPS: " << (1000.0 / m_stats.averageDeltaTime) << std::endl;
    }
    
    void SimulateGameLogic()
    {
        // 게임 로직 시뮬레이션 (매우 간단함)
        // 실제 게임에서는 여기서:
        // - 플레이어 입력 처리
        // - 게임 물리 계산
        // - AI 업데이트
        // - 클라이언트에 상태 전송 등이 수행됨
    }
};

IAsyncAction RunGameSimulation()
{
    GameTickSimulator simulator;
    simulator.Start();
    
    // 5초 동안 실행
    co_await resume_after(std::chrono::seconds(5));
```

### 코드 실행 흐름 설명

1. 시뮬레이터가 시작되면 16.67ms 주기의 타이머가 생성된다.
2. 매 틱마다 `OnGameTick()`이 호출된다.
3. 현재 시간을 기록하고, 이전 틱 이후의 경과 시간(델타 타임)을 계산한다.
4. 누적 오차를 추적하고, 1초마다 통계를 출력한다.
5. 오차가 커지면 보정을 수행한다(실제 코드에서는 타이머 주기를 조정할 수도 있음).
6. 게임 로직을 시뮬레이션한다(현재 버전은 빈 함수).

### 개선 방안과 확장 가능성

**현재 한계:**

첫 번째, 게임 로직이 구현되지 않았다. 실제 게임 서버에서는 `SimulateGameLogic()` 함수에 플레이어 상태 업데이트, 물리 계산, 네트워크 전송 등이 포함될 것이다.

두 번째, 드리프트 보정이 단순하다. 오차를 기록만 하고 실제로는 타이머 주기를 조정하지 않는다.

세 번째, 멀티스레딩 환경에서의 동기화가 없다. 게임 상태에 여러 스레드가 접근하는 경우 뮤텍스가 필요하다.

**개선 방안:**

첫 번째, 플레이어 입력 처리를 추가할 수 있다. 입력 큐에서 대기 중인 입력을 꺼내서 게임 상태를 업데이트한다.

두 번째, 멀티 스레드 지원을 추가할 수 있다. 네트워크 I/O, 물리 계산, 상태 전송을 별도 스레드에서 병렬로 수행한다.

세 번째, 적응형 프레임레이트를 구현할 수 있다. 시스템 부하에 따라 타이머 주기를 동적으로 조정한다.

네 번째, 프로파일링을 추가할 수 있다. 각 게임 로직 부분이 얼마나 오래 걸리는지 측정해서 병목을 식별한다.

**게임 서버 적용 예시:**

이 패턴은 실제 게임 서버에서 다음과 같이 사용된다:

```cpp
void SimulateGameLogic()
{
    // 플레이어 입력 처리 (네트워크에서 온 데이터)
    ProcessPlayerInputs();
    
    // 게임 상태 업데이트
    UpdatePlayerPositions(m_gameState.deltaTime);
    UpdateGameLogic(m_gameState.deltaTime);
    
    // 변경사항을 모든 클라이언트에 전송
    BroadcastGameState();
    
    // 정기적 데이터베이스 저장 (매 100틱)
    if (m_gameState.tickCount % (TARGET_FPS * 100) == 0)
    {
        SaveGameState();
    }
}
```

다음 장에서는 동기화 객체를 다룬다. 멀티스레드 환경에서 안전하게 데이터를 공유하기 위한 뮤텍스, 조건 변수, 원자적 연산 등을 학습한다.

---

이 장에서 배운 내용을 정리하면:

1. **ThreadPoolTimer API**: `CreateTimer()`로 일회성 타이머, `CreatePeriodicTimer()`로 주기적 타이머를 생성한다.

2. **타이머 생명주기**: 타이머가 시작되면 자동으로 실행되며, 주기적 타이머는 `Cancel()`로 명시적으로 취소해야 한다.

3. **타이머 정확도**: 타이머는 밀리초 수준의 정확도를 제공하지만, 완벽하지는 않으므로 지터와 드리프트를 고려해야 한다.

4. **안전한 정리**: RAII 패턴을 사용해서 타이머 리소스를 안전하게 관리한다.

5. **게임 루프 구현**: 고정 프레임레이트, 델타 타임, 드리프트 보정을 조합해서 안정적인 게임 루프를 구현할 수 있다.

6. **통계 추적**: 프레임레이트, 델타 타임, 프레임 드롭 등을 추적해서 게임 성능을 모니터링할 수 있다.

이제 안정적인 타이머 기반 게임 루프를 만들 수 있다. 다음 장에서는 멀티스레드 환경에서 이 게임 루프의 상태를 안전하게 보호하는 방법을 배운다.
