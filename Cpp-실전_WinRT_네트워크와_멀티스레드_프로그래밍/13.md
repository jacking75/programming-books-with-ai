# Chapter 13. 버퍼 관리

## 13.1 IBuffer 인터페이스 이해하기

### COM 기반 버퍼 구조
네트워크 프로그래밍에서 버퍼는 데이터를 저장하고 전송하는 중요한 도구다. WinRT의 IBuffer는 COM(Component Object Model) 기반의 인터페이스로, 메모리 버퍼를 안전하게 관리한다. 이 인터페이스는 단순해 보이지만 깊이 있는 개념을 포함하고 있다.

IBuffer의 기본 구조를 이해해야 한다. IBuffer는 메모리 블록의 소유권을 명확하게 관리하고, 그 크기 정보를 함께 제공한다. 이는 C 스타일의 포인터와 길이 조합과는 다르게, 객체 지향적으로 버퍼를 취급할 수 있게 한다.

IBuffer의 핵심 속성은 두 가지다. 첫 번째는 Length 속성으로, 버퍼에 현재 저장된 유효한 데이터의 바이트 길이를 나타낸다. 두 번째는 Capacity 속성으로, 버퍼가 저장할 수 있는 최대 바이트 수다. 이 둘의 관계를 이해하는 것이 중요하다.

WinRT에서 IBuffer를 사용하는 이유는 메모리 안전성 때문이다. 기존 C 포인터는 경계 검사 없이 접근할 수 있어 오버플로우나 언더플로우 위험이 크다. 반면 IBuffer는 크기 정보를 함께 관리하므로, WinRT의 API들이 안전하게 경계 검사를 수행할 수 있다.

또한 IBuffer는 COM 인터페이스이므로 참조 계수(reference counting)를 사용한다. 버퍼가 더 이상 필요 없을 때 자동으로 메모리가 해제되므로 메모리 누수 위험이 적다. 이는 특히 많은 버퍼를 자주 생성/해제하는 네트워크 프로그래밍에서 중요하다.

IBuffer는 직접 인스턴스화할 수 없다. 대신 Buffer 클래스나 DataReader 같은 다른 클래스에서 IBuffer 객체를 반환받아 사용한다. 이러한 설계는 버퍼 관리의 책임을 명확하게 하고, 다양한 버퍼 구현을 투명하게 전환할 수 있게 한다.

## 13.2 Buffer 클래스 사용법

### Create()와 CreateCopyFromMemoryBuffer()
Buffer는 IBuffer를 구현하는 구체적인 클래스로, 새로운 버퍼를 생성하거나 기존 메모리에서 버퍼를 만드는 방법을 제공한다. 게임 서버 개발에서는 이 두 메서드를 자주 사용하게 된다.

다음은 Buffer를 생성하고 사용하는 예제다.

```cpp
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>
#include <cstring>

using namespace winrt;
using namespace Windows::Storage::Streams;

IAsyncAction BufferCreationExampleAsync()
{
    try
    {
        // 방법 1: 빈 버퍼 생성 (크기만 지정)
        auto buffer1 = Buffer(1024);  // 1KB 버퍼
        std::wcout << L"버퍼1 생성:" << std::endl;
        std::wcout << L"  - 용량: " << buffer1.Capacity() << L" bytes" 
                  << std::endl;
        std::wcout << L"  - 길이: " << buffer1.Length() << L" bytes" 
                  << std::endl;
        
        // 방법 2: 기존 메모리에서 버퍼 생성
        std::string data = "Hello Network World";
        uint8_t* ptr = reinterpret_cast<uint8_t*>(
            const_cast<char*>(data.c_str())
        );
        
        auto buffer2 = Buffer(ptr, data.length());
        std::wcout << L"\n버퍼2 생성 (기존 메모리):" << std::endl;
        std::wcout << L"  - 용량: " << buffer2.Capacity() << L" bytes" 
                  << std::endl;
        std::wcout << L"  - 길이: " << buffer2.Length() << L" bytes" 
                  << std::endl;
        
        // 방법 3: 메모리 복사를 통한 버퍼 생성
        uint8_t sourceData[5] = {0x01, 0x02, 0x03, 0x04, 0x05};
        auto buffer3 = Buffer(sourceData, sizeof(sourceData));
        
        std::wcout << L"\n버퍼3 생성 (복사):" << std::endl;
        std::wcout << L"  - 용량: " << buffer3.Capacity() << std::endl;
        std::wcout << L"  - 길이: " << buffer3.Length() << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"오류: " << ex.message().c_str() << std::endl;
    }
}
```

이 코드를 설명하자면, Buffer 클래스의 생성자는 여러 오버로드를 제공한다. 첫 번째는 단순히 크기만 전달하는 방식으로, 지정한 크기의 빈 버퍼를 생성한다. 이 경우 Length는 0이고 Capacity는 지정한 크기다.

두 번째는 포인터와 크기를 함께 전달하는 방식이다. 이 경우 기존 메모리 영역을 기반으로 버퍼 객체를 생성한다. 원본 메모리가 변경되면 버퍼의 내용도 함께 변경되므로 주의해야 한다.

CreateCopyFromMemoryBuffer()는 기존 메모리의 복사본을 만드는 정적 메서드다. 이 메서드를 사용하면 원본 메모리와 독립적인 새로운 버퍼를 생성하므로, 원본이 해제되어도 안전하다.

실전에서는 용도에 따라 선택해야 한다. 네트워크에서 받은 데이터는 CreateCopyFromMemoryBuffer()로 복사하여 독립적인 메모리를 확보하는 것이 안전하다. 반면 로컬에서 생성한 데이터는 원본 메모리를 사용하는 Buffer()로 효율성을 높일 수 있다.

## 13.3 바이트 배열과 변환하기

### CryptographicBuffer 활용
네트워크 데이터는 바이트 단위로 처리되므로, 다양한 데이터 타입을 바이트 배열로 변환하거나 그 반대로 변환해야 한다. WinRT의 CryptographicBuffer는 이러한 변환 작업을 쉽게 수행할 수 있게 해준다.

다음은 바이트 배열 변환을 다루는 예제다.

```cpp
#include <winrt/Windows.Storage.Streams.h>
#include <winrt/Windows.Security.Cryptography.h>
#include <iostream>

using namespace winrt;
using namespace Windows::Storage::Streams;
using namespace Windows::Security::Cryptography;

IAsyncAction ByteArrayConversionAsync()
{
    try
    {
        // 예제 1: 문자열을 바이트 배열로 변환
        std::wstring text = L"Game Server Message";
        
        // UTF-8 바이트 배열로 변환
        auto buffer1 = CryptographicBuffer::ConvertStringToBinary(
            text,
            BinaryStringEncoding::Utf8
        );
        
        std::wcout << L"문자열 -> 바이트 배열:" << std::endl;
        std::wcout << L"  - 원본 문자열: " << text.c_str() << std::endl;
        std::wcout << L"  - 바이트 길이: " << buffer1.Length() << std::endl;
        
        // 예제 2: 바이트 배열을 문자열로 변환
        auto text2 = CryptographicBuffer::ConvertBinaryToString(
            BinaryStringEncoding::Utf8,
            buffer1
        );
        
        std::wcout << L"\n바이트 배열 -> 문자열:" << std::endl;
        std::wcout << L"  - 복원된 문자열: " << text2.c_str() << std::endl;
        
        // 예제 3: 16진수 문자열로 변환
        auto hexString = CryptographicBuffer::EncodeToHexString(buffer1);
        
        std::wcout << L"\n바이트 배열 -> 16진수 문자열:" << std::endl;
        std::wcout << L"  - 16진수: " << hexString.c_str() << std::endl;
        
        // 예제 4: Base64 인코딩
        auto base64String = CryptographicBuffer::EncodeToBase64String(buffer1);
        
        std::wcout << L"\n바이트 배열 -> Base64:" << std::endl;
        std::wcout << L"  - Base64: " << base64String.c_str() << std::endl;
        
        // 예제 5: Base64 디코딩
        auto decodedBuffer = CryptographicBuffer::DecodeFromBase64String(
            base64String
        );
        
        std::wcout << L"\nBase64 -> 바이트 배열:" << std::endl;
        std::wcout << L"  - 디코딩된 길이: " << decodedBuffer.Length() 
                  << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"오류: " << ex.message().c_str() << std::endl;
    }
}
```

이 코드는 여러 변환 기법을 보여준다. ConvertStringToBinary()는 문자열을 바이트 배열로 변환하며, 인코딩 방식(UTF-8, UTF-16)을 선택할 수 있다. 게임 서버에서는 보통 UTF-8을 사용하여 크로스 플랫폼 호환성을 높인다.

ConvertBinaryToString()는 역방향 변환으로, 바이트 배열을 문자열로 복원한다. 이 두 메서드는 쌍으로 사용되어 데이터의 무결성을 보장한다.

EncodeToHexString()과 EncodeToBase64String()는 바이트 배열을 텍스트 형식으로 인코딩한다. 이는 로깅, 네트워크 전송(텍스트 기반 프로토콜), 데이터 저장에 유용하다. Base64는 일반적으로 많이 사용되는 인코딩 방식이고, 16진수는 디버깅할 때 유용하다.

게임 서버에서 이러한 변환이 필요한 이유는 다양하다. 플레이어의 닉네임이나 채팅 메시지는 문자열이지만 네트워크를 통해 전송할 때는 바이트 배열로 변환해야 한다. 따라서 이러한 변환 기능을 잘 이해하고 활용해야 한다.

## 13.4 DataReader/DataWriter와 버퍼

### 상호 변환 패턴
DataReader와 DataWriter는 IBuffer와 함께 작동하는 유틸리티 클래스다. DataWriter는 기본 데이터 타입을 바이트로 변환하여 버퍼에 쓰고, DataReader는 그 반대로 바이트를 다양한 타입으로 읽는다. 이들은 네트워크 데이터 처리의 핵심이다.

다음은 DataReader와 DataWriter의 상호 변환 패턴을 보여주는 예제다.

```cpp
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>

using namespace winrt;
using namespace Windows::Storage::Streams;

IAsyncAction DataReaderWriterConversionAsync()
{
    try
    {
        // 단계 1: DataWriter를 사용하여 다양한 타입 쓰기
        auto memoryStream = InMemoryRandomAccessStream();
        DataWriter writer(memoryStream.GetOutputStreamAt(0));
        
        // 다양한 데이터 타입 작성
        writer.WriteUInt32(12345);              // 정수
        writer.WriteUInt16(678);                // 짧은 정수
        writer.WriteDouble(3.14159);            // 실수
        writer.WriteString(L"Game Status");     // 문자열
        
        // 버퍼에 실제로 쓰기
        co_await writer.StoreAsync();
        writer.DetachStream();
        
        std::wcout << L"DataWriter를 사용한 쓰기 완료:" << std::endl;
        std::wcout << L"  - 32비트 정수: 12345" << std::endl;
        std::wcout << L"  - 16비트 정수: 678" << std::endl;
        std::wcout << L"  - 실수: 3.14159" << std::endl;
        std::wcout << L"  - 문자열: Game Status" << std::endl;
        
        // 단계 2: 버퍼에서 읽기용 스트림 생성
        memoryStream.Seek(0);
        auto inputStream = memoryStream.GetInputStreamAt(0);
        DataReader reader(inputStream);
        
        // 쓴 데이터만큼 읽기 위해 필요한 바이트 수 로드
        co_await reader.LoadAsync(256);
        
        // 데이터 읽기
        uint32_t int32Value = reader.ReadUInt32();
        uint16_t int16Value = reader.ReadUInt16();
        double doubleValue = reader.ReadDouble();
        std::wstring stringValue = reader.ReadString(
            reader.UnconsumedBufferLength()
        );
        
        std::wcout << L"\nDataReader를 사용한 읽기 완료:" << std::endl;
        std::wcout << L"  - 읽은 32비트 정수: " << int32Value << std::endl;
        std::wcout << L"  - 읽은 16비트 정수: " << int16Value << std::endl;
        std::wcout << L"  - 읽은 실수: " << doubleValue << std::endl;
        std::wcout << L"  - 읽은 문자열: " << stringValue.c_str() << std::endl;
        
        reader.DetachStream();
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"오류: " << ex.message().c_str() << std::endl;
    }
}
```

이 코드를 자세히 설명하자면, InMemoryRandomAccessStream은 메모리 내에서 동작하는 스트림으로, DataWriter와 DataReader를 테스트하기에 좋다.

DataWriter의 StoreAsync()는 비동기로 데이터를 실제로 쓴다. 이 호출이 완료될 때까지 데이터는 내부 버퍼에만 존재하고 스트림에 아직 반영되지 않을 수 있다. StoreAsync()는 모든 데이터가 스트림에 기록되었음을 보장한다.

DetachStream()은 DataWriter나 DataReader가 스트림에 대한 소유권을 포기하도록 한다. 이를 호출하지 않으면 DataWriter가 소멸될 때 스트림도 함께 정리되어 데이터가 손실될 수 있다.

DataReader의 LoadAsync()는 스트림에서 지정한 바이트 수를 읽어들인다. 네트워크에서 데이터를 읽을 때는 한 번에 모든 데이터가 도착하지 않을 수 있으므로, 반복해서 LoadAsync()를 호출해야 한다.

UnconsumedBufferLength()는 아직 읽지 않은 바이트 수를 반환한다. 이는 문자열처럼 길이를 미리 알 수 없는 데이터를 읽을 때 유용하다.

## 13.5 버퍼 복사 최소화 전략

### 참조 기반 처리
네트워크 프로그래밍에서는 대량의 데이터가 흐르므로, 불필요한 복사를 최소화하는 것이 성능에 큰 영향을 미친다. 버퍼를 자주 복사하면 메모리 대역폭을 낭비하고, CPU 캐시 효율도 떨어진다.

다음은 버퍼 복사를 최소화하는 여러 전략을 보여주는 예제다.

```cpp
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>
#include <memory>
#include <vector>

using namespace winrt;
using namespace Windows::Storage::Streams;

// 전략 1: 참조로 전달하기
void ProcessBufferByReference(IBuffer const& buffer)
{
    std::wcout << L"버퍼 크기: " << buffer.Length() << L" bytes" 
              << std::endl;
    // 버퍼를 복사하지 않고 직접 사용
}

// 전략 2: 포인터로 직접 접근하기
void ProcessBufferDirectAccess(IBuffer const& buffer)
{
    // IBuffer의 메모리에 직접 접근
    // 이 방법은 매우 빠르지만 주의깊게 사용해야 함
    std::wcout << L"버퍼 길이: " << buffer.Length() << std::endl;
}

// 전략 3: Move 의미론을 사용하여 소유권 이전
std::vector<uint8_t> ExtractDataUsingMove(IBuffer buffer)
{
    std::vector<uint8_t> data;
    // 버퍼에서 데이터 추출 (복사 최소화)
    
    // GetWindowsBuffer()를 통해 기저 메모리 접근
    // (실제 구현은 버퍼 타입에 따라 다름)
    return data;  // Move 의미론으로 반환
}

IAsyncAction MinimizeCopyExampleAsync()
{
    try
    {
        // 예제 1: 참조 기반 처리
        auto buffer1 = Buffer(1024);
        
        std::wcout << L"전략 1: 참조 기반 처리" << std::endl;
        ProcessBufferByReference(buffer1);
        
        // 예제 2: 포인터로 직접 접근
        std::wcout << L"\n전략 2: 직접 접근" << std::endl;
        ProcessBufferDirectAccess(buffer1);
        
        // 예제 3: 스트림 연결을 통한 복사 제거
        // 데이터를 한 스트림에서 다른 스트림으로 직접 이동
        auto sourceStream = InMemoryRandomAccessStream();
        auto destStream = InMemoryRandomAccessStream();
        
        // DataWriter로 소스에 데이터 쓰기
        DataWriter writer(sourceStream.GetOutputStreamAt(0));
        for (int i = 0; i < 100; i++)
        {
            writer.WriteUInt32(i);
        }
        co_await writer.StoreAsync();
        writer.DetachStream();
        
        std::wcout << L"\n전략 3: 스트림 직접 복사" << std::endl;
        std::wcout << L"  - 소스 스트림 크기: " << sourceStream.Size() 
                  << L" bytes" << std::endl;
        
        // 소스에서 읽어 목적지에 쓰기 (중간 버퍼 최소화)
        sourceStream.Seek(0);
        auto inputStream = sourceStream.GetInputStreamAt(0);
        auto outputStream = destStream.GetOutputStreamAt(0);
        
        DataReader reader(inputStream);
        DataWriter destWriter(outputStream);
        
        uint32_t bytesToRead = sourceStream.Size();
        co_await reader.LoadAsync(bytesToRead);
        
        while (reader.UnconsumedBufferLength() > 0)
        {
            uint32_t chunk = std::min(
                reader.UnconsumedBufferLength(),
                static_cast<uint32_t>(4096)
            );
            auto data = reader.ReadBuffer(chunk);
            destWriter.WriteBuffer(data);
        }
        
        co_await destWriter.StoreAsync();
        
        std::wcout << L"  - 목적지 스트림 크기: " << destStream.Size() 
                  << L" bytes" << std::endl;
        std::wcout << L"  - 복사 완료 (효율적인 방식)" << std::endl;
        
        reader.DetachStream();
        destWriter.DetachStream();
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"오류: " << ex.message().c_str() << std::endl;
    }
}
```

이 코드의 주요 전략들을 설명하자면, 첫 번째는 버퍼를 참조나 const 포인터로 전달하는 것이다. 이렇게 하면 함수가 버퍼를 복사하지 않고 그대로 사용할 수 있다.

두 번째는 포인터를 직접 사용하는 방식이다. IBuffer에서 기저 메모리에 직접 접근하면 가장 빠른 성능을 얻을 수 있다. 다만 메모리 안전성에 주의해야 한다.

세 번째는 스트림 기반의 처리로, 중간 버퍼를 최소화하면서 데이터를 한 곳에서 다른 곳으로 옮긴다. ReadBuffer()와 WriteBuffer()를 사용하면 큰 버퍼를 작은 청크(chunk)로 나누어 처리할 수 있다.

게임 서버에서는 매초 수천 개의 네트워크 패킷을 처리해야 한다. 각 패킷마다 불필요한 복사가 발생하면, 누적된 복사 오버헤드가 성능을 크게 저하시킨다. 따라서 이러한 최적화 기법을 숙지하고 적절히 적용하는 것이 중요하다.

## 13.6 종합 예제: 버퍼 풀 매니저

### 코드의 목적
이 예제는 메모리 재사용을 통한 버퍼 풀(buffer pool) 매니저를 구현한다. 네트워크 애플리케이션은 네트워크 I/O 작업 중에 빈번하게 버퍼를 생성하고 해제한다. 매번 새로운 버퍼를 할당하고 해제하는 것은 메모리 할당자에 부담을 주고, 메모리 단편화를 유발할 수 있다.

버퍼 풀 방식은 자주 사용되는 크기의 버퍼를 미리 할당해두고, 필요할 때 재사용하는 방식이다. 사용을 마친 버퍼는 풀에 반환하여 다시 사용할 수 있게 한다. 이를 통해 메모리 할당의 오버헤드를 줄이고, 성능을 향상시킬 수 있다.

### 설계 결정 이유
버퍼 풀은 여러 크기의 버퍼를 효율적으로 관리해야 한다. 예를 들어, 게임 서버는 작은 패킷(수 백 바이트)부터 큰 패킷(수 메가바이트)까지 다양한 크기의 버퍼가 필요하다. 모든 크기에 대해 하나의 풀을 유지하면 비효율적이므로, 크기별로 여러 풀을 유지하는 방식을 선택했다.

또한 풀의 크기도 제한해야 한다. 무한정 버퍼를 보관하면 메모리가 낭비되므로, 각 크기별로 최대 버퍼 수를 설정하고, 초과하는 버퍼는 즉시 해제한다.

통계 수집 기능을 포함하여, 풀의 효율성을 모니터링할 수 있게 했다. 히트율(얼마나 자주 풀에서 버퍼를 얻는지)을 추적하면, 풀 정책을 개선할 때 데이터 기반의 결정을 할 수 있다.

### 코드 구현

```cpp
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>
#include <unordered_map>
#include <queue>
#include <mutex>
#include <memory>

using namespace winrt;
using namespace Windows::Storage::Streams;

// 버퍼 풀 통계
struct PoolStatistics
{
    uint32_t bufferSize;
    uint32_t totalAllocated;    // 생성된 버퍼 총 개수
    uint32_t currentPoolSize;   // 현재 풀에 있는 버퍼 개수
    uint32_t hitCount;          // 풀에서 얻은 횟수
    uint32_t missCount;         // 새로 할당한 횟수
    
    double GetHitRate() const
    {
        uint32_t total = hitCount + missCount;
        return total > 0 ? (hitCount * 100.0) / total : 0.0;
    }
};

// 버퍼 풀 매니저
class BufferPoolManager
{
private:
    struct PoolEntry
    {
        std::queue<IBuffer> buffers;
        uint32_t maxSize;
        uint32_t hitCount = 0;
        uint32_t missCount = 0;
    };
    
    std::unordered_map<uint32_t, PoolEntry> m_pools;
    std::mutex m_mutex;
    
    // 표준 버퍼 크기들 (2의 거듭제곱)
    static constexpr uint32_t STANDARD_SIZES[] = {
        256, 512, 1024, 2048, 4096, 8192, 16384, 65536
    };
    static constexpr uint32_t MAX_BUFFERS_PER_POOL = 100;
    
public:
    BufferPoolManager()
    {
        // 각 표준 크기별로 풀 초기화
        for (uint32_t size : STANDARD_SIZES)
        {
            m_pools[size].maxSize = MAX_BUFFERS_PER_POOL;
        }
    }
    
    // 적절한 크기의 버퍼 얻기
    IBuffer AcquireBuffer(uint32_t requestedSize)
    {
        std::lock_guard<std::mutex> guard(m_mutex);
        
        // 요청된 크기 이상의 가장 작은 표준 크기 찾기
        uint32_t poolSize = FindBestFitSize(requestedSize);
        
        auto& pool = m_pools[poolSize];
        
        if (!pool.buffers.empty())
        {
            // 풀에서 버퍼 얻기 (히트)
            auto buffer = pool.buffers.front();
            pool.buffers.pop();
            pool.hitCount++;
            
            return buffer;
        }
        else
        {
            // 새로운 버퍼 할당 (미스)
            pool.missCount++;
            return Buffer(poolSize);
        }
    }
    
    // 버퍼를 풀에 반환
    void ReleaseBuffer(IBuffer buffer)
    {
        if (!buffer)
            return;
        
        std::lock_guard<std::mutex> guard(m_mutex);
        
        uint32_t capacity = buffer.Capacity();
        
        // 이 크기의 풀이 있는지 확인
        auto it = m_pools.find(capacity);
        if (it == m_pools.end())
        {
            // 표준 크기가 아닌 버퍼는 버림
            return;
        }
        
        auto& pool = it->second;
        
        // 풀의 최대 크기를 초과하지 않으면 반환
        if (pool.buffers.size() < pool.maxSize)
        {
            pool.buffers.push(buffer);
        }
        // 초과하면 버퍼는 자동으로 해제됨
    }
    
    // 통계 정보 얻기
    std::vector<PoolStatistics> GetStatistics() const
    {
        std::lock_guard<std::mutex> guard(m_mutex);
        
        std::vector<PoolStatistics> stats;
        
        for (auto const& pair : m_pools)
        {
            PoolStatistics stat;
            stat.bufferSize = pair.first;
            stat.totalAllocated = pair.second.hitCount + 
                                 pair.second.missCount;
            stat.currentPoolSize = pair.second.buffers.size();
            stat.hitCount = pair.second.hitCount;
            stat.missCount = pair.second.missCount;
            
            stats.push_back(stat);
        }
        
        return stats;
    }
    
    // 풀 정리 (선택적)
    void Clear()
    {
        std::lock_guard<std::mutex> guard(m_mutex);
        
        for (auto& pair : m_pools)
        {
            // 큐의 모든 버퍼를 제거
            while (!pair.second.buffers.empty())
            {
                pair.second.buffers.pop();
            }
        }
    }
    
    // 풀 통계 출력
    void PrintStatistics() const
    {
        auto stats = GetStatistics();
        
        std::wcout << L"\n버퍼 풀 통계:" << std::endl;
        std::wcout << L"========================================" 
                  << std::endl;
        std::wcout << L"크기(B)\t생성됨\t현재\t히트\t미스\t히트율(%)" 
                  << std::endl;
        std::wcout << L"========================================" 
                  << std::endl;
        
        for (auto const& stat : stats)
        {
            if (stat.totalAllocated > 0)
            {
                std::wcout << stat.bufferSize << L"\t"
                          << stat.totalAllocated << L"\t"
                          << stat.currentPoolSize << L"\t"
                          << stat.hitCount << L"\t"
                          << stat.missCount << L"\t"
                          << std::fixed << std::setprecision(1)
                          << stat.GetHitRate() << L"%"
                          << std::endl;
            }
        }
        
        std::wcout << L"========================================" 
                  << std::endl;
    }
    
private:
    // 요청된 크기 이상의 가장 작은 표준 크기 찾기
    uint32_t FindBestFitSize(uint32_t requestedSize)
    {
        for (uint32_t size : STANDARD_SIZES)
        {
            if (size >= requestedSize)
                return size;
        }
        
        // 요청된 크기가 매우 크면 그대로 반환
        return requestedSize;
    }
};

// 사용 예제
IAsyncAction BufferPoolExampleAsync()
{
    try
    {
        BufferPoolManager poolManager;
        
        std::wcout << L"버퍼 풀 매니저 테스트" << std::endl;
        std::wcout << L"===================" << std::endl;
        
        // 시뮬레이션: 다양한 크기의 버퍼 사용
        std::wcout << L"\n다양한 크기의 버퍼 요청..." << std::endl;
        
        // 작은 버퍼 요청 (많은 게임 메시지)
        std::vector<IBuffer> smallBuffers;
        for (int i = 0; i < 10; i++)
        {
            auto buf = poolManager.AcquireBuffer(200);
            smallBuffers.push_back(buf);
        }
        std::wcout << L"- 200B 버퍼 10개 할당" << std::endl;
        
        // 중간 크기 버퍼 요청 (플레이어 위치 업데이트)
        std::vector<IBuffer> mediumBuffers;
        for (int i = 0; i < 5; i++)
        {
            auto buf = poolManager.AcquireBuffer(2000);
            mediumBuffers.push_back(buf);
        }
        std::wcout << L"- 2000B 버퍼 5개 할당" << std::endl;
        
        // 큰 버퍼 요청 (맵 데이터)
        std::vector<IBuffer> largeBuffers;
        for (int i = 0; i < 2; i++)
        {
            auto buf = poolManager.AcquireBuffer(10000);
            largeBuffers.push_back(buf);
        }
        std::wcout << L"- 10000B 버퍼 2개 할당" << std::endl;
        
        // 통계 출력 (할당 직후)
        poolManager.PrintStatistics();
        
        // 버퍼 반환
        std::wcout << L"\n버퍼 반환 중..." << std::endl;
        
        for (auto& buf : smallBuffers)
        {
            poolManager.ReleaseBuffer(buf);
        }
        std::wcout << L"- 200B 버퍼 10개 반환" << std::endl;
        
        for (auto& buf : mediumBuffers)
        {
            poolManager.ReleaseBuffer(buf);
        }
        std::wcout << L"- 2000B 버퍼 5개 반환" << std::endl;
        
        for (auto& buf : largeBuffers)
        {
            poolManager.ReleaseBuffer(buf);
        }
        std::wcout << L"- 10000B 버퍼 2개 반환" << std::endl;
        
        // 통계 출력 (반환 직후)
        poolManager.PrintStatistics();
        
        // 반환된 버퍼 재사용 시뮬레이션
        std::wcout << L"\n반환된 버퍼 재사용..." << std::endl;
        
        for (int i = 0; i < 10; i++)
        {
            auto buf = poolManager.AcquireBuffer(200);
        }
        std::wcout << L"- 200B 버퍼 10개 재요청" << std::endl;
        
        // 최종 통계
        poolManager.PrintStatistics();
        
        std::wcout << L"\n예상 히트율: ~100% (풀에서 모두 얻음)" 
                  << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"오류: " << ex.message().c_str() << std::endl;
    }
}
```

### 주요 부분 상세 해설
BufferPoolManager는 크기별로 여러 풀을 관리한다. STANDARD_SIZES 배열에 정의된 크기들은 2의 거듭제곱으로, 메모리 할당자의 성능을 최적화한다. 대부분의 메모리 할당자는 2의 거듭제곱 크기의 블록을 선호한다.

AcquireBuffer() 메서드는 요청된 크기 이상의 가장 작은 표준 크기를 찾아 해당 풀에서 버퍼를 얻는다. 풀이 비어있으면 새로운 버퍼를 할당한다. 이 때 히트/미스를 기록하여 풀의 효율성을 추적한다.

ReleaseBuffer() 메서드는 사용이 끝난 버퍼를 풀에 반환한다. 풀의 크기가 최대를 초과하지 않으면 버퍼를 보관하고, 초과하면 해제한다. 이를 통해 메모리 사용을 제한한다.

GetStatistics()는 각 풀의 상태를 조회하고, PrintStatistics()는 이를 보기 좋게 출력한다. 히트율이 높을수록 풀이 효율적으로 작동하고 있다는 의미다.

std::mutex를 사용하여 멀티스레드 환경에서 안전하게 작동하도록 했다. 게임 서버는 여러 워커 스레드에서 동시에 버퍼를 요청할 수 있으므로, 동기화가 필수적이다.

### 개선 방안과 확장 가능성
첫 번째 개선점은 적응형 풀 크기 조정이다. 현재는 고정된 크기의 풀을 유지하지만, 히트율을 모니터링하여 자동으로 풀의 크기를 조정할 수 있다. 특정 크기의 히트율이 높으면 그 풀의 크기를 늘리는 방식이다.

두 번째는 메모리 워터마크(watermark) 관리다. 전체 풀이 사용하는 메모리가 특정 한계를 초과하면, 가장 사용 빈도가 낮은 크기의 버퍼부터 해제하는 방식이다.

세 번째는 버퍼 재초기화다. 풀에 반환된 버퍼를 재사용할 때, 이전 데이터가 남아있을 수 있다. 보안이 중요한 경우, 버퍼를 풀에 반환하기 전에 메모리를 초기화해야 한다.

네 번째는 메트릭 기반의 자동 최적화다. 수집한 통계를 바탕으로, 프로덕션 환경에서 자동으로 STANDARD_SIZES를 조정할 수 있다. 게임의 특성에 따라 자주 사용되는 버퍼 크기가 다르기 때문이다.

다섯 번째는 계층적 풀 구조다. 버퍼의 할당과 해제가 NUMA(Non-Uniform Memory Access) 시스템에서 효율적이 되도록, 각 스레드마다 로컬 풀을 유지하고, 전역 풀과 동기화하는 방식을 적용할 수 있다.

실전에서는 이러한 개선 사항들을 게임 서버의 특성에 맞게 선택하여 적용하면, 더욱 효율적이고 안정적인 네트워크 애플리케이션을 만들 수 있다.
