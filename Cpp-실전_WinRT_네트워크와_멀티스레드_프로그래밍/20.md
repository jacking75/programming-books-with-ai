# Chapter 20. 세션 관리
온라인 게임 서버의 핵심은 연결된 클라이언트를 추적하고 관리하는 것이다. 플레이어가 서버에 접속하면 해당 플레이어에 대한 정보를 유지해야 하고, 플레이어가 어떤 상태(접속 중, 로비, 게임 진행, 접속 해제)에 있는지 알아야 하며, 타임아웃된 연결을 정리해야 한다. 이 모든 것을 담당하는 것이 세션 관리 시스템이다.

세션은 단순히 연결(connection)과는 다르다. 연결은 물리적인 TCP 소켓을 의미하지만, 세션은 클라이언트의 논리적 상태를 나타낸다. 예를 들어, 네트워크 끊김으로 인해 연결이 일시적으로 끊어져도, 세션이 유효하면 클라이언트가 재접속할 때 이전 상태를 복구할 수 있다. 이러한 복원력이 게임의 사용자 경험을 크게 개선한다.

---

## 20.1 세션의 개념과 필요성
게임 서버에서 세션이 필요한 이유를 구체적으로 이해해보자.

**이유 1: 플레이어 상태 관리**
플레이어가 서버에 접속하는 순간부터 접속 해제할 때까지 그의 모든 정보를 추적해야 한다. 예를 들어:
- 플레이어 ID와 계정 정보
- 현재 위치나 로비 상태
- 소유한 아이템 목록
- 파티나 길드 정보
- 현재 게임방의 ID

**이유 2: 타임아웃 처리**
네트워크는 항상 안정적이지 않다. 클라이언트가 갑자기 접속을 끊거나, 네트워크 지연으로 오래 응답이 없을 수 있다. 세션 시스템은 일정 시간 응답이 없는 플레이어의 세션을 자동으로 종료하여 서버 리소스를 절약한다.

**이유 3: 동시성 제어**
멀티스레드 환경에서 여러 네트워크 스레드가 동시에 다른 클라이언트의 세션에 접근할 수 있다. 세션 시스템은 이러한 동시 접근을 안전하게 제어한다.

**이유 4: 명확한 라이프사이클**
세션의 생성, 활성화, 검증, 종료의 각 단계를 명확하게 정의하면, 버그를 줄이고 유지보수가 쉬워진다.

구체적인 예를 들어보자. 한 플레이어가 채팅 서버에 로그인하면:
1. 새로운 세션이 생성되고 고유한 세션 ID가 할당된다.
2. 세션이 "활성" 상태로 설정되고, 마지막 활동 시간이 기록된다.
3. 플레이어가 메시지를 보내면 해당 세션의 마지막 활동 시간이 갱신된다.
4. 만약 5분 동안 활동이 없으면 세션이 자동으로 "타임아웃" 상태로 변경된다.
5. 세션을 정리하는 백그라운드 작업이 타임아웃된 세션을 삭제한다.

이러한 체계적인 관리 없이는 좀비 연결(zombie connection)이 쌓이고, 메모리 누수가 발생하며, 플레이어가 혼란스러운 상태에 빠질 수 있다.

## 20.2 세션 클래스 설계
기본적인 세션 클래스부터 만들어보자. 세션은 클라이언트를 식별하고 상태를 관리하는 핵심 객체이다.

```cpp
#include <string>
#include <chrono>
#include <cstdint>

enum class SessionState
{
    CREATED,        // 세션이 방금 생성됨
    CONNECTED,      // 클라이언트가 연결됨
    AUTHENTICATED,  // 사용자 인증 완료
    ACTIVE,         // 활발히 활동 중
    IDLE,           // 활동이 없음
    TIMEOUT,        // 타임아웃됨
    CLOSED          // 종료됨
};

class Session
{
private:
    std::string sessionId;          // 고유한 세션 식별자
    std::string userId;             // 로그인한 사용자 ID
    SessionState state;             // 현재 상태
    
    // 타임 스탬프 관리
    std::chrono::system_clock::time_point createdTime;
    std::chrono::system_clock::time_point lastActivityTime;
    
    // 연결 정보
    uint32_t clientIpHash;          // 클라이언트 IP 해시
    uint16_t clientPort;            // 클라이언트 포트
    
    // 추가 메타데이터
    int32_t messageCount = 0;       // 주고받은 메시지 수
    bool isEncrypted = false;       // 암호화 연결 여부

public:
    // 생성자
    explicit Session(const std::string& id)
        : sessionId(id), state(SessionState::CREATED),
          createdTime(std::chrono::system_clock::now()),
          lastActivityTime(createdTime), clientIpHash(0), clientPort(0)
    {
    }

    // 세션 식별자
    const std::string& GetSessionId() const { return sessionId; }

    // 사용자 정보 관리
    void SetUserId(const std::string& id) { userId = id; }
    const std::string& GetUserId() const { return userId; }

    // 상태 관리
    void SetState(SessionState newState) { state = newState; }
    SessionState GetState() const { return state; }

    // 활동 시간 갱신 (주기적으로 호출됨)
    void UpdateActivity()
    {
        lastActivityTime = std::chrono::system_clock::now();
        messageCount++;
    }

    // 마지막 활동으로부터 경과 시간 (초)
    int64_t GetIdleSeconds() const
    {
        auto now = std::chrono::system_clock::now();
        auto elapsed = std::chrono::duration_cast<std::chrono::seconds>(
            now - lastActivityTime
        );
        return elapsed.count();
    }

    // 세션 생성 이후 경과 시간 (초)
    int64_t GetAgeSeconds() const
    {
        auto now = std::chrono::system_clock::now();
        auto elapsed = std::chrono::duration_cast<std::chrono::seconds>(
            now - createdTime
        );
        return elapsed.count();
    }

    // 클라이언트 정보
    void SetClientInfo(uint32_t ipHash, uint16_t port)
    {
        clientIpHash = ipHash;
        clientPort = port;
    }

    uint32_t GetClientIpHash() const { return clientIpHash; }
    uint16_t GetClientPort() const { return clientPort; }

    // 연결 정보
    void SetEncrypted(bool encrypted) { isEncrypted = encrypted; }
    bool IsEncrypted() const { return isEncrypted; }

    // 통계
    int32_t GetMessageCount() const { return messageCount; }
};
```

이 세션 클래스의 설계 원칙은 다음과 같다. 첫째, **불변성(Immutability)**: 세션 ID는 생성 후 절대 변경되지 않으므로 const로 반환한다. 둘째, **시간 추적**: 생성 시간과 마지막 활동 시간을 기록하여 타임아웃 판정에 사용한다. 셋째, **단일 책임**: 세션은 자신의 상태만 관리하고, 비즈니스 로직(예: 로그인 처리)은 포함하지 않는다. 넷째, **메타데이터**: IP 주소 해시, 암호화 여부 등 연결 정보를 저장한다.

## 20.3 세션 식별자 관리
세션이 고유하려면 고유한 식별자가 필요하다. UUID(Universally Unique Identifier)는 전 세계 어디서나 충돌할 가능성이 거의 없는 128비트 식별자를 생성한다.

```cpp
#include <random>
#include <sstream>
#include <iomanip>
#include <array>

class SessionIdGenerator
{
private:
    std::random_device rd;
    std::mt19937_64 gen;
    std::uniform_int_distribution<uint64_t> dis;

public:
    SessionIdGenerator() : gen(rd()), dis(0, UINT64_MAX)
    {
    }

    // UUID v4 생성 (간단한 버전)
    std::string GenerateSessionId()
    {
        // 두 개의 64비트 난수를 생성
        uint64_t part1 = dis(gen);
        uint64_t part2 = dis(gen);

        // 16진법 문자열로 변환
        std::ostringstream oss;
        oss << std::hex << std::setfill('0')
            << std::setw(8) << (part1 >> 32)
            << "-"
            << std::setw(4) << ((part1 >> 16) & 0xFFFF)
            << "-"
            << std::setw(4) << (part1 & 0xFFFF)
            << "-"
            << std::setw(4) << (part2 >> 48)
            << "-"
            << std::setw(12) << (part2 & 0xFFFFFFFFFFFF);

        return oss.str();
    }

    // 더 간단한 버전: 타임스탬프 + 난수 조합
    std::string GenerateSimpleSessionId()
    {
        auto now = std::chrono::system_clock::now();
        auto timestamp = now.time_since_epoch().count();
        uint32_t random = dis(gen) & 0xFFFFFFFF;

        std::ostringstream oss;
        oss << std::hex << std::setfill('0')
            << std::setw(16) << timestamp
            << "-"
            << std::setw(8) << random;

        return oss.str();
    }
};
```

UUID 생성 방식은 몇 가지 선택지가 있다. **UUID v4 방식**(완전 난수)은 충돌 확률이 매우 낮지만 계산이 복잡하다. **타임스탐프 기반 방식**은 생성 순서를 예측할 수 있지만 더 빠르고, 디버깅 시 시간 순서를 알 수 있다는 장점이 있다. 게임 서버에서는 보안보다 성능이 중요한 경우가 많으므로, 타임스탬프 기반 방식이 실용적이다.

## 20.4 세션 상태 관리
세션의 생명주기를 **상태 머신**으로 관리하면 올바른 상태 전이를 보장할 수 있다.

```cpp
#include <stdexcept>

class SessionStateMachine
{
public:
    // 상태 전이 규칙 확인
    static bool CanTransition(SessionState from, SessionState to)
    {
        // 상태 전이 규칙 테이블
        switch (from)
        {
        case SessionState::CREATED:
            return to == SessionState::CONNECTED;

        case SessionState::CONNECTED:
            return to == SessionState::AUTHENTICATED || 
                   to == SessionState::CLOSED;

        case SessionState::AUTHENTICATED:
            return to == SessionState::ACTIVE ||
                   to == SessionState::CLOSED;

        case SessionState::ACTIVE:
            return to == SessionState::IDLE ||
                   to == SessionState::TIMEOUT ||
                   to == SessionState::CLOSED;

        case SessionState::IDLE:
            return to == SessionState::ACTIVE ||
                   to == SessionState::TIMEOUT ||
                   to == SessionState::CLOSED;

        case SessionState::TIMEOUT:
            return to == SessionState::CLOSED;

        case SessionState::CLOSED:
            return false;  // 닫힌 세션은 어디로도 갈 수 없음

        default:
            return false;
        }
    }

    // 안전한 상태 전이
    static void TransitionState(Session& session, SessionState newState)
    {
        SessionState currentState = session.GetState();

        if (!CanTransition(currentState, newState))
        {
            throw std::runtime_error(
                "Invalid state transition from " +
                StateToString(currentState) +
                " to " +
                StateToString(newState)
            );
        }

        session.SetState(newState);
    }

private:
    static std::string StateToString(SessionState state)
    {
        switch (state)
        {
        case SessionState::CREATED:        return "CREATED";
        case SessionState::CONNECTED:      return "CONNECTED";
        case SessionState::AUTHENTICATED:  return "AUTHENTICATED";
        case SessionState::ACTIVE:         return "ACTIVE";
        case SessionState::IDLE:           return "IDLE";
        case SessionState::TIMEOUT:        return "TIMEOUT";
        case SessionState::CLOSED:         return "CLOSED";
        default:                           return "UNKNOWN";
        }
    }
};
```

이 상태 머신은 논리적으로 불가능한 전이를 방지한다. 예를 들어, CLOSED 상태인 세션을 다시 ACTIVE로 돌릴 수 없다. 이렇게 하면 프로그래머의 실수로 인한 세션 관리 버그를 줄일 수 있다.

## 20.5 세션 컨테이너
많은 수의 세션을 효율적으로 관리하려면 **스레드 안전한 맵**이 필요하다. 멀티스레드 환경에서 여러 네트워크 스레드가 동시에 세션을 추가, 제거, 검색할 수 있기 때문이다.

```cpp
#include <map>
#include <memory>
#include <mutex>
#include <vector>

class SessionManager
{
private:
    std::map<std::string, std::shared_ptr<Session>> sessions;
    std::map<std::string, std::shared_ptr<Session>> sessionsByUserId;
    std::mutex sessionMutex;
    
    static const int64_t TIMEOUT_SECONDS = 300;  // 5분

public:
    // 새 세션 생성
    std::shared_ptr<Session> CreateSession(const std::string& sessionId)
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        auto session = std::make_shared<Session>(sessionId);
        sessions[sessionId] = session;
        return session;
    }

    // 세션 검색 (세션 ID로)
    std::shared_ptr<Session> GetSession(const std::string& sessionId)
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        auto it = sessions.find(sessionId);
        if (it != sessions.end())
            return it->second;
        return nullptr;
    }

    // 세션 검색 (사용자 ID로)
    std::shared_ptr<Session> GetSessionByUserId(const std::string& userId)
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        auto it = sessionsByUserId.find(userId);
        if (it != sessionsByUserId.end())
            return it->second;
        return nullptr;
    }

    // 사용자 로그인 처리
    void AuthenticateSession(std::shared_ptr<Session> session, 
                             const std::string& userId)
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        session->SetUserId(userId);
        SessionStateMachine::TransitionState(session, SessionState::AUTHENTICATED);
        sessionsByUserId[userId] = session;
    }

    // 세션 종료
    bool CloseSession(const std::string& sessionId)
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        auto it = sessions.find(sessionId);
        if (it == sessions.end())
            return false;

        auto session = it->second;
        
        // 사용자 ID 맵에서 제거
        if (!session->GetUserId().empty())
            sessionsByUserId.erase(session->GetUserId());
        
        // 세션 맵에서 제거
        sessions.erase(it);
        return true;
    }

    // 타임아웃된 세션 정리 (백그라운드 작업에서 호출)
    size_t CleanupTimeoutSessions()
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        size_t cleanedCount = 0;
        std::vector<std::string> toRemove;

        for (auto& entry : sessions)
        {
            auto session = entry.second;
            
            // 활성 상태가 아닌 세션만 타임아웃 체크
            if (session->GetState() != SessionState::ACTIVE &&
                session->GetState() != SessionState::IDLE)
                continue;

            // 타임아웃 여부 판정
            if (session->GetIdleSeconds() > TIMEOUT_SECONDS)
            {
                toRemove.push_back(entry.first);
                cleanedCount++;
            }
        }

        // 타임아웃된 세션 제거
        for (const auto& sessionId : toRemove)
            CloseSession(sessionId);

        return cleanedCount;
    }

    // 현재 활성 세션 수
    size_t GetActiveSessionCount() const
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        size_t count = 0;
        for (const auto& entry : sessions)
        {
            auto state = entry.second->GetState();
            if (state != SessionState::CLOSED &&
                state != SessionState::TIMEOUT)
                count++;
        }
        return count;
    }

    // 모든 세션 정보 (통계용)
    std::vector<std::pair<std::string, int64_t>> GetAllSessionStats() const
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        std::vector<std::pair<std::string, int64_t>> stats;
        for (const auto& entry : sessions)
        {
            stats.emplace_back(
                entry.first,
                entry.second->GetIdleSeconds()
            );
        }
        return stats;
    }
};
```

이 세션 컨테이너의 핵심 설계 요소는 다음과 같다.

**이중 인덱싱**: 세션을 세션 ID로도, 사용자 ID로도 검색할 수 있도록 두 개의 맵을 유지한다. 이렇게 하면 사용자가 재접속할 때 기존 세션을 빠르게 찾을 수 있다.

**뮤텍스 보호**: 모든 맵 접근을 뮤텍스로 보호하여 멀티스레드 환경에서의 경쟁 조건을 방지한다.

**스마트 포인터**: `std::shared_ptr`를 사용하여 메모리 관리를 자동화한다. 세션이 더 이상 필요 없으면 자동으로 메모리가 해제된다.

**타임아웃 관리**: `CleanupTimeoutSessions()` 메서드를 별도의 백그라운드 스레드에서 주기적으로 호출하여 좀비 세션을 정리한다.

## 20.6 종합 예제: 완전한 세션 매니저
이제 생성, 검증, 타임아웃, 정리를 모두 포함한 프로덕션급 세션 관리 시스템을 만들어보자.

**코드의 목적**: 멀티스레드 게임 서버에서 수천 개의 플레이어 세션을 안전하고 효율적으로 관리하는 완전한 시스템을 구현한다. 이 시스템은 플레이어의 접속, 인증, 활동, 타임아웃, 접속 해제의 전체 생명주기를 관리한다.

**설계 결정 이유**:
1. **이중 인덱싱**: 세션 ID와 사용자 ID 두 가지 방식으로 검색을 지원하여 다양한 상황에 대응한다.
2. **통합 상태 머신**: 불가능한 상태 전이를 방지하고 버그를 줄인다.
3. **백그라운드 정리 스레드**: 타임아웃된 세션을 자동으로 정리하여 메모리 누수를 방지한다.
4. **통계 수집**: 디버깅과 모니터링을 위해 세션 정보를 수집한다.
5. **예외 기반 에러 처리**: 명확한 에러 메시지로 디버깅을 쉽게 한다.

```cpp
#include <map>
#include <memory>
#include <mutex>
#include <thread>
#include <chrono>
#include <vector>
#include <queue>
#include <functional>
#include <random>
#include <sstream>
#include <iomanip>
#include <stdexcept>

// ============ 세션 상태 정의 ============
enum class SessionState
{
    CREATED,
    CONNECTED,
    AUTHENTICATED,
    ACTIVE,
    IDLE,
    TIMEOUT,
    CLOSED
};

// ============ 세션 클래스 ============
class Session
{
private:
    std::string sessionId;
    std::string userId;
    SessionState state;
    std::chrono::system_clock::time_point createdTime;
    std::chrono::system_clock::time_point lastActivityTime;
    uint32_t clientIpHash;
    uint16_t clientPort;
    int32_t messageCount;

public:
    explicit Session(const std::string& id)
        : sessionId(id), state(SessionState::CREATED),
          createdTime(std::chrono::system_clock::now()),
          lastActivityTime(createdTime), 
          clientIpHash(0), clientPort(0), messageCount(0)
    {
    }

    const std::string& GetSessionId() const { return sessionId; }
    void SetUserId(const std::string& id) { userId = id; }
    const std::string& GetUserId() const { return userId; }
    void SetState(SessionState newState) { state = newState; }
    SessionState GetState() const { return state; }
    
    void UpdateActivity()
    {
        lastActivityTime = std::chrono::system_clock::now();
        messageCount++;
    }

    int64_t GetIdleSeconds() const
    {
        auto now = std::chrono::system_clock::now();
        auto elapsed = std::chrono::duration_cast<std::chrono::seconds>(
            now - lastActivityTime
        );
        return elapsed.count();
    }

    int64_t GetAgeSeconds() const
    {
        auto now = std::chrono::system_clock::now();
        auto elapsed = std::chrono::duration_cast<std::chrono::seconds>(
            now - createdTime
        );
        return elapsed.count();
    }

    void SetClientInfo(uint32_t ipHash, uint16_t port)
    {
        clientIpHash = ipHash;
        clientPort = port;
    }

    uint32_t GetClientIpHash() const { return clientIpHash; }
    uint16_t GetClientPort() const { return clientPort; }
    int32_t GetMessageCount() const { return messageCount; }
};

// ============ 세션 ID 생성기 ============
class SessionIdGenerator
{
private:
    std::random_device rd;
    std::mt19937_64 gen;
    std::uniform_int_distribution<uint64_t> dis;

public:
    SessionIdGenerator() : gen(rd()), dis(0, UINT64_MAX) {}

    std::string Generate()
    {
        auto now = std::chrono::system_clock::now();
        auto timestamp = now.time_since_epoch().count();
        uint32_t random = dis(gen) & 0xFFFFFFFF;

        std::ostringstream oss;
        oss << std::hex << std::setfill('0')
            << std::setw(16) << timestamp
            << "-"
            << std::setw(8) << random;

        return oss.str();
    }
};

// ============ 세션 상태 머신 ============
class SessionStateMachine
{
public:
    static bool CanTransition(SessionState from, SessionState to)
    {
        switch (from)
        {
        case SessionState::CREATED:
            return to == SessionState::CONNECTED;
        case SessionState::CONNECTED:
            return to == SessionState::AUTHENTICATED || 
                   to == SessionState::CLOSED;
        case SessionState::AUTHENTICATED:
            return to == SessionState::ACTIVE || 
                   to == SessionState::CLOSED;
        case SessionState::ACTIVE:
            return to == SessionState::IDLE || 
                   to == SessionState::TIMEOUT || 
                   to == SessionState::CLOSED;
        case SessionState::IDLE:
            return to == SessionState::ACTIVE || 
                   to == SessionState::TIMEOUT || 
                   to == SessionState::CLOSED;
        case SessionState::TIMEOUT:
            return to == SessionState::CLOSED;
        case SessionState::CLOSED:
            return false;
        default:
            return false;
        }
    }

    static void Transition(Session& session, SessionState newState)
    {
        if (!CanTransition(session.GetState(), newState))
            throw std::runtime_error("Invalid state transition");
        session.SetState(newState);
    }
};

// ============ 메인 세션 매니저 ============
class ProductionSessionManager
{
private:
    std::map<std::string, std::shared_ptr<Session>> sessions;
    std::map<std::string, std::shared_ptr<Session>> sessionsByUserId;
    mutable std::mutex sessionMutex;
    
    SessionIdGenerator idGenerator;
    std::thread cleanupThread;
    volatile bool shouldStop = false;
    
    static const int64_t TIMEOUT_SECONDS = 300;
    static const int64_t CLEANUP_INTERVAL_SECONDS = 30;

public:
    ProductionSessionManager()
    {
        // 백그라운드 정리 스레드 시작
        cleanupThread = std::thread([this]() { CleanupWorker(); });
    }

    ~ProductionSessionManager()
    {
        shouldStop = true;
        if (cleanupThread.joinable())
            cleanupThread.join();
    }

    // ===== 세션 생성 및 검색 =====

    std::shared_ptr<Session> CreateSession()
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        std::string sessionId = idGenerator.Generate();
        auto session = std::make_shared<Session>(sessionId);
        sessions[sessionId] = session;
        
        return session;
    }

    std::shared_ptr<Session> GetSession(const std::string& sessionId)
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        auto it = sessions.find(sessionId);
        if (it != sessions.end())
            return it->second;
        return nullptr;
    }

    std::shared_ptr<Session> GetSessionByUserId(const std::string& userId)
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        auto it = sessionsByUserId.find(userId);
        if (it != sessionsByUserId.end())
            return it->second;
        return nullptr;
    }

    // ===== 세션 상태 전이 =====

    void ConnectSession(std::shared_ptr<Session> session,
                       uint32_t clientIpHash, uint16_t clientPort)
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        session->SetClientInfo(clientIpHash, clientPort);
        SessionStateMachine::Transition(*session, SessionState::CONNECTED);
    }

    void AuthenticateSession(std::shared_ptr<Session> session,
                            const std::string& userId)
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        // 같은 사용자의 기존 세션이 있으면 종료
        auto existingSession = sessionsByUserId[userId];
        if (existingSession)
        {
            try
            {
                SessionStateMachine::Transition(*existingSession, 
                                               SessionState::CLOSED);
                sessions.erase(existingSession->GetSessionId());
            }
            catch (...) { /* 무시 */ }
        }

        session->SetUserId(userId);
        SessionStateMachine::Transition(*session, SessionState::AUTHENTICATED);
        SessionStateMachine::Transition(*session, SessionState::ACTIVE);
        sessionsByUserId[userId] = session;
    }

    void UpdateSessionActivity(std::shared_ptr<Session> session)
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        if (session->GetState() == SessionState::TIMEOUT)
            return;  // 타임아웃된 세션은 업데이트하지 않음

        if (session->GetState() == SessionState::IDLE)
            SessionStateMachine::Transition(*session, SessionState::ACTIVE);

        session->UpdateActivity();
    }

    bool CloseSession(const std::string& sessionId)
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        auto it = sessions.find(sessionId);
        if (it == sessions.end())
            return false;

        auto session = it->second;

        if (!session->GetUserId().empty())
            sessionsByUserId.erase(session->GetUserId());

        try
        {
            SessionStateMachine::Transition(*session, SessionState::CLOSED);
        }
        catch (...) { /* 이미 닫혀있을 수 있음 */ }

        sessions.erase(it);
        return true;
    }

    // ===== 세션 검증 =====

    bool ValidateSession(const std::string& sessionId)
    {
        auto session = GetSession(sessionId);
        if (!session)
            return false;

        auto state = session->GetState();
        return state == SessionState::ACTIVE || 
               state == SessionState::IDLE;
    }

    // ===== 백그라운드 작업 =====

private:
    void CleanupWorker()
    {
        while (!shouldStop)
        {
            std::this_thread::sleep_for(
                std::chrono::seconds(CLEANUP_INTERVAL_SECONDS)
            );

            CleanupTimeoutSessions();
        }
    }

    void CleanupTimeoutSessions()
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        std::vector<std::string> toRemove;

        for (auto& entry : sessions)
        {
            auto session = entry.second;
            auto state = session->GetState();

            if (state != SessionState::ACTIVE &&
                state != SessionState::IDLE &&
                state != SessionState::CONNECTED)
                continue;

            if (session->GetIdleSeconds() > TIMEOUT_SECONDS)
            {
                toRemove.push_back(entry.first);
                
                if (state != SessionState::TIMEOUT)
                {
                    try
                    {
                        SessionStateMachine::Transition(*session, 
                                                       SessionState::TIMEOUT);
                    }
                    catch (...) { /* 무시 */ }
                }
            }
        }

        for (const auto& sessionId : toRemove)
        {
            auto it = sessions.find(sessionId);
            if (it != sessions.end())
            {
                auto session = it->second;
                if (!session->GetUserId().empty())
                    sessionsByUserId.erase(session->GetUserId());
                sessions.erase(it);
            }
        }
    }

public:
    // ===== 모니터링 및 통계 =====

    size_t GetActiveSessionCount() const
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        size_t count = 0;
        for (const auto& entry : sessions)
        {
            auto state = entry.second->GetState();
            if (state == SessionState::ACTIVE || 
                state == SessionState::IDLE)
                count++;
        }
        return count;
    }

    size_t GetTotalSessionCount() const
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        return sessions.size();
    }

    struct SessionStats
    {
        std::string sessionId;
        std::string userId;
        int64_t idleSeconds;
        int64_t ageSeconds;
        int32_t messageCount;
        std::string state;
    };

    std::vector<SessionStats> GetAllSessionStats() const
    {
        std::lock_guard<std::mutex> lock(sessionMutex);
        
        std::vector<SessionStats> stats;
        for (const auto& entry : sessions)
        {
            auto session = entry.second;
            stats.push_back({
                session->GetSessionId(),
                session->GetUserId(),
                session->GetIdleSeconds(),
                session->GetAgeSeconds(),
                session->GetMessageCount(),
                StateToString(session->GetState())
            });
        }
        return stats;
    }

private:
    std::string StateToString(SessionState state) const
    {
        switch (state)
        {
        case SessionState::CREATED:        return "CREATED";
        case SessionState::CONNECTED:      return "CONNECTED";
        case SessionState::AUTHENTICATED:  return "AUTHENTICATED";
        case SessionState::ACTIVE:         return "ACTIVE";
        case SessionState::IDLE:           return "IDLE";
        case SessionState::TIMEOUT:        return "TIMEOUT";
        case SessionState::CLOSED:         return "CLOSED";
        default:                           return "UNKNOWN";
        }
    }
};
```

**주요 부분 상세 해설**:

1. **CreateSession() 메서드**: 새로운 세션을 생성할 때 고유한 ID를 생성하고, 세션을 두 개의 맵에 저장한다. 생성된 세션은 CREATED 상태로 시작한다.

2. **AuthenticateSession() 메서드**: 사용자가 로그인할 때 호출된다. 중요한 점은 같은 사용자의 이전 세션이 있으면 먼저 종료한다는 것이다. 이렇게 하면 한 사용자가 여러 세션을 열 수 없도록 방지한다.

3. **UpdateSessionActivity() 메서드**: 플레이어가 메시지를 보낼 때마다 호출되어 마지막 활동 시간을 갱신한다. IDLE 상태면 ACTIVE로 복구한다.

4. **CleanupWorker() 백그라운드 스레드**: 30초마다 타임아웃된 세션을 확인하고 제거한다. 이 스레드는 생성자에서 시작되고 소멸자에서 종료된다.

5. **ValidateSession() 메서드**: 요청에 포함된 세션 ID가 유효한지 확인한다. 게임 로직 실행 전에 항상 호출해야 한다.

6. **통계 수집**: GetAllSessionStats()는 모든 세션 정보를 수집하여 로깅이나 모니터링에 사용할 수 있다.

**개선 방안과 확장 가능성**:

첫째, **세션 암호화 토큰**: 현재는 세션 ID만 검증하는데, 클라이언트가 수정하지 못하도록 세션 생성 시 암호화된 토큰을 발급할 수 있다. 클라이언트는 이 토큰을 모든 요청에 포함시켜야 한다.

둘째, **세션 메타데이터 저장**: 게임별로 필요한 추가 정보(플레이어의 현재 위치, 파티 ID, 길드 ID 등)를 세션에 저장할 수 있다. `std::map<std::string, std::string>` 형태의 속성 맵을 추가하면 된다.

셋째, **이벤트 콜백 시스템**: 세션 생성, 인증, 타임아웃, 종료 시점에 콜백 함수를 호출하여 상위 계층이 이에 대응할 수 있게 한다. 예를 들어, 세션이 타임아웃되면 플레이어의 진행 중인 게임을 자동으로 종료할 수 있다.

네 번째, **재접속 지원**: 타임아웃되더라도 일정 시간 내에 재접속하면 같은 세션을 복구하는 기능을 추가할 수 있다. 이렇게 하면 네트워크 끊김으로 인한 사용자 불편을 줄일 수 있다.

다섯 번째, **데이터베이스 동기화**: 세션 정보를 데이터베이스에 저장하여 서버 재시작 시에도 세션 상태를 복구할 수 있다.

여섯 번째, **분산 세션 관리**: 여러 서버가 공동으로 세션을 관리하는 경우, Redis 같은 메모리 캐시를 사용하여 세션을 중앙집중식으로 관리할 수 있다.

이 완전한 세션 매니저는 실제 게임 서버에서 즉시 사용 가능하며, 나머지 서버 로직(채팅, 게임, 로비 등)의 기초가 된다.
