# Chapter 9. TCP 소켓 - StreamSocket
TCP는 온라인 게임 서버 개발에서 가장 기본적이면서도 중요한 프로토콜이다. StreamSocket은 WinRT에서 TCP 통신을 담당하는 핵심 클래스로, 신뢰성 있는 양방향 통신을 제공한다. 이 장에서는 StreamSocket의 모든 측면을 학습하고, 실제 게임 서버에서 사용할 수 있는 수준의 HTTP 클라이언트를 구현해본다.

---

## 9.1 StreamSocket API 상세 설명
StreamSocket은 TCP 기반의 연결 지향 통신을 제공하는 WinRT 클래스다. 전통적인 Winsock의 socket과 connect, send, recv 함수를 모던하고 비동기적인 방식으로 대체한다.

### 생성자와 초기화
StreamSocket의 생성자는 매우 간단하다. 아무런 매개변수 없이 호출되며, 생성 후에 서버에 연결하거나 기존 연결에서 받은 소켓으로 초기화할 수 있다.

```cpp
#include <winrt/Windows.Networking.Sockets.h>
using namespace winrt::Windows::Networking::Sockets;

// 새로운 소켓 생성
StreamSocket socket;

// 또는 스마트 포인터 사용
auto socket = std::make_unique<StreamSocket>();
```

StreamSocket은 기본 생성자만 지원한다. 복사 생성자와 복사 대입 연산자는 삭제되어 있으므로, 이동 의미론을 사용하거나 스마트 포인터로 관리해야 한다.

### 주요 메서드
**ConnectAsync()** 메서드는 원격 서버에 연결하는 비동기 작업을 수행한다. 이 메서드는 **IAsyncAction** 코루틴을 반환하므로 co_await로 기다릴 수 있다.

```cpp
HostName remoteHost(L"game.example.com");
hstring serviceName(L"8888");

auto connectTask = socket.ConnectAsync(remoteHost, serviceName);
co_await connectTask;
// 여기서 연결이 완료됨
```

**GetOutputStreamAsync()** 메서드는 데이터를 보내기 위한 출력 스트림을 얻는다. 반환된 스트림에 DataWriter를 래핑하여 사용한다.

```cpp
auto outputStream = co_await socket.GetOutputStreamAsync();
DataWriter writer(outputStream);
```

**GetInputStreamAsync()** 메서드는 데이터를 받기 위한 입력 스트림을 얻는다. 반환된 스트림에 DataReader를 래핑하여 사용한다.

```cpp
auto inputStream = co_await socket.GetInputStreamAsync();
DataReader reader(inputStream);
```

**Close()** 메서드는 연결을 종료한다. 이 메서드는 동기적이며 즉시 소켓을 닫는다. 작업이 완료될 때까지 기다리지 않는다.

```cpp
socket.Close();
```

### 주요 속성

**Control** 속성은 **StreamSocketControl** 객체를 반환하여 소켓의 동작을 제어할 수 있게 한다. KeepAlive 활성화, Nagle 알고리즘 비활성화, 버퍼 크기 조정 등이 가능하다.

```cpp
StreamSocketControl control = socket.Control();
control.KeepAlive(true);
control.NoDelay(true);
```

**Information** 속성은 **StreamSocketInformation** 객체를 반환하여 연결의 현재 상태를 조회할 수 있게 한다. 로컬 주소, 로컬 포트, 원격 주소, 원격 포트 등을 확인할 수 있다.

```cpp
StreamSocketInformation info = socket.Information();
HostName localAddr = info.LocalAddress();
hstring localPort = info.LocalPort();
HostName remoteAddr = info.RemoteAddress();
hstring remotePort = info.RemotePort();
```

## 9.2 서버에 연결하기
StreamSocket을 사용하여 원격 서버에 연결하는 과정은 다음 단계로 진행된다. 먼저 대상 서버의 호스트명과 포트를 지정하고, ConnectAsync()를 호출하여 비동기적으로 연결을 시도한다. 연결이 성공하면 데이터를 송수신할 준비가 된다.

### ConnectAsync() 단계별 설명

```cpp
#include <winrt/Windows.Networking.h>
#include <winrt/Windows.Networking.Sockets.h>
#include <iostream>

using namespace winrt;
using namespace winrt::Windows::Networking;
using namespace winrt::Windows::Networking::Sockets;

IAsyncAction ConnectToServerExample()
{
    StreamSocket socket;

    try
    {
        // 1단계: 호스트명과 포트 설정
        HostName remoteHost(L"example.com");
        hstring remotePort(L"80");

        // 2단계: 서버에 연결 시도
        co_await socket.ConnectAsync(remoteHost, remotePort);
        
        std::wcout << L"서버 연결 성공" << std::endl;

        // 3단계: 연결 정보 확인
        StreamSocketInformation info = socket.Information();
        std::wcout << L"로컬 주소: " 
                   << info.LocalAddress().DisplayName().c_str() 
                   << std::endl;
        std::wcout << L"로컬 포트: " 
                   << info.LocalPort().c_str() << std::endl;
        std::wcout << L"원격 주소: " 
                   << info.RemoteAddress().DisplayName().c_str() 
                   << std::endl;
        std::wcout << L"원격 포트: " 
                   << info.RemotePort().c_str() << std::endl;

        // 4단계: 연결 종료
        socket.Close();
    }
    catch (hresult_error const& ex)
    {
        // 연결 실패 처리
        std::wcerr << L"연결 실패: " 
                   << ex.message().c_str() << std::endl;
    }
}
```

**1단계: 호스트명 지정**: HostName 객체를 생성할 때 도메인 이름이나 IP 주소를 전달한다. "example.com"은 DNS를 통해 해석되고, IP 주소는 바로 사용된다.

**2단계: ConnectAsync() 호출**: 이 비동기 메서드는 즉시 반환되고, 실제 연결은 백그라운드에서 진행된다. co_await를 사용하면 연결이 완료될 때까지 대기한다. 연결에 실패하면 예외가 발생한다.

**3단계: 연결 정보 확인**: 연결 후 Information 속성을 통해 실제 사용 중인 주소와 포트를 확인할 수 있다. 이는 자동으로 할당된 로컬 포트를 알아야 할 때 유용하다.

**4단계: 정리**: 작업이 끝나면 Close()를 호출하여 연결을 종료한다.

### 타임아웃 처리
실제 네트워크 환경에서는 연결이 평소보다 느릴 수 있다. WinRT는 기본적으로 장시간 타임아웃을 사용하지만, 필요시 타이머를 결합하여 타임아웃을 구현할 수 있다:

```cpp
#include <winrt/Windows.Foundation.h>

IAsyncAction ConnectWithTimeout()
{
    StreamSocket socket;
    HostName remoteHost(L"example.com");
    hstring remotePort(L"8888");

    try
    {
        // 5초 타임아웃으로 설정
        auto connectOperation = socket.ConnectAsync(
            remoteHost, remotePort);
        
        auto delayOperation = 
            std::chrono::milliseconds(5000);
        
        // 실전에서는 더 복잡한 타임아웃 처리가 필요
        co_await connectOperation;
    }
    catch (hresult_error const& ex)
    {
        std::wcerr << L"연결 타임아웃 또는 실패" << std::endl;
    }
}
```

## 9.3 DataWriter로 데이터 보내기
StreamSocket으로 데이터를 보내려면 **DataWriter** 클래스를 사용한다. DataWriter는 다양한 데이터 타입을 자동으로 올바른 바이트 형식으로 변환하고, 버퍼에 작성한 후 네트워크로 전송한다.

### 버퍼링과 StoreAsync()

```cpp
#include <winrt/Windows.Storage.Streams.h>
using namespace winrt::Windows::Storage::Streams;

IAsyncAction SendDataExample(StreamSocket socket)
{
    try
    {
        // 1단계: 출력 스트림 가져오기
        IOutputStream outputStream = 
            co_await socket.GetOutputStreamAsync();
        
        DataWriter writer(outputStream);

        // 2단계: 데이터 작성 (아직 전송 안됨)
        writer.WriteInt32(12345);           // 32비트 정수
        writer.WriteString(L"Hello Server"); // 문자열
        writer.WriteByte(0xFF);             // 바이트
        writer.WriteBytes(
            std::vector<uint8_t>{1, 2, 3, 4, 5});

        // 3단계: 버퍼 상태 확인
        uint32_t unstoredLength = 
            writer.UnstoredBufferLength();
        std::wcout << L"버퍼에 있는 데이터 크기: " 
                   << unstoredLength << L" 바이트" << std::endl;

        // 4단계: 버퍼의 데이터를 실제로 전송
        uint32_t bytesSent = 
            co_await writer.StoreAsync();
        
        std::wcout << L"전송된 데이터: " 
                   << bytesSent << L" 바이트" << std::endl;

        // 5단계: 플러시하여 모든 데이터 보내기
        co_await writer.FlushAsync();

        // 6단계: DataWriter 정리
        writer.DetachStream();
    }
    catch (hresult_error const& ex)
    {
        std::wcerr << L"데이터 전송 실패: " 
                   << ex.message().c_str() << std::endl;
    }
}
```

**1단계: 출력 스트림 가져오기**: GetOutputStreamAsync()는 비동기적으로 스트림을 반환한다. 이 스트림은 네트워크 송신 버퍼로 연결된다.

**2단계: DataWriter에 데이터 작성**: DataWriter는 WriteInt32, WriteString, WriteByte 등의 타입 특화 메서드를 제공한다. 이 메서드들은 데이터를 자동으로 올바른 바이트 순서(big-endian)로 변환하여 내부 버퍼에 작성한다. 중요한 점은 이 단계에서 아직 네트워크로 전송되지 않는다는 것이다.

**3단계: 버퍼 상태 확인**: UnstoredBufferLength()는 아직 전송되지 않은 데이터의 바이트 수를 반환한다. 이는 디버깅이나 성능 모니터링에 유용하다.

**4단계: StoreAsync() 호출**: 이 메서드는 버퍼의 모든 데이터를 스트림에 기록한다. 반환값은 실제로 전송된 바이트 수다. 네트워크 지연으로 인해 요청한 것보다 적은 바이트가 전송될 수 있다.

**5단계: FlushAsync()**: 이 메서드는 모든 버퍼링된 데이터를 강제로 네트워크로 전송한다. Nagle 알고리즘이 활성화되어 있으면 작은 패킷들이 결합되기를 기다리지만, FlushAsync()는 즉시 전송을 강제한다.

**6단계: 정리**: DetachStream()을 호출하면 DataWriter가 스트림의 소유권을 포기한다. 이후 스트림을 다른 목적으로 사용할 수 있다.

### 프로토콜 메시지 작성 예제
게임에서 자주 사용하는 패턴은 메시지 헤더와 바디를 함께 전송하는 것이다:

```cpp
IAsyncAction SendGameMessage(StreamSocket socket)
{
    IOutputStream outputStream = 
        co_await socket.GetOutputStreamAsync();
    DataWriter writer(outputStream);

    // 메시지 타입과 데이터
    uint16_t messageType = 1001;  // 플레이어 이동 메시지
    float positionX = 100.5f;
    float positionY = 200.3f;

    // 헤더 작성: 타입(2바이트) + 길이(2바이트)
    writer.WriteUInt16(messageType);
    writer.WriteUInt16(12);  // 데이터 길이

    // 바디 작성: 좌표 정보
    writer.WriteBytes(std::vector<uint8_t>(
        reinterpret_cast<uint8_t*>(&positionX),
        reinterpret_cast<uint8_t*>(&positionX) + 
        sizeof(float)));
    
    co_await writer.StoreAsync();
}
```

## 9.4 DataReader로 데이터 받기
DataReader는 DataWriter의 역할을 한다. 네트워크로부터 받은 바이트들을 타입별로 읽어서 원래의 데이터로 복원한다.

### LoadAsync()와 바이트 읽기

```cpp
#include <winrt/Windows.Storage.Streams.h>
using namespace winrt::Windows::Storage::Streams;

IAsyncAction ReceiveDataExample(StreamSocket socket)
{
    try
    {
        // 1단계: 입력 스트림 가져오기
        IInputStream inputStream = 
            co_await socket.GetInputStreamAsync();
        
        DataReader reader(inputStream);

        // 2단계: 최소 읽을 바이트 크기 설정
        reader.InputStreamOptions(
            InputStreamOptions::Partial);

        // 3단계: 네트워크에서 데이터 읽기
        // 최대 1024 바이트를 읽으려고 시도
        uint32_t bytesLoaded = 
            co_await reader.LoadAsync(1024);
        
        std::wcout << L"수신한 데이터: " 
                   << bytesLoaded << L" 바이트" << std::endl;

        // 4단계: 로드된 데이터가 있는지 확인
        if (bytesLoaded == 0)
        {
            std::wcout << L"연결이 종료되었습니다" << std::endl;
            return;
        }

        // 5단계: 데이터 읽기
        int32_t intValue = reader.ReadInt32();
        hstring stringValue = reader.ReadString(
            reader.UnconsumedBufferLength());
        uint8_t byteValue = reader.ReadByte();

        std::wcout << L"정수: " << intValue << std::endl;
        std::wcout << L"문자열: " 
                   << stringValue.c_str() << std::endl;
        std::wcout << L"바이트: " << static_cast<int>(byteValue) 
                   << std::endl;

        // 6단계: 남은 데이터 크기 확인
        uint32_t unconsumedLength = 
            reader.UnconsumedBufferLength();
        std::wcout << L"읽지 않은 바이트: " 
                   << unconsumedLength << std::endl;

        // 7단계: 특정 바이트만 읽기
        std::vector<uint8_t> buffer(10);
        reader.ReadBytes(buffer);

        reader.DetachStream();
    }
    catch (hresult_error const& ex)
    {
        std::wcerr << L"데이터 수신 실패: " 
                   << ex.message().c_str() << std::endl;
    }
}
```

**1단계: 입력 스트림 가져오기**: GetInputStreamAsync()는 네트워크 수신 버퍼에 연결된 스트림을 반환한다.

**2단계: InputStreamOptions 설정**: Partial 옵션을 지정하면 요청한 바이트 전부를 받을 때까지 기다리지 않고, 사용 가능한 만큼만 반환한다. 이는 실시간 데이터 처리에 중요하다.

**3단계: LoadAsync() 호출**: 이 메서드는 네트워크에서 데이터를 읽어서 내부 버퍼에 로드한다. Partial 옵션에서는 최대 요청한 바이트까지만 로드된다. 반환값은 실제로 로드된 바이트 수다. 0이면 연결이 종료되었다는 의미다.

**4단계: 로드된 데이터 확인**: LoadAsync()가 0을 반환하면 원격 호스트가 연결을 종료한 것이다. 이는 정상적인 종료인지 오류인지 판단하여 처리해야 한다.

**5단계: 데이터 읽기**: ReadInt32(), ReadString() 등의 메서드로 타입별로 데이터를 읽는다. DataWriter와 마찬가지로 바이트 순서를 자동으로 처리한다.

**6단계: 남은 데이터 확인**: UnconsumedBufferLength()는 아직 읽지 않은 버퍼의 크기를 반환한다. 부분적으로 읽은 패킷을 처리할 때 중요하다.

**7단계: 바이트 배열 읽기**: ReadBytes()를 사용하면 임의의 바이트 수를 벡터로 읽을 수 있다. 바이너리 데이터나 구조체를 읽을 때 유용하다.

### 연속 수신 루프 패턴
게임 서버에서는 연결이 종료될 때까지 계속 데이터를 받아야 한다:

```cpp
IAsyncAction ReceiveLoopExample(StreamSocket socket)
{
    IInputStream inputStream = 
        co_await socket.GetInputStreamAsync();
    DataReader reader(inputStream);
    reader.InputStreamOptions(InputStreamOptions::Partial);

    try
    {
        while (true)
        {
            // 4096 바이트 또는 사용 가능한 데이터까지 읽기
            uint32_t bytesLoaded = 
                co_await reader.LoadAsync(4096);
            
            if (bytesLoaded == 0)
            {
                // 연결 종료
                std::wcout << L"연결 종료" << std::endl;
                break;
            }

            // 로드된 데이터 처리
            while (reader.UnconsumedBufferLength() > 0)
            {
                // 메시지 파싱 로직
                uint16_t messageType = reader.ReadUInt16();
                uint16_t messageLength = reader.ReadUInt16();
                
                // 메시지 타입별 처리...
            }
        }
    }
    catch (hresult_error const& ex)
    {
        std::wcerr << L"수신 오류: " 
                   << ex.message().c_str() << std::endl;
    }
}
```

## 9.5 연결 종료와 정리
StreamSocket은 관리되지 않은 리소스(native socket, 메모리 등)를 사용하므로, 더 이상 필요 없을 때 반드시 적절히 정리해야 한다. 부정확한 정리는 리소스 누수와 접속자 수 제한 도달로 이어진다.

### 올바른 소켓 닫기

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <iostream>

using namespace winrt::Windows::Networking::Sockets;

void CloseSocketCorrectly(StreamSocket& socket)
{
    try
    {
        // 1단계: 기존 스트림들의 DataReader/DataWriter 정리
        // 이미 위에서 DetachStream()을 호출했다고 가정

        // 2단계: 소켓 연결 종료
        // Close()는 동기 메서드이고 모든 리소스를 해제한다
        socket.Close();

        std::wcout << L"소켓이 정상 종료되었습니다" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcerr << L"소켓 닫기 실패: " 
                   << ex.message().c_str() << std::endl;
    }
}

// RAII 패턴을 사용한 자동 정리
class ManagedSocket
{
private:
    StreamSocket socket_;

public:
    ManagedSocket() = default;

    ~ManagedSocket()
    {
        try
        {
            socket_.Close();
        }
        catch (...)
        {
            // 소멸자에서는 예외를 무시해야 함
        }
    }

    StreamSocket& GetSocket() { return socket_; }

    // 복사 방지
    ManagedSocket(const ManagedSocket&) = delete;
    ManagedSocket& operator=(const ManagedSocket&) = delete;

    // 이동은 허용
    ManagedSocket(ManagedSocket&& other) noexcept
        : socket_(std::move(other.socket_))
    {
    }

    ManagedSocket& operator=(ManagedSocket&& other) noexcept
    {
        if (this != &other)
        {
            socket_.Close();
            socket_ = std::move(other.socket_);
        }
        return *this;
    }
};

// 사용 예제
IAsyncAction UseSocketWithAutoCleanup()
{
    ManagedSocket managedSocket;
    StreamSocket& socket = managedSocket.GetSocket();

    try
    {
        HostName host(L"example.com");
        hstring port(L"80");
        
        co_await socket.ConnectAsync(host, port);

        // 데이터 송수신...

        // 함수 종료 시 자동으로 소켓이 닫힘
    }
    catch (hresult_error const& ex)
    {
        std::wcerr << L"오류: " 
                   << ex.message().c_str() << std::endl;
    }
}
```

**1단계: DataReader/DataWriter 정리**: 스트림을 사용한 후에는 DetachStream()을 호출하여 DataReader/DataWriter가 스트림의 소유권을 포기하게 한다. 이를 하지 않으면 스트림이 아직 사용 중인데 소켓을 닫으려고 시도할 수 있다.

**2단계: Close() 호출**: StreamSocket의 Close() 메서드는 동기적으로 모든 리소스를 해제한다. 이것이 소켓 정리의 핵심이다.

**RAII 패턴**: C++의 RAII(Resource Acquisition Is Initialization) 패턴을 사용하면 소켓이 스코프를 벗어날 때 자동으로 정리된다. 이는 예외 안전성을 제공한다.

## 9.6 소켓 오류 처리

```cpp
IAsyncAction ConnectWithErrorHandling()
{
    StreamSocket socket;

    try
    {
        HostName remoteHost(L"example.com");
        hstring remotePort(L"8888");

        switch (errorStatus)

// 재시도 로직을 포함한 연결
IAsyncAction ConnectWithRetry()
{
    StreamSocket socket;
    int maxRetries = 3;
    int retryCount = 0;
    int delayMs = 1000;

    while (retryCount < maxRetries)
    {
        try
        {
            HostName remoteHost(L"game.example.com");
            hstring remotePort(L"8888");

            co_await socket.ConnectAsync(remoteHost, remotePort);
            std::wcout << L"연결 성공" << std::endl;
            return;
        }
        catch (hresult_error const& ex)
        {
            SocketErrorStatus status = 
                SocketError::GetStatus(ex.code());
            
            // 재시도 가능한 오류인지 확인
            if (status == 
                SocketErrorStatus::ConnectionTimedOut ||
                status == SocketErrorStatus::NetworkIsDown)
            {
                retryCount++;
                if (retryCount < maxRetries)
                {
                    std::wcout << L"" << retryCount 
                               << L"번째 재시도 대기 중..." 
                               << std::endl;
                    
                    Sleep(delayMs);
                    delayMs *= 2;  // 지수 백오프
                    
                    socket.Close();
                    socket = StreamSocket();
                    continue;
                }
            }

            // 재시도 불가능한 오류
            std::wcerr << L"연결 실패: " 
                       << GetSocketErrorMessage(status)
                           .c_str() << std::endl;
            throw;
        }
    }

    std::wcerr << L"최대 재시도 횟수 초과" << std::endl;
}
```

**SocketError::GetStatus()**: HRESULT를 SocketErrorStatus 열거형으로 변환한다. 이를 통해 어떤 종류의 오류인지 정확히 파악할 수 있다.

**오류 분류**: 연결 타임아웃, DNS 실패, 연결 거부 등 각 오류는 다른 원인을 시사하고 다른 해결책을 요구한다.

**재시도 로직**: 일시적인 오류(타임아웃, 네트워크 다운)에 대해서는 재시도를 시도하지만, 영구적인 오류(연결 거부, DNS 실패)에 대해서는 재시도를 하지 않는다.

## 9.7 종합 예제: HTTP 클라이언트 구현
이번 종합 예제는 HTTP 프로토콜을 이해하고 구현하는 과정을 보여준다. StreamSocket의 모든 핵심 기능(연결, 송신, 수신, 오류 처리)이 통합된다.

### 코드의 목적과 설계
이 HTTP 클라이언트는 다음을 구현한다:
- HTTP GET 요청 작성 및 전송
- HTTP 응답 헤더 파싱
- 응답 바디 수신 및 저장
- 오류 처리 및 연결 재사용

설계 결정 사항은 다음과 같다. HTTP 1.0 방식을 사용하여 요청 후 연결 종료하도록 했다(1.1의 keep-alive는 복잡성이 높아 생략). GET 요청만 지원하여 단순성을 유지했다. 헤더와 바디를 구분하여 처리하되, Content-Length를 기반으로 바디 크기를 파악한다.

### 구현 코드

```cpp
#include <winrt/Windows.Networking.h>
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>
#include <sstream>
#include <map>

using namespace winrt;
using namespace winrt::Windows::Networking;
using namespace winrt::Windows::Networking::Sockets;
using namespace winrt::Windows::Storage::Streams;

// HTTP 응답 구조체
struct HttpResponse
{
    int StatusCode;
    hstring StatusMessage;
    std::map<hstring, hstring> Headers;
    std::string Body;
    bool IsSuccess() const { return StatusCode >= 200 && 
                                    StatusCode < 300; }
};

// HTTP 클라이언트 클래스
class SimpleHttpClient
{
private:
    hstring hostname_;
    hstring port_;
    StreamSocket socket_;

public:
    SimpleHttpClient(const hstring& hostname, 
                     const hstring& port = L"80")
        : hostname_(hostname), port_(port)
    {
    }

    // URL 경로와 쿼리 문자열로 GET 요청 수행
    IAsyncOperation<HttpResponse> GetAsync(
        const hstring& path)
    {
        try
        {
            // 1단계: 서버에 연결
            HostName remoteHost(hostname_);
            co_await socket_.ConnectAsync(remoteHost, port_);

            std::wcout << L"[HTTP] 서버 연결: " 
                       << hostname_.c_str() << std::endl;

            // 2단계: HTTP 요청 생성 및 전송
            IOutputStream outputStream = 
                co_await socket_.GetOutputStreamAsync();
            DataWriter writer(outputStream);

            // HTTP 요청 라인
            hstring requestLine = 
                L"GET " + path + L" HTTP/1.0\r\n";
            
            writer.WriteString(requestLine);

            // 요청 헤더
            writer.WriteString(L"Host: ");
            writer.WriteString(hostname_);
            writer.WriteString(L"\r\n");
            
            writer.WriteString(L"Connection: close\r\n");
            writer.WriteString(L"User-Agent: "
                              L"SimpleHttpClient/1.0\r\n");
            writer.WriteString(L"\r\n");

            co_await writer.StoreAsync();
            co_await writer.FlushAsync();

            std::wcout << L"[HTTP] 요청 전송됨" << std::endl;

            // 3단계: 응답 수신
            IInputStream inputStream = 
                co_await socket_.GetInputStreamAsync();
            DataReader reader(inputStream);
            reader.InputStreamOptions(
                InputStreamOptions::Partial);

            // 4단계: 상태 라인 파싱
            HttpResponse response;
            response.Body = "";

            std::string statusLine = 
                co_await ReadLineAsync(reader);
            
            std::istringstream statusStream(statusLine);
            hstring version;
            int statusCode;
            hstring statusMessage;

            // 상태 라인 분석: "HTTP/1.0 200 OK"
            size_t firstSpace = statusLine.find(' ');
            size_t secondSpace = 
                statusLine.find(' ', firstSpace + 1);

            if (firstSpace != std::string::npos && 
                secondSpace != std::string::npos)
            {
                response.StatusCode = 
                    std::stoi(statusLine.substr(
                        firstSpace + 1, 3));
                response.StatusMessage = 
                    hstring(statusLine.substr(
                        secondSpace + 1));
            }

            std::wcout << L"[HTTP] 상태: " 
                       << response.StatusCode 
                       << L" " << response
                           .StatusMessage.c_str() << std::endl;

            // 5단계: 응답 헤더 파싱
            uint32_t contentLength = 0;

            while (true)
            {
                std::string headerLine = 
                    co_await ReadLineAsync(reader);
                
                if (headerLine.empty())
                {
                    // 빈 줄은 헤더 종료
                    break;
                }

                size_t colonPos = 
                    headerLine.find(':');
                
                if (colonPos != std::string::npos)
                {
                    std::string headerName = 
                        headerLine.substr(0, colonPos);
                    std::string headerValue = 
                        headerLine.substr(colonPos + 2);

                    response.Headers[hstring(headerName)] = 
                        hstring(headerValue);

                    // Content-Length 추출
                    if (headerName == "Content-Length")
                    {
                        contentLength = 
                            std::stoi(headerValue);
                    }
                }
            }

            // 6단계: 응답 바디 수신
            if (contentLength > 0)
            {
                response.Body = 
                    co_await ReadBodyAsync(
                        reader, contentLength);
            }
            else
            {
                // Content-Length가 없으면 끝까지 읽기
                uint32_t bytesLoaded = 1;
                while (bytesLoaded > 0)
                {
                    bytesLoaded = 
                        co_await reader.LoadAsync(4096);
                    
                    if (bytesLoaded > 0)
                    {
                        std::vector<uint8_t> buffer(
                            bytesLoaded);
                        reader.ReadBytes(buffer);
                        
                        response.Body.append(
                            reinterpret_cast<char*>(
                                buffer.data()), 
                            buffer.size());
                    }
                }
            }

            std::wcout << L"[HTTP] 바디 크기: " 
                       << response.Body.size() 
                       << L" 바이트" << std::endl;

            // 7단계: 연결 종료
            socket_.Close();

            co_return response;
        }
        catch (hresult_error const& ex)
        {
            std::wcerr << L"[HTTP] 오류: " 
                       << ex.message().c_str() << std::endl;
            
            socket_.Close();
            
            HttpResponse emptyResponse;
            emptyResponse.StatusCode = -1;
            co_return emptyResponse;
        }
    }

private:
    // 한 줄을 읽는 헬퍼 함수
    IAsyncOperation<std::string> ReadLineAsync(
        DataReader& reader)
    {
        std::string line;

        while (true)
        {
            // 1바이트씩 읽기
            uint32_t bytesLoaded = 
                co_await reader.LoadAsync(1);
            
            if (bytesLoaded == 0)
            {
                break;
            }

            uint8_t byte = reader.ReadByte();

            if (byte == '\n')
            {
                // 라인 종료 (\r\n)
                if (!line.empty() && 
                    line.back() == '\r')
                {
                    line.pop_back();
                }
                break;
            }

            line += static_cast<char>(byte);
        }

        co_return line;
    }

    // 바디를 읽는 헬퍼 함수
    IAsyncOperation<std::string> ReadBodyAsync(
        DataReader& reader, 
        uint32_t contentLength)
    {
        std::string body;
        uint32_t remainingBytes = contentLength;

        while (remainingBytes > 0)
        {
            uint32_t toRead = 
                std::min(remainingBytes, 4096u);
            
            uint32_t bytesLoaded = 
                co_await reader.LoadAsync(toRead);
            
            if (bytesLoaded == 0)
            {
                break;
            }

            std::vector<uint8_t> buffer(bytesLoaded);
            reader.ReadBytes(buffer);

            body.append(
                reinterpret_cast<char*>(buffer.data()), 
                buffer.size());

            remainingBytes -= bytesLoaded;
        }

        co_return body;
    }
};

// 사용 예제
IAsyncAction RunHttpClientExample()
{
    SimpleHttpClient client(L"example.com", L"80");

    // GET 요청
    HttpResponse response = 
        co_await client.GetAsync(L"/index.html");

    if (response.IsSuccess())
    {
        std::wcout << L"\n[결과] 요청 성공" << std::endl;
        
        // 응답 헤더 출력
        std::wcout << L"응답 헤더:" << std::endl;
        for (const auto& [name, value] : response.Headers)
        {
            std::wcout << L"  " << name.c_str() << L": " 
                       << value.c_str() << std::endl;
        }

        // 응답 바디 출력 (처음 500자만)
        std::wcout << L"\n응답 바디 (처음 500자):" << std::endl;
        std::string preview = response.Body;
        if (preview.length() > 500)
        {
            preview = preview.substr(0, 500) + "...";
        }
        std::cout << preview << std::endl;
    }
    else
    {
        std::wcerr << L"[결과] 요청 실패: " 
                   << response.StatusCode << std::endl;
    }
}
```

### 주요 부분 상세 해설

**HTTP 요청 생성**: 3단계에서 HTTP GET 요청을 구성한다. HTTP 1.0 방식을 사용하며, 마지막에 빈 줄(`\r\n\r\n`)을 보내 헤더의 끝을 표시한다. 이 부분은 HTTP 프로토콜의 기본이며, 정확한 형식이 필수다.

**상태 라인 파싱**: 4단계에서 응답의 첫 줄(`HTTP/1.0 200 OK`)을 파싱한다. 공백으로 구분된 세 부분을 추출하는데, 상태 코드(200)는 정수로 변환하고, 상태 메시지(OK)는 문자열로 유지한다.

**헤더 파싱 루프**: 5단계에서 각 헤더 라인을 읽고 콜론으로 구분하여 이름과 값을 추출한다. Content-Length는 특히 중요하여 바디 크기 결정에 사용된다.

**바디 수신**: 6단계는 두 가지 방식을 지원한다. Content-Length가 있으면 정확히 그 크기만큼 읽고, 없으면 연결이 끝날 때까지 계속 읽는다. 이는 HTTP/1.1의 chunked transfer encoding은 지원하지 않지만, HTTP/1.0과의 호환성을 보장한다.

**라인 읽기 헬퍼**: ReadLineAsync는 1바이트씩 읽으면서 `\r\n`을 만날 때까지 계속한다. 네트워크 프로토콜 파싱에 자주 사용되는 패턴이다.

### 개선 방안과 확장 가능성

**리다이렉션 처리**: 현재는 3xx 상태 코드를 무시하지만, Location 헤더를 따라 새로운 URL로 요청을 보내는 기능을 추가할 수 있다.

**Chunked Transfer Encoding**: HTTP/1.1의 chunked 인코딩을 지원하면 Content-Length가 없는 응답도 처리할 수 있다. 각 청크는 크기 + `\r\n` + 데이터 형식이다.

**HTTPS 지원**: StreamSocketControl의 ClientCertificate 설정과 보안 프로토콜 설정을 통해 HTTPS를 지원할 수 있다.

**타임아웃**: 매우 느린 네트워크에서 응답을 계속 기다리면 클라이언트가 응답 불가 상태가 될 수 있다. 타이머를 결합하여 타임아웃을 구현할 수 있다.

**압축 지원**: gzip 또는 deflate 압축을 지원하면 응답 크기를 줄일 수 있다. Accept-Encoding 헤더를 보내고, Content-Encoding 헤더를 확인하여 압축 해제한다.

**쿠키 관리**: Set-Cookie 헤더를 파싱하고, 이후 요청에서 Cookie 헤더에 포함시켜 세션을 유지할 수 있다.

이 HTTP 클라이언트는 게임 서버의 버전 확인, 뉴스 다운로드, 사용자 데이터 업로드 등 다양한 목적으로 활용될 수 있다. 기본 구조를 이해하면, 더 복잡한 프로토콜도 같은 방식으로 구현할 수 있다.
