# Chapter 15. WebSocket 프로그래밍

## 15.1 MessageWebSocket API 설명

### WebSocket 프로토콜 개요
WebSocket은 TCP 연결 위에서 동작하는 양방향 통신 프로토콜로, 실시간 애플리케이션에 필수적이다. HTTP와는 달리 한 번 연결을 수립하면 그 연결을 통해 서버와 클라이언트가 자유롭게 메시지를 주고받을 수 있다.

전통적인 HTTP는 클라이언트가 요청을 보내고 서버가 응답하는 요청-응답 방식이다. 반면 WebSocket은 한 번 연결되면 서버가 언제든 클라이언트에게 데이터를 전송할 수 있다. 이는 게임 서버에서 매우 중요한데, 플레이어의 위치 변화나 채팅 메시지를 실시간으로 전달할 수 있기 때문이다.

WinRT의 MessageWebSocket은 WebSocket을 통해 메시지 단위로 통신하는 클래스다. 이와 유사한 StreamWebSocket도 있지만, MessageWebSocket이 더 간단하고 직관적이므로 대부분의 게임 서버에서는 MessageWebSocket을 사용한다.

WebSocket 연결의 흐름을 이해해야 한다. 첫 단계는 WebSocket 핸드셰이크로, HTTP 업그레이드 요청을 통해 일반 TCP 연결을 WebSocket으로 변환한다. 두 번째 단계는 메시지 송수신으로, 양쪽이 자유롭게 메시지를 주고받는다. 마지막 단계는 연결 종료로, 어느 한쪽이 연결을 닫으면 양쪽 모두 정리 작업을 수행한다.

MessageWebSocket은 텍스트와 바이너리 두 가지 메시지 타입을 지원한다. 텍스트 메시지는 JSON 같은 구조화된 데이터에 적합하고, 바이너리 메시지는 효율성이 중요한 경우에 사용된다. 게임 서버에서는 보통 바이너리 메시지를 사용하여 대역폭을 절약한다.

## 15.2 WebSocket 연결하기

### ConnectAsync()와 핸드셰이크
WebSocket 연결을 시작하려면 ConnectAsync() 메서드를 호출해야 한다. 이 메서드는 지정한 WebSocket 서버 주소에 연결을 시도하고, 핸드셰이크 과정을 자동으로 처리한다.

다음은 WebSocket에 연결하는 기본 예제다.

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Foundation.h>
#include <iostream>

using namespace winrt;
using namespace Windows::Networking::Sockets;
using namespace Windows::Foundation;

IAsyncAction ConnectToWebSocketAsync(
    std::wstring const& serverUri)
{
    MessageWebSocket webSocket;
    
    try
    {
        std::wcout << L"WebSocket 연결 중: " << serverUri.c_str() 
                  << std::endl;
        
        Uri uri(serverUri);
        
        // 서버에 연결
        co_await webSocket.ConnectAsync(uri);
        
        std::wcout << L"WebSocket 연결 성공" << std::endl;
        std::wcout << L"- 로컬 주소: " 
                  << webSocket.LocalHostName().DisplayName().c_str() 
                  << std::endl;
        std::wcout << L"- 상태: 연결됨" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"연결 실패: " << ex.message().c_str() 
                  << std::endl;
    }
}

// 더 자세한 연결 예제
IAsyncAction DetailedConnectionAsync()
{
    MessageWebSocket webSocket;
    
    try
    {
        // 연결 전에 이벤트 핸들러 등록
        webSocket.MessageReceived(
            [](MessageWebSocket const& socket,
               MessageWebSocketMessageReceivedEventArgs const& args)
        {
            std::wcout << L"메시지 수신" << std::endl;
        });
        
        // 서버 주소 설정
        std::wstring serverUri = L"ws://localhost:8080/game";
        Uri uri(serverUri);
        
        std::wcout << L"연결 시작..." << std::endl;
        
        // 비동기 연결
        co_await webSocket.ConnectAsync(uri);
        
        std::wcout << L"연결 완료!" << std::endl;
        std::wcout << L"WebSocket 프로토콜이 성공적으로 협상되었습니다." 
                  << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"오류: " << ex.message().c_str() 
                  << std::endl;
    }
}
```

ConnectAsync()를 호출할 때 중요한 점이 몇 가지 있다. 첫 번째는 올바른 URI 형식이다. WebSocket의 URI는 "ws://" (암호화되지 않은) 또는 "wss://" (암호화된) 프로토콜을 사용해야 한다.

두 번째는 핸드셰이크 과정이다. ConnectAsync()는 내부적으로 HTTP 업그레이드 요청을 보내고 서버의 응답을 기다린다. 이 과정이 성공하면 연결이 WebSocket으로 변환된다.

세 번째는 타이밍이다. ConnectAsync()가 성공하기 전에 MessageReceived 같은 이벤트 핸들러를 등록하는 것이 좋다. 연결이 수립되자마자 메시지가 도착할 수도 있기 때문이다.

## 15.3 MessageWebSocketControl 설정

### 메시지 타입과 버퍼 크기
MessageWebSocket의 동작을 세밀하게 조정하려면 Control 속성에 접근해야 한다. MessageWebSocketControl은 메시지 타입, 버퍼 크기, 데이터 압축 등을 설정할 수 있게 해준다.

다음은 MessageWebSocketControl을 사용하는 예제다.

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <iostream>

using namespace winrt;
using namespace Windows::Networking::Sockets;

IAsyncAction ConfigureWebSocketAsync()
{
    MessageWebSocket webSocket;
    
    try
    {
        // 연결 전에 컨트롤 설정
        auto control = webSocket.Control();
        
        // 메시지 타입 설정 (기본값은 Utf8)
        // Utf8: 텍스트 메시지
        // Binary: 바이너리 메시지
        control.MessageType(SocketMessageType::Binary);
        std::wcout << L"메시지 타입: Binary" << std::endl;
        
        // 최대 메시지 크기 설정 (바이트)
        // 게임 서버는 보통 수 MB 정도로 설정
        control.MaxMessageSize(1024 * 1024);  // 1MB
        std::wcout << L"최대 메시지 크기: 1MB" << std::endl;
        
        // 서버에 연결
        Uri uri(L"ws://localhost:8080/game");
        co_await webSocket.ConnectAsync(uri);
        
        std::wcout << L"설정된 상태로 WebSocket 연결 완료" << std::endl;
        
        // 현재 설정 확인
        auto currentMessageType = control.MessageType();
        auto currentMaxSize = control.MaxMessageSize();
        
        std::wcout << L"현재 설정:" << std::endl;
        std::wcout << L"- 메시지 타입: " 
                  << (currentMessageType == SocketMessageType::Binary ? 
                      L"Binary" : L"Text") << std::endl;
        std::wcout << L"- 최대 크기: " << currentMaxSize << L" bytes" 
                  << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"오류: " << ex.message().c_str() 
                  << std::endl;
    }
}

// 게임 서버용 설정 예제
IAsyncAction GameServerWebSocketConfigAsync()
{
    MessageWebSocket webSocket;
    
    try
    {
        auto control = webSocket.Control();
        
        // 게임 서버 최적화 설정
        // 1. 바이너리 메시지 사용 (효율성)
        control.MessageType(SocketMessageType::Binary);
        
        // 2. 충분한 메시지 크기 허용
        // 대규모 맵 데이터 동기화 등을 고려
        control.MaxMessageSize(10 * 1024 * 1024);  // 10MB
        
        std::wcout << L"게임 서버용 WebSocket 설정 완료" << std::endl;
        std::wcout << L"- 메시지 타입: Binary (효율적)" << std::endl;
        std::wcout << L"- 최대 메시지 크기: 10MB" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"설정 오류: " << ex.message().c_str() 
                  << std::endl;
    }
}
```

MessageType 설정은 중요한 결정이다. Binary를 선택하면 데이터를 바이트 단위로 전송할 수 있어 효율적이다. 반면 Utf8을 선택하면 텍스트 기반 메시지를 사용할 수 있어 디버깅과 상호 운용성이 좋다. 게임 서버에서는 보통 성능을 위해 Binary를 선택한다.

MaxMessageSize는 한 번에 수신할 수 있는 메시지의 최대 크기다. 너무 작으면 큰 메시지를 받지 못하고, 너무 크면 메모리를 낭비한다. 게임의 특성에 따라 적절한 값을 설정해야 한다.

중요한 주의사항은 이 설정들이 연결 전에 수행되어야 한다는 것이다. ConnectAsync() 이후에 변경하면 일부 설정이 적용되지 않을 수 있다.

## 15.4 메시지 보내기

### DataWriter를 통한 전송
WebSocket으로 메시지를 보내려면 OutputStream을 통해 DataWriter를 생성하고, 이를 사용하여 데이터를 작성한 후 전송한다.

다음은 WebSocket을 통해 메시지를 보내는 예제다.

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>
#include <cstring>

using namespace winrt;
using namespace Windows::Networking::Sockets;
using namespace Windows::Storage::Streams;

// 텍스트 메시지 전송
IAsyncAction SendTextMessageAsync(
    MessageWebSocket const& webSocket,
    std::wstring const& message)
{
    try
    {
        DataWriter writer(webSocket.OutputStream());
        
        // 텍스트 메시지 작성
        writer.WriteString(message);
        
        // 메시지 전송
        uint32_t bytesSent = co_await writer.StoreAsync();
        
        std::wcout << L"텍스트 메시지 전송: " << message.c_str() 
                  << std::endl;
        std::wcout << L"전송된 바이트: " << bytesSent << std::endl;
        
        writer.DetachStream();
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"전송 오류: " << ex.message().c_str() 
                  << std::endl;
    }
}

// 바이너리 메시지 전송
IAsyncAction SendBinaryMessageAsync(
    MessageWebSocket const& webSocket,
    uint8_t const* data,
    uint32_t length)
{
    try
    {
        DataWriter writer(webSocket.OutputStream());
        
        // 바이너리 데이터 작성
        writer.WriteBytes(
            winrt::array_view<uint8_t const>(data, length)
        );
        
        // 메시지 전송
        uint32_t bytesSent = co_await writer.StoreAsync();
        
        std::wcout << L"바이너리 메시지 전송: " << length 
                  << L" bytes" << std::endl;
        std::wcout << L"전송된 바이트: " << bytesSent << std::endl;
        
        writer.DetachStream();
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"전송 오류: " << ex.message().c_str() 
                  << std::endl;
    }
}

// 게임 패킷 전송 예제
IAsyncAction SendGamePacketAsync(
    MessageWebSocket const& webSocket,
    uint32_t playerId,
    float posX,
    float posY)
{
    try
    {
        DataWriter writer(webSocket.OutputStream());
        
        // 게임 패킷 구성
        // 형식: [PlayerId(4)] [PosX(4)] [PosY(4)]
        writer.WriteUInt32(playerId);
        writer.WriteFloat(posX);
        writer.WriteFloat(posY);
        
        // 전송
        co_await writer.StoreAsync();
        
        std::wcout << L"게임 패킷 전송: PlayerID=" << playerId 
                  << L", Pos=(" << posX << L", " << posY << L")" 
                  << std::endl;
        
        writer.DetachStream();
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"전송 실패: " << ex.message().c_str() 
                  << std::endl;
    }
}
```

메시지 전송에서 중요한 몇 가지 포인트가 있다.

첫 번째는 DataWriter의 생성이다. webSocket.OutputStream()으로 출력 스트림을 얻고, 이를 바탕으로 DataWriter를 생성한다. 이 DataWriter가 바로 데이터를 작성하는 도구다.

두 번째는 데이터 작성이다. WriteString(), WriteUInt32(), WriteFloat() 등 다양한 메서드를 사용하여 여러 타입의 데이터를 작성할 수 있다.

세 번째는 StoreAsync() 호출이다. 이 메서드는 작성한 데이터를 실제로 스트림에 보낸다. 이 호출이 완료될 때까지 대기해야 전송이 완료된다.

네 번째는 DetachStream() 호출이다. DataWriter를 사용한 후 반드시 이를 호출하여 스트림과의 연결을 끊어야 한다. 그렇지 않으면 DataWriter가 소멸될 때 스트림도 함께 정리될 수 있다.

## 15.5 메시지 받기

### MessageReceived 이벤트 처리
WebSocket에서 메시지를 수신하려면 MessageReceived 이벤트에 핸들러를 등록해야 한다. 서버에서 메시지를 보내면 이 이벤트가 발생하고, 등록된 핸들러가 호출된다.

다음은 WebSocket 메시지 수신을 다루는 예제다.

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>

using namespace winrt;
using namespace Windows::Networking::Sockets;
using namespace Windows::Storage::Streams;

class WebSocketMessageHandler
{
private:
    MessageWebSocket m_webSocket;
    
public:
    IAsyncAction InitializeAsync(std::wstring const& serverUri)
    {
        try
        {
            // MessageReceived 이벤트 핸들러 등록
            m_webSocket.MessageReceived(
                { this, &WebSocketMessageHandler::OnMessageReceived }
            );
            
            // 서버에 연결
            Uri uri(serverUri);
            co_await m_webSocket.ConnectAsync(uri);
            
            std::wcout << L"WebSocket 연결 및 메시지 핸들러 등록 완료" 
                      << std::endl;
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"초기화 실패: " << ex.message().c_str() 
                      << std::endl;
        }
    }
    
private:
    void OnMessageReceived(
        MessageWebSocket const& sender,
        MessageWebSocketMessageReceivedEventArgs const& args)
    {
        try
        {
            // 메시지 타입 확인
            auto messageType = args.MessageType();
            
            if (messageType == SocketMessageType::Text)
            {
                // 텍스트 메시지 처리
                DataReader reader = args.GetDataReader();
                uint32_t length = reader.UnconsumedBufferLength();
                std::wstring message = reader.ReadString(length);
                
                std::wcout << L"[TEXT] " << message.c_str() 
                          << std::endl;
            }
            else if (messageType == SocketMessageType::Binary)
            {
                // 바이너리 메시지 처리
                DataReader reader = args.GetDataReader();
                uint32_t length = reader.UnconsumedBufferLength();
                
                std::wcout << L"[BINARY] " << length 
                          << L" bytes 수신" << std::endl;
                
                // 바이너리 데이터 읽기
                auto buffer = reader.ReadBuffer(length);
                ProcessBinaryMessage(buffer);
            }
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"메시지 처리 오류: " 
                      << ex.message().c_str() << std::endl;
        }
    }
    
    void ProcessBinaryMessage(IBuffer const& buffer)
    {
        // 바이너리 메시지 파싱
        uint32_t length = buffer.Length();
        std::wcout << L"바이너리 메시지 처리: " << length 
                  << L" bytes" << std::endl;
        
        // 실제 게임 로직에서는 여기서 패킷을 파싱
        // 예: 플레이어 위치, 인벤토리 업데이트 등
    }
};

// 게임 메시지 핸들러 예제
class GameMessageHandler
{
private:
    MessageWebSocket m_webSocket;
    
public:
    IAsyncAction SetupAsync(std::wstring const& serverUri)
    {
        // 메시지 수신 핸들러 등록
        m_webSocket.MessageReceived(
            [this](MessageWebSocket const& socket,
                   MessageWebSocketMessageReceivedEventArgs const& args)
        {
            this->HandleGameMessage(args);
        });
        
        Uri uri(serverUri);
        co_await m_webSocket.ConnectAsync(uri);
    }
    
private:
    void HandleGameMessage(MessageWebSocketMessageReceivedEventArgs const& args)
    {
        DataReader reader = args.GetDataReader();
        
        // 게임 패킷 파싱
        // 형식: [MessageType(1)] [Payload(...)]
        uint8_t messageType = reader.ReadByte();
        
        switch (messageType)
        {
        case 1:  // 플레이어 위치 업데이트
        {
            uint32_t playerId = reader.ReadUInt32();
            float posX = reader.ReadFloat();
            float posY = reader.ReadFloat();
            
            std::wcout << L"[위치 업데이트] Player " << playerId 
                      << L" -> (" << posX << L", " << posY << L")" 
                      << std::endl;
            break;
        }
        case 2:  // 채팅 메시지
        {
            std::wstring message = reader.ReadString(
                reader.UnconsumedBufferLength()
            );
            
            std::wcout << L"[채팅] " << message.c_str() 
                      << std::endl;
            break;
        }
        default:
            std::wcout << L"[알 수 없는 메시지 타입] " 
                      << messageType << std::endl;
            break;
        }
    }
};
```

메시지 수신 처리에서 중요한 부분들을 설명하자면:

첫 번째는 이벤트 핸들러 등록이다. MessageReceived에 함수 포인터나 람다를 등록하면, 메시지가 도착할 때마다 이 핸들러가 호출된다.

두 번째는 메시지 타입 확인이다. Text인지 Binary인지 확인하여 적절한 방식으로 처리한다.

세 번째는 DataReader 사용이다. args.GetDataReader()로 메시지 데이터를 읽을 수 있는 리더를 얻는다.

네 번째는 UnconsumedBufferLength()다. 이 메서드로 읽지 않은 바이트 수를 알 수 있어, 정확한 길이의 데이터를 읽을 수 있다.

## 15.6 StreamWebSocket 사용하기

### 스트림 기반 통신
MessageWebSocket이 메시지 단위로 처리한다면, StreamWebSocket은 스트림 기반으로 처리한다. 대량의 데이터를 전송해야 하거나, 데이터의 경계가 명확하지 않은 경우 StreamWebSocket이 더 적합하다.

다음은 StreamWebSocket을 사용하는 예제다.

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>

using namespace winrt;
using namespace Windows::Networking::Sockets;
using namespace Windows::Storage::Streams;

// StreamWebSocket 기본 사용
IAsyncAction UseStreamWebSocketAsync()
{
    StreamWebSocket webSocket;
    
    try
    {
        // 서버에 연결
        Uri uri(L"ws://localhost:8080/stream");
        co_await webSocket.ConnectAsync(uri);
        
        std::wcout << L"StreamWebSocket 연결 완료" << std::endl;
        
        // 출력 스트림으로 데이터 전송
        DataWriter writer(webSocket.OutputStream());
        writer.WriteString(L"Hello from Stream WebSocket");
        co_await writer.StoreAsync();
        
        std::wcout << L"데이터 전송 완료" << std::endl;
        
        // 입력 스트림에서 데이터 수신
        DataReader reader(webSocket.InputStream());
        uint32_t bytesRead = co_await reader.LoadAsync(1024);
        
        if (bytesRead > 0)
        {
            std::wstring response = reader.ReadString(bytesRead);
            std::wcout << L"서버 응답: " << response.c_str() 
                      << std::endl;
        }
        
        writer.DetachStream();
        reader.DetachStream();
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"오류: " << ex.message().c_str() 
                  << std::endl;
    }
}

// 대용량 데이터 전송 예제
IAsyncAction SendLargeDataAsync()
{
    StreamWebSocket webSocket;
    
    try
    {
        Uri uri(L"ws://localhost:8080/upload");
        co_await webSocket.ConnectAsync(uri);
        
        std::wcout << L"대용량 데이터 전송 시작" << std::endl;
        
        DataWriter writer(webSocket.OutputStream());
        
        // 1MB 데이터 청크 단위로 전송
        const uint32_t CHUNK_SIZE = 1024 * 1024;
        const uint32_t TOTAL_SIZE = 10 * 1024 * 1024;  // 10MB
        
        for (uint32_t sent = 0; sent < TOTAL_SIZE; sent += CHUNK_SIZE)
        {
            uint32_t chunkSize = (TOTAL_SIZE - sent) < CHUNK_SIZE ?
                               (TOTAL_SIZE - sent) : CHUNK_SIZE;
            
            // 청크 데이터 작성
            for (uint32_t i = 0; i < chunkSize; i++)
            {
                writer.WriteByte(static_cast<uint8_t>(i % 256));
            }
            
            // 청크 전송
            co_await writer.StoreAsync();
            
            std::wcout << L"전송 진행: " << (sent + chunkSize) / (1024 * 1024) 
                      << L"MB / " << TOTAL_SIZE / (1024 * 1024) 
                      << L"MB" << std::endl;
        }
        
        std::wcout << L"전송 완료" << std::endl;
        
        writer.DetachStream();
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"전송 오류: " << ex.message().c_str() 
                  << std::endl;
    }
}

// 연속 데이터 수신 예제
IAsyncAction ReceiveStreamDataAsync()
{
    StreamWebSocket webSocket;
    
    try
    {
        Uri uri(L"ws://localhost:8080/stream");
        co_await webSocket.ConnectAsync(uri);
        
        std::wcout << L"스트림 데이터 수신 중..." << std::endl;
        
        DataReader reader(webSocket.InputStream());
        
        bool isReceiving = true;
        uint32_t totalBytesReceived = 0;
        
        while (isReceiving)
        {
            // 데이터 로드 (대기)
            uint32_t bytesLoaded = 
                co_await reader.LoadAsync(4096);
            
            if (bytesLoaded == 0)
            {
                // 연결 종료
                std::wcout << L"연결이 종료되었습니다" 
                          << std::endl;
                isReceiving = false;
            }
            else
            {
                totalBytesReceived += bytesLoaded;
                
                // 데이터 처리
                auto buffer = reader.ReadBuffer(bytesLoaded);
                
                std::wcout << L"청크 수신: " << bytesLoaded 
                          << L" bytes (총 " << totalBytesReceived 
                          << L" bytes)" << std::endl;
            }
        }
        
        reader.DetachStream();
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"수신 오류: " << ex.message().c_str() 
                  << std::endl;
    }
}
```

StreamWebSocket과 MessageWebSocket의 차이를 이해하는 것이 중요하다.

MessageWebSocket은 메시지 경계를 자동으로 관리한다. 한 번에 하나의 메시지를 보내고, 수신할 때도 메시지 단위로 처리된다.

StreamWebSocket은 데이터를 스트림 방식으로 처리한다. 바이트가 연속적으로 흐르며, 메시지 경계를 수동으로 관리해야 한다. 따라서 프로토콜 설계가 더 복잡할 수 있다.

선택 기준은 다음과 같다. 게임 상태 업데이트 같은 이산적인 메시지를 주고받는다면 MessageWebSocket을 사용한다. 영상 스트림이나 대용량 파일 전송처럼 연속적인 데이터 흐름이 필요하면 StreamWebSocket을 사용한다.

## 15.7 종합 예제: 실시간 게임 데이터 스트리밍

### 코드의 목적
이 예제는 WebSocket을 통해 게임 상태를 실시간으로 동기화하는 시스템을 구현한다. 플레이어의 위치 업데이트, 채팅 메시지, 게임 이벤트 등을 WebSocket으로 전송하고, 서버로부터 받은 데이터를 처리한다. 또한 연결 끊김을 감지하고 자동으로 재연결하는 기능도 포함한다.

이는 게임 서버 개발에서 실제로 필요한 핵심 기능이다. 모던 게임은 모두 실시간 통신을 기반으로 하며, WebSocket은 이를 구현하는 효율적인 방법이다.

### 설계 결정 이유
게임 데이터 스트리밍에는 여러 고려사항이 있다.

첫 번째는 메시지 형식이다. 효율성을 위해 바이너리 메시지를 사용하고, 고정 길이의 헤더로 메시지 타입을 식별하도록 설계했다.

두 번째는 하트비트(heartbeat) 메커니즘이다. 연결이 유지되는지 확인하기 위해 주기적으로 하트비트 메시지를 보낸다. 서버로부터 일정 시간 동안 응답이 없으면 연결을 끊었다고 판단한다.

세 번째는 재연결 로직이다. 연결이 끊어지면 자동으로 재연결을 시도한다. 지수 백오프(exponential backoff) 알고리즘을 사용하여 재연결 시도 간격을 점진적으로 늘린다.

네 번째는 메시지 큐다. 연결이 끊어진 동안 전송할 메시지를 큐에 저장했다가, 재연결 후 전송한다.

### 코드 구현

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Storage.Streams.h>
#include <winrt/Windows.Foundation.h>
#include <iostream>
#include <queue>
#include <mutex>
#include <chrono>
#include <cstring>

using namespace winrt;
using namespace Windows::Networking::Sockets;
using namespace Windows::Storage::Streams;
using namespace Windows::Foundation;

// 게임 메시지 타입
enum class GameMessageType : uint8_t
{
    PlayerPosition = 1,
    PlayerAction = 2,
    ChatMessage = 3,
    Heartbeat = 4,
    GameEvent = 5
};

// 플레이어 위치 데이터
struct PlayerPosition
{
    uint32_t playerId;
    float x;
    float y;
    float z;
};

// 게임 스트리밍 클라이언트
class GameStreamingClient
{
private:
    MessageWebSocket m_webSocket;
    std::queue<std::vector<uint8_t>> m_messageQueue;
    std::mutex m_queueMutex;
    
    uint32_t m_playerId = 0;
    bool m_isConnected = false;
    bool m_shouldReconnect = true;
    uint32_t m_reconnectAttempts = 0;
    static constexpr uint32_t MAX_RECONNECT_ATTEMPTS = 5;
    
public:
    // 서버에 연결
    IAsyncAction ConnectAsync(
        std::wstring const& serverUri,
        uint32_t playerId)
    {
        m_playerId = playerId;
        
        try
        {
            std::wcout << L"게임 서버에 연결 중... (Player " 
                      << playerId << L")" << std::endl;
            
            // 메시지 수신 핸들러 등록
            m_webSocket.MessageReceived(
                { this, &GameStreamingClient::OnMessageReceived }
            );
            
            // 연결
            Uri uri(serverUri);
            co_await m_webSocket.ConnectAsync(uri);
            
            m_isConnected = true;
            m_reconnectAttempts = 0;
            
            std::wcout << L"[✓] 게임 서버 연결 성공" << std::endl;
            
            // 하트비트 루프 시작
            co_await HeartbeatLoopAsync();
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"[✗] 연결 실패: " 
                      << ex.message().c_str() << std::endl;
            m_isConnected = false;
            
            // 재연결 시도
            co_await ReconnectAsync(serverUri);
        }
    }
    
    // 플레이어 위치 업데이트 전송
    IAsyncAction SendPlayerPositionAsync(
        float x, float y, float z)
    {
        try
        {
            if (!m_isConnected)
            {
                // 연결이 끊어진 경우 메시지를 큐에 저장
                QueuePositionMessage(x, y, z);
                return;
            }
            
            DataWriter writer(m_webSocket.OutputStream());
            
            // 메시지 구성: [Type(1)] [PlayerId(4)] [X(4)] [Y(4)] [Z(4)]
            writer.WriteByte(
                static_cast<uint8_t>(GameMessageType::PlayerPosition)
            );
            writer.WriteUInt32(m_playerId);
            writer.WriteFloat(x);
            writer.WriteFloat(y);
            writer.WriteFloat(z);
            
            co_await writer.StoreAsync();
            writer.DetachStream();
            
            // 디버깅 정보 (게임에서는 제거)
            // std::wcout << L"위치 전송: (" << x << L", " << y 
            //           << L", " << z << L")" << std::endl;
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"위치 전송 실패: " 
                      << ex.message().c_str() << std::endl;
            m_isConnected = false;
        }
    }
    
    // 채팅 메시지 전송
    IAsyncAction SendChatMessageAsync(
        std::wstring const& message)
    {
        try
        {
            if (!m_isConnected)
            {
                std::wcout << L"서버 연결 끊김, 메시지 저장" 
                          << std::endl;
                return;
            }
            
            DataWriter writer(m_webSocket.OutputStream());
            
            // 메시지 구성: [Type(1)] [PlayerId(4)] [Message(...)]
            writer.WriteByte(
                static_cast<uint8_t>(GameMessageType::ChatMessage)
            );
            writer.WriteUInt32(m_playerId);
            writer.WriteString(message);
            
            co_await writer.StoreAsync();
            writer.DetachStream();
            
            std::wcout << L"[채팅 전송] " << message.c_str() 
                      << std::endl;
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"채팅 전송 실패: " 
                      << ex.message().c_str() << std::endl;
        }
    }
    
    bool IsConnected() const { return m_isConnected; }
    
private:
    // 메시지 수신 처리
    void OnMessageReceived(
        MessageWebSocket const& sender,
        MessageWebSocketMessageReceivedEventArgs const& args)
    {
        try
        {
            DataReader reader = args.GetDataReader();
            uint32_t length = reader.UnconsumedBufferLength();
            
            if (length == 0) return;
            
            // 메시지 타입 읽기
            uint8_t messageType = reader.ReadByte();
            
            switch (static_cast<GameMessageType>(messageType))
            {
            case GameMessageType::PlayerPosition:
            {
                uint32_t playerId = reader.ReadUInt32();
                float x = reader.ReadFloat();
                float y = reader.ReadFloat();
                float z = reader.ReadFloat();
                
                std::wcout << L"[플레이어 위치] Player " 
                          << playerId << L" -> (" << x << L", " 
                          << y << L", " << z << L")" << std::endl;
                break;
            }
            case GameMessageType::ChatMessage:
            {
                uint32_t playerId = reader.ReadUInt32();
                std::wstring message = reader.ReadString(
                    reader.UnconsumedBufferLength()
                );
                
                std::wcout << L"[채팅] Player " << playerId 
                          << L": " << message.c_str() << std::endl;
                break;
            }
            case GameMessageType::GameEvent:
            {
                uint32_t eventId = reader.ReadUInt32();
                std::wcout << L"[게임 이벤트] ID=" << eventId 
                          << std::endl;
                break;
            }
            case GameMessageType::Heartbeat:
            {
                // 하트비트 응답 - 연결이 살아있음을 확인
                break;
            }
            default:
                std::wcout << L"[알 수 없는 메시지 타입] " 
                          << messageType << std::endl;
                break;
            }
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"메시지 처리 오류: " 
                      << ex.message().c_str() << std::endl;
        }
    }
    
    // 하트비트 루프
    IAsyncAction HeartbeatLoopAsync()
    {
        while (m_shouldReconnect && m_isConnected)
        {
            try
            {
                // 5초 대기
                co_await winrt::resume_after(
                    std::chrono::seconds(5)
                );
                
                // 하트비트 메시지 전송
                DataWriter writer(m_webSocket.OutputStream());
                writer.WriteByte(
                    static_cast<uint8_t>(GameMessageType::Heartbeat)
                );
                writer.WriteUInt32(m_playerId);
                
                co_await writer.StoreAsync();
                writer.DetachStream();
            }
            catch (hresult_error const&)
            {
                m_isConnected = false;
                break;
            }
        }
    }
    
    // 재연결
    IAsyncAction ReconnectAsync(std::wstring const& serverUri)
    {
        while (m_reconnectAttempts < MAX_RECONNECT_ATTEMPTS)
        {
            m_reconnectAttempts++;
            
            // 지수 백오프: 1초, 2초, 4초, 8초, 16초
            uint32_t delaySeconds = 1 << (m_reconnectAttempts - 1);
            
            std::wcout << L"[재연결] " << m_reconnectAttempts 
                      << L"번째 시도 (" << delaySeconds 
                      << L"초 후)" << std::endl;
            
            co_await winrt::resume_after(
                std::chrono::seconds(delaySeconds)
            );
            
            try
            {
                co_await ConnectAsync(serverUri, m_playerId);
                
                // 재연결 성공 후 큐에 있는 메시지 전송
                co_await FlushQueueAsync();
                
                return;
            }
            catch (hresult_error const& ex)
            {
                std::wcout << L"재연결 실패: " 
                          << ex.message().c_str() << std::endl;
            }
        }
        
        std::wcout << L"[✗] 최대 재연결 시도 횟수 초과" 
                  << std::endl;
    }
    
    // 메시지 큐에 저장
    void QueuePositionMessage(float x, float y, float z)
    {
        std::lock_guard<std::mutex> lock(m_queueMutex);
        
        std::vector<uint8_t> message;
        message.push_back(
            static_cast<uint8_t>(GameMessageType::PlayerPosition)
        );
        
        // PlayerId, X, Y, Z를 바이트로 변환하여 추가
        // (단순화된 구현)
        
        m_messageQueue.push(message);
    }
    
    // 큐에 있는 메시지 전송
    IAsyncAction FlushQueueAsync()
    {
        std::lock_guard<std::mutex> lock(m_queueMutex);
        
        while (!m_messageQueue.empty())
        {
            auto message = m_messageQueue.front();
            m_messageQueue.pop();
            
            try
            {
                DataWriter writer(m_webSocket.OutputStream());
                writer.WriteBytes(
                    winrt::array_view<uint8_t const>(
                        message.data(), message.size()
                    )
                );
                
                co_await writer.StoreAsync();
                writer.DetachStream();
            }
            catch (hresult_error const&)
            {
                // 전송 실패 시 다시 큐에 추가
                m_messageQueue.push(message);
                break;
            }
        }
    }
};

// 사용 예제
IAsyncAction GameStreamingExampleAsync()
{
    GameStreamingClient client;
    
    std::wcout << L"게임 스트리밍 시작" << std::endl;
    std::wcout << std::wstring(50, L'=') << std::endl;
    
    try
    {
        // 서버에 연결 (PlayerID = 1001)
        co_await client.ConnectAsync(L"ws://localhost:8080/game", 1001);
        
        // 위치 업데이트 전송 (게임 루프 시뮬레이션)
        for (int i = 0; i < 10; i++)
        {
            float x = 100.0f + i * 10.0f;
            float y = 200.0f + i * 5.0f;
            float z = 50.0f;
            
            co_await client.SendPlayerPositionAsync(x, y, z);
            
            // 채팅 메시지
            if (i == 5)
            {
                co_await client.SendChatMessageAsync(L"안녕하세요!");
            }
            
            // 100ms 대기 (게임 프레임)
            co_await winrt::resume_after(
                std::chrono::milliseconds(100)
            );
        }
        
        std::wcout << L"\n게임 스트리밍 완료" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"오류: " << ex.message().c_str() 
                  << std::endl;
    }
}
```

### 주요 부분 상세 해설
메시지 형식은 바이너리로 설계되어 있다. 첫 바이트는 메시지 타입이고, 나머지는 페이로드다. 이러한 구조는 고정 크기 헤더를 통해 메시지를 빠르게 식별할 수 있게 한다.

하트비트 로직은 주기적으로 메시지를 보내서 연결 상태를 확인한다. 5초마다 하트비트를 전송하므로, 연결이 끊어지면 최대 5초 내에 이를 감지할 수 있다.

재연결 로직은 지수 백오프를 사용한다. 첫 번째 시도는 1초, 두 번째는 2초, 세 번째는 4초 등으로 대기 시간을 늘린다. 이는 서버 부하를 고려한 설계다.

메시지 큐는 연결이 끊어진 동안 전송할 메시지를 저장한다. 재연결 후에 이 메시지들을 전송하여 데이터 손실을 최소화한다.

### 개선 방안과 확장 가능성
첫 번째 개선점은 메시지 압축이다. 위치 데이터 같은 경우 많은 소수점이 포함되므로, 좌표를 정수로 변환하거나 델타 압축을 적용하면 대역폭을 절약할 수 있다.

두 번째는 적응형 하트비트 간격이다. 네트워크 상태에 따라 하트비트 간격을 동적으로 조정할 수 있다.

세 번째는 메시지 우선순위 큐다. 중요한 메시지(게임 이벤트)를 먼저 전송하고, 덜 중요한 메시지(위치 업데이트)는 나중에 전송할 수 있다.

네 번째는 네트워크 통계 수집이다. 패킷 손실률, 평균 지연, 재연결 횟수 등을 기록하여 네트워크 상태를 모니터링할 수 있다.

다섯 번째는 서버 부하 분산이다. 여러 서버에 연결하거나, 메시지를 여러 경로로 전송하는 등 신뢰성을 높일 수 있다.

실전 게임 서버에서는 이러한 개선 사항들을 상황에 맞게 적용하여 강력하고 안정적인 실시간 통신 시스템을 구축할 수 있다.
