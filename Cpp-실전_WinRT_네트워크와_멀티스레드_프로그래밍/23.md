# Chapter 23. 실시간 게임 서버 기초
이번 장에서는 온라인 게임 서버의 핵심이 되는 실시간 네트워킹 기술을 다룬다. 게임 서버는 단순한 채팅이나 로비 서버와 달리, 매우 엄격한 시간 제약과 일관성 요구사항을 가진다. 플레이어들이 동시에 움직이고, 충돌하고, 상호작용하는 상황을 수십 밀리초 단위로 조율해야 하기 때문이다.

이 장을 통해 여러분은 게임 서버의 심장인 게임 루프를 구현하고, 네트워크 지연을 보상하며, 플레이어 상태를 효율적으로 동기화하는 방법을 배울 것이다.

---

## 23.1 실시간 게임 네트워킹 개요
게임 서버는 다른 서버 애플리케이션과 완전히 다른 특성을 가진다. 채팅 서버는 메시지가 조금 늦게 도착해도 괜찮지만, 게임 서버에서는 밀리초 단위의 지연이 게임 경험을 크게 해친다.

### TCP vs UDP 선택
이것이 TCP와 UDP의 선택이 중요한 이유다.

**TCP의 특징**
TCP는 신뢰성을 보장한다. 모든 패킷이 순서대로 도착한다. 하지만 패킷 손실 시 재전송으로 인해 지연이 발생한다. 만약 게임 중간의 한 프레임 데이터가 손실되었다면, 그 프레임만 다시 보내는 것이 아니라 이후의 모든 패킷을 기다려야 한다. 이를 "Head-of-Line Blocking"이라고 부른다. 100명의 플레이어가 있는 게임에서 누군가의 패킷이 손실되면 모든 플레이어가 영향을 받는다.

**UDP의 특징**
UDP는 신뢰성을 포기한다. 패킷이 도착하지 않을 수 있고, 순서대로 도착하지 않을 수도 있다. 하지만 매우 빠르다. 게임 상태는 지속적으로 업데이트되므로, 예전 프레임의 위치 정보를 받지 못해도 다음 프레임에 올바른 위치를 받으면 된다. 손실된 패킷은 잊고 계속 진행한다.

**어떤 게임이 무엇을 사용하는가?**
대부분의 빠른 액션 게임(FPS, MOBA, 격투 게임)은 UDP를 사용한다. 낮은 지연이 핵심이기 때문이다. 턴 기반 게임은 모든 입력이 정확해야 하므로 TCP를 사용한다. 혼합형도 있다. 플레이어의 위치 같은 지속적인 상태는 UDP로 보내고, 중요한 이벤트(아이템 획득, 레벨업)는 TCP로 보낸다.

이 장에서는 **UDP 기반 실시간 게임 서버**를 중심으로 설명한다. 왜냐하면 UDP가 더 어렵고, TCP는 이전 장들에서 이미 배웠기 때문이다.

## 23.2 게임 루프 구현
모든 게임 서버의 중심에는 **게임 루프**가 있다. 게임 루프는 무한히 반복되면서 다음을 수행한다.

1. 플레이어 입력을 받는다
2. 게임 상태를 업데이트한다
3. 모든 플레이어에게 업데이트된 상태를 보낸다

중요한 것은 이 루프가 **일정한 시간 간격**으로 실행되어야 한다는 것이다. 만약 어떤 틱에서는 10ms, 다음 틱에서는 30ms가 걸린다면, 플레이어들이 보는 게임 세계의 속도가 불균일해진다.

### 고정 타임스텝 구현
여기서는 고정 타임스텝을 구현한다. 이것은 현대 게임의 표준이다.

```cpp
// GameServer.h
#pragma once
#include <cstdint>
#include <vector>
#include <chrono>

// 게임 루프의 한 사이클을 나타내는 구조체
struct GameTick {
    uint32_t tickNumber;        // 이 틱의 번호 (0부터 시작)
    double tickTime;            // 이 틱이 시작된 시간 (서버 기준 초)
    float deltaTime;            // 이전 틱으로부터의 경과 시간 (항상 1/60초)
};

class SimpleGameServer {
public:
    SimpleGameServer(uint32_t tickRate = 60);
    
    // 게임 루프를 시작한다. 이 함수는 블로킹된다.
    void Run();
    
    // 게임 루프를 정지한다.
    void Stop();

private:
    uint32_t m_tickRate;        // 초당 틱 수 (기본값 60)
    uint32_t m_tickNumber;      // 현재 틱 번호
    bool m_isRunning;           // 게임 루프 실행 중인가?
    double m_accumulator;       // 누적된 시간
    
    // 한 틱을 실행한다
    void ExecuteTick(const GameTick& tick);
    
    // 플레이어 입력을 처리한다
    void ProcessInput();
    
    // 게임 상태를 업데이트한다
    void UpdateGame(float deltaTime);
    
    // 상태를 모든 클라이언트에 브로드캐스트한다
    void BroadcastState();
};
```

```cpp
// GameServer.cpp
#include "GameServer.h"
#include <thread>
#include <iostream>

SimpleGameServer::SimpleGameServer(uint32_t tickRate)
    : m_tickRate(tickRate), m_tickNumber(0), m_isRunning(false), 
      m_accumulator(0.0) {
}

void SimpleGameServer::Run() {
    m_isRunning = true;
    
    // 한 틱의 시간을 계산한다. tickRate가 60이면 deltaTime은 1/60초다.
    double tickDuration = 1.0 / m_tickRate;
    
    // 게임 시작 시간을 기록한다
    auto startTime = std::chrono::high_resolution_clock::now();
    auto lastTime = startTime;
    
    std::cout << "Game server started. Tick rate: " << m_tickRate << std::endl;
    
    while (m_isRunning) {
        auto currentTime = std::chrono::high_resolution_clock::now();
        
        // 이전 프레임 이후 경과 시간을 계산한다
        std::chrono::duration<double> elapsed = currentTime - lastTime;
        double deltaTime = elapsed.count();
        lastTime = currentTime;
        
        // 누적 시간이 한 틱 시간을 초과할 때까지 틱을 실행한다
        // 이것을 "고정 타임스텝"이라고 부른다.
        m_accumulator += deltaTime;
        
        while (m_accumulator >= tickDuration) {
            // 틱 정보를 준비한다
            auto totalElapsed = 
                std::chrono::duration<double>(currentTime - startTime);
            GameTick tick;
            tick.tickNumber = m_tickNumber;
            tick.tickTime = totalElapsed.count();
            tick.deltaTime = static_cast<float>(tickDuration);
            
            // 이번 틱을 실행한다
            ExecuteTick(tick);
            
            // 누적 시간을 감소시키고 틱 번호를 증가시킨다
            m_accumulator -= tickDuration;
            m_tickNumber++;
        }
        
        // 다음 틱까지 기다린다. 이것은 CPU를 과도하게 사용하지 않게 한다.
        std::this_thread::sleep_for(std::chrono::milliseconds(1));
    }
    
    std::cout << "Game server stopped." << std::endl;
}

void SimpleGameServer::ExecuteTick(const GameTick& tick) {
    // 이 함수는 정확히 고정된 시간 간격으로 호출된다.
    
    // 1단계: 플레이어 입력을 처리한다
    ProcessInput();
    
    // 2단계: 게임 상태를 업데이트한다
    UpdateGame(tick.deltaTime);
    
    // 3단계: 모든 플레이어에게 상태를 보낸다
    BroadcastState();
}

void SimpleGameServer::ProcessInput() {
    // 여기서는 네트워크로부터 입력을 받는다.
    // 자세한 구현은 23.3에서 다룬다.
}

void SimpleGameServer::UpdateGame(float deltaTime) {
    // 게임 로직을 업데이트한다.
    // 플레이어 이동, 충돌 감지 등을 수행한다.
}

void SimpleGameServer::BroadcastState() {
    // 업데이트된 상태를 모든 클라이언트에게 보낸다.
    // 자세한 구현은 23.4에서 다룬다.
}

void SimpleGameServer::Stop() {
    m_isRunning = false;
}
```

**이 구현의 핵심**

누적 시간(accumulator) 개념이 이 구현의 핵심이다. 실제 시간이 항상 일정하지 않으므로(다른 프로세스의 간섭, 시스템 부하 등), 우리는 실제 시간을 추적하고, 그것이 한 틱의 시간 이상이 될 때만 게임 상태를 업데이트한다. 이렇게 하면 네트워크 지연이나 시스템 부하가 있어도 게임의 물리 엔진과 모든 플레이어가 동일한 속도로 진행된다.

만약 시스템이 매우 과부하 상태라면, 여러 틱이 한 번에 실행될 수 있다. 이를 "틱 누적"이라고 부르며, 이것은 예상된 동작이다.

## 23.3 플레이어 입력 처리
게임 서버는 플레이어로부터 입력을 받아야 한다. "위로 이동하고 싶다", "공격한다" 같은 명령을 말한다.

### 입력 검증과 버퍼링
단순히 입력을 받고 곧바로 처리하면 안 된다. 두 가지 문제가 있다.

첫째, **네트워크 지연**이다. 플레이어 A의 입력이 50ms 늦게 도착한다면, 우리는 그 입력을 현재 틱에서 처리해야 하나, 아니면 대기했다가 올바른 타이밍에 처리해야 하나? 일반적으로 도착한 즉시 처리한다. 하지만 그 입력의 타임스탬프를 기록해두어야 한다.

둘째, **부정행위**다. 클라이언트는 악의적이거나 버그가 있을 수 있다. 플레이어가 초음속으로 이동하는 입력을 보낼 수도 있다. 이런 입력을 검증해야 한다.

여기서는 입력 검증과 버퍼링을 구현한다.

```cpp
// PlayerInput.h
#pragma once
#include <cstdint>
#include <vector>

// 플레이어가 보낼 수 있는 입력의 종류
enum class InputType : uint8_t {
    MoveForward = 0,
    MoveBackward = 1,
    MoveLeft = 2,
    MoveRight = 3,
    Jump = 4,
    Attack = 5
};

// 한 프레임의 플레이어 입력
struct PlayerInputFrame {
    uint32_t tickNumber;        // 이 입력이 해당하는 틱 번호
    uint32_t playerId;          // 입력을 보낸 플레이어 ID
    double clientTime;          // 클라이언트에서 기록한 시간
    std::vector<InputType> inputs;  // 이 틱에서 활성화된 입력들
};

class PlayerInputValidator {
public:
    // 플레이어의 입력이 유효한지 검사한다.
    // 부정행위 가능성이 있으면 false를 반환한다.
    static bool ValidateInput(const PlayerInputFrame& input, 
                             uint32_t currentTickNumber);
    
    // 한 틱 내에서 같은 입력이 중복되는지 검사한다
    static bool HasDuplicateInputs(const PlayerInputFrame& input);
    
    // 입력이 너무 많지는 않은지 검사한다
    // (한 틱에 10개 이상의 입력이 있으면 이상하다)
    static bool HasExcessiveInputs(const PlayerInputFrame& input);
    
    // 틱 번호가 합리적인지 검사한다
    // (미래의 틱에 대한 입력은 받으면 안 된다)
    static bool IsValidTickNumber(uint32_t inputTick, 
                                 uint32_t currentTickNumber);
};

// 플레이어 입력 버퍼
class InputBuffer {
public:
    InputBuffer(uint32_t maxSize = 1000);
    
    // 입력을 버퍼에 추가한다. 중복된 입력은 무시한다.
    void AddInput(const PlayerInputFrame& input);
    
    // 주어진 틱 번호의 입력을 가져온다. 없으면 기본값(입력 없음)을 반환한다.
    PlayerInputFrame GetInput(uint32_t tickNumber);
    
    // 주어진 틱보다 오래된 입력을 모두 제거한다.
    // (메모리 절약을 위해)
    void RemoveOldInputs(uint32_t tickNumber);
    
    // 현재 버퍼에 몇 개의 입력이 있는지 반환한다
    size_t GetBufferedInputCount() const;

private:
    struct BufferedInput {
        uint32_t tickNumber;
        PlayerInputFrame frame;
    };
    
    std::vector<BufferedInput> m_inputs;
    uint32_t m_maxSize;
    uint32_t m_lastReceivedTick;  // 가장 최근에 받은 입력의 틱 번호
};
```

```cpp
// PlayerInput.cpp
#include "PlayerInput.h"
#include <algorithm>

bool PlayerInputValidator::ValidateInput(const PlayerInputFrame& input,
                                        uint32_t currentTickNumber) {
    // 입력의 여러 측면을 검증한다
    
    if (!IsValidTickNumber(input.tickNumber, currentTickNumber)) {
        return false;  // 미래의 틱에 대한 입력이다
    }
    
    if (HasDuplicateInputs(input)) {
        return false;  // 중복된 입력이 있다
    }
    
    if (HasExcessiveInputs(input)) {
        return false;  // 입력이 너무 많다
    }
    
    return true;
}

bool PlayerInputValidator::HasDuplicateInputs(const PlayerInputFrame& input) {
    auto inputsCopy = input.inputs;
    std::sort(inputsCopy.begin(), inputsCopy.end());
    
    // 정렬 후, 인접한 같은 값이 있으면 중복이다
    auto it = std::adjacent_find(inputsCopy.begin(), inputsCopy.end());
    return it != inputsCopy.end();
}

bool PlayerInputValidator::HasExcessiveInputs(const PlayerInputFrame& input) {
    // 한 틱에 10개 이상의 입력이 있으면 부정행위로 의심한다
    return input.inputs.size() > 10;
}

bool PlayerInputValidator::IsValidTickNumber(uint32_t inputTick,
                                            uint32_t currentTickNumber) {
    // 입력의 틱 번호가 현재 틱 또는 과거여야 한다
    // 미래의 틱에 대한 입력은 거절한다
    return inputTick <= currentTickNumber;
}

InputBuffer::InputBuffer(uint32_t maxSize)
    : m_maxSize(maxSize), m_lastReceivedTick(0) {
}

void InputBuffer::AddInput(const PlayerInputFrame& input) {
    // 이미 받은 입력인지 검사한다 (중복 제거)
    auto it = std::find_if(m_inputs.begin(), m_inputs.end(),
        [&input](const BufferedInput& bi) {
            return bi.frame.tickNumber == input.tickNumber &&
                   bi.frame.playerId == input.playerId;
        });
    
    if (it != m_inputs.end()) {
        // 이미 있으면 새로운 것으로 덮어쓴다 (최신 입력이 맞을 것이라 가정)
        it->frame = input;
    } else {
        // 없으면 추가한다
        BufferedInput bi;
        bi.tickNumber = input.tickNumber;
        bi.frame = input;
        m_inputs.push_back(bi);
    }
    
    // 버퍼 크기 제한을 초과하면 가장 오래된 것을 제거한다
    if (m_inputs.size() > m_maxSize) {
        m_inputs.erase(m_inputs.begin());
    }
    
    // 마지막으로 받은 틱을 업데이트한다
    if (input.tickNumber > m_lastReceivedTick) {
        m_lastReceivedTick = input.tickNumber;
    }
}

PlayerInputFrame InputBuffer::GetInput(uint32_t tickNumber) {
    // 주어진 틱 번호의 입력을 찾는다
    auto it = std::find_if(m_inputs.begin(), m_inputs.end(),
        [tickNumber](const BufferedInput& bi) {
            return bi.tickNumber == tickNumber;
        });
    
    if (it != m_inputs.end()) {
        return it->frame;
    }
    
    // 없으면 기본값(입력 없음)을 반환한다
    PlayerInputFrame empty;
    empty.tickNumber = tickNumber;
    empty.playerId = 0;
    empty.inputs.clear();
    return empty;
}

void InputBuffer::RemoveOldInputs(uint32_t tickNumber) {
    // 주어진 틱보다 오래된 모든 입력을 제거한다
    m_inputs.erase(
        std::remove_if(m_inputs.begin(), m_inputs.end(),
            [tickNumber](const BufferedInput& bi) {
                return bi.tickNumber < tickNumber - 100;  // 100 틱 이전 데이터는 제거
            }),
        m_inputs.end()
    );
}

size_t InputBuffer::GetBufferedInputCount() const {
    return m_inputs.size();
}
```

**이 구현의 핵심**

입력 검증은 상당히 간단하지만 중요하다. 가장 자주 하는 실수는 클라이언트를 완전히 믿는 것이다. 온라인 게임에서는 모든 클라이언트 입력을 의심해야 한다. 또한 버퍼에서는 중복된 입력을 처리한다. 네트워크 상황이 좋지 않아 같은 입력이 여러 번 도착할 수 있기 때문이다.

## 23.4 상태 업데이트 브로드캐스트
게임 서버는 모든 플레이어의 위치, 상태를 계산한 후, 모든 플레이어에게 알려줘야 한다. 하지만 매 틱마다 모든 정보를 보내면 대역폭을 많이 사용한다.

### 델타 압축 기초
**델타 압축**은 변화한 정보만 보내는 기법이다. 예를 들어, 플레이어 A의 위치가 (10, 20)에서 (10, 21)로 변했다면, 변화가 아닌 X좌표(10)를 다시 보낼 이유가 없다. Y좌표 변화(+1)만 보낸다.

```cpp
// GameState.h
#pragma once
#include <cstdint>
#include <vector>
#include <cmath>

// 플레이어의 위치와 상태를 나타낸다
struct PlayerState {
    uint32_t playerId;
    float posX;
    float posY;
    uint16_t health;
    uint8_t direction;  // 0=up, 1=right, 2=down, 3=left
    bool isAlive;
};

// 플레이어 상태의 변화를 표현한다
struct PlayerStateDelta {
    uint32_t playerId;
    
    // 어떤 필드가 변했는지 표시한다 (비트 플래그)
    uint8_t changedFields;
    static constexpr uint8_t CHANGED_POS_X = 0x01;
    static constexpr uint8_t CHANGED_POS_Y = 0x02;
    static constexpr uint8_t CHANGED_HEALTH = 0x04;
    static constexpr uint8_t CHANGED_DIRECTION = 0x08;
    static constexpr uint8_t CHANGED_IS_ALIVE = 0x10;
    
    // 변화한 필드들의 값
    float posX;
    float posY;
    uint16_t health;
    uint8_t direction;
    bool isAlive;
};

// 한 틱의 모든 플레이어 상태 변화
struct GameStateSnapshot {
    uint32_t tickNumber;
    std::vector<PlayerStateDelta> playerDeltas;
};

// 게임 상태 관리자
class GameStateManager {
public:
    // 플레이어의 새로운 상태를 설정한다
    void SetPlayerState(const PlayerState& state);
    
    // 현재 상태를 가져온다
    PlayerState GetPlayerState(uint32_t playerId) const;
    
    // 이전 상태와 비교해서 변화분만 추출한다
    // 첫 번째 호출 후부터는 델타만 추출된다
    GameStateSnapshot GenerateDeltaSnapshot();
    
    // 모든 플레이어 상태를 초기화한다
    void Reset();
    
    // 현재 몇 명의 플레이어가 게임 중인지 반환한다
    size_t GetActivePlayerCount() const;

private:
    // 현재 상태와 이전 상태를 비교해서 변화를 감지한다
    static PlayerStateDelta ComputeDelta(const PlayerState& current,
                                         const PlayerState& previous);
    
    // 두 좌표가 유의미하게 다른지 검사한다 (float 비교는 부동소수점 오차 때문에 조심)
    static bool HasPositionChanged(float old_x, float old_y, 
                                   float new_x, float new_y);
    
    std::vector<PlayerState> m_currentState;      // 현재 상태
    std::vector<PlayerState> m_previousState;     // 이전 틱의 상태
    uint32_t m_snapshotCount;                    // 생성된 스냅샷 수
};
```

```cpp
// GameState.cpp
#include "GameState.h"
#include <algorithm>

void GameStateManager::SetPlayerState(const PlayerState& state) {
    // 같은 ID의 플레이어 상태가 있으면 업데이트하고, 없으면 추가한다
    auto it = std::find_if(m_currentState.begin(), m_currentState.end(),
        [state](const PlayerState& ps) { return ps.playerId == state.playerId; });
    
    if (it != m_currentState.end()) {
        *it = state;
    } else {
        m_currentState.push_back(state);
    }
}

PlayerState GameStateManager::GetPlayerState(uint32_t playerId) const {
    auto it = std::find_if(m_currentState.begin(), m_currentState.end(),
        [playerId](const PlayerState& ps) { return ps.playerId == playerId; });
    
    if (it != m_currentState.end()) {
        return *it;
    }
    
    // 없으면 기본값을 반환한다
    PlayerState empty = {};
    empty.playerId = playerId;
    return empty;
}

GameStateSnapshot GameStateManager::GenerateDeltaSnapshot() {
    GameStateSnapshot snapshot;
    snapshot.tickNumber = m_snapshotCount++;
    
    // 현재의 각 플레이어 상태와 이전 상태를 비교해서 델타를 생성한다
    for (const auto& currentState : m_currentState) {
        PlayerState previousState = {};
        
        // 이전 상태를 찾는다
        auto it = std::find_if(m_previousState.begin(), m_previousState.end(),
            [&currentState](const PlayerState& ps) {
                return ps.playerId == currentState.playerId;
            });
        
        if (it != m_previousState.end()) {
            previousState = *it;
        }
        
        // 변화를 계산한다
        PlayerStateDelta delta = ComputeDelta(currentState, previousState);
        
        // 변화가 있으면 스냅샷에 추가한다
        if (delta.changedFields != 0) {
            snapshot.playerDeltas.push_back(delta);
        }
    }
    
    // 현재 상태를 이전 상태로 저장한다 (다음 틱을 위해)
    m_previousState = m_currentState;
    
    return snapshot;
}

PlayerStateDelta GameStateManager::ComputeDelta(const PlayerState& current,
                                                const PlayerState& previous) {
    PlayerStateDelta delta;
    delta.playerId = current.playerId;
    delta.changedFields = 0;
    
    // 각 필드를 비교한다
    if (HasPositionChanged(previous.posX, previous.posY,
                          current.posX, current.posY)) {
        delta.changedFields |= PlayerStateDelta::CHANGED_POS_X;
        delta.changedFields |= PlayerStateDelta::CHANGED_POS_Y;
        delta.posX = current.posX;
        delta.posY = current.posY;
    }
    
    if (current.health != previous.health) {
        delta.changedFields |= PlayerStateDelta::CHANGED_HEALTH;
        delta.health = current.health;
    }
    
    if (current.direction != previous.direction) {
        delta.changedFields |= PlayerStateDelta::CHANGED_DIRECTION;
        delta.direction = current.direction;
    }
    
    if (current.isAlive != previous.isAlive) {
        delta.changedFields |= PlayerStateDelta::CHANGED_IS_ALIVE;
        delta.isAlive = current.isAlive;
    }
    
    return delta;
}

bool GameStateManager::HasPositionChanged(float old_x, float old_y,
                                          float new_x, float new_y) {
    // float 비교는 부동소수점 오차 때문에 조심해야 한다
    // 작은 변화는 무시하고 큰 변화만 전송한다
    const float epsilon = 0.01f;  // 1cm 이하의 변화는 무시
    
    float dx = new_x - old_x;
    float dy = new_y - old_y;
    
    return (std::abs(dx) > epsilon) || (std::abs(dy) > epsilon);
}

void GameStateManager::Reset() {
    m_currentState.clear();
    m_previousState.clear();
    m_snapshotCount = 0;
}

size_t GameStateManager::GetActivePlayerCount() const {
    return m_currentState.size();
}
```

**이 구현의 핵심**

델타 압축은 네트워크 대역폭을 크게 절약한다. 예를 들어, 플레이어가 움직이지 않고 있다면 위치 정보를 보낼 필요가 없다. 또한 비트 플래그를 사용해서 어떤 필드가 변했는지 간단히 표현한다. 클라이언트는 이 플래그를 읽고 변한 필드만 언팩한다. 부동소수점 위치 비교에 epsilon을 사용하는 것도 중요하다. 매우 작은 부동소수점 오차로 인해 불필요한 업데이트가 발생하는 것을 막는다.

## 23.5 지연 보상 기법
온라인 게임에서 가장 어려운 문제 중 하나는 **네트워크 지연**이다. 플레이어가 공격 버튼을 누르는 순간과 서버가 그 입력을 받는 순간 사이에 100ms의 지연이 있을 수 있다. 이 시간 동안 상대방이 이미 움직여 있을 것이다.

게임은 이 지연을 여러 방식으로 보상한다. 그중 가장 기본적인 것은 **타임스탐프 기반 처리**이다. 클라이언트가 입력할 때의 시간을 함께 보내고, 서버는 그 시간 기준으로 게임 상태를 보간한다.

```cpp
// TimestampedInput.h
#pragma once
#include <cstdint>
#include <vector>

// 타임스탐프가 포함된 입력
struct TimestampedInput {
    uint32_t playerId;
    double clientTime;      // 클라이언트에서 입력한 시간 (클라이언트 기준)
    double serverTime;      // 서버가 받은 시간 (서버 기준)
    float inputX;           // 입력 축 X (-1.0 ~ 1.0)
    float inputY;           // 입력 축 Y (-1.0 ~ 1.0)
};

// 플레이어의 움직임을 보간한다
class MotionInterpolator {
public:
    // 이전 위치와 현재 위치를 알고 있을 때,
    // 두 시점 사이의 임의의 시간에서의 위치를 계산한다
    static void InterpolatePosition(float prevX, float prevY,
                                    float currX, float currY,
                                    double prevTime, double currTime,
                                    double targetTime,
                                    float& outX, float& outY);
    
    // 플레이어의 움직임을 외삽(extrapolate)한다.
    // 즉, 현재 속도를 기반으로 미래 위치를 예측한다.
    static void ExtrapolatePosition(float currX, float currY,
                                    float velX, float velY,
                                    double currTime, double targetTime,
                                    float& outX, float& outY);
};

// 지연 보상을 관리한다
class LatencyCompensator {
public:
    LatencyCompensator();
    
    // 입력을 받았을 때의 네트워크 지연을 기록한다
    void RecordLatency(double clientTime, double serverTime);
    
    // 현재 예상되는 지연을 반환한다 (움직이는 평균)
    double GetEstimatedLatency() const;
    
    // 클라이언트 시간을 서버 시간으로 변환한다
    double ConvertClientTimeToServerTime(double clientTime) const;

private:
    std::vector<double> m_latencies;
    double m_averageLatency;
    static constexpr size_t LATENCY_HISTORY_SIZE = 30;  // 30개의 최근 지연을 추적
};

// 인터폴레이션 버퍼
// 플레이어들의 위치를 일정 기간 보관했다가, 과거 시점의 위치를 조회할 수 있다
class InterpolationBuffer {
public:
    struct StoredPosition {
        double time;
        float x, y;
    };
    
    InterpolationBuffer(size_t maxSize = 100);
    
    // 플레이어의 위치를 저장한다
    void StorePosition(uint32_t playerId, double time, float x, float y);
    
    // 지정된 과거 시점에서의 위치를 보간해서 구한다
    bool GetInterpolatedPosition(uint32_t playerId, double targetTime,
                                 float& outX, float& outY);

private:
    struct PlayerBuffer {
        uint32_t playerId;
        std::vector<StoredPosition> positions;
    };
    
    std::vector<PlayerBuffer> m_playerBuffers;
    size_t m_maxSize;
};
```

```cpp
// TimestampedInput.cpp
#include "TimestampedInput.h"
#include <cmath>
#include <algorithm>

void MotionInterpolator::InterpolatePosition(float prevX, float prevY,
                                             float currX, float currY,
                                             double prevTime, double currTime,
                                             double targetTime,
                                             float& outX, float& outY) {
    // 선형 보간(linear interpolation)을 사용한다
    // 두 점 사이의 거리와 시간을 이용해서 중간 위치를 계산한다
    
    if (currTime == prevTime) {
        // 시간이 같으면 현재 위치를 반환한다
        outX = currX;
        outY = currY;
        return;
    }
    
    // 0과 1 사이의 보간 계수를 계산한다
    // t=0이면 이전 위치, t=1이면 현재 위치를 의미한다
    double t = (targetTime - prevTime) / (currTime - prevTime);
    
    // t가 범위를 벗어나면 클램핑한다
    t = std::max(0.0, std::min(1.0, t));
    
    // 선형 보간 공식: P(t) = P0 + t * (P1 - P0)
    outX = prevX + static_cast<float>(t) * (currX - prevX);
    outY = prevY + static_cast<float>(t) * (currY - prevY);
}

void MotionInterpolator::ExtrapolatePosition(float currX, float currY,
                                             float velX, float velY,
                                             double currTime, double targetTime,
                                             float& outX, float& outY) {
    // 현재 속도를 기반으로 미래 위치를 예측한다
    // P_future = P_current + v * deltaTime
    
    double deltaTime = targetTime - currTime;
    
    // deltaTime이 너무 크면 외삽을 제한한다 (예측이 너무 부정확해진다)
    // 최대 200ms까지만 외삽한다
    deltaTime = std::min(deltaTime, 0.2);
    
    outX = currX + velX * static_cast<float>(deltaTime);
    outY = currY + velY * static_cast<float>(deltaTime);
}

LatencyCompensator::LatencyCompensator() : m_averageLatency(0.0) {
}

void LatencyCompensator::RecordLatency(double clientTime, double serverTime) {
    // 이 호출은 일반적으로 클라이언트가 현재 서버 시간을 받았을 때 사용된다
    // 클라이언트 시계와 서버 시계의 시차를 계산한다
    
    double latency = serverTime - clientTime;
    
    // 네거티브 지연은 말이 안 되므로 무시한다
    if (latency < 0) {
        return;
    }
    
    // 지연 기록을 추가한다
    m_latencies.push_back(latency);
    
    // 히스토리 크기를 초과하면 가장 오래된 것을 제거한다
    if (m_latencies.size() > LATENCY_HISTORY_SIZE) {
        m_latencies.erase(m_latencies.begin());
    }
    
    // 평균 지연을 다시 계산한다
    m_averageLatency = 0.0;
    for (double lat : m_latencies) {
        m_averageLatency += lat;
    }
    m_averageLatency /= m_latencies.size();
}

double LatencyCompensator::GetEstimatedLatency() const {
    return m_averageLatency;
}

double LatencyCompensator::ConvertClientTimeToServerTime(double clientTime) const {
    return clientTime + m_averageLatency;
}

InterpolationBuffer::InterpolationBuffer(size_t maxSize) : m_maxSize(maxSize) {
}

void InterpolationBuffer::StorePosition(uint32_t playerId, double time,
                                       float x, float y) {
    // 플레이어의 버퍼를 찾는다
    auto it = std::find_if(m_playerBuffers.begin(), m_playerBuffers.end(),
        [playerId](const PlayerBuffer& pb) {
            return pb.playerId == playerId;
        });
    
    if (it == m_playerBuffers.end()) {
        // 없으면 새로 만든다
        PlayerBuffer pb;
        pb.playerId = playerId;
        m_playerBuffers.push_back(pb);
        it = m_playerBuffers.end() - 1;
    }
    
    // 위치를 저장한다
    StoredPosition pos;
    pos.time = time;
    pos.x = x;
    pos.y = y;
    it->positions.push_back(pos);
    
    // 버퍼 크기를 초과하면 가장 오래된 것을 제거한다
    if (it->positions.size() > m_maxSize) {
        it->positions.erase(it->positions.begin());
    }
}

bool InterpolationBuffer::GetInterpolatedPosition(uint32_t playerId,
                                                 double targetTime,
                                                 float& outX, float& outY) {
    // 플레이어의 버퍼를 찾는다
    auto it = std::find_if(m_playerBuffers.begin(), m_playerBuffers.end(),
        [playerId](const PlayerBuffer& pb) {
            return pb.playerId == playerId;
        });
    
    if (it == m_playerBuffers.end() || it->positions.empty()) {
        return false;
    }
    
    auto& positions = it->positions;
    
    // 정확히 같은 시간의 위치가 있는지 찾는다
    auto exactMatch = std::find_if(positions.begin(), positions.end(),
        [targetTime](const StoredPosition& sp) {
            return std::abs(sp.time - targetTime) < 0.0001;
        });
    
    if (exactMatch != positions.end()) {
        outX = exactMatch->x;
        outY = exactMatch->y;
        return true;
    }
    
    // targetTime보다 이전과 이후의 위치를 찾는다
    StoredPosition* prevPos = nullptr;
    StoredPosition* nextPos = nullptr;
    
    for (auto& pos : positions) {
        if (pos.time <= targetTime) {
            prevPos = &pos;
        }
        if (pos.time >= targetTime && nextPos == nullptr) {
            nextPos = &pos;
        }
    }
    
    if (prevPos != nullptr && nextPos != nullptr && prevPos != nextPos) {
        // 두 위치 사이를 보간한다
        MotionInterpolator::InterpolatePosition(
            prevPos->x, prevPos->y,
            nextPos->x, nextPos->y,
            prevPos->time, nextPos->time,
            targetTime,
            outX, outY
        );
        return true;
    }
    
    if (prevPos != nullptr) {
        outX = prevPos->x;
        outY = prevPos->y;
        return true;
    }
    
    return false;
}
```

**이 구현의 핵심**

타임스탐프 기반 지연 보상은 두 가지를 한다. 첫째, 지연을 측정한다. 플레이어의 입력에 타임스탐프를 붙이고, 서버는 그 시간과 현재 시간의 차이를 계산해서 지연을 추정한다. 둘째, 과거 시점에서의 게임 상태를 재구성한다. 플레이어가 공격할 때, 서버는 과거(지연을 뺀) 시점의 상대방 위치를 조회해서 명중 판정을 한다. 이렇게 하면 플레이어는 지연을 느끼지 못한다.

## 23.6 종합 예제: 간단한 액션 게임 서버
이제 여기까지 배운 모든 개념을 통합해서 실제로 작동하는 게임 서버를 만들어본다. 이것은 간단한 스타일의 액션 게임이다. 플레이어들이 맵 위를 이동하고, 서로를 공격하고, 충돌을 감지하는 게임이다.

**설계 개요**

이 서버는 다음과 같이 작동한다:
1. 고정 60 틱/초의 게임 루프를 실행한다
2. 플레이어 입력을 받고 검증한다
3. 물리 엔진을 사용해서 위치를 업데이트한다
4. 충돌 감지를 수행한다
5. 델타 압축된 상태를 모든 클라이언트에게 브로드캐스트한다

이 예제는 약 200줄이지만, 매우 기본적인 예제다. 실전에서는 이것을 확장해야 한다.

```cpp
// SimpleActionGameServer.h
#pragma once
#include <cstdint>
#include <vector>
#include <cmath>
#include <algorithm>
#include <chrono>

// 게임 월드의 설정
namespace GameConfig {
    constexpr uint32_t TICK_RATE = 60;
    constexpr float MAP_WIDTH = 1000.0f;
    constexpr float MAP_HEIGHT = 1000.0f;
    constexpr float PLAYER_RADIUS = 5.0f;
    constexpr float PLAYER_MAX_SPEED = 200.0f;  // 픽셀/초
    constexpr uint16_t PLAYER_MAX_HEALTH = 100;
}

// 플레이어가 할 수 있는 입력
enum class PlayerAction : uint8_t {
    MoveUp = 0,
    MoveDown = 1,
    MoveLeft = 2,
    MoveRight = 3,
    Attack = 4
};

// 플레이어의 게임 상태
struct Player {
    uint32_t id;
    float posX, posY;           // 위치 (픽셀)
    float velX, velY;           // 속도 (픽셀/초)
    uint16_t health;
    bool isAlive;
    double lastActionTime;      // 마지막 행동 시간
    uint32_t lastAttackTick;    // 마지막 공격 틱 번호
};

// 플레이어의 상태 변화
struct PlayerDelta {
    uint32_t playerId;
    uint8_t changedFields;  // 비트 마스크
    float posX, posY;
    uint16_t health;
    bool isAlive;
    
    static constexpr uint8_t CHANGED_POS = 0x01;
    static constexpr uint8_t CHANGED_HEALTH = 0x02;
    static constexpr uint8_t CHANGED_IS_ALIVE = 0x04;
};

// 게임 상태 스냅샷
struct GameSnapshot {
    uint32_t tickNumber;
    std::vector<PlayerDelta> playerDeltas;
};

// 입력 버퍼 (지난 10틱의 입력을 보관)
struct InputFrame {
    std::vector<PlayerAction> actions;  // 이 프레임의 입력들
};

// 간단한 액션 게임 서버
class SimpleActionGameServer {
public:
    SimpleActionGameServer();

    void Stop();

    // 최근 게임 스냅샷을 가져온다 (클라이언트에게 보낼 데이터)
    GameSnapshot GetLatestSnapshot() const;
    
    // 현재 플레이어 상태를 가져온다 (디버깅용)
    Player GetPlayer(uint32_t playerId) const;
    
    // 현재 틱 번호를 가져온다
    uint32_t GetCurrentTick() const;

private:
    // 게임 루프의 한 사이클을 실행한다
    void UpdateTick(float deltaTime);
    
    // 플레이어 입력을 처리한다
    void ProcessInput(float deltaTime);
    
    // 플레이어 이동을 업데이트한다
    void UpdateMovement(float deltaTime);
    
    // 공격을 처리한다
    void ProcessAttacks();
    
    // 충돌을 감지한다
    void DetectCollisions();
    
    // 게임 상태를 기반으로 델타 스냅샷을 생성한다
    GameSnapshot GenerateSnapshot();
    
    // 좌표를 맵 경계 내로 클램핑한다
    void ClampToMap(float& x, float& y);
    
    // 두 플레이어 사이의 거리를 계산한다
    float Distance(const Player& a, const Player& b) const;
    
    std::vector<Player> m_players;
    std::vector<Player> m_previousPlayers;  // 델타 계산용
    std::vector<InputFrame> m_inputBuffers;  // 플레이어별 입력 버퍼
    uint32_t m_tickNumber;
    bool m_isRunning;

    GameSnapshot m_lastSnapshot;
```

```cpp
// SimpleActionGameServer.cpp
#include "SimpleActionGameServer.h"
#include <iostream>
#include <thread>

SimpleActionGameServer::SimpleActionGameServer()
    : m_tickNumber(0), m_isRunning(false), m_accumulator(0.0) {
}

void SimpleActionGameServer::Start() {
    m_isRunning = true;
    m_accumulator = 0.0;
    
    double tickDuration = 1.0 / GameConfig::TICK_RATE;
    auto startTime = std::chrono::high_resolution_clock::now();
    auto lastTime = startTime;
    
    std::cout << "[GameServer] Starting with " << m_players.size() 
              << " players at " << GameConfig::TICK_RATE << " ticks/sec" << std::endl;
    
    while (m_isRunning) {
        auto currentTime = std::chrono::high_resolution_clock::now();
        std::chrono::duration<double> elapsed = currentTime - lastTime;
        double deltaTime = elapsed.count();
        lastTime = currentTime;
        
        m_accumulator += deltaTime;
        
        while (m_accumulator >= tickDuration && m_isRunning) {
            UpdateTick(static_cast<float>(tickDuration));
            m_accumulator -= tickDuration;
            m_tickNumber++;
        }
        
        std::this_thread::sleep_for(std::chrono::milliseconds(1));
    }
    
    std::cout << "[GameServer] Stopped at tick " << m_tickNumber << std::endl;
}

void SimpleActionGameServer::Stop() {
    m_isRunning = false;
}

void SimpleActionGameServer::AddPlayer(uint32_t playerId, float startX, float startY) {
    Player player;
    player.id = playerId;
    player.posX = startX;
    player.posY = startY;
    player.velX = 0.0f;
    player.velY = 0.0f;
    player.health = GameConfig::PLAYER_MAX_HEALTH;
    player.isAlive = true;
    player.lastActionTime = 0.0;
    player.lastAttackTick = 0;
    
    m_players.push_back(player);
    
    // 입력 버퍼 크기를 플레이어 수에 맞춰 조정한다
    m_inputBuffers.resize(m_players.size());
    
    std::cout << "[GameServer] Player " << playerId << " joined at (" 
              << startX << ", " << startY << ")" << std::endl;
}

void SimpleActionGameServer::HandlePlayerInput(uint32_t playerId,
                                               const std::vector<PlayerAction>& actions) {
    // 플레이어를 찾는다
    auto it = std::find_if(m_players.begin(), m_players.end(),
        [playerId](const Player& p) { return p.id == playerId; });
    
    if (it == m_players.end()) {
        return;  // 플레이어가 없음
    }
    
    size_t playerIndex = std::distance(m_players.begin(), it);
    
    // 입력을 저장한다 (다음 틱에서 처리됨)
    if (playerIndex < m_inputBuffers.size()) {
        m_inputBuffers[playerIndex].actions = actions;
    }
}

void SimpleActionGameServer::UpdateTick(float deltaTime) {
    // 게임 로직의 순서:
    // 1. 입력 처리 (플레이어 속도 설정)
    // 2. 물리 업데이트 (위치 계산)
    // 3. 공격 처리 (피해 계산)
    // 4. 충돌 감지
    // 5. 스냅샷 생성 (델타)
    
    ProcessInput(deltaTime);
    UpdateMovement(deltaTime);
    ProcessAttacks();
    DetectCollisions();
    m_lastSnapshot = GenerateSnapshot();
}

void SimpleActionGameServer::ProcessInput(float deltaTime) {
    for (size_t i = 0; i < m_players.size(); ++i) {
        // 현재 입력을 가져온다
        const auto& inputFrame = m_inputBuffers[i];
        
        // 속도를 초기화한다
        m_players[i].velX = 0.0f;
        m_players[i].velY = 0.0f;
        
        // 입력에 따라 속도를 설정한다
        for (const auto& action : inputFrame.actions) {
            switch (action) {
                case PlayerAction::MoveUp:
                    m_players[i].velY -= GameConfig::PLAYER_MAX_SPEED;
                    break;
                case PlayerAction::MoveDown:
                    m_players[i].velY += GameConfig::PLAYER_MAX_SPEED;
                    break;
                case PlayerAction::MoveLeft:
                    m_players[i].velX -= GameConfig::PLAYER_MAX_SPEED;
                    break;
                case PlayerAction::MoveRight:
                    m_players[i].velX += GameConfig::PLAYER_MAX_SPEED;
                    break;
                case PlayerAction::Attack:
                    // Attack은 ProcessAttacks에서 처리된다
                    break;
            }
        }
        
        // 속도를 정규화한다 (대각선 이동이 더 빨라지지 않도록)
        float speedSquared = m_players[i].velX * m_players[i].velX + 
                             m_players[i].velY * m_players[i].velY;
        if (speedSquared > GameConfig::PLAYER_MAX_SPEED * GameConfig::PLAYER_MAX_SPEED) {
            float speed = std::sqrt(speedSquared);
            float scale = GameConfig::PLAYER_MAX_SPEED / speed;
            m_players[i].velX *= scale;
            m_players[i].velY *= scale;
        }
    }
}

void SimpleActionGameServer::UpdateMovement(float deltaTime) {
    for (auto& player : m_players) {
        if (!player.isAlive) continue;
        
        // 위치를 속도 기반으로 업데이트한다
        player.posX += player.velX * deltaTime;
        player.posY += player.velY * deltaTime;
        
        // 맵 경계를 벗어나지 않도록 한다
        ClampToMap(player.posX, player.posY);
    }
}

void SimpleActionGameServer::ProcessAttacks() {
    // 공격 행동을 감지하고 피해를 준다
    for (size_t i = 0; i < m_players.size(); ++i) {
        const auto& inputFrame = m_inputBuffers[i];
        
        // Attack 입력이 있는지 확인한다
        bool hasAttackInput = std::find(inputFrame.actions.begin(),
                                        inputFrame.actions.end(),
                                        PlayerAction::Attack) != inputFrame.actions.end();
        
        if (!hasAttackInput || !m_players[i].isAlive) {
            continue;
        }
        
        // 공격 쿨다운 체크 (매 틱마다 공격하지 못하도록)
        if (m_tickNumber - m_players[i].lastAttackTick < 10) {  // 10틱 = 약 167ms
            continue;
        }
        
        m_players[i].lastAttackTick = m_tickNumber;
        
        // 주변 플레이어에게 피해를 준다
        constexpr float ATTACK_RANGE = 50.0f;
        constexpr uint16_t ATTACK_DAMAGE = 10;
        
        for (size_t j = 0; j < m_players.size(); ++j) {
            if (i == j || !m_players[j].isAlive) continue;
            
            float dist = Distance(m_players[i], m_players[j]);
            if (dist <= ATTACK_RANGE) {
                m_players[j].health -= ATTACK_DAMAGE;
                
                // 체력이 0 이하이면 플레이어가 죽는다
                if (m_players[j].health <= 0) {
                    m_players[j].isAlive = false;
                    m_players[j].health = 0;
                }
            }
        }
    }
}

void SimpleActionGameServer::DetectCollisions() {
    // 단순 원형 충돌 감지
    constexpr float COLLISION_PUSH_FACTOR = 0.5f;
    
    for (size_t i = 0; i < m_players.size(); ++i) {
        if (!m_players[i].isAlive) continue;
        
        for (size_t j = i + 1; j < m_players.size(); ++j) {
            if (!m_players[j].isAlive) continue;
            
            float dist = Distance(m_players[i], m_players[j]);
            float minDist = GameConfig::PLAYER_RADIUS * 2;
            
            if (dist < minDist && dist > 0) {
                // 두 플레이어가 겹쳤으므로 밀어낸다
                float pushX = (m_players[j].posX - m_players[i].posX) / dist * COLLISION_PUSH_FACTOR;
                float pushY = (m_players[j].posY - m_players[i].posY) / dist * COLLISION_PUSH_FACTOR;
                
                m_players[i].posX -= pushX;
                m_players[i].posY -= pushY;
                m_players[j].posX += pushX;
                m_players[j].posY += pushY;
                
                // 맵 경계를 벗어나지 않도록 한다
                ClampToMap(m_players[i].posX, m_players[i].posY);
                ClampToMap(m_players[j].posX, m_players[j].posY);
            }
        }
    }
}

GameSnapshot SimpleActionGameServer::GenerateSnapshot() {
    GameSnapshot snapshot;
    snapshot.tickNumber = m_tickNumber;
    
    // 현재 상태와 이전 상태를 비교해서 변화분만 추출한다
    for (const auto& currentPlayer : m_players) {
        // 이전 상태에서 같은 ID의 플레이어를 찾는다
        auto prevIt = std::find_if(m_previousPlayers.begin(), m_previousPlayers.end(),
            [&currentPlayer](const Player& p) { return p.id == currentPlayer.id; });
        
        Player previousPlayer = {};
        if (prevIt != m_previousPlayers.end()) {
            previousPlayer = *prevIt;
        }
        
        // 변화를 감지한다
        PlayerDelta delta;
        delta.playerId = currentPlayer.id;
        delta.changedFields = 0;
        
        // 위치 변화
        const float POS_EPSILON = 0.1f;
        if (std::abs(currentPlayer.posX - previousPlayer.posX) > POS_EPSILON ||
            std::abs(currentPlayer.posY - previousPlayer.posY) > POS_EPSILON) {
            delta.changedFields |= PlayerDelta::CHANGED_POS;
            delta.posX = currentPlayer.posX;
            delta.posY = currentPlayer.posY;
        }
        
        // 체력 변화
        if (currentPlayer.health != previousPlayer.health) {
            delta.changedFields |= PlayerDelta::CHANGED_HEALTH;
            delta.health = currentPlayer.health;
        }
        
        // 생존 상태 변화
        if (currentPlayer.isAlive != previousPlayer.isAlive) {
            delta.changedFields |= PlayerDelta::CHANGED_IS_ALIVE;
            delta.isAlive = currentPlayer.isAlive;
        }
        
        // 변화가 있으면 스냅샷에 포함한다
        if (delta.changedFields != 0) {
            snapshot.playerDeltas.push_back(delta);
        }
    }
    
    // 현재 상태를 이전 상태로 저장한다 (다음 틱을 위해)
    m_previousPlayers = m_players;
    
    return snapshot;
}

void SimpleActionGameServer::ClampToMap(float& x, float& y) {
    x = std::max(GameConfig::PLAYER_RADIUS, 
                 std::min(x, GameConfig::MAP_WIDTH - GameConfig::PLAYER_RADIUS));
    y = std::max(GameConfig::PLAYER_RADIUS,
                 std::min(y, GameConfig::MAP_HEIGHT - GameConfig::PLAYER_RADIUS));
}

float SimpleActionGameServer::Distance(const Player& a, const Player& b) const {
    float dx = a.posX - b.posX;
    float dy = a.posY - b.posY;
    return std::sqrt(dx * dx + dy * dy);
}

GameSnapshot SimpleActionGameServer::GetLatestSnapshot() const {
    return m_lastSnapshot;
}

Player SimpleActionGameServer::GetPlayer(uint32_t playerId) const {
    auto it = std::find_if(m_players.begin(), m_players.end(),
        [playerId](const Player& p) { return p.id == playerId; });
    
    if (it != m_players.end()) {
        return *it;
    }
    
    return Player{};
}

uint32_t SimpleActionGameServer::GetCurrentTick() const {
    return m_tickNumber;
}
```

**이 구현의 설계 결정**

이 서버의 설계에는 여러 중요한 결정사항이 있다.

첫째, **고정 60틱 시스템**을 사용했다. 현대 게임의 표준은 60틱이다. 더 높은 틱레이트(120, 144)는 더 낮은 지연을 제공하지만 더 많은 서버 리소스가 필요하다.

둘째, **입력 버퍼**를 사용해서 입력을 모아둔 다음 한 틱에서 처리한다. 이렇게 하면 네트워크 지연의 영향을 줄일 수 있다.

셋째, **속도 정규화**를 했다. 플레이어가 위아래 화살표를 동시에 누르고 좌우 화살표도 동시에 누르면 대각선으로 이동할 때 더 빨라지는 문제가 생긴다. 이를 방지하기 위해 속도를 최대값으로 정규화한다.

넷째, **델타 압축**을 사용했다. 변화하지 않은 플레이어 상태는 스냅샷에 포함하지 않는다. 이 때문에 네트워크 대역폭을 크게 절약할 수 있다.

다섯째, **공격 쿨다운**을 구현했다. 플레이어가 매 틱마다 공격하는 것을 막기 위해 최소 10틱 간격으로만 공격할 수 있도록 했다.

**주요 부분 상세 해설**

게임 루프의 UpdateTick 함수는 매우 중요하다. 이 함수는 다음 순서를 반드시 지켜야 한다:
1. ProcessInput: 플레이어 입력으로부터 속도를 계산한다
2. UpdateMovement: 속도에 따라 위치를 계산한다
3. ProcessAttacks: 공격이 일어났는지 확인하고 피해를 준다
4. DetectCollisions: 플레이어 간 충돌을 처리한다
5. GenerateSnapshot: 변화분만 추출해서 클라이언트에게 보낼 데이터를 만든다

이 순서가 바뀌면 게임이 일관성 없게 작동한다. 예를 들어, 충돌 감지 후에 이동을 하면, 겹친 플레이어를 밀어낼 수 없다.

**개선 방안과 확장 가능성**

이 구현은 매우 기본적이며, 실제 게임 서버에는 많은 개선이 필요하다:

1. **더 정교한 물리**: 현재는 매우 기본적인 충돌 반응만 구현했다. 실제 게임에서는 속도 유지, 마찰, 중력 등이 필요하다.

2. **영역 기반 브로드캐스트**: 현재는 모든 플레이어에게 모든 플레이어의 상태를 보낸다. 큰 게임에서는 플레이어 근처의 다른 플레이어 정보만 보내야 한다.

3. **더 나은 공격 시스템**: 현재는 단순 원형 범위를 사용하지만, 실제 게임에서는 애니메이션 타이밍, 방향, 더 복잡한 히트박스가 필요하다.

4. **게임 이벤트**: 플레이어 죽음, 아이템 획득, 레벨업 같은 이벤트를 구분해서 처리해야 한다.

5. **부정행위 방지**: 클라이언트의 입력을 더 엄격히 검증해야 한다. 예를 들어, 플레이어가 1초 안에 이동할 수 없는 거리를 이동했다면 거절해야 한다.

6. **네트워크 최적화**: 중요한 정보(생사, 공격)와 덜 중요한 정보(위치)를 다르게 처리할 수 있다. 생사 정보는 신뢰성 있게(TCP), 위치는 빠르게(UDP) 보낼 수 있다.

7. **관찰자(Observer) 패턴**: 게임 이벤트(공격, 죽음)를 구독하는 시스템을 만들어서 다양한 로직이 게임 이벤트에 반응할 수 있도록 한다.

이러한 개선사항들을 추가하면 더 완성도 높은 실제 게임 서버를 만들 수 있다.

## 정리
이 장에서는 실시간 게임 서버의 핵심 개념들을 배웠다. TCP와 UDP의 선택부터 시작해서, 고정 타임스텝 게임 루프, 입력 검증과 버퍼링, 델타 압축을 통한 대역폭 절약, 지연 보상까지 다양한 주제를 다루었다.

게임 서버는 다른 서버 애플리케이션과 달리, 시간에 매우 민감하고, 일관성이 중요하다. 여기서 배운 기법들은 대부분 이런 특성에서 비롯된 것이다. 다음 장에서는 이런 게임 서버의 성능을 어떻게 최적화하는지 배울 것이다.
