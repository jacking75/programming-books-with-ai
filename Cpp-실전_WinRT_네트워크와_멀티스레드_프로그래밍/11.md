# Chapter 11. UDP 소켓 - DatagramSocket

## 11.1 DatagramSocket API 설명

### UDP의 특성과 사용 시나리오
UDP(User Datagram Protocol)는 TCP와 달리 연결을 설정하지 않고 메시지를 전송하는 비연결형 프로토콜이다. WinRT의 DatagramSocket은 이러한 UDP 통신을 구현하기 위한 핵심 클래스다.  
  
UDP의 주요 특성을 먼저 이해해야 한다. 첫째, UDP는 전송 순서를 보장하지 않는다. 패킷이 도착하지 않을 수도 있고, 도착 순서가 바뀔 수도 있다. 둘째, UDP는 낮은 지연 시간을 제공한다. 연결 설정 과정이 없으므로 즉시 데이터를 전송할 수 있다. 셋째, UDP는 원-투-매니(one-to-many) 통신을 지원한다. 브로드캐스트나 멀티캐스트를 통해 여러 수신자에게 동시에 메시지를 보낼 수 있다.

이러한 특성 때문에 UDP는 특정 상황에 매우 적합하다. 실시간 게임의 플레이어 위치 동기화, 온라인 비디오 스트리밍, DNS 쿼리, 네트워크 모니터링 도구 등에서 UDP를 사용한다. 이들은 모두 약간의 패킷 손실이나 순서 변경보다 낮은 지연 시간이 더 중요한 분야다.

DatagramSocket의 기본 구조를 보자. 이 클래스는 UDP 소켓의 생성, 바인딩, 메시지 송수신, 멀티캐스트 관리 등의 기능을 제공한다. TCP의 StreamSocket과는 달리, DatagramSocket은 연결 개념이 없다. 로컬 포트에 바인딩하고 바로 송수신을 시작할 수 있다.

DatagramSocket을 사용하는 일반적인 흐름은 다음과 같다. 첫 번째 단계에서는 DatagramSocket 인스턴스를 생성한다. 두 번째 단계에서는 로컬 포트에 바인딩한다. 세 번째 단계에서는 MessageReceived 이벤트를 등록하여 수신할 메시지를 처리한다. 네 번째 단계에서는 특정 주소로 메시지를 전송한다. 마지막 단계에서는 소켓을 정리한다.

## 11.2 로컬 포트 바인딩

### BindServiceNameAsync() 메서드 이해
DatagramSocket을 사용하기 위한 첫 번째 단계는 로컬 포트에 바인딩하는 것이다. BindServiceNameAsync() 메서드를 호출하면 지정한 포트에서 UDP 메시지를 수신할 수 있게 된다. 이 메서드는 비동기이므로 co_await를 사용하여 호출해야 한다.

다음은 로컬 포트 5000에 바인딩하는 기본 예제다.

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Networking.h>
#include <iostream>

using namespace winrt;
using namespace Windows::Networking::Sockets;
using namespace Windows::Networking;

IAsyncAction BindUdpSocketAsync()
{
    DatagramSocket socket;
    
    try
    {
        // 로컬 포트 5000에 바인딩
        co_await socket.BindServiceNameAsync(L"5000");
        
        std::wcout << L"UDP 소켓이 포트 5000에 바인딩되었습니다." << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"바인딩 실패: " << ex.message().c_str() << std::endl;
    }
}
```

이 코드를 설명하자면, 먼저 DatagramSocket 객체를 생성한다. 그 다음 BindServiceNameAsync() 메서드에 포트 번호를 문자열로 전달한다. WinRT에서는 포트 번호를 숫자가 아닌 문자열로 전달해야 한다. 이는 포트 번호 대신 서비스 이름(예: "http", "ftp")을 전달할 수 있도록 하기 위함이다.

BindServiceNameAsync()가 성공하면 소켓은 지정한 포트에서 모든 네트워크 인터페이스의 UDP 메시지를 수신하기 시작한다. 포트가 이미 다른 애플리케이션에서 사용 중이면 hresult_error 예외가 발생한다.

포트 번호 대신 서비스 이름을 사용하는 방식도 있다. 예를 들어 "5353" 대신 "mdns"를 사용할 수 있지만, 시스템에서 해당 서비스 이름이 정의되어 있어야 한다.

## 11.3 메시지 보내기

### GetOutputStreamAsync()와 전송 메커니즘
UDP 메시지를 전송하려면 원격 호스트의 주소와 포트를 알아야 한다. DatagramSocket의 GetOutputStreamAsync() 메서드를 호출하면 특정 주소로 메시지를 보낼 수 있는 스트림을 얻을 수 있다.

다음은 특정 주소로 메시지를 보내는 예제다.

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Networking.h>
#include <winrt/Windows.Storage.Streams.h>
#include <string>
#include <iostream>

using namespace winrt;
using namespace Windows::Networking::Sockets;
using namespace Windows::Networking;
using namespace Windows::Storage::Streams;

IAsyncAction SendUdpMessageAsync()
{
    DatagramSocket socket;
    
    try
    {
        // 로컬 포트에 바인딩
        co_await socket.BindServiceNameAsync(L"5001");
        std::wcout << L"소켓이 포트 5001에 바인딩되었습니다." << std::endl;
        
        // 원격 호스트 정보
        HostName remoteHost(L"192.168.1.100");
        std::wstring message = L"Hello UDP Server!";
        
        // 원격 호스트로의 출력 스트림 획득
        auto outputStream = co_await socket.GetOutputStreamAsync(
            remoteHost, 
            L"5002"
        );
        
        // DataWriter를 사용하여 메시지 작성
        DataWriter writer(outputStream);
        writer.WriteString(message);
        
        // 메시지 전송
        co_await writer.StoreAsync();
        writer.DetachStream();
        
        std::wcout << L"메시지 전송 완료: " << message.c_str() << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcout << L"오류: " << ex.message().c_str() << std::endl;
    }
}
```

이 코드의 흐름을 자세히 설명하자. 먼저 로컬 포트 5001에 바인딩한다. 그 다음 원격 호스트의 주소와 포트를 지정하여 GetOutputStreamAsync()를 호출한다. 이 메서드는 해당 주소로 데이터를 보낼 수 있는 출력 스트림을 반환한다.

DataWriter는 기본 데이터 타입을 바이트로 변환하여 스트림에 쓰는 유틸리티 클래스다. WriteString() 메서드로 문자열을 작성한 후 StoreAsync()를 호출하면 실제로 네트워크를 통해 전송된다. 마지막으로 DetachStream()을 호출하여 스트림과의 연결을 끊는다.

중요한 점은 매번 메시지를 보낼 때마다 GetOutputStreamAsync()를 호출할 수 있다는 것이다. UDP는 연결 개념이 없으므로, 같은 소켓에서 여러 다른 주소로 메시지를 보낼 수 있다.

## 11.4 메시지 받기

### MessageReceived 이벤트 처리
UDP 소켓이 메시지를 수신하면 MessageReceived 이벤트가 발생한다. 이 이벤트에 이벤트 핸들러를 등록하여 들어온 메시지를 처리할 수 있다.

다음은 UDP 메시지를 수신하는 완전한 예제다.

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>
#include <string>

using namespace winrt;
using namespace Windows::Networking::Sockets;
using namespace Windows::Storage::Streams;

class UdpReceiver
{
private:
    DatagramSocket m_socket;
    
public:
    IAsyncAction StartListeningAsync()
    {
        try
        {
            // 로컬 포트 5002에 바인딩
            co_await m_socket.BindServiceNameAsync(L"5002");
            std::wcout << L"리스닝 시작: 포트 5002" << std::endl;
            
            // MessageReceived 이벤트 등록
            m_socket.MessageReceived(
                { this, &UdpReceiver::OnMessageReceived }
            );
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"바인딩 실패: " << ex.message().c_str() << std::endl;
        }
    }
    
private:
    void OnMessageReceived(
        DatagramSocket const& socket,
        DatagramSocketMessageReceivedEventArgs const& args)
    {
        try
        {
            // 수신한 메시지의 입력 스트림 획득
            auto inputStream = args.GetDataStream();
            DataReader reader(inputStream);
            
            // 메시지를 읽을 때 필요한 바이트 수를 미리 알아야 함
            // 여기서는 간단히 255바이트까지 읽는다고 가정
            uint32_t availableBytes = reader.UnconsumedBufferLength();
            std::wstring message = reader.ReadString(availableBytes);
            
            // 송신자 정보 출력
            std::wcout << L"송신자: " 
                      << args.RemoteAddress().DisplayName().c_str() << L":"
                      << args.RemotePort().c_str() << std::endl;
            std::wcout << L"메시지: " << message.c_str() << std::endl;
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"메시지 처리 실패: " 
                      << ex.message().c_str() << std::endl;
        }
    }
};
```

이 코드를 설명하자면, MessageReceived 이벤트에 등록한 OnMessageReceived() 메서드가 호출될 때마다 새로운 UDP 메시지가 들어온 것이다. 이벤트 핸들러의 두 번째 파라미터인 args는 메시지와 송신자 정보를 담고 있다.

args.GetDataStream()으로 메시지 데이터를 포함한 스트림을 획득하고, DataReader를 사용하여 이를 읽는다. UnconsumedBufferLength()는 아직 읽지 않은 바이트 수를 반환한다. 송신자의 IP 주소와 포트는 각각 args.RemoteAddress().DisplayName()과 args.RemotePort()로 얻을 수 있다.

주의할 점은 UDP는 데이터 길이를 보장하지 않으므로, 메시지의 실제 크기를 미리 알아야 한다는 것이다. 고정 크기 메시지를 사용하거나, 메시지 앞에 길이 정보를 포함시키는 방식을 권장한다.

## 11.5 멀티캐스트 사용하기

### JoinMulticastGroup() 메서드 활용
멀티캐스트는 한 송신자에서 여러 수신자에게 효율적으로 메시지를 전달하는 방식이다. DatagramSocket의 JoinMulticastGroup() 메서드를 사용하여 특정 멀티캐스트 그룹에 참여할 수 있다. 멀티캐스트 주소는 224.0.0.0부터 239.255.255.255 범위의 IPv4 주소다.

다음은 멀티캐스트 그룹에 참여하여 메시지를 주고받는 예제다.

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Networking.h>
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>

using namespace winrt;
using namespace Windows::Networking::Sockets;
using namespace Windows::Networking;
using namespace Windows::Storage::Streams;

class MulticastPeer
{
private:
    DatagramSocket m_socket;
    static constexpr wchar_t* MULTICAST_ADDRESS = L"224.0.0.100";
    static constexpr wchar_t* MULTICAST_PORT = L"5003";
    
public:
    IAsyncAction JoinMulticastGroupAsync()
    {
        try
        {
            // 로컬 포트에 바인딩
            co_await m_socket.BindServiceNameAsync(MULTICAST_PORT);
            std::wcout << L"포트 " << MULTICAST_PORT 
                      << L"에 바인딩 완료" << std::endl;
            
            // 멀티캐스트 그룹에 참여
            HostName multicastAddr(MULTICAST_ADDRESS);
            m_socket.JoinMulticastGroup(multicastAddr);
            std::wcout << L"멀티캐스트 그룹 " << MULTICAST_ADDRESS 
                      << L"에 참여했습니다." << std::endl;
            
            // MessageReceived 이벤트 등록
            m_socket.MessageReceived(
                { this, &MulticastPeer::OnMulticastMessageReceived }
            );
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"오류: " << ex.message().c_str() << std::endl;
        }
    }
    
    IAsyncAction SendMulticastMessageAsync(std::wstring const& message)
    {
        try
        {
            HostName multicastHost(MULTICAST_ADDRESS);
            
            // 멀티캐스트 주소로 출력 스트림 획득
            auto outputStream = co_await m_socket.GetOutputStreamAsync(
                multicastHost,
                MULTICAST_PORT
            );
            
            DataWriter writer(outputStream);
            writer.WriteString(message);
            
            // 전송
            co_await writer.StoreAsync();
            writer.DetachStream();
            
            std::wcout << L"멀티캐스트 메시지 전송: " 
                      << message.c_str() << std::endl;
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"전송 실패: " << ex.message().c_str() << std::endl;
        }
    }
    
private:
    void OnMulticastMessageReceived(
        DatagramSocket const& socket,
        DatagramSocketMessageReceivedEventArgs const& args)
    {
        try
        {
            auto inputStream = args.GetDataStream();
            DataReader reader(inputStream);
            
            uint32_t availableBytes = reader.UnconsumedBufferLength();
            std::wstring message = reader.ReadString(availableBytes);
            
            std::wcout << L"[멀티캐스트] 송신자: "
                      << args.RemoteAddress().DisplayName().c_str()
                      << L" | 메시지: " << message.c_str() << std::endl;
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"메시지 처리 실패: " 
                      << ex.message().c_str() << std::endl;
        }
    }
};
```

이 코드를 설명하자면, JoinMulticastGroup() 메서드에 멀티캐스트 주소를 전달하면 해당 그룹에 참여하게 된다. 멀티캐스트 그룹에 참여한 후에는 다른 사람들이 그 그룹 주소로 보낸 메시지를 수신할 수 있다.

중요한 구조는 다음과 같다. 로컬 포트에 먼저 바인딩한 후, 멀티캐스트 그룹에 참여한다. 그 후 다른 피어와 동일한 방식으로 메시지를 주고받을 수 있다. SendMulticastMessageAsync() 메서드는 멀티캐스트 주소로 메시지를 보내므로, 같은 그룹에 참여한 모든 피어가 이 메시지를 받게 된다.

멀티캐스트의 장점은 효율성이다. 여러 클라이언트에게 같은 메시지를 보낼 때, 각 클라이언트에게 별도로 메시지를 보내지 않고 한 번에 멀티캐스트 주소로 보내면 네트워크 대역폭을 크게 절약할 수 있다.

## 11.6 종합 예제: P2P 채팅 애플리케이션

### 코드의 목적
이 예제는 UDP를 활용한 완전한 P2P(피어-투-피어) 채팅 애플리케이션을 구현한다. 다수의 피어가 멀티캐스트를 사용하여 메시지를 주고받으며, 새로운 피어의 참여를 감지하고 피어 목록을 유지한다. 이는 로컬 네트워크 내에서 별도의 중앙 서버 없이 피어들이 직접 통신하는 시스템이다.

### 설계 결정 이유

멀티캐스트를 사용하는 이유는 그룹 내 모든 피어에게 효율적으로 메시지를 전달하기 위함이다. 또한 각 피어가 자신의 상태(이름, 접속 상태)를 멀티캐스트 메시지로 알림으로써, 중앙 서버 없이도 피어 발견과 상태 관리가 가능하다.

메시지 형식은 간단한 프로토콜을 사용한다. 메시지 타입(HELLO, CHAT, GOODBYE), 송신자 이름, 메시지 내용으로 구성된다. 이를 통해 피어들은 새로운 사람의 참여, 채팅 메시지, 사람의 퇴장을 구분할 수 있다.

스레드 안전성을 위해 피어 목록(m_peers)을 std::map으로 관리하고, 실제 멀티스레드 환경에서는 뮤텍스를 사용해야 한다. 이 예제에서는 간단히 원리를 보여주기 위해 간단하게 구현했다.

### 코드 구현

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Networking.h>
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>
#include <string>
#include <map>
#include <sstream>
#include <chrono>

using namespace winrt;
using namespace Windows::Networking::Sockets;
using namespace Windows::Networking;
using namespace Windows::Storage::Streams;

// 메시지 타입 정의
enum class MessageType
{
    HELLO,      // 피어 참여 알림
    CHAT,       // 채팅 메시지
    GOODBYE     // 피어 퇴장 알림
};

// 피어 정보 구조
struct PeerInfo
{
    std::wstring name;
    std::wstring address;
    std::chrono::system_clock::time_point lastHeartbeat;
};

// P2P 채팅 애플리케이션 클래스
class P2PChatApplication
{
private:
    DatagramSocket m_socket;
    std::wstring m_localPeerName;
    std::map<std::wstring, PeerInfo> m_peers;
    
    static constexpr wchar_t* MULTICAST_ADDRESS = L"224.0.0.100";
    static constexpr wchar_t* MULTICAST_PORT = L"5004";
    static constexpr wchar_t* MESSAGE_DELIMITER = L"|";
    
public:
    P2PChatApplication(std::wstring const& peerName)
        : m_localPeerName(peerName)
    {
    }
    
    // 애플리케이션 초기화 및 시작
    IAsyncAction InitializeAsync()
    {
        try
        {
            // 로컬 포트에 바인딩
            co_await m_socket.BindServiceNameAsync(MULTICAST_PORT);
            std::wcout << L"[초기화] 포트 " << MULTICAST_PORT 
                      << L"에 바인딩 완료" << std::endl;
            
            // 멀티캐스트 그룹에 참여
            HostName multicastAddr(MULTICAST_ADDRESS);
            m_socket.JoinMulticastGroup(multicastAddr);
            std::wcout << L"[초기화] 멀티캐스트 그룹 " 
                      << MULTICAST_ADDRESS << L"에 참여" << std::endl;
            
            // MessageReceived 이벤트 등록
            m_socket.MessageReceived(
                { this, &P2PChatApplication::OnMessageReceived }
            );
            
            // HELLO 메시지로 자신의 참여 알림
            co_await SendMessageAsync(
                MessageType::HELLO, 
                m_localPeerName,
                L"입장했습니다."
            );
            
            std::wcout << L"[시작] P2P 채팅 애플리케이션 시작됨" << std::endl;
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"[오류] 초기화 실패: " 
                      << ex.message().c_str() << std::endl;
        }
    }
    
    // 채팅 메시지 전송
    IAsyncAction SendChatMessageAsync(std::wstring const& message)
    {
        co_await SendMessageAsync(MessageType::CHAT, m_localPeerName, message);
    }
    
    // 피어 목록 출력
    void PrintPeers() const
    {
        std::wcout << L"\n[피어 목록]" << std::endl;
        for (auto const& pair : m_peers)
        {
            std::wcout << L"  - " << pair.first << L" ("
                      << pair.second.address << L")" << std::endl;
        }
        std::wcout << std::endl;
    }
    
    // 애플리케이션 종료
    IAsyncAction ShutdownAsync()
    {
        try
        {
            // GOODBYE 메시지 전송
            co_await SendMessageAsync(
                MessageType::GOODBYE,
                m_localPeerName,
                L"퇴장합니다."
            );
            
            std::wcout << L"[종료] P2P 채팅 애플리케이션 종료" << std::endl;
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"[오류] 종료 중 오류: " 
                      << ex.message().c_str() << std::endl;
        }
    }
    
private:
    // 메시지 전송 내부 메서드
    IAsyncAction SendMessageAsync(
        MessageType type,
        std::wstring const& peerName,
        std::wstring const& content)
    {
        try
        {
            // 메시지 구성: TYPE|PEER_NAME|CONTENT
            std::wstring typeStr = (type == MessageType::HELLO) ? L"HELLO" :
                                   (type == MessageType::CHAT) ? L"CHAT" :
                                   L"GOODBYE";
            
            std::wstring fullMessage = typeStr + MESSAGE_DELIMITER + 
                                      peerName + MESSAGE_DELIMITER + 
                                      content;
            
            HostName multicastHost(MULTICAST_ADDRESS);
            auto outputStream = co_await m_socket.GetOutputStreamAsync(
                multicastHost,
                MULTICAST_PORT
            );
            
            DataWriter writer(outputStream);
            writer.WriteString(fullMessage);
            co_await writer.StoreAsync();
            writer.DetachStream();
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"[오류] 메시지 전송 실패: " 
                      << ex.message().c_str() << std::endl;
        }
    }
    
    // 수신 메시지 처리
    void OnMessageReceived(
        DatagramSocket const& socket,
        DatagramSocketMessageReceivedEventArgs const& args)
    {
        try
        {
            auto inputStream = args.GetDataStream();
            DataReader reader(inputStream);
            uint32_t availableBytes = reader.UnconsumedBufferLength();
            
            if (availableBytes == 0) return;
            
            // 메시지 파싱
            std::wstring fullMessage = reader.ReadString(availableBytes);
            std::wstring remoteAddress = args.RemoteAddress().DisplayName().c_str();
            
            // 메시지를 파이프 구분자로 분할
            auto parts = ParseMessage(fullMessage);
            if (parts.size() < 3) return;
            
            std::wstring typeStr = parts[0];
            std::wstring peerName = parts[1];
            std::wstring content = parts[2];
            
            // 자신의 메시지는 무시
            if (peerName == m_localPeerName) return;
            
            // 메시지 타입별 처리
            if (typeStr == L"HELLO")
            {
                HandleHelloMessage(peerName, remoteAddress);
            }
            else if (typeStr == L"CHAT")
            {
                HandleChatMessage(peerName, remoteAddress, content);
            }
            else if (typeStr == L"GOODBYE")
            {
                HandleGoodbyeMessage(peerName);
            }
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"[오류] 메시지 처리 실패: " 
                      << ex.message().c_str() << std::endl;
        }
    }
    
    // HELLO 메시지 처리
    void HandleHelloMessage(
        std::wstring const& peerName,
        std::wstring const& address)
    {
        if (m_peers.find(peerName) == m_peers.end())
        {
            // 새로운 피어 추가
            PeerInfo info;
            info.name = peerName;
            info.address = address;
            info.lastHeartbeat = std::chrono::system_clock::now();
            m_peers[peerName] = info;
            
            std::wcout << L"[알림] 피어 참여: " << peerName 
                      << L" (" << address << L")" << std::endl;
        }
        else
        {
            // 기존 피어 하트비트 업데이트
            m_peers[peerName].lastHeartbeat = std::chrono::system_clock::now();
        }
    }
    
    // CHAT 메시지 처리
    void HandleChatMessage(
        std::wstring const& peerName,
        std::wstring const& address,
        std::wstring const& content)
    {
        // 피어 목록 업데이트
        if (m_peers.find(peerName) != m_peers.end())
        {
            m_peers[peerName].lastHeartbeat = std::chrono::system_clock::now();
        }
        
        // 채팅 메시지 출력
        std::wcout << L"[" << peerName << L"] " << content << std::endl;
    }
    
    // GOODBYE 메시지 처리
    void HandleGoodbyeMessage(std::wstring const& peerName)
    {
        auto it = m_peers.find(peerName);
        if (it != m_peers.end())
        {
            m_peers.erase(it);
            std::wcout << L"[알림] 피어 퇴장: " << peerName << std::endl;
        }
    }
    
    // 메시지 파싱 (구분자로 분할)
    std::vector<std::wstring> ParseMessage(std::wstring const& message)
    {
        std::vector<std::wstring> result;
        std::wistringstream stream(message);
        std::wstring token;
        
        while (std::getline(stream, token, L'|'))
        {
            result.push_back(token);
        }
        
        return result;
    }
};

// 사용 예제
IAsyncAction RunP2PChatAsync()
{
    P2PChatApplication chat(L"Alice");
    
    co_await chat.InitializeAsync();
    
    // 채팅 메시지 전송
    co_await chat.SendChatMessageAsync(L"안녕하세요!");
    
    // 피어 목록 출력
    chat.PrintPeers();
    
    // 추가 메시지 전송
    co_await chat.SendChatMessageAsync(L"오늘 날씨가 좋네요.");
    
    // 종료
    co_await chat.ShutdownAsync();
}
```

### 주요 부분 상세 해설
이 구현에서 가장 중요한 부분은 메시지 프로토콜이다. 각 메시지는 "TYPE|PEER_NAME|CONTENT" 형식으로, 파이프 문자(|)로 구분된다. 이는 간단하지만 확장 가능한 설계이다.

메시지 타입은 세 가지다. HELLO는 새로운 피어의 참여를 알리거나 기존 피어의 하트비트 역할을 한다. CHAT은 실제 채팅 메시지고, GOODBYE는 피어의 정리를 알린다.

피어 목록은 std::map으로 관리되며, 각 피어는 이름, 주소, 마지막 하트비트 시간을 기록한다. 실제 프로덕션 환경에서는 이를 기반으로 타임아웃된 피어를 자동으로 제거하는 로직을 추가해야 한다.

자신의 메시지는 무시하는 로직(`if (peerName == m_localPeerName) return;`)이 중요하다. 멀티캐스트 메시지는 송신자도 수신하게 되므로, 자신의 메시지를 다시 처리하지 않아야 한다.

### 개선 방안과 확장 가능성
현재 구현은 여러 방면에서 개선할 수 있다. 첫 번째는 타임아웃 처리다. 일정 시간 동안 메시지를 보내지 않은 피어를 자동으로 제거하는 기능을 추가할 수 있다.

두 번째는 메시지 인증이다. 악의적인 피어가 다른 피어의 이름으로 메시지를 보내는 것을 방지하려면, 메시지에 디지털 서명을 추가해야 한다.

세 번째는 비공개 메시지 기능이다. 현재는 모든 메시지가 멀티캐스트되지만, 특정 피어에게만 메시지를 보내려면 유니캐스트 통신을 추가할 수 있다.

네 번째는 메시지 순서 보장이다. UDP는 패킷의 순서를 보장하지 않으므로, 메시지에 시퀀스 번호를 붙여서 순서를 재정렬하는 로직을 추가할 수 있다.

다섯 번째는 압축과 암호화다. 대역폭을 절약하고 보안을 강화하려면 메시지를 압축하고 암호화해야 한다.

이러한 개선 사항들은 실제 게임 서버나 협업 애플리케이션에서 필수적인 기능들이다.
