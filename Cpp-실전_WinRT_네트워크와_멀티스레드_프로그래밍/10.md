# Chapter 10. TCP 서버 - StreamSocketListener
지금까지 StreamSocket을 사용하여 클라이언트 입장에서 서버에 연결하는 방법을 학습했다면, 이번 장에서는 반대 방향을 다룬다. StreamSocketListener는 특정 포트에서 클라이언트의 연결을 기다리고, 연결이 들어오면 이를 수락하여 통신하는 TCP 서버를 구현하게 해준다. 온라인 게임 서버 개발자라면 반드시 숙달해야 할 핵심 기술이다.

---

## 10.1 StreamSocketListener API 설명
StreamSocketListener는 TCP 서버의 핵심 클래스로, 클라이언트의 연결을 수락하고 관리하는 역할을 한다. 전통적인 Winsock의 listen() 함수와 accept() 함수의 역할을 하지만, WinRT의 비동기 모델에 맞게 재설계되었다.

### 리스너 생성과 설정
StreamSocketListener는 아무런 매개변수 없는 기본 생성자로 만들어진다. 생성 후 바인딩과 설정을 거쳐 연결 수락을 시작하게 된다.

```cpp
#include <winrt/Windows.Networking.Sockets.h>
using namespace winrt::Windows::Networking::Sockets;

// 리스너 생성
StreamSocketListener listener;

// 또는 스마트 포인터 사용
auto listener = std::make_unique<StreamSocketListener>();
```

리스너는 내부적으로 상태를 유지한다. 리스닝 상태가 되기 전에는 연결을 수락할 수 없으며, Close()가 호출되면 더 이상 새로운 연결을 수락하지 않는다.

### 주요 메서드
**BindServiceNameAsync()** 메서드는 특정 포트 번호 또는 서비스 이름에 바인딩한다. 이 메서드는 비동기 작업을 반환하므로 co_await로 기다릴 수 있다.

```cpp
hstring portNumber(L"8888");
co_await listener.BindServiceNameAsync(portNumber);
```

**BindEndpointAsync()** 메서드는 특정 네트워크 인터페이스와 포트에 바인딩한다. 이는 BindServiceNameAsync()보다 더 세밀한 제어를 제공한다.

```cpp
HostName localAddress(L"192.168.1.10");
hstring portNumber(L"8888");
co_await listener.BindEndpointAsync(localAddress, portNumber);
```

**ConnectionReceived** 이벤트는 클라이언트가 연결을 시도할 때 발생한다. 이 이벤트에 핸들러를 등록하여 연결을 처리한다.

```cpp
listener.ConnectionReceived(
    [](const StreamSocketListener& sender, 
       const StreamSocketListenerConnectionReceivedEventArgs& args) {
        // 연결 처리 로직
    }
);
```

**Close()** 메서드는 리스너를 종료한다. 이 메서드는 동기적이며, 호출 후 더 이상 새로운 연결을 수락하지 않는다.

```cpp
listener.Close();
```

### 주요 속성

**Control** 속성은 **StreamSocketListenerControl** 객체를 반환하여 리스너의 동작을 제어할 수 있게 한다. 예를 들어, 수신 버퍼 크기를 조정하거나 특정 QoS 설정을 할 수 있다.

```cpp
StreamSocketListenerControl control = listener.Control();
control.QualityOfService(SocketQualityOfService::LowLatency);
```

**Information** 속성은 **StreamSocketListenerInformation** 객체를 반환하여 리스너의 현재 상태를 조회할 수 있게 한다. 바인드된 로컬 주소와 포트를 확인할 수 있다.

```cpp
StreamSocketListenerInformation info = listener.Information();
HostName localAddress = info.LocalAddress();
hstring localPort = info.LocalPort();
```

## 10.2 포트 바인딩하기
서버가 클라이언트 연결을 받으려면 먼저 특정 포트에 바인딩해야 한다. WinRT에서는 두 가지 바인딩 방식을 제공한다.

### BindServiceNameAsync()와 BindEndpointAsync()

```cpp
#include <winrt/Windows.Networking.h>
#include <winrt/Windows.Networking.Sockets.h>
#include <iostream>

using namespace winrt;
using namespace winrt::Windows::Networking;
using namespace winrt::Windows::Networking::Sockets;

// 방식 1: 포트 번호로 바인딩 (모든 네트워크 인터페이스)
IAsyncAction BindToPortExample()
{
    StreamSocketListener listener;

    try
    {
        // 포트 8888로 바인딩
        hstring servicePort(L"8888");
        co_await listener.BindServiceNameAsync(servicePort);

        // 바인딩 정보 확인
        StreamSocketListenerInformation info = 
            listener.Information();
        std::wcout << L"바인드 성공" << std::endl;
        std::wcout << L"로컬 주소: " 
                   << info.LocalAddress().DisplayName()
                       .c_str() << std::endl;
        std::wcout << L"로컬 포트: " 
                   << info.LocalPort().c_str() << std::endl;

        // 서버 종료 시까지 대기
        listener.Close();
    }
    catch (hresult_error const& ex)
    {
        std::wcerr << L"바인드 실패: " 
                   << ex.message().c_str() << std::endl;
    }
}

// 방식 2: 특정 주소와 포트로 바인딩
IAsyncAction BindToSpecificEndpointExample()
{
    StreamSocketListener listener;

    try
    {
        // 로컬호스트 127.0.0.1의 8888 포트로 바인딩
        HostName localAddress(L"127.0.0.1");
        hstring servicePort(L"8888");

        co_await listener.BindEndpointAsync(
            localAddress, servicePort);

        std::wcout << L"특정 엔드포인트 바인드 성공" << std::endl;

        listener.Close();
    }
    catch (hresult_error const& ex)
    {
        std::wcerr << L"바인드 실패: " 
                   << ex.message().c_str() << std::endl;
    }
}

// 방식 3: 자동 포트 할당
IAsyncAction BindWithAutomaticPort()
{
    StreamSocketListener listener;

    try
    {
        // 포트를 "0"으로 지정하면 시스템이 자동으로 할당
        hstring automaticPort(L"0");
        co_await listener.BindServiceNameAsync(automaticPort);

        StreamSocketListenerInformation info = 
            listener.Information();
        std::wcout << L"자동 할당된 포트: " 
                   << info.LocalPort().c_str() << std::endl;

        listener.Close();
    }
    catch (hresult_error const& ex)
    {
        std::wcerr << L"바인드 실패: " 
                   << ex.message().c_str() << std::endl;
    }
}

// 방식 4: IPv6로 바인딩
IAsyncAction BindToIPv6Example()
{
    StreamSocketListener listener;

    try
    {
        // IPv6 주소로 바인딩
        HostName ipv6Address(L"::1");  // 루프백 주소
        hstring servicePort(L"8888");

        co_await listener.BindEndpointAsync(
            ipv6Address, servicePort);

        std::wcout << L"IPv6 바인드 성공" << std::endl;

        listener.Close();
    }
    catch (hresult_error const& ex)
    {
        std::wcerr << L"바인드 실패: " 
                   << ex.message().c_str() << std::endl;
    }
}
```

**BindServiceNameAsync() vs BindEndpointAsync()**: BindServiceNameAsync()는 모든 사용 가능한 네트워크 인터페이스에서 수신하므로, 대부분의 게임 서버에 적합하다. BindEndpointAsync()는 특정 네트워크 인터페이스에만 바인딩하므로, 여러 네트워크가 있는 서버에서 특정 인터페이스만 사용하고 싶을 때 유용하다.

**포트 범위**: 0~65535 범위의 포트를 사용할 수 있다. 1024 이하의 포트는 관리자 권한이 필요할 수 있다. 게임 서버는 보통 1024 이상의 포트를 사용한다.

**자동 포트 할당**: "0"을 포트로 지정하면 시스템이 사용 가능한 포트를 자동으로 선택한다. 이는 여러 서버 인스턴스를 한 기계에서 실행할 때 유용하다.

## 10.3 연결 수락 이벤트
클라이언트가 서버에 연결을 시도하면 ConnectionReceived 이벤트가 발생한다. 이 이벤트의 핸들러에서 클라이언트와의 통신 작업을 수행한다.

### ConnectionReceived 핸들러 구조

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <iostream>

using namespace winrt::Windows::Networking::Sockets;

IAsyncAction ConnectionReceivedHandlerExample()
{
    StreamSocketListener listener;

    try
    {
        // 1단계: 리스너 바인딩
        hstring servicePort(L"8888");
        co_await listener.BindServiceNameAsync(servicePort);

        std::wcout << L"서버가 포트 8888에서 대기 중..." 
                   << std::endl;

        // 2단계: ConnectionReceived 이벤트 핸들러 등록
        listener.ConnectionReceived(
            [](const StreamSocketListener& sender,
               const StreamSocketListenerConnectionReceivedEventArgs& 
                   args)
            {
                // 이 코드는 클라이언트 연결 시 호출됨
                // 주의: 이벤트 핸들러 내에서는 co_await를 직접 사용할 수 없음
                // 별도의 비동기 함수를 호출해야 함
                
                try
                {
                    // 3단계: 수락된 소켓 얻기
                    StreamSocket clientSocket = 
                        args.Socket();

                    // 4단계: 클라이언트 정보 출력
                    StreamSocketInformation info = 
                        clientSocket.Information();

                    std::wcout << L"[연결 수락]" << std::endl;
                    std::wcout << L"  원격 주소: " 
                               << info.RemoteAddress()
                                   .DisplayName().c_str() 
                               << std::endl;
                    std::wcout << L"  원격 포트: " 
                               << info.RemotePort().c_str() 
                               << std::endl;
                    std::wcout << L"  로컬 주소: " 
                               << info.LocalAddress()
                                   .DisplayName().c_str() 
                               << std::endl;
                    std::wcout << L"  로컬 포트: " 
                               << info.LocalPort().c_str() 
                               << std::endl;

                    // 5단계: 클라이언트 처리를 위한 별도 함수 호출
                    // (비동기 작업은 이 함수 내에서 수행)
                    // HandleClientConnection(clientSocket);
                }
                catch (hresult_error const& ex)
                {
                    std::wcerr << L"[오류] 연결 수락 중 오류: " 
                               << ex.message().c_str() 
                               << std::endl;
                }
            }
        );

        // 6단계: 이벤트를 처리하기 위해 대기
        // 실전에서는 서버 종료 신호를 받을 때까지 대기
        Sleep(60000);  // 60초 대기

        listener.Close();
        std::wcout << L"서버 종료" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcerr << L"[오류] 서버 실패: " 
                   << ex.message().c_str() << std::endl;
    }
}

// 클라이언트 연결 처리를 위한 비동기 함수
IAsyncAction HandleClientConnection(StreamSocket clientSocket)
{
    try
    {
        // 여기서 비동기 작업(co_await)을 사용할 수 있음
        IInputStream inputStream = 
            co_await clientSocket.GetInputStreamAsync();
        IOutputStream outputStream = 
            co_await clientSocket.GetOutputStreamAsync();

        DataReader reader(inputStream);
        reader.InputStreamOptions(InputStreamOptions::Partial);

        DataWriter writer(outputStream);

        // 클라이언트와 통신...
        std::wcout << L"클라이언트 처리 시작" << std::endl;

        clientSocket.Close();
    }
    catch (hresult_error const& ex)
    {
        std::wcerr << L"클라이언트 처리 중 오류: " 
                   << ex.message().c_str() << std::endl;
    }
}
```

**ConnectionReceived 이벤트의 특징**: 이 이벤트 핸들러는 동기적으로 실행되므로, 내부에서 직접 co_await를 사용할 수 없다. 비동기 작업이 필요하면 별도의 비동기 함수를 호출해야 한다.

**StreamSocketListenerConnectionReceivedEventArgs**: 이벤트 매개변수 객체로, Socket() 메서드를 통해 수락된 클라이언트 소켓을 얻을 수 있다. 이 소켓은 이미 연결 상태이므로 바로 데이터 송수신을 시작할 수 있다.

**클라이언트 정보 추출**: 수락된 소켓의 Information 속성으로부터 클라이언트의 IP 주소와 포트 정보를 얻을 수 있다. 이는 로깅, 접근 제어, 통계 수집 등에 유용하다.

## 10.4 클라이언트와 통신하기
ConnectionReceived 이벤트에서 수락한 소켓을 사용하여 클라이언트와 데이터를 송수신한다. StreamSocket의 사용 방법과 동일하지만, 서버 쪽 특성을 고려한 패턴을 알아야 한다.

### 소켓에서 스트림 얻기

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>

using namespace winrt;
using namespace winrt::Windows::Networking::Sockets;
using namespace winrt::Windows::Storage::Streams;

// 클라이언트 통신을 위한 구조체
struct ClientConnection
{
    StreamSocket Socket;
    hstring RemoteAddress;
    hstring RemotePort;
};

// 클라이언트와의 통신 처리
IAsyncAction HandleClientCommunication(
    ClientConnection client)
{
    try
    {
        std::wcout << L"[처리 시작] " 
                   << client.RemoteAddress.c_str() 
                   << L":" << client.RemotePort.c_str() 
                   << std::endl;

        // 1단계: 입출력 스트림 획득
        IInputStream inputStream = 
            co_await client.Socket.GetInputStreamAsync();
        IOutputStream outputStream = 
            co_await client.Socket.GetOutputStreamAsync();

        std::wcout << L"[스트림 획득 완료]" << std::endl;

        // 2단계: DataReader와 DataWriter 생성
        DataReader reader(inputStream);
        reader.InputStreamOptions(InputStreamOptions::Partial);

        DataWriter writer(outputStream);

        // 3단계: 클라이언트로부터 데이터 수신
        uint32_t bytesLoaded = 
            co_await reader.LoadAsync(1024);

        if (bytesLoaded == 0)
        {
            std::wcout << L"[연결 종료] 클라이언트가 연결을 끊음" 
                       << std::endl;
            client.Socket.Close();
            co_return;
        }

        std::wcout << L"[수신] " << bytesLoaded 
                   << L" 바이트" << std::endl;

        // 4단계: 수신한 데이터 읽기
        hstring receivedMessage = 
            reader.ReadString(bytesLoaded);
        
        std::wcout << L"[메시지] " 
                   << receivedMessage.c_str() 
                   << std::endl;

        // 5단계: 응답 데이터 작성
        hstring responseMessage = 
            L"[서버 응답] 메시지를 받았습니다: " + 
            receivedMessage;
        
        writer.WriteString(responseMessage);
        uint32_t bytesSent = 
            co_await writer.StoreAsync();

        std::wcout << L"[송신] " << bytesSent 
                   << L" 바이트" << std::endl;

        // 6단계: 정리
        writer.DetachStream();
        client.Socket.Close();

        std::wcout << L"[처리 종료]" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcerr << L"[오류] " 
                   << ex.message().c_str() << std::endl;
        
        try
        {
            client.Socket.Close();
        }
        catch (...)
        {
        }
    }
}

// 서버의 메인 루프
IAsyncAction RunSimpleEchoServer()
{
    StreamSocketListener listener;

    try
    {
        // 포트 8888에 바인딩
        hstring servicePort(L"8888");
        co_await listener.BindServiceNameAsync(servicePort);

        std::wcout << L"[서버 시작] 포트 8888에서 대기 중..." 
                   << std::endl;

        // ConnectionReceived 이벤트 처리
        listener.ConnectionReceived(
            [](const StreamSocketListener& sender,
               const StreamSocketListenerConnectionReceivedEventArgs& 
                   args)
            {
                try
                {
                    StreamSocket clientSocket = args.Socket();
                    StreamSocketInformation info = 
                        clientSocket.Information();

                    ClientConnection client;
                    client.Socket = std::move(clientSocket);
                    client.RemoteAddress = 
                        info.RemoteAddress().DisplayName();
                    client.RemotePort = info.RemotePort();

                    // 비동기 함수 호출 (결과를 기다리지 않음)
                    auto _ = HandleClientCommunication(client);
                }
                catch (hresult_error const& ex)
                {
                    std::wcerr << L"[오류] 연결 수락 실패: " 
                               << ex.message().c_str() 
                               << std::endl;
                }
            }
        );

        // 30초 동안 실행
        Sleep(30000);

        listener.Close();
        std::wcout << L"[서버 종료]" << std::endl;
    }
    catch (hresult_error const& ex)
    {
        std::wcerr << L"[오류] 서버 시작 실패: " 
                   << ex.message().c_str() << std::endl;
    }
}
```

**스트림 획득의 순차성**: GetInputStreamAsync()와 GetOutputStreamAsync()는 소켓이 완전히 연결된 상태에서만 호출되어야 한다. 이벤트 핸들러에서 수락한 소켓은 이미 연결된 상태이므로 안전하게 호출할 수 있다.

**DataReader/DataWriter 사용**: 클라이언트 소켓에서 얻은 스트림을 DataReader와 DataWriter로 래핑하여 타입 안전한 데이터 송수신을 한다.

**비동기 작업 관리**: ConnectionReceived 이벤트 핸들러는 동기적이므로, 클라이언트 처리 작업을 별도의 비동기 함수로 분리하고 호출한다. 이렇게 하면 다음 클라이언트 연결을 빠르게 수락할 수 있다.

## 10.5 연결 관리하기
실전 게임 서버에서는 여러 클라이언트가 동시에 연결된다. 각 클라이언트의 연결 상태를 추적하고, 필요시 특정 클라이언트에 메시지를 보내거나 모든 클라이언트에게 브로드캐스트하는 기능이 필요하다.

### 클라이언트 목록 유지

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <iostream>
#include <vector>
#include <mutex>
#include <memory>

using namespace winrt::Windows::Networking::Sockets;
using namespace winrt::Windows::Storage::Streams;

// 클라이언트 정보
struct ClientSession
{
    uint32_t SessionId;
    hstring RemoteAddress;
    hstring RemotePort;
    StreamSocket Socket;
    std::chrono::system_clock::time_point ConnectTime;
    bool IsActive;
};

// 클라이언트 관리자
class ClientManager
{
private:
    std::vector<std::shared_ptr<ClientSession>> clients_;
    std::mutex clientsMutex_;
    uint32_t nextSessionId_;

public:
    ClientManager() : nextSessionId_(1) {}

    // 새 클라이언트 등록
    std::shared_ptr<ClientSession> RegisterClient(
        StreamSocket socket)
    {
        std::lock_guard<std::mutex> lock(clientsMutex_);

        auto session = 
            std::make_shared<ClientSession>();
        
        StreamSocketInformation info = 
            socket.Information();

        session->SessionId = nextSessionId_++;
        session->RemoteAddress = 
            info.RemoteAddress().DisplayName();
        session->RemotePort = info.RemotePort();
        session->Socket = std::move(socket);
        session->ConnectTime = 
            std::chrono::system_clock::now();
        session->IsActive = true;

        clients_.push_back(session);

        std::wcout << L"[세션 생성] ID=" << session->SessionId 
                   << L" Address=" 
                   << session->RemoteAddress.c_str() 
                   << std::endl;

        return session;
    }

    // 클라이언트 제거
    void UnregisterClient(uint32_t sessionId)
    {
        std::lock_guard<std::mutex> lock(clientsMutex_);

        auto it = std::find_if(
            clients_.begin(), clients_.end(),
            [sessionId](const 
                std::shared_ptr<ClientSession>& session) {
                return session->SessionId == sessionId;
            });

        if (it != clients_.end())
        {
            (*it)->IsActive = false;
            clients_.erase(it);
            std::wcout << L"[세션 제거] ID=" << sessionId 
                       << std::endl;
        }
    }

    // 모든 활성 클라이언트 개수
    int GetActiveClientCount() const
    {
        std::lock_guard<std::mutex> lock(clientsMutex_);
        return static_cast<int>(clients_.size());
    }

    // 모든 클라이언트 정보 출력
    void PrintAllClients() const
    {
        std::lock_guard<std::mutex> lock(clientsMutex_);

        std::wcout << L"\n=== 연결된 클라이언트 ===" << std::endl;
        std::wcout << L"총 개수: " << clients_.size() 
                   << std::endl;

        for (size_t i = 0; i < clients_.size(); ++i)
        {
            const auto& client = clients_[i];
            std::wcout << L"  [" << (i + 1) << L"] ID=" 
                       << client->SessionId << L" | " 
                       << client->RemoteAddress.c_str() 
                       << L":" 
                       << client->RemotePort.c_str() 
                       << std::endl;
        }
    }

    // 특정 세션 ID로 클라이언트 찾기
    std::shared_ptr<ClientSession> FindClient(
        uint32_t sessionId)
    {
        std::lock_guard<std::mutex> lock(clientsMutex_);

        auto it = std::find_if(
            clients_.begin(), clients_.end(),
            [sessionId](const 
                std::shared_ptr<ClientSession>& session) {
                return session->SessionId == sessionId;
            });

        if (it != clients_.end())
        {
            return *it;
        }

        return nullptr;
    }

    // 특정 클라이언트에게 메시지 전송
    IAsyncAction SendToClient(
        uint32_t sessionId, 
        const hstring& message)
    {
        auto client = FindClient(sessionId);
        
        if (!client)
        {
            std::wcerr << L"[오류] 세션을 찾을 수 없음: " 
                       << sessionId << std::endl;
            co_return;
        }

        try
        {
            IOutputStream outputStream = 
                co_await client->Socket
                    .GetOutputStreamAsync();
            
            DataWriter writer(outputStream);
            writer.WriteString(message);
            
            co_await writer.StoreAsync();
            co_await writer.FlushAsync();
            
            std::wcout << L"[메시지 송신] ID=" << sessionId 
                       << std::endl;
        }
        catch (hresult_error const& ex)
        {
            std::wcerr << L"[오류] 메시지 송신 실패: " 
                       << ex.message().c_str() 
                       << std::endl;
        }
    }

    // 모든 클라이언트에게 브로드캐스트
    IAsyncAction BroadcastToAll(
        const hstring& message)
    {
        std::vector<std::shared_ptr<ClientSession>> 
            clientsCopy;
        
        {
            std::lock_guard<std::mutex> lock(clientsMutex_);
            clientsCopy = clients_;
        }

        for (const auto& client : clientsCopy)
        {
            co_await SendToClient(
                client->SessionId, message);
        }

        std::wcout << L"[브로드캐스트] " 
                   << clientsCopy.size() 
                   << L"명에게 전송" << std::endl;
    }

    // 통계 정보
    void PrintStatistics() const
    {
        std::lock_guard<std::mutex> lock(clientsMutex_);

        std::wcout << L"\n=== 서버 통계 ===" << std::endl;
        std::wcout << L"활성 연결: " << clients_.size() 
                   << std::endl;
        std::wcout << L"다음 세션 ID: " 
                   << nextSessionId_ << std::endl;
    }
};

// 사용 예제
IAsyncAction ManageClientsExample()
{
    ClientManager manager;

    // 클라이언트 시뮬레이션을 위한 더미 소켓 정보
    // (실제로는 ConnectionReceived 이벤트에서 수락된 소켓을 사용)

    std::wcout << L"클라이언트 관리 예제" << std::endl;
    
    manager.PrintStatistics();
    manager.PrintAllClients();

    // 모든 클라이언트에게 브로드캐스트
    co_await manager.BroadcastToAll(
        L"[공지] 모든 클라이언트에게 전송되는 메시지");
}
```

**스레드 안전성**: ClientManager는 std::mutex를 사용하여 클라이언트 목록을 보호한다. 여러 스레드가 동시에 접근해도 데이터 손상이 발생하지 않는다.

**세션 ID 관리**: 각 클라이언트에 고유한 세션 ID를 부여하여, IP:포트 쌍이 변할 수 있는 네트워크 환경에서도 클라이언트를 추적할 수 있다.

**스마트 포인터 사용**: std::shared_ptr을 사용하여 클라이언트 세션의 생명주기를 관리한다. 클라이언트가 연결을 끊으면 자동으로 메모리가 해제된다.

**브로드캐스트 패턴**: BroadcastToAll은 모든 클라이언트에게 동일한 메시지를 전송한다. 게임 상태 업데이트, 공지 사항, 채팅 메시지 등에 사용된다.

## 10.6 종합 예제: 멀티 클라이언트 에코 서버
이번 종합 예제는 이 장의 모든 개념을 통합하여 실전 게임 서버의 기초가 되는 멀티 클라이언트 에코 서버를 구현한다.

### 코드의 목적과 설계

이 에코 서버는 다음을 구현한다:
- 여러 클라이언트의 동시 연결 수용
- 각 클라이언트로부터 데이터 수신
- 수신한 데이터를 에코(반복)하여 클라이언트에게 전송
- 연결된 클라이언트 목록 관리
- 모든 클라이언트에게 주기적 상태 브로드캐스트
- 클라이언트 연결/종료 이벤트 추적

설계 결정 사항은 다음과 같다. 각 클라이언트 연결을 별도의 비동기 작업으로 처리하여 한 클라이언트의 지연이 다른 클라이언트에 영향을 주지 않도록 했다. 클라이언트 목록은 뮤텍스로 보호하여 스레드 안전하게 관리했다. 브로드캐스트 기능을 통해 게임 서버의 필수 요소인 다중 클라이언트 메시지 배포를 시연했다.

### 구현 코드

```cpp
#include <winrt/Windows.Networking.h>
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>
#include <vector>
#include <mutex>
#include <memory>
#include <chrono>

using namespace winrt;
using namespace winrt::Windows::Networking;
using namespace winrt::Windows::Networking::Sockets;
using namespace winrt::Windows::Storage::Streams;

// 클라이언트 세션 구조
struct ClientSession
{
    uint32_t SessionId;
    hstring RemoteAddress;
    hstring RemotePort;
    StreamSocket Socket;
    std::chrono::system_clock::time_point ConnectTime;
    bool IsActive;
    int MessageCount;
};

// 에코 서버 클래스
class EchoServer
{
private:
    StreamSocketListener listener_;
    std::vector<std::shared_ptr<ClientSession>> clients_;
    std::mutex clientsMutex_;
    uint32_t nextSessionId_;
    bool isRunning_;

public:
    EchoServer() 
        : nextSessionId_(1), isRunning_(false)
    {
    }

    // 서버 시작
    IAsyncAction StartAsync(const hstring& port)
    {
        try
        {
            // 포트에 바인딩
            co_await listener_.BindServiceNameAsync(port);
            isRunning_ = true;

            std::wcout << L"[서버 시작] 포트 " 
                       << port.c_str() 
                       << L"에서 수신 대기 중..." 
                       << std::endl;

            StreamSocketListenerInformation info = 
                listener_.Information();
            std::wcout << L"[바인드] " 
                       << info.LocalAddress()
                           .DisplayName().c_str() 
                       << L":" << info.LocalPort().c_str() 
                       << std::endl;

            // 연결 수락 이벤트 등록
            listener_.ConnectionReceived(
                [this](const StreamSocketListener& sender,
                       const StreamSocketListenerConnectionReceivedEventArgs& 
                           args)
                {
                    this->OnConnectionReceived(args);
                }
            );
        }
        catch (hresult_error const& ex)
        {
            std::wcerr << L"[오류] 서버 시작 실패: " 
                       << ex.message().c_str() 
                       << std::endl;
        }
    }

    // 서버 종료
    void Stop()
    {
        isRunning_ = false;
        listener_.Close();
        std::wcout << L"[서버 종료]" << std::endl;
    }

    // 통계 출력
    void PrintStatistics()
    {
        std::lock_guard<std::mutex> lock(clientsMutex_);

        std::wcout << L"\n=== 서버 통계 ===" << std::endl;
        std::wcout << L"활성 연결: " << clients_.size() 
                   << std::endl;
        
        for (size_t i = 0; i < clients_.size(); ++i)
        {
            const auto& client = clients_[i];
            std::wcout << L"  [" << (i + 1) << L"] ID=" 
                       << client->SessionId << L" | " 
                       << client->RemoteAddress.c_str() 
                       << L":" 
                       << client->RemotePort.c_str() 
                       << L" | 메시지=" 
                       << client->MessageCount 
                       << std::endl;
        }
    }

private:
    // 새 연결 수락 시 호출
    void OnConnectionReceived(
        const StreamSocketListenerConnectionReceivedEventArgs& args)
    {
        try
        {
            StreamSocket clientSocket = args.Socket();
            StreamSocketInformation info = 
                clientSocket.Information();

            // 클라이언트 세션 생성
            auto session = 
                std::make_shared<ClientSession>();
            
            session->SessionId = nextSessionId_++;
            session->RemoteAddress = 
                info.RemoteAddress().DisplayName();
            session->RemotePort = info.RemotePort();
            session->Socket = std::move(clientSocket);
            session->ConnectTime = 
                std::chrono::system_clock::now();
            session->IsActive = true;
            session->MessageCount = 0;

            // 클라이언트 목록에 추가
            {
                std::lock_guard<std::mutex> lock(clientsMutex_);
                clients_.push_back(session);
            }

            std::wcout << L"[연결] ID=" << session->SessionId 
                       << L" From=" 
                       << session->RemoteAddress.c_str() 
                       << L":" 
                       << session->RemotePort.c_str() 
                       << std::endl;

            // 비동기로 클라이언트 처리
            auto _ = HandleClientAsync(session);
        }
        catch (hresult_error const& ex)
        {
            std::wcerr << L"[오류] 연결 수락 실패: " 
                       << ex.message().c_str() 
                       << std::endl;
        }
    }

    // 클라이언트 연결 처리
    IAsyncAction HandleClientAsync(
        std::shared_ptr<ClientSession> session)
    {
        try
        {
            IInputStream inputStream = 
                co_await session->Socket
                    .GetInputStreamAsync();
            IOutputStream outputStream = 
                co_await session->Socket
                    .GetOutputStreamAsync();

            DataReader reader(inputStream);
            reader.InputStreamOptions(
                InputStreamOptions::Partial);
            
            DataWriter writer(outputStream);

            // 클라이언트로부터 데이터 수신 루프
            while (session->IsActive)
            {
                // 데이터 수신
                uint32_t bytesLoaded = 
                    co_await reader.LoadAsync(1024);

                if (bytesLoaded == 0)
                {
                    // 클라이언트가 연결 종료
                    std::wcout << L"[종료] ID=" 
                               << session->SessionId 
                               << std::endl;
                    break;
                }

                // 수신 데이터 읽기
                hstring receivedMessage = 
                    reader.ReadString(bytesLoaded);

                session->MessageCount++;

                std::wcout << L"[수신] ID=" 
                           << session->SessionId 
                           << L" | " 
                           << receivedMessage.c_str() 
                           << L" (" << bytesLoaded 
                           << L" bytes)" 
                           << std::endl;

                // 에코 응답 전송 (수신한 데이터 그대로 반환)
                hstring echoMessage = 
                    L"[에코] " + receivedMessage;
                
                writer.WriteString(echoMessage);
                co_await writer.StoreAsync();
                co_await writer.FlushAsync();

                std::wcout << L"[송신] ID=" 
                           << session->SessionId 
                           << L" | 에코 메시지 전송" 
                           << std::endl;
            }

            // 연결 정리
            writer.DetachStream();
            session->Socket.Close();
            session->IsActive = false;

            // 클라이언트 목록에서 제거
            {
                std::lock_guard<std::mutex> lock(clientsMutex_);
                auto it = std::find(
                    clients_.begin(), clients_.end(), 
                    session);
                if (it != clients_.end())
                {
                    clients_.erase(it);
                }
            }

            std::wcout << L"[정리 완료] ID=" 
                       << session->SessionId 
                       << std::endl;
        }
        catch (hresult_error const& ex)
        {
            std::wcerr << L"[오류] 클라이언트 처리 중: " 
                       << ex.message().c_str() 
                       << std::endl;

            session->IsActive = false;
            try
            {
                session->Socket.Close();
            }
            catch (...)
            {
            }

            // 클라이언트 목록에서 제거
            {
                std::lock_guard<std::mutex> lock(clientsMutex_);
                auto it = std::find(
                    clients_.begin(), clients_.end(), 
                    session);
                if (it != clients_.end())
                {
                    clients_.erase(it);
                }
            }
        }
    }
};

// 서버 실행 예제
IAsyncAction RunEchoServerDemo()
{
    EchoServer server;

    // 서버 시작 (포트 8888)
    co_await server.StartAsync(L"8888");

    // 30초 동안 실행
    std::wcout << L"30초 동안 서버 실행 중..." << std::endl;
    
    for (int i = 0; i < 6; ++i)
    {
        Sleep(5000);
        server.PrintStatistics();
    }

    server.Stop();
    std::wcout << L"프로그램 종료" << std::endl;
}
```

### 주요 부분 상세 해설
**OnConnectionReceived 메서드**: 클라이언트 연결이 들어오면 즉시 호출되는 콜백이다. 여기서 새로운 ClientSession을 생성하고, 뮤텍스를 사용하여 스레드 안전하게 clients_ 벡터에 추가한다. 그 후 비동기 함수 HandleClientAsync를 호출하여 클라이언트를 처리한다. 이렇게 분리하면 다음 클라이언트의 연결을 빠르게 수락할 수 있다.

**HandleClientAsync 메서드**: 각 클라이언트를 처리하는 비동기 코루틴이다. GetInputStreamAsync와 GetOutputStreamAsync로 스트림을 얻고, while 루프에서 지속적으로 데이터를 수신한다. LoadAsync가 0을 반환하면 클라이언트가 연결을 종료했으므로 루프를 빠져나간다. 수신한 메시지를 "[에코]" 접두사와 함께 다시 전송한다.

**스레드 안전성**: clients_ 벡터에 접근할 때마다 std::lock_guard로 뮤텍스를 잠금한다. 클라이언트 처리 중에는 뮤텍스를 풀어서 다른 스레드가 클라이언트 목록을 조회할 수 있게 한다.

**리소스 정리**: 클라이언트 처리가 끝나면 반드시 DetachStream()과 Close()를 호출하여 리소스를 정리한다. 또한 clients_ 벡터에서도 제거하여 메모리 누수를 방지한다.

**통계 수집**: MessageCount를 추적하여 각 클라이언트가 몇 개의 메시지를 받았는지 기록한다. 이는 성능 모니터링과 디버깅에 유용하다.

### 개선 방안과 확장 가능성
**메시지 프로토콜**: 현재는 단순 문자열 에코만 수행하지만, 바이너리 프로토콜을 추가하여 게임 메시지를 처리할 수 있다. 메시지 타입, ID, 페이로드 등을 포함하는 구조화된 형식을 사용하면 더 복잡한 게임 로직을 구현할 수 있다.

**브로드캐스트 기능**: 한 클라이언트의 메시지를 다른 모든 클라이언트에게 전달하는 기능을 추가할 수 있다. 이는 게임 플레이어의 이동, 채팅 메시지 등을 모두에게 전파하는 데 필수적이다.

**타임아웃 처리**: 클라이언트가 일정 시간 동안 데이터를 보내지 않으면 자동으로 연결을 끊는 기능을 추가할 수 있다. 이는 유령 연결(ghost connection)을 정리하는 데 도움이 된다.

**메시지 큐**: 각 클라이언트마다 송신 큐를 두어, 빠른 속도로 전송 명령이 들어와도 안정적으로 처리할 수 있게 할 수 있다.

**암호화**: TLS/SSL을 사용하여 클라이언트와의 통신을 암호화할 수 있다. StreamSocket의 Control 속성으로 보안 설정을 할 수 있다.

**인증**: 클라이언트 연결 시 사용자 이름과 비밀번호를 확인하는 로직을 추가할 수 있다. 이는 온라인 게임의 필수 요소이다.

**데이터베이스 연동**: 클라이언트 정보나 게임 상태를 데이터베이스에 저장할 수 있다. 예를 들어 플레이어의 위치, 아이템, 점수 등을 영속화할 수 있다.

이 에코 서버는 간단하지만 게임 서버의 핵심 구조를 모두 포함한다. 여기서 배운 패턴을 확장하면 실제 게임 서버를 구현할 수 있다.
