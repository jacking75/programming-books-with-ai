# Chapter 16. 네트워크와 스레드 통합

## 16.1 네트워크 I/O 스레딩 모델

### WinRT의 비동기 처리 방식
네트워크 프로그래밍에서 스레드와 비동기를 올바르게 이해하는 것은 매우 중요하다. WinRT는 전통적인 멀티스레드 프로그래밍과는 다른 접근 방식을 제공한다.

전통적인 멀티스레드 모델에서는 각 클라이언트마다 하나의 스레드를 할당했다. 1000명의 클라이언트가 연결되면 1000개의 스레드가 필요했고, 이는 메모리와 CPU 오버헤드를 초래했다. 게다가 스레드 간 동기화도 복잡했다.

WinRT의 비동기 모델은 이러한 문제를 해결한다. co_await를 사용하면 실제로는 스레드를 블로킹하지 않고 비동기 작업의 완료를 기다린다. 따라서 적은 수의 스레드로 많은 클라이언트를 처리할 수 있다.

핵심 개념을 설명하자면, 비동기 작업(async operation)은 시간이 걸리는 작업(네트워크 I/O, 파일 I/O 등)을 나타낸다. co_await를 만나면 코루틴은 일시 중지되고, 작업이 완료되면 재개된다. 이 과정에서 스레드는 다른 작업을 처리할 수 있다.

WinRT의 비동기 처리 방식은 ThreadPool과 긴밀하게 연동된다. 비동기 작업이 완료되면, ThreadPool의 워커 스레드 중 하나에서 코루틴을 재개한다. 이를 통해 자동으로 부하 분산이 이루어진다.

게임 서버에서 이를 적용하면, 수천 명의 플레이어 연결을 상대적으로 적은 수의 스레드로 처리할 수 있다. 예를 들어, 10,000명의 플레이어를 50-100개 정도의 워커 스레드로 처리하는 것이 가능하다.

하지만 주의할 점이 있다. 모든 작업이 비동기 친화적인 것은 아니다. CPU 집약적인 작업(복잡한 게임 로직 계산)은 여전히 스레드가 필요하다. 따라서 I/O 바운드 작업(네트워크)과 CPU 바운드 작업(게임 로직)을 분리하는 것이 중요하다.

## 16.2 수신 전용 스레드 패턴

### 지속적인 데이터 수신
게임 서버는 클라이언트로부터 지속적으로 데이터를 수신해야 한다. 플레이어가 움직이면 위치 정보가 오고, 채팅을 입력하면 메시지가 온다. 이러한 데이터를 효율적으로 처리하는 방법이 수신 전용 스레드 패턴이다.

수신 전용 스레드는 데이터 수신에만 집중하는 스레드다. 이 스레드는 클라이언트로부터 도착한 데이터를 읽고, 이를 처리 큐에 넣는다. 다른 워커 스레드들은 큐에서 데이터를 꺼내 처리한다.

다음은 수신 전용 스레드 패턴의 기본 예제다.

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>
#include <thread>
#include <queue>
#include <mutex>

using namespace winrt;
using namespace Windows::Networking::Sockets;
using namespace Windows::Storage::Streams;

// 수신된 데이터 구조
struct ReceivedMessage
{
    uint32_t clientId;
    std::vector<uint8_t> data;
    std::chrono::system_clock::time_point timestamp;
};

class ReceiveThread
{
private:
    std::thread m_receiveThread;
    std::queue<ReceivedMessage> m_messageQueue;
    std::mutex m_queueMutex;
    bool m_shouldRun = true;
    
public:
    // 수신 스레드 시작
    void Start(StreamSocket const& socket, uint32_t clientId)
    {
        m_receiveThread = std::thread(
            [this, socket = socket, clientId]() 
        {
            ReceiveLoop(socket, clientId);
        });
    }
    
    // 수신 메시지 꺼내기
    bool TryGetMessage(ReceivedMessage& message)
    {
        std::lock_guard<std::mutex> lock(m_queueMutex);
        
        if (m_messageQueue.empty())
            return false;
        
        message = m_messageQueue.front();
        m_messageQueue.pop();
        return true;
    }
    
    void Stop()
    {
        m_shouldRun = false;
        if (m_receiveThread.joinable())
            m_receiveThread.join();
    }
    
private:
    void ReceiveLoop(StreamSocket const& socket, uint32_t clientId)
    {
        DataReader reader(socket.InputStream());
        
        while (m_shouldRun)
        {
            try
            {
                // 1024 바이트 읽기 대기
                uint32_t bytesLoaded = reader.LoadAsync(1024).get();
                
                if (bytesLoaded == 0)
                {
                    // 클라이언트 연결 종료
                    std::wcout << L"클라이언트 " << clientId 
                              << L" 연결 종료" << std::endl;
                    m_shouldRun = false;
                    break;
                }
                
                // 데이터를 벡터로 변환
                std::vector<uint8_t> data(bytesLoaded);
                auto buffer = reader.ReadBuffer(bytesLoaded);
                
                // 메시지 구성
                ReceivedMessage msg;
                msg.clientId = clientId;
                msg.timestamp = std::chrono::system_clock::now();
                
                // 버퍼에서 데이터 추출
                uint8_t* rawData = buffer.data();
                data.assign(rawData, rawData + bytesLoaded);
                msg.data = std::move(data);
                
                // 큐에 추가
                {
                    std::lock_guard<std::mutex> lock(m_queueMutex);
                    m_messageQueue.push(msg);
                }
                
                std::wcout << L"[수신] 클라이언트 " << clientId 
                          << L": " << bytesLoaded << L" bytes" 
                          << std::endl;
            }
            catch (hresult_error const& ex)
            {
                std::wcout << L"수신 오류: " 
                          << ex.message().c_str() << std::endl;
                m_shouldRun = false;
            }
        }
    }
};
```

이 패턴의 핵심은 수신과 처리를 분리한다는 것이다. 수신 스레드는 읽기만 하고, 처리는 별도의 워커 스레드가 한다. 이를 통해 다음과 같은 장점을 얻는다:

첫 번째는 응답성 향상이다. 수신 스레드가 항상 데이터를 기다리고 있으므로, 새로운 메시지가 도착하면 즉시 큐에 추가된다.

두 번째는 처리 유연성이다. 처리하는 워커 스레드가 바쁘더라도, 수신 스레드는 계속 데이터를 받을 수 있다.

세 번째는 버퍼링이다. 처리보다 수신이 빠르면, 큐에 메시지가 쌓인다. 처리가 따라잡으면 큐가 감소한다.

## 16.3 전송 큐 구현

### 스레드 안전 메시지 큐
데이터를 받는 것만큼 중요한 것이 데이터를 보내는 것이다. 여러 워커 스레드가 동시에 메시지를 보내려고 하면 문제가 발생할 수 있다. 이를 해결하기 위해 스레드 안전한 전송 큐를 구현해야 한다.

다음은 스레드 안전 메시지 큐의 예제다.

```cpp
#include <queue>
#include <mutex>
#include <condition_variable>
#include <memory>

// 스레드 안전 메시지 큐 (전송용)
template<typename T>
class ThreadSafeQueue
{
private:
    mutable std::mutex m_mutex;
    std::queue<T> m_queue;
    std::condition_variable m_condVar;
    
public:
    ThreadSafeQueue() = default;
    
    // 복사 금지
    ThreadSafeQueue(ThreadSafeQueue const&) = delete;
    ThreadSafeQueue& operator=(ThreadSafeQueue const&) = delete;
    
    // 큐에 항목 추가
    void Push(T value)
    {
        {
            std::lock_guard<std::mutex> lock(m_mutex);
            m_queue.push(std::move(value));
        }
        m_condVar.notify_one();  // 대기 중인 스레드 깨우기
    }
    
    // 큐에서 항목 추출 (대기 없음)
    bool TryPop(T& value)
    {
        std::lock_guard<std::mutex> lock(m_mutex);
        
        if (m_queue.empty())
            return false;
        
        value = std::move(m_queue.front());
        m_queue.pop();
        return true;
    }
    
    // 큐에서 항목 추출 (타임아웃 있음)
    bool WaitAndPop(T& value, std::chrono::milliseconds timeout)
    {
        std::unique_lock<std::mutex> lock(m_mutex);
        
        if (!m_condVar.wait_for(
                lock, 
                timeout,
                [this] { return !m_queue.empty(); }
            ))
        {
            return false;  // 타임아웃
        }
        
        value = std::move(m_queue.front());
        m_queue.pop();
        return true;
    }
    
    // 큐의 크기
    size_t Size() const
    {
        std::lock_guard<std::mutex> lock(m_mutex);
        return m_queue.size();
    }
    
    // 큐가 비어있는지 확인
    bool Empty() const
    {
        std::lock_guard<std::mutex> lock(m_mutex);
        return m_queue.empty();
    }
    
    // 큐 비우기
    void Clear()
    {
        std::lock_guard<std::mutex> lock(m_mutex);
        while (!m_queue.empty())
            m_queue.pop();
    }
};

// 메시지 타입
struct SendMessage
{
    uint32_t clientId;
    std::vector<uint8_t> data;
    int priority;  // 우선순위 (높을수록 먼저 전송)
};

// 사용 예제
IAsyncAction SendQueueExampleAsync()
{
    ThreadSafeQueue<SendMessage> sendQueue;
    
    // 전송 워커 스레드
    auto senderThread = std::thread([&sendQueue]()
    {
        SendMessage msg;
        
        while (sendQueue.WaitAndPop(msg, std::chrono::milliseconds(1000)))
        {
            std::wcout << L"전송: 클라이언트 " << msg.clientId 
                      << L", 크기 " << msg.data.size() 
                      << L", 우선순위 " << msg.priority << std::endl;
            
            // 실제 전송 로직...
        }
    });
    
    // 여러 워커 스레드가 메시지를 큐에 추가
    for (int i = 0; i < 5; i++)
    {
        SendMessage msg;
        msg.clientId = i;
        msg.data.resize(256);
        msg.priority = i % 2;
        
        sendQueue.Push(msg);
        
        co_await winrt::resume_after(std::chrono::milliseconds(100));
    }
    
    // 스레드 종료 대기
    senderThread.join();
    
    std::wcout << L"전송 완료, 큐 크기: " << sendQueue.Size() 
              << std::endl;
}
```

ThreadSafeQueue는 여러 스레드의 동시 접근을 안전하게 처리한다:

첫 번째는 뮤텍스 보호다. 모든 작업이 뮤텍스로 보호되므로, 동시에 여러 스레드가 접근해도 데이터 무결성이 유지된다.

두 번째는 조건 변수다. WaitAndPop()은 큐가 비어있으면 대기한다. 새로운 항목이 Push되면 깨어나서 처리한다.

세 번째는 Move 의미론이다. std::move를 사용하여 불필요한 복사를 피한다. 큰 데이터를 전송할 때 성능 향상이 된다.

## 16.4 이벤트 기반 아키텍처

### 콜백과 이벤트 디스패치
게임 서버는 다양한 이벤트가 발생한다: 클라이언트 연결, 데이터 수신, 에러 발생 등. 이러한 이벤트들을 체계적으로 처리하기 위해 이벤트 기반 아키텍처를 사용한다.

다음은 이벤트 기반 아키텍처의 예제다.

```cpp
#include <functional>
#include <map>
#include <vector>
#include <mutex>

// 이벤트 타입 정의
enum class EventType
{
    ClientConnected,
    ClientDisconnected,
    DataReceived,
    ErrorOccurred,
    ServerStarted,
    ServerStopped
};

// 이벤트 정보
struct Event
{
    EventType type;
    uint32_t clientId;
    std::string message;
    std::chrono::system_clock::time_point timestamp;
};

// 이벤트 핸들러 타입
using EventHandler = std::function<void(Event const&)>;

// 이벤트 디스패처
class EventDispatcher
{
private:
    std::map<EventType, std::vector<EventHandler>> m_handlers;
    std::mutex m_mutex;
    
public:
    // 이벤트 핸들러 등록
    void Subscribe(EventType type, EventHandler handler)
    {
        std::lock_guard<std::mutex> lock(m_mutex);
        m_handlers[type].push_back(handler);
        
        std::wcout << L"이벤트 핸들러 등록: " 
                  << static_cast<int>(type) << std::endl;
    }
    
    // 이벤트 발생
    void Dispatch(Event const& event)
    {
        std::vector<EventHandler> handlersToCall;
        
        {
            std::lock_guard<std::mutex> lock(m_mutex);
            auto it = m_handlers.find(event.type);
            
            if (it != m_handlers.end())
            {
                handlersToCall = it->second;
            }
        }
        
        // 뮤텍스 락을 해제한 후 핸들러 호출
        for (auto const& handler : handlersToCall)
        {
            try
            {
                handler(event);
            }
            catch (std::exception const& ex)
            {
                std::wcout << L"이벤트 핸들러 실행 중 예외: " 
                          << ex.what() << std::endl;
            }
        }
    }
    
    // 특정 이벤트 타입의 핸들러 개수
    size_t GetHandlerCount(EventType type) const
    {
        std::lock_guard<std::mutex> lock(m_mutex);
        auto it = m_handlers.find(type);
        return it != m_handlers.end() ? it->second.size() : 0;
    }
};

// 게임 서버의 이벤트 기반 처리
class GameServer
{
private:
    EventDispatcher m_dispatcher;
    
public:
    void Initialize()
    {
        // 클라이언트 연결 이벤트 핸들러
        m_dispatcher.Subscribe(
            EventType::ClientConnected,
            [this](Event const& event)
        {
            std::wcout << L"[연결] 클라이언트 " << event.clientId 
                      << L" 연결됨" << std::endl;
            // 로그인 프로세스 시작
            OnClientConnected(event.clientId);
        });
        
        // 데이터 수신 이벤트 핸들러
        m_dispatcher.Subscribe(
            EventType::DataReceived,
            [this](Event const& event)
        {
            std::wcout << L"[메시지] 클라이언트 " << event.clientId 
                      << L": " << event.message << std::endl;
            // 메시지 처리
            OnDataReceived(event.clientId, event.message);
        });
        
        // 클라이언트 연결 해제 이벤트 핸들러
        m_dispatcher.Subscribe(
            EventType::ClientDisconnected,
            [this](Event const& event)
        {
            std::wcout << L"[연결해제] 클라이언트 " 
                      << event.clientId << L" 해제됨" << std::endl;
            // 정리 작업
            OnClientDisconnected(event.clientId);
        });
        
        // 에러 이벤트 핸들러
        m_dispatcher.Subscribe(
            EventType::ErrorOccurred,
            [this](Event const& event)
        {
            std::wcout << L"[에러] " << event.message << std::endl;
        });
    }
    
    void RaiseEvent(Event const& event)
    {
        m_dispatcher.Dispatch(event);
    }
    
private:
    void OnClientConnected(uint32_t clientId)
    {
        // 클라이언트 인증, 세션 생성 등
    }
    
    void OnDataReceived(uint32_t clientId, std::string const& message)
    {
        // 메시지 파싱, 게임 로직 실행 등
    }
    
    void OnClientDisconnected(uint32_t clientId)
    {
        // 세션 정리, 리소스 해제 등
    }
};
```

이벤트 기반 아키텍처의 장점을 살펴보자:

첫 번째는 느슨한 결합(loose coupling)이다. 이벤트를 발생시키는 코드와 처리하는 코드가 분리되어 있다. 새로운 이벤트 핸들러를 추가해도 기존 코드를 수정할 필요가 없다.

두 번째는 확장성이다. 새로운 이벤트 타입을 쉽게 추가할 수 있고, 기존 이벤트에 대한 핸들러도 언제든 추가할 수 있다.

세 번째는 유지보수성이다. 각 이벤트 핸들러가 독립적이므로 이해하고 수정하기 쉽다.

## 16.5 종합 예제: 비동기 TCP 서버 프레임워크

### 코드의 목적

이 예제는 연결 관리, 메시지 큐, 워커 스레드 풀을 포함한 완전한 비동기 TCP 서버 프레임워크를 구현한다. 이는 실제 게임 서버 개발에서 재사용할 수 있는 프레임워크다.

이 프레임워크는 다음을 포함한다:
- 여러 클라이언트 연결 관리
- 각 클라이언트로부터의 데이터 수신
- 스레드 안전한 메시지 전송 큐
- 워커 스레드 풀을 통한 비동기 작업 처리
- 이벤트 기반 알림 시스템

### 설계 결정 이유

프레임워크 설계에는 여러 중요한 결정이 있다.

첫 번째는 클라이언트별 수신 스레드 사용이다. 각 클라이언트마다 독립적인 수신 스레드를 두면, 한 클라이언트의 느린 수신이 다른 클라이언트에 영향을 미치지 않는다.

두 번째는 중앙집중식 전송 큐다. 모든 클라이언트의 전송 요청이 하나의 큐로 들어가므로, 공정한 전송 순서를 보장한다.

세 번째는 워커 스레드 풀이다. 게임 로직 처리를 위해 고정 크기의 스레드 풀을 유지한다. 이는 메모리 사용량을 예측 가능하게 한다.

네 번째는 이벤트 기반 설계다. 각 단계에서 이벤트를 발생시키므로, 사용자가 필요한 부분만 처리하는 콜백을 등록할 수 있다.

### 코드 구현

```cpp
#include <winrt/Windows.Networking.Sockets.h>
#include <winrt/Windows.Storage.Streams.h>
#include <iostream>
#include <memory>
#include <map>
#include <queue>
#include <mutex>
#include <thread>
#include <vector>
#include <functional>
#include <chrono>

using namespace winrt;
using namespace Windows::Networking::Sockets;
using namespace Windows::Storage::Streams;

// 이벤트 타입
enum class ServerEventType
{
    ClientConnected,
    ClientDisconnected,
    MessageReceived,
    Error
};

// 서버 이벤트
struct ServerEvent
{
    ServerEventType type;
    uint32_t clientId;
    std::string data;
    std::chrono::system_clock::time_point timestamp;
};

// 메시지 구조
struct Message
{
    uint32_t clientId;
    std::vector<uint8_t> data;
};

// 클라이언트 세션
class ClientSession
{
public:
    uint32_t clientId;
    StreamSocket socket;
    std::thread receiveThread;
    bool isActive = true;
    
    ClientSession(uint32_t id, StreamSocket sock)
        : clientId(id), socket(std::move(sock))
    {
    }
    
    ~ClientSession()
    {
        isActive = false;
        if (receiveThread.joinable())
            receiveThread.join();
    }
};

// 비동기 TCP 서버 프레임워크
class AsyncTcpServerFramework
{
private:
    StreamSocketListener m_listener;
    std::map<uint32_t, std::shared_ptr<ClientSession>> m_clients;
    std::mutex m_clientsMutex;
    
    std::queue<Message> m_sendQueue;
    std::mutex m_sendQueueMutex;
    
    std::vector<std::thread> m_workerThreads;
    std::queue<ServerEvent> m_eventQueue;
    std::mutex m_eventQueueMutex;
    
    bool m_isRunning = false;
    uint32_t m_nextClientId = 1;
    
    static constexpr int NUM_WORKER_THREADS = 4;
    
    using EventCallback = std::function<void(ServerEvent const&)>;
    std::map<ServerEventType, std::vector<EventCallback>> m_eventHandlers;
    std::mutex m_handlersMutex;
    
public:
    // 서버 시작
    IAsyncAction StartAsync(std::wstring const& port)
    {
        try
        {
            m_isRunning = true;
            
            // 워커 스레드 시작
            for (int i = 0; i < NUM_WORKER_THREADS; i++)
            {
                m_workerThreads.emplace_back(
                    [this]() { WorkerThread(); }
                );
            }
            
            // 전송 스레드 시작
            std::thread senderThread([this]() { SenderThread(); });
            senderThread.detach();
            
            // 리스너 시작
            auto condition = m_listener.ConnectionReceived(
                { this, &AsyncTcpServerFramework::OnConnectionReceived }
            );
            
            co_await m_listener.BindServiceNameAsync(port);
            
            std::wcout << L"[서버] 포트 " << port.c_str() 
                      << L"에서 대기 중..." << std::endl;
            
            RaiseEvent(ServerEvent{
                ServerEventType::Error,
                0,
                "Server started"
            });
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"[오류] 서버 시작 실패: " 
                      << ex.message().c_str() << std::endl;
        }
    }
    
    // 이벤트 핸들러 등록
    void Subscribe(ServerEventType type, EventCallback handler)
    {
        std::lock_guard<std::mutex> lock(m_handlersMutex);
        m_eventHandlers[type].push_back(handler);
    }
    
    // 메시지 전송
    void SendMessage(uint32_t clientId, 
                    std::vector<uint8_t> const& data)
    {
        std::lock_guard<std::mutex> lock(m_sendQueueMutex);
        m_sendQueue.push({clientId, data});
    }
    
    // 클라이언트 수
    size_t GetClientCount() const
    {
        std::lock_guard<std::mutex> lock(m_clientsMutex);
        return m_clients.size();
    }
    
    void Stop()
    {
        m_isRunning = false;
        
        std::lock_guard<std::mutex> lock(m_clientsMutex);
        m_clients.clear();
    }
    
private:
    // 연결 수신
    void OnConnectionReceived(
        StreamSocketListener const&,
        StreamSocketListenerConnectionReceivedEventArgs const& args)
    {
        try
        {
            StreamSocket socket = args.Socket();
            uint32_t clientId = m_nextClientId++;
            
            std::wcout << L"[연결] 클라이언트 " << clientId 
                      << L" 연결됨" << std::endl;
            
            auto session = std::make_shared<ClientSession>(
                clientId, 
                std::move(socket)
            );
            
            {
                std::lock_guard<std::mutex> lock(m_clientsMutex);
                m_clients[clientId] = session;
            }
            
            // 수신 스레드 시작
            session->receiveThread = std::thread(
                [this, session]() { ReceiveFromClient(session); }
            );
            
            RaiseEvent(ServerEvent{
                ServerEventType::ClientConnected,
                clientId,
                "Client connected"
            });
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"[오류] 연결 처리 실패: " 
                      << ex.message().c_str() << std::endl;
        }
    }
    
    // 클라이언트로부터 수신
    void ReceiveFromClient(std::shared_ptr<ClientSession> session)
    {
        try
        {
            DataReader reader(session->socket.InputStream());
            
            while (session->isActive && m_isRunning)
            {
                uint32_t bytesRead = reader.LoadAsync(1024).get();
                
                if (bytesRead == 0)
                {
                    break;  // 클라이언트 연결 종료
                }
                
                std::vector<uint8_t> data(bytesRead);
                auto buffer = reader.ReadBuffer(bytesRead);
                uint8_t* rawData = buffer.data();
                data.assign(rawData, rawData + bytesRead);
                
                // 이벤트 발생
                RaiseEvent(ServerEvent{
                    ServerEventType::MessageReceived,
                    session->clientId,
                    std::string(data.begin(), data.end())
                });
            }
        }
        catch (hresult_error const& ex)
        {
            std::wcout << L"[오류] 수신 실패 (클라이언트 " 
                      << session->clientId << L"): " 
                      << ex.message().c_str() << std::endl;
        }
        
        // 클라이언트 제거
        {
            std::lock_guard<std::mutex> lock(m_clientsMutex);
            m_clients.erase(session->clientId);
        }
        
        std::wcout << L"[연결해제] 클라이언트 " 
                  << session->clientId << L" 연결 해제" << std::endl;
        
        RaiseEvent(ServerEvent{
            ServerEventType::ClientDisconnected,
            session->clientId,
            "Client disconnected"
        });
    }
    
    // 전송 스레드
    void SenderThread()
    {
        while (m_isRunning)
        {
            Message msg;
            
            {
                std::lock_guard<std::mutex> lock(m_sendQueueMutex);
                if (m_sendQueue.empty())
                {
                    std::this_thread::sleep_for(
                        std::chrono::milliseconds(10)
                    );
                    continue;
                }
                
                msg = m_sendQueue.front();
                m_sendQueue.pop();
            }
            
            // 클라이언트에 전송
            std::shared_ptr<ClientSession> session;
            {
                std::lock_guard<std::mutex> lock(m_clientsMutex);
                auto it = m_clients.find(msg.clientId);
                if (it != m_clients.end())
                    session = it->second;
            }
            
            if (session)
            {
                try
                {
                    DataWriter writer(session->socket.OutputStream());
                    writer.WriteBytes(
                        winrt::array_view<uint8_t const>(
                            msg.data.data(), msg.data.size()
                        )
                    );
                    writer.StoreAsync().get();
                    writer.DetachStream();
                }
                catch (hresult_error const& ex)
                {
                    std::wcout << L"[오류] 전송 실패: " 
                              << ex.message().c_str() << std::endl;
                }
            }
        }
    }
    
    // 워커 스레드
    void WorkerThread()
    {
        while (m_isRunning)
        {
            // 이벤트 처리
            ServerEvent event;
            
            {
                std::lock_guard<std::mutex> lock(m_eventQueueMutex);
                if (m_eventQueue.empty())
                {
                    std::this_thread::sleep_for(
                        std::chrono::milliseconds(10)
                    );
                    continue;
                }
                
                event = m_eventQueue.front();
                m_eventQueue.pop();
            }
            
            // 등록된 핸들러 호출
            std::vector<EventCallback> handlers;
            {
                std::lock_guard<std::mutex> lock(m_handlersMutex);
                auto it = m_eventHandlers.find(event.type);
                if (it != m_eventHandlers.end())
                    handlers = it->second;
            }
            
            for (auto const& handler : handlers)
            {
                try
                {
                    handler(event);
                }
                catch (std::exception const& ex)
                {
                    std::wcout << L"[오류] 이벤트 핸들러 실행 중 "
                              L"예외: " << ex.what() << std::endl;
                }
            }
        }
    }
    
    // 이벤트 발생
    void RaiseEvent(ServerEvent const& event)
    {
        std::lock_guard<std::mutex> lock(m_eventQueueMutex);
        m_eventQueue.push(event);
    }
};

// 사용 예제
IAsyncAction ServerFrameworkExampleAsync()
{
    AsyncTcpServerFramework server;
    
    // 이벤트 핸들러 등록
    server.Subscribe(
        ServerEventType::ClientConnected,
        [](ServerEvent const& event)
    {
        std::wcout << L">> 클라이언트 연결: " << event.clientId 
                  << std::endl;
    });
    
    server.Subscribe(
        ServerEventType::MessageReceived,
        [](ServerEvent const& event)
    {
        std::wcout << L">> 메시지 수신 (클라이언트" 
                  << event.clientId << L"): " 
                  << event.data << std::endl;
    });
    
    server.Subscribe(
        ServerEventType::ClientDisconnected,
        [](ServerEvent const& event)
    {
        std::wcout << L">> 클라이언트 연결 해제: " 
                  << event.clientId << std::endl;
    });
    
    // 서버 시작
    co_await server.StartAsync(L"8080");
    
    // 30초 실행
    co_await winrt::resume_after(std::chrono::seconds(30));
    
    server.Stop();
    
    std::wcout << L"서버 종료, 최종 클라이언트 수: " 
              << server.GetClientCount() << std::endl;
}
```

### 주요 부분 상세 해설
AsyncTcpServerFramework는 여러 핵심 컴포넌트로 구성된다.

**클라이언트 관리**: m_clients 맵이 모든 연결된 클라이언트를 관리한다. shared_ptr을 사용하여 자동 메모리 관리를 한다.

**메시지 큐**: m_sendQueue는 모든 클라이언트의 전송 요청을 저장한다. SenderThread에서 이 큐를 처리하므로, 스레드 안전성이 보장된다.

**워커 스레드 풀**: NUM_WORKER_THREADS개의 워커 스레드가 이벤트를 처리한다. 고정 크기의 스레드 풀이므로 메모리 사용량이 예측 가능하다.

**이벤트 기반 처리**: RaiseEvent()는 이벤트를 큐에 추가하고, 워커 스레드가 이를 처리한다. 이를 통해 비동기 처리가 가능해진다.

### 개선 방안과 확장 가능성
첫 번째 개선점은 메시지 우선순위다. 게임 이벤트는 빨리 전송하고, 일반 데이터는 나중에 전송할 수 있도록 우선순위 큐를 사용할 수 있다.

두 번째는 동적 워커 스레드 조정이다. 이벤트 큐의 크기가 커지면 워커 스레드를 추가하고, 작아지면 제거하는 방식으로 자동 조정할 수 있다.

세 번째는 연결 타임아웃 관리다. 일정 시간 동안 활동이 없는 클라이언트를 자동으로 연결 해제할 수 있다.

네 번째는 통계 수집이다. 초당 처리 메시지 수, 평균 지연, 에러율 등을 기록하여 모니터링할 수 있다.

다섯 번째는 graceful shutdown이다. 서버를 종료할 때 기존 클라이언트들과의 연결을 안전하게 정리하는 프로세스를 구현할 수 있다.

실전 게임 서버 개발에서는 이 프레임워크를 기반으로 게임 특화 로직을 추가하여 완성된 서버를 구축할 수 있다.
