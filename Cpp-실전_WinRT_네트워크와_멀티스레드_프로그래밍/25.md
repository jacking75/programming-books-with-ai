# Chapter 25. 테스트와 디버깅
이제 여러분이 만든 게임 서버가 정말로 작동하는지 확인하는 방법을 배울 차례다. 완벽하게 설계하고 구현했다고 생각한 코드도 실제로 돌려보면 예상 밖의 문제들이 발생한다. 네트워크는 불완전하고, 플레이어들의 행동은 예측 불가능하며, 타이밍 문제는 항상 숨어있다.

이 장에서는 게임 서버를 체계적으로 테스트하고 문제를 찾아내며 해결하는 방법을 배운다. 단순한 버그 찾기가 아니라, 부하 상황에서도 안정적으로 작동하는지 확인하고, 예상 밖의 네트워크 상황에서도 견딜 수 있는지 검증하는 것이다.

## 25.1 단위 테스트 작성하기
단위 테스트(Unit Test)는 함수나 클래스 하나하나를 독립적으로 테스트하는 것이다. 게임 서버에서는 프로토콜 처리, 상태 업데이트, 패킷 생성 같은 핵심 로직을 먼저 테스트해야 한다.

### 프로토콜 테스트
게임 서버의 프로토콜은 매우 중요하다. 프로토콜이 잘못 구현되면 클라이언트와 서버가 통신할 수 없다.

```cpp
// GameProtocol.h
#pragma once
#include <cstdint>
#include <vector>
#include <cstring>

// 프로토콜 메시지 타입
enum class MessageType : uint8_t {
    PlayerMove = 0x01,
    PlayerAttack = 0x02,
    PlayerJoin = 0x03,
    PlayerLeave = 0x04,
    GameStateUpdate = 0x05,
    Error = 0xFF
};

// 플레이어 이동 메시지
struct MoveMessage {
    MessageType type;
    uint32_t playerId;
    float posX;
    float posY;
    uint8_t direction;
    
    static constexpr size_t SERIALIZED_SIZE = 1 + 4 + 4 + 4 + 1;  // 14바이트
    
    // 메시지를 바이트 배열로 직렬화한다
    void Serialize(uint8_t* buffer, size_t bufferSize) const {
        if (bufferSize < SERIALIZED_SIZE) {
            throw std::runtime_error("Buffer too small");
        }
        
        size_t offset = 0;
        buffer[offset++] = static_cast<uint8_t>(type);
        
        std::memcpy(buffer + offset, &playerId, 4);
        offset += 4;
        
        std::memcpy(buffer + offset, &posX, 4);
        offset += 4;
        
        std::memcpy(buffer + offset, &posY, 4);
        offset += 4;
        
        buffer[offset++] = direction;
    }
    
    // 바이트 배열로부터 메시지를 역직렬화한다
    static MoveMessage Deserialize(const uint8_t* buffer, size_t bufferSize) {
        if (bufferSize < SERIALIZED_SIZE) {
            throw std::runtime_error("Buffer too small");
        }
        
        MoveMessage msg;
        size_t offset = 0;
        
        msg.type = static_cast<MessageType>(buffer[offset++]);
        
        std::memcpy(&msg.playerId, buffer + offset, 4);
        offset += 4;
        
        std::memcpy(&msg.posX, buffer + offset, 4);
        offset += 4;
        
        std::memcpy(&msg.posY, buffer + offset, 4);
        offset += 4;
        
        msg.direction = buffer[offset++];
        
        return msg;
    }
};

// 공격 메시지
struct AttackMessage {
    MessageType type;
    uint32_t playerId;
    uint32_t targetPlayerId;
    
    static constexpr size_t SERIALIZED_SIZE = 1 + 4 + 4;  // 9바이트
    
    void Serialize(uint8_t* buffer, size_t bufferSize) const {
        if (bufferSize < SERIALIZED_SIZE) {
            throw std::runtime_error("Buffer too small");
        }
        
        size_t offset = 0;
        buffer[offset++] = static_cast<uint8_t>(type);
        std::memcpy(buffer + offset, &playerId, 4);
        offset += 4;
        std::memcpy(buffer + offset, &targetPlayerId, 4);
    }
    
    static AttackMessage Deserialize(const uint8_t* buffer, size_t bufferSize) {
        if (bufferSize < SERIALIZED_SIZE) {
            throw std::runtime_error("Buffer too small");
        }
        
        AttackMessage msg;
        size_t offset = 0;
        
        msg.type = static_cast<MessageType>(buffer[offset++]);
        std::memcpy(&msg.playerId, buffer + offset, 4);
        offset += 4;
        std::memcpy(&msg.targetPlayerId, buffer + offset, 4);
        
        return msg;
    }
};
```

**단위 테스트 코드**

이제 이 프로토콜이 정확히 작동하는지 테스트한다. C++에서는 Google Test(gtest)나 Catch2 같은 프레임워크를 사용할 수 있다. 여기서는 간단한 테스트 구조를 보여준다.

```cpp
// ProtocolTest.cpp
#include "GameProtocol.h"
#include <iostream>
#include <cassert>

// 테스트 프레임워크 (매우 간단함)
class TestRunner {
public:
    static int RunAllTests() {
        int passed = 0;
        int failed = 0;
        
        std::cout << "Running Protocol Tests...\n" << std::endl;
        
        try {
            TestMoveMessageSerialization();
            std::cout << "[PASS] TestMoveMessageSerialization" << std::endl;
            passed++;
        } catch (const std::exception& e) {
            std::cout << "[FAIL] TestMoveMessageSerialization: " << e.what() << std::endl;
            failed++;
        }
        
        try {
            TestMoveMessageDeserialization();
            std::cout << "[PASS] TestMoveMessageDeserialization" << std::endl;
            passed++;
        } catch (const std::exception& e) {
            std::cout << "[FAIL] TestMoveMessageDeserialization: " << e.what() << std::endl;
            failed++;
        }
        
        try {
            TestRoundTripSerialization();
            std::cout << "[PASS] TestRoundTripSerialization" << std::endl;
            passed++;
        } catch (const std::exception& e) {
            std::cout << "[FAIL] TestRoundTripSerialization: " << e.what() << std::endl;
            failed++;
        }
        
        try {
            TestAttackMessageSerialization();
            std::cout << "[PASS] TestAttackMessageSerialization" << std::endl;
            passed++;
        } catch (const std::exception& e) {
            std::cout << "[FAIL] TestAttackMessageSerialization: " << e.what() << std::endl;
            failed++;
        }
        
        try {
            TestBufferTooSmall();
            std::cout << "[PASS] TestBufferTooSmall" << std::endl;
            passed++;
        } catch (const std::exception& e) {
            std::cout << "[FAIL] TestBufferTooSmall: " << e.what() << std::endl;
            failed++;
        }
        
        std::cout << "\n--- Test Results ---" << std::endl;
        std::cout << "Passed: " << passed << ", Failed: " << failed << std::endl;
        
        return failed;
    }

private:
    // 테스트: MoveMessage가 올바르게 직렬화되는가?
    static void TestMoveMessageSerialization() {
        MoveMessage msg;
        msg.type = MessageType::PlayerMove;
        msg.playerId = 12345;
        msg.posX = 123.456f;
        msg.posY = 789.012f;
        msg.direction = 2;
        
        uint8_t buffer[100];
        msg.Serialize(buffer, 100);
        
        // 첫 바이트는 메시지 타입이어야 한다
        assert(buffer[0] == static_cast<uint8_t>(MessageType::PlayerMove));
        
        // 다른 필드들을 검증한다
        uint32_t playerId;
        std::memcpy(&playerId, buffer + 1, 4);
        assert(playerId == 12345);
        
        float posX;
        std::memcpy(&posX, buffer + 5, 4);
        assert(posX == 123.456f);
    }
    
    // 테스트: MoveMessage가 올바르게 역직렬화되는가?
    static void TestMoveMessageDeserialization() {
        MoveMessage original;
        original.type = MessageType::PlayerMove;
        original.playerId = 54321;
        original.posX = 456.789f;
        original.posY = 123.456f;
        original.direction = 3;
        
        uint8_t buffer[100];
        original.Serialize(buffer, 100);
        
        MoveMessage deserialized = MoveMessage::Deserialize(buffer, 100);
        
        assert(deserialized.type == original.type);
        assert(deserialized.playerId == original.playerId);
        assert(deserialized.posX == original.posX);
        assert(deserialized.posY == original.posY);
        assert(deserialized.direction == original.direction);
    }
    
    // 테스트: 직렬화 후 역직렬화하면 원래대로 복구되는가?
    static void TestRoundTripSerialization() {
        MoveMessage original;
        original.type = MessageType::PlayerMove;
        original.playerId = 99999;
        original.posX = 111.222f;
        original.posY = 333.444f;
        original.direction = 1;
        
        uint8_t buffer[MoveMessage::SERIALIZED_SIZE];
        original.Serialize(buffer, MoveMessage::SERIALIZED_SIZE);
        
        MoveMessage recovered = MoveMessage::Deserialize(buffer, MoveMessage::SERIALIZED_SIZE);
        
        // 모든 필드가 같아야 한다
        assert(recovered.type == original.type);
        assert(recovered.playerId == original.playerId);
        assert(recovered.posX == original.posX);
        assert(recovered.posY == original.posY);
        assert(recovered.direction == original.direction);
    }
    
    // 테스트: AttackMessage도 올바르게 직렬화/역직렬화되는가?
    static void TestAttackMessageSerialization() {
        AttackMessage original;
        original.type = MessageType::PlayerAttack;
        original.playerId = 100;
        original.targetPlayerId = 200;
        
        uint8_t buffer[100];
        original.Serialize(buffer, 100);
        
        AttackMessage deserialized = AttackMessage::Deserialize(buffer, 100);
        
        assert(deserialized.type == original.type);
        assert(deserialized.playerId == original.playerId);
        assert(deserialized.targetPlayerId == original.targetPlayerId);
    }
    
    // 테스트: 버퍼가 너무 작으면 예외를 던지는가?
    static void TestBufferTooSmall() {
        MoveMessage msg;
        msg.type = MessageType::PlayerMove;
        msg.playerId = 1;
        msg.posX = 1.0f;
        msg.posY = 1.0f;
        msg.direction = 0;
        
        uint8_t buffer[5];  // 너무 작은 버퍼
        
        try {
            msg.Serialize(buffer, 5);
            throw std::runtime_error("Expected exception was not thrown");
        } catch (const std::runtime_error& e) {
            // 예상된 예외다
        }
    }
};

// 사용 예제
int main() {
    return TestRunner::RunAllTests();
}
```

**단위 테스트의 중요성**

단위 테스트는 다음과 같은 이점을 제공한다:

1. **초기 버그 발견**: 작은 단위로 테스트하면 버그를 빨리 찾을 수 있다.

2. **회귀 방지(Regression Prevention)**: 나중에 코드를 수정했을 때, 이전 테스트를 다시 돌려서 새로운 버그가 생기지 않았는지 확인할 수 있다.

3. **문서화**: 테스트 코드는 함수가 어떻게 사용되어야 하는지 보여주는 예제가 된다.

4. **자신감**: 테스트가 통과하면 코드가 정말로 작동한다는 확신을 가질 수 있다.

## 25.2 부하 테스트 도구
단위 테스트는 개별 함수를 테스트하지만, 실제 게임 서버는 수백 명의 플레이어가 동시에 접속한다. 이런 상황을 테스트하는 것을 부하 테스트(Load Test)라고 부른다.

### 다중 클라이언트 시뮬레이터
실제 플레이어를 수천 명 고용할 수는 없으므로, 클라이언트를 시뮬레이션하는 프로그램을 만들어야 한다.

```cpp
// LoadTestSimulator.h
#pragma once
#include <cstdint>
#include <vector>
#include <thread>
#include <atomic>
#include <chrono>
#include <random>
#include <memory>

// 시뮬레이션된 플레이어
class SimulatedPlayer {
public:
    explicit SimulatedPlayer(uint32_t playerId);
    
    // 플레이어를 실행한다 (블로킹)
    void Run(uint32_t durationSeconds);
    
    // 플레이어를 중지한다
    void Stop();
    
    // 통계를 가져온다
    uint64_t GetMessagesSent() const { return m_messagesSent; }
    uint64_t GetMessagesReceived() const { return m_messagesReceived; }
    uint64_t GetErrors() const { return m_errors; }

private:
    // 게임 로직을 시뮬레이션한다
    void SimulateGameplay();
    
    // 랜덤 이동을 한다
    void SendMoveMessage();
    
    // 공격을 한다
    void SendAttackMessage();
    
    // 서버로부터 메시지를 받는다 (시뮬레이션)
    void ReceiveMessage();
    
    uint32_t m_playerId;
    std::atomic<uint64_t> m_messagesSent;
    std::atomic<uint64_t> m_messagesReceived;
    std::atomic<uint64_t> m_errors;
    std::atomic<bool> m_isRunning;
    
    float m_currentX;
    float m_currentY;
    std::mt19937 m_randomGenerator;
};

// 부하 테스트 코디네이터
class LoadTestCoordinator {
public:
    LoadTestCoordinator(uint32_t numPlayers, uint32_t durationSeconds);
    
    // 테스트를 실행한다
    void Run();
    
    // 결과를 출력한다
    void PrintResults() const;

private:
    uint32_t m_numPlayers;
    uint32_t m_durationSeconds;
    std::vector<std::unique_ptr<SimulatedPlayer>> m_players;
    std::vector<std::thread> m_threads;
};
```

```cpp
// LoadTestSimulator.cpp
#include "LoadTestSimulator.h"
#include <iostream>
#include <iomanip>

SimulatedPlayer::SimulatedPlayer(uint32_t playerId)
    : m_playerId(playerId), m_messagesSent(0), m_messagesReceived(0),
      m_errors(0), m_isRunning(false), m_currentX(500.0f), m_currentY(500.0f) {
    
    // 무작위 생성기 초기화
    m_randomGenerator.seed(playerId);
}

void SimulatedPlayer::Run(uint32_t durationSeconds) {
    m_isRunning = true;
    auto startTime = std::chrono::high_resolution_clock::now();
    
    std::cout << "Player " << m_playerId << " started" << std::endl;
    
    while (m_isRunning) {
        auto currentTime = std::chrono::high_resolution_clock::now();
        auto elapsed = std::chrono::duration_cast<std::chrono::seconds>(
            currentTime - startTime).count();
        
        if (elapsed >= durationSeconds) {
            break;
        }
        
        // 게임플레이를 시뮬레이션한다
        SimulateGameplay();
        
        // 약간의 지연을 추가한다 (타이밍 시뮬레이션)
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
    }
    
    std::cout << "Player " << m_playerId << " stopped. "
              << "Sent: " << m_messagesSent << ", Received: " << m_messagesReceived << std::endl;
}

void SimulatedPlayer::Stop() {
    m_isRunning = false;
}

void SimulatedPlayer::SimulateGameplay() {
    // 이 플레이어가 무엇을 할지 랜덤하게 결정한다
    std::uniform_int_distribution<int> actionDist(0, 9);
    int action = actionDist(m_randomGenerator);
    
    if (action < 6) {
        // 60% 확률로 이동
        SendMoveMessage();
    } else if (action < 9) {
        // 30% 확률로 공격
        SendAttackMessage();
    } else {
        // 10% 확률로 대기
        ReceiveMessage();
    }
}

void SimulatedPlayer::SendMoveMessage() {
    // 이동 방향을 랜덤하게 결정한다
    std::uniform_real_distribution<float> moveDist(-10.0f, 10.0f);
    m_currentX += moveDist(m_randomGenerator);
    m_currentY += moveDist(m_randomGenerator);
    
    // 맵 경계를 벗어나지 않도록 한다
    m_currentX = std::max(0.0f, std::min(m_currentX, 1000.0f));
    m_currentY = std::max(0.0f, std::min(m_currentY, 1000.0f));
    
    // 실제로는 서버로 패킷을 보낼 것이다
    // 여기서는 시뮬레이션만 한다
    m_messagesSent++;
}

void SimulatedPlayer::SendAttackMessage() {
    // 공격 대상을 랜덤하게 선택한다
    std::uniform_int_distribution<uint32_t> targetDist(1, 1000);
    uint32_t targetPlayerId = targetDist(m_randomGenerator);
    
    // 실제로는 서버로 패킷을 보낼 것이다
    m_messagesSent++;
}

void SimulatedPlayer::ReceiveMessage() {
    // 서버로부터 메시지를 받는다 (시뮬레이션)
    // 실제로는 네트워크에서 기다릴 것이다
    m_messagesReceived++;
}

LoadTestCoordinator::LoadTestCoordinator(uint32_t numPlayers, uint32_t durationSeconds)
    : m_numPlayers(numPlayers), m_durationSeconds(durationSeconds) {
    
    // 각 플레이어를 생성한다
    for (uint32_t i = 0; i < numPlayers; ++i) {
        m_players.push_back(std::make_unique<SimulatedPlayer>(i + 1));
    }
}

void LoadTestCoordinator::Run() {
    std::cout << "Starting load test with " << m_numPlayers << " players for "
              << m_durationSeconds << " seconds" << std::endl;
    
    auto startTime = std::chrono::high_resolution_clock::now();
    
    // 각 플레이어를 별도 스레드에서 실행한다
    for (auto& player : m_players) {
        m_threads.push_back(std::thread(
            [&player, this]() { player->Run(m_durationSeconds); }
        ));
    }
    
    // 모든 스레드가 완료될 때까지 기다린다
    for (auto& thread : m_threads) {
        if (thread.joinable()) {
            thread.join();
        }
    }
    
    auto endTime = std::chrono::high_resolution_clock::now();
    auto duration = std::chrono::duration_cast<std::chrono::milliseconds>(
        endTime - startTime).count();
    
    std::cout << "\nLoad test completed in " << duration << " ms" << std::endl;
    
    PrintResults();
}

void LoadTestCoordinator::PrintResults() const {
    uint64_t totalMessagesSent = 0;
    uint64_t totalMessagesReceived = 0;
    uint64_t totalErrors = 0;
    
    std::cout << "\n--- Load Test Results ---" << std::endl;
    std::cout << std::left << std::setw(10) << "Player ID"
              << std::setw(15) << "Sent"
              << std::setw(15) << "Received"
              << std::setw(10) << "Errors" << std::endl;
    std::cout << std::string(50, '-') << std::endl;
    
    for (const auto& player : m_players) {
        uint64_t sent = player->GetMessagesSent();
        uint64_t received = player->GetMessagesReceived();
        uint64_t errors = player->GetErrors();
        
        totalMessagesSent += sent;
        totalMessagesReceived += received;
        totalErrors += errors;
    }
    
    std::cout << "\nTotal Messages Sent: " << totalMessagesSent << std::endl;
    std::cout << "Total Messages Received: " << totalMessagesReceived << std::endl;
    std::cout << "Total Errors: " << totalErrors << std::endl;
    
    if (totalMessagesSent > 0) {
        double errorRate = (static_cast<double>(totalErrors) / totalMessagesSent) * 100.0;
        std::cout << "Error Rate: " << std::fixed << std::setprecision(2)
                  << errorRate << "%" << std::endl;
    }
    
    double messagesPerSecond = static_cast<double>(totalMessagesSent) / m_durationSeconds;
    std::cout << "Messages Per Second: " << std::fixed << std::setprecision(0)
              << messagesPerSecond << std::endl;
}

// 사용 예제
int main() {
    // 100명의 플레이어로 60초간 테스트한다
    LoadTestCoordinator coordinator(100, 60);
    coordinator.Run();
    
    return 0;
}
```

**부하 테스트의 의미**

부하 테스트를 통해 우리는 다음을 확인할 수 있다:

1. **처리량(Throughput)**: 서버가 초당 몇 개의 메시지를 처리할 수 있는가?

2. **응답 시간(Response Time)**: 클라이언트가 보낸 요청에 서버가 얼마나 빨리 응답하는가?

3. **안정성**: 장시간 부하가 걸린 상태에서 서버가 계속 작동하는가? 메모리 누수나 성능 저하가 일어나는가?

4. **병목 지점**: 어디서 성능이 떨어지는가? CPU, 메모리, 네트워크 중 어디가 문제인가?

## 25.3 네트워크 조건 시뮬레이션
실제 네트워크는 완벽하지 않다. 지연(latency), 패킷 손실(packet loss), 대역폭 제약(bandwidth limitation) 같은 문제들이 일어난다. 게임 서버는 이런 악조건에서도 작동해야 한다.

### 지연과 패킷 손실 주입
네트워크 문제를 시뮬레이션하는 도구를 만들어보자.

```cpp
// NetworkConditionSimulator.h
#pragma once
#include <cstdint>
#include <vector>
#include <deque>
#include <random>
#include <mutex>
#include <chrono>

// 네트워크 상태를 정의한다
struct NetworkCondition {
    uint32_t latencyMs;         // 패킷이 도착하는 데 걸리는 시간 (밀리초)
    float packetLossRate;       // 패킷 손실 비율 (0.0 ~ 1.0)
    uint32_t bandwidthMbps;     // 대역폭 (Mbps)
    
    // 정상 네트워크 조건
    static NetworkCondition Normal() {
        return {10, 0.0f, 100};
    }
    
    // 나쁜 네트워크 조건 (WiFi)
    static NetworkCondition Poor() {
        return {100, 0.05f, 10};
    }
    
    // 매우 나쁜 네트워크 조건 (모바일)
    static NetworkCondition VeryPoor() {
        return {500, 0.2f, 2};
    }
};

// 전송 대기 중인 패킷
struct PendingPacket {
    std::vector<uint8_t> data;
    std::chrono::high_resolution_clock::time_point deliveryTime;
    bool shouldDrop;
};

// 네트워크 조건을 시뮬레이션한다
class NetworkConditionSimulator {
public:
    NetworkConditionSimulator(const NetworkCondition& condition);
    
    // 패킷을 전송한다
    void SendPacket(const uint8_t* data, size_t size);
    
    // 대기 중인 패킷 중 전달 가능한 것들을 반환한다
    std::vector<std::vector<uint8_t>> ReceivePackets();
    
    // 네트워크 조건을 변경한다 (동적 변경 가능)
    void SetNetworkCondition(const NetworkCondition& condition);
    
    // 현재 통계
    uint64_t GetTotalSent() const { return m_totalSent; }
    uint64_t GetTotalDropped() const { return m_totalDropped; }
    uint64_t GetTotalDelivered() const { return m_totalDelivered; }

private:
    // 패킷이 손실될지 결정한다
    bool ShouldDropPacket() const;
    
    NetworkCondition m_condition;
    std::deque<PendingPacket> m_pendingPackets;
    mutable std::mutex m_mutex;
    
    std::mt19937 m_randomGenerator;
    
    uint64_t m_totalSent;
    uint64_t m_totalDropped;
    uint64_t m_totalDelivered;
};
```

```cpp
// NetworkConditionSimulator.cpp
#include "NetworkConditionSimulator.h"
#include <algorithm>
#include <iostream>

NetworkConditionSimulator::NetworkConditionSimulator(const NetworkCondition& condition)
    : m_condition(condition), m_totalSent(0), m_totalDropped(0), m_totalDelivered(0) {
    
    m_randomGenerator.seed(std::random_device{}());
}

void NetworkConditionSimulator::SendPacket(const uint8_t* data, size_t size) {
    std::lock_guard<std::mutex> lock(m_mutex);
    
    PendingPacket packet;
    packet.data.assign(data, data + size);
    packet.shouldDrop = ShouldDropPacket();
    
    // 지연을 계산한다
    auto now = std::chrono::high_resolution_clock::now();
    auto latency = std::chrono::milliseconds(m_condition.latencyMs);
    packet.deliveryTime = now + latency;
    
    m_pendingPackets.push_back(packet);
    m_totalSent++;
    
    if (packet.shouldDrop) {
        m_totalDropped++;
    }
}

std::vector<std::vector<uint8_t>> NetworkConditionSimulator::ReceivePackets() {
    std::lock_guard<std::mutex> lock(m_mutex);
    
    std::vector<std::vector<uint8_t>> result;
    auto now = std::chrono::high_resolution_clock::now();
    
    // 전달 시간이 된 패킷들을 찾는다
    auto it = m_pendingPackets.begin();
    while (it != m_pendingPackets.end()) {
        if (it->deliveryTime <= now) {
            // 손실되지 않은 패킷만 반환한다
            if (!it->shouldDrop) {
                result.push_back(it->data);
                m_totalDelivered++;
            }
            
            it = m_pendingPackets.erase(it);
        } else {
            ++it;
        }
    }
    
    return result;
}

void NetworkConditionSimulator::SetNetworkCondition(const NetworkCondition& condition) {
    std::lock_guard<std::mutex> lock(m_mutex);
    m_condition = condition;
}

bool NetworkConditionSimulator::ShouldDropPacket() const {
    std::uniform_real_distribution<float> dist(0.0f, 1.0f);
    float random = dist(m_randomGenerator);
    return random < m_condition.packetLossRate;
}
```

## 25.4 로깅 시스템
게임 서버에서 무엇이 일어나고 있는지 알아야 한다. 어떤 플레이어가 접속했고, 언제 에러가 발생했으며, 서버의 어느 부분이 느린지 알아야 문제를 찾을 수 있다.

### 비동기 로거 구현
로깅 성능이 중요하므로 비동기로 작동하는 로거를 만들자.

```cpp
// AsyncLogger.h
#pragma once
#include <string>
#include <queue>
#include <thread>
#include <mutex>
#include <condition_variable>
#include <memory>
#include <fstream>
#include <chrono>
#include <sstream>

// 로그 레벨
enum class LogLevel {
    DEBUG = 0,
    INFO = 1,
    WARNING = 2,
    ERROR = 3,
    CRITICAL = 4
};

// 하나의 로그 메시지
struct LogMessage {
    LogLevel level;
    std::string message;
    std::chrono::system_clock::time_point timestamp;
    uint32_t threadId;
};

// 비동기 로거
class AsyncLogger {
public:
    AsyncLogger(const std::string& filename);
    ~AsyncLogger();
    
    // 로그를 기록한다 (빠른 추가, 실제 쓰기는 비동기)
    void Log(LogLevel level, const std::string& message);
    
    // 편의 함수
    void Debug(const std::string& msg) { Log(LogLevel::DEBUG, msg); }
    void Info(const std::string& msg) { Log(LogLevel::INFO, msg); }
    void Warning(const std::string& msg) { Log(LogLevel::WARNING, msg); }
    void Error(const std::string& msg) { Log(LogLevel::ERROR, msg); }
    void Critical(const std::string& msg) { Log(LogLevel::CRITICAL, msg); }
    
    // 모든 로그를 파일에 쓴다
    void Flush();
    
    // 로거를 중지한다
    void Stop();

private:
    // 로깅 워커 스레드
    void LoggingThread();
    
    // 로그 메시지를 문자열로 포맷한다
    std::string FormatMessage(const LogMessage& msg);
    
    std::string m_filename;
    std::ofstream m_file;
    
    std::queue<LogMessage> m_logQueue;
    mutable std::mutex m_queueMutex;
    std::condition_variable m_queueCV;
    
    std::thread m_workerThread;
    std::atomic<bool> m_isRunning;
};

// 글로벌 로거
extern AsyncLogger& GetGlobalLogger();
```

```cpp
// AsyncLogger.cpp
#include "AsyncLogger.h"
#include <iostream>
#include <iomanip>
#include <sstream>
#include <ctime>

AsyncLogger::AsyncLogger(const std::string& filename)
    : m_filename(filename), m_isRunning(true) {
    
    m_file.open(filename, std::ios::app);
    if (!m_file.is_open()) {
        throw std::runtime_error("Failed to open log file: " + filename);
    }
    
    m_workerThread = std::thread(&AsyncLogger::LoggingThread, this);
}

AsyncLogger::~AsyncLogger() {
    Stop();
}

void AsyncLogger::Log(LogLevel level, const std::string& message) {
    LogMessage msg;
    msg.level = level;
    msg.message = message;
    msg.timestamp = std::chrono::system_clock::now();
    msg.threadId = std::this_thread::get_id();  // 간단한 구현
    
    {
        std::lock_guard<std::mutex> lock(m_queueMutex);
        m_logQueue.push(msg);
    }
    
    // 큐가 비어있지 않으면 워커 스레드를 깨운다
    m_queueCV.notify_one();
}

void AsyncLogger::Flush() {
    // 모든 로그가 파일에 쓰일 때까지 기다린다
    {
        std::unique_lock<std::mutex> lock(m_queueMutex);
        m_queueCV.wait(lock, [this] { return m_logQueue.empty(); });
    }
    
    m_file.flush();
}

void AsyncLogger::Stop() {
    m_isRunning = false;
    m_queueCV.notify_one();
    
    if (m_workerThread.joinable()) {
        m_workerThread.join();
    }
    
    // 남은 로그를 모두 쓴다
    while (!m_logQueue.empty()) {
        LogMessage msg = m_logQueue.front();
        m_logQueue.pop();
        
        m_file << FormatMessage(msg) << std::endl;
    }
    
    m_file.close();
}

void AsyncLogger::LoggingThread() {
    while (m_isRunning) {
        std::unique_lock<std::mutex> lock(m_queueMutex);
        
        // 로그가 들어올 때까지 기다린다
        m_queueCV.wait(lock, [this] { return !m_logQueue.empty() || !m_isRunning; });
        
        // 큐에 있는 모든 로그를 처리한다
        while (!m_logQueue.empty()) {
            LogMessage msg = m_logQueue.front();
            m_logQueue.pop();
            
            lock.unlock();
            m_file << FormatMessage(msg) << std::endl;
            lock.lock();
        }
    }
}

std::string AsyncLogger::FormatMessage(const LogMessage& msg) {
    std::ostringstream oss;
    
    // 시간
    auto time = std::chrono::system_clock::to_time_t(msg.timestamp);
    auto ms = std::chrono::duration_cast<std::chrono::milliseconds>(
        msg.timestamp.time_since_epoch()) % 1000;
    
    oss << std::put_time(std::localtime(&time), "%Y-%m-%d %H:%M:%S");
    oss << "." << std::setfill('0') << std::setw(3) << ms.count();
    
    // 로그 레벨
    oss << " [";
    switch (msg.level) {
        case LogLevel::DEBUG: oss << "DEBUG"; break;
        case LogLevel::INFO: oss << "INFO"; break;
        case LogLevel::WARNING: oss << "WARN"; break;
        case LogLevel::ERROR: oss << "ERROR"; break;
        case LogLevel::CRITICAL: oss << "CRIT"; break;
    }
    oss << "]";
    
    // 메시지
    oss << " " << msg.message;
    
    return oss.str();
}

// 글로벌 로거 구현
AsyncLogger& GetGlobalLogger() {
    static AsyncLogger logger("server.log");
    return logger;
}

// 사용 예제
void ExampleLogging() {
    auto& logger = GetGlobalLogger();
    
    logger.Info("Server started");
    logger.Debug("Debug information");
    logger.Warning("This is a warning");
    logger.Error("An error occurred");
    
    logger.Flush();  // 모든 로그를 파일에 쓴다
}
```

**로깅의 중요성**

로깅은 다음과 같은 이점을 제공한다:

1. **문제 추적**: 에러가 발생했을 때, 로그를 통해 무엇이 일어났는지 알 수 있다.

2. **성능 분석**: 언제 어떤 작업이 얼마나 오래 걸렸는지 기록할 수 있다.

3. **보안**: 누가 언제 무엇을 했는지 기록해서 부정행위를 감지할 수 있다.

4. **배포**: 프로덕션 서버에서 발생한 문제를 로그를 통해 원격으로 진단할 수 있다.

## 25.5 디버깅 기법
문제가 발생했을 때 어떻게 찾아내고 해결할까?

### 패킷 덤프와 분석
게임 서버의 문제는 대부분 네트워크 패킷과 관련이 있다. 패킷이 올바르게 전송되고 있는지 확인하는 것이 중요하다.

```cpp
// PacketDumper.h
#pragma once
#include <cstdint>
#include <vector>
#include <string>
#include <iostream>
#include <iomanip>

// 패킷을 분석하고 출력한다
class PacketDumper {
public:
    // 패킷을 16진수로 덤프한다
    static void DumpPacket(const uint8_t* data, size_t size,
                          const std::string& label = "") {
        if (!label.empty()) {
            std::cout << "=== " << label << " ===" << std::endl;
        }
        
        std::cout << "Size: " << size << " bytes" << std::endl;
        std::cout << std::hex << std::setfill('0');
        
        for (size_t i = 0; i < size; ++i) {
            if (i % 16 == 0) {
                std::cout << std::endl << std::setw(4) << i << ": ";
            }
            std::cout << std::setw(2) << static_cast<int>(data[i]) << " ";
        }
        
        std::cout << std::endl;
        std::cout << std::dec << std::setfill(' ');
    }
    
    // 패킷을 분석해서 의미 있는 필드를 출력한다
    static void AnalyzePacket(const uint8_t* data, size_t size) {
        if (size < 1) {
            std::cout << "Error: Packet too small" << std::endl;
            return;
        }
        
        uint8_t messageType = data[0];
        std::cout << "Message Type: 0x" << std::hex << static_cast<int>(messageType)
                  << std::dec << std::endl;
        
        // 메시지 타입별로 분석한다
        switch (messageType) {
            case 0x01:  // PlayerMove
                if (size >= 14) {
                    uint32_t playerId;
                    float posX, posY;
                    uint8_t direction;
                    
                    std::memcpy(&playerId, data + 1, 4);
                    std::memcpy(&posX, data + 5, 4);
                    std::memcpy(&posY, data + 9, 4);
                    direction = data[13];
                    
                    std::cout << "PlayerMove: ID=" << playerId
                              << " Pos=(" << posX << ", " << posY << ")"
                              << " Direction=" << static_cast<int>(direction) << std::endl;
                }
                break;
                
            case 0x02:  // PlayerAttack
                if (size >= 9) {
                    uint32_t playerId, targetId;
                    std::memcpy(&playerId, data + 1, 4);
                    std::memcpy(&targetId, data + 5, 4);
                    
                    std::cout << "PlayerAttack: Attacker=" << playerId
                              << " Target=" << targetId << std::endl;
                }
                break;
                
            default:
                std::cout << "Unknown message type" << std::endl;
        }
    }
    
    // 두 패킷을 비교한다 (테스트할 때 유용)
    static bool ComparePackets(const uint8_t* packet1, size_t size1,
                              const uint8_t* packet2, size_t size2) {
        if (size1 != size2) {
            std::cout << "Size mismatch: " << size1 << " vs " << size2 << std::endl;
            return false;
        }
        
        for (size_t i = 0; i < size1; ++i) {
            if (packet1[i] != packet2[i]) {
                std::cout << "Difference at byte " << i << ": "
                          << std::hex << std::setw(2) << std::setfill('0')
                          << static_cast<int>(packet1[i]) << " vs "
                          << static_cast<int>(packet2[i]) << std::dec << std::endl;
                return false;
            }
        }
        
        return true;
    }
};

// 사용 예제
void DebugPackets() {

    testPacket[0] = 0x01;  // PlayerMove

    std::memcpy(testPacket + 1, &playerId, 4);

    std::memcpy(testPacket + 5, &posX, 4);
    
    float posY = 789.012f;
    std::memcpy(testPacket + 9, &posY, 4);
    
    testPacket[13] = 2;  // Direction
    
    // 패킷을 분석한다
```

```cpp
bool LoadTestScenario::Execute() {
    // 간단한 부하 테스트: 메시지를 빠르게 생성하고 처리한다
    uint64_t totalMessages = 0;
    uint64_t totalErrors = 0;
    
    for (uint32_t i = 0; i < m_numPlayers; ++i) {
        for (uint32_t j = 0; j < 100; ++j) {  // 플레이어당 100개 메시지
            try {
                MoveMessage msg;
                msg.type = MessageType::PlayerMove;
                msg.playerId = i;
                msg.posX = static_cast<float>(i * 10);
                msg.posY = static_cast<float>(j * 10);
                msg.direction = j % 4;
                
                uint8_t buffer[MoveMessage::SERIALIZED_SIZE];
                msg.Serialize(buffer, MoveMessage::SERIALIZED_SIZE);
                
                MoveMessage recovered = MoveMessage::Deserialize(buffer, MoveMessage::SERIALIZED_SIZE);
                totalMessages++;
            } catch (...) {
                totalErrors++;
            }
        }
    }
    
    // 에러율이 1% 미만이면 성공
    return totalErrors < (totalMessages / 100);
}

void IntegratedTestFramework::AddTestScenario(std::shared_ptr<TestScenario> scenario) {
    m_scenarios.push_back(scenario);

void IntegratedTestFramework::RunAllTests() {
    std::cout << "\n========== Running Integrated Tests ==========" << std::endl;
    std::cout << "Total scenarios: " << m_scenarios.size() << std::endl;
    std::cout << std::string(45, '=') << "\n" << std::endl;
    
    for (auto& scenario : m_scenarios) {
        std::cout << "Running: " << scenario->GetName() << "... ";
        std::cout.flush();
        
        auto startTime = std::chrono::high_resolution_clock::now();
        
        bool passed = false;
        std::string errorMsg;
        
        try {
            passed = scenario->Execute();
        } catch (const std::exception& e) {
            errorMsg = e.what();
            passed = false;
        }
        
        auto endTime = std::chrono::high_resolution_clock::now();
        auto duration = std::chrono::duration_cast<std::chrono::milliseconds>(
            endTime - startTime).count();
        
        TestResult result;
        result.testName = scenario->GetName();
        result.passed = passed;
        result.errorMessage = errorMsg;
        result.executionTimeMs = duration;
        m_results.push_back(result);
        
        if (passed) {
            std::cout << "[PASS]";
        } else {
            std::cout << "[FAIL]";
        }
        std::cout << " (" << duration << " ms)" << std::endl;
    }
}

void IntegratedTestFramework::PrintResults() const {
    std::cout << "\n========== Test Results Summary ==========" << std::endl;
    
    int passCount = 0;
    int failCount = 0;
    uint64_t totalTime = 0;
    
    std::cout << std::left << std::setw(30) << "Test Name"
              << std::setw(10) << "Status"
              << std::setw(15) << "Time (ms)" << std::endl;
    std::cout << std::string(55, '-') << std::endl;
    
    for (const auto& result : m_results) {
        std::cout << std::left << std::setw(30) << result.testName
                  << std::setw(10) << (result.passed ? "PASS" : "FAIL")
                  << std::setw(15) << result.executionTimeMs << std::endl;
        
        if (result.passed) {
            passCount++;
        } else {
            failCount++;
            if (!result.errorMessage.empty()) {
                std::cout << "  Error: " << result.errorMessage << std::endl;
            }
        }
        
        totalTime += result.executionTimeMs;
    }
    
    std::cout << std::string(55, '=') << std::endl;
    std::cout << "Passed: " << passCount << ", Failed: " << failCount
              << ", Total Time: " << totalTime << " ms" << std::endl;
}

void IntegratedTestFramework::SaveResults(const std::string& filename) const {
    std::ofstream file(filename);
    
    file << "Test Results Report\n";
    file << "==================\n\n";
    
    for (const auto& result : m_results) {
        file << "Test: " << result.testName << "\n";
        file << "Status: " << (result.passed ? "PASS" : "FAIL") << "\n";
        file << "Time: " << result.executionTimeMs << " ms\n";
        
        if (!result.errorMessage.empty()) {
            file << "Error: " << result.errorMessage << "\n";
        }
        
        file << "\n";
    }
}
```

**이 구현의 설계 결정**

1. **확장 가능한 구조**: 새로운 테스트 시나리오를 추가하려면 `TestScenario`를 상속해서 `Execute()` 메서드를 구현하기만 하면 된다.

2. **자동화**: 모든 테스트가 자동으로 실행되고 결과가 수집된다. 개발자는 결과만 확인하면 된다.

3. **측정 가능성**: 각 테스트의 실행 시간을 기록해서 성능 저하를 감지할 수 있다.

4. **결과 저장**: 테스트 결과를 파일에 저장해서 나중에 분석할 수 있다.

**주요 부분 상세 해설**

`ProtocolTestScenario::Execute()` 메서드는 게임 서버의 핵심인 프로토콜이 올바르게 작동하는지 확인한다. 메시지를 직렬화한 후 역직렬화해서 원래 값과 비교한다. 이것은 가장 기본적이면서도 가장 중요한 테스트다.

`LoadTestScenario::Execute()` 메서드는 많은 양의 메시지를 빠르게 처리해서 서버의 처리량을 확인한다. 에러 없이 처리된 메시지의 비율이 높으면 서버가 부하를 잘 견딘다는 의미다.

`NetworkConditionTestScenario::Execute()` 메서드는 다양한 네트워크 조건에서 서버가 작동하는지 확인한다. 정상적인 네트워크에서는 대부분의 패킷이 도착해야 하고, 나쁜 네트워크에서도 언젠가는 패킷이 도착해야 한다.

**개선 방안과 확장 가능성**

현재 프레임워크는 기본 구조만 제공한다. 다음과 같은 개선이 가능하다:

1. **성능 메트릭 계산**: 메시지당 지연 시간, 처리량(throughput), 메모리 사용량 등을 자동으로 계산할 수 있다.

2. **병렬 테스트**: 여러 테스트 시나리오를 동시에 실행해서 테스트 시간을 줄일 수 있다.

3. **동적 테스트 조정**: 시스템의 응답에 따라 부하를 동적으로 조정할 수 있다. 예를 들어, 시스템이 느려지면 부하를 줄인다.

4. **리포팅**: 테스트 결과를 HTML이나 JSON 형식으로 생성해서 더 읽기 쉽게 만들 수 있다.

5. **CI/CD 통합**: Jenkins나 GitHub Actions 같은 CI/CD 시스템과 통합해서 자동으로 테스트를 실행하고 결과를 보고할 수 있다.

6. **회귀 테스트**: 이전 테스트의 성능을 기록해두고, 새로운 변경사항이 성능을 악화시키지 않는지 확인할 수 있다.

## 정리

이 장에서는 게임 서버를 체계적으로 테스트하고 디버깅하는 방법을 배웠다. 단위 테스트로 개별 기능을 확인하고, 부하 테스트로 많은 플레이어를 처리할 수 있는지 검증하고, 네트워크 조건 시뮬레이션으로 악조건에서도 작동하는지 확인했다.

특히 마지막의 통합 테스트 프레임워크는 이 모든 것을 자동으로 수행한다. 개발자는 단지 코드를 커밋하면, 프레임워크가 자동으로 테스트를 실행하고 결과를 보여준다.

게임 서버 개발에서 테스트와 디버깅은 개발 시간의 상당 부분을 차지한다. 초기에 좋은 테스트 프레임워크를 만들어두면, 나중에 버그를 찾고 수정하는 시간을 크게 줄일 수 있다. 이것이 전문적인 개발팀이 항상 강조하는 "테스트를 먼저 작성하라(TDD - Test Driven Development)"는 말의 의미다.

지금까지 여러분은 네트워크 프로그래밍의 기초부터 실제 게임 서버 구현, 성능 최적화, 테스트까지 게임 서버 개발의 전체 과정을 배웠다. 이제 여러분은 실제로 온라인 게임 서버를 만들 수 있는 기초를 갖추었다. 다음 단계는 실제로 프로젝트를 만들어보고, 문제를 마주치고, 해결하면서 경험을 쌓는 것이다.
