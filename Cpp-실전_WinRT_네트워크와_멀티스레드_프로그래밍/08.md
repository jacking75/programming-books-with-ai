# Chapter 8. WinRT 네트워킹 시작
네트워크 프로그래밍은 온라인 게임 서버 개발의 핵심이다. 이 장에서는 WinRT의 네트워킹 기초를 다루면서 네트워크 통신을 위한 기본 개념과 API를 학습한다. Windows.Networking 네임스페이스는 현대적이고 비동기 지향적인 네트워크 API를 제공하며, 이것이 다른 플랫폼의 소켓 API와 어떻게 다른지 이해하는 것이 중요하다.

---

## 8.1 Windows.Networking 네임스페이스 소개
Windows.Networking 네임스페이스는 WinRT의 네트워크 프로그래밍을 위한 중심이다. 전통적인 Winsock API와 달리 이 네임스페이스는 비동기 프로그래밍을 기본으로 설계되었고, 모던 C++의 코루틴과 자연스럽게 통합된다.  
  
이 네임스페이스가 제공하는 주요 컴포넌트는 다음과 같다. **HostName**은 도메인 이름이나 IP 주소를 표현하는 객체로, DNS 해석을 수행한다. **EndpointPair**는 통신의 양 끝점을 정의하는데, 로컬 주소와 포트, 원격 주소와 포트를 한데 포함한다. **StreamSocket**과 **StreamSocketListener**는 TCP 기반 통신을 담당하고, **DatagramSocket**은 UDP 통신을 제공한다.

또한 **Windows.Networking.Connectivity** 서브네임스페이스는 네트워크 연결 상태를 조회하고 모니터링하는 API를 제공한다. 이를 통해 현재 기기의 인터넷 연결 여부, 활성 네트워크 어댑터, 연결 타입 등을 확인할 수 있다.  
  
WinRT 네트워킹의 특징은 비동기 우선이라는 점이다. 모든 I/O 작업은 **IAsyncOperation** 또는 **IAsyncAction** 코루틴으로 반환되며, co_await를 사용하여 결과를 기다린다. 이는 스레드 블로킹을 방지하고 시스템 리소스를 효율적으로 사용하게 한다.

## 8.2 HostName API 이해하기
**HostName** 클래스는 네트워크 주소를 나타내는 기본 단위다. 문자열 형태의 도메인 이름이나 IP 주소를 받아서 이를 검증하고 내부적으로 적절한 형식으로 변환한다.

### 호스트 이름 생성과 검증
HostName을 생성할 때는 생성자에 주소 문자열을 전달한다. 문자열이 유효한 도메인 이름인지 IP 주소인지 자동으로 판단되며, 필요시 DNS 해석이 발생한다.

```cpp
#include <winrt/Windows.Networking.h>
using namespace winrt::Windows::Networking;

// 호스트 이름 생성과 검증
void HostNameExample()
{
    // IPv4 주소로 생성
    HostName ipv4(L"192.168.1.1");
    winrt::hstring type1 = ipv4.Type().ToString();
    // Type은 Ipv4 또는 Ipv6, DomainName 등의 값

    // 도메인 이름으로 생성
    HostName domain(L"example.com");
    winrt::hstring rawName = domain.RawName();
    // RawName()은 원본 문자열을 반환

    // IPv6 주소
    HostName ipv6(L"::1");
    bool isIpv6 = (ipv6.Type() == 
        HostNameType::Ipv6);

    // 호스트 이름 출력
    winrt::hstring displayName = domain.DisplayName();
}
```

HostName의 **Type** 속성은 주소의 종류를 **HostNameType** 열거형으로 반환한다. 가능한 값은 Ipv4, Ipv6, DomainName이다. **RawName**은 입력받은 원본 문자열을 반환하고, **DisplayName**은 표시용 형식의 문자열을 반환한다.

HostName 객체는 비교 연산을 지원한다. 두 HostName 객체가 같은 주소를 나타내는지 확인할 수 있으며, 이는 내부적으로 DNS 해석 결과를 기반으로 비교된다.

## 8.3 EndpointPair 사용하기
**EndpointPair**는 네트워크 통신의 양쪽 끝점을 정의하는 구조다. TCP 연결에서 클라이언트는 자신의 로컬 주소와 포트를 정하고, 연결할 서버의 주소와 포트를 지정해야 한다. EndpointPair는 이 네 가지 정보를 모두 포함한다.

### 로컬/원격 엔드포인트 설정

```cpp
#include <winrt/Windows.Networking.h>
using namespace winrt::Windows::Networking;

void EndpointPairExample()
{
    // 원격 서버 엔드포인트 생성
    HostName remoteHost(L"game.server.com");
    winrt::hstring remoteServiceName(L"8888");
    
    HostName localHost(L"0.0.0.0");
    winrt::hstring localServiceName(L"0");

    // EndpointPair 생성
    // 로컬 호스트, 로컬 포트, 원격 호스트, 원격 포트 순서
    EndpointPair pair(
        localHost,
        localServiceName,
        remoteHost,
        remoteServiceName
    );

    // 엔드포인트 정보 조회
    HostName local = pair.LocalHostName();
    HostName remote = pair.RemoteHostName();
    winrt::hstring localPort = pair.LocalServiceName();
    winrt::hstring remotePort = pair.RemoteServiceName();

    // 표시용 문자열 생성
    winrt::hstring displayStr = pair.LocalHostName().DisplayName();
}
```

EndpointPair의 생성자는 네 개의 매개변수를 받는다. 첫 번째와 두 번째는 로컬 호스트와 포트이고, 세 번째와 네 번째는 원격 호스트와 포트다. 로컬 호스트에 "0.0.0.0"을 지정하면 모든 사용 가능한 네트워크 인터페이스를 의미한다.

포트는 문자열로 지정된다는 점이 특이하다. "8888"이나 "http" 같은 서비스 이름을 전달할 수 있으며, 시스템이 자동으로 해석한다. 로컬 포트에 "0"을 지정하면 시스템이 사용 가능한 포트를 자동으로 할당한다.

EndpointPair는 불변 객체이므로 생성 후 개별 속성을 변경할 수 없다. 다른 설정이 필요하면 새로운 EndpointPair 객체를 생성해야 한다.

## 8.4 네트워크 권한과 Capability
WinRT 애플리케이션이 네트워크에 접근하려면 먼저 애플리케이션 매니페스트 파일에 적절한 권한을 선언해야 한다. 이는 Windows의 샌드박스 모델의 일부이며, 사용자가 애플리케이션이 어떤 리소스에 접근하는지 명확히 알 수 있게 한다.

### 매니페스트 설정 방법
애플리케이션의 **Package.appxmanifest** 파일을 XML 편집기로 열면 다음과 같은 Capabilities 섹션을 찾을 수 있다:

```xml
<Capabilities>
  <Capability Name="internetClient" />
  <Capability Name="internetClientServer" />
  <Capability Name="privateNetworkClientServer" />
</Capabilities>
```

**internetClient** 권한은 인터넷으로의 아웃바운드 연결을 허용한다. 이는 게임 클라이언트가 서버에 연결할 때 필요하다.

**internetClientServer** 권한은 인터넷을 통한 인바운드와 아웃바운드 모두를 허용한다. 서버로 작동하는 경우나 P2P 통신을 할 때 필요하다.

**privateNetworkClientServer** 권한은 로컬 네트워크(LAN)에서의 모든 연결을 허용한다. 회사 네트워크나 가정 네트워크에서 서버에 접근할 때 필요하다.

Visual Studio의 매니페스트 편집기를 사용하면 UI를 통해 이러한 권한을 추가할 수 있다. **Package.appxmanifest** 파일을 더블클릭하고 "Capabilities" 탭으로 이동하여 필요한 권한을 체크박스로 선택하면 된다.

권한이 없으면 런타임에 **0x80072F0D** (HRESULT_FROM_WIN32(ERROR_FILE_NOT_FOUND)) 같은 에러가 발생할 수 있다. 따라서 개발 초기 단계에 필요한 권한을 모두 선언하는 것이 좋다.

## 8.5 네트워크 연결 상태 확인
애플리케이션이 네트워크 작업을 시작하기 전에 연결 상태를 확인하는 것은 중요하다. **Windows.Networking.Connectivity** 네임스페이스는 이를 위한 API를 제공한다. 현재 인터넷 연결 여부, 활성 네트워크 어댑터, 연결 타입(Wi-Fi, 유선, 모바일 등)을 조회할 수 있다.

### NetworkInformation API 사용

```cpp
#include <winrt/Windows.Networking.Connectivity.h>
#include <iostream>
using namespace winrt::Windows::Networking::Connectivity;

void CheckNetworkConnection()
{
    // 인터넷 연결 여부 확인
    NetworkConnectivityLevel level = 
        NetworkInformation::GetInternetConnectionProfile() != nullptr ?
        NetworkConnectivityLevel::InternetAccess :
        NetworkConnectivityLevel::None;

    if (level == NetworkConnectivityLevel::InternetAccess) {
        std::wcout << L"인터넷 연결 가능" << std::endl;
    } else {
        std::wcout << L"인터넷 연결 불가" << std::endl;
    }

    // 활성 네트워크 어댑터 조회
    auto connectionProfiles = 
        NetworkInformation::GetConnectionProfiles();
    
    int profileCount = 0;
    for (const auto& profile : connectionProfiles) {
        profileCount++;
        
        // 연결 이름
        winrt::hstring profileName = profile.ProfileName();
        
        // 연결 타입
        NetworkConnectivityLevel connLevel = 
            profile.GetNetworkConnectivityLevel();
        
        // 네트워크 비용 정보
        ConnectionCost cost = profile.GetConnectionCost();
        bool isMetered = cost.NetworkCostType() == 
            NetworkCostType::Fixed || 
            cost.NetworkCostType() == NetworkCostType::Variable;
        
        std::wcout << L"프로필 " << profileCount << L": " 
                   << profileName.c_str() << std::endl;
        std::wcout << L"  측정된 연결: " 
                   << (isMetered ? L"예" : L"아니오") << std::endl;
    }

    // 구체적 어댑터 정보
    auto adapters = NetworkInformation::GetHostNames();
    
    for (const auto& hostName : adapters) {
        if (hostName.Type() == 
            winrt::Windows::Networking::HostNameType::Ipv4 ||
            hostName.Type() == 
            winrt::Windows::Networking::HostNameType::Ipv6) {
            
            std::wcout << L"주소: " 
                       << hostName.DisplayName().c_str() 
                       << std::endl;
        }
    }
}

// 네트워크 변경 이벤트 모니터링
IInspectable NetworkStatusChanged;

void SetupNetworkChangeNotification()
{
    // 네트워크 상태 변경 시 콜백
    NetworkInformation::NetworkStatusChanged(
        [](const IInspectable& sender) {
            // 네트워크 상태가 변경되면 여기서 처리
            std::wcout << L"네트워크 상태 변경됨" << std::endl;
            CheckNetworkConnection();
        }
    );
}
```

**GetInternetConnectionProfile()** 함수는 현재 인터넷에 연결된 프로필을 반환한다. null이 아니면 인터넷 연결이 가능하다는 뜻이다.

**GetConnectionProfiles()** 함수는 모든 활성 네트워크 연결의 목록을 반환한다. 각 프로필로부터 연결 이름, 타입, 비용 정보를 얻을 수 있다. 측정된 연결(metered connection)은 이동 통신망이나 제한된 데이터 플랜을 의미하므로, 대용량 데이터 전송 시 사용자에게 경고할 때 유용하다.

**GetHostNames()** 함수는 기기의 모든 네트워크 주소를 반환한다. IPv4, IPv6, 로컬 호스트명 등이 포함되며, HostNameType으로 필터링하여 원하는 주소만 얻을 수 있다.

**NetworkStatusChanged** 이벤트는 네트워크 연결 상태가 변할 때 발생한다. Wi-Fi가 연결되거나 끊어질 때, 유선 연결 상태가 변할 때 등이 이에 해당한다. 게임 서버 클라이언트는 이 이벤트를 수신하여 연결을 재시도하거나 사용자에게 알림을 줄 수 있다.

## 8.6 종합 예제: 네트워크 모니터링 도구
이번 종합 예제는 네트워크의 현재 상태를 주기적으로 수집하고 변화를 감지하는 도구를 구현한다. 이는 온라인 게임 클라이언트가 연결을 안정적으로 유지하기 위한 기초가 된다.

### 코드의 목적과 설계
이 예제는 다음을 구현한다:
- 현재 네트워크 연결 상태 조회
- 활성 네트워크 어댑터 정보 수집
- 연결 상태 변경 감지
- 주기적 모니터링과 통계

설계 결정 사항은 다음과 같다. **NetworkMonitor** 클래스는 싱글톤 패턴으로 설계하여 애플리케이션 전체에서 공유되도록 했다. 네트워크 상태 변경 콜백을 통해 실시간으로 변화를 감지하고, 애플리케이션의 다른 부분으로 알림을 전달한다. 타이머를 사용하여 주기적으로 상세 정보를 업데이트하여 느린 변경사항도 감지할 수 있게 했다.

### 구현 코드

```cpp
#include <winrt/Windows.Networking.Connectivity.h>
#include <winrt/Windows.Foundation.h>
#include <iostream>
#include <vector>
#include <memory>
#include <chrono>

using namespace winrt;
using namespace winrt::Windows::Networking;
using namespace winrt::Windows::Networking::Connectivity;
using namespace winrt::Windows::Foundation;

// 네트워크 어댑터 정보 구조체
struct AdapterInfo
{
    hstring ProfileName;
    Networking::HostNameType AddressType;
    hstring Address;
    NetworkConnectivityLevel ConnectivityLevel;
    bool IsMetered;
    std::chrono::system_clock::time_point CaptureTime;
};

// 네트워크 상태 변경 콜백 타입
using NetworkStatusCallback = 
    std::function<void(const std::vector<AdapterInfo>&)>;

class NetworkMonitor
{
private:
    std::vector<AdapterInfo> currentAdapters_;
    std::vector<NetworkStatusCallback> callbacks_;
    bool isInternetAvailable_;
    std::chrono::system_clock::time_point lastStatusChange_;

    // 싱글톤 인스턴스
    static NetworkMonitor* instance_;

    NetworkMonitor() 
        : isInternetAvailable_(false)
    {
        lastStatusChange_ = std::chrono::system_clock::now();
        RefreshAdapterInfo();
        RegisterNetworkChangeHandler();
    }

public:
    ~NetworkMonitor()
    {
        // 이벤트 핸들러 정리
    }

    // 싱글톤 접근
    static NetworkMonitor* GetInstance()
    {
        if (!instance_) {
            instance_ = new NetworkMonitor();
        }
        return instance_;
    }

    // 어댑터 정보 새로고침
    void RefreshAdapterInfo()
    {
        currentAdapters_.clear();
        
        // 인터넷 연결 확인
        auto profile = NetworkInformation::GetInternetConnectionProfile();
        isInternetAvailable_ = (profile != nullptr);

        // 활성 연결 프로필 조회
        auto profiles = NetworkInformation::GetConnectionProfiles();
        
        for (const auto& connProfile : profiles) {
            // 이 프로필의 모든 호스트명 조회
            auto hostNames = NetworkInformation::GetHostNames();
            
            for (const auto& hostName : hostNames) {
                // IPv4와 IPv6만 수집
                if (hostName.Type() == Networking::HostNameType::Ipv4 ||
                    hostName.Type() == Networking::HostNameType::Ipv6) {
                    
                    AdapterInfo info;
                    info.ProfileName = connProfile.ProfileName();
                    info.AddressType = hostName.Type();
                    info.Address = hostName.DisplayName();
                    info.ConnectivityLevel = 
                        connProfile.GetNetworkConnectivityLevel();
                    
                    ConnectionCost cost = 
                        connProfile.GetConnectionCost();
                    info.IsMetered = 
                        (cost.NetworkCostType() == 
                            NetworkCostType::Fixed ||
                         cost.NetworkCostType() == 
                            NetworkCostType::Variable);
                    
                    info.CaptureTime = 
                        std::chrono::system_clock::now();
                    
                    currentAdapters_.push_back(info);
                }
            }
        }

        lastStatusChange_ = std::chrono::system_clock::now();
    }

    // 네트워크 상태 변경 핸들러 등록
    void RegisterNetworkChangeHandler()
    {
        NetworkInformation::NetworkStatusChanged(
            [this](const IInspectable& sender) {
                OnNetworkStatusChanged();
            }
        );
    }

    // 상태 변경 콜백 등록
    void Subscribe(NetworkStatusCallback callback)
    {
        callbacks_.push_back(callback);
    }

    // 현재 어댑터 정보 반환
    std::vector<AdapterInfo> GetAdapterInfo() const
    {
        return currentAdapters_;
    }

    // 인터넷 연결 여부
    bool IsInternetAvailable() const
    {
        return isInternetAvailable_;
    }

    // 마지막 상태 변경 시간
    std::chrono::system_clock::time_point GetLastStatusChangeTime() const
    {
        return lastStatusChange_;
    }

    // 활성 어댑터 개수
    int GetAdapterCount() const
    {
        return static_cast<int>(currentAdapters_.size());
    }

private:
    // 네트워크 상태 변경 처리
    void OnNetworkStatusChanged()
    {
        RefreshAdapterInfo();
        
        // 모든 구독자에게 알림
        for (const auto& callback : callbacks_) {
            callback(currentAdapters_);
        }
    }
};

{

    }
    
    std::wcout << L"활성 어댑터: " 
               << monitor->GetAdapterCount() << std::endl;
    
    auto adapters = monitor->GetAdapterInfo();
    for (size_t i = 0; i < adapters.size(); ++i) {
        const auto& info = adapters[i];
        std::wcout << L"\n  어댑터 " << (i + 1) << L":" << std::endl;
        std::wcout << L"    이름: " << info.ProfileName.c_str() << std::endl;
        std::wcout << L"    주소: " << info.Address.c_str() << std::endl;
        std::wcout << L"    타입: " 
                   << (info.AddressType == 
                       Networking::HostNameType::Ipv4 ? 
                       L"IPv4" : L"IPv6") << std::endl;
        std::wcout << L"    측정 연결: " 
                   << (info.IsMetered ? L"예" : L"아니오") 
                   << std::endl;
        
        if (info.ConnectivityLevel == 
            NetworkConnectivityLevel::InternetAccess) {
            std::wcout << L"    인터넷 접근: 가능" << std::endl;
        } else if (info.ConnectivityLevel == 
                   NetworkConnectivityLevel::LocalAccess) {
            std::wcout << L"    인터넷 접근: 로컬 전용" << std::endl;
        } else {
            std::wcout << L"    인터넷 접근: 없음" << std::endl;
        }
    }
}
```

### 주요 부분 상세 해설

**NetworkMonitor 클래스 설계**: 이 클래스는 싱글톤 패턴을 사용하여 애플리케이션 전체에서 유일한 인스턴스만 존재하도록 한다. 네트워크 모니터링 로직을 중앙화하고 여러 곳에서 쉽게 접근할 수 있게 한다.

**RefreshAdapterInfo 함수**: 이 함수는 현재 활성 네트워크 연결의 완전한 목록을 구성한다. GetInternetConnectionProfile()로 인터넷 연결 가능 여부를 확인하고, GetConnectionProfiles()로 모든 프로필을 순회한다. 각 프로필마다 GetHostNames()를 호출하여 관련된 IP 주소를 수집한다. 주의할 점은 하나의 프로필이 여러 IP 주소를 가질 수 있다는 것이다(IPv4와 IPv6 모두 등).

**네트워크 상태 변경 감지**: RegisterNetworkChangeHandler()에서 NetworkInformation::NetworkStatusChanged 이벤트에 람다를 등록한다. 네트워크 변경이 감지되면 OnNetworkStatusChanged()가 호출되고, 모든 구독자 콜백에 새로운 어댑터 정보를 전달한다.

**구독 패턴**: Subscribe() 메서드를 통해 애플리케이션의 여러 부분이 네트워크 상태 변경을 감시할 수 있다. 예를 들어 게임 클라이언트의 연결 관리자는 이벤트를 받으면 서버 재연결을 시도할 수 있다.

### 개선 방안과 확장 가능성
현재 구현은 기본적인 네트워크 모니터링을 제공하지만, 다음과 같이 개선할 수 있다.

**네트워크 상태 기록**: 시간별 어댑터 변화를 기록하여 연결 불안정성을 분석할 수 있다. 예를 들어 하루 동안 몇 번 연결이 끊어졌는지, 각 끊김이 얼마나 오래 지속되었는지 추적할 수 있다.

**특정 어댑터 모니터링**: 게임이 특정 네트워크를 사용해야 하는 경우, 그 어댑터만 집중 모니터링할 수 있다. 예를 들어 회사 네트워크에 연결되었을 때만 특정 서버에 접근하도록 할 수 있다.

**DNS 해석 결과 캐싱**: 자주 접근하는 서버의 IP를 캐시하여 DNS 해석 지연을 줄일 수 있다. NetworkInformation API는 로컬 캐시를 지원하므로 이를 활용하면 성능이 향상된다.

**네트워크 대역폭 추정**: GetConnectionCost()의 반환값을 더 상세히 분석하여 현재 대역폭을 추정할 수 있다. 측정 연결에서는 압축을 강화하고, 무제한 연결에서는 해상도를 높이는 식의 적응적 처리가 가능하다.

**프로필별 분리**: 여러 네트워크가 동시에 활성화될 때, 어느 것을 사용할지 선택하는 로직을 추가할 수 있다. 예를 들어 유선이 더 안정적이므로 우선순위를 높이는 방식이다.

이러한 개선사항들은 더 강건한 게임 클라이언트를 만드는 데 도움이 된다. 불안정한 네트워크에서도 플레이어가 최대한 오래 서버와 연결을 유지하고, 문제가 발생했을 때 빠르게 재연결할 수 있게 된다.
