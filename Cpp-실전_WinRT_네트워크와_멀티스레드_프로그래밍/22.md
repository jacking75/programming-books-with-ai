# Chapter 22. 게임 로비 서버
게임 로비 서버는 플레이어들이 실제 게임에 진입하기 전에 거쳐야 하는 관문이다. 채팅 서버가 실시간 통신에 집중한다면, 로비 서버는 게임 시작 전의 모든 준비 작업을 담당한다. 플레이어 매칭, 게임방 생성, 준비 상태 확인, 게임 시작 조율 등이 로비 서버의 핵심 기능이다.

온라인 게임에서 로비 서버의 중요성은 매우 크다. 수천 명의 플레이어가 동시에 접속할 수 있고, 각각 다른 매칭 조건(실력, 지역, 플레이 시간대)을 원한다. 로비 서버는 이들을 빠르고 정공하게 매칭하여 게임 시작 속도를 결정한다. 또한 게임이 시작되면 플레이어들의 상태를 동기화하여 공정한 게임 환경을 만들어야 한다.

이 장에서는 매칭 알고리즘부터 게임 시작 조율까지의 전체 과정을 다루며, 실전 게임 로비 시스템을 만들어본다.

---

## 22.1 로비 서버의 역할과 기능
로비 서버는 게임 진행 중인 서버와는 다른 독립적인 서버로 동작한다. 각 서버의 역할을 정확하게 이해하는 것이 중요하다.

**로비 서버의 주요 역할**

**1. 플레이어 매칭(Matchmaking)**
로비 서버는 게임을 원하는 플레이어들을 찾아서 적절하게 그룹을 만든다. 단순한 경우는 "5명이 모을 때까지 기다린다"이지만, 실제로는 훨씬 복잡하다.

- **레이팅 기반 매칭**: 플레이어의 실력(MMR, Skill Rating)을 고려하여 비슷한 수준의 플레이어끼리 매칭한다. 신참 플레이어를 고수와 섞으면 게임이 재미없어진다.
- **지역 기반 매칭**: 같은 대륙의 플레이어끼리 매칭하여 핑을 줄인다.
- **보상 기반 매칭**: 최근 플레이 기록이 없는 플레이어와 자주 플레이하는 플레이어를 섞어서 균형을 맞춘다.

**2. 게임방(Room) 관리**
로비 서버는 각 게임방의 정보를 관리한다. 방이 생성되고, 플레이어가 들어오고, 게임이 시작되고, 게임이 끝날 때까지의 전체 라이프사이클을 관리해야 한다.

- **공개방 vs 비공개방**: 공개방은 누구나 입장 가능하지만, 비공개방은 초대받은 사람만 입장 가능하다.
- **방 상태 추적**: WAITING(기다리는 중), READY(시작 준비됨), IN_GAME(게임 진행 중), FINISHED(게임 완료)

**3. 게임 시작 조율**
모든 플레이어가 준비되면 게임을 시작한다. 하지만 네트워크 지연으로 일부 플레이어의 준비 신호가 늦어질 수 있다.

- **준비 확인**: 모든 플레이어가 "준비됨" 상태를 보냈는지 확인한다.
- **타임아웃 처리**: 일정 시간 내에 준비하지 않는 플레이어는 자동으로 제거한다.
- **동기화**: 모든 클라이언트가 같은 시각에 게임을 시작하도록 명령한다.

**4. 로비 정보 제공**
플레이어가 로비에 들어올 때, 현재 어떤 게임들이 진행 중인지, 어떤 플레이어들이 매칭을 기다리고 있는지 정보를 제공한다.

- **방 목록**: 현재 열려있는 공개방의 목록
- **플레이어 수**: 각 방에 몇 명이 있는지
- **게임 정보**: 게임 맵, 게임 모드, 최소 레이팅 등

**로비 서버와 게임 서버의 관계**

```
┌─────────────────────────────────────────────────┐
│             클라이언트들                          │
└──────────────────┬──────────────────────────────┘
                   │
          ┌────────┴─────────┐
          │                  │
    ┌─────▼─────┐      ┌────▼──────┐
    │ Lobby     │      │ Game      │
    │ Server    │      │ Server    │
    │           │      │           │
    │ -매칭      │      │ -게임진행 │
    │ -방관리    │      │ -상태동기 │
    │ -시작조율  │      │ -결과처리 │
    └─────┬─────┘      └────┬──────┘
          │                  │
          └────────┬─────────┘
                   │
          ┌────────▼─────────┐
          │  데이터베이스      │
          │  (게임 기록)       │
          └───────────────────┘
```

플레이어가 로비에 있을 때는 로비 서버와만 통신한다. 게임이 시작되면 로비 서버는 클라이언트에게 게임 서버의 주소를 알려주고, 클라이언트는 게임 서버로 연결을 전환한다. 로비 서버는 게임 진행 중에는 대기하다가, 게임이 끝나면 결과를 받아서 기록한다.

## 22.2 플레이어 매칭 시스템
플레이어 매칭은 로비 서버의 가장 복잡한 부분이다. 빠르면서도 정공한 매칭을 하려면 여러 전략이 필요하다.

```cpp
#include <vector>
#include <queue>
#include <map>
#include <memory>
#include <mutex>
#include <chrono>
#include <algorithm>
#include <cmath>

enum class MatchmakingState
{
    SEARCHING,    // 매칭 중
    FOUND,        // 매칭됨
    CANCELLED,    // 취소됨
    TIMEOUT       // 타임아웃
};

struct Player
{
    std::string playerId;
    std::string playerName;
    int32_t rating;           // 플레이어 실력 점수
    int32_t wins;
    int32_t losses;
    std::chrono::system_clock::time_point searchStartTime;
    MatchmakingState state;
};

class MatchmakingQueue
{
private:
    struct QueueEntry
    {
        std::shared_ptr<Player> player;
        std::chrono::system_clock::time_point queueTime;
    };

    std::queue<QueueEntry> queue;
    std::map<std::string, QueueEntry> playerMap;
    std::mutex queueMutex;

    // 매칭 조건
    static const int32_t INITIAL_RATING_RANGE = 100;  // 처음에는 ±100
    static const int32_t MAX_RATING_RANGE = 500;      // 최대 ±500
    static const int64_t MAX_WAIT_TIME = 60;          // 최대 대기 시간 60초

public:
    // 플레이어를 매칭 큐에 추가
    bool AddPlayerToQueue(std::shared_ptr<Player> player)
    {
        std::lock_guard<std::mutex> lock(queueMutex);

        // 이미 큐에 있으면 추가 안 함
        if (playerMap.find(player->playerId) != playerMap.end())
            return false;

        QueueEntry entry;
        entry.player = player;
        entry.queueTime = std::chrono::system_clock::now();

        queue.push(entry);
        playerMap[player->playerId] = entry;
        player->state = MatchmakingState::SEARCHING;

        return true;
    }

    // 플레이어를 큐에서 제거
    bool RemovePlayerFromQueue(const std::string& playerId)
    {
        std::lock_guard<std::mutex> lock(queueMutex);
        playerMap.erase(playerId);
        return true;
    }

    // 현재 큐에 있는 플레이어 수
    size_t GetQueueSize() const
    {
        std::lock_guard<std::mutex> lock(queueMutex);
        return playerMap.size();
    }

    // 매칭 가능한 플레이어들을 찾기
    std::vector<std::shared_ptr<Player>> FindMatchableGroup(
        size_t groupSize = 5)
    {
        std::lock_guard<std::mutex> lock(queueMutex);

        std::vector<std::shared_ptr<Player>> group;
        std::vector<std::string> toRemove;

        // 큐 앞에서부터 플레이어를 확인
        std::vector<QueueEntry> tempQueue;
        while (!queue.empty())
        {
            tempQueue.push_back(queue.front());
            queue.pop();
        }

        // 첫 번째 플레이어를 기준으로 그룹 구성
        if (tempQueue.empty())
        {
            // 큐를 다시 채우기
            for (const auto& entry : tempQueue)
                queue.push(entry);
            return group;
        }

        auto firstPlayer = tempQueue[0].player;
        auto firstQueueTime = tempQueue[0].queueTime;
        auto now = std::chrono::system_clock::now();
        auto waitTime = std::chrono::duration_cast<std::chrono::seconds>(
            now - firstQueueTime
        ).count();

        // 대기 시간에 따라 매칭 범위 결정
        int32_t ratingRange = INITIAL_RATING_RANGE + 
                             (waitTime * 10);
        ratingRange = std::min(ratingRange, MAX_RATING_RANGE);

        // 첫 번째 플레이어를 그룹에 추가
        group.push_back(firstPlayer);
        toRemove.push_back(firstPlayer->playerId);

        // 나머지 플레이어들을 확인
        for (size_t i = 1; i < tempQueue.size() && 
             group.size() < groupSize; i++)
        {
            auto player = tempQueue[i].player;
            
            // 레이팅이 범위 내인지 확인
            if (std::abs(player->rating - firstPlayer->rating) <= 
                ratingRange)
            {
                group.push_back(player);
                toRemove.push_back(player->playerId);
            }
        }

        // 그룹이 완성되었는지 확인
        if (group.size() < groupSize)
        {
            // 그룹이 완성되지 않으면 원래대로 복원
            for (const auto& entry : tempQueue)
                queue.push(entry);
            return {};
        }

        // 제거할 플레이어들을 큐에서 빼기
        for (const auto& id : toRemove)
        {
            playerMap.erase(id);
        }

        // 남은 플레이어들을 다시 큐에 넣기
        for (const auto& entry : tempQueue)
        {
            if (std::find(toRemove.begin(), toRemove.end(), 
                         entry.player->playerId) == toRemove.end())
            {
                queue.push(entry);
            }
        }

        return group;
    }

    // 타임아웃된 플레이어들 찾기 (60초 이상 대기)
    std::vector<std::shared_ptr<Player>> FindTimeoutPlayers()
    {
        std::lock_guard<std::mutex> lock(queueMutex);

        std::vector<std::shared_ptr<Player>> timeoutPlayers;
        auto now = std::chrono::system_clock::now();

        for (auto& entry : playerMap)
        {
            auto waitTime = std::chrono::duration_cast<
                std::chrono::seconds>(
                    now - entry.second.queueTime
                ).count();

            if (waitTime > MAX_WAIT_TIME)
            {
                timeoutPlayers.push_back(entry.second.player);
            }
        }

        return timeoutPlayers;
    }
};

class MatchmakingEngine
{
private:
    MatchmakingQueue queue;

public:
    // 플레이어의 매칭 요청 처리
    bool RequestMatch(std::shared_ptr<Player> player)
    {
        player->searchStartTime = 
            std::chrono::system_clock::now();
        return queue.AddPlayerToQueue(player);
    }

    // 매칭 취소
    bool CancelMatchRequest(const std::string& playerId)
    {
        return queue.RemovePlayerFromQueue(playerId);
    }

    // 주기적으로 호출되는 매칭 처리 (백그라운드 작업)
    std::vector<std::vector<std::shared_ptr<Player>>> 
    ProcessMatching()
    {
        std::vector<std::vector<std::shared_ptr<Player>>> 
            matchedGroups;

        // 5명의 플레이어 그룹을 만들 때까지 반복
        while (true)
        {
            auto group = queue.FindMatchableGroup(5);
            if (group.empty())
                break;

            // 그룹을 매칭된 것으로 표시
            for (auto& player : group)
            {
                player->state = MatchmakingState::FOUND;
            }

            matchedGroups.push_back(group);
        }

        // 타임아웃된 플레이어들 처리
        auto timeoutPlayers = queue.FindTimeoutPlayers();
        for (auto& player : timeoutPlayers)
        {
            queue.RemovePlayerFromQueue(player->playerId);
            player->state = MatchmakingState::TIMEOUT;
        }

        return matchedGroups;
    }

    // 현재 매칭 큐 상태
    size_t GetQueueSize() const
    {
        return queue.GetQueueSize();
    }
};
```

이 매칭 시스템의 핵심 알고리즘은 **동적 레이팅 범위**이다. 처음에는 비슷한 실력의 플레이어를 찾으려고 하지만, 대기 시간이 길어질수록 범위를 점점 넓혀서 빠른 매칭을 보장한다. 이렇게 하면 신참 플레이어와 고수의 게임 경험을 모두 고려할 수 있다.

**주요 특징**:

- **FIFO 순서 보장**: 먼저 온 플레이어를 먼저 매칭한다.
- **적응형 범위**: 대기 시간이 길어질수록 매칭 범위를 넓힌다.
- **타임아웃 처리**: 60초 이상 대기하면 자동으로 취소한다.
- **스레드 안전**: 뮤텍스로 보호된 큐를 사용한다.

## 22.3 방 생성과 관리
매칭이 완료되면 게임방을 생성하고, 플레이어들을 그 방으로 이동시킨다. 게임방은 공개방과 비공개방을 지원해야 한다.

```cpp
#include <string>
#include <vector>
#include <map>
#include <memory>
#include <mutex>

enum class RoomState
{
    WAITING,      // 플레이어 대기 중
    READY,        // 모두 준비됨
    IN_GAME,      // 게임 진행 중
    FINISHED      // 게임 완료
};

enum class RoomType
{
    PUBLIC,       // 누구나 입장 가능
    PRIVATE       // 초대된 사람만 입장 가능
};

struct GameRoom
{
    std::string roomId;
    std::string roomName;
    RoomType roomType;
    std::vector<std::string> players;  // 플레이어 ID 목록
    std::string gameServerId;          // 게임 서버 주소
    int32_t maxPlayers;
    RoomState state;
    std::map<std::string, bool> readyStatus;  // playerId -> ready
};

class RoomManager
{
private:
    std::map<std::string, std::shared_ptr<GameRoom>> rooms;
    std::map<std::string, std::string> playerToRoom;  // playerId -> roomId
    std::mutex roomMutex;
    int32_t roomIdCounter = 0;

public:
    // 새로운 방 생성
    std::shared_ptr<GameRoom> CreateRoom(
        const std::vector<std::string>& playerIds,
        RoomType roomType = RoomType::PUBLIC)
    {
        std::lock_guard<std::mutex> lock(roomMutex);

        // 방 ID 생성
        std::string roomId = "room_" + 
                            std::to_string(roomIdCounter++);

        auto room = std::make_shared<GameRoom>();
        room->roomId = roomId;
        room->roomName = "Game_" + roomId;
        room->roomType = roomType;
        room->players = playerIds;
        room->maxPlayers = playerIds.size();
        room->state = RoomState::WAITING;

        // 모든 플레이어의 준비 상태를 false로 초기화
        for (const auto& playerId : playerIds)
        {
            room->readyStatus[playerId] = false;
            playerToRoom[playerId] = roomId;
        }

        rooms[roomId] = room;
        return room;
    }

    // 방에 플레이어 추가 (공개방에만 가능)
    bool AddPlayerToRoom(const std::string& roomId,
                        const std::string& playerId)
    {
        std::lock_guard<std::mutex> lock(roomMutex);

        auto it = rooms.find(roomId);
        if (it == rooms.end())
            return false;

        auto room = it->second;

        // 비공개방이면 입장 불가
        if (room->roomType == RoomType::PRIVATE)
            return false;

        // 방이 가득 찼는지 확인
        if (room->players.size() >= room->maxPlayers)
            return false;

        // 이미 방에 있으면 안 됨
        if (playerToRoom.find(playerId) != playerToRoom.end())
            return false;

        room->players.push_back(playerId);
        room->readyStatus[playerId] = false;
        playerToRoom[playerId] = roomId;

        return true;
    }

    // 방에서 플레이어 제거
    bool RemovePlayerFromRoom(const std::string& roomId,
                             const std::string& playerId)
    {
        std::lock_guard<std::mutex> lock(roomMutex);

        auto it = rooms.find(roomId);
        if (it == rooms.end())
            return false;

        auto room = it->second;

        auto playerIt = std::find(room->players.begin(),
                                 room->players.end(),
                                 playerId);
        if (playerIt == room->players.end())
            return false;

        room->players.erase(playerIt);
        room->readyStatus.erase(playerId);
        playerToRoom.erase(playerId);

        // 방이 비어지면 삭제
        if (room->players.empty())
            rooms.erase(it);

        return true;
    }

    // 플레이어가 속한 방 조회
    std::shared_ptr<GameRoom> GetPlayerRoom(
        const std::string& playerId)
    {
        std::lock_guard<std::mutex> lock(roomMutex);

        auto it = playerToRoom.find(playerId);
        if (it == playerToRoom.end())
            return nullptr;

        auto roomIt = rooms.find(it->second);
        if (roomIt == rooms.end())
            return nullptr;

        return roomIt->second;
    }

    // 방 ID로 직접 조회
    std::shared_ptr<GameRoom> GetRoom(const std::string& roomId)
    {
        std::lock_guard<std::mutex> lock(roomMutex);

        auto it = rooms.find(roomId);
        if (it == rooms.end())
            return nullptr;

        return it->second;
    }

    // 공개방 목록 조회
    std::vector<std::shared_ptr<GameRoom>> GetPublicRooms()
    {
        std::lock_guard<std::mutex> lock(roomMutex);

        std::vector<std::shared_ptr<GameRoom>> publicRooms;
        for (auto& entry : rooms)
        {
            if (entry.second->roomType == RoomType::PUBLIC &&
                entry.second->state == RoomState::WAITING)
            {
                publicRooms.push_back(entry.second);
            }
        }
        return publicRooms;
    }

    // 모든 방의 개수
    size_t GetRoomCount() const
    {
        std::lock_guard<std::mutex> lock(roomMutex);
        return rooms.size();
    }
};
```

이 방 관리 시스템의 특징:

**이중 인덱싱**: 플레이어 ID로도, 방 ID로도 빠르게 검색할 수 있다.

**공개/비공개 구분**: 비공개방은 초대된 플레이어만 입장 가능하다.

**자동 정리**: 방이 비어지면 자동으로 삭제하여 메모리 낭비를 방지한다.

**상태 추적**: 방의 상태를 WAITING, READY, IN_GAME, FINISHED로 구분하여 각 단계를 명확하게 한다.

## 22.4 게임 시작 프로토콜
모든 플레이어가 게임 준비를 마치면 서버는 게임을 시작해야 한다. 이를 위해 준비 확인과 동기화 과정이 필요하다.

```cpp
#include <string>
#include <memory>
#include <vector>
#include <mutex>
#include <chrono>

class GameStartCoordinator
{
private:
    struct RoomReadyState
    {
        std::shared_ptr<GameRoom> room;
        std::chrono::system_clock::time_point readyStartTime;
        bool allPlayersReady = false;
    };

    std::map<std::string, RoomReadyState> readyStates;
    std::mutex coordinatorMutex;

    static const int64_t READY_TIMEOUT = 10;  // 10초 대기

public:
    // 플레이어가 준비됐음을 표시
    bool MarkPlayerReady(const std::string& roomId,
                        const std::string& playerId)
    {
        std::lock_guard<std::mutex> lock(coordinatorMutex);

        auto it = readyStates.find(roomId);
        if (it == readyStates.end())
            return false;

        it->second.room->readyStatus[playerId] = true;
        return true;
    }

    // 방의 모든 플레이어가 준비 완료했는지 확인
    bool AreAllPlayersReady(const std::string& roomId)
    {
        std::lock_guard<std::mutex> lock(coordinatorMutex);

        auto it = readyStates.find(roomId);
        if (it == readyStates.end())
            return false;

        auto room = it->second.room;
        for (const auto& player : room->players)
        {
            if (!room->readyStatus[player])
                return false;
        }
        return true;
    }

    // 게임 시작 신호 전송 (모두 준비됨)
    bool TryStartGame(const std::string& roomId)
    {
        std::lock_guard<std::mutex> lock(coordinatorMutex);

        auto it = readyStates.find(roomId);
        if (it == readyStates.end())
            return false;

        auto room = it->second.room;

        // 모든 플레이어가 준비했는지 최종 확인
        for (const auto& player : room->players)
        {
            if (!room->readyStatus[player])
                return false;
        }

        // 게임 상태 변경
        room->state = RoomState::IN_GAME;

        // 실제로는 여기서 클라이언트에 게임 서버 주소와
        // 게임 시작 신호를 보냄
        // SendGameStartSignal(roomId);

        return true;
    }

    // 게임 시작 준비 시간 초과 처리
    std::vector<std::string> GetReadyTimeoutRooms()
    {
        std::lock_guard<std::mutex> lock(coordinatorMutex);

        std::vector<std::string> timeoutRooms;
        auto now = std::chrono::system_clock::now();

        for (auto& entry : readyStates)
        {
            if (entry.second.room->state != RoomState::WAITING)
                continue;

            auto waitTime = std::chrono::duration_cast<
                std::chrono::seconds>(
                    now - entry.second.readyStartTime
                ).count();

            if (waitTime > READY_TIMEOUT)
            {
                timeoutRooms.push_back(entry.first);
            }
        }

        return timeoutRooms;
    }

    // 준비 안 한 플레이어 목록 조회
    std::vector<std::string> GetNotReadyPlayers(
        const std::string& roomId)
    {
        std::lock_guard<std::mutex> lock(coordinatorMutex);

        std::vector<std::string> notReady;
        auto it = readyStates.find(roomId);
        if (it == readyStates.end())
            return notReady;

        auto room = it->second.room;
        for (const auto& player : room->players)
        {
            if (!room->readyStatus[player])
                notReady.push_back(player);
        }

        return notReady;
    }

    // 방의 준비 상태 초기화
    void InitializeRoomReadyState(
        std::shared_ptr<GameRoom> room)
    {
        std::lock_guard<std::mutex> lock(coordinatorMutex);

        RoomReadyState state;
        state.room = room;
        state.readyStartTime = 
            std::chrono::system_clock::now();
        state.allPlayersReady = false;

        readyStates[room->roomId] = state;
    }

    // 방의 준비 상태 제거 (게임 시작 후)
    void RemoveRoomReadyState(const std::string& roomId)
    {
        std::lock_guard<std::mutex> lock(coordinatorMutex);
        readyStates.erase(roomId);
    }

    // 준비 진행률 조회 (UI 표시용)
    double GetReadyProgress(const std::string& roomId)
    {
        std::lock_guard<std::mutex> lock(coordinatorMutex);

        auto it = readyStates.find(roomId);
        if (it == readyStates.end())
            return 0.0;

        auto room = it->second.room;
        int32_t readyCount = 0;

        for (const auto& player : room->players)
        {
            if (room->readyStatus[player])
                readyCount++;
        }

        return static_cast<double>(readyCount) / 
               room->players.size();
    }
};
```

게임 시작 프로토콜의 핵심은 **타임아웃 처리**다. 만약 10초 후에도 모든 플레이어가 준비하지 않으면, 준비하지 않은 플레이어들을 자동으로 제거하고 게임을 시작해야 한다. 이렇게 하면 한두 명의 느린 플레이어 때문에 전체 게임 시작이 지연되는 것을 방지할 수 있다.

## 22.5 상태 동기화
로비의 정보(현재 방 목록, 매칭 상태, 플레이어 수)는 주기적으로 모든 클라이언트에게 브로드캐스트되어야 한다.

```cpp
#include <string>
#include <vector>
#include <memory>
#include <mutex>

struct LobbyState
{
    size_t totalPlayers;      // 로비의 전체 플레이어
    size_t waitingInQueue;    // 매칭 대기 중
    size_t inGames;           // 게임 진행 중
    std::vector<std::pair<std::string, size_t>> 
        publicRooms;          // (방이름, 플레이어수)
};

class LobbyStateSynchronizer
{
private:
    RoomManager& roomManager;
    MatchmakingEngine& matchmakingEngine;
    std::mutex syncMutex;

public:
    LobbyStateSynchronizer(RoomManager& rm, 
                          MatchmakingEngine& me)
        : roomManager(rm), matchmakingEngine(me)
    {
    }

    // 현재 로비 상태 스냅샷 생성
    LobbyState GetCurrentLobbyState()
    {
        std::lock_guard<std::mutex> lock(syncMutex);

        LobbyState state;

        // 매칭 큐의 플레이어 수
        state.waitingInQueue = matchmakingEngine.GetQueueSize();

        // 게임 방의 정보
        auto publicRooms = roomManager.GetPublicRooms();
        for (const auto& room : publicRooms)
        {
            state.publicRooms.emplace_back(
                room->roomName,
                room->players.size()
            );
        }

        state.totalPlayers = state.waitingInQueue + 
                            ComputeInGamePlayers();
        state.inGames = ComputeInGamePlayers();

        return state;
    }

    // 로비 상태를 문자열로 직렬화
    std::string SerializeLobbyState(const LobbyState& state)
    {
        std::string result = "LOBBY_UPDATE|";
        result += std::to_string(state.totalPlayers) + "|";
        result += std::to_string(state.waitingInQueue) + "|";
        result += std::to_string(state.inGames) + "|";
        result += std::to_string(state.publicRooms.size()) + "|";

        for (const auto& room : state.publicRooms)
        {
            result += room.first + ":" + 
                     std::to_string(room.second) + ";";
        }

        return result;
    }

    // 플레이어별 개인 상태 정보
    std::string GetPlayerStatus(const std::string& playerId)
    {
        std::lock_guard<std::mutex> lock(syncMutex);

        // 플레이어의 현재 상태를 조회
        // WAITING_MATCH / IN_ROOM / IN_GAME 중 하나

        std::string status = "PLAYER_STATUS|";
        status += playerId + "|";

        // 플레이어가 속한 방 조회
        auto room = roomManager.GetPlayerRoom(playerId);
        if (room)
        {
            status += room->roomId + "|";
            status += std::to_string(room->players.size()) + "|";

            // 준비 상태
            if (room->readyStatus.count(playerId) > 0)
            {
                status += room->readyStatus.at(playerId) ? 
                         "READY" : "WAITING";
            }
        }
        else
        {
            status += "SEARCHING|";
        }

        return status;
    }

private:
    size_t ComputeInGamePlayers()
    {
        // 모든 게임 방의 플레이어 수 합산
        // (실제 구현에서는 더 효율적으로 할 수 있음)
        return 0;
    }
};
```

상태 동기화의 핵심은 **주기적 브로드캐스트**다. 5초마다 로비 상태를 모든 클라이언트에게 보내면, 클라이언트는 항상 최신 정보를 표시할 수 있다. 또한 개인별 상태 정보를 별도로 제공하여, 각 플레이어가 자신의 상태를 정확히 파악할 수 있도록 한다.

## 22.6 종합 예제: 완전한 게임 로비 시스템
이제 매칭, 방 관리, 게임 시작을 모두 통합한 실제로 작동하는 로비 서버를 만들어보자.

**코드의 목적**: 게임 플레이어들의 매칭 요청부터 게임 서버로의 이동까지의 전체 과정을 관리하는 프로덕션급 로비 서버를 구현한다. 플레이어가 로비에 들어와서 "게임 시작" 버튼을 누르면, 다른 플레이어들과 빠르게 매칭되고, 모두 준비하면 게임 서버 주소를 받아 게임에 진입할 수 있어야 한다.

**설계 결정 이유**:
1. **백그라운드 매칭 워커**: 매칭 작업을 별도 스레드에서 주기적으로 처리하여, 클라이언트 요청을 방해하지 않는다.
2. **이벤트 기반 알림**: 매칭 완료, 게임 준비 완료 등의 이벤트를 명확하게 정의하여, 상위 계층이 대응하기 쉽게 한다.
3. **상태 머신**: 각 플레이어의 상태(SEARCHING, MATCHED, READY, IN_GAME)를 명확하게 추적한다.
4. **타임아웃 안전성**: 타임아웃된 요청이나 준비하지 않은 플레이어를 자동으로 처리한다.

```cpp
#include <string>
#include <memory>
#include <vector>
#include <map>
#include <mutex>
#include <thread>
#include <queue>
#include <chrono>
#include <functional>

// ============ 이벤트 정의 ============

enum class LobbyEvent
{
    MATCH_FOUND,
    MATCH_TIMEOUT,
    GAME_READY,
    GAME_START,
    PLAYER_DISCONNECTED
};

struct LobbyEventData
{
    LobbyEvent eventType;
    std::string playerId;
    std::string roomId;
    std::string gameServerId;
    std::vector<std::string> otherPlayers;
};

// ============ 플레이어 상태 ============

enum class PlayerLobbyState
{
    IDLE,             // 대기 중
    SEARCHING,        // 매칭 대기 중
    MATCHED,          // 매칭됨
    READY,            // 게임 준비 완료
    IN_GAME,          // 게임 진행 중
    LEFT_LOBBY        // 로비 떠남
};

// ============ 통합 로비 서버 ============

class ProductionLobbyServer
{
private:
    MatchmakingEngine matchmakingEngine;
    RoomManager roomManager;
    GameStartCoordinator startCoordinator;
    LobbyStateSynchronizer synchronizer;

    std::map<std::string, PlayerLobbyState> playerStates;
    std::queue<LobbyEventData> eventQueue;
    std::mutex serverMutex;

    std::thread matchingWorkerThread;
    std::thread startCoordinatorThread;
    volatile bool shouldStop = false;

    // 이벤트 콜백
    std::function<void(const LobbyEventData&)> onEventCallback;

public:
    ProductionLobbyServer()
        : synchronizer(roomManager, matchmakingEngine)
    {
        // 백그라운드 워커 스레드 시작
        matchingWorkerThread = std::thread(
            [this]() { MatchingWorker(); }
        );
        startCoordinatorThread = std::thread(
            [this]() { StartCoordinatorWorker(); }
        );
    }

    ~ProductionLobbyServer()
    {
        shouldStop = true;
        if (matchingWorkerThread.joinable())
            matchingWorkerThread.join();
        if (startCoordinatorThread.joinable())
            startCoordinatorThread.join();
    }

    // ===== 플레이어 매칭 요청 =====

    bool RequestMatchmaking(const std::string& playerId,
                           const std::string& playerName,
                           int32_t rating)
    {
        std::lock_guard<std::mutex> lock(serverMutex);

        // 플레이어가 이미 로비에 있으면 거절
        if (playerStates.find(playerId) != playerStates.end())
            return false;

        auto player = std::make_shared<Player>();
        player->playerId = playerId;
        player->playerName = playerName;
        player->rating = rating;
        player->wins = 0;
        player->losses = 0;
        player->state = MatchmakingState::SEARCHING;

        if (!matchmakingEngine.RequestMatch(player))
            return false;

        playerStates[playerId] = PlayerLobbyState::SEARCHING;
        return true;
    }

    // 매칭 취소
    bool CancelMatchmaking(const std::string& playerId)
    {
        std::lock_guard<std::mutex> lock(serverMutex);

        auto it = playerStates.find(playerId);
        if (it == playerStates.end() || 
            it->second != PlayerLobbyState::SEARCHING)
            return false;

        matchmakingEngine.CancelMatchRequest(playerId);
        playerStates.erase(it);
        return true;
    }

    // ===== 게임 준비 상태 =====

    bool MarkPlayerReady(const std::string& playerId)
    {
        std::lock_guard<std::mutex> lock(serverMutex);

        auto playerState = playerStates[playerId];
        if (playerState != PlayerLobbyState::MATCHED)
            return false;

        auto room = roomManager.GetPlayerRoom(playerId);
        if (!room)
            return false;

        startCoordinator.MarkPlayerReady(room->roomId, 
                                        playerId);
        playerStates[playerId] = PlayerLobbyState::READY;

        return true;
    }

    // ===== 로비 상태 조회 =====

    LobbyState GetLobbyState()
    {
        return synchronizer.GetCurrentLobbyState();
    }

    std::string GetPlayerStatus(const std::string& playerId)
    {
        return synchronizer.GetPlayerStatus(playerId);
    }

    // ===== 이벤트 콜백 설정 =====

    void SetEventCallback(
        std::function<void(const LobbyEventData&)> callback)
    {
        onEventCallback = callback;
    }

    // ===== 이벤트 처리 =====

private:
    void PublishEvent(const LobbyEventData& event)
    {
        std::lock_guard<std::mutex> lock(serverMutex);
        eventQueue.push(event);

        if (onEventCallback)
            onEventCallback(event);
    }

    // ===== 백그라운드 워커 1: 매칭 처리 =====

    void MatchingWorker()
    {
        while (!shouldStop)
        {
            std::this_thread::sleep_for(
                std::chrono::milliseconds(500)
            );

            // 매칭 처리
            auto matchedGroups = 
                matchmakingEngine.ProcessMatching();

            for (const auto& group : matchedGroups)
            {
                // 매칭된 플레이어들로 방 생성
                std::vector<std::string> playerIds;
                for (const auto& player : group)
                    playerIds.push_back(player->playerId);

                auto room = roomManager.CreateRoom(
                    playerIds, 
                    RoomType::PRIVATE
                );

                if (room)
                {
                    {
                        std::lock_guard<std::mutex> lock(
                            serverMutex
                        );

                        // 매칭된 각 플레이어의 상태 업데이트
                        for (const auto& playerId : playerIds)
                        {
                            playerStates[playerId] = 
                                PlayerLobbyState::MATCHED;
                        }

                        // 게임 준비 상태 초기화
                        startCoordinator.InitializeRoomReadyState(
                            room
                        );
                    }

                    // 매칭 완료 이벤트 발행
                    LobbyEventData event;
                    event.eventType = LobbyEvent::MATCH_FOUND;
                    event.roomId = room->roomId;
                    for (const auto& playerId : playerIds)
                    {
                        event.otherPlayers.push_back(playerId);
                    }

                    for (const auto& playerId : playerIds)
                    {
                        event.playerId = playerId;
                        PublishEvent(event);
                    }
                }
            }
        }
    }

    // ===== 백그라운드 워커 2: 게임 시작 조율 =====

    void StartCoordinatorWorker()
    {
        while (!shouldStop)
        {
            std::this_thread::sleep_for(
                std::chrono::milliseconds(1000)
            );

            std::lock_guard<std::mutex> lock(serverMutex);

            // 모든 방을 확인하여 게임 시작 여부 판단
            auto publicRooms = roomManager.GetPublicRooms();
            
            for (const auto& room : publicRooms)
            {
                if (room->state != RoomState::WAITING)
                    continue;

                // 모든 플레이어가 준비했는지 확인
                bool allReady = true;
                for (const auto& playerId : room->players)
                {
                    if (!room->readyStatus[playerId])
                    {
                        allReady = false;
                        break;
                    }
                }

                if (allReady)
                {
                    // 게임 시작
                    if (startCoordinator.TryStartGame(
                        room->roomId))
                    {
                        // 게임 시작 이벤트 발행
                        for (const auto& playerId : 
                             room->players)
                        {
                            LobbyEventData event;
                            event.eventType = LobbyEvent::GAME_START;
                            event.playerId = playerId;
                            event.roomId = room->roomId;
                            event.gameServerId = 
                                "game.server.com:9000";

                            PublishEvent(event);

                            playerStates[playerId] = 
                                PlayerLobbyState::IN_GAME;
                        }

                        startCoordinator.RemoveRoomReadyState(
                            room->roomId
                        );
                    }
                }
            }

            // 타임아웃된 방 처리
            auto timeoutRooms = 
                startCoordinator.GetReadyTimeoutRooms();
            
            for (const auto& roomId : timeoutRooms)
            {
                auto room = roomManager.GetRoom(roomId);
                if (!room)
                    continue;

                // 준비하지 않은 플레이어들을 방에서 제거
                auto notReady = 
                    startCoordinator.GetNotReadyPlayers(
                        roomId
                    );

                for (const auto& playerId : notReady)
                {
                    roomManager.RemovePlayerFromRoom(
                        roomId, 
                        playerId
                    );
                    playerStates.erase(playerId);

                    // 타임아웃 이벤트
                    LobbyEventData event;
                    event.eventType = 
                        LobbyEvent::MATCH_TIMEOUT;
                    event.playerId = playerId;
                    PublishEvent(event);
                }

                // 남은 플레이어가 충분하면 게임 시작
                if (room->players.size() >= 2)
                {
                    // 모든 남은 플레이어를 준비 상태로 표시
                    for (const auto& playerId : 
                         room->players)
                    {
                        room->readyStatus[playerId] = true;
                    }
                }
            }
        }
    }

public:
    // ===== 통계 및 모니터링 =====

    struct LobbyStats
    {
        size_t totalPlayers;
        size_t searchingPlayers;
        size_t matchedPlayers;
        size_t inGamePlayers;
        size_t totalRooms;
    };

    LobbyStats GetLobbyStats()
    {
        std::lock_guard<std::mutex> lock(serverMutex);

        LobbyStats stats = {0};
        stats.totalRooms = roomManager.GetRoomCount();

        for (const auto& entry : playerStates)
        {
            stats.totalPlayers++;
            switch (entry.second)
            {
            case PlayerLobbyState::SEARCHING:
                stats.searchingPlayers++;
                break;
            case PlayerLobbyState::MATCHED:
                stats.matchedPlayers++;
                break;
            case PlayerLobbyState::IN_GAME:
                stats.inGamePlayers++;
                break;
            default:
                break;
            }
        }

        return stats;
    }
};
```

**주요 부분 상세 해설**:

1. **MatchingWorker()**: 500ms마다 매칭 처리를 수행한다. 매칭이 완료되면 방을 생성하고, 모든 플레이어에게 매칭 완료 이벤트를 발행한다.

2. **StartCoordinatorWorker()**: 1초마다 모든 방을 확인하여 게임 시작 여부를 판단한다. 모든 플레이어가 준비하면 게임 서버 주소를 클라이언트에게 전달한다.

3. **타임아웃 처리**: 10초 내에 준비하지 않은 플레이어는 방에서 자동으로 제거되고, 남은 플레이어가 충분하면 게임을 시작한다.

4. **이벤트 기반 설계**: 매칭 완료, 게임 준비, 게임 시작 등의 이벤트를 콜백 함수로 처리하여, 상위 계층(클라이언트, 게임 서버 연결 모듈)과의 느슨한 결합을 유지한다.

5. **멀티스레드 안전성**: 모든 공유 데이터 접근을 뮤텍스로 보호하여, 워커 스레드와 메인 스레드 간의 경쟁 조건을 방지한다.

**개선 방안과 확장 가능성**:

첫째, **매칭 알고리즘 고도화**: 현재는 단순한 레이팅 기반 매칭이지만, 게임 모드별, 맵별, 지역별 등 다양한 조건을 추가할 수 있다.

둘째, **동적 서버 할당**: 현재는 고정된 게임 서버 주소를 반환하지만, 실제로는 부하가 가장 낮은 게임 서버를 동적으로 선택할 수 있다.

셋째, **플레이어 선호도 추적**: 자주 함께 게임하는 플레이어들을 우선적으로 매칭하여 커뮤니티 형성을 장려한다.

네 번째, **스킬 기반 매칭 균형**: 한 팀이 너무 강하고 다른 팀이 약한 경우를 감지하여 플레이어를 교환한다.

다섯 번째, **순위 시스템 통합**: 매칭 결과에 따른 순위 변화를 계산하고 저장한다.

여섯 번째, **보상 시스템**: 승리한 팀과 패배한 팀에 경험치, 골드, 배지 등의 보상을 차등 지급한다.

일곱째, **실패 복구**: 게임 중 플레이어가 나가면 대체 플레이어를 찾아서 투입하거나, AI가 그 자리를 대신한다.

이 완전한 게임 로비 시스템은 실제 온라인 게임에 적용할 수 있으며, 수만 명의 플레이어를 동시에 처리할 수 있도록 확장할 수 있다. 데이터베이스 연동, 마이크로서비스 아키텍처, 분산 시스템 등으로 더욱 고도화할 수 있다.
