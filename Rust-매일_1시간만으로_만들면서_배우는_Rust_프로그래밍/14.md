# Day 14: 여러 클라이언트 순차 처리
지난 시간까지 우리는 enum으로 프로토콜을 정의하고 한 번에 한 클라이언트를 처리하는 서버를 만들었다. 하지만 현재 서버는 한 클라이언트가 연결을 끊어야만 다음 클라이언트를 받을 수 있다. 오늘은 여러 클라이언트를 순차적으로 처리하는 방법과, 효율적인 버퍼링을 위한 `BufReader`와 `BufWriter`를 배운다. 그리고 이 과정에서 참조와 버퍼의 관계를 깊이 이해하게 될 것이다.

## **현재 서버의 문제점**

지금까지 만든 서버는 다음과 같은 구조였다.

```rust
fn main() -> io::Result<()> {
    let listener = TcpListener::bind("127.0.0.1:8080")?;
    
    for stream in listener.incoming() {
        let stream = stream?;
        handle_client(stream)?; // 여기서 블록됨!
    }
    
    Ok(())
}
```

문제는 `handle_client()`가 완전히 끝날 때까지 다음 클라이언트를 받을 수 없다는 것이다. 실제 상황을 시뮬레이션해보자.

```
시간 흐름:
─────────────────────────────────────────────────

Client A 연결 ───┐
                │
                └─> handle_client(A) 실행 중...
                    (Client A가 종료할 때까지 대기)
                
Client B 연결 시도 ─> 서버가 응답 안함 (대기 중)
                
Client A 종료 ──┐
                │
                └─> handle_client(A) 종료
                
                    이제서야 Client B 수락 가능
```

이것은 받아들일 수 없는 동작이다. 다행히 오늘은 순차 처리만 다루므로 간단하게 개선할 수 있다. 내일은 스레드를 사용해서 진짜 동시 처리를 구현할 것이다.

## **에러 처리 개선: 클라이언트 에러로 서버 중단 방지**

첫 번째 개선은 한 클라이언트의 에러가 전체 서버를 멈추지 않도록 하는 것이다.

```rust
fn main() -> io::Result<()> {
    let listener = TcpListener::bind("127.0.0.1:8080")?;
    println!("Server listening on 127.0.0.1:8080");
    println!("Press Ctrl+C to stop\n");
    
    for stream in listener.incoming() {
        match stream {
            Ok(stream) => {
                // 클라이언트 처리 중 에러가 발생해도 서버는 계속 실행
                if let Err(e) = handle_client(stream) {
                    eprintln!("Error handling client: {}", e);
                    // 다음 클라이언트로 계속 진행
                }
            }
            Err(e) => {
                eprintln!("Failed to accept connection: {}", e);
                // 연결 수락 실패도 치명적이지 않음
            }
        }
    }
    
    Ok(())
}
```

이제 한 클라이언트가 문제를 일으켜도 서버는 계속 실행되고 다음 클라이언트를 받을 수 있다.

```
Client A → 에러 발생 → 로그 출력 → 다음 클라이언트 대기
           ↓
        서버 계속 실행
```

## **BufReader와 BufWriter: 왜 필요한가**

지금까지 우리는 `TcpStream`에서 직접 읽고 썼다. 하지만 이것은 비효율적이다. 네트워크 I/O는 시스템 콜을 사용하는데, 이것은 비용이 크다. 한 바이트씩 읽는다면 매번 시스템 콜이 발생한다.

```
효율이 낮은 방식:
┌──────────┐      read(1 byte)      ┌────────┐
│   앱     │ ───────────────────────>│  커널  │  시스템 콜
└──────────┘                        └────────┘
     │
     │ 다시 1 byte 필요
     ▼
┌──────────┐      read(1 byte)      ┌────────┐
│   앱     │ ───────────────────────>│  커널  │  또 시스템 콜!
└──────────┘                        └────────┘
```

`BufReader`는 큰 청크로 읽어서 내부 버퍼에 저장한다. 그리고 애플리케이션은 이 버퍼에서 데이터를 가져온다.

```
효율적인 방식 (BufReader):
┌──────────┐                        ┌────────┐
│   앱     │      read(4096 bytes)  │  커널  │  시스템 콜 1번
│          │ ───────────────────────>│        │
│          │                        └────────┘
│ BufReader│                             │
│ ┌──────┐ │                             │
│ │ 버퍼  │ │ <───────────────────────────┘
│ │4096  │ │
│ └──────┘ │
└────┬─────┘
     │
     │ 앱이 필요할 때마다 버퍼에서 가져옴
     │ (시스템 콜 없음!)
     ▼
  read_line()
```

## **BufReader로 라인 기반 프로토콜 구현하기**

우리의 프로토콜은 라인 기반이다. 클라이언트가 `ADD 1 2\n`처럼 개행 문자로 끝나는 명령을 보낸다. `BufReader`의 `read_line()` 메서드는 이런 상황에 완벽하다.

```rust
use std::io::{BufRead, BufReader, Write};
use std::net::TcpStream;

fn handle_client(stream: TcpStream) -> io::Result<()> {
    let peer_addr = stream.peer_addr()?;
    println!("[{}] Connected", peer_addr);
    
    // TcpStream을 BufReader로 감싸기
    let mut reader = BufReader::new(stream.try_clone()?);
    let mut writer = stream;
    
    // 환영 메시지
    writer.write_all(b"Welcome to the server!\n")?;
    writer.flush()?;
    
    loop {
        let mut line = String::new();
        
        // 개행 문자까지 읽기
        let bytes_read = reader.read_line(&mut line)?;
        
        if bytes_read == 0 {
            // 클라이언트가 연결을 닫음
            println!("[{}] Disconnected", peer_addr);
            break;
        }
        
        println!("[{}] Received: {:?}", peer_addr, line.trim());
        
        // 명령 처리...
        writer.write_all(b"OK\n")?;
        writer.flush()?;
    }
    
    Ok(())
}
```

여기서 중요한 점이 몇 가지 있다. 첫째, `stream.try_clone()?`을 사용해서 스트림을 복제한다. 이것은 같은 네트워크 연결을 가리키는 두 개의 핸들을 만든다. 하나는 읽기용 `BufReader`로 감싸고, 다른 하나는 쓰기용으로 직접 사용한다.

```
    TcpStream (원본)
         │
         ├─> try_clone() ─> TcpStream (복제)
         │                       │
         │                  BufReader로 감싸기
         │                       │
         │                    reader (읽기용)
         │
         └─> writer (쓰기용)
```

둘째, `flush()`를 호출한다. 이것은 버퍼에 있는 데이터를 즉시 네트워크로 보낸다. 그렇지 않으면 데이터가 버퍼에 남아있을 수 있다.

## **참조와 버퍼의 관계 이해하기**

`BufReader`의 내부를 들여다보면 소유권과 빌림의 실전 사례를 볼 수 있다.

```rust
// BufReader의 개념적 구조 (실제는 더 복잡함)
pub struct BufReader<R> {
    inner: R,           // TcpStream을 소유
    buf: Vec<u8>,       // 내부 버퍼를 소유
    pos: usize,         // 현재 읽기 위치
    cap: usize,         // 버퍼에 있는 유효한 데이터 양
}
```

`BufReader::new(stream)`을 호출하면 `TcpStream`의 소유권이 `BufReader`로 이동한다. 이제 스트림은 `BufReader` 안에 있고, 우리는 직접 접근할 수 없다.

```rust
let stream = TcpStream::connect("127.0.0.1:8080")?;
let reader = BufReader::new(stream);
// 이제 stream은 사용할 수 없음!
// stream.write_all(...)?; // 컴파일 에러!
```

이것이 우리가 `try_clone()`을 사용한 이유다. 스트림을 복제해서 하나는 읽기용, 하나는 쓰기용으로 사용한다.

```rust
let stream = TcpStream::connect("127.0.0.1:8080")?;
let reader = BufReader::new(stream.try_clone()?);
let mut writer = stream;

// 이제 둘 다 사용 가능
reader.read_line(&mut line)?;
writer.write_all(b"OK\n")?;
```

## **BufWriter로 쓰기 최적화하기**

읽기에 `BufReader`가 있다면, 쓰기에는 `BufWriter`가 있다. 이것도 같은 원리로 작동한다. 작은 쓰기들을 버퍼에 모았다가 한꺼번에 보낸다.

```rust
use std::io::{BufRead, BufReader, BufWriter, Write};

fn handle_client_buffered(stream: TcpStream) -> io::Result<()> {
    let peer_addr = stream.peer_addr()?;
    println!("[{}] Connected", peer_addr);
    
    let reader = BufReader::new(stream.try_clone()?);
    let mut writer = BufWriter::new(stream);
    
    writer.write_all(b"Welcome!\n")?;
    writer.flush()?; // 즉시 전송하려면 flush 필요
    
    for line in reader.lines() {
        let line = line?;
        println!("[{}] Received: {}", peer_addr, line);
        
        // 여러 번 write_all을 호출해도 버퍼에 모임
        writer.write_all(b"You said: ")?;
        writer.write_all(line.as_bytes())?;
        writer.write_all(b"\n")?;
        writer.flush()?; // 응답을 즉시 보냄
    }
    
    println!("[{}] Disconnected", peer_addr);
    Ok(())
}
```

`BufWriter`를 사용하면 여러 작은 쓰기 작업이 하나의 시스템 콜로 합쳐진다.

```
BufWriter 없이:
write("You said: ") ─────> 시스템 콜
write("hello")      ─────> 시스템 콜
write("\n")         ─────> 시스템 콜

BufWriter 사용:
write("You said: ") ──┐
write("hello")        ├─> 버퍼에 모음
write("\n")           ┘
flush()               ─────> 시스템 콜 1번
```

## **완전한 서버 구현**

이제 배운 모든 것을 합쳐서 개선된 서버를 만들어보자.

```rust
use std::io::{self, BufRead, BufReader, BufWriter, Write};
use std::net::{TcpListener, TcpStream};
use std::fmt;
use std::num::ParseIntError;

// Request와 Response enum (Day 13에서 정의한 것과 동일)
#[derive(Debug, Clone)]
enum Request {
    Ping,
    Echo(String),
    Add(i32, i32),
    Subtract(i32, i32),
    Multiply(i32, i32),
    Divide(i32, i32),
    Reverse(String),
    Quit,
}

#[derive(Debug, Clone)]
enum Response {
    Pong,
    Message(String),
    Number(i32),
    Error(String),
    Goodbye,
}

#[derive(Debug)]
enum ParseError {
    EmptyCommand,
    UnknownCommand(String),
    InvalidArgument(String),
    ParseIntError(ParseIntError),
}

impl fmt::Display for ParseError {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        match self {
            ParseError::EmptyCommand => write!(f, "Empty command"),
            ParseError::UnknownCommand(cmd) => write!(f, "Unknown command: {}", cmd),
            ParseError::InvalidArgument(msg) => write!(f, "Invalid argument: {}", msg),
            ParseError::ParseIntError(e) => write!(f, "Parse error: {}", e),
        }
    }
}

impl std::error::Error for ParseError {}

impl From<ParseIntError> for ParseError {
    fn from(error: ParseIntError) -> Self {
        ParseError::ParseIntError(error)
    }
}

// 파싱 함수
fn parse_request(line: &str) -> Result<Request, ParseError> {
    let parts: Vec<&str> = line.trim().split_whitespace().collect();
    
    if parts.is_empty() {
        return Err(ParseError::EmptyCommand);
    }
    
    match parts[0].to_uppercase().as_str() {
        "PING" => Ok(Request::Ping),
        "QUIT" => Ok(Request::Quit),
        
        "ECHO" => {
            if parts.len() < 2 {
                return Err(ParseError::InvalidArgument(
                    "ECHO requires a message".to_string()
                ));
            }
            Ok(Request::Echo(parts[1..].join(" ")))
        }
        
        "ADD" => {
            if parts.len() != 3 {
                return Err(ParseError::InvalidArgument(
                    "ADD requires exactly two numbers".to_string()
                ));
            }
            let x = parts[1].parse::<i32>()?;
            let y = parts[2].parse::<i32>()?;
            Ok(Request::Add(x, y))
        }
        
        "SUB" => {
            if parts.len() != 3 {
                return Err(ParseError::InvalidArgument(
                    "SUB requires exactly two numbers".to_string()
                ));
            }
            let x = parts[1].parse::<i32>()?;
            let y = parts[2].parse::<i32>()?;
            Ok(Request::Subtract(x, y))
        }
        
        "MUL" => {
            if parts.len() != 3 {
                return Err(ParseError::InvalidArgument(
                    "MUL requires exactly two numbers".to_string()
                ));
            }
            let x = parts[1].parse::<i32>()?;
            let y = parts[2].parse::<i32>()?;
            Ok(Request::Multiply(x, y))
        }
        
        "DIV" => {
            if parts.len() != 3 {
                return Err(ParseError::InvalidArgument(
                    "DIV requires exactly two numbers".to_string()
                ));
            }
            let x = parts[1].parse::<i32>()?;
            let y = parts[2].parse::<i32>()?;
            Ok(Request::Divide(x, y))
        }
        
        "REVERSE" => {
            if parts.len() < 2 {
                return Err(ParseError::InvalidArgument(
                    "REVERSE requires a string".to_string()
                ));
            }
            Ok(Request::Reverse(parts[1..].join(" ")))
        }
        
        cmd => Err(ParseError::UnknownCommand(cmd.to_string())),
    }
}

// 요청 처리
fn handle_request(request: Request) -> Response {
    match request {
        Request::Ping => Response::Pong,
        Request::Echo(msg) => Response::Message(msg),
        Request::Add(x, y) => Response::Number(x + y),
        Request::Subtract(x, y) => Response::Number(x - y),
        Request::Multiply(x, y) => Response::Number(x * y),
        Request::Divide(x, y) => {
            if y == 0 {
                Response::Error("Division by zero".to_string())
            } else {
                Response::Number(x / y)
            }
        }
        Request::Reverse(text) => {
            Response::Message(text.chars().rev().collect())
        }
        Request::Quit => Response::Goodbye,
    }
}

// 응답 포맷
fn format_response(response: &Response) -> String {
    match response {
        Response::Pong => "PONG".to_string(),
        Response::Message(msg) => msg.clone(),
        Response::Number(n) => n.to_string(),
        Response::Error(msg) => format!("ERROR: {}", msg),
        Response::Goodbye => "Goodbye!".to_string(),
    }
}

// 클라이언트 처리 (개선된 버전)
fn handle_client(stream: TcpStream) -> io::Result<()> {
    let peer_addr = stream.peer_addr()?;
    println!("[{}] Connected", peer_addr);
    
    // 버퍼링된 reader와 writer 생성
    let reader = BufReader::new(stream.try_clone()?);
    let mut writer = BufWriter::new(stream);
    
    // 환영 메시지
    writeln!(writer, "===== Buffered Protocol Server =====")?;
    writeln!(writer, "Commands: PING, ECHO <msg>, ADD/SUB/MUL/DIV <x> <y>")?;
    writeln!(writer, "          REVERSE <text>, QUIT")?;
    writeln!(writer, "====================================")?;
    writer.flush()?;
    
    // lines() 이터레이터 사용
    for line_result in reader.lines() {
        let line = match line_result {
            Ok(line) => line,
            Err(e) => {
                eprintln!("[{}] Read error: {}", peer_addr, e);
                break;
            }
        };
        
        // 빈 라인 무시
        if line.trim().is_empty() {
            continue;
        }
        
        println!("[{}] Request: {}", peer_addr, line.trim());
        
        // 파싱 및 처리
        let response = match parse_request(&line) {
            Ok(request) => {
                let is_quit = matches!(request, Request::Quit);
                let response = handle_request(request);
                
                if is_quit {
                    let output = format_response(&response);
                    writeln!(writer, "{}", output)?;
                    writer.flush()?;
                    println!("[{}] Client quit", peer_addr);
                    break;
                }
                
                response
            }
            Err(e) => {
                eprintln!("[{}] Parse error: {}", peer_addr, e);
                Response::Error(e.to_string())
            }
        };
        
        // 응답 전송
        let output = format_response(&response);
        writeln!(writer, "{}", output)?;
        writer.flush()?;
        
        println!("[{}] Response: {:?}", peer_addr, response);
    }
    
    println!("[{}] Disconnected", peer_addr);
    Ok(())
}

// 메인 함수
fn main() -> io::Result<()> {
    let listener = TcpListener::bind("127.0.0.1:8080")?;
    println!("Buffered server listening on 127.0.0.1:8080");
    println!("Press Ctrl+C to stop\n");
    
    let mut client_count = 0u32;
    
    for stream in listener.incoming() {
        match stream {
            Ok(stream) => {
                client_count += 1;
                println!("==> Client #{} connected", client_count);
                
                if let Err(e) = handle_client(stream) {
                    eprintln!("Error handling client #{}: {}", client_count, e);
                }
                
                println!("<== Client #{} session ended\n", client_count);
            }
            Err(e) => {
                eprintln!("Failed to accept connection: {}", e);
            }
        }
    }
    
    Ok(())
}
```

## **lines() 이터레이터의 장점**

우리는 `reader.lines()`를 사용했다. 이것은 `Iterator` trait를 구현하는데, 매우 편리한 기능들을 제공한다.

```rust
// 명시적 루프
loop {
    let mut line = String::new();
    let bytes_read = reader.read_line(&mut line)?;
    if bytes_read == 0 {
        break;
    }
    // 라인 처리...
}

// lines() 이터레이터 사용 (더 간결함)
for line_result in reader.lines() {
    let line = line_result?;
    // 라인 처리...
    // EOF는 자동으로 처리됨
}
```

`lines()` 이터레이터는 각 라인의 끝에서 개행 문자를 제거해준다. 또한 EOF(파일 끝)를 자동으로 처리한다.

## **참조와 버퍼: 깊이 이해하기**

`BufReader`와 `BufWriter`를 사용할 때 소유권과 빌림이 어떻게 작동하는지 자세히 살펴보자.

```rust
fn demonstrate_buffer_ownership() -> io::Result<()> {
    let stream = TcpStream::connect("127.0.0.1:8080")?;
    
    // 방법 1: stream의 소유권을 BufReader에게 줌
    let mut reader = BufReader::new(stream);
    // 이제 stream은 사용할 수 없음
    
    // 방법 2: 복제해서 각각 사용
    let stream = TcpStream::connect("127.0.0.1:8080")?;
    let mut reader = BufReader::new(stream.try_clone()?);
    let mut writer = stream;
    // 둘 다 사용 가능
    
    // reader가 참조를 반환함을 주목
    let mut line = String::new();
    reader.read_line(&mut line)?; // line의 가변 참조를 빌림
    
    Ok(())
}
```

`read_line(&mut self, buf: &mut String)`의 시그니처를 보자. 이것은 `String`의 가변 참조를 받는다. 이미 할당된 버퍼를 재사용할 수 있다는 의미다.

```rust
// 비효율적: 매번 새로운 String 할당
loop {
    let mut line = String::new();
    reader.read_line(&mut line)?;
    // line은 여기서 드롭됨
}

// 효율적: String 재사용
let mut line = String::new();
loop {
    line.clear(); // 내용만 지우고 할당된 메모리는 유지
    reader.read_line(&mut line)?;
    // 같은 String 객체 재사용
}
```

이것은 참조와 빌림의 실용적인 예다. `read_line`은 `String`을 소유하지 않고 빌려서 내용을 채운다.

```
┌──────────────┐
│   line       │  String 객체 (스택)
│  ┌────────┐  │
│  │ptr     │──┼─────> 힙의 버퍼
│  │len     │  │
│  │capacity│  │
│  └────────┘  │
└──────┬───────┘
       │
       │ &mut 빌림
       ▼
┌──────────────┐
│ read_line()  │  내용을 채움
└──────────────┘
       │
       ▼
line은 여전히 소유권을 가짐
```

## **버퍼 크기와 성능**

`BufReader`와 `BufWriter`의 기본 버퍼 크기는 8KB다. 필요하면 변경할 수 있다.

```rust
// 기본 크기 (8KB)
let reader = BufReader::new(stream);

// 커스텀 크기 (64KB)
let reader = BufReader::with_capacity(64 * 1024, stream);

// 작은 크기 (1KB)
let writer = BufWriter::with_capacity(1024, stream);
```

일반적으로 기본 크기가 적절하다. 하지만 매우 큰 데이터를 처리하거나 메모리가 제한적인 환경에서는 조정이 필요할 수 있다.

```
작은 버퍼 (1KB):
┌──────┐
│ 1KB  │ ─> 빈번한 시스템 콜
└──────┘

큰 버퍼 (64KB):
┌──────────────────────────────────┐
│            64KB                  │ ─> 드문 시스템 콜
└──────────────────────────────────┘
```

## **순차 처리의 한계**

현재 서버는 여러 클라이언트를 받을 수 있지만, 여전히 순차적이다. 한 클라이언트가 오래 걸리는 작업을 하면 다른 클라이언트들이 기다려야 한다.

```
Client A: 긴 작업 중 ──────────────────>
                        │
Client B: 기다림        │
Client C: 기다림        │
                        ▼
                     A 완료 후
                     B 시작 ──────>
                               │
                               ▼
                            B 완료 후
                            C 시작 ─>
```

다음 시간에는 스레드를 사용해서 각 클라이언트를 동시에 처리하는 방법을 배운다. 그러면 한 클라이언트의 작업이 다른 클라이언트를 블록하지 않는다.

## **정리하면**

오늘 우리는 여러 클라이언트를 순차적으로 처리하는 서버를 만들었다. `BufReader`와 `BufWriter`로 네트워크 I/O를 효율적으로 버퍼링하는 방법을 배웠고, `lines()` 이터레이터로 라인 기반 프로토콜을 깔끔하게 처리했다. 또한 `try_clone()`으로 스트림을 복제해서 읽기와 쓰기를 분리하는 패턴을 익혔다.

참조와 버퍼의 관계를 통해 Rust가 어떻게 안전하면서도 효율적인 코드를 작성하게 하는지 실전에서 경험했다. `read_line`은 버퍼를 빌려서 재사용할 수 있게 하고, 버퍼링 계층은 시스템 콜을 최소화한다.

다음 시간에는 스레드를 도입해서 진정한 동시 처리를 구현할 것이다. 그리고 소유권이 스레드 경계를 넘을 때 어떻게 작동하는지 배우게 될 것이다.
