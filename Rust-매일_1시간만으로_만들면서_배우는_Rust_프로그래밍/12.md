# Day 12: 에러 처리 심화

지난 시간까지 우리는 간단한 TCP 에코 서버를 만들었다. 서버는 클라이언트의 연결을 받고, 데이터를 읽어서 그대로 돌려보낸다. 하지만 실제 세계에서는 많은 것들이 잘못될 수 있다. 네트워크가 끊길 수 있고, 잘못된 데이터가 들어올 수 있으며, 파일 시스템 접근이 실패할 수도 있다. 오늘은 Rust에서 이러한 에러를 우아하게 처리하는 방법을 배운다.

## **에러는 왜 중요한가**

프로그램이 에러를 제대로 처리하지 못하면 사용자는 "무엇이 잘못되었는지" 알 수 없다. 더 나쁜 경우, 프로그램이 예상치 못한 상태로 계속 실행되어 데이터가 손상될 수도 있다. Rust는 에러를 타입 시스템의 일부로 만들어서 컴파일러가 우리에게 에러 처리를 강제한다. 이것은 처음에는 번거로울 수 있지만, 결국 더 견고한 프로그램을 만들게 해준다.

## **? 연산자: 에러 전파의 마법**

이전에 `Result<T, E>`를 사용할 때 `match`나 `unwrap()`을 써서 처리했다. 하지만 실제 프로그램에서는 여러 단계에서 에러가 발생할 수 있고, 매번 `match`를 쓰면 코드가 지저분해진다. `?` 연산자는 이 문제를 해결한다.

```rust
use std::fs::File;
use std::io::{self, Read};

// ? 연산자 없이 작성한 코드
fn read_file_verbose(path: &str) -> Result<String, io::Error> {
    let mut file = match File::open(path) {
        Ok(f) => f,
        Err(e) => return Err(e),
    };
    
    let mut contents = String::new();
    match file.read_to_string(&mut contents) {
        Ok(_) => Ok(contents),
        Err(e) => Err(e),
    }
}

// ? 연산자를 사용한 간결한 코드
fn read_file_simple(path: &str) -> Result<String, io::Error> {
    let mut file = File::open(path)?;
    let mut contents = String::new();
    file.read_to_string(&mut contents)?;
    Ok(contents)
}
```

`?` 연산자는 `Result`가 `Ok`면 그 안의 값을 꺼내고, `Err`면 현재 함수에서 즉시 그 에러를 반환한다. 이것은 에러를 "위로 전파"하는 깔끔한 방법이다. 단, `?`를 사용하는 함수는 반드시 `Result`를 반환해야 한다.

```
┌─────────────────────────────────────┐
│   read_file_simple("data.txt")      │
│                                     │
│   File::open("data.txt")?           │
│        │                            │
│        ├─ Ok(file) ──────────────┐  │
│        │                         │  │
│        └─ Err(e) ─────> return Err(e)
│                                  │  │
│   file.read_to_string()?         │  │
│        │                         │  │
│        ├─ Ok(()) ──────────────┐ │  │
│        │                       │ │  │
│        └─ Err(e) ─> return Err(e) │
│                                │ │  │
│   Ok(contents) <───────────────┴─┘  │
└─────────────────────────────────────┘
```

## **TCP 서버에 ? 연산자 적용하기**

지난 시간의 에코 서버를 개선해보자. 기존 코드는 `unwrap()`을 많이 사용했는데, 이것은 에러가 발생하면 프로그램이 패닉을 일으킨다. 프로덕션 서버에서는 이것은 받아들일 수 없다.

```rust
use std::net::{TcpListener, TcpStream};
use std::io::{self, Read, Write};

fn handle_client(mut stream: TcpStream) -> io::Result<()> {
    let mut buffer = [0; 512];
    
    loop {
        let bytes_read = stream.read(&mut buffer)?;
        
        if bytes_read == 0 {
            // 클라이언트가 연결을 닫음
            println!("Client disconnected");
            return Ok(());
        }
        
        stream.write_all(&buffer[..bytes_read])?;
    }
}

fn main() -> io::Result<()> {
    let listener = TcpListener::bind("127.0.0.1:8080")?;
    println!("Server listening on port 8080");
    
    for stream in listener.incoming() {
        match stream {
            Ok(stream) => {
                println!("New connection: {}", stream.peer_addr()?);
                
                if let Err(e) = handle_client(stream) {
                    eprintln!("Error handling client: {}", e);
                }
            }
            Err(e) => {
                eprintln!("Connection failed: {}", e);
            }
        }
    }
    
    Ok(())
}
```

이 코드에서 주목할 점이 몇 가지 있다. 첫째, `main` 함수가 이제 `io::Result<()>`를 반환한다. Rust는 `main`이 `Result`를 반환하는 것을 허용하며, 에러가 발생하면 자동으로 에러 메시지를 출력하고 프로그램을 종료한다. 둘째, `handle_client` 함수에서 `?` 연산자를 사용해서 읽기와 쓰기 에러를 전파한다. 셋째, 메인 루프에서는 `match`를 사용해서 각 클라이언트의 에러를 처리하지만 서버 전체를 중단하지는 않는다.

## **커스텀 에러 타입 만들기**

서버가 복잡해지면 여러 종류의 에러가 발생한다. 네트워크 에러, 파싱 에러, 인증 에러 등이 있을 수 있다. 이들을 하나의 타입으로 통합하면 에러 처리가 일관되고 명확해진다.

```rust
use std::fmt;
use std::io;
use std::num::ParseIntError;

#[derive(Debug)]
enum ServerError {
    IoError(io::Error),
    ParseError(ParseIntError),
    ProtocolError(String),
    AuthError(String),
}

impl fmt::Display for ServerError {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        match self {
            ServerError::IoError(e) => write!(f, "IO error: {}", e),
            ServerError::ParseError(e) => write!(f, "Parse error: {}", e),
            ServerError::ProtocolError(msg) => write!(f, "Protocol error: {}", msg),
            ServerError::AuthError(msg) => write!(f, "Authentication error: {}", msg),
        }
    }
}

impl std::error::Error for ServerError {}

// io::Error를 ServerError로 자동 변환
impl From<io::Error> for ServerError {
    fn from(error: io::Error) -> Self {
        ServerError::IoError(error)
    }
}

// ParseIntError를 ServerError로 자동 변환
impl From<ParseIntError> for ServerError {
    fn from(error: ParseIntError) -> Self {
        ServerError::ParseError(error)
    }
}
```

`ServerError`는 `enum`으로 여러 종류의 에러를 표현한다. 각 변형(variant)은 다른 에러 타입을 감싸거나 메시지를 포함한다. `Display` trait를 구현해서 에러를 사용자가 읽을 수 있는 형태로 출력할 수 있다. `std::error::Error` trait를 구현하면 이 타입이 표준 에러 타입으로 인식된다.

특히 중요한 것은 `From` trait 구현이다. 이것은 `?` 연산자가 자동으로 에러를 변환할 수 있게 해준다. 예를 들어 `stream.read(&mut buffer)?`는 `io::Error`를 반환하지만, `From<io::Error>` 구현 덕분에 자동으로 `ServerError`로 변환된다.

```
         ┌──────────────┐
         │  io::Error   │
         └──────┬───────┘
                │
                │ From<io::Error>
                │
                ▼
         ┌──────────────────┐
         │  ServerError::   │
         │  IoError(...)    │
         └──────────────────┘
```

## **실전: 프로토콜 파서에 에러 처리 추가하기**

이제 간단한 프로토콜을 파싱하는 함수를 만들어보자. 클라이언트가 "PING", "ECHO message", "ADD x y" 같은 명령을 보낼 수 있다고 가정한다.

```rust
use std::io::{self, BufRead, BufReader, Write};
use std::net::TcpStream;

type Result<T> = std::result::Result<T, ServerError>;

enum Command {
    Ping,
    Echo(String),
    Add(i32, i32),
    Quit,
}

fn parse_command(line: &str) -> Result<Command> {
    let parts: Vec<&str> = line.trim().split_whitespace().collect();
    
    if parts.is_empty() {
        return Err(ServerError::ProtocolError("Empty command".to_string()));
    }
    
    match parts[0] {
        "PING" => Ok(Command::Ping),
        "QUIT" => Ok(Command::Quit),
        "ECHO" => {
            if parts.len() < 2 {
                return Err(ServerError::ProtocolError(
                    "ECHO requires a message".to_string()
                ));
            }
            Ok(Command::Echo(parts[1..].join(" ")))
        }
        "ADD" => {
            if parts.len() != 3 {
                return Err(ServerError::ProtocolError(
                    "ADD requires exactly two numbers".to_string()
                ));
            }
            let x = parts[1].parse::<i32>()?; // From<ParseIntError> 덕분에 자동 변환
            let y = parts[2].parse::<i32>()?;
            Ok(Command::Add(x, y))
        }
        cmd => Err(ServerError::ProtocolError(
            format!("Unknown command: {}", cmd)
        )),
    }
}

fn execute_command(command: Command, stream: &mut TcpStream) -> Result<bool> {
    match command {
        Command::Ping => {
            stream.write_all(b"PONG\n")?;
        }
        Command::Echo(msg) => {
            stream.write_all(msg.as_bytes())?;
            stream.write_all(b"\n")?;
        }
        Command::Add(x, y) => {
            let result = x + y;
            stream.write_all(format!("{}\n", result).as_bytes())?;
        }
        Command::Quit => {
            stream.write_all(b"Goodbye\n")?;
            return Ok(false); // 연결 종료 신호
        }
    }
    Ok(true) // 연결 유지
}

fn handle_client_v2(stream: TcpStream) -> Result<()> {
    let peer_addr = stream.peer_addr()?;
    println!("Client connected: {}", peer_addr);
    
    let mut reader = BufReader::new(stream.try_clone()?);
    let mut writer = stream;
    
    writer.write_all(b"Welcome! Commands: PING, ECHO <msg>, ADD <x> <y>, QUIT\n")?;
    
    loop {
        let mut line = String::new();
        let bytes_read = reader.read_line(&mut line)?;
        
        if bytes_read == 0 {
            println!("Client {} disconnected", peer_addr);
            break;
        }
        
        match parse_command(&line) {
            Ok(command) => {
                match execute_command(command, &mut writer) {
                    Ok(should_continue) => {
                        if !should_continue {
                            println!("Client {} quit", peer_addr);
                            break;
                        }
                    }
                    Err(e) => {
                        eprintln!("Error executing command: {}", e);
                        writer.write_all(format!("ERROR: {}\n", e).as_bytes())?;
                    }
                }
            }
            Err(e) => {
                eprintln!("Error parsing command: {}", e);
                writer.write_all(format!("ERROR: {}\n", e).as_bytes())?;
            }
        }
    }
    
    Ok(())
}
```

이 코드는 여러 층의 에러 처리를 보여준다. `parse_command`는 명령을 파싱하면서 프로토콜 에러나 파싱 에러를 반환할 수 있다. `execute_command`는 네트워크 쓰기 에러를 반환할 수 있다. `handle_client_v2`는 이 모든 에러를 처리하며, 각 에러에 따라 적절히 반응한다. 치명적이지 않은 에러는 클라이언트에게 에러 메시지를 보내고 연결을 유지하지만, I/O 에러 같은 심각한 에러는 함수를 종료한다.

## **에러 컨텍스트 추가하기**

에러가 발생했을 때 "무엇을 하다가" 에러가 발생했는지 알면 디버깅이 훨씬 쉽다. 단순히 "파일을 열 수 없음"보다는 "설정 파일 /etc/server.conf를 열 수 없음"이 더 유용하다.

```rust
fn load_config(path: &str) -> Result<String> {
    std::fs::read_to_string(path)
        .map_err(|e| ServerError::ProtocolError(
            format!("Failed to load config from {}: {}", path, e)
        ))
}
```

`map_err` 메서드는 에러를 변환하면서 추가 정보를 넣을 수 있다. 이것은 에러가 여러 층을 거쳐 올라올 때 각 층에서 컨텍스트를 추가하는 패턴이다.

```
    ┌────────────────────────────────────────┐
    │         Application Layer              │
    │  "Failed to start server"              │
    └───────────────┬────────────────────────┘
                    │ context added
    ┌───────────────▼────────────────────────┐
    │         Configuration Layer            │
    │  "Failed to load config from path"     │
    └───────────────┬────────────────────────┘
                    │ context added
    ┌───────────────▼────────────────────────┐
    │         File System Layer              │
    │  "Permission denied"                   │
    └────────────────────────────────────────┘
```

## **실전 예제: 완전한 서버**

이제 배운 모든 것을 합쳐서 견고한 서버를 만들어보자.

```rust
use std::io::{self, BufRead, BufReader, Write};
use std::net::{TcpListener, TcpStream};
use std::fmt;
use std::num::ParseIntError;

// 에러 타입 정의
#[derive(Debug)]
enum ServerError {
    IoError(io::Error),
    ParseError(ParseIntError),
    ProtocolError(String),
}

impl fmt::Display for ServerError {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        match self {
            ServerError::IoError(e) => write!(f, "IO error: {}", e),
            ServerError::ParseError(e) => write!(f, "Parse error: {}", e),
            ServerError::ProtocolError(msg) => write!(f, "Protocol error: {}", msg),
        }
    }
}

impl std::error::Error for ServerError {}

impl From<io::Error> for ServerError {
    fn from(error: io::Error) -> Self {
        ServerError::IoError(error)
    }
}

impl From<ParseIntError> for ServerError {
    fn from(error: ParseIntError) -> Self {
        ServerError::ParseError(error)
    }
}

type Result<T> = std::result::Result<T, ServerError>;

// 명령 타입
enum Command {
    Ping,
    Echo(String),
    Add(i32, i32),
    Multiply(i32, i32),
    Quit,
}

// 명령 파싱
fn parse_command(line: &str) -> Result<Command> {
    let parts: Vec<&str> = line.trim().split_whitespace().collect();
    
    if parts.is_empty() {
        return Err(ServerError::ProtocolError("Empty command".to_string()));
    }
    
    match parts[0].to_uppercase().as_str() {
        "PING" => Ok(Command::Ping),
        "QUIT" => Ok(Command::Quit),
        "ECHO" => {
            if parts.len() < 2 {
                return Err(ServerError::ProtocolError(
                    "ECHO requires a message".to_string()
                ));
            }
            Ok(Command::Echo(parts[1..].join(" ")))
        }
        "ADD" => {
            if parts.len() != 3 {
                return Err(ServerError::ProtocolError(
                    "ADD requires exactly two numbers".to_string()
                ));
            }
            let x = parts[1].parse::<i32>()?;
            let y = parts[2].parse::<i32>()?;
            Ok(Command::Add(x, y))
        }
        "MUL" => {
            if parts.len() != 3 {
                return Err(ServerError::ProtocolError(
                    "MUL requires exactly two numbers".to_string()
                ));
            }
            let x = parts[1].parse::<i32>()?;
            let y = parts[2].parse::<i32>()?;
            Ok(Command::Multiply(x, y))
        }
        cmd => Err(ServerError::ProtocolError(
            format!("Unknown command: {}", cmd)
        )),
    }
}

// 명령 실행
fn execute_command(command: Command, stream: &mut TcpStream) -> Result<bool> {
    match command {
        Command::Ping => {
            stream.write_all(b"PONG\n")?;
        }
        Command::Echo(msg) => {
            stream.write_all(msg.as_bytes())?;
            stream.write_all(b"\n")?;
        }
        Command::Add(x, y) => {
            let result = x + y;
            stream.write_all(format!("Result: {}\n", result).as_bytes())?;
        }
        Command::Multiply(x, y) => {
            let result = x * y;
            stream.write_all(format!("Result: {}\n", result).as_bytes())?;
        }
        Command::Quit => {
            stream.write_all(b"Goodbye!\n")?;
            return Ok(false);
        }
    }
    stream.flush()?; // 버퍼를 즉시 전송
    Ok(true)
}

// 클라이언트 처리
fn handle_client(stream: TcpStream) -> Result<()> {
    let peer_addr = stream.peer_addr()?;
    println!("[{}] Connected", peer_addr);
    
    let mut reader = BufReader::new(stream.try_clone()?);
    let mut writer = stream;
    
    // 환영 메시지
    writer.write_all(b"===== Simple Calculator Server =====\n")?;
    writer.write_all(b"Commands:\n")?;
    writer.write_all(b"  PING          - Check connection\n")?;
    writer.write_all(b"  ECHO <msg>    - Echo message\n")?;
    writer.write_all(b"  ADD <x> <y>   - Add two numbers\n")?;
    writer.write_all(b"  MUL <x> <y>   - Multiply two numbers\n")?;
    writer.write_all(b"  QUIT          - Disconnect\n")?;
    writer.write_all(b"====================================\n")?;
    writer.flush()?;
    
    loop {
        let mut line = String::new();
        let bytes_read = reader.read_line(&mut line)?;
        
        if bytes_read == 0 {
            println!("[{}] Disconnected (connection closed)", peer_addr);
            break;
        }
        
        println!("[{}] Received: {}", peer_addr, line.trim());
        
        match parse_command(&line) {
            Ok(command) => {
                match execute_command(command, &mut writer) {
                    Ok(should_continue) => {
                        if !should_continue {
                            println!("[{}] Disconnected (quit command)", peer_addr);
                            break;
                        }
                    }
                    Err(e) => {
                        eprintln!("[{}] Execution error: {}", peer_addr, e);
                        let error_msg = format!("ERROR: {}\n", e);
                        writer.write_all(error_msg.as_bytes())?;
                        writer.flush()?;
                    }
                }
            }
            Err(e) => {
                eprintln!("[{}] Parse error: {}", peer_addr, e);
                let error_msg = format!("ERROR: {}\n", e);
                writer.write_all(error_msg.as_bytes())?;
                writer.flush()?;
            }
        }
    }
    
    Ok(())
}

// 메인 함수
fn main() -> io::Result<()> {
    let listener = TcpListener::bind("127.0.0.1:8080")?;
    println!("Server listening on 127.0.0.1:8080");
    println!("Press Ctrl+C to stop");
    
    for stream in listener.incoming() {
        match stream {
            Ok(stream) => {
                // 각 클라이언트를 순차적으로 처리
                // (다음 시간에 스레드로 개선할 예정)
                if let Err(e) = handle_client(stream) {
                    eprintln!("Client handler error: {}", e);
                }
            }
            Err(e) => {
                eprintln!("Failed to accept connection: {}", e);
            }
        }
    }
    
    Ok(())
}
```

이 서버를 실행하고 `telnet localhost 8080` 또는 `nc localhost 8080`으로 연결해보자. 다양한 명령을 시도해볼 수 있다.

```
  QUIT          - Disconnect

Hello, Rust!

```

## **에러 처리의 소유권 관점**

에러 처리를 할 때 소유권과 빌림이 어떻게 작동하는지 이해하는 것이 중요하다. `Result<T, E>`는 값을 소유한다. `?` 연산자를 사용하면 에러의 소유권이 호출자에게 이동한다. 이것은 에러 객체를 여러 번 사용할 수 없다는 의미이기도 하다.

```rust
fn example() -> Result<()> {
    let result = parse_command("INVALID");
    
    // 에러를 로깅
    if let Err(e) = result {
        eprintln!("Error: {}", e);
        // e의 소유권이 여기서 소비됨
        // return Err(e); // 이것은 작동하지 않음!
    }
    
    Ok(())
}
```

에러를 여러 곳에서 사용해야 한다면 참조를 사용하거나 에러를 복제해야 한다.

```rust
fn example_fixed() -> Result<()> {
    let result = parse_command("INVALID");
    
    if let Err(ref e) = result {
        // ref로 빌림
        eprintln!("Error: {}", e);
    }
    
    result?; // 여전히 result를 사용 가능
    Ok(())
}
```

## **정리하면**

오늘 우리는 Rust의 에러 처리 시스템을 깊이 있게 살펴봤다. `?` 연산자는 에러를 간결하게 전파하게 해주고, 커스텀 에러 타입은 다양한 에러를 통합적으로 관리하게 해준다. `From` trait는 에러 변환을 자동화하고, 에러 컨텍스트는 디버깅을 쉽게 만든다. 이 모든 것이 결합되어 견고하고 유지보수하기 쉬운 서버 코드를 만들 수 있다.

다음 시간에는 열거형(enum)의 강력함을 더 깊이 탐구하고, 패턴 매칭으로 복잡한 프로토콜을 우아하게 처리하는 방법을 배운다. 오늘 만든 서버가 그 토대가 될 것이다.
