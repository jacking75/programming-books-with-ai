## 불변성의 의미와 mut 키워드

### Rust의 불변성 철학
Rust에서 변수는 기본적으로 불변(immutable)이다. 이것은 단순한 문법 규칙이 아니라 안전한 프로그램을 작성하기 위한 핵심 설계 철학이다. 불변 변수는 한번 값이 할당되면 절대 변경되지 않는다는 보장을 컴파일러가 제공한다.

간단한 예제로 시작해보자:

```rust
fn main() {
    let config_name = "database.conf";
    println!("설정 파일: {}", config_name);
    
    // 다음 줄의 주석을 제거하면 컴파일 에러가 발생한다
    // config_name = "server.conf";
}
```

위 코드를 실행하면 정상 동작한다. 하지만 주석 처리된 줄의 주석을 제거하면:

```
error[E0384]: cannot assign twice to immutable variable `config_name`
```

컴파일러가 즉시 문제를 감지한다. 이것이 불변성의 힘이다. 실수로 값을 변경하는 버그를 컴파일 시점에 방지한다.

### 가변 변수: mut 키워드
값을 변경해야 하는 경우 명시적으로 `mut` 키워드를 사용한다:

```rust
fn main() {
    let mut line_count = 0;
    println!("처리된 라인 수: {}", line_count);
    
    line_count = line_count + 1;
    println!("처리된 라인 수: {}", line_count);
    
    line_count += 1;  // 축약형
    println!("처리된 라인 수: {}", line_count);
}
```

실행 결과:

```
처리된 라인 수: 0
처리된 라인 수: 1
처리된 라인 수: 2
```

`mut`는 "이 변수는 변경될 수 있다"는 의도를 명시적으로 표현한다. 코드를 읽는 사람은 이 변수가 어디선가 값이 바뀔 수 있음을 즉시 알 수 있다.

### 불변성과 가변성의 선택 기준
불변성과 가변성을 언제 사용할지 다이어그램으로 살펴보자:

```
┌─ 변수 선언 시 결정 ──────────────────────────┐
│                                            │
│  값이 변경되지 않는가?                        │
│         │                                  │
│         ├─ YES → let name = value;         │
│         │        (불변 변수 사용)            │
│         │                                  │
│         └─ NO  → let mut name = value;     │
│                  (가변 변수 사용)            │
│                                            │
│  기본 원칙:                                 │
│  - 가능하면 불변으로 시작                     │
│  - 필요할 때만 mut 추가                      │
│  - 불변이 더 안전하고 이해하기 쉬움            │
│                                            │
└────────────────────────────────────────────┘
```

실용적인 예제를 보자:

```rust
fn main() {
    // 설정 파일 경로 - 변경 없음
    let config_path = "app.conf";
    
    // 파싱된 설정 개수 - 계속 증가
    let mut parsed_count = 0;
    
    // 최대 설정 항목 수 - 변경 없음
    let max_config_items = 100;
    
    println!("설정 파일: {}", config_path);
    println!("최대 항목: {}", max_config_items);
    
    // 설정을 파싱하는 과정 시뮬레이션
    parsed_count += 5;
    println!("파싱된 항목: {}", parsed_count);
}
```

### 섀도잉(Shadowing)
Rust는 같은 이름의 변수를 새로 선언하여 이전 값을 "가리는" 섀도잉을 허용한다:

```rust
fn main() {
    let config_value = "123";
    println!("문자열: {}", config_value);
    
    // 같은 이름으로 새 변수 선언 (타입도 바꿀 수 있다)
    let config_value: i32 = config_value.parse().expect("숫자 파싱 실패");
    println!("숫자: {}", config_value);
    
    // 또 다시 섀도잉
    let config_value = config_value * 2;
    println!("계산 결과: {}", config_value);
}
```

실행 결과:

```
문자열: 123
숫자: 123
계산 결과: 246
```

섀도잉은 `mut`와 다르다. `mut`는 같은 변수의 값을 변경하지만, 섀도잉은 새로운 변수를 만든다. 섀도잉은 타입도 바꿀 수 있다는 장점이 있다.

## 기본 타입들 (정수, 부동소수점, 문자열)

### 정수 타입

Rust는 다양한 크기의 정수 타입을 제공한다:

```
┌─ 정수 타입 ─────────────────────────────────┐
│                                              │
│  부호 있음 (signed)    부호 없음 (unsigned) │
│  ─────────────────    ──────────────────── │
│  i8   (-128 ~ 127)    u8   (0 ~ 255)       │
│  i16  (-32K ~ 32K)    u16  (0 ~ 65K)       │
│  i32  (-2B ~ 2B)      u32  (0 ~ 4B)        │
│  i64  (-9E ~ 9E)      u64  (0 ~ 18E)       │
│  i128 (매우 큰 수)    u128 (매우 큰 수)    │
│                                              │
│  isize (포인터 크기)  usize (포인터 크기)   │
│  - 32비트 시스템: 32비트                    │
│  - 64비트 시스템: 64비트                    │
│                                              │
└──────────────────────────────────────────────┘
```

실제 코드로 살펴보자:

```rust
fn main() {
    // 기본 정수 타입은 i32
    let port = 8080;
    println!("포트: {}", port);
    
    // 명시적 타입 지정
    let max_connections: u32 = 1000;
    println!("최대 연결: {}", max_connections);
    
    // 타입 접미사 사용
    let timeout = 30u64;
    println!("타임아웃: {}초", timeout);
    
    // 언더스코어로 가독성 향상
    let max_buffer_size = 1_000_000;
    println!("버퍼 크기: {} 바이트", max_buffer_size);
    
    // 다양한 진법 표현
    let hex_value = 0xFF;        // 16진수
    let octal_value = 0o77;      // 8진수
    let binary_value = 0b1111;   // 2진수
    println!("16진수 0xFF = {}", hex_value);
    println!("8진수 0o77 = {}", octal_value);
    println!("2진수 0b1111 = {}", binary_value);
}
```

실행 결과:

```
포트: 8080
최대 연결: 1000
타임아웃: 30초
버퍼 크기: 1000000 바이트
16진수 0xFF = 255
8진수 0o77 = 63
2진수 0b1111 = 15
```

### 부동소수점 타입

부동소수점 타입은 두 가지다:

```rust
fn main() {
    // f64가 기본 타입 (더 정밀함)
    let cpu_usage = 23.5;
    println!("CPU 사용률: {}%", cpu_usage);
    
    // f32 명시
    let memory_usage: f32 = 67.8;
    println!("메모리 사용률: {}%", memory_usage);
    
    // 산술 연산
    let total = cpu_usage + memory_usage as f64;
    println!("총합: {}", total);
    
    // 과학적 표기법
    let large_number = 1.5e6;  // 1,500,000
    let small_number = 3.2e-3;  // 0.0032
    println!("큰 수: {}", large_number);
    println!("작은 수: {}", small_number);
}
```

### 불리언 타입

불리언은 `true`와 `false` 두 값만 가진다:

```rust
fn main() {
    let is_enabled = true;

    println!("활성화됨: {}", is_enabled);
```

### 문자와 문자열

Rust에서 문자열은 다소 복잡하다. 두 가지 주요 타입이 있다:

**char 타입**: 유니코드 문자 하나를 표현한다.

```rust
fn main() {
    let letter = 'A';
    let emoji = '😊';
    let korean = '한';
    
    println!("문자: {}, {}, {}", letter, emoji, korean);
}
```

**문자열 타입**: `String`과 `&str`이 있다. 

```rust
fn main() {
    // &str: 문자열 리터럴 (불변, 읽기 전용)
    let config_key = "database.host";
    
    // String: 소유된 문자열 (가변 가능, 힙에 저장)
    let mut config_value = String::from("localhost");
    
    println!("키: {}", config_key);
    println!("값: {}", config_value);
    
    // String은 수정 가능
    config_value.push_str(":5432");
    println!("수정된 값: {}", config_value);
    
    // String을 &str로 변환 (자동)
    let value_ref: &str = &config_value;
    println!("참조: {}", value_ref);
}
```

문자열 타입의 관계를 다이어그램으로 표현하면:

```
┌─ 문자열 타입의 세계 ────────────────────────┐
│                                              │
│  "hello"                                     │
│  문자열 리터럴                               │
│  타입: &str                                  │
│  위치: 프로그램 바이너리 (읽기 전용)        │
│      │                                       │
│      │ String::from()                        │
│      ▼                                       │
│  String::from("hello")                       │
│  소유된 문자열                               │
│  타입: String                                │
│  위치: 힙 메모리 (수정 가능)                │
│      │                                       │
│      │ &를 이용한 빌림                       │
│      ▼                                       │
│  &my_string                                  │
│  문자열 슬라이스                             │
│  타입: &str                                  │
│  위치: String을 가리키는 참조                │
│                                              │
└──────────────────────────────────────────────┘
```

지금은 이 차이를 완전히 이해하지 못해도 된다. 앞으로 계속 사용하면서 자연스럽게 익숙해질 것이다.

## 간단한 키-값 설정 파서 구현

이제 배운 내용을 활용하여 실용적인 설정 파서를 만들어보자. 다음과 같은 형식의 설정 파일을 파싱한다:

```
database.host=localhost
database.port=5432
database.name=myapp
server.port=8080
server.timeout=30
debug.enabled=true
```

### 기본 파서 구조

먼저 간단한 버전부터 시작한다. `src/main.rs`를 다음과 같이 작성한다:

```rust
fn main() {
    println!("=== 설정 파일 파서 ===\n");
    
    // 설정 파일 내용을 문자열로 시뮬레이션
    let config_content = "\
database.host=localhost
database.port=5432
server.port=8080
debug.enabled=true
";
    
    println!("원본 설정:\n{}", config_content);
    println!("─────────────────────────────");
    
    // 한 줄씩 처리
    let mut line_number = 0;
    
    for line in config_content.lines() {
        line_number += 1;
        
        // 빈 줄 건너뛰기
        if line.trim().is_empty() {
            continue;
        }
        
        println!("{}번째 줄: {}", line_number, line);
    }
}
```

실행 결과:

```
=== 설정 파일 파서 ===

원본 설정:
database.host=localhost
database.port=5432
server.port=8080
debug.enabled=true

─────────────────────────────
1번째 줄: database.host=localhost
2번째 줄: database.port=5432
3번째 줄: server.port=8080
4번째 줄: debug.enabled=true
```

코드를 살펴보자:

**백슬래시 이스케이프**: 문자열 앞의 `\`는 여러 줄 문자열을 만들 때 줄바꿈을 포함하지 않도록 한다.

**lines() 메서드**: 문자열을 줄 단위로 나누어 반복한다.

**trim() 메서드**: 앞뒤 공백을 제거한다.

**is_empty() 메서드**: 빈 문자열인지 확인한다.

### 키-값 분리하기

이제 각 줄을 키와 값으로 분리해보자:

```rust
fn main() {
    println!("=== 설정 파일 파서 ===\n");
    
    let config_content = "\
database.host=localhost
database.port=5432
server.port=8080
debug.enabled=true
max.connections=100
";
    
    let mut parsed_count = 0;
    
    for line in config_content.lines() {
        // 빈 줄 건너뛰기
        let line = line.trim();
        if line.is_empty() {
            continue;
        }
        
        // '=' 기준으로 분리
        if let Some(equal_pos) = line.find('=') {
            let key = &line[..equal_pos];
            let value = &line[equal_pos + 1..];
            
            println!("키: {:20} 값: {}", key, value);
            parsed_count += 1;
        } else {
            println!("경고: 잘못된 형식 - {}", line);
        }
    }
    
    println!("\n총 {}개의 설정을 파싱했습니다.", parsed_count);
}
```

실행 결과:

```
=== 설정 파일 파서 ===

키: database.host        값: localhost
키: database.port        값: 5432
키: server.port          값: 8080
키: debug.enabled        값: true
키: max.connections      값: 100

총 5개의 설정을 파싱했습니다.
```

새로운 개념을 살펴보자:

**if let 구문**: `find('=')`는 `Option<usize>`를 반환한다. 값이 있으면(Some) 처리하고, 없으면(None) 건너뛴다.

**문자열 슬라이싱**: `&line[..equal_pos]`는 문자열의 일부를 참조한다. `..`는 범위를 나타낸다.

```
line = "database.host=localhost"
       ├────────────┼──────────┤
       0          13 14       22

&line[..13]    → "database.host"
&line[14..]    → "localhost"
```

### 타입별 값 파싱

설정 값을 적절한 타입으로 변환해보자:

```rust
fn main() {
    println!("=== 타입별 설정 파서 ===\n");
    
    let config_content = "\
database.host=localhost
database.port=5432
server.port=8080
debug.enabled=true
max.connections=100
timeout.seconds=30.5
";
    
    for line in config_content.lines() {
        let line = line.trim();
        if line.is_empty() {
            continue;
        }
        
        if let Some(equal_pos) = line.find('=') {
            let key = &line[..equal_pos];
            let value = &line[equal_pos + 1..];
            
            // 키 이름에 따라 적절한 타입으로 파싱
            parse_config_value(key, value);
        }
    }
}

fn parse_config_value(key: &str, value: &str) {
    // 키가 "port"나 "connections"를 포함하면 정수로 파싱
    if key.contains("port") || key.contains("connections") {
        match value.parse::<u32>() {
            Ok(num) => println!("[정수] {}: {}", key, num),
            Err(_) => println!("[에러] {}: '{}' - 정수 파싱 실패", key, value),
        }
    }
    // 키가 "timeout"이나 "seconds"를 포함하면 부동소수점으로 파싱
    else if key.contains("timeout") || key.contains("seconds") {
        match value.parse::<f64>() {
            Ok(num) => println!("[실수] {}: {}", key, num),
            Err(_) => println!("[에러] {}: '{}' - 실수 파싱 실패", key, value),
        }
    }
    // 키가 "enabled"를 포함하면 불리언으로 파싱
    else if key.contains("enabled") {
        match value.parse::<bool>() {
            Ok(b) => println!("[불리언] {}: {}", key, b),
            Err(_) => println!("[에러] {}: '{}' - 불리언 파싱 실패", key, value),
        }
    }
    // 그 외는 문자열로 처리
    else {
        println!("[문자열] {}: {}", key, value);
    }
}
```

실행 결과:

```
=== 타입별 설정 파서 ===

[문자열] database.host: localhost
[정수] database.port: 5432
[정수] server.port: 8080
[불리언] debug.enabled: true
[정수] max.connections: 100
[실수] timeout.seconds: 30.5
```

**match 표현식**: `Result` 타입을 처리하는 강력한 방법이다. `Ok`는 성공, `Err`는 실패를 나타낸다.

**parse::<Type>()**: 터보피쉬(turbofish) 문법으로 파싱할 타입을 명시한다.

### 주석과 공백 처리 개선

실용적인 파서는 주석도 처리해야 한다:

```rust
fn main() {
    println!("=== 고급 설정 파서 ===\n");
    
    let config_content = "\
# 데이터베이스 설정
database.host=localhost
database.port=5432  # 기본 PostgreSQL 포트

# 서버 설정
server.port=8080
server.timeout=30

# 디버그 옵션
debug.enabled=true
";
    
    let mut parsed_count = 0;
    let mut comment_count = 0;
    
    for (line_num, line) in config_content.lines().enumerate() {
        let line = line.trim();
        
        // 빈 줄
        if line.is_empty() {
            continue;
        }
        
        // 주석 줄 (# 으로 시작)
        if line.starts_with('#') {
            comment_count += 1;
            println!("{}번 줄: 주석 - {}", line_num + 1, line);
            continue;
        }
        
        // 라인 끝 주석 제거
        let line = if let Some(comment_pos) = line.find('#') {
            &line[..comment_pos].trim()
        } else {
            line
        };
        
        // 키-값 파싱
        if let Some(equal_pos) = line.find('=') {
            let key = line[..equal_pos].trim();
            let value = line[equal_pos + 1..].trim();
            
            println!("{}번 줄: {} = {}", line_num + 1, key, value);
            parsed_count += 1;
        }
    }
    
    println!("\n통계:");
    println!("- 파싱된 설정: {}개", parsed_count);
    println!("- 주석 줄: {}개", comment_count);
}
```

실행 결과:

```

1번 줄: 주석 - # 데이터베이스 설정

3번 줄: database.port = 5432
5번 줄: 주석 - # 서버 설정
6번 줄: server.port = 8080
7번 줄: server.timeout = 30
9번 줄: 주석 - # 디버그 옵션
10번 줄: debug.enabled = true

통계:
- 파싱된 설정: 5개
- 주석 줄: 3개
```

**enumerate()**: 반복하면서 인덱스도 함께 제공한다.

**starts_with()**: 문자열이 특정 패턴으로 시작하는지 확인한다.

**중첩된 if let**: 여러 단계의 옵셔널 처리를 연쇄적으로 수행한다.

## println! 매크로로 디버깅하기

`println!`은 단순한 출력 도구가 아니라 강력한 디버깅 도구다. 다양한 포맷팅 기능을 활용하면 개발 과정이 훨씬 수월해진다.

### 기본 포맷팅

```rust
fn main() {
    let key = "database.port";
    let value = 5432;
    let is_valid = true;
    
    // 기본 출력
    println!("키: {}", key);
    
    // 여러 값 출력
    println!("키: {}, 값: {}, 유효: {}", key, value, is_valid);
    
    // 위치 지정자 사용
    println!("{0}의 값은 {1}이며, {1}은 정수입니다", key, value);
    
    // 이름 지정자 사용
    println!("{name}={val}", name=key, val=value);
}
```


### 실전 디버깅 예제

설정 파서에 디버깅 출력을 추가해보자:

```rust
fn main() {
    println!("=== 디버깅이 추가된 설정 파서 ===\n");
    
    let config_content = "\
database.host=localhost
database.port=5432
invalid_line_without_equals
server.port=8080
";
    
    let mut stats = ParserStats {
        total_lines: 0,
        parsed_lines: 0,
        error_lines: 0,
    };
    
    for (line_num, line) in config_content.lines().enumerate() {
        stats.total_lines += 1;
        let line = line.trim();
        
        println!("[DEBUG] {}번 줄 처리 중: {:?}", line_num + 1, line);
        
        if line.is_empty() {
            println!("  → 빈 줄 건너뜀\n");
            continue;
        }
        
        if let Some(equal_pos) = line.find('=') {
            let key = line[..equal_pos].trim();
            let value = line[equal_pos + 1..].trim();
            
            println!("  → 파싱 성공!");
            println!("     키:   {:?}", key);
            println!("     값:   {:?}", value);
            println!("     타입: {}\n", get_value_type(value));
            
            stats.parsed_lines += 1;
        } else {
            println!("  → ❌ 에러: '=' 기호를 찾을 수 없음\n");
            stats.error_lines += 1;
        }
    }
    
    println!("─────────────────────────────");
    println!("파싱 통계:");
    println!("{:#?}", stats);
}

struct ParserStats {
    total_lines: u32,
    parsed_lines: u32,
    error_lines: u32,
}

fn get_value_type(value: &str) -> &str {
    if value.parse::<i32>().is_ok() {
        "정수"
    } else if value.parse::<f64>().is_ok() {
        "실수"
    } else if value.parse::<bool>().is_ok() {
        "불리언"
    } else {
        "문자열"
    }
}
```

## 정리

오늘 1시간 동안 다음을 학습했다:

**불변성과 가변성**: Rust의 기본 철학인 불변성을 이해하고, `mut` 키워드로 필요한 경우에만 가변성을 부여하는 법을 배웠다. 섀도잉으로 같은 이름의 새 변수를 만드는 방법도 익혔다.

**기본 데이터 타입**:
- 정수 타입 (i8, i16, i32, i64, u8, u16, u32, u64)
- 부동소수점 타입 (f32, f64)
- 불리언 타입 (bool)
- 문자 타입 (char)
- 문자열 타입 (String, &str)

**실전 프로젝트**: 키-값 형식의 설정 파일을 파싱하는 프로그램을 처음부터 만들었다. 한 줄씩 읽고, '=' 기호로 분리하고, 주석을 처리하고, 타입별로 값을 변환하는 전체 과정을 구현했다.

**디버깅 기술**: `println!` 매크로의 다양한 포맷팅 옵션을 배워 효과적으로 디버깅하는 방법을 익혔다.

### 핵심 개념 정리

