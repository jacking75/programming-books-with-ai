# Day 5: 컬렉션으로 데이터 관리하기
오늘은 Rust의 핵심 컬렉션 타입인 `Vec<T>`, `String`, `HashMap<K, V>`를 배운다. 이들은 동적으로 크기가 변하는 데이터를 다루는 필수 도구다. Day 4까지 만든 설정 파서를 `HashMap` 기반으로 재구성하면서 실용적인 사용법을 익힌다.

## Vec<T>: 동적 배열
`Vec<T>`는 같은 타입의 값을 순서대로 저장하는 동적 배열이다. 배열과 달리 크기가 컴파일 타임에 정해지지 않고, 런타임에 자유롭게 늘리거나 줄일 수 있다.

```rust
fn main() {
    // 빈 벡터 생성
    let mut numbers: Vec<i32> = Vec::new();
    
    // 값 추가
    numbers.push(10);
    numbers.push(20);
    numbers.push(30);
    
    println!("벡터: {:?}", numbers);
    println!("길이: {}", numbers.len());
    
    // vec! 매크로로 초기값과 함께 생성
    let mut fruits = vec!["사과", "바나나", "오렌지"];
    println!("과일: {:?}", fruits);
    
    // 인덱스로 접근
    let first = fruits[0];
    println!("첫 번째 과일: {}", first);
    
    // get 메서드는 Option을 반환한다 (안전하다)
    match fruits.get(1) {
        Some(fruit) => println!("두 번째 과일: {}", fruit),
        None => println!("인덱스를 벗어났다"),
    }
    
    // 마지막 요소 제거
    if let Some(last) = fruits.pop() {
        println!("제거된 과일: {}", last);
    }
    
    println!("남은 과일: {:?}", fruits);
}
```

벡터는 힙에 데이터를 저장한다. 벡터가 스코프를 벗어나면 벡터와 그 안의 모든 요소가 자동으로 정리된다. 인덱스 연산자 `[]`는 범위를 벗어나면 패닉을 발생시키지만, `get` 메서드는 `Option`을 반환해서 안전하게 처리할 수 있다.

```
    Vec<i32> 메모리 구조
    
    스택                    힙
    ┌──────────┐          ┌─────┐
    │ ptr      │─────────>│ 10  │
    │ len: 3   │          │ 20  │
    │ cap: 4   │          │ 30  │
    └──────────┘          │ ... │
                          └─────┘
    
    ptr: 데이터 위치
    len: 현재 요소 개수
    cap: 할당된 용량
```

벡터는 필요에 따라 자동으로 용량을 늘린다. 하지만 미리 용량을 예약하면 재할당을 줄여서 성능을 개선할 수 있다.

```rust
fn main() {
    // 용량 10으로 벡터 생성
    let mut numbers = Vec::with_capacity(10);
    println!("길이: {}, 용량: {}", numbers.len(), numbers.capacity());
    
    for i in 0..10 {
        numbers.push(i);
    }
    
    println!("길이: {}, 용량: {}", numbers.len(), numbers.capacity());
    
    // 11번째 요소를 추가하면 용량이 늘어난다
    numbers.push(10);
    println!("길이: {}, 용량: {}", numbers.len(), numbers.capacity());
}
```

벡터를 순회하는 방법은 여러 가지가 있다.

```rust
fn main() {
    let numbers = vec![1, 2, 3, 4, 5];
    
    // 불변 참조로 순회
    for num in &numbers {
        println!("{}", num);
    }
    
    // 가변 참조로 순회하면서 수정
    let mut numbers = vec![1, 2, 3, 4, 5];
    for num in &mut numbers {
        *num *= 2;  // 역참조로 값을 수정한다
    }
    println!("두 배: {:?}", numbers);
    
    // 소유권을 가져와서 순회 (이후 numbers를 사용할 수 없다)
    for num in numbers {
        println!("{}", num);
    }
    // println!("{:?}", numbers);  // 컴파일 에러! 소유권이 이동했다
}
```

벡터에 다른 타입을 함께 저장하려면 열거형을 사용한다.

```rust
#[derive(Debug)]
enum ConfigValue {
    Text(String),
    Number(i32),
    Boolean(bool),
}

fn main() {
    let settings = vec![
        ConfigValue::Text(String::from("localhost")),
        ConfigValue::Number(8080),
        ConfigValue::Boolean(true),
    ];
    
    for setting in &settings {
        match setting {
            ConfigValue::Text(s) => println!("문자열: {}", s),
            ConfigValue::Number(n) => println!("숫자: {}", n),
            ConfigValue::Boolean(b) => println!("불린: {}", b),
        }
    }
}
```
  

## String과 &str의 차이
Rust에서 문자열은 복잡하다고 느낄 수 있지만, 이유를 이해하면 명확해진다. `String`과 `&str`은 가장 혼란스러운 부분 중 하나다.

**String:** 소유된 문자열이다. 힙에 할당되고 크기가 가변적이며, 수정할 수 있다. `Vec<u8>`과 비슷하지만 항상 유효한 UTF-8을 보장한다.

**&str:** 문자열 슬라이스다. 어딘가에 있는 문자열 데이터를 참조한다. 크기가 고정되어 있고 수정할 수 없다. 문자열 리터럴의 타입도 `&str`이다.

```rust
fn main() {
    // 문자열 리터럴은 &str 타입이다
    let greeting: &str = "안녕하세요";
    
    // String::from이나 to_string()으로 String 생성
    let mut owned = String::from("Hello");
    let owned2 = "Hello".to_string();
    
    // String은 수정할 수 있다
    owned.push_str(", world!");
    println!("{}", owned);
    
    // String을 &str로 변환 (자동으로 일어난다)
    print_str(&owned);
    
    // &str을 String으로 변환
    let s: String = greeting.to_string();
    println!("{}", s);
}

fn print_str(s: &str) {
    println!("문자열: {}", s);
}
```

일반적으로 함수 매개변수로는 `&str`을 받는 것이 좋다. `String`과 `&str` 모두 전달할 수 있고, 소유권을 가져가지 않기 때문이다.

```rust
fn process_string(s: &str) {
    println!("처리: {}", s);
}

fn main() {
    let owned = String::from("owned");
    let borrowed = "borrowed";
    
    process_string(&owned);     // String을 &str로 전달
    process_string(borrowed);   // &str을 그대로 전달
    
    // 둘 다 여전히 사용할 수 있다
    println!("{}, {}", owned, borrowed);
}
```

`String`은 여러 유용한 메서드를 제공한다.

```rust
fn main() {
    let mut s = String::from("hello");
    
    // 문자열 추가
    s.push(' ');
    s.push_str("world");
    println!("{}", s);  // "hello world"
    
    // + 연산자로 연결 (왼쪽 String의 소유권을 가져간다)
    let s1 = String::from("Hello, ");
    let s2 = String::from("world!");
    let s3 = s1 + &s2;  // s1은 더 이상 사용할 수 없다
    println!("{}", s3);
    
    // format! 매크로는 소유권을 가져가지 않는다
    let s1 = String::from("Hello");
    let s2 = String::from("world");
    let s3 = format!("{}, {}!", s1, s2);
    println!("{}", s3);
    // s1, s2 모두 여전히 사용할 수 있다
    
    // 문자열 슬라이싱
    let s = String::from("안녕하세요");
    let hello = &s[0..9];  // UTF-8 바이트 단위로 슬라이싱 (한글 3글자는 9바이트)
    println!("{}", hello);
    
    // 문자열 검색
    if s.contains("안녕") {
        println!("인사말을 포함한다");
    }
    
    // 문자열 분리
    let words: Vec<&str> = s.split('하').collect();
    println!("{:?}", words);
}
```

```
    String vs &str
    
    String (소유)               &str (참조)
    ┌──────────┐               ┌──────────┐
    │ ptr      │──┐            │ ptr      │──┐
    │ len      │  │            │ len      │  │
    │ capacity │  │            └──────────┘  │
    └──────────┘  │                          │
                  v                          v
              힙 메모리                   어딘가의 메모리
              ┌─────────┐                ┌─────────┐
              │ H e l l │                │ W o r l │
              │ o       │                │ d       │
              └─────────┘                └─────────┘
              
    수정 가능                    읽기 전용
    크기 가변                    크기 고정
```
  

## HashMap으로 키-값 저장소 만들기
`HashMap<K, V>`는 키와 값의 쌍을 저장하는 컬렉션이다. 키를 통해 값을 빠르게 찾을 수 있다. 설정 데이터를 저장하기에 완벽하다.

```rust
use std::collections::HashMap;

fn main() {
    // 빈 HashMap 생성
    let mut scores = HashMap::new();
    
    // 키-값 쌍 삽입
    scores.insert(String::from("Blue"), 10);
    scores.insert(String::from("Yellow"), 50);
    
    println!("{:?}", scores);
    
    // 값 가져오기 (Option을 반환한다)
    let team_name = String::from("Blue");
    match scores.get(&team_name) {
        Some(score) => println!("{} 팀 점수: {}", team_name, score),
        None => println!("팀을 찾을 수 없다"),
    }
    
    // 키가 없을 때만 삽입
    scores.entry(String::from("Blue")).or_insert(25);
    scores.entry(String::from("Red")).or_insert(25);
    println!("{:?}", scores);
    
    // 모든 키-값 순회
    for (key, value) in &scores {
        println!("{}: {}", key, value);
    }
}
```

`HashMap`은 `Vec`나 `String`과 달리 표준 라이브러리의 프렐루드에 포함되지 않아서 `use` 선언이 필요하다. `insert` 메서드는 키가 이미 존재하면 기존 값을 덮어쓴다.

`entry` API는 키가 없을 때만 값을 삽입하는 편리한 방법을 제공한다. 이는 불필요한 삽입을 방지하고 효율적이다.

```rust
use std::collections::HashMap;

fn main() {
    let text = "hello world wonderful world";
    let mut word_count = HashMap::new();
    
    for word in text.split_whitespace() {
        // entry는 키에 대한 Entry를 반환한다
        // or_insert는 키가 없으면 값을 삽입하고, 값에 대한 가변 참조를 반환한다
        let count = word_count.entry(word).or_insert(0);
        *count += 1;  // 역참조로 값을 증가시킨다
    }
    
    println!("{:?}", word_count);
}
```

`HashMap`의 키는 `Eq`와 `Hash` trait을 구현해야 한다. 대부분의 기본 타입은 이미 구현되어 있다. 커스텀 타입을 키로 사용하려면 이 trait들을 구현해야 한다.

```
    HashMap 내부 구조 (단순화)
    
    ┌─────────────────────────┐
    │  Hash Table (버킷 배열)  │
    ├─────────────────────────┤
    │ 0: [("Blue", 10)]       │
    │ 1: []                   │
    │ 2: [("Yellow", 50)]     │
    │ 3: [("Red", 25)]        │
    │ ...                     │
    └─────────────────────────┘
    
    키를 해싱 ───> 버킷 인덱스
    충돌 시 같은 버킷에 여러 쌍 저장
```
  

## 설정 파서를 HashMap 기반으로 개선
이제 Day 2-4에서 만든 설정 파서를 `HashMap`을 사용해서 더 유연하게 만든다. 어떤 키든 동적으로 저장하고 조회할 수 있다.

```rust
use std::collections::HashMap;

#[derive(Debug)]
enum ConfigError {
    ParseError(String),
    MissingKey(String),
    InvalidValue { key: String, reason: String },
}

#[derive(Debug)]
struct Config {
    data: HashMap<String, String>,
}

impl Config {
    fn new() -> Self {
        Config {
            data: HashMap::new(),
        }
    }
    
    fn from_text(text: &str) -> Result<Self, ConfigError> {
        let mut config = Config::new();
        
        for (line_num, line) in text.lines().enumerate() {
            let line = line.trim();
            
            // 빈 줄이나 주석 건너뛰기
            if line.is_empty() || line.starts_with('#') {
                continue;
            }
            
            // 키=값 파싱
            let parts: Vec<&str> = line.split('=').collect();
            if parts.len() != 2 {
                return Err(ConfigError::ParseError(
                    format!("{}번째 줄: '{}' (형식: key=value)", line_num + 1, line)
                ));
            }
            
            let key = parts[0].trim();
            let value = parts[1].trim();
            
            if key.is_empty() {
                return Err(ConfigError::ParseError(
                    format!("{}번째 줄: 키가 비어있다", line_num + 1)
                ));
            }
            
            config.data.insert(key.to_string(), value.to_string());
        }
        
        Ok(config)
    }
    
    // 문자열 값 가져오기
    fn get(&self, key: &str) -> Option<&String> {
        self.data.get(key)
    }
    
    // 문자열 값 가져오기 (없으면 에러)
    fn get_required(&self, key: &str) -> Result<&String, ConfigError> {
        self.data.get(key)
            .ok_or_else(|| ConfigError::MissingKey(key.to_string()))
    }
    
    // 정수로 파싱해서 가져오기
    fn get_int(&self, key: &str) -> Result<i32, ConfigError> {
        let value = self.get_required(key)?;
        value.parse::<i32>()
            .map_err(|_| ConfigError::InvalidValue {
                key: key.to_string(),
                reason: format!("'{}' 는 유효한 정수가 아니다", value),
            })
    }
    
    // 정수로 파싱 (없으면 기본값)
    fn get_int_or(&self, key: &str, default: i32) -> i32 {
        self.get_int(key).unwrap_or(default)
    }
    
    // 불린으로 파싱해서 가져오기
    fn get_bool(&self, key: &str) -> Result<bool, ConfigError> {
        let value = self.get_required(key)?;
        match value.to_lowercase().as_str() {
            "true" | "yes" | "1" => Ok(true),
            "false" | "no" | "0" => Ok(false),
            _ => Err(ConfigError::InvalidValue {
                key: key.to_string(),
                reason: format!("'{}' 는 유효한 불린 값이 아니다", value),
            }),
        }
    }
    
    // 불린으로 파싱 (없으면 기본값)
    fn get_bool_or(&self, key: &str, default: bool) -> bool {
        self.get_bool(key).unwrap_or(default)
    }
    
    // 값 설정
    fn set(&mut self, key: String, value: String) {
        self.data.insert(key, value);
    }
    
    // 모든 키 가져오기
    fn keys(&self) -> Vec<&String> {
        self.data.keys().collect()
    }
    
    // 키가 존재하는지 확인
    fn contains_key(&self, key: &str) -> bool {
        self.data.contains_key(key)
    }
    
    // 특정 접두사를 가진 설정들 가져오기
    fn get_section(&self, prefix: &str) -> HashMap<String, String> {
        self.data
            .iter()
            .filter(|(key, _)| key.starts_with(prefix))
            .map(|(key, value)| {
                // 접두사 제거
                let new_key = key.strip_prefix(prefix)
                    .unwrap_or(key)
                    .trim_start_matches('.')
                    .to_string();
                (new_key, value.clone())
            })
            .collect()
    }
}

fn main() {
    let config_text = r#"
# 서버 설정
server.host=0.0.0.0
server.port=8080
server.max_connections=100
server.enable_logging=true

# 데이터베이스 설정
db.host=localhost
db.port=5432
db.name=myapp
db.pool_size=20

# 기타
debug_mode=false
"#;
    
    match Config::from_text(config_text) {
        Ok(config) => {
            println!("설정 로드 성공!\n");
            
            // 개별 값 접근
            if let Some(host) = config.get("server.host") {
                println!("서버 호스트: {}", host);
            }
            
            // 필수 값 가져오기
            match config.get_required("server.port") {
                Ok(port) => println!("서버 포트: {}", port),
                Err(e) => println!("에러: {:?}", e),
            }
            
            // 타입 변환과 함께 가져오기
            let port = config.get_int_or("server.port", 3000);
            println!("포트 (정수): {}", port);
            
            let logging = config.get_bool_or("server.enable_logging", false);
            println!("로깅 활성화: {}", logging);
            
            // 섹션별로 가져오기
            println!("\n서버 설정:");
            let server_config = config.get_section("server");
            for (key, value) in &server_config {
                println!("  {}: {}", key, value);
            }
            
            println!("\n데이터베이스 설정:");
            let db_config = config.get_section("db");
            for (key, value) in &db_config {
                println!("  {}: {}", key, value);
            }
            
            // 모든 키 출력
            println!("\n모든 설정 키:");
            for key in config.keys() {
                println!("  - {}", key);
            }
        }
        Err(e) => {
            println!("설정 로드 실패: {:?}", e);
        }
    }
}
```

이 구현은 실제 프로덕션에서 사용할 수 있는 수준이다. `HashMap`을 사용해서 동적으로 어떤 키든 저장하고, 타입 변환 메서드를 제공해서 사용하기 편하며, 섹션별로 설정을 그룹화해서 관리할 수 있다.

```
    Config 구조
    
    Config
    └── data: HashMap<String, String>
        ├── "server.host" → "0.0.0.0"
        ├── "server.port" → "8080"
        ├── "db.host" → "localhost"
        └── ...
        
    get_section("server")
    └── 필터링
        ├── "host" → "0.0.0.0"
        └── "port" → "8080"
```
  

## 컬렉션과 소유권
컬렉션을 다룰 때 소유권 규칙은 특히 중요하다. 값을 컬렉션에 넣으면 소유권이 이동한다.

```rust
fn main() {
    let mut v = Vec::new();
    let s = String::from("hello");
    
    v.push(s);
    // println!("{}", s);  // 컴파일 에러! s의 소유권이 벡터로 이동했다
    
    // 대신 참조를 저장하거나
    let s1 = String::from("hello");
    let v: Vec<&String> = vec![&s1];
    println!("{}", s1);  // 가능하다
    
    // clone으로 복사본을 만들 수 있다
    let s2 = String::from("hello");
    let mut v2 = Vec::new();
    v2.push(s2.clone());
    println!("{}", s2);  // 가능하다 (원본이 남아있다)
}
```

## 실전 예제: 로그 분석기
배운 컬렉션을 모두 활용해서 로그 파일을 분석하는 프로그램을 만들어본다.

```rust
use std::collections::HashMap;

#[derive(Debug, Clone)]
enum LogLevel {
    Debug,
    Info,
    Warning,
    Error,
}

impl LogLevel {
    fn from_str(s: &str) -> Option<Self> {
        match s.to_uppercase().as_str() {
            "DEBUG" => Some(LogLevel::Debug),
            "INFO" => Some(LogLevel::Info),
            "WARNING" | "WARN" => Some(LogLevel::Warning),
            "ERROR" => Some(LogLevel::Error),
            _ => None,
        }
    }
}

#[derive(Debug)]
struct LogEntry {
    timestamp: String,
    level: LogLevel,
    message: String,
}

impl LogEntry {
    fn parse(line: &str) -> Option<Self> {
        let line = line.trim();
        if !line.starts_with('[') {
            return None;
        }
        
        let close_bracket = line.find(']')?;
        let timestamp = line[1..close_bracket].to_string();
        
        let rest = line[close_bracket + 1..].trim();
        let parts: Vec<&str> = rest.splitn(2, ':').collect();
        if parts.len() != 2 {
            return None;
        }
        
        let level = LogLevel::from_str(parts[0].trim())?;
        let message = parts[1].trim().to_string();
        
        Some(LogEntry {
            timestamp,
            level,
            message,
        })
    }
}

struct LogAnalyzer {
    entries: Vec<LogEntry>,
}

impl LogAnalyzer {
    fn new() -> Self {
        LogAnalyzer {
            entries: Vec::new(),
        }
    }
    
    fn add_entry(&mut self, entry: LogEntry) {
        self.entries.push(entry);
    }
    
    fn parse_logs(&mut self, text: &str) {
        for line in text.lines() {
            if let Some(entry) = LogEntry::parse(line) {
                self.add_entry(entry);
            }
        }
    }
    
    fn count_by_level(&self) -> HashMap<String, usize> {
        let mut counts = HashMap::new();
        
        for entry in &self.entries {
            let level_name = format!("{:?}", entry.level);
            *counts.entry(level_name).or_insert(0) += 1;
        }
        
        counts
    }
    
    fn find_errors(&self) -> Vec<&LogEntry> {
        self.entries
            .iter()
            .filter(|entry| matches!(entry.level, LogLevel::Error))
            .collect()
    }
    
    fn search_keyword(&self, keyword: &str) -> Vec<&LogEntry> {
        self.entries
            .iter()
            .filter(|entry| entry.message.contains(keyword))
            .collect()
    }
    
    fn get_summary(&self) -> String {
        let total = self.entries.len();
        let counts = self.count_by_level();
        
        format!(
            "총 로그: {}\n  Debug: {}\n  Info: {}\n  Warning: {}\n  Error: {}",
            total,
            counts.get("Debug").unwrap_or(&0),
            counts.get("Info").unwrap_or(&0),
            counts.get("Warning").unwrap_or(&0),
            counts.get("Error").unwrap_or(&0)
        )
    }
}

fn main() {
    let logs = r#"
[2024-01-01 10:00:00] INFO: 서버 시작
[2024-01-01 10:00:01] DEBUG: 설정 로드 중
[2024-01-01 10:00:02] INFO: 설정 로드 완료
[2024-01-01 10:00:05] WARNING: 연결 시도 실패, 재시도 중
[2024-01-01 10:00:10] ERROR: 데이터베이스 연결 실패
[2024-01-01 10:00:15] INFO: 재연결 성공
[2024-01-01 10:00:20] DEBUG: 요청 처리 중
[2024-01-01 10:00:25] ERROR: 메모리 부족
[2024-01-01 10:00:30] WARNING: 높은 부하 감지
"#;
    
    let mut analyzer = LogAnalyzer::new();
    analyzer.parse_logs(logs);
    
    println!("=== 로그 분석 결과 ===\n");
    println!("{}\n", analyzer.get_summary());
    
    println!("=== 에러 로그 ===");
    let errors = analyzer.find_errors();
    for error in errors {
        println!("[{}] {}", error.timestamp, error.message);
    }
    
    println!("\n=== '연결' 키워드 검색 ===");
    let connection_logs = analyzer.search_keyword("연결");
    for log in connection_logs {
        println!("[{}] {:?}: {}", log.timestamp, log.level, log.message);
    }
}
```

이 예제는 `Vec`로 로그 엔트리를 저장하고, `HashMap`으로 레벨별 개수를 집계하며, 이터레이터와 필터링으로 검색 기능을 구현한다. 실제로 사용할 수 있는 수준의 로그 분석 도구의 기초다.
  

## 정리하며
오늘은 Rust의 핵심 컬렉션 타입을 배웠다. 동적 배열 `Vec`, 소유된 문자열 `String`, 키-값 저장소 `HashMap`은 거의 모든 Rust 프로그램에서 사용된다.

핵심 포인트를 정리하면 다음과 같다.

`Vec<T>`는 같은 타입의 값을 순서대로 저장하는 동적 배열이다. `push`, `pop`, `get` 등의 메서드로 데이터를 관리한다. 인덱스 접근보다 `get` 메서드가 안전하다.

`String`은 소유된 UTF-8 문자열이고 `&str`은 문자열 슬라이스다. 함수 매개변수로는 `&str`을 받는 것이 유연하다. `format!` 매크로는 소유권을 가져가지 않고 새 문자열을 만든다.

`HashMap<K, V>`는 키로 값을 빠르게 찾을 수 있다. `entry` API로 키가 없을 때만 값을 삽입하는 패턴을 효율적으로 구현할 수 있다.

컬렉션에 값을 넣으면 소유권이 이동한다. 원본을 유지하려면 `clone`하거나 참조를 저장한다. 참조를 저장하면 라이프타임 문제가 생기는데 Day 9에서 배운다.

내일부터는 Part 2로 넘어가서 Rust의 가장 중요한 개념인 소유권과 빌림을 본격적으로 배운다. 오늘까지 만든 설정 파서와 로그 분석기에서 소유권이 어떻게 작동하는지 깊이 이해하게 될 것이다.
