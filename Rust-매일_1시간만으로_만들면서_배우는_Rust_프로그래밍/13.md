# Day 13: 열거형으로 프로토콜 정의하기
지난 시간까지 우리는 에러 처리를 개선하고 간단한 계산기 서버를 만들었다. 하지만 문자열 파싱만으로 프로토콜을 처리하는 것은 한계가 있다. 코드가 분산되어 있고, 새로운 명령을 추가하기 어려우며, 컴파일러가 우리를 도와줄 수 없다. 오늘은 Rust의 가장 강력한 기능 중 하나인 열거형(enum)을 사용해서 프로토콜을 타입 안전하게 정의하는 방법을 배운다.

## **열거형이란 무엇인가**

열거형은 여러 가능한 값 중 하나를 표현하는 타입이다. 다른 언어의 enum과 비슷해 보이지만, Rust의 enum은 훨씬 강력하다. 각 변형(variant)이 서로 다른 타입의 데이터를 가질 수 있기 때문이다.

```rust
// 간단한 열거형
enum TrafficLight {
    Red,
    Yellow,
    Green,
}

// 데이터를 가진 열거형
enum Message {
    Quit,                       // 데이터 없음
    Move { x: i32, y: i32 },   // 익명 구조체
    Write(String),              // 단일 값
    ChangeColor(u8, u8, u8),   // 튜플
}
```

이것이 강력한 이유는 컴파일러가 모든 경우를 처리했는지 확인할 수 있기 때문이다. 문자열로 "MOVE", "WRITE" 같은 명령을 처리하면 오타가 있어도 런타임에야 알 수 있다. 하지만 enum을 사용하면 컴파일 타임에 모든 경우를 처리했는지 확인된다.

## **패턴 매칭: enum의 완벽한 파트너**

enum을 사용하려면 패턴 매칭이 필요하다. `match` 표현식은 enum의 모든 변형을 처리하도록 강제한다.

```rust
fn handle_message(msg: Message) {
    match msg {
        Message::Quit => {
            println!("Quitting...");
        }
        Message::Move { x, y } => {
            println!("Moving to ({}, {})", x, y);
        }
        Message::Write(text) => {
            println!("Writing: {}", text);
        }
        Message::ChangeColor(r, g, b) => {
            println!("Changing color to RGB({}, {}, {})", r, g, b);
        }
    }
}
```

만약 하나의 변형이라도 빠뜨리면 컴파일러가 에러를 낸다. 이것은 안전성의 핵심이다.

```
    ┌─────────────┐
    │   Message   │
    └──────┬──────┘
           │
    ┌──────┴──────────────────────────┐
    │                                  │
    ▼                ▼                 ▼
┌────────┐    ┌──────────┐    ┌─────────────┐
│  Quit  │    │   Move   │    │    Write    │
└────────┘    │  {x, y}  │    │  (String)   │
              └──────────┘    └─────────────┘
                       ▼
                ┌──────────────┐
                │ ChangeColor  │
                │  (r, g, b)   │
                └──────────────┘
```

## **프로토콜을 enum으로 정의하기**

지난 시간의 서버를 개선해보자. 먼저 프로토콜의 모든 명령을 enum으로 정의한다.

```rust
#[derive(Debug, Clone)]
enum Request {
    Ping,
    Echo(String),
    Add(i32, i32),
    Subtract(i32, i32),
    Multiply(i32, i32),
    Divide(i32, i32),
    Uppercase(String),
    Lowercase(String),
    Quit,
}

#[derive(Debug, Clone)]
enum Response {
    Pong,
    Message(String),
    Number(i32),
    Error(String),
    Goodbye,
}
```

`Request`는 클라이언트가 보내는 명령이고, `Response`는 서버가 반환하는 응답이다. 이제 프로토콜이 명확하게 타입으로 표현되었다. `#[derive(Debug, Clone)]`은 디버깅 출력과 복제를 자동으로 구현해준다.

## **문자열을 enum으로 파싱하기**

네트워크를 통해서는 여전히 문자열이 오기 때문에, 문자열을 `Request` enum으로 변환하는 함수가 필요하다.

```rust
use std::num::ParseIntError;
use std::fmt;

#[derive(Debug)]
enum ParseError {
    EmptyCommand,
    UnknownCommand(String),
    InvalidArgument(String),
    ParseIntError(ParseIntError),
}

impl fmt::Display for ParseError {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        match self {
            ParseError::EmptyCommand => write!(f, "Empty command"),
            ParseError::UnknownCommand(cmd) => write!(f, "Unknown command: {}", cmd),
            ParseError::InvalidArgument(msg) => write!(f, "Invalid argument: {}", msg),
            ParseError::ParseIntError(e) => write!(f, "Parse error: {}", e),
        }
    }
}

impl std::error::Error for ParseError {}

impl From<ParseIntError> for ParseError {
    fn from(error: ParseIntError) -> Self {
        ParseError::ParseIntError(error)
    }
}

fn parse_request(line: &str) -> Result<Request, ParseError> {
    let parts: Vec<&str> = line.trim().split_whitespace().collect();
    
    if parts.is_empty() {
        return Err(ParseError::EmptyCommand);
    }
    
    match parts[0].to_uppercase().as_str() {
        "PING" => Ok(Request::Ping),
        "QUIT" => Ok(Request::Quit),
        
        "ECHO" => {
            if parts.len() < 2 {
                return Err(ParseError::InvalidArgument(
                    "ECHO requires a message".to_string()
                ));
            }
            Ok(Request::Echo(parts[1..].join(" ")))
        }
        
        "ADD" => {
            if parts.len() != 3 {
                return Err(ParseError::InvalidArgument(
                    "ADD requires exactly two numbers".to_string()
                ));
            }
            let x = parts[1].parse::<i32>()?;
            let y = parts[2].parse::<i32>()?;
            Ok(Request::Add(x, y))
        }
        
        "SUB" => {
            if parts.len() != 3 {
                return Err(ParseError::InvalidArgument(
                    "SUB requires exactly two numbers".to_string()
                ));
            }
            let x = parts[1].parse::<i32>()?;
            let y = parts[2].parse::<i32>()?;
            Ok(Request::Subtract(x, y))
        }
        
        "MUL" => {
            if parts.len() != 3 {
                return Err(ParseError::InvalidArgument(
                    "MUL requires exactly two numbers".to_string()
                ));
            }
            let x = parts[1].parse::<i32>()?;
            let y = parts[2].parse::<i32>()?;
            Ok(Request::Multiply(x, y))
        }
        
        "DIV" => {
            if parts.len() != 3 {
                return Err(ParseError::InvalidArgument(
                    "DIV requires exactly two numbers".to_string()
                ));
            }
            let x = parts[1].parse::<i32>()?;
            let y = parts[2].parse::<i32>()?;
            Ok(Request::Divide(x, y))
        }
        
        "UPPER" => {
            if parts.len() < 2 {
                return Err(ParseError::InvalidArgument(
                    "UPPER requires a string".to_string()
                ));
            }
            Ok(Request::Uppercase(parts[1..].join(" ")))
        }
        
        "LOWER" => {
            if parts.len() < 2 {
                return Err(ParseError::InvalidArgument(
                    "LOWER requires a string".to_string()
                ));
            }
            Ok(Request::Lowercase(parts[1..].join(" ")))
        }
        
        cmd => Err(ParseError::UnknownCommand(cmd.to_string())),
    }
}
```

이 함수는 문자열을 받아서 `Request` enum을 반환한다. 파싱 과정에서 발생할 수 있는 모든 에러를 `ParseError` enum으로 표현한다.

## **요청 처리하기**

이제 `Request`를 받아서 처리하고 `Response`를 반환하는 함수를 만든다.

```rust
fn handle_request(request: Request) -> Response {
    match request {
        Request::Ping => Response::Pong,
        
        Request::Echo(msg) => Response::Message(msg),
        
        Request::Add(x, y) => Response::Number(x + y),
        
        Request::Subtract(x, y) => Response::Number(x - y),
        
        Request::Multiply(x, y) => Response::Number(x * y),
        
        Request::Divide(x, y) => {
            if y == 0 {
                Response::Error("Division by zero".to_string())
            } else {
                Response::Number(x / y)
            }
        }
        
        Request::Uppercase(text) => Response::Message(text.to_uppercase()),
        
        Request::Lowercase(text) => Response::Message(text.to_lowercase()),
        
        Request::Quit => Response::Goodbye,
    }
}
```

이 함수는 놀랍도록 깔끔하다. 각 요청 타입에 대한 처리가 명확하게 분리되어 있고, 컴파일러가 모든 경우를 처리했는지 확인해준다. 새로운 명령을 추가하면 컴파일러가 이 함수를 업데이트하라고 알려줄 것이다.

## **응답을 문자열로 변환하기**

서버는 `Response`를 네트워크를 통해 보내야 하므로 문자열로 변환해야 한다.

```rust
fn format_response(response: &Response) -> String {
    match response {
        Response::Pong => "PONG\n".to_string(),
        Response::Message(msg) => format!("{}\n", msg),
        Response::Number(n) => format!("{}\n", n),
        Response::Error(msg) => format!("ERROR: {}\n", msg),
        Response::Goodbye => "Goodbye!\n".to_string(),
    }

```

여기서 `response`를 빌려온다는 점에 주목하자. 응답을 포맷한 후에도 여전히 사용할 수 있도록 소유권을 가져오지 않는다.

## **완전한 서버 구현**

이제 모든 조각을 합쳐서 완전한 서버를 만들어보자.

```rust
use std::io::{self, BufRead, BufReader, Write};
use std::net::{TcpListener, TcpStream};
use std::fmt;
use std::num::ParseIntError;

// Request enum 정의
#[derive(Debug, Clone)]
enum Request {
    Ping,
    Echo(String),
    Add(i32, i32),
    Subtract(i32, i32),
    Multiply(i32, i32),
    Divide(i32, i32),

    Lowercase(String),

// 클라이언트 처리
fn handle_client(stream: TcpStream) -> io::Result<()> {
    let peer_addr = stream.peer_addr()?;
    println!("[{}] Connected", peer_addr);
    
    let mut reader = BufReader::new(stream.try_clone()?);
    let mut writer = stream;
    
    // 환영 메시지
    writer.write_all(b"===== Rust Protocol Server =====\n")?;
    writer.write_all(b"Commands:\n")?;
    writer.write_all(b"  PING              - Check connection\n")?;
    writer.write_all(b"  ECHO <msg>        - Echo message\n")?;
    writer.write_all(b"  ADD <x> <y>       - Add two numbers\n")?;
    writer.write_all(b"  SUB <x> <y>       - Subtract y from x\n")?;
    writer.write_all(b"  MUL <x> <y>       - Multiply two numbers\n")?;
    writer.write_all(b"  DIV <x> <y>       - Divide x by y\n")?;
    writer.write_all(b"  UPPER <text>      - Convert to uppercase\n")?;
    writer.write_all(b"  LOWER <text>      - Convert to lowercase\n")?;
    writer.write_all(b"  QUIT              - Disconnect\n")?;
    writer.write_all(b"=================================\n")?;
    writer.flush()?;
    
    loop {
        let mut line = String::new();
        let bytes_read = reader.read_line(&mut line)?;
        
        if bytes_read == 0 {
            println!("[{}] Disconnected (connection closed)", peer_addr);
            break;
        }
        
        println!("[{}] Received: {}", peer_addr, line.trim());
        
        // 파싱 시도
        let response = match parse_request(&line) {
            Ok(request) => {
                println!("[{}] Parsed request: {:?}", peer_addr, request);
                
                // 종료 요청 확인
                let is_quit = matches!(request, Request::Quit);
                
                // 요청 처리
                let response = handle_request(request);
                
                if is_quit {
                    let output = format_response(&response);
                    writer.write_all(output.as_bytes())?;
                    writer.flush()?;
                    println!("[{}] Client quit", peer_addr);
                    break;
                }
                
                response
            }
            Err(e) => {
                println!("[{}] Parse error: {}", peer_addr, e);
                Response::Error(e.to_string())
            }
        };
        
        // 응답 전송
        let output = format_response(&response);
        writer.write_all(output.as_bytes())?;
        writer.flush()?;
        
        println!("[{}] Sent response: {:?}", peer_addr, response);
    }
    
    Ok(())
}

// 메인 함수
fn main() -> io::Result<()> {
    let listener = TcpListener::bind("127.0.0.1:8080")?;
    println!("Server listening on 127.0.0.1:8080");
    println!("Press Ctrl+C to stop");
    println!();
    
    for stream in listener.incoming() {
        match stream {
            Ok(stream) => {
                if let Err(e) = handle_client(stream) {
                    eprintln!("Client handler error: {}", e);
                }
            }
            Err(e) => {
                eprintln!("Failed to accept connection: {}", e);
            }
        }
    }
    
    Ok(())
}
```

## **enum의 메모리 레이아웃**

enum은 메모리에서 어떻게 표현될까? Rust는 가장 큰 변형을 저장할 수 있는 크기와 어떤 변형인지를 나타내는 태그(tag)를 사용한다.

```
Response enum 메모리 레이아웃:

┌─────────────────────────────────────┐
│  Tag (어떤 변형인지)                  │  예: 0=Pong, 1=Message, ...
├─────────────────────────────────────┤
│                                     │
│  Data (가장 큰 변형의 크기)           │  예: String의 경우 24바이트
│                                     │
└─────────────────────────────────────┘

Response::Pong:
┌─────┬──────────────────────┐
│  0  │  (사용 안 함)         │
└─────┴──────────────────────┘

Response::Message("Hello"):
┌─────┬──────────────────────┐
│  1  │  String("Hello")     │
└─────┴──────────────────────┘

Response::Number(42):
┌─────┬──────────────────────┐
│  2  │  42  (나머지 미사용)  │
└─────┴──────────────────────┘
```

이것은 enum이 타입 안전성을 제공하면서도 효율적이라는 것을 보여준다.

## **패턴 매칭의 고급 기능**

패턴 매칭은 단순히 enum을 분해하는 것 이상을 할 수 있다.

```rust
// 조건부 매칭
fn describe_number(response: &Response) -> String {
    match response {
        Response::Number(n) if *n > 0 => format!("Positive: {}", n),
        Response::Number(n) if *n < 0 => format!("Negative: {}", n),
        Response::Number(_) => "Zero".to_string(),
        _ => "Not a number".to_string(),
    }
}

// 여러 패턴 동시에
fn is_success(response: &Response) -> bool {
    matches!(response, Response::Pong | Response::Message(_) | Response::Number(_))
}

// 중첩 패턴
fn analyze_request(request: &Request) -> &str {
    match request {
        Request::Add(x, y) | Request::Multiply(x, y) if *x == *y => {
            "Operating on equal numbers"
        }
        Request::Divide(_, 0) => {
            "Warning: division by zero!"
        }
        Request::Echo(msg) if msg.is_empty() => {
            "Empty echo"
        }
        _ => "Normal operation"
    }
}
```

`if` 가드는 패턴에 추가 조건을 붙이고, `|`는 여러 패턴을 하나로 합친다. `matches!` 매크로는 특정 패턴과 일치하는지 빠르게 확인한다.

## **enum과 소유권**

enum을 사용할 때 소유권 규칙은 여전히 적용된다. enum의 각 변형에 저장된 데이터는 그 enum이 소유한다.

```rust
fn process_message(response: Response) {
    match response {
        Response::Message(msg) => {
            // msg의 소유권이 여기로 이동함
            println!("Message: {}", msg);
            // msg는 여기서 드롭됨
        }
        other => {
            // other가 나머지 변형들을 소유
            println!("Other response: {:?}", other);
        }
    }
    // response는 이미 이동되어 여기서 사용할 수 없음
}

// 소유권을 가져오지 않으려면 참조를 사용
fn process_message_ref(response: &Response) {
    match response {
        Response::Message(msg) => {
            // msg는 &String 타입
            println!("Message: {}", msg);
        }
        _ => println!("Other response"),
    }
    // response는 여전히 사용 가능
}
```

match에서 변수를 바인딩하면 기본적으로 소유권이 이동한다. 참조를 사용하면 빌려오기만 한다.

## **실전 패턴: Result와 Option도 enum이다**

지금까지 사용해온 `Result<T, E>`와 `Option<T>`도 사실 enum이다.

```rust
// 표준 라이브러리의 실제 정의 (단순화)
enum Option<T> {
    Some(T),
    None,
}

enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

이제 `?` 연산자와 `match`가 왜 그렇게 작동하는지 이해가 된다. 이것들은 단지 enum 패턴 매칭의 특별한 경우일 뿐이다.

```rust
// 이 두 코드는 동일하다
fn method1() -> Result<i32, ParseError> {
    let request = parse_request("ADD 1 2")?;
    Ok(42)
}

fn method2() -> Result<i32, ParseError> {
    let request = match parse_request("ADD 1 2") {
        Ok(req) => req,
        Err(e) => return Err(e),
    };
    Ok(42)
}
```

## **프로토콜 확장하기**

enum을 사용하는 큰 장점은 프로토콜을 확장하기 쉽다는 것이다. 새로운 명령을 추가해보자.

```rust
#[derive(Debug, Clone)]
enum Request {
    // 기존 명령들...
    Ping,
    Echo(String),
    // ... 생략 ...
    
    // 새로운 명령 추가
    Reverse(String),
    Length(String),
    Stats,
}

#[derive(Debug, Clone)]
enum Response {
    // 기존 응답들...
    Pong,
    Message(String),
    Number(i32),
    Error(String),
    Goodbye,
    
    // 새로운 응답 추가
    Stats { 
        requests_handled: u64, 
        uptime_seconds: u64 
    },
}
```

이제 컴파일러가 `parse_request`, `handle_request`, `format_response` 함수를 업데이트하라고 알려줄 것이다. 이것은 놓치는 부분 없이 전체 시스템을 일관되게 업데이트할 수 있게 해준다.

```rust
// parse_request에 추가
"REVERSE" => {
    if parts.len() < 2 {
        return Err(ParseError::InvalidArgument(
            "REVERSE requires a string".to_string()
        ));
    }
    Ok(Request::Reverse(parts[1..].join(" ")))
}
"LENGTH" => {
    if parts.len() < 2 {
        return Err(ParseError::InvalidArgument(
            "LENGTH requires a string".to_string()
        ));
    }
    Ok(Request::Length(parts[1..].join(" ")))
}
"STATS" => Ok(Request::Stats),

// handle_request에 추가 (간단한 버전)
Request::Reverse(text) => {
    Response::Message(text.chars().rev().collect())
}
Request::Length(text) => {
    Response::Number(text.len() as i32)
}
Request::Stats => {
    Response::Stats {
        requests_handled: 100, // 실제로는 카운터 필요
        uptime_seconds: 3600,
    }
}

// format_response에 추가
Response::Stats { requests_handled, uptime_seconds } => {
    format!("Requests: {}, Uptime: {}s\n", requests_handled, uptime_seconds)
}
```

## **정리하면**

오늘 우리는 Rust의 enum이 얼마나 강력한지 배웠다. enum은 단순한 열거가 아니라 서로 다른 타입의 데이터를 가질 수 있는 타입 안전한 합집합(union)이다. 패턴 매칭은 enum의 모든 경우를 처리하도록 강제해서 버그를 컴파일 타임에 잡아낸다. 

프로토콜을 enum으로 정의하면 코드가 명확해지고, 확장하기 쉬워지며, 컴파일러가 우리를 도와줄 수 있다. 이것은 문자열 파싱만 사용하는 것보다 훨씬 안전하고 유지보수하기 쉽다.

다음 시간에는 여러 클라이언트를 순차적으로 처리하는 서버로 개선하고, `BufReader`와 `BufWriter`로 버퍼링을 효율적으로 관리하는 방법을 배운다. 그리고 참조와 버퍼의 관계를 더 깊이 이해하게 될 것이다.
