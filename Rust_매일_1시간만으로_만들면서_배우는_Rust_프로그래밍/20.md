# Day 20: 우아한 종료 처리

지금까지 우리는 동작하는 채팅 서버를 만들었지만, 한 가지 중요한 문제가 남아있다. Ctrl+C로 서버를 종료하면 클라이언트들이 갑자기 연결이 끊어지고, 진행 중이던 작업들이 불완전한 상태로 남는다. 프로덕션 환경에서는 서버를 종료할 때 **우아한 종료(graceful shutdown)**가 필수다. 모든 클라이언트에게 종료를 알리고, 진행 중인 요청을 완료하고, 리소스를 정리한 후에 깨끗하게 종료해야 한다.

오늘은 시그널 처리, 종료 신호 전파, 스레드 정리, 그리고 Rust의 `Drop` trait을 활용한 자동 리소스 관리를 배운다. 이 과정에서 소유권과 빌림이 어떻게 안전한 정리를 보장하는지 깊이 이해하게 될 것이다.

## 우아한 종료의 흐름

우아한 종료는 여러 단계로 이루어진다. 전체 흐름을 시각화해보자.

```
1. Ctrl+C 시그널 감지
         │
         ▼
2. TcpListener 종료 (새 연결 거부)
         │
         ▼
3. 모든 클라이언트에게 종료 알림
         │
         ▼
4. 진행 중인 작업 완료 대기
         │
         ▼
5. 모든 스레드 join
         │
         ▼
6. 리소스 정리 (파일, 연결 등)
         │
         ▼
7. 프로세스 종료
```

각 단계가 순서대로 진행되어야 한다. 중간에 스킵하거나 역순으로 하면 데이터 손실이나 불완전한 상태가 발생할 수 있다.

## Ctrl+C 시그널 처리하기

Unix 계열 시스템에서 Ctrl+C는 SIGINT 시그널을 보낸다. Rust 표준 라이브러리에는 시그널 처리 기능이 없지만, `ctrlc` 크레이트를 사용하면 쉽게 처리할 수 있다.

먼저 `Cargo.toml`에 의존성을 추가한다.

```toml
[dependencies]
ctrlc = "3.4"
```

시그널을 처리하려면 여러 스레드에 종료 신호를 전달할 방법이 필요하다. 가장 간단한 방법은 `Arc<AtomicBool>`을 사용하는 것이다.

```rust
use std::sync::Arc;
use std::sync::atomic::{AtomicBool, Ordering};

fn main() {
    let shutdown = Arc::new(AtomicBool::new(false));
    let shutdown_clone = Arc::clone(&shutdown);

    ctrlc::set_handler(move || {
        println!("\n종료 신호 받음. 서버를 종료합니다...");
        shutdown_clone.store(true, Ordering::SeqCst);
    }).expect("시그널 핸들러 설정 실패");

    // 서버 로직...
    
    while !shutdown.load(Ordering::SeqCst) {
        // 작업 수행
    }

    println!("정리 작업 중...");
}
```

`AtomicBool`은 락 없이도 여러 스레드에서 안전하게 읽고 쓸 수 있는 불리언 값이다. `Ordering::SeqCst`는 가장 강력한 메모리 순서 보장을 제공한다. 자세한 내용은 고급 주제지만, 기본적으로 모든 스레드가 일관된 값을 보도록 보장한다고 이해하면 된다.

## 채널을 통한 종료 신호 전파

`AtomicBool`은 간단하지만, 채널을 사용하면 더 명확한 종료 패턴을 만들 수 있다. 특별한 종료 메시지를 정의해보자.

```rust
use std::sync::mpsc::{self, Sender, Receiver};

#[derive(Debug, Clone)]
enum ServerMessage {
    Shutdown,
}

type ShutdownSender = Sender<ServerMessage>;
type ShutdownReceiver = Receiver<ServerMessage>;
```

이제 메인 스레드에서 모든 스레드로 종료 신호를 보낼 수 있다. 하지만 여기에 문제가 있다. `mpsc` 채널은 단일 소비자만 가능하므로, 여러 스레드에 동시에 알릴 수 없다. 해결책은 각 스레드마다 별도의 채널을 만들고, 모든 송신자를 목록에 저장하는 것이다.

```rust
use std::sync::{Arc, Mutex};

type ShutdownChannels = Arc<Mutex<Vec<ShutdownSender>>>;

fn main() {
    let shutdown_channels: ShutdownChannels = Arc::new(Mutex::new(Vec::new()));
    let shutdown_channels_clone = Arc::clone(&shutdown_channels);

    ctrlc::set_handler(move || {
        println!("\n종료 신호 받음!");
        let channels = shutdown_channels_clone.lock().unwrap();
        for tx in channels.iter() {
            let _ = tx.send(ServerMessage::Shutdown);
        }
    }).expect("시그널 핸들러 설정 실패");

    // 스레드 생성 시
    let (shutdown_tx, shutdown_rx) = mpsc::channel();
    {
        let mut channels = shutdown_channels.lock().unwrap();
        channels.push(shutdown_tx);
    }

    // 스레드에서 shutdown_rx를 사용해서 종료 신호 확인
}
```

이 패턴은 작동하지만 복잡하다. 더 간단한 방법은 **브로드캐스트 채널**을 사용하는 것인데, 표준 라이브러리에는 없으므로 실무에서는 `tokio::sync::broadcast`나 비슷한 크레이트를 쓴다. 오늘은 교육 목적으로 `AtomicBool`과 일반 채널을 조합한 실용적인 패턴을 사용하겠다.

## 채팅 서버에 우아한 종료 추가하기

Day 19에서 만든 채팅 서버를 확장해서 우아한 종료를 구현해보자. 전체 구조는 다음과 같다.

```
┌──────────────────────────────────────────┐
│           메인 스레드                    │
│  - TcpListener 루프                      │
│  - Ctrl+C 시그널 감지                    │
│  - shutdown flag 설정                    │
└──────────────┬───────────────────────────┘
               │
    ┌──────────┼──────────┐
    ▼          ▼          ▼
┌─────────┐ ┌─────────┐ ┌─────────┐
│Client 1 │ │Client 2 │ │Client 3 │
│ Thread  │ │ Thread  │ │ Thread  │
└─────────┘ └─────────┘ └─────────┘
    │          │          │
    └──────────┴──────────┘
         종료 확인
```

먼저 필요한 타입과 구조를 정의한다.

```rust
use std::sync::{Arc, Mutex};
use std::sync::atomic::{AtomicBool, Ordering};
use std::sync::mpsc::{self, Sender, Receiver};
use std::net::{TcpListener, TcpStream};
use std::io::{BufRead, BufReader, Write};
use std::thread::{self, JoinHandle};
use std::time::Duration;

type ClientList = Arc<Mutex<Vec<(usize, Sender<String>)>>>;

struct Server {
    clients: ClientList,
    shutdown: Arc<AtomicBool>,
    client_handles: Vec<JoinHandle<()>>,
}

impl Server {
    fn new() -> Self {
        Server {
            clients: Arc::new(Mutex::new(Vec::new())),
            shutdown: Arc::new(AtomicBool::new(false)),
            client_handles: Vec::new(),
        }
    }

    fn is_shutting_down(&self) -> bool {
        self.shutdown.load(Ordering::SeqCst)
    }

    fn initiate_shutdown(&self) {
        self.shutdown.store(true, Ordering::SeqCst);
    }
}
```

`Server` 구조체는 서버의 상태를 캡슐화한다. 클라이언트 목록, 종료 플래그, 그리고 생성된 스레드 핸들들을 추적한다. 스레드 핸들을 저장하면 나중에 모든 스레드가 완료될 때까지 `join()`할 수 있다.

## TcpListener의 non-blocking 모드

`TcpListener::incoming()`은 블로킹 호출이므로 종료 신호를 받아도 즉시 반응하지 않는다. 새 연결이 들어올 때까지 계속 기다린다. 이를 해결하려면 `set_nonblocking(true)`를 사용해서 논블로킹 모드로 전환한다.

```rust
fn run_server(&mut self) -> std::io::Result<()> {
    let listener = TcpListener::bind("127.0.0.1:8080")?;
    listener.set_nonblocking(true)?;

    println!("╔════════════════════════════════════╗");
    println!("║    Rust 채팅 서버 v2.0            ║");
    println!("║    127.0.0.1:8080                 ║");
    println!("║    Ctrl+C로 종료                  ║");
    println!("╚════════════════════════════════════╝");

    let mut next_id = 1;

    loop {
        // 종료 신호 확인
        if self.is_shutting_down() {
            println!("새 연결 수락 중단...");
            break;
        }

        // non-blocking으로 연결 시도
        match listener.accept() {
            Ok((stream, addr)) => {
                println!("[클라이언트 {}] 연결: {}", next_id, addr);
                let handle = self.spawn_client_handler(stream, next_id);
                self.client_handles.push(handle);
                next_id += 1;
            }
            Err(ref e) if e.kind() == std::io::ErrorKind::WouldBlock => {
                // 연결이 없으면 잠시 대기
                thread::sleep(Duration::from_millis(100));
            }
            Err(e) => {
                eprintln!("연결 수락 에러: {}", e);
            }
        }
    }

    Ok(())
}
```

논블로킹 모드에서는 `accept()`가 즉시 반환된다. 연결이 있으면 `Ok`를 반환하고, 없으면 `WouldBlock` 에러를 반환한다. 이 경우 짧게 sleep한 후 다시 시도한다. 이렇게 하면 종료 플래그를 주기적으로 확인할 수 있다.

## 클라이언트 핸들러 수정

클라이언트 핸들러도 종료 신호를 확인하도록 수정해야 한다. 읽기 루프에서 주기적으로 플래그를 체크한다.

```rust
impl Server {
    fn spawn_client_handler(&self, stream: TcpStream, client_id: usize) -> JoinHandle<()> {
        let clients = Arc::clone(&self.clients);
        let shutdown = Arc::clone(&self.shutdown);

        thread::spawn(move || {
            handle_client(stream, client_id, clients, shutdown);
        })
    }
}

fn handle_client(
    stream: TcpStream,
    client_id: usize,
    clients: ClientList,
    shutdown: Arc<AtomicBool>,
) {
    println!("[클라이언트 {}] 핸들러 시작", client_id);

    let (tx, rx) = mpsc::channel::<String>();

    // 클라이언트 등록
    {
        let mut client_list = clients.lock().unwrap();
        client_list.push((client_id, tx.clone()));
        println!("총 접속자: {}", client_list.len());
    }

    // 환영 메시지
    tx.send(format!("환영합니다! 당신은 클라이언트 {}입니다.", client_id)).unwrap();

    // 입장 알림
    broadcast_message(&clients, client_id, 
        format!("*** [{}]님이 입장하셨습니다 ***", client_id));

    // TCP 스트림을 논블로킹으로 설정
    stream.set_nonblocking(true).unwrap();
    let read_stream = stream.try_clone().unwrap();
    let mut write_stream = stream;

    // 쓰기 스레드
    let shutdown_clone = Arc::clone(&shutdown);
    let writer_id = client_id;
    let write_handle = thread::spawn(move || {
        for message in rx {
            if shutdown_clone.load(Ordering::SeqCst) {
                println!("[클라이언트 {}] 쓰기 스레드 종료", writer_id);
                break;
            }

            if writeln!(write_stream, "{}", message).is_err() {
                break;
            }
        }
    });

    // 읽기 루프
    let mut reader = BufReader::new(read_stream);
    let mut line_buffer = String::new();

    loop {
        // 종료 신호 확인
        if shutdown.load(Ordering::SeqCst) {
            println!("[클라이언트 {}] 종료 신호 감지", client_id);
            tx.send(String::from("서버가 종료됩니다. 연결을 끊습니다.")).unwrap();
            break;
        }

        line_buffer.clear();
        match reader.read_line(&mut line_buffer) {
            Ok(0) => {
                println!("[클라이언트 {}] 연결 끊김", client_id);
                break;
            }
            Ok(_) => {
                let message = line_buffer.trim();
                if message.is_empty() {
                    continue;
                }

                println!("[{}] {}", client_id, message);
                let formatted = format!("[{}]: {}", client_id, message);
                broadcast_message(&clients, client_id, formatted);
            }
            Err(ref e) if e.kind() == std::io::ErrorKind::WouldBlock => {
                // 데이터 없음, 잠시 대기
                thread::sleep(Duration::from_millis(50));
            }
            Err(e) => {
                eprintln!("[클라이언트 {}] 읽기 에러: {}", client_id, e);
                break;
            }
        }
    }

    // 퇴장 알림
    let leave_message = format!("*** [{}]님이 퇴장하셨습니다 ***", client_id);
    broadcast_message(&clients, client_id, leave_message);

    // 클라이언트 제거
    {
        let mut client_list = clients.lock().unwrap();
        client_list.retain(|(id, _)| *id != client_id);
        println!("[클라이언트 {}] 제거. 남은 인원: {}", client_id, client_list.len());
    }

    // 쓰기 스레드 종료 대기 (최대 1초)
    let join_result = write_handle.join();
    if join_result.is_err() {
        eprintln!("[클라이언트 {}] 쓰기 스레드 join 실패", client_id);
    }

    println!("[클라이언트 {}] 핸들러 종료", client_id);
}
```

스트림을 논블로킹으로 설정하면 `read_line()`이 즉시 반환된다. 데이터가 없으면 `WouldBlock` 에러가 발생하고, 이 경우 짧게 sleep한 후 종료 플래그를 다시 확인한다. 종료 신호를 감지하면 클라이언트에게 메시지를 보내고 루프를 빠져나온다.

## 메인 함수와 시그널 핸들러 연결

이제 모든 부분을 메인 함수에서 연결한다.

```rust
fn main() {
    let mut server = Server::new();

    // Ctrl+C 핸들러 설정
    let shutdown_flag = Arc::clone(&server.shutdown);
    ctrlc::set_handler(move || {
        println!("\n╔════════════════════════════════════╗");
        println!("║    종료 신호 받음 (Ctrl+C)        ║");
        println!("╚════════════════════════════════════╝");
        shutdown_flag.store(true, Ordering::SeqCst);
    }).expect("Ctrl+C 핸들러 설정 실패");

    // 서버 실행
    if let Err(e) = server.run_server() {
        eprintln!("서버 실행 에러: {}", e);
    }

    // 종료 알림
    println!("\n모든 클라이언트에게 종료 알림 전송 중...");
    {
        let client_list = server.clients.lock().unwrap();
        for (id, tx) in client_list.iter() {
            let _ = tx.send(String::from("*** 서버가 종료됩니다 ***"));
            println!("  클라이언트 {}에게 알림 전송", id);
        }
    }

    // 모든 클라이언트 스레드가 종료될 때까지 대기
    println!("모든 클라이언트 스레드 종료 대기 중...");
    let total = server.client_handles.len();
    for (i, handle) in server.client_handles.into_iter().enumerate() {
        print!("  스레드 {}/{} 종료 대기... ", i + 1, total);
        handle.join().unwrap();
        println!("완료");
    }

    println!("\n╔════════════════════════════════════╗");
    println!("║    서버가 안전하게 종료되었습니다  ║");
    println!("╚════════════════════════════════════╝");
}
```

이 메인 함수는 우아한 종료의 모든 단계를 명확하게 보여준다. 먼저 서버를 실행하고, Ctrl+C를 받으면 새 연결 수락을 중단한다. 그 다음 모든 클라이언트에게 종료를 알리고, 각 클라이언트 스레드가 정리를 마칠 때까지 기다린다. 마지막으로 깨끗한 종료 메시지를 출력한다.

## Drop trait으로 자동 리소스 정리

Rust의 강력한 특징 중 하나는 `Drop` trait이다. 값이 스코프를 벗어나면 자동으로 `drop()` 메서드가 호출된다. 이를 활용해서 리소스 정리를 자동화할 수 있다.

예를 들어, 클라이언트 연결을 나타내는 구조체를 만들어보자.

```rust
struct ClientConnection {
    id: usize,
    clients: ClientList,
}

impl ClientConnection {
    fn new(id: usize, clients: ClientList) -> Self {
        // 등록
        {
            let mut client_list = clients.lock().unwrap();
            println!("[클라이언트 {}] 등록됨", id);
        }

        ClientConnection { id, clients }
    }
}

impl Drop for ClientConnection {
    fn drop(&mut self) {
        // 자동 정리
        let mut client_list = self.clients.lock().unwrap();
        client_list.retain(|(id, _)| *id != self.id);
        println!("[클라이언트 {}] Drop으로 자동 제거됨", self.id);
    }
}
```

이제 `ClientConnection`을 사용하면 명시적으로 제거하지 않아도 스코프를 벗어날 때 자동으로 목록에서 제거된다.

```rust
fn handle_client_with_raii(
    stream: TcpStream,
    client_id: usize,
    clients: ClientList,
    shutdown: Arc<AtomicBool>,
) {
    // 이 변수가 스코프를 벗어나면 자동으로 Drop::drop 호출
    let _connection = ClientConnection::new(client_id, Arc::clone(&clients));

    // ... 클라이언트 처리 로직 ...
    // 함수가 어떻게 끝나든 (정상, 에러, panic) _connection이 드롭됨
}
```

`Drop` trait은 **RAII(Resource Acquisition Is Initialization)** 패턴을 구현한다. 리소스를 획득할 때 객체를 초기화하고, 객체가 파괴될 때 자동으로 리소스를 해제한다. 파일 핸들, 네트워크 연결, 락 등 모든 리소스에 적용할 수 있다.

실제로 `Mutex`의 락도 `MutexGuard`가 `Drop`을 구현해서 스코프를 벗어날 때 자동으로 언락된다.

```rust
{
    let guard = mutex.lock().unwrap();
    // guard를 사용
} // 여기서 guard가 드롭되고 자동으로 언락됨
```

## 타임아웃을 이용한 강제 종료

때로는 클라이언트가 응답하지 않거나 무한 루프에 빠질 수 있다. 우아한 종료에 타임아웃을 추가해서 일정 시간 후에는 강제로 종료하도록 할 수 있다.

```rust
use std::time::Instant;

fn main() {
    let mut server = Server::new();

    // ... (Ctrl+C 핸들러 설정) ...

    // 서버 실행
    if let Err(e) = server.run_server() {
        eprintln!("서버 실행 에러: {}", e);
    }

    // 종료 프로세스 시작
    println!("\n종료 프로세스 시작...");
    let shutdown_start = Instant::now();
    let timeout = Duration::from_secs(10);

    // 모든 클라이언트에게 알림
    {
        let client_list = server.clients.lock().unwrap();
        for (id, tx) in client_list.iter() {
            let _ = tx.send(String::from("*** 서버가 10초 후 종료됩니다 ***"));
        }
    }

    // 스레드 종료 대기 (타임아웃 있음)
    for (i, handle) in server.client_handles.into_iter().enumerate() {
        let elapsed = shutdown_start.elapsed();
        if elapsed >= timeout {
            println!("타임아웃! 남은 스레드 강제 종료");
            break;
        }

        let remaining = timeout - elapsed;
        println!("스레드 {} 대기 중... (남은 시간: {}초)", 
                 i + 1, remaining.as_secs());
        
        // join은 타임아웃을 지원하지 않으므로 별도 처리 필요
        handle.join().unwrap();
    }

    let total_time = shutdown_start.elapsed();
    println!("종료 완료 (소요 시간: {:.2}초)", total_time.as_secs_f64());
}
```

더 정교한 타임아웃 처리를 원한다면 `thread::spawn`과 채널을 조합해서 구현할 수 있다.

```rust
use std::sync::mpsc;

fn join_with_timeout(handle: JoinHandle<()>, timeout: Duration) -> Result<(), String> {
    let (tx, rx) = mpsc::channel();

    thread::spawn(move || {
        let _ = handle.join();
        let _ = tx.send(());
    });

    match rx.recv_timeout(timeout) {
        Ok(_) => Ok(()),
        Err(_) => Err(String::from("타임아웃")),
    }
}
```

이 함수는 스레드가 완료될 때까지 기다리되, 지정된 시간이 지나면 타임아웃을 반환한다. 실제로는 스레드가 계속 실행될 수 있지만, 더 이상 기다리지 않고 진행한다.

## 시그널과 멀티스레드의 상호작용

시그널 핸들러는 특별한 컨텍스트에서 실행되며, 할 수 있는 작업이 제한된다. 일반적으로 시그널 핸들러에서는 최소한의 작업만 수행하고, 실제 처리는 메인 로직에서 해야 한다.

우리 코드에서는 시그널 핸들러가 단순히 `AtomicBool`을 설정하기만 한다. 이것은 안전하다. 하지만 복잡한 작업(락 획득, 메모리 할당, I/O 등)은 피해야 한다.

```rust
// 안전: atomic 연산만 수행
ctrlc::set_handler(move || {
    shutdown_flag.store(true, Ordering::SeqCst);
}).unwrap();

// 위험: 락을 획득하려고 시도 (데드락 가능)
ctrlc::set_handler(move || {
    let mut list = clients.lock().unwrap(); // 나쁜 예!
    list.clear();
}).unwrap();
```

만약 시그널이 `Mutex`를 잡고 있는 스레드를 중단시키면, 시그널 핸들러에서 같은 락을 얻으려고 할 때 데드락이 발생한다. 이것이 시그널 핸들러를 단순하게 유지해야 하는 이유다.

## 전체 코드: 우아한 종료가 적용된 채팅 서버

모든 개념을 종합한 완전한 예제다.

```rust
use std::sync::{Arc, Mutex};
use std::sync::atomic::{AtomicBool, Ordering};
use std::sync::mpsc::{self, Sender};
use std::net::{TcpListener, TcpStream};
use std::io::{BufRead, BufReader, Write};
use std::thread::{self, JoinHandle};
use std::time::Duration;

type ClientList = Arc<Mutex<Vec<(usize, Sender<String>)>>>;

struct Server {
    clients: ClientList,
    shutdown: Arc<AtomicBool>,
    client_handles: Vec<JoinHandle<()>>,
}

impl Server {
    fn new() -> Self {
        Server {
            clients: Arc::new(Mutex::new(Vec::new())),
            shutdown: Arc::new(AtomicBool::new(false)),
            client_handles: Vec::new(),
        }
    }

    fn is_shutting_down(&self) -> bool {
        self.shutdown.load(Ordering::SeqCst)
    }

    fn run(&mut self) -> std::io::Result<()> {
        let listener = TcpListener::bind("127.0.0.1:8080")?;
        listener.set_nonblocking(true)?;

        println!("╔════════════════════════════════════╗");
        println!("║  Rust 채팅 서버 v2.0 (우아한 종료)║");
        println!("║  127.0.0.1:8080                   ║");
        println!("║  Ctrl+C로 종료                    ║");
        println!("╚════════════════════════════════════╝\n");

        let mut next_id = 1;

        loop {
            if self.is_shutting_down() {
                println!("→ 새 연결 수락 중단");
                break;
            }

            match listener.accept() {
                Ok((stream, addr)) => {
                    println!("[시스템] 클라이언트 {} 연결: {}", next_id, addr);
                    let handle = self.spawn_client_handler(stream, next_id);
                    self.client_handles.push(handle);
                    next_id += 1;
                }
                Err(ref e) if e.kind() == std::io::ErrorKind::WouldBlock => {
                    thread::sleep(Duration::from_millis(100));
                }
                Err(e) => {
                    eprintln!("[에러] 연결 수락 실패: {}", e);
                }
            }
        }

        Ok(())
    }

    fn spawn_client_handler(&self, stream: TcpStream, id: usize) -> JoinHandle<()> {
        let clients = Arc::clone(&self.clients);
        let shutdown = Arc::clone(&self.shutdown);

        thread::spawn(move || {
            handle_client(stream, id, clients, shutdown);
        })
    }

    fn shutdown_all_clients(&self) {
        println!("\n→ 모든 클라이언트에게 종료 알림");
        let client_list = self.clients.lock().unwrap();
        let count = client_list.len();
        
        for (id, tx) in client_list.iter() {
            if tx.send(String::from("━━━ 서버가 종료됩니다 ━━━")).is_ok() {
                println!("  ✓ 클라이언트 {}에게 알림 전송", id);
            }
        }

        println!("  총 {}명에게 알림 완료\n", count);
    }

    fn wait_for_all_threads(&mut self) {
        let total = self.client_handles.len();
        if total == 0 {
            println!("→ 종료할 클라이언트 스레드 없음\n");
            return;
        }

        println!("→ {} 개의 클라이언트 스레드 종료 대기", total);

        for (i, handle) in self.client_handles.drain(..).enumerate() {
            print!("  [{}/{}] 종료 대기... ", i + 1, total);
            if handle.join().is_ok() {
                println!("✓");
            } else {
                println!("✗ (에러)");
            }
        }

        println!("  모든 스레드 종료 완료\n");
    }
}

fn handle_client(
    stream: TcpStream,
    client_id: usize,
    clients: ClientList,
    shutdown: Arc<AtomicBool>,
) {
    let (tx, rx) = mpsc::channel::<String>();

    // 등록
    {
        let mut client_list = clients.lock().unwrap();
        client_list.push((client_id, tx.clone()));
    }

    // 환영 메시지
    let _ = tx.send(format!("━━━━━━━━━━━━━━━━━━━━━━━━"));
    let _ = tx.send(format!("  환영합니다!"));
    let _ = tx.send(format!("  당신은 클라이언트 {} 입니다", client_id));
    let _ = tx.send(format!("━━━━━━━━━━━━━━━━━━━━━━━━"));

    broadcast_message(&clients, client_id, 
        format!(">>> [{}]님이 입장하셨습니다", client_id));

    // 스트림 설정
    stream.set_nonblocking(true).ok();
    let read_stream = stream.try_clone().unwrap();
    let mut write_stream = stream;

    // 쓰기 스레드
    let shutdown_clone = Arc::clone(&shutdown);
    let write_handle = thread::spawn(move || {
        for message in rx {
            if shutdown_clone.load(Ordering::SeqCst) {
                break;
            }
            if writeln!(write_stream, "{}", message).is_err() {
                break;
            }
        }
    });

    // 읽기 루프
    let mut reader = BufReader::new(read_stream);
    let mut line = String::new();

    loop {
        if shutdown.load(Ordering::SeqCst) {
            let _ = tx.send(String::from("\n서버가 종료됩니다. 안녕히 가세요!"));
            thread::sleep(Duration::from_millis(100)); // 메시지 전송 시간
            break;
        }

        line.clear();
        match reader.read_line(&mut line) {
            Ok(0) => break, // 연결 끊김
            Ok(_) => {
                let msg = line.trim();
                if !msg.is_empty() {
                    let formatted = format!("[{}]: {}", client_id, msg);
                    broadcast_message(&clients, client_id, formatted);
                }
            }
            Err(ref e) if e.kind() == std::io::ErrorKind::WouldBlock => {
                thread::sleep(Duration::from_millis(50));
            }
            Err(_) => break,
        }
    }

    // 정리
    broadcast_message(&clients, client_id, 
        format!("<<< [{}]님이 퇴장하셨습니다", client_id));

    {
        let mut client_list = clients.lock().unwrap();
        client_list.retain(|(id, _)| *id != client_id);
    }

    write_handle.join().ok();
    println!("[시스템] 클라이언트 {} 핸들러 종료", client_id);
}

fn broadcast_message(clients: &ClientList, sender_id: usize, message: String) {
    let client_list = clients.lock().unwrap();
    for (id, tx) in client_list.iter() {
        if *id != sender_id {
            let _ = tx.send(message.clone());
        }
    }
}

fn main() {
    let mut server = Server::new();

    // Ctrl+C 핸들러
    let shutdown_flag = Arc::clone(&server.shutdown);
    ctrlc::set_handler(move || {
        println!("\n\n╔════════════════════════════════════╗");
        println!("║   Ctrl+C 감지: 종료 시작...       ║");
        println!("╚════════════════════════════════════╝");
        shutdown_flag.store(true, Ordering::SeqCst);
    }).expect("Ctrl+C 핸들러 설정 실패");

    // 서버 실행
    if let Err(e) = server.run() {
        eprintln!("서버 에러: {}", e);
        return;
    }

    // 우아한 종료
    server.shutdown_all_clients();
    server.wait_for_all_threads();

    println!("╔════════════════════════════════════╗");
    println!("║   서버가 안전하게 종료되었습니다   ║");
    println!("╚════════════════════════════════════╝");
}
```

## 소유권과 우아한 종료

우아한 종료 과정에서 Rust의 소유권 시스템이 어떻게 안전성을 보장하는지 살펴보자.

스레드 핸들 `JoinHandle<T>`는 소유권을 가진다. `join()`을 호출하면 핸들을 소비(consume)한다. 즉, 한 번만 호출할 수 있다.

```rust
let handle = thread::spawn(|| { /* ... */ });
handle.join().unwrap(); // 소유권 이동
// handle.join(); // 컴파일 에러! handle은 이미 소비됨
```

이것은 실수로 같은 스레드를 두 번 join하는 것을 방지한다. 벡터에서 핸들을 꺼낼 때도 `drain()`이나 `into_iter()`를 사용해서 소유권을 이동시킨다.

```rust
for handle in self.client_handles.drain(..) {
    handle.join().unwrap(); // 각 핸들의 소유권을 가져와서 join
}
// 이후 client_handles는 비어있음
```

채널의 송신자 `Sender`도 비슷하다. 송신자가 모두 드롭되면 수신자의 `recv()`는 에러를 반환한다. 이것은 자동으로 종료 신호 역할을 한다.

```rust
// 클라이언트가 목록에서 제거되면 Sender가 드롭됨
{
    let mut client_list = clients.lock().unwrap();
    client_list.retain(|(id, _)| *id != client_id);
} // 여기서 제거된 (id, Sender) 튜플이 드롭됨

// 쓰기 스레드에서
for message in rx { // 모든 Sender가 드롭되면 이 루프 종료
    // ...
}
```

이런 자동 정리 메커니즘 덕분에 명시적으로 종료 신호를 보내지 않아도 리소스가 깔끔하게 정리된다.

## 정리

오늘 우리는 실용적인 서버에 필수적인 우아한 종료를 구현했다. Ctrl+C 시그널을 처리하고, 모든 클라이언트에게 알리고, 진행 중인 작업을 완료하고, 스레드를 정리하는 전체 과정을 배웠다.

`AtomicBool`을 사용해서 여러 스레드에 종료 신호를 전파하는 패턴을 익혔다. 논블로킹 I/O를 활용해서 블로킹 호출 중에도 주기적으로 종료 플래그를 확인할 수 있었다. `JoinHandle`을 저장해서 모든 스레드가 완전히 종료될 때까지 기다리는 방법도 배웠다.

`Drop` trait을 통해 Rust의 RAII 패턴을 이해했다. 리소스를 자동으로 정리하는 타입을 만들어서 누수나 불완전한 정리를 방지할 수 있다. 소유권 시스템과 결합하면 컴파일 타임에 리소스 관리의 정확성을 보장받는다.

시그널 핸들러에서는 최소한의 작업만 수행해야 한다는 것도 배웠다. 복잡한 로직은 메인 스레드로 위임하고, 핸들러는 단순히 플래그를 설정하거나 채널로 신호를 보내는 정도만 한다.

이제 우리는 동기 멀티스레드 네트워크 프로그래밍의 기초를 완성했다. Arc, Mutex, 채널, 스레드, 시그널 처리 등 실무에서 필요한 모든 도구를 다뤘다. 내일부터는 비동기 프로그래밍으로 넘어가서 훨씬 더 많은 동시 연결을 효율적으로 처리하는 방법을 배울 것이다. `async/await` 문법과 `tokio` 런타임을 사용하면 스레드 기반 코드보다 적은 리소스로 더 높은 성능을 달성할 수 있다.
