Rust 여행의 첫걸음을 내딛는 순간이다. 오늘은 개발 환경을 구축하고 첫 프로그램을 작성한다. 문법을 외우는 대신 실제로 동작하는 코드를 만들면서 Rust의 개발 방식을 체험한다. 1시간이라는 시간 동안 Rust 설치부터 간단한 계산기 프로그램까지 완성할 것이다.
  

## Rust 설치와 Cargo 소개

### Rust 설치하기
Rust는 rustup이라는 도구를 통해 설치한다. rustup은 Rust의 버전 관리자이자 설치 도구로, 여러 버전의 Rust를 관리하고 업데이트를 쉽게 해준다.

**Linux 또는 macOS:**  
터미널을 열고 다음 명령을 실행한다:

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

**Windows:**  
https://rustup.rs 에서 rustup-init.exe를 다운로드하고 실행한다.

설치 과정에서 기본 설정을 선택하면 된다. 설치가 완료되면 터미널을 다시 시작한다.

설치 확인을 위해 다음 명령을 입력한다:

```bash
rustc --version
```

다음과 같은 출력이 나타난다:

```
rustc 1.75.0 (82e1608df 2023-12-21)
```

버전 번호는 다를 수 있지만 정상적으로 출력되면 설치가 완료된 것이다.  
  
  
### Cargo: Rust의 빌드 도구이자 패키지 매니저
Cargo는 rustup과 함께 자동으로 설치되는 Rust의 핵심 도구다. Cargo는 다음과 같은 역할을 수행한다:

- 프로젝트 생성 및 관리
- 코드 빌드 및 컴파일
- 의존성(라이브러리) 관리
- 테스트 실행
- 문서 생성

Cargo가 제대로 설치되었는지 확인한다:

```bash
cargo --version
```

```
cargo 1.75.0 (1d8b05cdd 2023-11-20)
```

Cargo는 Rust 개발의 중심이다. 앞으로 모든 프로젝트는 Cargo를 통해 관리할 것이다.
  

## "Hello, World!"와 Cargo 프로젝트 구조 이해

### 첫 프로젝트 생성하기
작업할 디렉토리로 이동한 뒤, 첫 Rust 프로젝트를 생성한다:

```bash
cargo new hello_rust
cd hello_rust
```

`cargo new` 명령은 새로운 Rust 프로젝트를 생성한다. `hello_rust`라는 이름의 디렉토리가 만들어지고 그 안에 필요한 파일들이 자동으로 생성된다.

### 프로젝트 구조 살펴보기

생성된 프로젝트의 구조는 다음과 같다:

```
hello_rust/
├── Cargo.toml
├── .git/
├── .gitignore
└── src/
    └── main.rs
```

이를 시각적으로 표현하면:

```
┌─ hello_rust ─────────────────────┐
│                                   │
│  ┌─ Cargo.toml ─────────────┐   │
│  │ 프로젝트 설정 파일        │   │
│  │ (의존성, 버전 등)         │   │
│  └──────────────────────────┘   │
│                                   │
│  ┌─ src/ ──────────────────┐    │
│  │  ┌─ main.rs ─────────┐  │    │
│  │  │ 실제 코드가       │  │    │
│  │  │ 위치하는 곳       │  │    │
│  │  └──────────────────┘  │    │
│  └─────────────────────────┘    │
│                                   │
│  ┌─ .git/ ──────────────────┐   │
│  │ Git 저장소               │   │
│  └──────────────────────────┘   │
└───────────────────────────────────┘
```

**Cargo.toml** 파일을 살펴보자:

```toml
[package]
name = "hello_rust"
version = "0.1.0"
edition = "2021"

[dependencies]
```

이 파일은 프로젝트의 메타데이터를 담고 있다. `[package]` 섹션은 프로젝트 이름, 버전, 사용할 Rust 에디션을 정의한다. `[dependencies]` 섹션은 나중에 외부 라이브러리를 추가할 때 사용한다.

**src/main.rs** 파일을 열어보자:

```rust
fn main() {
    println!("Hello, world!");
}
```

이것이 Rust의 가장 기본적인 프로그램이다. `main` 함수가 프로그램의 시작점이다. `println!`은 매크로로, 텍스트를 화면에 출력한다. 느낌표(!)는 이것이 함수가 아닌 매크로임을 나타낸다.
  

### 프로그램 실행하기
프로그램을 빌드하고 실행해보자:

```bash
cargo run
```

처음 실행하면 컴파일 과정이 진행되고 다음과 같은 출력이 나타난다:

```
   Compiling hello_rust v0.1.0 (/path/to/hello_rust)
    Finished dev [unoptimized + debuginfo] target(s) in 1.23s
     Running `target/debug/hello_rust`
Hello, world!
```
  
`cargo run` 명령은 코드를 컴파일하고 실행까지 한 번에 수행한다. 코드가 변경되지 않았다면 다음 실행 시 컴파일을 건너뛴다.

컴파일만 하고 싶다면:

```bash
cargo build
```

개발 모드로 빌드된 실행 파일은 `target/debug/` 디렉토리에 생성된다.
  

## 간단한 계산기 만들기
이제 실제로 의미 있는 프로그램을 만들어보자. 사용자로부터 두 숫자를 입력받아 사칙연산을 수행하는 간단한 계산기를 작성한다.

### 사용자 입력 받기

`src/main.rs` 파일을 다음과 같이 수정한다:

```rust
use std::io;

fn main() {
    println!("=== 간단한 계산기 ===");
    
    // 첫 번째 숫자 입력
    println!("첫 번째 숫자를 입력하세요:");
    let mut first_number = String::new();
    io::stdin()
        .read_line(&mut first_number)
        .expect("입력을 읽는 데 실패했습니다");
    
    // 두 번째 숫자 입력
    println!("두 번째 숫자를 입력하세요:");
    let mut second_number = String::new();
    io::stdin()
        .read_line(&mut second_number)
        .expect("입력을 읽는 데 실패했습니다");
    
    println!("입력받은 값: {} 와 {}", first_number, second_number);
}
```
  
코드를 하나씩 살펴보자.

**use std::io;**
표준 라이브러리의 입출력 모듈을 가져온다. 이를 통해 `io::stdin()`을 사용할 수 있다.

**let mut first_number = String::new();**
새로운 변수를 선언한다. `let`은 변수 선언 키워드고, `mut`는 이 변수가 변경 가능(mutable)함을 의미한다. Rust의 변수는 기본적으로 불변(immutable)이므로 값을 변경하려면 `mut`를 명시해야 한다. `String::new()`는 빈 문자열을 생성한다.

**io::stdin().read_line(&mut first_number)**
표준 입력(키보드)에서 한 줄을 읽어 `first_number`에 저장한다. `&mut`는 변경 가능한 참조를 의미한다.

**expect("입력을 읽는 데 실패했습니다")**
`read_line`은 성공 또는 실패를 나타내는 `Result` 타입을 반환한다. `expect`는 에러가 발생하면 프로그램을 종료하고 메시지를 출력한다.

프로그램을 실행해보자:

```bash
cargo run
```
  
```
=== 간단한 계산기 ===
첫 번째 숫자를 입력하세요:
42
두 번째 숫자를 입력하세요:
58
입력받은 값: 42
 와 58
```

입력받은 값이 출력되지만 줄바꿈이 포함되어 있다. 이를 수정하고 실제 계산을 수행하도록 개선해보자.
   

### 문자열을 숫자로 변환하고 계산하기
코드를 다음과 같이 수정한다:

```rust
use std::io;

fn main() {
    println!("=== 간단한 계산기 ===");
    
    // 첫 번째 숫자 입력
    println!("첫 번째 숫자를 입력하세요:");
    let mut first_input = String::new();
    io::stdin()
        .read_line(&mut first_input)
        .expect("입력을 읽는 데 실패했습니다");
    
    // 문자열을 숫자로 변환
    let first_number: f64 = first_input
        .trim()
        .parse()
        .expect("숫자를 파싱하는 데 실패했습니다");
    
    // 두 번째 숫자 입력
    println!("두 번째 숫자를 입력하세요:");
    let mut second_input = String::new();
    io::stdin()
        .read_line(&mut second_input)
        .expect("입력을 읽는 데 실패했습니다");
    
    // 문자열을 숫자로 변환
    let second_number: f64 = second_input
        .trim()
        .parse()
        .expect("숫자를 파싱하는 데 실패했습니다");
    
    // 계산 수행
    println!("\n결과:");
    println!("{} + {} = {}", first_number, second_number, first_number + second_number);
    println!("{} - {} = {}", first_number, second_number, first_number - second_number);
    println!("{} × {} = {}", first_number, second_number, first_number * second_number);
    println!("{} ÷ {} = {}", first_number, second_number, first_number / second_number);
}
```

새로 추가된 부분을 살펴보자.

**let first_number: f64**
`f64`는 64비트 부동소수점 숫자 타입이다. 타입을 명시적으로 지정했다.

**first_input.trim()**
문자열 앞뒤의 공백과 줄바꿈 문자를 제거한다.

**parse()**
문자열을 지정된 타입(여기서는 `f64`)으로 변환한다. 이 메서드도 `Result` 타입을 반환하므로 `expect`로 처리한다.

데이터 변환 과정을 다이어그램으로 표현하면:

```
사용자 입력 "42\n"
       ↓
   [read_line]
       ↓
String "42\n"
       ↓
    [trim]
       ↓
   String "42"
       ↓
    [parse]
       ↓
    f64: 42.0
```

프로그램을 실행해보자:

```bash
cargo run
```

```
=== 간단한 계산기 ===
첫 번째 숫자를 입력하세요:
15.5
두 번째 숫자를 입력하세요:
4.5

결과:
15.5 + 4.5 = 20
15.5 - 4.5 = 11
15.5 × 4.5 = 69.75
15.5 ÷ 4.5 = 3.4444444444444446
```

완벽하게 동작한다!

### 연산자 선택 기능 추가하기
계산기를 더욱 발전시켜 사용자가 원하는 연산을 선택할 수 있도록 개선한다:

```rust
use std::io;

fn main() {
    println!("=== 간단한 계산기 ===");
    
    // 첫 번째 숫자 입력
    println!("첫 번째 숫자를 입력하세요:");
    let mut first_input = String::new();
    io::stdin()
        .read_line(&mut first_input)
        .expect("입력을 읽는 데 실패했습니다");
    
    let first_number: f64 = first_input
        .trim()
        .parse()
        .expect("숫자를 파싱하는 데 실패했습니다");
    
    // 연산자 입력
    println!("연산자를 입력하세요 (+, -, *, /):");
    let mut operator = String::new();
    io::stdin()
        .read_line(&mut operator)
        .expect("입력을 읽는 데 실패했습니다");
    
    let operator = operator.trim();
    
    // 두 번째 숫자 입력
    println!("두 번째 숫자를 입력하세요:");
    let mut second_input = String::new();
    io::stdin()
        .read_line(&mut second_input)
        .expect("입력을 읽는 데 실패했습니다");
    
    let second_number: f64 = second_input
        .trim()
        .parse()
        .expect("숫자를 파싱하는 데 실패했습니다");
    
    // 연산 수행
    let result = if operator == "+" {
        first_number + second_number
    } else if operator == "-" {
        first_number - second_number
    } else if operator == "*" {
        first_number * second_number
    } else if operator == "/" {
        first_number / second_number
    } else {
        println!("알 수 없는 연산자입니다!");
        return;
    };
    
    println!("\n결과: {} {} {} = {}", first_number, operator, second_number, result);
}
```

새로운 개념을 살펴보자.

**if-else 표현식**
Rust에서 `if`는 표현식(expression)이다. 즉, 값을 반환한다. 각 분기의 마지막 표현식이 반환값이 되며, 세미콜론을 붙이지 않는다.

**return 키워드**
함수를 조기에 종료한다. 잘못된 연산자가 입력되면 프로그램을 종료한다.

실행 결과:

```bash
cargo run
```

```
=== 간단한 계산기 ===
첫 번째 숫자를 입력하세요:
100
연산자를 입력하세요 (+, -, *, /):
*
두 번째 숫자를 입력하세요:
2.5

결과: 100 * 2.5 = 250
```

## 컴파일러 메시지 읽는 법
Rust 컴파일러는 매우 친절하고 상세한 에러 메시지를 제공한다. 이것이 Rust 학습의 핵심 도구다. 의도적으로 에러를 만들어보며 컴파일러 메시지를 읽는 법을 익혀보자.

### 예제 1: 불변 변수 수정 시도
코드를 다음과 같이 수정해보자:

```rust
fn main() {
    let x = 5;
    println!("x의 값: {}", x);
    x = 10;
    println!("x의 새 값: {}", x);
}
```

`cargo run`을 실행하면:

```
error[E0384]: cannot assign twice to immutable variable `x`
 --> src/main.rs:4:5
  |
2 |     let x = 5;
  |         -
  |         |
  |         first assignment to `x`
  |         help: consider making this binding mutable: `mut x`
3 |     println!("x의 값: {}", x);
4 |     x = 10;
  |     ^^^^^^ cannot assign twice to immutable variable

For more information about this error, try `rustc --explain E0384`.
error: could not compile `hello_rust` (bin "hello_rust") due to 1 previous error
```

컴파일러 메시지를 해석해보자:

```
┌─ 에러 메시지 구조 ──────────────────────────────┐
│                                               │
│  error[E0384]: 에러 코드와 간단한 설명           │
│                                               │
│  --> src/main.rs:4:5                          │
│      파일 위치와 줄:컬럼 번호                    │
│                                               │
│  코드 조각과 화살표로 문제 위치 표시               │
│                                               │
│  help: 해결 방법 제안                           │
│                                               │
└───────────────────────────────────────────────┘
```

**에러 코드**: `E0384`는 불변 변수에 두 번 할당을 시도했다는 의미다.

**위치 정보**: `src/main.rs:4:5`는 4번째 줄, 5번째 컬럼을 가리킨다.

**해결 제안**: `mut x`로 변경하라고 친절하게 알려준다.

수정된 코드:

```rust
fn main() {
    let mut x = 5;
    println!("x의 값: {}", x);
    x = 10;
    println!("x의 새 값: {}", x);
}
```

이제 정상적으로 컴파일된다.

### 예제 2: 타입 불일치
다음 코드를 시도해보자:

```rust
fn main() {
    let number: i32 = "42";
    println!("{}", number);
}
```

컴파일러 출력:

```
error[E0308]: mismatched types
 --> src/main.rs:2:23
  |
2 |     let number: i32 = "42";
  |                 ---   ^^^^ expected `i32`, found `&str`
  |                 |
  |                 expected due to this

For more information about this error, try `rustc --explain E0308`.
```

**에러 코드**: `E0308`은 타입 불일치를 의미한다.

**설명**: `i32` 타입을 기대했지만 `&str`(문자열 슬라이스)가 왔다고 명확히 알려준다.

올바른 코드:

```rust
fn main() {
    let number: i32 = "42".parse().expect("숫자 파싱 실패");
    println!("{}", number);
}
```

### 예제 3: 더 자세한 정보 얻기
에러 메시지에 나온 `rustc --explain` 명령을 사용하면 더 상세한 설명을 볼 수 있다:

```bash
rustc --explain E0384
```

이 명령은 해당 에러에 대한 긴 문서와 예제를 제공한다. 처음 보는 에러를 만났을 때 매우 유용하다.

### 컴파일러 메시지 읽기 팁
Rust 컴파일러와 대화하는 방법:

1. **당황하지 않기**: 에러 메시지가 길어도 차근차근 읽는다.

2. **위치 확인**: 먼저 파일과 줄 번호를 확인한다.

3. **설명 읽기**: 무엇이 잘못되었는지 영어로 설명되어 있다.

4. **제안 따르기**: `help:` 부분의 제안을 먼저 시도해본다.

5. **explain 활용**: 이해가 안 되면 `rustc --explain` 명령을 사용한다.

컴파일러를 친구로 생각하자. Rust 컴파일러는 버그를 실행 전에 잡아주는 강력한 동료다.
