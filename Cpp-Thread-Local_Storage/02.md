# 2. Win32 API TLS

## 2.1 TLS 함수들 (TlsAlloc, TlsSetValue, TlsGetValue, TlsFree)
Win32 API의 Thread Local Storage(TLS)는 Windows에서 스레드별 독립적인 데이터를 관리하는 가장 기초적이면서도 강력한 메커니즘입니다. 네 개의 핵심 함수로 구성되어 있습니다.

### TLS의 작동 원리
TLS는 각 프로세스가 최대 1088개의 TLS 슬롯을 가질 수 있으며, 각 스레드는 이 슬롯들에 독립적인 값을 저장할 수 있습니다. 마치 모든 스레드가 공유하는 큰 배열이 있지만, 각 스레드에서 접근할 때는 자신의 값만 보이는 것과 같습니다.

```
프로세스 메모리 구조:

┌─────────────────────────────────────────────────────────┐
│                    공유 메모리                           │
│  (TLS 슬롯 메타데이터, 글로벌 변수 등)                 │
└─────────────────────────────────────────────────────────┘

TLS 슬롯 0    TLS 슬롯 1    TLS 슬롯 2
    ↓             ↓             ↓
┌─────────────────────────────────────────────────────────┐
│  스레드 1의 TLS 데이터                                  │
│  [값1]        [값2]        [값3]                        │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  스레드 2의 TLS 데이터                                  │
│  [값1']       [값2']       [값3']                       │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  스레드 3의 TLS 데이터                                  │
│  [값1'']      [값2'']      [값3'']                      │
└─────────────────────────────────────────────────────────┘
```

### 핵심 함수 설명

**DWORD TlsAlloc(void)**  
새로운 TLS 슬롯을 할당합니다. 성공 시 슬롯 인덱스(0~1087)를 반환하고, 실패 시 TLS_OUT_OF_INDEXES를 반환합니다. 일반적으로 프로그램 시작 시 **한 번만** 호출하며, 반환된 인덱스를 전역변수로 저장해 모든 스레드가 공유해서 사용합니다.

각 스레드는 최대 1088개(인덱스 0~1087)의 독립적인 저장 슬롯을 가질 수 있습니다. 단, 슬롯을 실제로 사용하려면 반드시 `TlsAlloc()`으로 해당 인덱스를 먼저 예약해야 하며, 프로세스 전체에서 `TlsAlloc()` 호출 횟수의 합산이 1088을 초과하면 더 이상 새로운 슬롯을 예약할 수 없습니다.

> ⚠️ **주의:** `TlsAlloc()`은 스레드마다 호출하는 것이 아닙니다. 스레드마다 호출하면 인덱스를 낭비하여 1088개 한도를 빠르게 소진하게 됩니다.

```cpp
// ✅ 올바른 사용법: 프로그램 시작 시 한 번만 호출, 전역변수로 보관
DWORD tlsIndex = TlsAlloc();
if (tlsIndex == TLS_OUT_OF_INDEXES) {
    // 에러 처리
    return FALSE;
}

// ❌ 잘못된 사용법: 스레드마다 TlsAlloc() 호출
// Thread A: DWORD tlsIndex = TlsAlloc(); // 인덱스 0 소비
// Thread B: DWORD tlsIndex = TlsAlloc(); // 인덱스 1 소비 ← 낭비!
```

**BOOL TlsSetValue(DWORD dwTlsIndex, LPVOID lpTlsValue)**  
현재 스레드의 특정 TLS 슬롯에 값을 저장합니다. 인덱스 번호가 같더라도 스레드가 다르면 완전히 다른 메모리 공간에 저장되므로, 어떤 스레드가 저장한 값도 다른 스레드에서는 보이지 않습니다. `TlsSetValue()`는 내부적으로 항상 "현재 이 함수를 호출하고 있는 스레드"의 공간에만 접근하도록 OS가 보장합니다.

```
tlsIndex = 3 (전역 공유, 모든 스레드가 동일한 인덱스를 사용)

┌─────────────┬──────────────────────────────────────┐
│             │         TLS 슬롯 테이블               │
│             │  [0]  [1]  [2]  [3]  [4] ...         │
├─────────────┼──────────────────────────────────────┤
│  Thread A   │   -    -    -   42    -  ...          │
├─────────────┼──────────────────────────────────────┤
│  Thread B   │   -    -    -   99    -  ...          │
├─────────────┼──────────────────────────────────────┤
│  Thread C   │   -    -    -  NULL   -  ...          │
└─────────────┴──────────────────────────────────────┘
번호는 같지만 각 스레드의 슬롯은 물리적으로 완전히 다른 메모리
```

```cpp
// Thread A에서 실행
TlsSetValue(tlsIndex, new int(42));  // Thread A 슬롯에 42 저장

// Thread B에서 실행 (동일한 tlsIndex 사용)
TlsSetValue(tlsIndex, new int(99));  // Thread B 슬롯에 99 저장
                                     // Thread A의 42는 덮어써지지 않음

LPVOID myData = new int(42);
BOOL success = TlsSetValue(tlsIndex, myData);
```

**LPVOID TlsGetValue(DWORD dwTlsIndex)**  
현재 스레드의 특정 TLS 슬롯 값을 읽습니다. 동일한 인덱스로 호출하더라도 항상 현재 스레드의 슬롯에서만 값을 읽어오므로, 다른 스레드의 값과 절대 혼용되지 않습니다. 반환 값이 NULL일 수 있으므로, 실제 오류 여부를 확인하려면 `GetLastError()`를 호출해야 합니다.

```cpp
// Thread A에서 실행
LPVOID data = TlsGetValue(tlsIndex); // Thread A의 슬롯 값만 반환
if (data == NULL) {
    DWORD error = GetLastError();
    if (error != ERROR_SUCCESS) {
        // 실제 오류 발생
    }
}
```

**BOOL TlsFree(DWORD dwTlsIndex)**  
할당한 TLS 슬롯을 해제합니다. 프로그램 종료 시 호출합니다. 주의할 점은 이 함수를 호출한다고 해서 각 스레드에 저장된 데이터가 자동으로 정리되지 않는다는 것입니다. 따라서 `TlsFree()` 호출 전에 각 스레드에서 저장한 데이터를 직접 해제해야 메모리 누수를 방지할 수 있습니다.

```cpp
// TlsFree 전에 각 스레드에서 저장한 데이터를 먼저 해제
LPVOID data = TlsGetValue(tlsIndex);
if (data != NULL) {
    delete data; // 각 스레드에서 직접 해제
}

TlsFree(tlsIndex); // 슬롯 인덱스 반환
```
  

## 2.2 기본 사용 예제

### 예제 1: 간단한 스레드별 상태 저장

```cpp
#include <windows.h>
#include <stdio.h>
#include <iostream>

// TLS에 저장할 데이터 구조
struct ThreadData {
    int threadId;
    int requestCount;
    char buffer[256];
};

DWORD g_tlsIndex;

DWORD WINAPI WorkerThread(LPVOID param) {
    int threadNum = (int)(intptr_t)param;
    
    // 이 스레드용 TLS 데이터 생성
    ThreadData* pData = new ThreadData();
    pData->threadId = GetCurrentThreadId();
    pData->requestCount = 0;
    sprintf_s(pData->buffer, sizeof(pData->buffer), 
              "Thread %d data", threadNum);
    
    // TLS 슬롯에 저장
    if (!TlsSetValue(g_tlsIndex, (LPVOID)pData)) {
        printf("TlsSetValue failed\n");
        delete pData;
        return 1;
    }
    
    // 작업 수행
    for (int i = 0; i < 5; i++) {
        // TLS에서 데이터 읽기
        ThreadData* pLocalData = (ThreadData*)TlsGetValue(g_tlsIndex);
        if (pLocalData) {
            pLocalData->requestCount++;
            printf("[Thread %d] Request %d: %s\n", 
                   threadNum, 
                   pLocalData->requestCount,
                   pLocalData->buffer);
        }
        Sleep(100);
    }
    
    // 정리
    ThreadData* pCleanup = (ThreadData*)TlsGetValue(g_tlsIndex);
    if (pCleanup) {
        printf("[Thread %d] Processed %d requests total\n", 
               threadNum, pCleanup->requestCount);
        delete pCleanup;
    }
    
    return 0;
}

int main() {
    // TLS 슬롯 할당
    g_tlsIndex = TlsAlloc();
    if (g_tlsIndex == TLS_OUT_OF_INDEXES) {
        printf("TlsAlloc failed\n");
        return 1;
    }
    
    // 스레드 생성
    HANDLE threads[3];
    for (int i = 0; i < 3; i++) {
        threads[i] = CreateThread(NULL, 0, WorkerThread, 
                                 (LPVOID)(intptr_t)i, 0, NULL);
    }
    
    // 모든 스레드 종료 대기
    WaitForMultipleObjects(3, threads, TRUE, INFINITE);
    
    // 정리
    for (int i = 0; i < 3; i++) {
        CloseHandle(threads[i]);
    }
    
    TlsFree(g_tlsIndex);
    
    printf("All threads completed\n");
    return 0;
}
```

### 예제 2: 에러 코드 저장 (errno 스타일)
Win32 TLS의 실제 사용 사례로, 스레드별 에러 상태를 관리하는 패턴입니다.

```cpp
#include <windows.h>
#include <stdio.h>

DWORD g_tlsErrorIndex;

// 스레드별 에러 정보
struct ThreadError {
    DWORD errorCode;
    char errorMsg[256];
};

void SetThreadError(DWORD errorCode, const char* message) {
    ThreadError* pError = (ThreadError*)TlsGetValue(g_tlsErrorIndex);
    
    // 첫 사용 시 초기화
    if (pError == NULL) {
        pError = new ThreadError();
        TlsSetValue(g_tlsErrorIndex, pError);
    }
    
    pError->errorCode = errorCode;
    strcpy_s(pError->errorMsg, sizeof(pError->errorMsg), message);
}

DWORD GetThreadError(char* pBuffer, size_t bufferSize) {
    ThreadError* pError = (ThreadError*)TlsGetValue(g_tlsErrorIndex);
    
    if (pError == NULL) {
        return ERROR_SUCCESS;
    }
    
    if (pBuffer && bufferSize > 0) {
        strcpy_s(pBuffer, bufferSize, pError->errorMsg);
    }
    
    return pError->errorCode;
}

void ClearThreadError() {
    ThreadError* pError = (ThreadError*)TlsGetValue(g_tlsErrorIndex);
    if (pError) {
        pError->errorCode = ERROR_SUCCESS;
        pError->errorMsg[0] = '\0';
    }
}

DWORD WINAPI ProcessingThread(LPVOID param) {
    int operationNum = 0;
    
    for (int i = 0; i < 3; i++) {
        operationNum++;
        
        // 성공 사례
        if (i == 0) {
            ClearThreadError();
            printf("[Thread %d] Operation %d: Success\n", 
                   GetCurrentThreadId(), operationNum);
        }
        // 실패 사례
        else if (i == 1) {
            SetThreadError(ERROR_ACCESS_DENIED, 
                          "Access denied to resource");
            char errMsg[256];
            DWORD errCode = GetThreadError(errMsg, sizeof(errMsg));
            printf("[Thread %d] Operation %d: Error 0x%X - %s\n", 
                   GetCurrentThreadId(), operationNum, errCode, errMsg);
        }
        // 또 다른 실패 사례
        else {
            SetThreadError(ERROR_NOT_FOUND, 
                          "Resource not found");
            char errMsg[256];
            DWORD errCode = GetThreadError(errMsg, sizeof(errMsg));
            printf("[Thread %d] Operation %d: Error 0x%X - %s\n", 
                   GetCurrentThreadId(), operationNum, errCode, errMsg);
        }
        
        Sleep(200);
    }
    
    return 0;
}

int main() {
    g_tlsErrorIndex = TlsAlloc();
    if (g_tlsErrorIndex == TLS_OUT_OF_INDEXES) {
        printf("Failed to allocate TLS index\n");
        return 1;
    }
    
    HANDLE threads[2];
    for (int i = 0; i < 2; i++) {
        threads[i] = CreateThread(NULL, 0, ProcessingThread, NULL, 0, NULL);
    }
    
    WaitForMultipleObjects(2, threads, TRUE, INFINITE);
    
    for (int i = 0; i < 2; i++) {
        CloseHandle(threads[i]);
    }
    
    TlsFree(g_tlsErrorIndex);
    return 0;
}
```
  

## 2.3 한계점과 주의사항

### 1. 슬롯 수 제한
프로세스당 최대 1088개의 슬롯만 할당 가능합니다. 이는 대규모 애플리케이션에서 제약이 될 수 있습니다.

```
최대 슬롯 수: 1088개

많은 라이브러리가 각각 TLS를 사용하는 경우:
- Windows 런타임: 여러 개
- C 런타임 라이브러리: 여러 개
- OpenSSL: 여러 개
- 사용자 코드: 여러 개

→ TLS_OUT_OF_INDEXES 에러 발생 가능성
```

### 2. 자동 정리 없음
TlsFree()는 슬롯만 해제하며, 각 스레드에 저장된 데이터는 수동으로 정리해야 합니다. 스레드가 종료될 때 자동으로 메모리가 해제되지 않습니다.

```cpp
// 잘못된 코드 - 메모리 누수 발생
DWORD WINAPI BadThread(LPVOID param) {
    ThreadData* pData = new ThreadData();
    TlsSetValue(g_tlsIndex, pData);
    // pData를 delete하지 않고 종료
    return 0;  // 메모리 누수!
}
```

### 3. NULL 값의 모호성
TlsGetValue()가 NULL을 반환한다고 해서 반드시 오류가 아닙니다. 실제로 NULL을 저장한 것일 수도 있습니다.

```cpp
// NULL 확인 방법
LPVOID data = TlsGetValue(tlsIndex);
DWORD lastError = GetLastError();

if (data == NULL && lastError != ERROR_SUCCESS) {
    // 실제 오류
} else if (data == NULL && lastError == ERROR_SUCCESS) {
    // NULL이 실제 저장 값
}
```

### 4. 동적 라이브러리에서의 문제
DLL이 TLS를 사용할 때 DLL_THREAD_ATTACH/DETACH 메시지를 처리해야 합니다.

```cpp
BOOL WINAPI DllMain(HINSTANCE hinstDLL, DWORD fdwReason, LPVOID lpvReserved) {
    switch (fdwReason) {
        case DLL_PROCESS_ATTACH:
            // 프로세스 초기화
            g_tlsIndex = TlsAlloc();
            break;
            
        case DLL_THREAD_ATTACH:
            // 새 스레드가 생성될 때마다 호출
            // 각 스레드별 초기화
            break;
            
        case DLL_THREAD_DETACH:
            // 스레드 종료 시 정리
            {
                void* pData = TlsGetValue(g_tlsIndex);
                if (pData) delete[] pData;
            }
            break;
            
        case DLL_PROCESS_DETACH:
            // 프로세스 종료
            TlsFree(g_tlsIndex);
            break;
    }
    return TRUE;
}
```

### 5. 성능 특성
TLS 접근은 매우 빠르지만(대략 메모리 접근과 유사한 수준), 대량의 작은 할당을 관리할 때는 오버헤드가 존재합니다.

```
TLS 접근 성능 (대략):
- TlsGetValue(): ~100-200 사이클
- TlsSetValue(): ~200-300 사이클
- 일반 메모리 접근: ~1-3 사이클

→ 매우 자주 호출되는 경로에서는 결과를 캐시할 것
```

### 6. 초기화 시점의 함정
TLS 슬롯 자체는 NULL로 초기화되므로, 첫 접근 시 명시적으로 초기화해야 합니다.

```cpp
// 각 스레드에서 이런 패턴을 사용해야 함
void* data = TlsGetValue(tlsIndex);
if (data == NULL && GetLastError() == ERROR_SUCCESS) {
    // 첫 접근 - 초기화 필요
    data = new MyData();
    TlsSetValue(tlsIndex, data);
}
```

이러한 한계점들이 C++11의 `thread_local` 키워드 도입의 동기가 되었으며, 다음 섹션에서 더 나은 대안을 살펴보겠습니다.
