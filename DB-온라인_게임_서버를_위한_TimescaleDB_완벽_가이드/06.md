# Chapter 6: 서버 성능 메트릭 수집 시스템

게임 서버를 운영하다 보면 가장 먼저 마주치는 질문이 있다. "지금 서버가 정상인가?" 이 단순해 보이는 질문에 답하기 위해서는 서버의 상태를 지속적으로 관찰하고 기록해야 한다. CPU 사용률이 갑자기 치솟았는가? 메모리 누수가 발생하고 있는가? 네트워크 대역폭이 포화 상태인가? 이런 질문들에 즉각적으로 답할 수 있어야 장애를 예방하고, 문제가 발생했을 때 원인을 빠르게 파악할 수 있다.

이번 챕터에서는 게임 서버의 핵심 성능 지표들을 실시간으로 수집하여 TimescaleDB에 저장하는 시스템을 직접 구축한다. C#의 성능 카운터를 사용해 Windows 시스템의 메트릭을 수집하고, SQLKata를 통해 효율적으로 데이터를 저장하는 방법을 배운다. 단순히 데이터를 저장하는 것을 넘어서, 배치 삽입과 트랜잭션 처리를 통해 초당 수천 건의 메트릭을 처리할 수 있는 고성능 시스템을 만들어본다.

## 6.1 메트릭 데이터 모델 설계

메트릭 데이터를 저장하기 전에 먼저 어떤 구조로 데이터를 설계할지 결정해야 한다. 게임 서버의 성능 메트릭은 전형적인 시계열 데이터다. 특정 시점에 측정된 값들이 시간 순서대로 쌓이고, 나중에 시간 범위를 지정해서 조회하거나 집계하는 패턴이 반복된다.

먼저 수집할 메트릭의 종류를 정의한다. 일반적인 게임 서버에서 모니터링해야 할 핵심 지표는 다음과 같다.

**시스템 리소스 메트릭**은 서버의 하드웨어 자원 사용 현황을 나타낸다. CPU 사용률은 프로세서가 얼마나 바쁜지를 백분율로 표현하며, 메모리 사용량은 물리 메모리와 가용 메모리를 바이트 단위로 추적한다. 디스크 사용량은 읽기/쓰기 속도와 큐 길이를 측정하고, 네트워크 사용량은 초당 송수신 바이트 수를 기록한다.

**애플리케이션 메트릭**은 게임 서버 프로세스 자체의 상태를 나타낸다. 활성 연결 수는 현재 접속 중인 플레이어 수를, 요청 처리 시간은 각 게임 액션의 응답 속도를, 에러 발생 횟수는 시스템의 안정성을 나타낸다.

이제 이 데이터를 저장할 테이블을 설계한다. pgAdmin을 열고 TimescaleDB에 접속한 후 다음 SQL을 실행한다.

```sql
-- 서버 메트릭 저장 테이블 생성
CREATE TABLE server_metrics (
    time TIMESTAMPTZ NOT NULL,
    server_id VARCHAR(50) NOT NULL,
    metric_name VARCHAR(100) NOT NULL,
    metric_value DOUBLE PRECISION NOT NULL,
    unit VARCHAR(20),
    tags JSONB,
    PRIMARY KEY (time, server_id, metric_name)
);

-- Hypertable로 변환
SELECT create_hypertable('server_metrics', 'time');

-- 자주 사용하는 조회 패턴을 위한 인덱스
CREATE INDEX idx_server_metrics_server_id_time 
ON server_metrics (server_id, time DESC);

CREATE INDEX idx_server_metrics_name_time 
ON server_metrics (metric_name, time DESC);
```

이 설계의 핵심은 **넓은 형식(wide format)** 대신 **좁은 형식(narrow format)**을 사용한다는 점이다. CPU, 메모리, 네트워크를 각각의 컬럼으로 만드는 대신, `metric_name`과 `metric_value`라는 범용적인 컬럼에 모든 메트릭을 저장한다. 이렇게 하면 나중에 새로운 메트릭을 추가할 때 테이블 구조를 변경할 필요가 없다.

`tags` 컬럼은 JSONB 타입으로 설정했는데, 이는 메트릭에 대한 추가 메타데이터를 유연하게 저장하기 위함이다. 예를 들어 CPU 메트릭이라면 어떤 코어인지, 네트워크 메트릭이라면 어떤 네트워크 인터페이스인지 등의 정보를 저장할 수 있다.

```json
{
  "core": "0",
  "processor": "Intel Xeon",
  "zone": "ap-northeast-2"
}
```

C# 코드에서 이 테이블 구조를 표현하는 모델 클래스를 작성한다.

```csharp
using System;
using Newtonsoft.Json;

namespace GameServerMonitoring.Models
{
    /// <summary>
    /// 서버 성능 메트릭을 표현하는 모델
    /// </summary>
    public class ServerMetric
    {
        /// <summary>
        /// 메트릭이 수집된 시각 (UTC)
        /// </summary>
        public DateTime Time { get; set; }

        /// <summary>
        /// 서버 식별자 (예: "game-server-01")
        /// </summary>
        public string ServerId { get; set; }

        /// <summary>
        /// 메트릭 이름 (예: "cpu_usage", "memory_used")
        /// </summary>
        public string MetricName { get; set; }

        /// <summary>
        /// 메트릭 값
        /// </summary>
        public double MetricValue { get; set; }

        /// <summary>
        /// 측정 단위 (예: "percent", "bytes", "ms")
        /// </summary>
        public string Unit { get; set; }

        /// <summary>
        /// 추가 태그 정보 (JSONB로 저장됨)
        /// </summary>
        public Dictionary<string, string> Tags { get; set; }

        public ServerMetric()
        {
            Tags = new Dictionary<string, string>();
            Time = DateTime.UtcNow;
        }
    }
}
```

## 6.2 C# 성능 카운터로 시스템 메트릭 수집

Windows에서 시스템 성능 정보를 수집하려면 `System.Diagnostics.PerformanceCounter` 클래스를 사용한다. 이 클래스는 Windows Performance Monitor와 동일한 데이터 소스에서 메트릭을 읽어온다.

먼저 NuGet 패키지를 설치한다. Visual Studio에서 프로젝트를 열고 패키지 관리자 콘솔에 다음을 입력한다.

```powershell
Install-Package System.Diagnostics.PerformanceCounter
```

성능 카운터를 관리하는 클래스를 작성한다.

```csharp
using System;
using System.Collections.Generic;
using System.Diagnostics;
using System.Linq;

namespace GameServerMonitoring.Collectors
{
    /// <summary>
    /// Windows 시스템 메트릭을 수집하는 클래스
    /// </summary>
    public class SystemMetricsCollector : IDisposable
    {
        private readonly string _serverId;
        private readonly Dictionary<string, PerformanceCounter> _counters;

        public SystemMetricsCollector(string serverId)
        {
            _serverId = serverId;
            _counters = new Dictionary<string, PerformanceCounter>();
            
            InitializeCounters();
        }

        /// <summary>
        /// 성능 카운터를 초기화한다
        /// </summary>
        private void InitializeCounters()
        {
            // CPU 사용률 카운터
            _counters["cpu_usage"] = new PerformanceCounter(
                "Processor", 
                "% Processor Time", 
                "_Total"
            );

            // 사용 가능한 메모리 (MB)
            _counters["memory_available"] = new PerformanceCounter(
                "Memory", 
                "Available MBytes"
            );

            // 커밋된 메모리 비율
            _counters["memory_committed_percent"] = new PerformanceCounter(
                "Memory", 
                "% Committed Bytes In Use"
            );

            // 디스크 읽기 속도 (bytes/sec)
            _counters["disk_read_bytes"] = new PerformanceCounter(
                "PhysicalDisk", 
                "Disk Read Bytes/sec", 
                "_Total"
            );

            // 디스크 쓰기 속도 (bytes/sec)
            _counters["disk_write_bytes"] = new PerformanceCounter(
                "PhysicalDisk", 
                "Disk Write Bytes/sec", 
                "_Total"
            );

            // 네트워크 송신 속도 (bytes/sec)
            // 주의: 네트워크 인터페이스 이름은 환경에 따라 다를 수 있음
            try
            {
                var category = new PerformanceCounterCategory("Network Interface");
                var instanceNames = category.GetInstanceNames();
                
                // 첫 번째 활성 네트워크 인터페이스 사용
                if (instanceNames.Length > 0)
                {
                    var interfaceName = instanceNames[0];
                    
                    _counters["network_sent_bytes"] = new PerformanceCounter(
                        "Network Interface", 
                        "Bytes Sent/sec", 
                        interfaceName
                    );

                    _counters["network_received_bytes"] = new PerformanceCounter(
                        "Network Interface", 
                        "Bytes Received/sec", 
                        interfaceName
                    );
                }
            }
            catch (Exception ex)
            {
                Console.WriteLine($"네트워크 카운터 초기화 실패: {ex.Message}");
            }

            // 첫 번째 샘플을 위한 대기
            // PerformanceCounter는 두 번의 샘플이 필요하므로 초기화 시 한 번 읽어둠
            foreach (var counter in _counters.Values)
            {
                counter.NextValue();
            }
        }

        /// <summary>
        /// 현재 시스템 메트릭을 수집한다
        /// </summary>
        /// <returns>수집된 메트릭 목록</returns>
        public List<ServerMetric> CollectMetrics()
        {
            var metrics = new List<ServerMetric>();
            var timestamp = DateTime.UtcNow;

            foreach (var kvp in _counters)
            {
                try
                {
                    var value = kvp.Value.NextValue();
                    var metricName = kvp.Key;
                    
                    // 단위 결정
                    string unit = DetermineUnit(metricName);

                    // 메트릭 생성
                    var metric = new ServerMetric
                    {
                        Time = timestamp,
                        ServerId = _serverId,
                        MetricName = metricName,
                        MetricValue = value,
                        Unit = unit,
                        Tags = new Dictionary<string, string>
                        {
                            { "collector", "system" },
                            { "os", "windows" }
                        }
                    };

                    metrics.Add(metric);
                }
                catch (Exception ex)
                {
                    Console.WriteLine($"메트릭 수집 실패 ({kvp.Key}): {ex.Message}");
                }
            }

            return metrics;
        }

        /// <summary>
        /// 메트릭 이름으로부터 측정 단위를 결정한다
        /// </summary>
        private string DetermineUnit(string metricName)
        {
            if (metricName.Contains("usage") || metricName.Contains("percent"))
                return "percent";
            if (metricName.Contains("bytes"))
                return "bytes_per_sec";
            if (metricName.Contains("memory_available"))
                return "megabytes";
            
            return "count";
        }

        public void Dispose()
        {
            foreach (var counter in _counters.Values)
            {
                counter?.Dispose();
            }
            _counters.Clear();
        }
    }
}
```

이 코드는 Windows의 주요 성능 지표를 수집한다. `PerformanceCounter`는 Windows가 내부적으로 관리하는 성능 데이터에 접근하는 표준 방법이다. 각 카운터는 **카테고리(Category)**, **카운터 이름(Counter Name)**, **인스턴스(Instance)**의 세 가지 요소로 구성된다.

예를 들어 CPU 사용률을 수집하려면 다음과 같이 지정한다.

$$\text{카테고리: "Processor", 카운터: "\% Processor Time", 인스턴스: "\_Total"}$$

`_Total` 인스턴스는 모든 CPU 코어의 평균 사용률을 의미한다. 개별 코어의 사용률을 측정하려면 인스턴스를 `"0"`, `"1"` 등으로 지정하면 된다.

성능 카운터를 사용할 때 주의할 점이 있다. `NextValue()` 메서드는 두 번의 샘플 간의 차이를 계산하여 값을 반환한다. 따라서 처음 호출할 때는 항상 0이 반환된다. 이를 해결하기 위해 `InitializeCounters()` 메서드에서 모든 카운터를 한 번씩 읽어두었다.

## 6.3 배치 삽입으로 성능 최적화

메트릭을 수집하는 것은 시작일 뿐이다. 이제 이 데이터를 TimescaleDB에 저장해야 한다. 만약 메트릭을 하나씩 INSERT한다면 성능이 매우 느려진다. 초당 10개의 메트릭을 수집하는 서버가 100대라면 초당 1,000번의 INSERT 쿼리가 발생한다. 이는 데이터베이스에 엄청난 부하를 주고 네트워크 오버헤드도 크다.

해결책은 **배치 삽입(Batch Insert)**이다. 여러 개의 메트릭을 모아서 한 번의 쿼리로 삽입하면 성능이 극적으로 향상된다.

SQLKata는 기본적으로 배치 삽입을 지원하지 않으므로, Npgsql의 `COPY` 기능을 활용하거나 다중 행 INSERT를 구성해야 한다. 여기서는 Npgsql의 바이너리 COPY를 사용하는 방법을 보여준다.

```csharp
using System;
using System.Collections.Generic;
using System.Threading.Tasks;
using Npgsql;
using Newtonsoft.Json;

namespace GameServerMonitoring.Storage
{
    /// <summary>
    /// TimescaleDB에 메트릭을 저장하는 리포지토리
    /// </summary>
    public class MetricsRepository
    {
        private readonly string _connectionString;

        public MetricsRepository(string connectionString)
        {
            _connectionString = connectionString;
        }

        /// <summary>
        /// 메트릭 목록을 배치로 삽입한다
        /// </summary>
        public async Task InsertMetricsBatchAsync(List<ServerMetric> metrics)
        {
            if (metrics == null || metrics.Count == 0)
                return;

            await using var connection = new NpgsqlConnection(_connectionString);
            await connection.OpenAsync();

            // COPY 명령을 사용한 고속 삽입
            var copyCommand = @"
                COPY server_metrics (time, server_id, metric_name, metric_value, unit, tags) 
                FROM STDIN (FORMAT BINARY)";

            await using (var writer = await connection.BeginBinaryImportAsync(copyCommand))
            {
                foreach (var metric in metrics)
                {
                    await writer.StartRowAsync();
                    await writer.WriteAsync(metric.Time, NpgsqlTypes.NpgsqlDbType.TimestampTz);
                    await writer.WriteAsync(metric.ServerId, NpgsqlTypes.NpgsqlDbType.Varchar);
                    await writer.WriteAsync(metric.MetricName, NpgsqlTypes.NpgsqlDbType.Varchar);
                    await writer.WriteAsync(metric.MetricValue, NpgsqlTypes.NpgsqlDbType.Double);
                    await writer.WriteAsync(metric.Unit, NpgsqlTypes.NpgsqlDbType.Varchar);
                    
                    // Dictionary를 JSON으로 직렬화
                    var tagsJson = JsonConvert.SerializeObject(metric.Tags);
                    await writer.WriteAsync(tagsJson, NpgsqlTypes.NpgsqlDbType.Jsonb);
                }

                await writer.CompleteAsync();
            }

            Console.WriteLine($"✓ {metrics.Count}개 메트릭 삽입 완료");
        }

        /// <summary>
        /// 특정 서버의 최근 메트릭을 조회한다
        /// </summary>
        public async Task<List<ServerMetric>> GetRecentMetricsAsync(
            string serverId, 
            string metricName, 
            TimeSpan timeRange)
        {
            var metrics = new List<ServerMetric>();
            var startTime = DateTime.UtcNow - timeRange;

            await using var connection = new NpgsqlConnection(_connectionString);
            await connection.OpenAsync();

            var query = @"
                SELECT time, server_id, metric_name, metric_value, unit, tags
                FROM server_metrics
                WHERE server_id = @serverId
                  AND metric_name = @metricName
                  AND time >= @startTime
                ORDER BY time DESC
                LIMIT 1000";

            await using var command = new NpgsqlCommand(query, connection);
            command.Parameters.AddWithValue("serverId", serverId);
            command.Parameters.AddWithValue("metricName", metricName);
            command.Parameters.AddWithValue("startTime", startTime);

            await using var reader = await command.ExecuteReaderAsync();
            while (await reader.ReadAsync())
            {
                var metric = new ServerMetric
                {
                    Time = reader.GetDateTime(0),
                    ServerId = reader.GetString(1),
                    MetricName = reader.GetString(2),
                    MetricValue = reader.GetDouble(3),
                    Unit = reader.GetString(4),
                    Tags = JsonConvert.DeserializeObject<Dictionary<string, string>>(
                        reader.GetString(5))
                };
                metrics.Add(metric);
            }

            return metrics;
        }
    }
}
```

Npgsql의 `BeginBinaryImportAsync` 메서드는 PostgreSQL의 `COPY` 프로토콜을 사용하여 대량의 데이터를 매우 빠르게 삽입한다. 이 방법은 일반 INSERT보다 10배에서 100배까지 빠를 수 있다. 데이터가 바이너리 형식으로 전송되므로 파싱 오버헤드도 최소화된다.

배치 크기는 상황에 따라 조정해야 한다. 너무 작으면 배치의 이점이 줄어들고, 너무 크면 메모리 사용량이 증가하고 오류 발생 시 재시도가 어려워진다. 일반적으로 100~1,000개 정도가 적절하다.

## 6.4 실시간 CPU/메모리 사용률 저장

이제 수집기와 저장소를 연결하여 실제로 메트릭을 수집하고 저장하는 애플리케이션을 만든다. 백그라운드에서 주기적으로 메트릭을 수집하는 서비스를 구현한다.

```csharp
using System;
using System.Collections.Generic;
using System.Threading;
using System.Threading.Tasks;
using Microsoft.Extensions.Hosting;
using Microsoft.Extensions.Logging;

namespace GameServerMonitoring.Services
{
    /// <summary>
    /// 백그라운드에서 메트릭을 주기적으로 수집하는 서비스
    /// </summary>
    public class MetricsCollectionService : BackgroundService
    {
        private readonly ILogger<MetricsCollectionService> _logger;
        private readonly MetricsRepository _repository;
        private readonly SystemMetricsCollector _collector;
        private readonly TimeSpan _collectionInterval;
        private readonly List<ServerMetric> _buffer;
        private readonly int _batchSize;

        public MetricsCollectionService(
            ILogger<MetricsCollectionService> logger,
            MetricsRepository repository,
            string serverId,
            TimeSpan collectionInterval,
            int batchSize = 100)
        {
            _logger = logger;
            _repository = repository;
            _collector = new SystemMetricsCollector(serverId);
            _collectionInterval = collectionInterval;
            _batchSize = batchSize;
            _buffer = new List<ServerMetric>();
        }

        protected override async Task ExecuteAsync(CancellationToken stoppingToken)
        {
            _logger.LogInformation("메트릭 수집 서비스 시작");

            while (!stoppingToken.IsCancellationRequested)
            {
                try
                {
                    // 메트릭 수집
                    var metrics = _collector.CollectMetrics();
                    _buffer.AddRange(metrics);

                    _logger.LogDebug($"메트릭 {metrics.Count}개 수집 (버퍼: {_buffer.Count})");

                    // 배치 크기에 도달하면 저장
                    if (_buffer.Count >= _batchSize)
                    {
                        await FlushBufferAsync();
                    }

                    // 지정된 간격만큼 대기
                    await Task.Delay(_collectionInterval, stoppingToken);
                }
                catch (Exception ex)
                {
                    _logger.LogError(ex, "메트릭 수집 중 오류 발생");
                }
            }

            // 종료 시 남은 버퍼 저장
            if (_buffer.Count > 0)
            {
                await FlushBufferAsync();
            }

            _collector.Dispose();
            _logger.LogInformation("메트릭 수집 서비스 종료");
        }

        /// <summary>
        /// 버퍼의 메트릭을 데이터베이스에 저장한다
        /// </summary>
        private async Task FlushBufferAsync()
        {
            try
            {
                var metricsToSave = new List<ServerMetric>(_buffer);
                _buffer.Clear();

                await _repository.InsertMetricsBatchAsync(metricsToSave);
                
                _logger.LogInformation($"✓ {metricsToSave.Count}개 메트릭 저장 완료");
            }
            catch (Exception ex)
            {
                _logger.LogError(ex, "메트릭 저장 중 오류 발생");
                // 실패한 메트릭은 버퍼에 다시 추가할 수도 있지만,
                // 여기서는 단순히 로그만 남기고 계속 진행
            }
        }
    }
}
```

이 서비스는 ASP.NET Core의 `BackgroundService` 클래스를 상속받아 애플리케이션이 실행되는 동안 계속 백그라운드에서 작동한다. 메트릭을 수집할 때마다 버퍼에 추가하고, 버퍼가 일정 크기에 도달하면 한 번에 저장한다.

## 6.5 서버별 메트릭 조회 및 비교

메트릭을 저장했으니 이제 조회하고 분석할 차례다. SQLKata를 사용하여 복잡한 조회 쿼리를 작성한다. 특정 서버의 CPU 사용률 추이를 조회하거나, 여러 서버를 비교하는 쿼리를 만들어본다.

```csharp
using System;
using System.Collections.Generic;
using System.Linq;
using System.Threading.Tasks;
using SqlKata;
using SqlKata.Execution;
using Npgsql;

namespace GameServerMonitoring.Queries
{
    /// <summary>
    /// 메트릭 조회를 위한 쿼리 빌더
    /// </summary>
    public class MetricsQueryService
    {
        private readonly QueryFactory _db;

        public MetricsQueryService(string connectionString)
        {
            var connection = new NpgsqlConnection(connectionString);
            var compiler = new SqlKata.Compilers.PostgresCompiler();
            _db = new QueryFactory(connection, compiler);
        }

        /// <summary>
        /// 특정 서버의 최근 메트릭을 조회한다
        /// </summary>
        public async Task<IEnumerable<dynamic>> GetRecentMetricsAsync(
            string serverId,
            string metricName,
            int hours = 1)
        {
            var startTime = DateTime.UtcNow.AddHours(-hours);

            var query = new Query("server_metrics")
                .Select("time", "metric_value", "unit")
                .Where("server_id", serverId)
                .Where("metric_name", metricName)
                .Where("time", ">=", startTime)
                .OrderByDesc("time");

            return await _db.GetAsync(query);
        }

        /// <summary>
        /// 시간 범위별 평균 메트릭을 계산한다
        /// </summary>
        public async Task<IEnumerable<dynamic>> GetAverageMetricsByTimeAsync(
            string serverId,
            string metricName,
            string interval = "5 minutes",
            int hours = 24)
        {
            var startTime = DateTime.UtcNow.AddHours(-hours);

            // SQLKata에서 지원하지 않는 time_bucket 함수는 RawQuery 사용
            var sql = $@"
                SELECT 
                    time_bucket('{interval}', time) AS bucket,
                    AVG(metric_value) AS avg_value,
                    MIN(metric_value) AS min_value,
                    MAX(metric_value) AS max_value,
                    COUNT(*) AS sample_count
                FROM server_metrics
                WHERE server_id = @serverId
                  AND metric_name = @metricName
                  AND time >= @startTime
                GROUP BY bucket
                ORDER BY bucket DESC";

            return await _db.SelectAsync(sql, new
            {
                serverId,
                metricName,
                startTime
            });
        }

        /// <summary>
        /// 여러 서버의 현재 메트릭을 비교한다
        /// </summary>
        public async Task<IEnumerable<dynamic>> CompareServersCurrentMetricsAsync(
            List<string> serverIds,
            string metricName)
        {
            // 각 서버의 최근 5분 평균값 조회
            var fiveMinutesAgo = DateTime.UtcNow.AddMinutes(-5);

            var query = new Query("server_metrics")
                .Select("server_id")
                .SelectRaw("AVG(metric_value) AS avg_value")
                .SelectRaw("MAX(metric_value) AS max_value")
                .SelectRaw("MIN(metric_value) AS min_value")
                .WhereIn("server_id", serverIds)
                .Where("metric_name", metricName)
                .Where("time", ">=", fiveMinutesAgo)
                .GroupBy("server_id")
                .OrderBy("server_id");

            return await _db.GetAsync(query);
        }

        /// <summary>
        /// 메트릭 값이 임계값을 초과하는 시점을 찾는다
        /// </summary>
        public async Task<IEnumerable<dynamic>> FindThresholdViolationsAsync(
            string serverId,
            string metricName,
            double threshold,
            int hours = 24)
        {
            var startTime = DateTime.UtcNow.AddHours(-hours);

            var query = new Query("server_metrics")
                .Select("time", "metric_value")
                .Where("server_id", serverId)
                .Where("metric_name", metricName)
                .Where("metric_value", ">", threshold)
                .Where("time", ">=", startTime)
                .OrderByDesc("time");

            return await _db.GetAsync(query);
        }
    }
}
```

## 6.6 이상 징후 감지 쿼리 작성

메트릭을 수집하고 조회하는 것만으로는 충분하지 않다. 실제 운영 환경에서는 문제가 발생하기 전에 미리 감지하는 것이 중요하다. 이상 징후를 자동으로 탐지하는 쿼리를 작성한다.

이상 징후 감지에는 여러 가지 접근 방법이 있다. 가장 단순한 방법은 **정적 임계값(Static Threshold)**을 설정하는 것이다. 예를 들어 CPU 사용률이 90%를 초과하면 알림을 보낸다. 하지만 이 방법은 서버의 정상적인 부하 패턴을 고려하지 못한다.

더 정교한 방법은 **통계적 이상 탐지(Statistical Anomaly Detection)**다. 과거 데이터의 평균과 표준편차를 계산하고, 현재 값이 평균에서 몇 표준편차 이상 벗어났는지 확인한다.

$$\text{Z-Score} = \frac{\text{현재값} - \text{평균}}{\text{표준편차}}$$

Z-Score의 절대값이 3을 초과하면 이상 징후로 판단할 수 있다.

```csharp
using System;
using System.Collections.Generic;
using System.Threading.Tasks;
using SqlKata.Execution;
using Npgsql;

namespace GameServerMonitoring.Anomaly
{
    /// <summary>
    /// 이상 징후 탐지를 위한 쿼리 서비스
    /// </summary>
    public class AnomalyDetectionService
    {
        private readonly QueryFactory _db;

        public AnomalyDetectionService(string connectionString)
        {
            var connection = new NpgsqlConnection(connectionString);
            var compiler = new SqlKata.Compilers.PostgresCompiler();
            _db = new QueryFactory(connection, compiler);
        }

        /// <summary>
        /// 갑작스러운 메트릭 증가를 감지한다
        /// </summary>
        public async Task<IEnumerable<dynamic>> DetectSuddenSpikeAsync(
            string serverId,
            string metricName,
            double spikeThreshold = 2.0, // 2배 증가
            int minutes = 5)
        {
            // 현재 값과 5분 전 값을 비교하여 급증 감지
            var sql = @"
                WITH current_metrics AS (
                    SELECT 
                        time_bucket('1 minute', time) AS bucket,
                        AVG(metric_value) AS current_value
                    FROM server_metrics
                    WHERE server_id = @serverId
                      AND metric_name = @metricName
                      AND time >= NOW() - INTERVAL '1 minute'
                    GROUP BY bucket
                ),
                previous_metrics AS (
                    SELECT 
                        AVG(metric_value) AS baseline_value
                    FROM server_metrics
                    WHERE server_id = @serverId
                      AND metric_name = @metricName
                      AND time BETWEEN NOW() - INTERVAL '10 minutes' 
                                   AND NOW() - INTERVAL '5 minutes'
                )
                SELECT 
                    c.bucket,
                    c.current_value,
                    p.baseline_value,
                    (c.current_value / NULLIF(p.baseline_value, 0)) AS spike_ratio
                FROM current_metrics c
                CROSS JOIN previous_metrics p
                WHERE (c.current_value / NULLIF(p.baseline_value, 0)) > @spikeThreshold";

            return await _db.SelectAsync(sql, new
            {
                serverId,
                metricName,
                spikeThreshold
            });
        }

        /// <summary>
        /// 통계적 이상값을 감지한다 (Z-Score 기반)
        /// </summary>
        public async Task<IEnumerable<dynamic>> DetectStatisticalAnomaliesAsync(
            string serverId,
            string metricName,
            double zScoreThreshold = 3.0,
            int baselineHours = 24)
        {
            var sql = @"
                WITH baseline_stats AS (
                    SELECT 
                        AVG(metric_value) AS mean_value,
                        STDDEV(metric_value) AS stddev_value
                    FROM server_metrics
                    WHERE server_id = @serverId
                      AND metric_name = @metricName
                      AND time >= NOW() - INTERVAL '@baselineHours hours'
                ),
                recent_metrics AS (
                    SELECT 
                        time,
                        metric_value
                    FROM server_metrics
                    WHERE server_id = @serverId
                      AND metric_name = @metricName
                      AND time >= NOW() - INTERVAL '10 minutes'
                )
                SELECT 
                    r.time,
                    r.metric_value,
                    s.mean_value,
                    s.stddev_value,
                    ABS((r.metric_value - s.mean_value) / NULLIF(s.stddev_value, 0)) AS z_score
                FROM recent_metrics r
                CROSS JOIN baseline_stats s
                WHERE ABS((r.metric_value - s.mean_value) / NULLIF(s.stddev_value, 0)) > @zScoreThreshold
                ORDER BY r.time DESC";

            return await _db.SelectAsync(sql, new
            {
                serverId,
                metricName,
                baselineHours,
                zScoreThreshold
            });
        }

        /// <summary>
        /// 여러 서버에서 동시에 발생하는 문제를 감지한다
        /// </summary>
        public async Task<IEnumerable<dynamic>> DetectCrossServerAnomaliesAsync(
            string metricName,
            double threshold,
            int affectedServerThreshold = 3)
        {
            var sql = @"
                WITH recent_violations AS (
                    SELECT 
                        time_bucket('1 minute', time) AS bucket,
                        server_id,
                        AVG(metric_value) AS avg_value
                    FROM server_metrics
                    WHERE metric_name = @metricName
                      AND time >= NOW() - INTERVAL '5 minutes'
                    GROUP BY bucket, server_id
                    HAVING AVG(metric_value) > @threshold
                )
                SELECT 
                    bucket,
                    COUNT(DISTINCT server_id) AS affected_servers,
                    ARRAY_AGG(DISTINCT server_id) AS server_list,
                    AVG(avg_value) AS avg_metric_value
                FROM recent_violations
                GROUP BY bucket
                HAVING COUNT(DISTINCT server_id) >= @affectedServerThreshold
                ORDER BY bucket DESC";

            return await _db.SelectAsync(sql, new
            {
                metricName,
                threshold,
                affectedServerThreshold
            });
        }

        /// <summary>
        /// 메트릭의 변화율을 계산한다
        /// </summary>
        public async Task<IEnumerable<dynamic>> CalculateMetricChangeRateAsync(
            string serverId,
            string metricName,
            int minutes = 10)
        {
            var sql = @"
                WITH time_series AS (
                    SELECT 
                        time_bucket('1 minute', time) AS bucket,
                        AVG(metric_value) AS avg_value
                    FROM server_metrics
                    WHERE server_id = @serverId
                      AND metric_name = @metricName
                      AND time >= NOW() - INTERVAL '@minutes minutes'
                    GROUP BY bucket
                    ORDER BY bucket
                )
                SELECT 
                    bucket,
                    avg_value,
                    LAG(avg_value) OVER (ORDER BY bucket) AS prev_value,
                    avg_value - LAG(avg_value) OVER (ORDER BY bucket) AS absolute_change,
                    ((avg_value - LAG(avg_value) OVER (ORDER BY bucket)) / 
                     NULLIF(LAG(avg_value) OVER (ORDER BY bucket), 0) * 100) AS percent_change
                FROM time_series
                WHERE bucket >= NOW() - INTERVAL '5 minutes'";

            return await _db.SelectAsync(sql, new
            {
                serverId,
                metricName,
                minutes
            });
        }
    }
}
```

이제 이상 징후 감지를 실제로 실행하는 모니터링 에이전트를 만든다.

```csharp
using System;
using System.Linq;
using System.Threading;
using System.Threading.Tasks;
using Microsoft.Extensions.Logging;

namespace GameServerMonitoring.Monitoring
{
    /// <summary>
    /// 이상 징후를 지속적으로 모니터링하는 에이전트
    /// </summary>
    public class AnomalyMonitoringAgent
    {
        private readonly AnomalyDetectionService _detectionService;
        private readonly ILogger _logger;
        private readonly TimeSpan _checkInterval;

        public AnomalyMonitoringAgent(
            AnomalyDetectionService detectionService,
            ILogger logger,
            TimeSpan checkInterval)
        {
            _detectionService = detectionService;
            _logger = logger;
            _checkInterval = checkInterval;
        }

        public async Task RunAsync(CancellationToken cancellationToken)
        {
            _logger.LogInformation("이상 징후 모니터링 시작");

            while (!cancellationToken.IsCancellationRequested)
            {
                try
                {
                    await CheckAnomaliesAsync();
                    await Task.Delay(_checkInterval, cancellationToken);
                }
                catch (Exception ex)
                {
                    _logger.LogError(ex, "이상 징후 검사 중 오류 발생");
                }
            }
        }

        private async Task CheckAnomaliesAsync()
        {
            // 1. CPU 급증 감지
            var cpuSpikes = await _detectionService.DetectSuddenSpikeAsync(
                "GAME-SERVER-01",
                "cpu_usage",
                spikeThreshold: 2.0
            );

            foreach (var spike in cpuSpikes)
            {
                _logger.LogWarning(
                    $"⚠️  CPU 사용률 급증 감지: {spike.bucket} - " +
                    $"현재 {spike.current_value:F2}%, " +
                    $"기준 {spike.baseline_value:F2}%, " +
                    $"증가율 {spike.spike_ratio:F2}x"
                );
            }

            // 2. 통계적 이상값 감지
            var anomalies = await _detectionService.DetectStatisticalAnomaliesAsync(
                "GAME-SERVER-01",
                "cpu_usage",
                zScoreThreshold: 3.0
            );

            foreach (var anomaly in anomalies)
            {
                _logger.LogWarning(
                    $"⚠️  통계적 이상값 감지: {anomaly.time:HH:mm:ss} - " +
                    $"값 {anomaly.metric_value:F2}%, " +
                    $"Z-Score {anomaly.z_score:F2}"
                );
            }

            // 3. 다중 서버 동시 문제 감지
            var crossServerIssues = await _detectionService.DetectCrossServerAnomaliesAsync(
                "cpu_usage",
                threshold: 80.0,
                affectedServerThreshold: 3
            );

            foreach (var issue in crossServerIssues)
            {
                var servers = string.Join(", ", ((string[])issue.server_list));
                _logger.LogError(
                    $"🚨 다중 서버 이상: {issue.bucket} - " +
                    $"{issue.affected_servers}대 영향, " +
                    $"평균 {issue.avg_metric_value:F2}% " +
                    $"({servers})"
                );
            }
        }
    }
}
```

이 챕터에서 구축한 시스템은 게임 서버의 성능을 실시간으로 추적하고, 문제가 발생하기 전에 미리 경고할 수 있는 기반을 제공한다. 메트릭 수집부터 저장, 조회, 이상 탐지까지 전체 파이프라인을 구현했다.

실제 운영 환경에서는 이 시스템을 더욱 확장할 수 있다. Grafana와 연동하여 시각적인 대시보드를 만들거나, Slack이나 Discord로 알림을 보내거나, PagerDuty 같은 온콜 시스템과 통합할 수 있다. 하지만 그 모든 것의 기초는 바로 여기서 배운 메트릭 수집과 TimescaleDB 저장 시스템이다.
