# Chapter 9: 데이터 압축과 보관 정책

당신의 게임이 성공적으로 런칭한 지 3개월이 지났다. 서버 모니터링 시스템은 완벽하게 작동하고 있고, 플레이어 로그도 실시간으로 잘 쌓이고 있다. 그런데 어느 날 인프라 담당자로부터 연락이 온다. "데이터베이스 디스크 사용량이 급격히 증가하고 있습니다. 이대로라면 한 달 안에 스토리지를 증설해야 합니다." 

초당 수백 건씩 쌓이는 로그 데이터는 시간이 지날수록 기하급수적으로 늘어난다. 하지만 모든 데이터가 같은 중요도를 가지는 것은 아니다. 어제 발생한 서버 메트릭은 실시간으로 조회하고 분석해야 하지만, 6개월 전의 데이터는 가끔 통계 목적으로만 확인한다. 이러한 데이터의 생명주기를 이해하고 효율적으로 관리하는 것이 이번 챕터의 핵심이다.

TimescaleDB는 시간이 지난 데이터를 자동으로 압축하고, 필요 없는 데이터를 자동으로 삭제하는 강력한 기능을 제공한다. 이를 통해 스토리지 비용을 90% 이상 절감하면서도 쿼리 성능은 유지할 수 있다. 실제로 압축을 적용하면 수백 GB의 데이터가 수십 GB로 줄어드는 마법 같은 경험을 하게 된다.

## 9.1 데이터 압축의 원리와 이점

TimescaleDB의 압축은 단순히 파일을 zip으로 압축하는 것과는 완전히 다르다. 시계열 데이터의 특성을 활용한 컬럼 기반 압축 알고리즘을 사용하여 데이터를 재구성한다.

**압축 전후의 데이터 구조 비교**

```
압축 전 (행 기반 저장):
┌─────────────────────────────────────────────────────────┐
│ Row 1: [2024-01-01 00:00:00][server-1][CPU][45.2]      │
│ Row 2: [2024-01-01 00:00:01][server-1][CPU][45.3]      │
│ Row 3: [2024-01-01 00:00:02][server-1][CPU][45.1]      │
│ Row 4: [2024-01-01 00:00:03][server-1][CPU][45.4]      │
│ ... 수백만 개의 행 ...                                  │
└─────────────────────────────────────────────────────────┘

압축 후 (컬럼 기반 저장):
┌──────────────────────────────────────────┐
│ Timestamp Column (델타 인코딩):          │
│ [Base: 2024-01-01 00:00:00]             │
│ [+1초, +1초, +1초, +1초, ...]           │
├──────────────────────────────────────────┤
│ Server Column (딕셔너리 압축):          │
│ [Dictionary: 1=server-1]                │
│ [1, 1, 1, 1, ...]                       │
├──────────────────────────────────────────┤
│ Metric Column (딕셔너리 압축):          │
│ [Dictionary: 1=CPU]                     │
│ [1, 1, 1, 1, ...]                       │
├──────────────────────────────────────────┤
│ Value Column (델타 인코딩):             │
│ [Base: 45.2]                            │
│ [+0.1, -0.2, +0.3, ...]                 │
└──────────────────────────────────────────┘
```

시계열 데이터는 연속된 값들이 비슷한 경향을 보인다. CPU 사용률이 45.2%에서 45.3%로 변하는 것처럼 작은 변화만 있는 경우가 많다. 압축 알고리즘은 이러한 패턴을 감지하여 기준값과 차이만 저장하는 델타 인코딩을 사용한다. 또한 서버 이름처럼 반복되는 문자열은 딕셔너리로 변환하여 한 번만 저장하고 나머지는 숫자 참조로 대체한다.

**압축의 이점을 구체적인 수치로 살펴보자:**

게임 서버 10대에서 5초마다 10개의 메트릭을 수집한다고 가정하면 하루에 발생하는 로그는 다음과 같다.

$$\text{하루 로그 수} = 10 \text{대} \times \frac{86400 \text{초}}{5 \text{초}} \times 10 \text{메트릭} = 1,728,000 \text{건}$$

각 로그 레코드가 평균 200바이트라면 압축 전 데이터 크기는 다음과 같다.

$$\text{일일 데이터} = 1,728,000 \times 200 \text{바이트} = 345.6 \text{MB}$$

한 달이면 약 10GB, 1년이면 120GB가 쌓인다. 하지만 압축을 적용하면 평균 90% 압축률을 달성하여 연간 12GB로 줄일 수 있다.

압축된 데이터는 여전히 쿼리가 가능하다. TimescaleDB는 쿼리 실행 시 필요한 컬럼만 자동으로 압축 해제한다. 따라서 사용자는 압축 여부를 신경 쓰지 않고 동일한 SQL 쿼리를 사용할 수 있다.

**압축이 적합한 데이터와 그렇지 않은 데이터**

압축은 모든 데이터에 적합한 것은 아니다. 최근 데이터는 계속 INSERT와 UPDATE가 발생하므로 압축하지 않는 것이 좋다. 일반적으로 7일~30일이 지난 데이터부터 압축을 시작한다.

압축된 Chunk는 읽기 전용이 된다. 따라서 압축 후에는 해당 시간 범위의 데이터를 수정하거나 삭제할 수 없다. 만약 수정이 필요하다면 먼저 압축을 해제하고 작업을 진행한 후 다시 압축해야 한다.

## 9.2 압축 정책 설정하기

압축 정책을 설정하는 것은 매우 간단하다. 어떤 Hypertable을 압축할지, 얼마나 오래된 데이터를 압축할지만 결정하면 나머지는 TimescaleDB가 자동으로 처리한다.

먼저 게임 서버 메트릭 테이블을 만들어보자. 이전 챕터에서 사용했던 테이블 구조를 그대로 활용한다.

```sql
-- 서버 메트릭 테이블 생성
CREATE TABLE server_metrics (
    time TIMESTAMPTZ NOT NULL,
    server_name TEXT NOT NULL,
    metric_name TEXT NOT NULL,
    value DOUBLE PRECISION,
    tags JSONB
);

-- Hypertable로 변환
SELECT create_hypertable('server_metrics', 'time');
```

이제 압축을 활성화한다. 압축 설정에서 가장 중요한 것은 어떤 컬럼을 기준으로 데이터를 묶을지 결정하는 것이다. 일반적으로 자주 함께 조회되는 컬럼들을 `segment_by`로 지정한다.

```sql
-- 압축 활성화 (server_name과 metric_name 기준으로 세그먼트 분할)
ALTER TABLE server_metrics SET (
    timescaledb.compress,
    timescaledb.compress_segmentby = 'server_name, metric_name',
    timescaledb.compress_orderby = 'time DESC'
);
```

`segment_by` 파라미터는 압축 시 데이터를 그룹화하는 기준이다. 같은 서버의 같은 메트릭은 하나의 압축 단위로 묶인다. 이렇게 하면 특정 서버의 특정 메트릭을 조회할 때 압축 해제 오버헤드가 최소화된다.

`compress_orderby`는 압축된 데이터 내부의 정렬 순서를 지정한다. 시간 역순으로 정렬하면 최근 데이터부터 빠르게 접근할 수 있다.

이제 자동 압축 정책을 추가한다. 7일이 지난 데이터는 자동으로 압축되도록 설정한다.

```sql
-- 7일 지난 데이터 자동 압축 정책
SELECT add_compression_policy('server_metrics', INTERVAL '7 days');
```

이 명령을 실행하면 TimescaleDB의 백그라운드 작업 스케줄러가 정기적으로 7일 이상 된 Chunk를 찾아서 압축한다. 기본적으로 하루에 한 번씩 실행되지만, 필요에 따라 주기를 조정할 수 있다.

**수동으로 특정 Chunk 압축하기**

자동 정책을 기다리지 않고 즉시 압축하고 싶다면 수동 압축을 실행할 수 있다.

```sql
-- 특정 시간 범위의 Chunk 수동 압축
SELECT compress_chunk(i) 
FROM show_chunks('server_metrics', older_than => INTERVAL '7 days') i;
```

이 쿼리는 7일 이상 된 모든 Chunk를 찾아서 즉시 압축한다. 대량의 데이터를 압축할 때는 시간이 오래 걸릴 수 있으므로 서비스 부하가 적은 시간대에 실행하는 것이 좋다.

**C#에서 압축 정책 관리하기**

SQLKata는 DDL 명령을 직접 지원하지 않으므로 Npgsql을 직접 사용하여 압축 정책을 관리한다.

```csharp
using Npgsql;
using System;
using System.Threading.Tasks;

public class CompressionManager
{
    private readonly string _connectionString;

    public CompressionManager(string connectionString)
    {
        _connectionString = connectionString;
    }

    // 압축 활성화 및 정책 설정
    public async Task EnableCompressionAsync(
        string tableName, 
        string[] segmentByColumns, 
        string orderByColumn = "time DESC",
        int compressAfterDays = 7)
    {
        await using var connection = new NpgsqlConnection(_connectionString);
        await connection.OpenAsync();

        // 압축 활성화
        var segmentBy = string.Join(", ", segmentByColumns);
        var alterTableSql = $@"
            ALTER TABLE {tableName} SET (
                timescaledb.compress,
                timescaledb.compress_segmentby = '{segmentBy}',
                timescaledb.compress_orderby = '{orderByColumn}'
            );";

        await using (var cmd = new NpgsqlCommand(alterTableSql, connection))
        {
            await cmd.ExecuteNonQueryAsync();
        }

        Console.WriteLine($"압축 설정 완료: {tableName}");

        // 자동 압축 정책 추가
        var policyJson = $@"
            SELECT add_compression_policy('{tableName}', INTERVAL '{compressAfterDays} days');";

        await using (var cmd = new NpgsqlCommand(policyJson, connection))
        {
            await cmd.ExecuteNonQueryAsync();
        }

        Console.WriteLine($"자동 압축 정책 추가 완료: {compressAfterDays}일 후 압축");
    }

    // 수동 압축 실행
    public async Task CompressOldChunksAsync(string tableName, int olderThanDays)
    {
        await using var connection = new NpgsqlConnection(_connectionString);
        await connection.OpenAsync();

        var compressSql = $@"
            SELECT compress_chunk(i, if_not_compressed => true) 
            FROM show_chunks('{tableName}', older_than => INTERVAL '{olderThanDays} days') i;";

        await using var cmd = new NpgsqlCommand(compressSql, connection);
        var compressed = await cmd.ExecuteScalarAsync();

        Console.WriteLine($"압축 완료: {compressed}개 Chunk");
    }

    // 압축 상태 조회
    public async Task<CompressionStats> GetCompressionStatsAsync(string tableName)
    {
        await using var connection = new NpgsqlConnection(_connectionString);
        await connection.OpenAsync();

        var statsSql = $@"
            SELECT 
                pg_size_pretty(before_compression_total_bytes) as before_size,
                pg_size_pretty(after_compression_total_bytes) as after_size,
                ROUND(100 - (after_compression_total_bytes::numeric / 
                      before_compression_total_bytes::numeric * 100), 2) as compression_ratio
            FROM timescaledb_information.compression_settings
            WHERE hypertable_name = '{tableName}';";

        await using var cmd = new NpgsqlCommand(statsSql, connection);
        await using var reader = await cmd.ExecuteReaderAsync();

        if (await reader.ReadAsync())
        {
            return new CompressionStats
            {
                BeforeSize = reader.GetString(0),
                AfterSize = reader.GetString(1),
                CompressionRatio = reader.GetDecimal(2)
            };
        }

        return null;
    }
}

public class CompressionStats
{
    public string BeforeSize { get; set; }
    public string AfterSize { get; set; }
    public decimal CompressionRatio { get; set; }

    public override string ToString()
    {
        return $"압축 전: {BeforeSize}, 압축 후: {AfterSize}, 압축률: {CompressionRatio}%";
    }
}
```

이 클래스를 사용하면 C# 코드에서 압축 관련 작업을 쉽게 수행할 수 있다.

```csharp
// 사용 예제
var compressionManager = new CompressionManager(
    "Host=localhost;Database=gamedb;Username=postgres;Password=yourpassword"
);

// 압축 활성화 및 정책 설정
await compressionManager.EnableCompressionAsync(
    tableName: "server_metrics",
    segmentByColumns: new[] { "server_name", "metric_name" },
    orderByColumn: "time DESC",
    compressAfterDays: 7
);

// 수동 압축 실행
await compressionManager.CompressOldChunksAsync("server_metrics", olderThanDays: 7);

// 압축 통계 확인
var stats = await compressionManager.GetCompressionStatsAsync("server_metrics");
Console.WriteLine(stats);
```

## 9.3 자동 데이터 삭제 정책 (Retention Policy)

압축으로 스토리지를 절약했지만, 언젠가는 오래된 데이터를 완전히 삭제해야 한다. 법적 보관 의무가 없거나 분석 가치가 없는 오래된 로그는 과감히 삭제하는 것이 좋다.

TimescaleDB의 Retention Policy는 지정한 기간이 지난 데이터를 자동으로 삭제한다. 예를 들어 게임 서버 메트릭은 6개월만 보관하고, 플레이어 로그인 기록은 1년간 보관하는 식으로 테이블마다 다른 정책을 적용할 수 있다.

**Retention Policy 설정하기**

```sql
-- 90일 지난 데이터 자동 삭제 정책
SELECT add_retention_policy('server_metrics', INTERVAL '90 days');
```

이 정책을 설정하면 90일이 지난 Chunk가 자동으로 삭제된다. 삭제는 Chunk 단위로 이루어지므로 매우 빠르다. 개별 행을 DELETE하는 것이 아니라 Chunk 전체를 DROP하기 때문이다.

**압축과 삭제 정책을 함께 사용하는 전략**

일반적으로 다음과 같은 3단계 데이터 생명주기를 설정한다.

```
┌─────────────┬──────────────┬─────────────┬──────────────┐
│  최근 7일   │  8일~30일    │ 31일~90일   │   90일 초과  │
├─────────────┼──────────────┼─────────────┼──────────────┤
│ 압축 안 함  │  자동 압축   │  압축 상태  │   자동 삭제  │
│ 빠른 쓰기   │  (90% 절감)  │  유지       │              │
│ 빠른 조회   │              │             │              │
└─────────────┴──────────────┴─────────────┴──────────────┘
```

최근 7일 데이터는 압축하지 않고 빠른 쓰기와 조회를 보장한다. 8일~30일 데이터는 압축하여 스토리지를 절약한다. 31일~90일 데이터는 압축된 상태로 장기 보관하며, 필요할 때만 조회한다. 90일이 지난 데이터는 자동으로 삭제되어 더 이상 스토리지를 차지하지 않는다.

이러한 정책을 코드로 구현하면 다음과 같다.

```sql
-- 1단계: 7일 후 압축
SELECT add_compression_policy('server_metrics', INTERVAL '7 days');

-- 2단계: 90일 후 삭제
SELECT add_retention_policy('server_metrics', INTERVAL '90 days');
```

**C#에서 Retention Policy 관리**

```csharp
public class RetentionManager
{
    private readonly string _connectionString;

    public RetentionManager(string connectionString)
    {
        _connectionString = connectionString;
    }

    // 자동 삭제 정책 추가
    public async Task AddRetentionPolicyAsync(string tableName, int retentionDays)
    {
        await using var connection = new NpgsqlConnection(_connectionString);
        await connection.OpenAsync();

        var policySql = $@"
            SELECT add_retention_policy('{tableName}', INTERVAL '{retentionDays} days');";

        await using var cmd = new NpgsqlCommand(policySql, connection);
        await cmd.ExecuteNonQueryAsync();

        Console.WriteLine($"데이터 보관 정책 설정 완료: {tableName} - {retentionDays}일");
    }

    // 수동으로 오래된 Chunk 삭제
    public async Task<int> DropOldChunksAsync(string tableName, int olderThanDays)
    {
        await using var connection = new NpgsqlConnection(_connectionString);
        await connection.OpenAsync();

        var dropSql = $@"
            SELECT drop_chunks('{tableName}', INTERVAL '{olderThanDays} days');";

        await using var cmd = new NpgsqlCommand(dropSql, connection);
        var result = await cmd.ExecuteScalarAsync();

        var droppedCount = result != null ? Convert.ToInt32(result) : 0;
        Console.WriteLine($"삭제된 Chunk: {droppedCount}개");

        return droppedCount;
    }

    // 현재 보관 정책 조회
    public async Task<List<RetentionPolicyInfo>> GetRetentionPoliciesAsync()
    {
        await using var connection = new NpgsqlConnection(_connectionString);
        await connection.OpenAsync();

        var querySql = @"
            SELECT 
                hypertable_name,
                older_than,
                schedule_interval
            FROM timescaledb_information.jobs
            WHERE proc_name = 'policy_retention';";

        await using var cmd = new NpgsqlCommand(querySql, connection);
        await using var reader = await cmd.ExecuteReaderAsync();

        var policies = new List<RetentionPolicyInfo>();
        while (await reader.ReadAsync())
        {
            policies.Add(new RetentionPolicyInfo
            {
                HypertableName = reader.GetString(0),
                OlderThan = reader.GetString(1),
                ScheduleInterval = reader.GetString(2)
            });
        }

        return policies;
    }

    // 테이블별 데이터 보관 현황 조회
    public async Task<DataRetentionStatus> GetRetentionStatusAsync(string tableName)
    {
        await using var connection = new NpgsqlConnection(_connectionString);
        await connection.OpenAsync();

        var statusSql = $@"
            SELECT 
                MIN(time) as oldest_data,
                MAX(time) as newest_data,
                COUNT(*) as total_rows,
                pg_size_pretty(pg_total_relation_size('{tableName}')) as total_size
            FROM {tableName};";

        await using var cmd = new NpgsqlCommand(statusSql, connection);
        await using var reader = await cmd.ExecuteReaderAsync();

        if (await reader.ReadAsync())
        {
            return new DataRetentionStatus
            {
                TableName = tableName,
                OldestData = reader.GetDateTime(0),
                NewestData = reader.GetDateTime(1),
                TotalRows = reader.GetInt64(2),
                TotalSize = reader.GetString(3)
```

## 9.4 콜드 스토리지 전략

완전히 삭제하기 전에 오래된 데이터를 저렴한 스토리지로 옮기는 것도 좋은 전략이다. 예를 들어 90일 이후 데이터는 AWS S3나 Azure Blob Storage 같은 콜드 스토리지로 아카이브하고, TimescaleDB에서는 삭제한다.

이렇게 하면 필요할 때 언제든지 복원할 수 있으면서도 데이터베이스 성능은 최적으로 유지할 수 있다.

**콜드 스토리지 아카이브 프로세스**

```
┌──────────────────────────────────────────────────────────┐
│                    데이터 생명주기                        │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  0~7일      : Hot Storage (압축 안 함)                  │
│              → TimescaleDB (빠른 쓰기/읽기)              │
│                                                          │
│  8~90일     : Warm Storage (압축됨)                     │
│              → TimescaleDB (압축, 가끔 조회)             │
│                                                          │
│  91~365일   : Cold Storage (아카이브)                   │
│              → S3/Blob Storage (CSV/Parquet)            │
│                                                          │
│  365일 이후 : 완전 삭제 또는 장기 백업                   │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

**C#으로 콜드 스토리지 아카이브 구현**

```csharp
using System.IO;
using System.Text;
using SqlKata;
using SqlKata.Execution;

public class ColdStorageArchiver
{
    private readonly QueryFactory _db;
    private readonly string _archivePath;

    public ColdStorageArchiver(QueryFactory db, string archivePath)
    {
        _db = db;
        _archivePath = archivePath;
    }

    // 오래된 데이터를 CSV로 추출하여 아카이브
    public async Task<ArchiveResult> ArchiveOldDataAsync(
        string tableName, 
        int olderThanDays,
        bool deleteAfterArchive = false)
    {
        var cutoffDate = DateTime.UtcNow.AddDays(-olderThanDays);
        var archiveFileName = $"{tableName}_{cutoffDate:yyyyMMdd}_archive.csv";
        var fullPath = Path.Combine(_archivePath, archiveFileName);

        // 아카이브할 데이터 조회
        var data = await _db.Query(tableName)
            .Where("time", "<", cutoffDate)
            .OrderBy("time")
            .GetAsync();

        var records = data.ToList();
        if (records.Count == 0)
        {
            Console.WriteLine("아카이브할 데이터가 없다.");
            return new ArchiveResult { Success = false, Message = "No data to archive" };
        }

        // CSV 파일로 저장
        var csvContent = new StringBuilder();
        
        // 헤더 작성
        var firstRecord = records[0] as IDictionary<string, object>;
        csvContent.AppendLine(string.Join(",", firstRecord.Keys));

        // 데이터 작성
        foreach (var record in records)
        {
            var dict = record as IDictionary<string, object>;
            var values = dict.Values.Select(v => 
                v?.ToString()?.Replace(",", ";") ?? "");
            csvContent.AppendLine(string.Join(",", values));
        }

        await File.WriteAllTextAsync(fullPath, csvContent.ToString());

        Console.WriteLine($"아카이브 완료: {records.Count}건 → {fullPath}");

        // 선택적으로 데이터베이스에서 삭제
        if (deleteAfterArchive)
        {
            var deleted = await _db.Query(tableName)
                .Where("time", "<", cutoffDate)
                .DeleteAsync();

            Console.WriteLine($"데이터베이스에서 {deleted}건 삭제");
        }

        return new ArchiveResult
        {
            Success = true,
            ArchiveFile = fullPath,
            RecordCount = records.Count,
            FileSize = new FileInfo(fullPath).Length
        };
    }

    // 아카이브된 데이터 복원
    public async Task<int> RestoreFromArchiveAsync(string tableName, string archiveFilePath)
    {
        if (!File.Exists(archiveFilePath))
        {
            throw new FileNotFoundException($"아카이브 파일을 찾을 수 없다: {archiveFilePath}");
        }

        var lines = await File.ReadAllLinesAsync(archiveFilePath);
        if (lines.Length < 2)
        {
            return 0;
        }

        var headers = lines[0].Split(',');
        var insertCount = 0;

        // 배치 삽입을 위한 데이터 준비
        var batchSize = 1000;
        var batch = new List<Dictionary<string, object>>();

        for (int i = 1; i < lines.Length; i++)
        {
            var values = lines[i].Split(',');
            var record = new Dictionary<string, object>();

            for (int j = 0; j < headers.Length; j++)
            {
                record[headers[j]] = values[j];
            }

            batch.Add(record);

            if (batch.Count >= batchSize)
            {
                insertCount += await _db.Query(tableName).InsertAsync(batch);
                batch.Clear();
            }
        }

        // 남은 데이터 삽입
        if (batch.Count > 0)
        {
            insertCount += await _db.Query(tableName).InsertAsync(batch);
        }

        Console.WriteLine($"복원 완료: {insertCount}건 → {tableName}");
        return insertCount;
    }

    // 아카이브 파일 목록 조회
    public List<ArchiveFileInfo> GetArchiveFiles()
    {
        var directory = new DirectoryInfo(_archivePath);
        if (!directory.Exists)
        {
            return new List<ArchiveFileInfo>();
        }

        return directory.GetFiles("*_archive.csv")
            .Select(f => new ArchiveFileInfo
            {
                FileName = f.Name,
                FilePath = f.FullName,
                FileSize = f.Length,
                CreatedDate = f.CreationTime
            })
            .OrderByDescending(f => f.CreatedDate)
            .ToList();
    }
}

public class ArchiveResult
{
    public bool Success { get; set; }
    public string Message { get; set; }
    public string ArchiveFile { get; set; }
    public int RecordCount { get; set; }
    public long FileSize { get; set; }

    public override string ToString()
    {
        return $"아카이브: {RecordCount:N0}건, 크기: {FileSize / 1024.0 / 1024.0:F2}MB, 파일: {ArchiveFile}";
    }
}

public class ArchiveFileInfo
{
    public string FileName { get; set; }
    public string FilePath { get; set; }
    public long FileSize { get; set; }
    public DateTime CreatedDate { get; set; }

    public override string ToString()
    {
        return $"{FileName} ({FileSize / 1024.0 / 1024.0:F2}MB) - {CreatedDate:yyyy-MM-dd}";
    }
}
```

## 9.5 압축된 데이터 조회하기

압축된 데이터를 조회하는 것은 일반 데이터와 완전히 동일하다. SQLKata를 사용하여 평소와 같이 쿼리하면 TimescaleDB가 자동으로 압축 해제를 처리한다.

```csharp
public class CompressedDataQuery
{
    private readonly QueryFactory _db;

    public CompressedDataQuery(QueryFactory db)
    {
        _db = db;
    }

    // 압축된 데이터 조회 (일반 쿼리와 동일)
    public async Task<IEnumerable<dynamic>> GetServerMetricsAsync(
        string serverName, 
        DateTime startTime, 
        DateTime endTime)
    {
        return await _db.Query("server_metrics")
            .Where("server_name", serverName)
            .WhereBetween("time", startTime, endTime)
            .OrderBy("time")
            .GetAsync();
    }

    // 압축된 데이터로 집계 쿼리
    public async Task<dynamic> GetAverageCpuUsageAsync(
        string serverName, 
        int lastDays = 30)
    {
        var startTime = DateTime.UtcNow.AddDays(-lastDays);

        return await _db.Query("server_metrics")
            .Where("server_name", serverName)
            .Where("metric_name", "CPU")
            .Where("time", ">=", startTime)
            .SelectRaw("AVG(value) as avg_cpu")
            .SelectRaw("MIN(value) as min_cpu")
            .SelectRaw("MAX(value) as max_cpu")
            .FirstAsync();
    }

    // time_bucket으로 압축된 데이터 그룹화
    public async Task<IEnumerable<dynamic>> GetHourlyMetricsAsync(
        string serverName,
        DateTime startTime,
        DateTime endTime)
    {
        var query = @"
            SELECT 
                time_bucket('1 hour', time) as bucket,
                metric_name,
                AVG(value) as avg_value,
                MAX(value) as max_value
            FROM server_metrics
            WHERE server_name = @serverName
              AND time BETWEEN @startTime AND @endTime
            GROUP BY bucket, metric_name
            ORDER BY bucket, metric_name";

        return await _db.StatementAsync(query, new
        {
            serverName,
            startTime,
            endTime
        });
    }
}
```

압축된 데이터를 조회할 때 성능은 어떨까? 일반적으로 압축된 데이터는 디스크 I/O가 줄어들어 오히려 쿼리 속도가 빨라진다. 특히 시간 범위 조회나 집계 쿼리에서 효과가 크다.

## 9.6 실전: 3개월 이상 데이터 자동 압축

이제 실전 예제로 완전한 데이터 생명주기 관리 시스템을 구축해보자. 게임 서버에서 수집되는 다양한 로그 테이블에 압축과 삭제 정책을 적용하여 자동으로 관리되도록 한다.

```csharp
public class DataLifecycleManager
{
    private readonly string _connectionString;
    private readonly QueryFactory _db;

    public DataLifecycleManager(string connectionString, QueryFactory db)
    {
        _connectionString = connectionString;
        _db = db;
    }

    // 전체 데이터 생명주기 정책 설정
    public async Task SetupDataLifecyclePoliciesAsync()
    {
        Console.WriteLine("=== 데이터 생명주기 정책 설정 시작 ===\n");

        // 1. 서버 메트릭: 7일 후 압축, 90일 후 삭제
        await SetupTablePolicyAsync(
            tableName: "server_metrics",
            compressAfterDays: 7,
            deleteAfterDays: 90,
            segmentBy: new[] { "server_name", "metric_name" }
        );

        // 2. 플레이어 로그인 로그: 30일 후 압축, 365일 후 삭제
        await SetupTablePolicyAsync(
            tableName: "player_login_logs",
            compressAfterDays: 30,
            deleteAfterDays: 365,
            segmentBy: new[] { "player_id" }
        );

        // 3. 게임 이벤트 로그: 14일 후 압축, 180일 후 삭제
        await SetupTablePolicyAsync(
            tableName: "game_event_logs",
            compressAfterDays: 14,
            deleteAfterDays: 180,
            segmentBy: new[] { "event_type", "server_name" }
        );

        // 4. 채팅 로그: 3일 후 압축, 30일 후 삭제
        await SetupTablePolicyAsync(
            tableName: "chat_logs",
            compressAfterDays: 3,
            deleteAfterDays: 30,
            segmentBy: new[] { "channel_id" }
        );

        Console.WriteLine("\n=== 모든 정책 설정 완료 ===");
    }

    // 개별 테이블 정책 설정
    private async Task SetupTablePolicyAsync(
        string tableName,
        int compressAfterDays,
        int deleteAfterDays,
        string[] segmentBy)
    {
        Console.WriteLine($"\n[{tableName}] 정책 설정 중...");

        await using var connection = new NpgsqlConnection(_connectionString);
        await connection.OpenAsync();

        try
        {
            // 압축 설정
            var segmentByColumns = string.Join(", ", segmentBy);
            var alterTableSql = $@"
                ALTER TABLE {tableName} SET (
                    timescaledb.compress,
                    timescaledb.compress_segmentby = '{segmentByColumns}',
                    timescaledb.compress_orderby = 'time DESC'
                );";

            await using (var cmd = new NpgsqlCommand(alterTableSql, connection))
            {
                await cmd.ExecuteNonQueryAsync();
            }

            Console.WriteLine($"  ✓ 압축 활성화 (segment by: {segmentByColumns})");

            // 압축 정책 추가
            var compressionPolicySql = $@"
                SELECT add_compression_policy('{tableName}', INTERVAL '{compressAfterDays} days');";

            await using (var cmd = new NpgsqlCommand(compressionPolicySql, connection))
            {
                await cmd.ExecuteNonQueryAsync();
            }

            Console.WriteLine($"  ✓ 자동 압축 정책: {compressAfterDays}일 후");

            // 삭제 정책 추가
            var retentionPolicySql = $@"
                SELECT add_retention_policy('{tableName}', INTERVAL '{deleteAfterDays} days');";

            await using (var cmd = new NpgsqlCommand(retentionPolicySql, connection))
            {
                await cmd.ExecuteNonQueryAsync();
            }

            Console.WriteLine($"  ✓ 자동 삭제 정책: {deleteAfterDays}일 후");
        }
        catch (Exception ex)
        {
            Console.WriteLine($"  ✗ 오류 발생: {ex.Message}");
        }
    }

    // 전체 테이블 현황 조회
    public async Task PrintDataLifecycleStatusAsync()
    {
        Console.WriteLine("\n=== 데이터 생명주기 현황 ===\n");

        var tables = new[] 
        { 
            "server_metrics", 
            "player_login_logs", 
            "game_event_logs", 
            "chat_logs" 
        };

        foreach (var table in tables)
        {
            await PrintTableStatusAsync(table);
        }
    }

    // 개별 테이블 현황 출력
    private async Task PrintTableStatusAsync(string tableName)
    {
        await using var connection = new NpgsqlConnection(_connectionString);
        await connection.OpenAsync();

        // 테이블 기본 정보
        var infoSql = $@"
            SELECT 
                COUNT(*) as total_rows,
                MIN(time) as oldest_data,
                MAX(time) as newest_data,
                pg_size_pretty(pg_total_relation_size('{tableName}')) as total_size
            FROM {tableName};";

        await using var infoCmd = new NpgsqlCommand(infoSql, connection);
        await using var infoReader = await infoCmd.ExecuteReaderAsync();

        if (await infoReader.ReadAsync())
        {
            var totalRows = infoReader.GetInt64(0);
            var oldestData = infoReader.IsDBNull(1) ? (DateTime?)null : infoReader.GetDateTime(1);
            var newestData = infoReader.IsDBNull(2) ? (DateTime?)null : infoReader.GetDateTime(2);
            var totalSize = infoReader.GetString(3);

            Console.WriteLine($"[{tableName}]");
            Console.WriteLine($"  전체 레코드: {totalRows:N0}건");
            Console.WriteLine($"  데이터 기간: {oldestData:yyyy-MM-dd} ~ {newestData:yyyy-MM-dd}");
            Console.WriteLine($"  전체 크기: {totalSize}");
        }
        await infoReader.CloseAsync();

        // 압축 통계
        var compressionSql = $@"
            SELECT 
                COUNT(*) FILTER (WHERE is_compressed = true) as compressed_chunks,
                COUNT(*) FILTER (WHERE is_compressed = false) as uncompressed_chunks,
                pg_size_pretty(SUM(before_compression_total_bytes)) as before_size,
                pg_size_pretty(SUM(after_compression_total_bytes)) as after_size
            FROM timescaledb_information.chunks
            WHERE hypertable_name = '{tableName}';";

        await using var compCmd = new NpgsqlCommand(compressionSql, connection);
        await using var compReader = await compCmd.ExecuteReaderAsync();

        if (await compReader.ReadAsync())
        {
            var compressedChunks = compReader.GetInt64(0);
            var uncompressedChunks = compReader.GetInt64(1);
            var beforeSize = compReader.IsDBNull(2) ? "0" : compReader.GetString(2);
            var afterSize = compReader.IsDBNull(3) ? "0" : compReader.GetString(3);

            Console.WriteLine($"  압축된 Chunk: {compressedChunks}개");
            Console.WriteLine($"  압축 안 된 Chunk: {uncompressedChunks}개");
            
            if (compressedChunks > 0)
            {
                Console.WriteLine($"  압축 효과: {beforeSize} → {afterSize}");
            }
        }

        Console.WriteLine();
    }

    // 수동으로 즉시 압축 실행 (유지보수 작업)
    public async Task ManualCompressionAsync(string tableName, int olderThanDays)
    {
        Console.WriteLine($"\n[{tableName}] 수동 압축 실행 중...");

        await using var connection = new NpgsqlConnection(_connectionString);
        await connection.OpenAsync();

        var sql = $@"
            SELECT compress_chunk(i, if_not_compressed => true)
            FROM show_chunks('{tableName}', older_than => INTERVAL '{olderThanDays} days') i;";

        await using var cmd = new NpgsqlCommand(sql, connection);
        
        var stopwatch = System.Diagnostics.Stopwatch.StartNew();
        await cmd.ExecuteNonQueryAsync();
        stopwatch.Stop();

        Console.WriteLine($"  완료! (소요 시간: {stopwatch.ElapsedMilliseconds}ms)");
    }

    // 전체 시스템 최적화 (야간 유지보수 작업)
    public async Task RunNightlyMaintenanceAsync()
    {
        Console.WriteLine($"\n=== 야간 유지보수 작업 시작 ({DateTime.Now:yyyy-MM-dd HH:mm:ss}) ===\n");

        var tables = new[] 
        { 
            "server_metrics", 
            "player_login_logs", 
            "game_event_logs", 
            "chat_logs" 
        };

        foreach (var table in tables)
        {
            // 1. 수동 압축 실행 (자동 정책을 기다리지 않고)
            await ManualCompressionAsync(table, 7);

            // 2. VACUUM ANALYZE 실행 (성능 최적화)
            await VacuumTableAsync(table);

            // 3. 통계 갱신
            await UpdateTableStatisticsAsync(table);
        }

        Console.WriteLine("\n=== 야간 유지보수 작업 완료 ===");
    }

    private async Task VacuumTableAsync(string tableName)
    {
        Console.WriteLine($"[{tableName}] VACUUM 실행 중...");

        await using var connection = new NpgsqlConnection(_connectionString);
        await connection.OpenAsync();

        var sql = $"VACUUM ANALYZE {tableName};";
        await using var cmd = new NpgsqlCommand(sql, connection);
        await cmd.ExecuteNonQueryAsync();

        Console.WriteLine($"  ✓ VACUUM 완료");
    }

    private async Task UpdateTableStatisticsAsync(string tableName)
    {
        Console.WriteLine($"[{tableName}] 통계 갱신 중...");

        await using var connection = new NpgsqlConnection(_connectionString);
        await connection.OpenAsync();

        var sql = $"ANALYZE {tableName};";
        await using var cmd = new NpgsqlCommand(sql, connection);
        await cmd.ExecuteNonQueryAsync();

        Console.WriteLine($"  ✓ 통계 갱신 완료");
    }
}
```

**핵심 요약**

이번 챕터에서는 TimescaleDB의 데이터 압축과 보관 정책을 배웠다. 시계열 데이터의 특성을 활용한 컬럼 기반 압축 알고리즘으로 90% 이상의 스토리지를 절감할 수 있다. 자동 압축 정책과 자동 삭제 정책을 설정하면 데이터 생명주기가 자동으로 관리되어 운영 부담이 크게 줄어든다. 콜드 스토리지 전략을 활용하면 장기 보관이 필요한 데이터를 저렴한 스토리지로 아카이브할 수 있다. 압축된 데이터도 일반 데이터와 동일하게 쿼리할 수 있으며, 오히려 디스크 I/O가 줄어들어 성능이 향상된다. C#과 SQLKata를 사용하여 이러한 정책들을 코드로 자동화하면 라이브 게임 서버에서 안정적으로 대용량 로그 데이터를 관리할 수 있다.
