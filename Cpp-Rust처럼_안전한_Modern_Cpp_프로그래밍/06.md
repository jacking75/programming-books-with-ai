# 6장: constexpr과 컴파일 타임 안전성

## 6.1 C++26 constexpr 확장 기능
constexpr은 C++11에서 도입된 이후 계속 발전해왔으며, C++26에서는 더욱 강력한 기능들이 추가되었다. 컴파일 타임에 계산을 수행함으로써 런타임 오류를 사전에 방지하고, 성능도 향상시킬 수 있다.

### constexpr의 기본 개념

```cpp
// 기본적인 constexpr 함수
constexpr int square(int x) {
    return x * x;
}

// 컴파일 타임 상수로 사용
constexpr int result = square(5);  // 25, 컴파일 타임에 계산됨

// 배열 크기로 사용 가능
int array[square(10)];  // int array[100]과 동일

// 런타임에도 사용 가능
int runtime_value = 7;
int runtime_result = square(runtime_value);  // 런타임 계산

// 안전하지 않은 예시: constexpr 없이
#define SQUARE(x) ((x) * (x))  // 매크로 - 타입 안전하지 않음

int bad_result = SQUARE(5);  // 작동하지만 위험
// SQUARE(x++) 같은 경우 예상치 못한 동작
```

constexpr의 이점:

1. **컴파일 타임 계산**: 실행 파일에 결과만 포함되어 성능 향상
2. **타입 안전성**: 매크로와 달리 타입 검사를 받음
3. **디버깅 용이**: 일반 함수처럼 디버깅 가능
4. **오류 조기 발견**: 컴파일 시점에 오류 검출

### constexpr 예외 처리 (C++26)
주의: C++26의 constexpr 예외 처리는 제안 단계에 있으며, 2025년 11월 현재 주요 컴파일러에서 아직 완전히 구현되지 않았다. 이 섹션은 향후 도입될 기능과 현재 사용 가능한 대안을 함께 다룬다.

```cpp
// 현재 C++23까지: constexpr에서 예외 사용 불가
constexpr int divide_old(int a, int b) {
    // return a / b;  // b가 0이면 UB, 컴파일 타임에 검증 안 됨
    
    // 현재 방식: assert 또는 조건문으로 처리
    if (b == 0) {
        // 컴파일 타임에는 컴파일 에러
        // 런타임에는 프로그램 종료
        std::abort();  // constexpr 컨텍스트에서는 컴파일 에러 발생
    }
    return a / b;
}

// C++26 예정: constexpr에서 예외 처리 가능
constexpr int divide_new(int a, int b) {
    if (b == 0) {
        throw std::invalid_argument("0으로 나눌 수 없다");
    }
    return a / b;
}

// 사용 예시 (C++26)
constexpr int safe_calculation() {
    try {
        return divide_new(10, 2);  // 컴파일 타임: 5
        // return divide_new(10, 0);  // 컴파일 에러!
    } catch (const std::invalid_argument& e) {
        return -1;  // 예외 처리
    }
}

// 현재 사용 가능한 안전한 대안: std::optional
constexpr std::optional<int> safe_divide(int a, int b) {
    if (b == 0) {
        return std::nullopt;  // 오류 표시
    }
    return a / b;
}

// 사용
constexpr auto result1 = safe_divide(10, 2);   // std::optional<int>{5}
constexpr auto result2 = safe_divide(10, 0);   // std::nullopt

static_assert(result1.has_value(), "값이 있어야 함");
static_assert(!result2.has_value(), "값이 없어야 함");
```

### constexpr structured binding (C++26)
C++26에서는 structured binding을 constexpr 컨텍스트에서 사용할 수 있다.

```cpp
// 구조체 정의
struct Point {
    int x;
    int y;
    
    constexpr Point(int x_, int y_) : x(x_), y(y_) {}
};

// constexpr structured binding
constexpr Point get_origin() {
    return Point{0, 0};
}

constexpr auto calculate_distance() {
    constexpr auto [x, y] = get_origin();  // C++26: constexpr structured binding
    return x * x + y * y;
}

static_assert(calculate_distance() == 0, "원점의 거리 제곱은 0이다");

// 복잡한 예시: 튜플 분해
constexpr std::tuple<int, int, int> get_rgb() {
    return {255, 128, 64};
}

constexpr int calculate_brightness() {
    constexpr auto [r, g, b] = get_rgb();
    return (r + g + b) / 3;
}

static_assert(calculate_brightness() == 147, "밝기 계산 검증");

// 배열 structured binding
constexpr std::array<int, 3> get_coordinates() {
    return {10, 20, 30};
}

constexpr int sum_coordinates() {
    constexpr auto [x, y, z] = get_coordinates();
    return x + y + z;
}

static_assert(sum_coordinates() == 60, "좌표 합 검증");
```

### 안전하지 않은 방법과 안전한 방법의 비교

```cpp
// === 안전하지 않은 방법: 매크로 ===
#define MAX(a, b) ((a) > (b) ? (a) : (b))
#define ARRAY_SIZE 100

// 문제점들
int x = 5;
int bad1 = MAX(x++, 10);  // x가 두 번 증가할 수 있음
// 타입 안전하지 않음
// 디버깅 어려움
// 네임스페이스 오염

// === 안전한 방법: constexpr ===
constexpr int max_value(int a, int b) {
    return (a > b) ? a : b;
}

constexpr std::size_t array_size = 100;

// 이점들
int y = 5;
int good1 = max_value(y++, 10);  // 명확한 동작, y는 한 번만 증가
// 타입 안전
// 디버깅 가능
// 스코프 제한 가능

// === 복잡한 계산 비교 ===

// 안전하지 않은 방법: 런타임 계산
int factorial_runtime(int n) {
    int result = 1;
    for (int i = 2; i <= n; ++i) {
        result *= i;
    }
    return result;
}

// 사용 시마다 런타임 계산 발생
int array1[factorial_runtime(5)];  // 컴파일 에러: 상수가 아님

// 안전한 방법: 컴파일 타임 계산
constexpr int factorial(int n) {
    int result = 1;
    for (int i = 2; i <= n; ++i) {
        result *= i;
    }
    return result;
}

// 컴파일 타임에 계산됨
constexpr int fact5 = factorial(5);  // 120
int array2[factorial(5)];  // int array2[120]과 동일

static_assert(factorial(5) == 120, "5! = 120");
static_assert(factorial(0) == 1, "0! = 1");
```
  


## 6.2 컴파일 타임 계산으로 런타임 오류 방지
constexpr을 활용하면 많은 런타임 오류를 컴파일 타임으로 이동시킬 수 있다. 이는 프로그램의 안전성을 크게 향상시킨다.

### 범위 검증

```cpp
// 컴파일 타임 범위 검증
template<int Min, int Max>
class BoundedInt {
    int value;
    
    constexpr void validate(int v) const {
        if (v < Min || v > Max) {
            throw std::out_of_range("값이 범위를 벗어났다");
        }
    }
    
public:
    constexpr BoundedInt(int v) : value(v) {
        validate(v);
    }
    
    constexpr int get() const { return value; }
    
    constexpr BoundedInt& operator=(int v) {
        validate(v);
        value = v;
        return *this;
    }
};

// 사용
constexpr BoundedInt<0, 100> percentage(50);  // OK
// constexpr BoundedInt<0, 100> invalid(150);  // 컴파일 에러!

// 함수에서 사용
constexpr int calculate_score(int base) {
    BoundedInt<0, 100> score(base);
    // score = 150;  // 컴파일 에러!
    return score.get() * 2;
}

static_assert(calculate_score(50) == 100, "점수 계산 검증");

// 실용적인 예시: RGB 색상
class RGBColor {
    using ColorValue = BoundedInt<0, 255>;
    ColorValue r, g, b;
    
public:
    constexpr RGBColor(int red, int green, int blue)
        : r(red), g(green), b(blue) {}
    
    constexpr int red() const { return r.get(); }
    constexpr int green() const { return g.get(); }
    constexpr int blue() const { return b.get(); }
    
    constexpr uint32_t to_hex() const {
        return (r.get() << 16) | (g.get() << 8) | b.get();
    }
};

constexpr RGBColor white(255, 255, 255);  // OK
// constexpr RGBColor invalid(256, 0, 0);  // 컴파일 에러!

static_assert(white.to_hex() == 0xFFFFFF, "흰색 검증");
```

### 배열 인덱스 안전성

```cpp
// 컴파일 타임 배열 접근 검증
template<typename T, std::size_t N>
class SafeArray {
    std::array<T, N> data;
    
public:
    constexpr SafeArray() : data{} {}
    
    constexpr SafeArray(std::initializer_list<T> init) {
        if (init.size() > N) {
            throw std::out_of_range("초기화 리스트가 너무 크다");
        }
        std::copy(init.begin(), init.end(), data.begin());
    }
    
    // 컴파일 타임 인덱스 접근
    template<std::size_t Index>
        requires (Index < N)
    constexpr T& get() {
        return data[Index];
    }
    
    template<std::size_t Index>
        requires (Index < N)
    constexpr const T& get() const {
        return data[Index];
    }
    
    // 런타임 인덱스 접근 (경계 검사 포함)
    constexpr T& at(std::size_t index) {
        if (index >= N) {
            throw std::out_of_range("인덱스가 범위를 벗어났다");
        }
        return data[index];
    }
    
    constexpr const T& at(std::size_t index) const {
        if (index >= N) {
            throw std::out_of_range("인덱스가 범위를 벗어났다");
        }
        return data[index];
    }
    
    constexpr std::size_t size() const { return N; }
};

// 사용
constexpr SafeArray<int, 5> arr{1, 2, 3, 4, 5};

// 컴파일 타임 접근
constexpr int first = arr.get<0>();   // OK
constexpr int third = arr.get<2>();   // OK
// constexpr int invalid = arr.get<10>();  // 컴파일 에러!

static_assert(first == 1, "첫 번째 요소 검증");
static_assert(third == 3, "세 번째 요소 검증");

// 런타임 접근 (constexpr 컨텍스트에서도 가능)
constexpr int get_element(std::size_t index) {
    return arr.at(index);  // 범위 검사 수행
}

static_assert(get_element(0) == 1, "동적 접근 검증");
// constexpr int bad = get_element(10);  // 컴파일 에러!
```

### 문자열 처리 안전성

```cpp
// 컴파일 타임 문자열 검증
class ConstexprString {
    const char* str;
    std::size_t len;
    
    constexpr std::size_t compute_length(const char* s) const {
        std::size_t length = 0;
        while (s[length] != '\0') {
            ++length;
        }
        return length;
    }
    
public:
    constexpr ConstexprString(const char* s) 
        : str(s), len(compute_length(s)) {}
    
    constexpr std::size_t length() const { return len; }
    
    constexpr char operator[](std::size_t index) const {
        if (index >= len) {
            throw std::out_of_range("인덱스가 범위를 벗어났다");
        }
        return str[index];
    }
    
    constexpr bool starts_with(const char* prefix) const {
        std::size_t prefix_len = compute_length(prefix);
        if (prefix_len > len) {
            return false;
        }
        
        for (std::size_t i = 0; i < prefix_len; ++i) {
            if (str[i] != prefix[i]) {
                return false;
            }
        }
        return true;
    }
    
    constexpr bool contains(char ch) const {
        for (std::size_t i = 0; i < len; ++i) {
            if (str[i] == ch) {
                return true;
            }
        }
        return false;
    }
};

// 사용
constexpr ConstexprString greeting("Hello, World!");

static_assert(greeting.length() == 13, "길이 검증");
static_assert(greeting[0] == 'H', "첫 문자 검증");
static_assert(greeting.starts_with("Hello"), "접두사 검증");
static_assert(greeting.contains('W'), "문자 포함 검증");
static_assert(!greeting.contains('X'), "문자 미포함 검증");

// 이메일 검증 예시
constexpr bool is_valid_email(const char* email) {
    ConstexprString str(email);
    
    // 간단한 검증: @ 포함 여부
    if (!str.contains('@')) {
        return false;
    }
    
    // 최소 길이 확인
    if (str.length() < 3) {  // a@b 형식
        return false;
    }
    
    return true;
}

static_assert(is_valid_email("user@example.com"), "유효한 이메일");
static_assert(!is_valid_email("invalid"), "유효하지 않은 이메일");
static_assert(!is_valid_email("@"), "유효하지 않은 이메일");
```

### 수학적 불변식 검증

```cpp
// 기하학적 불변식 검증
class Triangle {
    double a, b, c;  // 세 변의 길이
    
    constexpr bool is_valid_triangle() const {
        // 삼각형 부등식: 두 변의 합이 나머지 한 변보다 커야 함
        return (a + b > c) && (b + c > a) && (c + a > b);
    }
    
public:
    constexpr Triangle(double side_a, double side_b, double side_c)
        : a(side_a), b(side_b), c(side_c) {
        if (!is_valid_triangle()) {
            throw std::invalid_argument("유효하지 않은 삼각형");
        }
    }
    
    constexpr double perimeter() const {
        return a + b + c;
    }
    
    constexpr double area() const {
        // 헤론의 공식
        double s = perimeter() / 2.0;
        return std::sqrt(s * (s - a) * (s - b) * (s - c));
    }
};

// 사용
constexpr Triangle valid(3, 4, 5);  // 유효한 삼각형
// constexpr Triangle invalid(1, 2, 10);  // 컴파일 에러!

static_assert(valid.perimeter() == 12.0, "둘레 검증");

// 행렬 차원 검증
template<std::size_t Rows, std::size_t Cols>
class Matrix {
    std::array<std::array<double, Cols>, Rows> data{};
    
public:
    constexpr Matrix() = default;
    
    constexpr double& at(std::size_t row, std::size_t col) {
        if (row >= Rows || col >= Cols) {
            throw std::out_of_range("행렬 인덱스 범위 초과");
        }
        return data[row][col];
    }
    
    constexpr const double& at(std::size_t row, std::size_t col) const {
        if (row >= Rows || col >= Cols) {
            throw std::out_of_range("행렬 인덱스 범위 초과");
        }
        return data[row][col];
    }
    
    // 행렬 곱셈: 차원 호환성을 컴파일 타임에 검증
    template<std::size_t OtherCols>
    constexpr Matrix<Rows, OtherCols> multiply(
        const Matrix<Cols, OtherCols>& other) const {
        Matrix<Rows, OtherCols> result;
        
        for (std::size_t i = 0; i < Rows; ++i) {
            for (std::size_t j = 0; j < OtherCols; ++j) {
                double sum = 0.0;
                for (std::size_t k = 0; k < Cols; ++k) {
                    sum += data[i][k] * other.data[k][j];
                }
                result.data[i][j] = sum;
            }
        }
        
        return result;
    }
    
    constexpr std::size_t rows() const { return Rows; }
    constexpr std::size_t cols() const { return Cols; }
};

// 사용
constexpr Matrix<2, 3> mat1;  // 2x3 행렬
constexpr Matrix<3, 2> mat2;  // 3x2 행렬

// 차원이 맞으므로 곱셈 가능
constexpr auto result = mat1.multiply(mat2);  // 2x2 행렬
static_assert(result.rows() == 2 && result.cols() == 2, "결과 행렬 차원 검증");

// 차원이 맞지 않으면 컴파일 에러
// constexpr auto invalid = mat2.multiply(mat1);  // 컴파일 에러!
```

```cpp
// 애플리케이션 설정 검증
struct ServerConfig {
    int port;
    int max_connections;
    int timeout_seconds;
    
    constexpr ServerConfig(int p, int max_conn, int timeout)
        : port(p)
        , max_connections(max_conn)
        , timeout_seconds(timeout) {
        validate();
    }
    
private:
    constexpr void validate() const {
        if (port < 1 || port > 65535) {
            throw std::out_of_range("포트는 1-65535 범위여야 한다");
        }
        
        if (max_connections < 1 || max_connections > 10000) {
            throw std::out_of_range("최대 연결 수는 1-10000 범위여야 한다");
        }
        
        if (timeout_seconds < 0) {
            throw std::invalid_argument("타임아웃은 음수일 수 없다");
        }
    }
};

// 컴파일 타임 설정 검증
constexpr ServerConfig production_config(8080, 1000, 30);  // OK
// constexpr ServerConfig bad_config(99999, 1000, 30);  // 컴파일 에러!

static_assert(production_config.port == 8080, "포트 검증");
static_assert(production_config.max_connections == 1000, "최대 연결 수 검증");

// 빌드 시점 환경별 설정
enum class Environment { Development, Staging, Production };

template<Environment Env>
constexpr ServerConfig get_config() {
    if constexpr (Env == Environment::Development) {
        return ServerConfig(3000, 10, 60);
    } else if constexpr (Env == Environment::Staging) {
        return ServerConfig(8080, 100, 45);
    } else {
        return ServerConfig(80, 1000, 30);
    }
}

constexpr auto dev_config = get_config<Environment::Development>();
constexpr auto prod_config = get_config<Environment::Production>();

static_assert(dev_config.port == 3000, "개발 환경 포트");
static_assert(prod_config.port == 80, "운영 환경 포트");
```
  


## 6.3 #embed 지시문으로 안전한 리소스 포함
주의: `#embed` 지시문은 C++23에서 제안되었으나 C++26에도 아직 채택되지 않았다. 2025년 11월 현재 일부 컴파일러에서 실험적으로 지원하고 있다. 이 섹션은 향후 도입될 기능과 현재 사용 가능한 대안을 함께 다룬다.

### 전통적인 방법의 문제점

```cpp
// === 안전하지 않은 방법 1: 런타임 파일 읽기 ===
std::vector<uint8_t> load_file_runtime(const char* filename) {
    std::ifstream file(filename, std::ios::binary);
    if (!file) {
        throw std::runtime_error("파일 열기 실패");
    }
    
    return std::vector<uint8_t>(
        std::istreambuf_iterator<char>(file),
        std::istreambuf_iterator<char>()
    );
}

// 문제점:
// 1. 런타임 오버헤드
// 2. 파일이 없으면 런타임 에러
```

### 빌드 시스템 통합 예시

```python
# embed_file.py - 파일을 C++ 배열로 변환
import sys

def embed_file(input_file, output_file, var_name):
    with open(input_file, 'rb') as f:
        data = f.read()
    
    with open(output_file, 'w') as f:
        f.write(f'// Auto-generated from {input_file}\n')
        f.write(f'#include <array>\n')
        f.write(f'#include <cstdint>\n\n')
        f.write(f'namespace embedded {{\n')
        f.write(f'constexpr std::array<uint8_t, {len(data)}> {var_name} = {{\n')
        
        for i, byte in enumerate(data):
            if i % 12 == 0:
                f.write('    ')
            f.write(f'0x{byte:02x}')
            if i < len(data) - 1:
                f.write(', ')
            if i % 12 == 11:
                f.write('\n')
        
        f.write('\n};\n')
        f.write('}\n')

if __name__ == '__main__':
    embed_file(sys.argv[1], sys.argv[2], sys.argv[3])
```
