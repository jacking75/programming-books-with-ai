# 1장: 왜 안전한 C++인가?

## 1.1 C++의 메모리 안전성 문제들
C++는 강력한 성능과 유연성을 제공하지만, 그만큼 개발자에게 많은 책임을 요구한다. 메모리 안전성 문제는 C++ 프로그램에서 가장 흔하게 발생하는 버그의 원인이며, 보안 취약점의 주요 원인이기도 하다.

### 버퍼 오버플로우

```cpp
// 안전하지 않은 코드
void unsafe_copy() {
    char buffer[10];
    char source[] = "This is a very long string";
    strcpy(buffer, source);  // 버퍼 오버플로우 발생!
}
```

위 코드는 전형적인 버퍼 오버플로우 예제다. `buffer`는 10바이트만 할당되어 있지만, `source`는 훨씬 더 긴 문자열을 담고 있다. `strcpy` 함수는 경계 검사를 하지 않기 때문에 버퍼를 넘어서는 메모리 영역에 쓰기를 시도하며, 이는 프로그램 크래시나 보안 취약점으로 이어진다.

```cpp
// 안전한 코드
void safe_copy() {
    std::string buffer;
    std::string source = "This is a very long string";
    buffer = source;  // 자동으로 메모리 관리
}
```

`std::string`을 사용하면 필요한 만큼 자동으로 메모리를 할당하고 관리한다. 버퍼 오버플로우가 원천적으로 차단된다.

### Use-After-Free

```cpp
// 안전하지 않은 코드
void unsafe_pointer_usage() {
    int* ptr = new int(42);
    delete ptr;
    *ptr = 10;  // 이미 해제된 메모리에 접근!
}
```

메모리를 해제한 후에도 포인터가 여전히 같은 주소를 가리키고 있어서, 이미 해제된 메모리에 접근하는 use-after-free 버그가 발생한다. 이런 버그는 탐지하기 어렵고, 프로그램이 간헐적으로 크래시하거나 예측 불가능한 동작을 보이게 만든다.

```cpp
// 안전한 코드
void safe_pointer_usage() {
    std::unique_ptr<int> ptr = std::make_unique<int>(42);
    // ptr은 스코프를 벗어날 때 자동으로 해제됨
    // 명시적 delete 불필요
}
```

스마트 포인터를 사용하면 메모리 라이프사이클이 자동으로 관리된다. 객체가 스코프를 벗어나면 자동으로 메모리가 해제되며, use-after-free 버그가 방지된다.

### Null 포인터 역참조

```cpp
// 안전하지 않은 코드
void unsafe_null_check(int* ptr) {
    *ptr = 42;  // ptr이 nullptr이면 크래시!
}
```

포인터가 null인지 확인하지 않고 역참조하면 프로그램이 즉시 크래시한다. 특히 복잡한 코드에서는 포인터가 null일 수 있는 경로를 추적하기 어렵다.

```cpp
// 안전한 코드
void safe_null_check(std::optional<int>& value) {
    if (value.has_value()) {
        *value = 42;  // 값이 있을 때만 접근
    }
}
```

`std::optional`을 사용하면 값이 있을 수도, 없을 수도 있다는 것을 타입 시스템으로 명시한다. 값에 접근하기 전에 반드시 존재 여부를 확인해야 하므로, null 포인터 역참조를 방지할 수 있다.

  
### 데이터 레이스

```cpp
// 안전하지 않은 코드
int shared_counter = 0;

void unsafe_increment() {
    for (int i = 0; i < 1000; ++i) {
        shared_counter++;  // 여러 스레드에서 동시 접근 시 레이스 발생
    }
}
```

여러 스레드가 동시에 같은 변수에 접근하여 수정하면 데이터 레이스가 발생한다. `shared_counter++` 연산은 원자적이지 않기 때문에, 읽기-수정-쓰기 과정에서 다른 스레드의 업데이트가 손실될 수 있다.

```cpp
// 안전한 코드
std::atomic<int> shared_counter{0};

void safe_increment() {
    for (int i = 0; i < 1000; ++i) {
        shared_counter.fetch_add(1);  // 원자적 연산
    }
}
```

`std::atomic`을 사용하면 연산이 원자적으로 수행되어 데이터 레이스가 방지된다. 여러 스레드가 동시에 접근해도 안전하다.

## 1.2 Rust가 보여준 안전한 시스템 프로그래밍
Rust는 메모리 안전성과 스레드 안전성을 컴파일 타임에 보장하는 시스템 프로그래밍 언어다. Rust의 핵심 개념들은 C++ 커뮤니티에 큰 영향을 주었으며, 많은 아이디어가 현대적인 C++ 개발 패턴으로 채택되었다.

### 소유권 시스템
Rust의 소유권 시스템은 각 값이 정확히 하나의 소유자를 가지며, 소유자가 스코프를 벗어나면 값이 자동으로 정리된다는 원칙을 따른다. 이는 메모리 누수와 이중 해제를 컴파일 타임에 방지한다.

```rust
// Rust 예제
fn rust_ownership() {
    let s1 = String::from("hello");
    let s2 = s1;  // s1의 소유권이 s2로 이동
    // println!("{}", s1);  // 컴파일 에러! s1은 더 이상 유효하지 않음
}
```

C++에서는 이동 시맨틱과 스마트 포인터를 통해 유사한 개념을 구현할 수 있다.

```cpp
// C++에서 소유권 개념 구현
void cpp_ownership() {
    std::unique_ptr<std::string> s1 = 
        std::make_unique<std::string>("hello");
    std::unique_ptr<std::string> s2 = std::move(s1);  // 소유권 이동
    // s1은 이제 nullptr
    if (s1) {  // 런타임 검사 필요
        std::cout << *s1;
    }
}
```

C++의 경우 컴파일러가 이동 후 사용을 막아주지는 않지만, 스마트 포인터를 올바르게 사용하면 유사한 안전성을 달성할 수 있다.

### 빌림(Borrowing)과 라이프타임
Rust는 참조의 라이프타임을 컴파일 타임에 추적하여 댕글링 포인터를 방지한다.

```rust
// Rust 예제
fn rust_borrowing() {
    let s = String::from("hello");
    let r = &s;  // s를 빌림
    println!("{}", r);
}  // s와 r 모두 스코프를 벗어남, 안전함
```

C++26에서는 명시적인 라이프타임 표기는 없지만, 스마트 포인터와 참조를 신중하게 사용하여 유사한 안전성을 달성할 수 있다.

```cpp
// C++에서 안전한 참조 사용
void cpp_borrowing() {
    std::string s = "hello";
    const std::string& r = s;  // s를 참조
    std::cout << r;
}  // s와 r 모두 스코프를 벗어남, 안전함
```

### 타입 안전성과 패턴 매칭
Rust의 `Result`와 `Option` 타입은 에러 처리를 타입 시스템의 일부로 만들어, 에러를 무시하기 어렵게 만든다.

```rust
// Rust 예제
fn divide(a: i32, b: i32) -> Result<i32, String> {
    if b == 0 {
        Err("Division by zero".to_string())
    } else {
        Ok(a / b)
    }
}
```

C++에서는 `std::expected`(C++23)나 `std::optional`을 사용하여 유사한 패턴을 구현할 수 있다.

```cpp
// C++에서 타입 안전한 에러 처리
std::optional<int> divide(int a, int b) {
    if (b == 0) {
        return std::nullopt;
    }
    return a / b;
}

void use_divide() {
    auto result = divide(10, 2);
    if (result.has_value()) {
        std::cout << "결과: " << *result << std::endl;
    } else {
        std::cout << "0으로 나눌 수 없다" << std::endl;
    }
}
```

## 1.3 C++26에서 추가된 안전성 기능들
C++26은 메모리 안전성과 타입 안전성을 크게 개선하는 여러 기능을 도입한다. 이 기능들은 Rust와 같은 언어에서 영감을 받았지만, C++의 철학인 "사용하지 않는 것에 대해서는 비용을 지불하지 않는다"는 원칙을 유지한다.

### Contract 기능
Contract는 함수의 사전조건, 사후조건, 불변식을 명시적으로 표현할 수 있게 해준다. 컴파일 타임이나 런타임에 이러한 조건들을 검증하여 버그를 조기에 발견할 수 있다.

```cpp
// Contract 없는 코드
int unsafe_divide(int a, int b) {
    return a / b;  // b가 0이면 미정의 동작
}

// Contract를 사용한 코드
int safe_divide(int a, int b)
    pre(b != 0)  // 사전조건: b는 0이 아니어야 함
{
    return a / b;
}

void use_safe_divide() {
    int result = safe_divide(10, 0);  // 사전조건 위반, 검출됨
}
```

Contract를 사용하면 함수 호출 시점에 조건이 만족되는지 자동으로 검사된다. 위반 시 프로그램이 종료되거나 로그를 남기는 등의 정책을 설정할 수 있다.

### Trivial Relocatability
많은 타입들은 메모리를 단순히 복사하는 것만으로도 안전하게 이동할 수 있다. C++26은 `[[trivially_relocatable]]` 속성을 도입하여 컴파일러가 이를 최적화할 수 있게 한다.

```cpp
// 기존 방식
class OldString {
    char* data;
public:
    // 복잡한 이동 생성자와 소멸자 필요
    OldString(OldString&& other) noexcept {
        data = other.data;
        other.data = nullptr;
    }
    ~OldString() {
        delete[] data;
    }
};

// C++26 방식
class [[trivially_relocatable]] NewString {
    char* data;
public:
    // 컴파일러가 최적화된 이동 연산 생성
};
```

`[[trivially_relocatable]]` 속성은 벡터의 재할당 등에서 성능을 크게 개선하면서도 안전성을 유지한다.

### Erroneous Behavior와 초기화 검증
C++26은 초기화되지 않은 메모리 읽기를 탐지할 수 있는 메커니즘을 제공한다.

```cpp
// 미정의 동작
void undefined_behavior() {
    int x;
    int y = x;  // x가 초기화되지 않음, 미정의 동작
}

// C++26 Erroneous Behavior
void erroneous_behavior() {
    [[indeterminate]] int x;
    int y = x;  // 컴파일러가 경고하거나 런타임에 탐지 가능
}
```

`[[indeterminate]]` 속성을 사용하면 변수가 의도적으로 초기화되지 않았음을 표시할 수 있으며, 초기화 전 읽기를 시도하면 탐지된다.

### Pack Indexing
템플릿 파라미터 팩을 인덱스로 접근할 수 있게 되어, 복잡한 메타프로그래밍이 더 안전하고 명확해진다.

```cpp
// 기존 방식: 복잡한 재귀 템플릿
template<size_t I, typename... Args>
struct GetTypeHelper;

template<typename T, typename... Rest>
struct GetTypeHelper<0, T, Rest...> {
    using type = T;
};

template<size_t I, typename T, typename... Rest>
struct GetTypeHelper<I, T, Rest...> {
    using type = typename GetTypeHelper<I-1, Rest...>::type;
};

// C++26 방식: 직관적인 인덱싱
template<size_t I, typename... Args>
using GetType = Args...[I];  // 훨씬 간단하고 안전함

// 사용 예
template<typename... Args>
void process(Args... args) {
    using FirstType = Args...[0];
    FirstType first = args...[0];
}
```

### Hazard Pointer와 RCU
동시성 프로그래밍을 위한 안전한 메커니즘이 표준 라이브러리에 추가된다.

```cpp
// Lock-free 스택 구현
template<typename T>
class LockFreeStack {
    struct Node {
        T data;
        std::atomic<Node*> next;
    };
    
    std::atomic<Node*> head{nullptr};
    std::hazard_pointer_obj_base<Node> hazard;
    
public:
    void push(T value) {
        Node* new_node = new Node{std::move(value)};
        new_node->next = head.load();
        while (!head.compare_exchange_weak(
            new_node->next, new_node));
    }
    
    std::optional<T> pop() {
        std::hazard_pointer hp = std::make_hazard_pointer(hazard);
        Node* old_head = hp.protect(head);
        
        if (!old_head) return std::nullopt;
        
        while (!head.compare_exchange_weak(
            old_head, old_head->next.load())) {
            old_head = hp.protect(head);
            if (!old_head) return std::nullopt;
        }
        
        T result = std::move(old_head->data);
        hp.reset_protection();
        hazard.retire(old_head);
        return result;
    }
};
```

Hazard Pointer는 메모리 재사용 시점을 안전하게 관리하여 use-after-free를 방지한다.

### std::inplace_vector
스택에 할당되는 고정 크기 벡터로, 동적 할당 없이도 벡터의 편의성을 제공한다.

```cpp
// 동적 할당을 사용하는 기존 방식
void use_vector() {
    std::vector<int> vec;  // 힙 할당
    vec.push_back(1);
    vec.push_back(2);
}

// C++26 스택 기반 벡터
void use_inplace_vector() {
    std::inplace_vector<int, 10> vec;  // 스택 할당
    vec.push_back(1);
    vec.push_back(2);
    // vec.push_back(...);  // 11번째 요소 추가 시 예외 발생
}
```

`std::inplace_vector`는 크기 제한을 컴파일 타임에 알고 있어서, 오버플로우를 안전하게 처리할 수 있다.

## 1.4 현대적 C++ 개발 환경 구축 (GCC 14+, Clang 18+)
C++26의 기능들을 활용하려면 최신 컴파일러가 필요하다. 이 섹션에서는 GCC 14 이상과 Clang 18 이상을 사용하여 개발 환경을 구축하는 방법을 설명한다.

### GCC 14+ 설치와 설정
GCC 14는 C++26의 많은 기능을 실험적으로 지원한다.

```bash
# Ubuntu/Debian 기반 시스템
sudo add-apt-repository ppa:ubuntu-toolchain-r/test
sudo apt update
sudo apt install gcc-14 g++-14

# 기본 컴파일러로 설정
sudo update-alternatives --install /usr/bin/gcc gcc /usr/bin/gcc-14 100
sudo update-alternatives --install /usr/bin/g++ g++ /usr/bin/g++-14 100
```

컴파일 시 C++26 기능을 활성화하려면 다음 플래그를 사용한다.

```bash
# C++26 표준 활성화
g++-14 -std=c++26 -Wall -Wextra -pedantic main.cpp -o program

# 추가 안전성 검사 활성화
g++-14 -std=c++26 -Wall -Wextra -pedantic \
       -fsanitize=address \
       -fsanitize=undefined \
       -D_GLIBCXX_DEBUG \
       main.cpp -o program
```

### Clang 18+ 설치와 설정
Clang 18은 많은 C++26 기능을 지원하며, 특히 우수한 진단 메시지를 제공한다.

```bash
# Ubuntu/Debian 기반 시스템
wget https://apt.llvm.org/llvm.sh
chmod +x llvm.sh
sudo ./llvm.sh 18

# Clang 18을 기본 컴파일러로 설정
sudo update-alternatives --install /usr/bin/clang clang /usr/bin/clang-18 100
sudo update-alternatives --install /usr/bin/clang++ clang++ /usr/bin/clang++-18 100
```

Clang 컴파일 옵션은 다음과 같다.

```bash
# C++26 표준 활성화
clang++-18 -std=c++2c -Wall -Wextra -pedantic main.cpp -o program

# 추가 안전성 검사 활성화
clang++-18 -std=c++2c -Wall -Wextra -pedantic \
          -fsanitize=address \
          -fsanitize=memory \
          -fsanitize=thread \
          -fsanitize=undefined \
          main.cpp -o program
```

### CMake 프로젝트 설정
대부분의 C++ 프로젝트는 CMake를 사용하여 빌드 시스템을 구성한다. C++26을 사용하는 프로젝트의 CMakeLists.txt 예제는 다음과 같다.

```cmake
cmake_minimum_required(VERSION 3.28)
project(SafeCppProject LANGUAGES CXX)

# C++26 표준 요구
set(CMAKE_CXX_STANDARD 26)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_CXX_EXTENSIONS OFF)

# 컴파일러 경고 활성화
if(CMAKE_CXX_COMPILER_ID MATCHES "GNU|Clang")
    add_compile_options(-Wall -Wextra -pedantic -Werror)
endif()

# 실행 파일 생성
add_executable(my_program
    src/main.cpp
    src/safe_container.cpp
)

# 디버그 빌드에 sanitizer 추가
if(CMAKE_BUILD_TYPE MATCHES Debug)
    target_compile_options(my_program PRIVATE
        -fsanitize=address
        -fsanitize=undefined
    )
    target_link_options(my_program PRIVATE
        -fsanitize=address
        -fsanitize=undefined
    )
endif()
```

프로젝트 빌드는 다음과 같이 수행한다.

```bash
mkdir build
cd build
cmake -DCMAKE_BUILD_TYPE=Debug ..
cmake --build .
```

### 컴파일러 기능 확인
현재 컴파일러가 어떤 C++26 기능을 지원하는지 확인하는 간단한 프로그램을 작성할 수 있다.

```cpp
#include <iostream>

int main() {
    std::cout << "C++ 표준: " << __cplusplus << std::endl;
    
    #ifdef __cpp_pack_indexing
    std::cout << "Pack indexing: 지원됨" << std::endl;
    #else
    std::cout << "Pack indexing: 미지원" << std::endl;
    #endif
    
    #ifdef __cpp_contracts
    std::cout << "Contracts: 지원됨" << std::endl;
    #else
    std::cout << "Contracts: 미지원" << std::endl;
    #endif
    
    #ifdef __cpp_static_assert
    std::cout << "Static assert: 지원됨" << std::endl;
    #endif
    
    return 0;
}
```

### 통합 개발 환경 설정
Visual Studio Code를 사용하는 경우, `.vscode/settings.json` 파일에 다음 설정을 추가한다.

```json
{
    "C_Cpp.default.cppStandard": "c++26",
    "C_Cpp.default.compilerPath": "/usr/bin/g++-14",
    "C_Cpp.default.intelliSenseMode": "gcc-x64",
    "C_Cpp.errorSquiggles": "enabled",
    "files.associations": {
        "*.cpp": "cpp",
        "*.h": "cpp"
    }
}
```

CLion을 사용하는 경우, Settings > Build, Execution, Deployment > Toolchains에서 컴파일러 경로를 설정하고, CMake 옵션에 `-DCMAKE_CXX_STANDARD=26`을 추가한다.

### 코드 정적 분석 도구 설정
안전한 C++ 코드를 작성하려면 정적 분석 도구를 활용하는 것이 좋다.

```bash
# Clang-Tidy 설치
sudo apt install clang-tidy-18

# 프로젝트 루트에 .clang-tidy 파일 생성
cat > .clang-tidy << EOF
Checks: >
  *,
  -fuchsia-*,
  -google-*,
  -llvm-*,
  -modernize-use-trailing-return-type,
  clang-analyzer-*,
  cppcoreguidelines-*,
  modernize-*,
  performance-*,
  readability-*

CheckOptions:
  - key: readability-identifier-naming.VariableCase
    value: lower_case
  - key: readability-identifier-naming.FunctionCase
    value: lower_case
EOF

# 분석 실행
clang-tidy-18 src/*.cpp -- -std=c++26
```

이제 개발 환경이 준비되었다. 다음 장부터는 C++26의 구체적인 기능들을 활용하여 안전한 코드를 작성하는 방법을 배우게 된다. 메모리 관리, 타입 안전성, 동시성 프로그래밍 등 각 주제를 실제 예제와 함께 깊이 있게 다룰 것이다.
