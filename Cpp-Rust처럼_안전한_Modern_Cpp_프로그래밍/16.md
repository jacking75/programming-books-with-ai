# 16장: 안전성 검증 도구들
안전한 C++ 코드를 작성하는 것만큼 중요한 것이 작성된 코드의 안전성을 검증하는 것이다. 아무리 주의 깊게 코드를 작성해도 메모리 안전성 문제, 데이터 레이스, 정의되지 않은 동작 등은 쉽게 발생할 수 있다. 이 장에서는 C++26 환경에서 사용할 수 있는 다양한 검증 도구들을 살펴본다.

## 16.1 컴파일러 내장 검사 옵션

현대 C++ 컴파일러들은 코드의 안전성을 검증하는 강력한 옵션들을 제공한다. 이러한 옵션들을 활용하면 런타임 이전에 많은 문제를 발견할 수 있다.

### 16.1.1 경고 옵션 활성화

**안전하지 않은 방법: 기본 경고 수준 사용**

```cpp
// 컴파일: g++ main.cpp
#include <vector>
#include <iostream>

int process_data(const std::vector<int>& data, int index) {
    return data[index];  // 경계 검사 없음
}

int main() {
    std::vector<int> numbers = {1, 2, 3};
    int result = process_data(numbers, 10);  // 위험하지만 경고 없음
    
    int x;
    std::cout << x << std::endl;  // 초기화되지 않은 변수
    
    return 0;
}
```

기본 컴파일 옵션으로는 많은 잠재적 문제가 경고 없이 컴파일된다.

**안전한 방법: 엄격한 경고 옵션 활성화**

```cpp
// 컴파일: g++ -std=c++26 -Wall -Wextra -Wpedantic -Werror main.cpp
#include <vector>
#include <iostream>
#include <stdexcept>

int process_data(const std::vector<int>& data, int index) {
    if (index < 0 || index >= static_cast<int>(data.size())) {
        throw std::out_of_range("Index out of bounds");
    }
    return data[index];
}

int main() {
    std::vector<int> numbers = {1, 2, 3};
    
    try {
        int result = process_data(numbers, 2);  // 안전한 접근
        std::cout << "Result: " << result << std::endl;
    } catch (const std::out_of_range& e) {
        std::cerr << "Error: " << e.what() << std::endl;
    }
    
    int x = 0;  // 명시적 초기화
    std::cout << x << std::endl;
    
    return 0;
}
```

권장하는 GCC/Clang 컴파일 옵션들은 다음과 같다.

```bash
g++ -std=c++26 \
    -Wall \              # 일반적인 경고
    -Wextra \            # 추가 경고
    -Wpedantic \         # 표준 준수 경고
    -Werror \            # 경고를 에러로 처리
    -Wconversion \       # 암시적 타입 변환 경고
    -Wsign-conversion \  # 부호 변환 경고
    -Wshadow \           # 변수 섀도잉 경고
    -Wnull-dereference \ # null 역참조 경고
    -Wformat=2 \         # printf 형식 문자열 검사
    main.cpp
```

### 16.1.2 런타임 검사 활성화

**안전하지 않은 방법: 최적화만 고려**

```cpp
// 컴파일: g++ -O3 main.cpp
#include <vector>
#include <iostream>

int main() {
    std::vector<int> vec = {1, 2, 3};
    
    // 범위를 벗어난 접근
    for (int i = 0; i <= 10; ++i) {
        std::cout << vec[i] << " ";  // 정의되지 않은 동작
    }
    
    return 0;
}
```

최적화만 활성화하면 런타임 검사가 수행되지 않아 문제를 발견하기 어렵다.

**안전한 방법: 디버그 빌드와 검사 활성화**

```cpp
// 컴파일: g++ -std=c++26 -D_GLIBCXX_ASSERTIONS -g -O0 main.cpp
#include <vector>
#include <iostream>

int main() {
    std::vector<int> vec = {1, 2, 3};
    
    // _GLIBCXX_ASSERTIONS가 활성화되어 있으면
    // operator[]가 범위 검사를 수행한다
    try {
        for (int i = 0; i < static_cast<int>(vec.size()); ++i) {
            std::cout << vec[i] << " ";
        }
        std::cout << std::endl;
    } catch (...) {
        std::cerr << "Error occurred" << std::endl;
    }
    
    return 0;
}
```

디버그 빌드에서 사용할 수 있는 옵션들은 다음과 같다.

```bash
# GCC/libstdc++ 디버그 빌드
g++ -std=c++26 \
    -D_GLIBCXX_ASSERTIONS \      # 기본 검사 활성화
    -D_GLIBCXX_DEBUG \            # 완전한 디버그 모드
    -D_GLIBCXX_DEBUG_PEDANTIC \   # 엄격한 디버그 모드
    -g \                          # 디버그 정보 포함
    -O0 \                         # 최적화 비활성화
    main.cpp

# Clang/libc++ 디버그 빌드
clang++ -std=c++26 \
    -D_LIBCPP_ENABLE_ASSERTIONS \  # 런타임 검사 활성화
    -D_LIBCPP_DEBUG=1 \            # 디버그 모드
    -g \
    -O0 \
    main.cpp
```

### 16.1.3 미정의 동작 검사 활성화

**활용 예: 다양한 미정의 동작 탐지**

```cpp
// 컴파일: g++ -std=c++26 -fsanitize=undefined -g main.cpp
#include <iostream>
#include <limits>

void demonstrate_ub_checks() {
    // 1. 정수 오버플로우
    {
        int max = std::numeric_limits<int>::max();
        int overflow = max + 1;  // UBSan이 탐지
        std::cout << "Overflow: " << overflow << std::endl;
    }
    
    // 2. 부호 있는 정수 이동 연산
    {
        int value = 1;
        int shift = value << 31;  // UBSan이 탐지
        std::cout << "Shift: " << shift << std::endl;
    }
    
    // 3. null 포인터 역참조
    {
        int* ptr = nullptr;
        // *ptr = 42;  // UBSan이 탐지 (주석 처리)
    }
    
    // 4. 정렬되지 않은 포인터 접근
    {
        char buffer[10];
        int* misaligned = reinterpret_cast<int*>(buffer + 1);
        // *misaligned = 42;  // UBSan이 탐지 (주석 처리)
    }
}

int main() {
    demonstrate_ub_checks();
    return 0;
}
```

UBSan(Undefined Behavior Sanitizer)은 다음과 같은 문제들을 탐지한다.

```bash
# UBSan 활성화
g++ -std=c++26 \
    -fsanitize=undefined \           # 기본 UB 검사
    -fsanitize=float-divide-by-zero \ # 부동소수점 0 나누기
    -fsanitize=bounds \              # 배열 경계 검사
    -fsanitize=alignment \           # 정렬 검사
    -fsanitize=null \                # null 포인터 검사
    -fsanitize=signed-integer-overflow \ # 부호 있는 정수 오버플로우
    -fno-sanitize-recover=all \      # 첫 에러에서 중단
    -g \
    main.cpp
```

## 16.2 AddressSanitizer와 ThreadSanitizer

### 16.2.1 AddressSanitizer (ASan)

AddressSanitizer는 메모리 안전성 문제를 탐지하는 강력한 도구다.

**안전하지 않은 방법: 메모리 문제가 있는 코드**

```cpp
#include <iostream>
#include <vector>

void heap_buffer_overflow() {
    int* array = new int[10];
    array[10] = 42;  // 범위 초과 - ASan이 탐지
    delete[] array;
}

void use_after_free() {
    int* ptr = new int(42);
    delete ptr;
    *ptr = 100;  // 해제된 메모리 사용 - ASan이 탐지
}

void stack_buffer_overflow() {
    int array[10];
    array[10] = 42;  // 스택 버퍼 오버플로우 - ASan이 탐지
}

void memory_leak() {
    int* ptr = new int(42);
    // delete를 하지 않음 - ASan이 탐지
}

int main() {
    // 각 함수를 개별적으로 테스트
    // heap_buffer_overflow();
    // use_after_free();
    // stack_buffer_overflow();
    // memory_leak();
    
    return 0;
}
```

**안전한 방법: 메모리 안전성 보장**

```cpp
// 컴파일: g++ -std=c++26 -fsanitize=address -g main.cpp
#include <iostream>
#include <vector>
#include <memory>
#include <array>

void safe_heap_usage() {
    std::vector<int> vec(10);
    vec.at(9) = 42;  // 범위 검사 수행
    // vec.at(10) = 42;  // 예외 발생
}

void safe_smart_pointer() {
    auto ptr = std::make_unique<int>(42);
    std::cout << *ptr << std::endl;
    // 자동으로 메모리 해제
}

void safe_stack_usage() {
    std::array<int, 10> arr;
    arr.at(9) = 42;  // 범위 검사 수행
}

void safe_resource_management() {
    auto resource = std::make_unique<int>(42);
    std::cout << *resource << std::endl;
    // RAII로 자동 정리
}

int main() {
    safe_heap_usage();
    safe_smart_pointer();
    safe_stack_usage();
    safe_resource_management();
    
    return 0;
}
```

ASan 사용 시 컴파일 옵션은 다음과 같다.

```bash
# AddressSanitizer 활성화
g++ -std=c++26 \
    -fsanitize=address \              # ASan 활성화
    -fsanitize-address-use-after-scope \ # 스코프 이후 사용 탐지
    -fno-omit-frame-pointer \         # 정확한 스택 추적
    -g \                              # 디버그 정보
    -O1 \                             # 적절한 최적화
    main.cpp

# 실행 시 옵션 설정
ASAN_OPTIONS=detect_leaks=1:check_initialization_order=1 ./a.out
```

### 16.2.2 ThreadSanitizer (TSan)

ThreadSanitizer는 데이터 레이스와 동시성 문제를 탐지한다.

**안전하지 않은 방법: 데이터 레이스가 있는 코드**

```cpp
#include <thread>
#include <iostream>

int shared_counter = 0;  // 보호되지 않은 공유 변수

void unsafe_increment() {
    for (int i = 0; i < 100000; ++i) {
        ++shared_counter;  // 데이터 레이스 - TSan이 탐지
    }
}

int main() {
    std::thread t1(unsafe_increment);
    std::thread t2(unsafe_increment);
    
    t1.join();
    t2.join();
    
    std::cout << "Counter: " << shared_counter << std::endl;
    return 0;
}
```

**안전한 방법: 동기화를 사용한 안전한 코드**

```cpp
// 컴파일: g++ -std=c++26 -fsanitize=thread -g main.cpp
#include <thread>
#include <iostream>
#include <mutex>
#include <atomic>
#include <vector>

// 방법 1: mutex 사용
class SafeCounterMutex {
    int counter = 0;
    std::mutex mtx;
    
public:
    void increment() {
        std::lock_guard<std::mutex> lock(mtx);
        ++counter;
    }
    
    int get() const {
        return counter;
    }
};

// 방법 2: atomic 사용
class SafeCounterAtomic {
    std::atomic<int> counter{0};
    
public:
    void increment() {
        counter.fetch_add(1, std::memory_order_relaxed);
    }
    
    int get() const {
        return counter.load(std::memory_order_relaxed);
    }
};

void test_mutex_counter() {
    SafeCounterMutex counter;
    
    auto increment_task = [&counter]() {
        for (int i = 0; i < 100000; ++i) {
            counter.increment();
        }
    };
    
    std::vector<std::thread> threads;
    for (int i = 0; i < 4; ++i) {
        threads.emplace_back(increment_task);
    }
    
    for (auto& t : threads) {
        t.join();
    }
    
    std::cout << "Mutex Counter: " << counter.get() << std::endl;
}

void test_atomic_counter() {
    SafeCounterAtomic counter;
    
    auto increment_task = [&counter]() {
        for (int i = 0; i < 100000; ++i) {
            counter.increment();
        }
    };
    
    std::vector<std::thread> threads;
    for (int i = 0; i < 4; ++i) {
        threads.emplace_back(increment_task);
    }
    
    for (auto& t : threads) {
        t.join();
    }
    
    std::cout << "Atomic Counter: " << counter.get() << std::endl;
}

int main() {
    test_mutex_counter();
    test_atomic_counter();
    
    return 0;
}
```

TSan 사용 시 컴파일 옵션은 다음과 같다.

```bash
# ThreadSanitizer 활성화
g++ -std=c++26 \
    -fsanitize=thread \    # TSan 활성화
    -g \                   # 디버그 정보
    -O1 \                  # 적절한 최적화
    main.cpp

# 실행 시 옵션 설정
TSAN_OPTIONS=second_deadlock_stack=1:history_size=7 ./a.out
```

### 16.2.3 MemorySanitizer (MSan)

MemorySanitizer는 초기화되지 않은 메모리 읽기를 탐지한다.

**안전하지 않은 방법: 초기화되지 않은 메모리 사용**

```cpp
#include <iostream>

int uninitialized_read() {
    int x;
    return x;  // MSan이 탐지
}

struct Data {
    int a;
    int b;
};

void partial_initialization() {
    Data data;
    data.a = 42;
    // data.b는 초기화되지 않음
    std::cout << data.a + data.b << std::endl;  // MSan이 탐지
}

int main() {
    // int result = uninitialized_read();
    // partial_initialization();
    
    return 0;
}
```

**안전한 방법: 명시적 초기화**

```cpp
// 컴파일: clang++ -std=c++26 -fsanitize=memory -g main.cpp
#include <iostream>

int initialized_read() {
    int x = 0;  // 명시적 초기화
    return x;
}

struct Data {
    int a = 0;  // 기본 멤버 초기화
    int b = 0;
};

void safe_initialization() {
    Data data{42, 100};  // 완전한 초기화
    std::cout << data.a + data.b << std::endl;
}

void aggregate_initialization() {
    Data data{.a = 42, .b = 100};  // 지명 초기화
    std::cout << data.a + data.b << std::endl;
}

int main() {
    int result = initialized_read();
    std::cout << "Result: " << result << std::endl;
    
    safe_initialization();
    aggregate_initialization();
    
    return 0;
}
```

## 16.3 정적 분석 도구 활용

### 16.3.1 Clang-Tidy

Clang-Tidy는 코드 품질과 안전성을 검사하는 정적 분석 도구다.

**안전하지 않은 방법: 안전성 문제가 있는 코드**

```cpp
#include <vector>
#include <memory>
#include <iostream>

class UnsafeClass {
public:
    // 문제 1: 원시 포인터 사용
    int* data;
    
    // 문제 2: 암시적 복사 생성자 (얕은 복사)
    UnsafeClass(int value) : data(new int(value)) {}
    
    ~UnsafeClass() {
        delete data;
    }
};

void unsafe_patterns() {
    // 문제 3: C 스타일 캐스트
    double x = 3.14;
    int y = (int)x;
    
    // 문제 4: 매직 넘버
    std::vector<int> vec(100);
    
    // 문제 5: 원시 루프
    for (size_t i = 0; i < vec.size(); ++i) {
        vec[i] = i;
    }
}

int main() {
    unsafe_patterns();
    return 0;
}
```

**안전한 방법: Clang-Tidy 권장사항 적용**

```cpp
#include <vector>
#include <memory>
#include <iostream>
#include <algorithm>
#include <ranges>

class SafeClass {
public:
    // 스마트 포인터 사용
    std::unique_ptr<int> data;
    
    explicit SafeClass(int value) : data(std::make_unique<int>(value)) {}
    
    // Rule of Five 명시
    SafeClass(const SafeClass& other) 
        : data(other.data ? std::make_unique<int>(*other.data) : nullptr) {}
    
    SafeClass(SafeClass&&) noexcept = default;
    SafeClass& operator=(const SafeClass& other) {
        if (this != &other) {
            data = other.data ? std::make_unique<int>(*other.data) : nullptr;
        }
        return *this;
    }
    SafeClass& operator=(SafeClass&&) noexcept = default;
    ~SafeClass() = default;
};

void safe_patterns() {
    // C++ 스타일 캐스트
    double x = 3.14;
    int y = static_cast<int>(x);
    
    // 상수 정의
    constexpr size_t BUFFER_SIZE = 100;
    std::vector<int> vec(BUFFER_SIZE);
    
    // 범위 기반 for 또는 알고리즘 사용
    std::ranges::for_each(vec, [](int& val) {
        static int counter = 0;
        val = counter++;
    });
    
    // 또는 views 사용
    auto view = std::views::iota(0, static_cast<int>(BUFFER_SIZE));
    std::ranges::copy(view, vec.begin());
}

int main() {
    safe_patterns();
    
    SafeClass obj1(42);
    SafeClass obj2 = obj1;  // 안전한 깊은 복사
    
    return 0;
}
```

### 16.3.2 Cppcheck

Cppcheck는 C/C++ 코드의 버그와 안전성 문제를 찾는 정적 분석 도구다.

**활용 예: Cppcheck로 문제 탐지**

```cpp
#include <vector>
#include <iostream>
#include <memory>

void demonstrate_cppcheck_issues() {
    // 문제 1: 메모리 누수
    {
        int* ptr = new int(42);
        if (true) {
            return;  // Cppcheck: 메모리 누수
        }
        delete ptr;
    }
    
    // 문제 2: 배열 범위 초과
    {
        int arr[10];
        for (int i = 0; i <= 10; ++i) {  // Cppcheck: 범위 초과
            arr[i] = i;
        }
    }
    
    // 문제 3: null 포인터 역참조
    {
        int* ptr = nullptr;
        if (false) {
            ptr = new int(42);
        }
        // *ptr = 100;  // Cppcheck: null 포인터 역참조 가능
    }
}

// 안전한 버전
void safe_version() {
    // 스마트 포인터 사용
    {
        auto ptr = std::make_unique<int>(42);
        if (true) {
            return;  // 자동 정리
        }
    }
    
    // 범위 기반 for
    {
        std::vector<int> vec(10);
        for (auto& val : vec) {
            val = 0;
        }
    }
    
    // null 체크
    {
        std::unique_ptr<int> ptr;
        if (false) {
            ptr = std::make_unique<int>(42);
        }
        if (ptr) {
            *ptr = 100;
        }
    }
}

int main() {
    safe_version();
    return 0;
}
```

### 16.3.3 통합 검증 워크플로우

**활용 예: 종합적인 안전성 검증**

```cpp
// safe_math.hpp
#ifndef SAFE_MATH_HPP
#define SAFE_MATH_HPP

#include <stdexcept>
#include <limits>
#include <concepts>

template<std::integral T>
class SafeMath {
public:
    static T add(T a, T b) {
        if (a > 0 && b > std::numeric_limits<T>::max() - a) {
            throw std::overflow_error("Addition overflow");
        }
        if (a < 0 && b < std::numeric_limits<T>::min() - a) {
            throw std::underflow_error("Addition underflow");
        }
        return a + b;
    }
    
    static T multiply(T a, T b) {
        if (a == 0 || b == 0) {
            return 0;
        }
        
        if (a > 0) {
            if (b > 0 && a > std::numeric_limits<T>::max() / b) {
                throw std::overflow_error("Multiplication overflow");
            }
            if (b < 0 && b < std::numeric_limits<T>::min() / a) {
                throw std::underflow_error("Multiplication underflow");
            }
        } else {
            if (b > 0 && a < std::numeric_limits<T>::min() / b) {
                throw std::underflow_error("Multiplication underflow");
            }
            if (b < 0 && a < std::numeric_limits<T>::max() / b) {
                throw std::overflow_error("Multiplication overflow");
            }
        }
        
        return a * b;
    }
    
    static T divide(T a, T b) {
        if (b == 0) {
            throw std::domain_error("Division by zero");
        }
        if (a == std::numeric_limits<T>::min() && b == -1) {
            throw std::overflow_error("Division overflow");
        }
        return a / b;
    }
};

#endif // SAFE_MATH_HPP
```

```cpp
// main.cpp
#include "safe_math.hpp"
#include <iostream>
#include <cassert>

void test_safe_math() {
    // 정상 케이스
    assert(SafeMath<int>::add(10, 20) == 30);
    assert(SafeMath<int>::multiply(5, 6) == 30);
    assert(SafeMath<int>::divide(20, 4) == 5);
    
    // 오버플로우 케이스
    try {
        SafeMath<int>::add(
            std::numeric_limits<int>::max(),
            1
        );
        assert(false && "Should throw overflow_error");
    } catch (const std::overflow_error&) {
        std::cout << "Overflow correctly detected" << std::endl;
    }
    
    // 0으로 나누기
    try {
        SafeMath<int>::divide(10, 0);
        assert(false && "Should throw domain_error");
    } catch (const std::domain_error&) {
        std::cout << "Division by zero correctly detected" << std::endl;
    }
}

int main() {
    test_safe_math();
    std::cout << "All tests passed!" << std::endl;
    return 0;
}
```

### 16.3.4 실전 프로젝트 예시

모든 검증 도구를 통합한 실전 예시를 살펴본다.

```cpp
// buffer_manager.hpp
#ifndef BUFFER_MANAGER_HPP
#define BUFFER_MANAGER_HPP

#include <memory>
#include <vector>
#include <span>
#include <stdexcept>
#include <mutex>
#include <shared_mutex>

template<typename T>
class ThreadSafeBuffer {
private:
    std::vector<T> data_;
    mutable std::shared_mutex mutex_;
    
public:
    explicit ThreadSafeBuffer(size_t size) : data_(size) {}
    
    // 읽기 전용 접근
    T read(size_t index) const {
        std::shared_lock lock(mutex_);
        
        if (index >= data_.size()) {
            throw std::out_of_range("Index out of range");
        }
        
        return data_[index];
    }
    
    // 쓰기 접근
    void write(size_t index, const T& value) {
        std::unique_lock lock(mutex_);
        
        if (index >= data_.size()) {
            throw std::out_of_range("Index out of range");
        }
        
        data_[index] = value;
    }
    
    // 일괄 읽기 (읽기 전용 span 반환)
    std::vector<T> read_all() const {
        std::shared_lock lock(mutex_);
        return data_;
    }
    
    // 일괄 쓰기
    void write_all(std::span<const T> values) {
        std::unique_lock lock(mutex_);
        
        if (values.size() != data_.size()) {
            throw std::invalid_argument("Size mismatch");
        }
        
        std::ranges::copy(values, data_.begin());
    }
    
    size_t size() const {
        std::shared_lock lock(mutex_);
        return data_.size();
    }
};

#endif // BUFFER_MANAGER_HPP
```

```cpp
// test_buffer.cpp
#include "buffer_manager.hpp"
#include <iostream>
#include <thread>
#include <cassert>

void test_basic_operations() {
    ThreadSafeBuffer<int> buffer(10);
    
    // 쓰기 테스트
    for (size_t i = 0; i < buffer.size(); ++i) {
        buffer.write(i, static_cast<int>(i * 2));
    }
    
    // 읽기 테스트
    for (size_t i = 0; i < buffer.size(); ++i) {
        assert(buffer.read(i) == static_cast<int>(i * 2));
    }
    
    std::cout << "Basic operations test passed" << std::endl;
}

void test_boundary_conditions() {
    ThreadSafeBuffer<int> buffer(10);
    
    // 경계 내 접근
    buffer.write(0, 100);
    buffer.write(9, 200);
    assert(buffer.read(0) == 100);
    assert(buffer.read(9) == 200);
    
    // 경계 초과 접근
    try {
        buffer.read(10);
        assert(false && "Should throw out_of_range");
    } catch (const std::out_of_range&) {
        std::cout << "Boundary check works correctly" << std::endl;
    }
}

void test_concurrent_access() {
    ThreadSafeBuffer<int> buffer(1000);
    
    // 여러 스레드에서 동시 쓰기
    std::vector<std::thread> writers;
    for (int t = 0; t < 4; ++t) {
        writers.emplace_back([&buffer, t]() {
            for (size_t i = t * 250; i < (t + 1) * 250; ++i) {
                buffer.write(i, static_cast<int>(i));
            }
        });
    }
    
    for (auto& writer : writers) {
        writer.join();
    }
    
    // 검증
    for (size_t i = 0; i < buffer.size(); ++i) {
        assert(buffer.read(i) == static_cast<int>(i));
    }
    
    std::cout << "Concurrent access test passed" << std::endl;
}

int main() {
    test_basic_operations();
    test_boundary_conditions();
    test_concurrent_access();
    
    std::cout << "All buffer tests passed!" << std::endl;
    return 0;
}
```

## 정리

이 장에서는 안전한 C++ 코드를 검증하는 다양한 도구들을 살펴보았다. 컴파일러 경고 옵션부터 시작하여 AddressSanitizer, ThreadSanitizer, 정적 분석 도구까지 각각의 도구가 특정 유형의 문제를 탐지하는 데 특화되어 있다. 실전에서는 이러한 도구들을 CI/CD 파이프라인에 통합하여 지속적으로 코드 품질을 검증하는 것이 중요하다.

핵심 교훈은 다음과 같다.

첫째, 개발 초기부터 엄격한 컴파일러 옵션을 사용하여 많은 문제를 사전에 방지한다. 둘째, 각 Sanitizer는 특정 유형의 문제에 특화되어 있으므로 용도에 맞게 선택하여 사용한다. 셋째, 정적 분석 도구를 활용하여 런타임 이전에 코드 품질 문제를 발견한다. 넷째, 모든 검증 도구를 자동화된 워크플로우에 통합하여 일관성 있는 품질을 유지한다.

다음 장에서는 이러한 검증 도구들을 단위 테스트 및 속성 기반 테스트와 결합하여 더욱 견고한 코드를 작성하는 방법을 살펴본다.
