## 10.1 동적 할당 없는 벡터

### 전통적인 std::vector의 메모리 안전성 문제
`std::vector`는 C++에서 가장 널리 사용되는 컨테이너이지만, 동적 메모리 할당으로 인한 여러 안전성 문제를 내포하고 있다.

```cpp
// 안전하지 않은 예: 힙 할당 실패 가능성
void process_data(const std::vector<int>& input) {
    std::vector<int> temp;
    temp.reserve(input.size());  // 메모리 부족 시 std::bad_alloc 발생
    
    for (int value : input) {
        temp.push_back(value * 2);  // 재할당 발생 가능, 메모리 단편화
    }
    
    // 처리...
}
```

이 코드는 다음과 같은 문제를 가진다.

1. **힙 할당 실패**: 메모리가 부족한 환경에서 예외가 발생할 수 있다
2. **예측 불가능한 성능**: 재할당 시 모든 요소를 복사하므로 성능이 급격히 저하된다
3. **메모리 단편화**: 반복적인 할당/해제로 메모리가 단편화된다
4. **캐시 지역성 저하**: 힙 메모리는 스택에 비해 캐시 효율이 낮다

### C++26 std::inplace_vector 소개
`std::inplace_vector`는 고정된 최대 크기를 가지며 스택이나 정적 저장소에 요소를 저장하는 컨테이너다. 동적 할당을 완전히 배제함으로써 메모리 안전성을 크게 향상시킨다.

```cpp
#include <inplace_vector>

// 안전한 예: 스택 기반 벡터
void process_data_safe(const std::vector<int>& input) {
    // 최대 100개 요소를 스택에 저장
    std::inplace_vector<int, 100> temp;
    
    for (int value : input) {
        if (temp.size() < temp.capacity()) {
            temp.push_back(value * 2);  // 동적 할당 없음, 예외 안전
        } else {
            // 용량 초과 처리
            break;
        }
    }
    
    // 처리...
}
```

### 주요 특징과 안전성 이점
`std::inplace_vector`는 다음과 같은 안전성 특징을 제공한다.

```cpp
#include <inplace_vector>
#include <iostream>

void demonstrate_safety() {
    // 1. 컴파일 타임에 최대 크기 결정
    std::inplace_vector<double, 50> measurements;
    
    // 2. 동적 할당 없음 - bad_alloc 불가능
    for (int i = 0; i < 40; ++i) {
        measurements.push_back(3.14 * i);  // 안전, 예외 발생 안 함
    }
    
    // 3. 크기 제한 명확
    static_assert(std::inplace_vector<int, 50>::static_capacity == 50);
    
    // 4. 스택 저장으로 자동 정리
    // 함수 종료 시 자동으로 소멸, 메모리 누수 불가능
}
```

### std::vector와의 비교
두 컨테이너의 차이를 명확히 보여주는 예제다.

```cpp
#include <vector>
#include <inplace_vector>
#include <chrono>

// std::vector 사용 - 동적 할당
double benchmark_vector() {
    auto start = std::chrono::high_resolution_clock::now();
    
    for (int iteration = 0; iteration < 10000; ++iteration) {
        std::vector<int> v;
        v.reserve(20);  // 힙 할당 발생
        
        for (int i = 0; i < 20; ++i) {
            v.push_back(i);
        }
        
        int sum = 0;
        for (int val : v) {
            sum += val;
        }
    }
    
    auto end = std::chrono::high_resolution_clock::now();
    return std::chrono::duration<double, std::milli>(end - start).count();
}

// std::inplace_vector 사용 - 스택 할당
double benchmark_inplace_vector() {
    auto start = std::chrono::high_resolution_clock::now();
    
    for (int iteration = 0; iteration < 10000; ++iteration) {
        std::inplace_vector<int, 20> v;  // 스택 할당, 초기화 비용 없음
        
        for (int i = 0; i < 20; ++i) {
            v.push_back(i);
        }
        
        int sum = 0;
        for (int val : v) {
            sum += val;
        }
    }
    
    auto end = std::chrono::high_resolution_clock::now();
    return std::chrono::duration<double, std::milli>(end - start).count();
}

void compare_performance() {
    double vector_time = benchmark_vector();
    double inplace_time = benchmark_inplace_vector();
    
    std::cout << "std::vector 시간: " << vector_time << "ms\n";
    std::cout << "std::inplace_vector 시간: " << inplace_time << "ms\n";
    std::cout << "성능 향상: " << (vector_time / inplace_time) << "배\n";
}
```
  

## 10.2 크기 제한과 오버플로우 방지

### 용량 초과 시 안전한 처리
`std::inplace_vector`는 용량 초과를 안전하게 처리할 수 있는 여러 메커니즘을 제공한다.

```cpp
#include <inplace_vector>
#include <optional>
#include <iostream>

// 안전하지 않은 예: 용량 확인 없이 추가
void unsafe_add(std::inplace_vector<int, 10>& vec, int value) {
    vec.push_back(value);  // 용량 초과 시 std::bad_alloc 발생
}

// 안전한 예 1: 추가 전 용량 확인
bool safe_add_with_check(std::inplace_vector<int, 10>& vec, int value) {
    if (vec.size() < vec.capacity()) {
        vec.push_back(value);
        return true;
    }
    return false;  // 용량 부족 명시적 표시
}

// 안전한 예 2: try_push_back 사용 (C++26)
std::optional<int> safe_add_with_try(
    std::inplace_vector<int, 10>& vec, 
    int value
) {
    if (auto result = vec.try_push_back(value)) {
        return value;
    }
    return std::nullopt;  // 실패 시 명시적 반환
}

// 안전한 예 3: 범위 기반 안전 삽입
template<std::ranges::range R>
size_t safe_append_range(
    std::inplace_vector<int, 100>& vec,
    R&& range
) {
    size_t added = 0;
    size_t available = vec.capacity() - vec.size();
    
    for (int value : range) {
        if (added >= available) {
            break;  // 용량 제한 존중
        }
        vec.push_back(value);
        ++added;
    }
    
    return added;  // 실제 추가된 요소 수 반환
}
```

### 컴파일 타임 크기 검증
템플릿과 `static_assert`를 활용하여 컴파일 타임에 크기를 검증할 수 있다.

```cpp
#include <inplace_vector>
#include <array>

// 컴파일 타임 크기 검증
template<size_t N>
class FixedQueue {
    std::inplace_vector<int, N> data;
    
public:
    static_assert(N > 0, "큐 크기는 0보다 커야 한다");
    static_assert(N <= 1000, "큐 크기가 너무 크다 (스택 오버플로우 위험)");
    
    bool enqueue(int value) {
        if (data.size() < N) {
            data.push_back(value);
            return true;
        }
        return false;
    }
    
    std::optional<int> dequeue() {
        if (data.empty()) {
            return std::nullopt;
        }
        
        int value = data.front();
        data.erase(data.begin());
        return value;
    }
    
    constexpr size_t max_size() const noexcept {
        return N;
    }
    
    size_t size() const noexcept {
        return data.size();
    }
};

// 사용 예
void test_fixed_queue() {
    FixedQueue<20> queue;
    
    // 크기 제한 내에서 안전하게 작동
    for (int i = 0; i < 20; ++i) {
        bool success = queue.enqueue(i);
        assert(success);
    }
    
    // 용량 초과 시 실패 반환
    bool overflow = queue.enqueue(999);
    assert(!overflow);  // 예상된 실패
}
```

### 동적 배열 vs 고정 배열 선택 가이드
언제 `std::vector`를 사용하고 언제 `std::inplace_vector`를 사용해야 하는지 판단하는 기준을 제시한다.

```cpp
#include <inplace_vector>
#include <vector>
#include <string>

// std::inplace_vector가 적합한 경우
class Parser {
    // 1. 최대 크기가 명확하고 작은 경우
    std::inplace_vector<char, 256> token_buffer;
    
    // 2. 임시 버퍼로 사용하는 경우
    std::inplace_vector<int, 32> temp_indices;

    void parse_token_realtime(std::string_view input) {

        for (char c : input) {
            if (token_buffer.size() < token_buffer.capacity()) {
                token_buffer.push_back(c);
            } else {
                break;  // 안전하게 제한
            }
        }
    }
};

// std::vector가 적합한 경우
class DynamicCollection {
    // 1. 크기를 예측할 수 없는 경우
    std::vector<std::string> user_messages;
    
    // 2. 크기가 매우 클 수 있는 경우
    std::vector<double> large_dataset;

        user_messages.resize(count);
    }

// 하이브리드 접근: 작은 경우 스택, 큰 경우 힙
template<typename T, size_t SmallSize = 16>
class SmallVector {
    std::inplace_vector<T, SmallSize> small_storage;
    std::vector<T> large_storage;
    bool using_heap = false;
    
public:
    void push_back(const T& value) {
        if (!using_heap && small_storage.size() < SmallSize) {
            small_storage.push_back(value);
        } else {
            if (!using_heap) {
                // 스택에서 힙으로 전환
                large_storage.assign(
                    small_storage.begin(), 
                    small_storage.end()
                );
                using_heap = true;
            }
            large_storage.push_back(value);
        }
    }
    
    size_t size() const noexcept {
        return using_heap ? large_storage.size() : small_storage.size();
    }
};
```

## 10.3 성능과 안전성의 균형

### 메모리 지역성과 캐시 효율
`std::inplace_vector`는 스택 할당으로 인해 우수한 캐시 지역성을 제공한다.

```cpp
#include <inplace_vector>
#include <vector>
#include <numeric>
#include <chrono>

// 캐시 친화적인 처리
struct Point3D {
    double x, y, z;
    
    double magnitude() const {
        return std::sqrt(x*x + y*y + z*z);
    }
};

// std::vector: 힙에 분산된 데이터
double process_heap_points(const std::vector<Point3D>& points) {
    double sum = 0.0;
    
    for (const auto& point : points) {
        sum += point.magnitude();  // 캐시 미스 가능성 높음
    }
    
    return sum;
}

// std::inplace_vector: 스택에 연속된 데이터
double process_stack_points(const std::inplace_vector<Point3D, 100>& points) {
    double sum = 0.0;
    
    for (const auto& point : points) {
        sum += point.magnitude();  // 캐시 히트 가능성 높음
    }
    
    return sum;
}

void benchmark_cache_locality() {
    const int iterations = 100000;
    
    // 힙 기반 벤치마크
    std::vector<Point3D> heap_points(50);
    for (auto& p : heap_points) {
        p = {1.0, 2.0, 3.0};
    }
    
    auto start_heap = std::chrono::high_resolution_clock::now();
    volatile double heap_result = 0.0;
    for (int i = 0; i < iterations; ++i) {
        heap_result += process_heap_points(heap_points);
    }
    auto end_heap = std::chrono::high_resolution_clock::now();
    
    // 스택 기반 벤치마크
    std::inplace_vector<Point3D, 100> stack_points;
    for (int i = 0; i < 50; ++i) {
        stack_points.push_back({1.0, 2.0, 3.0});
    }
    
    auto start_stack = std::chrono::high_resolution_clock::now();
    volatile double stack_result = 0.0;
    for (int i = 0; i < iterations; ++i) {
        stack_result += process_stack_points(stack_points);
    }
    auto end_stack = std::chrono::high_resolution_clock::now();
    
    auto heap_time = std::chrono::duration<double, std::micro>(
        end_heap - start_heap
    ).count();
    auto stack_time = std::chrono::duration<double, std::micro>(
        end_stack - start_stack
    ).count();
    
    std::cout << "힙 기반 처리 시간: " << heap_time << "μs\n";
    std::cout << "스택 기반 처리 시간: " << stack_time << "μs\n";
    std::cout << "성능 향상: " << (heap_time / stack_time) << "배\n";
}
```

### 예외 안전성 보장
`std::inplace_vector`는 동적 할당이 없어 강력한 예외 안전성을 제공한다.

```cpp
#include <inplace_vector>
#include <stdexcept>

class SafeProcessor {
    std::inplace_vector<int, 50> results;
    
public:
    // 강력한 예외 보장: 실패 시 상태 변경 없음
    void process_batch(const std::inplace_vector<int, 50>& inputs) {
        std::inplace_vector<int, 50> temp;  // 스택 할당, 예외 불가
        
        for (int value : inputs) {
            if (value < 0) {
                throw std::invalid_argument("음수는 허용되지 않는다");
                // temp는 자동으로 소멸, 메모리 누수 없음
            }
            
            // 용량 체크 - 예외 없는 실패 처리
            if (temp.size() < temp.capacity()) {
                temp.push_back(value * 2);
            } else {
                throw std::length_error("처리할 데이터가 너무 많다");
            }
        }
        
        // 모든 처리가 성공한 경우에만 결과 저장
        results = std::move(temp);  // noexcept 이동
    }
    
    const std::inplace_vector<int, 50>& get_results() const noexcept {
        return results;
    }
};

void test_exception_safety() {
    SafeProcessor processor;
    
    try {
        std::inplace_vector<int, 50> valid_input;
        for (int i = 0; i < 10; ++i) {
            valid_input.push_back(i);
        }
        
        processor.process_batch(valid_input);
        assert(processor.get_results().size() == 10);
        
        // 잘못된 입력으로 예외 발생
        std::inplace_vector<int, 50> invalid_input;
        invalid_input.push_back(-5);
        
        processor.process_batch(invalid_input);  // 예외 발생
        
    } catch (const std::invalid_argument& e) {
        // 예외 발생 시에도 이전 상태 유지
        assert(processor.get_results().size() == 10);
        std::cout << "예외 처리 완료: " << e.what() << "\n";
    }
}
```

### 실시간 시스템에서의 활용
실시간 제약이 있는 시스템에서 `std::inplace_vector`의 활용 예다.

```cpp
#include <inplace_vector>
#include <chrono>
#include <iostream>

// 실시간 제약: 10ms 이내 처리 필요
class RealtimeController {
    static constexpr size_t MAX_SAMPLES = 1000;
    std::inplace_vector<double, MAX_SAMPLES> sensor_data;
    
public:
    // 동적 할당 없는 데이터 수집 - 결정론적 시간 보장
    void collect_sample(double value) noexcept {
        if (sensor_data.size() < MAX_SAMPLES) {
            sensor_data.push_back(value);  // O(1), 할당 없음
        } else {
            // 오래된 데이터 제거 (링 버퍼 동작)
            sensor_data.erase(sensor_data.begin());
            sensor_data.push_back(value);
        }
    }
    
    // 빠른 통계 계산 - 캐시 친화적
    double calculate_average() const noexcept {
        if (sensor_data.empty()) {
            return 0.0;
        }
        
        double sum = 0.0;
        for (double value : sensor_data) {
            sum += value;  // 연속 메모리 접근, 캐시 효율 우수
        }
        
        return sum / sensor_data.size();
    }
    
    // 이상치 탐지 - 예측 가능한 실행 시간
    size_t detect_anomalies(double threshold) const noexcept {
        double avg = calculate_average();
        size_t count = 0;
        
        for (double value : sensor_data) {
            if (std::abs(value - avg) > threshold) {
                ++count;
            }
        }
        
        return count;
    }
};

void test_realtime_performance() {
    RealtimeController controller;
    
    auto start = std::chrono::steady_clock::now();
    
    // 1000개 샘플 수집 및 처리
    for (int i = 0; i < 1000; ++i) {
        controller.collect_sample(100.0 + (i % 20));
    }
    
    double avg = controller.calculate_average();
    size_t anomalies = controller.detect_anomalies(15.0);
    
    auto end = std::chrono::steady_clock::now();
    auto duration = std::chrono::duration<double, std::milli>(
        end - start
    ).count();
    
    std::cout << "처리 시간: " << duration << "ms\n";
    std::cout << "평균값: " << avg << "\n";
    std::cout << "이상치 수: " << anomalies << "\n";
    
    // 실시간 제약 확인
    assert(duration < 10.0);  // 10ms 이내 완료
}
```

### 실전 응용: 고성능 로깅 시스템
`std::inplace_vector`를 활용한 실용적인 로깅 시스템 구현이다.

```cpp
#include <inplace_vector>
#include <string_view>
#include <chrono>
#include <iostream>
#include <fstream>

enum class LogLevel {
    Debug,
    Info,
    Warning,
    Error
};

struct LogEntry {
    std::chrono::system_clock::time_point timestamp;
    LogLevel level;
    char message[128];  // 고정 크기 메시지
    
    LogEntry(LogLevel lvl, std::string_view msg) 
        : timestamp(std::chrono::system_clock::now())
        , level(lvl) 
    {
        size_t copy_len = std::min(msg.size(), sizeof(message) - 1);
        msg.copy(message, copy_len);
        message[copy_len] = '\0';
    }
};

class FastLogger {
    static constexpr size_t BUFFER_SIZE = 100;
    std::inplace_vector<LogEntry, BUFFER_SIZE> buffer;
    
public:
    // 빠른 로그 추가 - 동적 할당 없음
    bool log(LogLevel level, std::string_view message) noexcept {
        if (buffer.size() >= BUFFER_SIZE) {
            flush();  // 버퍼가 차면 비우기
        }
        
        if (buffer.size() < BUFFER_SIZE) {
            buffer.emplace_back(level, message);
            return true;
        }
        
        return false;
    }
    
    // 버퍼 비우기
    void flush() {
        if (buffer.empty()) {
            return;
        }
        
        // 실제 로그 출력 (파일, 네트워크 등)
        for (const auto& entry : buffer) {
            print_entry(entry);
        }
        
        buffer.clear();  // 스택 메모리, 즉시 재사용 가능
    }
    
    ~FastLogger() {
        flush();  // 소멸 시 남은 로그 출력
    }
    
private:
    void print_entry(const LogEntry& entry) const {
        const char* level_str = "UNKNOWN";
        switch (entry.level) {
            case LogLevel::Debug:   level_str = "DEBUG"; break;
            case LogLevel::Info:    level_str = "INFO"; break;
            case LogLevel::Warning: level_str = "WARN"; break;
            case LogLevel::Error:   level_str = "ERROR"; break;
        }
        
        std::cout << "[" << level_str << "] " << entry.message << "\n";
    }
};

void test_fast_logger() {
    FastLogger logger;
    
    // 대량 로깅 - 할당 없이 빠르게 처리
    auto start = std::chrono::high_resolution_clock::now();
    
    for (int i = 0; i < 1000; ++i) {
        logger.log(LogLevel::Info, "처리 중...");
        
        if (i % 100 == 0) {
            logger.log(LogLevel::Debug, "진행 상황 체크포인트");
        }
    }
    
    auto end = std::chrono::high_resolution_clock::now();
    auto duration = std::chrono::duration<double, std::micro>(
        end - start
    ).count();
    
    std::cout << "1000개 로그 처리 시간: " << duration << "μs\n";
    std::cout << "로그당 평균 시간: " << (duration / 1000.0) << "μs\n";
}
```

### 요약 및 모범 사례
`std::inplace_vector`를 효과적으로 사용하기 위한 가이드라인이다.

```cpp
#include <inplace_vector>

// 모범 사례 1: 적절한 크기 선택
template<size_t N>
class GoodPractice {
    // ✓ 작은 크기: 스택 사용량 최소화
    std::inplace_vector<int, 32> small_buffer;
    
    // ✗ 너무 큰 크기: 스택 오버플로우 위험
    // std::inplace_vector<int, 1000000> too_large;  // 피해야 함
    
public:
    // 모범 사례 2: 용량 확인 후 추가
    bool safe_add(int value) {
        if (small_buffer.size() < small_buffer.capacity()) {
            small_buffer.push_back(value);
            return true;
        }
        return false;
    }
    
    // 모범 사례 3: noexcept 함수에서 사용
    size_t count_positive() const noexcept {
        size_t count = 0;
        for (int value : small_buffer) {
            if (value > 0) {
                ++count;
            }
        }
        return count;
    }
};

// 언제 std::inplace_vector를 사용할 것인가?
namespace UseCases {
    // ✓ 좋은 사용 사례
    void good_use_cases() {
        // 1. 최대 크기가 명확하고 작음
        std::inplace_vector<char, 64> command_buffer;
        
        // 2. 임시 계산 버퍼
        std::inplace_vector<double, 16> temp_values;
        
        // 3. 실시간/임베디드 시스템
        std::inplace_vector<int, 32> sensor_readings;
        
        // 4. 성능 크리티컬 경로
        std::inplace_vector<float, 8> transform_matrix;
    }
    
    // ✗ 피해야 할 사용 사례
    void bad_use_cases() {
        // 1. 크기를 예측할 수 없음
        // std::inplace_vector<std::string, ???> user_input;
        
        // 2. 매우 큰 데이터
        // std::inplace_vector<Image, 10000> image_cache;
        
        // 3. 크기가 런타임에 결정됨
        // std::inplace_vector<Data, ???> dynamic_data;
    }
}
```

`std::inplace_vector`는 Rust의 고정 크기 배열과 유사한 안전성을 C++에 제공한다. 동적 할당을 제거함으로써 메모리 안전성, 예측 가능한 성능, 예외 안전성을 동시에 달성할 수 있다. 적절한 상황에서 사용하면 성능과 안전성을 모두 향상시킬 수 있는 강력한 도구다.
