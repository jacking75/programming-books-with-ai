# 3장: 바이너리 프로토콜 설계
이제부터는 실제로 게임에서 사용할 프로토콜을 설계하고 구현해보자. 2장에서 배운 바이너리 직렬화의 개념을 실제 코드로 옮겨서, 서버와 클라이언트 사이에 데이터를 주고받을 수 있게 만들 것이다.
  

## 3.1 패킷 구조 설계
게임이 복잡해질수록 다양한 종류의 정보를 주고받아야 한다. 플레이어 위치, 체력, 공격, 채팅 등 여러 종류의 메시지가 섞여서 전송된다. 이들을 구분하고 올바르게 처리하려면 체계적인 패킷 구조가 필요하다.

### 패킷의 기본 구조
모든 패킷이 공통적으로 가져야 할 헤더를 먼저 정의하자. 헤더는 패킷의 "명함"이라고 생각하면 된다. 누가 보냈는지, 어떤 종류의 메시지인지, 얼마나 큰지를 알려준다.

```
패킷 헤더 구조:

┌──────────────┬──────────────┬──────────────┐
│  패킷 타입    │ 패킷 길이      │  페이로드     │
│  (1 바이트)   │ (2 바이트)    │  (가변 길이)  │
└──────────────┴──────────────┴──────────────┘
```

**패킷 타입**은 이 패킷이 무엇을 의미하는지를 나타낸다. 0은 로그인 요청, 1은 위치 업데이트, 2는 채팅 메시지 같은 식으로 정의한다. 1바이트로는 최대 255개까지 정의할 수 있으므로 대부분의 게임에 충분하다.

**패킷 길이**는 페이로드(실제 데이터)의 크기를 나타낸다. 2바이트면 0부터 65535까지의 크기를 표현할 수 있으므로, 초소형 게임부터 대규모 게임까지 커버할 수 있다.

**페이로드**는 실제 데이터로, 패킷 타입에 따라 내용이 달라진다.

### 패킷 타입 정의
게임에서 사용할 패킷 타입들을 먼저 정의해두자. 이것이 게임의 프로토콜이 된다.

```csharp
/// <summary>
/// 네트워크에서 사용할 패킷 타입을 정의합니다.
/// 서버와 클라이언트가 이 타입들을 공유해야 하므로,
/// 일반적으로 별도의 파일에서 정의합니다.
/// </summary>
public static class PacketType
{
    // 연결 관련
    public const byte Login = 0;           // 로그인 요청
    public const byte LoginResponse = 1;   // 로그인 응답
    
    // 게임 상태 관련
    public const byte PositionUpdate = 2;  // 플레이어 위치 업데이트
    public const byte PlayerJoined = 3;    // 새 플레이어 입장
    public const byte PlayerLeft = 4;      // 플레이어 퇴장
    
    // 통신 관련
    public const byte Chat = 5;            // 채팅 메시지
    public const byte Ping = 6;            // 연결 확인 신호
    public const byte Pong = 7;            // 연결 확인 응답
}
```

이렇게 상수로 정의하면 코드에서 타입을 사용할 때 매직 넘버(의미 모를 숫자)를 피할 수 있고, 실수로 잘못된 숫자를 쓸 가능성도 줄어든다.

### 패킷 구조를 정의하는 방법
이제 각 패킷 타입별로 페이로드의 구조를 정의해야 한다. 다음과 같이 표로 정리하면 체계적이다.

**로그인 요청 패킷 (PacketType.Login)**

| 필드 | 타입 | 크기 | 설명 |
|------|------|------|------|
| 플레이어 이름 | string | 가변 | 게임에서 사용할 이름 |

예: "철수"라는 플레이어가 로그인하려고 한다. 이 경우 패킷은 다음과 같이 구성된다.

```
헤더: [패킷타입: 0] [길이: 5]
페이로드: ["철수" (5바이트, string의 길이 정보 포함)]
전체 패킷 바이트: 0x00, 0x05, 0x00, ... (실제로는 더 복잡함)
```

**플레이어 위치 업데이트 패킷 (PacketType.PositionUpdate)**

| 필드 | 타입 | 크기 | 설명 |
|------|------|------|------|
| 플레이어 ID | int | 4 | 어떤 플레이어인지 구분 |
| X 좌표 | float | 4 | 화면 상의 X 좌표 |
| Y 좌표 | float | 4 | 화면 상의 Y 좌표 |

**채팅 메시지 패킷 (PacketType.Chat)**

| 필드 | 타입 | 크기 | 설명 |
|------|------|------|------|
| 플레이어 ID | int | 4 | 누가 말했는가 |
| 메시지 | string | 가변 | 실제 채팅 내용 |

이렇게 정의해두면, 나중에 패킷을 만들거나 받을 때 정확히 어떤 순서로 데이터를 읽고 써야 하는지 알 수 있다.

### 패킷 설계 시 주의사항
패킷을 설계할 때 몇 가지 주의할 점이 있다.

**1. 바이트 정렬(Alignment)**

일부 시스템에서는 데이터가 특정 바운더리에 정렬되어야 한다. 하지만 우리가 바이너리 데이터를 직접 제어하면, 이런 정렬을 무시할 수 있다. 그래서 바이트 단위로 정확히 패킷을 만들 수 있다.

**2. 버전 관리**

게임이 업데이트되면서 패킷 형식이 바뀔 수 있다. 이 경우를 대비해서, 처음부터 패킷에 버전 번호를 포함시키는 것이 좋다. 하지만 이 책의 간단한 게임에서는 생략하겠다.

**3. 효율성**

패킷을 자주 보내므로, 꼭 필요한 데이터만 포함시켜야 한다. 예를 들어, 게임 시작할 때만 보내는 정보와 매 프레임 보내는 정보는 분리하는 것이 좋다.

**4. 확장성**

나중에 기능을 추가할 때를 대비해서, 예비 필드를 두는 것도 고려할 수 있다. 하지만 초기에는 필요한 것만 넣는 것이 낫다.

## 3.2 BinaryPrimitives를 이용한 직렬화
2장에서 `BinaryWriter`와 `BinaryReader`를 배웠다면, 이번에는 더 효율적인 방법인 `BinaryPrimitives`를 사용해보자. `BinaryPrimitives`는 .NET에서 제공하는 클래스로, 직접 바이트 배열에 데이터를 읽고 쓸 수 있게 해준다.

### BinaryPrimitives의 장점
`BinaryWriter`는 스트림을 사용하기 때문에 메모리 할당이 여러 번 일어난다. 하지만 `BinaryPrimitives`는 미리 할당된 바이트 배열에 직접 데이터를 쓰므로 더 효율적이다. 게임에서는 매초 수십 개의 패킷이 오가므로, 이런 효율성이 중요하다.

```csharp
using System;
using System.Buffers.Binary;

// BinaryPrimitives 사용 예제
byte[] buffer = new byte[12];  // 4 + 4 + 4 = 12바이트 미리 할당

// 데이터 쓰기
int playerId = 42;
float x = 100.5f;
float y = 200.3f;

// offset 0부터 playerId를 int(4바이트)로 쓴다
BinaryPrimitives.WriteInt32LittleEndian(buffer.AsSpan(0), playerId);
// offset 4부터 x를 float(4바이트)로 쓴다
BinaryPrimitives.WriteSingleLittleEndian(buffer.AsSpan(4), x);
// offset 8부터 y를 float(4바이트)로 쓴다
BinaryPrimitives.WriteSingleLittleEndian(buffer.AsSpan(8), y);

// 데이터 읽기
int readId = BinaryPrimitives.ReadInt32LittleEndian(buffer.AsSpan(0));
float readX = BinaryPrimitives.ReadSingleLittleEndian(buffer.AsSpan(4));
float readY = BinaryPrimitives.ReadSingleLittleEndian(buffer.AsSpan(8));
```

### Endian(엔디안)이란
위의 코드에서 `LittleEndian`이라는 단어를 봤을 것이다. 이는 바이트의 순서를 정하는 방식이다.

숫자 42를 4바이트 int로 표현한다고 하자. 16진수로 0x0000002A다.

**Little Endian** (작은 것부터): `2A 00 00 00`
**Big Endian** (큰 것부터): `00 00 00 2A`

어느 것이 맞다는 것은 없고, 일관되기만 하면 된다. 다행히 대부분의 현대 컴퓨터와 .NET은 Little Endian을 사용한다. 따라서 `LittleEndian` 메서드를 사용하면 된다.

### 패킷 클래스 만들기
이제 실제로 패킷을 만들고 읽는 클래스를 작성해보자. 먼저 기본이 되는 `Packet` 클래스를 만든다.

```csharp
/// <summary>
/// 모든 패킷이 상속할 기본 클래스입니다.
/// 패킷의 타입과 직렬화/역직렬화 로직을 정의합니다.
/// </summary>
public abstract class Packet
{
    /// <summary>
    /// 이 패킷의 타입을 반환합니다.
    /// 파생 클래스에서 반드시 오버라이드해야 합니다.
    /// </summary>
    public abstract byte PacketType { get; }

    /// <summary>
    /// 이 패킷을 바이트 배열로 변환합니다 (직렬화).
    /// </summary>
    public abstract byte[] Serialize();

    /// <summary>
    /// 바이트 배열로부터 패킷을 복원합니다 (역직렬화).
    /// payload는 헤더를 제외한 순수 데이터입니다.
    /// </summary>
    public abstract void Deserialize(byte[] payload);
}
```

이제 구체적인 패킷들을 만들어보자. 먼저 로그인 요청 패킷이다.

```csharp
/// <summary>
/// 플레이어가 로그인할 때 보내는 패킷입니다.
/// 페이로드: [플레이어 이름 (string)]
/// </summary>
public class LoginPacket : Packet
{
    public string PlayerName { get; set; }

    public override byte PacketType => PacketType.Login;

    public LoginPacket() { }

    public LoginPacket(string playerName)
    {
        PlayerName = playerName;
    }

    /// <summary>
    /// LoginPacket을 바이트 배열로 변환합니다.
    /// 로직: string을 인코딩하여 바이트로 변환하고,
    /// 앞에 문자열의 길이를 정보로 붙입니다.
    /// </summary>
    public override byte[] Serialize()
    {
        // 플레이어 이름을 UTF-8로 인코딩
        byte[] nameBytes = System.Text.Encoding.UTF8.GetBytes(PlayerName);
        
        // 길이(int, 4바이트) + 이름 데이터
        byte[] payload = new byte[4 + nameBytes.Length];
        
        // 0-3번 바이트: 문자열 길이 (Little Endian)
        BinaryPrimitives.WriteInt32LittleEndian(payload.AsSpan(0), nameBytes.Length);
        
        // 4번부터: 실제 문자열 데이터
        nameBytes.CopyTo(payload, 4);
        
        return payload;
    }

    /// <summary>
    /// 바이트 배열로부터 LoginPacket을 복원합니다.
    /// 로직: 먼저 길이를 읽고, 그 길이만큼 문자열을 읽습니다.
    /// </summary>
    public override void Deserialize(byte[] payload)
    {
        // 0-3번 바이트: 문자열 길이 읽기
        int nameLength = BinaryPrimitives.ReadInt32LittleEndian(payload.AsSpan(0));
        
        // 4번부터 nameLength만큼: 실제 문자열 읽기
        PlayerName = System.Text.Encoding.UTF8.GetString(payload, 4, nameLength);
    }
}
```

다음은 위치 업데이트 패킷이다.

```csharp
/// <summary>
/// 플레이어의 위치를 업데이트하는 패킷입니다.
/// 페이로드: [플레이어 ID (int)] [X 좌표 (float)] [Y 좌표 (float)]
/// 전체 크기: 4 + 4 + 4 = 12바이트 (고정 크기)
/// </summary>
public class PositionUpdatePacket : Packet
{
    public int PlayerId { get; set; }
    public float X { get; set; }
    public float Y { get; set; }

    public override byte PacketType => PacketType.PositionUpdate;

    public PositionUpdatePacket() { }

    public PositionUpdatePacket(int playerId, float x, float y)
    {
        PlayerId = playerId;
        X = x;
        Y = y;
    }

    /// <summary>
    /// PositionUpdatePacket을 바이트 배열로 변환합니다.
    /// 고정 크기 패킷이므로 항상 12바이트입니다.
    /// 순서: PlayerId(4) -> X(4) -> Y(4)
    /// </summary>
    public override byte[] Serialize()
    {
        byte[] payload = new byte[12];
        
        // 0-3: PlayerId (int)
        BinaryPrimitives.WriteInt32LittleEndian(payload.AsSpan(0), PlayerId);
        
        // 4-7: X (float)
        BinaryPrimitives.WriteSingleLittleEndian(payload.AsSpan(4), X);
        
        // 8-11: Y (float)
        BinaryPrimitives.WriteSingleLittleEndian(payload.AsSpan(8), Y);
        
        return payload;
    }

    /// <summary>
    /// 바이트 배열로부터 PositionUpdatePacket을 복원합니다.
    /// 고정 크기이므로 정확히 12바이트가 들어올 것으로 예상합니다.
    /// </summary>
    public override void Deserialize(byte[] payload)
    {
        // 0-3: PlayerId
        PlayerId = BinaryPrimitives.ReadInt32LittleEndian(payload.AsSpan(0));
        
        // 4-7: X
        X = BinaryPrimitives.ReadSingleLittleEndian(payload.AsSpan(4));
        
        // 8-11: Y
        Y = BinaryPrimitives.ReadSingleLittleEndian(payload.AsSpan(8));
    }
}
```

마지막으로 채팅 메시지 패킷이다.

```csharp
/// <summary>
/// 채팅 메시지를 보내는 패킷입니다.
/// 페이로드: [플레이어 ID (int)] [메시지 (string)]
/// </summary>
public class ChatPacket : Packet
{
    public int PlayerId { get; set; }
    public string Message { get; set; }

    public override byte PacketType => PacketType.Chat;

    public ChatPacket() { }

    public ChatPacket(int playerId, string message)
    {
        PlayerId = playerId;
        Message = message;
    }

    /// <summary>
    /// ChatPacket을 바이트 배열로 변환합니다.
    /// 순서: PlayerId(4) -> Message길이(4) -> Message 데이터
    /// 가변 크기 패킷입니다.
    /// </summary>
    public override byte[] Serialize()
    {
        byte[] messageBytes = System.Text.Encoding.UTF8.GetBytes(Message);
        
        // PlayerId(4) + 메시지길이(4) + 메시지데이터
        byte[] payload = new byte[4 + 4 + messageBytes.Length];
        
        // 0-3: PlayerId
        BinaryPrimitives.WriteInt32LittleEndian(payload.AsSpan(0), PlayerId);
        
        // 4-7: 메시지 길이
        BinaryPrimitives.WriteInt32LittleEndian(payload.AsSpan(4), messageBytes.Length);
        
        // 8부터: 실제 메시지 데이터
        messageBytes.CopyTo(payload, 8);
        
        return payload;
    }

    /// <summary>
    /// 바이트 배열로부터 ChatPacket을 복원합니다.
    /// 메시지 길이를 먼저 읽고, 그 길이만큼 메시지를 읽습니다.
    /// </summary>
    public override void Deserialize(byte[] payload)
    {
        // 0-3: PlayerId
        PlayerId = BinaryPrimitives.ReadInt32LittleEndian(payload.AsSpan(0));
        
        // 4-7: 메시지 길이
        int messageLength = BinaryPrimitives.ReadInt32LittleEndian(payload.AsSpan(4));
        
        // 8부터: 메시지 데이터
        Message = System.Text.Encoding.UTF8.GetString(payload, 8, messageLength);
    }
}
```

### 패킷 팩토리 만들기
이제 바이트 배열로부터 올바른 타입의 패킷 객체를 만드는 팩토리를 만들어보자. 패킷의 타입을 읽고, 그에 맞는 패킷 객체를 생성하는 것이다.

```csharp
/// <summary>
/// 바이트 데이터로부터 패킷 객체를 생성하는 팩토리입니다.
/// 패킷 타입을 읽고 올바른 패킷 클래스를 선택합니다.
/// </summary>
public static class PacketFactory
{
    /// <summary>
    /// 패킷 헤더를 읽습니다.
    /// 바이트 배열의 처음 3바이트를 읽어서 타입과 길이를 반환합니다.
    /// 반환값: (패킷타입, 페이로드길이)
    /// </summary>
    public static (byte type, int length) ReadHeader(byte[] data)
    {
        if (data.Length < 3)
            throw new ArgumentException("패킷 데이터가 너무 작습니다.");
        
        byte type = data[0];
        int length = BinaryPrimitives.ReadInt16LittleEndian(data.AsSpan(1));
        
        return (type, length);
    }

    /// <summary>
    /// 패킷 헤더를 만듭니다.
    /// 패킷 타입과 페이로드를 받아서 헤더를 생성합니다.
    /// </summary>
    public static byte[] CreateHeader(byte packetType, byte[] payload)
    {
        byte[] header = new byte[3 + payload.Length];
        
        header[0] = packetType;
        BinaryPrimitives.WriteInt16LittleEndian(header.AsSpan(1), (short)payload.Length);
        payload.CopyTo(header, 3);
        
        return header;
    }

    /// <summary>
    /// 바이트 배열로부터 패킷 객체를 생성합니다.
    /// 페이로드(헤더를 제외한 데이터)를 받아서 적절한 Packet 객체를 반환합니다.
    /// </summary>
    public static Packet CreatePacket(byte packetType, byte[] payload)
    {
        Packet packet = null;

        switch (packetType)
        {
            case PacketType.Login:
                packet = new LoginPacket();
                break;

            case PacketType.PositionUpdate:
                packet = new PositionUpdatePacket();
                break;

            case PacketType.Chat:
                packet = new ChatPacket();
                break;

            default:
                throw new InvalidOperationException(
                    $"알 수 없는 패킷 타입입니다: {packetType}");
        }

        // 패킷 객체에 페이로드를 역직렬화
        packet.Deserialize(payload);
        return packet;
    }
}
```

위의 코드를 설명하면, `ReadHeader` 메서드는 패킷의 첫 3바이트를 읽는다. 첫 바이트는 타입, 다음 2바이트는 길이다. 이렇게 헤더를 읽으면, 실제 페이로드 데이터가 몇 바이트인지 알 수 있어서, 정확히 그만큼 읽을 수 있다.

`CreateHeader` 메서드는 반대로 타입과 페이로드를 받아서 헤더를 만든다. 헤더와 페이로드를 합쳐서 네트워크로 보낼 완전한 패킷을 만드는 것이다.

`CreatePacket` 메서드는 타입을 보고 올바른 Packet 클래스의 인스턴스를 만들고, 페이로드를 역직렬화한다.

## 3.3 패킷 송수신 기본 구현
이제 실제로 네트워크를 통해 패킷을 보내고 받는 코드를 구현해보자.

### 패킷 전송 구현
먼저 클라이언트가 서버로 패킷을 보내는 방법이다.

```csharp
using System;
using System.Net.Sockets;

/// <summary>
/// 게임 클라이언트의 네트워크 부분을 담당합니다.
/// TCP 소켓을 사용하여 서버와 통신합니다.
/// </summary>
public class GameNetworkClient
{
    private TcpClient _tcpClient;
    private NetworkStream _stream;
    private const int ReceiveBufferSize = 4096;

    public GameNetworkClient()
    {
        _tcpClient = null;
        _stream = null;
    }

    /// <summary>
    /// 서버에 연결합니다.
    /// serverAddress: 서버의 IP 주소 (예: "127.0.0.1")
    /// serverPort: 서버의 포트 번호 (예: 9000)
    /// </summary>
    public void Connect(string serverAddress, int serverPort)
    {
        try
        {
            _tcpClient = new TcpClient();
            _tcpClient.Connect(serverAddress, serverPort);
            _stream = _tcpClient.GetStream();
            Console.WriteLine($"서버 {serverAddress}:{serverPort}에 연결되었습니다.");
        }
        catch (Exception ex)
        {
            Console.WriteLine($"연결 실패: {ex.Message}");
            throw;
        }
    }

    /// <summary>
    /// 패킷을 서버로 전송합니다.
    /// 이 메서드는 동기 방식으로 작동합니다 (전송이 완료될 때까지 대기).
    /// 나중에는 비동기 방식으로 바꿀 것입니다.
    /// </summary>
    public void SendPacket(Packet packet)
    {
        if (_stream == null)
        {
            Console.WriteLine("서버에 연결되어 있지 않습니다.");
            return;
        }

        try
        {
            // 패킷의 페이로드를 직렬화
            byte[] payload = packet.Serialize();
            
            // 헤더를 포함한 완전한 패킷 데이터 생성
            byte[] fullPacket = PacketFactory.CreateHeader(packet.PacketType, payload);
            
            // 네트워크 스트림으로 전송
            _stream.Write(fullPacket, 0, fullPacket.Length);
            _stream.Flush();
            
            Console.WriteLine($"패킷 전송: 타입={packet.PacketType}, 크기={fullPacket.Length}바이트");
        }
        catch (Exception ex)
        {
            Console.WriteLine($"패킷 전송 실패: {ex.Message}");
        }
    }

    /// <summary>
    /// 서버와의 연결을 종료합니다.
    /// </summary>
    public void Disconnect()
    {
        try
        {
            _stream?.Close();
            _tcpClient?.Close();
            Console.WriteLine("서버 연결이 종료되었습니다.");
        }
        catch (Exception ex)
        {
            Console.WriteLine($"연결 종료 중 오류: {ex.Message}");
        }
    }
}
```

### 패킷 수신 구현
받는 쪽은 조금 더 복잡하다. 네트워크에서 받은 데이터가 완전한 패킷이 아닐 수 있기 때문이다. 한 번의 네트워크 수신으로 여러 개의 패킷이 올 수도 있고, 하나의 패킷이 여러 번에 나눠올 수도 있다.

```csharp
using System;
using System.Net.Sockets;
using System.Collections.Generic;

/// <summary>
/// 패킷 수신을 위한 버퍼를 관리합니다.
/// 불완전한 패킷들을 보관했다가, 완전한 패킷이 되면 처리합니다.
/// </summary>
public class PacketBuffer
{
    private byte[] _buffer;
    private int _dataSize;
    private const int MaxBufferSize = 65536;

    public PacketBuffer()
    {
        _buffer = new byte[MaxBufferSize];
        _dataSize = 0;
    }

    /// <summary>
    /// 새로운 데이터를 버퍼에 추가합니다.
    /// </summary>
    public void AddData(byte[] data, int length)
    {
        if (_dataSize + length > MaxBufferSize)
        {
            throw new InvalidOperationException("버퍼가 꽉 찼습니다.");
        }

        Array.Copy(data, 0, _buffer, _dataSize, length);
        _dataSize += length;
    }

    /// <summary>
    /// 버퍼에서 완전한 패킷을 추출합니다.
    /// 패킷이 완전하면 true를 반환하고, packetType과 payload를 설정합니다.
    /// 불완전하면 false를 반환합니다.
    /// </summary>
    public bool TryExtractPacket(out byte packetType, out byte[] payload)
    {
        packetType = 0;
        payload = null;

        // 헤더를 읽을 수 있는 최소 크기는 3바이트
        if (_dataSize < 3)
            return false;

        // 헤더 읽기: 패킷 타입(1) + 길이(2)
        packetType = _buffer[0];
        int packetLength = BinaryPrimitives.ReadInt16LittleEndian(_buffer.AsSpan(1));

        // 헤더(3) + 페이로드가 모두 들어왔는지 확인
        int totalPacketSize = 3 + packetLength;
        if (_dataSize < totalPacketSize)
            return false;

        // 페이로드 추출
        payload = new byte[packetLength];
        Array.Copy(_buffer, 3, payload, 0, packetLength);

        // 사용한 데이터를 버퍼에서 제거
        Array.Copy(_buffer, totalPacketSize, _buffer, 0, _dataSize - totalPacketSize);
        _dataSize -= totalPacketSize;

        return true;
    }

    /// <summary>
    /// 버퍼에 남아있는 데이터의 크기를 반환합니다.
    /// 디버깅 용도로 유용합니다.
    /// </summary>
    public int GetBufferedDataSize() => _dataSize;
}
```

이 코드의 핵심은 `TryExtractPacket` 메서드다. 이 메서드는 버퍼에서 완전한 패킷이 될 때까지 데이터를 모아두고, 완전한 패킷이 되면 그것을 추출해서 반환한다. 이렇게 하면 불완전한 패킷을 처리하는 복잡성을 숨길 수 있다.

이제 패킷 수신을 담당하는 클래스를 만들자.

```csharp
using System;
using System.Net.Sockets;
using System.Collections.Generic;
using System.Threading;
using System.Threading.Tasks;

/// <summary>
/// 네트워크에서 패킷을 수신합니다.
/// 별도의 스레드에서 실행되므로, 게임 루프를 방해하지 않습니다.
/// 이 코드의 목적:
/// - 게임 루프와 무관하게 백그라운드에서 패킷을 수신
/// - 받은 패킷을 큐에 저장
/// - 메인 게임 루프에서는 큐를 읽어서 패킷을 처리
/// </summary>
public class PacketReceiver
{
    private NetworkStream _stream;
    private PacketBuffer _buffer;
    private Queue<Packet> _receivedPackets;
    private bool _isRunning;
    private Thread _receiveThread;
    private readonly object _lockObject = new object();
    private const int ReceiveBufferSize = 4096;

    public PacketReceiver(NetworkStream stream)
    {
        _stream = stream;
        _buffer = new PacketBuffer();
        _receivedPackets = new Queue<Packet>();
        _isRunning = false;
        _receiveThread = null;
    }

    /// <summary>
    /// 백그라운드 수신을 시작합니다.
    /// 별도의 스레드를 만들어서 계속 패킷을 기다립니다.
    /// </summary>
    public void Start()
    {
        if (_isRunning)
            return;

        _isRunning = true;
        _receiveThread = new Thread(ReceiveThreadFunction)
        {
            IsBackground = true
        };
        _receiveThread.Start();
    }

    /// <summary>
    /// 백그라운드 수신을 중지합니다.
    /// </summary>
    public void Stop()
    {
        _isRunning = false;
        _receiveThread?.Join(1000);
    }

    /// <summary>
    /// 백그라운드 스레드에서 실행되는 메인 수신 루프입니다.
    /// 이 코드의 자세한 설명:
    /// 1. NetworkStream에서 데이터를 읽는다
    /// 2. 읽은 데이터를 버퍼에 추가한다
    /// 3. 버퍼에 완전한 패킷이 있는지 확인한다
    /// 4. 완전한 패킷이면 큐에 추가한다
    /// 5. 이 과정을 반복한다
    /// </summary>
    private void ReceiveThreadFunction()
    {
        byte[] receiveBuffer = new byte[ReceiveBufferSize];

        try
        {
            while (_isRunning)
            {
                // 네트워크 스트림에서 최대 4096바이트까지 읽음
                int bytesRead = _stream.Read(receiveBuffer, 0, ReceiveBufferSize);

                if (bytesRead <= 0)
                {
                    // 서버 연결 끊김
                    Console.WriteLine("서버 연결이 끊어졌습니다.");
                    _isRunning = false;
                    break;
                }

                // 읽은 데이터를 버퍼에 추가
                _buffer.AddData(receiveBuffer, bytesRead);

                // 버퍼에서 완전한 패킷들을 추출
                while (_buffer.TryExtractPacket(out byte packetType, out byte[] payload))
                {
                    try
                    {
                        // 패킷 객체 생성
                        Packet packet = PacketFactory.CreatePacket(packetType, payload);

                        // 다른 스레드에서 안전하게 접근할 수 있도록 lock을 사용
                        lock (_lockObject)
                        {
                            _receivedPackets.Enqueue(packet);
                        }

                        Console.WriteLine($"패킷 수신: 타입={packetType}, 페이로드크기={payload.Length}");
                    }
                    catch (Exception ex)
                    {
                        Console.WriteLine($"패킷 처리 오류: {ex.Message}");
                    }
                }
            }
        }
        catch (Exception ex)
        {
            Console.WriteLine($"수신 오류: {ex.Message}");
            _isRunning = false;
        }
    }

    /// <summary>
    /// 큐에 있는 모든 패킷을 가져옵니다.
    /// 이 메서드는 게임의 메인 루프에서 호출됩니다.
    /// 반환된 리스트의 패킷들을 처리하면 된다.
    /// </summary>
    public List<Packet> GetReceivedPackets()
    {
        List<Packet> packets = new List<Packet>();

        lock (_lockObject)
        {
            while (_receivedPackets.Count > 0)
            {
                packets.Add(_receivedPackets.Dequeue());
            }
        }

        return packets;
    }

    /// <summary>
    /// 현재 큐에 있는 패킷의 개수를 반환합니다.
    /// 디버깅할 때 유용합니다.
    /// </summary>
    public int GetPendingPacketCount()
    {
        lock (_lockObject)
        {
            return _receivedPackets.Count;
        }
    }
}
```

### 게임 네트워크 관리자
이제 송수신을 모두 합쳐서 게임이 사용할 통합 네트워크 관리자를 만들자.

```csharp
using System;
using System.Net.Sockets;
using System.Collections.Generic;

/// <summary>
/// 게임의 네트워크 부분을 통합적으로 관리합니다.
/// 송신과 수신, 연결 관리를 담당합니다.
/// 이 코드의 목적:
/// - 게임 클래스가 간단하게 네트워크를 사용할 수 있도록 추상화
/// - 내부적으로 복잡한 버퍼링, 스레드 관리를 담당
/// - 게임에서는 이 매니저만 사용하면 된다
/// </summary>
public class NetworkManager
{
    private GameNetworkClient _client;
    private PacketReceiver _receiver;
    private bool _isConnected;

    public NetworkManager()
    {
        _client = new GameNetworkClient();
        _receiver = null;
        _isConnected = false;
    }

    /// <summary>
    /// 서버에 연결합니다.
    /// </summary>
    public void Connect(string serverAddress, int serverPort)
    {
        _client.Connect(serverAddress, serverPort);
        
        // TcpClient로부터 NetworkStream을 가져와서 receiver를 초기화
        _receiver = new PacketReceiver(_client._tcpClient.GetStream());
        _receiver.Start();
        
        _isConnected = true;
    }

    /// <summary>
    /// 패킷을 보냅니다.
    /// 게임 코드에서는 이 메서드만 사용하면 된다.
    /// </summary>
    public void SendPacket(Packet packet)
    {
        if (!_isConnected)
        {
            Console.WriteLine("서버에 연결되어 있지 않습니다.");
            return;
        }

        _client.SendPacket(packet);
    }

    /// <summary>
    /// 수신된 패킷들을 모두 가져옵니다.
    /// 게임의 메인 루프에서 매 프레임마다 이 메서드를 호출합니다.
    /// </summary>
    public List<Packet> GetReceivedPackets()
    {
        if (!_isConnected || _receiver == null)
            return new List<Packet>();

        return _receiver.GetReceivedPackets();
    }

    /// <summary>
    /// 서버와의 연결을 종료합니다.
    /// </summary>
    public void Disconnect()
    {
        if (_receiver != null)
        {
            _receiver.Stop();
        }

        _client.Disconnect();
        _isConnected = false;
    }

    /// <summary>
    /// 연결 상태를 반환합니다.
    /// </summary>
    public bool IsConnected => _isConnected;
}
```

### 실제 사용 예제
이제 위에서 만든 모든 클래스들이 어떻게 함께 작동하는지 보자.

```csharp
using System;
using System.Collections.Generic;
using System.Threading;

/// <summary>
/// 프로토콜과 네트워크를 실제로 사용하는 예제입니다.
/// 이 코드의 흐름:
/// 1. 네트워크 매니저를 만들고 서버에 연결
/// 2. 로그인 패킷을 보냄
/// 3. 위치 업데이트 패킷을 보냄
/// 4. 받은 패킷을 처리
/// 5. 연결 종료
/// </summary>
class Program
{
    static void Main(string[] args)
    {
        // 네트워크 매니저 생성
        NetworkManager networkManager = new NetworkManager();

        try
        {
            // 서버에 연결 (로컬 테스트용)
            Console.WriteLine("서버에 연결 중...");
            networkManager.Connect("127.0.0.1", 9000);
            Console.WriteLine("연결 성공!");

            // 로그인 패킷 보내기
            LoginPacket loginPacket = new LoginPacket("철수");
            networkManager.SendPacket(loginPacket);

            // 조금 대기 (네트워크 지연 시뮬레이션)
            Thread.Sleep(100);

            // 위치 업데이트 패킷 보내기
            PositionUpdatePacket posPacket = 
                new PositionUpdatePacket(1, 100.5f, 200.3f);
            networkManager.SendPacket(posPacket);

            // 채팅 메시지 보내기
            ChatPacket chatPacket = 
                new ChatPacket(1, "안녕하세요!");
            networkManager.SendPacket(chatPacket);

            // 수신된 패킷 처리 (5초 동안)
            for (int i = 0; i < 5; i++)
            {
                List<Packet> receivedPackets = 
                    networkManager.GetReceivedPackets();

                foreach (var packet in receivedPackets)
                {
                    if (packet is LoginPacket login)
                    {
                        Console.WriteLine(
                            $"로그인 응답: {login.PlayerName}");
                    }
                    else if (packet is PositionUpdatePacket pos)
                    {
                        Console.WriteLine(
                            $"위치 업데이트: ID={pos.PlayerId}, " +
                            $"X={pos.X}, Y={pos.Y}");
                    }
                    else if (packet is ChatPacket chat)
                    {
                        Console.WriteLine(
                            $"채팅: ID={chat.PlayerId}, " +
                            $"메시지={chat.Message}");
                    }
                }

                Thread.Sleep(1000);
            }

            // 연결 종료
            networkManager.Disconnect();
            Console.WriteLine("연결 종료.");
        }
        catch (Exception ex)
        {
            Console.WriteLine($"오류: {ex.Message}");
        }
    }
}
```

이 예제 코드가 50줄이 넘어가므로 자세히 설명하겠다.

**전체 흐름**:
1. `NetworkManager`를 만들어서 네트워크를 관리할 준비를 한다.
2. `Connect` 메서드로 서버에 연결한다. 이 과정에서 내부적으로 `PacketReceiver`도 함께 초기화되고, 백그라운드 스레드가 시작된다.
3. 여러 종류의 패킷을 만들어서 `SendPacket`으로 보낸다. 각 패킷은 자동으로 직렬화되어 서버로 전송된다.
4. 루프를 돌면서 `GetReceivedPackets`를 호출한다. 이 메서드는 백그라운드 스레드가 수신한 패킷들을 반환한다.
5. 받은 패킷을 타입에 따라 처리한다.
6. 마지막으로 `Disconnect`로 연결을 종료한다.

**중요한 세부사항**:
- `GameNetworkClient._tcpClient`를 바로 접근하는 것은 좋지 않은 설계지만, 여기서는 간단히 하기 위해 이렇게 했다. 실제로는 `_client` 내부에서 NetworkStream을 얻는 메서드를 제공하는 것이 나을 것이다.
- 스레드 안전성을 위해 `PacketReceiver`에서 `lock`을 사용했다. 백그라운드 스레드(수신)와 메인 스레드(게임 루프)가 동시에 큐에 접근할 수 있기 때문이다.

## 학습 확인 과제
이 장에서 배운 내용을 확인하기 위해 다음 과제를 수행해보자.

### 과제 1: 패킷 타입 추가
이 장에서는 Login, PositionUpdate, Chat 패킷을 만들었다. 이제 새로운 패킷을 추가해보자.

**공격 패킷(AttackPacket)** 을 만들어보자. 이 패킷은 다음 정보를 포함해야 한다:
- 공격자 플레이어 ID (int)
- 피해자 플레이어 ID (int)  
- 공격 데미지 (int)

1. `PacketType` 클래스에 `Attack = 8`을 추가하자.
