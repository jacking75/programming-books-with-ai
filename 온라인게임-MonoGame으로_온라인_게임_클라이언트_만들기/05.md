# 5장: MonoGame 프로젝트 시작
이제 앞서 배운 네트워크 통신의 기초를 바탕으로, 실제 게임을 만들어보자. 이 장에서는 MonoGame 프로젝트를 생성하고, 게임 루프의 기본 구조를 이해하며, 화면에 무언가를 그리고 업데이트하는 방법을 배운다.

## 5.1 MonoGame 프로젝트 생성

### MonoGame 설치 및 프로젝트 템플릿
MonoGame 3.8을 사용하여 .NET 9.0 기반의 프로젝트를 만들기 위해서는 먼저 MonoGame 템플릿을 설치해야 한다.

**터미널(또는 PowerShell)에서 다음 명령을 실행한다:**

```bash
dotnet new install MonoGame.Templates.CSharp
```

이 명령을 실행하면 MonoGame의 프로젝트 템플릿이 .NET CLI에 등록된다. 이제 이 템플릿을 사용하여 새로운 MonoGame 프로젝트를 생성할 수 있다.

**프로젝트를 생성하려면:**

```bash
dotnet new mgdesktopgl -n OnlineGameClient -f net9.0
```

이 명령의 의미를 설명한다:
- `mgdesktopgl`: MonoGame Desktop OpenGL 템플릿을 사용한다는 뜻이다. 데스크톱 환경에서 OpenGL을 사용하여 그래픽을 렌더링한다.
- `-n OnlineGameClient`: 프로젝트 이름을 "OnlineGameClient"로 설정한다.
- `-f net9.0`: .NET 9.0 프레임워크를 대상으로 한다.

생성된 프로젝트 폴더로 이동하면 다음과 같은 구조가 보인다:

```
OnlineGameClient/
├── OnlineGameClient.csproj
├── Program.cs
├── Game1.cs
├── Content/
└── bin/
```

**프로젝트 파일(OnlineGameClient.csproj) 확인:**

생성된 프로젝트 파일을 열어보면 MonoGame 관련 NuGet 패키지가 자동으로 포함되어 있다. 우리는 추가로 네트워크 통신용 클래스들을 이 프로젝트에 포함시킬 것이다.

### 프로젝트 구조 설계
게임이 커질수록 체계적인 폴더 구조가 필요하다. 다음과 같이 폴더를 조직할 것을 권장한다:

```
OnlineGameClient/
├── Game1.cs                    (메인 게임 클래스)
├── Program.cs                  (진입점)
├── Content/                    (게임 리소스)
│   ├── Graphics/               (이미지, 스프라이트)
│   └── Fonts/                  (폰트 파일)
├── Network/                    (네트워크 관련 클래스)
│   ├── NetworkClient.cs
│   ├── PacketHandler.cs
│   └── MessageProcessor.cs
├── Game/                       (게임 로직)
│   ├── Player.cs
│   ├── GameState.cs
│   └── GameManager.cs
└── Utils/                      (유틸리티)
    └── Constants.cs
```

이런 폴더 구조를 만들어두면 나중에 기능을 추가할 때 코드를 찾기가 쉬워진다.

### MonoGame 기본 파일 이해
MonoGame 템플릿으로 생성된 `Program.cs`를 보자:

```csharp
using var game = new Game1();
game.Run();
```

이 파일은 매우 간단한 진입점이다. `Game1` 클래스를 인스턴스화한 후 `Run()` 메서드를 호출한다. `Run()` 메서드가 호출되면 게임 루프가 시작되고, 게임이 종료될 때까지 계속 실행된다.

`Game1.cs` 파일을 살펴보자. 이것이 우리 게임의 핵심 클래스다:

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;

public class Game1 : Game
{
    private GraphicsDeviceManager _graphics;
    private SpriteBatch _spriteBatch;

    public Game1()
    {
        _graphics = new GraphicsDeviceManager(this);
        Content.RootDirectory = "Content";
        IsMouseVisible = true;
    }

    protected override void Initialize()
    {
        base.Initialize();
    }

    protected override void LoadContent()
    {
        _spriteBatch = new SpriteBatch(GraphicsDevice);
    }

    protected override void Update(GameTime gameTime)
    {
        if (GamePad.GetState(PlayerIndex.One).Buttons.Back == ButtonState.Pressed)
            Exit();

        base.Update(gameTime);
    }

    protected override void Draw(GameTime gameTime)
    {
        GraphicsDevice.Clear(Color.CornflowerBlue);

        _spriteBatch.Begin();
        _spriteBatch.End();

        base.Draw(gameTime);
    }
}
```

각 메서드의 역할을 설명한다:

**생성자**: 게임이 처음 만들어질 때 호출된다. 여기서 그래픽 설정을 초기화하고, 리소스 폴더 경로를 설정한다.

**Initialize()**: 게임 로직의 초기화가 필요할 때 사용한다. 게임에 필요한 변수들을 여기서 초기화한다.

**LoadContent()**: 게임에 필요한 이미지, 사운드, 폰트 등의 리소스를 여기서 로드한다. 이 메서드는 `Initialize()` 다음에 한 번만 호출된다.

**Update()**: 게임 로직을 업데이트하는 메서드다. 플레이어 입력 처리, 캐릭터 위치 업데이트, 네트워크 메시지 처리 등이 여기서 일어난다. 기본적으로 초당 60번 호출된다.

**Draw()**: 화면에 게임을 그리는 메서드다. 배경 그리기, 캐릭터 그리기, 텍스트 표시 등이 여기서 일어난다.

## 5.2 기본 게임 루프 이해

### 게임 루프의 구조
MonoGame의 게임 루프는 다음과 같은 패턴으로 반복된다:

```
[게임 시작]
    ↓
[Initialize() 호출]
    ↓
[LoadContent() 호출]
    ↓
┌─→ [Update() 호출 - 게임 로직 업데이트]
│   ↓
│   [Draw() 호출 - 화면에 그리기]
│   ↓
│   [화면 표시]
│   ↓
│   [다음 프레임 준비]
└─── [반복]
    ↓
[게임 종료 요청?]
    ├─ No → 위의 반복 루프로 돌아감
    └─ Yes → [정리 작업 후 종료]
```

이 루프는 매 프레임마다 실행된다. 기본 설정에서는 초당 60프레임(FPS)으로 동작한다. 즉, `Update()`와 `Draw()`가 1초에 60번씩 호출된다.

### GameTime 객체 활용
`Update()`와 `Draw()` 메서드에는 `GameTime gameTime` 매개변수가 전달된다. 이 객체에는 게임의 시간 정보가 들어있다:

```csharp
protected override void Update(GameTime gameTime)
{
    // 이전 프레임 이후 경과한 시간(초 단위)
    double deltaTime = gameTime.ElapsedGameTime.TotalSeconds;
    
    // 게임이 시작된 후 총 경과 시간
    double totalTime = gameTime.TotalGameTime.TotalSeconds;
    
    // 현재가 고정 타임스탭 업데이트인지 여부
    bool isRunningSlowly = gameTime.IsRunningSlowly;
    
    // 매 프레임마다 캐릭터를 이동시키는 예시
    // deltaTime을 사용하면 프레임 레이트가 변해도 이동 속도가 일정하다
    playerPosition += playerVelocity * (float)deltaTime;
}
```

`deltaTime`을 사용하는 이유는 매우 중요하다. 만약 deltaTime 없이 `playerPosition += playerVelocity;`로 하면, 컴퓨터 성능이 좋아서 100FPS로 실행되는 경우와 30FPS로 실행되는 경우에 캐릭터의 이동 속도가 달라진다. `deltaTime`을 곱하면 어떤 프레임 레이트에서도 일정한 속도로 이동한다.

### 초당 프레임 설정
게임의 초당 프레임 수를 변경하려면 생성자에서 설정한다:

```csharp
public Game1()
{
    _graphics = new GraphicsDeviceManager(this);
    Content.RootDirectory = "Content";
    IsMouseVisible = true;
    
    // 초당 30 프레임으로 설정
    TargetElapsedTime = TimeSpan.FromSeconds(1.0 / 30.0);
}
```

온라인 게임에서는 네트워크 지연을 고려하여 프레임 레이트를 설정해야 한다. 일반적으로 30FPS나 60FPS를 사용한다.

### 게임 루프와 네트워크 통신의 통합
우리가 앞서 배운 네트워크 클라이언트를 게임 루프와 어떻게 통합할지 생각해보자:

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using System.Threading.Tasks;

public class Game1 : Game
{
    private GraphicsDeviceManager _graphics;
    private SpriteBatch _spriteBatch;
    
    // 네트워크 관련 필드
    private NetworkClient _networkClient;
    private MessageProcessor _messageProcessor;
    private bool _isNetworkInitialized;

    public Game1()
    {
        _graphics = new GraphicsDeviceManager(this);
        Content.RootDirectory = "Content";
        IsMouseVisible = true;
    }

    protected override void Initialize()
    {
        // 네트워크 클라이언트 초기화
        InitializeNetworkAsync();
        
        base.Initialize();
    }
    
    /// <summary>
    /// 비동기로 네트워크 연결을 초기화한다.
    /// 게임 시작과 동시에 서버 연결을 시도한다.
    /// </summary>
    private async void InitializeNetworkAsync()
    {
        _networkClient = new NetworkClient();
        
        // 서버에 연결 시도 (localhost, 포트 5000)
        bool connected = await _networkClient.ConnectAsync("127.0.0.1", 5000);
        
        if (connected)
        {
            // 메시지 프로세서 생성
            _messageProcessor = new MessageProcessor(_networkClient);
            
            // 핸들러 등록 (후속 장에서 구현할 예정)
            
            // 백그라운드에서 패킷 수신 시작
            _ = _networkClient.StartReceivingAsync();
            
            _isNetworkInitialized = true;
        }
        else
        {
            _isNetworkInitialized = false;
        }
    }

    protected override void LoadContent()
    {
        _spriteBatch = new SpriteBatch(GraphicsDevice);
    }

    protected override void Update(GameTime gameTime)
    {
        // 네트워크 메시지 처리
        if (_isNetworkInitialized && _messageProcessor != null)
        {
            _messageProcessor.ProcessMessages();
        }
        
        // 게임 로직 업데이트
        // (캐릭터 이동, 충돌 감지 등)
        
        base.Update(gameTime);
    }

    protected override void Draw(GameTime gameTime)
    {
        GraphicsDevice.Clear(Color.CornflowerBlue);

        _spriteBatch.Begin();
        
        // 게임 요소 렌더링
        // (배경, 플레이어, 다른 플레이어 등)
        
        _spriteBatch.End();

        base.Draw(gameTime);
    }
}
```

이 구조의 중요한 점들을 설명한다:

**비동기 네트워크 초기화**: `InitializeNetworkAsync()` 메서드를 `Initialize()`에서 호출한다. 이것은 서버 연결을 시도하면서도 게임이 멈추지 않도록 비동기로 실행된다.

**Update() 메서드에서 메시지 처리**: 매 프레임마다 `ProcessMessages()`를 호출하여 서버에서 받은 메시지를 처리한다. 이렇게 하면 게임 로직과 네트워크 통신이 동기화된다.

## 5.3 렌더링과 업데이트
이 섹션에서는 실제로 게임에 무언가를 그리고 업데이트하는 방법을 배운다.

### SpriteBatch를 이용한 2D 렌더링
MonoGame에서 2D 게임을 만들 때는 `SpriteBatch` 클래스를 사용한다. 이것은 효율적으로 2D 이미지를 화면에 그릴 수 있게 해준다.

가장 간단한 예로, 화면의 특정 위치에 색상이 있는 사각형을 그려보자:

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;

public class Game1 : Game
{
    private GraphicsDeviceManager _graphics;
    private SpriteBatch _spriteBatch;
    
    // 화면 중앙의 사각형 위치
    private Vector2 _rectanglePosition;
    private const int RectangleSize = 32;

    public Game1()
    {
        _graphics = new GraphicsDeviceManager(this);
        Content.RootDirectory = "Content";
        IsMouseVisible = true;
    }

    protected override void Initialize()
    {
        // 화면 중앙에 사각형을 배치
        _rectanglePosition = new Vector2(
            _graphics.PreferredBackBufferWidth / 2,
            _graphics.PreferredBackBufferHeight / 2
        );
        
        base.Initialize();
    }

    protected override void LoadContent()
    {
        _spriteBatch = new SpriteBatch(GraphicsDevice);
    }

    protected override void Update(GameTime gameTime)
    {
        base.Update(gameTime);
    }

    protected override void Draw(GameTime gameTime)
    {
        GraphicsDevice.Clear(Color.CornflowerBlue);

        _spriteBatch.Begin();
        
        // 하얀색 사각형을 그린다
        // Draw(texture, position, color)
        // texture가 null이면 흰색 픽셀을 사용한다
        _spriteBatch.Draw(
            texture: null,
            destinationRectangle: new Rectangle(
                (int)_rectanglePosition.X,
                (int)_rectanglePosition.Y,
                RectangleSize,
                RectangleSize
            ),
            color: Color.White
        );
        
        _spriteBatch.End();

        base.Draw(gameTime);
    }
}
```

`SpriteBatch.Draw()` 메서드의 매개변수들을 설명한다:

- `texture`: 그릴 이미지. `null`이면 흰색 픽셀을 사용한다.
- `destinationRectangle`: 화면 위의 위치와 크기.
- `color`: 사각형의 색상.

### 키보드 입력으로 캐릭터 이동
이제 키보드로 사각형을 움직여보자:

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;

public class Game1 : Game
{
    private GraphicsDeviceManager _graphics;
    private SpriteBatch _spriteBatch;
    
    private Vector2 _playerPosition;
    private const int PlayerSize = 32;
    private const float PlayerSpeed = 200f;  // 픽셀/초

    public Game1()
    {
        _graphics = new GraphicsDeviceManager(this);
        Content.RootDirectory = "Content";
        IsMouseVisible = true;
    }

    protected override void Initialize()
    {
        _playerPosition = new Vector2(400, 300);
        base.Initialize();
    }

    protected override void LoadContent()
    {
        _spriteBatch = new SpriteBatch(GraphicsDevice);
    }

    protected override void Update(GameTime gameTime)
    {
        KeyboardState keyboardState = Keyboard.GetState();
        
        // deltaTime 가져오기
        float deltaTime = (float)gameTime.ElapsedGameTime.TotalSeconds;
        
        // 키보드 입력 처리
        if (keyboardState.IsKeyDown(Keys.Up))
        {
            _playerPosition.Y -= PlayerSpeed * deltaTime;
        }
        
        if (keyboardState.IsKeyDown(Keys.Down))
        {
            _playerPosition.Y += PlayerSpeed * deltaTime;
        }
        
        if (keyboardState.IsKeyDown(Keys.Left))
        {
            _playerPosition.X -= PlayerSpeed * deltaTime;
        }
        
        if (keyboardState.IsKeyDown(Keys.Right))
        {
            _playerPosition.X += PlayerSpeed * deltaTime;
        }
        
        // 화면 경계 내에서만 이동하도록 제한
        int screenWidth = _graphics.PreferredBackBufferWidth;
        int screenHeight = _graphics.PreferredBackBufferHeight;
        
        _playerPosition.X = MathHelper.Clamp(
            _playerPosition.X,
            0,
            screenWidth - PlayerSize
        );
        
        _playerPosition.Y = MathHelper.Clamp(
            _playerPosition.Y,
            0,
            screenHeight - PlayerSize
        );
        
        base.Update(gameTime);
    }

    protected override void Draw(GameTime gameTime)
    {
        GraphicsDevice.Clear(Color.CornflowerBlue);

        _spriteBatch.Begin();
        
        // 플레이어 캐릭터 그리기
        _spriteBatch.Draw(
            texture: null,
            destinationRectangle: new Rectangle(
                (int)_playerPosition.X,
                (int)_playerPosition.Y,
                PlayerSize,
                PlayerSize
            ),
            color: Color.Red
        );
        
        _spriteBatch.End();

        base.Draw(gameTime);
    }
}
```

중요한 개념들을 설명한다:

**KeyboardState**: `Keyboard.GetState()`를 통해 매 프레임마다 현재 키보드 상태를 확인한다.

**IsKeyDown()**: 특정 키가 눌려있는지 확인한다. 여러 키가 동시에 눌려있을 수 있으므로 여러 방향키를 동시에 처리할 수 있다.

**deltaTime 사용**: 이동 거리를 계산할 때 `PlayerSpeed * deltaTime`을 사용한다. 이렇게 하면 초당 200픽셀씩 일정하게 이동한다.

**MathHelper.Clamp()**: 플레이어가 화면 밖으로 나가지 않도록 위치를 제한한다.

### 텍스트 표시
화면에 게임 정보를 표시해야 한다. 먼저 MonoGame에서 기본으로 제공하는 폰트를 사용해보자:

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Content;

public class Game1 : Game
{
    private GraphicsDeviceManager _graphics;
    private SpriteBatch _spriteBatch;
    private SpriteFont _font;
    
    private Vector2 _playerPosition;
    private string _statusText;

    public Game1()
    {
        _graphics = new GraphicsDeviceManager(this);
        Content.RootDirectory = "Content";
        IsMouseVisible = true;
    }

    protected override void Initialize()
    {
        _playerPosition = new Vector2(400, 300);
        base.Initialize();
    }

    protected override void LoadContent()
    {
        _spriteBatch = new SpriteBatch(GraphicsDevice);
        
        // Arial 폰트 로드 (MonoGame 기본 폰트)
        _font = Content.Load<SpriteFont>("Arial");
    }

    protected override void Update(GameTime gameTime)
    {
        // 상태 텍스트 업데이트
        _statusText = $"위치: ({_playerPosition.X:F0}, {_playerPosition.Y:F0})";
        
        base.Update(gameTime);
    }

    protected override void Draw(GameTime gameTime)
    {
        GraphicsDevice.Clear(Color.CornflowerBlue);

        _spriteBatch.Begin();
        
        // 플레이어 그리기
        _spriteBatch.Draw(
            texture: null,
            destinationRectangle: new Rectangle(
                (int)_playerPosition.X,
                (int)_playerPosition.Y,
                32,
                32
            ),
            color: Color.Red
        );
        
        // 텍스트 그리기
        _spriteBatch.DrawString(
            spriteFont: _font,
            text: _statusText,
            position: new Vector2(10, 10),
            color: Color.White
        );
        
        _spriteBatch.End();

        base.Draw(gameTime);
    }
}
```

폰트 파일을 로드하려면 먼저 Content 폴더에 폰트를 준비해야 한다. MonoGame에서는 특정한 형식의 폰트 파일을 사용한다.

### 더 체계적인 구조: GameManager 클래스
게임이 커질수록 `Game1` 클래스가 너무 많은 책임을 가지게 된다. 게임 로직을 관리하는 별도의 클래스를 만드는 것이 좋다:

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using Microsoft.Xna.Framework.Input;

/// <summary>
/// 게임의 전체 로직을 관리하는 클래스다.
/// 플레이어, 네트워크, 게임 상태 등을 조율한다.
/// </summary>
public class GameManager
{
    private Vector2 _playerPosition;
    private const int PlayerSize = 32;
    private const float PlayerSpeed = 200f;
    
    private int _screenWidth;
    private int _screenHeight;
    
    private NetworkClient _networkClient;
    private MessageProcessor _messageProcessor;
    private bool _isNetworkInitialized;

    public GameManager(int screenWidth, int screenHeight)
    {
        _screenWidth = screenWidth;
        _screenHeight = screenHeight;
        _playerPosition = new Vector2(screenWidth / 2, screenHeight / 2);
    }

    /// <summary>
    /// 네트워크를 비동기로 초기화한다.
    /// 이 메서드는 별도의 스레드에서 실행되므로 게임 루프를 블로킹하지 않는다.
    /// </summary>
    public async void InitializeNetworkAsync()
    {
        _networkClient = new NetworkClient();
        
        bool connected = await _networkClient.ConnectAsync("127.0.0.1", 5000);
        
        if (connected)
        {
            _messageProcessor = new MessageProcessor(_networkClient);
            _ = _networkClient.StartReceivingAsync();
            _isNetworkInitialized = true;
        }
    }

    /// <summary>
    /// 게임 로직을 매 프레임마다 업데이트한다.
    /// </summary>
    public void Update(GameTime gameTime)
    {
        // 네트워크 메시지 처리
        if (_isNetworkInitialized && _messageProcessor != null)
        {
            _messageProcessor.ProcessMessages();
        }
        
        // 입력 처리
        HandleInput(gameTime);
    }

    /// <summary>
    /// 화면에 게임을 렌더링한다.
    /// </summary>
    public void Draw(SpriteBatch spriteBatch, SpriteFont font)
    {
        // 플레이어 그리기
        spriteBatch.Draw(
            texture: null,
            destinationRectangle: new Rectangle(
                (int)_playerPosition.X,
                (int)_playerPosition.Y,
                PlayerSize,
                PlayerSize
            ),
            color: Color.Red
        );
        
        // 상태 정보 그리기
        string statusText = $"위치: ({_playerPosition.X:F0}, {_playerPosition.Y:F0}) | 네트워크: {(_isNetworkInitialized ? "연결됨" : "연결 안됨")}";
        spriteBatch.DrawString(font, statusText, new Vector2(10, 10), Color.White);
    }

    /// <summary>
    /// 키보드 입력을 처리한다.
    /// </summary>
    private void HandleInput(GameTime gameTime)
    {
        KeyboardState keyboardState = Keyboard.GetState();
        float deltaTime = (float)gameTime.ElapsedGameTime.TotalSeconds;
        
        if (keyboardState.IsKeyDown(Keys.Up))
        {
            _playerPosition.Y -= PlayerSpeed * deltaTime;
        }
        
        if (keyboardState.IsKeyDown(Keys.Down))
        {
            _playerPosition.Y += PlayerSpeed * deltaTime;
        }
        
        if (keyboardState.IsKeyDown(Keys.Left))
        {
            _playerPosition.X -= PlayerSpeed * deltaTime;
        }
        
        if (keyboardState.IsKeyDown(Keys.Right))
        {
            _playerPosition.X += PlayerSpeed * deltaTime;
        }
        
        // 경계 제한
        _playerPosition.X = MathHelper.Clamp(_playerPosition.X, 0, _screenWidth - PlayerSize);
        _playerPosition.Y = MathHelper.Clamp(_playerPosition.Y, 0, _screenHeight - PlayerSize);
    }

    /// <summary>
    /// 게임을 종료한다.
    /// 네트워크 연결을 정리한다.
    /// </summary>
    public void Shutdown()
    {
        if (_networkClient != null)
        {
            _networkClient.Disconnect();
        }
    }
}
```

이제 `Game1`은 훨씬 간단해진다:

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;

public class Game1 : Game
{
    private GraphicsDeviceManager _graphics;
    private SpriteBatch _spriteBatch;
    private SpriteFont _font;
    private GameManager _gameManager;

    public Game1()
    {
        _graphics = new GraphicsDeviceManager(this);
        Content.RootDirectory = "Content";
        IsMouseVisible = true;
    }

    protected override void Initialize()
    {
        int width = _graphics.PreferredBackBufferWidth;
        int height = _graphics.PreferredBackBufferHeight;
        
        _gameManager = new GameManager(width, height);
        _gameManager.InitializeNetworkAsync();
        
        base.Initialize();
    }

    protected override void LoadContent()
    {
        _spriteBatch = new SpriteBatch(GraphicsDevice);
        _font = Content.Load<SpriteFont>("Arial");
    }

    protected override void Update(GameTime gameTime)
    {
        _gameManager.Update(gameTime);
        base.Update(gameTime);
    }

    protected override void Draw(GameTime gameTime)
    {
        GraphicsDevice.Clear(Color.CornflowerBlue);

        _spriteBatch.Begin();
        _gameManager.Draw(_spriteBatch, _font);
        _spriteBatch.End();

        base.Draw(gameTime);
    }

    protected override void OnExiting(object sender, System.EventArgs args)
    {
        _gameManager.Shutdown();
        base.OnExiting(sender, args);
    }
}
```

`GameManager` 클래스를 분리한 이점들을 설명한다:

**관심사의 분리**: `Game1`은 MonoGame의 기본 구조만 담당하고, `GameManager`는 게임 로직을 담당한다.

**테스트 용이성**: `GameManager`를 독립적으로 테스트할 수 있다.

**코드 재사용**: `GameManager`를 다른 MonoGame 프로젝트에서도 사용할 수 있다.

**유지보수성**: 게임 로직이 커질수록 이런 구조가 코드 관리를 쉽게 만든다.

### 다양한 색상과 도형 그리기
MonoGame에서 다양한 색상의 도형을 그리는 방법을 보자:

```csharp
protected override void Draw(GameTime gameTime)

    _spriteBatch.Begin();

        texture: null,

    );
    
    // 초록색 사각형
```

```csharp
protected override void Update(GameTime gameTime)
```

```csharp
protected override void Update(GameTime gameTime)
```

**문제 8** (심화): `Initialize()` 메서드와 `LoadContent()` 메서드의 차이점을 설명하시오. 각각 어떤 작업을 하기에 적합한가?

**예시 답안**: 
- `Initialize()`: 게임의 게임 로직 초기화에 사용된다. 변수 초기화, 게임 상태 설정, 네트워크 연결 시작 등이 여기에 해당한다. 리소스 파일이 필요 없는 작업을 한다.
- `LoadContent()`: 디스크에서 파일을 읽어 메모리에 로드하는 작업에 사용된다. 이미지, 소리, 폰트 등의 리소스를 로드한다. 이 메서드는 `GraphicsDevice`가 준비된 후에 호출되므로 그래픽 리소스를 안전하게 로드할 수 있다.

예를 들어, 플레이어의 초기 위치를 설정하는 것은 `Initialize()`에서, 플레이어 이미지를 로드하는 것은 `LoadContent()`에서 한다.
