# 6장: 간단한 플레이어 캐릭터 만들기

이제 본격적으로 게임에 캐릭터를 추가하겠다. 5장에서는 간단한 사각형을 그렸지만, 이번 장에서는 실제 이미지(스프라이트)를 사용하여 플레이어 캐릭터를 구현한다. 스프라이트는 2D 게임의 기본이 되는 그래픽 요소다.

## 6.1 스프라이트 기초

### 스프라이트란 무엇인가
스프라이트는 2D 게임에서 사용되는 2차원 이미지다. 배경, 캐릭터, 아이템 등 화면에 그려지는 모든 요소가 스프라이트다. MonoGame에서 스프라이트는 `Texture2D` 클래스로 표현된다.

```
┌─────────────────┐
│                 │
│    스프라이트     │  ← 이미지 파일 (PNG, JPG 등)
│                 │
└─────────────────┘
        ↓
  Texture2D 객체로 로드
        ↓
  스프라이트배치(SpriteBatch)로 화면에 그리기
```

### 스프라이트 파일 준비
게임에 사용할 스프라이트 이미지를 준비해야 한다. 이 책의 예제에서는 간단한 사각형 이미지를 사용한다.

다음과 같은 방법으로 스프라이트를 준비할 수 있다:

**방법 1: 외부 이미지 에디터로 만들기**

그림판, GIMP, Photoshop 등을 사용하여 32x32 픽셀의 PNG 이미지를 만든다. 플레이어 캐릭터를 나타내는 간단한 도형이나 색상의 사각형이면 된다.

**방법 2: 코드로 동적 생성하기**

런타임에 이미지를 동적으로 생성할 수도 있다. 이는 스프라이트 파일을 준비할 수 없을 때 유용하다:

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;

/// <summary>
/// 게임 시작 시 빨간색 사각형 스프라이트를 동적으로 생성한다.
/// 이 방법은 파일 없이 색상만으로 캐릭터를 표현할 때 사용한다.
/// </summary>
public class SpriteGenerator
{
    /// <summary>
    /// 단색 스프라이트를 생성한다.
    /// </summary>
    /// <param name="graphicsDevice">MonoGame의 그래픽 디바이스</param>
    /// <param name="width">스프라이트 너비</param>
    /// <param name="height">스프라이트 높이</param>
    /// <param name="color">스프라이트 색상</param>
    /// <returns>생성된 Texture2D 객체</returns>
    public static Texture2D CreateColoredSquare(
        GraphicsDevice graphicsDevice,
        int width,
        int height,
        Color color)
    {
        // 지정된 크기의 텍스처 생성
        Texture2D texture = new Texture2D(graphicsDevice, width, height);
        
        // 텍스처를 채울 색상 데이터 배열
        Color[] colorData = new Color[width * height];
        
        // 모든 픽셀을 지정된 색상으로 채운다
        for (int i = 0; i < colorData.Length; i++)
        {
            colorData[i] = color;
        }
        
        // 색상 데이터를 텍스처에 설정
        texture.SetData(colorData);
        
        return texture;
    }
    
    /// <summary>
    /// 테두리가 있는 사각형 스프라이트를 생성한다.
    /// </summary>
    /// <param name="graphicsDevice">MonoGame의 그래픽 디바이스</param>
    /// <param name="width">스프라이트 너비</param>
    /// <param name="height">스프라이트 높이</param>
    /// <param name="fillColor">채우기 색상</param>
    /// <param name="borderColor">테두리 색상</param>
    /// <returns>생성된 Texture2D 객체</returns>
    public static Texture2D CreateBorderedSquare(
        GraphicsDevice graphicsDevice,
        int width,
        int height,
        Color fillColor,
        Color borderColor)
    {
        Texture2D texture = new Texture2D(graphicsDevice, width, height);
        Color[] colorData = new Color[width * height];
        
        // 모든 픽셀을 채우기 색상으로 초기화
        for (int i = 0; i < colorData.Length; i++)
        {
            colorData[i] = fillColor;
        }
        
        // 테두리 부분을 다른 색상으로 칠한다
        int borderThickness = 2;
        
        // 위쪽과 아래쪽 테두리
        for (int x = 0; x < width; x++)
        {
            for (int y = 0; y < borderThickness; y++)
            {
                colorData[y * width + x] = borderColor;
                colorData[(height - 1 - y) * width + x] = borderColor;
            }
        }
        
        // 왼쪽과 오른쪽 테두리
        for (int y = 0; y < height; y++)
        {
            for (int x = 0; x < borderThickness; x++)
            {
                colorData[y * width + x] = borderColor;
                colorData[y * width + (width - 1 - x)] = borderColor;
            }
        }
        
        texture.SetData(colorData);
        return texture;
    }
}
```

이 코드의 목적과 동작을 설명한다:

**CreateColoredSquare**: 간단한 색상의 사각형을 만든다. 먼저 `Texture2D` 객체를 만들고, 배열에 색상 정보를 채운 다음, `SetData()`로 텍스처에 적용한다. 이렇게 하면 PNG 파일 없이도 색상 이미지를 사용할 수 있다.

**CreateBorderedSquare**: 테두리가 있는 사각형을 만든다. 가장자리 부분(borderThickness 크기)은 다른 색으로 칠해서 테두리 효과를 낸다.

### 스프라이트 로드
파일에서 스프라이트를 로드하는 방법을 보자. 먼저 스프라이트 이미지를 `Content/Graphics` 폴더에 저장해야 한다:

```csharp
protected override void LoadContent()
{
    _spriteBatch = new SpriteBatch(GraphicsDevice);
    
    // 외부 파일에서 스프라이트 로드
    _playerTexture = Content.Load<Texture2D>("Graphics/Player");
    
    // 또는 동적으로 스프라이트 생성
    _playerTexture = SpriteGenerator.CreateBorderedSquare(
        GraphicsDevice,
        32,
        32,
        Color.Red,
        Color.DarkRed
    );
}
```

MonoGame의 Content 시스템에서 `Content.Load<Texture2D>("Graphics/Player")`를 호출하면, 자동으로 `Content/Graphics/Player.png` 파일을 찾아서 로드한다. 파일의 확장자를 포함하면 안 된다.

## 6.2 키보드 입력 처리
이제 플레이어가 키보드 입력으로 캐릭터를 움직이도록 구현하겠다. MonoGame에서 입력 처리는 매우 간단하다.

### 키보드 상태 확인

```csharp
using Microsoft.Xna.Framework.Input;

protected override void Update(GameTime gameTime)
{
    // 현재 키보드 상태를 가져온다
    KeyboardState keyboardState = Keyboard.GetState();
    
    // 특정 키가 눌려있는지 확인
    if (keyboardState.IsKeyDown(Keys.Up))
    {
        // 위쪽 화살표 키가 눌림
    }
    
    if (keyboardState.IsKeyDown(Keys.Down))
    {
        // 아래쪽 화살표 키가 눌림
    }
    
    if (keyboardState.IsKeyDown(Keys.Left))
    {
        // 왼쪽 화살표 키가 눌림
    }
    
    if (keyboardState.IsKeyDown(Keys.Right))
    {
        // 오른쪽 화살표 키가 눌림
    }
    
    base.Update(gameTime);
}
```

### 이전 프레임과의 비교
키 입력을 더 세밀하게 제어하려면, 현재 프레임의 키 상태와 이전 프레임의 키 상태를 비교해야 한다. 예를 들어 "방금 누른 순간" 또는 "방금 뗀 순간"을 감지할 수 있다:

```csharp
public class InputManager
{
    private KeyboardState _currentKeyboardState;
    private KeyboardState _previousKeyboardState;

    public void Update()
    {
        _previousKeyboardState = _currentKeyboardState;
        _currentKeyboardState = Keyboard.GetState();
    }

    /// <summary>
    /// 현재 키가 눌려있는지 확인한다.
    /// </summary>
    public bool IsKeyDown(Keys key)
    {
        return _currentKeyboardState.IsKeyDown(key);
    }

    /// <summary>
    /// 이전 프레임에는 눌려있지 않았는데, 현재 프레임에는 눌려있는지 확인한다.
    /// 즉, 방금 누른 순간을 감지한다.
    /// </summary>
    public bool IsKeyPressed(Keys key)
    {
        return _currentKeyboardState.IsKeyDown(key) && 
               !_previousKeyboardState.IsKeyDown(key);
    }

    /// <summary>
    /// 이전 프레임에는 눌려있었는데, 현재 프레임에는 눌려있지 않은지 확인한다.
    /// 즉, 방금 뗀 순간을 감지한다.
    /// </summary>
    public bool IsKeyReleased(Keys key)
    {
        return !_currentKeyboardState.IsKeyDown(key) && 
               _previousKeyboardState.IsKeyDown(key);
    }
}
```

이 클래스의 세 가지 메서드를 구분하여 사용한다:

- **IsKeyDown()**: 키가 계속 눌려있는 동안 매 프레임마다 true를 반환한다. 연속적인 이동에 사용한다.
- **IsKeyPressed()**: 키를 방금 눌렀을 때만 한 번 true를 반환한다. 점프나 공격 같은 순간적인 동작에 사용한다.
- **IsKeyReleased()**: 키를 방금 뗐을 때만 한 번 true를 반환한다. 키를 떼는 순간을 감지해야 할 때 사용한다.

### WASD와 화살표 키 모두 지원
많은 게임이 WASD와 화살표 키를 모두 지원한다. 이를 구현하려면:

```csharp
public class InputManager
{
    private KeyboardState _currentKeyboardState;
    private KeyboardState _previousKeyboardState;

    public void Update()
    {
        _previousKeyboardState = _currentKeyboardState;
        _currentKeyboardState = Keyboard.GetState();
    }

    /// <summary>
    /// 위쪽 방향 입력 여부를 반환한다.
    /// 화살표 위 또는 W 키로 입력할 수 있다.
    /// </summary>
    public bool IsMoveUpPressed()
    {
        return _currentKeyboardState.IsKeyDown(Keys.Up) || 
               _currentKeyboardState.IsKeyDown(Keys.W);
    }

    /// <summary>
    /// 아래쪽 방향 입력 여부를 반환한다.
    /// 화살표 아래 또는 S 키로 입력할 수 있다.
    /// </summary>
    public bool IsMoveDownPressed()
    {
        return _currentKeyboardState.IsKeyDown(Keys.Down) || 
               _currentKeyboardState.IsKeyDown(Keys.S);
    }

    /// <summary>
    /// 왼쪽 방향 입력 여부를 반환한다.
    /// 화살표 왼쪽 또는 A 키로 입력할 수 있다.
    /// </summary>
    public bool IsMoveLeftPressed()
    {
        return _currentKeyboardState.IsKeyDown(Keys.Left) || 
               _currentKeyboardState.IsKeyDown(Keys.A);
    }

    /// <summary>
    /// 오른쪽 방향 입력 여부를 반환한다.
    /// 화살표 오른쪽 또는 D 키로 입력할 수 있다.
    /// </summary>
    public bool IsMoveRightPressed()
    {
        return _currentKeyboardState.IsKeyDown(Keys.Right) || 
               _currentKeyboardState.IsKeyDown(Keys.D);
    }

    /// <summary>
    /// ESC 키를 방금 눌렀는지 확인한다.
    /// 게임 종료나 메뉴 열기에 사용된다.
    /// </summary>
    public bool IsExitPressed()
    {
        return IsKeyPressed(Keys.Escape);
    }

    private bool IsKeyPressed(Keys key)
    {
        return _currentKeyboardState.IsKeyDown(key) && 
               !_previousKeyboardState.IsKeyDown(key);
    }
}
```

## 6.3 플레이어 객체 구현
이제 플레이어 캐릭터를 나타내는 `Player` 클래스를 만들겠다. 이 클래스는 위치, 스프라이트, 이동 속도 등을 관리한다.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;

/// <summary>
/// 게임의 플레이어 캐릭터를 나타내는 클래스다.
/// 위치, 스프라이트, 이동 속도 등을 관리하며,
/// 입력 처리와 렌더링을 담당한다.
/// 
/// 이 클래스는 플레이어의 로컬 상태(로컬에서만 알면 되는 정보)만 관리한다.
/// 네트워크 동기화는 GameManager에서 처리한다.
/// </summary>
public class Player
{
    /// <summary>
    /// 플레이어의 위치 (화면 상의 X, Y 좌표)
    /// </summary>
    public Vector2 Position { get; set; }

    /// <summary>
    /// 플레이어의 현재 속도 벡터 (X, Y 방향 속도)
    /// </summary>
    public Vector2 Velocity { get; set; }

    /// <summary>
    /// 플레이어 스프라이트의 텍스처
    /// </summary>
    public Texture2D Texture { get; private set; }

    /// <summary>
    /// 플레이어의 너비
    /// </summary>
    public int Width { get; private set; }

    /// <summary>
    /// 플레이어의 높이
    /// </summary>
    public int Height { get; private set; }

    /// <summary>
    /// 플레이어의 이동 속도 (픽셀/초)
    /// </summary>
    private float _moveSpeed;

    /// <summary>
    /// 화면의 너비 (경계 체크 용)
    /// </summary>
    private int _screenWidth;

    /// <summary>
    /// 화면의 높이 (경계 체크 용)
    /// </summary>
    private int _screenHeight;

    /// <summary>
    /// 플레이어를 생성한다.
    /// </summary>
    /// <param name="texture">플레이어 스프라이트</param>
    /// <param name="initialPosition">초기 위치</param>
    /// <param name="screenWidth">화면 너비</param>
    /// <param name="screenHeight">화면 높이</param>
    /// <param name="moveSpeed">이동 속도 (픽셀/초)</param>
    public Player(
        Texture2D texture,
        Vector2 initialPosition,
        int screenWidth,
        int screenHeight,
        float moveSpeed = 200f)
    {
        Texture = texture;
        Position = initialPosition;
        Width = texture.Width;
        Height = texture.Height;
        _screenWidth = screenWidth;
        _screenHeight = screenHeight;
        _moveSpeed = moveSpeed;
        Velocity = Vector2.Zero;
    }

    /// <summary>
    /// 플레이어를 업데이트한다.
    /// 입력 처리, 위치 갱신, 경계 체크 등을 수행한다.
    /// </summary>
    /// <param name="gameTime">게임 시간 정보</param>
    /// <param name="inputManager">입력 정보</param>
    public void Update(GameTime gameTime, InputManager inputManager)
    {
        float deltaTime = (float)gameTime.ElapsedGameTime.TotalSeconds;

        // 속도 초기화 (이전 속도를 유지하지 않도록)
        Velocity = Vector2.Zero;

        // 입력 처리
        HandleInput(inputManager);

        // 위치 업데이트
        Position += Velocity * _moveSpeed * deltaTime;

        // 화면 경계 체크
        ClampToScreen();
    }

    /// <summary>
    /// 입력에 따라 속도를 설정한다.
    /// 여러 방향 입력을 동시에 처리할 수 있다.
    /// </summary>
    private void HandleInput(InputManager inputManager)
    {
        if (inputManager.IsMoveUpPressed())
        {
            Velocity.Y -= 1;
        }

        if (inputManager.IsMoveDownPressed())
        {
            Velocity.Y += 1;
        }

        if (inputManager.IsMoveLeftPressed())
        {
            Velocity.X -= 1;
        }

        if (inputManager.IsMoveRightPressed())
        {
            Velocity.X += 1;
        }

        // 대각선 이동 시 속도 정규화
        // 예: 위쪽과 오른쪽을 동시에 입력하면 대각선 이동
        if (Velocity.Length() > 0)
        {
            Velocity.Normalize();
        }
    }

    /// <summary>
    /// 플레이어가 화면 경계를 벗어나지 않도록 위치를 제한한다.
    /// </summary>
    private void ClampToScreen()
    {
        if (Position.X < 0)
        {
            Position.X = 0;
        }

        if (Position.X + Width > _screenWidth)
        {
            Position.X = _screenWidth - Width;
        }

        if (Position.Y < 0)
        {
            Position.Y = 0;
        }

        if (Position.Y + Height > _screenHeight)
        {
            Position.Y = _screenHeight - Height;
        }
    }

    /// <summary>
    /// 플레이어를 화면에 그린다.
    /// </summary>
    /// <param name="spriteBatch">스프라이트 배치</param>
    public void Draw(SpriteBatch spriteBatch)
    {
        spriteBatch.Draw(
            texture: Texture,
            position: Position,
            sourceRectangle: null,
            color: Color.White,
            rotation: 0f,
            origin: Vector2.Zero,
            scale: 1f,
            effects: SpriteEffects.None,
            layerDepth: 0f
        );
    }

    /// <summary>
    /// 플레이어의 좌상단 좌표와 크기로 이루어진 사각형을 반환한다.
    /// 충돌 감지에 사용된다.
    /// </summary>
    public Rectangle GetBounds()
    {
        return new Rectangle(
            (int)Position.X,
            (int)Position.Y,
            Width,
            Height
        );
    }
}
```

**클래스의 목적**: `Player` 클래스는 플레이어 캐릭터의 모든 상태와 동작을 캡슐화한다. 위치, 속도, 스프라이트, 입력 처리 등을 한 곳에서 관리하여 코드를 체계적으로 만든다.

**주요 필드들**:
- `Position`: 화면에서의 실제 위치다. 매 프레임마다 업데이트된다.
- `Velocity`: 이동 방향을 나타낸다. 크기가 1인 정규화된 벡터이며, `_moveSpeed`와 곱해진다.
- `Texture`: 플레이어 이미지다. 생성할 때 한 번만 설정되고, 이후 변경되지 않는다.

**Update 메서드**: 이것이 클래스의 핵심이다. 매 프레임마다 호출되며:
1. 입력 처리로 `Velocity` 설정
2. 속도와 시간을 곱해 위치 계산
3. 화면 경계 체크

**대각선 이동 정규화**: 사용자가 위쪽과 오른쪽을 동시에 누르면 벡터는 (−1, 1)이 되는데, 이를 정규화하면 크기가 1인 대각선 방향이 된다. 이렇게 하면 대각선 이동도 수평 이동과 같은 속도로 이루어진다.

**Draw 메서드**: `SpriteBatch.Draw()`의 모든 매개변수를 명시적으로 전달한다. 기본값을 사용하되, 각 매개변수의 의미를 이해하는 것이 중요하다.

### InputManager와 Player를 GameManager에 통합
이제 앞서 만든 `InputManager`와 `Player`를 `GameManager`에 통합하자:

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;

/// <summary>
/// 게임의 전체 로직을 관리하는 클래스다.
/// 플레이어, 입력, 네트워크 등을 조율한다.
/// </summary>
public class GameManager
{
    private Player _player;
    private InputManager _inputManager;
    
    private NetworkClient _networkClient;
    private MessageProcessor _messageProcessor;
    private bool _isNetworkInitialized;
    
    private int _screenWidth;
    private int _screenHeight;

    public GameManager(int screenWidth, int screenHeight)
    {
        _screenWidth = screenWidth;
        _screenHeight = screenHeight;
        _inputManager = new InputManager();
    }

    /// <summary>
    /// 플레이어를 초기화한다.
    /// LoadContent 이후에 호출되어야 한다.
    /// </summary>
    public void InitializePlayer(Texture2D playerTexture)
    {
        Vector2 initialPosition = new Vector2(
            _screenWidth / 2 - playerTexture.Width / 2,
            _screenHeight / 2 - playerTexture.Height / 2
        );

        _player = new Player(
            texture: playerTexture,
            initialPosition: initialPosition,
            screenWidth: _screenWidth,
            screenHeight: _screenHeight,
            moveSpeed: 200f
        );
    }

    /// <summary>
    /// 네트워크를 비동기로 초기화한다.
    /// </summary>
    public async void InitializeNetworkAsync()
    {
        _networkClient = new NetworkClient();
        
        bool connected = await _networkClient.ConnectAsync("127.0.0.1", 5000);
        
        if (connected)
        {
            _messageProcessor = new MessageProcessor(_networkClient);
            _ = _networkClient.StartReceivingAsync();
            _isNetworkInitialized = true;
        }
    }

    /// <summary>
    /// 게임 로직을 매 프레임마다 업데이트한다.
    /// </summary>
    public void Update(GameTime gameTime)
    {
        // 입력 상태 업데이트
        _inputManager.Update();

        // 게임 종료 요청 확인
        if (_inputManager.IsExitPressed())
        {
            // 게임 종료 신호는 Game1에서 처리
        }

        // 플레이어 업데이트
        if (_player != null)
        {
            _player.Update(gameTime, _inputManager);
        }

        // 네트워크 메시지 처리
        if (_isNetworkInitialized && _messageProcessor != null)
        {
            _messageProcessor.ProcessMessages();
        }
    }

    /// <summary>
    /// 화면에 게임을 렌더링한다.
    /// </summary>
    public void Draw(SpriteBatch spriteBatch, SpriteFont font)
    {
        // 플레이어 그리기
        if (_player != null)
        {
            _player.Draw(spriteBatch);
        }

        // 디버그 정보 표시
        DrawDebugInfo(spriteBatch, font);
    }

    /// <summary>
    /// 디버그 정보를 화면에 표시한다.
    /// </summary>
    private void DrawDebugInfo(SpriteBatch spriteBatch, SpriteFont font)
    {
        if (_player == null)
            return;

        string positionText = $"Position: ({_player.Position.X:F0}, {_player.Position.Y:F0})";
        string velocityText = $"Velocity: ({_player.Velocity.X:F2}, {_player.Velocity.Y:F2})";
        string networkText = $"Network: {(_isNetworkInitialized ? "Connected" : "Disconnected")}";

        spriteBatch.DrawString(font, positionText, new Vector2(10, 10), Color.White);
        spriteBatch.DrawString(font, velocityText, new Vector2(10, 35), Color.White);
        spriteBatch.DrawString(font, networkText, new Vector2(10, 60), Color.White);
    }

    /// <summary>
    /// 게임을 종료한다.
    /// 네트워크 연결을 정리한다.
    /// </summary>
    public void Shutdown()
    {
        if (_networkClient != null)
        {
            _networkClient.Disconnect();
        }
    }
}
```

### Game1에서 GameManager 사용
마지막으로 `Game1` 클래스를 업데이트하여 새로운 구조를 사용하도록 한다:

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;

public class Game1 : Game
{
    private GraphicsDeviceManager _graphics;
    private SpriteBatch _spriteBatch;
    private SpriteFont _font;
    private GameManager _gameManager;
    private Texture2D _playerTexture;

    public Game1()
    {
        _graphics = new GraphicsDeviceManager(this);
        Content.RootDirectory = "Content";
        IsMouseVisible = true;
    }

    protected override void Initialize()
    {
        int width = _graphics.PreferredBackBufferWidth;
        int height = _graphics.PreferredBackBufferHeight;
        
        _gameManager = new GameManager(width, height);
        _gameManager.InitializeNetworkAsync();
        
        base.Initialize();
    }

    protected override void LoadContent()
    {
        _spriteBatch = new SpriteBatch(GraphicsDevice);
        
        // 플레이어 스프라이트 로드 또는 생성
        try
        {
            _playerTexture = Content.Load<Texture2D>("Graphics/Player");
        }
        catch
        {
            // 파일이 없으면 동적으로 생성
            _playerTexture = SpriteGenerator.CreateBorderedSquare(
                GraphicsDevice,
                32,
                32,
                Color.Red,
                Color.DarkRed
            );
        }
        
        // 폰트 로드
        try
        {
            _font = Content.Load<SpriteFont>("Arial");
        }
        catch
        {
            // 폰트 파일이 없으면 기본 폰트 사용
            _font = null;
        }

        _gameManager.InitializePlayer(_playerTexture);
    }

    protected override void Update(GameTime gameTime)
    {
        _gameManager.Update(gameTime);
        base.Update(gameTime);
    }

    protected override void Draw(GameTime gameTime)
    {
        GraphicsDevice.Clear(Color.CornflowerBlue);

        _spriteBatch.Begin(SpriteSortMode.Deferred, BlendState.AlphaBlend);
        
        if (_font != null)
        {
            _gameManager.Draw(_spriteBatch, _font);
        }
        else
        {
            // 폰트 없이 플레이어만 그리기
            _gameManager.Draw(_spriteBatch, null);
        }
        
        _spriteBatch.End();

        base.Draw(gameTime);
    }

    protected override void OnExiting(object sender, System.EventArgs args)
    {
        _gameManager.Shutdown();
        base.OnExiting(sender, args);
    }
}
```

## 종합 정리
6장에서 배운 내용을 정리한다. MonoGame에서 스프라이트는 2D 게임의 기본 요소이며, `Texture2D` 클래스로 표현된다. 파일에서 로드하거나 코드로 동적으로 생성할 수 있다.

입력 처리는 `KeyboardState`를 사용하여 구현하며, `InputManager` 클래스를 만들어 입력 로직을 캡슐화할 수 있다. `Player` 클래스는 위치, 속도, 스프라이트를 관리하고, 입력에 따라 움직이며, 화면 경계를 벗어나지 않도록 제한한다.

이러한 구조를 사용하면 나중에 여러 플레이어를 추가하거나, 복잡한 게임 로직을 구현할 때 코드를 체계적으로 관리할 수 있다. 다음 장에서는 네트워크 통신과 게임을 통합하여 여러 플레이어가 같은 게임 월드에서 상호작용할 수 있도록 할 것이다.

### 객관식 문제
**문제 1**: MonoGame에서 스프라이트를 나타내는 클래스는?
- ① Sprite
- ② Texture2D
- ③ SpriteImage
- ④ GraphicsObject

**정답**: ②

**해설**: MonoGame에서 2D 이미지는 `Texture2D` 클래스로 표현된다. 이것은 GPU 메모리에 저장된 텍스처 데이터를 나타낸다.

---

**문제 2**: 벡터를 정규화(Normalize)하는 이유는?
- ① 벡터의 방향만 유지하고 크기를 1로 만들기 위해
- ② 벡터의 크기만 유지하고 방향을 바꾸기 위해
- ③ 벡터를 더 빠르게 계산하기 위해
- ④ 벡터의 색상을 변경하기 위해

**정답**: ①

**해설**: 정규화된 벡터는 방향은 유지하면서 크기가 1인 벡터가 된다. 이렇게 하면 대각선 이동도 직선 이동과 같은 속도로 이루어진다.

---

**문제 3**: `InputManager.IsKeyPressed()`와 `InputManager.IsKeyDown()`의 차이는?
- ① 차이 없다
- ② IsKeyPressed는 한 번만 true, IsKeyDown은 계속 true
- ③ IsKeyDown은 한 번만 true, IsKeyPressed는 계속 true
- ④ IsKeyPressed는 마우스 입력, IsKeyDown은 키보드 입력

**정답**: ②

**해설**: `IsKeyDown()`은 키가 눌려있는 동안 매 프레임마다 true이고, `IsKeyPressed()`는 방금 눌렀을 때만 한 번 true다.

- ④ 플레이어의 투명도를 설정한다

**정답**: ②

### 서술형 문제

**문제 5**: 스프라이트를 파일에서 로드하는 방법과 동적으로 생성하는 방법의 장단점을 비교하시오.

**예시 답안**:

**파일에서 로드하는 방법**:
- 장점: 원하는 그래픽을 정확하게 표현할 수 있다. 품질이 높다.
- 단점: 파일을 준비해야 하고, 파일이 없으면 오류가 발생한다.

**동적으로 생성하는 방법**:
- 장점: 코드만으로 간단하게 이미지를 만들 수 있다. 파일 의존성이 없다.
- 단점: 복잡한 그래픽을 만들 수 없다. 성능이 좋지 않을 수 있다.



**문제 6**: 다음 코드의 문제점을 찾고 수정하시오.

```csharp
public void Update(GameTime gameTime, InputManager inputManager)
{
    // 이 코드는 플레이어를 8배 빠르게 이동시킨다
    if (inputManager.IsMoveUpPressed())
        Velocity.Y -= 1;
    if (inputManager.IsMoveDownPressed())
        Velocity.Y += 1;
    if (inputManager.IsMoveLeftPressed())
        Velocity.X -= 1;
    if (inputManager.IsMoveRightPressed())
        Velocity.X += 1;
    
    if (Velocity.Length() > 0)
        Velocity.Normalize();
    
    // 위치 업데이트
    for (int i = 0; i < 8; i++)  // 이 부분이 문제!
    {
        Position += Velocity * _moveSpeed * (float)gameTime.ElapsedGameTime.TotalSeconds;
    }
}
```

**예시 답안**: 문제는 위치 업데이트를 반복문으로 8번 수행한다는 것이다. 이는 의도하지 않은 8배 빠른 이동을 일으킨다.

**수정된 코드**:
```csharp
public void Update(GameTime gameTime, InputManager inputManager)
{
    float deltaTime = (float)gameTime.ElapsedGameTime.TotalSeconds;
    
    Velocity = Vector2.Zero;
    
    if (inputManager.IsMoveUpPressed())
        Velocity.Y -= 1;
    if (inputManager.IsMoveDownPressed())
        Velocity.Y += 1;
    if (inputManager.IsMoveLeftPressed())
        Velocity.X -= 1;
    if (inputManager.IsMoveRightPressed())
        Velocity.X += 1;
    
    if (Velocity.Length() > 0)
        Velocity.Normalize();
    
    // 위치 업데이트 (한 번만)
    Position += Velocity * _moveSpeed * deltaTime;
    
    ClampToScreen();
}
```


**문제 7**: `Player` 클래스의 생성자에서 `Position`을 초기화할 때, 화면의 정확한 중앙에 배치하려면 어떻게 해야 하는가?

**예시 답안**:
```csharp
Vector2 initialPosition = new Vector2(
    _screenWidth / 2 - Width / 2,    // 가로 중앙
    _screenHeight / 2 - Height / 2   // 세로 중앙
);
```

플레이어 스프라이트의 좌상단이 좌표계의 원점이므로, 정확한 중앙에 배치하려면 스크린 중앙에서 스프라이트의 절반 크기를 빼야 한다.
  
  
**문제 8** (심화): `Player` 클래스에 `public float Velocity { get; set; }` 형태의 속성이 아닌 `private float _moveSpeed`처럼 `private` 필드를 사용하는 이유를 설명하시오.

**예시 답안**: `_moveSpeed`는 플레이어의 이동 속도 상수로, 게임 중에 변경되지 않아야 하는 값이다. `private` 필드로 만들면 클래스 외부에서 실수로 변경할 수 없다. 반면 `Velocity`는 입력에 따라 매 프레임 변경되는 값이므로 public으로 공개한다. 이렇게 하면 캡슐화 원칙을 지키면서도 필요한 데이터는 공개할 수 있다.

  

**문제 9** (과제): 다음 기능을 가진 향상된 `Player` 클래스를 만드시오.

1. **플레이어 색상 변경**: 1, 2, 3, 4 키를 눌러 색상을 빨강, 파랑, 초록, 노랑으로 변경할 수 있어야 한다.

2. **속도 부스트**: SHIFT 키를 누르고 있을 때는 이동 속도가 1.5배가 되어야 한다.

3. **방향 표시**: 플레이어가 어느 방향으로 이동 중인지 보여주기 위해, 플레이어 방향을 기반으로 스프라이트를 회전시킨다 (또는 색상으로 표시).

4. **이동 발자국**: 플레이어가 이동했던 경로를 남기도록 이전 5개의 위치를 저장하고, 화면에 점으로 표시한다 (선택사항).

**요구사항**:
- 색상 변경이 즉시 반영되어야 한다.
- 속도 부스트 중에도 정확한 속도가 유지되어야 한다.
- 모든 기능이 `Update()` 메서드 내에서 처리되어야 한다.

**제출 방법**: 수정된 `Player.cs`와 `InputManager.cs` 파일을 제출한다.
