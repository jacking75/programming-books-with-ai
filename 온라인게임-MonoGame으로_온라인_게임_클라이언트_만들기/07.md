# 7장: 플레이어 위치 동기화
지금까지 우리는 로컬 게임(혼자 하는 게임)을 만들었다. 이제 온라인 게임의 핵심 기능인 플레이어 위치 동기화를 구현하겠다. 여러 플레이어가 같은 월드에 있을 때, 한 플레이어의 움직임을 다른 플레이어들이 볼 수 있도록 해야 한다.

## 7.1 위치 패킷 프로토콜

### 패킷 설계의 필요성
온라인 게임에서 플레이어의 위치를 동기화하려면, 먼저 위치 정보를 어떻게 전송할지 정의해야 한다. 이를 프로토콜이라고 하며, 클라이언트와 서버가 같은 형식으로 데이터를 주고받을 수 있도록 표준화된다.

위치 패킷은 다음과 같은 정보를 포함해야 한다:

- **플레이어 ID**: 어느 플레이어의 위치인지 식별
- **X 좌표**: 플레이어의 가로 위치
- **Y 좌표**: 플레이어의 세로 위치
- **패킷 타입**: 이것이 위치 정보라는 것을 명시
- **(선택사항) 방향 정보**: 플레이어가 보고 있는 방향

### 바이너리 형식 정의
3장에서 배운 바이너리 직렬화를 사용하여 위치 패킷의 형식을 정의하자:

```
┌────────────────────────────────────────────┐
│ 위치 동기화 패킷 (Position Update Packet)    │
├────────────────────────────────────────────┤
│ 바이트 0        : 패킷 타입 (1)              │
│ 바이트 1-4      : 플레이어 ID (정수)          │
│ 바이트 5-8      : X 좌표 (실수)              │
│ 바이트 9-12     : Y 좌표 (실수)              │
│ 바이트 13-14    : 방향 정보 (short)          │
├────────────────────────────────────────────┤
│ 총 크기: 15 바이트                           │
└────────────────────────────────────────────┘
```

### PacketHandler에 위치 패킷 추가
4장에서 만든 `PacketHandler` 클래스를 확장하여 위치 패킷을 처리하도록 한다:

```csharp
/// <summary>
/// 패킷 핸들러에 위치 관련 상수와 메서드를 추가한다.
/// </summary>
public partial class PacketHandler
{
    // 기존 패킷 타입들
    public const byte PACKET_TYPE_POSITION = 1;
    public const byte PACKET_TYPE_CHAT = 2;
    public const byte PACKET_TYPE_LOGIN = 3;
    public const byte PACKET_TYPE_PLAYER_JOIN = 4;
    public const byte PACKET_TYPE_PLAYER_LEAVE = 5;
    
    // 위치 패킷의 구조 정의
    public const int POSITION_PACKET_SIZE = 15;
    private const int POSITION_PACKET_TYPE_OFFSET = 0;
    private const int POSITION_PLAYER_ID_OFFSET = 1;
    private const int POSITION_X_OFFSET = 5;
    private const int POSITION_Y_OFFSET = 9;
    private const int POSITION_DIRECTION_OFFSET = 13;

    /// <summary>
    /// 위치 정보 패킷을 생성한다.
    /// 플레이어의 현재 위치를 서버로 전송할 때 사용한다.
    /// </summary>
    /// <param name="playerId">플레이어 ID</param>
    /// <param name="x">X 좌표</param>
    /// <param name="y">Y 좌표</param>
    /// <param name="direction">방향 (라디안, 0-360도)</param>
    /// <returns>패킷 데이터 (바이트 배열)</returns>
    public static byte[] CreatePositionPacket(
        int playerId,
        float x,
        float y,
        short direction = 0)
    {
        byte[] packet = new byte[POSITION_PACKET_SIZE];
        
        // 패킷 타입
        packet[POSITION_PACKET_TYPE_OFFSET] = PACKET_TYPE_POSITION;
        
        // 플레이어 ID (4바이트)
        Array.Copy(
            BitConverter.GetBytes(playerId),
            0,
            packet,
            POSITION_PLAYER_ID_OFFSET,
            4
        );
        
        // X 좌표 (4바이트 float)
        Array.Copy(
            BitConverter.GetBytes(x),
            0,
            packet,
            POSITION_X_OFFSET,
            4
        );
        
        // Y 좌표 (4바이트 float)
        Array.Copy(
            BitConverter.GetBytes(y),
            0,
            packet,
            POSITION_Y_OFFSET,
            4
        );
        
        // 방향 정보 (2바이트 short)
        Array.Copy(
            BitConverter.GetBytes(direction),
            0,
            packet,
            POSITION_DIRECTION_OFFSET,
            2
        );
        
        return packet;
    }

    /// <summary>
    /// 위치 정보 패킷을 파싱한다.
    /// 서버로부터 받은 다른 플레이어의 위치 정보를 해석할 때 사용한다.
    /// </summary>
    /// <param name="packetData">수신한 패킷 데이터</param>
    /// <returns>파싱된 위치 메시지 객체</returns>
    public static PositionMessage ParsePositionMessage(byte[] packetData)
    {
        if (packetData == null || packetData.Length < POSITION_PACKET_SIZE)
        {
            throw new ArgumentException("위치 패킷이 너무 짧습니다");
        }

        int playerId = BitConverter.ToInt32(packetData, POSITION_PLAYER_ID_OFFSET);
        float x = BitConverter.ToSingle(packetData, POSITION_X_OFFSET);
        float y = BitConverter.ToSingle(packetData, POSITION_Y_OFFSET);
        short direction = BitConverter.ToInt16(packetData, POSITION_DIRECTION_OFFSET);

        return new PositionMessage
        {
            PlayerId = playerId,
            X = x,
            Y = y,
            Direction = direction
        };
    }
}

/// <summary>
/// 서버로부터 수신한 위치 정보를 나타내는 메시지 클래스다.
/// </summary>
public class PositionMessage
{
    public int PlayerId { get; set; }
    public float X { get; set; }
    public float Y { get; set; }
    public short Direction { get; set; }
}
```

이 코드의 목적을 설명한다:

**CreatePositionPacket**: 로컬 플레이어의 현재 위치를 패킷으로 변환한다. 이 패킷을 서버로 보내면 서버가 모든 클라이언트에게 브로드캐스트하여 다른 플레이어들의 화면에 위치가 업데이트된다.

**ParsePositionMessage**: 서버에서 받은 위치 패킷을 `PositionMessage` 객체로 변환한다. 이렇게 하면 게임 로직에서 쉽게 위치 정보에 접근할 수 있다.

**오프셋 상수**: 각 데이터가 패킷의 어느 위치에 있는지 명확하게 정의한다. 이렇게 하면 코드가 가독성이 높아지고 유지보수가 쉬워진다.

### 네트워크 프로토콜 흐름

```
┌──────────────────────────────────────────────────────┐
│ 플레이어 1의 위치가 변경됨                           │
│ Position: (100, 150)                                 │
└──────────────────────────────────────────────────────┘
                         ↓
┌──────────────────────────────────────────────────────┐
│ 위치 패킷 생성                                       │
│ [타입: 1][ID: 1][X: 100][Y: 150][방향: 0]         │
└──────────────────────────────────────────────────────┘
                         ↓
┌──────────────────────────────────────────────────────┐
│ 네트워크를 통해 서버로 전송                          │
└──────────────────────────────────────────────────────┘
                         ↓
┌──────────────────────────────────────────────────────┐
│ 서버가 받은 패킷을 모든 클라이언트에게 브로드캐스트  │
└──────────────────────────────────────────────────────┘
                         ↓
┌──────────────────────────────────────────────────────┐
│ 플레이어 2가 패킷 수신                               │
│ 다른 플레이어(플레이어 1)의 위치 업데이트           │
│ 화면에 플레이어 1을 새로운 위치에 그림              │
└──────────────────────────────────────────────────────┘
```

## 7.2 플레이어 움직임 전송

### 위치 업데이트 주기 결정
온라인 게임에서 매 프레임마다 위치 정보를 보내면 네트워크 트래픽이 너무 많아진다. 실제 게임들은 일정한 시간 간격으로 위치를 업데이트한다.

```csharp
/// <summary>
/// 플레이어의 위치 동기화를 관리하는 클래스다.
/// 로컬 플레이어의 위치를 일정한 간격으로 서버로 전송하고,
/// 다른 플레이어들의 위치 업데이트를 처리한다.
/// </summary>
public class PositionSynchronizer
{
    private NetworkClient _networkClient;
    
    /// <summary>
    /// 위치 업데이트 전송 간격 (초 단위)
    /// 0.1초 간격은 초당 10번 위치를 전송한다는 뜻이다.
    /// 네트워크 트래픽과 동기화 정확도의 균형을 맞춘다.
    /// </summary>
    private const float POSITION_UPDATE_INTERVAL = 0.1f;
    
    /// <summary>
    /// 마지막으로 위치를 전송한 이후 경과 시간
    /// </summary>
    private float _timeSinceLastPositionUpdate;
    
    /// <summary>
    /// 로컬 플레이어의 ID
    /// 서버에서 로그인 시 할당받는다.
    /// </summary>
    private int _localPlayerId;

    public PositionSynchronizer(NetworkClient networkClient)
    {
        _networkClient = networkClient;
        _timeSinceLastPositionUpdate = 0f;
    }

    /// <summary>
    /// 로컬 플레이어의 ID를 설정한다.
    /// </summary>
    public void SetLocalPlayerId(int playerId)
    {
        _localPlayerId = playerId;
    }

    /// <summary>
    /// 플레이어의 위치가 변경되었을 때, 서버로 전송할지 결정하고 전송한다.
    /// 매 프레임마다 Update 메서드에서 호출된다.
    /// </summary>
    /// <param name="gameTime">게임 시간 정보</param>
    /// <param name="player">로컬 플레이어 객체</param>
    public async void Update(GameTime gameTime, Player player)
    {
        // 지난 프레임 이후 경과한 시간을 누적
        _timeSinceLastPositionUpdate += (float)gameTime.ElapsedGameTime.TotalSeconds;

        // 충분한 시간이 경과했으면 위치 정보 전송
        if (_timeSinceLastPositionUpdate >= POSITION_UPDATE_INTERVAL)
        {
            await SendPositionUpdate(player);
            _timeSinceLastPositionUpdate = 0f;
        }
    }

    /// <summary>
    /// 로컬 플레이어의 현재 위치를 서버로 전송한다.
    /// </summary>
    private async Task SendPositionUpdate(Player player)
    {
        if (!_networkClient.IsConnected)
        {
            return;
        }

        try
        {
            // 위치 패킷 생성
            byte[] packet = PacketHandler.CreatePositionPacket(
                playerId: _localPlayerId,
                x: player.Position.X,
                y: player.Position.Y,
                direction: 0 // 나중에 방향 정보 추가 가능
            );

            // 서버로 전송
            await _networkClient.SendPacketAsync(packet);
        }
        catch (Exception ex)
        {
            Console.WriteLine($"[오류] 위치 전송 중 예외 발생: {ex.Message}");
        }
    }
}
```

이 클래스의 설계를 설명한다:

**POSITION_UPDATE_INTERVAL**: 0.1초 간격으로 위치를 전송한다는 것은 초당 10번 전송한다는 뜻이다. 이는 네트워크 대역폭을 과도하게 사용하지 않으면서도 충분히 부드러운 이동을 보여준다.

**시간 누적**: `_timeSinceLastPositionUpdate`를 사용하여 정확히 0.1초 간격으로 전송하도록 한다. 프레임 레이트가 변해도 일정한 간격으로 동작한다.

**비동기 전송**: `SendPositionUpdate` 메서드를 `async`로 만들어 네트워크 전송이 게임 루프를 블로킹하지 않도록 한다.

### GameManager에 PositionSynchronizer 통합

```csharp
/// <summary>
/// 게임의 전체 로직을 관리하는 클래스다.
/// 플레이어, 입력, 네트워크, 위치 동기화 등을 조율한다.
/// </summary>
public class GameManager
{
    private Player _player;
    private InputManager _inputManager;
    
    private NetworkClient _networkClient;
    private MessageProcessor _messageProcessor;
    private PositionSynchronizer _positionSynchronizer;
    
    private bool _isNetworkInitialized;
    
    private int _screenWidth;
    private int _screenHeight;

    public GameManager(int screenWidth, int screenHeight)
    {
        _screenWidth = screenWidth;
        _screenHeight = screenHeight;
        _inputManager = new InputManager();
    }

    /// <summary>
    /// 플레이어를 초기화한다.
    /// </summary>
    public void InitializePlayer(Texture2D playerTexture)
    {
        Vector2 initialPosition = new Vector2(
            _screenWidth / 2 - playerTexture.Width / 2,
            _screenHeight / 2 - playerTexture.Height / 2
        );

        _player = new Player(
            texture: playerTexture,
            initialPosition: initialPosition,
            screenWidth: _screenWidth,
            screenHeight: _screenHeight,
            moveSpeed: 200f
        );
    }

    /// <summary>
    /// 네트워크를 비동기로 초기화한다.
    /// </summary>
    public async void InitializeNetworkAsync()
    {
        _networkClient = new NetworkClient();
        
        bool connected = await _networkClient.ConnectAsync("127.0.0.1", 5000);
        
        if (connected)
        {
            _messageProcessor = new MessageProcessor(_networkClient);
            _positionSynchronizer = new PositionSynchronizer(_networkClient);
            
            // 핸들러 등록
            _messageProcessor.RegisterHandler(
                typeof(PositionMessage),
                msg => HandlePositionMessage((PositionMessage)msg)
            );
            
            _ = _networkClient.StartReceivingAsync();
            _isNetworkInitialized = true;
            
            Console.WriteLine("[정보] 네트워크 초기화 완료");
        }
    }

    /// <summary>
    /// 게임 로직을 매 프레임마다 업데이트한다.
    /// </summary>
    public void Update(GameTime gameTime)
    {
        _inputManager.Update();

        if (_player != null)
        {
            _player.Update(gameTime, _inputManager);
        }

        // 위치 동기화
        if (_isNetworkInitialized && _positionSynchronizer != null && _player != null)
        {
            _positionSynchronizer.Update(gameTime, _player);
        }

        // 네트워크 메시지 처리
        if (_isNetworkInitialized && _messageProcessor != null)
        {
            _messageProcessor.ProcessMessages();
        }
    }

    /// <summary>
    /// 다른 플레이어로부터 받은 위치 메시지를 처리한다.
    /// </summary>
    private void HandlePositionMessage(PositionMessage msg)
    {
        Console.WriteLine($"[정보] 플레이어 {msg.PlayerId}의 위치 업데이트: ({msg.X:F0}, {msg.Y:F0})");
        // 이후 다른 플레이어 객체를 관리할 때 여기서 위치를 업데이트한다
    }

    /// <summary>
    /// 화면에 게임을 렌더링한다.
    /// </summary>
    public void Draw(SpriteBatch spriteBatch, SpriteFont font)
    {
        if (_player != null)
        {
            _player.Draw(spriteBatch);
        }

        DrawDebugInfo(spriteBatch, font);
    }

    /// <summary>
    /// 디버그 정보를 화면에 표시한다.
    /// </summary>
    private void DrawDebugInfo(SpriteBatch spriteBatch, SpriteFont font)
    {
        if (_player == null || font == null)
            return;

        string positionText = $"로컬 위치: ({_player.Position.X:F0}, {_player.Position.Y:F0})";
        string syncText = $"위치 동기화: {(_isNetworkInitialized ? "활성화" : "비활성화")}";
        string networkText = $"네트워크: {(_isNetworkInitialized ? "연결됨" : "연결 안됨")}";

        spriteBatch.DrawString(font, positionText, new Vector2(10, 10), Color.White);
        spriteBatch.DrawString(font, syncText, new Vector2(10, 35), Color.White);
        spriteBatch.DrawString(font, networkText, new Vector2(10, 60), Color.White);
    }

    /// <summary>
    /// 게임을 종료한다.
    /// </summary>
    public void Shutdown()
    {
        if (_networkClient != null)
        {
            _networkClient.Disconnect();
        }
    }
}
```

## 7.3 다른 플레이어 위치 수신 및 표시

### RemotePlayer 클래스 구현
이제 서버에서 받은 다른 플레이어의 위치 정보를 표시해야 한다. 원격 플레이어(remote player)를 나타내는 새로운 클래스를 만들자:

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;

/// <summary>
/// 다른 플레이어를 나타내는 클래스다.
/// 로컬 플레이어와 달리, 입력 처리가 없고 네트워크로부터 받은 위치만 업데이트된다.
/// </summary>
public class RemotePlayer
{
    /// <summary>
    /// 원격 플레이어의 고유 ID
    /// </summary>
    public int PlayerId { get; private set; }

    /// <summary>
    /// 원격 플레이어의 현재 위치
    /// </summary>
    public Vector2 Position { get; set; }

    /// <summary>
    /// 원격 플레이어의 스프라이트
    /// </summary>
    public Texture2D Texture { get; private set; }

    /// <summary>
    /// 원격 플레이어의 너비
    /// </summary>
    public int Width { get; private set; }

    /// <summary>
    /// 원격 플레이어의 높이
    /// </summary>
    public int Height { get; private set; }

    /// <summary>
    /// 마지막으로 위치 업데이트를 받은 이후 경과 시간
    /// 일정 시간 동안 업데이트가 없으면 플레이어를 제거할 수 있다.
    /// </summary>
    public float TimeSinceLastUpdate { get; set; }

    /// <summary>
    /// 플레이어의 이름 (나중에 표시할 예정)
    /// </summary>
    public string PlayerName { get; set; }

    /// <summary>
    /// 원격 플레이어를 생성한다.
    /// </summary>
    /// <param name="playerId">플레이어의 고유 ID</param>
    /// <param name="texture">플레이어 스프라이트</param>
    /// <param name="initialPosition">초기 위치</param>
    /// <param name="playerName">플레이어 이름</param>
    public RemotePlayer(
        int playerId,
        Texture2D texture,
        Vector2 initialPosition,
        string playerName = "Unknown")
    {
        PlayerId = playerId;
        Texture = texture;
        Position = initialPosition;
        Width = texture.Width;
        Height = texture.Height;
        PlayerName = playerName;
        TimeSinceLastUpdate = 0f;
    }

    /// <summary>
    /// 원격 플레이어를 업데이트한다.
    /// 마지막 위치 업데이트 이후의 시간을 누적한다.
    /// </summary>
    public void Update(GameTime gameTime)
    {
        TimeSinceLastUpdate += (float)gameTime.ElapsedGameTime.TotalSeconds;
    }

    /// <summary>
    /// 원격 플레이어의 위치를 업데이트한다.
    /// 서버로부터 새로운 위치 정보를 받았을 때 호출된다.
    /// </summary>
    /// <param name="newPosition">새로운 위치</param>
    public void UpdatePosition(Vector2 newPosition)
    {
        Position = newPosition;
        TimeSinceLastUpdate = 0f;
    }

    /// <summary>
    /// 원격 플레이어를 화면에 그린다.
    /// 로컬 플레이어와 다른 색상으로 표시된다.
    /// </summary>
    public void Draw(SpriteBatch spriteBatch)
    {
        // 원격 플레이어는 파란색으로 표시
        spriteBatch.Draw(
            texture: Texture,
            position: Position,
            sourceRectangle: null,
            color: Color.Blue,
            rotation: 0f,
            origin: Vector2.Zero,
            scale: 1f,
            effects: SpriteEffects.None,
            layerDepth: 0f
        );
    }

    /// <summary>
    /// 원격 플레이어의 이름을 화면에 그린다.
    /// </summary>
    public void DrawName(SpriteBatch spriteBatch, SpriteFont font)
    {
        if (font == null)
            return;

        Vector2 namePosition = Position + new Vector2(0, Height + 5);
        spriteBatch.DrawString(font, PlayerName, namePosition, Color.White);
    }

    /// <summary>
    /// 원격 플레이어의 경계를 나타내는 사각형을 반환한다.
    /// 충돌 감지에 사용된다.
    /// </summary>
    public Rectangle GetBounds()
    {
        return new Rectangle(
            (int)Position.X,
            (int)Position.Y,
            Width,
            Height
        );
    }
}
```

### RemotePlayerManager 클래스
여러 원격 플레이어를 관리하는 클래스를 만들자:

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;
using System;
using System.Collections.Generic;

/// <summary>
/// 게임 월드에 있는 모든 원격 플레이어를 관리하는 클래스다.
/// 플레이어 추가, 제거, 업데이트, 렌더링 등을 담당한다.
/// </summary>
public class RemotePlayerManager
{
    /// <summary>
    /// 플레이어 ID를 키로, RemotePlayer 객체를 값으로 하는 딕셔너리
    /// </summary>
    private Dictionary<int, RemotePlayer> _remotePlayers;

    /// <summary>
    /// 원격 플레이어에 사용할 스프라이트 텍스처
    /// </summary>
    private Texture2D _playerTexture;

    /// <summary>
    /// 플레이어 타임아웃 시간 (초 단위)
    /// 이 시간 동안 업데이트가 없는 플레이어는 제거된다.
    /// </summary>
    private const float PLAYER_TIMEOUT = 5f;

    public RemotePlayerManager(Texture2D playerTexture)
    {
        _remotePlayers = new Dictionary<int, RemotePlayer>();
        _playerTexture = playerTexture;
    }

    /// <summary>
    /// 새로운 원격 플레이어를 추가한다.
    /// </summary>
    /// <param name="playerId">플레이어의 고유 ID</param>
    /// <param name="position">플레이어의 초기 위치</param>
    /// <param name="playerName">플레이어의 이름</param>
    public void AddPlayer(int playerId, Vector2 position, string playerName = "Unknown")
    {
        if (_remotePlayers.ContainsKey(playerId))
        {
            Console.WriteLine($"[경고] 플레이어 {playerId}가 이미 존재합니다");
            return;
        }

        RemotePlayer remotePlayer = new RemotePlayer(
            playerId: playerId,
            texture: _playerTexture,
            initialPosition: position,
            playerName: playerName
        );

        _remotePlayers[playerId] = remotePlayer;
        Console.WriteLine($"[정보] 플레이어 {playerName}(ID: {playerId})이 추가되었습니다");
    }

    /// <summary>
    /// 원격 플레이어를 제거한다.
    /// </summary>
    /// <param name="playerId">제거할 플레이어의 ID</param>
    public void RemovePlayer(int playerId)
    {
        if (_remotePlayers.Remove(playerId))
        {
            Console.WriteLine($"[정보] 플레이어 {playerId}가 제거되었습니다");
        }
    }

    /// <summary>
    /// 특정 플레이어의 위치를 업데이트한다.
    /// 서버로부터 받은 위치 메시지를 처리할 때 호출된다.
    /// </summary>
    /// <param name="playerId">업데이트할 플레이어의 ID</param>
    /// <param name="newPosition">새로운 위치</param>
    public void UpdatePlayerPosition(int playerId, Vector2 newPosition)
    {
        if (_remotePlayers.TryGetValue(playerId, out var player))
        {
            player.UpdatePosition(newPosition);
        }
        else
        {
            Console.WriteLine($"[경고] 플레이어 {playerId}를 찾을 수 없습니다");
        }
    }

    /// <summary>
    /// 모든 원격 플레이어를 업데이트한다.
    /// 타임아웃된 플레이어를 제거한다.
    /// </summary>
    public void Update(GameTime gameTime)
    {
        List<int> playersToRemove = new List<int>();

        foreach (var kvp in _remotePlayers)
        {
            kvp.Value.Update(gameTime);

            // 일정 시간 동안 업데이트가 없으면 제거
            if (kvp.Value.TimeSinceLastUpdate > PLAYER_TIMEOUT)
            {
                playersToRemove.Add(kvp.Key);
            }
        }

        // 타임아웃된 플레이어 제거
        foreach (int playerId in playersToRemove)
        {
            RemovePlayer(playerId);
        }
    }

    /// <summary>
    /// 모든 원격 플레이어를 화면에 그린다.
    /// </summary>
    public void Draw(SpriteBatch spriteBatch)
    {
        foreach (var player in _remotePlayers.Values)
        {
            player.Draw(spriteBatch);
        }
    }

    /// <summary>
    /// 모든 원격 플레이어의 이름을 화면에 그린다.
    /// </summary>
    public void DrawNames(SpriteBatch spriteBatch, SpriteFont font)
    {
        if (font == null)
            return;

        foreach (var player in _remotePlayers.Values)
        {
            player.DrawName(spriteBatch, font);
        }
    }

    /// <summary>
    /// 현재 게임 월드에 있는 원격 플레이어 수를 반환한다.
    /// </summary>
    public int GetPlayerCount()
    {
        return _remotePlayers.Count;
    }

    /// <summary>
    /// 특정 ID의 플레이어를 반환한다.
    /// 플레이어가 없으면 null을 반환한다.
    /// </summary>
    public RemotePlayer GetPlayer(int playerId)
    {
        _remotePlayers.TryGetValue(playerId, out var player);
        return player;
    }
}
```

이 코드의 중요한 부분들을 설명한다:

**Dictionary 사용**: 플레이어 ID를 키로 사용하여 빠르게 특정 플레이어를 찾을 수 있다. List를 사용하면 매번 순회해야 하므로 느리다.

**타임아웃 처리**: 네트워크 연결이 끊기거나 클라이언트가 게임을 나가도 서버가 즉시 알지 못할 수 있다. 따라서 일정 시간 동안 업데이트가 없는 플레이어를 제거하여 메모리 누수를 방지한다.

**리스트 순회 중 제거**: `_remotePlayers` 딕셔너리를 순회 중에 직접 제거하면 오류가 발생한다. 따라서 제거할 플레이어 ID를 먼저 수집한 후 순회가 끝난 후에 제거한다.

### GameManager에 원격 플레이어 통합

```csharp
/// <summary>
/// 게임의 전체 로직을 관리하는 클래스다.
/// 로컬 플레이어, 원격 플레이어, 네트워크 등을 조율한다.
/// </summary>
public class GameManager
{
    private Player _player;
    private InputManager _inputManager;
    private RemotePlayerManager _remotePlayerManager;
    
    private NetworkClient _networkClient;
    private MessageProcessor _messageProcessor;
    private PositionSynchronizer _positionSynchronizer;
    
    private bool _isNetworkInitialized;
    private int _localPlayerId;
    
    private int _screenWidth;
    private int _screenHeight;

    public GameManager(int screenWidth, int screenHeight)
    {
        _screenWidth = screenWidth;
        _screenHeight = screenHeight;
        _inputManager = new InputManager();
    }

    /// <summary>
    /// 플레이어를 초기화한다.
    /// </summary>
    public void InitializePlayer(Texture2D playerTexture)
    {
        Vector2 initialPosition = new Vector2(
            _screenWidth / 2 - playerTexture.Width / 2,
            _screenHeight / 2 - playerTexture.Height / 2
        );

        _player = new Player(
            texture: playerTexture,
            initialPosition: initialPosition,
            screenWidth: _screenWidth,
            screenHeight: _screenHeight,
            moveSpeed: 200f
        );

        // 원격 플레이어 관리자 생성
        _remotePlayerManager = new RemotePlayerManager(playerTexture);
    }

    /// <summary>
    /// 네트워크를 비동기로 초기화한다.
    /// </summary>
    public async void InitializeNetworkAsync()
    {
        _networkClient = new NetworkClient();
        
        bool connected = await _networkClient.ConnectAsync("127.0.0.1", 5000);
        
        if (connected)
        {
            _messageProcessor = new MessageProcessor(_networkClient);
            _positionSynchronizer = new PositionSynchronizer(_networkClient);
            
            // 메시지 핸들러 등록
            _messageProcessor.RegisterHandler(
                typeof(PositionMessage),
                msg => HandlePositionMessage((PositionMessage)msg)
            );

            _messageProcessor.RegisterHandler(
                typeof(PlayerJoinMessage),
                msg => HandlePlayerJoinMessage((PlayerJoinMessage)msg)
            );

            _messageProcessor.RegisterHandler(
                typeof(PlayerLeaveMessage),
                msg => HandlePlayerLeaveMessage((PlayerLeaveMessage)msg)
            );

            _ = _networkClient.StartReceivingAsync();
            _isNetworkInitialized = true;
            
            Console.WriteLine("[정보] 네트워크 초기화 완료");
        }
    }

    /// <summary>
    /// 로컬 플레이어의 ID를 설정한다.
    /// 서버의 로그인 응답에서 할당받는다.
    /// </summary>
    public void SetLocalPlayerId(int playerId)
    {
        _localPlayerId = playerId;
        _positionSynchronizer?.SetLocalPlayerId(playerId);
    }

    /// <summary>
    /// 게임 로직을 매 프레임마다 업데이트한다.
    /// </summary>
    public void Update(GameTime gameTime)
    {
        _inputManager.Update();

        // 로컬 플레이어 업데이트
        if (_player != null)
        {
            _player.Update(gameTime, _inputManager);
        }

        // 원격 플레이어 업데이트
        if (_remotePlayerManager != null)
        {
            _remotePlayerManager.Update(gameTime);
        }

        // 위치 동기화
        if (_isNetworkInitialized && _positionSynchronizer != null && _player != null)
        {
            _positionSynchronizer.Update(gameTime, _player);
        }

        // 네트워크 메시지 처리
        if (_isNetworkInitialized && _messageProcessor != null)
        {
            _messageProcessor.ProcessMessages();
        }
    }

    /// <summary>
    /// 다른 플레이어로부터 받은 위치 메시지를 처리한다.
    /// </summary>
    private void HandlePositionMessage(PositionMessage msg)
    {
        // 로컬 플레이어의 위치 업데이트는 무시 (자신의 입력으로 이미 업데이트됨)
        if (msg.PlayerId == _localPlayerId)
        {
            return;
        }

        // 원격 플레이어의 위치 업데이트
        Vector2 newPosition = new Vector2(msg.X, msg.Y);
        _remotePlayerManager?.UpdatePlayerPosition(msg.PlayerId, newPosition);
    }

    /// <summary>
    /// 새로운 플레이어가 게임에 입장했을 때 처리한다.
    /// </summary>
    private void HandlePlayerJoinMessage(PlayerJoinMessage msg)
    {
        // 로컬 플레이어는 무시
        if (msg.PlayerId == _localPlayerId)
        {
            return;
        }

        // 새로운 원격 플레이어 추가
        _remotePlayerManager?.AddPlayer(
            msg.PlayerId,
            Vector2.Zero,  // 서버에서 초기 위치를 보낼 때까지는 (0, 0)
            msg.PlayerName
        );
    }

    /// <summary>
    /// 플레이어가 게임을 나갔을 때 처리한다.
    /// </summary>
    private void HandlePlayerLeaveMessage(PlayerLeaveMessage msg)
    {
        _remotePlayerManager?.RemovePlayer(msg.PlayerId);
    }

    /// <summary>
    /// 화면에 게임을 렌더링한다.
    /// 로컬 플레이어와 원격 플레이어를 모두 그린다.
    /// </summary>
    public void Draw(SpriteBatch spriteBatch, SpriteFont font)
    {
        // 원격 플레이어 먼저 그리기 (배경처럼)
        if (_remotePlayerManager != null)
        {
            _remotePlayerManager.Draw(spriteBatch);
        }

        // 로컬 플레이어 그리기 (위에)
        if (_player != null)
        {
            _player.Draw(spriteBatch);
        }

        // 모든 플레이어의 이름 표시
        if (_remotePlayerManager != null && font != null)
        {
            _remotePlayerManager.DrawNames(spriteBatch, font);
        }

        // 디버그 정보
        DrawDebugInfo(spriteBatch, font);
    }

    /// <summary>
    /// 디버그 정보를 화면에 표시한다.
    /// </summary>
    private void DrawDebugInfo(SpriteBatch spriteBatch, SpriteFont font)
    {
        if (_player == null || font == null)
            return;

        string positionText = $"로컬 위치: ({_player.Position.X:F0}, {_player.Position.Y:F0})";
        string playerCountText = $"게임 내 플레이어: {(_remotePlayerManager?.GetPlayerCount() ?? 0) + 1}";
        string networkText = $"네트워크: {(_isNetworkInitialized ? "연결됨" : "연결 안됨")}";

        spriteBatch.DrawString(font, positionText, new Vector2(10, 10), Color.White);
        spriteBatch.DrawString(font, playerCountText, new Vector2(10, 35), Color.White);
        spriteBatch.DrawString(font, networkText, new Vector2(10, 60), Color.White);
    }

    /// <summary>
    /// 게임을 종료한다.
    /// </summary>
    public void Shutdown()
    {
        if (_networkClient != null)
        {
            _networkClient.Disconnect();
        }
    }
}
```

## 실제 동작 확인
이제 실제 서버를 실행하고 여러 클라이언트를 연결하여 위치 동기화가 작동하는지 확인해야 한다. 이 장의 코드는 서버의 다음 기능을 가정한다:

1. 클라이언트가 연결되면 플레이어 ID를 할당
2. 클라이언트로부터 위치 패킷을 받으면 모든 클라이언트에게 브로드캐스트
3. 새 플레이어가 입장하면 알림을 전송
4. 플레이어가 나가면 알림을 전송

서버 구현은 이 책의 범위를 벗어나지만, 테스트를 위해 간단한 에코 서버를 만들 수 있다.

</br>
  

## 종합 정리
7장에서 배운 내용을 정리한다. 온라인 게임에서 플레이어 위치 동기화는 가장 중요한 기능이다. 먼저 패킷 프로토콜을 설계하여 위치 정보를 어떻게 표현할지 정의한다. 그 다음 일정한 시간 간격으로 위치 정보를 서버로 전송하며, 서버로부터 받은 다른 플레이어의 위치 정보를 게임 월드에 표시한다.

`PositionSynchronizer` 클래스는 로컬 플레이어의 위치 전송을 담당하고, `RemotePlayerManager` 클래스는 다른 플레이어들의 위치를 관리한다. 이러한 구조를 사용하면 나중에 게임이 복잡해져도 코드를 체계적으로 관리할 수 있다.

### 객관식 문제

**문제 1**: 위치 패킷의 기본 크기는?
- ① 9 바이트
- ② 13 바이트
- ③ 15 바이트
- ④ 17 바이트

**정답**: ③ 15 바이트

**해설**: 패킷 타입(1) + 플레이어 ID(4) + X 좌표(4) + Y 좌표(4) + 방향(2) = 15 바이트다.

---

**문제 2**: `POSITION_UPDATE_INTERVAL`을 0.05초로 설정하면 초당 몇 번 위치를 전송하는가?
- ① 5회
- ② 10회
- ③ 15회
- ④ 20회

**정답**: ④ 20회

**해설**: 1초 ÷ 0.05초 = 20회다. 간격이 작을수록 더 자주 전송된다.

---

**문제 3**: `RemotePlayer`가 `Player`와 다른 점은?
- ① RemotePlayer가 더 크다
- ② RemotePlayer는 입력 처리가 없다
- ③ RemotePlayer는 텍스처를 로드할 수 없다
- ④ RemotePlayer는 화면에 그릴 수 없다

**정답**: ②

**해설**: `RemotePlayer`는 네트워크로부터 받은 위치 정보만으로 이동하므로 입력 처리가 필요 없다.

---

**문제 4**: `RemotePlayerManager`에서 타임아웃된 플레이어를 순회 중에 바로 제거하지 않는 이유는?
- ① 성능 저하
- ② 메모리 누수
- ③ 컬렉션 수정으로 인한 오류
- ④ 네트워크 지연

**정답**: ③

**해설**: 컬렉션을 순회 중에 요소를 제거하면 `InvalidOperationException`이 발생한다. 따라서 제거할 항목을 먼저 수집한 후, 순회가 끝난 후에 제거해야 한다.

**문제 6**: 다음 코드는 로컬 플레이어의 위치 업데이트를 원격으로도 적용하려고 한다. 문제점을 찾고 수정하시오.

```csharp
private void HandlePositionMessage(PositionMessage msg)
{
    // 모든 플레이어 위치 업데이트
    Vector2 newPosition = new Vector2(msg.X, msg.Y);
    _remotePlayerManager?.UpdatePlayerPosition(msg.PlayerId, newPosition);
}
```

**예시 답안**: 문제는 로컬 플레이어의 위치까지 업데이트한다는 것이다. 로컬 플레이어의 경우 사용자 입력으로 이미 위치가 업데이트되어 있으므로, 서버로부터 받은 위치로 다시 업데이트하면 입력 지연이 발생할 수 있다.

**수정된 코드**:
```csharp
private void HandlePositionMessage(PositionMessage msg)
{
    // 로컬 플레이어의 메시지는 무시
    if (msg.PlayerId == _localPlayerId)
    {
        return;
    }
    
    // 원격 플레이어 위치만 업데이트
    Vector2 newPosition = new Vector2(msg.X, msg.Y);
    _remotePlayerManager?.UpdatePlayerPosition(msg.PlayerId, newPosition);
}
```

**예시 답안**: `TimeSinceLastUpdate`는 다음 두 가지 경우에 초기화된다:

**문제 10** (심화 과제): 현재 구현에서는 플레이어의 이동을 서버 지연 없이 즉시 표시한다. 하지만 실제 온라인 게임에서는 이동 예측(movement prediction)을 사용한다. 

다음을 구현하시오:

1. **속도 정보 추가**: 위치 패킷에 플레이어의 속도(벡터)를 추가한다.

2. **클라이언트 예측**: 위치 업데이트를 받은 후, 다음 업데이트까지 속도 정보를 사용하여 플레이어의 위치를 보간(interpolation)한다.

3. **부드러운 이동**: 이렇게 하면 네트워크 지연이 있어도 원격 플레이어의 이동이 더 부드러워 보인다.

**패킷 형식** (확장):
```
[타입: 1][ID: 4][X: 4][Y: 4][방향: 2][VX: 4][VY: 4] = 23 바이트
```

**예시 답안 개요**:
- `PositionMessage`에 `VelocityX`, `VelocityY` 필드 추가
- `RemotePlayer`의 `Update()` 메서드에서 속도를 사용하여 위치 보간
- 다음 패킷을 받을 때까지 계속해서 속도만큼 이동

이 기법을 사용하면 네트워크 지연이 있어도 원격 플레이어의 이동이 자연스러워 보인다.
