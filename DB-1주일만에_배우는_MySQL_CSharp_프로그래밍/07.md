## 7.1 트랜잭션과 동시성 제어
다중 유저가 동시에 접속하는 온라인 게임에서는 동시성 제어가 매우 중요하다. 여러 유저가 동시에 같은 데이터를 수정할 때 데이터 일관성을 보장하는 방법을 알아본다.

### 7.1.1 트랜잭션 격리 수준(Isolation Level)
트랜잭션 격리 수준은 동시에 실행되는 트랜잭션들이 서로 어느 정도 격리되어 실행될지를 결정한다. MySQL InnoDB는 4가지 격리 수준을 제공한다.

```
┌─────────────────────────────────────────────────────────────┐
│              Transaction Isolation Levels                    │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  READ UNCOMMITTED (가장 낮은 격리)                           │
│  ├─ Dirty Read 발생 가능                                    │
│  ├─ Non-Repeatable Read 발생 가능                           │
│  └─ Phantom Read 발생 가능                                  │
│                                                              │
│  READ COMMITTED                                              │
│  ├─ Dirty Read 방지                                         │
│  ├─ Non-Repeatable Read 발생 가능                           │
│  └─ Phantom Read 발생 가능                                  │
│                                                              │
│  REPEATABLE READ (InnoDB 기본값)                             │
│  ├─ Dirty Read 방지                                         │
│  ├─ Non-Repeatable Read 방지                                │
│  └─ Phantom Read 대부분 방지 (MVCC)                         │
│                                                              │
│  SERIALIZABLE (가장 높은 격리)                               │
│  ├─ 모든 이상 현상 방지                                     │
│  ├─ 가장 안전하지만 성능 저하                               │
│  └─ 게임에서는 거의 사용하지 않음                           │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

각 격리 수준에서 발생할 수 있는 이상 현상은 다음과 같다.

**Dirty Read**: 커밋되지 않은 데이터를 읽는 현상
```
Transaction A          Transaction B
─────────────          ─────────────
BEGIN;
UPDATE users 
SET gold = 1000 
WHERE user_id = 1;
                       BEGIN;
                       SELECT gold FROM users 
                       WHERE user_id = 1;
                       -- 1000을 읽음 (커밋 전)
ROLLBACK;
                       -- 실제로는 롤백되어 잘못된 값
```

**Non-Repeatable Read**: 같은 쿼리를 두 번 실행했을 때 결과가 다른 현상
```
Transaction A          Transaction B
─────────────          ─────────────
BEGIN;
SELECT gold FROM users 
WHERE user_id = 1;
-- 결과: 500
                       BEGIN;
                       UPDATE users 
                       SET gold = 1000 
                       WHERE user_id = 1;
                       COMMIT;
SELECT gold FROM users 
WHERE user_id = 1;
-- 결과: 1000 (다름!)
```

**Phantom Read**: 같은 조건의 쿼리를 두 번 실행했을 때 레코드 수가 달라지는 현상

MySQL에서 격리 수준을 설정하는 방법은 다음과 같다.

```sql
-- 현재 격리 수준 확인
SELECT @@transaction_isolation;

-- 세션 격리 수준 변경
SET SESSION TRANSACTION ISOLATION LEVEL READ COMMITTED;
SET SESSION TRANSACTION ISOLATION LEVEL REPEATABLE READ;

-- 글로벌 격리 수준 변경 (모든 새 연결에 적용)
SET GLOBAL TRANSACTION ISOLATION LEVEL REPEATABLE READ;
```

C#에서는 트랜잭션 시작 시 격리 수준을 지정할 수 있다.

```csharp
using MySqlConnector;
using System.Data;

public class TransactionIsolationExample
{
    private readonly MySqlConnection _connection;

    public TransactionIsolationExample(MySqlConnection connection)
    {
        _connection = connection;
    }

    // REPEATABLE READ로 트랜잭션 실행
    public async Task ExecuteWithRepeatableReadAsync()
    {
        using var transaction = await _connection.BeginTransactionAsync(
            IsolationLevel.RepeatableRead);
        
        try
        {
            // 트랜잭션 작업 수행
            await DoSomethingAsync(transaction);
            
            await transaction.CommitAsync();
        }
        catch
        {
            await transaction.RollbackAsync();
            throw;
        }
    }

    // READ COMMITTED로 트랜잭션 실행
    public async Task ExecuteWithReadCommittedAsync()
    {
        using var transaction = await _connection.BeginTransactionAsync(
            IsolationLevel.ReadCommitted);
        
        try
        {
            await DoSomethingAsync(transaction);
            await transaction.CommitAsync();
        }
        catch
        {
            await transaction.RollbackAsync();
            throw;
        }
    }

    private async Task DoSomethingAsync(MySqlTransaction transaction)
    {
        using var cmd = _connection.CreateCommand();
        cmd.Transaction = transaction;
        cmd.CommandText = "SELECT * FROM users WHERE user_id = @userId";
        cmd.Parameters.AddWithValue("@userId", 1);
        
        await cmd.ExecuteNonQueryAsync();
    }
}
```

### 7.1.2 락(Lock)의 이해와 데드락 방지
락은 동시성 제어의 핵심 메커니즘이다. InnoDB는 행 수준 락(Row-level Lock)을 사용하여 높은 동시성을 제공한다.

**락의 종류**

```
┌──────────────────────────────────────────────────┐
│              InnoDB Lock Types                    │
├──────────────────────────────────────────────────┤
│                                                   │
│  Shared Lock (S Lock)                            │
│  ├─ 읽기 락                                      │
│  ├─ 여러 트랜잭션이 동시에 획득 가능            │
│  └─ SELECT ... FOR SHARE                         │
│                                                   │
│  Exclusive Lock (X Lock)                         │
│  ├─ 쓰기 락                                      │
│  ├─ 다른 락과 함께 획득 불가                    │
│  └─ SELECT ... FOR UPDATE, UPDATE, DELETE        │
│                                                   │
│  Record Lock                                     │
│  └─ 인덱스 레코드에 대한 락                     │
│                                                   │
│  Gap Lock                                        │
│  └─ 인덱스 레코드 사이의 간격에 대한 락        │
│                                                   │
│  Next-Key Lock (Record + Gap)                    │
│  └─ REPEATABLE READ의 Phantom Read 방지         │
│                                                   │
└──────────────────────────────────────────────────┘
```

락을 명시적으로 사용하는 방법은 다음과 같다.

```sql
-- 공유 락 (Shared Lock)
SELECT * FROM users WHERE user_id = 1 FOR SHARE;

-- 배타 락 (Exclusive Lock)
SELECT * FROM users WHERE user_id = 1 FOR UPDATE;

-- 특정 행만 락 걸기
SELECT * FROM inventory 
WHERE user_id = 1 AND item_id = 100 
FOR UPDATE;
```

**데드락(Deadlock)**

데드락은 두 개 이상의 트랜잭션이 서로가 보유한 락을 기다리며 무한 대기 상태에 빠지는 현상이다.

```
Transaction A              Transaction B
─────────────              ─────────────
BEGIN;                     BEGIN;
UPDATE users               UPDATE inventory
SET gold = gold - 100      SET quantity = quantity - 1
WHERE user_id = 1;         WHERE user_id = 1;
                          
-- 락 획득: users(1)      -- 락 획득: inventory(1)

UPDATE inventory           UPDATE users
SET quantity = quantity-1  SET gold = gold - 100
WHERE user_id = 1;         WHERE user_id = 1;

-- 대기: inventory(1)     -- 대기: users(1)
-- ⚠️ DEADLOCK!           -- ⚠️ DEADLOCK!
```

InnoDB는 데드락을 자동으로 감지하고 한 트랜잭션을 롤백시킨다. 하지만 데드락을 방지하는 것이 더 좋은 방법이다.

**데드락 방지 전략**

1. **항상 같은 순서로 테이블/행 접근하기**
2. **트랜잭션을 짧게 유지하기**
3. **적절한 인덱스 사용하기**
4. **락 타임아웃 설정하기**

```csharp
using SqlKata;
using SqlKata.Execution;
using MySqlConnector;
using System;
using System.Threading.Tasks;

public class DeadlockPreventionExample
{
    private readonly QueryFactory _db;

    public DeadlockPreventionExample(MySqlConnection connection)
    {
        var compiler = new SqlKata.Compilers.MySqlCompiler();
        _db = new QueryFactory(connection, compiler);
    }

    // ❌ 나쁜 예: 데드락 발생 가능
    public async Task BadTransferItemAsync(long fromUserId, long toUserId, int itemId)
    {
        using var transaction = await _db.Connection.BeginTransactionAsync();
        
        try
        {
            // 순서가 보장되지 않아 데드락 가능
            await _db.Query("inventory")
                .Where("user_id", fromUserId)
                .Where("item_id", itemId)
                .DecrementAsync("quantity", 1);

            await _db.Query("inventory")
                .Where("user_id", toUserId)
                .Where("item_id", itemId)
                .IncrementAsync("quantity", 1);

            await transaction.CommitAsync();
        }
        catch
        {
            await transaction.RollbackAsync();
            throw;
        }
    }

    // ✅ 좋은 예: 항상 같은 순서로 락 획득
    public async Task GoodTransferItemAsync(long fromUserId, long toUserId, int itemId)
    {
        // user_id가 작은 것부터 처리하여 항상 같은 순서 보장
        var firstUserId = Math.Min(fromUserId, toUserId);
        var secondUserId = Math.Max(fromUserId, toUserId);

        using var transaction = await _db.Connection.BeginTransactionAsync();
        
        try
        {
            // 항상 작은 user_id부터 락 획득
            var firstInv = await LockInventoryAsync(firstUserId, itemId);
            var secondInv = await LockInventoryAsync(secondUserId, itemId);

            // 실제 작업 수행
            if (firstUserId == fromUserId)
            {
                await UpdateInventoryAsync(fromUserId, itemId, -1);
                await UpdateInventoryAsync(toUserId, itemId, 1);
            }
            else
            {
                await UpdateInventoryAsync(toUserId, itemId, 1);
                await UpdateInventoryAsync(fromUserId, itemId, -1);
            }

            await transaction.CommitAsync();
        }
        catch
        {
            await transaction.RollbackAsync();
            throw;
        }
    }

    private async Task<dynamic> LockInventoryAsync(long userId, int itemId)
    {
        using var cmd = _db.Connection.CreateCommand();
        cmd.CommandText = @"
            SELECT * FROM inventory 
            WHERE user_id = @userId AND item_id = @itemId 
            FOR UPDATE";
        cmd.Parameters.AddWithValue("@userId", userId);
        cmd.Parameters.AddWithValue("@itemId", itemId);

        using var reader = await cmd.ExecuteReaderAsync();
        if (await reader.ReadAsync())
        {
            return reader;
        }
        return null;
    }

    private async Task UpdateInventoryAsync(long userId, int itemId, int delta)
    {
        await _db.Query("inventory")
            .Where("user_id", userId)
            .Where("item_id", itemId)
            .IncrementAsync("quantity", delta);
    }

    // 데드락 발생 시 재시도 로직
    public async Task<bool> ExecuteWithRetryAsync(
        Func<Task> action, 
        int maxRetries = 3)
    {
        for (int i = 0; i < maxRetries; i++)
        {
            try
            {
                await action();
                return true;
            }
            catch (MySqlException ex) when (ex.Number == 1213) // Deadlock
            {
                if (i == maxRetries - 1)
                {
                    throw;
                }
                
                // 짧은 대기 후 재시도
                await Task.Delay(100 * (i + 1));
            }
        }
        return false;
    }
}
```

### 7.1.3 낙관적 락 vs 비관적 락
게임에서는 상황에 따라 낙관적 락과 비관적 락을 적절히 사용해야 한다.

**비관적 락 (Pessimistic Lock)**

충돌이 자주 발생할 것으로 예상되는 경우 사용한다. `SELECT ... FOR UPDATE`로 미리 락을 획득한다.

```csharp
public class PessimisticLockExample
{
    private readonly QueryFactory _db;

    public PessimisticLockExample(MySqlConnection connection)
    {
        var compiler = new SqlKata.Compilers.MySqlCompiler();
        _db = new QueryFactory(connection, compiler);
    }

    // 아이템 구매 (비관적 락)
    public async Task<bool> PurchaseItemAsync(long userId, int itemId, int price)
    {
        using var transaction = await _db.Connection.BeginTransactionAsync();
        
        try
        {
            // 유저의 골드를 락으로 보호
            using var cmd = _db.Connection.CreateCommand();
            cmd.Transaction = transaction;
            cmd.CommandText = @"
                SELECT gold FROM users 
                WHERE user_id = @userId 
                FOR UPDATE";
            cmd.Parameters.AddWithValue("@userId", userId);

            var gold = (int)await cmd.ExecuteScalarAsync();

            if (gold < price)
            {
                throw new InvalidOperationException("골드가 부족하다.");
            }

            // 골드 차감
            await _db.Query("users")
                .Where("user_id", userId)
                .DecrementAsync("gold", price);

            // 아이템 추가
            await _db.Query("inventory").InsertAsync(new
            {
                user_id = userId,
                item_id = itemId,
                quantity = 1
            });

            await transaction.CommitAsync();
            return true;
        }
        catch
        {
            await transaction.RollbackAsync();
            throw;
        }
    }
}
```

**낙관적 락 (Optimistic Lock)**

충돌이 드물게 발생할 것으로 예상되는 경우 사용한다. 버전 컬럼을 사용하여 충돌을 감지한다.

```sql
-- 버전 컬럼 추가
ALTER TABLE users ADD COLUMN version INT NOT NULL DEFAULT 0;
```

```csharp
public class OptimisticLockExample
{
    private readonly QueryFactory _db;

    public OptimisticLockExample(MySqlConnection connection)
    {
        var compiler = new SqlKata.Compilers.MySqlCompiler();
        _db = new QueryFactory(connection, compiler);
    }

    // 프로필 업데이트 (낙관적 락)
    public async Task<bool> UpdateProfileAsync(
        long userId, 
        string nickname, 
        int currentVersion)
    {
        // 버전을 확인하며 업데이트
        var affected = await _db.Query("users")
            .Where("user_id", userId)
            .Where("version", currentVersion)
            .UpdateAsync(new
            {
                nickname = nickname,
                version = currentVersion + 1,
                updated_at = DateTime.Now
            });

        if (affected == 0)
        {
            // 다른 트랜잭션이 먼저 수정함
            throw new InvalidOperationException(
                "데이터가 다른 사용자에 의해 수정되었다. 다시 시도해야 한다.");
        }

        return true;
    }

    // 낙관적 락 재시도 로직
    public async Task<bool> UpdateProfileWithRetryAsync(
        long userId, 
        string nickname, 
        int maxRetries = 3)
    {
        for (int i = 0; i < maxRetries; i++)
        {
            try
            {
                // 현재 버전 조회
                var user = await _db.Query("users")
                    .Where("user_id", userId)
                    .Select("version")
                    .FirstOrDefaultAsync<dynamic>();

                if (user == null)
                {
                    return false;
                }

                // 업데이트 시도
                await UpdateProfileAsync(userId, nickname, (int)user.version);
                return true;
            }
            catch (InvalidOperationException)
            {
                if (i == maxRetries - 1)
                {
                    throw;
                }
                // 짧은 대기 후 재시도
                await Task.Delay(50);
            }
        }
        return false;
    }
}
```

## 7.2 성능 최적화 전략
실제 게임 서비스에서는 수만 명의 동시 접속자를 처리해야 한다. 효율적인 성능 최적화 전략을 알아본다.

### 7.2.1 쿼리 최적화 체크리스트
쿼리 성능을 최적화하기 위한 핵심 체크리스트다.

```
┌─────────────────────────────────────────────────────┐
│         Query Optimization Checklist                 │
├─────────────────────────────────────────────────────┤
│                                                      │
│  ✓ SELECT 시 필요한 컬럼만 조회                     │
│    ❌ SELECT *                                       │
│    ✅ SELECT user_id, username, level                │
│                                                      │
│  ✓ WHERE 절에 인덱스 컬럼 사용                      │
│    ❌ WHERE YEAR(created_at) = 2025                  │
│    ✅ WHERE created_at >= '2025-01-01'               │
│                                                      │
│  ✓ 함수 사용 최소화                                 │
│    ❌ WHERE LOWER(username) = 'test'                 │
│    ✅ WHERE username = 'test'                        │
│                                                      │
│  ✓ 적절한 인덱스 사용                               │
│    - 카디널리티가 높은 컬럼                         │
│    - WHERE, JOIN, ORDER BY에 자주 사용되는 컬럼     │
│                                                      │
│  ✓ LIMIT 사용하여 결과 제한                         │
│    ✅ SELECT * FROM users LIMIT 100                  │
│                                                      │
│  ✓ EXISTS vs IN 적절히 선택                         │
│    - 서브쿼리 결과가 작으면 IN                      │
│    - 서브쿼리 결과가 크면 EXISTS                    │
│                                                      │
│  ✓ JOIN 순서 최적화                                 │
│    - 작은 테이블을 먼저                             │
│    - 결과가 적어지는 조건을 먼저                    │
│                                                      │
└─────────────────────────────────────────────────────┘
```

```csharp
public class QueryOptimizationExamples
{
    private readonly QueryFactory _db;

    public QueryOptimizationExamples(MySqlConnection connection)
    {
        var compiler = new SqlKata.Compilers.MySqlCompiler();
        _db = new QueryFactory(connection, compiler);
    }

    // ❌ 나쁜 예: SELECT * 사용
    public async Task<List<dynamic>> BadGetUsersAsync()
    {
        var users = await _db.Query("users")
            .GetAsync();  // 모든 컬럼 조회
        
        return users.ToList();
    }

    // ✅ 좋은 예: 필요한 컬럼만 조회
    public async Task<List<UserBasicInfo>> GoodGetUsersAsync()
    {
        var users = await _db.Query("users")
            .Select("user_id", "username", "level")
            .GetAsync<UserBasicInfo>();
        
        return users.ToList();
    }

    // ❌ 나쁜 예: 함수 사용으로 인덱스 미사용
    public async Task<List<dynamic>> BadGetUsersByDateAsync(int year)
    {
        using var cmd = _db.Connection.CreateCommand();
        cmd.CommandText = @"
            SELECT * FROM users 
            WHERE YEAR(created_at) = @year";
        cmd.Parameters.AddWithValue("@year", year);

        var users = new List<dynamic>();
        using var reader = await cmd.ExecuteReaderAsync();
        while (await reader.ReadAsync())
        {
            users.Add(reader);
        }
        return users;
    }

    // ✅ 좋은 예: 인덱스를 활용할 수 있는 조건
    public async Task<List<UserBasicInfo>> GoodGetUsersByDateAsync(int year)
    {
        var startDate = new DateTime(year, 1, 1);
        var endDate = new DateTime(year, 12, 31, 23, 59, 59);

        var users = await _db.Query("users")
            .WhereBetween("created_at", startDate, endDate)
            .Select("user_id", "username", "level")
            .GetAsync<UserBasicInfo>();
        
        return users.ToList();
    }

    // ✅ 페이징 처리
    public async Task<List<UserBasicInfo>> GetUsersWithPagingAsync(
        int page, 
        int pageSize = 20)
    {
        var offset = (page - 1) * pageSize;

        var users = await _db.Query("users")
            .Select("user_id", "username", "level")
            .OrderByDesc("user_id")
            .Limit(pageSize)
            .Offset(offset)
            .GetAsync<UserBasicInfo>();
        
        return users.ToList();
    }

    // ✅ EXISTS 사용
    public async Task<List<UserBasicInfo>> GetUsersWithInventoryAsync()
    {
        using var cmd = _db.Connection.CreateCommand();
        cmd.CommandText = @"
            SELECT user_id, username, level
            FROM users u
            WHERE EXISTS (
                SELECT 1 FROM inventory i 
                WHERE i.user_id = u.user_id
            )";

        var users = new List<UserBasicInfo>();
        using var reader = await cmd.ExecuteReaderAsync();
        while (await reader.ReadAsync())
        {
            users.Add(new UserBasicInfo
            {
                UserId = reader.GetInt64(0),
                Username = reader.GetString(1),
                Level = reader.GetInt32(2)
            });
        }
        return users;
    }
```

### 7.2.4 캐싱 전략
자주 조회되지만 변경이 적은 데이터는 캐싱하여 데이터베이스 부하를 줄인다.

```csharp
using System;
using System.Collections.Concurrent;
using System.Threading.Tasks;
using Microsoft.Extensions.Caching.Memory;

public class CachingStrategyExample
{
    private readonly QueryFactory _db;
    private readonly IMemoryCache _cache;

    public CachingStrategyExample(
        MySqlConnection connection, 
        IMemoryCache cache)
    {
        var compiler = new SqlKata.Compilers.MySqlCompiler();
        _db = new QueryFactory(connection, compiler);
        _cache = cache;
    }

    // 아이템 마스터 데이터 캐싱 (변경이 거의 없음)
    public async Task<ItemMaster> GetItemMasterAsync(int itemId)
    {
        string cacheKey = $"item_master_{itemId}";

        // 캐시 확인
        if (_cache.TryGetValue(cacheKey, out ItemMaster cachedItem))
        {
            return cachedItem;
        }

        // DB 조회
        var item = await _db.Query("item_master")
            .Where("item_id", itemId)
            .FirstOrDefaultAsync<ItemMaster>();

        if (item != null)
        {
            // 캐시에 저장 (1시간 유지)
            var cacheOptions = new MemoryCacheEntryOptions
            {
                AbsoluteExpirationRelativeToNow = TimeSpan.FromHours(1),
                SlidingExpiration = TimeSpan.FromMinutes(30)
            };

            _cache.Set(cacheKey, item, cacheOptions);
        }

        return item;
    }

    // 유저 정보 캐싱 (변경이 있을 때 갱신)
    public async Task<UserInfo> GetUserInfoAsync(long userId)
    {
        string cacheKey = $"user_info_{userId}";

        if (_cache.TryGetValue(cacheKey, out UserInfo cachedUser))
        {
            return cachedUser;
        }

        var user = await _db.Query("users")
            .Where("user_id", userId)
            .FirstOrDefaultAsync<UserInfo>();

        if (user != null)
        {
            var cacheOptions = new MemoryCacheEntryOptions
            {
                AbsoluteExpirationRelativeToNow = TimeSpan.FromMinutes(10)
            };

            _cache.Set(cacheKey, user, cacheOptions);
        }

        return user;
    }

    // 유저 정보 업데이트 시 캐시 무효화
    public async Task UpdateUserInfoAsync(long userId, UserInfo updatedInfo)
    {
        await _db.Query("users")
            .Where("user_id", userId)
            .UpdateAsync(new
            {
                username = updatedInfo.Username,
                level = updatedInfo.Level,
                updated_at = DateTime.Now
            });

        // 캐시 무효화
        string cacheKey = $"user_info_{userId}";
        _cache.Remove(cacheKey);
    }

    // 랭킹 데이터 캐싱 (주기적 갱신)
    public async Task<List<RankingInfo>> GetRankingAsync(int limit = 100)
    {
        string cacheKey = "ranking_top_100";

        if (_cache.TryGetValue(cacheKey, out List<RankingInfo> cachedRanking))
        {
            return cachedRanking;
        }

        var ranking = await _db.Query("users")
            .Select("user_id", "username", "level", "exp")
            .OrderByDesc("level")
            .ThenByDesc("exp")
            .Limit(limit)
            .GetAsync<RankingInfo>();

        var rankingList = ranking.ToList();

        // 5분간 캐시 유지
        var cacheOptions = new MemoryCacheEntryOptions
        {
            AbsoluteExpirationRelativeToNow = TimeSpan.FromMinutes(5)
        };

        _cache.Set(cacheKey, rankingList, cacheOptions);

        return rankingList;
    }
}
```

### 7.3.2 연속 출석 계산 로직

```csharp
using SqlKata;
using SqlKata.Execution;
using MySqlConnector;
using System;
using System.Collections.Generic;
using System.Linq;
using System.Threading.Tasks;

public class AttendanceService
{
    private readonly QueryFactory _db;

    public AttendanceService(MySqlConnection connection)
    {
        var compiler = new SqlKata.Compilers.MySqlCompiler();
        _db = new QueryFactory(connection, compiler);
    }

    // 출석 체크
    public async Task<AttendanceResult> CheckAttendanceAsync(long userId)
    {
        var today = DateTime.Today;

        using var transaction = await _db.Connection.BeginTransactionAsync();
        
        try
        {
            // 오늘 이미 출석했는지 확인
            var todayAttendance = await _db.Query("attendance")
                .Where("user_id", userId)
                .Where("attendance_date", today)
                .FirstOrDefaultAsync<Attendance>();

            if (todayAttendance != null)
            {
                throw new InvalidOperationException("오늘은 이미 출석했다.");
            }

            // 어제 출석 기록 조회
            var yesterday = today.AddDays(-1);
            var yesterdayAttendance = await _db.Query("attendance")
                .Where("user_id", userId)
                .Where("attendance_date", yesterday)
                .FirstOrDefaultAsync<Attendance>();

            // 연속 출석 일수 계산
            int consecutiveDays = 1;
            if (yesterdayAttendance != null)
            {
                // 어제 출석했으면 연속 일수 증가
                consecutiveDays = yesterdayAttendance.ConsecutiveDays + 1;
            }

            // 출석 기록 생성
            var attendanceId = await _db.Query("attendance")
                .InsertGetIdAsync<long>(new
                {
                    user_id = userId,
                    attendance_date = today,
                    consecutive_days = consecutiveDays,
                    reward_claimed = false
                });

            // 보상 조회
            var reward = await GetAttendanceRewardAsync(consecutiveDays);

            await transaction.CommitAsync();

            return new AttendanceResult
            {
                AttendanceId = attendanceId,
                ConsecutiveDays = consecutiveDays,
                Reward = reward,
                AttendanceDate = today
            };
        }
        catch
        {
            await transaction.RollbackAsync();
            throw;
        }
    }

    // 출석 보상 조회
    private async Task<AttendanceReward> GetAttendanceRewardAsync(int dayNumber)
    {
        // 정확한 일수의 보상이 있는지 확인
        var exactReward = await _db.Query("attendance_rewards")
            .Where("day_number", dayNumber)
            .FirstOrDefaultAsync<AttendanceReward>();

        if (exactReward != null)
        {
            return exactReward;
        }

        // 없으면 기본 보상 (예: 1일차 보상)
        var defaultReward = await _db.Query("attendance_rewards")
            .Where("day_number", 1)
            .FirstOrDefaultAsync<AttendanceReward>();

        return defaultReward;
    }

    // 보상 수령
    public async Task<bool> ClaimAttendanceRewardAsync(long userId, long attendanceId)
    {
        using var transaction = await _db.Connection.BeginTransactionAsync();
        
        try
        {
            // 출석 기록 확인
            var attendance = await _db.Query("attendance")
                .Where("attendance_id", attendanceId)
                .Where("user_id", userId)
                .Where("reward_claimed", false)
                .FirstOrDefaultAsync<Attendance>();

            if (attendance == null)
            {
                throw new InvalidOperationException("수령 가능한 보상이 없다.");
            }

            // 보상 조회
            var reward = await GetAttendanceRewardAsync(attendance.ConsecutiveDays);

            if (reward == null)
            {
                throw new InvalidOperationException("보상 정보를 찾을 수 없다.");
            }

            // 인벤토리에 보상 추가
            await AddRewardToInventoryAsync(userId, reward.ItemId, reward.Quantity);

            // 보상 수령 처리
            await _db.Query("attendance")
                .Where("attendance_id", attendanceId)
                .UpdateAsync(new { reward_claimed = true });

            await transaction.CommitAsync();
            return true;
        }
        catch
        {
            await transaction.RollbackAsync();
            throw;
        }
    }

    // 인벤토리에 보상 추가
    private async Task AddRewardToInventoryAsync(long userId, int itemId, int quantity)
    {
        var existing = await _db.Query("user_inventory")
            .Where("user_id", userId)
            .Where("item_id", itemId)
            .FirstOrDefaultAsync<dynamic>();

        if (existing != null)
        {
            await _db.Query("user_inventory")
                .Where("user_id", userId)
                .Where("item_id", itemId)
                .IncrementAsync("quantity", quantity);
        }
        else
        {
            await _db.Query("user_inventory").InsertAsync(new
            {
                user_id = userId,
                item_id = itemId,
                quantity = quantity
            });
        }
    }

    // 연속 출석 일수 조회
    public async Task<int> GetConsecutiveDaysAsync(long userId)
    {
        var latest = await _db.Query("attendance")
            .Where("user_id", userId)
            .OrderByDesc("attendance_date")
            .FirstOrDefaultAsync<Attendance>();

        if (latest == null)
        {
            return 0;
        }

        // 어제까지 연속 출석했는지 확인
        var yesterday = DateTime.Today.AddDays(-1);
        if (latest.AttendanceDate < yesterday)
        {
            // 연속 출석이 끊김
            return 0;
        }

        return latest.ConsecutiveDays;
    }

    // 오늘 출석 가능 여부 확인
    public async Task<bool> CanCheckAttendanceTodayAsync(long userId)
    {
        var today = DateTime.Today;

        var count = await _db.Query("attendance")
            .Where("user_id", userId)
            .Where("attendance_date", today)
            .CountAsync<int>();

        return count == 0;
    }
}

public class Attendance
{
    public long AttendanceId { get; set; }
    public long UserId { get; set; }
    public DateTime AttendanceDate { get; set; }
    public bool RewardClaimed { get; set; }
    public int ConsecutiveDays { get; set; }
    public DateTime CreatedAt { get; set; }
}

public class AttendanceReward
{

    public int DayNumber { get; set; }

    public int Quantity { get; set; }
}

public class AttendanceResult
{
    public long AttendanceId { get; set; }
    public int ConsecutiveDays { get; set; }
    public AttendanceReward Reward { get; set; }

}
```

```csharp
        var compiler = new SqlKata.Compilers.MySqlCompiler();
        _db = new QueryFactory(connection, compiler);
    }

    // 월별 출석 현황 조회
    public async Task<MonthlyAttendance> GetMonthlyAttendanceAsync(
        long userId, 
        int year, 
        int month)
    {
        var startDate = new DateTime(year, month, 1);
        var endDate = startDate.AddMonths(1).AddDays(-1);

        var attendances = await _db.Query("attendance")
            .Where("user_id", userId)
            .WhereBetween("attendance_date", startDate, endDate)
            .OrderBy("attendance_date")
            .GetAsync<Attendance>();

        var attendanceList = attendances.ToList();

        // 출석한 날짜 목록
        var attendedDates = attendanceList
            .Select(a => a.AttendanceDate.Day)
            .ToList();

        // 총 출석 일수
        var totalDays = attendanceList.Count;

        // 최대 연속 출석 일수
        var maxConsecutive = attendanceList.Any() 
            ? attendanceList.Max(a => a.ConsecutiveDays) 
            : 0;

        // 수령한 보상 개수
        var claimedRewards = attendanceList.Count(a => a.RewardClaimed);

        return new MonthlyAttendance
        {
            Year = year,
            Month = month,
            AttendedDates = attendedDates,
            TotalDays = totalDays,
            MaxConsecutiveDays = maxConsecutive,
            ClaimedRewards = claimedRewards,
            DaysInMonth = DateTime.DaysInMonth(year, month)
        };
    }

    // 전체 출석 통계
    public async Task<AttendanceStatistics> GetAttendanceStatisticsAsync(long userId)
    {
        var allAttendances = await _db.Query("attendance")
            .Where("user_id", userId)
            .OrderBy("attendance_date")
            .GetAsync<Attendance>();

        var attendanceList = allAttendances.ToList();

        if (!attendanceList.Any())
        {
            return new AttendanceStatistics
            {
                TotalDays = 0,
                CurrentStreak = 0,
                MaxStreak = 0,
                FirstAttendance = null,
                LastAttendance = null
            };
        }

        // 현재 연속 출석 일수
        var latest = attendanceList.Last();
        var currentStreak = latest.AttendanceDate >= DateTime.Today.AddDays(-1)
            ? latest.ConsecutiveDays
            : 0;

        return new AttendanceStatistics
        {
            TotalDays = attendanceList.Count,
            CurrentStreak = currentStreak,
            MaxStreak = attendanceList.Max(a => a.ConsecutiveDays),
            FirstAttendance = attendanceList.First().AttendanceDate,
            LastAttendance = latest.AttendanceDate
        };
    }

    // 출석률 순위 (이번 달 기준)
    public async Task<List<AttendanceRanking>> GetMonthlyAttendanceRankingAsync(
        int year, 
        int month, 
        int limit = 100)
    {
        var startDate = new DateTime(year, month, 1);
        var endDate = startDate.AddMonths(1);

        using var cmd = _db.Connection.CreateCommand();
        cmd.CommandText = @"
            SELECT 
                u.user_id,
                u.username,
                COUNT(a.attendance_id) as attendance_count,
                MAX(a.consecutive_days) as max_consecutive
            FROM users u
            LEFT JOIN attendance a ON u.user_id = a.user_id
                AND a.attendance_date >= @startDate
                AND a.attendance_date < @endDate
            GROUP BY u.user_id, u.username
            HAVING attendance_count > 0
            ORDER BY attendance_count DESC, max_consecutive DESC
            LIMIT @limit";
        
        cmd.Parameters.AddWithValue("@startDate", startDate);
        cmd.Parameters.AddWithValue("@endDate", endDate);
        cmd.Parameters.AddWithValue("@limit", limit);

        var rankings = new List<AttendanceRanking>();
        using var reader = await cmd.ExecuteReaderAsync();
        
        int rank = 1;
        while (await reader.ReadAsync())
        {
            rankings.Add(new AttendanceRanking
            {
                Rank = rank++,
                UserId = reader.GetInt64(0),
                Username = reader.GetString(1),
                AttendanceCount = reader.GetInt32(2),
                MaxConsecutive = reader.GetInt32(3)
            });
        }

        return rankings;
    }
}
```

### 7.4.1 백업과 복구 기본
데이터베이스 백업은 서비스 운영의 필수 요소다.

```bash
# 전체 데이터베이스 백업
mysqldump -u game_user -p game_db > game_db_backup_20251226.sql

# 특정 테이블만 백업
mysqldump -u game_user -p game_db users characters inventory > partial_backup.sql

# 압축하여 백업
mysqldump -u game_user -p game_db | gzip > game_db_backup_20251226.sql.gz

# 백업 복구
mysql -u game_user -p game_db < game_db_backup_20251226.sql

# 압축 파일 복구
gunzip < game_db_backup_20251226.sql.gz | mysql -u game_user -p game_db
```

C#에서 백업 스크립트를 실행하는 방법도 있다.

```csharp
using System;
using System.Diagnostics;
using System.IO;
using System.Threading.Tasks;

public class DatabaseBackupService
{
    private readonly string _mysqlDumpPath = "mysqldump";
    private readonly string _backupDirectory = @"C:\Backups\GameDB";

    public async Task<string> CreateBackupAsync(
        string host, 
        string database, 
        string user, 
        string password)
    {
        if (!Directory.Exists(_backupDirectory))
        {
            Directory.CreateDirectory(_backupDirectory);
        }

        var timestamp = DateTime.Now.ToString("yyyyMMdd_HHmmss");
        var backupFileName = $"{database}_backup_{timestamp}.sql";
        var backupFilePath = Path.Combine(_backupDirectory, backupFileName);

        var arguments = $"-h {host} -u {user} -p{password} {database}";

        var processInfo = new ProcessStartInfo
        {
            FileName = _mysqlDumpPath,
            Arguments = arguments,
            RedirectStandardOutput = true,
            UseShellExecute = false,
            CreateNoWindow = true
        };

        using var process = Process.Start(processInfo);
        using var output = File.CreateText(backupFilePath);
        
        await output.WriteAsync(await process.StandardOutput.ReadToEndAsync());
        await process.WaitForExitAsync();

        if (process.ExitCode == 0)
        {
            return backupFilePath;
        }
        else
        {
            throw new Exception("백업 실패");
        }
    }

    // 오래된 백업 파일 정리
    public void CleanOldBackups(int daysToKeep = 30)
    {
        var files = Directory.GetFiles(_backupDirectory, "*.sql");
        var cutoffDate = DateTime.Now.AddDays(-daysToKeep);

        foreach (var file in files)
        {
            var fileInfo = new FileInfo(file);
            if (fileInfo.CreationTime < cutoffDate)
            {
                File.Delete(file);
            }
        }
    }
}
```

```csharp
using System;
using System.Security.Cryptography;
using System.Text;

public class SecurityHelper
{
    // 비밀번호 해싱 (BCrypt 권장)
    public static string HashPassword(string password)
    {
        // BCrypt.Net-Next 라이브러리 사용
        return BCrypt.Net.BCrypt.HashPassword(password);
    }

    // 비밀번호 검증
    public static bool VerifyPassword(string password, string hashedPassword)
    {
        return BCrypt.Net.BCrypt.Verify(password, hashedPassword);
    }

    // 민감 정보 암호화 (AES)
    public static string EncryptData(string plainText, string key)
    {
        using var aes = Aes.Create();
        aes.Key = Encoding.UTF8.GetBytes(key.PadRight(32).Substring(0, 32));
        aes.IV = new byte[16]; // 실제로는 랜덤 IV 사용

        var encryptor = aes.CreateEncryptor(aes.Key, aes.IV);
        using var ms = new System.IO.MemoryStream();
        using var cs = new CryptoStream(ms, encryptor, CryptoStreamMode.Write);
        using var sw = new System.IO.StreamWriter(cs);
        
        sw.Write(plainText);
        sw.Flush();
        cs.FlushFinalBlock();

        return Convert.ToBase64String(ms.ToArray());
    }

    // SQL Injection 방지를 위한 입력 검증
    public static bool IsValidUsername(string username)
    {
        // 영문, 숫자, 언더스코어만 허용
        return System.Text.RegularExpressions.Regex.IsMatch(
            username, 
            @"^[a-zA-Z0-9_]{3,20}$");
    }
```

```csharp
using System;

        var sw = Stopwatch.StartNew();
        
        try
        {
            _logger.LogInformation($"[{queryName}] 시작");

            _logger.LogInformation(

            // 느린 쿼리 경고
            if (sw.ElapsedMilliseconds > 1000)
            {
                _logger.LogWarning(
                    $"[{queryName}] 느린 쿼리 감지 - {sw.ElapsedMilliseconds}ms");
            }
            
            return result;
        }
        catch (Exception ex)
        {
            sw.Stop();
            _logger.LogError(ex,

        }

    {

            if (_connection.State != System.Data.ConnectionState.Open)
            {
                await _connection.OpenAsync();

            return true;
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "데이터베이스 연결 실패");
            return false;
        }
    }

                State = reader.IsDBNull(6) ? null : reader.GetString(6),
```
