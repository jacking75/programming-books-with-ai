# Day 4: SQLKata를 이용한 쿼리 빌더
Day 3에서는 MySqlConnector를 사용하여 직접 SQL 문자열을 작성했다. 하지만 복잡한 쿼리를 문자열로 작성하면 오타가 발생하기 쉽고, 동적 조건을 처리할 때 코드가 복잡해진다. SQLKata는 이러한 문제를 해결해주는 강력한 쿼리 빌더 라이브러리다.

---

## 4.1 SQLKata 라이브러리 소개

### Query Builder 패턴의 장점
Query Builder는 SQL 쿼리를 메서드 체인 방식으로 작성하는 디자인 패턴이다. 다음과 같은 장점이 있다.

**1. 타입 안정성**

문자열로 쿼리를 작성하면 컴파일 시점에 오류를 찾을 수 없다. Query Builder는 메서드 호출 방식이므로 IDE의 자동완성과 컴파일 타임 체크를 활용할 수 있다.

```csharp
// ❌ 문자열 방식 - 오타를 찾기 어렵다
string sql = "SELCT * FROM usres WHERE levle > 10"; // SELCT, usres, levle 오타

// ✅ Query Builder - 컴파일 시점에 오류 발견
var query = new Query("users")
    .Select("*")
    .Where("level", ">", 10);
```

**2. 동적 쿼리 작성의 용이성**

검색 조건이 동적으로 변하는 경우, 문자열 결합 방식은 복잡하고 가독성이 떨어진다.

```csharp
// ❌ 문자열 결합 방식
string sql = "SELECT * FROM users WHERE 1=1";
if (!string.IsNullOrEmpty(username))
{
    sql += " AND username LIKE @username";
}
if (minLevel > 0)
{
    sql += " AND level >= @minLevel";
}
if (isOnline)
{
    sql += " AND is_online = 1";
}

// ✅ Query Builder 방식
var query = new Query("users");
if (!string.IsNullOrEmpty(username))
{
    query.WhereLike("username", username);
}
if (minLevel > 0)
{
    query.Where("level", ">=", minLevel);
}
if (isOnline)
{
    query.Where("is_online", true);
}
```

**3. 데이터베이스 독립성**

SQLKata는 MySQL, PostgreSQL, SQL Server 등 다양한 데이터베이스를 지원한다. 코드 변경 없이 데이터베이스를 교체할 수 있다.

**4. 가독성 향상**

복잡한 조인이나 서브쿼리도 메서드 체인으로 명확하게 표현할 수 있다.

```csharp
// 복잡한 조인 쿼리도 읽기 쉽다
var query = new Query("users as u")
    .Join("characters as c", "u.user_id", "c.user_id")
    .Join("inventories as i", "c.character_id", "i.character_id")
    .Where("u.level", ">", 50)
    .Where("i.item_type", "legendary")
    .Select("u.username", "c.character_name", "i.item_name");
```

### SQLKata 설치 및 초기 설정
NuGet 패키지를 설치한다:

```bash
dotnet add package SqlKata
dotnet add package SqlKata.Execution
```

`SqlKata.Execution` 패키지는 쿼리를 실제로 실행할 수 있는 확장 기능을 제공한다.

프로젝트 파일(`.csproj`)에 다음 참조가 추가된다:

```xml

</ItemGroup>
```

### MySqlConnector와 SQLKata 통합
SQLKata를 사용하려면 QueryFactory 객체를 생성해야 한다. QueryFactory는 데이터베이스 연결과 쿼리 컴파일러를 관리한다.

```csharp
using MySqlConnector;
using SqlKata;
using SqlKata.Compilers;
using SqlKata.Execution;

public class DatabaseContext
{
    private readonly string _connectionString;
    
    public DatabaseContext(string connectionString)
    {
        _connectionString = connectionString;
    }
    
    // QueryFactory 생성
    public QueryFactory CreateQueryFactory()
    {
        var connection = new MySqlConnection(_connectionString);
        var compiler = new MySqlCompiler();
        return new QueryFactory(connection, compiler);
    }
}
```

기본 사용 예제:

```csharp
class Program
{
    static void Main(string[] args)
    {
        string connectionString = "Server=localhost;Database=game_db;User=root;Password=1234;";
        var dbContext = new DatabaseContext(connectionString);
        
        using (var db = dbContext.CreateQueryFactory())
        {
            // 쿼리 작성 및 실행
            var users = db.Query("users")
                         .Where("level", ">", 10)
                         .Get();
            
            foreach (var user in users)
            {
                Console.WriteLine($"ID: {user.user_id}, Name: {user.username}");
            }
        }
    }
}
```

**SQLKata 실행 흐름:**

```
┌──────────────────┐
│   Query Builder  │
│  new Query()     │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  MySqlCompiler   │
│  (SQL 생성)      │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│     SQL 문        │
│ SELECT * FROM... │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ MySqlConnection  │
│  (실행)          │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│     결과          │
└──────────────────┘
```

## 4.2 SQLKata 기본 쿼리 작성

### Select, Where, Join 쿼리 빌딩

#### 기본 SELECT 쿼리

```csharp
using (var db = dbContext.CreateQueryFactory())
{
    // 전체 컬럼 조회
    var allUsers = db.Query("users").Get();
    
    // 특정 컬럼만 조회
    var userNames = db.Query("users")
                     .Select("user_id", "username", "level")
                     .Get();
    
    // 별칭 사용
    var usersWithAlias = db.Query("users")
                          .Select("user_id as id", "username as name")
                          .Get();
    
    // Distinct 사용
    var levels = db.Query("users")
                  .Distinct()
                  .Select("level")
                  .Get();
}
```

생성되는 SQL:

```sql
-- Get()
SELECT * FROM users

-- Select with columns
SELECT user_id, username, level FROM users

-- With alias
SELECT user_id as id, username as name FROM users

-- Distinct
SELECT DISTINCT level FROM users
```

#### WHERE 조건 쿼리

```csharp
// 단일 조건
var highLevelUsers = db.Query("users")
                      .Where("level", ">", 50)
                      .Get();

// 여러 조건 (AND)
var activeUsers = db.Query("users")
                   .Where("level", ">", 10)
                   .Where("is_banned", false)
                   .Get();

// OR 조건
var vipUsers = db.Query("users")
                .Where("level", ">", 100)
                .OrWhere("vip_level", ">", 0)
                .Get();

// IN 조건
var selectedUsers = db.Query("users")
                     .WhereIn("user_id", new[] { 1, 2, 3, 4, 5 })
                     .Get();

// BETWEEN 조건
var mediumLevelUsers = db.Query("users")
                        .WhereBetween("level", 10, 50)
                        .Get();

// LIKE 조건
var searchUsers = db.Query("users")
                   .WhereLike("username", "%hero%")
                   .Get();

// NULL 체크
var usersWithoutEmail = db.Query("users")
                         .WhereNull("email")
                         .Get();

var usersWithEmail = db.Query("users")
                      .WhereNotNull("email")
                      .Get();
```

생성되는 SQL:

```sql
-- Where
SELECT * FROM users WHERE level > 50

-- Multiple Where (AND)
SELECT * FROM users WHERE level > 10 AND is_banned = 0

-- OrWhere
SELECT * FROM users WHERE level > 100 OR vip_level > 0

-- WhereIn
SELECT * FROM users WHERE user_id IN (1, 2, 3, 4, 5)

-- WhereBetween
SELECT * FROM users WHERE level BETWEEN 10 AND 50

-- WhereLike
SELECT * FROM users WHERE username LIKE '%hero%'
```

#### JOIN 쿼리

```csharp
// INNER JOIN
var usersWithCharacters = db.Query("users as u")
                           .Join("characters as c", "u.user_id", "c.user_id")
                           .Select("u.username", "c.character_name", "c.level")

                              .LeftJoin("characters as c", "u.user_id", "c.user_id")

                              .Get();

// 다중 조인
var inventoryData = db.Query("users as u")
                     .Join("characters as c", "u.user_id", "c.user_id")
                     .Join("inventories as i", "c.character_id", "i.character_id")
                     .Join("items as it", "i.item_id", "it.item_id")
                     .Where("it.item_type", "weapon")
                     .Select("u.username", "c.character_name", "it.item_name")
                     .Get();

// 조인 조건이 복잡한 경우
var complexJoin = db.Query("users as u")
                   .Join("characters as c", j => j
                       .On("u.user_id", "c.user_id")
                       .Where("c.is_deleted", false)
                   )

```

생성되는 SQL:

```sql
-- INNER JOIN
SELECT u.username, c.character_name, c.level 
FROM users as u 
INNER JOIN characters as c ON u.user_id = c.user_id

-- LEFT JOIN
SELECT u.username, c.character_name 
FROM users as u 
LEFT JOIN characters as c ON u.user_id = c.user_id

-- Multiple joins
SELECT u.username, c.character_name, it.item_name 
FROM users as u 
INNER JOIN characters as c ON u.user_id = c.user_id
INNER JOIN inventories as i ON c.character_id = i.character_id
INNER JOIN items as it ON i.item_id = it.item_id
WHERE it.item_type = 'weapon'
```

생성되는 SQL:

```sql
-- WhereIn with subquery

WHERE user_id IN (SELECT user_id FROM rankings WHERE rank <= 100)

WHERE EXISTS (SELECT * FROM characters as c WHERE c.user_id = u.user_id)

-- Subquery in SELECT
SELECT u.user_id, u.username, 
       (SELECT COUNT(*) FROM characters WHERE user_id = u.user_id) as character_count
FROM users as u
```

## 4.3 게임 인벤토리 시스템 구현
실제 게임에서 사용하는 인벤토리 시스템을 SQLKata로 구현한다.

### 아이템 테이블 설계

먼저 필요한 테이블을 설계한다.

```sql
-- 아이템 마스터 테이블 (게임 기획 데이터)
CREATE TABLE items (
    item_id INT AUTO_INCREMENT PRIMARY KEY,
    item_name VARCHAR(100) NOT NULL,
    item_type ENUM('weapon', 'armor', 'potion', 'material', 'quest') NOT NULL,
    rarity ENUM('common', 'uncommon', 'rare', 'epic', 'legendary') NOT NULL,
    max_stack INT NOT NULL DEFAULT 1,
    description TEXT,
    icon_path VARCHAR(255),
    sell_price INT NOT NULL DEFAULT 0,
    INDEX idx_type (item_type),
    INDEX idx_rarity (rarity)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 유저 인벤토리 테이블
CREATE TABLE inventories (
    inventory_id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    item_id INT NOT NULL,
    quantity INT NOT NULL DEFAULT 1,
    slot_position INT NULL,
    acquired_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    
    FOREIGN KEY (user_id) REFERENCES users(user_id) ON DELETE CASCADE,
    FOREIGN KEY (item_id) REFERENCES items(item_id),
    INDEX idx_user_item (user_id, item_id),
    INDEX idx_user (user_id),
    
    -- 같은 유저가 같은 슬롯을 중복 사용하지 않도록
    UNIQUE KEY uk_user_slot (user_id, slot_position)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 거래 내역 테이블
CREATE TABLE trade_logs (
    trade_id BIGINT AUTO_INCREMENT PRIMARY KEY,
    sender_id BIGINT NOT NULL,
    receiver_id BIGINT NOT NULL,
    item_id INT NOT NULL,
    quantity INT NOT NULL,
    trade_price INT NOT NULL DEFAULT 0,
    trade_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    
    FOREIGN KEY (sender_id) REFERENCES users(user_id),
    FOREIGN KEY (receiver_id) REFERENCES users(user_id),
    FOREIGN KEY (item_id) REFERENCES items(item_id),
    INDEX idx_sender (sender_id, trade_time),
    INDEX idx_receiver (receiver_id, trade_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

샘플 아이템 데이터 삽입:

```sql
INSERT INTO items (item_name, item_type, rarity, max_stack, description, sell_price) VALUES
('철 검', 'weapon', 'common', 1, '평범한 철제 검', 100),
```

```csharp
using SqlKata.Execution;
using System;
using System.Collections.Generic;
using System.Linq;

public class InventoryService
{
    private readonly DatabaseContext _dbContext;
    
    public InventoryService(DatabaseContext dbContext)
    {
        _dbContext = dbContext;

    /// 유저의 전체 인벤토리 조회
    /// </summary>
    public List<InventoryItem> GetUserInventory(long userId)
    {
        using (var db = _dbContext.CreateQueryFactory())
        {

                         .Join("items as it", "inv.item_id", "it.item_id")

                             "it.rarity",
                             "it.max_stack",
                             "it.sell_price"
                         )
                         .OrderBy("inv.slot_position")
                         .Get<InventoryItem>();
            
            return items.ToList();
        }
    }
    
    /// <summary>
    /// 특정 타입의 아이템만 조회
    /// </summary>
    public List<InventoryItem> GetItemsByType(long userId, string itemType)
    {

        {

                         .Where("inv.user_id", userId)

                         .Get<InventoryItem>();

    /// <summary>
    /// 아이템 추가 (스택 가능 아이템은 기존에 합산)
    /// </summary>
    public bool AddItem(long userId, int itemId, int quantity)
    {
        using (var db = _dbContext.CreateQueryFactory())
        {
            // 트랜잭션 시작
            db.Connection.Open();
            using (var transaction = db.Connection.BeginTransaction())
            {
                try
                {
                    db.SetTransaction(transaction);
                    
                    // 아이템 정보 조회
                    var item = db.Query("items")
                                .Where("item_id", itemId)
                                .First<dynamic>();
                    
                    if (item == null)
                    {
                        Console.WriteLine("존재하지 않는 아이템이다.");
                        transaction.Rollback();
                        return false;
                    }
                    
                    int maxStack = item.max_stack;
                    
                    // 스택 가능한 아이템이면 기존 수량에 추가
                    if (maxStack > 1)
                    {
                        var existingItem = db.Query("inventories")
                                            .Where("user_id", userId)
                                            .Where("item_id", itemId)
                                            .First<dynamic>();
                        
                        if (existingItem != null)
                        {
                            int newQuantity = (int)existingItem.quantity + quantity;
                            
                            // 최대 스택 수 체크
                            if (newQuantity > maxStack)
                            {
                                Console.WriteLine($"최대 스택 수({maxStack})를 초과한다.");
                                transaction.Rollback();
                                return false;
                            }
                            
                            db.Query("inventories")
                              .Where("inventory_id", existingItem.inventory_id)
                              .Update(new { quantity = newQuantity });
                            
                            transaction.Commit();
                            Console.WriteLine($"아이템 수량 증가: {item.item_name} x{quantity}");
                            return true;
                        }
                    }
                    
                    // 새로운 아이템 추가
                    // 빈 슬롯 찾기
                    int? emptySlot = FindEmptySlot(db, userId);
                    
                    db.Query("inventories").Insert(new
                    {
                        user_id = userId,
                        item_id = itemId,
                        quantity = quantity,
                        slot_position = emptySlot,
                        acquired_at = DateTime.Now
                    });
                    
                    transaction.Commit();
                    Console.WriteLine($"아이템 획득: {item.item_name} x{quantity}");
                    return true;
                }
                catch (Exception ex)
                {
                    transaction.Rollback();
                    Console.WriteLine($"아이템 추가 실패: {ex.Message}");
                    return false;
                }
            }
        }
    }
    
    /// <summary>
    /// 빈 슬롯 번호 찾기
    /// </summary>
    private int? FindEmptySlot(QueryFactory db, long userId)
    {
        var usedSlots = db.Query("inventories")
                         .Where("user_id", userId)
                         .WhereNotNull("slot_position")
                         .Select("slot_position")
                         .Get<dynamic>()
                         .Select(x => (int)x.slot_position)
                         .OrderBy(x => x)
                         .ToList();
        
        // 0번부터 순차적으로 빈 슬롯 찾기
        for (int i = 0; i < 100; i++)
        {
            if (!usedSlots.Contains(i))
            {
                return i;
            }
        }
        
        return null; // 인벤토리 가득 찼음
    }
    
    /// <summary>
    /// 아이템 삭제 또는 수량 감소
    /// </summary>
    public bool RemoveItem(long userId, int itemId, int quantity)
    {
        using (var db = _dbContext.CreateQueryFactory())
        {
            db.Connection.Open();
            using (var transaction = db.Connection.BeginTransaction())
            {
                try
                {
                    db.SetTransaction(transaction);
                    
                    var inventoryItem = db.Query("inventories")
                                         .Where("user_id", userId)
                                         .Where("item_id", itemId)
                                         .First<dynamic>();
                    
                    if (inventoryItem == null)
                    {
                        Console.WriteLine("해당 아이템을 보유하고 있지 않다.");
                        transaction.Rollback();
                        return false;
                    }
                    
                    int currentQuantity = inventoryItem.quantity;
                    
                    if (currentQuantity < quantity)
                    {
                        Console.WriteLine($"수량이 부족하다. (보유: {currentQuantity}, 필요: {quantity})");
                        transaction.Rollback();
                        return false;
                    }
                    
                    if (currentQuantity == quantity)
                    {
                        // 전부 제거
                        db.Query("inventories")
                          .Where("inventory_id", inventoryItem.inventory_id)
                          .Delete();
                        
                        Console.WriteLine($"아이템 제거 완료");
                    }
                    else
                    {
                        // 수량만 감소
                        db.Query("inventories")
                          .Where("inventory_id", inventoryItem.inventory_id)
                          .Update(new { quantity = currentQuantity - quantity });
                        
                        Console.WriteLine($"아이템 수량 감소: {quantity}개");
                    }
                    
                    transaction.Commit();
                    return true;
                }
                catch (Exception ex)
                {
                    transaction.Rollback();
                    Console.WriteLine($"아이템 제거 실패: {ex.Message}");
                    return false;
                }
            }
        }
    }
    
    /// <summary>
    /// 인벤토리 정렬 (타입별, 레어도별)
    /// </summary>
    public List<InventoryItem> GetSortedInventory(long userId, string sortBy = "slot")
    {
        using (var db = _dbContext.CreateQueryFactory())
        {
            var query = db.Query("inventories as inv")
                         .Join("items as it", "inv.item_id", "it.item_id")
                         .Where("inv.user_id", userId)
                         .Select("inv.*", "it.item_name", "it.item_type", "it.rarity");
            
            switch (sortBy)
            {
                case "type":
                    query.OrderBy("it.item_type").ThenBy("it.rarity");
                    break;
                case "rarity":
                    query.OrderByDesc("it.rarity").ThenBy("it.item_type");
                    break;
                case "name":
                    query.OrderBy("it.item_name");
                    break;
                default:
                    query.OrderBy("inv.slot_position");
                    break;
            }
            
            return query.Get<InventoryItem>().ToList();
        }
    }
}
```

### 아이템 거래 시스템 구현

```csharp
public class TradeService
{
    private readonly DatabaseContext _dbContext;
    private readonly InventoryService _inventoryService;
    
    public TradeService(DatabaseContext dbContext, InventoryService inventoryService)
    {
        _dbContext = dbContext;
        _inventoryService = inventoryService;
    }
    
    /// <summary>
    /// 아이템 거래 처리
    /// </summary>
    public bool TradeItem(long senderId, long receiverId, int itemId, int quantity, int price)
    {
        using (var db = _dbContext.CreateQueryFactory())
        {
            db.Connection.Open();
            using (var transaction = db.Connection.BeginTransaction())
            {
                try
                {
                    db.SetTransaction(transaction);
                    
                    // 1. 보내는 사람이 아이템을 보유하고 있는지 확인
                    var senderItem = db.Query("inventories")
                                      .Where("user_id", senderId)
                                      .Where("item_id", itemId)
                                      .First<dynamic>();
                    
                    if (senderItem == null || senderItem.quantity < quantity)
                    {
                        Console.WriteLine("보내는 사람의 아이템이 부족하다.");
                        transaction.Rollback();
                        return false;
                    }
                    
                    // 2. 받는 사람의 골드 확인
                    var receiver = db.Query("users")
                                    .Where("user_id", receiverId)
                                    .First<dynamic>();
                    
                    if (receiver == null)
                    {
                        Console.WriteLine("받는 사람이 존재하지 않는다.");
                        transaction.Rollback();
                        return false;
                    }
                    
                    if (receiver.gold < price)
                    {
                        Console.WriteLine("받는 사람의 골드가 부족하다.");
                        transaction.Rollback();
                        return false;
                    }
                    
                    // 3. 보내는 사람의 아이템 제거
                    if (senderItem.quantity == quantity)
                    {
                        db.Query("inventories")
                          .Where("inventory_id", senderItem.inventory_id)
                          .Delete();
                    }
                    else
                    {
                        db.Query("inventories")
                          .Where("inventory_id", senderItem.inventory_id)
                          .Decrement("quantity", quantity);
                    }
                    
                    // 4. 받는 사람에게 아이템 추가
                    var item = db.Query("items")
                                .Where("item_id", itemId)
                                .First<dynamic>();
                    
                    var receiverItem = db.Query("inventories")
                                        .Where("user_id", receiverId)
                                        .Where("item_id", itemId)
                                        .First<dynamic>();
                    
                    if (receiverItem != null && item.max_stack > 1)
                    {
                        // 기존 아이템에 수량 추가
                        db.Query("inventories")
                          .Where("inventory_id", receiverItem.inventory_id)
                          .Increment("quantity", quantity);
                    }
                    else
                    {
                        // 새 아이템 추가
                        db.Query("inventories").Insert(new
                        {
                            user_id = receiverId,
                            item_id = itemId,
                            quantity = quantity,
                            acquired_at = DateTime.Now
                        });
                    }
                    
                    // 5. 골드 거래
                    db.Query("users")
                      .Where("user_id", senderId)
                      .Increment("gold", price);
                    
                    db.Query("users")
                      .Where("user_id", receiverId)
                      .Decrement("gold", price);
                    
                    // 6. 거래 로그 기록
                    db.Query("trade_logs").Insert(new
                    {
                        sender_id = senderId,
                        receiver_id = receiverId,
                        item_id = itemId,
                        quantity = quantity,
                        trade_price = price,
                        trade_time = DateTime.Now
                    });
                    
                    transaction.Commit();
                    Console.WriteLine($"거래 완료: {item.item_name} x{quantity}, 가격: {price} 골드");
                    return true;
                }
                catch (Exception ex)
                {
                    transaction.Rollback();
                    Console.WriteLine($"거래 실패: {ex.Message}");
                    return false;
                }
            }
        }
    }
    
    /// <summary>
    /// 유저의 거래 내역 조회
    /// </summary>
    public List<TradeHistory> GetTradeHistory(long userId, int limit = 20)
    {
        using (var db = _dbContext.CreateQueryFactory())
        {
            var trades = db.Query("trade_logs as tl")
                          .Join("items as it", "tl.item_id", "it.item_id")
                          .LeftJoin("users as sender", "tl.sender_id", "sender.user_id")
                          .LeftJoin("users as receiver", "tl.receiver_id", "receiver.user_id")
                          .Where(q => q
                              .Where("tl.sender_id", userId)
                              .OrWhere("tl.receiver_id", userId)
                          )
                          .Select(
                              "tl.trade_id",
                              "tl.sender_id",
                              "tl.receiver_id",
                              "sender.username as sender_name",
                              "receiver.username as receiver_name",
                              "it.item_name",
                              "tl.quantity",
                              "tl.trade_price",
                              "tl.trade_time"
                          )
                          .OrderByDesc("tl.trade_time")
                          .Limit(limit)
                          .Get<TradeHistory>();
            
            return trades.ToList();
        }
    }
}

public class TradeHistory
{
    public long trade_id { get; set; }
    public long sender_id { get; set; }
    public long receiver_id { get; set; }
    public string sender_name { get; set; }
    public string receiver_name { get; set; }
    public string item_name { get; set; }
    public int quantity { get; set; }
    public int trade_price { get; set; }
    public DateTime trade_time { get; set; }
}
```

## 정리

Day 4에서는 SQLKata를 사용하여 SQL 쿼리를 더욱 안전하고 직관적으로 작성하는 방법을 배웠다. Query Builder 패턴을 사용하면 문자열 기반 쿼리의 단점을 극복하고, 동적 조건 처리가 쉬워진다.

게임 인벤토리 시스템 구현을 통해 실전에서 SQLKata를 어떻게 활용하는지 익혔다. 아이템 추가, 삭제, 거래 등의 복잡한 로직도 메서드 체인으로 깔끔하게 표현할 수 있다.

다음 Day 5에서는 JOIN을 활용한 고급 쿼리와 인덱스를 통한 성능 최적화를 배운다. 게임 랭킹 시스템을 구현하며 대용량 데이터를 효율적으로 조회하는 방법을 익힌다.

**핵심 요약:**

- SQLKata는 타입 안정성과 가독성을 제공하는 Query Builder다
- Select, Where, Join 등 모든 SQL 구문을 메서드 체인으로 작성한다
- Insert, Update, Delete도 직관적인 메서드로 처리한다
- 복잡한 조건문, 서브쿼리, 집계 함수도 지원한다
- 트랜잭션과 결합하여 안전한 게임 시스템을 구현한다
