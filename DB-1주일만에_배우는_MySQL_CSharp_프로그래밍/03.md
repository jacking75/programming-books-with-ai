## 3.1 개발 환경 구성

### .NET 프로젝트 생성
먼저 새로운 콘솔 애플리케이션 프로젝트를 생성한다. 터미널이나 명령 프롬프트에서 다음 명령을 실행한다.

```bash
dotnet new console -n GameServer
cd GameServer
```

프로젝트가 생성되면 `.csproj` 파일이 생성된다. 이 파일은 프로젝트의 설정과 의존성을 관리한다.

### MySqlConnector 라이브러리 설치 및 설정
MySqlConnector는 MySQL과 통신하기 위한 오픈소스 ADO.NET 드라이버다. 공식 MySQL Connector보다 성능이 우수하고 비동기 처리를 완벽하게 지원한다.

NuGet 패키지를 설치한다:

```bash
dotnet add package MySqlConnector
```

설치가 완료되면 `.csproj` 파일에 다음과 같은 참조가 추가된다:

```xml
<ItemGroup>
  <PackageReference Include="MySqlConnector" Version="2.3.0" />
</ItemGroup>
```

### 연결 문자열(Connection String) 구성
MySQL 서버에 접속하기 위해서는 연결 문자열이 필요하다. 연결 문자열은 다음과 같은 정보를 포함한다:

```
서버 주소, 포트 번호, 데이터베이스 이름, 사용자 계정, 비밀번호
```

기본적인 연결 문자열 형식은 다음과 같다:

```csharp
string connectionString = "Server=localhost;Port=3306;Database=game_db;User=root;Password=yourpassword;";
```

보안을 위해 연결 문자열을 코드에 직접 작성하지 않고 설정 파일로 분리하는 것이 좋다. `appsettings.json` 파일을 생성하고 다음과 같이 작성한다:

```json
{
  "ConnectionStrings": {
    "GameDatabase": "Server=localhost;Port=3306;Database=game_db;User=game_user;Password=game1234;Pooling=true;MinimumPoolSize=5;MaximumPoolSize=20;"
  }
}
```

연결 문자열의 주요 옵션들:

- **Server**: MySQL 서버 주소 (localhost, IP 주소, 도메인)
- **Port**: MySQL 포트 번호 (기본값: 3306)
- **Database**: 접속할 데이터베이스 이름
- **User**: MySQL 사용자 계정
- **Password**: 계정 비밀번호
- **Pooling**: 연결 풀링 활성화 여부 (기본값: true)
- **MinimumPoolSize**: 최소 연결 풀 크기
- **MaximumPoolSize**: 최대 연결 풀 크기
- **ConnectionTimeout**: 연결 타임아웃 시간(초)

## 3.2 MySqlConnector 기본 사용법

### 데이터베이스 연결과 종료
MySqlConnection 객체를 사용하여 데이터베이스에 연결한다. using 문을 사용하면 연결이 자동으로 종료되므로 리소스 누수를 방지할 수 있다.

```csharp
using MySqlConnector;

string connectionString = "Server=localhost;Database=game_db;User=root;Password=1234;";

using (var connection = new MySqlConnection(connectionString))
{
    connection.Open();
    Console.WriteLine("데이터베이스 연결 성공!");
    // 여기서 쿼리 실행
} // using 블록을 벗어나면 자동으로 연결 종료
```

연결 상태를 확인하려면 `State` 속성을 사용한다:

```csharp
if (connection.State == System.Data.ConnectionState.Open)
{
    Console.WriteLine("연결됨");
}
```

**연결 흐름도:**

```
┌─────────────┐
│ Application │
└──────┬──────┘
       │ new MySqlConnection()
       ▼
┌─────────────────┐
│ MySqlConnection │
└──────┬──────────┘
       │ Open()
       ▼
┌─────────────────┐
│  MySQL Server   │
│   (Port 3306)   │
└──────┬──────────┘
       │ Query Execution
       ▼
┌─────────────────┐
│     Result      │
└──────┬──────────┘
       │ Close() / Dispose()
       ▼
    [종료]
```

### 쿼리 실행 메서드
MySqlConnector는 다양한 쿼리 실행 메서드를 제공한다.

#### ExecuteNonQuery - INSERT, UPDATE, DELETE용
결과 집합을 반환하지 않는 쿼리에 사용한다. 영향받은 행의 개수를 반환한다.

```csharp
using (var connection = new MySqlConnection(connectionString))
{
    connection.Open();
    
    string sql = "INSERT INTO users (username, email, created_at) VALUES ('player1', 'player1@game.com', NOW())";
    
    using (var command = new MySqlCommand(sql, connection))
    {
        int affectedRows = command.ExecuteNonQuery();
        Console.WriteLine($"{affectedRows}개 행이 삽입되었다.");
    }
}
```

#### ExecuteScalar - 단일 값 조회용
쿼리 결과의 첫 번째 행, 첫 번째 열 값을 반환한다. 카운트나 합계 같은 집계 값을 가져올 때 유용하다.

```csharp
using (var connection = new MySqlConnection(connectionString))
{
    connection.Open();
    
    string sql = "SELECT COUNT(*) FROM users";
    
    using (var command = new MySqlCommand(sql, connection))
    {
        long userCount = (long)command.ExecuteScalar();
        Console.WriteLine($"총 유저 수: {userCount}");
    }
}
```

#### ExecuteReader - 다중 행 조회용
여러 행의 결과를 순차적으로 읽을 때 사용한다.

```csharp
using (var connection = new MySqlConnection(connectionString))
{
    connection.Open();
    
    string sql = "SELECT user_id, username, level FROM users WHERE level >= 10";
    
    using (var command = new MySqlCommand(sql, connection))
    using (var reader = command.ExecuteReader())
    {
        while (reader.Read())
        {
            int userId = reader.GetInt32("user_id");
            string username = reader.GetString("username");
            int level = reader.GetInt32("level");
            
            Console.WriteLine($"ID: {userId}, 이름: {username}, 레벨: {level}");
        }
    }
}
```

**Reader 사용 시 타입별 메서드:**

- `GetInt32()`: int 타입
- `GetInt64()`: long 타입
- `GetString()`: string 타입
- `GetDateTime()`: DateTime 타입
- `GetDecimal()`: decimal 타입
- `GetBoolean()`: bool 타입
- `IsDBNull()`: NULL 체크

### 파라미터를 이용한 안전한 쿼리 실행
사용자 입력을 쿼리에 직접 연결하면 SQL 인젝션 공격에 취약하다. 반드시 파라미터를 사용해야 한다.

**❌ 잘못된 방법 (SQL 인젝션 위험):**

```csharp
string username = userInput; // 사용자 입력값
string sql = $"SELECT * FROM users WHERE username = '{username}'";
// username이 "admin' OR '1'='1" 같은 값이면 모든 데이터 조회 가능
```

**✅ 올바른 방법 (파라미터 사용):**

```csharp
using (var connection = new MySqlConnection(connectionString))
{
    connection.Open();
    
    string sql = "SELECT * FROM users WHERE username = @username AND level >= @minLevel";
    
    using (var command = new MySqlCommand(sql, connection))
    {
        // 파라미터 추가
        command.Parameters.AddWithValue("@username", "player1");
        command.Parameters.AddWithValue("@minLevel", 10);
        
        using (var reader = command.ExecuteReader())
        {
            while (reader.Read())
            {
                Console.WriteLine(reader.GetString("username"));
            }
        }
    }
}
```

### 트랜잭션 처리
트랜잭션은 여러 개의 쿼리를 하나의 작업 단위로 묶어서 모두 성공하거나 모두 실패하도록 보장한다. 게임에서 아이템 거래, 재화 이동 등의 작업에 필수적이다.

```csharp
using (var connection = new MySqlConnection(connectionString))
{
    connection.Open();
    
    using (var transaction = connection.BeginTransaction())
    {
        try
        {
            // 첫 번째 쿼리: 보내는 사람의 골드 차감
            string sql1 = "UPDATE users SET gold = gold - @amount WHERE user_id = @senderId";
            using (var cmd1 = new MySqlCommand(sql1, connection, transaction))
            {
                cmd1.Parameters.AddWithValue("@amount", 1000);
                cmd1.Parameters.AddWithValue("@senderId", 1);
                cmd1.ExecuteNonQuery();
            }
            
            // 두 번째 쿼리: 받는 사람의 골드 증가
            string sql2 = "UPDATE users SET gold = gold + @amount WHERE user_id = @receiverId";
            using (var cmd2 = new MySqlCommand(sql2, connection, transaction))
            {
                cmd2.Parameters.AddWithValue("@amount", 1000);
                cmd2.Parameters.AddWithValue("@receiverId", 2);
                cmd2.ExecuteNonQuery();
            }
            
            // 세 번째 쿼리: 거래 기록 저장
            string sql3 = "INSERT INTO trade_logs (sender_id, receiver_id, amount, trade_time) VALUES (@senderId, @receiverId, @amount, NOW())";
            using (var cmd3 = new MySqlCommand(sql3, connection, transaction))
            {
                cmd3.Parameters.AddWithValue("@senderId", 1);
                cmd3.Parameters.AddWithValue("@receiverId", 2);
                cmd3.Parameters.AddWithValue("@amount", 1000);
                cmd3.ExecuteNonQuery();
            }
            
            // 모든 쿼리 성공 시 커밋
            transaction.Commit();
            Console.WriteLine("거래가 성공적으로 완료되었다.");
        }
        catch (Exception ex)
        {
            // 하나라도 실패하면 롤백
            transaction.Rollback();
            Console.WriteLine($"거래 실패: {ex.Message}");
        }
    }
}
```

**트랜잭션 흐름:**

```
BEGIN TRANSACTION
    │
    ├─ UPDATE users (골드 차감)
    │   └─ 성공 ✓
    │
    ├─ UPDATE users (골드 증가)
    │   └─ 성공 ✓
    │
    ├─ INSERT trade_logs
    │   └─ 성공 ✓
    │
    └─ COMMIT (모든 변경사항 확정)

만약 중간에 실패하면:
    └─ ROLLBACK (모든 변경사항 취소)
```

## 3.3 게임 회원가입/로그인 구현
실제 게임 서버에서 사용하는 회원가입과 로그인 기능을 구현한다.

### 테이블 설계
먼저 유저 정보를 저장할 테이블을 생성한다:

```sql
CREATE TABLE users (
    user_id BIGINT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    salt VARCHAR(255) NOT NULL,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    last_login DATETIME NULL,
    is_banned BOOLEAN NOT NULL DEFAULT FALSE,
    INDEX idx_username (username),
    INDEX idx_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 회원가입 기능 개발
회원가입 시 비밀번호를 평문으로 저장하면 안 된다. 반드시 해싱해서 저장해야 한다.

```csharp
using System;
using System.Security.Cryptography;
using System.Text;
using MySqlConnector;

public class UserService
{
    private readonly string _connectionString;
    
    public UserService(string connectionString)
    {
        _connectionString = connectionString;
    }
    
    // 회원가입 메서드
    public bool Register(string username, string email, string password)
    {
        // 입력값 검증
        if (string.IsNullOrWhiteSpace(username) || username.Length < 3 || username.Length > 50)
        {
            Console.WriteLine("유저명은 3~50자 사이여야 한다.");
            return false;
        }
        
        if (string.IsNullOrWhiteSpace(password) || password.Length < 6)
        {
            Console.WriteLine("비밀번호는 최소 6자 이상이어야 한다.");
            return false;
        }
        
        // 비밀번호 해싱
        string salt = GenerateSalt();
        string passwordHash = HashPassword(password, salt);
        
        using (var connection = new MySqlConnection(_connectionString))
        {
            connection.Open();
            
            string sql = @"INSERT INTO users (username, email, password_hash, salt, created_at) 
                          VALUES (@username, @email, @passwordHash, @salt, @createdAt)";
            
            try
            {
                using (var command = new MySqlCommand(sql, connection))
                {
                    command.Parameters.AddWithValue("@username", username);
                    command.Parameters.AddWithValue("@email", email);
                    command.Parameters.AddWithValue("@passwordHash", passwordHash);
                    command.Parameters.AddWithValue("@salt", salt);
                    command.Parameters.AddWithValue("@createdAt", DateTime.Now);
                    
                    int rows = command.ExecuteNonQuery();
                    
                    if (rows > 0)
                    {
                        Console.WriteLine($"회원가입 성공: {username}");
                        return true;
                    }
                }
            }
            catch (MySqlException ex)
            {
                // Unique 제약 위반 (중복 유저명 또는 이메일)
                if (ex.Number == 1062)
                {
                    Console.WriteLine("이미 존재하는 유저명 또는 이메일이다.");
                }
                else
                {
                    Console.WriteLine($"회원가입 실패: {ex.Message}");
                }
            }
        }
        
        return false;
    }
    
    // Salt 생성 (보안 강화)
    private string GenerateSalt()
    {
        byte[] saltBytes = new byte[32];
        using (var rng = RandomNumberGenerator.Create())
        {
            rng.GetBytes(saltBytes);
        }
        return Convert.ToBase64String(saltBytes);
    }
    
    // 비밀번호 해싱
    private string HashPassword(string password, string salt)
    {
        using (var sha256 = SHA256.Create())
        {
            string saltedPassword = password + salt;
            byte[] bytes = Encoding.UTF8.GetBytes(saltedPassword);
            byte[] hash = sha256.ComputeHash(bytes);
            return Convert.ToBase64String(hash);
        }
    }
}
```

### 로그인 인증 로직
로그인 시 입력받은 비밀번호를 해싱하여 저장된 해시값과 비교한다.

```csharp
public class LoginResult
{
    public bool Success { get; set; }
    public long UserId { get; set; }
    public string Username { get; set; }
    public string Message { get; set; }
}

public class UserService
{
    // ... 이전 코드 ...
    
    public LoginResult Login(string username, string password)
    {
        var result = new LoginResult { Success = false };
        
        using (var connection = new MySqlConnection(_connectionString))
        {
            connection.Open();
            
            // 유저 정보 조회
            string sql = @"SELECT user_id, username, password_hash, salt, is_banned 
                          FROM users 
                          WHERE username = @username";
            
            using (var command = new MySqlCommand(sql, connection))
            {
                command.Parameters.AddWithValue("@username", username);
                
                using (var reader = command.ExecuteReader())
                {
                    if (!reader.Read())
                    {
                        result.Message = "존재하지 않는 유저명이다.";
                        return result;
                    }
                    
                    // 계정 정지 확인
                    bool isBanned = reader.GetBoolean("is_banned");
                    if (isBanned)
                    {
                        result.Message = "정지된 계정이다.";
                        return result;
                    }
                    
                    // 비밀번호 검증
                    string storedHash = reader.GetString("password_hash");
                    string salt = reader.GetString("salt");
                    string inputHash = HashPassword(password, salt);
                    
                    if (storedHash != inputHash)
                    {
                        result.Message = "비밀번호가 일치하지 않는다.";
                        return result;
                    }
                    
                    // 로그인 성공
                    result.Success = true;
                    result.UserId = reader.GetInt64("user_id");
                    result.Username = reader.GetString("username");
                    result.Message = "로그인 성공";
                }
            }
            
            // 마지막 로그인 시간 업데이트
            if (result.Success)
            {
                string updateSql = "UPDATE users SET last_login = @loginTime WHERE user_id = @userId";
                using (var updateCmd = new MySqlCommand(updateSql, connection))
                {
                    updateCmd.Parameters.AddWithValue("@loginTime", DateTime.Now);
                    updateCmd.Parameters.AddWithValue("@userId", result.UserId);
                    updateCmd.ExecuteNonQuery();
                }
            }
        }
        
        return result;
    }
}
```

### 비밀번호 해싱 처리
비밀번호 보안은 게임 서버에서 매우 중요하다. 다음 원칙을 따른다:

**✅ 해야 할 것:**

- 비밀번호를 평문으로 저장하지 않는다
- Salt를 사용하여 레인보우 테이블 공격을 방지한다
- SHA256 이상의 해시 알고리즘을 사용한다
- 유저마다 다른 Salt를 생성한다

**❌ 하지 말아야 할 것:**

- 비밀번호를 평문으로 저장
- 모든 유저에게 같은 Salt 사용
- MD5 같은 약한 해시 알고리즘 사용
- 비밀번호를 로그에 출력

**해싱 과정:**

```
사용자 입력: "mypassword123"
    ↓
Salt 생성: "a8f5f167f44f4964e6c998dee827110c"
    ↓
결합: "mypassword123a8f5f167f44f4964e6c998dee827110c"
    ↓
SHA256 해싱
    ↓
저장: "5e884898da28047151d0e56f8dc6292773603d0d6aabbdd62a11ef721d1542d8"
```

로그인 시 검증:

```
사용자 입력: "mypassword123"
    ↓
DB에서 Salt 조회: "a8f5f167f44f4964e6c998dee827110c"
    ↓
결합: "mypassword123a8f5f167f44f4964e6c998dee827110c"
    ↓
SHA256 해싱
    ↓
결과: "5e884898da28047151d0e56f8dc6292773603d0d6aabbdd62a11ef721d1542d8"
    ↓
DB 저장값과 비교 → 일치하면 로그인 성공
```
